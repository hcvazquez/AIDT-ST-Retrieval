   <div class="sect1"> 
    <h2 id="mergesort"><a class="link" href="#mergesort">1. Mergesort</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="mergesort_overview"><a class="link" href="#mergesort_overview">1.1. Overview</a></h3> 
      <div class="paragraph"> 
       <p>The <em>Mergesort</em> algorithm can be used to sort a collection of objects. <em>Mergesort</em> is a so called <em>divide and conquer</em> algorithm. <em>Divide and conquer</em> algorithms divide the original data into smaller sets of data to solve the problem.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="mergesort"><a class="link" href="#mergesort">1.2. Mergesort</a></h3> 
      <div class="paragraph"> 
       <p>During the <em>Mergesort</em> process the objects of the collection are divided into two collections. To split a collection, <em>Mergesort</em> will take the middle of the collection and split the collection into its left and its right part.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The resulting collections are again recursively sorted via the <em>Mergesort</em> algorithm.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Once the sorting process of the two collections is finished, the result of the two collections is combined. To combine both collections <em>Mergesort</em> starts in each collection at the beginning. It picks the object which is smaller and inserts this object into the new collection. For this collection it now selects the next elements and selects the smaller element from both collections.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Once all elements from both collections have been inserted in the new collection, <em>Mergesort</em> has successfully sorted the collection.</p> 
      </div> 
      <div class="paragraph"> 
       <p>To avoid the creation of too many collections, typically one new collection is created and the left and right side are treated as different collections.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="mergesort_quicksort"><a class="link" href="#mergesort_quicksort">1.3. Comparison with Quicksort</a></h3> 
      <div class="paragraph"> 
       <p>In comparison to <a href="http://www.vogella.com/tutorials/JavaAlgorithmsQuicksort/article.html">Quicksort</a> the divide part in <em>Mergesort</em> is simple, while the merging part is complex.</p> 
      </div> 
      <div class="paragraph"> 
       <p><em>Quicksort</em> can sort "inline" in an existing collection, e.g. it does not have to create a copy of the collection while <em>Mergesort</em> requires a copy.</p> 
      </div> 
     </div> 
    </div> 
   </div> 
   <div class="sect1"> 
    <h2 id="mergesort"><a class="link" href="#mergesort">2. Mergesort in Java</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="mergesort_implementation"><a class="link" href="#mergesort_implementation">2.1. Implementation</a></h3> 
      <div class="paragraph"> 
       <p>Create a Java project called <em>de.vogella.algorithms.sort.mergesort</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Create the following program.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">de.vogella.algorithms.sort.mergesort</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">Mergesort</span> {
    <span class="directive">private</span> <span class="type">int</span><span class="type">[]</span> numbers;
    <span class="directive">private</span> <span class="type">int</span><span class="type">[]</span> helper;

    <span class="directive">private</span> <span class="type">int</span> number;

    <span class="directive">public</span> <span class="type">void</span> sort(<span class="type">int</span><span class="type">[]</span> values) {
        <span class="local-variable">this</span>.numbers = values;
        number = values.length;
        <span class="local-variable">this</span>.helper = <span class="keyword">new</span> <span class="type">int</span>[number];
        mergesort(<span class="integer">0</span>, number - <span class="integer">1</span>);
    }

    <span class="directive">private</span> <span class="type">void</span> mergesort(<span class="type">int</span> low, <span class="type">int</span> high) {
        <span class="comment">// check if low is smaller than high, if not then the array is sorted</span>
        <span class="keyword">if</span> (low &lt; high) {
            <span class="comment">// Get the index of the element which is in the middle</span>
            <span class="type">int</span> middle = low + (high - low) / <span class="integer">2</span>;
            <span class="comment">// Sort the left side of the array</span>
            mergesort(low, middle);
            <span class="comment">// Sort the right side of the array</span>
            mergesort(middle + <span class="integer">1</span>, high);
            <span class="comment">// Combine them both</span>
            merge(low, middle, high);
        }
    }

    <span class="directive">private</span> <span class="type">void</span> merge(<span class="type">int</span> low, <span class="type">int</span> middle, <span class="type">int</span> high) {

        <span class="comment">// Copy both parts into the helper array</span>
        <span class="keyword">for</span> (<span class="type">int</span> i = low; i &lt;= high; i++) {
            helper[i] = numbers[i];
        }

        <span class="type">int</span> i = low;
        <span class="type">int</span> j = middle + <span class="integer">1</span>;
        <span class="type">int</span> k = low;
        <span class="comment">// Copy the smallest values from either the left or the right side back</span>
        <span class="comment">// to the original array</span>
        <span class="keyword">while</span> (i &lt;= middle &amp;&amp; j &lt;= high) {
            <span class="keyword">if</span> (helper[i] &lt;= helper[j]) {
                numbers[k] = helper[i];
                i++;
            } <span class="keyword">else</span> {
                numbers[k] = helper[j];
                j++;
            }
            k++;
        }
        <span class="comment">// Copy the rest of the left side of the array into the target array</span>
        <span class="keyword">while</span> (i &lt;= middle) {
            numbers[k] = helper[i];
            k++;
            i++;
        }
        <span class="comment">// Since we are sorting in-place any leftover elements from the right side</span>
        <span class="comment">// are already at the right position.</span>

    }
}</code></pre>

    <h2 id="complexity"><a class="link" href="#complexity">3. Complexity Analysis</a></h2> 
    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>The following describes the <a href="http://www.vogella.com/tutorials/ComplexityAnalysis/article.html">runtime complexity of mergesort</a>.</p> 
     </div> 
     <div class="paragraph"> 
      <p>Mergesort sorts in worst case in O(n log n) time. Due to the required copying of the collection Mergesort is in the average case slower than <a href="http://www.vogella.com/tutorials/JavaAlgorithmsQuicksort/article.html">Quicksort</a>.</p>
