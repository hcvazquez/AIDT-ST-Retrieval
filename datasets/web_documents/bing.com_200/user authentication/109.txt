       <h1><span class="mw-headline" id="Advanced"><span class="mw-headline-number">4</span> Advanced</span></h1> 
       <p>Axis is essentially a SOAP engine -- a framework for constructing SOAP processors such as clients, servers, gateways, etc. The current version of Axis is written in Java.&nbsp; The examples in this article show how to construct and use an Axis2 client to consume a ServiceNow Web Service.&nbsp; This content is intended for system admins with a light development background in Java.&nbsp; To begin you would need Java JDK version 1.4.2 or higher and Axis2 version 1.0 or higher.<br> </p> 
       <h2><span class="mw-headline" id="Creating_Axis2_Client_Java_Classes"><span class="mw-headline-number">4.1</span> Creating Axis2 Client Java Classes</span></h2> 
       <h3><span class="mw-headline" id="Create_a_Java_Project"><span class="mw-headline-number">4.1.1</span> Create a Java Project</span></h3> 
       <p>This example uses Eclipse SDK Version: 3.4.2 for managing the source code and executing the web request.&nbsp; Eclipse is not required. </p> 
       <ul>
        <li>Open Eclipse and from the menu select <b>File -&gt; New -&gt; Project -&gt; Java Project</b>. </li> 
        <li>Give the project a name .</li> 
        <li>Verify that the correct JRE is specified. 
         <ul>
          <li>If using wsdl2java run "java -version" on the command line and this will be the version to specify for the project specific JRE. </li> 
          <li>If using the Axis2 Codegen plugin use default JRE.</li>
         </ul></li>
       </ul> 
       <p><a href="/index.php?title=File:JavaProject.png" class="image"><img alt="JavaProject.png" src="/images/c/cb/JavaProject.png" width="561" height="757"></a> </p> 
       <h3><span class="mw-headline" id="Generate_your_Axis2_client_code"><span class="mw-headline-number">4.1.2</span> Generate your Axis2 client code</span></h3> 
       <ul>
        <li>From a command line in the bin directory of the axis folder:</li>
       </ul> 
       <pre>  ./wsdl2java.sh -uri https://&lt;instance name&gt;.service-now.com/incident.do?WSDL -o /glide/workspace/TestWebService/

</pre> 
       <p><br> </p> 
       <ul>
        <li>In the above example: 
         <ul>
          <li>The "-uri" is either the path where you have saved a copy of the wsdl to either ".wsdl" or ".xml", or the URL the WSDL resides at. </li> 
          <li>The "-o" is the path where you want the files to be written out to. If not specified, the files will be written out to the current bin location. </li>
         </ul></li> 
        <li>In Eclipse refresh the project and the generated Stub and CallbackHandler should now be displayed</li>
       </ul>

       <h2><span class="mw-headline" id="Compatibility_with_Axis2_Versions_1.1_and_higher"><span class="mw-headline-number">4.3</span> Compatibility with Axis2 Versions 1.1 and higher</span></h2> 
       <p>Chunking support is only available in HTTP Version 1.1. By default chunking is enabled in Axis2.xml for versions 1.1 and higher. ServiceNow does not support Chuncking, so you will need to disable chunking at deployment time or at runtime. </p> 
       <ul>
        <li>Deployment time: One can disable HTTP chunking by removing or commenting out the following element from Axis2.xml</li>
       </ul> 
       <div class="mw-highlight mw-content-ltr" dir="ltr">
        <pre><span class="nt">&lt;parameter</span> <span class="na">name=</span><span class="s">"Transfer-Encoding"</span><span class="nt">&gt;</span>chunked<span class="nt">&lt;/parameter&gt;</span>
</pre>
       </div> 
       <ul>
        <li>Runtime: User can disable the chunking using following property set in Client or Stub, versions 1.1.1 and higher only<br></li>
       </ul> 
       <div class="mw-highlight mw-content-ltr" dir="ltr">
        <pre><span class="n">options</span><span class="o">.</span><span class="na">setProperty</span><span class="o">(</span><span class="n">org</span><span class="o">.</span><span class="na">apache</span><span class="o">.</span><span class="na">axis2</span><span class="o">.</span><span class="na">transport</span><span class="o">.</span><span class="na">http</span><span class="o">.</span><span class="na">HTTPConstants</span><span class="o">.</span><span class="na">CHUNKED</span><span class="o">,</span> <span class="n">Boolean</span><span class="o">.</span><span class="na">FALSE</span><span class="o">);</span>
</pre>
       </div> 
       <h2><span class="mw-headline" id="Creating_Unique_Packages"><span class="mw-headline-number">4.4</span> Creating Unique Packages</span></h2> 
       <p>You can use the Axis2 parameter namespace2package (ns2p) to create unique package names. The parameter uses this <a target="_blank" rel="nofollow noreferrer noopener" class="external text" href="http://axis.apache.org/axis2/java/core/tools/CodegenToolReference.html#cmdref">format</a>: </p> 
       <pre>
&lt;Axis path&gt;\bin\wsdl2java.bat -u -p cr2 -ns2p &lt;namespace&gt;=&lt;package name&gt; -uri &lt;wsdl to convert&gt;
</pre> 
       <p>For example: </p> 
       <pre>
&lt;Axis path&gt;\bin\wsdl2java.bat -u -p cr2 -ns2p http://www.service-now.com/change_request=my.change_request -uri change_request
</pre>
