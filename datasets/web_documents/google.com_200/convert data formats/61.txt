            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>you can do</p> 
              <pre><code>function formatDate(date) {
    var d = new Date(date),
        month = '' + (d.getMonth() + 1),
        day = '' + d.getDate(),
        year = d.getFullYear();

    if (month.length &lt; 2) month = '0' + month;
    if (day.length &lt; 2) day = '0' + day;

    return [year, month, day].join('-');
}
</code></pre> 
              <p>usage example:</p> 
              <pre><code>alert(formatDate('Sun May 11,2014'));
</code></pre> 
              <p>Output:</p> 
              <pre><code>2014-05-11
</code></pre> 
              <p>Demo on fiddle: <a href="http://jsfiddle.net/abdulrauf6182012/2Frm3/">http://jsfiddle.net/abdulrauf6182012/2Frm3/</a></p>

             <div class="post-text" itemprop="text"> 
              <pre><code>format = function date2str(x, y) {
    var z = {
        M: x.getMonth() + 1,
        d: x.getDate(),
        h: x.getHours(),
        m: x.getMinutes(),
        s: x.getSeconds()
    };
    y = y.replace(/(M+|d+|h+|m+|s+)/g, function(v) {
        return ((v.length &gt; 1 ? "0" : "") + eval('z.' + v.slice(-1))).slice(-2)
    });

    return y.replace(/(y+)/g, function(v) {
        return x.getFullYear().toString().slice(-v.length)
    });
}
</code></pre>

             <div class="post-text" itemprop="text"> 
              <p><code>toISOString()</code> assumes your date is local time and converts it to UTC. You will get incorrect date string.</p> 
              <p>The following method should return what you need.</p> 
              <pre><code>Date.prototype.yyyymmdd = function() {         

    var yyyy = this.getFullYear().toString();                                    
    var mm = (this.getMonth()+1).toString(); // getMonth() is zero-based         
    var dd  = this.getDate().toString();             

    return yyyy + '-' + (mm[1]?mm:"0"+mm[0]) + '-' + (dd[1]?dd:"0"+dd[0]);
};
</code></pre> 
              <p>Source: <a href="https://blog.justin.kelly.org.au/simple-javascript-function-to-format-the-date-as-yyyy-mm-dd/" rel="nofollow noreferrer">https://blog.justin.kelly.org.au/simple-javascript-function-to-format-the-date-as-yyyy-mm-dd/</a></p>

              <p>Reformatting a date string is fairly straight forward, e.g.</p>

              <div class="snippet" data-lang="js" data-hide="false" data-console="true" data-babel="false"> 
               <div class="snippet-code"> 
                <pre class="snippet-code-js lang-js prettyprint-override"><code>var s = 'Sun May 11,2014';

function reformatDate(s) {
  function z(n){return ('0' + n).slice(-2)}
  var months = [,'jan','feb','mar','apr','may','jun',
                 'jul','aug','sep','oct','nov','dec'];
  var b = s.split(/\W+/);
  return b[3] + '-' +
    z(months.indexOf(b[1].substr(0,3).toLowerCase())) + '-' +
    z(b[2]);
}

             <div class="post-text" itemprop="text"> 
              <p>None of these answers quite satisfied me. I wanted a cross platform solution that gives me the day in the local timezone without using any external libraries.</p> 
              <p>This is what I came up with:</p> 
              <pre><code>function localDay(time) {
  var minutesOffset = time.getTimezoneOffset()
  var millisecondsOffset = minutesOffset*60*1000
  var local = new Date(time - millisecondsOffset)
  return local.toISOString().substr(0, 10)
}
</code></pre> 
              <p>That should return the day of the date, in YYYY-MM-DD format, in the timezone the date references.</p> 
              <p>So for example <code>localDay(new Date("2017-08-24T03:29:22.099Z"))</code> will return <code>"2017-08-23"</code> even though it's already the 24th at UTC.</p> 
              <p>You'll need to <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date/toISOString" rel="nofollow noreferrer">polyfill</a> Date.prototype.toISOString for it to work in IE8, but it should be supported everywhere else.</p>
