     <h1> <span class="mw-headline" id="Introduction:_Building_Web_pages_with_Video"> Introduction: Building Web pages with Video </span></h1> 
     <p>The <a href="/VLC_media_player/" title="VLC media player">VLC media player</a> webplugins are native browser plugins, similar to Flash or Silverlight plugins and allow playback inside the browser of all the videos that <a href="/VLC_media_player/" title="VLC media player">VLC media player</a> can read. </p>
     <p>Additionally to viewing video on all pages, you can build custom pages that will use the advanced features of the plugin, using Javascript functions to control playback or extract information from the plugin. </p>
     <p>There are 2 main plugins: one is ActiveX for IE, the other is NPAPI for the other browsers. They feature the same amount of features. </p>
     <p>In older versions, those plugins were very crashy. <b>We URGE YOU</b> to use VLC <b>2.0.0</b> or newer versions. </p>

     <h1> <span class="mw-headline" id="Embed_tag_attributes"> Embed tag attributes </span></h1> 
     <p>To embed the plugin into a webpage, use the following &lt;embed&gt; template: </p> 
     <div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
      <div class="html5 source-html5">
       <ol>
        <li class="li1"><pre class="de1"><span class="sc2">&lt;<span class="kw2">embed</span> <span class="kw3">type</span><span class="sy0">=</span><span class="st0">"application/x-vlc-plugin"</span> pluginspage<span class="sy0">=</span><span class="st0">"http://www.videolan.org"</span> <span class="sy0">/</span>&gt;</span></pre></li>
       </ol>
      </div>
     </div> 
     <p><br> If you are using vlc version &lt; 2.2.0 with Internet Explorer, use instead the following &lt;object&gt; template: </p> 
     <div dir="ltr" class="mw-geshi mw-code mw-content-ltr">
      <div class="html5 source-html5">
       <ol>
        <li class="li1"><pre class="de1"><span class="sc2">&lt;<span class="kw2">object</span> <span class="kw3">classid</span><span class="sy0">=</span><span class="st0">"clsid:9BE31822-FDAD-461B-AD51-BE1D1C159921"</span> <span class="kw3">codebase</span><span class="sy0">=</span><span class="st0">"http://download.videolan.org/pub/videolan/vlc/last/win32/axvlc.cab"</span>&gt;&lt;<span class="sy0">/</span><span class="kw2">object</span>&gt;</span></pre></li>
       </ol>

     <h3> <span class="mw-headline" id="Required_elements"> Required elements </span></h3> 
     <p>These are <b>required</b> attributes for the &lt;embed&gt; tag: </p> 
     <ul>
      <li><b>width</b>: Specifies the width of the plugin. </li>
      <li><b>height</b>: Specifies the height of the plugin. </li>
      <li><b>target</b> (or one of these alias: <b>mrl</b>, <b>filename</b>, <b>src</b>): Specifies the source location (URL) of the video to load. </li>
     </ul> 
     <h3> <span class="mw-headline" id="Optional_elements"> Optional elements </span></h3> 
     <p>These are additional attributes for the &lt;embed&gt; tag: </p> 
     <ul>
      <li><b>autoplay</b>, <b>autostart</b>: Specifies whether the plugin starts playing on load. Default: <i>true</i> </li>
      <li><b>allowfullscreen</b> (or <b>fullscreenEnabled</b>, <b>fullscreen</b>): (since VLC version 2.0.0) Specifies whether the user can switch into fullscreen mode. Default: <i>true</i> </li>
      <li><b>windowless</b>: (since VLC version 2.0.6, only for Mozilla) Draw the video on a window-less (non-accelerated) surface and allow styling (CSS overlay, 3D transformations, and much more). Default: <i>false</i> </li>
      <li><b>mute</b>: Specifies whether the audio volume is initially muted. Default: <i>false</i> </li>
      <li><b>volume</b>: (since VLC version 2.2.2) Specifies the initial audio volume as a percentage. Default: <i>100</i> </li>
      <li><b>loop</b>, <b>autoloop</b>: Specifies whether the video loops on end. Default: <i>false</i> </li>
      <li><b>controls</b> (or <b>toolbar</b>): Specifies whether the controls are shown by default. Default: <i>true</i> </li>
      <li><b>bgcolor</b>: Specifies the background color of the video player. Default: <i>#000000</i> </li>
      <li><b>text</b>: (only for Mozilla on MacOS) Specifies a text displayed as long as no video is shown. Default: empty </li>
      <li><b>branding</b>: (in vlc version &lt; 2.2.2 only for Mozilla on MacOS) Specifies whether VLC branding should be displayed in the web plugin's drawing context. Default: <i>true</i> </li>
     </ul>

     <h1> <span class="mw-headline" id="Javascript_API_description"> Javascript API description </span></h1> 
     <p>The vlc plugin exports several objects that can be accessed for setting and getting information. When used improperly the API's will throw an exception that includes a string that explains what happened. For example when you set vlc.audio.track out of range. </p> 
     <h2> <span class="mw-headline" id="VLC_objects"> VLC objects </span></h2> 
     <p>The vlc plugin knows the following objects: </p> 
     <ul>
      <li><b>audio</b>: Access audio properties. </li>
      <li><b>input</b>: Access input properties. 
       <ul>
        <li><b>input.title</b>: Access title properties (available in vlc version &gt;= 2.2.2, supported only &gt;= 3.0.0) </li>
        <li><b>input.chapter</b>: Access chapter properties (available in vlc version &gt;= 2.2.2, supported only &gt;= 3.0.0) </li>
       </ul> </li>
      <li><b>playlist</b>: Access playlist properties. 
       <ul>
        <li><b>playlist.items</b>: Access playlist items properties. </li>
       </ul> </li>
      <li><b>subtitle</b>: Access subtitle properties. </li>
      <li><b>video</b>: Access video properties. 
       <ul>
        <li><b>video.deinterlace</b>: Access deinterlace properties. </li>
        <li><b>video.marquee</b>: Access marquee video filter properties. </li>
        <li><b>video.logo</b>: Access logo video filter properties. </li>
       </ul> </li>
      <li><b>mediaDescription</b>: Access media info properties (available in vlc version &gt;= 2.0.2). </li>
     </ul> 
     <p>The following are deprecated: </p> 
     <ul>
      <li><b>log</b>: Access log properties (only available in vlc version &lt;= 1.0.0-rc1). </li>
      <li><b>messages</b>: Access to log message properties (only available in vlc version &lt;= 1.0.0-rc1). </li>
      <li><b>iterator</b>: Access to log iterator properties (only available in vlc version &lt;= 1.0.0-rc1). </li>
      <li><b>message</b>: Access to log message properties (only available in vlc version &lt;= 1.0.0-rc1). </li>
     </ul>

     <h2> <span class="mw-headline" id="Root_object"> Root object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.VersionInfo</b>: returns version information string </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>vlc.versionInfo()</b>: (only for Mozilla) returns version information string (same as VersionInfo) </li>
      <li><b>vlc.getVersionInfo()</b>: (supported in vlc version &gt;= 2.2.2) returns version information string (same as VersionInfo and versionInfo()) </li>
     </ul> 
     <ul>
      <li><b>vlc.addEventListener(eventname, callback, bubble)</b>: (only for Mozilla) add a listener for mentioned event name, callback expects a function and bubble influences the order of eventhandling by JS (usually it is set to false). </li>
      <li><b>vlc.removeEventListener(eventname, callback, bubble)</b>: (only for Mozilla) remove listener for mentioned event name, callback expects a function and bubble influences the order of eventhandling by JS (usually it is set to false). </li>
     </ul> 
     <ul>
      <li><b>vlc.attachEvent(eventname, callback)</b>: (only for ActiveX) add listener for mentioned event name, callback expects a function </li>
      <li><b>vlc.detachEvent(eventname, callback)</b>: (only for ActiveX) remove listener for mentioned event name, callback expects a function </li>
     </ul> 
     <p>events </p> 
     <ul>
      <li><b>MediaPlayerNothingSpecial</b>: vlc is in idle state doing nothing but waiting for a command to be issued </li>
      <li><b>MediaPlayerOpening</b>: vlc is opening an media resource locator (MRL) </li>
      <li><b>MediaPlayerBuffering(int cache)</b>: vlc is buffering </li>
      <li><b>MediaPlayerPlaying</b>: vlc is playing a media </li>
      <li><b>MediaPlayerPaused</b>: vlc is in paused state </li>
      <li><b>MediaPlayerStopped</b>: vlc is in stopped state </li>
      <li><b>MediaPlayerForward</b>: vlc is fastforwarding through the media (this never gets invoked) </li>
      <li><b>MediaPlayerBackward</b>: vlc is going backwards through the media (this never gets invoked) </li>
      <li><b>MediaPlayerEncounteredError</b>: vlc has encountered an error and is unable to continue </li>
      <li><b>MediaPlayerEndReached</b>: vlc has reached the end of current playlist </li>
      <li><b>MediaPlayerTimeChanged(int time)</b>: time has changed </li>
      <li><b>MediaPlayerPositionChanged(float position)</b>: media position has changed </li>
      <li><b>MediaPlayerSeekableChanged(bool seekable)</b>: media seekable flag has changed (true means media is seekable, false means it is not) </li>
      <li><b>MediaPlayerPausableChanged(bool pausable)</b>: media pausable flag has changed (true means media is pauseable, false means it is not) </li>
      <li><b>MediaPlayerMediaChanged</b>: (supported in vlc version &gt;= 2.2.0) media has changed </li>
      <li><b>MediaPlayerTitleChanged(int title)</b>: (in vlc version &lt; 2.2.0 only for Mozilla) title has changed (DVD/Blu-ray) </li>
      <li><b>MediaPlayerChapterChanged(int chapter)</b>: (supported in vlc version &gt;= 3.0.0) chapter has changed (DVD/Blu-ray) </li>
      <li><b>MediaPlayerLengthChanged(int length)</b>: (in vlc version &lt; 2.2.0 only for Mozilla) length has changed </li>
      <li><b>MediaPlayerVout(int count)</b>: (supported in vlc version &gt;= 2.2.7) the number of video output has changed </li>
      <li><b>MediaPlayerMuted</b>: (supported in vlc version &gt;= 2.2.7) audio volume was muted </li>
      <li><b>MediaPlayerUnmuted</b>: (supported in vlc version &gt;= 2.2.7) audio volume was unmuted </li>
      <li><b>MediaPlayerAudioVolume(float volume)</b>: (supported in vlc version &gt;= 2.2.7) audio volume has changed </li>
     </ul>

     <h2> <span class="mw-headline" id="Audio_object"> Audio object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.audio.count</b>: (supported in vlc version &gt;= 1.1.0) returns the number of audio track available. </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.audio.mute</b>: boolean value to mute and unmute the audio. </li>
      <li><b>vlc.audio.volume</b>: a value between [0-200] which indicates a percentage of the volume. </li>
      <li><b>vlc.audio.track</b>: (supported in vlc version &gt; 0.8.6) a value between [1-65535] which indicates the audio track to play or that is playing. a value of 0 means the audio is/will be disabled. </li>
      <li><b>vlc.audio.channel</b>: (supported in vlc version &gt; 0.8.6) integer value between [1-5] that indicates which audio channel mode is used, values can be: "1=stereo", "2=reverse stereo", "3=left", "4=right", "5=dolby". Use vlc.audio.channel to check if setting of the audio channel mode has succeeded. </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>vlc.audio.toggleMute()</b>: boolean toggle that mutes and unmutes the audio based upon the previous state. </li>
      <li><b>vlc.audio.description(int i)</b>: (supported in vlc version &gt;= 1.1.0) give the i-th audio track name. 0 corresponds to disable and 1 to the first audio track. </li>
     </ul>

     <h2> <span class="mw-headline" id="Input_object"> Input object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.input.length</b>: length of the input file in number of milliseconds. 0 is returned for 'live' streams or clips whose length cannot be determined by VLC. It returns -1 if no input is playing. </li>
      <li><b>vlc.input.fps</b>: frames per second returned as a float (typically 60.0, 50.0, 23.976, etc...) </li>
      <li><b>vlc.input.hasVout</b>: a boolean that returns true when the video is being displayed, it returns false when video is not displayed </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.input.position</b>: normalized position in multimedia stream item given as a float value between [0.0 - 1.0] </li>
      <li><b>vlc.input.time</b>: the absolute position in time given in milliseconds, this property can be used to seek through the stream </li>
     </ul> 
     <pre>&lt;!-- absolute seek in stream&nbsp;--&gt;

     <h3> <span class="mw-headline" id="Title_object"> Title object </span></h3> 
     <p>readonly properties </p>

      <li><b>vlc.input.title.track</b>: (supported in vlc version &gt;= 2.2.2) get and set the title track. The property takes an integer as input value [0..65535]. It returns -1 if no titles are available. </li>

     <h3> <span class="mw-headline" id="Chapter_object"> Chapter object </span></h3> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.input.chapter.count</b>: (supported in vlc version &gt;= 2.2.2) returns the number of chapter available in the current title. </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.input.chapter.track</b>: (supported in vlc version &gt;= 2.2.2) get and set the chapter track. The property takes an integer as input value [0..65535]. It returns -1 if no chapters are available. </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>vlc.input.chapter.description(int i)</b>: (supported in vlc version &gt;= 2.2.2) give the i-th chapter name. </li>
      <li><b>vlc.input.chapter.countForTitle(int i)</b>: (supported in vlc version &gt;= 2.2.2) returns the number of chapter available for a specific title. </li>
      <li><b>vlc.input.chapter.prev()</b>: (supported in vlc version &gt;= 2.2.2) play the previous chapter. </li>
      <li><b>vlc.input.chapter.next()</b>: (supported in vlc version &gt;= 2.2.2) play the next chapter. </li>
     </ul> 
     <h2> <span class="mw-headline" id="Playlist_object"> Playlist object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.playlist.itemCount</b>: number that returns the amount of items currently in the playlist (<b>deprecated</b>, do not use, see <a href="#Playlist_items_object">Playlist items</a>) </li>
      <li><b>vlc.playlist.isPlaying</b>: a boolean that returns true if the current playlist item is playing and false when it is not playing </li>
      <li><b>vlc.playlist.currentItem</b>: (supported in vlc version &gt;= 2.2.0) number that returns the index of the current item in the playlist. It returns -1 if the playlist is empty or no item is active. </li>
      <li><b>vlc.playlist.items</b>: return the playlist items collection, see <a href="#Playlist_items_object">Playlist items</a> </li>

     <p>read/write properties </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>vlc.playlist.add(mrl)</b>: add a playlist item as <a href="/MRL/" title="MRL" class="mw-redirect">MRL</a>. The MRL must be given as a string. Returns the index of the just added item in the playlist as a number. </li>
      <li><b>vlc.playlist.add(mrl,name,options)</b>: add a playlist item as MRL, with metaname 'name' and options 'options'. options are text arguments which can be provided either as a single string containing space separated values, akin to VLC command line, or as an array of string values. Returns the index of the just added item in the playlist as a number. </li>
     </ul>

     <h3> <span class="mw-headline" id="Playlist_items_object"> Playlist items object </span></h3>

     <h2> <span class="mw-headline" id="Subtitle_object"> Subtitle object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.subtitle.count</b>: (supported in vlc version &gt;= 1.1.0) returns the number of subtitle available. </li>

     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.subtitle.track</b>: (supported in vlc version &gt;= 1.1.0) get and set the subtitle track to show on the video screen. The property takes an integer as input value [1..65535]. If subtitle track is set to 0, the subtitles will be disabled. </li>

     <h2> <span class="mw-headline" id="Video_object"> Video object </span></h2> 
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.video.width</b>: returns the horizontal size of the video </li>
      <li><b>vlc.video.height</b>: returns the vertical size of the video </li>
      <li><b>vlc.video.count</b>: (supported in vlc version &gt;= 2.2.7) returns the number of video track available. </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.video.fullscreen</b>: when set to true the video will be displayed in fullscreen mode, when set to false the video will be shown inside the video output size. The property takes a boolean as input. </li>
      <li><b>vlc.video.aspectRatio</b>: get and set the aspect ratio to use in the video screen. The property takes a string as input value. Typical values are: "1:1", "4:3", "16:9", "16:10", "221:100" and "5:4" </li>
      <li><b>vlc.video.subtitle</b>: (supported in vlc version &gt; 0.8.6a) get and set the subtitle track to show on the video screen. The property takes an integer as input value [1..65535]. If subtitle track is set to 0, the subtitles will be disabled. </li>
      <li><b>vlc.video.crop</b>: get and set the geometry of the zone to crop. This is set as &lt;width&gt; x &lt;height&gt; + &lt;left offset&gt; + &lt;top offset&gt;. A possible value is: "120x120+10+10" </li>
      <li><b>vlc.video.teletext</b>: (supported in vlc version &gt;= 0.9.0) get and set teletext page to show on the video stream. This will only work if a teletext elementary stream is available in the video stream. The property takes an integer as input value [0..1000] for indicating the teletext page to view, setting the value to 0 means hide teletext. </li>
      <li><b>vlc.video.track</b>: (supported in vlc version &gt;= 2.2.7) a value between [1-65535] which indicates the video track to play or that is playing. a value of 0 means the video is/will be disabled. </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>vlc.video.takeSnapshot()</b>: (supported in vlc version &gt;= 0.9.0, only for ActiveX) generates a snapshot and saves it on the desktop </li>
      <li><b>vlc.video.toggleFullscreen()</b>: toggle the fullscreen mode based on the previous setting </li>
      <li><b>vlc.video.toggleTeletext()</b>: (supported in vlc version &gt;= 0.9.0) toggle the teletext page to overlay transparent or not, based on the previous setting </li>
      <li><b>vlc.video.description(int i)</b>: (supported in vlc version &gt;= 2.2.7) give the i-th video track name. 0 corresponds to disable and 1 to the first video track. </li>
     </ul>

     <h3> <span class="mw-headline" id="Marquee_Object"> Marquee Object </span></h3> 
     <p>readonly properties </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.video.marquee.text</b>: (supported in vlc version &gt;= 1.1.0) display my text on the screen. </li>
      <li><b>vlc.video.marquee.color</b>: (supported in vlc version &gt;= 1.1.0) change the text color. val is the new color to use (WHITE=0x000000, BLACK=0xFFFFFF, RED=0xFF0000, GREEN=0x00FF00, BLUE=0x0000FF...). </li>

      <li><b>vlc.video.marquee.size</b>: (supported in vlc version &gt;= 1.1.0) val define the new size for the text displayed on the screen. If the text is bigger than the screen then the text is not displayed. </li>

      <li><b>vlc.video.marquee.x</b>: (supported in vlc version &gt;= 1.1.0) change text abscissa. </li>
      <li><b>vlc.video.marquee.y</b>: (supported in vlc version &gt;= 1.1.0) change text ordinate. </li>

     <h2> <span class="mw-headline" id="DEPRECATED_APIs"> DEPRECATED APIs </span></h2> 
     <h3> <span class="mw-headline" id="DEPRECATED:_Log_object"> DEPRECATED: Log object </span></h3> 
     <dl>
      <dd>
       <b>CAUTION</b>: For security concern, VLC 1.0.0-rc1 is the latest (near-to-stable) version in which this object and its children are supported. 
      </dd>
     </dl> 
     <p>This object allows accessing VLC main message logging queue. Typically this queue capacity is very small (no nore than 256 entries) and can easily overflow, therefore messages should be read and cleared as often as possible. </p>
     <p>readonly properties </p> 
     <ul>
      <li><b>vlc.log.messages</b>: returns the message collection, see <a href="#DEPRECATED:_Messages_object">Messages object</a> </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><b>vlc.log.verbosity</b>: write number [-1,0,1,2,3] for changing the verbosity level of the log messages; messages whose verbosity is higher than set will be not be logged in the queue. The numbers have the following meaning: -1 disable, 0 info, 1 error, 2 warning, 3 debug. </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <h3> <span class="mw-headline" id="DEPRECATED:_Messages_object"> DEPRECATED: Messages object </span></h3> 
     <dl>
      <dd>
       <b>CAUTION</b>: For security concern, VLC 1.0.0-rc1 is the latest (near-to-stable) version in which this object and its children are supported. 
      </dd>
     </dl> 
     <p>readonly properties </p> 
     <ul>
      <li><b>messages.count</b>: returns number of messages in the log </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>messages.clear()</b>: clear the current log buffer. It should be called as frequently as possible to not overflow the message queue. Call this method after the log messages of interest are read. </li>
      <li><b>messages.iterator()</b>: creates and returns an iterator object, used to iterate over the messages in the log. <b>Don't clear the log buffer while holding an iterator object.</b> </li>
     </ul> 
     <h3> <span class="mw-headline" id="DEPRECATED:_Messages_Iterator_object"> DEPRECATED: Messages Iterator object </span></h3> 
     <dl>
      <dd>
       <b>CAUTION</b>: For security concern, VLC 1.0.0-rc1 is the latest (near-to-stable) version in which this object and its children are supported. 
      </dd>
     </dl> 
     <p>readonly properties </p> 
     <ul>
      <li><b>iterator.hasNext</b>: returns a boolean that indicates whether <i>vlc.log.messages.next()</i> will return the next message. </li>
     </ul> 
     <p>read/write properties </p> 
     <ul>
      <li><i>none</i> </li>
     </ul> 
     <p>methods </p> 
     <ul>
      <li><b>iterator.next()</b>: returns the next message object in the log, see <a href="#DEPRECATED:_Message_subobject">Message object</a> </li>
     </ul> 
     <h3> <span class="mw-headline" id="DEPRECATED:_Message_subobject"> DEPRECATED: Message subobject </span></h3> 
     <dl>
      <dd>
       <b>CAUTION</b>: For security concern, VLC 1.0.0-rc1 is the latest (near-to-stable) version in which this object and its children are supported. 
      </dd>
     </dl> 
     <ul>
      <li><b>message.severity</b>: number that indicates the severity of the log message (0 = info, 1 = error, 2 = warning, 3 = debug) </li>
      <li><b>message.name</b>: name of VLC module that printed the log message (e.g: main, http, directx, etc...) </li>
      <li><b>message.type</b>: type of VLC module that printed the log message (eg: input, access, vout, sout, etc...) </li>
      <li><b>message.message</b>: the message text </li>
     </ul>
