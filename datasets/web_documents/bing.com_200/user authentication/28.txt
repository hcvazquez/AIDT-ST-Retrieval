         <p><a href="https://travis-ci.org/AzureAD/azure-activedirectory-library-for-js"><img src="https://camo.githubusercontent.com/760caafd2944ba1d727d758d255681b2d16c211b/68747470733a2f2f7472617669732d63692e6f72672f417a75726541442f617a7572652d6163746976656469726563746f72792d6c6962726172792d666f722d6a732e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/AzureAD/azure-activedirectory-library-for-js.svg?branch=master" style="max-width:100%;"></a></p> 
         <p>Active Directory Authentication Library for JavaScript (ADAL JS) helps you to use Azure AD for handling authentication in your single page applications. This library is optimized for working together with AngularJS.</p>

         <p><a href="https://github.com/azure-samples?query=active-directory">We provide a full suite of sample applications and documentation on GitHub</a> to help you get started with learning the Azure Identity system. We also provide full walkthroughs for authentication flows such as OAuth2, OpenID Connect, Graph API, and other awesome features.</p>

           </svg></a>Getting Started</h1> 
         <p><strong>Quick usage guide</strong></p> 
         <p>Below you can find a quick reference for the most common operations you need to perform to use ADAL JS.</p> 
         <p>1- Include references to angular.js libraries, adal.js, adal-angular.js in your main app page. The ADAL should be included after Angular, but before your application scripts as shown below.</p> 
         <p>2- Include a reference to the ADAL module in your app module.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> app <span class="pl-k">=</span> <span class="pl-smi">angular</span>.<span class="pl-en">module</span>(<span class="pl-s"><span class="pl-pds">'</span>demoApp<span class="pl-pds">'</span></span>, [<span class="pl-s"><span class="pl-pds">'</span>ngRoute<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>AdalAngular<span class="pl-pds">'</span></span>]);</pre>
         </div> 
         <p>3- <em><strong>When HTML5 mode is configured</strong></em>, ensure the $locationProvider hashPrefix is set</p> 
         <div class="highlight highlight-source-js">
          <pre>	<span class="pl-c"><span class="pl-c">//</span> using '!' as the hashPrefix but can be a character of your choosing</span>
	<span class="pl-smi">app</span>.<span class="pl-en">config</span>([<span class="pl-s"><span class="pl-pds">'</span>$locationProvider<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">$locationProvider</span>) {
		<span class="pl-smi">$locationProvider</span>.<span class="pl-en">html5Mode</span>(<span class="pl-c1">true</span>).<span class="pl-en">hashPrefix</span>(<span class="pl-s"><span class="pl-pds">'</span>!<span class="pl-pds">'</span></span>);
	}]);</pre>
         </div> 
         <p>Without the hashPrefix set, the AAD login will loop indefinitely as the callback URL from AAD (in the form of, {yourBaseUrl}/#{AADTokenAndState}) will be rewritten to remove the '#' causing the token parsing to fail and login sequence to occur again.</p> 
         <p>4- Initialize ADAL with the AAD app coordinates at app config time. The minimum required object to initialize ADAL is:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">adalAuthenticationServiceProvider</span>.<span class="pl-en">init</span>({
    <span class="pl-c"><span class="pl-c">//</span> clientId is the identifier assigned to your app by Azure Active Directory.</span>
    clientId<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>e9a5a8b6-8af7-4719-9821-0deef255f68e<span class="pl-pds">"</span></span> 
})</pre>
         </div> 
         <p>A single-tenant configuration, with CORS, looks like this:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c"><span class="pl-c">//</span> endpoint to resource mapping(optional)</span>
    <span class="pl-k">var</span> endpoints <span class="pl-k">=</span> {
        <span class="pl-s"><span class="pl-pds">"</span>https://yourhost/api<span class="pl-pds">"</span></span><span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>b6a68585-5287-45b2-ba82-383ba1f60932<span class="pl-pds">"</span></span>,
    };
<span class="pl-smi">adalAuthenticationServiceProvider</span>.<span class="pl-en">init</span>(
        {
            <span class="pl-c"><span class="pl-c">//</span> Config to specify endpoints and similar for your app</span>
            tenant<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>52d4b072-9470-49fb-8721-bc3a1c9912a1<span class="pl-pds">"</span></span>, <span class="pl-c"><span class="pl-c">//</span> Optional by default, it sends common</span>
            clientId<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>e9a5a8b6-8af7-4719-9821-0deef255f68e<span class="pl-pds">"</span></span>, <span class="pl-c"><span class="pl-c">//</span> Required</span>
            <span class="pl-c"><span class="pl-c">//</span>localLoginUrl: "/login",  // optional</span>
            <span class="pl-c"><span class="pl-c">//</span>redirectUri : "your site", optional</span>
            endpoints<span class="pl-k">:</span> endpoints  <span class="pl-c"><span class="pl-c">//</span> If you need to send CORS api requests.</span>
        },
        $httpProvider   <span class="pl-c"><span class="pl-c">//</span> pass http provider to inject request interceptor to attach tokens</span>
        );</pre>
         </div> 
         <p>5- Define which routes you want to secure via ADAL - by adding <code>requireADLogin: true</code> to their definition</p> 
         <div class="highlight highlight-source-js">
          <pre>$routeProvider.
    <span class="pl-en">when</span>(<span class="pl-s"><span class="pl-pds">"</span>/todoList<span class="pl-pds">"</span></span>, {
        controller<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>todoListController<span class="pl-pds">"</span></span>,
        templateUrl<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>/App/Views/todoList.html<span class="pl-pds">"</span></span>,
        requireADLogin<span class="pl-k">:</span> <span class="pl-c1">true</span>
    });
</pre>
         </div> 
         <p>6- Any service invocation code you might have will remain unchanged. ADAL's interceptor will automatically add tokens for every outgoing call.</p> 
         <p>Anonymous endpoints, introduced in version 1.0.10, is an array of values that will be ignored by the ADAL route/state change handlers. ADAL will not attach a token to outgoing requests that have these keywords or URI. Routes that <em>do not</em> specify the <code>requireADLogin=true</code> property are added to the <code>anonymousEndpoints</code> array automatically.</p>

           </svg></a>Multi-Tenant</h3> 
         <p>By default, you have multi-tenant support. ADAL will set tenant to 'common', if it is not specified in the config. This allows any Microsoft account to authenticate to your application. If you are not interested in multi-tenant behavior, you will need to set the <code>tenant</code> property as shown above.</p> 
         <p>If you allow multi-tenant authentication, and you do not wish to allow all Microsoft account users to use your application, you must provide your own method of filtering the token issuers to only those tenants who are allowed to login.</p>

           </svg></a>CORS API usage and IE</h3> 
         <p>ADAL will get access token for given CORS API endpoints in the config. Access token is requested using Iframe. Iframe needs to access the cookies for the same domain that you did the initial sign in. IE does not allow to access cookies in IFrame for localhost. Your url needs to be fully qualified domain i.e <a href="http://yoursite.azurewebsites.com">http://yoursite.azurewebsites.com</a>. Chrome does not have this restriction.</p> 
         <p>To make CORS API call, you need to specify endpoints in the config for your CORS API. Your service will be similar to this to make the call from JS. In your API project, you need to enable CORS API requests to receive flight requests. You can check the sample for CORS API <a href="https://github.com/AzureADSamples/SinglePageApp-WebAPI-AngularJS-DotNet">sample</a>.</p>
