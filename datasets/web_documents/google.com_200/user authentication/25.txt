            <p>In this article, I’ll be walking you through 5 steps with which you can integrate JWT authentication into your existing project.</p> 
            <p>For those following my series, we’ve got a todo list app, and we have written tests for the app. By the end of this tutorial, only registered user will be able to create a todo task, which means the app users are required to register.</p> 
            <p>Now, let's start with some basics first.</p> 
            <h2 id="what-is-jwt"><a class="header-anchor" href="#what-is-jwt" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> What is JWT?</h2> 
            <p>A JSON Web Token(JWT), defines an explicit, compact, and self-containing secured protocol for transmitting restricted informations. This is often used to send information that can be verified and trusted by means of a digital signature.</p> 
            <p>The JWT Claims Set represents a compact URL-safe JSON object, that is base64url encoded and digitally signed and/or encrypted. The JSON object consists of zero or more name/value pairs (or members), where the names are strings and the values are arbitrary JSON values. These members are the claims represented by the JWT.<br> Here's an example of JWT:</p> 
            <pre><code>eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJlbWFpbCI6Im9sYXR1bmRlZ2FydWJhQGdtYWlsLmNvbSIsImZ1bGxOYW1lIjoiT2xhdHVuZGUgR2FydWJhIiwiX2lkIjoiNThmMjYzNDdiMTY1YzUxODM1NDMxYTNkIiwiaWF0IjoxNDkyMjgwMTk4fQ.VcMpybz08cB5PsrMSr25En4_EwCGWZVFgciO4M-3ENE
</code></pre> 
            <p>JSON Web Tokens consist of three parts separated by dots(i.e. <code>header.payload.signature</code>)</p> 
            <h2 id="how-jwt-works"><a class="header-anchor" href="#how-jwt-works" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> How JWT works</h2> 
            <p>JWT works as a two way protocol where a request is made and the response is generated from a server.<br> <img class="b-lazy" data-src="https://process.filestackapi.com/cache=expiry:max/te8ALKiFSFawLNZnIxqA" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" alt="Screen Shot 2017-04-15 at 9.59.42 AM.png"></p> 
            <p>The browser or the requesting device makes the request(user login information for authentication) for JWT encoded data, the server generates the signed token and return to the client(Mobile device/browser) as shown in the diagram above.</p> 
            <p>Subsequently, the token can be sent over to the http request for every other request that needs authentication on the server. The server then validates the token and, if it’s valid, returns the secure resource to the client.</p> 
            <p>This token is often signed using any secure signature method (e.g Asymmetric key algorithm such as HMAC SHA-256 or Asymmetric, public-key system, such as RSA).</p> 
            <h2 id="advantages-jwt"><a class="header-anchor" href="#advantages-jwt" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Advantages JWT?</h2> 
            <ul> 
             <li> <p><strong>Compact:</strong> JWT is compact, which means it can be sent along with http request either as body or as a header attribute.</p> </li> 
             <li> <p><strong>Stateless/self-contained:</strong> The token contains all the information to identify the user, which eliminates the need for the session state. If we use a load balancer, we can pass the user to any server, instead of being bound to the same server we logged in on. This also enhance performance, since there is no server-side lookup for deserialization on each request</p> </li> 
             <li> <p><strong>Reusability:</strong> We can have many separate servers that run on multiple platforms and domains and reuse the same token for authenticating the user. It is easy to build an application that shares permissions with another application.</p> </li> 
             <li> <p><strong>Security:</strong> There is no need to worry about cross-site request forgery (CSRF) attacks.</p> </li> 
            </ul> 
            <h3 id="prerequisite"><a class="header-anchor" href="#prerequisite" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Prerequisite:</h3> 
            <ul> 
             <li>Node.js</li> 
             <li>Node.js app</li> 
            </ul>

      user.hash_password = undefined;
      return res.json(user);
    }
  });
};
</code></pre> 
            <p><em>Note:</em> A hash password was saved in the database using bcrypt.</p> 
            <p>The <code>sign_in</code> handler handles the user sign in activities. Hence, we need to check and see if the user is already saved in the database. If yes, we have to check if the entered parameters matches the one saved by the user in the database.</p> 
            <p>If the user or the entered parameters do not match with the saved data, we return an error with a defined message.</p> 
            <pre><code>exports.sign_in = function(req, res) {
  User.findOne({
    email: req.body.email
  }, function(err, user) {
    if (err) throw err;
    if (!user) {
      res.status(401).json({ message: 'Authentication failed. User not found.' });
    } else if (user) {
      if (!user.comparePassword(req.body.password)) {
        res.status(401).json({ message: 'Authentication failed. Wrong password.' });
      } else {
        return res.json({token: jwt.sign({ email: user.email, fullName: user.fullName, _id: user._id}, 'RESTFULAPIs')});
      }
    }
  });
};
</code></pre> 
            <p>In <code>loginRequired</code>,&nbsp;we check to see if the user is signed in.<br> If the user is signed in, the user is allowed to carry out the next activities; else, we return a user unauthorized message in that instance, like so:</p> 
            <pre><code>exports.loginRequired = function(req, res, next) {
  if (req.user) {
    next();
  } else {
    return res.status(401).json({ message: 'Unauthorized user!' });
  }
};
</code></pre>

            <p>As shown in the diagram above, our model and JWT were made available to the files that require them first and foremost.<br> In the register handler, we instantiated our User model with the form data, after which we saved the document in the database.</p> 
            <p><em>Note:</em> Always perform form validation on the body properties before saving them to User model.</p> 
            <h4 id="4-create-the-user-routes-and-update-tasks-route"><a class="header-anchor" href="#4-create-the-user-routes-and-update-tasks-route" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> 4. Create the user routes and update “/tasks” route,</h4> 
            <p>Since we have some routes before, let’s update it by adding two other routes for the user activities.</p> 
            <p>First and foremost, let’s make the user handlers available in the routes, before we create the routes by updating:</p> 
            <pre><code>var todoList = require(‘../controllers/todoListController’);
</code></pre> 
            <p>to</p> 
            <pre><code>var todoList = require(‘../controllers/todoListController’),
 userHandlers = require(‘../controllers/userController.js’);
</code></pre>

            <p><img class="b-lazy" data-src="https://process.filestackapi.com/cache=expiry:max/utOuyaCRTASM5P5DJAQ7" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" alt="Screen Shot 2017-04-09 at 1.36.02 AM.png"></p> 
            <p>After creating the route, add the <code>loginRequired</code> handler to the post request on "<code>/tasks</code>".</p> 
            <p>This will ensure that the user is logged in before he/she can create a todo task.</p> 
            <h4 id="5-updating-the-file"><a class="header-anchor" href="#5-updating-the-file" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> 5. Updating the <code>server.js</code> file</h4> 
            <p>Going over to the <code>server.js</code> file, we need to add the newly created User model and also add a middleware to the express server that will check the state of the user(logged in or not logged in)</p> 
            <p>To add the user, add</p> 
            <pre><code>var User = require('./api/models/userModel'),
jsonwebtoken = require("jsonwebtoken");
</code></pre> 
            <p>to the required file at the top of the file.</p>
