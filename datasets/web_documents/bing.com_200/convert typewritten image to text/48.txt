           <h1 class="post-title">PDF Generation in Java using iText JAR</h1> 
           <p class="post-byline">by <a href="http://viralpatel.net/blogs/author/kiran/" title="Posts by Kiran Hegde" rel="author">Kiran Hegde</a> · April 1, 2009</p> 
           <div class="clear"></div> 
           <div class="entry"> 
            <div class="entry-inner"> 
             <p><img class="alignleft size-full wp-image-1075" title="java-pdf-logo" src="//img.viralpatel.net/java-pdf-logo.jpg" alt="java-pdf-logo" width="125" height="120"><strong>Generating PDF</strong> files in today’s enterprise applications is quite common. Doing this with <strong>Java</strong> is not an easy task as <strong>Java</strong> does not gives default api’s to handle <strong>PDF</strong> files. No worries, <strong>iText</strong> <strong>jar</strong> is for you.</p> 
             <p><strong>iText</strong> is a <strong>free Java-PDF library</strong> that allows you to generate <strong>PDF</strong> files <strong>on the fly</strong> (<strong>dynamically</strong>). <strong>iText </strong>is an ideal library for developers looking to enhance web- and other applications with <strong>dynamic PDF</strong> document <strong>generation</strong> and/or <strong>manipulation</strong>. iText is not an end-user tool. Typically you won’t use it on your Desktop as you would use Acrobat or any other PDF application. Rather, you’ll build iText into your own applications so that you can automate the PDF creation and manipulation process.</p>

             <div style="float:none;margin:10px 0 10px 0;text-align:center;">

             <h3>iText (Java-PDF Library) can be used to:</h3> 
             <ol> 
              <li>Serve PDF to a browser</li> 
              <li>Generate dynamic documents from XML files or databases</li> 
              <li>Use PDF’s many interactive features</li> 
              <li>Add bookmarks, page numbers, watermarks, etc.</li> 
              <li>Split, concatenate, and manipulate PDF pages</li> 
              <li>Automate filling out of PDF forms</li> 
              <li>Add digital signatures to a PDF file</li> 
             </ol> 
             <h2>Technical Requirements to use iText</h2> 
             <p>You should have <strong>JDK 1.4</strong> or later to integrate <strong>iText PDF</strong> <strong>generation</strong> in your application.</p> 
             <h2>Getting iText</h2> 
             <p>Download iText jar from its home page <a href="http://www.lowagie.com/iText/download.html">http://www.lowagie.com/iText/download.html</a></p> 
             <p>iText core: <a href="http://sourceforge.net/projects/itext/files/latest/download?source=files">iText-5.2.1.jar</a></p> 
             <h2>Generate simple PDF in Java using free Java-PDF library</h2> 
             <p><strong>It is very easy to generate a simple PDF file in Java using iText</strong>. All you have to do is to put <strong>itext.jar </strong>in your class path and paste following code in <strong>GeneratePDF.java</strong> class and compile and execute it. After you execute this, a file <strong>Test.pdf</strong> will be created in C: drive (If you are using Linux, you may want to have /usr/test.pdf as path).</p> 
             <pre><code class="language-java">package net.viralpatel.pdf;

import java.io.File;
import java.io.FileOutputStream;
import java.io.OutputStream;
import java.util.Date;

import com.itextpdf.text.Document;
import com.itextpdf.text.Paragraph;
import com.itextpdf.text.pdf.PdfWriter;

public class GeneratePDF {

	public static void main(String[] args) {
		try {
			OutputStream file = new FileOutputStream(new File("D:\\Test.pdf"));

			Document document = new Document();
			PdfWriter.getInstance(document, file);

			document.open();
			document.add(new Paragraph("Hello World, iText"));
			document.add(new Paragraph(new Date().toString()));

			document.close();
			file.close();

		} catch (Exception e) {

			e.printStackTrace();
		}
	}
}
</code></pre> 
             <p>In above code snippet we have created Document object which represents our <strong>PDF document</strong>. Also, by supplying <strong>OutputStream</strong> object to getInstance() method sends the output to OutputStream. Thus in our case we have created a output file and sent output to it.</p> 
             <h2>Generate PDF as Output Stream in HTTP request</h2> 
             <p>Sometime we may want to add the <strong>PDF generation</strong> functionality to a web application, where user on clicking some link or button is <strong>served with PDF</strong> output. Hence the <strong>PDF should be generated on fly</strong> and sent to client browser.</p> 
             <p>Consider following simple Struts Action class which uses this mechanism to <strong>generate a dummy PDF</strong> and <strong>sent the output to browser</strong>.</p> 
             <pre><code class="language-java">package net.viralpatel.struts.helloworld.action;

import java.util.Date;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import org.apache.struts.action.Action;
import org.apache.struts.action.ActionForm;
import org.apache.struts.action.ActionForward;
import org.apache.struts.action.ActionMapping;

import com.itextpdf.text.Document;
import com.itextpdf.text.Paragraph;
import com.itextpdf.text.pdf.PdfWriter;

/**
 * @author KiranRavi_Hegde
 *
 */
public class PdfHelloWorldAction extends Action {
	
	public ActionForward execute(ActionMapping mapping, ActionForm form,
			HttpServletRequest request, HttpServletResponse response)
			throws Exception {
	
	Document document = new Document();
	try{
		response.setContentType("application/pdf");
		PdfWriter.getInstance(document, response.getOutputStream());
		document.open();
		document.add(new Paragraph("Hello Kiran"));
		document.add(new Paragraph(new Date().toString()));
	}catch(Exception e){
		e.printStackTrace();
	}
	document.close();
	return null;

	}
}
</code></pre> 
             <p>If you notice in above code, we have passed <code>response.getOutputStream()</code> object to <code>getInstance()</code> method. Thus the output generated by iText will be sent directly to the response. Also don’t forget to set the content type of the response to <strong>application/pdf</strong>.</p> 
             <h2>Setting attributes of PDF using free Java-PDF library</h2> 
             <p>While you generate a PDF, you may want to set its different attribute like: author name, title, file description etc. iText jar can help you to set different attributes of a PDF file. Document object provide different methods to add various attributes to a PDF file.</p> 
             <pre><code class="language-java">document.addAuthor("Kiran Hegde");
document.addCreationDate();
document.addCreator("iText library");
document.addTitle("Hello World PDF");
</code></pre>

              <p>you can write the above example in servlet also.</p> 
              <pre><code class="language-java">package com.vamsi;

import java.io.IOException;
import java.util.Date;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

import com.itextpdf.text.Document;
import com.itextpdf.text.Paragraph;
import com.itextpdf.text.pdf.PdfWriter;

public class PdfServlet extends HttpServlet 
{
	private static final long serialVersionUID = 1L;
    public PdfServlet() 
    {
        super();
    }

	protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException 
	{
		Document document= new Document();
		try
		{
			response.setContentType("application/pdf");
			PdfWriter.getInstance(document, response.getOutputStream());
			document.open();
			document.add(new Paragraph("Hello vamsi"));
			document.add(new Paragraph(new Date().toString()));
		}
		catch(Exception e)
		{
			System.out.println(e);
		}
		document.close();

	}
}
</code></pre>
