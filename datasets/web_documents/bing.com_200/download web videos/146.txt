         <h2>Introduction</h2>
         <br>
         <p>This page describes the nature and structure of an Alfresco Module Package (AMP) file, which can be installed to an Alfresco server in order to extend it in some way.<br></p>
         <p>For information related to other ways of extending the repository without using AMP files, see <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FPackaging_And_Deploying_Extensions" rel="nofollow" target="_blank">Packaging And Deploying Extensions</a>.<br></p>
         <br>
         <h2>What is an AMP file?</h2>
         <br>
         <p>An AMP file is a collection of code, XML, images, CSS, etc. that collectively extend the functionality or data provided by Alfresco. For more information, see <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FDeveloping_an_Alfresco_Module" rel="nofollow" target="_blank">Developing an Alfresco Module</a>.<br></p>
         <p>An AMP file can contain as little as a set of custom templates or a new category.&nbsp; It can contain a custom model and associated UI customisations.&nbsp; It could contain a complete new set of functionality, for example records management.<br></p>
         <p>As a general rule of thumb, anything that is considered to be an 'installable' extension to Alfresco should be called a module and packaged as one or more AMP files (best practice is to provide one AMP for the repository tier and another for the Share UI tier).<br></p>
         <p>AMP files can be installed into the Alfresco and Share WAR files using the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a>.&nbsp; An AMP file has a standard structure (described below), which can be customised if required.<br></p>
         <p>AMP files can also be created via Apache Maven archetypes and have their lifecycle and deployment into Alfresco managed by the Maven Alfresco AMP archetype. For a full description of Maven AMPs Alfresco support see <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FManaging_Alfresco_Lifecycle_with_Maven" rel="nofollow" target="_blank">Managing Alfresco Lifecycle with Maven</a>.<br></p>
         <p>Once the contents of the AMP file has been installed into a WAR file using the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a>, the WAR can be deployed to the application server.&nbsp; When Alfresco is next started, the installed module configuration will be detected, and the repository will be bootstrapped to include the new module functionality and data.<br></p>
         <br>
         <h2>The structure of an AMP file</h2>
         <br>
         <p>An AMP file is ZIP compressed and has a default structure as shown below:<br></p>
         <br>
         <pre><br>&nbsp; /<br>&nbsp; |<br>&nbsp; |- /config<br>&nbsp; |<br>&nbsp; |- /lib<br>&nbsp; |<br>&nbsp; |- /licenses<br>&nbsp; |<br>&nbsp; |- /web<br>&nbsp;&nbsp;&nbsp; |<br>&nbsp;&nbsp;&nbsp; |- /jsp<br>&nbsp;&nbsp;&nbsp; |<br>&nbsp;&nbsp;&nbsp; |- /css<br>&nbsp;&nbsp;&nbsp; |<br>&nbsp;&nbsp;&nbsp; |- /images<br>&nbsp;&nbsp;&nbsp; |<br>&nbsp;&nbsp;&nbsp; |- /scripts<br>&nbsp; |<br>&nbsp; |- module.properties<br>&nbsp; |<br>&nbsp; |- file-mapping.properties<br></pre>
         <br>
         <ul>
          <li><strong>/config</strong><br></li>
         </ul>
         <p>This directory will be mapped into the <code>/WEB-INF/classes</code> directory in the WAR file.&nbsp; Generally your Spring and UI config will reside in the standard package structure within this directory. <br></p>
         <p>Resources that are used by your extension, such as XML import files or ACP's, may also reside in here, as it can often be convenient to place such things on the classpath for loading from Spring.<br></p>
         <p>As a module developer, you will be required to provide a <code>module-context.xml</code> (and optionally a <code>module-disable-context.xml</code> and <code>module-uninstall-context.xml</code>) in the <strong>alfresco.module.&lt;moduleId&gt;</strong> package (see <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FDeveloping_an_Alfresco_Module" rel="nofollow" target="_blank">Developing an Alfresco Module</a> for more details).&nbsp; These will reside in the <code>/config</code> directory.<br></p>
         <p>One should place server side webscripts here. AMP -&gt; war file mappings are:<br></p>
         <p>config/alfresco/templates/webscripts -&gt; $CATALINA_HOME/webapps/alfresco/WEB-INF/classes/alfresco/templates/webscripts<br></p>
         <p>config/alfresco/extension/templates/webscripts -&gt; $CATALINA_HOME/webapps/alfresco/WEB-INF/classes/alfresco/extension/templates/webscripts <br></p>
         <p>To be more specific /config/alfresco resides in&nbsp; $CATALINA_HOME/webapps/alfresco/WEB-INF/classes/alfresco/ <br></p>
         <br>
         <ul>
          <li><strong>/lib</strong><br></li>
         </ul>
         <p>This directory will be mapped into <code>/WEB-INF/lib</code>.&nbsp; It should contain any jar files that relate to your module.<br></p>
         <br>
         <ul>
          <li><strong>/licenses</strong><br></li>
         </ul>
         <p>If your module includes any 3rd party jars that require the inclusion of licences that are not currently included in the standard repository WAR, then these should be placed here.<br></p>
         <br>
         <ul>
          <li><strong>/web/jsp</strong><br></li>
         </ul>
         <p>This directory should contain any custom or modified JSP's that relate to your module.&nbsp; The contents are mapped into the <code>/jsp</code> directory in the WAR file.<br></p>
         <br>
         <ul>
          <li><strong>/web/css</strong><br></li>
         </ul>
         <p>This directory should contain any CSS files that relate to your module.&nbsp; The contents are mapped into the <code>/css</code> directory in the WAR file.<br></p>
         <br>
         <ul>
          <li><strong>/web/images</strong><br></li>
         </ul>
         <p>Any images that relate to your module should be placed here.&nbsp; The contents are mapped into the <code>/images</code> directory in the WAR file.<br></p>
         <br>
         <ul>
          <li><strong>/web/scripts</strong><br></li>
         </ul>
         <p>JavaScript files that are used by the user interface should be placed here.&nbsp; The contents are mapped into the <code>/scripts</code> directory in the WAR file.<br></p>
         <p>Any folder structures found in any of these directories are mapped, as they are found, into the destination folders in the WAR.<br></p>
         <p>If a file already exists it is overridden in the WAR; however, a recoverable backup is taken by the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a>.<br></p>
         <p>Any of these directories can be missing or empty.<br></p>
         <br>
         <ul>
          <li><strong>module.properties</strong><br></li>
         </ul>
         <p>The <code>module.properties</code> file is required to be present in an AMP file.&nbsp; It contains metadata about the module, most importantly the id and version of the module that this AMP file relates to.&nbsp; See <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FDeveloping_an_Alfresco_Module" rel="nofollow" target="_blank">Developing an Alfresco Module</a> for more details.<br></p>
         <br>
         <ul>
          <li><strong>file-mapping.properties</strong><br></li>
         </ul>
         <p>If you wish to customise the way in which your AMP file is mapped into the WAR file using the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a>, then this file should be provided in the AMP file.&nbsp; If it is not provided, then the default mapping information will be used.&nbsp; See <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FAMP_Files%23Customising_the_structure_of_an_AMP_file" rel="nofollow" target="_blank">Customising the structure of an AMP file</a> for more information.<br></p>
         <br>
         <h2>Customising the structure of an AMP file</h2>
         <br>
         <p>The structure of an AMP file may be customised via the use of 'file mappings' - these are the rules that govern where the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a> copies directories in the AMP file to in the target WAR file.&nbsp; By default, the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a> uses the following file mappings:<br></p>
         <br>
         <pre><br># The default AMP =&gt; WAR file mappings<br>/config=/WEB-INF/classes<br>/lib=/WEB-INF/lib<br>/licenses=/WEB-INF/licenses<br>/web/jsp=/jsp<br>/web/css=/css<br>/web/images=/images<br>/web/scripts=/scripts<br>/web/php=/php<br></pre>
         <br>
         <p>You can customise the structure of your AMP file by providing a <strong>file-mapping.properties</strong> file in the root directory of the AMP file.&nbsp; This is a file in standard Java properties format where the key of each property is the source path in the AMP (with a leading forward slash), and the value of that property is that destination path in the target WAR file (also with a leading forward slash).&nbsp; The contents of each mapped path will be recursively copied to the target at the time the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a> installs the AMP into a WAR file.<br></p>
         <p>You may control whether the default mappings are applied or not via the <strong>include.default</strong> setting - this setting is also provided in the file-mapping.properties.&nbsp; This boolean property defaults to true (inherit default mappings), but may be set to false as follows:<br></p>
         <br>
         <pre><br>include.default=false<br></pre>
         <br>
         <p>Custom file mappings always take precedence over the defaults, so you may include the defaults and then override them on a case-by-case basis.<br></p>
         <p>If the source directory does not exist in the AMP file, then the mapping will be ignored; however, the destination directory in the WAR file must exist or a runtime exception will be raised when the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FModule_Management_Tool" rel="nofollow" target="_blank">Module Management Tool</a> tries to install the AMP.<br></p>
         <br>
         <h3>Example</h3>
         <br>
         <pre><br># Custom AMP to WAR location mappings<br><br>#<br># The following property can be used to include the standard set of mappings.<br># The contents of this file will override any defaults.&nbsp; The default is<br># 'true', i.e. the default mappings will be augmented or modified by values in<br># this file.<br>#<br>include.default=false<br><br>#<br># Custom mappings.&nbsp; If 'include.default' is false, then this is the complete set.<br>#<br>/WEB-INF=/WEB-INF<br>/web=/<br></pre>
         <br>
         <h2>AMP Examples</h2>
         <br>
         <p>Many of the add-ons available on the <a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Faddons.alfresco.com%2F" rel="nofollow" target="_blank">Alfresco Add-ons Site</a> are delivered as AMP files, and their source code (where available) provides numerous examples of AMP file construction.<br></p>
         <p><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3AAdministration" rel="nofollow" target="_blank">Administration</a><br><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3AFile_Types" rel="nofollow" target="_blank">File Types</a><br><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3ADeployment" rel="nofollow" target="_blank">Deployment</a><br><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3ACustomizing_and_Extending" rel="nofollow" target="_blank">Customizing and Extending</a><br><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3APackaging" rel="nofollow" target="_blank">Packaging</a><br><a class="jive-link-external-small" href="https://community.alfresco.com/external-link.jspa?url=http%3A%2F%2Fwikitest.alfresco.com%2Fwiki%2FCategory%3AAMP" rel="nofollow" target="_blank">AMP</a><br></p>
