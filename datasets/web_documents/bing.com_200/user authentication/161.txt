       <p>In this article we will be implementing User Authentication in an ASP.NET MVC 6 application. Just like MVC 5, we have an Authentication Action Filter in MVC 6.</p> 
       <p>The following diagram gives an idea of Authentication when the end-user makes a call to an MVC 6 application.</p> 
       <p id="aspnet" class="articlead">&nbsp;</p> 
       <p><img title="aspnet-mvc-user-auth" style="background-image:none;padding-top:0;padding-left:0;display:inline;padding-right:0;border-width:0;" border="0" alt="aspnet-mvc-user-auth" src="http://www.dotnetcurry.com/images/mvc/Implementing-User-Authenticati.NET-MVC-6_E4AD/aspnet-mvc-user-auth.png" width="1383" height="724"></p> 
       <p>When the end-user makes a call to an MVC 6 application requesting a View, a response in the form of a View is returned when the action is executed. However if the Controller or the Action is applied with the <b>Authorize </b>attribute, then the request processing on the server sends the Login Page response to the client. Once the end-user sends the request with credentials, the Authentication Filter is executed which is responsible for validating Credentials against the Database server where the application users are stored. If the credentials are validated, then the Users will be <b>Logged In</b> and response of the View will be sent back to the user.</p> 
       <h3>Implementing Authentication in ASP.NET MVC</h3> 
       <p>To implement this application, we will be using <a href="http://www.bitly.com/dncvscommunity">Visual Studio 2015 Community Edition</a> and ASP.NET 5 RC1. ASP.NET 5 RC 1 can be downloaded from <a href="http://docs.asp.net/en/latest/getting-started/installing-on-windows.html">this link</a>.</p> 
       <p><b>Step 1: </b>Open Visual Studio 2015 and create a new ASP.NET application of name ASPNET5_Auth as shown in the following Image</p> 
       <p><img title="create-aspnet-project" border="0" alt="create-aspnet-project" src="http://www.dotnetcurry.com/images/mvc/Implementing-User-Authenticati.NET-MVC-6_E4AD/create-aspnet-project.png" width="986" height="555"></p> 
       <p>Click on <b>OK </b>and from the next window select ASP.NET Empty application from ASP.NET 5 templates as shown in the following image</p> 
       <p><img title="aspnet-empty-app" border="0" alt="aspnet-empty-app" src="http://www.dotnetcurry.com/images/mvc/Implementing-User-Authenticati.NET-MVC-6_E4AD/aspnet-empty-app.png" width="313" height="138"></p>

       <p><b>Step 3: </b>Since we need to store the Application users information in a SQL Server database, open Sql Server and create database of the name <b>Security.</b></p> 
       <p><b>Step 4: </b>In the project, add a new ASP.NET Configuration file, this will add appSettings.json file in the project. In this file add the following connection string</p>

            "ConnectionString": "Server=.;Database=Security;
                                    Trusted_Connection=True;"
        }

       <pre class="brush: csharp;">
using System.Threading.Tasks;
using Microsoft.AspNet.Mvc;
using Microsoft.AspNet.Authorization;
using ASPNET5_Auth.Models;
using Microsoft.AspNet.Identity;

 

namespace ASPNET5_Auth.Controllers
{

    public class AccountController : Controller
    {
        //1
        private readonly UserManager&lt;ApplicationUser&gt; _securityManager;
        private readonly SignInManager&lt;ApplicationUser&gt; _loginManager;
        //2
        public AccountController(UserManager&lt;ApplicationUser&gt; secMgr, SignInManager&lt;ApplicationUser&gt; loginManager)
        {
            _securityManager = secMgr;
            _loginManager = loginManager;
        }

        //3
        [HttpGet]
        [AllowAnonymous]
        public IActionResult Register()
        {
            return View();
        }

        //4
 
        [HttpPost]
        [AllowAnonymous]
        [ValidateAntiForgeryToken]
        public async Task&lt;IActionResult&gt; Register(Register model)
        {
            if (ModelState.IsValid)
            {
                var user = new ApplicationUser
                {
                    UserName = model.Email,
                    Email = model.Email
                };
                var result = await _securityManager.CreateAsync(user, model.Password);
                if (result.Succeeded)
                {
                    await _loginManager.SignInAsync(user, isPersistent: false);
                    return RedirectToAction(nameof(HomeController.Index), "Home");
                }
            }
            
            return View(model);
        }

        //5
        [HttpGet]
        [AllowAnonymous]
        public IActionResult Login(string returnUrl = null)
        {
            ViewData["ReturnUrl"] = returnUrl;
            return View();
        }

        //6
        [HttpPost]
        [AllowAnonymous]
        [ValidateAntiForgeryToken]
        public async Task&lt;IActionResult&gt; Login(Login model, string returnUrl = null)
        {
            ViewData["ReturnUrl"] = returnUrl;
            if (ModelState.IsValid)
            {
                var result = await _loginManager.PasswordSignInAsync(model.Email, model.Password, false, lockoutOnFailure: false);
                if (result.Succeeded)
                {
                    return RedirectToReturnUrl(returnUrl);
                }
            }

             
            return View(model);
        }

        //7
        [HttpPost]
        [ValidateAntiForgeryToken]
        public async Task&lt;IActionResult&gt; LogOff()
        {
            await _loginManager.SignOutAsync();
            
            return RedirectToAction(nameof(HomeController.Index), "Home");
        }
        //8
        private IActionResult RedirectToReturnUrl(string returnUrl)
        {
            if (Url.IsLocalUrl(returnUrl))
            {
                return Redirect(returnUrl);
            }
            else
            {
                return RedirectToAction(nameof(HomeController.Index), "Home");
            }
        }
    }
}</pre> 
       <p>This above code has the following specifications:</p> 
       <p>1. It declares objects of <b>UserManager&lt;T&gt; </b>and <b>SignInManager&lt;T&gt; </b>classes. The UserManager&lt;T&gt; is used to manage users from the persistent store i.e. database and the SignInManager&lt;T&gt; class is used to manager APIs for user Sign In. The <b>T </b>here is the ApplicationUser class which contains user information.</p> 
       <p>2. The constructor of the controller class is injected with the UserManager and SignInManager dependencies. These dependencies will be registered in future steps.</p> 
       <p>3. Step for responding with the <b>Register </b>view.</p> 
       <p>4. HttpPost <b>Register </b>method which accepts the <b>Register </b>model object. This method will create an application method by using <b>CreateAsync() </b>method of the SignInManager class. If the user is registered successfully then the user will be signed using <b>SignAsync() </b>method of the UserManager class.</p> 
       <p>5. The HttpGet <b>Login() </b>method which responds with a Login View.</p> 
       <p>6. The <b>Login() </b>HttpPost method is used for the model containing the Email and password. If the Login is successful, then it will redirect to the page from which the request for Login initiated.</p> 
       <p>7. The LogOff () method.</p> 
       <p>8. A private method to redirect to the url from where the login request initiated.</p>

public IConfigurationRoot Configuration { get; set; }

// This method gets called by the runtime. Use this method to add services to the container.
