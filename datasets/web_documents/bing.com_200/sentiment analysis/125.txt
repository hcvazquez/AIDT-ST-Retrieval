        <p align="justify"></p>
        <p><strong>Scope<br> </strong><br> This article is about demonstrating the use of Machine Learning to do Sentiment Analysis on texts.<br> <br> <strong>Introduction<br> </strong><br> Sentiment Analysis is the process of detecting the feeling or the mood of a person when writing a text (technically called contextual polarity). In other words, it determines whether a piece of writing is positive, negative or neutral.<br> <br> <strong>Uses of Sentiment Analysis</strong></p> 
        <ol> 
         <li>Product reviews<br> 
          <ul> 
           <li>Is the review positive or negative</li> 
          </ul> </li> 
         <li>Analyzing customer emails</li> 
         <li>Social Media Analytics
          <ul> 
           <li>What do customers think about my company</li> 
          </ul> </li> 
        </ol> 
        <p>If one has about 10 mails or products, this will be quite a simple task, but what if some store has thousands of products each of them with &nbsp;hundreds of reviews daily.<br> <br>Then, automating this process makes sense. And this is where Text Analytics comes into play.<br><br><strong>Text Analytics API<br> </strong><br>Text Analytics API is a suite of text analytics web services built with Azure Machine Learning. The API can be used to analyze unstructured text for tasks such as sentiment analysis and key phrase extraction.<br><br>The API returns a numeric score between 0 &amp; 1. Scores close to 1 indicate positive sentiment, while scores close to 0 indicate negative sentiment.<br> <br>The advantage of this API is that a new model need not be designed and trained, the user only needs to bring the data and call the service to get the sentiment results.<br> <br>However, because this is only the initial release of this service only English is supported right now.<br><br><strong>How it works<br> </strong><br>Text Analytics API does not simply use a lexicon approach to map words such as “good or “bad” to return the results.<br> <br>Instead, it uses advanced natural language processing techniques under the hood.<br> <br>For even more details of the service, consult the documentation <a href="http://blogs.technet.com/b/machinelearning/archive/2015/04/08/introducing-text-analytics-in-the-azure-ml-marketplace.aspx">here</a>.<br> <br><strong>Signing up for the Text Analytics Service</strong></p> 
        <ol> 
         <li>Go to <span style="color: #0000ff;"><span style="color: #0000ff;"><a href="https://datamarket.azure.com/dataset/amla/text-analytics">Text Analytics</a><br><br></span></span></li> 
         <li>Select the required package. <br> <br>For testing purposed, up to 10,000 transactions are available for free.<br> <br>The following are the prices as on October 10, 2015.<br><br> <img src="http://csharpcorner.mindcrackerinc.netdna-cdn.com/UploadFile/a5470d/sentiment-analysis-api-with-azure-machine-learning/Images/prices.jpg" alt="prices "> <br><br> </li> 
         <li>Once the registration is done, you shall be provided with an account key. Keep it safely, this shall be used in the applications.</li> 
        </ol> 
        <p><strong>Sample Chat Application<br> </strong><br>One cool application of this API can be to use it in a chat application which analyse the sentiment of each chat and automatically places a smiley for the user.<br><br><strong>Expected Outcome</strong><br><br><img src="http://csharpcorner.mindcrackerinc.netdna-cdn.com/UploadFile/a5470d/sentiment-analysis-api-with-azure-machine-learning/Images/chat example.jpg" border="0" alt="" width="648" height="192"><br><br><strong>High Level Application Design<br> </strong><br>When a user sends a message, the message is first broadcast to all the connected clients. Then, an asynchronous call is made to the Web Service to get the sentiment. After the sentiment score is obtained, the chat is updated with the corresponding smiley.<br><br><img src="http://csharpcorner.mindcrackerinc.netdna-cdn.com/UploadFile/a5470d/sentiment-analysis-api-with-azure-machine-learning/Images/High Level Application Design.jpg" alt="High Level Application Design"><strong></strong><br> <br><strong>Diving in the Codes<br> </strong><br>This is an ASP.NET MVC Application which makes calls to the Web Service asynchronously and place the smiley only after the result is obtained.<br> <br><strong>Calling the Text Analytics Service</strong></p>

        <p><strong>The JavaScript<br> <br></strong>There are 2 main functions on the JavaScript part, to send a message and to append messages to the div.</p> 
        <ol type="a"> 
         <li>Append the messages to the div<br> <br>Here, 2 actions are defined. Action 1 is when a broadcast happen and action 2 is when the result from the Web Service is obtained and it appends the smiley to the existing div.<br>

           <div class="bar"></div>

            <li class="alt"><span>chat.client.addNewMessageToPageChat&nbsp;=&nbsp;function(message,&nbsp;id,&nbsp;action)&nbsp;{&nbsp;&nbsp;</span></li>

            <li class="alt"><span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;document.getElementById(<span class="string">'message'</span><span>).innerHTML&nbsp;=&nbsp;message&nbsp;+&nbsp;</span><span class="string">'&lt;label&nbsp;id&nbsp;='</span><span>&nbsp;+&nbsp;id&nbsp;+&nbsp;</span><span class="string">'&gt;&nbsp;&lt;/label&gt;&nbsp;'</span><span>&nbsp;+&nbsp;</span><span class="string">'&lt;br/&gt;'</span><span>&nbsp;+&nbsp;text;&nbsp;&nbsp;</span></span></li>
            <li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;}&nbsp;&nbsp;</span></li>
            <li class="alt"><span>&nbsp;&nbsp;&nbsp;&nbsp;<span class="comment">//update</span><span>&nbsp;&nbsp;</span></span></li>

            <li class="alt"><span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;var&nbsp;name&nbsp;=&nbsp;$(<span class="string">'#name'</span><span>).val();&nbsp;&nbsp;</span></span></li>

            <li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;chat.server.send(tosen);&nbsp;&nbsp;</span></li>
