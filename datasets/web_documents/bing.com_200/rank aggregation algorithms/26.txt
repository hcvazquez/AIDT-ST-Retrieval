             <div class="post-text" itemprop="text"> 
              <p><strong>Prehistory</strong></p> 
              <p>I am trying to make some statistical analysis of some experimental data, arises from measurements made ??on an ordinal scale.</p> 
              <p>I faced with the problem of <em>rank aggregation</em>: to get from many "individuals" orderings (on the same set of objects) one "collective" ordering.</p> 
              <p>The most "natural" approach to this problem is <em><a href="http://en.wikipedia.org/wiki/Kemeny%E2%80%93Young_method" rel="nofollow noreferrer">Kemeny-Young method</a></em> (better look <a href="http://babel.hathitrust.org/cgi/pt?id=mdp.39015002661802;view=1up;seq=21" rel="nofollow noreferrer">primary source</a>).</p> 
              <p>Surprisingly I found out that there is no program application for this method!!! (There is <a href="http://www.nr.com/whp/ky/kemenyyoung.html" rel="nofollow noreferrer">one C++ code</a>, but it does not allow weak orderings, i.e., does not allow cases when several objects share same position at ordering).</p> 
              <p>Previously I asked some points (<a href="https://mathematica.stackexchange.com/questions/37312/how-to-get-all-possible-orderings-equal-ranks-allowed-for-given-set-of-n-obj">one</a>, <a href="https://mathematica.stackexchange.com/questions/37813/how-to-constrain-the-generation-of-all-possible-orderings">two</a>) for constructing needed code, but now I have decide to tell all at once — because the problem is more complicated than I thought at first, and I can miss some points since I am null in <em>Mathematica</em> and programming.</p> 
              <p><strong>Description</strong></p> 
              <p>Let $R_1, R_2, R_3, ..., R_N$ denote "individuals" weak orderings of $N$ given objects $a, b, c, ...$.</p> 
              <p>Let consider $R$ as set of ordered pairs of objects. Then $(a, b)\in R$ has the usual interpretation: "$a$ is <em>ordered at least as good as b</em> at $R$". In case $(b, a)$ is also in $R$ we say that "<em>both are ordered equally good</em>". Where in case $(b,a)$ is not in $R$ we say that "$a$ is <em>ordered above</em> $b$" or "$b$ is <em>ordered below</em> $a$".</p> 
              <p>(After introducing metric we will see that it is possible to neglect — since they don't make further difference — such pairs as $(a, a), (b, b), (c, c)...$.)</p> 
              <blockquote> 
               <p>To demonstrate orderings notation, here are all possible orderings for $N=3$:</p> 
               <p>$\ R_1=\{(a, b), (a, c), (b, c)\}$</p> 
               <p>$\ R_2=\{(a, c), (a, b), (c, b)\}$</p> 
               <p>$\ R_3=\{(a, b), (a, c), (b, c), (c, b)\}$</p> 
               <p>$\ R_4=\{(b, a), (b, c), (a, c)\}$</p> 
               <p>$\ R_5=\{(b, c), (b, a), (c, a)\}$</p> 
               <p>$\ R_6=\{(b, a), (b, c), (a, c), (c, a)\}$</p> 
               <p>$\ R_7=\{(c, a), (c, b), (a, b)\}$</p> 
               <p>$\ R_8=\{(c, b), (c, a), (b, a)\}$</p> 
               <p>$\ R_9=\{(c, a), (c, b), (a, b), (b, a)\}$</p> 
               <p>$R_{10}=\{(a, b), (b, a), (a, c), (b, c)\}$</p> 
               <p>$R_{11}=\{(a, c), (c, a), (a, b), (c, b)\}$</p> 
               <p>$R_{12}=\{(b, c), (c, b), (b, a), (c, a)\}$</p> 
               <p>$R_{13}=\{(a, b), (b, a), (a, c), (c, a), (b, c), (c, b)\}$</p> 
              </blockquote> 
              <p>For such notation we may introduce metric $\delta$ (so-called <em>Kemeny distance</em>) between any two orderings $R_1$ and $R_2$ by next way: $$\delta(R_1,R_2)=|R_1\setminus R_2|+|R_2\setminus R_1|$$ where $\setminus$ means set-theoretic difference; and $||$ means cardinality of set (i.e., number of elements, because sets are finite).</p> 
              <p>The required "collective" ordering $?$ (so-called <em>Kemeny mean</em>) is such ordering of given objects, that minimizes the sum of the squares of Kemeny distances to all "individuals" orderings, i.e.: $$?=\min \sum_{i=1}^{N } \delta(R_i,?)^2$$</p> 
              <p>In fact, $?$ may be not unique, there may be several $?_1, ?_2, ...$ for which the appropriate sums of the squares of Kemeny distances are equal and minimal. So in general case $?$ is set of such $?_i$.</p> 
              <p><strong>How to attack</strong></p> 
              <p>So we have several of $R_i$ as input and the goal is to output $?$.</p> 
              <p>It looks, like there should be brute-force with one-by-one generating of possible ordering, calculating the sum of the squares of Kemeny distances to all input orderings and verifiaction for minimum.</p> 
              <p>Even for small $N$ number of possible orderings is huge (for my data case $N=10$, there are over $10^8$ possible orderings), so we should keep less data.</p>

              <p>[<strong>Warning:</strong> There could be issues with the averaging procedure. At least one should pay attention to <code>allPossibleOrderings</code>; its definition probably needs to be improved. See other two *<em>edit</em>*s in text below.]</p> 
              <p>I'm probably doing something nasty here but if RAM is <em>really</em> not a problem for you, then… here you go, a blunt brute-force approach:</p> 
              <pre><code>KemenyDistance =
Plus @@
Composition[Length, Complement] @@@
Permutations[{##}, {2}] &amp;;

FindKemenyMean[setOfOrderings:{__List}, allObjects:_List:{}] :=
Block[{ weakAndNonWeakOrderings, allPossibleOrderings
      , kemenySumOfSquares
      , minFound, data, \[CapitalOmega]candidate },

weakAndNonWeakOrderings@objects_List :=
With[{orderingFromOrderedList = Partition[#, 2, 1]&amp;},
Block[{nonWeakOrderingsOnly, addWeakOrderings},
  nonWeakOrderingsOnly =
  orderingFromOrderedList /@ Permutations[#, {Length@#}]&amp;;

  addWeakOrderings@nonWeakOrdering_ :=
  Join[#, nonWeakOrdering]&amp; /@
  (Permutations[Reverse/@#, Length@#]&amp;@nonWeakOrdering);

  Flatten[addWeakOrderings /@ nonWeakOrderingsOnly@objects, 1]]];

allPossibleOrderings =
weakAndNonWeakOrderings @
If[allObjects === {}
,  Composition[DeleteDuplicates, Flatten]@setOfOrderings
,  allObjects];

kemenySumOfSquares[\[CapitalOmega]_, orderingsToAverage_List] :=
Plus @@ (KemenyDistance[#, \[CapitalOmega]]^2 &amp; /@ orderingsToAverage);

Fold[
  With[
    { currentMin = First@Cases[#1, minFound@n_ :&gt; n, {1}, 1]
    , currentOrdering = #2 }
  , With[
     { currentSum =
       kemenySumOfSquares[currentOrdering, setOfOrderings] }
    , If[currentSum &lt; currentMin
      ,  data[minFound@currentSum, \[CapitalOmega]candidate@currentOrdering]
      ,  #1]]] &amp;
  , data[minFound[\[Infinity]], \[CapitalOmega]candidate[]]
  , allPossibleOrderings]]
</code></pre> 
              <p>These two definitions give you <code>KemenyDistance</code> and <code>FindKemenyMean</code> precisely in the form you described. <code>FindKemenyMean</code> fetches initial objects from the orderings list which will always work if your orderings are linear (= complete).</p> 
              <pre><code>In[1]:= FindKemenyMean[
          { {{a, b}, {b, c}, {c, b}, {c, d}},
            {{b, d}, {d, c}, {c, a}},
            {{c, a}, {a, d}, {d, b}}  }]
Out[1]= data[minFound[45], \[CapitalOmega]candidate[{{b, c}, {c, a}, {a, d}}]]
</code></pre> 
              <p>You can provide list of all objects independently as its second argument but I don't guarantee anything for incomplete orderings (just don't have enough mathematical intuition for that).</p> 
              <p>Notes:</p> 
              <p>1) My calculation of all orderings, with weak ones included, for $N = 3$, evaluates list that is somewhat longer than the one you provided. I probably generate equivalent (hence redundant) weak orderings! [<strong>edit:</strong> Or maybe I should have converted them to some canonical form.]</p> 
              <p>2) The $R_i$ data structure for orderings is obviously redundant, hence definitely inefficient. [<strong>edit:</strong> not so if you don't presume that the relation is transitive, or if one <em>must</em> get to some kind of “complete form” of all $R_i$'s in order to calculate Kemeny mean w.r.t. them] I don't understand a thing in statistics but I suggest describing orderings in a more concise manner; Mma has gives lots of possibilites to do that. For example, you could use weighted sums, like $a + 2b + 3c$, to describe that $a &lt; b &lt; c$ (or maybe $a + (1+\varepsilon)b + (1+\varepsilon)^2c$ would be a better idea). Calculation of average ordering then would become an immensely more simple and efficient procedure (you could just use arithmetic mean, or something close to it), although I can't say much about its sanity. At least orderings that treat $a$ and $b$ as equal would always get averaged to orderings with the same property in case you pick simple arithmetic mean.</p> 
              <p>3) I use a lot of nested scoping constructs. Normally, one would write a small package instead. I hope this won't make it significantly more dificult for you to find out what's going on here in case you're interested in <em>Mathematica</em>'s core functions. Hopefully, explicit names of symbols (I could say it's a [really good] Mma tradition) will be of help.</p>
