   <h1>4.3 &nbsp; Stacks and Queues</h1> 
   <p><br> In this section, we introduce two closely-related data types for manipulating arbitrarily large collections of objects: the <em>stack</em> and the <em>queue</em>. Stacks and queues are special cases of the idea of a <em>collection</em>. Each is characterized by four operations: <em>create</em> the collection, <em>insert</em> an item, <em>remove</em> an item, and test whether the collection is <em>empty</em>.

   <h2>Array implementations of stacks.</h2> Representing stacks with arrays is a natural idea. In particular, we maintain an instance variable 
   <tt>n</tt> that stores the number of items in the stack and an array 
   <tt>items[]</tt> that stores the 
   <tt>n</tt> items, with the most recently inserted item in 
   <tt>items[n-1]</tt> and the least recently inserted item in 
   <tt>items[0]</tt>. This policy allows us to add and remove items at the end without moving any of the other items in the stack. 
   <ul> 
    <a name="4.3.1"></a> 
    <p></p>
    <li> <em>Fixed-length array implementation of a stack of strings.</em> <a href="ArrayStackOfStrings.java.html">ArrayStackOfStrings.java</a> implements this approach for a stack of strings whose maximum capacity is specified by the argument to the constructor. To remove an item, we decrement <tt>n</tt> and then return <tt>a[n]</tt>; to insert a new item, we set <tt>a[n]</tt> equal to the new item and then increment <tt>n</tt>. <p> </p>
     <blockquote> 
      <img SRC="images/stack-array-trace.png" width="300" alt="Trace of ArrayStackOfStrings test client"> 
     </blockquote> 
     <!--
The primary characteristic of this implementation is that
<em>the push and pop operations take constant time</em>. The drawback 
of this implementation is that it requires the client to estimate the maximum 
size of the stack ahead of time and always uses space proportional to that 
maximum.
--> </li>
    <li><em>Resizing array implementation of a stack of strings.</em> <a href="ResizingArrayStackOfStrings.java.html">ResizingArrayStackOfStrings.java</a> is a version of <a href="ArrayStackOfStrings.java.html">ArrayStackOfStrings.java</a> that dynamically adjusts the length of the array <tt>items[]</tt> so that it is sufficiently large to hold all of the items and but not so large as to waste an excessive amount of space. First, in <tt>push()</tt>, we check whether there is room for the new item; if not, we create a new array of <em>double</em> the length of the old array and copy the items from the old array to the new array. Similarly, in <tt>pop()</tt>, we check whether the array is too large, and we <em>halve</em> its length if that is the case. <p> </p>
     <blockquote> 
      <img SRC="images/stack-resizing-array-trace.png" width="500" alt="trace of a stack implementation using array doubling"> 
     </blockquote> This doubling-and-halving strategy guarantees that that the stack never overflows and never becomes less than one-quarter full. <p></p></li>
    <li><em>Resizing array implementation of a generic stack.</em> <a href="ResizingArrayStack.java.html">ResizingArrayStack.java</a> implements a generic stack using a resizing array. For technical reasons, a cast is needed when allocating the array of generics. </li>
   </ul>

   <h2>Linked lists.</h2> A 
   <em>singly linked list</em> comprises a sequence of 
   <em>nodes</em>, with each node containing a reference (or 
   <em>link</em>) to its successor. By convention, the link in the last node is 
   <em>null</em>, to indicate that it terminates the list. With object-oriented programming, implementing linked lists is not difficult. We define a class for the node abstraction that is 
   <em>recursive</em> in nature: 
   <blockquote> 
    <table> 
     <tbody>
      <tr>
       <td><pre>
class Node { 
   String item; 
   Node next; 
} 
</pre></td> 
       <td> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </td> 
       <td> <img SRC="images/linked-list-anatomy.png" width="400" align="right" hspace="25" alt="Anatomy of a singly linked list"> </td> 
      </tr> 
     </tbody>
    </table> 
   </blockquote> A 
   <tt>Node</tt> object has two instance variables: a 
   <tt>String</tt> and a 
   <tt>Node</tt>. The 
   <tt>String</tt> is a placeholder in this example for any data that we might want to structure with a linked list (we can use any set of instance variables); the instance variable of type 
   <tt>Node</tt> characterizes the linked nature of the data structure. 
   <ul> 
    <li><em>Linking together a linked list.</em> For example, to build a linked list that contains the items <tt>"to"</tt>, <tt>"be"</tt>, and <tt>"or"</tt>, we create a <tt>Node</tt> for each item: <p> </p>
     <blockquote> 
      <img SRC="images/linked-list.png" width="250" alt="linking together a linked list"> 
     </blockquote> </li>
    <li><em>Insert.</em> Suppose that you want to insert a new node into a linked list. The easiest place to do so is at the beginning of the list. For example, to insert the string <tt>not</tt> at the beginning of a given linked list whose first node is <tt>first</tt>, we save <tt>first</tt> in a temporary variable <tt>oldFirst</tt>, assign to <tt>first</tt> a new <tt>Node</tt>, and assign its <tt>item</tt> field to <tt>not</tt> and its <tt>next</tt> field to <tt>oldFirst</tt>. <p> </p>
     <blockquote> 
      <img SRC="images/linked-list-insert.png" width="350" alt="inserting an item into a linked list"> 
     </blockquote> </li> 
    <li><em>Remove.</em> Suppose that you want to remove the first node from a list. This operation is even easier: simply assign to <tt>first</tt> the value <tt>first.next</tt>. <p> </p>
     <blockquote> 
      <img SRC="images/linked-list-remove.png" width="250" alt="removing an item from a linked list"> 
     </blockquote> </li> 
    <li><em>Traversal.</em> To examine every item in a linked list, we initialize a loop index variable <tt>x</tt> that references the the first <tt>Node</tt> of the linked list. Then, we find the value of the item associated with <tt>x</tt> by accessing <tt>x.item</tt>, and then update <tt>x</tt> to refer to the next <tt>Node</tt> in the linked list, assigning to it the value of <tt>x.next</tt> and repeating this process until <tt>x</tt> is <tt>null</tt> (which indicates that we have reached the end of the linked list). This process is known as <em>traversing</em> the list, and is succinctly expressed in this code fragment: 
     <blockquote> 
      <table> 
       <tbody>
        <tr>
         <td><pre>
for (Node x = first; x != null; x = x.next) 
    StdOut.println(x.item);
</pre></td> 
         <td> </td>
        </tr>
       </tbody>
      </table> 
     </blockquote> 
     <blockquote> 
      <img SRC="images/linked-list-traverse.png" width="300" alt="Traversing a singly linked list"> 
     </blockquote> </li>
   </ul>

   <h2>Implementing stacks with linked lists.</h2> Representing stacks with linked lists is a natural idea. In particular, we maintain an instance variable 
   <tt>first</tt> that stores a reference to the most recently inserted item. This policy allows us to add and remove items at the beginning of the linked list without accessing the links of any other items in the linked list. 
   <ul> 
    <li><em>Linked-list implementation of a stack of strings.</em> <a href="LinkedStackOfStrings.java.html">LinkedStackOfStrings.java</a> uses a linked list to implement a stack of strings. The implementation is based on a <em>nested class</em> <tt>Node</tt> like the one we have been using. Java allows us to define and use other classes within class implementations in this natural way. We designate the nested class as <tt>private</tt> because clients do not need to know any of the details of the linked lists. <p> </p>

     </blockquote> </li>
    <li><em>Linked-list implementation of a generic stack.</em> <a href="Stack.java.html">Stack.java</a> implements a generic stack using a singly linked list. </li>
   </ul>

    <li><b>Indexing a book.</b> Write a program that reads in a text file from standard input and compiles an alphabetical index of which words appear on which lines, as in the following input. Ignore case and punctuation. Similar to FrequencyCount, but for each word maintain a list of location on which it appears. </li>
    <li><b>Copy constructor for a resizing array implementation of a stack.</b> Add a copy constructor to <a href="ArrayStackOfStrings.java.html">ArrayStackOfStrings.java</a> </li>
    <li><b>Reorder linked list.</b> Given a singly linked list x1 -&gt; x2 -&gt; ... -&gt; x_2n containing 2n nodes, rearrnage the nodes to be x1 -&gt; x2n -&gt; x2 -&gt; x_2n-1 -&gt; x3 -&gt; .... <em>Hint</em>: break the linked list in half; reverse the order of the nodes in the second linked list; merge the two lists together. </li>
