      <p>Strictly speaking, LaTeX source can be used to directly generate two formats:</p> 
      <ul> 
       <li>DVI using <tt>latex</tt>, the first one to be supported;</li> 
       <li>PDF using <code class="mw-highlight" dir="ltr">pdflatex</code>, more recent.</li> 
      </ul> 
      <p>Using other software freely available on Internet, you can easily convert DVI and PDF to other document formats. In particular, you can obtain the PostScript version using software which is included in your LaTeX distribution. Some LaTeX IDE will give you the possibility to generate the PostScript version directly (even if it uses internally a DVI mid-step, e.g. LaTeX ? DVI ? PS). It is also possible to create PDF from DVI and vice versa. It doesn't seem logical to create a file with two steps when you can create it straight away, but some users might need it because, as you remember from the first chapters, the format you can generate depends upon the formats of the images you want to include (EPS for DVI, PNG and JPG for PDF). Here you will find sections about different formats with description about how to get it.</p> 
      <p>Other formats can be produced, such as RTF (which can be used in Microsoft Word) and HTML. However, these documents are produced from software that parses and interprets the LaTeX files, and do not implement all the features available for the primary DVI and PDF outputs. Nonetheless, they do work, and can be crucial tools for collaboration with colleagues who do not edit documents with LaTeX.</p>

      <h2><span class="mw-headline" id="Tools_installation">Tools installation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=1" title="Edit section: Tools installation">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>This chapter features a lot of third-party tools; most of them are installed independently of your TeX distribution.</p> 
      <p>Some tools are Unix-specific (*BSD, GNU/Linux and Mac OS X), but it may be possible to make them work on Windows. If you have the choice, it is often easier with Unix systems for command line tools.</p> 
      <p>Some tools may already be installed. For instance, you can check if dvipng is installed and ready to use (Unix only):</p> 
      <pre>
type dvipng
</pre> 
      <p>Most of these tools are installable using your package manager or portage tree (Unix only).</p> 
      <h2><span class="mw-headline" id="Preview_mode">Preview mode</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=2" title="Edit section: Preview mode">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

First, start by making sure you have <span style="font-family: monospace; color: #2020C0; font-weight: normal;">preview</span>. See <a href="/wiki/LaTeX/Installing_Extra_Packages" title="LaTeX/Installing Extra Packages">Installing Extra Packages</a>.</p>

      <h3><span class="mw-headline" id="Directly">Directly</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=4" title="Edit section: Directly">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

</pre> 
      <h3><span class="mw-headline" id="DVI_to_PDF">DVI to PDF</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=5" title="Edit section: DVI to PDF">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <pre>

      <p>will create <tt>my_file.pdf</tt>. Another way is to pass through PS generation:</p>

      <h3><span class="mw-headline" id="Merging_PDF">Merging PDF</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=6" title="Edit section: Merging PDF">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>If you have created different PDF documents and you want to merge them into one single PDF file you can use the following command-line command. You need to have Ghostscript installed:</p> 
      <h4><span class="mw-headline" id="Using_Windows">Using Windows</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=7" title="Edit section: Using Windows">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <pre>
gswin32 -dNOPAUSE -sDEVICE=pdfwrite -sOUTPUTFILE=Merged.pdf -dBATCH 1.pdf 2.pdf 3.pdf
</pre> 
      <h4><span class="mw-headline" id="Using_Linux">Using Linux</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=8" title="Edit section: Using Linux">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <pre>
gs -dNOPAUSE -sDEVICE=pdfwrite -sOUTPUTFILE=Merged.pdf -dBATCH 1.pdf 2.pdf 3.pdf
</pre> 
      <p>Alternatively, <a rel="nofollow" class="external text" href="http://pdfshuffler.sourceforge.net/">PDF-Shuffler</a> is a small python-gtk application, which helps the user to merge or split pdf documents and rotate, crop and rearrange their pages using an interactive and intuitive graphical interface. This program may be available in your Linux distribution's repository.</p> 
      <p>Another option to check out is <a rel="nofollow" class="external text" href="http://www.accesspdf.com/">pdftk</a> (or PDF toolkit), which is a command-line tool that can manipulate PDFs in many ways. To merge one or more files, use:</p> 
      <pre>
pdftk 1.pdf 2.pdf 3.pdf cat output 123.pdf
</pre> 
      <h4><span class="mw-headline" id="Using_pdfLaTeX">Using pdfLaTeX</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=9" title="Edit section: Using pdfLaTeX">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p><i>Note:</i> If you are merging external PDF documents into a LaTeX document which is compiled with <code>pdflatex</code>, a much simpler option is to use the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">pdfpages</span> package, <i>e.g.</i>:</p> 
      <table style="margin-top: 1.5em; margin-bottom: 1.5em"> 
       <tbody>

...

      <p>Three simple <a href="https://en.wikipedia.org/wiki/Shell_(computing)" class="extiw" title="wikipedia:Shell (computing)">shell</a> scripts using the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">pdfpages</span> package are provided in the <a rel="nofollow" class="external text" href="http://www2.warwick.ac.uk/fac/sci/statistics/staff/academic/firth/software/pdfjam">pdfjam bundle</a> by D.       <h3><span class="mw-headline" id="XeTeX">XeTeX</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=10" title="Edit section: XeTeX">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>You can also use XeTeX (or, more precisely, XeLaTeX), which works in the same way as <tt>pdflatex</tt>: it creates a PDF file directly from LaTeX source. One advantage of XeTeX over standard LaTeX is support for Unicode and modern typography. See <a href="https://en.wikipedia.org/wiki/XeTeX" class="extiw" title="w:XeTeX">its Wikipedia entry</a> for more details.</p> 
      <p>Customization of PDF output in XeTeX (setting document title, author, keywords etc.) is done using the configuration of <a href="/wiki/LaTeX/Hyperlinks#Customization" title="LaTeX/Hyperlinks">hyperref</a> package.</p>

      <h2><span class="mw-headline" id="Convert_to_RTF">Convert to <a href="https://en.wikipedia.org/wiki/Rich_Text_Format" class="extiw" title="w:Rich Text Format">RTF</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=12" title="Edit section: Convert to RTF">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>LaTeX can be converted into an RTF file, which in turn can be opened by a word processor such as <a href="https://en.wikipedia.org/wiki/LibreOffice_Writer" class="extiw" title="w:LibreOffice Writer">LibreOffice Writer</a> or <a href="https://en.wikipedia.org/wiki/Microsoft_Word" class="extiw" title="w:Microsoft Word">Microsoft Word</a>. This conversion is done through <a rel="nofollow" class="external text" href="http://latex2rtf.sourceforge.net/">latex2rtf</a>, which may run on any computer platform, however is only actively supported on Windows, Linux and BSD, with the last mac update being from 2001 (a recent version for OSX is available via MacPorts). The program operates by reading the LaTeX source, and mimicking the behaviour of the LaTeX program. <code>latex2rtf</code> supports most of the standard implementations of LaTeX, such as standard formatting, some math typesetting, inclusion of EPS, PNG or JPG graphics, and tables. As well, it has some limited support for packages, such as varioref, and natbib. However, many other packages are not supported.</p> 
      <p><tt>latex2rtf</tt> is simple to use. The Windows version has a GUI (<tt>l2rshell.exe</tt>), which is straightforward to use. The command-line version is offered for all platforms, and can be used on an example <tt>mypaper.tex</tt> file:</p> 
      <pre>
latex mypaper
bibtex mypaper # if you use bibtex
latex2rtf mypaper
</pre> 
      <p>Both <code>latex</code> and (if needed) <code>bibtex</code> commands need to be run <i>before</i> <code>latex2rtf</code>, because the <tt>.aux</tt> and <tt>.bbl</tt> files are needed to produce the proper output. The result of this conversion will create <tt>myfile.rtf</tt>, which you may open in many word processors such as Microsoft Word or LibreOffice.</p> 
      <h2><span class="mw-headline" id="Convert_to_HTML">Convert to HTML</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=13" title="Edit section: Convert to HTML">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>There are many converters to HTML. Some of them use an intermediate file which then will be converted to the destination format.</p> 
      <dl> 
       <dt>
        <a rel="nofollow" class="external text" href="http://hevea.inria.fr">HEVEA</a>
       </dt> 
      </dl> 
      <pre>
hevea mylatexfile
</pre> 
      <dl> 
       <dt>
        latex2html
       </dt> 
      </dl> 
      <pre>
latex2html -html_version 4.0,latin1,unicode -split 1 -nonavigation -noinfo -title "MyDocument" MyDocument.tex
</pre> 
      <dl> 
       <dt>
        <a href="https://en.wikipedia.org/wiki/LaTeXML" class="extiw" title="w:LaTeXML">LaTeXML</a>
       </dt> 
      </dl> 
      <pre>
latexmlc paper.tex --destination=paper.html
</pre> 
      <dl> 
       <dt>
        <a rel="nofollow" class="external text" href="https://github.com/coolwanglu/pdf2htmlEX">pdf2htmlEX</a>
       </dt> 
      </dl> 
      <pre>
pdf2htmlEX [options] &lt;input.pdf&gt; [&lt;output.html&gt;]
</pre> 
      <p>pdf2htmlEX can convert PDF to HTML without losing text or format. It is designed as a general PDF to HTML converter, not only restricted to the PDF generated by LaTeX source. LaTeX users can compile the LaTeX source code to PDF, and then convert the PDF to HTML via pdf2htmlEX. Some introductions of pdf2htmlEX can be found on its own <a rel="nofollow" class="external text" href="https://github.com/coolwanglu/pdf2htmlEX/wiki">wiki page</a>. More technical details can be found on the paper published on TUGboat: <i>Online publishing via pdf2htmlEX</i> <a rel="nofollow" class="external text" href="http://coolwanglu.github.io/pdf2htmlEX/doc/tb108wang.html">HTML</a> / <a rel="nofollow" class="external text" href="http://coolwanglu.github.io/pdf2htmlEX/doc/tb108wang.pdf">PDF</a>. The Figure 3 of the paper gives different work-flows of publishing HTML online.</p> 
      <dl> 
       <dt>
        TeX4ht
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://tug.org/tex4ht/">TeX4ht</a> has many options and possible configurations, but for a basic conversion,</p> 
      <pre>
 htlatex myfile.tex
</pre> 
      <p>will usually result in a reasonable HTML approximation. An introduction by the original author was published in TUGboat <a rel="nofollow" class="external autonumber" href="http://tug.org/TUGboat/tb25-1/gurari.pdf">[1]</a>.</p> 
      <dl> 
       <dt>
        <a rel="nofollow" class="external text" href="http://freecode.com/projects/bibtex2html">bibtex2html</a>
       </dt> 
      </dl> 
      <p>For exporting the BibTeX file only.</p> 
      <pre>
bibtex2html mybibtexfile
</pre> 
      <h2><span class="mw-headline" id="Convert_to_image_formats">Convert to image formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=14" title="Edit section: Convert to image formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>It is sometimes useful to convert LaTeX output to image formats for use in systems that do not support DVI nor PDF files, such as Wikipedia.</p> 
      <p>There are two families of graphics:</p> 
      <ul> 
       <li>Vector graphics can be scaled to any size, thus do not suffer from quality loss. <a href="https://en.wikipedia.org/wiki/Scalable_Vector_Graphics" class="extiw" title="w:Scalable Vector Graphics">SVG</a> is a vector format.</li> 
       <li>Raster graphics define every pixel explicitly. <a href="https://en.wikipedia.org/wiki/PNG" class="extiw" title="w:PNG">PNG</a> is a raster format.</li> 
      </ul> 
      <p>So vector graphics are usually preferred. There is still some cases where raster graphics are used:</p> 
      <ul> 
       <li>The target system does not handle vector graphics, only raster graphics are supported.</li> 
       <li>SVG can not embed fonts. So either the font will be rendered using a local .ttf or .otf font (which will mostly change the output), or all characters must be turned to vector graphics. This last method makes the SVG big and slow. If the input LaTeX file contains a lot of text which formatting must be preserved, SVG is not that great.</li> 
      </ul> 
      <p>So SVG is great for drawings and a small amount of text. JPG is a well known raster formats, however it is usually not as good as PNG for text.</p> 
      <p>In some cases it may be sufficient to simply copy a region of a PDF (or PS) file using the tools available in a PDF viewer (for example using LaTeX to typeset a formula for pasting into a presentation). This however will not generally have sufficient resolution for whole pages or large areas.</p> 
      <h3><span class="mw-headline" id="Multiple_formats">Multiple formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=15" title="Edit section: Multiple formats">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        pdftocairo
       </dt> 
      </dl> 
      <p>There is <tt>pdftocairo</tt> featured in the poppler toolset.</p> 
      <pre>
pdftocairo -svg latexdoc.pdf output.svg
</pre> 
      <p><tt>pdftocairo</tt> also supports various raster graphic formats.</p> 
      <h3><span class="mw-headline" id="Vector_graphics">Vector graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=16" title="Edit section: Vector graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        pdf2svg
       </dt> 
      </dl> 
      <p>Direct conversion from PDF to SVG can be done using the command line tool <a rel="nofollow" class="external text" href="http://www.cityinthesky.co.uk/opensource/pdf2svg/">pdf2svg</a>.</p> 
      <pre>
pdf2svg file.pdf file.svg
</pre> 
      <dl> 
       <dt>
        ps2svg
       </dt> 
      </dl> 
      <p>Alternatively DVI or PDF can be converted to PS as described before, then the bash script <a class="external text" href="http://en.wikipedia.org/wiki/Wikipedia:WikiProject_Electronics/Ps2svg.sh">ps2svg.sh</a> can be used (as all the software used by this script is multiplatform, this is also possible in Windows, a step-by-step guide could be written).</p> 
      <dl> 
       <dt>
        dvisvgm
       </dt> 
      </dl> 
      <p>One can also use <a rel="nofollow" class="external text" href="http://dvisvgm.sourceforge.net/">dvisvgm</a>, an open source utility that converts from DVI to SVG.</p> 
      <pre>
dvisvgm -n file.dvi
</pre> 
      <dl> 
       <dt>
        Inkscape
       </dt> 
      </dl> 
      <p>Inkscape is able to convert to SVG, PDF, EPS, and other vector graphic formats.</p> 
      <pre>
inkscape --export-area-drawing --export-ps=OUTPUT INPUT
inkscape --export-area-page --export-plain-svg=OUTPUT INPUT
</pre> 
      <h3><span class="mw-headline" id="Raster_graphics">Raster graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=17" title="Edit section: Raster graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        JPEG
       </dt> 
      </dl> 
      <p>Run <tt>ghostscript</tt> on the PostScript file created by <tt>pdf2ps</tt> as follows:</p> 
      <pre>
echo "quit" | gs -sDEVICE=jpeg -sOutputFile=document.jpg -r300 document.ps
</pre> 
      <dl> 
       <dt>
        GIMP
       </dt> 
      </dl> 
      <p>Open your file with <a href="/wiki/GIMP" title="GIMP">GIMP</a>. It will ask you which page you want to convert, whether you want to use anti-aliasing (choose <i>strong</i> if you want to get something similar to what you see on the screen). Try different resolutions to fit your needs, but 100 dpi should be enough. Once you have the image within GIMP, you can post-process it as you like and save it to any format supported by GIMP, as PNG for example.</p> 
      <dl> 
       <dt>
        dvipng
       </dt> 
      </dl> 
      <p>A method for DVI files is <a rel="nofollow" class="external text" href="http://savannah.nongnu.org/projects/dvipng/">dvipng</a>. Usage is the same as <tt>dvipdfm</tt>.</p> 
      <p>Run <tt>latex</tt> as usual to generate the dvi file. Now, we want an X font size formula, where X is measure in pixels. You need to convert this, to dots per inch (dpi). The formula is: <tt>&lt;dpi&gt; = &lt;font_px&gt;*72.27/10</tt>. If you want, for instance, X = 32, then the size in dpi corresponds to 231.26. This value will be passed to <tt>dvipng</tt> using the flag <tt>-D</tt>. To generate the desired png file run the command as follows:</p> 
      <pre>
dvipng -T tight -D 231.26 -o foo.png foo.dvi
</pre> 
      <p>The flag <tt>-T</tt> sets the size of the image. The option <tt>tight</tt> will only include all ink put on the page. The option <tt>-o</tt> sends the output to the file name <tt>foo.png</tt>.</p> 
      <dl> 
       <dt>
        ImageMagick
       </dt> 
      </dl> 
      <p>The <tt>convert</tt> command from the <a rel="nofollow" class="external text" href="http://www.imagemagick.org/">ImageMagick</a> suite can convert both DVI and PDF files to PNG.</p> 
      <pre>
convert input.pdf output.png
</pre> 
      <dl> 
       <dt>
        optipng
       </dt> 
      </dl> 
      <p>You can optimize the resulting image using <a rel="nofollow" class="external text" href="http://optipng.sourceforge.net/">optipng</a> so that it will take up less space.</p> 
      <h2><span class="mw-headline" id="Convert_to_plain_text">Convert to plain text</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Export_To_Other_Formats&amp;action=edit&amp;section=18" title="Edit section: Convert to plain text">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>If you are thinking of converting to plain text for spell-checking or to count words, there may be an easier way — read <a href="/wiki/LaTeX/Tips_and_Tricks#Spell-checking_and_Word_Counting" title="LaTeX/Tips and Tricks">Tips and Tricks</a> first.</p> 
      <p>Most LaTeX distributions come with <tt>detex</tt> program, which strips LaTeX commands. It can handle multi-file projects, so all you need is to give one command:</p> 
      <pre>
detex yourfile
</pre> 
      <p>(note the omission of .tex extension). This will output result to standard output. If you want the plain text go to a file, use</p> 
      <pre>
detex yourfile &gt; yourfile.txt
</pre> 
      <p>If the output from <tt>detex</tt> does not satisfy you, you can try a newer version available on <a rel="nofollow" class="external text" href="http://code.google.com/p/opendetex/">Google Code</a>, or use HTML conversion first and then copy text from your browser.</p> 
      <p>If you want to keep the formating, you can use a <i>DVI-to-plain text</i> converter, like <tt>catdvi</tt>. Example:</p> 
      <pre>
catdvi yourfile.dvi | fmt -u
</pre> 
      <p>The use of <i>fmt -u</i> (available on most Unices) will remove the justification.</p>
