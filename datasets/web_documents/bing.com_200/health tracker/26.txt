            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>You can only import a function from another class if that function is static; so for example this will work:</p> 
              <pre><code>public class Gets {
    public static void foo() {}
}
</code></pre> 
              <p>and then import it:</p> 
              <pre><code>import static acs.Gets.foo;
</code></pre> 
              <p>If it isn't static however you won't be able to import it.</p> 
              <p><strong>EDIT:</strong> As was pointed out in the comments, using static imports can make code more difficult to read, so they should be used with caution. They can be useful if used correctly though.</p>

             <div class="post-text" itemprop="text"> 
              <p>Java is Object oriented. You need to create an object of the class for you to invoke methods of the class. </p> 
              <p>So something like</p> 
              <p><code>new Gets().myFunction();</code></p> 
              <p>is bound to work. The only exception to this rule is to create a <code>static</code> method. You can find out when to create static methods and if they suit your requirement, but to access a method of a class, you need an object. </p>

             <div class="post-text" itemprop="text"> 
              <p>You would still need to create an object to access the non-static method:</p> 
              <pre><code>(new Gets()).foo();
</code></pre> 
              <p>And you will still need to declare the class to access a static method:</p> 
              <pre><code>Gets.staticFoo();
</code></pre> 
              <p>You <strong>don't</strong> need to import a class from the <em>same package</em>. It is superfluous as all classes in a package are already visible to each other.</p> 
              <p>Also, you are also confusing yourself about how imports work. Importing a class <strong>does not</strong> mean that its method or fields become accessible without qualification.</p> 
              <p>The reason why one imports a class from <em>another package</em>, is so that one does not have to declare the package everywhere in code. Instead of writing:</p> 
              <pre><code>acs.anotherpkg.Blah testblah = new acs.anotherpkg.Blah();
</code></pre> 
              <p>You can simply write:</p> 
              <pre><code>import acs.anotherpkg.Blah;
...
Blah testblah = new Blah();
</code></pre>
