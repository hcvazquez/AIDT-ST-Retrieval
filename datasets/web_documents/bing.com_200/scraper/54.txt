             <div class="post-text" itemprop="text"> 
              <p>When I install e.g. jQuery from Nuget into my VS C# project, jquery.js ends up in my packages folder. This is good, since I don't want to check this file into my source control, since it's not my code and I'm never going to change it - in this context it's the equivalent of a DLL I might get via a different package. So far, so good...</p> 
              <p>Why, then, does the package installer also copy the file into the project Scripts folder, when surely it would be better to add it to the project as a link to the file in the packages folder, much as a Nuget package DLL would be referenced by the project.</p>

                 <div style="display: block;" class="comment-body"> 
                  <span class="comment-copy">The scripts folder is your project. It's where you application loads its scripts from. The packages folder is the "source" for the packages - ie your local cache so it doesn't need to download in future. Your application does <i>not</i> read files from the packages folder (or at least, it/you shouldn't). If you did the same with a dll, it would put the dll in packages <i>and</i> the bin folder. Packages are not part of the project, they're a means to deliver content to your project.</span> –&nbsp;
