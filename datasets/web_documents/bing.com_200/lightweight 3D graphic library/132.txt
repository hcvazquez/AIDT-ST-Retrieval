       <h2>Graphics2D: A better Graphics class</h2>

       <p>Java 2D allows for the specification of arbitrary shapes using the <code>java.awt.Shape</code> interface. I'll leave the specifics of such transforms to the reader (see <a href="#resources" rel="nofollow">Resources</a> for articles that discuss this in greater detail), but suffice it to say that <code>AffineTransform</code>s allow you to operate on any Java 2D graphic to translate (move) it, rotate it, scale it, shear it, or perform combinations of the these manipulations.</p>
       <p>The key to <code>AffineTransform</code> lies in the concept of <em>Device Space</em> and <em>User Space</em>. Device Space is that area into which the graphics will be rendered on the screen. This is analogous to the coordinates that are used when one creates regular AWT-style <code>Graphics</code>-based 2D graphics. User Space, however, is a translatable, rotatable coordinate system that may be operated on by one or more <code>AffineTransform</code>s.</p>
       <p>Device Space and User Space coordinate systems initially overlap, with the origin at the upper left of the rendering surface (here, a Frame). The positive x axis moves right from the origin, while the positive y axis moves down.</p>

       <h2>Of Bezier and higher-ordered curves</h2>
       <p>Now that we have examined how transforms can be used to manipulate graphical objects, let's reexamine how we build complex and interesting arbitrary shapes.</p>
       <p>Curves are used throughout mathematics and computer graphics to approximate complex shapes using a finite, well-defined (and ideally small) number of mathematical points. Whereas the standard AWT did not directly support drawing with arbitrary curves in the past (Java 1.0 or 1.1 platforms), Java 2D adds built-in support for first-, second-, and third-order curves. You can draw curves with two <em>end points</em> and zero, one, or two <em>control points</em>. Java 2D computes first- and second-order curves using linear and quadratic formulas and cubic, or third-order, curves using Bezier curves.</p>
       <p>(Bezier curves are a type of parametric polynomial curve that have some very desirable properties related to computation of closed curves and surfaces. They are used in numerous graphics applications. Please refer to the <a href="#resources" rel="nofollow">Resources</a> for more information on the use of parametric polynomials and Bezier curves in computer graphics.) The <code>GeneralPath</code> methods that draw each of these curves are:</p>
       <ul>
        <li><code>lineTo()</code> for straight segments (specify only end points)</li>
        <li><code>quadTo()</code> for quadratic curves (specify one control point)</li>
        <li><code>curveTo()</code> for third-ordered curves (specify two control points, drawn using cubic Bezier curve)</li>
       </ul>
