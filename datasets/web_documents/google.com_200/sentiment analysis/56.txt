    <h2 id="why-use-server-side-programming">Why use server-side programming?</h2> 
    <p>In <a href="/a2z/server-node/">session four’s notes</a> on node and twitter bots I covered the basics of working with <a href="https://nodejs.org/en/">node</a>, <a href="https://www.npmjs.com/">npm</a>, and building an API with “RESTian” routes.</p> 
    <p>This page picks up on that thread and looks at a few scenarios where running server side code to work with text has advantages over running everything on the client.</p> 
    <p>One of the main reasons you might go down this road is if you have a large dataset that takes a significant amount of time to process. For example, let’s say you want to build a word counting app and you have one million documents to process. This would be unrealistic to do on the client-side, but reasonable on the server.</p> 
    <h3 id="moduleexports">module.exports</h3> 
    <p>To build this example the first thing I’ll do is go and grab <a href="https://github.com/shiffman/A2Z-F16/blob/gh-pages/week5-analysis/01_concordance/concordance.js"><code class="highlighter-rouge">concordance.js</code></a> from the <a href="/a2z/text-analysis">text analysis examples</a>. Functions and objects from separate JS files can be used in node just like in an HTML page. However, this must be done via <a href="https://nodejs.org/api/modules.html">node modules</a> and the <code class="highlighter-rouge">require()</code> function.</p> 
    <p>For example, if you have the following constructor function in a file called <code class="highlighter-rouge">concordance.js</code>:</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">function</span> <span class="nx">Concordance</span><span class="p">()</span> <span class="p">{</span>
  <span class="k">this</span><span class="p">.</span><span class="nx">dict</span> <span class="o">=</span> <span class="p">{};</span>
  <span class="k">this</span><span class="p">.</span><span class="nx">keys</span> <span class="o">=</span> <span class="p">[];</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>You can have access to this constructor function in your main app (<code class="highlighter-rouge">server.js</code>) with two additional steps. First, you must add <code class="highlighter-rouge">Concordance</code> to <code class="highlighter-rouge">module.exports</code> in <code class="highlighter-rouge">concordance.js</code>. <code class="highlighter-rouge">module.exports</code> is <a href="http://stackoverflow.com/questions/5311334/what-is-the-purpose-of-node-js-module-exports-and-how-do-you-use-it">the object that’s actually returned as when you call require</a>.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span> <span class="nx">Concordance</span><span class="p">;</span></code></pre>
    </figure> 
    <p>Once you’ve done this, you can now get access to <code class="highlighter-rouge">Concordance</code> with a call to <code class="highlighter-rouge">require()</code> like so:</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// Require it</span>
<span class="kd">var</span> <span class="nx">Concordance</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'./concordance'</span><span class="p">);</span>
<span class="c1">// Now make an object with it</span>
<span class="kd">var</span> <span class="nx">wordcounts</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Concordance</span><span class="p">();</span></code></pre>
    </figure> 
    <h3 id="reading-files">Reading files</h3> 
    <iframe width="560" height="315" src="https://www.youtube.com/embed/6iZiqQZBQJY?list=PLRqwX-V7Uu6Yyn-fBtGHfN0_xCtBwUkBp" frameborder="0" allowfullscreen></iframe> 
    <p>Now that I have a <code class="highlighter-rouge">Concordance</code> object I can start filling it with data on the server. Let’s say I had a sequence of numbered files sitting on the server that I want to process. I can read those files and pass the contents to the concordance with node’s <a href="https://nodejs.org/api/fs.html">file system module</a> (aka <code class="highlighter-rouge">fs</code>). The <code class="highlighter-rouge">fs</code> module has functions for grabbing a list of files in a directory as well as reading specific files.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">var</span> <span class="nx">fs</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'fs'</span><span class="p">);</span>

<span class="c1">// What are all the files in the directory "data"?</span>
<span class="kd">var</span> <span class="nx">files</span> <span class="o">=</span> <span class="nx">fs</span><span class="p">.</span><span class="nx">readdirSync</span><span class="p">(</span><span class="s1">'data'</span><span class="p">);</span>
<span class="c1">// Read all the files one by one</span>
<span class="k">for</span> <span class="p">(</span><span class="kd">var</span> <span class="nx">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="nx">i</span>  <span class="o">&lt;</span> <span class="nx">files</span><span class="p">.</span><span class="nx">length</span><span class="p">;</span> <span class="nx">i</span><span class="o">++</span><span class="p">)</span> <span class="p">{</span>
  <span class="kd">var</span> <span class="nx">txt</span> <span class="o">=</span> <span class="nx">fs</span><span class="p">.</span><span class="nx">readFileSync</span><span class="p">(</span><span class="s1">'data/'</span><span class="o">+</span><span class="nx">files</span><span class="p">[</span><span class="nx">i</span><span class="p">],</span> <span class="s1">'utf8'</span><span class="p">);</span>
  <span class="nx">wordcounts</span><span class="p">.</span><span class="nx">process</span><span class="p">(</span><span class="nx">txt</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>One thing you might notice about the above is the use of <code class="highlighter-rouge">readdirSync()</code> and <code class="highlighter-rouge">readFileSync()</code> as opposed to <code class="highlighter-rouge">readdir()</code> and <code class="highlighter-rouge">readFile()</code>. The “sync” refers to “synchronous” meaning these lines of code are “blocking”. The data has to be read before moving onto the next line. This is unusual in JavaScript in that typically a callback is required to be executed when the data is read. This is a case where I am happy for the program to stop and wait because I want to process all of the data before the server starts listening for connections. It’s ok if it takes a long time because this only happens once when the server starts. (This, however, would not be advisable at other points in the code like handling a client request to the server.)</p> 
    <h3 id="routes-for-results">Routes for results</h3> 
    <p>Now that the data is read, I can create routes that send the data to a client making a <code class="highlighter-rouge">loadJSON()</code> request with p5 (or pick your function using any JS library that can make HTTP requests.) Here’s one that sends everything in the concordance object.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// Route for sending all the concordance data</span>
<span class="nx">app</span><span class="p">.</span><span class="nx">get</span><span class="p">(</span><span class="s1">'/all'</span><span class="p">,</span> <span class="nx">showAll</span><span class="p">);</span>

<span class="kd">function</span> <span class="nx">showAll</span><span class="p">(</span><span class="nx">req</span><span class="p">,</span> <span class="nx">res</span><span class="p">)</span> <span class="p">{</span>
  <span class="c1">// Send the entire concordance</span>
  <span class="nx">res</span><span class="p">.</span><span class="nx">send</span><span class="p">(</span><span class="nx">wordcounts</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>I can also get fancier and make up my own protocol for sending back pieces of data from the concordance. For example, here is some code that returns the count for a specific word or “word not found” if it is not present in the concordance. The point is to send back a JavaScript object — it’s up to you to put in the object what you think makes the most sense.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// Now a route for data about one word</span>
<span class="nx">app</span><span class="p">.</span><span class="nx">get</span><span class="p">(</span><span class="s1">'/search/:word'</span><span class="p">,</span> <span class="nx">showOne</span><span class="p">);</span>

<span class="kd">function</span> <span class="nx">showOne</span><span class="p">(</span><span class="nx">req</span><span class="p">,</span> <span class="nx">res</span><span class="p">)</span> <span class="p">{</span>
  <span class="kd">var</span> <span class="nx">word</span> <span class="o">=</span> <span class="nx">req</span><span class="p">.</span><span class="nx">params</span><span class="p">[</span><span class="s1">'word'</span><span class="p">];</span>

  <span class="kd">var</span> <span class="nx">reply</span> <span class="o">=</span> <span class="p">{</span> <span class="p">};</span>
  <span class="kd">var</span> <span class="nx">count</span> <span class="o">=</span> <span class="nx">wordcounts</span><span class="p">.</span><span class="nx">getCount</span><span class="p">(</span><span class="nx">word</span><span class="p">);</span>

  <span class="c1">// If it's not part of concordance send back a message</span>
  <span class="k">if</span> <span class="p">(</span><span class="nx">count</span> <span class="o">===</span> <span class="kc">undefined</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">reply</span><span class="p">.</span><span class="nx">status</span> <span class="o">=</span> <span class="s1">'word not found'</span><span class="p">;</span>
  <span class="c1">// Otherwise send back the word and count</span>
  <span class="p">}</span> <span class="k">else</span> <span class="p">{</span>
    <span class="nx">reply</span><span class="p">.</span><span class="nx">status</span> <span class="o">=</span> <span class="s1">'success'</span><span class="p">;</span>
    <span class="nx">reply</span><span class="p">.</span><span class="nx">count</span> <span class="o">=</span> <span class="nx">count</span><span class="p">;</span>
  <span class="p">}</span>
  <span class="c1">// It's useful to send back the word to</span>
  <span class="c1">// so the client can match the count with a given API call</span>
  <span class="nx">reply</span><span class="p">.</span><span class="nx">word</span> <span class="o">=</span> <span class="nx">word</span><span class="p">;</span>

  <span class="nx">res</span><span class="p">.</span><span class="nx">send</span><span class="p">(</span><span class="nx">reply</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <h3 id="what-does-the-client-say">What does the client say?</h3>

    <p>The client can then access this data with <code class="highlighter-rouge">loadJSON()</code>.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript">  <span class="c1">// Getting all the data</span>
  <span class="nx">loadJSON</span><span class="p">(</span><span class="s1">'/all'</span><span class="p">,</span> <span class="nx">gotAll</span><span class="p">);</span>

  <span class="c1">// Getting data for one word -- 'apple'</span>
  <span class="nx">loadJSON</span><span class="p">(</span><span class="s1">'/search/apple'</span><span class="p">,</span> <span class="nx">gotOne</span><span class="p">);</span>

  <span class="kd">function</span> <span class="nx">gotAll</span><span class="p">(</span><span class="nx">data</span><span class="p">)</span> <span class="p">{</span>
    <span class="c1">// deal with all the data</span>
  <span class="p">}</span>

  <span class="kd">function</span> <span class="nx">gotOne</span><span class="p">(</span><span class="nx">data</span><span class="p">)</span> <span class="p">{</span>
    <span class="c1">// deal with the results of the search for a single word</span>
  <span class="p">}</span></code></pre>
    </figure> 
    <h2 id="cors">CORS</h2> 
    <p>One thing you might notice about the above <code class="highlighter-rouge">loadJSON()</code> calls is that they do not reference the domain itself, simply the route “all” or “search”. This is because I am assuming that the p5 sketch will be hosted by the same node app that is running the API code. In fact, in my example I’m doing exactly this by placing the p5 sketch in a “public” folder and serving those files statically using node and express:</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="nx">app</span><span class="p">.</span><span class="nx">use</span><span class="p">(</span><span class="nx">express</span><span class="p">.</span><span class="kr">static</span><span class="p">(</span><span class="s1">'public'</span><span class="p">));</span></code></pre>
    </figure> 
    <p>However, let’s say you want others to be able to access your API from their code. In order for this to be possible you must enable something called <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Access_control_CORS">CORS (Cross-origin resource sharing)</a>. This prevents others from getting that nasty error: <em>XMLHttpRequest cannot load. No ‘Access-Control-Allow-Origin’ header is present on the requested resource.</em> (<a href="https://jvaneyck.wordpress.com/2014/01/07/cross-domain-requests-in-javascript/">More about Cross Domain Requests in JS</a>.)</p> 
    <p>This is easy enough to do with the <a href="https://github.com/expressjs/cors">Node CORS package</a>.</p> 
    <figure class="highlight">
     <pre><code class="language-text" data-lang="text">$ npm install cors --save</code></pre>
    </figure> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">var</span> <span class="nx">cors</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'cors'</span><span class="p">);</span>
<span class="nx">app</span><span class="p">.</span><span class="nx">use</span><span class="p">(</span><span class="nx">cors</span><span class="p">());</span></code></pre>
    </figure> 
    <h2 id="persistance">Persistance</h2> 
    <p>Another topic relevant to server-side programming is “persistance”. In other words, let’s say you want to build a text classifier. <a href="https://github.com/NaturalNode/natural">NaturalNode</a> includes as one of its features <a href="https://web.stanford.edu/class/cs124/lec/naivebayes.pdf">Bayesian Text Classification</a> which I briefly covered in <a href="/a2z/text-analysis">text analysis</a>.</p> 
    <p>Let’s assume your application classifies text as “happy” or “sad”. The system is “trained” by users submitting text tagged with the appropriate category (happy or sad). The server passes all this text to a <code class="highlighter-rouge">Classifier</code> object which stores all the relevant counts and probabilities for the submitted text. After running your application for a week, you have hundreds of submissions. What would happen if you have to close and restart the server?</p> 
    <p>If all of the data is just stored in memory in the <code class="highlighter-rouge">Classifier</code> object, it will all be lost as soon as the server quits. A solution is to save the data somewhere permanent that persists even when the server stops running. One option is to use a database (like <a href="https://docs.mongodb.org/ecosystem/drivers/node-js/">mongodb</a> or the simpler <a href="https://github.com/louischatriot/nedb">nedb</a>) but for some basic scenarios these solutions can be overkill.</p> 
    <p>One option is to just write out a text file filled with JSON. The <code class="highlighter-rouge">fs</code> module can handle the reading and writing and the <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/JSON">JSON</a> object has functions <code class="highlighter-rouge">parse()</code> and <code class="highlighter-rouge">stringify()</code> to convert back and forth from JS object to raw text.</p> 
    <p>Here is the skeleton of code used in the examples here. Step 1 for the server is to check and see if the file exists. (The code below is from a sentiment analysis scenario where words and their positive/negative score are stored in a file called <code class="highlighter-rouge">additional.json</code>.)</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">var</span> <span class="nx">exists</span> <span class="o">=</span> <span class="nx">fs</span><span class="p">.</span><span class="nx">existsSync</span><span class="p">(</span><span class="s1">'additional.json'</span><span class="p">);</span></code></pre>
    </figure> 
    <p>If it does in fact exist, the data can be read using <code class="highlighter-rouge">fs</code> and stored in a variable using <code class="highlighter-rouge">JSON.parse()</code>.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">var</span> <span class="nx">additional</span><span class="p">;</span>
<span class="k">if</span> <span class="p">(</span><span class="nx">exists</span><span class="p">)</span> <span class="p">{</span>
  <span class="kd">var</span> <span class="nx">txt</span> <span class="o">=</span> <span class="nx">fs</span><span class="p">.</span><span class="nx">readFileSync</span><span class="p">(</span><span class="s1">'additional.json'</span><span class="p">,</span> <span class="s1">'utf8'</span><span class="p">);</span>
  <span class="nx">additional</span> <span class="o">=</span> <span class="nx">JSON</span><span class="p">.</span><span class="nx">parse</span><span class="p">(</span><span class="nx">txt</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>In the case of it not existing, the code simply makes an empty JavaScript object.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="p">}</span> <span class="k">else</span> <span class="p">{</span>
  <span class="nx">additional</span> <span class="o">=</span> <span class="p">{};</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>The example includes an API call to add a word and score to the sentiment analysis. The server can then just write out a new JSON file using <code class="highlighter-rouge">JSON.stringify()</code> on the variable <code class="highlighter-rouge">additional</code>. Here, the writing of a file must be done asynchronously since the action is associated with a web request (rather than just when the server starts up) and you don’t want the server to get stuck on an operation while other clients could be trying to connect.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// Assuming some new word and score is added</span>
<span class="nx">additional</span><span class="p">[</span><span class="s1">'new word'</span><span class="p">]</span> <span class="o">=</span> <span class="mi">5</span><span class="p">;</span>

<span class="c1">// Turn additional back into text</span>
<span class="kd">var</span> <span class="nx">json</span> <span class="o">=</span> <span class="nx">JSON</span><span class="p">.</span><span class="nx">stringify</span><span class="p">(</span><span class="nx">additional</span><span class="p">);</span>

<span class="c1">// Write out the file</span>
<span class="nx">fs</span><span class="p">.</span><span class="nx">writeFile</span><span class="p">(</span><span class="s1">'additional.json'</span><span class="p">,</span> <span class="nx">json</span><span class="p">,</span> <span class="s1">'utf8'</span><span class="p">,</span> <span class="nx">finished</span><span class="p">);</span>

<span class="c1">// Callback for when file is finished</span>
<span class="kd">function</span> <span class="nx">finished</span><span class="p">(</span><span class="nx">err</span><span class="p">)</span> <span class="p">{</span>
  <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="s1">'Finished writing additional.json'</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>Writing out a text file is a nice quick and dirty solution that can work for many small-scale creative projects in a prototyping stage. Often, however, a more involved database is required. In addition to creating a database on your own server, there is also the option of using an API service like <a href="https://firebase.google.com">Firebase</a> to store your data. I have <a href="/a2z/firebase/">notes and examples on firebase here</a>.</p> 
    <h2 id="get-vs-post">GET vs POST</h2> 
    <iframe width="560" height="315" src="https://www.youtube.com/embed/GZ2nwxhQUTU?list=PLRqwX-V7Uu6Yyn-fBtGHfN0_xCtBwUkBp" frameborder="0" allowfullscreen></iframe> 
    <p>The <a href="https://en.wikipedia.org/wiki/Hypertext_Transfer_Protocol">HTTP (Hypertext Transfer Protocol)</a> includes several different kinds of requests. The one you are likely most commonly familiar with is a <code class="highlighter-rouge">GET</code> request. This is the request that happens when you type a URL into an address bar. You are asking the server if you can “get” something, and what is sent back is some sort of data, often in the form of HTML, but in can be anything.</p> 
    <p>In fact, you are also making GET requests in code all the time. If you pass a url into the <code class="highlighter-rouge">loadJSON()</code> function, a GET request is made.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="nx">loadJSON</span><span class="p">(</span><span class="s1">'http://api.com/some/api/call'</span><span class="p">,</span> <span class="nx">gotData</span><span class="p">);</span>

<span class="kd">function</span> <span class="nx">gotData</span><span class="p">(</span><span class="nx">data</span><span class="p">)</span> <span class="p">{</span>
  <span class="c1">// the response is in data</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>In this week’s examples, I am handling GET requests in node as well, by using the <code class="highlighter-rouge">get()</code> function on the <code class="highlighter-rouge">app</code> object, i.e.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="nx">loadJSON</span><span class="p">(</span><span class="s1">'http://api.com/some/api/call'</span><span class="p">,</span> <span class="nx">gotData</span><span class="p">);</span>

<span class="kd">function</span> <span class="nx">gotData</span><span class="p">(</span><span class="nx">data</span><span class="p">)</span> <span class="p">{</span>
  <span class="c1">// the response is in data</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>There are some scenarios, however, where a POST request is preferable to a GET. POST requests are designed for instances where the data sent would be stored on the server (or affect some sort of change of state by the server like deleting a database record.) They are also useful in the context of sending sensitive data like passwords since the data of a POST is not visible via the URL address.</p> 
    <p>In the case of creative ITP projects we can be a little loosey goosey about these distinctions. I’m using a post here because my examples might send a large paragraph (or even more) of text (GET requests have a data length limit).</p> 
    <p>To send a POST from p5 the <code class="highlighter-rouge">httpPost()</code> method is available. Simply pass a JavaScript object with the data for the post to the appropriate url. You can then also define success and error callbacks to track the request.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// Data to be posted</span>
<span class="kd">var</span> <span class="nx">params</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s2">"daniel"</span><span class="p">,</span>
  <span class="na">password</span><span class="p">:</span> <span class="s2">"rainb0w"</span>
<span class="p">}</span>

<span class="c1">// Making the post</span>
<span class="nx">httpPost</span><span class="p">(</span><span class="s1">'/path/to/post'</span><span class="p">,</span> <span class="nx">params</span><span class="p">,</span> <span class="nx">finished</span><span class="p">);</span>

<span class="c1">// Handling the result</span>
<span class="kd">function</span> <span class="nx">finished</span><span class="p">(</span><span class="nx">reply</span><span class="p">)</span> <span class="p">{</span>
  <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">reply</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>On the server side, receiving the data looks almost identical to a <code class="highlighter-rouge">GET</code>.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// "post" instead of "get"</span>
<span class="nx">app</span><span class="p">.</span><span class="nx">post</span><span class="p">(</span><span class="s1">'/path/to/post'</span><span class="p">,</span> <span class="nx">gotData</span><span class="p">);</span>

<span class="kd">function</span> <span class="nx">gotData</span><span class="p">(</span><span class="nx">req</span><span class="p">,</span> <span class="nx">res</span><span class="p">)</span> <span class="p">{</span>
  <span class="c1">// Look at the "body" of a POST to get</span>
  <span class="c1">// data, not the "params" like with a GET</span>
  <span class="kd">var</span> <span class="nx">name</span> <span class="o">=</span> <span class="nx">req</span><span class="p">.</span><span class="nx">body</span><span class="p">.</span><span class="nx">name</span><span class="p">;</span>
  <span class="kd">var</span> <span class="nx">password</span> <span class="o">=</span> <span class="nx">req</span><span class="p">.</span><span class="nx">body</span><span class="p">.</span><span class="nx">password</span><span class="p">;</span>
  <span class="kd">var</span> <span class="nx">reply</span> <span class="o">=</span> <span class="p">{</span>
    <span class="na">message</span><span class="p">:</span> <span class="s2">"thank you"</span>
  <span class="p">}</span>
  <span class="nx">res</span><span class="p">.</span><span class="nx">send</span><span class="p">(</span><span class="nx">reply</span><span class="p">);</span>
<span class="p">}</span></code></pre>
    </figure> 
    <p>In the above code, you’ll notice that the data is pulled from <code class="highlighter-rouge">req.body</code> rather than <code class="highlighter-rouge">req.params</code> as with a <code class="highlighter-rouge">GET</code> request. This only works because earlier in my code I am including the express <a href="https://github.com/expressjs/body-parser">body-parser</a> package.</p> 
    <figure class="highlight">
     <pre><code class="language-javascript" data-lang="javascript"><span class="kd">var</span> <span class="nx">bodyParser</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'body-parser'</span><span class="p">);</span>
<span class="nx">app</span><span class="p">.</span><span class="nx">use</span><span class="p">(</span><span class="nx">bodyParser</span><span class="p">.</span><span class="nx">json</span><span class="p">());</span> <span class="c1">// support json encoded bodies</span></code></pre>
    </figure> 
    <p>This package handles the parsing of a <code class="highlighter-rouge">POST</code> body and gives you easy access in <code class="highlighter-rouge">request.body</code>.</p>
