         <p> Text messaging has become extremely widespread throughout the world -- to the point where an increasing number of web applications have integrated SMS to notify users of events, sales or coupons directly through their mobile devices.</p> 
         <p> In this tutorial, we will cover the fundamentals of sending text messages with PHP.</p>
         <h2>Premium Option</h2>
         <p>Before we get into the step-by-step process, you might want to look at a ready-made solution: <a href="https://codecanyon.net/item/sms-sender/10369066" target="_self">SMS Sender</a>, available on Envato Market. This PHP script lets you:</p>
         <ul> 
          <li>create and import contacts and groups</li> 
          <li>send a single SMS, or send bulk SMS to a group or multiple groups</li> 
          <li>connect to any SMS gateway with minimal configuration<br> </li> 
          <li>customize your SMS or email</li> 
          <li>add a link to your SMS and email</li> 
          <li>and much more</li> 
         </ul>
         <figure class="post_image">
          <a href="https://codecanyon.net/item/sms-sender/10369066" target="_self"><img alt="SMS Sender on Envato Market" src="https://cms-assets.tutsplus.com/uploads/users/48/posts/10978/image/SMS-sender.png"></a>
          <figcaption>
           <a href="https://codecanyon.net/item/sms-sender/10369066" target="_self">SMS Sender</a> on Envato Market
          </figcaption>
         </figure>
         <p>If you'd prefer to build your text-message sender from scratch, here's how to do it.</p> 
         <hr> 
         <h2> Overview </h2> 
         <blockquote>
          <p>Sending a text message (SMS) message is actually pretty easy.</p>
         </blockquote> 
         <p> Below is a simplified diagram of how a message can be sent from a web application to a wireless device. </p> 
         <div>
          <img alt="" src="https://cdn.tutsplus.com/net/uploads/legacy/902_smswithPHP/3-sms-with-php-http-to-phone.png">
         </div> 
         <p> We'll break this down -- one piece at a time:</p> 
         <ul> 
          <li> The message is composed using a web application that is stored and executed on a HTTP server and then sent through the internet ("the cloud") as an <em>email</em> message. </li> 
          <li> The email is received by a Short Message Service Gateway (SMS Gateway), which converts the message from an email message to a SMS message. </li> 
          <li> The SMS message is then handed to a Short Message Service Center (SMSC), which is a server that routes data to specific mobile devices. </li> 
          <li> The message is finally transmitted over the wireless network to the recipient. </li> 
         </ul> 
         <p> Most wireless networks have a SMS gateway through which email messages can be sent as text messages to a mobile device. This is nice, because, from a developer's standpoint, it is generally freehowever, it is of course not a free service for the end user. Fees still apply to the recipient of the message and messages sent via email will be billed as a non-network text message. </p> 
         <hr> 
         <h2> Email to SMS </h2> 
         <p> To send a SMS via email, you'll generally require only two things: </p> 
         <ul> 
          <li>The phone number or unique identifier of the mobile device you want to reach.</li> 
          <li>And the wireless network's domain name (many can be found in <a href="http://www.venture-ware.com/kevin/?p=83">this </a>list of email to SMS addresses)</li> 
         </ul> 
         <p> The following convention can be followed for most carriers: </p> 
         <pre class="brush: html noskimlinks noskimwords">phoneNumber@domainName.com</pre> 
         <p><code>phoneNumber</code> is the phone number of the mobile device to send the message to, and <code>domainName.com</code> is the address for the network's SMS Gateway. </p> 
         <p>To send a text to Mr. Example, you could simply add 3855550168@vtext.com to any email client, type a message and hit send. This will send a text message to phone number +1 (385) 555-0168 on the Verizon Wireless Network. </p> 
         <p> For example, I'll send a text message to myself using Gmail. </p> 
         <div>
          <img alt="" src="https://cdn.tutsplus.com/net/uploads/legacy/902_smswithPHP/sms-with-php-gmail.png">
         </div> 
         <p> When my phone receives the message, it should look like so: </p> 
         <div> 
          <img alt="" src="https://cdn.tutsplus.com/net/uploads/legacy/902_smswithPHP/sms-with-php-sms-on-cell-phone.png"> 
          <p> Pretty awesome! </p> 
         </div> 
         <hr> 
         <h2> PHP's <code>mail</code> Function </h2> 
         <p> Let's take things a step further. Using the SMS Gateway, we can send a text message via email using PHP's <code>mail</code> function. The <code>mail</code> function has the following signature: </p> 
         <pre class="brush: php noskimlinks noskimwords">bool mail ( string $to , string $subject , string $message [, string $additional_headers [, string $additional_parameters ]] )</pre> 
         <p>You can read more about it <a href="http://us2.php.net/manual/en/function.mail.php">here</a>.</p> 
         <ul> 
          <li> <code>$to</code> defines the receiver or receivers of the message. Valid examples include:<p></p> 
           <ul> 
            <li>user@example.com</li> 
            <li>user1@example.com, user2@example.com</li> 
            <li>User &lt;user@example.com&gt;</li> 
            <li>User1 &lt;user1@example.com&gt;, User2 &lt;user2@example.com&gt;</li> 
           </ul> </li> 
          <li> <code>$subject</code> is rather self explanatory; it should be a string containing the desired subject. However, SMS <em>do not require</em> a subject. </li> 
          <li> <code>$message</code> is the message to be delivered. As mentioned in the <a href="http://us2.php.net/manual/en/function.mail.php">PHP manual</a>, "each line should be separated with a LF (\n). Lines should not be larger than 70 characters." </li> 
         </ul> 
         <p> To replicate the earlier functionality, we could write the following PHP code: </p> 
         <pre class="brush: php noskimlinks noskimwords">mail( '3855550168@vtext.com', '', 'Testing' );</pre> 
         <hr> 
         <h2> A Test Drive </h2> 
         <p> Let's run a test with PHP to make that sure everything is setup correctly and that the <code>mail</code> function will, in fact, send a text message. Using the following code, we can run: </p> 
         <pre class="brush: php noskimlinks noskimwords">&lt;?php

var_dump( mail( '##########@vtext.com', '', 'This was sent with PHP.' ) ); // bool(true)

?&gt;</pre>

          <p> If you are getting an error, see the <a href="#troubleshooting">troubleshooting</a> section. </p>

         <p> As you can see in the image above, the message shows that it is from Gmail. This is because I route all my outgoing messages from my local server through that service. Unfortunately, as of this writing, I have been unsuccessful at altering the <code>From</code> header to reflect an alternate address. It seems that the email headers are stripped and replaced with headers prepared by the SMS gateway. If anyone knows of a workaround, please leave a comment and let the rest of us know! </p>

         <h2> The Script </h2> 
         <p> The most important part to this is the PHP script. We'll write that bit of code now: </p> 
         <pre class="brush: php noskimlinks noskimwords">&lt;?php

if ( isset( $_REQUEST ) &amp;&amp; !empty( $_REQUEST ) ) {
 if (
 isset( $_REQUEST['phoneNumber'], $_REQUEST['carrier'], $_REQUEST['smsMessage'] ) &amp;&amp;
  !empty( $_REQUEST['phoneNumber'] ) &amp;&amp;
  !empty( $_REQUEST['carrier'] )
 ) {
  $message = wordwrap( $_REQUEST['smsMessage'], 70 );
  $to = $_REQUEST['phoneNumber'] . '@' . $_REQUEST['carrier'];
  $result = @mail( $to, '', $message );
  print 'Message was sent to ' . $to;
 } else {
  print 'Not all information was submitted.';
 }
}

?&gt;
&lt;!DOCTYPE html&gt;</pre> 
         <ul> 
          <li>The script first checks to see if the form has been submitted. </li> 
          <li>If yes, it checks to see if the <code>phoneNumber</code>, <code>carrier</code> and <code>smsMessage</code> variables were sent. This is useful in the case where there may be more than one form on the page. </li> 
          <li>If <code>phoneNumber</code>, <code>carrier</code> and <code>smsMessage</code> are available and <code>phoneNumber</code> and <code>carrier</code> are not empty, it is okay to attempt to send the message. </li> 
          <li>The message argument in the <code>mail</code> function should be 70 characters in length per line. We can chop the message into 70 character chunks using the <code>wordwrap</code> function.</li> 
          <li> <code>phoneNumber</code> and <code>carrier</code> are concatenated and then the message is sent using the <code>mail</code> function. </li> 
          <li>If data is missing or it cannot be validated, the script simply returns <em>Not all information was submitted</em>. </li> 
          <li>Finally, <code>mail</code> returns a boolean indicating whether it was successful or not. The value is stored in <code>$result</code> in case I needed to verify that the message was in fact sent. </li> 
         </ul> 
         <p><strong>Note:</strong> The <code>mail</code> method only notifies whether the message was sent or not. It does not provide a way to check to see if the message was successfully received by the recipient server or mailbox.</p>

         <h2> Troubleshooting </h2> 
         <h3> Localhost Error </h3> 
         <p> In order to use the <code>mail</code> function, you must have a mail server running. If you're running this on a web host, you're probably okay. But if you are unsure, I recommend talking to an administrator. This also holds true for personal machines. So if you get errors like.. </p> 
         <pre class="brush: html noskimlinks noskimwords">Warning: mail() [function.mail]: Failed to connect to mailserver at "localhost" port 25, verify your "SMTP" and "smtp_port" setting in php.ini or use ini_set() in C:\wamp\www\sms\mail-test.php</pre> 
         <p> ...you will have to install and configure a mail server. This is out of the scope of this tutorial. However, if you are working on your local machine, switching to something like <a href="http://www.apachefriends.org/en/xampp.html">XAMPP</a> might solve this problem. Alternatively, try installing <a href="http://www.pmail.com/">Mercury Mail</a> alongside <a href="http://www.wampserver.com/en/">WAMP</a>, <a href="http://www.mamp.info/en/index.html">MAMP</a> or on a LAMP (or SAMP or OAMP, etc.) system (that's a lot of 'AMPs'). </p> 
         <h3> PHPMailer </h3> 
         <p> Another option (which is the method I prefer) is to use <a href="http://phpmailer.worxware.com/">PHPMailer</a>. Below is an example of how to use PHPMailer to connect to Gmail's SMTP server and send the message. </p> 
         <p> Using it is as simple as including a class in your script. </p> 
         <pre class="brush: php noskimlinks noskimwords">require 'class.phpmailer.php';

// Instantiate Class
$mail = new PHPMailer();

// Set up SMTP
$mail-&gt;IsSMTP();                // Sets up a SMTP connection
$mail-&gt;SMTPDebug  = 2;          // This will print debugging info
$mail-&gt;SMTPAuth = true;         // Connection with the SMTP does require authorization
$mail-&gt;SMTPSecure = "tls";      // Connect using a TLS connection
$mail-&gt;Host = "smtp.gmail.com";
$mail-&gt;Port = 587;
$mail-&gt;Encoding = '7bit';       // SMS uses 7-bit encoding

// Authentication
$mail-&gt;Username   = "email.address@gmail.com"; // Login
$mail-&gt;Password   = "password"; // Password

// Compose
$mail-&gt;Subject = "Testing";     // Subject (which isn't required)
$mail-&gt;Body = "Testing";        // Body of our message

// Send To
$mail-&gt;AddAddress( "##########@vtext.com" ); // Where to send it
var_dump( $mail-&gt;send() );      // Send!</pre> 
         <p> This should print out something along the lines of: </p> 
         <div>
          <img alt="" src="https://cdn.tutsplus.com/net/uploads/legacy/902_smswithPHP/5-sms-with-php-phpmailer-result.png">
         </div> 
         <p> It may take a little more to set up the connection depending on your situation. If you're planning on using Gmail, Google has provided <a href="http://mail.google.com/support/bin/answer.py?hl=en&amp;answer=13287">information on connecting</a>. </p> 
         <hr> 
         <h2> Conclusion </h2> 
         <p> There are a myriad of methods to accomplish the task of sending a SMS through a web application. This method is really meant for low volume messaging (most likely less than 1,000 text messages per month) and developers looking to get their feet wet without forking out cash. Other options include: </p> 
         <ul> 
          <li> <strong>Using a SMS Gateway Provider</strong><p></p> 
           <ul> 
            <li> Doing a <a href="http://www.google.com/search?q=sms+gateway+provider">Google search</a> will return plenty of options.</li> 
            <li> Most SMS gateway providers include an API for sending messages through a web application. </li> 
            <li> You can usually sign up for service for a reasonable price, assuming you are planning on sending at least 1,000 SMS message per month. </li> 
            <li> You can rent a <a href="http://en.wikipedia.org/wiki/Short_code">short code number</a>. </li> 
           </ul> </li> 
          <li> <strong>Using a GSM modem</strong><p></p> 
           <ul> 
            <li>This can be a costly and slow way to do it, since you have to buy a modem and have a contract with a wireless network </li> 
            <li> You will also have to use the <a href="http://en.wikipedia.org/wiki/AT_Commands">AT (Hayes) command set</a>. </li> 
           </ul> </li> 
          <li> <strong>Use a direct connection to a wireless network, which will require some strong negotiating and a whole lot of money.</strong> </li> 
         </ul> 
         <p>This tutorial is in no way a comprehensive review of sending tex messages with PHP; but it should get you started! I hope this tutorial has been of interest to you. Thank you so much for reading!</p>
         <p>If you still need help with this or any other PHP issue, try contacting one of the experienced <a href="https://studio.envato.com/search?utf8=%E2%9C%93&amp;search%5Bquery%5D=php" target="_self">PHP developers</a> on Envato Studio.</p>
