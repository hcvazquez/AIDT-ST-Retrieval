        <a href=""><h1>Passport.JS Authentication for your Node.JS Sleek Application</h1></a> 
        <span class="aut"> <a href="https://www.cubettech.com/blog/author/admin/" class="aut-pic"> <img src="http://cubettech.com/blog/wp-content/uploads/2015/12/21ef211.jpg"></a> <a href="https://www.cubettech.com/blog/author/admin/">Lovegin</a> . <span class="date">August 12th, 2014</span> </span>

        <p dir="ltr"><a href="http://blog.cubettech.com/wp-content/uploads/2014/08/passport.jpg"><img class="alignnone size-full wp-image-411" alt="Passport.js Authentication of Node.js" src="http://blog.cubettech.com/wp-content/uploads/2014/08/passport.jpg" width="770" height="300" srcset="https://www.cubettech.com/blog/wp-content/uploads/2014/08/passport.jpg 770w, https://www.cubettech.com/blog/wp-content/uploads/2014/08/passport-300x116.jpg 300w" sizes="(max-width: 770px) 100vw, 770px"></a></p> 
        <p dir="ltr"> </p>
        <p dir="ltr"> </p>
        <p>Node JS is one of the software platforms, formulated to support scalable server side and networking applications. Node JS applications, by utilizing non-blocking I/O as well as asynchronous events increases the throughput and efficiency to the core. There are several boons to Node JS applications. For instance, Node JS application in conjunction with JSON, a browser, and a document DB like Mongo DB caters a unified JavaScript development stack. Node JS based authentication and login can be a multifaceted task as it is with any other applications. Authenticating Node JS using Passport has been proved to be an effective strategy in the recent past. In this post, we will see how such an authentication process can be carried out with package Passport.</p> 
        <p>Brief introduction to Passport</p> 
        <p>Passport is a recognized middleware for Node JS authentication. The package Passport is uniquely formulated for the purpose of authenticating requests. It supports encapsulation during code creation by delegating all the other functionality to the application. Such an aspect plays a crucial role in keeping the code clean and maintainable.</p> 
        <p>Passport has libraries for common social networks like facebook, google , twitter, linkedIn etc. In order to setup a social login with passport, you need to obtain the developer keys from respective sites. It is also capable of recognizing inimitable authentication requirements of each application. Solutions to such requests are catered with the help of strategies that are usually packaged as individual modules. Some of the commonly used Passport authentication strategies are listed below.</p> 
        <p>– passport-local</p> 
        <p>– passport-facebook</p> 
        <p>– passport-google-oauth</p> 
        <p>– passport-twitter</p> 
        <p>– passport-linkedin</p> 
        <p>Setting-up the api keys, and routes for facebook and twitter are demonstrated below.</p> 
        <p>Stepwise creation of developer keys in Facebook.</p> 
        <p><a href="http://blog.cubettech.com/wp-content/uploads/2014/08/Picture1.jpg"><img class="alignnone size-full wp-image-419" alt="Facebook Key Development" src="http://blog.cubettech.com/wp-content/uploads/2014/08/Picture1.jpg" width="758" height="378" srcset="https://www.cubettech.com/blog/wp-content/uploads/2014/08/Picture1.jpg 758w, https://www.cubettech.com/blog/wp-content/uploads/2014/08/Picture1-300x149.jpg 300w" sizes="(max-width: 758px) 100vw, 758px"></a></p> 
        <p>Stepwise creation of developer keys in Twitter.<a href="http://blog.cubettech.com/wp-content/uploads/2014/08/Picture2.png"><img class="alignnone size-full wp-image-420" alt="Twitter Key Development" src="http://blog.cubettech.com/wp-content/uploads/2014/08/Picture2.png" width="942" height="505" srcset="https://www.cubettech.com/blog/wp-content/uploads/2014/08/Picture2.png 942w, https://www.cubettech.com/blog/wp-content/uploads/2014/08/Picture2-300x160.png 300w" sizes="(max-width: 942px) 100vw, 942px"></a></p> 
        <p>We could easily choose the package of authentication by making some minute changes in the code. &nbsp;For instance, the code used for choosing local passport authentication will look like:</p> 
        <p>app.post (‘/login’, passport.authenticate(‘local’, { successRedirect: ‘/’,</p> 
        <p>failureRedirect: ‘/login’ }));</p> 
        <p>&nbsp;</p> 
        <p>Local Passport.JS authentication of Node.JS can be easily studied by skimming through the steps listed below. The following code should work perfectly with the latest version of sleek.js.</p> 
        <p>1. First of all you should install passport &amp; passport-local,</p> 
        <p>Then require passport in app.js and modify it in the following format</p> 
        <p>var passport = require(‘passport’);</p> 
        <p>…</p> 
        <p>…</p> 
        <p>app.configure(function(){</p> 
        <p>…</p> 
        <p>app.use(express.session());</p> 
        <p>app.use(express.static(path.join(__dirname, ‘public’)));</p> 
        <p>app.use(passport.initialize());//use passport after express.session.</p> 
        <p>app.use(passport.session());</p> 
        <p>});</p> 
        <p>2. Create a passport.js file in your ‘lib’ folder and write your configuration strategy for example:</p> 
        <p>var passport = require(‘passport’);</p> 
        <p>passport.serializeUser(function(user, done) {</p> 
        <p>done(null, user);</p> 
        <p>});</p> 
        <p>passport.deserializeUser(function(obj, done) {</p> 
        <p>done(null, obj);</p> 
        <p>});</p> 
        <p>var LocalStrategy = require(‘passport-local’).Strategy;</p> 
        <p>passport.use(new LocalStrategy(</p> 
        <p>function(username,password,done) {</p> 
        <p>console.log(username)</p> 
        <p>if (username!=’abc’) {</p> 
        <p>return done(null, false);</p> 
        <p>}</p> 
        <p>else{</p> 
        <p>return done(null, true);</p> 
        <p>}</p> 
        <p>}</p> 
        <p>));</p> 
        <p>3. Include this file by updating application/config/config.js file like below</p> 
        <p>global.sleekConfig.configLibs = [‘passport’];</p> 
        <p>4. Add the route like this in your ‘application/config/routes.js’ or you can add the function as commonRouteFunction</p> 
        <p>{route: ‘/logincheck’, controller: ‘index’, action:’logincheck’,type:”POST”,fn:[‘checkPassport’]}</p> 
        <p>5. You should write the passport authentication function in the route helper in ‘application/helpers/roues.js’, inorder to call the function as a middleware. for example,</p> 
        <p>var passport = require(‘passport’);</p> 
        <p>var routeHelper = {</p> 
        <p>checkPassport:</p> 
        <p>passport.authenticate(‘local’, { successRedirect: ‘/success’,</p> 
        <p>failureRedirect: ‘/login’ })</p> 
        <p>}</p> 
        <p>module.exports = routeHelper;</p> 
        <p>Here we have literally formulated a new application that will enable user to register and login as per the norms set by the developer. The application will also support flash messages and hashing password. One of the other unique features of this type of application is that we could literally filter the traffic to certain sections/pages in our site that utilizes route middleware by means of username and password.</p> 
        <p>I hope this post clearly depicts the local Passport JS authentication process of Node JS Sleek application. I will strike again soon with another post on passport authentication of Facebook, Twitter and Google.</p>
