     <h1> Authenticate users with Azure Mobile Apps or the Active Directory Authentication Library for Cordova </h1> 
     <div class="wa-navigationScroll fixed-element" data-top-spacing="100" data-resting-position="absolute"> 
      <span>In this article:</span> 
      <ul class="scrollspy"> 
       <li><a href="#azure-app-service-auth-and-azure-mobile-apps">Azure App Service Auth and Azure Mobile Apps</a></li> 
       <li><a href="#active-directory-authentication-library-for-cordova">Active Directory Authentication Library for Cordova</a></li> 
       <li><a href="#javascript-amp-3rd-party-options">JavaScript &amp; 3rd Party Options</a></li> 
       <li><a href="#additional-security-topics">Additional Security Topics</a></li> 
       <li><a href="#comments">Comments</a></li> 
      </ul> 
     </div>
     <p>Security is a very broad topic that covers a number of different aspects of an app's lifecycle. Securing an app often represents a number of tradeoffs and key decisions. For the most part you should apply the same <a href="https://code.google.com/archive/p/browsersec/wikis/Main.wiki">best practices to your code as you do for web apps</a>. However, given the increased capabilities Cordova apps are afforded, it is important to limit your risk as much as possible.</p> 
     <p>A critical task for application security is authenticating and authorizing users accessing your app and its associated local or remote data. In this article we'll touch on some options that can help get you get auth up and running.</p> 
     <h2><a id="azure-app-service-auth-and-azure-mobile-apps" name="azure-app-service-auth-and-azure-mobile-apps" href="#azure-app-service-auth-and-azure-mobile-apps"></a>Azure App Service Auth and Azure Mobile Apps</h2> 
     <p><a href="https://azure.microsoft.com/en-us/services/app-service/">Azure App Service</a> is a suite of services designed to help you build great web and mobile apps. <a href="https://azure.microsoft.com/en-us/services/app-service/mobile/">Azure Mobile Apps</a> are mobile integrated client apps that take advantage of features within the broader Azure App Service including Web Apps and API Apps along with some additional useful features and client libraries.</p> 
     <p>A core first step in accessing all of these great services, however, is authorizing users both to access the app for the app to then access data in the cloud. Fortunately, the Cordova plugin for Azure Mobile Apps has an unified authentication interface that currently supports authenticating against Azure Active Directory, Facebook, Google, Twitter, and Microsoft accounts. The unified interface means that you're abstracted from downstream changes and can expect additional provider options and features in the future to streamline things even more.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/cordova-security-auth/20161018070757/azure-mobile-apps.png" alt="Azure Mobile App Auth"> </p> 
     <p>You can add the Azure Mobile Apps plugin your app as follows:</p> 
     <ol> 
      <li><p>Visual Studio Tools for Apache Cordova <strong>Update 8 and up,</strong> simply click "Add" on the <strong>Azure Mobile Apps</strong> plugin in the <strong>config.xml designer.</strong> </p></li> 
      <li><p>When using the command line or Visual Studio Code, you can add the plugin using the Cordova CLI as follows:</p> <pre class="prettyprint copy-to-clipboard ">cordova plugin add cordova-plugin-ms-azure-mobile-apps --save
</pre></li> 
     </ol> 
     <p>Note that you can also get basic information about the user logged in by making a REST call to the <strong>/.auth/me</strong> service endpoint and passing the appropriate auth token. See the <strong><a href="https://azure.microsoft.com/en-us/documentation/articles/app-service-mobile-cordova-get-started-users/">Azure Mobile Apps authentication documentation</a></strong> and the article on <strong><a href="../cordova-security-xmit/">securely transmitting data</a></strong> for additional details on token passing.</p> 
     <h3><a id="try-the-azure-connected-services-sample-app" name="try-the-azure-connected-services-sample-app" href="#try-the-azure-connected-services-sample-app"></a>Try the Azure connected services sample app</h3> 
     <p>To get the sample, find the <a href="https://github.com/Microsoft/cordova-samples/tree/master/azure-connected-services">Azure connected services sample</a> on GitHub.</p> 
     <p>Then follow these steps: 1. Download the sample and open it in Visual Studio.</p> 
     <ol> 
      <li><p>On the Azure portal, create an Azure Mobile App backend. Follow steps in this video to create the Azure Mobile App backend. You will need to create an Azure account if you don't have one.</p> <p><a href="https://www.youtube.com/watch?v=oKWEXU8i5oc" class="video" title="Create an Azure Mobile App"><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/cordova-security-auth/20161018070757/thmb-azure-mobile-app.png" alt="Create an Azure Mobile App"> </a></p> <p>For documentation that shows similar steps, see the <a href="https://azure.microsoft.com/en-us/documentation/articles/app-service-mobile-cordova-get-started/">Getting Started Tutorial</a>. You can skip the section on downloading the client app.</p> 
       <blockquote> 
        <p><strong>Note</strong>: You can follow steps described in the <a href="https://azure.microsoft.com/en-us/documentation/articles/app-service-mobile-cordova-get-started/">Getting Started Tutorial</a> to download the Azure sample, but if you use the getting started sample instead of the Azure connected services sample, you must take extra steps to add required plugins, to add login code, and to update the Cordova version in your app.</p> 
       </blockquote></li> 
      <li><p>Watch this video to add authentication to the Azure connected services sample. The video shows adding authentication with Twitter, but steps are similar for other services as well.</p> <p><a href="https://www.youtube.com/watch?v=PTrxocL2lQg" class="video" title="Add authentication to your app"><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/cordova-security-auth/20161018070757/thmb-authentication.png" alt="Add authentication to your app"> </a></p> <p>To follow similar steps in the tutorial, see <a href="https://azure.microsoft.com/en-us/documentation/articles/app-service-mobile-cordova-get-started-users/">How to: Authenticate users</a></p></li> 
      <li><p>Update code in the login callback function.</p> <p>If you're not using push notifications, comment out this code.</p> <pre class="prettyprint copy-to-clipboard lang-javascript">// Added to register for push notifications.
registerForPushNotifications();
</pre></li> 
     </ol> 
     <h2><a id="active-directory-authentication-library-for-cordova" name="active-directory-authentication-library-for-cordova" href="#active-directory-authentication-library-for-cordova"></a>Active Directory Authentication Library for Cordova</h2> 
     <p>Active Directory (AD) provides an industry leading identity server both in the cloud and on-premises through Azure Active Directory (AAD) and Active Directory Federation Services (ADFS). You can securely authenticate, authorize, access information in AD, and take advantage of device level single sign on and multi-factor authentication (MFA) capabilities through the powerful Active Directory Authentication Library (ADAL) available for all major native and cross-platform mobile and server side technologies.</p> 
     <p>For Cordova this functionality is provided via the ADAL plugin which can be used both with AAD and on-premises ADFS v3 and up. Note that the ADAL plugin is activley maintained but currently in <strong>preview.</strong> It uses the Android, iOS, and .NET ADAL native libraries under the covers and therefore persists auth tokens in a secure cache that you can then query to pass to downstream services.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/cordova-security-auth/20161018070757/adal.png" alt="ADAL"> </p> 
     <p>Adding the plugin is easy.</p> 
     <ol> 
      <li>In Visual Studio, simply click "Add" on the <strong>ADAL for Cordova</strong> plugin in the <strong>config.xml designer.</strong></li> 
      <li><p>When using the command line or Visual Studio Code, you can add the plugin using the Cordova CLI as follows:</p> <pre class="prettyprint copy-to-clipboard ">cordova plugin add cordova-plugin-ms-adal --save
</pre></li> 
     </ol> 
     <p>See the <strong><a href="https://azure.microsoft.com/en-us/documentation/articles/active-directory-devquickstarts-cordova/">Active Directory Quick Start for Cordova</a></strong> for additional details on setup. You can also read <a href="http://www.cloudidentity.com/blog/2015/04/06/adal-plugin-for-apache-cordova-deep-dive/">this blog post</a> on some of the internals and the advantages it provides over other methods.</p> 
     <p>While the quick start uses Azure AD, the plugin also works with <strong>ADFS v3</strong> and up by simply changing the authority and redirect URIs to the appropriate ones for your ADFS installation.</p> 
     <p>The quick start also has code that demonstrates calling the <a href="https://msdn.microsoft.com/en-us/library/azure/hh974476.aspx">Azure AD Graph REST API</a> directly using an AD token from the plugin. This approach can be reused across Azure services and O365 services. See the article on <a href="../cordova-security-xmit/">securely transmitting data</a> along with documentation on <a href="https://msdn.microsoft.com/en-us/library/azure/hh974476.aspx">Azure JSON based REST APIs</a> and <a href="http://dev.office.com/getting-started/office365apis">O365</a> for additional details on token passing to downstream services.</p>

     <h2><a id="javascript-amp-3rd-party-options" name="javascript-amp-3rd-party-options" href="#javascript-amp-3rd-party-options"></a>JavaScript &amp; 3rd Party Options</h2> 
     <p>Cordova also can take advantage of pure JavaScript based solutions to authenticate against Oauth providers thanks to the <strong>InAppBrowser</strong> plugin. See <a href="http://phonegap-tips.com/articles/google-api-oauth-with-phonegaps-inappbrowser.html">this article</a> for an excellent overview on the general approach JavaScript based solutions take. These options can be useful when client libraries are not available. The <strong><a href="https://github.com/nraboy/ng-cordova-oauth">ngCordovaOauth</a></strong> library encapsulates this approach to allow you to target an impressive array of different Oauth providors including on-prem ADFS. However, in general we recommend using provider Cordova plugins, the ADAL plugin, or Azure when security is of paramount concern.</p> 
     <p>If neither of the above options meet your needs, there are a number of 3rd party solutions that may be of use. First note that many Single Sign-On (SSO) solutions including <a href="https://auth0.com/">Auth0</a> actually explictly support PhoneGap/Cordova in their JavaScript client libraries or related Cordova plugins. If you already have a SSO provider, be sure to check with them to see what best practices they provide for Cordova apps.</p>
