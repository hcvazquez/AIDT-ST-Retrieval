       </svg></a>Need help on SharePoint with Node.JS? Join our gitter chat and ask question! <a href="https://gitter.im/sharepoint-node/Lobby"><img src="https://badges.gitter.im/gitterHQ/gitter.png" alt="Gitter chat"></a></h3> 
     <p><code>node-sp-auth</code> allows you to perform SharePoint unattended (without user interaction) http authentication with nodejs using different authentication techniques. <code>node-sp-auth</code> also takes care about caching authentication data for performance (no need for you to think about how long authentication will be available, that's a task for <code>node-sp-auth</code>, as soon as authentication will be expired, <code>node-sp-auth</code> will renew it internally).</p> 
     <p>Versions supported:</p> 
     <ul> 
      <li>SharePoint 2013, 2016</li> 
      <li>SharePoint Online</li> 
     </ul> 
     <p>Authentication options:</p> 
     <ul> 
      <li>SharePoint 2013, 2016: 
       <ul> 
        <li>Addin only permissions</li> 
        <li>User credentials through the http ntlm handshake</li> 
        <li>Form-based authentication (FBA)</li> 
       </ul> </li> 
      <li>SharePoint Online: 
       <ul> 
        <li>Addin only permissions</li> 
        <li>SAML based with user credentials</li> 
       </ul> </li> 
      <li>ADFS user credentials (works with both SharePoint on-premise and Online)</li> 
      <li>On demand authentication. Uses interactive browser session for asking credentials. Supports third-party authentication providers for SharePoint Online and SharePoint on-premise. Doesn't support integrated windows authentication (NTLM).</li> 
     </ul> 
     <p><a href="https://github.com/s-KaiNet/node-sp-auth/wiki">Wiki</a> contains detailed steps you need to perform in order to use any of authentication options as well as sample using.</p>

        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>API:</h2> 
     <h3><a id="user-content-getauthurl-credentialoptions" class="deep-link" href="#getauthurl-credentialoptions" aria-hidden="true">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" viewbox="0 0 16 16" width="16">

       </svg></a>getAuth(url, credentialOptions)</h3> 
     <h4><a id="user-content-return-value" class="deep-link" href="#return-value" aria-hidden="true">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" viewbox="0 0 16 16" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>return value:</h4> 
     <p>Promise resolving into object with following properties:</p> 
     <ul> 
      <li><code>headers</code> - http headers (normally contain <code>Authorization</code> header, may contain any other heraders as well)</li> 
      <li><code>options</code> - any additional options you may need to include for succesful request. For example, in case of on premise user credentials authentication, you need to set <code>agent</code> property on corresponding http client</li> 
     </ul> 
     <h4><a id="user-content-params" class="deep-link" href="#params" aria-hidden="true">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" viewbox="0 0 16 16" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>params:</h4> 
     <ul> 
      <li><code>url</code> - required, string, url to SharePoint site, <code>https://sp2013/sites/dev/</code> or <code>https:/[your tenant].sharepoint.com/sites/dev/</code></li> 
      <li><code>credentialOptions</code> - optional, object in a form of key-value. Each authentication option requires predefined credential object, depending on authentication type. Based on credentials provided, <code>node-sp-auth</code> automatically determines which authentication strategy to use (strategies listed in the top of the readme file).</li> 
     </ul> 
     <p>Possible values for <code>credentialOptions</code> (depending on authentication strategy):</p> 
     <ul> 
      <li> <p>SharePoint on premise (2013, 2016):</p> 
       <ul> 
        <li><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/SharePoint%20on-premise%20addin%20only%20authentication">Addin only permissions:</a><br> <code>clientId</code>, <code>issuerId</code>, <code>realm</code>, <code>rsaPrivateKeyPath</code>, <code>shaThumbprint</code></li> 
        <li><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/SharePoint%20on-premise%20user%20credentials%20authentication">User credentials through the http ntlm handshake:</a><br> <code>username</code>, <code>password</code>, <code>domain</code>, <code>workstation</code></li> 
        <li><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/SharePoint%20on-premise%20FBA%20authentication">User credentials for form-based authentication (FBA):</a><br> <code>username</code>, <code>password</code>, <code>fba</code> = true</li> 
       </ul> </li> 
      <li> <p>SharePoint Online:</p> 
       <ul> 
        <li><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/SharePoint%20Online%20addin%20only%20authentication">Addin only permissions:</a><br> <code>clientId</code>, <code>clientSecret</code></li> 
        <li><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/SharePoint%20Online%20user%20credentials%20authentication">SAML based with user credentials</a><br> <code>username</code> , <code>password</code></li> 
       </ul> </li> 
      <li> <p><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/ADFS%20user%20credentials%20authentication">ADFS user credentials:</a><br> <code>username</code>, <code>password</code>, <code>relyingParty</code>, <code>adfsUrl</code>, <code>adfsCookie</code></p> </li> 
      <li> <p><a href="https://github.com/s-KaiNet/node-sp-auth/wiki/On%20demand%20authentication">On demand authentication</a><br> <code>ondemand</code> = true, <code>electron</code>, <code>force</code>, <code>persist</code></p> </li> 
      <li> <p>no authentication - do not provide any authentication data at all, like <code>spauth.getAuth(url).then(...)</code>. In that case <code>node-sp-auth</code> will ask you for the site url and credentials. You will have to select any of the credential options listed above. Credentials will be stored in a user folder in an encrypted manner.<br> Credits: Andrew Koltyakov <a href="https://github.com/koltyakov">@koltyakov</a> and his awesome <a href="https://github.com/koltyakov/node-sp-auth-config">node-sp-auth-config</a></p> </li> 
     </ul> 
     <p>Please, use <a href="https://github.com/s-KaiNet/node-sp-auth/wiki/">Wiki</a> to see how you can configure your environment in order to use any of this authentication options.</p>
