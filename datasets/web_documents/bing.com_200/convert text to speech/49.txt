        <h3><a name="where" id="where"></a>Where can I get the Java Speech API (JSAPI)?</h3> 
        <p>The Java Speech API (JSAPI) is not part of the JDK and Sun does not ship an implementation of JSAPI. Instead, we work with third party speech companies to encourage the availability of <a href="/technetwork/java/jsapifaq-135248.html#implementation">multiple implementations</a>.

        <h3><a name="what" id="what"></a>What is the Java Speech API (JSAPI)?</h3> 
        <p>The Java Speech API allows Java applications to incorporate speech technology into their user interfaces. It defines a cross-platform API to support command and control recognizers, dictation systems and speech synthesizers. 
         <!--=== Return to top line ===--></p> 
        <p><img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <a href="/technetwork/java/jsapifaq-135248.html#top-question">Question Index</a> <img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> </p> 
        <h3><a name="spec-available" id="spec-available"></a>When will the JSAPI specification be available?</h3> 
        <p>The Java Speech 1.0 API specification was released on October 26, 1998, and is available <a href="/technetwork/java/speech-138007.html">here</a> 
         <!--=== Return to top line ===--></p>

        <h3><a name="spec" id="spec"></a>What does the Java Speech API specification include?</h3> 
        <p>The Java Speech API specification includes the Javadoc-style API documentation for the approximately 70 classes and interfaces in the API. The specification also includes a detailed Programmer's Guide which explains both introductory and advanced speech application programming with JSAPI. Two companion specifications are available: JSML and JSGF.</p> 
        <p>The specification is not yet provided with the .class files needed to compile applications with JSAPI. 
         <!--=== Return to top line ===--></p>

        <h3><a name="JSXX" id="JSXX"></a>What are JSML and JSGF?</h3> 
        <p>The <strong>Java Speech API Markup Language (JSML)</strong> and the <strong>Java Speech API Grammar Format (JSGF)</strong> are companion specifications to the Java Speech API. JSML (currently in beta) defines a standard text format for marking up text for input to a speech synthesizer. JSGF version 1.0 defines a standard text format for providing a grammar to a speech recognizer. Both specifications are available <a href="/technetwork/java/speech-138007.html">here</a> 
         <!--=== Return to top line ===--></p>

        <h3><a name="applet-use" id="applet-use"></a>How do I use JSAPI in an applet?</h3> 
        <p>It is possible to use JSAPI in an applet. In order to do this, users will need the Java Plug-in (see <a href="/technetwork/java/plugin-137649.html">here</a>). The reason for this is that JSAPI implementations require access to the AWT EventQueue, and the built-in JDK support in the browsers we've worked with denies any applet access to the AWT EventQueue. The Java Plug-in doesn't have this restriction, and users can configure the Java Plug-in to grant or deny applet access to the AWT Queue.</p> 
        <h3>If you are using JRE 1.1:</h3> 
        <p>Have your users follow these steps if your applet is based upon JDK 1.1:</p> 
        <ul> 
         <li>Obtain a JDK 1.1.7 or better Java Runtime Environment (JRE). The reason for this is we have had problems with applet security being denied with JDK 1.1.6. Please note that the user needs the JRE and not the JDK. The JRE is freely available for download from the following URL: <p><a href="/technetwork/java/j2se-eol-137618.html">link</a></p> </li> 
         <li>Before running the browser, have the user modify their CLASSPATH environment variable to include the supporting classes for JSAPI. For example, if the user has IBM's Speech for Java, have the user include the ibmjs.jar file in CLASSPATH.</li> 
         <li>Make sure any shared libraries for the JSAPI support are in the user's PATH. For example, if the user has IBM's Speech For Java, have the user include the ibmjs lib directory in their PATH (e.g., c:\ibmjs\lib).</li> 
         <li>Have the user copy the speech.properties to their home directory. A user can determine their home directory by enabling the console for the Java Plug-in. When the user accesses a page that uses the Java Plug-in, the Java Plug-in console will tell the user what it thinks the user's home directory is.</li> 
         <li>Use javakey to add your identity to their signature database (i.e., identitydb.obj). This will tell the Java Plug-in to trust applets signed by you.</li> 
         <li>Copy the identitydb.obj that was created or updated in previous step to the user's home directory (the same place where the user copied speech.properties).</li> 
        </ul> 
        <p>Then perform these steps on your applet:</p> 
        <ul> 
         <li>Use javakey to both create a signature database for your system and to sign your applet's jar file. This will allow the applet to participate in the security model.</li> 
         <li>Create an HTML page that uses your applet in the Plug-in.</li> 
         <li>If the user experiences a "checkread" exception while attempting to run your applet, it's most likely due to a mismatch between the user's identitydb.obj file and the signature on your applet's jar file. A way to remedy this is to recreate your identitydb.obj and re-sign your jar file.</li> 
        </ul> 
        <h3>If you are using JRE 1.2:</h3> 
        <p>The Java 2 platform's security model allows signing as done with JDK 1.1, but it also permits finer grained access control. The following are just some examples, and we recommend you read the Java Security Architecture Specification at the following URL before deciding what to do:</p> 
        <p><a href="/technetwork/java/j2se-eol-137618.html">link</a></p> 
        <p>For a quick start, have your users do the following if your applet uses the Java 2 (i.e., JDK 1.2) platform:</p> 
        <ul> 
         <li>Obtain the JDK 1.2 Plug-in.</li> 
         <li>Before running the browser, have the user modify their CLASSPATH environment variable to include the supporting classes for JSAPI. For example, if the user has IBM's Speech for Java, have the user include the ibmjs.jar file in CLASSPATH.</li> 
         <li>Make sure any shared libraries for the JSAPI support are in the user's PATH. For example, if the user has IBM's Speech For Java, have the user include the ibmjs lib directory in their PATH (e.g., c:\ibmjs\lib).</li> 
         <li>Have the user copy the speech.properties to their home directory. A user can determine their home directory by enabling the console for the Java Plug-in. When the user accesses a page that uses the Java Plug-in, the Java Plug-in console will tell the user what it thinks the user's home directory is.</li> 
         <li>Have the user modify their java.policy file in the Java Plug-in's security directory appropriately. There are many ways to do this. The following are just a few possibilities: 
          <ul> 
           <li>Grant all applets the AllPermission property. This is extremely dangerous and is only provided as an example. To do this, have the user modify their java.policy file to contain only the following lines: <pre>
grant {
 permission java.security.AllPermission;
}
 
</pre> </li> 
           <li>Grant permissions to a particular URL (e.g., the URL containing your applet). To do this, have the user add the following lines to their java.policy file: <pre>
grant codeBase "http://your.url.here" {
 permission java.security.AllPermission;
}
 
</pre> </li> 
          </ul> </li> 
        </ul> 
        <p>The information in this FAQ is not meant to be a complete tutorial on the JDK 1.1 and JDK 1.2 architecture. Instead, it is meant to be hopefully enough to get you started with running JSAPI applets in a browser. We suggest you visit the following URLs to obtain more information on the Java Security models:</p> 
        <p>Java Security Home Page: <br> <a href="/technetwork/java/javase/tech/index-jsp-136007.html">link</a></p> 
        <p>Tutorial on JDK 1.1 Security: <br> <a href="http://docs.oracle.com/javase/tutorial/security1.1/">link</a></p> 
        <p>Tutorial on JDK 1.2 Security: <br> <a href="http://docs.oracle.com/javase/tutorial/security1.2">link</a> 
         <!--=== Return to top line ===--></p> 
        <p><img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <a href="/technetwork/java/jsapifaq-135248.html#top-question">Question Index</a> <img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> </p> 
        <h3><a name="security-exception" id="security-exception"></a>Why does Netscape Navigator or Internet Explorer throw a security exception when I use JSAPI in an applet?</h3> 
        <p>JSAPI implementations require access to the AWT EventQueue. The built-in Java platform support in the browsers we've worked with denies an applet access to the AWT EventQueue. As a result, JSAPI implementations will be denied access to the AWT EventQueue. In addition, we are not aware of a way to configure the built-in Java platform support in these environments to allow access to the AWT EventQueue.</p> 
        <p>The Java Plug-in (see <a href="/technetwork/java/plugin-137649.html">link</a>), however, can be configured to allow an applet the necessary permissions it needs to use an implementation of JSAPI. As a result, we currently recommend using the Java Plug-in for applets that use JSAPI. 
         <!--=== Return to top line ===--></p> 
        <p><img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <a href="/technetwork/java/jsapifaq-135248.html#top-question">Question Index</a> <img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> </p> 
        <h3><a name="bugging" id="bugging"></a>I'm concerned about JSAPI applets "bugging" my office. What are the plans for JSAPI and security on JDK 1.2?</h3> 
        <p>The JSAPI 1.0 specification includes the SpeechPermission class that currently only supports one SpeechPermission: javax.speech. When that permission is granted, an application or applet has access to all the capabilities provided by installed speech recognizers and synthesizers. Without that permission, an application or applet has no access to speech capabilities.</p> 
        <p>As speech technology matures it is anticipated that a finer-grained permission model will be introduced to provide access by applications and applets to some, but not all, speech capabilities.</p> 
        <p>Before granting speech permission, developers and users should consider the potential impact of the grant. 
         <!--=== Return to top line ===--></p> 
        <p><img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <a href="/technetwork/java/jsapifaq-135248.html#top-question">Question Index</a> <img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> </p> 
        <h3><a name="audio" id="audio"></a>Does JSAPI allow me to control the audio input source of a recognizer or redirect the audio output of a speech synthesizer?</h3> 
        <p>This support is currently not in JSAPI. We plan to use the <a href="/technetwork/java/sound-136584.html">Java Sound API</a> to help provide this support in the future. We purposely left room for expansion in the <a href="http://docs.oracle.com/cd/E17802_01/products/products/java-media/speech/forDevelopers/jsapi-doc/javax/speech/AudioManager.html">javax.speech.AudioManager</a> interface and will further investigate this support after the Java Sound API is finalized. 
         <!--=== Return to top line ===--></p> 
        <p><img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <a href="/technetwork/java/jsapifaq-135248.html#top-question">Question Index</a> <img width="112" height="7" src="/ocom/groups/public/@otn/documents/digitalasset/149593.gif" alt=""> <br>
