      <h2><span class="mw-headline" id="Legal_issues">Legal issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=1" title="Edit section: Legal issues">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id=".NET">.NET</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=2" title="Edit section: .NET">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The Mono project aims to avoid infringing on any patents or copyrights, and to the extent that they are successful, the project can be safely distributed and used under the <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GPL</a>. On November 2, 2006, Microsoft and Novell announced a joint agreement whereby Microsoft promised not to sue Novell or its customers for patent infringement.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> According to a statement on the blog of Mono project leader <a href="/wiki/Miguel_de_Icaza" title="Miguel de Icaza">Miguel de Icaza</a>, this agreement only extends to Mono for Novell developers and users.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> Because of the possible threat of Microsoft patents, the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">FSF</a> recommends that people avoid creating software that depends on Mono or <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup><sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p> 
      <p>The Microsoft/Novell agreement was criticized by some in the <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> community because it violates the principles of giving equal rights to all users of a particular program (see <a href="/wiki/Microsoft-Novell_agreement" class="mw-redirect" title="Microsoft-Novell agreement">Agreement with Microsoft</a> and <a href="/wiki/Mono_(software)#Mono_and_Microsoft.27s_patents" title="Mono (software)">Mono and Microsoft's patents</a>).</p> 
      <p>In response to the Microsoft/Novell agreement, the <a href="/wiki/Free_Software_Foundation" title="Free Software Foundation">Free Software Foundation</a> revised its <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">GNU General Public License</a> to close the loophole used by Microsoft and Novell to bypass the GPL's very restrictive provisions on patent deals.<sup id="cite_ref-gnuvsmicrosoft_5-0" class="reference"><a href="#cite_note-gnuvsmicrosoft-5">[5]</a></sup> The FSF also stated that by selling coupons for Novell's Linux software, the mechanism by which Microsoft circumvented the GNU license, it considers Microsoft to be a Linux vendor, and thereby subject to the full terms and conditions laid out in the GPL.<sup id="cite_ref-gnuvsmicrosoft_5-1" class="reference"><a href="#cite_note-gnuvsmicrosoft-5">[5]</a></sup></p> 
      <p>The .NET landscape started to change in 2013, when Microsoft decided to open source many of its core .NET technologies under <a href="/wiki/Apache_License" title="Apache License">Apache License</a>,<sup id="cite_ref-msopensource1_6-0" class="reference"><a href="#cite_note-msopensource1-6">[6]</a></sup> with even more donated to newly formed <a href="/wiki/.NET_Foundation" title=".NET Foundation">.NET Foundation</a> in 2014.<sup id="cite_ref-msopensource2_7-0" class="reference"><a href="#cite_note-msopensource2-7">[7]</a></sup> Open sourced technologies include <a href="/wiki/ASP.NET_MVC" title="ASP.NET MVC">ASP.NET MVC</a>, <a href="/wiki/Entity_Framework" title="Entity Framework">Entity Framework</a>, <a href="/wiki/Managed_Extensibility_Framework" title="Managed Extensibility Framework">Managed Extensibility Framework</a>, <a href="/wiki/Microsoft_Roslyn" class="mw-redirect" title="Microsoft Roslyn">Roslyn</a> compiler-as-a-service infrastructure (together with C# and <a href="/wiki/Visual_Basic.NET" class="mw-redirect" title="Visual Basic.NET">Visual Basic.NET</a> compilers), <a href="/wiki/F_Sharp_(programming_language)" title="F Sharp (programming language)">F#</a> functional-first language compiler,<sup id="cite_ref-fsharpfoundation_8-0" class="reference"><a href="#cite_note-fsharpfoundation-8">[8]</a></sup> and many more.<sup id="cite_ref-netfoundationprojectlist_9-0" class="reference"><a href="#cite_note-netfoundationprojectlist-9">[9]</a></sup> Microsoft and <a href="/wiki/Xamarin" title="Xamarin">Xamarin</a> announced collaboration, with the intent to increase cross-platform availability of .NET on Mac OS, Linux, and mobile devices.<sup id="cite_ref-msxamarincollaboration_10-0" class="reference"><a href="#cite_note-msxamarincollaboration-10">[10]</a></sup></p> 
      <h2><span class="mw-headline" id="Traditional_computer_applications">Traditional computer applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=3" title="Edit section: Traditional computer applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="Desktop_applications">Desktop applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=4" title="Edit section: Desktop applications">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Although Java's <a href="/wiki/Abstract_Window_Toolkit" title="Abstract Window Toolkit">AWT</a> (Abstract Window Toolkit) and <a href="/wiki/Swing_(Java)" title="Swing (Java)">Swing</a> libraries are not shy of features, Java has struggled to establish a foothold in the desktop market. <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> was also slow, in the eyes of some,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2007)">citation needed</span></a></i>]</sup> to promote Java to developers and end users alike in a way that makes it an appealing choice for desktop software. Even technologies such as <a href="/wiki/Java_Web_Start" title="Java Web Start">Java Web Start</a>, which have few parallels within rival languages and platforms, have barely been promoted.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2007)">citation needed</span></a></i>]</sup></p> 
      <p>The release of Java version 6.0 on December 11, 2006, saw a renewed focus on the desktop market with an extensive set of new tools for closer integration with the desktop. At the 2007 <a href="/wiki/JavaOne" title="JavaOne">JavaOne</a> conference Sun made further desktop related announcements, including a new language aimed at taking on <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a> (<a href="/wiki/JavaFX" title="JavaFX">JavaFX</a>), a new lightweight way of downloading the JRE that sees the initial footprint reduced to under 2&nbsp;<a href="/wiki/Megabyte" title="Megabyte">Mb</a>, and a renewed focus on <a href="/wiki/Multimedia" title="Multimedia">multimedia</a> libraries.</p> 
      <p>An alternative to AWT and Swing is the <a href="/wiki/Standard_Widget_Toolkit" title="Standard Widget Toolkit">Standard Widget Toolkit</a> (<a href="/wiki/Standard_Widget_Toolkit" title="Standard Widget Toolkit">SWT</a>), which was originally developed by IBM and now maintained by the <a href="/wiki/Eclipse_Foundation" title="Eclipse Foundation">Eclipse Foundation</a>. It attempts to achieve improved performance and visualization of Java desktop applications by relying on underlying native libraries where possible.</p> 
      <p>On Windows, Microsoft's .NET is a popular desktop development providing both Windows Forms (a lightweight wrapper around the Win32 API), Windows Presentation Foundation, and Silverlight. With the integration of .NET into the Windows platform, .NET apps are first class citizens in the Windows environment with tighter OS integration and native look and feel compared to Java's Swing.</p> 
      <p>Outside of Windows, Silverlight is portable to the Mac OS X desktop. Mono is also becoming more common in open source and free software systems due to its inclusion<sup id="cite_ref-monoapplications_11-0" class="reference"><a href="#cite_note-monoapplications-11">[11]</a></sup> on many Linux desktop environments.</p> 
      <h3><span class="mw-headline" id="Server_applications">Server applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=5" title="Edit section: Server applications">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>This is probably the arena in which the two platforms are closest to being considered rivals. Java, through its <a href="/wiki/Java_EE" class="mw-redirect" title="Java EE">Java EE</a> (a.k.a. Java Platform Enterprise Edition) platform, and .NET through <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, compete to create web-based dynamic content and applications.</p> 
      <p>Both platforms are well used and supported in this market. Of the top 1,000 websites, approximately 24% use ASP.net and also 24% use Java, whereas of all the websites approximately 17% use ASP.net and 3% use Java.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p> 
      <p>Some of Oracle's Java-related license agreements for Java EE define aspects of the Java platform as a trade secret,<sup id="cite_ref-sunj2eesdk5update2_13-0" class="reference"><a href="#cite_note-sunj2eesdk5update2-13">[13]</a></sup> and prohibit the end user from contributing to a third-party Java environment. Specifically, at least one current license for an Oracle Java EE development package contains the following terms: <i>"You may make a single archival copy of Software, but otherwise may not copy, modify, or distribute Software." — "Unless enforcement is prohibited by applicable law, you may not decompile, or reverse engineer Software." — "You may not publish or provide the results of any benchmark or comparison tests run on Software to any third party without the prior written consent of Oracle." — "Software is confidential and copyrighted."</i><sup id="cite_ref-sunj2eesdk5update2_13-1" class="reference"><a href="#cite_note-sunj2eesdk5update2-13">[13]</a></sup> However, while Oracle's software is subject to the above license terms, Oracle's Java EE <a href="/wiki/API" class="mw-redirect" title="API">API</a> reference has been implemented under an open-source license by the <a href="/wiki/JBoss" class="mw-redirect" title="JBoss">JBoss</a> and <a href="/wiki/JOnAS" title="JOnAS">JOnAS</a> projects.</p> 
      <p>Microsoft's implementation of <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a> is not part of the standardized CLI, and while Microsoft's runtime environment and development tools are not subject to comparable secrecy agreements to Java EE, the official Microsoft tools are not <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> or <a href="/wiki/Free_software" title="Free software">free software</a>, and require Windows servers. However, a <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a> free software ASP.NET implementation is part of the <a href="/wiki/Mono_(software)" title="Mono (software)">Mono project</a> (minus webparts and <a href="/wiki/Web_Services_Enhancements" title="Web Services Enhancements">Web Services Enhancements</a>).<sup id="cite_ref-mono_asp_net_14-0" class="reference"><a href="#cite_note-mono_asp_net-14">[14]</a></sup> Mono supports ASP.NET 4.0 including Web Forms, Microsoft AJAX, and ASP.NET MVC.<sup id="cite_ref-mono_asp_net_4_15-0" class="reference"><a href="#cite_note-mono_asp_net_4-15">[15]</a></sup></p> 
      <h2><span class="mw-headline" id="Embedded_applications">Embedded applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=6" title="Edit section: Embedded applications">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="Mobile_applications">Mobile applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=7" title="Edit section: Mobile applications">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Google's popular <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> platform for mobile application is based on Java. Google adopted a customised virtual machine called Dalvik to optimise the execution of Java code for mobile devices.</p> 
      <p>Oracle provides <a href="/wiki/Java_ME" class="mw-redirect" title="Java ME">Java ME</a>; a reference implementation for mobile OEM vendors. Java ME is made up of various profiles that are subsets of the Java desktop environment with additional libraries targeted at mobile and set-top-box development. Java ME has a very large base within the mobile phone and <a href="/wiki/Personal_digital_assistant" title="Personal digital assistant">PDA</a> markets, with only the cheapest devices now devoid of a <a href="/wiki/K_virtual_machine" title="K virtual machine">KVM</a> (a cut down <a href="/wiki/JVM" class="mw-redirect" title="JVM">Java Virtual Machine</a> for use on devices with limited processing power). Java software, including many games, is commonplace.</p> 
      <p>While many feature phones include a JVM, they are not always heavily used by users (particularly in South Africa). Initially Java applications on most phones typically consisted of menuing systems, small games, or systems to download ringtones etc. However, more powerful phones are increasingly being sold with simple applications pre-loaded, such as translation dictionaries, world clock displays (darkness/light, timezones, etc.) and calculators. Some of these are written in Java, although how often phone owners actually use them is probably unknown.</p> 
      <p>Microsoft currently ships the <a href="/wiki/.NET_Compact_Framework" title=".NET Compact Framework">.NET Compact Framework</a> that runs on Windows CE and mobile devices, set-top boxes, and PDAs as well as the <a href="/wiki/Xbox_360" title="Xbox 360">Xbox 360</a>. Microsoft also provides the <a href="/wiki/.NET_Micro_Framework" title=".NET Micro Framework">.NET Micro Framework</a> for embedded developers with limited resources. Alternatively, Novell licenses embeddable versions of Mono to 3rd parties to use in their devices, and <a href="/wiki/Xamarin" title="Xamarin">Xamarin</a> commercially distributes the <a href="/wiki/MonoDroid" class="mw-redirect" title="MonoDroid">MonoDroid</a> and <a href="/wiki/MonoTouch" class="mw-redirect" title="MonoTouch">MonoTouch</a> framework for Android and iPhone development, respectively. Windows Phone 7 uses Silverlight for native apps, but Windows Phone 8 has C# and XAML as the main languages.</p> 
      <h3><span class="mw-headline" id="Home_entertainment_technologies">Home entertainment technologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=8" title="Edit section: Home entertainment technologies">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Java has found a market in digital television, where it can be used to provide software that sits alongside programming, or extends the capabilities of a given <a href="/wiki/Set_Top_Box" class="mw-redirect" title="Set Top Box">Set Top Box</a>. <a href="/wiki/TiVo" title="TiVo">TiVo</a>, for example, has a facility called "<a href="/w/index.php?title=Home_Media_Engine&amp;action=edit&amp;redlink=1" class="new" title="Home Media Engine (page does not exist)">Home Media Engine</a>", which allows <a href="/wiki/JavaTV" class="mw-redirect" title="JavaTV">JavaTV</a> software to be transmitted to an appropriate TiVo device to complement programming or provide extra functionality (for example, personalized stock tickers on a business news program.)</p> 
      <p>A variant of Java has been accepted as the official software tool for use on the next generation optical disc technology <a href="/wiki/Blu-ray" title="Blu-ray">Blu-ray</a>, via the <a href="/wiki/BD-J" title="BD-J">BD-J</a> interactive platform. This will mean that interactive content, such as menus, games, downloadables, etc. on all Blu-ray optical discs will be created under a variant of the Java platform.</p> 
      <p>Rather than using Java, <a href="/wiki/HD_DVD" title="HD DVD">HD DVD</a> (the defunct high definition successor to <a href="/wiki/DVD" title="DVD">DVD</a>) used a technology jointly developed by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> and <a href="/wiki/The_Walt_Disney_Company" title="The Walt Disney Company">Disney</a> called <a href="/wiki/HDi_Interactive_Format" class="mw-redirect" title="HDi Interactive Format">HDi</a> that was based on <a href="/wiki/Extensible_Markup_Language" class="mw-redirect" title="Extensible Markup Language">XML</a>, <a href="/wiki/Cascading_Style_Sheets" title="Cascading Style Sheets">CSS</a>, <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, and other technologies that are comparable to those used by standard <a href="/wiki/Web_browser" title="Web browser">Web browsers</a>.</p> 
      <p>The BD-J platform <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a> is more extensive than its iHD rival, with an alleged 8,000 methods and interfaces, as opposed to iHD's 400.<sup id="cite_ref-bludvd1_16-0" class="reference"><a href="#cite_note-bludvd1-16">[16]</a></sup> And while Microsoft is pushing iHD's XML presentation layer by including it with Windows Vista, iHD is still a newcomer in a market sector where Java technologies are already commonplace.<sup id="cite_ref-bluedvd2_17-0" class="reference"><a href="#cite_note-bluedvd2-17">[17]</a></sup></p> 
      <p>However, the fact that the HD DVD format has been abandoned in favor of Blu-ray means that HDi is no longer supported on any <a href="/wiki/Optical_disc" title="Optical disc">optical disc</a> format, making the BD-J format a clear winner.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup></p> 
      <h2><span class="mw-headline" id="Runtime_inclusion_in_operating_systems">Runtime inclusion in operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=9" title="Edit section: Runtime inclusion in operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span id=".NET/Mono"></span><span class="mw-headline" id=".NET.2FMono">.NET/Mono</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=10" title="Edit section: .NET/Mono">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>On Windows, Microsoft has promoted .NET as its flagship development platform<sup id="cite_ref-mscsharpflagship_19-0" class="reference"><a href="#cite_note-mscsharpflagship-19">[19]</a></sup> by including the .NET runtime in Windows XP Service Pack 2 and 3, Windows Server 2003, Windows Vista, Windows Server 2008 and Windows 7. Microsoft also distributes the Visual Studio Express development environment at no cost,<sup id="cite_ref-vsexpress_20-0" class="reference"><a href="#cite_note-vsexpress-20">[20]</a></sup> and the Visual Studio Community development environment at no cost, with limited use for organizations.<sup id="cite_ref-vscommunity_21-0" class="reference"><a href="#cite_note-vscommunity-21">[21]</a></sup></p> 
      <p>.NET Framework 3.5 runtime is not pre-installed on versions of Windows prior to Vista SP1, and must be downloaded by the user, which has been criticized because of its large size (65 MB download for .NET 3.5).<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup><sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup></p> 
      <p>While neither .NET nor Mono are installed with Mac OS X out-of-the-box, the Mono project can be downloaded and installed separately, for free, for any Mac user who wants to build or run C# and .NET software. As of 13 May 2008, Mono's System.Windows.Forms 2.0 is API-complete (contains 100% of classes, methods etc. in Microsoft's System.Windows.Forms 2.0); also System.Windows.Forms 2.0 works natively on Mac OS X.</p> 
      <p>C# and the CLI are included and used in a number of Linux- and BSD-based operating systems by way of including the free software Mono Project.<sup id="cite_ref-fedoramono_24-0" class="reference"><a href="#cite_note-fedoramono-24">[24]</a></sup><sup id="cite_ref-debianmono_25-0" class="reference"><a href="#cite_note-debianmono-25">[25]</a></sup><sup id="cite_ref-ubuntumono_26-0" class="reference"><a href="#cite_note-ubuntumono-26">[26]</a></sup></p> 
      <p>As a result of inclusion of .NET or Mono runtimes in the distributions of Windows and Linux, non-GUI applications that use the programming interfaces that are common to both .NET and Mono can be developed in C# or any other .NET language and then deployed across many operating systems and processor architectures using a runtime environment that is available as a part of the operating system's installation.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup><sup id="cite_ref-28" class="reference"><a href="#cite_note-28">[28]</a></sup><sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[29]</a></sup> Both Microsoft .NET and the Mono project have complete support for the Ecma- and ISO-standardized C# language and .NET runtime, and many of Microsoft's non-standardized .NET programming interfaces have been implemented or are under development in Mono,<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[30]</a></sup> but each environment includes many components that have not been implemented in the other.</p> 
      <h3><span class="mw-headline" id="Java">Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Comparison_of_the_Java_and_.NET_platforms&amp;action=edit&amp;section=11" title="Edit section: Java">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Starting with XP SP1a, Windows does not ship with a Java runtime environment.</p> 
      <p>Java was pre-installed on all new Apple computers beginning with <a href="/wiki/Mac_OS_X_10.0" title="Mac OS X 10.0">Mac OS X 10.0</a> and ending with <a href="/wiki/Mac_OS_X_Snow_Leopard" title="Mac OS X Snow Leopard">10.6</a>, after which Java 6 became an optional Apple download.<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[31]</a></sup><sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[32]</a></sup> Java 7 and later releases are provided by Oracle.<sup id="cite_ref-33" class="reference"><a href="#cite_note-33">[33]</a></sup></p> 
      <p>Java comes pre-installed with many commercial Unix flavors, including those from <a href="/wiki/Hewlett_Packard" class="mw-redirect" title="Hewlett Packard">Hewlett Packard</a>, <a href="/wiki/IBM" title="IBM">IBM</a> and <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>. As of June 2009, the <a href="/wiki/Debian" title="Debian">Debian</a>,<sup id="cite_ref-34" class="reference"><a href="#cite_note-34">[34]</a></sup> <a href="/wiki/Fedora_(Linux_distribution)" class="mw-redirect" title="Fedora (Linux distribution)">Fedora 9</a>,<sup id="cite_ref-fedora9redhat_35-0" class="reference"><a href="#cite_note-fedora9redhat-35">[35]</a></sup> <a href="/wiki/Mandriva" title="Mandriva">Mandriva</a><sup id="cite_ref-36" class="reference"><a href="#cite_note-36">[36]</a></sup> <a href="/wiki/OpenSUSE" title="OpenSUSE">OpenSUSE</a>,<sup id="cite_ref-37" class="reference"><a href="#cite_note-37">[37]</a></sup> <a href="/wiki/Slackware" title="Slackware">Slackware</a> extra,<sup id="cite_ref-38" class="reference"><a href="#cite_note-38">[38]</a></sup> and <a href="/wiki/Ubuntu_(operating_system)" title="Ubuntu (operating system)">Ubuntu 8.04</a><sup id="cite_ref-ubuntu_openjdk-6_39-0" class="reference"><a href="#cite_note-ubuntu_openjdk-6-39">[39]</a></sup> distributions are available with OpenJDK, based completely on <a href="/wiki/Free_software" title="Free software">free</a> and <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> code.<sup id="cite_ref-40" class="reference"><a href="#cite_note-40">[40]</a></sup> Since June 2008, OpenJDK passed all of the compatibility tests in the Java SE 6 <a href="/wiki/Technology_Compatibility_Kit" title="Technology Compatibility Kit">JCK</a> and can claim to be a fully compatible Java 6 implementation.<sup id="cite_ref-41" class="reference"><a href="#cite_note-41">[41]</a></sup><sup id="cite_ref-fedora9la_42-0" class="reference"><a href="#cite_note-fedora9la-42">[42]</a></sup> OpenJDK can run complex applications such as <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a>, <a href="/wiki/GlassFish" title="GlassFish">GlassFish</a>, <a href="/wiki/JBoss" class="mw-redirect" title="JBoss">JBoss</a>, or <a href="/wiki/Netbeans" class="mw-redirect" title="Netbeans">Netbeans</a>.</p> 
      <p>The Operating System Distributor License for Java (DLJ)<sup id="cite_ref-dlj_43-0" class="reference"><a href="#cite_note-dlj-43">[43]</a></sup> was a Sun initiative to ease distribution issues with operating systems based on Linux or OpenSolaris.</p> 
      <p>If Java is not installed on a computer by default, it may be downloaded by the user as a Web plugin. The Web plugin process has been criticized because of the size of the Java plugin. Unlike other plugins, the Java download is a full runtime environment capable of running not just applets, but full applications and dynamic WebStart apps. Because of this, the perceived download footprint is larger than some web plugins. However, compared to Java, other popular browser plugins have larger sizes: Java 6 <a href="/wiki/Java_Runtime_Environment" class="mw-redirect" title="Java Runtime Environment">JRE</a> is 13 MB, but <a href="/wiki/Adobe_Acrobat" title="Adobe Acrobat">Acrobat Reader</a> is 33 MB, <a href="/wiki/QuickTime" title="QuickTime">QuickTime</a> 19 MB, <a href="/wiki/Windows_Media_Player" title="Windows Media Player">Windows Media Player</a> 25 MB, the <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework 3.0</a> runtime is 54 MB, and the .NET Framework 3.5 runtime is 197 MB (it's a united package for x86, x64 and IA-64, each part has approximately 60 MB).</p> 
      <p>At the <a href="/wiki/JavaOne" title="JavaOne">JavaOne</a> event in May 2007 Sun announced that the deployment issues with Java would be solved in two major updates during the lifespan of Java 6 (the changes will not be held over to Java 7.) These include:</p> 
      <ul> 
       <li>The introduction of a new consumer JRE edition, with an initial 2&nbsp;Mb footprint and the ability to download the remaining 9&nbsp;Mb in sections using an on-demand methodology.</li> 
       <li>The development of drop-in cross platform <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> code, which can be used from a Web page to install the necessary JRE for a given applet or <a href="/wiki/Rich_Internet_Application" class="mw-redirect" title="Rich Internet Application">Rich Internet Application</a> to run, if necessary.</li> 
       <li>An improvement in support for automatically downloading updates to the JRE.</li> 
       <li>support for pre-loading of the JRE, so applets and applications written in Java start up almost instantaneously.</li> 
      </ul>
