     <h1>Building a Sentiment Analysis Bot with IBM Watson and Raspberry Pi</h1> 
     <div class="api_doc api_articles page_watson-sentiment card"> 
      <div class="article"> 
       <span class="subtle_silver">By Tomomi Imura</span> 
       <span class="float_right article_tag"><a href="/tutorials/tags/node.js">Node.js</a></span> 
       <span class="float_right article_tag"><a href="/tutorials/tags/events-api">Events API</a></span> 
       <span class="float_right article_tag"><a href="/tutorials/tags/ibm-watson">IBM Watson</a></span> 
       <span class="float_right article_tag"><a href="/tutorials/tags/cool-tricks">Cool tricks</a></span> 
       <hr> 
       <p>Do you ever wonder how your teammates perceive your Slack chat messages? Maybe you sound heroic! Or unpleasant. Perhaps you project harsh undertones you haven’t even noticed!</p> 
       <p>Imagine a helpful robot that analyzes your written communications. Now-- stop imagining, because I created this (<em>physical!</em>) bot that analyzes message tone using IBM Watson, “reading” the emotion of each posted message!</p> 
       <p>In this article, I’ll share how I created the bot.</p> 
       <iframe src="https://player.vimeo.com/video/219731310" width="640" height="360" frameborder="0" webkitallowfullscreen mozallowfullscreen allowfullscreen></iframe> 
       <p><small><a href="https://vimeo.com/219731310">Slack Sentiment Analysis Bot</a> from <a href="https://vimeo.com/user2524947">Tomomi Imura</a> on <a href="https://vimeo.com">Vimeo</a>.</small></p> 
       <p>First I’ll show you how to use the <a href="https://api.slack.com/events-api"><strong>Events API</strong></a> to capture messages, send them to IBM Watson for analysis, and then post the result using <a href="https://api.slack.com/methods/chat.postMessage"><code>chat.postMessage</code></a>.</p> 
       <p>Then I will present a totally optional but <strong>fun</strong> exercise! We will port our bot to a Raspberry Pi and reflect emotions using colors produced by an RGB LED.</p> 
       <p>?? <a href="https://github.com/girliemac/slack-sentiment">The source code</a> is available on GitHub.</p> 
       <a name="before_starting_this_project"></a>
       <h3>Before starting this project</h3> 
       <p>This project uses Node.js, so make sure <a href="https://nodejs.org/">Node.js</a> is installed on your machine. You’ll also need an <a href="https://console.ng.bluemix.net">IBM Bluemix account</a> and <a href="https://console.ng.bluemix.net/services/tone_analyzer/cbe7a324-0794-46d3-a6be-db4e58604273/?paneId=manage">add the Watson Tone Analyzer</a>. Your service credentials will be required later.</p> 
       <p>In addition to the software bot, if you want to build a physical Raspberry Pi bot that shows the results with color LEDs, you’ll need some affordable hack-friendly hardware. You can either buy the IBM <a href="https://ibmtjbot.github.io/">TJBot Kit</a> that includes the cardboard robot, or prepare separately:</p> 
       <ul> 
        <li>1 <a href="https://smile.amazon.com/Raspberry-Pi-RASPBERRYPI3-MODB-1GB-Model-Motherboard/dp/B01CD5VC92/">Raspberry Pi 3</a> (If you are using Pi 2, you need a WiFi dongle too)</li> 
        <li>1 <a href="https://www.adafruit.com/products/1734">NeoPixel RGB LED</a></li> 
        <li>3 Female/female jumper wires</li> 
       </ul> 
       <p>You need to install the latest <a href="https://www.raspberrypi.org/downloads/raspbian/">Raspbian OS</a>, connect to WiFi, and update and upgrade the system. See <a href="https://www.raspberrypi.org/downloads/raspbian/">Raspberrypi.org</a> for additional instruction.</p> 
       <a name="building_a_sentiment_analysis_bot"></a>
       <h2>Building a sentiment analysis bot</h2> 
       <p>OK, let’s build a Slack bot that reads messages and analyzes for emotional content!</p> 
       <p>Our bot’s workflow:</p> 
       <ol> 
        <li>Read each message on a channel</li> 
        <li>Send the message to IBM Watson for examination</li> 
        <li>If the likelihood of an emotion is above the given confidence threshold, post the most prominent emotion to the Slack channel</li> 
       </ol> 
       <a name="creating_and_configuring_your_slack_app"></a>
       <h3>Creating and configuring your Slack app</h3> 
       <p>First, let’s create a new app at <a href="https://api.slack.com/apps?new_app=1">https://api.slack.com/apps?new_app=1</a>. Fill out the (1) <strong>App Name</strong> and (2) choose a slack team that you are allowed to develop apps on:</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-create-app.png" alt="Create an App"></p> 
       <p>Click <strong>Create App</strong> when finished. Next, you need to configure your App.</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-basic-info.gif" alt="App Basic Info"></p> 
       <p>At <strong>Add features and functionality</strong>, enable “Bots”, “Event Subscription”, and “Permissions”. We will walk through them soon.</p> 
       <p>Also, you will need your API credentials you can find at the <strong>App Credentials</strong> section when you code. They identify your app to the Slack platform.</p> 
       <a name="setting_up_a_bot_user"></a>
       <h4>Setting up a bot user</h4> 
       <p>Click the <strong>Bots</strong> button under <strong>Add features and functionality,</strong> or the (1) <strong>Bot Users</strong> menu from the left navigation to set up a bot user. Then (2) give your bot a username you want:</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-botuser.png" alt="App - Bot Users"></p> 
       <p>Make sure to click the <strong>Save Changes</strong> every time you make a change in this configuration process.</p> 
       <a name="setting_oauth__amp__permissions"></a>
       <h4>Setting OAuth &amp; permissions</h4> 
       <p>Click <strong>OAuth &amp; Permissions</strong> from the left menu.</p> 
       <p>You will need a bot user token generated when installing your app, beginning with <code>xoxb-</code> during development. <em>(Note: To distribute your bot to the rest of the world, you will need to set up an OAuth button, etc, however, this tutorial does not cover the process.)</em></p> 
       <p>Scroll down to <strong>Permission Scopes</strong> section to add the scopes you need to use for the bot. At the pulldown menu, choose <code>bot</code>:</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-permission-scopes.png" alt="App - Permission"></p> 
       <a name="setting_event_subscriptions"></a>
       <h4>Setting event subscriptions</h4> 
       <p>To receive channel messages, we are going to use the <a href="https://api.slack.com/events-api">Events API</a>. Click <strong>Event Subscriptions</strong>. You need to enable the feature by setting the <strong>Request URL</strong>, but let’s skip this for now.</p> 
       <p>Scroll down to <strong>Subscribe to Bot Events.</strong> Click the <strong>Add Bot User Event button</strong>, and select <em>message.channels:</em></p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-botuser-event.png" alt="App - Event"></p> 
       <p>Next, you will need to write an webhook scaffold with Node.js, and come back to the page for the <strong>Request URL</strong>, and finish the rest of the App configuration.</p> 
       <p>The Request URL used by the Events API is a kind of webhook, where the URL you provide will receive a HTTP request for each posted message.</p> 
       <a name="setting_up_your_request_url"></a>
       <h3>Setting up your Request URL</h3> 
       <p>During the development, I am using a temporary URL from <a href="https://ngrok.com/">ngrok</a>, which serves my localhost to a public URL. <a href="https://ngrok.com/download">Download ngrok</a> for your OS, unzip it, and install it by moving the file to the appropriate directory, for example, if you are using Mac, use the command, <code>mv ngrok /usr/local/bin</code>. You can learn more about ngrok at <a href="https://api.slack.com/tutorials/tunneling-with-ngrok">Using ngrok to develop locally for Slack</a>.</p> 
       <p>Once you install it, run it on terminal:</p> 
       <pre><code class="bash">$ ngrok http 5000
</code></pre> 
       <p>The ngrok tool generates a URL looking like <code>http://618830b2.ngrok.io</code> for your local server at <code>localhost:5000</code>. Copy the ngrok URL and paste it into the <strong>configuration</strong> setup, but <em>not so fast</em>! The URL must be validated, and in order to do that you’ll need to finally write some code. We will come back here later.</p> 
       <a name="running_an_express_server"></a>
       <h4>Running an Express server</h4> 
       <p>Express is a simple, modular web framework for node.js. We’ll use it to quickly create our Request URL webhook receiver.</p> 
       <p>First, set up your node application:</p> 
       <pre><code class="bash">$ npm init
</code></pre> 
       <p>And configure your application. Then install Express.JS and body-parser (for POST):</p> 
       <pre><code class="bash">$ npm install express body-parser --save
</code></pre> 
       <p>Once finished installing these modules, create a <strong>index.js</strong> file, and instantiate the express server, listening on port 5000. Since you’ve set ngrok to <code>localhost:5000</code>, you must use the same port!</p> 
       <pre><code class="javascript">'use strict';
const express = require('express');
const bodyParser = require('body-parser');
const app = express();
app.use(bodyParser.json());
app.use(bodyParser.urlencoded({ extended: true }));

const server = app.listen(5000, () =&gt; {  
  console.log('Express server listening on port %d in %s mode', server.address().port, app.settings.env);});
</code></pre> 
       <p>Now, create HTTP POST route to handle the validation code. Let’s create an <code>/event</code> route. This will be triggered every time an event arrives, like the <code>message.channels</code> event we want from public Slack channels.</p> 
       <p>When you enter the webhook URL on your configuration page, Slack will send a request with a <code>challenge</code> parameter, so your endpoint must respond with the challenge value to complete the handshake.</p> 
       <pre><code class="javascript">app.post('/event', (req, res) =&gt; {
  if (req.body.type === 'url_verification') {
    res.send(q.challenge);
  }

  // will implement the bot here ...
});
</code></pre> 
       <p>Now, let’s run this node script.</p> 
       <p>Then (1) go back to the <strong>Event Subscriptions</strong>, and (2) enter your ngrok URL with the route, for instance, <code>https://618830b2.ngrok.io/event</code>. If everything works fine, the Request URL will verify, and you should see a green checkmark like this:</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/slack-event.png" alt="App - Events"></p> 
       <a name="handling_event_subscriptions"></a>
       <h3>Handling event subscriptions</h3> 
       <p>Let’s start coding your bot!</p> 
       <p>Go back to your <strong>index.js</strong> file and at the <code>/events</code> POST route you defined, we’ll receive and handle the message payload.</p> 
       <p>Within the request handler, you need to check the payload to see (1) if the <code>token</code> value matches with your verification token (which you can find at the <em>App Credentials</em> section of the <strong>Basic Information</strong>), and (2) if the <code>type</code> value is the <code>event_callback</code>, extract the message text to be analyzed by Watson later:</p> 
       <pre><code class="javascript">app.post('/events', (req, res) =&gt; {
  let q = req.body;
  // 1. To see if the request is coming from Slack
  if (q.token !== process.env.SLACK_VERIFICATION_TOKEN) {
    res.sendStatus(400);
    return;
  }
  // 2. Events - get the message text
  else if (q.type === 'event_callback') {
    if(!q.event.text) return;
    analyzeTone(q.event); // sentiment analysis
  }
});
</code></pre> 
       <p><em>If you just want to run the code locally, you may hard-code your token here. There are multiple ways to set your environment variables, but I usually set a .env file to include the variables. In the source code, you can take a look at the <code>.env-test</code> file to see how it is set.</em></p> 
       <p>Now, create a function, <code>analyzeTone</code>, which take the message text to be analyzed and some other info from the payload.</p> 
       <a name="using_the_sentiment_analysis_with_watson"></a>
       <h3>Using the sentiment analysis with Watson</h3> 
       <p>Now, let’s use IBM Watson for the sentiment analysis, so get your Service credentials from your IBM Bluemix account ready!</p> 
       <p>Go to your Bluemix console, and choose <strong>Tone Analyzer</strong> from the catalog under Watson.</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/ibm-bluemix-catalog.png" alt="Bluemix"></p> 
       <p>At the next screen, click <strong>Create</strong>.</p> 
       <p>Once you activate the Tone Analyzer, (1) click the <strong>Service Credentials</strong> from the left menu, and you should be able to obtain your username and password for this specific service by (2) clicking the little triangle by “View credentials”.</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/ibm-bluemix-credentials.png" alt="Bluemix Tone Analyzer"></p> 
       <p>To use the Watson Node.js library, install the <code>watson-developer-cloud</code> module from npm:</p> 
       <pre><code class="bash">$ npm install watson-developer-cloud --save
</code></pre> 
       <p>Then instantiate it with your credentials:</p> 
       <pre><code class="javascript">const watson = require('watson-developer-cloud');
let tone_analyzer = watson.tone_analyzer({
    username: process.env.WATSON_TONE_ANALYSIS_USERNAME,
    password: process.env.WATSON_TONE_ANALYSIS_PASSWORD,
    version: 'v3',
    version_date: '2016-05-19'
  });
</code></pre> 
       <p>Now, you sent the Slack message text to the Tone Analysis:</p> 
       <pre><code class="javascript">const confidencethreshold = 0.55;
tone_analyzer.tone({text: text}, (err, tone) =&gt; {
    tone.document_tone.tone_categories.forEach((tonecategory) =&gt; {
      if(tonecategory.category_id === 'emotion_tone'){
        console.log(tonecategory.tones);
        tonecategory.tones.forEach((emotion) =&gt; {
          if(emotion.score &gt;= confidencethreshold) {
            postEmotion(emotion, ev)
          }
        })
      }
    })
  });
</code></pre> 
       <p>Watson can analyze a lot more from a given text, but for this project we are only using the emotion analysis. The results will be returned as a JSON that provides a hierarchical representation of the analysis. Each emotion has an index, so define the confidence threshold (let’s set it 0.55 for now. You can adjust it later!) and use only the emotion the exceeded the threshold value! Learn more about the <code>watson-developer-cloud</code> module on <a href="https://github.com/watson-developer-cloud/node-sdk">the GitHub repo</a>.</p> 
       <a name="posting_a_sentiment_back_on_a_channel"></a>
       <h3>Posting a sentiment back on a channel</h3> 
       <p>Now, let’s create a <code>postEmotion</code> function and post the result back on the Slack channel! To post a message, use <a href="https://api.slack.com/methods/chat.postMessage"><code>chat.postMessage</code></a> API. This call requires values for the <code>token</code>, <code>channel</code>, and <code>text</code>. The token should come from an authentication of a user with the OAuth, however, during the development, use the token that begins with <code>xoxb-</code> that you can find at the <strong>OAuth &amp; Permissions</strong> section.</p> 
       <pre><code class="javascript">function postEmotion(emotion, ev) {
  let message = 'feeling ' + emotion.tone_id;
  let options = {
    method: 'POST',
    uri: 'https://slack.com/api/chat.postMessage',
    form: {
      token: 'xoxb-.....', // Your Slack OAuth token
      channel: ev.channel,
      text: message,
      as_user: false,
      username: 'Watson Bot'
    }
  };
  // Use Request module to POST
  request(options, (error, response, body) =&gt; {
    if (error) {
      console.log(error)
    }
  });
}
</code></pre> 
       <p>Yipee, your bot should be working now! Let’s run your node code. Add the “sentiment_analysis” bot to your Slack channel, and try posting some emotional messages!</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/giphy-bot-demo.gif" alt="Bot demo GIF"></p> 
       <p>The code samples used here are simplified to fit in the tutorial, however, if you wish to include the teammates’ usernames in the result messages as seen in this screenshot, view <a href="https://github.com/girliemac/slack-sentiment">the entire source code</a> on the GitHub repo!</p> 
       <p>You can end the project right here, or proceed this tutorial for more fun with hardware!</p> 
       <a name="making_the_bot_into_a_physical_bot_with_raspberry_pi"></a>
       <h2>Making the bot into a physical bot with Raspberry Pi!</h2> 
       <p>More bots are more fun! Let’s port this code to Raspberry Pi to make it as a physical bot!</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/tjbot.jpg" alt="TJ Bot"></p> 
       <a name="installing_node.js_on_pi"></a>
       <h3>Installing Node.js on Pi</h3> 
       <p>You need to boot up your Pi with Raspbian, connect to Internet, and upgrade the system. If you are new to Raspberry Pi, I recommend to learn how to set up on <a href="https://www.raspberrypi.org/documentation/">RaspberryPi.org</a>!</p> 
       <p>Then you can either directly work on the terminal on Pi, or SSH into the Pi from your computer, and install Node.js for Linux ARM:</p> 
       <pre><code class="bash">$ curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
$ sudo apt-get install -y nodejs
</code></pre> 
       <a name="installing_ngrok_on_pi"></a>
       <h3>Installing ngrok on Pi</h3> 
       <p>A nice thing about ngrok is that it supports ARM, so you can use in on a Raspberry Pi too!</p> 
       <pre><code class="bash">$ cd /tmp
$ wget https://bin.equinox.io/c/4VmDzA7iaHb/ngrok-stable-linux-arm.zip
$ unzip ngrok*
$ sudo mv ngrok /usr/bin/
</code></pre> 
       <p><a href="https://www.raspberrypi.org/documentation/remote-access/ssh/">SSH</a> into the Pi to copy your <strong>index.js</strong> and all the module dependencies over to the Pi from your computer. Run the index.js, and now you should be able to tunnel requests to your Pi, just like you did before on more expensive computer:</p> 
       <pre><code class="bash">$ ngrok http 5000
</code></pre> 
       <p>This gives you a new ngrok forwarding URL, so don’t forget to update the <em>Request URL</em> at the <strong>Event Subscription</strong> on your Slack App setting page!</p> 
       <p>Try posting some message on Slack and see if it works.</p> 
       <p>Now, let’s work on hardware!</p> 
       <a name="wiring_up_the_pi_and_an_led"></a>
       <h3>Wiring up the Pi and an LED</h3> 
       <p>Follow the diagram below and wire a <a href="https://www.adafruit.com/products/1734">Neopixel RGB LED</a>. Find a flat side of the LED, and the leg closest to the flat side is a data-out (no wire), then ground, power, and data. Connect each leg to a corresponding pin:</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/pi-neopixelled-800.png" alt="Raspberry Pi wiring"></p> 
       <a name="light_up_an_emotion"></a>
       <h4>Light up an Emotion!</h4> 
       <p>Now you programmatically light the LED up, in a color depending on the emotion. First, install the <code>rpi-ws281x-native</code> via npm:</p> 
       <pre><code class="bash">$ npm install rpi-ws281x-native --save
</code></pre> 
       <p>Then write the code to turn the LED on. Let’s define the colors too:</p> 
       <pre><code class="javascript">const ws281x = require('rpi-ws281x-native');
const NUM_LEDS = 1;
ws281x.init(NUM_LEDS);
const color = new Uint32Array(NUM_LEDS);
process.on('SIGINT', () =&gt; {
  ws281x.reset();
  process.nextTick(() =&gt; { process.exit(0); });
});
const red = 0x00ff00;
const green = 0xff0000;
const blue = 0x0000ff;
const yellow = 0xffff00;
const purple = 0x00ffff;

// Show a specific color for each emotion
function colorEmotion(emotion) {
  if (emotion.tone_id === 'anger') {
    setLED(red);
  } else if(emotion.tone_id === 'joy') {
    setLED(yellow);
  } else if(emotion.tone_id === 'fear') {
    setLED(purple);
  } else if(emotion.tone_id === 'disgust') {
    setLED(green);
  } else if(emotion.tone_id === 'sadness') {
    setLED(blue);
  }
}
// Set the LED to the given color value
function setLED(colorval){
  color[0] = colorval ;
  ws281x.render(color);
}
</code></pre> 
       <p>Now, call the <code>colorEmotion</code> function at where you are calling the <code>postEmotion()</code>.</p> 
       <p>Run the Node code, type some message on Slack and see the LED lights up to match your feelings. Hooray!</p> 
       <p><img src="https://a.slack-edge.com/436da/img/api/articles/hooray.png" alt="Hooray!"></p> 
       <p>I hope you enjoyed the tutorial!</p>
