         <h1><a href="#translations-for-the-meteor-standard-packages-almost-i18n" aria-hidden="true" class="anchor" id="user-content-translations-for-the-meteor-standard-packages-almost-i18n">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Translations for the meteor standard packages (almost i18n)</h1> 
         <p>This package offers translations for accounts-base, accounts-passwords, accounts-entry, accounts-templates-core and billing. Contributions for other packages are welcome. We try to translate only messages that might pop up at a users screen as developers are expected to understand English errors anyway.</p> 
         <p>Translations are currently available for Arabic, Catalan, Chinese (Mandarin), Chinese (Traditional), Chinese (Hong Kong), Croatian, Czech, Danish, Dutch, Estoninan, Farsi, Finish, French, French (Canada), German, Greek, Hebrew, Hungarian, Indonesian, Italian, Japanese, Khmer, Korean, Norwegian, Polish, Portuguese (Brasil), Portugues (Portugal), Romanian, Russian, Slovak, Slovenian, Spanish (South America), Spanish (Spain), Swedish, Thai, Turkish, Ukrainian and Vietnamese.</p> 
         <h2><a href="#version-200" aria-hidden="true" class="anchor" id="user-content-version-200">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Version 2.0.0</h2> 
         <p>The second generation of meteor-accounts-t9n is able to run with <code>npm</code> alone. This means it can even be used with plain Node.js applications, not only within Meteor, see in <code>./examples/npm</code> how this could work.</p> 
         <p>When using <code>npm</code> the translation files will no longer be eagerly loaded. Several people asked for this feature to safe bandwidth and it is possible now, this closes #161.</p> 
         <p>To make it work with <code>npm</code> the files have been translated to JavaScript, so if you use CoffeeScript you can require from the main directory and if you use JavaScript you would import/require from <code>build</code>.</p> 
         <p>These <code>examples</code> show different use cases:</p> 
         <ul> 
          <li>meteorPackage: This is way it has been used in former versions as a meteor package.</li> 
          <li>meteorNpm: Use t9n as npm package within Meteor.</li> 
         </ul>

         <p><code>meteor add softwarerero:accounts-t9n</code></p> 
         <p>Off course one can easily add translations for applications. By putting a file like <code>myapp.es</code> in a folder under <code>lib</code> translations can be used on the client and on the server. Example:</p>

         <h2><a href="#get-a-localized-text-in-a-template" aria-hidden="true" class="anchor" id="user-content-get-a-localized-text-in-a-template">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Get a localized text in a template</h2> 
         <p><code>{{t9n code}}</code> <code>{{t9n code parameter1='value'}}</code></p> 
         <p>Example: <code>{{t9n "store.purchase"}}</code>.</p> 
         <p>If a translation is not found the key is displayed. To spot not translated keys a prefix and a postfix can surround the key, they default to "&gt;" and "&lt;" so a you would see "&gt;nonExistantKey&lt;". You can change the pre- and postfix:</p> 
         <p><code>T9n.missingPrefix = "&gt;"</code> <code>T9n.missingPostfix = "&lt;"</code></p> 
         <p>If you use <code>get</code> you can also suppress printing of the prefix and postfix if you set the second parameter to false (it defaults to true). <code>T9n.get code, false</code></p> 
         <h2><a href="#get-a-localized-text-with-parameters" aria-hidden="true" class="anchor" id="user-content-get-a-localized-text-with-parameters">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Get a localized text with parameters</h2> 
         <p>Optionally named parameters can be used, naming them allows for repetition.</p> 
         <p><code>T9n.get code, true, args</code></p> 
         <p>Example:</p> 
         <p>If you define a string in your language file like</p> 
         <pre><code>'pun': '@{subject} @{predicate} @{adverb} @{object}. Frische @{object} @{predicate} @{subject}.'
</code></pre> 
         <p>and have an object like</p> 
         <pre><code>args =
  subject: "Fischer's Fritz"
  predicate: 'fischt'
  object: 'Fische'
  adverb: 'frische'
</code></pre> 
         <p>you could call</p> 
         <pre><code>T9n.get 'pun', true, args
</code></pre> 
         <p>and that should give you</p> 
         <pre><code>'Fischer's Fritz fischt frische Fische. Frische Fische fischt Fischer's Fritz.'
</code></pre> 
         <p>You must specify the second argument for prefix/postfix too, I am sorry.</p> 
         <h2><a href="#define-translations" aria-hidden="true" class="anchor" id="user-content-define-translations">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Define translations</h2> 
         <p><code>T9n.map language, yourMap</code></p> 
         <p>Example:</p> 
         <pre><code>T9n.map 'en',
  hello: 'world'
  store:
    purchase: 'buy now'
    basket: 'basket'
</code></pre> 
         <p>Tip: If you do not want to expose the reason why a login was unsuccessful for security reasons. They could overwrite the corresponding messages:</p> 
         <pre><code>T9n.map 'en',
  error:
    accounts:
      'User not found': 'Not for you'
      'Incorrect password': 'Not for you'
</code></pre>

T9n.map 'en_GB', 'error.accounts.Incorrect password': 'You might want to enter a more correct version of your password'

           </svg></a>Send only some language files to the client (new in version 1.1.0)</h1> 
         <p>If you have a need to reduce bandwidth you can specify which languages to send to the client by setting an environment variable like <code>T9N_LANGUAGES='es,de'</code>. Now only Spanish and German should go over the wire instead of all translations. With the current package manager this does not work anymore. If you need that functionality delete ~/.meteor/packages/softwarerero_accounts-t9n and instead copy it directly into your project (you can go to myproject/packages and issue git clone <a href="https://github.com/softwarerero/meteor-accounts-t9n.git">https://github.com/softwarerero/meteor-accounts-t9n.git</a>). Now it should work again.</p>
