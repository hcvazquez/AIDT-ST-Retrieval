                  <h2> General instructions for a non-HubSpot hosted website </h2> 
                  <p><span>If you don't use a content management system, like WordPress or Joomla, follow these steps to install the tracking code on your website.</span></p> 
                  <div class="article-img-wrap bottom "> 
                  </div> 
                 </div>
                </div> 
                <div id="hs_cos_wrapper_widget_1449076699657" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_custom_widget" style="" data-hs-cos-general-type="widget" data-hs-cos-type="custom_widget">
                 <div class="alert step thin">
                  <span>Remember that if your website is hosted on the HubSpot COS or CMS that you do not need to follow these steps to install the tracking code. The tracking code is installed on all pages you create in HubSpot.</span>
                 </div>
                </div>

                 <div class="step"> 
                  <h2> 1. Assign this task to someone on your team </h2> 
                  <ul> 
                   <li><span>If you </span><span><strong>DO</strong></span><span> have access to the HTML of your website (or to your content management system) you can complete the steps below yourself.</span></li> 
                   <li><span>If you </span><span><strong>DO NOT</strong></span><span> have access to your website's HTML, email the URL of this article to your IT department or web developer.&nbsp;</span></li> 
                  </ul>

                 <div class="alert step thin">
                  <span>Whoever is installing the tracking code will have to log in to HubSpot to complete the steps, so you need to create a HubSpot user account for them. For a refresher on creating a new HubSpot user account, </span>

                 <div class="step"> 
                  <h2> How to install the tracking code on a WordPress website </h2> 
                  <p><span>To install the HubSpot tracking code on a WordPress site, install the </span><strong>HubSpot Tracking Code for WordPress&nbsp;</strong><span>plugin, within your WordPress instance.</span></p> 
                  <div class="article-img-wrap bottom "> 
                  </div> 
                 </div>
                </div> 
                <div id="hs_cos_wrapper_widget_3520693964" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_custom_widget" style="" data-hs-cos-general-type="widget" data-hs-cos-type="custom_widget"> 
                 <div class="substep"> 
                  <h3>What does the plugin do?</h3> 
                  <ul> 
                   <li><span>Installs the HubSpot tracking code on your WordPress site.</span></li> 
                   <li><span>Sets content type for filtering between pages and posts in the Page Performance tool.</span></li> 
                   <li><span>Creates contacts with Lifecycle Stage of Subscriber when non-HubSpot forms are filled out on the site.</span></li> 
                  </ul> 
                  <div class="article-img-wrap bottom "> 
                  </div> 
                 </div>
                </div> 
                <div id="hs_cos_wrapper_widget_1449077617154" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_custom_widget" style="" data-hs-cos-general-type="widget" data-hs-cos-type="custom_widget">
                 <div class="alert step thin">
                  <span>The "</span>
                  <span>HubSpot Tracking Code for WordPress" plugin replaces the "HubSpot for WordPress" plugin. Unlike the older plugin, this plugin does not include various deprecated features and improves the reliability of the plugin.&nbsp;<a href="//help.hubspot.com/articles/KCS_Article/Forms/How-to-embed-a-form-or-CTA-in-a-Wordpress-post-or-page" target="_blank">Forms and CTAs should now be embeded directly in the WordPress editor</a>. WordPress no longer transmits meta information such as <span>publish date and comment count to Page Performance.&nbsp;</span></span>
                 </div>
                </div> 
                <div id="hs_cos_wrapper_widget_3520693969" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_custom_widget" style="" data-hs-cos-general-type="widget" data-hs-cos-type="custom_widget"> 
                 <div class="substep"> 
                  <h3>Requirements</h3> 
                  <ul> 
                   <li><span>Your website or blog must be using a WordPress.org install. The Tracking code plugin cannot be installed on WordPress.com sites, as they do not allow you to add plugins or Javascript.</span></li> 
                   <li><span>You must have WordPress version 3.0 or higher installed on your sever (the latest version is recommended).</span></li> 
                   <li><span>The&nbsp;</span><span><a href="http://codex.wordpress.org/Function_Reference/wp_footer">wp_footer function</a></span><span>&nbsp;must be included in your WordPress themes footer file (<strong>footer.php</strong>). This function is required for our tracking code to be installed on every page of your site.</span></li> 
                   <li><span>You must be a WordPress admin to be able to install plugins for your site. If you are not the admin, you can forward these instructions to the person who manages your WordPress install.</span></li> 
                  </ul> 
                  <div class="article-img-wrap bottom "> 
                  </div> 
                 </div>
                </div>
