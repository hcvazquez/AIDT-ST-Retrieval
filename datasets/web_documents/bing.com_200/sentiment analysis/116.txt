           <h1 name="51f0" id="51f0" class="graf graf--h3 graf-after--figure graf--title">Sentiment Analysis Tools Overview, Part&nbsp;2</h1>
           <p name="11c4" id="11c4" class="graf graf--p graf-after--h3"><em class="markup--em markup--p-em">by Olga Davydova</em></p>
           <p name="3539" id="3539" class="graf graf--p graf-after--p"><em class="markup--em markup--p-em">Information contained in this article has been mostly obtained from sentiment analysis tools’ official websites.</em></p>
           <p name="5f23" id="5f23" class="graf graf--p graf-after--p">The most common application of sentiment analysis is in consumer products and services reviews. The main task of sentiment analysis is to determine whether the text expresses a positive or a negative sentiment and to assign it a polarity value. The goal of this article is to review the most known sentiment analysis tools. We will cover lexicon-based analysis methods, rule-based analysis methods, and machine learning techniques.</p>
           <h3 name="2a27" id="2a27" class="graf graf--h3 graf-after--p">NLTK SentimentAnalyzer</h3>
           <p name="6536" id="6536" class="graf graf--p graf-after--h3"><a href="http://www.nltk.org/api/nltk.sentiment.html" data-href="http://www.nltk.org/api/nltk.sentiment.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">This Natural Language Toolkit (NLTK), a Sentiment Analysis tool</a>, is based on machine learning approaches. The module nltk.sentiment.util contains a special function: demo_liu_hu_lexicon(sentence, plot=False). The function counts the number of positive, negative and neutral words in the input and classifies them depending on which polarity is most frequently represented <a href="http://www.nltk.org/api/nltk.sentiment.html" data-href="http://www.nltk.org/api/nltk.sentiment.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[1]</a>. Words that do not appear in the lexicon are considered as neutral. This is an example of sentiment classification using Liu and Hu opinion lexicon. The Liu and Hu opinion lexicon is a list of positive and negative words.</p>
           <p name="eb72" id="eb72" class="graf graf--p graf-after--p">Many sentiment analysis tools rely on lists of words and phrases with positive and negative connotations. Many lists are already available. You can read about the most known lists <a href="https://medium.com/@datamonsters/sentiment-analysis-tools-overview-part-1-positive-and-negative-words-databases-ae35431a470c" data-href="https://medium.com/@datamonsters/sentiment-analysis-tools-overview-part-1-positive-and-negative-words-databases-ae35431a470c" class="markup--anchor markup--p-anchor" target="_blank">in the previous article</a>.</p>
           <p name="8457" id="8457" class="graf graf--p graf-after--p">Another useful function is demo_vader_instance(text). It returns polarity scores for a text using VADER approach.</p>

           <h3 name="ed5d" id="ed5d" class="graf graf--h3 graf-after--figure">Pattern</h3>
           <p name="f41f" id="f41f" class="graf graf--p graf-after--h3"><a href="http://www.clips.ua.ac.be/pages/pattern-en#sentiment" data-href="http://www.clips.ua.ac.be/pages/pattern-en#sentiment" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Pattern</a> is a web mining module for the Python programming language <a href="http://www.clips.ua.ac.be/pages/pattern-en#sentiment" data-href="http://www.clips.ua.ac.be/pages/pattern-en#sentiment" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[2]</a>. It has many tools for data mining including sentiment analysis tools. Input text that can be a string, text, sentence, chunk, word or a synset (a set of one or more synonyms), is divided into two types: facts and opinions. Opinions carry people’s sentiments. The module has several useful functions:</p>
           <p name="bca4" id="bca4" class="graf graf--p graf-after--p">The sentiment() function returns a (polarity, subjectivity) tuple, an ordered set of values, for the given sentence, based on the adjectives it contains, where polarity is a value between -1.0 and +1.0 and subjectivity between 0.0 and 1.0.</p>
           <p name="5797" id="5797" class="graf graf--p graf-after--p">The positive() function returns True if the given sentence’s polarity is above the threshold.</p>

           <h3 name="375f" id="375f" class="graf graf--h3 graf-after--figure">TextBlob</h3>
           <p name="2621" id="2621" class="graf graf--p graf-after--h3"><a href="http://textblob.readthedocs.io/en/dev/quickstart.html#sentiment-analysis" data-href="http://textblob.readthedocs.io/en/dev/quickstart.html#sentiment-analysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">TextBlob</a> is another text processing Python library. The sentiment property returns a named tuple of the form Sentiment (polarity, subjectivity). The polarity score is a float within the range [-1.0, 1.0]. The subjectivity is a float within the range [0.0, 1.0] where 0.0 is very objective and 1.0 is very subjective <a href="http://textblob.readthedocs.io/en/dev/quickstart.html#sentiment-analysis" data-href="http://textblob.readthedocs.io/en/dev/quickstart.html#sentiment-analysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[3]</a>.</p>

           <h3 name="abc3" id="abc3" class="graf graf--h3 graf-after--figure">Stanford CoreNLP</h3>
           <p name="3068" id="3068" class="graf graf--p graf-after--h3">Many sentiment analysis tools lose important information by using a lexicon-based approach. A new deep learning model used in <a href="https://nlp.stanford.edu/sentiment/index.html" data-href="https://nlp.stanford.edu/sentiment/index.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Stanford CoreNLP</a> makes it possible to compute the sentiment based on how individual words change the meaning of longer phrases. A new type of recursive neural network that builds on grammatical structures was created, and a <a href="https://nlp.stanford.edu/sentiment/treebank.html" data-href="https://nlp.stanford.edu/sentiment/treebank.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">sentiment treebank</a> was developed. The sentiment treebank includes fine-grained sentiment labels for 215,154 phrases in the parse trees of 11,855 sentences. To address them use the Recursive Neural Tensor Network <a href="https://nlp.stanford.edu/sentiment/index.html" data-href="https://nlp.stanford.edu/sentiment/index.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[4]</a>. The code is written in Java. It is possible to use Stanford CoreNLP from the command-line, via its Java programmatic API, via third party APIs for most major modern programming languages, or via a web service. It works on Linux, macOS, and Windows.</p>

           <h3 name="23a1" id="23a1" class="graf graf--h3 graf-after--figure">RapidMiner</h3>

It is possible to crowd-source your customers’ feedback, conduct social market research, determine the polarity of product reviews, and hold discussions around the web <a href="https://rapidminer.com/solutions/text-mining/" data-href="https://rapidminer.com/solutions/text-mining/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[5]</a>.            <h3 name="38aa" id="38aa" class="graf graf--h3 graf-after--figure">General Architecture for Text Engineering (GATE)</h3>
           <p name="2b2e" id="2b2e" class="graf graf--p graf-after--h3"><a href="https://gate.ac.uk/sentiment/" data-href="https://gate.ac.uk/sentiment/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">General Architecture for Text Engineering (GATE)</a> is a Java open-source, natural language processing tool developed at the University of Sheffield in 1995. It contains several sentiment analysis systems. One of them uses supervised machine learning methods, trained on human-annotated data, co-occurrence statistics, and lexicons of positive and negative words <a href="https://gate.ac.uk/sentiment/" data-href="https://gate.ac.uk/sentiment/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[6]</a>. Others use lightly-supervised and unsupervised approaches. Developers also consider richer user-sensitive models of opinions to reflect the fact that the same opinions could be positive for one user group and negative for another. Author authority models and temporal elements are used that make it possible to detect shifts in attitudes over time.</p>

           <h3 name="dfd7" id="dfd7" class="graf graf--h3 graf-after--figure">R (Package: Syuzhet)</h3>
           <p name="1873" id="1873" class="graf graf--p graf-after--h3"><a href="https://github.com/mjockers/syuzhet" data-href="https://github.com/mjockers/syuzhet" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Syuzhet</a> extracts sentiment and sentiment-derived plot arcs from text using four sentiment dictionaries <a href="https://github.com/mjockers/syuzhet" data-href="https://github.com/mjockers/syuzhet" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[7]</a>. Its get_sentiment function assesses the sentiment of each word or sentence. The function takes two arguments: a character vector of sentences or words and a method. The method determines which sentiment extraction methods to use. After the sentiment values are determined, it is possible to obtain a measure of the overall emotional valence in the text by summing the determined values <a href="https://cran.r-project.org/web/packages/syuzhet/vignettes/syuzhet-vignette.html" data-href="https://cran.r-project.org/web/packages/syuzhet/vignettes/syuzhet-vignette.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[8]</a>.</p>
           <h3 name="aded" id="aded" class="graf graf--h3 graf-after--p">R (Package: RSentiment)</h3>
           <p name="f174" id="f174" class="graf graf--p graf-after--h3"><a href="https://cran.r-project.org/web/packages/RSentiment/index.html" data-href="https://cran.r-project.org/web/packages/RSentiment/index.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">The RSentiment</a> package lets users analyze the sentiment of a sentence and assign a score to it. A special function calculate_sentiment predicts the sentiment of sentences. Sentences can be classified into six categories: positive, negative, very positive, very negative, sarcasm and neutral <a href="https://cran.r-project.org/web/packages/RSentiment/index.html" data-href="https://cran.r-project.org/web/packages/RSentiment/index.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[9]</a>.</p>
           <h3 name="ea8f" id="ea8f" class="graf graf--h3 graf-after--p">R (Package: Sentiment Analysis)</h3>
           <p name="5b3c" id="5b3c" class="graf graf--p graf-after--h3"><a href="https://github.com/sfeuerriegel/SentimentAnalysis" data-href="https://github.com/sfeuerriegel/SentimentAnalysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">SentimentAnalysis</a> uses various existing dictionaries of positive and negative words and can create customized dictionaries using the generateDictionary() function. The latter uses LASSO regularization as a statistical approach to select relevant terms based on an exogenous response variable <a href="https://github.com/sfeuerriegel/SentimentAnalysis" data-href="https://github.com/sfeuerriegel/SentimentAnalysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[10]</a>. The function analyzeSentiment() returns sentiment scores from contents stored in different formats. ConvertToBinary() and convertToDirection() functions convert the continuous scores to either binary sentiment classes (negative or positive) or tertiary directions (negative, neutral or positive). CompareToResponse() function performs a statistical evaluation, plotSentimentResponse()?—?enables a visual comparison.</p>

           <h3 name="9257" id="9257" class="graf graf--h3 graf-after--figure">Google Cloud Prediction API</h3>
           <p name="b8df" id="b8df" class="graf graf--p graf-after--h3"><a href="https://cloud.google.com/prediction/docs/sentiment_analysis" data-href="https://cloud.google.com/prediction/docs/sentiment_analysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Google Cloud Prediction API</a> provides a representational state transfer (RESTful) API to build machine learning models. It helps to analyze a text string and classify it with one of the labels that the user provides. Labels could be positive and negative, or happy, frustrated, or sad <a href="https://cloud.google.com/prediction/docs/sentiment_analysis" data-href="https://cloud.google.com/prediction/docs/sentiment_analysis" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[11]</a>. After collecting training samples, users need to pre-classify each sample with a label then upload the data to Google Cloud Storage. Use the prediction.trainedmodels.insert() method to train the model and make predictions with the Google Prediction API in your application. If needed, you can improve your model.</p>
           <h3 name="df64" id="df64" class="graf graf--h3 graf-after--p">Sentiment Classifier using Word Sense Disambiguation (WSD)</h3>
           <p name="6958" id="6958" class="graf graf--p graf-after--h3"><a href="https://github.com/kevincobain2000/sentiment_classifier" data-href="https://github.com/kevincobain2000/sentiment_classifier" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Sentiment Classifier</a> uses Word Sense Disambiguation (WSD) drawing on WordNet and word occurrence statistics from the movie review corpus NLTK. WSD determines which meaning of a word is used in a sentence if the word has more than one meaning. As features, bigrams on Naive Bayes and maximum entropy classifier are used. The sentiment analysis tool classifies input into positive or negative labels <a href="https://github.com/kevincobain2000/sentiment_classifier" data-href="https://github.com/kevincobain2000/sentiment_classifier" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[12]</a>.</p>

           <h3 name="36a2" id="36a2" class="graf graf--h3 graf-after--figure">IBM Watson Natural Language Understanding</h3>
           <p name="93f2" id="93f2" class="graf graf--p graf-after--h3"><a href="https://www.ibm.com/watson/developercloud/doc/natural-language-understanding/#sentiment" data-href="https://www.ibm.com/watson/developercloud/doc/natural-language-understanding/#sentiment" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Watson</a> enables data scientists to analyze the sentiment of a specific target phrases or the sentiment of the whole document. You can also obtain sentiment information for detected entities and keywords <a href="https://www.ibm.com/watson/developercloud/doc/natural-language-understanding/#sentiment" data-href="https://www.ibm.com/watson/developercloud/doc/natural-language-understanding/#sentiment" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">[13]</a>. A deeper analysis of the text permits the determination of a positive tone while a customer review may have an overall negative sentiment. The Sentiment object has a score property for the concept ranging from -1 to 1. Negative scores indicate negative sentiments, and positive scores indicate positive sentiments. The tool supports Arabic, English, French, German, Italian, Portuguese, Russian, and Spanish. It has Node SDK, Java SDK, Python SDK, iOS SDK, and Unity SDK.</p>

           <h3 name="dd27" id="dd27" class="graf graf--h3 graf-after--p">Conclusion</h3>
           <p name="9dfd" id="9dfd" class="graf graf--p graf-after--h3">Some popular sentiment analysis tools and methods were presented above. All these tools make it possible to determine the sentiment of texts, which can be positive or negative. A number of tools classify text into additional categories: very positive, very negative, sarcastic, neutral, happy, frustrated, and sad. We described several Python libraries including NLTK, Pattern, TextBlob, Sentiment Classifier, some R packages like Syuzhet, RSentiment, Sentiment Analysis, and other tools like IBM Watson Natural Language Understanding, Stanford CoreNLP, RapidMiner, GATE, Google Prediction API, SentiGrade and Sent??otion.</p>
