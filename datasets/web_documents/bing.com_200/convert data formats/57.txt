              <p>If you are developing javascript for SharePoint and can use its javascript libraries then it might be a good idea to be using standart approach which take care of all the rest</p> 
              <pre><code>var date = new Date();
var arg = { date: date };
var str = Sys.Serialization.JavaScriptSerializer.serialize(arg);// "{"date":"\/Date(1377757755979)\/"}"
var newObject = Sys.Serialization.JavaScriptSerializer.deserialize(str);
var newDate = newObject.date; // Date {Thu Aug 29 2013 10:29:15 GMT+0400}
</code></pre> 
              <p>The class is a part of <a href="http://msdn.microsoft.com/en-us/magazine/cc163300.aspx" rel="nofollow">Microsoft Ajax Library</a> so it is a good idea to use it whenever you want to communicate with standart Microsoft Web Services.</p>

              <p>As part of .Net Framework at least two options are available:</p> 
              <ul> 
               <li><code>Deserialize</code> method from <a href="http://msdn.microsoft.com/en-us/library/system.web.script.serialization.javascriptserializer.aspx" rel="nofollow">System.Web.Script.Serialization.JavaScriptSerializer</a></li> 
               <li><code>ReadObject</code> method from <a href="http://msdn.microsoft.com/en-us/library/system.runtime.serialization.json.datacontractjsonserializer.aspx" rel="nofollow">System.Runtime.Serialization.Json.DataContractJsonSerializer</a></li> 
              </ul> 
              <h2>Example</h2> 
              <pre><code>string dateString = @"""\/Date(1377683175000)\/""";
var serializer = new JavaScriptSerializer();
var date = serializer.Deserialize&lt;DateTime&gt;(dateString);  //returns {28.8.2013 9:46:15}
</code></pre>
