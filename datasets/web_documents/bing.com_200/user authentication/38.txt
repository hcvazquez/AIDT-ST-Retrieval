       </svg></a>ActiveDirectory for Node</h1> 
     <p class="package-description-redundant">ActiveDirectory is an ldapjs client for authN (authentication) and authZ (authorization) for Microsoft Active Directory with range retrieval support for large Active Directory installations. This code was a port of an existing C# library (not published) that I had written a few years ago. Here are the key features</p> 
     <ul> 
      <li>Authenticate</li> 
      <li>Authorization (via group membership information)</li> 
      <li>Nested groups support</li> 
      <li>Range specifier / retrieval support (http://msdn.microsoft.com/en-us/library/dd358433.aspx)</li> 
      <li>Automatic paging support (Active Directory results (MaxPageSize) limited to 1000 per request by default)</li> 
      <li>Recycle bin (tombstone) query support</li> 
      <li>Referral support</li> 
     </ul>

     <p>ActiveDirectory uses the following additional node modules:</p> 
     <ul> 
      <li><a href="http://underscorejs.org/">underscore</a> - a utility-belt library for JavaScript that provides a lot of the functional programming support</li> 
      <li><a href="https://github.com/caolan/async">async</a> - Async utilities for node and the browser</li> 
      <li><a href="http://ldapjs.org/">ldapjs</a> - A pure JavaScript, from-scratch framework for implementing LDAP clients and servers in Node.js</li> 
      <li><a href="https://github.com/trentm/node-bunyan">bunyan</a> - A simple and fast JSON logging module for node.js services</li> 
     </ul>

      <li>groupName - The group to check for membership. Can be a commonName (cn) or a distinguishedName (dn)</li>

     <a name="opts"> ### Optional Parameters / Extended Functionality <p>Any method which takes an 'opts' parameter allows for additional options. Options for both activedirectory.js and the internal ldapjs client are supported.</p> <p>Currently supported ldapjs opts are:</p> 
      <ul> 
       <li>url - a valid LDAP url.</li> 
       <li>host - the host name to connect to (used with port in lieu of url)</li> 
       <li>port - the port to connect to (used with hostname in lieu of url)</li> 
       <li>secure - indicates if ldaps:// vs ldap:// is used. (used with hostname/port in lieu of url)</li> 
       <li>tlsOptions - additrional tls options (see ldapjs for more information)</li> 
       <li>socketPath - If you're running an LDAP server over a Unix Domain Socket, use this.</li> 
       <li>log - You can optionally pass in a bunyan instance the client will use to acquire a logger. The client logs all messages at the trace level.</li> 
       <li>timeout - How long the client should let operations live for before timing out. Default is Infinity.</li> 
       <li>idleTimeout - How long the client should wait before timing out on TCP connections. Default is up to the OS.</li> 
       <li>bindDN - The DN all connections should be bound as.</li> 
       <li>bindCredentials - The credentials to use with bindDN.</li> 
       <li>scope - One of base, one, or sub. Defaults to base.</li> 
       <li>filter - A string version of an LDAP filter (see below), or a programatically constructed Filter object. Defaults to (objectclass=*).</li> 
       <li>attributes - attributes to select and return (if these are set, the server will return only these attributes). Defaults to the empty set, which means all attributes.</li> 
       <li>sizeLimit - the maximum number of entries to return. Defaults to 0 (unlimited).</li> 
       <li>timeLimit - the maximum amount of time the server should take in responding, in seconds. Defaults to 10. Lots of servers will ignore this.</li> 
      </ul> <p>Options for activedirectory.js:</p> </a>
     <ul>
      <a name="opts"> <li>baseDN - The alternative baseDN to use than the one specified in the ctor.</li> <li>includeMembership - Indicates that a search or find operation should enumerate the group memberships of the specified result types. Supported values are [ 'user', 'group', 'all' ].</li> <li>includeDeleted - Indicates that results should include tombstoned / deleted items. Please see <a href="#findDeletedObjects">findDeletedObject</a> for additional notes and caveats.</li></a> 
      <li>entryParser - Allows for a custom function to be specified for parsing of the resulting ldap object. Examples include augmenting ldap data with external data from an RDBMs. function onParse(entry, raw, callback) { callback(entry); } If null is returned, the result is excluded.</li> 
     </ul>
