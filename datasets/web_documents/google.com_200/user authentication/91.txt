   <h1><a name="kanchor6"></a>Logic - API Management</h1> 
   <p>The logic services feature in MobileFabric helps you import and integrate Node.js services (APIs) directly into MobileFabric for developing server-side and networking applications. You can now enhance the capability of your MobileFabric applications by adding custom services that are created within Node.js packages/projects. Node.js is an open-source and cross-platform runtime environment. Node.js applications are written in JavaScript, and can be run within the Node.js runtime on Mac OS, Microsoft Windows, and Linux. Node.js also provides a rich library of JavaScript modules that simplify the development of web applications using Node.js. Currently, MobileFabric supports Node.js V6.2.2 or above.</p> 
   <h2><a name="How"></a>How Node.js Works in MobileFabric</h2> 
   <ol> 
    <li value="1">Once you build a node.js package (in a .zip file), you need to import the package file into MobileFabric Console.</li> 
    <li value="2">Publish the Node.js package into the Node.js runtime server through the console. <p>Node.js services published from the package are available in the <b>MobileFabric Console &gt; Apps &gt; Configure Services &gt;&nbsp;Logic</b> tab.</p></li> 
    <li value="3"> You can then integrate/associate services created in Node.js packages to MobileFabric applications. </li> 
    <li value="4">Finally, the app needs to be published by selecting a Node.js environment.</li> 
   </ol> 
   <p class="Important" data-mc-autonum="<b><i><span style=&quot;color: #ff6600;&quot; class=&quot;mcFormatColor&quot;>Important: </span></i></b>"><span class="autonumber"><span><b><i><span style="color: #ff6600;" class="mcFormatColor">Important: </span></i></b></span></span> Ensure that your package details (files and folders) are zipped in the root of the .zip file.</p> 
   <h2><a name="Benefits"></a>Benefits of Node.js App Development</h2> 
   <p>The logic services feature offers the following advantages for quickly mobilizing Node.js services in MobileFabric for building apps:</p> 
   <ul> 
    <li value="1">Logic services exposes services created in Node.js helps you use custom Node.js services for building mobile apps.</li> 
    <li value="2">Integrate custom services (APIs) created in Node.js language directly into MobileFabric apps, and develop server-side and networking applications.</li> 
    <li value="3">Protect Node.js services (APIs) by setting security levels that control MobileFabric Identity services for authentication of apps users.</li> 
    <li value="4">Build scalable network applications using Node.js technology in MobileFabric apps.</li> 
    <li value="5">Node.js supports a non-blocking input/output model that makes it lightweight and efficient.</li> 
   </ul> 
   <h2><a name="Use"></a>Use Cases</h2> 
   <ul> 
    <li value="1">For <b>creating real-time web apps</b>: Node.js technology improves the performance and development of web apps.</li> 
    <li value="2"> <p>For <b>creating Web sites</b>: Node.js technology helps you share code between the browser and the back end.</p> </li> 
   </ul> 
   <h2><a name="NodeJS_Limits"></a>Limitations</h2> 
   <p>Read the following limitations for Node.js services before associating the services with MobileFabric apps.</p> 
   <ul> 
    <li value="1">Currently, one Node.js package for one user is allowed.</li> 
    <li value="2">Logic services tab support is available only for Kony Cloud.</li> 
   </ul> 
   <h2><a name="Prerequi"></a>Prerequisites</h2> 
   <ul> 
    <li value="1">Always use the <code class="codefirst">npm install save</code> option to install any modules.<p>For example, to add the package in dependencies, follow the commands:</p><pre>npm install --save my_dep</pre><p>or</p><pre>npm install -S my_dep</pre><p>The above commands ensure that all the dependencies are updated to <code class="codefirst">package.json</code> file.</p></li> 
    <li value="2">Ensure that your package details (files and folders) are zipped in the root of the .zip file.</li> 
    <li value="3">Ensure the port specified in swagger.json range starts from 9000 to 10000. </li> 
    <li value="4">Ensure that the port in swagger.json should match with the one specified in the application code.</li> 
   </ul> 
   <ul> 
    <li value="1"> Avoid the <code class="codefirst">npm_modules</code> folder in the final .zip file, which you want to upload to MobileFabric. The <code class="codefirst">npm_module</code> folder increases unnecessary package file size.<p class="Important" data-mc-autonum="<b><i><span style=&quot;color: #ff6600;&quot; class=&quot;mcFormatColor&quot;>Important: </span></i></b>"><span class="autonumber"><span><b><i><span style="color: #ff6600;" class="mcFormatColor">Important: </span></i></b></span></span>&nbsp;The user app should not depend on PORT environment variable. By default, if you are using <b>express-generator</b> to generate an express basic project, the <code class="file_names">bin/www</code> folder will have a PORT environment variable to refer to port value. In the <code class="file_names">bin/www</code> folder, use port number directly. <br><br>For example, <br></p><p class="code"><code class="codefirst">var port = normalizePort(process.env.PORT || '3000'); </code><br></p><p>will become <br><br><code class="codefirst">var port = normalizePort('9000');</code></p><p></p></li> 
   </ul> 
   <h2><a name="Logging"></a>Logging Service Support for User App</h2> 
   <p>Use the following is a sample code to enable logging in the user apps. So that user app logging statements can be viewed MobileFabric logging services.</p>
   <pre>let mflogger;

// Kony logger to capture application logs to display using logging service
try {
  mflogger = require(process.env.KONY_LOGIC_LOGGER);
} catch(e) {
  mflogger = console;
}  </pre> 
   <p>Use the <code class="codefirst">mflogger </code>instance for logging in the user app. Supported logging methods: <code class="codefirst">debug</code>, <code class="codefirst">info</code>, <code class="codefirst">warn</code>, and <code class="codefirst">error</code>.</p>
   <pre>mflogger.info('Message to log ...');</pre> 
   <h2><a name="NodeJSTroubleshooting"></a>Troubleshooting</h2> 
   <p><b>Unauthorized</b>: If you see the following message indicates that your endpoint resource is protected by MobileFabric identity services configured for your application. So you need to generate MobileFabric login token and provide it as part of header with name as <code class="codefirst">x-kony-Authorization</code> </p>
   <pre>{
    "message": "Authentication failed"
} </pre> 
   <p><b>Error</b>: If you see the following error, unpublish the app from API Management tab and publish the app. </p>
   <pre>{
    "error": "Connection error."
}  </pre> 
   <p>For technical questions, suggestions, and comments, or to report problems on Kony's product line, contact <a href="mailto:support@kony.com">support@kony.com</a>.</p> 
   <h2><a name="Node.js"></a>Node.js Package Structure</h2> 
   <p>A Node.js package contains the following folders and files.</p> 
   <p class="proc_subheading"><b>Swagger.json: </b> </p> 
   <ul> 
    <li value="1">The Swagger.json file includes all paths for services / APIs of the Node.js package.</li> 
    <li value="2">The <code class="codefirst">host</code> key should be present and hostname should be <b>localhost</b> and port value must be within <b>9000 - 10000.</b></li> 
    <li value="3">The <code class="codefirst">basePath</code>&nbsp;key should be present and value should with <code class="codefirst">/services/</code> and followed by user app base path.</li> 
    <li value="4">The <code class="codefirst">mfidentitylevel</code>, which is optional, is used to enable auth protection for the resources. Possible values are: <code class="codefirst">public</code>, <code class="codefirst">protected</code>, and <code class="codefirst">anonymous</code>.</li> 
   </ul> 
   <p>The following is a sample code for swagger.json. </p>
   <pre>{

   <p><b>Package.json: </b> </p> 
   <ul>

    <li value="2">The script <code class="codefirst">start</code> key must be present. MobileFabric logic services uses the start key to start the user app. The following is a sample code for package.json. </li>
