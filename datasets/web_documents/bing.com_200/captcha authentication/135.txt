       <h2>Introduction</h2> 
       <p>Authentication is one of the big parts of every application. Security is always something that is changing and evolving. In the past, we have gone over <a href="https://scotch.io/courses/easy-node-authentication">Node authentication</a> using the great <a href="http://passportjs.org/">Passport</a> npm package.</p> 
       <p>Those articles used the session based authentication however, which has problems when we talk about scaling web services and creating an API that can be consumed across many devices and services.</p> 
       <p>As a primer to this article, go ahead and get yourself familiar with <a href="https://scotch.io/bar-talk/the-ins-and-outs-of-token-based-authentication">token based authentication principles</a> and the standard used for token based authentication, <a href="https://scotch.io/tutorials/the-anatomy-of-a-json-web-token">JSON Web Tokens</a>.</p> 
       <p>Now that we've got all the important information about token based authentication out of the way, let's build a very simple Node API and use tokens to authenticate users that request access.</p> 
       <p><em>Note</em>: Updated to fix a security vulnerability. Only passing in a set payload when creating our JWT as pointed out by <a href="https://github.com/Mydayyy">Mydayyy</a> on <a href="https://github.com/scotch-io/node-token-authentication/pull/9">GitHub</a>.</p> 
       <h2>What We'll Be Building</h2> 
       <p>We'll build a quick API using Node and Express and we'll be using <a href="https://www.getpostman.com/">POSTman</a> to test it.</p> 
       <p>The main workflow of this is that we will:</p> 
       <ol> 
        <li>Have unprotected and protected routes</li> 
        <li>A user will authenticate by passing in a name and a password and get back a token</li> 
        <li>The user will store this token on their client-side and send it for every request</li> 
        <li>We will validate this token, and if all is good, pass back information in JSON format</li> 
       </ol> 
       <p>Our API will be built with:</p> 
       <ul> 
        <li>normal routes (not authenticated)</li> 
        <li>route middleware to authenticate the token</li> 
        <li>route to authenticate a user and password and get a token</li> 
        <li>authenticated routes to get all users</li> 
       </ul> 
       <p><strong>Further Reading</strong>: For some information on Express routing and middleware, check out our article on <a href="https://scotch.io/tutorials/javascript/learn-to-use-the-new-router-in-expressjs-4">Express 4.0 routing</a>.</p> 
       <h3>Tools Needed</h3> 
       <ul> 
        <li>node and npm</li> 
        <li><a href="https://www.getpostman.com/">POSTman</a></li> 
       </ul> 
       <p>We know what we're going to build, now let's get to it!</p> 
       <h2>Getting Started</h2> 
       <p>Let's take a look at our file structure for our Node application. This will be simplified and we'll be putting a lot of stuff into the <code>server.js</code> file.</p> 
       <h3>File Structure</h3> 
       <pre><code class="language-javascript">- app/
----- models/
---------- user.js
- config.js
- package.json
- server.js</code></pre> 
       <p>For a fully fledged application, you'd want to move some of this out of the main file and into separate files (particularly the routes).</p> 
       <h2>Set Up Our Node Application (package.json)</h2> 
       <p>First, we need to set up our <code>package.json</code> file. This is our beginning file for our Node application.</p> 
       <pre><code class="language-javascript">{
  "name": "node-token-jwt",
  "main": "server.js"
}</code></pre> 
       <p>Now that we have our <code>package.json</code> set, let's install our packages.</p> 
       <pre><code class="language-javascript">$ npm install express body-parser morgan mongoose jsonwebtoken --save</code></pre> 
       <ul> 
        <li><strong>express</strong> is the popular Node framework</li> 
        <li><strong>mongoose</strong> is how we interact with our MongoDB database</li> 
        <li><strong>morgan</strong> will log requests to the console so we can see what is happening</li> 
        <li><strong>body-parser</strong> will let us get parameters from our POST requests</li> 
        <li><strong>jsonwebtoken</strong> is how we create and verify our JSON Web Tokens</li> 
       </ul> 
       <p>The <code>--save</code> modifier will also save these packages to our <code>package.json</code> file. How convenient!</p> 
       <p>Next let's take care of a few files that we'll need for our project.</p> 
       <h3>User Model (app/models/user.js)</h3> 
       <p>The user model that we define will be used when creating and getting users. To create a Mongoose model, let's create the file <code>app/models/user.js</code></p> 
       <pre><code class="language-javascript">// get an instance of mongoose and mongoose.Schema
var mongoose = require('mongoose');
var Schema = mongoose.Schema;

// set up a mongoose model and pass it using module.exports
module.exports = mongoose.model('User', new Schema({ 
    name: String, 
    password: String, 
    admin: Boolean 
}));</code></pre> 
       <p>The other file we'll need to create is our <code>config.js</code> file. This is where we can store different variables and configuration for our application.</p> 
       <h3>Config File (config.js)</h3> 
       <p>For this file, <strong>you will need to create MongoDB database</strong>. You can either create one locally or easily use one online at <a href="https://modulus.io">modulus.io</a> for free. Either way, you will be able to get a URI string to use as your database configuration.</p> 
       <pre><code class="language-javascript">module.exports = {

    'secret': 'ilovescotchyscotch',
    'database': 'mongodb://noder:noderauth&amp;54;proximus.modulusmongo.net:27017/so9pojyN'

};</code></pre> 
       <ul> 
        <li><strong>secret</strong>: used when we create and verify JSON Web Tokens</li> 
        <li><strong>database</strong>: the URI with username and password to your MongoDB installation</li> 
       </ul> 
       <p>With all that out of the way, we can get to the big parts of our tutorial. We still haven't defined our main file (<code>server.js</code>), so let's get to that.</p> 
       <h2>The Actual Node Application (server.js)</h2> 
       <p>In this file, we will:</p> 
       <p><strong>Grab All the Packages</strong> This will include the packages we installed earlier (express, body-parser, morgan, mongoose, and jsonwebtoken) and also we'll be grabbing the model and config that we created.</p> 
       <p><strong>Configure Our Application</strong> We will set our important variables, configure our packages, and connect to our database here.</p> 
       <p><strong>Create Basic Routes</strong> These are the unprotected routes like the home page (<code>http://localhost:8080</code>). We'll also create a <strong>/setup</strong> route here so that we can create a sample user in our new database.</p> 
       <p><strong>Create API Routes</strong> This includes the following routes:</p> 
       <ul> 
        <li><code>POST http://localhost:8080/api/authenticate</code> Check name and password against the database and provide a token if authentication successful. This route will not require a token because this is where we get the token.</li> 
        <li><code>GET http://localhost:8080/api</code> Show a random message. This route is protected and will require a token.</li> 
        <li><code>GET http://localhost:8080/api/users</code> List all users. This route is protected and will require a token.</li> 
       </ul> 
       <p>With those things in our mind, let's start our <code>server.js</code> file:</p> 
       <pre><code class="language-javascript">// =======================
// get the packages we need ============
// =======================
var express     = require('express');
var app         = express();
var bodyParser  = require('body-parser');
var morgan      = require('morgan');
var mongoose    = require('mongoose');

var jwt    = require('jsonwebtoken'); // used to create, sign, and verify tokens
var config = require('./config'); // get our config file
var User   = require('./app/models/user'); // get our mongoose model

// =======================
// configuration =========
// =======================
var port = process.env.PORT || 8080; // used to create, sign, and verify tokens
mongoose.connect(config.database); // connect to database
app.set('superSecret', config.secret); // secret variable

// use body parser so we can get info from POST and/or URL parameters
app.use(bodyParser.urlencoded({ extended: false }));
app.use(bodyParser.json());

// use morgan to log requests to the console
app.use(morgan('dev'));

// =======================
// routes ================
// =======================
// basic route
app.get('/', function(req, res) {
    res.send('Hello! The API is at http://localhost:' + port + '/api');
});

// API ROUTES -------------------
// we'll get to these in a second

// =======================
// start the server ======
// =======================
app.listen(port);
console.log('Magic happens at http://localhost:' + port);
</code></pre> 
       <p>With that, we should be able to start up our Node server (make sure you have a valid database configured in <code>config.js</code>). Now if we start the server with:</p> 
       <pre><code class="language-javascript">$ node server.js</code></pre> 
       <p><strong>Tip:</strong> Use nodemon to have your server restart on file changes. Install nodemon using <code>npm install -g nodemon</code>. Then start your server with <code>nodemon server.js</code>.</p> 
       <p>We should be able to go to our browser and see the message from the route we created. Go to <a href="http://localhost:8080">http://localhost:8080</a> and you'll see:</p> 
       <div class="text-center"> 
        <a href="https://cask.scotch.io/2014/11/2014-11-13-2.png"><img src="https://cask.scotch.io/2014/11/2014-11-13-2.png" alt="2014-11-13 (2)" width="756" height="455" class="aligncenter size-full wp-image-5262 img-responsive img-thumbnail"></a> 
       </div> 
       <p>As a bonus, since we used morgan, we are able to see the request logged to our console, which helps with development.</p> 
       <div class="text-center"> 
        <a href="https://cask.scotch.io/2014/11/2014-11-13-4.png"><img src="https://cask.scotch.io/2014/11/2014-11-13-4.png" alt="2014-11-13 (4)" width="889" height="434" class="aligncenter size-full wp-image-5266 img-responsive img-thumbnail"></a> 
       </div> 
       <h2>Creating a Sample User</h2> 
       <p>Now we know our application is up and running! Let's create a sample user using the model that we created earlier.</p> 
       <p>This is a very simple process, we'll just create a quick route that will create a user of our choosing.</p> 
       <p>Here's that route. Just add it to the routes section of our <code>server.js</code>:</p> 
       <pre><code class="language-javascript">app.get('/setup', function(req, res) {

  // create a sample user
  var nick = new User({ 
    name: 'Nick Cerminara', 
    password: 'password',
    admin: true 
  });

  // save the sample user
  nick.save(function(err) {
    if (err) throw err;

    console.log('User saved successfully');
    res.json({ success: true });
  });
});</code></pre> 
       <div class="alert alert-info">
         It's important to note that we would **never** save a password to the database plain text like this. You would protect your passwords by hashing it. 
       </div> 
       <p>Go ahead and visit your application at <code>http://localhost:8080/setup</code>. You should see the message 'User saved successfully' logged to your console and the JSON object with <code>{ success: true }</code> in your browser.</p> 
       <div class="text-center"> 
        <a href="https://cask.scotch.io/2014/11/2014-11-13-3.png"><img src="https://cask.scotch.io/2014/11/2014-11-13-3.png" alt="2014-11-13 (3)" width="809" height="435" class="aligncenter size-full wp-image-5264 img-responsive img-thumbnail"></a>

       <p>We'll now create a route to get the users in our database and return them as JSON.</p> 
       <h2>Showing Our User</h2> 
       <p>Now let's create our API routes and create one to return all our users in JSON format. We'll use an instance of the Express router for this. We'll place a few placeholders so we can see where things will go. Here is the code for that:</p> 
       <pre><code class="language-javascript">// API ROUTES -------------------

// get an instance of the router for api routes
var apiRoutes = express.Router(); 

// TODO: route to authenticate a user (POST http://localhost:8080/api/authenticate)

// TODO: route middleware to verify a token

// route to show a random message (GET http://localhost:8080/api/)
apiRoutes.get('/', function(req, res) {
  res.json({ message: 'Welcome to the coolest API on earth!' });
});

// route to return all users (GET http://localhost:8080/api/users)
apiRoutes.get('/users', function(req, res) {
  User.find({}, function(err, users) {
    res.json(users);
  });
});   

// apply the routes to our application with the prefix /api
app.use('/api', apiRoutes);
</code></pre> 
       <p>We now have two routes that we can use. We can see these in our browser again, but at this point, let's switch over to POSTman.</p> 
       <p>For the URL, use: <code>http://localhost:8080/api/</code> and we will be able to see the message.</p> 
       <div class="text-center"> 
        <a href="https://cask.scotch.io/2014/11/2014-11-13-6.png"><img src="https://cask.scotch.io/2014/11/2014-11-13-6.png" alt="2014-11-13 (6)" width="1437" height="969" class="aligncenter size-full wp-image-5271 img-responsive img-thumbnail"></a> 
       </div> 
       <p>Then we can go to: <code>http://localhost:8080/api/users</code> and see the list of users.</p> 
       <div class="text-center"> 
        <a href="https://cask.scotch.io/2014/11/2014-11-13-5.png"><img src="https://cask.scotch.io/2014/11/2014-11-13-5.png" alt="2014-11-13 (5)" width="1425" height="959" class="aligncenter size-full wp-image-5270 img-responsive img-thumbnail"></a> 
       </div> 
       <p>This is great that we have been able to create a user and show them. We probably don't want any random person to see our list of users however.</p> 
       <p>Next, let's make sure that we can <strong>authenticate a user</strong> and then <strong>protect those routes</strong> using Express route middleware and requiring a token.</p> 
       <h2>Authenticating and Creating a Token</h2> 
       <p>Let's make our <code>POST http://localhost:8080/api/authenticate</code> route where we will accept a name and a password (probably from a form). If the name and password validate, then we will create a token and pass that back.</p> 
       <p>Once the user has that token, they will store it client side and pass it with every request for information after that and we will validate the token on every request using route middleware.</p> 
       <p>Here's the code for our POST route:</p> 
       <pre><code class="language-javascript">// API ROUTES -------------------

// get an instance of the router for api routes
var apiRoutes = express.Router(); 

// route to authenticate a user (POST http://localhost:8080/api/authenticate)
apiRoutes.post('/authenticate', function(req, res) {

  // find the user
  User.findOne({
    name: req.body.name
  }, function(err, user) {

    if (err) throw err;

    if (!user) {
      res.json({ success: false, message: 'Authentication failed. User not found.' });
    } else if (user) {

      // check if password matches
      if (user.password != req.body.password) {
        res.json({ success: false, message: 'Authentication failed. Wrong password.' });
      } else {

        // if user is found and password is right
        // create a token with only our given payload
    // we don't want to pass in the entire user since that has the password
    const payload = {
      admin: user.admin 
    };
        var token = jwt.sign(payload, app.get('superSecret'), {
          expiresInMinutes: 1440 // expires in 24 hours
        });

        // return the information including token as JSON
        res.json({
          success: true,
          message: 'Enjoy your token!',
          token: token
        });
      }   

    }

  });
});

...</code></pre> 
       <p>With this code, we can check our user and password and pass back a token in a JSON response. We are using <strong>mongoose to find the user</strong> and <strong>jsonwebtoken to create the token</strong>.</p>

       <h2>Route Middleware to Protect API Routes</h2> 
       <p>At this point, we have 3 routes defined on our API routes (<code>/api/authenticate</code>, <code>/api</code>, and <code>/api/users</code>). Let's create route middleware to protect the last 2 routes. We won't want to protect the <code>/api/authenticate</code> route so what we'll do is place our middleware beneath that route. Order is important here.</p> 
       <p>Let's take a look at the code:</p> 
       <pre><code class="language-javascript">// API ROUTES -------------------

// get an instance of the router for api routes
var apiRoutes = express.Router(); 

// route to authenticate a user (POST http://localhost:8080/api/authenticate)
...

// route middleware to verify a token
apiRoutes.use(function(req, res, next) {

  // check header or url parameters or post parameters for token
  var token = req.body.token || req.query.token || req.headers['x-access-token'];

  // decode token
  if (token) {

    // verifies secret and checks exp
    jwt.verify(token, app.get('superSecret'), function(err, decoded) {      
      if (err) {
        return res.json({ success: false, message: 'Failed to authenticate token.' });    
      } else {
        // if everything is good, save to request for use in other routes
        req.decoded = decoded;    
        next();
      }
    });

  } else {

    // if there is no token
    // return an error
    return res.status(403).send({ 
        success: false, 
        message: 'No token provided.' 
    });

  }
});

// route to show a random message (GET http://localhost:8080/api/)
...

// route to return all users (GET http://localhost:8080/api/users)
...

// apply the routes to our application with the prefix /api
app.use('/api', apiRoutes);
</code></pre> 
       <p>We are using the <code>jsonwebtoken</code> package again, but this time we are going to verify the token that was passed in. It is important that our secret used here matches the secret that was used to create the token. We are also making sure to send the right HTTP response code as 403 forbidden and our user was not authenticated to view any data.</p> 
       <p>If everything looks good the token was able to be verified, we'll take the information that came out of the token and pass it to the other routes in the <code>req</code> object.</p>
