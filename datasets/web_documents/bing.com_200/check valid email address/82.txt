         <ul>
          <li><em>all_lowercase: true</em> - Transforms the local part (before the @ symbol) of all email addresses to lowercase.           <li><em>outlookdotcom_lowercase: true</em> - Outlook.com addresses (including Windows Live and Hotmail) are known to be case-insensitive, so this switch allows lowercasing them even when <em>all_lowercase</em> is set to false. Please note that when <em>all_lowercase</em> is true, Outlook.com addresses are lowercased regardless of the value of this setting.</li>
          <li><em>outlookdotcom_remove_subaddress: true</em>: Normalizes addresses by removing "sub-addresses", which is the part following a "+" sign (e.g. <a href="mailto:%22foo+bar@outlook.com">"foo+bar@outlook.com</a>" becomes <a href="mailto:%22foo@outlook.com">"foo@outlook.com</a>").</li>
          <li><em>yahoo_lowercase: true</em> - Yahoo Mail addresses are known to be case-insensitive, so this switch allows lowercasing them even when <em>all_lowercase</em> is set to false. Please note that when <em>all_lowercase</em> is true, Yahoo Mail addresses are lowercased regardless of the value of this setting.</li>
          <li><em>yahoo_remove_subaddress: true</em>: Normalizes addresses by removing "sub-addresses", which is the part following a "-" sign (e.g. <a href="mailto:%22foo-bar@yahoo.com">"foo-bar@yahoo.com</a>" becomes <a href="mailto:%22foo@yahoo.com">"foo@yahoo.com</a>").</li>
          <li><em>icloud_lowercase: true</em> - iCloud addresses (including MobileMe) are known to be case-insensitive, so this switch allows lowercasing them even when <em>all_lowercase</em> is set to false. Please note that when <em>all_lowercase</em> is true, iCloud addresses are lowercased regardless of the value of this setting.</li>
          <li><em>icloud_remove_subaddress: true</em>: Normalizes addresses by removing "sub-addresses", which is the part following a "+" sign (e.g. <a href="mailto:%22foo+bar@icloud.com">"foo+bar@icloud.com</a>" becomes <a href="mailto:%22foo@icloud.com">"foo@icloud.com</a>").</li>
