           <h1>Slimsy</h1> 
           <p><strong>Effortless Responsive Images with Slimmage and Umbraco</strong></p> 
           <p><strong>Note</strong>&nbsp;Slimsy v1.1.6+ requires Umbraco v7.3.0+ (for earler versions of Umbraco use Slimsy v1.1.5 from the archive)</p> 
           <p>Slimmage.js used in conjunction with ImageProcessor.Web (included in Umbraco v7.1) and the built-in Umbraco Image Cropper will make your responsive websites images both adaptive and "retina" quality (if supported by the client browser).</p> 
           <p>Slimsy installs everything you need to use Slimmage.js in Umbraco v7.1. It includes Slimmage.js, SlimResponse and some helper methods that work with both dynamic and typed published content models.</p> 
           <h2><a href="https://github.com/Jeavon/Slimsy#implementing-post-package-installation" target="_blank" rel="nofollow"></a>Implementing post package installation</h2> 
           <h3><a href="https://github.com/Jeavon/Slimsy#1-add-slimmagejs-to-your-pages" target="_blank" rel="nofollow"></a>1. Add slimmage.js to your pages</h3> 
           <p>In your master template add the Slimmage Javascript file(s) to the top of your head section (Slimmage should be the first js library)</p> 
           <p><strong>without bundling scripts (plain HTTP requests)</strong></p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;script type="text/javascript"&gt;
        window.slimmage = { verbose: false };
    &lt;/script&gt;
    &lt;script src="/scripts/slimmage.min.js"&gt;&lt;/script&gt;
</code></pre>
           </div> 
           <p><strong>with bundling of your scripts you can fetch the configuration from a separate file</strong></p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;script src="/scripts/slimmage.settings.js"&gt;&lt;/script&gt;
    &lt;script src="/scripts/slimmage.min.js"&gt;&lt;/script&gt;
</code></pre>
           </div> 
           <p>If you don't already have js bundling in place you should take a look at the&nbsp;<a href="../../../projects/developer-tools/optimus" target="_blank" rel="nofollow">Optimus</a>&nbsp;package, it will allow you to bundle them together in minutes.</p> 
           <h3><a href="https://github.com/Jeavon/Slimsy#2-adjust-your-image-src-attributes" target="_blank" rel="nofollow"></a>2. Adjust your image src attributes</h3> 
           <p>Use the GetResponsiveImageUrl or GetResponsiveCropUrl methods on your dynamic or typed content/media items. For these methods to function correctly your image property types should use the built-in&nbsp;<strong>Image Cropper</strong>.</p> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsiveimageurlwidth-height" target="_blank" rel="nofollow"></a>GetResponsiveImageUrl(width, height)</h4> 
           <p>use this method for setting the crop dimensions in your Razor code, assumes your image cropper property alias is "umbracoFile"</p> 
           <p>e.g. An initial image size of 270 x 161. This example is looping pages, each page has a media picker with property alias "Image"</p> 
           <div class="body markdown-syntax">
            <pre><code>@foreach (var feature in homePage.umbTextPages.Where("featuredPage"))
{
    &lt;div class="3u"&gt;
        &lt;!-- Feature --&gt;
        &lt;section class="is-feature"&gt;
            @if (feature.HasValue("Image"))
            {
                var featureImage = Umbraco.Media(feature.Image);
                &lt;a href="@feature.Url" class="image image-full"&gt;
                &lt;img src="@featureImage.GetResponsiveImageUrl(270, 161)" alt="" /&gt;
                &lt;/a&gt;
            }
            &lt;h3&gt;&lt;a href="@feature.Url"&gt;@feature.Name&lt;/a&gt;&lt;/h3&gt;
            @Umbraco.Truncate(feature.BodyText, 100)
        &lt;/section&gt;
        &lt;!-- /Feature --&gt;
    &lt;/div&gt;
}
</code></pre>
           </div> 
           <p>e.g. If you need only a width dimension (and a flexible height) set height parameter to 0</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveImageUrl(270, 0)" alt="" /&gt;
</code></pre>
           </div> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsiveimageurlwidth-height-propertyalias" target="_blank" rel="nofollow"></a>GetResponsiveImageUrl(width, height, propertyAlias)</h4> 
           <p>Overloaded method, with additional propertyAlias parameter to specify your image cropper property alias.</p> 
           <p>e.g. An initial image size of 270 x 161, Image Cropper property alias of "Image"</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveImageUrl(270, 161, "Image")" alt="" /&gt;
</code></pre>
           </div> 
           <p>e.g. If you need only a width dimension (and a flexible height) set height parameter to 0</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featuredNewsItem.GetResponsiveImageUrl(859, 0, "Image")" alt="" /&gt;
</code></pre>
           </div> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsiveimageurlwidth-height-propertyalias-outputformat" target="_blank" rel="nofollow"></a>GetResponsiveImageUrl(width, height, propertyAlias, outputFormat)</h4> 
           <p>Overloaded method, with additional outputFormat parameter so you can specify if this image should be returned as a specific format type.</p> 
           <p>e.g. If you wanted a image to always be returned as a png</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveImageUrl(270, 161, "Image", "png")" alt="" /&gt;
</code></pre>
           </div> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsivecropurlcropalias" target="_blank" rel="nofollow"></a>GetResponsiveCropUrl(cropAlias)</h4> 
           <p>use this method when you want to use a predefined crop, assumes your image cropper property alias is "umbracoFile".</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveCropUrl("home")" alt="" /&gt;
</code></pre>
           </div> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsivecropurlcropalias-propertyalias" target="_blank" rel="nofollow"></a>GetResponsiveCropUrl(cropAlias, propertyAlias)</h4> 
           <p>Overloaded method, with additional propertyAlias parameter to specify your image cropper property alias.</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveCropUrl("home", "Image")" alt="" /&gt;
</code></pre>
           </div> 
           <h4><a href="https://github.com/Jeavon/Slimsy#getresponsivecropurlcropalias-propertyalias-outputformat" target="_blank" rel="nofollow"></a>GetResponsiveCropUrl(cropAlias, propertyAlias, outputFormat)</h4> 
           <p>Overloaded method, with additional outputFormat parameter so you can specify if this image should be returned as a specific format type.</p> 
           <div class="body markdown-syntax">
            <pre><code>    &lt;img src="@featureImage.GetResponsiveCropUrl("home", "Image", "png")" alt="" /&gt;
</code></pre>
           </div> 
           <h1><a href="https://github.com/Jeavon/Slimsy#advanced-options" target="_blank" rel="nofollow"></a>Advanced Options</h1> 
           <p>By default Slimsy v1.1.0+ will request that all images generated by ImageProcessor are jpg as this provides the best compression and optimisation, this is regardless of the source image type. You can disable this behaviour by adding the following appsetting to web.config.</p> 
           <div class="body markdown-syntax">
            <pre><code>&lt;add key="Slimsy:Format" value="false"/&gt;
</code></pre>
           </div> 
           <p>The above overloaded methods specifying outputFormat will of course take precedence over this default.</p> 
           <p>If you really wanted to have all default URLs return pngs you could also do this</p> 
           <div class="body markdown-syntax">
            <pre><code>&lt;add key="Slimsy:Format" value="png"/&gt;</code></pre>
           </div> 
           <p>Added in Slimsy v1.1.2 - you can now specify the default background color by added another appsetting to web.config. As an exmaple this setting is used if ImageProcessor is converting a png to a jpg and it as some transparency</p> 
           <div class="body markdown-syntax">
            <pre><code>&lt;add key="Slimsy:BGColor" value="fff"/&gt;
</code></pre>
           </div>

    $(document).ready(function () {
        $("#reportCompatibility").click(function () {
            $(".report-compat-item").show();
            $(this).hide();
            $("#reportCompatibilityPanel").show();
        });
        $("#cancelCompatibility").click(function () {
            $(".report-compat-item").hide();
            $("#reportCompatibility").show();
            $("#reportCompatibilityPanel").hide();
        });
        $("#reportCompatibilityPanel button").click(function (e) {
            var apiUrl = "/umbraco/Api/ProjectCompatibility/UpdateCompatibility";
            e.preventDefault();

            var report = {};

            $(".versions > div").each(function() {
                var version = $(this).find("input").val();
                var val = $(this).find("select").val();
                if (val === "1") {
                    report[version] = true;
                }
                else if (val === "2") {
                    report[version] = false;
                }
            });

            var model = {
                "projectId": 125049,
                "fileId": 15885,
                "report": report
            }

            $.post(apiUrl, model,
                function(data) {
                    $(".report-compat-item").hide();
                    $("#reportingTools").html("<strong>Thanks!</strong>");
                    $(".versions > div").each(function() {
                        var version = $(this).find("input").val();
                        var found = _.find(data, function(item) {
                            return item.version === version;
                        });
                        if (found) {
                            var span = $(this).find("span.smiley");
                            span.css("smiley " + found.smiley);
                            span.html(found.version + " " + (found.smiley === "untested" ? "(untested)" : ("(" + found.percentage + "%)")));
                        }
                    });
                });

        });
    });
