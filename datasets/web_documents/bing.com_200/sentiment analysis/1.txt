         <p>Sentiment is a Node.js module that uses the <a href="http://www2.imm.dtu.dk/pubdb/views/publication_details.php?id=6010">AFINN-165</a> wordlist and <a href="http://journals.plos.org/plosone/article?id=10.1371/journal.pone.0144296">Emoji Sentiment Ranking</a> to perform <a href="http://en.wikipedia.org/wiki/Sentiment_analysis">sentiment analysis</a> on arbitrary blocks of input text. Sentiment provides several things:</p> 
         <ul> 
          <li>Performance (see benchmarks below)</li> 
          <li>The ability to append and overwrite word / value pairs from the AFINN wordlist</li> 
          <li>A build process that makes updating sentiment to future wordlists trivial</li> 
         </ul>

         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> sentiment <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>sentiment<span class="pl-pds">'</span></span>);

});

           </svg></a>Benchmarks</h3> 
         <p>A primary motivation for designing <code>sentiment</code> was performance. As such, it includes a benchmark script within the test directory that compares it against the <a href="https://github.com/thinkroth/Sentimental">Sentimental</a> module which provides a nearly equivalent interface and approach. Based on these benchmarks, running on a MacBook Pro with Node v6.9.1, <code>sentiment</code> is <strong>twice as fast</strong> as alternative implementations:</p> 
         <div class="highlight highlight-source-shell">
          <pre>sentiment (Latest) x 448,788 ops/sec ±1.02% (88 runs sampled)
Sentimental (1.0.1) x 240,103 ops/sec ±5.13% (81 runs sampled)</pre>
         </div> 
         <p>To run the benchmarks yourself:</p> 
         <div class="highlight highlight-source-shell">
          <pre>make benchmark</pre>
         </div>

           </svg></a>Validation</h3> 
         <p>While the accuracy provided by AFINN is quite good considering it's computational performance (see above) there is always room for improvement. Therefore the <code>sentiment</code> module is open to accepting PRs which modify or amend the AFINN / Emoji datasets or implementation given that they improve accuracy and maintain similar performance characteristics. In order to establish this, we test the <code>sentiment</code> module against <a href="https://archive.ics.uci.edu/ml/datasets/Sentiment+Labelled+Sentences">three labelled datasets provided by UCI</a>.</p> 
         <p>To run the validation tests yourself:</p> 
         <div class="highlight highlight-source-shell">
          <pre>make validate</pre>
