             <h2>Introduction&nbsp;&nbsp;&nbsp;&nbsp;</h2> 
             <p>This is <strong>light, fast and simple to understand</strong> mathematical parser designed in <strong>one class</strong>, which receives as input a mathematical expression (System.String) and returns the output value (System.Double). For example, if your input string is "?(625)+25*(3/3)" then parser returns double value — 50.&nbsp;</p>
             <h2>Background&nbsp;&nbsp;&nbsp;</h2>
             <p>The idea was to create a string calculator for educational goals.&nbsp;&nbsp;</p>
             <h2>How it work&nbsp;&nbsp;&nbsp;</h2>
             <p>&nbsp;<img src="/KB/recipes/381509/arcticleImg2.png"></p>
             <p>For more please look at the code, I tried to explain the work of the parser in the comments and how it can be changed and extended.&nbsp;&nbsp;</p>
             <p>Convert to RPN:</p>
             <p><img src="/KB/recipes/381509/arcticleImg3.png"></p>
             <p>*&nbsp;Input operators are replaced by the corresponding token (its necessary to distinguish unary from binary).&nbsp;</p>
             <h2>Using the code&nbsp;&nbsp;</h2>
             <p>Example:&nbsp;&nbsp;&nbsp;&nbsp;</p>
             <pre lang="cs"><span class="code-keyword">public</span> <span class="code-keyword">static</span> <span class="code-keyword">void</span> Main()
{
        MathParser parser = <span class="code-keyword">new</span> MathParser();
    <span class="code-keyword">string</span> s1 = <span class="code-string">"</span><span class="code-string">pi+5*5+5*3-5*5-5*3+1E1"</span>;
    <span class="code-keyword">string</span> s2 = <span class="code-string">"</span><span class="code-string">sin(cos(tg(sh(ch(th(100))))))"</span>;
        <span class="code-keyword">bool</span> isRadians = <span class="code-keyword">false</span>;
        <span class="code-keyword">double</span> d1 = parser.Parse(s1, isRadians);
    <span class="code-keyword">double</span> d2 = parser.Parse(s2, isRadians);

        Console.WriteLine(d1); <span class="code-comment">//</span><span class="code-comment"> 13.141592...</span>
    Console.WriteLine(d2); <span class="code-comment">//</span><span class="code-comment"> 0.0174524023974442</span>
        Console.ReadKey(<span class="code-keyword">true</span>);
} &nbsp;
</pre>
             <h2>Features&nbsp;&nbsp;</h2> 
             <p><strong>Ariphmetical operators&nbsp;</strong>: </p>
             <p> </p>
             <ul>
              <li>() — parentheses;&nbsp;&nbsp;</li>
              <li>+ —&nbsp;&nbsp;plus (a + b);&nbsp;</li>
              <li>- —&nbsp;minus (a - b);&nbsp;</li>
              <li>* — multiplycation symbol (a * b);&nbsp;</li>
              <li>/ — divide symbol&nbsp;(a / b);&nbsp;</li>
              <li>^ — degree symbol (a ^ b).&nbsp;&nbsp;</li>
             </ul>
             <p> </p>
             <p><strong>Functions&nbsp;</strong>: &nbsp;&nbsp;&nbsp;</p>
             <p>Trigonometric functions:&nbsp;</p>
             <div>
              <ul>
               <li>sin(x); </li>
               <li>cos(x); </li>
               <li>tg(x); &nbsp;</li>
               <li>ctg(x).&nbsp;</li>
              </ul>
              <p>Hyperbolic functions:</p>
              <ul>
               <li>sh(x); </li>
               <li>ch(x); </li>
               <li>th(x).&nbsp;&nbsp;</li>
              </ul>
             </div>
             <div>
              <p> </p>Other functions:&nbsp;
              <br>
              <ul>
               <li>?(a), sqrt(a) — square root of a number;&nbsp;</li>
               <li>exp(x) &nbsp;— exponential function (or just use e^x);&nbsp;&nbsp;</li>
               <li>(a)log(b) — logarithm;&nbsp;</li>
               <li>ln(x) — natural logarithm;&nbsp;</li>
               <li>abs(x) — absolute value.&nbsp;&nbsp;</li>
              </ul>
              <p> </p>
             </div>
             <div>
              <p><strong>Constants</strong>: </p>
              <p> </p>
              <ul>
               <li>pi — 3.14159265...;&nbsp;&nbsp;</li>
               <li>e — 2.71828183....&nbsp;&nbsp;</li>
              </ul>
              <p> </p>
              <p>Arguments of the trigonometric functions can be expressed as radians (true) or degrees (false) by sending appropriate bool value to parse method.&nbsp;Example (as radians):&nbsp;&nbsp;&nbsp;</p>
              <pre lang="cs">parser.Parse(<span class="code-string">"</span><span class="code-string">sin(90)"</span>, <span class="code-keyword">true</span>);</pre>
              <p>Work with any char decimal separator in real number (regional settings).</p>
              <p>New operators, functions and constants can be easily added to the code.&nbsp;</p>
              <p>This parser is simple (special designed in single class), convenient for the further implementation, expansion and modification.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</p>

                               </div>I know what you mean, however, you are always allowed to specify a standard, i.e. in<br> <br> 68^-16^5^72<br> <br> there is a standard defined (otherwise you need to use brackets). The standard is the same as<br> <br> 68^((-16)^(5^72)).<br> <br> One example of such a case is 2^2^2^2. If you go from left to right you end up with 256 (2^2 = 4, 4^2 = 16, 16^2 = 256).                                </div>This is a commonly known topic: <a href="http://en.wikipedia.org/wiki/Operator_associativity">Operator Associativity</a>[<a href="http://en.wikipedia.org/wiki/Operator_associativity" target="_blank" title="New Window">^</a>]. Each decent math expression parser knows about this and makes proper decisions while parsing the passed expression string.<br> E.g. <code>a&lt;sup&gt;b&lt;sup&gt;c&lt;/sup&gt;&lt;/sup&gt;</code> is <b>right</b>-associative as described e.g. in <a href="http://en.wikipedia.org/wiki/Tetration">Tetration</a>[<a href="http://en.wikipedia.org/wiki/Tetration" target="_blank" title="New Window">^</a>].<br> Associativity says, where to start drawing parethesis if you have a chain of operators of the same <a href="http://en.wikipedia.org/wiki/Order_of_operations">Precedence</a>[<a href="http://en.wikipedia.org/wiki/Order_of_operations" target="_blank" title="New Window">^</a>].<br> <br> The same holds for the unary <code>+</code> and <code>-</code> sign operator. And on more complex languages: casting, indexing, member access, etc. They are all <b>right</b>-associative.<br> The vast majority of operators are <b>left</b>-associative, as the standard math operators dictate.<br> <br> All other interpretations are unexpected behaviour and misleading, i.e. violating one of the base rules of user interaction and programming (avoid unexpected/misleading/unintuitive behaviour).<br> <br> Your argument to inhibit that is a bit awkward, since the rule is not discussable from a standard point of view (see <a href="http://en.wikipedia.org/wiki/Tetration">http://en.wikipedia.org/wiki/Tetration</a>[<a href="http://en.wikipedia.org/wiki/Tetration" target="_blank" title="New Window">^</a>]).<br> <br> Cheers<br> Andi<br></td>
