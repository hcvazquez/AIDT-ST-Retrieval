      <h1>Stripe.js Reference</h1> 
      <p>Stripe.js makes it easy to collect certain kinds of sensitive information without having it touch your server.</p> 
     </header> 
     <section> 
      <p>This is the comprehensive API reference for Stripe.js. With Stripe.js, you can create tokens, insert and customize expertly-crafted form elements, and accept payments with Apple Pay and the Payment Request API.</p> 
      <aside class="note"> 
       <h3 id="quickstart">Quickstart</h3> 
       <p>Looking for something lighter weight? Check out the <a href="/docs/elements">Elements Quickstart</a> or the <a href="/docs/elements/payment-request-button">Payment Request Button Guide</a>.</p>

      <h2 id="including-stripejs">Including Stripe.js</h2> 
      <p>However you’re using Stripe.js, you always begin by including the library and setting your API key. To get started, include this script on your pages—it should always be loaded directly from <strong>https://js.stripe.com</strong>:</p> 
      <div class="code" data-language="html"> 
       <pre class="language-html numbered"><code>&lt;script src="https://js.stripe.com/v3/"&gt;&lt;/script&gt;</code></pre> 
      </div> 
      <p>To best leverage Stripe’s <a href="/docs/radar">advanced fraud functionality</a>, include this script on every page on your site, not just the checkout page. This allows Stripe to detect anomalous behavior that may be indicative of fraud as users browse your website.</p> 
      <aside class="important"> 
       <p>This version of Stripe.js uses a different method for <a href="#stripe-create-token">creating tokens</a> than <a href="/docs/stripe.js/v2">Stripe.js v2</a>. We recommend existing users upgrade at their earliest convenience. If you’re currently using Stripe.js v2 to collect card information, check out our <a href="/docs/elements/migrating">Elements migration guide</a> to learn how to switch. </p> 
      </aside> 
      <h3 id="stripe-function"><code>Stripe(<span class="token string">publishableKey[, options]</span>)</code></h3> 
      <p>Use <code>Stripe(publishableKey[, options])</code> to create an instance of the Stripe object. Your Stripe publishable <a href="/docs/dashboard#api-keys">API key</a> is required when calling this function, as it identifies your website to Stripe:</p> 
      <div class="tabs tabs-code animated"> 
       <nav class="nav-languages"> 
        <a data-language="javascript">JavaScript</a> 
        <a data-language="javascript_esnext">JavaScript (ESNext)</a> 
       </nav> 
       <div class="tabs-content"> 
        <div class="code" data-language="javascript"> 
         <pre class="language-javascript numbered"><code>var stripe = Stripe('pk_test_6pRNASCoBOKtIshFeQd4XMUh');</code></pre> 
        </div> 
        <div class="code" data-language="javascript_esnext"> 
         <pre class="language-javascript numbered"><code>const stripe = Stripe('pk_test_6pRNASCoBOKtIshFeQd4XMUh');</code></pre> 
        </div> 
       </div> 
      </div> 
      <p>We‘ve placed a random API key in the code. Replace it with your <a href="https://dashboard.stripe.com/account/apikeys">actual publishable API key</a> to test this code through your Stripe account.</p> 
      <p>When you’re ready to accept live payments, replace the test key with your live key in production. Learn more about how API keys work in <a href="/docs/dashboard#livemode-and-testing">test mode and live mode</a>.</p> 
      <p>This function accepts an optional <code><span class="token string" < span>options</span></code> object. Available options are documented below:</p> 
      <table class="reference" id="stripe-options"> 
       <tbody> 
        <tr> 
         <td style="width:25%;"><strong>stripeAccount</strong> 
          <div class="type">
           string
          </div></td> 
         <td><a href="/docs/connect">Connect</a> only. Specifying a connected account ID (e.g., <strong>acct_24BFMpJ1svR5A89k</strong>) will allow you to perform actions on behalf of that account.</td> 
        </tr> 
       </tbody> 
      </table> 
     </section>

      <h3 id="stripe-elements"><code>stripe.elements(<span class="token string">[options]</span>)</code></h3> 
      <p>Create pre-built UI components to collect sensitive card information with <a href="/docs/elements">Elements</a>.</p> 
      <div class="tabs tabs-code animated"> 
       <nav class="nav-languages"> 
        <a data-language="javascript">JavaScript</a> 
        <a data-language="javascript_esnext">JavaScript (ESNext)</a> 
       </nav> 
       <div class="tabs-content"> 
        <div class="code" data-language="javascript"> 
         <pre class="language-javascript numbered"><code>var elements = stripe.elements();</code></pre> 
        </div> 
        <div class="code" data-language="javascript_esnext"> 
         <pre class="language-javascript numbered"><code>const elements = stripe.elements();</code></pre> 
        </div> 
       </div> 
      </div> 
      <p>This method creates an instance of <code>elements</code>, which manages a group of Elements. It accepts an optional <code><span class="token string">options</span></code> object. Available options are documented below:</p> 
      <table class="reference" id="elements-options"> 
       <thead> 
        <tr> 
         <th>Option</th> 
         <th>Description</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td class="attr" style="width:18%;"> fonts 
          <div class="type">
           <span class="tag optional">optional</span> array
          </div> </td> 
         <td> An array of custom fonts, which Elements created from the <code>elements</code> object can use. <br> Fonts can either be loaded via a CSS file by passing an object with the <strong>cssSrc</strong> attribute: 
          <table> 
           <tbody> 
            <tr> 
             <td class="attr" style="width:20%;"> cssSrc 
              <div class="type">
               string
              </div> </td> 
             <td> A relative or absolute URL pointing to a CSS file with <a href="https://developer.mozilla.org/en/docs/Web/CSS/@font-face">@font-face</a> definitions, for example:<br> <code>"https://fonts.googleapis.com/css?family=Open+Sans"</code><br> Note that if you are using a <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy">content security policy</a> (CSP), <a href="/docs/security#content-security-policy">additional directives</a> may be necessary. </td> 
            </tr> 
           </tbody> 
          </table> or they can be loaded directly by passing an object with the following attributes: 
          <table> 
           <tbody> 
            <tr>

      <h3 id="element-mount"><code>element.mount(<span class="token string">domElement</span>)</code></h3> 
      <p>You will need to create a container DOM element in order to <code>mount</code> an Element. If it has a label, it will be automatically focused when its label is clicked. There are two ways to do this:</p> 
      <ol> 
       <li>Mount the instance within a <code>&lt;label&gt;</code>. 
        <div class="code" data-language="html"> 
         <pre class="language-html numbered"><code>&lt;label&gt;Card
  &lt;div id="card-element"&gt;&lt;/div&gt;
&lt;/label&gt;</code></pre> 
        </div> </li> 
       <li>Create a <code>&lt;label&gt;</code> with a <code>for</code> attribute, referencing the ID of your container. 
        <div class="code" data-language="html"> 
         <pre class="language-html numbered"><code>&lt;label for="card-element"&gt;Card&lt;/label&gt;
&lt;div id="card-element"&gt;&lt;/div&gt;</code></pre> 
        </div> </li> 
      </ol> 
      <p>The <code>mount</code> method attaches your element to the DOM. <code>mount</code> accepts either a CSS Selector (e.g., <code>'#card-element'</code>) or a DOM element.</p> 
      <div class="code" data-language="javascript"> 
       <pre class="language-javascript numbered"><code>cardElement.mount('#card-element');</code></pre> 
      </div> 
      <h3 id="element-on"><code>element.on(<span class="token string">event, handler</span>)</code></h3> 
      <p>The only way to communicate with your element is by listening to an <code><span class="token string">event</span></code>. Elements might emit any of these events:</p> 
      <table class="reference"> 
       <tbody> 
        <tr> 
         <td class="attr" style="width:10%;"> blur </td> 
         <td>Triggered when the Element loses focus.</td> 
        </tr> 
        <tr> 
         <td class="attr"> change </td> 
         <td>Triggered when any of the following values changes on the Element:<br> 
          <table class="reference"> 
           <tbody> 
            <tr> 
             <td class="attr"> empty 
              <div class="type">
               boolean
              </div> </td> 
             <td><code>true</code> if the value is empty.</td> 
            </tr> 
            <tr> 
             <td class="attr"> complete 
              <div class="type">
               boolean
              </div> </td> 
             <td> <code>true</code> if the value is well-formed and potentially complete.<br> <strong>complete</strong> can be used to progressively disclose the next parts of your form or to enable form submission.<br> <strong>complete</strong> <em>is not</em> an indicator of whether a user is done with their input—it only indicates that the Element contains a potentially complete, well-formed value. In many cases the user could still add further input.<br> <strong>complete</strong> <em>should not</em> be used to perform an action such as advancing the cursor to a subsequent field or performing a tokenization request. </td> 
            </tr> 
            <tr> 
             <td class="attr"> brand 
              <div class="type">
               string
              </div> </td> 
             <td>Applies to the <code><span class="token string">card</span></code> and <code><span class="token string">cardNumber</span></code> Elements only. Contains the card brand (e.g., <code>visa</code> or <code>amex</code>) of the card number being entered.</td> 
            </tr> 
            <tr> 
             <td class="attr"> error

               object
              </div> </td>

            <tr> 
             <td class="attr"> value 
              <div class="type">
               string, object
              </div> </td> 
             <td>An object similar to the one used for <code>value</code> when <a href="#element-options">creating an Element</a>. Only present for non-sensitive values, like <code>postalCode</code>.</td> 
            </tr> 
           </tbody> 
          </table> </td> 
        </tr> 
        <tr> 
         <td class="attr"> focus </td> 
         <td>Triggered when the Element gains focus.</td> 
        </tr> 
        <tr> 
         <td class="attr"> ready </td> 
         <td>Triggered when the Element is fully rendered and can accept <code>element.focus()</code> calls.</td> 
        </tr> 
       </tbody> 
      </table>

      <h4 id="postal-code-formatting">Postal code formatting</h4> 
      <p>Elements automatically determines your customer’s billing address country based on their card number. Using this information, the postal code field validation reflects whether that country uses numeric or alphanumeric-formatted postal codes, or if the country uses postal codes at all. For instance, if a U.S. card is entered, the postal code field only accepts a five-digit numeric value. If it’s a UK card, an alphanumeric value can be provided instead.</p> 
      <p>Many of our test cards have a U.S. billing address country. When using these to test your payment form, you must also use a five-digit U.S. ZIP code (e.g., <strong>12345</strong>). To test Elements with other postal code formats, use our <a href="/docs/testing#international-cards">international test card numbers</a>.</p>

      <h2 id="the-element-container">The Element container</h2> 
      <p>You should style the container you mount an Element to as if it were an <code>&lt;input&gt;</code> on your page. For example, to control <code>padding</code> and <code>border</code> on an Element, set these properties on the container. This is usually done by re-using the classes that you have applied to your DOM <code>&lt;input&gt;</code>s. Example:</p> 
      <div class="code" data-language="html"> 
       <pre class="language-html numbered"><code>&lt;style&gt;
 .my-input {
   padding: 10px;
   border: 1px solid #ccc;
 }
&lt;/style&gt;

&lt;form&gt;
  &lt;div&gt;
    &lt;label&gt;Name&lt;/label&gt;
    &lt;input class="my-input"&gt;
  &lt;/div&gt;
  &lt;div&gt;
    &lt;label&gt;Card&lt;/label&gt;
    &lt;!-- Using the same "my-input" class on the --&gt;
    &lt;!-- regular input above and on this container. --&gt;
    &lt;div class="my-input" id="card-element"&gt;&lt;/div&gt;
  &lt;/div&gt;
&lt;/form&gt;</code></pre> 
      </div> 
      <p>After the Element is mounted, the <code>.StripeElement</code> class is added to the container. Additionally, the following classes are automatically added to the container when the Element is complete, empty, focused, invalid, or autofilled by the browser:</p> 
      <ul> 
       <li><code>.StripeElement--complete</code></li> 
       <li><code>.StripeElement--empty</code></li> 
       <li><code>.StripeElement--focus</code></li> 
       <li><code>.StripeElement--invalid</code></li> 
       <li><code>.StripeElement--webkit-autofill</code> (Chrome and Safari only)</li> 
      </ul> 
      <p>You can override these class names using the <code>classes</code> <a href="#element-options">option</a> when you <code>create</code> an Element.</p>

      <h2 id="supported-browsers">Supported browsers</h2> 
      <p>Stripe.js strives to support all recent versions of major browsers. For the sake of security and providing the best experience to the majority of customers, we do not support browsers that are no longer receiving security updates and represent a small minority of traffic.</p> 
      <ul> 
       <li>We support Internet Explorer and Edge per <a href="https://support.microsoft.com/en-us/gp/microsoft-internet-explorer">Microsoft's lifecycle policy</a>. As of January 12, 2016, we support Internet Explorer 9 and above.</li> 
       <li>We support Chrome and Safari on all platforms and Firefox on desktop platforms.</li> 
       <li>We support the Android native browser on Android 4.4 and later.</li> 
       <li>We require TLS 1.2 to be supported by the browser.</li> 
       <li>We respond to bug reports but do not proactively test other mobile browsers.</li> 
      </ul> 
      <p>If you have an issue with Stripe.js on a specific browser, please <a href="https://support.stripe.com/email">contact us</a> so we can improve its support.</p>
