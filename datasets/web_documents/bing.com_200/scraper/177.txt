     <h1> <span class="mw-headline" id="Java_on_Fedora"> Java on Fedora </span></h1> 
     <p>Fedora uses a Free software stack that consists of <a class="external text" href="http://en.wikipedia.org/wiki/Openjdk">OpenJDK</a> and IcedTea. Refer to our <a href="/wiki/Java/FAQ" title="Java/FAQ">Java/FAQ</a> for more information on that. Software mentioned on this page may come in different packaging formats. </p>
     <p>Fedora's own packages (<i>RPM</i>) are easy to install with <a href="/wiki/Dnf" title="Dnf" class="mw-redirect">Dnf</a>. As simple as "<code>sudo dnf install java</code>". </p>
     <p>Third party packages may be in archive formats (.zip, tar) for which case-specific instructions should be provided. </p> 
     <h1> <span class="mw-headline" id="Terminology"> Terminology </span></h1> 
     <p>In Java context, terms may be a bit confusing for newcomers. <b>JRE</b> (<i>Java Runtime Environment</i>) stands for <i>virtual machine</i>, which is able to run Java programs. </p>
     <p><b>JDK</b> (<i>Java Development Kit</i>) or <b>SDK</b> (<i>Software Development Kit</i>) is needed to develop Java programs. Many SDK packages these days also ship JRE, or to confuse more, no separate JRE is provided and thus terminal installations are supposed to include the development environment. </p>
     <p>Terms <b>AOT</b> (<i><a class="external text" href="http://en.wikipedia.org/wiki/AOT_compiler">Ahead of Time</a>]</i>), <b>SE</b>, <b>J2EE</b> need some more explanation. </p>
     <p><br> </p> 
     <h1> <span class="mw-headline" id="Java_Runtime_Environments_.28JRE.29"> Java Runtime Environments (JRE) </span></h1> 
     <p>Java has different JRE implementations and wide variety of combinations which work together. Different implementations can be installed simultaneously to Fedora and activated one at a time. Activation is done using the <a href="/w/index.php?title=Alternatives&amp;action=edit&amp;redlink=1" class="new" title="Alternatives (page does not exist)">Alternatives</a> system (<i>also used to change some other subsystems</i>). Java's subsystem name is surprisingly <i>java</i> and typical commands include: </p> 
     <pre># alternatives --display java</pre> 
     <pre># alternatives --config java</pre> 
     <p>See alternative's own documentation for more information for usage and parts involved. </p>
     <p>Should be noted that JRE implementations installed outside Fedora distribution, may not support alternatives and thus not be visible there. Then the symbolic links under directory <i>/etc/alternatives</i> must be manually fixed. </p> 
     <h2> <span class="mw-headline" id="OpenJDK_and_project_IcedTea"> OpenJDK and project IcedTea </span></h2> 
     <p>Fedora has shipped OpenJDK as default JRE implementation. It is based on Sun Microsystem's/Oracle's <a class="external text" href="http://en.wikipedia.org/wiki/JavaOne">JavaOne</a> open source release and complemented by Red Hat's <a class="external text" href="http://en.wikipedia.org/wiki/IcedTea">IcedTea</a> project that implements the missing third party components that Sun/Oracle could not release under free License. </p>
     <p>OpenJDK's <b>java.library.path</b>, shared library paths for i386 are: </p> 
     <pre>
/usr/lib
/usr/lib/jvm/java-1.?.0-openjdk-1.?.0.0.x86_64/jre/lib/
</pre> 
     <p>and for x86_64: </p> 
     <pre>
/usr/lib64
/usr/lib/jvm/java-1.?.0-openjdk-1.?.0.0.x86_64/jre/lib/amd64/
</pre> 
     <p>OpenJDK package name on Fedora is <i>java-1.?.0-openjdk</i>. </p> 
     <h2> <span class="mw-headline" id="Oracle.27s_Java_SE"> Oracle's Java SE </span></h2> 
     <p>Original Oracle's Java SE (Standard Edition) can be downloaded directly from <a class="external free" href="http://www.java.com/en/download/">http://www.java.com/en/download/</a> and installed manually. </p>
     <p>You're, however, encouraged to try out the OpenJDK runtime first. </p>
     <p><br> </p> 
     <h2> <span class="mw-headline" id="Troubleshooting_JRE"> Troubleshooting JRE </span></h2> 
     <p>Fedora's JRE in browser environment can be tested using the following test pages: </p> 
     <ul>
      <li> <a class="external free" href="http://www.java.com/en/download/help/testvm.xml">http://www.java.com/en/download/help/testvm.xml</a> </li>
      <li> <a class="external free" href="http://java.com/en/download/installed.jsp?detect=jre&amp;try=1">http://java.com/en/download/installed.jsp?detect=jre&amp;try=1</a> </li>
     </ul> 
     <p>If these don't work, proceed with <a href="/wiki/Java/Troubleshooting" title="Java/Troubleshooting">Java/Troubleshooting</a> page. </p> 
     <h1> <span class="mw-headline" id="Java_Development_Tools_.28SDK.29"> Java Development Tools (SDK) </span></h1> 
     <p>The following SDKs are listed in alphabetic order. </p> 
     <h2> <span class="mw-headline" id="Eclipse"> Eclipse </span></h2> 
     <p>See our <a href="/wiki/Eclipse" title="Eclipse">Eclipse</a> page for an integrated development environment platform that itself is written in Java and has plugin support for many programming languages. </p> 
     <h2> <span class="mw-headline" id="IntelliJ_IDEA"> IntelliJ IDEA </span></h2> 
     <p>IntelliJ IDEA is considered by many the ultimate Java development environment. A commercial project for many years, IntelliJ IDEA became open-source in 2009 by releasing the IntelliJ IDEA Community Edition. IntelliJ is written in Java, features a portable Swing UI and is extensible via plugins. </p> 
     <h2> <span class="mw-headline" id="Maven"> Maven </span></h2> 
     <p>Fedora includes a somewhat customized version of Maven in the distribution. The customization is purely to make Maven work well in offline mode with the rest of the system. Details on how packagers can use this customized Maven are located <a class="external text" href="http://sochotni.fedorapeople.org/java-packaging-howto/">here</a> . Generally the latest stable Maven version is shipped. Also related to maven, a feasibility study is being performed to change the installation of Java packages to become a valid maven repository. It will make it easier to support parallel installation of either "legacy" or "progressive" versions of Java packages. Once some positive results are obtained, it will be discussed on the fedora-devel-java-list. Maven itself is no longer patched to add support for Fedora's file layout but a pristine copy is shipped. Customizations are done via the <a class="external text" href="https://github.com/mizdebsk/xmvn">XMvn project</a>. It is recommended to install maven-local rpm which will bring in xmvn as dependency but will also install some other helper scripts and customizations for Fedora. </p> 
     <h2> <span class="mw-headline" id="NetBeans_IDE"> NetBeans IDE </span></h2> 
     <p>The "reference" Java IDE, developed for many years by Sun(now Oracle). NetBeans is written in Java, features a portable Swing UI, out of the box support for many Java tools and frameworks and it's extensible via plugins. </p> 
     <h1> <span class="mw-headline" id="Java_Server_Side_Environments"> Java Server Side Environments </span></h1> 
     <h2> <span class="mw-headline" id="Apache_Tomcat"> Apache Tomcat </span></h2> 
     <p>Fedora ships <a class="external text" href="http://en.wikipedia.org/wiki/Apache_Tomcat">Apache Tomcat</a> as part of standard distribution. </p>
     <p>Apache Tomcat package name on Fedora is <i>tomcat</i> and is currently at version 7.x. </p> 
     <h2> <span class="mw-headline" id="JBoss_AS"> JBoss AS </span></h2> 
     <p>The <a href="/wiki/SIGs/Java" title="SIGs/Java">Java SIG</a> is working on packaging <a class="external text" href="http://jboss.org/as7">JBoss AS 7</a>, along with a minimal set of required JARs, to make this open-source Java EE application server available as a service in Fedora. </p>
     <p>You can learn more about the effort and the current progress on the <a href="/wiki/JBossAS7" title="JBossAS7">JBoss AS 7 on Fedora</a> page. </p>
     <p><a href="/wiki/JBossAS7_in_Fedora_marketing_collateral" title="JBossAS7 in Fedora marketing collateral">Talking points</a> about JBoss AS 7 on Fedora 17 are available and are used to create marketing cards and information sheets. </p> 
     <h1> <span class="mw-headline" id="Teaching_and_Learning_Java"> Teaching and Learning Java </span></h1> 
     <p>As part of planning and implementing new 100% FLOSS Java components into Fedora, this draft document has been opened to give developers a place to teach each other about best practices, patterns, etc. </p>
