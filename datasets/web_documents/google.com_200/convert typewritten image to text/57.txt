        </div> 
        <!-- for Cascading Toc --> 
       </div> 
       <div class="note parbase section"> 
        <div class="help-note"> 
         <p> <span class="help-note-title">Note:</span> </p>
         <p>The wiggle method—which is used to randomly vary a property value—is in the Property attributes and methods category. (See <a href="expression-language-reference.html#property_attributes_and_methods_expression_reference">Property attributes and methods (expression reference)</a>.)</p> 
         <p></p> 
        </div>

       <div class="variable parbase section"> 
        <!-- Commented out by cmiqueo --> 
        <!-- <a name="seedRandom(offset, timeless=false)" href="./"></a><div class="variable-title">seedRandom(offset, timeless=false)</div> --> 
        <a name="seedRandom(offset, timeless=false)"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">seedRandom(offset, timeless=false)</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Return type: none. </p>
         <p>Argument type: <i>offset</i> is a Number,<i> timeless</i> is a Boolean. </p>
         <p>The <span class="code">random</span> and <span class="code">gaussRandom</span> methods use a seed value that controls the sequence of numbers. By default, the seed is computed as a function of a unique layer identifier, the property within the layer, the current time, and an offset value of 0. Call <span class="code">seedRandom</span> to set the offset to something other than 0 to create a different random sequence. </p>
         <p>Use <span class="code">true</span> for the <i>timeless</i> argument to not use the current time as input to the random seed. Using <span class="code">true</span> for the <i>timeless</i> argument allows you to generate a random number that doesn’t vary depending on the time of evaluation. </p>
         <p>The <i>offset</i> value, but not the <i>timeless</i> value, is also used to control the initial value of the <span class="code">wiggle</span> function.</p>
         <p>For example, this expression on the Opacity property sets the Opacity value to a random value that does not vary with time:</p>
         <span class="code">&nbsp;&nbsp;seedRandom(123456, true); &nbsp;&nbsp;random()*100</span>

         <h2 id="interpolation_methods_expression_reference"> Interpolation methods (expression reference) </h2> 
        </div> 
        <!-- for Cascading Toc --> 
       </div> 
       <div class="text parbase section"> 
        <a name="main-pars_text_2"></a> 
        <div class="text">
         <p>For all the Interpolation methods, the argument <span class="code"><em>t</em></span> is often <span class="code">time</span> or <span class="code">value</span>, though it can have other values, instead. If <span class="code"><em>t</em></span> is <span class="code">time</span>, the interpolation between values happens over a duration. If <span class="code"><em>t</em></span> is <span class="code">value</span>, then the expression maps one range of values to a new range of values.</p> 
         <p>For additional explanations and examples of the Interpolation methods, see <a href="http://www.adobe.com/go/learn_ae_jjgiffordexpressionsinterpolation">JJ Gifford’s website</a>.</p> 
         <p>Chris and Trish Meyer provide additional information and examples for these methods in an article on the <a href="http://provideocoalition.com/index.php/cmg_keyframes/story/deeper_modes_of_expression_part_2_interpolation_methods/">ProVideo Coalition website</a>.</p> 
         <p>Ian Haigh provides a script on <a href="http://aescripts.com/ease-and-wizz/">After Effects Scripts website</a> that you can use to easily apply advanced interpolation method expressions—such as bounces—to properties.</p> 
         <p>Andrew Devis provides a <a href="http://blogs.adobe.com/toddkopriva/2010/10/tutorials-on-using-linear-expression-method.html">pair of video tutorials</a> on the Creative COW website that show in detail how to use the <span class="code">linear</span> expression method along with the Convert Audio To Keyframes command.</p>

        <a name="linear(t, tMin, tMax, value1, value2)"></a>

         <p class="text"> <span class="help-variable-title">linear(t, tMin, tMax, value1, value2)</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Return type: Number or Array.</p> 
         <p>Argument type: <em>t</em>, <em>tMin</em>, and <em>tMax</em> are Numbers, and <em>value1</em> and <em>value2</em> are Numbers or Arrays.</p>

         <p><span class="code">&nbsp;&nbsp;linear(time, 0, 6, 20, 80)</span></p> 
         <p>This method—like all the Interpolation methods—can also be used to convert from one range of values to another. For example, this expression on the Opacity property converts the Opacity values from the range 0%-100% to the range 20%-80%:</p> 
         <p><span class="code">&nbsp;&nbsp;linear(value, 0, 100, 20, 80)</span></p>

        <a name="sampleImage(point, radius = [.5, .5], postEffect=true,
t=time)"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">sampleImage(point, radius = [.5, .5], postEffect=true, t=time)</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Return type: Array [4].</p> 
         <p>Argument type: <em>point</em> is an Array [2], <em>radius</em> is an Array [2], <em>postEffect</em> is a Boolean, and <em>t</em> is a Number.</p> 
         <p>Samples the color and alpha channel values of a layer and returns the average alpha-weighted value of the pixels within the specified distance of the point as an array: [<em>red</em>, <em>green</em>, <em>blue</em>, <em>alpha</em>]. If <em>postEffect</em> is true, the sampled values are for the layer after masks and effects on that layer have been rendered; if <em>postEffect</em> is false, the sampled values are for the layer before masks and effects have been rendered. The input value <em>point</em> is in layer space; the point [0,0] is the center of the upper-left pixel in the layer. The input value <em>radius</em> specifies the horizontal and vertical distance from the sample center to the edges of the sampled rectangle. The default value samples one pixel.</p> 
         <p></p>
        </div> 
       </div> 
       <div class="note parbase section"> 
        <div class="help-note"> 
         <p> <span class="help-note-title">Note:</span> </p>
         <p>The postEffect parameter refers to effects applied directly to the layer, not to effects applied indirectly, such as with an adjustment layer.</p> 
         <p></p> 
        </div> 
       </div> 
       <div class="note parbase section"> 
        <div class="help-note"> 
         <p> <span class="help-note-title">Note:</span> </p>
         <p>Using sampleImage in an expression no longer disables multiprocessing.</p> 
         <p></p> 
        </div> 
       </div> 
       <div class="text parbase section"> 
        <a name="main-pars_text"></a> 
        <div class="text">
         <p>This example samples a rectangle 4 pixels wide and 3 pixels high, centered around a point 100 pixels down and to the right of the upper-left corner of the layer:</p> 
         <p><span class="code">&nbsp;&nbsp;thisComp.layer(1).sampleImage([100, 100], [2, 1.5])</span></p> 
         <p>Dan Ebberts provides an example of how to use the <span class="code">sampleImage</span> method on his <a href="http://www.adobe.com/go/learn_ae_danobscuration">MotionScript website</a>.</p> 
         <p>Todd Kopriva provides instructions for using the <span class="code">sampleImage</span> method and the Point Control effect to monitor colors for a specified point during color correction on his <a href="http://blogs.adobe.com/toddkopriva/2009/07/color-sampler-using-sampleimag.html">After Effects Region of Interest blog</a>.</p>

         <h2 id="layer_space_transforms_methods_expression_reference"> Layer Space Transforms methods (expression reference) </h2> 
        </div> 
        <!-- for Cascading Toc --> 
       </div> 
       <div class="text parbase section"> 
        <a name="main-pars_text_7"></a> 
        <div class="text">
         <p>Use layer space transform methods to transform values from one space to another, such as from layer space to world space. The “from” methods transform values from the named space (composition or world) to the layer space. The “to” methods transform values from the layer space to the named space (composition or world). Each transform method takes an optional argument to determine the time at which the transform is computed; however, you can almost always use the current (default) time.</p> 
         <p>Use “Vec” transform methods when transforming a direction vector, such as the difference between two position values. Use the plain (non-”Vec”) transform methods when transforming a point, such as position. Composition (comp) and world space are the same for 2D layers. For 3D layers, however, composition space is relative to the active camera, and world space is independent of the camera.</p> 
        </div> 
       </div>

        <!-- <a name="wiggle(freq, amp, octaves=1, amp_mult=.5, t=time)" href="./"></a><div class="variable-title">wiggle(freq, amp, octaves=1, amp_mult=.5, t=time)</div> --> 
        <a name="wiggle(freq, amp, octaves=1, amp_mult=.5, t=time)"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">wiggle(freq, amp, octaves=1, amp_mult=.5, t=time)</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Return type: Number or Array.</p> 
         <p>Argument type: <em>freq</em>, <em>amp</em>, <em>octaves</em>, <em>amp_mult</em>, and <em>t</em> are Numbers.</p> 
         <p>Randomly shakes (wiggles) the value of the property.</p> 
         <p><em>freq</em> value is the frequency in wiggles per second.</p> 
         <p><em>amp</em> value is the amplitude in units of the property to which it is applied.</p> 
         <p><em>octaves</em> is the number of octaves of noise to add together. This value controls how much detail is in the wiggle. Make this value higher than the default of 1 to include higher frequencies or lower to include amplitude harmonics in the wiggle.</p> 
         <p><em>amp_mult</em> is the amount that <em>amp</em> is multiplied by for each octave. This value controls how fast the harmonics drop off. The default is 0.5; make it closer to 1 to have the harmonics added at the same amplitude as the base frequency, or closer to 0 to add in less detail.</p> 
         <p><em>t</em> is the base start time. This value defaults to the current time. Use this parameter if you want the output to be a wiggle of the property value sampled at a different time.</p> 
         <p>Example: <span class="code">position.wiggle(5, 20, 3, .5)</span> produces about 5 wiggles per second with an average size of about 20 pixels. In addition to the main wiggle, two more levels of detailed wiggles occur with a frequency of 10 and 20 wiggles per second, and sizes of 10 and 5 pixels, respectively.</p> 
         <p>This example, on a two-dimensional property such as Scale, wiggles both dimensions by the same amount:</p> 
         <p><span class="code">&nbsp;&nbsp;v = wiggle(5, 10); &nbsp;&nbsp;[v[0], v[0]]</span></p> 
         <p>This example, on a two-dimensional property, wiggles only along the y-axis:</p> 
         <p><span class="code">&nbsp;&nbsp;freq = 3; &nbsp;&nbsp;amp = 50; &nbsp;&nbsp;w = wiggle(freq,amp); &nbsp;&nbsp;[value[0],w[1]];</span></p> 
         <p>Dan Ebberts provides an example expression and a detailed explanation on his <a href="http://www.motionscript.com/design-guide/looping-wiggle.html">MotionScript website</a> that shows how to use the time parameter of the <span class="code">wiggle</span> method to create a looping animation.</p>

        <div class="variable-text">
         <p class="text"></p>
         <p>Return type: Number or Array.</p> 
         <p>Loops a segment of time that is measured from the first keyframe on the layer forward toward the Out point of the layer. The loop plays from the In point of the layer. The <em>numKeyframes</em> value determines what segment is looped: The segment looped is the portion of the layer from the first keyframe to the <em>numKeyframes</em>+1 keyframe. For example, <span class="code">loopIn("cycle", 3) </span>loops the segment bounded by the first and fourth keyframes. The default value of 0 means that all keyframes loop.</p> 
         <p>You can use keyframe-looping methods to repeat a series of keyframes. You can use these methods on most properties. Exceptions include properties that can’t be expressed by simple numeric values in the Timeline panel, such as the Source Text property, path shape properties, and the Histogram property for the Levels effect. Keyframes or duration values that are too large are clipped to the maximum allowable value. Values that are too small result in a constant loop.</p>

        </div> 
        <!-- for Cascading Toc -->

       <div class="text parbase section"> 
        <a name="main-pars_text_13"></a> 
        <div class="text">

For example, the following expression gives you the value of the third Position keyframe: <span class="code">position.key(3).value</span>. </p>
         <p>The following expression, when written on an Opacity property with keyframes, ignores the keyframe values and uses only the placement of the keyframes in time to determine where a flash should occur: <span class="code">&nbsp;&nbsp;d = Math.abs(time - nearestKey(time).time); &nbsp;&nbsp;easeOut(d, 0, .1, 100, 0)</span></p>

        <div class="header  header-top"> 
         <h2 id="markerkey_attributes_expression_reference"> MarkerKey attributes (expression reference) </h2> 
        </div> 
        <!-- for Cascading Toc --> 
       </div> 
       <div class="text parbase section"> 
        <a name="main-pars_text_14"></a> 
        <div class="text">
         <p>You can access values for composition markers and layer markers using the same methods. Access layer markers through the <span class="code">thisLayer.marker</span> object; access composition markers through the <span class="code">thisComp.marker</span> object.</p> 
         <p>For the purpose of expressions, markers are a special type of Key object, so you can use methods such as <span class="code">nearestKey(time)</span> to access markers, and markers also have <span class="code">time</span> and <span class="code">index</span> attributes. The <span class="code">index</span> attribute is not the number (name) of the marker; it is the keyframe index number, representing the order of the marker in the time ruler.</p> 
         <p>Expressions have access to all the values for a marker that you can set in the Composition Marker or Layer Marker dialog box. This expression on the Source Text property of a text layer displays the time, duration, index, comment (name), chapter, URL, frame target, and cue point name for the layer marker nearest the current time, and whether the marker is for an event cue point:</p> 
        </div> 
       </div> 
       <div class="codeblock parbase section">
        <link> 
        <link> 
        <!-- Linewrap override --> 
        <pre class="brush: as3; gutter: true">&nbsp;&nbsp;m = thisLayer.marker.nearestKey(time);&nbsp; 
&nbsp;&nbsp;s = "time:" + timeToCurrentFormat(m.time) + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"duration: " + m.duration + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"key index: " + m.index + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"comment:" + m.comment + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"chapter:" + m.chapter + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"URL:" + m.url + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"frame target: " + m.frameTarget + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"cue point name: " + m.cuePointName + "\r" +&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"Event cue point? " + m.eventCuePoint + "\r";&nbsp; 
&nbsp;&nbsp;for (param in m.parameters){&nbsp; 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;s += "parameter: " + param + " value: " + m.parameters[param] + "\r";&nbsp; 
&nbsp;&nbsp;} 
&nbsp;&nbsp;s</pre> 
       </div> 
       <div class="text parbase section"> 
        <a name="main-pars_text_15"></a> 
        <div class="text">
         <p>Because the XMP metadata in a footage item can be converted into layer markers for a layer based on that item, expressions can interact with XMP metadata. For information, see <a href="/after-effects/using/xmp-metadata.html#xmp_metadata_in_after_effects">XMP metadata in After Effects</a>. </p>
         <p>Dan Ebberts provides a tutorial on the <a href="http://www.adobe.com/devnet/aftereffects/">After Effects Developer Center</a> that includes an example of using XMP metadata with expressions.</p>
