        <p align="justify"></p>
        <p><b><font face="Segoe UI" size="2">Introduction</font></b></p> 
        <p><font face="Segoe UI" size="2">In today's article we discuss the<b>&nbsp;</b></font><strong style="outline: 0px; text-align: justify; color: rgb(51, 51, 51); line-height: 24px; font-family: &quot;Segoe UI&quot;; font-size: 13px; background-color: rgb(255, 255, 255);"><font size="2" style="outline: 0px;">Radix Sort In Java.</font></strong></p> 
        <p><strong><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">Radix Sort</font></span></strong></p> 
        <p><font face="Segoe UI"><span style="color: rgb(34, 34, 34); line-height: 18px; font-size: 13px; background-color: rgb(255, 255, 255);">A Radix Sort is a type of bucket sort. This type of sort is based on a radix. Arrays are sorted in a total of "d" passes where "d" is the number of digits of the highest number. In computer science, a Radix Sort is a non-comparative integer sorting algorithm that sorts data with integer keys by grouping keys by the individual digits that share the same significant position and value. A multiple pass distribution sort algorithm that distributes each item to a bucket according to part of the item's key beginning with the least significant part of the key. After each pass, items are collected from the buckets, keeping the items in order, then redistributed according to the next most significant part of the key. </span></font></p> 
        <p><font face="Segoe UI"><font color="#800000"><span style="line-height: 18px; font-size: 13px; background-color: rgb(255, 255, 255);">For decimal number</span></font><span style="color: rgb(34, 34, 34); line-height: 18px; font-size: 13px; background-color: rgb(255, 255, 255);">: In the first pass the array is sorted&nbsp;by the units digit, then in the next pass it is sorted&nbsp;by the tens digit and so on. This process goes on until all the digits of the highest numbers are traversed.<br></span><font color="#800000"><span style="line-height: 18px; font-size: 13px; background-color: rgb(255, 255, 255);">For words</span></font><span style="color: rgb(34, 34, 34); line-height: 18px; font-size: 13px; background-color: rgb(255, 255, 255);">: In the first pass the array is sorted&nbsp;by the first letter, and in the second pass the second character is considered and so on.</span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">A positional notation is required, but because integers can represent strings of characters (e.g., names or dates) and specially formatted floating point numbers, a Radix Sort is not limited to integers. </font></span></strong></p> 
        <p><b><font face="Segoe UI" size="2">Code Description</font></b></p> 
        <p><font face="Segoe UI" size="2">A Radix Sort is an algorithm that sorts a list of numbers and is in the distribution sort category. This sorting algorithm doesn't compare the numbers but distributes them, it works as follows:</font></p> 
        <ol> 
         <li><font face="Segoe UI" size="2">&nbsp;Sorting takes place by distributing the list of number into a bucket by passing through the individual digits of a given number one-by-one.</font></li> 
         <li><font face="Segoe UI" size="2">&nbsp;Beginning with the least significant digit (1st place). Here, the number of buckets are a total of ten (0 to 9).</font></li> 
         <li><font face="Segoe UI" size="2">&nbsp;Now start by arranging the values as in the example. The least value comes first and so on in increasing order.</font></li> 
         <li><font face="Segoe UI" size="2">&nbsp;After the first pass move to </font><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong>next digit </strong><strong style="font-weight: 400;">(10s place).</strong></span></font></li> 
         <li><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">&nbsp;Arrange those values in incresing order.</font></span></strong></li> 
         <li><font face="Segoe UI" size="2">&nbsp;After each pass, the numbers are collected from the buckets, keeping the numbers in order.</font></li> 
         <li><font face="Segoe UI" size="2">&nbsp;Now, recursively redistribute the numbers as in the step "1" above but with a following reconsideration: use the next most significant part of the number, that is then followed by step "2" above.</font></li>
        </ol> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong>An example </strong></span></font></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong style="font-weight: 400;">Original, </strong><strong>unsorted list:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#800000" face="Segoe UI" size="2">155, 34, 76, 80, 502, 150, 20, 4</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong>Sorting by the least significant</strong><strong style="font-weight: 400;"> digit (1s place) gives:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#800000" face="Segoe UI" size="2">8<u>0</u>, 15<u>0</u>, 2<u>0</u>,</font></span></strong><font color="#800000" face="Segoe UI" size="2"> 50<u>2</u>, 3<u>4</u>, <u>4</u>, 15<u>5</u>, 7<u>6</u> </font></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong>Sorting by the next digit </strong><strong style="font-weight: 400;">(10s place) gives:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#800000" face="Segoe UI" size="2">4, 5<u>0</u>2, <u>2</u>0, <u>3</u>4, 1<u>5</u>0, 1<u>5</u>5, <u>7</u>6, <u>8</u>0</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong>Sorting by the most significant digit </strong><strong style="font-weight: 400;">(100s place) gives a sorted array:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#800000" face="Segoe UI" size="2">4, 20, 34, 76, 80, <u>1</u>50, <u>1</u>55, <u>5</u>02</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);">It is important to realize that each of the steps above requires just a single pass over the data, since each item can be placed in its correct bucket without having to be compared with other items.<br>Some LSD Radix Sort implementations allocate space for buckets by first counting the number of keys that belong in each bucket before moving keys into those buckets. The number of times that each digit occurs is stored in an array. Consider the previous list of keys viewed in a different way:</span></strong></font></p> 
        <p><font color="#800000" face="Segoe UI" size="2">155, 034, 076, 080, 502, 150, 020, 004</font></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong style="font-weight: 400;">The </strong><strong>first counting pass</strong><strong style="font-weight: 400;"> starts on the least significant digit of each key, producing an array of bucket sizes:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">3 (bucket size for digits of 0: </font><font color="#800000" face="Segoe UI" size="2">080, 150, 020</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 2:</font><font color="#800000" face="Segoe UI" size="2"> 50<u>2</u></font><font color="#333333" face="Segoe UI" size="2">)<br>2 (bucket size for digits of 4: </font><font color="#800000" face="Segoe UI" size="2">03<u>4</u>, 00<u>4</u></font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 5: </font><font color="#800000" face="Segoe UI" size="2">15<u>5</u></font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 6: </font><font color="#800000" face="Segoe UI" size="2">07<u>6</u></font><font color="#333333" face="Segoe UI" size="2">)</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong style="font-weight: 400;">A </strong><strong>second counting pass</strong><strong style="font-weight: 400;"> on the next more significant digit of each key will produce an array of bucket sizes:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">2 (bucket size for digits of 0: </font><font color="#800000" face="Segoe UI" size="2">0<u>0</u>4, 5<u>0</u>2</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 2: </font><font color="#800000" face="Segoe UI" size="2">0<u>2</u>0</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 3:</font><font color="#800000" face="Segoe UI" size="2"> 0<u>3</u>4</font><font color="#333333" face="Segoe UI" size="2">)<br>2 (bucket size for digits of 5: </font><font color="#800000" face="Segoe UI" size="2">1<u>5</u>0, 1<u>5</u>5</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 7: </font><font color="#800000" face="Segoe UI" size="2">0<u>7</u>6</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 8: </font><font color="#800000" face="Segoe UI" size="2">0<u>8</u>0</font><font color="#333333" face="Segoe UI" size="2">)</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"><strong style="font-weight: 400;">A </strong><strong>third and final counting pass</strong><strong style="font-weight: 400;"> on the most significant digit of each key will produce an array of bucket sizes:</strong></span></font></p> 
        <p><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);"><font color="#333333" face="Segoe UI" size="2">5</font></span><font color="#333333" face="Segoe UI" size="2"><span style="background-color: rgb(255, 255, 255);"> (bucket size for digits of 0:</span></font><span style="background-color: rgb(255, 255, 255);"><font color="#800000" face="Segoe UI" size="2"> <u>0</u>04, <u>0</u>20, ,<u>0</u>34, <u>0</u>76, <u>0</u>80</font><font color="#333333" face="Segoe UI" size="2">)<br>2 (bucket size for digits of 1: </font><font color="#800000" face="Segoe UI" size="2"><u>1</u>50, <u>1</u>55</font><font color="#333333" face="Segoe UI" size="2">)<br>1 (bucket size for digits of 5: </font><font face="Segoe UI" size="2"><font color="#800000"><u>5</u>02</font>)</font></span></strong></p> 
        <p><font color="#333333" face="Segoe UI" size="2"><strong style="font-weight: 400;"><span style="background-color: rgb(255, 255, 255);">At least one LSD Radix Sort implementation now counts the number of times that each digit occurs in each column for all columns in a single counting pass. (See the external links section.) Other LSD Radix Sort implementations allocate space for buckets dynamically as the space is needed.</span></strong></font></p> 
        <p><font face="Segoe UI" size="2">A Radix Sort can be particularly effective on a linked list. Rather than buckets, put items in linked lists. At the end of a pass collect the items by "sewing" the lists together; make the tail of each list point to the head of the next list.</font></p> 
        <p><b><font face="Segoe UI" size="2">Algorithm of Radix Sort</font></b></p> 
        <p><font face="Segoe UI" size="2">Every algorithm always depends on the input. Now we see that the effectiveness of algorithms depends greatly on the input. For input that is almost sorted, an Insertion Sort may be preferred instead of a Quicksort,&nbsp;which is generally a faster algorithm. On the other hand, a Quicksort is considered one of the best general purpose sorting algorithms, but while it's a great algorithm when the data is randomized, it's practically as slow as a Bubble Sort when the input is almost or fully sorted.<br><br>Because the input is so important for an algorithm's efficiency, we may ask if there are any sorting algorithms that are faster than O(n.log(n)), which is the average-case complexity for a Merge Sort and a Quicksort. And the answer is yes there are faster, linear complexity algorithms, that can sort data faster than a Quicksort, Merge Sort and Heapsort. But there are some constraints!<br><br>Everything sounds great but we can't sort any particular data with linear complexity, so the question is what rules must the input follow in order to be sorted in linear time?<br><br>Such an algorithm that is capable of sorting data in linear O(n) time is a Radix Sort and the domain of the input is restricted; it must consist of only integers.</font></p> 
        <p><b><font face="Segoe UI" size="2">Overview</font></b></p>
