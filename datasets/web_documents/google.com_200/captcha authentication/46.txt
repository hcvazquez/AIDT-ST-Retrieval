               <p>This article will explain you to <b>salt hash passwords</b> using Node.js <b>Crypto</b>.<br> Here well not go into details comparing the pros and cons of different ways of storing passwords, rather well see how we can implement salt hashing mechanism for storing passwords in NodeJS. Stay rest assured that this is one of the better ways if not the best way to store passwords.</p> 
               <h3>What is this technique?</h3> 
               <p>Salt hashing is a technique in which we take the user entered password and a random string of characters called as salt, hash the combined string with a sutaible hashing algorithm and store the result in the database.</p> 
               <h3>Why Salt Hash?</h3> 
               <p>Since hashes of the same password are the same, they are much easier to crack using lookup tables and rainbow tables, here someone pre-calculates the hashes of common passwords and stores them for others to use. Having same password hash for two or more users also makes it easier for the attacker to predict the password. So as a rule of thumb, no two users should have the same password hash. Adding salt to a password and then hashing the result reduces the possibility of having duplicate hashes and if your salt length is long enough, chances are minimal. </p> 
               <h3>Practical Implementation</h3> 
               <p>Well in practice you would do the following.</p> 
               <h4>Creating and Storing password</h4> 
               <ol> 
                <li>Take the user password</li> 
                <li>Generate a salt(String of random characters)</li> 
                <li>Combine the salt with the user entered password</li> 
                <li>Hash the combined string with a suitable cryptographic algorithm.</li> 
                <li>Store the result as the password and also store the salt along side.</li> 
               </ol> 
               <h4>Validating user password</h4> 
               <ol> 
                <li>Validate the username and fetch the hashed result and salt from the database</li> 
                <li>Combine the user entered password with the salt stored for that user.</li> 
                <li>Hash the combined string with the same cryptographic algorithm used at the time of creating user.</li> 
                <li>Compare the result with the stored hash.</li> 
               </ol> 
               <p>In this article we wont be building a complete end to end application, instead we will see the code to <strong>salt-hash passwords using Node.js Crypto</strong><br> &nbsp;</p>

Mind you the emphasis here is to explain to the readers the concept of salt-hash with an example and not on the hashing algorithm used. And I dont see the above article explaining what salt hash is.
