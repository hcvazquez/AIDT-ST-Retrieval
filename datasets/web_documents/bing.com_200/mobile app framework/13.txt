       <div class="panel-pane pane-node-title"> 
        <h1>Top 15 Android-Ready Application Development Frameworks </h1> 
       </div> 
       <div class="panel-pane pane-entity-field pane-node-body"> 
        <div class="field field-name-body field-type-text-with-summary field-label-hidden">
         <div class="field-items">
          <div class="field-item even" property="content:encoded">
           <div class="article-content"> 
            <p>On July 20, Adobe unveiled version 2.0 of the open source PhoneGap, a leader among the growing crowd of cross-platform, Android-compatible, mobile app frameworks. Open source developers welcomed new PhoneGap features such as a "Cordova WebView" function that enables developers to integrate code into larger native applications.</p> 
            <p><img src="/sites/lcom/files/joomla/images/stories/41373/Cordova.jpg" width="450" height="243" alt="Cordova" title="Getting started in Android with Cordova, formerly known as PhoneGap." class="caption">There are scores of such frameworks to choose from these days, and as seen in our Top 15 list below, the best are getting quite sophisticated. Yet there is still considerable grumbling about the state of mobile cross-platform frameworks. They may be fine for the majority of Android apps being developed, yet few seem to be capable of handling all the requirements of a professional-quality enterprise or consumer app.</p> 
            <p>If you're familiar with Java and Eclipse, and Android is initially the sole destination, Google's Android SDK and related Android Development Tools (ADT) Eclipse plugin are probably the better choices. The problem is that most app publishers prefer to start with iOS, or else ship on iOS and Android simultaneously, with perhaps a BlackBerry or Windows Phone version as well. Others lack the experience to go native.</p> 
            <p>Nevertheless, the official tools deserve a look, as it's usually difficult to port one's cross-platform effort to the Android SDK in mid-stream. So we'll start with Google's tools first before moving on to the multi-platform frameworks.</p> 
            <h2>Google's Tools Get Friendlier</h2> 
            <p>Earlier this month, Google released <a href="http://android-developers.blogspot.com/2012/06/android-sdk-tools-revision-20.html">Revision 20</a>&nbsp;of the SDK and ADT in conjunction with Android 4.1 ("Jelly Bean"), adding new debugging tools, application templates, and performance tweaks. Other Google tools include a native development kit (NDK) for hardware optimization and the Android Accessory gadget control application development kit (ADK).</p> 
            <p>Android now offers an arguably superior platform to iOS. It provides much more flexibility, better app testing, and easier app approvals. Yet when it comes to overall ease of use, Apple may still have the edge. Although Android uses an easier, more widely known language in Java, Android's rich feature set and multitasking features are harder to master, and version- and device fragmentation can slow things down considerably.</p> 
            <p>There are no easy fixes for these problems, but Google has at least worked hard to reduce the learning curve. Last month it revamped its Android Developers website to make it more accessible, following up on its previous launch of an Android training program and the publication of a style guide.</p> 
            <h2>Cross-Platform Frameworks Duke it Out</h2> 
            <p>According to the <a href="http://ianskerrett.wordpress.com/2012/06/08/eclipse-community-survey-result-for-2012/">Eclipse Open Source Developer Report 2012</a>, 60 percent of open source developers writing Android or iOS apps use only the official SDK.&nbsp;Among those who use cross-platform frameworks, the choices, ranked from first to last were:</p> 
            <p>- jQuery Mobile (28.6 percent)</p> 
            <p>- PhoneGap (17.9)</p> 
            <p>- Sencha Touch (7.9)</p> 
            <p>- Dojo Mobile (4.9)</p> 
            <p>- Titanium (2.8).</p> 
            <p>These multi-platform options are typically open source JavaScript frameworks with support for HTML5 and CSS. Aimed primarily at web developers, they are often used for migrating website content to app form.</p> 
            <p>The frameworks support Android and iOS at a minimum, and often target BlackBerry, Windows Phone, and Symbian. Although they typically come with a "write once, run anywhere" promise, the amount of tweaking required for each version can still be considerable, and optimization of memory, battery life, and performance is often limited.</p> 
            <p>Most of the frameworks offer drag-and-drop GUI design tools, and many incorporate APIs aimed at exploiting specific components like audio and GPS. Quite a few are built on the Model View Controller (MVC) UI and component interaction model.</p> 
            <h2>Platforms for Coding Beyond JavaScript</h2> 
            <p>While PhoneGap and most of the leading frameworks are web-oriented HTML5/JavaScript apps that can tap native functions, some, like Appcelerator's Titanium, generate native Java code. While this approach has potential performance advantages, it tends to limit code reusability.</p> 
            <p>Some frameworks specialize in programming environments beyond JavaScript. For example:</p> 
            <p>-&nbsp;&nbsp; RhoMobile is designed for Ruby developers.</p> 
            <p>-&nbsp;&nbsp; MoSync is aimed at C++ hackers.</p> 
            <p>-&nbsp;&nbsp; C#-flavored Mono for Android appeals to enterprise-focused developers familiar with Visual Studio.</p> 
            <p>Game developers, meanwhile, tend to use gaming-focused frameworks that offer specialized level-creation features, game engines, and 3D animation support:</p> 
            <p>-&nbsp;&nbsp; Corona SDK (the main player here)</p> 
            <p>-&nbsp;&nbsp; Flixel</p> 
            <p>-&nbsp;&nbsp; Unity3D.</p> 
            <p>Educational organizations and others with limited needs and limited resources can turn to codeless, cross-platform app-building environments like TheAppBuilder. Google abandoned its own codeless App Inventor, but it was recently re-launched by MIT as the <a href="http://explore.appinventor.mit.edu/">MIT App Inventor</a>.</p> 
            <h2>Other Android-Ready Options</h2> 
            <p>On the opposite extreme, those aiming to develop complex applications, often in conjunction with new hardware, may prefer the robust, commercial Android-compatible platforms from embedded Linux OS vendors like Wind River, MontaVista, and Mentor Graphics. These are especially useful when targeting form factors beyond smartphone and tablets. The <a href="http://www.windriver.com/products/mobile/">Wind River Platform for Android </a>run-time environment includes an optimized Android SDK, middleware, device drivers, and testing suites, as well as vertical market accelerators.</p> 
            <p>Those looking to optimize their apps for particular processors can also find Android-ready tools from major ARM semiconductor vendors like Texas Instruments and Freescale, as well as ARM itself. Open source <a href="/news/embedded-mobile/mobile-linux/594494-beyond-the-beagle-seven-open-source-board-projects-that-count">development board projects</a> offer similar tools. MIPS and Intel, meanwhile, are building Android tools to support their own respective architectures.</p> 
            <p>Other Android-ready tools focus on particular steps in the development process. These include:</p> 
            <p>-&nbsp;&nbsp; Testing (Testroid, Appthwack)</p> 
            <p>-&nbsp;&nbsp; Performance management (Crittercism NDK)</p> 
            <p>-&nbsp;&nbsp; GUI design (DroidDraw, SPB UI Builder).</p> 
            <p>The latter category includes GUI tools from Motorola, HTC, and Samsung, designed for their respective <a href="/news/embedded-mobile/mobile-linux/577965-android-skins-refuse-to-go-away-">UI skins</a>. Finally, new <a href="/news/embedded-mobile/mobile-linux/591545-open-source-android-push-evolution-of-mobile-cloud-apps">cloud-oriented tools</a> such as OpenMobster and Cumulus provide sync and other cloud support for Android apps.</p> 
            <h2>15 Android-Ready Development Frameworks</h2> 
            <p>The following are 15 of the more popular Android development tools. Unless otherwise noted, they are open source, cross-platform frameworks:</p> 
            <p><strong>Basic4android:</strong>&nbsp;Anywhere Software's commercial RAD tool and IDE for Android provides a comprehensive feature set and an object-oriented programming language similar to Visual Basic.</p> 
            <p><a href="http://www.basic4ppc.com/android/why.html">http://www.basic4ppc.com/android/why.html</a></p> 
            <p><strong>Corona SDK:&nbsp;</strong>Widely used among game developers, Corona is also a popular, general-purpose framework. Corona Labs (formerly Ansca Mobile) claims an installed base of 120,000 developers. This high-end, commercial SDK offers over 500 APIs, as well as advertising and native UI support, and a built-in physics engine.</p> 
            <p><a href="http://www.coronalabs.com/">http://www.coronalabs.com/</a></p> 
            <p><strong>DHTMLX Touch:</strong>&nbsp;This JavaScript and AJAX library focuses on UI widgets, and is aimed at building HTML5-based apps.</p> 
            <p><a href="http://www.dhtmlx.com/touch/">http://www.dhtmlx.com/touch/</a></p> 
            <p><strong>Dojo Mobile:</strong>&nbsp;The Dojo community's BSD-licensed HTML5/JavaScript framework has added MVC and app-controller packages, as well as mobile-specific components such as switches and sliders. A degree of PhoneGap compatibility is also available.</p> 
            <p><a href="http://dojotoolkit.org/features/mobile">http://dojotoolkit.org/features/mobile</a></p> 
            <p><strong>iUI:</strong>&nbsp;This lightweight web UI framework includes a JavaScript library, CSS support, and development images.</p> 
            <p><a href="http://www.iui-js.org/">http://www.iui-js.org/</a></p> 
            <p><strong>jQuery Mobile:</strong>&nbsp;This popular, lightweight HTML5-based framework is built on jQuery, and focuses on semantic markup, progressive enhancement, and themable design. It's the leading cross-platform framework among Eclipse open source developers.</p> 
            <p><a href="http://jquerymobile.com/">http://jquerymobile.com/</a></p> 
            <p><strong>Kendo UI:</strong>&nbsp;Telerik's HTML5/JavaScript framework is available in open source and commercial versions. Kendo UI offers a wide selection of UI widgets and plugins, and provides an MVVM framework, performance optimization, and validation and internationalization features.</p> 
            <p><a href="http://www.kendoui.com/">http://www.kendoui.com/</a></p> 
            <p><strong>Mono for Android:</strong>&nbsp;Xamarin's C#- and enterprise-oriented package is compatible with a similar iOS-based MonoTouch version, and can also share code with the C#-based Windows Phone. Mono supplies an environment conducive to Visual Basic developers, and is touted for its debugger and native binary compiler.</p> 
            <p><a href="http://xamarin.com/">http://xamarin.com/</a></p> 
            <p><strong>MoSync SDK:</strong>&nbsp;MoSync supports C++, HTML5/JavaScript, or both on up to nine different platforms. The SDK is touted as being compatible with PhoneGap, as well MoSync's own new HTML5/JavaScript-based native mobile app developer/simulator, MoSync Reload.</p> 
            <p><a href="http://www.mosync.com/">http://www.mosync.com/</a></p> 
            <p><strong>PhoneGap:</strong>&nbsp;Designed for JavaScript, HTML5, and CSS development, PhoneGap is now sponsored by Adobe and the Apache Foundation. The 2.0 version adds Windows Phone support, new CLI functions, and overhauled JavaScript libraries. It also debuts Cordova WebView, an embeddable HTML rendering control that uses Apache's Cordova-JS API for tasks such as integrating PhoneGap code into larger native apps.</p> 
            <p><a href="http://phonegap.com/">http://phonegap.com/</a></p> 
            <p><strong>RhoMobile Suite:</strong>&nbsp;Motorola's mature, business-oriented framework features RhoConnect, RhoStudio, RhoElements, and a new RhoHub used for cloud app-building. RhoMobile is built on the Ruby language, the Rails Frameworks, and the MVC model.</p> 
            <p><a href="http://www.motorola.com/Business/US-EN/Business+Product+and+Services/Software+and+Applications/RhoMobile+Suite">http://www.motorola.com/Business/US-EN/Business+Product+and+Services/Software+and+Applications/RhoMobile+Suite</a></p> 
            <p><strong>Sencha Touch 2:</strong>&nbsp;Sencha's popular HTML5/JavaScript framework provides 50 built-in components, state management, and an integrated MVC system. It now offers a free native packager that streamlines distribution to stores like Google Play.</p> 
            <p><a href="http://www.sencha.com/products/touch/">http://www.sencha.com/products/touch/</a></p> 
            <p><strong>SproutCore:</strong>&nbsp;This HTML5-driven framework offers a "clean" MVC architecture, and emphasizes performance optimization and scalability.</p> 
            <p><a href="http://sproutcore.com/">http://sproutcore.com/</a></p> 
            <p><strong>TheAppBuilder:</strong>&nbsp;JamPot's new HTML5-based native app-building app has received plenty of buzz. It features a codeless, drag-and-drop interface that lets users quickly build fairly rudimentary apps by filling in Q&amp;A checklists. Highlights include extensive social networking integration and automated submissions to Google Play.</p> 
            <p><a href="http://www.theappbuilder.com/">http://www.theappbuilder.com/</a></p> 
            <p><strong>Titanium:</strong>&nbsp;Appcelerator claims its Android/iOS framework supports over 5,000 device and mobile-OS APIs. Unlike the more web-oriented frameworks, Titanium uses JavaScript to create native code, with claimed benefits in performance.</p> 
            <p><a href="http://www.appcelerator.com/platform/titanium-sdk">http://www.appcelerator.com/platform/titanium-sdk</a></p> 
            <p>Additional Android-compatible development options include Andromo, Application Craft, Hypernext Android Creator (HAC), Jo, jQTouch, MIT App Inventor, Togosoft Device Browser, Unity Mobile, WebApp.Net, Wink Toolkit, xUI, and Zepto.js. For more options, check out these roundups of Android development software from <a href="http://buildmobile.com/android-development-do-you-know-your-options/#fbid=-4eQmhlnOOF">BuildMobile</a>, <a href="http://dailytekk.com/2012/04/02/100-tools-to-develop-a-killer-ios-or-android-app/">Daily Tekk</a>, <a href="http://www.mobigeni.com/2011/08/23/20-mobile-frameworks-to-build-apps-for-ios-android-blackberry-and-more/">MobiGeni</a>, and <a href="http://setandbma.wordpress.com/2011/12/06/wora-platform-comparison/">Technology Trend Analysis</a>. Meanwhile, post your own favorites in the comments section below.</p>
