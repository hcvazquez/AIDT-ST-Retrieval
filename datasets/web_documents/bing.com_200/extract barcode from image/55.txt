       <p>The <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html" rel="nofollow">Javadoc documentation for the Package class</a> states:</p>
       <blockquote>
        Package objects contain version information about the implementation and specification of a Java package. This versioning information is retrieved and made available by the ClassLoader instance that loaded the class(es). Typically, it is stored in the manifest that is distributed with the classes.
       </blockquote>
       <p>In this blog post, I look at how Java code can use the Package class to access version information about a particular Java package's specification and implementation.</p>
       <p> The <a href="http://download.oracle.com/javase/tutorial/" rel="nofollow">Java Tutorials</a>' <a href="http://download.oracle.com/javase/tutorial/deployment/TOC.html" rel="nofollow">Deployment Trail</a> features the <a href="http://download.oracle.com/javase/tutorial/deployment/jar/" rel="nofollow">Packaging Programs in JAR Files</a> lesson that includes the relevant section <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html" rel="nofollow">Setting Package Version Information</a>. That <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html" rel="nofollow">short page</a> demonstrates how to <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html" rel="nofollow">provide package version information</a> inside a <a href="http://download.oracle.com/javase/tutorial/deployment/jar/basicsindex.html" rel="nofollow">JAR</a>'s <a href="http://download.oracle.com/javase/tutorial/deployment/jar/manifestindex.html" rel="nofollow">manifest file</a>.</p>
       <p> The <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html" rel="nofollow">Setting Package Version Information page</a> indicates that seven pieces of information (seven lines in the manifest file) can be specified for package version information. The first line of the seven is the name of the package and is indicated by <code>Name</code>. The other six pieces of version information that can be specified are title, version, and vendor for the specification and the same title, version, and vendor for the implementation. An example of what this might look like is shown next in this example <code>MANIFEST.MF</code> file.</p>
       <aside class="nativo-promo smartphone" id=""> 
       </aside>
       <p> <strong>Example MANIFEST.MF File With Package Version Information</strong></p>
       <pre class="prettyprint"><code>Name: dustin/examples/
Specification-Title: Dustin's Java/Groovy Examples
Specification-Version: 2011.06.17
Specification-Vendor: Marxian Software
Implementation-Title: Dustin Examples
Implementation-Version: 2011.06.17.02
Implementation-Vendor: Marxian Software
</code></pre>
       <p> In the above example, the various version information is provided for the package <code>dustin.examples</code> because that (<code>dustin/examples/</code>) is listed after <code>Name: </code>.</p>
       <p> These manifest entries can be created in an <a href="http://ant.apache.org/" rel="nofollow">Ant</a> build as shown in the next excerpt from an <a href="http://ant.apache.org/" rel="nofollow">Ant</a> <code>build.xml</code> file.</p>
       <p> <strong>Excerpt from build.xml for Building JAR with Manifest with Package Version Information</strong></p>

       <p> It is very easy to access the package information at this point. Use of <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html#getPackages()" rel="nofollow">Package.getPackages()</a> to get all packages associated with the current class loader and <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html#getPackage(java.lang.String)" rel="nofollow">Package.getPackage(String)</a> to get a particular package by name from the current class loader are demonstrated in the next code listing for <code>Main.java</code>.</p>
       <p> <strong>Main.java</strong></p>
       <pre class="prettyprint"><code>package dustin.examples;

import static java.lang.System.out;

/**
 * Simple class demonstrating in-code access of package-specific information as
 * specified in the Manifest file.
 */
public class Main
{
   /**
    * Display (to standard output) package details for provided Package.
    *
    * @param pkg Package whose details need to be printed to standard output.
    */
   private void displayPackageDetails(final Package pkg)
   {
      final String name = pkg.getName();
      out.println(name);
      out.println("\tSpec Title/Version: " + pkg.getSpecificationTitle() + " " + pkg.getSpecificationVersion());
      out.println("\tSpec Vendor: " +  pkg.getSpecificationVendor());
      out.println("\tImplementation: " + pkg.getImplementationTitle() + " " + pkg.getImplementationVersion());
      out.println("\tImplementation Vendor: " + pkg.getImplementationVendor());
   }

   /**
    * Display all packages associated with the class loader that do not start
    * with "sun", "com", "java", or "org".
    */
   private void displayCustomPackage()
   {
      final Package[] packages = Package.getPackages();
      for (final Package pkg : packages)
      {
         final String name = pkg.getName();
         if (   !name.startsWith("sun") &amp;&amp; !name.startsWith("java")
             &amp;&amp; !name.startsWith("com") &amp;&amp; !name.startsWith("org") )
         {
            displayPackageDetails(pkg);
         }
      }
   }

   /**
    * Main executable that demonstrates use of Package.getPackages() and
    * Package.getPackage(String).
    *
    * @param arguments Command-line arguments; none expected.
    */
   public static void main(final String[] arguments)
   {
      final Main me = new Main();
      me.displayCustomPackage();
      me.displayPackageDetails(Package.getPackage("dustin.examples"));
   }
}
</code></pre>
       <p> The output from running the above is shown next.</p>
       <p><a href="http://4.bp.blogspot.com/-TT5BSlUbhM4/TgA1buoPHzI/AAAAAAAACSc/K2wrPbQcvCI/s1600/outputExecutingPackageMethodsOnPackageDescribedVersioningInManifest.png" imageanchor="1" rel="nofollow"></a></p>
       <figure>
        <a href="http://4.bp.blogspot.com/-TT5BSlUbhM4/TgA1buoPHzI/AAAAAAAACSc/K2wrPbQcvCI/s1600/outputExecutingPackageMethodsOnPackageDescribedVersioningInManifest.png" imageanchor="1" rel="nofollow"><img border="0" height="104" src="http://4.bp.blogspot.com/-TT5BSlUbhM4/TgA1buoPHzI/AAAAAAAACSc/K2wrPbQcvCI/s320/outputExecutingPackageMethodsOnPackageDescribedVersioningInManifest.png" width="320"></a>
       </figure>
       <p></p>
       <p> The output shows that the package information described in the MANIFEST.MF's file is available to Java code. I intentionally included three attributes in the MANIFEST.MF example above that are not accessible from Java's <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html" rel="nofollow">Package</a> class. These are <code>Package-Title</code>, <code>Package-Version</code>, and <code>Package-Vendor</code>. The <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html" rel="nofollow">Setting Package Version Information page</a> links to the <a href="http://download.oracle.com/javase/6/docs/technotes/guides/versioning/spec/versioning2.html#wp89936" rel="nofollow">Package Version Specification</a>, which has a section <a href="http://download.oracle.com/javase/6/docs/technotes/guides/versioning/spec/versioning2.html#wp91706" rel="nofollow">1.5.8 JAR Manifest Format</a> that shows use of these three attributes beginning with <code>Package-</code> rather than the three I showed earlier beginning with <code>Implementation-</code>. My example intentionally included both sets with different values for each to prove that its the three <code>Implementation-</code> attributes that are available via the <code>Package</code> class rather than the three <code>Package-</code> attributes.</p>
       <p> <strong>Conclusion</strong></p>
       <p> I demonstrated in this blog post how to use Java's <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html" rel="nofollow">Package class</a> to access version information about a particular Java package. To demonstrate this, I used Ant to specify the package version information in a JAR's MANIFEST.MF file as part of the <a href="http://ant.apache.org/manual/Tasks/javac.html" rel="nofollow">javac task</a>.</p>
