         <p>[1] In this library, Levenshtein edit distance, LCS distance and their sibblings are computed using the <strong>dynamic programming</strong> method, which has a cost O(m.n). For Levenshtein distance, the algorithm is sometimes called <strong>Wagner-Fischer algorithm</strong> ("The string-to-string correction problem", 1974). The original algorithm uses a matrix of size m x n to store the Levenshtein distance between string prefixes.</p> 
         <p>If the alphabet is finite, it is possible to use the <strong>method of four russians</strong> (Arlazarov et al. "On economic construction of the transitive closure of a directed graph", 1970) to speedup computation. This was published by Masek in 1980 ("A Faster Algorithm Computing String Edit Distances"). This method splits the matrix in blocks of size t x t. Each possible block is precomputed to produce a lookup table. This lookup table can then be used to compute the string similarity (or distance) in O(nm/t). Usually, t is choosen as log(m) if m &gt; n. The resulting computation cost is thus O(mn/log(m)). This method has not been implemented (yet).</p> 
         <p>[2] In "Length of Maximal Common Subsequences", K.S. Larsen proposed an algorithm that computes the length of LCS in time O(log(m).log(n)). But the algorithm has a memory requirement O(m.n²) and was thus not implemented here.</p> 
         <p>[3] There are two variants of Damerau-Levenshtein string distance: Damerau-Levenshtein with adjacent transpositions (also sometimes called unrestricted Damerau–Levenshtein distance) and Optimal String Alignment (also sometimes called restricted edit distance). For Optimal String Alignment, no substring can be edited more than once.</p>

           </svg></a>Normalized, metric, similarity and distance</h2> 
         <p>Although the topic might seem simple, a lot of different algorithms exist to measure text similarity or distance. Therefore the library defines some interfaces to categorize them.</p> 
         <h3><a id="user-content-normalized-similarity-and-distance" class="anchor" href="#normalized-similarity-and-distance" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>(Normalized) similarity and distance</h3> 
         <ul> 
          <li>StringSimilarity : Implementing algorithms define a similarity between strings (0 means strings are completely different).</li> 
          <li>NormalizedStringSimilarity : Implementing algorithms define a similarity between 0.0 and 1.0, like Jaro-Winkler for example.</li> 
          <li>StringDistance : Implementing algorithms define a distance between strings (0 means strings are identical), like Levenshtein for example. The maximum distance value depends on the algorithm.</li> 
          <li>NormalizedStringDistance : This interface extends StringDistance. For implementing classes, the computed distance value is between 0.0 and 1.0. NormalizedLevenshtein is an example of NormalizedStringDistance.</li> 
         </ul> 
         <p>Generally, algorithms that implement NormalizedStringSimilarity also implement NormalizedStringDistance, and similarity = 1 - distance. But there are a few exceptions, like N-Gram similarity and distance (Kondrak)...</p> 
         <h3><a id="user-content-metric-distances" class="anchor" href="#metric-distances" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Metric distances</h3> 
         <p>The MetricStringDistance interface : A few of the distances are actually metric distances, which means that verify the triangle inequality d(x, y) &lt;= d(x,z) + d(z,y). For example, Levenshtein is a metric distance, but NormalizedLevenshtein is not.</p> 
         <p>A lot of nearest-neighbor search algorithms and indexing structures rely on the triangle inequality. You can check "Similarity Search, The Metric Space Approach" by Zezula et al. for a survey. These cannot be used with non metric similarity measures.</p> 
         <p><a href="http://www.javadoc.io/doc/info.debatty/java-string-similarity">Read Javadoc for a detailed description</a></p> 
         <h2><a id="user-content-shingles-n-gram-based-similarity-and-distance" class="anchor" href="#shingles-n-gram-based-similarity-and-distance" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Shingles (n-gram) based similarity and distance</h2> 
         <p>A few algorithms work by converting strings into sets of n-grams (sequences of n characters, also sometimes called k-shingles). The similarity or distance between the strings is then the similarity or distance between the sets.</p> 
         <p>Some of them, like jaccard, consider strings as sets of shingles, and don't consider the number of occurences of each shingle. Others, like cosine similarity, work using what is sometimes called the profile of the strings, which takes into account the number of occurences of each shingle.</p> 
         <p>For these algorithms, another use case is possible when dealing with large datasets:</p> 
         <ol> 
          <li>compute the set or profile representation of all the strings</li> 
          <li>compute the similarity between sets or profiles</li> 
         </ol> 
         <h2><a id="user-content-levenshtein" class="anchor" href="#levenshtein" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Levenshtein</h2> 
         <p>The Levenshtein distance between two words is the minimum number of single-character edits (insertions, deletions or substitutions) required to change one word into the other.</p> 
         <p>It is a metric string distance. This implementation uses dynamic programming (Wagner–Fischer algorithm), with only 2 rows of data. The space requirement is thus O(m) and the algorithm runs in O(m.n).</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.*</span>;

<span class="pl-k">public</span> <span class="pl-k">class</span> <span class="pl-en">MyApp</span> {
    
    <span class="pl-k">public</span> <span class="pl-k">static</span> <span class="pl-k">void</span> <span class="pl-en">main</span> (<span class="pl-k">String</span>[] <span class="pl-v">args</span>) {
        <span class="pl-smi">Levenshtein</span> l <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">Levenshtein</span>();

         <h2><a id="user-content-longest-common-subsequence" class="anchor" href="#longest-common-subsequence" aria-hidden="true">

           </svg></a>Longest Common Subsequence</h2> 
         <p>The longest common subsequence (LCS) problem consists in finding the longest subsequence common to two (or more) sequences. It differs from problems of finding common substrings: unlike substrings, subsequences are not required to occupy consecutive positions within the original sequences.</p> 
         <p>It is used by the diff utility, by Git for reconciling multiple changes, etc.</p> 
         <p>The LCS distance between strings X (of length n) and Y (of length m) is n + m - 2 |LCS(X, Y)| min = 0 max = n + m</p> 
         <p>LCS distance is equivalent to Levenshtein distance when only insertion and deletion is allowed (no substitution), or when the cost of the substitution is the double of the cost of an insertion or deletion.</p> 
         <p>This class implements the dynamic programming approach, which has a space requirement O(m.n), and computation cost O(m.n).</p> 
         <p>In "Length of Maximal Common Subsequences", K.S. Larsen proposed an algorithm that computes the length of LCS in time O(log(m).log(n)). But the algorithm has a memory requirement O(m.n²) and was thus not implemented here.</p>

          <pre><span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.*</span>;

        <span class="pl-smi">LongestCommonSubsequence</span> lcs <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">LongestCommonSubsequence</span>();

        <span class="pl-c"><span class="pl-c">//</span> Will produce 4.0</span>

           </svg></a>Metric Longest Common Subsequence</h2> 
         <p>Distance metric based on Longest Common Subsequence, from the notes "An LCS-based string metric" by Daniel Bakkelund. <a href="http://heim.ifi.uio.no/%7Edanielry/StringMetric.pdf">http://heim.ifi.uio.no/~danielry/StringMetric.pdf</a></p> 
         <p>The distance is computed as 1 - |LCS(s1, s2)| / max(|s1|, |s2|)</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">public</span> <span class="pl-k">class</span> <span class="pl-en">MyApp</span> {

        <span class="pl-k">public</span> <span class="pl-k">static</span> <span class="pl-k">void</span> <span class="pl-en">main</span>(<span class="pl-k">String</span>[] <span class="pl-v">args</span>) {

        <span class="pl-smi">info.debatty.java.stringsimilarity<span class="pl-k">.</span>MetricLCS</span> lcs <span class="pl-k">=</span> 
                <span class="pl-k">new</span> <span class="pl-smi">info.debatty.java.stringsimilarity<span class="pl-k">.</span>MetricLCS</span>();

        <span class="pl-smi">String</span> s1 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>ABCDEFG<span class="pl-pds">"</span></span>;   
        <span class="pl-smi">String</span> s2 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>ABCDEFHJKL<span class="pl-pds">"</span></span>;
        <span class="pl-c"><span class="pl-c">//</span> LCS: ABCDEF =&gt; length = 6</span>
        <span class="pl-c"><span class="pl-c">//</span> longest = s2 =&gt; length = 10</span>
        <span class="pl-c"><span class="pl-c">//</span> =&gt; 1 - 6/10 = 0.4</span>
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(lcs<span class="pl-k">.</span>distance(s1, s2));

        <span class="pl-c"><span class="pl-c">//</span> LCS: ABDF =&gt; length = 4</span>
        <span class="pl-c"><span class="pl-c">//</span> longest = ABDEF =&gt; length = 5</span>
        <span class="pl-c"><span class="pl-c">//</span> =&gt; 1 - 4 / 5 = 0.2</span>
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(lcs<span class="pl-k">.</span>distance(<span class="pl-s"><span class="pl-pds">"</span>ABDEF<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>ABDIF<span class="pl-pds">"</span></span>));
    }
}</pre>
         </div> 
         <h2><a id="user-content-n-gram" class="anchor" href="#n-gram" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>N-Gram</h2> 
         <p>Normalized N-Gram distance as defined by Kondrak, "N-Gram Similarity and Distance", String Processing and Information Retrieval, Lecture Notes in Computer Science Volume 3772, 2005, pp 115-126.</p> 
         <p><a href="http://webdocs.cs.ualberta.ca/%7Ekondrak/papers/spire05.pdf">http://webdocs.cs.ualberta.ca/~kondrak/papers/spire05.pdf</a></p> 
         <p>The algorithm uses affixing with special character '\n' to increase the weight of first characters. The normalization is achieved by dividing the total similarity score the original length of the longest word.</p> 
         <p>In the paper, Kondrak also defines a similarity measure, which is not implemented (yet).</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.*</span>;

<span class="pl-k">public</span> <span class="pl-k">class</span> <span class="pl-en">MyApp</span> {

    <span class="pl-k">public</span> <span class="pl-k">static</span> <span class="pl-k">void</span> <span class="pl-en">main</span>(<span class="pl-k">String</span>[] <span class="pl-v">args</span>) {
        
        <span class="pl-c"><span class="pl-c">//</span> produces 0.416666</span>
        <span class="pl-smi">NGram</span> twogram <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">NGram</span>(<span class="pl-c1">2</span>);
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(twogram<span class="pl-k">.</span>distance(<span class="pl-s"><span class="pl-pds">"</span>ABCD<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>ABTUIO<span class="pl-pds">"</span></span>));
        
        <span class="pl-c"><span class="pl-c">//</span> produces 0.97222</span>
        <span class="pl-smi">String</span> s1 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>Adobe CreativeSuite 5 Master Collection from cheap 4zp<span class="pl-pds">"</span></span>;
        <span class="pl-smi">String</span> s2 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>Adobe CreativeSuite 5 Master Collection from cheap d1x<span class="pl-pds">"</span></span>;
        <span class="pl-smi">NGram</span> ngram <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">NGram</span>(<span class="pl-c1">4</span>);
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(ngram<span class="pl-k">.</span>distance(s1, s2));
    }
}</pre>
         </div> 
         <h2><a id="user-content-shingle-n-gram-based-algorithms" class="anchor" href="#shingle-n-gram-based-algorithms" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Shingle (n-gram) based algorithms</h2> 
         <p>A few algorithms work by converting strings into sets of n-grams (sequences of n characters, also sometimes called k-shingles). The similarity or distance between the strings is then the similarity or distance between the sets.</p> 
         <p>The cost for computing these similarities and distances is mainly domnitated by k-shingling (converting the strings into sequences of k characters). Therefore there are typically two use cases for these algorithms:</p> 
         <p>Directly compute the distance between strings:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.*</span>;

<span class="pl-k">public</span> <span class="pl-k">class</span> <span class="pl-en">MyApp</span> {
    
    <span class="pl-k">public</span> <span class="pl-k">static</span> <span class="pl-k">void</span> <span class="pl-en">main</span>(<span class="pl-k">String</span>[] <span class="pl-v">args</span>) {
        <span class="pl-smi">QGram</span> dig <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">QGram</span>(<span class="pl-c1">2</span>);
        
        <span class="pl-c"><span class="pl-c">//</span> AB BC CD CE</span>
        <span class="pl-c"><span class="pl-c">//</span> 1  1  1  0</span>
        <span class="pl-c"><span class="pl-c">//</span> 1  1  0  1</span>
        <span class="pl-c"><span class="pl-c">//</span> Total: 2</span>

        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(dig<span class="pl-k">.</span>distance(<span class="pl-s"><span class="pl-pds">"</span>ABCD<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>ABCE<span class="pl-pds">"</span></span>));
    }
}</pre>
         </div> 
         <p>Or, for large datasets, pre-compute the profile or set representation of all strings. The similarity can then be computed between profiles or sets:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.KShingling</span>;
<span class="pl-k">import</span> <span class="pl-smi">info.debatty.java.stringsimilarity.StringProfile</span>;


<span class="pl-k">public</span> <span class="pl-k">class</span> <span class="pl-en">PrecomputedCosine</span> {

    <span class="pl-c"><span class="pl-c">/*</span>*</span>
<span class="pl-c">     * @param args the command line arguments</span>
<span class="pl-c">     <span class="pl-c">*/</span></span>
    <span class="pl-k">public</span> <span class="pl-k">static</span> <span class="pl-k">void</span> <span class="pl-en">main</span>(<span class="pl-k">String</span>[] <span class="pl-v">args</span>) <span class="pl-k">throws</span> <span class="pl-smi">Exception</span> {
        <span class="pl-smi">String</span> s1 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>My first string<span class="pl-pds">"</span></span>;
        <span class="pl-smi">String</span> s2 <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>My other string...<span class="pl-pds">"</span></span>;
        
        <span class="pl-c"><span class="pl-c">//</span> Let's work with sequences of 2 characters...</span>
        <span class="pl-smi">KShingling</span> ks <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-smi">KShingling</span>(<span class="pl-c1">2</span>);
        
        <span class="pl-c"><span class="pl-c">//</span> For cosine similarity I need the profile of strings</span>
        <span class="pl-smi">StringProfile</span> profile1 <span class="pl-k">=</span> ks<span class="pl-k">.</span>getProfile(s1);
        <span class="pl-smi">StringProfile</span> profile2 <span class="pl-k">=</span> ks<span class="pl-k">.</span>getProfile(s2);
        
        <span class="pl-c"><span class="pl-c">//</span> Prints 0.516185</span>
        <span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(profile1<span class="pl-k">.</span>cosineSimilarity(profile2));
        
    }
    
}</pre>
         </div> 
         <p>Pay attention, this only works if the same KShingling object is used to parse all input strings !</p> 
         <h3><a id="user-content-q-gram" class="anchor" href="#q-gram" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Q-Gram</h3> 
         <p>Q-gram distance, as defined by Ukkonen in "Approximate string-matching with q-grams and maximal matches" <a href="http://www.sciencedirect.com/science/article/pii/0304397592901434">http://www.sciencedirect.com/science/article/pii/0304397592901434</a></p> 
         <p>The distance between two strings is defined as the L1 norm of the difference of their profiles (the number of occurences of each n-gram): SUM( |V1_i - V2_i| ). Q-gram distance is a lower bound on Levenshtein distance, but can be computed in O(m + n), where Levenshtein requires O(m.n)</p> 
         <h3><a id="user-content-cosine-similarity" class="anchor" href="#cosine-similarity" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Cosine similarity</h3> 
         <p>The similarity between the two strings is the cosine of the angle between these two vectors representation, and is computed as V1 . V2 / (|V1| * |V2|)</p> 
         <p>Distance is computed as 1 - cosine similarity.</p> 
         <h3><a id="user-content-jaccard-index" class="anchor" href="#jaccard-index" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Jaccard index</h3> 
         <p>Like Q-Gram distance, the input strings are first converted into sets of n-grams (sequences of n characters, also called k-shingles), but this time the cardinality of each n-gram is not taken into account. Each input string is simply a set of n-grams. The Jaccard index is then computed as |V1 inter V2| / |V1 union V2|.</p> 
         <p>Distance is computed as 1 - cosine similarity. Jaccard index is a metric distance.</p> 
         <h3><a id="user-content-sorensen-dice-coefficient" class="anchor" href="#sorensen-dice-coefficient" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Sorensen-Dice coefficient</h3> 
         <p>Similar to Jaccard index, but this time the similarity is computed as 2 * |V1 inter V2| / (|V1| + |V2|).</p> 
         <p>Distance is computed as 1 - cosine similarity.</p> 
         <h2><a id="user-content-experimental" class="anchor" href="#experimental" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Experimental</h2> 
         <h3><a id="user-content-sift4" class="anchor" href="#sift4" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>SIFT4</h3> 
         <p>SIFT4 is a general purpose string distance algorithm inspired by JaroWinkler and Longest Common Subsequence. It was developed to produce a distance measure that matches as close as possible to the human perception of string distance. Hence it takes into account elements like character substitution, character distance, longest common subsequence etc. It was developed using experimental testing, and without theoretical background.</p> 
         <pre><code>import info.debatty.java.stringsimilarity.experimental.Sift4;

public class MyApp {

    public static void main(String[] args) {
        String s1 = "This is the first string";
        String s2 = "And this is another string";
        Sift4 sift4 = new Sift4();
        sift4.setMaxOffset(5);
        double expResult =  11.0;
        double result = sift4.distance(s1, s2);
        assertEquals(expResult, result, 0.0);
    }
}
</code></pre>
