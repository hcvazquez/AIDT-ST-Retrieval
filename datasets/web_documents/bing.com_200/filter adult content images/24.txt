    <h2 id="background-of-the-eclipse-project"><a class="link" href="#background-of-the-eclipse-project">1. Background of the Eclipse project</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="the-eclipse-ide-other-projects-and-a-short-history"><a class="link" href="#the-eclipse-ide-other-projects-and-a-short-history">1.1. The Eclipse IDE, other projects and a short history</a></h3> 
      <div class="paragraph"> 
       <p>While the Eclipse IDE for Java development is widely known, the Eclipse open source community is much bigger. Currently it consists of more than 150 projects covering different aspects of software development. For example, Eclipse projects provide tooling for JavaScript development and runtime components, like the popular <em>Jetty</em> webserver.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The roots of Eclipse go back to 2001. The initial code base was provided by IBM. In November 2001, a consortium was formed to support the development of Eclipse as open source software. This consortium was controlled by IBM.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In 2004 it became the <em>Eclipse Foundation</em>, which is a vendor neutral foundation where no single company has control of the direction.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The <em>Eclipse</em> name at this time was viewed by many as declaration of war against <em>Sun Microsystems</em>, the company responsible for developing the Java programming language. IBM stated that the name was targeting at "Eclipsing" Microsoft. See <a href="http://www.eweek.com/c/a/Application-Development/Eclipse-Behind-the-Name/">Eclipse: Behind the Name</a> for details.</p> 
      </div> 
      <div class="paragraph"> 
       <p>With the purchase of <em>Sun Microsystems</em> by Oracle this conflict finally went away. Oracle is currently among the 5 largest contributor companies of the Eclipse project.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="eclipse-releases"><a class="link" href="#eclipse-releases">1.2. Eclipse releases</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse open source project has a simultaneous release every year. Eclipse projects can choose to join this release but must fulfill the requirements described in the <a href="https://wiki.eclipse.org/SimRel/Simultaneous_Release_Requirements">Release requirements wiki</a>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>This simultaneous release includes new versions of the Eclipse IDE distributions. Before 2012 Eclipse was released in version 3.x, e.g., Eclipse 3.6, Eclipse 3.7. These releases and the corresponding API are referred to as Eclipse 3.x.</p> 
      </div>

     <div class="sect2"> 
      <h3 id="eclipse-foundation"><a class="link" href="#eclipse-foundation">1.3. Eclipse foundation</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse projects are governed by the <em>Eclipse Foundation</em>. The Eclipse Foundation is a non-profit, member supported corporation that hosts the Eclipse Open Source projects. It helps to cultivate both its open source community and its ecosystem of complementary products and services.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Eclipse Foundation does not work on the Eclipse code base, i.e., it does not have employee developers working on Eclipse. The mission of the Eclipse Foundation is <strong>enable</strong> the development by providing the infrastructure (Git, Gerrit, Hudson build server, the download sites, etc.) and a structured process. Eclipse projects follow a very well defined development process. To learn more about this process see the <a href="http://www.eclipse.org/projects/dev_process/development_process.php">Eclipse development process description</a>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>There is also an Eclipse Foundation Europe based in Germany.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In 2016 the Eclipse Foundation had approximately 30 employees. See <a href="http://www.eclipse.org/org/foundation/staff.php">Eclipse Foundation Staff</a> for details.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="the-eclipse-public-license"><a class="link" href="#the-eclipse-public-license">1.4. The Eclipse Public License</a></h3> 
      <div class="paragraph"> 
       <p>The <em>Eclipse Public License</em> (EPL) is an open source software license. The EPL is designed to be business-friendly. EPL licensed programs can be used, modified, copied and distributed free of charge. The consumer of EPL-licensed software can choose to use this software in closed source programs.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Only modifications in the original EPL code must be released as EPL code. This can for example be done by filling a bug report at the public Eclipse bug tracker and by uploading a Gerrit change.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Eclipse Foundation validates that source code contributed to Eclipse projects is free of intellectual property (IP) issues. This process is known as IP cleansing. Contributions with more than 1000 lines of code require the creation of a Contribution Questionnaire, and a review and approval by the IP team.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The permissive EPL and the IP cleansing effort of the Eclipse Foundation makes reusing the source code of Eclipse projects attractive to companies.</p> 
      </div> 
     </div>

    <h2 id="different-eclipse-ide-distributions"><a class="link" href="#different-eclipse-ide-distributions">2. Different Eclipse IDE distributions</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="the-eclipse-ide-for-java-development"><a class="link" href="#the-eclipse-ide-for-java-development">2.1. The Eclipse IDE for Java development</a></h3> 
      <div class="paragraph"> 
       <p>Most people know Eclipse as an integrated development environment (IDE) for Java. In 2014 the Eclipse IDE is the leading development environment for Java with a market share of approximately 65%.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Eclipse IDE can be extended with additional software components. Eclipse calls these software components <em>plug-ins</em>. Plug-in can be grouped into <em>features</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Several projects and companies have extended the Eclipse IDE or created stand-alone applications (Eclipse Rich Client Platform) on top of the Eclipse framework.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="the-eclipse-ide-distributions-from-the-eclipse-packaging-project-epp"><a class="link" href="#the-eclipse-ide-distributions-from-the-eclipse-packaging-project-epp">2.2. The Eclipse IDE distributions from the Eclipse Packaging Project (EPP)</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse IDE is also available as an IDE for other languages, ranging from C, C++ to Lua, Python, Perl and PHP. Several pre-packaged Eclipse distributions are available for download. These pre-packaged solutions are provided by an Eclipse project called the Eclipse Packaging Project (EPP).</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="developer-and-milestone-downloads"><a class="link" href="#developer-and-milestone-downloads">2.3. Developer and milestone downloads</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse project has a simultaneous release every year at the end of June. In June 2016 the Eclipse 4.6 (Neon) version was released.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The top-level Eclipse project creates regular builds of the next releases including JDT, PDT and the Eclipse platform projects. This is called the Eclipse SDK.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You find <em>Stable Builds</em> which are tested by the community. These <em>milestone</em> (ending with M and a number) and <em>release candidate</em> (RC) builds are created based on a predefined time schedule. <em>Integration</em> (I) and <em>Nightly</em> (N) builds are test builds which are automatically created. They are not manually tested.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In general, milestone and RC builds are relative stable compared to integration builds, but may not contain the latest features and patches.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You find one of latest Eclipse SDK builds under the <a href="http://download.eclipse.org/eclipse/downloads/" class="bare">http://download.eclipse.org/eclipse/downloads/</a> - Eclipse project download page link.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The packaged solutions are also available as developer release. You find them by clicking on the <em>Developer Builds</em> tab from the <a href="https://www.eclipse.org/downloads">Eclipse download page</a>.</p> 
      </div>

      <h3 id="install-the-eclipse-ide"><a class="link" href="#install-the-eclipse-ide">4.1. Install the Eclipse IDE</a></h3> 
      <div class="paragraph"> 
       <p>After you downloaded the file with the Eclipse distribution, unpack it to a local directory. Most operating systems can extract zip or tar.gz files in their file browser. For example, in <em>Windows 7</em> with a right-click on the file and selecting "Extract all…?".</p>

           </div> 
           <div class="paragraph"> 
            <p>The Eclipse IDE contains its custom Java compiler hence a JRE is sufficient for most tasks with Eclipse. The <em>JDK</em> version of Java is required to compile Java source code on the command line. Also Java web development sometimes requires the presences of <em>JDK</em>.</p>

      <h3 id="what-to-do-if-the-eclipse-ide-does-not-start"><a class="link" href="#what-to-do-if-the-eclipse-ide-does-not-start">4.2. What to do if the Eclipse IDE does not start</a></h3> 
      <div class="paragraph"> 
       <p>A common startup error happens when the user tries to launch a 64-bit version of Eclipse using a 32 bit JVM or vice versa. In this case Eclipse does not start and the user gets a message containing <strong>exit code=13</strong>. This happens when the version of Eclipse is not matching the JVM version. A 32-bit Eclipse must run with Java 32 bit, and 64-bit Eclipse must use a 64-bit JVM.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Use</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="console">java -version</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>on the command line. If the output does contain the word "Server" you are using the 64 bit version of Java and must use a 64 bit version of Eclipse. If it does not contain "Server", you have to use the 32 bit version of Eclipse.</p> 
      </div>

    <h2 id="important-eclipse-terminology"><a class="link" href="#important-eclipse-terminology">5. Important Eclipse terminology</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="workspace"><a class="link" href="#workspace">5.1. Workspace</a></h3> 
      <div class="paragraph"> 
       <p>The <em>workspace</em> is the physical location (file path) you are storing certain meta-data and (optional) your development artifacts. Your projects, source files, images and other artifacts can be stored inside or outside your workspace. The meta-data stored for the workspace contains preferences settings, plug-in specific meta data, logs etc.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You can choose the workspace during startup of Eclipse or via the <span class="menuseq"><span class="menu">File</span>&nbsp;? <span class="submenu">Switch Workspace</span>&nbsp;? <span class="menuitem">Others</span></span> menu entry. You typically use different <em>workspaces</em> if you require different settings or if you want to divide your work.</p> 
      </div> 
      <div class="admonitionblock note"> 
       <table> 
        <tbody>
         <tr> 
          <td class="icon"> <i class="fa icon-note" title="Note"></i> </td> 
          <td class="content"> 
           <div class="paragraph"> 
            <p>It is not mandatory that a project resides within the workspace directory. It is possible to refer to external resources (e.g., projects) from the <em>workspace</em>.</p> 
           </div> </td> 
         </tr> 
        </tbody>
       </table> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="views-and-editors"><a class="link" href="#views-and-editors">5.2. Views and editors</a></h3> 
      <div class="paragraph"> 
       <p>Eclipse provides <em>views</em> and <em>editors</em> to navigate and change content. View and editors can be grouped into <em>perspectives</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A view is typically used to work on a set of data. This data might be a hierarchical structure. If data is changed via the view, the underlying data is directly changed, without the need to save.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, the <em>Project Explorer</em> view allows you to browse and modify files of Eclipse projects. Any change in the <em>Project Explorer</em> is directly applied to the files, e.g., if you rename a file, the file system is directly changed.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Editors are typically used to modify a single data element, e.g., the content of a file or a data object. Change in an editor are only applied once the user saves.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, the <em>Java</em> editor is used to modify Java source files. Changes to the source file are applied once the user selects the <b class="button">Save</b> button. A editor with changed data (a dirty editor) is marked with an asterisk left to the name of the modified file.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xdirtyeditor10.png.pagespeed.ic.zpeGuSuRAc.png" alt="Editor marked as dirty" data-pagespeed-url-hash="2791624452" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="eclipse-projects"><a class="link" href="#eclipse-projects">5.3. Eclipse projects</a></h3> 
      <div class="paragraph"> 
       <p>An Eclipse project contains source, configuration and binary files related to a certain task. It groups them into buildable and reusable units. An Eclipse project can have <em>natures</em> assigned to it which describe the purpose of this project. For example, the Java <em>nature</em> defines a project as Java project. Projects can have multiple natures combined to model different technical aspects.</p> 
      </div> 
      <div class="paragraph"> 
       <p><em>Natures</em> for a project are defined via the <em>.project</em> file in the project directory.</p> 
      </div> 
     </div> 
    </div> 
   </div>

    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="content-assist"><a class="link" href="#content-assist">13.1. Content assist</a></h3> 
      <div class="paragraph"> 
       <p><em>Content assist</em> is a functionality in Eclipse which allows the developer to get context-sensitive code completion in an editor upon user request.</p> 
      </div> 
      <div class="paragraph"> 
       <p>It can be invoked by pressing <span class="keyseq"><kbd>Ctrl</kbd>+<kbd>Space</kbd></span>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, type <code>syso</code> in the editor of a Java source file and then press <span class="keyseq"><kbd>Ctrl</kbd>+<kbd>Space</kbd></span>. This will replace <code>syso</code> with <code>System.out.println("")</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If you have a reference to an object, for example, the object <code>person</code> of the type <code>Person</code> and need to see its methods, type <code>person.</code> and press <span class="keyseq"><kbd>Ctrl</kbd>+<kbd>Space</kbd></span>.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xcontentassists10.png.pagespeed.ic.FKaZeMbJnk.png" alt="Content Assists" data-pagespeed-url-hash="3782029076" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="quick-fix"><a class="link" href="#quick-fix">13.2. Quick Fix</a></h3> 
      <div class="paragraph"> 
       <p>Whenever Eclipse detects a problem, it will underline the problematic text in the editor. Select the underlined text and press <span class="keyseq"><kbd>Ctrl</kbd>+<kbd>1</kbd></span> to see proposals how to solve this problem. This functionality is called <em>Quick Fix</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, type <code>myBoolean = true;</code> If myBoolean is not yet defined, Eclipse will highlight it as an error. Select the variable and press <span class="keyseq"><kbd>Ctrl</kbd>+<kbd>1</kbd></span>. Eclipse will suggest creating a field or local variable.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xquickfix10.png.pagespeed.ic.NafZ5uGlm-.png" alt="Using Quickfix Example" data-pagespeed-url-hash="2042121195" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p><em>Quick Fix</em> is extremely powerful. For example, it allows you to create new local variables and fields as well as new methods and new classes. Or it can put <code>try/catch</code> statements around your exceptions. It can also assign a statement to a variable and much more.</p> 
      </div> 
      <div class="paragraph"> 
       <p><em>Quick Fix</em> also gives several options for code changes on code which does not contain errors, e.g., it allows you to convert a local variable to a field.</p> 
      </div>

      <div class="paragraph"> 
       <p><em>Refactoring</em> is the process of restructuring the code without changing its behavior. For example, renaming a Java class or method is a refactoring activity.</p> 
      </div> 
     </div> 
     <div class="sect2">

      </div>

Eclipse will make sure that all calls in your Workspace to your class or method are renamed.</p> 
      </div> 
      <div class="paragraph">

    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="eclipse-update-manager"><a class="link" href="#eclipse-update-manager">21.1. Eclipse update manager</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse IDE contains a software component called <em>Update Manager</em> which allows you to install and update software components. Installable software components are <code>features</code> or <code>plug-ins</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>These features are located in <em>update sites</em> or <em>software sites</em>. An <em>update site</em> contains installable software components and additional configuration files. It can be located in various places, e.g., on a web server or on the local filesystem.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The configuration files provide aggregated information about the software components in the <em>update site</em>. The update functionality in Eclipse uses this information to determine which software components are available in which version. This allows the Eclipse update functionality to download only components which are new or updated.</p> 
      </div> 
     </div> 
     <div class="sect2">

      <h3 id="manual-installation-of-plug-ins-dropins-folder"><a class="link" href="#manual-installation-of-plug-ins-dropins-folder">23.1. Manual installation of plug-ins (dropins folder)</a></h3> 
      <div class="paragraph"> 
       <p>Eclipse plug-ins are distributed as <em>jar</em> files. If you want to use an Eclipse plug-in directly or do not know the <em>update site</em> for it, you can place it in the <em>dropins</em> folder of your Eclipse installation directory. Eclipse monitors this directory and during a (re-)start of your IDE, the Eclipse update manager installs and removes plug-in based on the files contained in this directory.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You should not modify the content of the Eclipse <em>plugins</em> directory directly. If you want to install plug-ins, put them into the <em>dropins</em> folder. If you want to remove it, delete the JAR from this folder.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Plug-ins are typically distributed as <em>jar</em> files. To add a plug-in to your Eclipse installation, put the plug-in .jar file into the Eclipse <em>dropins</em> folder and restart Eclipse. Eclipse should detect the new plug-in and install it for you.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If you remove plug-ins from the <em>dropins</em> folder and restart Eclipse, these plug-ins are automatically removed from your Eclipse installation.</p> 
      </div> 
     </div>

     <div class="sect2"> 
      <h3 id="what-are-preferences"><a class="link" href="#what-are-preferences">24.1. What are preferences?</a></h3> 
      <div class="paragraph"> 
       <p>The behavior of the Eclipse IDE can be controlled via key value pairs stores as <em>preference</em> settings. Each Eclipse software component can define such perferences and use the values to configure itself. This allows you for example to configure how long the Eclipse waits before the code completion or if the import statements in your source code should be automatically adjusted if you save your source code.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Which preferences are key values stored on the file system, the Eclipse IDE allows the user to configure most of these values via the preference dialog.</p> 
      </div> 
     </div>

      <h3 id="task-management"><a class="link" href="#task-management">33.1. Task management</a></h3> 
      <div class="paragraph"> 
       <p>You can place markers in the code which you can later access via the <em>Task</em> view.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You can use <code>// TODO</code>, <code>// FIXME</code> or <code>// XXX</code> tags in your code to add task reminders.</p> 
      </div> 
      <div class="paragraph"> 
       <p>This indicates a task for Eclipse. You find those in the <em>Task</em> view of Eclipse. Via double-clicking on the task, you can navigate to the corresponding code.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You can open this view via <span class="menuseq"><span class="menu">Window</span>&nbsp;? <span class="submenu">Show View</span>&nbsp;? <span class="menuitem">Tasks</span></span>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, add a TODO to your <code>MyFirstClass</code> class to see it in the <em>Tasks</em> view.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.eclipse.ide.first</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyFirstClass</span> {

  <span class="directive">private</span> <span class="directive">static</span> <span class="directive">final</span> <span class="predefined-type">String</span> HELLO = <span class="string"><span class="delimiter">"</span><span class="content">Hello Eclipse!</span><span class="delimiter">"</span></span>;

  <span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {
    <span class="comment">// TODO Provide user interface</span>
    <span class="predefined-type">System</span>.out.println(HELLO);
    <span class="type">int</span> sum = <span class="integer">0</span>;
    sum = calculateSum(sum);
    <span class="predefined-type">System</span>.out.println(sum);
  }

  <span class="directive">private</span> <span class="directive">static</span> <span class="type">int</span> calculateSum(<span class="type">int</span> sum) {
    <span class="keyword">for</span> (<span class="type">int</span> i = <span class="integer">0</span>; i &lt;= <span class="integer">100</span>; i++) {
      sum += i;
    }
    <span class="keyword">return</span> sum;
  }
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Close the editor for the <code>MyFirstClass</code> class. If you now double-click on the tasks, the Java editor opens again and the TODO comment is selected.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xtodo10.png.pagespeed.ic.YqwGwA5noG.png" alt="todo10" data-pagespeed-url-hash="1595101713" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>TIP:The <em>Task</em> view shows only the tasks from the currently open projects.</p> 
      </div> 
     </div>

   <div class="sect1"> 
    <h2 id="asking-and-answering-questions"><a class="link" href="#asking-and-answering-questions">36. Asking (and answering) questions</a></h2> 
    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>Due to the complexity and extensibility of Eclipse, you will need additional resources to help you solve your specific problems. Fortunately, the web contains several resources which can help you with your Eclipse problems.</p> 
     </div> 
     <div class="paragraph"> 
      <p>Currently, the best places to find, ask and answer questions are the <a href="http://eclipse.org/forums">Eclipse forums</a> and <a href="http://stackoverflow.com">Stack Overflow</a>. Try to stay polite with your postings, as the Eclipse community values polite behavior.</p> 
     </div> 
     <div class="paragraph"> 
      <p>The <em>Eclipse forums</em> offer several topic-specific forums in which you can post and answer questions. To post or to answer questions in the Eclipse forums, you need a valid user account in the Eclipse bug tracker. <em>Stack Overflow</em> also requires a user account and its community is very active. <em>Stack Overflow</em> allows to tag questions with the relevant keyword, e.g., <em>Eclipse</em> and people search for them or subscribe to these theses.</p> 
     </div> 
     <div class="admonitionblock note"> 
      <table> 
       <tbody>
        <tr> 
         <td class="icon"> <i class="fa icon-note" title="Note"></i> </td> 
         <td class="content"> 
          <div class="paragraph"> 
           <p>Ensure that you search the forums and mailing lists for solutions for your problem. Somebody else might has asked the same question earlier and the answer is already available.</p> 
          </div> </td> 
        </tr> 
       </tbody>
      </table> 
     </div>

   <div class="sect1"> 
    <h2 id="eclipse-bug-reports-and-feature-requests"><a class="link" href="#eclipse-bug-reports-and-feature-requests">37. Eclipse bug reports and feature requests</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="reporting-bugs-and-feature-requests"><a class="link" href="#reporting-bugs-and-feature-requests">37.1. Reporting bugs and feature requests</a></h3> 
      <div class="paragraph"> 
       <p>If you encounter a problem with the Eclipse IDE or think about a potential improvement for it, you should report this to the Eclipse project. The Eclipse bug and feature tracker is using the open source <em>Bugzilla</em> project from <em>Mozilla</em>. In this system, you enter Eclipse error reports. You can also request new features or improvements of existing features.</p> 
      </div> 
      <div class="admonitionblock tip"> 
       <table> 
        <tbody>
         <tr> 
          <td class="icon"> <i class="fa icon-tip" title="Tip"></i> </td> 
          <td class="content"> Most Eclipse projects receive lots of bug and feature requests. So if you want something fixed or enhanced you may have to provide a Gerrit review for it. If the Eclipse developer sees that you try to fix the problem yourself, you typically receive more support from them. </td> 
         </tr> 
        </tbody>
       </table> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="using-the-eclipse-bugzilla-system"><a class="link" href="#using-the-eclipse-bugzilla-system">37.2. Using the Eclipse bugzilla system</a></h3> 
      <div class="paragraph"> 
       <p>This bug tracker can be found under <a href="https://bugs.eclipse.org/bugs/">Eclipse Bugzilla</a>. Here you can search for existing bugs and review them.</p> 
      </div> 
      <div class="paragraph"> 
       <p>To participate actively in the Eclipse bug tracker, you need to create a new account. This can be done by clicking the <em>Create a New Account</em> link.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xbugzilla10.png.pagespeed.ic.K_k1kFtpIU.png" alt="Opening an account at Eclipse bugzilla" data-pagespeed-url-hash="1669693161" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Once you have a user account, you can login to the Eclipse bug tracker. This allows you to comment on existing bugs and report new ones. The user data for the all Eclipse sites are the same, i.e, the forum, marketplace, bug tracker, etc. Only for the Gerrit access, different user data is used.</p> 
      </div> 
      <div class="paragraph"> 
       <p>As example you can report bugs for the Eclipse platform via the following link: <a href="https://bugs.eclipse.org/bugs/enter_bug.cgi?product=Platform">Bug report for the Eclipse platform</a>.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="eclipse-bug-priorities"><a class="link" href="#eclipse-bug-priorities">37.3. Eclipse bug priorities</a></h3> 
      <div class="paragraph"> 
       <p>The Eclipse Bugzilla system allows you and the Eclipse committer to enter the bug priority. But overall, it is up to each project do decide how they handle bugs so some variation from project to project will occur. The following rules can be used as guideline.</p> 
      </div> 
      <table class="tableblock frame-all grid-all spread"> 
       <caption class="title">
        Table 4. Bug priorities
       </caption> 
       <colgroup> 
        <col style="width: 33.3333%;"> 
        <col style="width: 66.6667%;"> 
       </colgroup> 
       <thead> 
        <tr> 
         <th class="tableblock halign-left valign-top">Priority</th> 
         <th class="tableblock halign-left valign-top">Description</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">blocker</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">The bug blocks development or testing of the build and no workaround is known.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">critical</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Implies "loss of data" or frequent crashes or a severe memory leak.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">major</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Implies a "major loss of function".</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">normal</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">This is the default value for new bug reports. Implies some loss of functionality under specific circumstances, typically the correct setting unless one of the other levels fit.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">minor</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Something is wrong, but doesn’t affect function significantly or other problem where easy workaround is present.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">trivial</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">This describes a cosmetic problem like misspelled words or misaligned text, but doesn’t affect function.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">enhancement</p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Represents a request for enhancement (also for "major" features that would be really nice to have).</p></td> 
        </tr> 
       </tbody> 
      </table> 
     </div>

    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>In this exercise you use the Bugzilla system to review some of the Eclipse platform bugs. No action is excepted from you, but if you find an updated bug, you should update the bug report and describe that the problem is solved.</p>

      <p>This exercise uses the Eclipse platform as example but you can use any Eclipse project of your choice.</p>

Select the <em>Advanced Search</em> tab and search for <span class="menuseq"><span class="menu">Eclipse</span>&nbsp;? <span class="submenu">Platform</span>&nbsp;? <span class="menuitem">UI</span></span> for all bugs in status <em>NEW</em>, <em>ASSIGNED</em>, <em>UNCONFIRMED</em> and <em>REOPENED</em>.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xbugtriage10.png.pagespeed.ic.mqGbhPqUqg.png" alt="bugtriage10" data-pagespeed-url-hash="3645258169" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="imageblock"> 
       <div class="content">

       </div>
