             <div class="post-text" itemprop="text"> 
              <p>I need to run the latest version of Node and NPM on Windows. I installed Node 0.5.8 and downloaded the sources of NPM from GitHub. The steps I followed to install NPM were listed on its GitHub site but I have a problem running the following command: </p> 
              <pre><code>node cli.js install npm -gf
</code></pre> 
              <p>but it fails with the following error message: </p> 
              <pre><code>Error: connect UNKNOWN
at errnoException (net_uv.js:566:11)
at Object.afterConnect [as oncomplete] (net_uv.js:557:18)

System Windows_NT 5.1.2600
command "...\\Node\\bin\\node.exe" "...\\npm\\cli.js" "install" "npm" "-gf"
cwd ...\npm
node -v v0.5.8
npm -v 1.0.94
code UNKNOWN
</code></pre> 
              <p>I think that this is a problem because I need authentication at my proxy to connect to the Internet. But I found no way to tell the installer to use my credentials for login. Is there a possibility to provide my proxy IP and login information to npm installation maybe via command-line arguments?</p> 
              <p>I can provide the full log (but seems to have no more relevant information) using pastebin if needed.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p><code>set http_proxy</code> worked really well for me but I had to enter it in every time I opened the command prompt. So I had to combine multiple answers and now mine is permanent.</p> 
              <p>My sequence went as follows:</p> 
              <ul> 
               <li>Go to <code>C:\Users\YourUserName</code></li> 
               <li>Create a file named <code>.npmrc</code></li> 
               <li>Inside that file type the following (if you are on an AD domain):</li> 
              </ul> 
              <p><code>proxy = http://domain\\username:password@ip:port</code></p> 
              <ul> 
               <li>Or use this is you are NOT on an AD domain:</li> 
              </ul> 
              <p><code>proxy = http://username:password@ip:port</code></p> 
              <ul> 
               <li>Save the File</li> 
               <li>Open a command prompt and try to use <code>npm</code></li> 
              </ul> 
              <p>Others have had extra success with the following extra command:</p> 
              <p><code>strict-ssl = false</code></p> 
              <p>*You should be able to use an IP address or the URL to the proxy in place of "IP" above.</p> 
              <p>Your proxy string may need to be tweaked a bit, but this made it so that I didn't have to add this every single time.</p> 
              <p>Cheers</p>

              <p>After some research, I could use it in this way:</p> 
              <ol> 
               <li><p>Install NTLM Authorization Proxy Server or another proxy server for NTLM, like Cntlm. Personally, I prefer the python server, because I can tinker with it, and performance is not a problem.</p></li> 
               <li><p>Set up the configuration; note that in my case, I had to enable both LM and NT mode, and I would suspect it being the normal case, nowadays.</p></li> 
               <li><p>Set http_proxy and https_proxy environment variable to point to your local proxy:</p> <p>set http_proxy="localhost:5865"<br> set https_proxy="localhost:5865"</p></li> 
               <li><p>npm should work now, of course it should be executed from a shell where the above environment variables are defined.</p></li> 
              </ol>

              <p>npm config set http_proxy http://<em>username</em>:<em>password</em>@<em>host/IP</em>:<em>port</em></p> 
              <p>npm config set proxy http://<em>username</em>:<em>password</em>@<em>host/IP</em>:<em>port</em></p> 
              <p>replace the parameters with the values, in my case </p> 
              <p><strong>username</strong>: (empty)</p> 
              <p><strong>password</strong>: (empty)</p> 
              <p><strong>host/IP</strong> : 192.36.36.110</p> 
              <p><strong>port</strong> : 8080</p> 
              <p>so My commands are </p> 
              <p>npm config set http_proxy <a href="http://:@192.36.36.110:8080" rel="nofollow noreferrer">http://:@192.36.36.110:8080</a></p> 
              <p>npm config set proxy <a href="http://:@192.36.36.110:8080" rel="nofollow noreferrer">http://:@192.36.36.110:8080</a></p>

              <p>CNTLM didn't work for me. I tried all possible combinations. NPM was giving Authentication error. Fiddler came for rescue and saved my time. It is easy to install and configure. Set Fiddler Rule to Automatically Authenticated.In .npmrc set these</p> 
              <pre><code>registry=http://registry.npmjs.org
proxy=http://127.0.0.1:8888
https-proxy=http://127.0.0.1:8888
http-proxy=http://127.0.0.1:8888
strict-ssl=false
</code></pre> 
              <p>It worked for me :)</p>
