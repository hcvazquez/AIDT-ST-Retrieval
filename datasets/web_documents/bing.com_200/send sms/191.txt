You probably want to review the <a href="license.shtml">license</a>. </p><p>If you're going to compile Kannel yourself, you need to download and install the Gnome XML library, libxml, from <a href="http://xmlsoft.org/xml.html">http://xmlsoft.org/xml.html</a>.<br> If you are installing it from your distribution's packages, you'll need <tt>libxml2-dev</tt> in addition to run-time <tt>libxml2</tt> package libraries. </p><h2><a name="status">Current status</a></h2> <p>The Kannel Group has made a new <strong>stable release 1.4.4</strong> available. This release is intended for production systems. </p> <p>If you are interested in contributing to the development, please use the <strong>SVN trunk</strong> tree and send suggestions and patches in <strong>diff -u</strong> notation to the developer's mailing list. We use SVN, so you need to install that first. (See for example <a href="http://subversion.apache.org/">SVNhome.org</a> for a source distribution, or your Linux distribution for ready-to-install binaries.) </p>
      <ol> 
       <li>Get sources using the following command 
        <blockquote> 
         <tt>svn co https://svn.kannel.org/&lt;project&gt;/trunk</tt> 
         <br>
         <br>where &lt;project&gt; is: gateway, sqlbox or opensmppbox. 
        </blockquote> </li>
       <li>Later, when you want to check whether there have been changes, do the following: 
        <blockquote> 
         <tt>cd gateway<br> svn up</tt> 
        </blockquote> </li>
      </ol> <p>If you want to send us patches, please send the output of <tt>svn diff</tt>.
