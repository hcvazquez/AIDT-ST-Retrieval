                     <div class="CommentText">
                      <p>Thanks for the hint Paul - I had a similar challenge to Jorg, and after reading your comment, ended up at DateTime.ParseExact. &nbsp;I think DateTime.Parse wouldn't work as my input dates are in many weird formats. Jorg's example would probably have similar problems with DateTime.Parse.</p> 
                      <p>I'm using the form of DateTime.ParseExact that accepts an array of date formats ( <a rel="nofollow" target="_new" href="http://msdn.microsoft.com/en-us/library/332de853">http://msdn.microsoft.com/en-us/library/332de853</a>(VS.90).aspx ). &nbsp;I prefer to do each conversion in a single line of code (rather than call a function), wrapped in a Try ... Catch so that I get Null dates output if all the formats fail. &nbsp;E.g. (in Sub Input0_ProcessInputRow):</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp;Dim dateFormats() As String = {"dd/MM/yyyy", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "dd/MM/yy", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "yyMMdd", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "yy-MM-dd", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "yyyyMMdd", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "yyyy-MM-dd", _</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; "yyyy0000"}</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp;Try</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;Row.MarriageDate = DateTime.ParseExact(Row.MARRIAGEDATERET, dateFormats, Nothing, Globalization.DateTimeStyles.None)</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp;Catch ex As Exception</p> 
                      <p> &nbsp; &nbsp; &nbsp; &nbsp;End Try</p> 
                      <p>(repeat the Try ... Catch block for other columns).</p>
