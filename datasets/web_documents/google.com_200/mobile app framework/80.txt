          <p><strong>TL;DR:</strong> Apache Cordova is a good option for those who already have a web application and want to port it to the different mobile platforms. This post show you how yo can do this, even if your app is already hosted and live. It also provides several tips on how to create a successful Cordova app. Check out <a href="https://github.com/auth0/cordova-sample">the repo</a> to get the code.</p> 
          <hr> 
          <p>Mobile applications is one of the fastest-growing categories in the mobility market today. If you already have a web application up and running and want to get on board with this trend, you can do it very quickly using Apache Cordova.</p> 
          <blockquote class="tweet-quote"> 
           <a href="https://twitter.com/intent/tweet?text=&quot;Mobile applications is one of the fastest-growing categories in the mobility market today.&quot; via @auth0 http://auth0.com/blog/converting-your-web-app-to-mobile/"> <p>"Mobile applications is one of the fastest-growing categories in the mobility market today."</p> <p class="tweet-link">TWEET THIS <img src="https://cdn.auth0.com/blog/resources/twitter.svg"></p> </a> 
          </blockquote> 
          <script>
    $(function() {
        $('.tweet-quote').on('click', function() {
            metricsLib.track('blog:share:twitter_quote');
        });
    });
</script> 
          <p>The promise of Cordova development is quite charming: you can use your existing HTML + JavaScript + CSS application to build cross-platform mobile applications for iOS, Android, Windows Phone, and other platforms.</p> 
          <h2>What is Apache Cordova?</h2> 
          <p><a href="https://cordova.apache.org/">Apache Cordova</a> is a free and open-source platform for building native mobile applications using HTML, CSS, and JavaScript. This allows you to target multiple platforms with just one codebase.</p> 
          <p>Basically, Cordova is a wrapper, an application that has an embedded web browser where your web app is loaded.</p> 
          <blockquote>
           <p><strong>Note:</strong> There is some confusion between Cordova and PhoneGap. Let's clarify this: Cordova is owned and maintained by Apache and will always be maintained as an open-source project. On the other hand, PhoneGap is a distribution of Cordova owned by Adobe which includes additional build services that may not be free. Currently, both have a very similar API. Unless you need one of the additional build services that PhoneGap provides, you should go with Cordova.</p>
          </blockquote> 
          <h2>Installing and setting up Apache Cordova</h2> 
          <p>Before installing and running anything related to Cordova, you will need to install the SDK for each platform that you intend to build your application for. In this post we will focus on iOS and Android; nevertheless, the process for other platforms is quite similar.</p> 
          <p>You will need a Mac computer with <a href="https://developer.apple.com/xcode/download/">Xcode</a> to specify iOS as the target platform, and the <a href="http://developer.android.com/sdk/index.html">Android SDK</a> to target the Android platform.</p> 
          <blockquote>
           <p>For instructions on downloading and configuring the Android SDK, go to the <a href="http://developer.android.com/sdk/installing/index.html">Official Android SDK installation documentation</a>.</p>
          </blockquote> 
          <p>First, install the Cordova command-line utility by running the following command:</p> 
          <pre><code>npm install -g cordova
</code></pre> 
          <blockquote>
           <p>If you are on a Mac computer, you must add <code>sudo</code> at the start of the aforementioned command.</p>
          </blockquote> 
          <p>Create a new Cordova project by running the following command, where <code>[your-app-name]</code> is the folder where the app will be created:</p> 
          <pre><code>cordova create your-app-name com.example.myapp MyAppName
</code></pre> 
          <blockquote>
           <p>The <code>create</code> command has three parameters: the first specifies the folder where the solution will be created, the second is a reverse domain-style identifier which must be unique, and the last one is the application display name. The last two paramenters can be omitted and configured later in the <code>config.xml</code> file.</p>
          </blockquote> 
          <p>Browse to the project folder to add the platforms that you want to support. To do this, once you are in the project foler, run the following command for each platform you want to add:</p> 
          <pre><code>cordova platform add [platform]
</code></pre> 
          <p>Where <code>[platform]</code> can be:</p> 
          <ul> 
           <li>ios</li> 
           <li>android</li> 
           <li>windows</li> 
           <li>wp8</li> 
           <li>blackberry10</li> 
           <li>firefoxos</li> 
           <li>amazon-fireos</li> 
          </ul> 
          <blockquote>
           <p>If you want to check which platforms are available in your system, run <code>cordova platform ls</code>.</p>
          </blockquote> 
          <p>Each time you run the <code>platform add</code> command, it will add native files to build apps for the specified platform. After running the above command, you should find a folder named ios containing .xcodeproj files and a bunch of other files.</p> 
          <h2>Porting your web app to Cordova</h2> 
          <p>In your project folder, you should find the <code>/www</code> directory. This is where your web app is located.</p> 
          <blockquote>
           <p>For this example we used the TodoMVC Vanilla JavaScript example that you can download from <a href="https://github.com/tastejs/todomvc/tree/master/examples/vanillajs">here</a>. Remember to run <code>npm install</code> to get the app dependencies.</p>
          </blockquote> 
          <p>Your app's <code>index.html</code> body content goes into the body of <code>/www/index.html</code>. Perform these three simple steps to update it:</p> 
          <ol> 
           <li>Update the <strong>head</strong> of the file carefully so as not to remove or modify Cordova's required meta tags, but add the elements necessary to your app. In this case, we added our app's title, CSS references, and the charset meta tag.</li> 
           <li>Update the <strong>body</strong> content with your app content.</li> 
           <li>Add the references to your app's required .js files (typically, at the end of the <em>body</em>).</li> 
          </ol> 
          <p>The <code>/js/index.js</code> includes functions that are required for Cordova. You just need to call your app init logic inside the <strong>onDeviceReady</strong> method, as shown in the example below:</p> 
          <pre><code>var app = {
    initialize: function() { ... }    
    bindEvents: function() { ... }    

    onDeviceReady: function() {
        // Your app init code
        ...
    }
};

app.initialize();
</code></pre> 
          <blockquote>
           <p>As the TodoMVC is very simple, just ensuring that the cordova.js and index.js files are executed before the app logic is sufficient.</p>
          </blockquote> 
          <p>Finally, all the CSS files, images, and JavaScript files should be moved to the corresponding folder inside the <code>/www/</code> folder.</p> 
          <p><img src="https://cdn.auth0.com/blog/converting-your-app-to-mobile/cordova-app-structure.png" alt="Cordova app structure"></p> 
          <h3>Add hosted webpage as an app</h3> 
          <p>If your application is already hosted on a server, converting your web app is very easy independent of the framework and languages used to create it.</p> 
          <p>Add the <strong>dialogs</strong> and <strong>network information</strong> plugins to your application by running the following commands:</p> 
          <pre><code>cordova plugin add cordova-plugin-dialogs

cordova plugin add cordova-plugin-network-information
</code></pre> 
          <p>To do it, after the <strong>onDeviceReady</strong> event occurs, just navigate to the URL of your hosted app, and that's it.</p> 
          <p>This is shown in the following code:</p> 
          <blockquote>
           <p>Continuing with the ToDoMVC app sample, we added this code at the bottom of the <code>app.js</code> file.</p>
          </blockquote> 
          <pre><code> &lt;script&gt;
  function onDeviceReady() {
    if (navigator.connection.type == Connection.NONE) {
      navigator.notification.alert('An internet connection is required to continue');
    } else {
      window.location="http://www.myapp.com";
    }
  }
  document.addEventListener("deviceready", onDeviceReady, false);
&lt;/script&gt;
</code></pre> 
          <p>Additionally, as the hosted app will require an internet connection, we added a simple check to verify that this condition is fullfilled.</p> 
          <blockquote>
           <p><strong>Note:</strong> By default, the Cordova app has the access origin policy set to all: <code>*</code>. If you change this policy, remember to add a rule to allow access to the domain on your hosted app in the <code>config.xml</code> file, as shown in the following code:</p>
          </blockquote> 
          <pre><code>&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;cordova&gt;
    &lt;access origin="http://www.myapp.com*"/&gt;
&lt;/cordova&gt;
</code></pre> 
          <h3>Define the application icon and splash screen</h3> 
          <p>These two assets are very simple to add and very important to your application, as they will give the look and feel of a mobile app.</p> 
          <p>You can set a generic icon in the <code>config.xml</code> file for all platforms by using the <strong>icon</strong> element. If you do not specify an icon then the <em>Apache Cordova</em> logo is used.</p> 
          <p>However, an icon that looks good on one platform may not look good on another. For this reason, you have the option to include specific icons for each platform. If you specify a platform icon, this will override the generic icon setting.</p> 
          <p>In the other hand, you can use the <code>splash</code> element to define the splash screens for each platform. In this case you will have one element per resolution. You should also consider the device's orientations; that is, splash screens for portrait and landscape orientations.</p> 
          <blockquote>
           <p>See the <a href="http://cordova.apache.org/docs/en/latest/config_ref/images.html">Cordova documentation</a> for the available resolutions for each platform.</p>
          </blockquote> 
          <p>The following code snippet shows the definition of a generic icon and platform-specfic icons; it also shows the splash screen definition for iOS.</p> 
          <pre><code>&lt;?xml version='1.0' encoding='utf-8'?&gt;
&lt;widget id="com.example.myapp" version="0.0.1" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0"&gt;
    &lt;name&gt;MyApp&lt;/name&gt;
    &lt;icon src="generic-icon.png" /&gt;
    ...
    &lt;platform name="ios"&gt;
    &lt;icon src="ios-icon.png" /&gt;
    &lt;splash src="res/screen/ios/Default~iphone.png" width="320" height="480"/&gt;
    &lt;splash src="res/screen/ios/Default@2x~iphone.png" width="640" height="960"/&gt;
        ...
    &lt;/platform&gt;
        &lt;platform name="android"&gt;
        &lt;icon src="android-icon.png" /&gt;
        ...
    &lt;/platform&gt;
&lt;/widget&gt;
</code></pre> 
          <blockquote>
           <p>Notice that the value of the <code>src</code> attribute is relative to the project directory and not to the www directory.</p>
          </blockquote> 
          <h2>Adding native features</h2> 
          <p>Most native features can be easily added to your Cordova app by using its plugin system. You can browse <a href="https://cordova.apache.org/docs/en/latest/cordova/plugins/pluginapis.html">Cordova Plugin APIs page</a> to see the most used plugins.</p> 
          <p>In this blog post we will show you how to add geolocation support to make your applications location-aware.</p> 
          <p>First, install the <strong>cordova-plugin-geolocation</strong> plugin by running the following command:</p> 
          <pre><code>cordova plugin add cordova-plugin-geolocation
</code></pre> 
          <p>Once the plugin is installed, just add a call to the <strong>navigator.geolocation.getCurrentPosition</strong> method in your app's code, and that's it.</p> 
          <pre><code>navigator.geolocation.getCurrentPosition(geolocationSuccessCallback,
                                         [geolocationError],
                                         [geolocationOptions]);
</code></pre> 
          <blockquote>
           <p><strong>Note:</strong> You can only call the <strong>getCurrentPosition</strong> method once the <strong>onDeviceReady</strong> event has been fired.</p>
          </blockquote> 
          <p>The following is a very simple example of code that gets the current location.</p> 
          <pre><code>var onSuccess = function(position) {
    console.log('Latitude: ' + position.coords.latitude + '\n' +
          'Longitude: ' + position.coords.longitude+ '\n');
};

var onError = function onError(error) {
    console.log('code: '    + error.code    + '\n' +
          'message: ' + error.message + '\n');
}

navigator.geolocation.getCurrentPosition(onSuccess, onError);
</code></pre> 
          <p>The following screenshot shows how we implemented this code in our simple app:</p> 
          <p><img src="https://cdn.auth0.com/blog/converting-your-app-to-mobile/using-geolocation.png" alt="Using GeoLocation"></p> 
          <h2>Testing your Cordova app</h2> 
          <p>To run your application you will need a device or emulator of the corresponding platform. Build your application by running the following command:</p> 
          <pre><code>cordova build [platform]
</code></pre> 
          <p>If you specify a platform, only that platform will be built, otherwise all added platforms will be built one by one.</p> 
          <p>The following subsections show instructions to install and set up the iOS and Android emulators.</p> 
          <h3>Installing an iOS simulator</h3> 
          <p>If you are using a Mac computer, you can install the iOS simulator by running the following command:</p> 
          <pre><code>sudo npm install -g ios-sim
</code></pre> 
          <h3>Creating the Android virtual device</h3> 
          <p>Once you have installed the Android SDK and configured the enviroment variables, run the following command to list the available targets:</p> 
          <pre><code>android list targets
</code></pre> 
          <p>Note the id number of the desired target and use it in the following command to create the Android virtual device.</p> 
          <pre><code>android create avd --name myCordova --target 1
</code></pre> 
          <blockquote>
           <p><strong>Note:</strong> If multiple ABIs are installed for the selected target, you will have to run the command specifying one ABI using the <code>--abi</code> parameter. The available ABIs are listed when the <code>create avd</code> command fails or using the <code>android list targets</code> command.</p>
          </blockquote> 
          <p>Try the emulator by running:</p> 
          <pre><code>cordova emulate android
</code></pre> 
          <p><img src="https://cdn.auth0.com/blog/converting-your-app-to-mobile/android-app.png" alt="Running the app in Android"></p> 
          <h3>Running the app</h3> 
          <p>If the build went well and the emulators are installed, run the following command specifying the desired platform to start the application:</p> 
          <pre><code>cordova run [platform]
</code></pre> 
          <p><img src="https://cdn.auth0.com/blog/converting-your-app-to-mobile/iphone-app.png" alt="Running the app in iOS"></p> 
          <h2>6 tips for migrating your application</h2> 
          <div class="" style="text-align: center;">
           <img style="margin: 0; max-width: 250px;" src="https://cdn.auth0.com/blog/this-the-season-for-cyber-criminals/recommendations.png" alt="Tips for creating your Cordova app"> 
          </div> 
          <p>Based on our experience with this type of app, we created the following list of tips for creating your own Cordova apps.</p> 
          <ol> 
           <li><p><strong>Single Page Apps work better.</strong> Remember that the recommended architecture for Cordova applications is Single-Page Applications (SPAs). This way, all of the resources are only loaded once when the app starts, and can stay in the web view for as long as the application is running. In addition, with SPAs, the user will not have page reloads, which are simply not typical for native applications.</p></li> 
           <li><p><strong>Make your app responsive.</strong> Ensure your app is responsive, in that it adapts gracefully to different devices and resolutions. Using CSS media queries is a good approach for this.</p></li> 
           <li><p><strong>Add smooth transitions when needed.</strong> If your app is not SPA, add transitions when navigating to other pages so the navigation goes unnoticed.</p></li> 
           <li><p><strong>Decide what form factors you will support.</strong> Will you support only mobile phone or tablet form factors, or provide a responsive, universal experience? Just as each platform and platform version will introduce an extra level of effort, each form-factor/breakpoint setting will introduce a new level of effort for designers, developers, and, eventually, testers.</p></li> 
           <li><p><strong>Add Offline Support.</strong> Decide if it is important to add offline support. While some apps don’t require any internet access for normal behavior, some do (hosted apps, for example). Even some particular features of the app may require internet. Consider the user experience of your app when no internet connection is present. For example, some features, such as sharing, can be disabled when offline. You can use the <a href="https://www.npmjs.com/package/cordova-plugin-network-information">Cordova Network Information plugin</a> to detect the connection type and to handle online and offline detection.</p></li> 
           <li><p><strong>Test on devices.</strong> It is almost mandatory to have at least one physical device for each platform/version that you are supporting. There are emulators and simulators that can help you to test the application and identify some major issues, but nothing is as good as having a real device for testing.</p></li> 
          </ol> 
          <h2>Aside: Adding Auth0 to your Cordova App</h2> 
          <p>Auth0 can be easily added to your Cordova application. In a few minutes you can add authentication to your platforms by following these simple steps:</p> 
          <ol> 
           <li>Set up the callback URL in Auth0.</li> 
           <li>You must install the <strong>InAppBrowser</strong> plugin from Cordova to be able to show the <em>Login popup</em>, and then update the config.xml file.</li> 
           <li>Follow the guide specific to the front-end technology you're using.</li> 
           <li>That's it!</li> 
          </ol> 
          <p>For detailed instructions, browse to the <a href="https://auth0.com/docs/native-platforms/cordova">Auth0 Cordova Tutorial</a>.</p> 
          <h2>Wrapping up</h2> 
          <p>You can create mobile apps quickly and easily with Cordova, but making a successful Cordova app means that the end user doesn't notice that the app is a wrapped web app. This is very simple if you have a responsive single-page app; if this is not the case, consider tweaking your app to make it look like native apps---that is, add page transitions, handle offline scenarios gracefully, disable navigation to external sites, adding platform-specific icons and splash screens, and so on.</p>
