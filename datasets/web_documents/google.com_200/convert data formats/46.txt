       <h1 class="article-header">Data Formatting</h1> 
       <p>In the widgets provided by the DevExtreme Data Visualization library, you can easily customize the automatically generated text for the following widget elements.</p> 
       <p><strong>Chart</strong> and <strong>PolarChart</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxChart/Configuration/legend/">legend items</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxChart/Configuration/series/label/">point labels</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxChart/Configuration/commonAxisSettings/label/">axis labels</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxChart/Configuration/tooltip/">tooltips</a> </li> 
       </ul> 
       <p><strong>PieChart</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxPieChart/Configuration/legend/">legend items</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxPieChart/Configuration/series/">point labels</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxPieChart/Configuration/tooltip/">tooltips</a> </li> 
       </ul> 
       <p><strong>TreeMap</strong> </p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxTreeMap/Configuration/tooltip/">tooltips</a> </li> 
       </ul> 
       <p><strong>CircularGauge</strong> and <strong>LinearGauge</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxCircularGauge/Configuration/subvalueIndicator/">subvalue indicators</a> <span style="font-size:11px">of the <em>textCloud</em> type</span></li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxCircularGauge/Configuration/valueIndicator/">value indicator</a> <span style="font-size:11px">of the <em>rangeBar</em> type</span> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxCircularGauge/Configuration/tooltip/">tooltips</a> </li> 
       </ul> 
       <p><strong>dxBarGuage</strong> </p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxBarGauge/Configuration/label/">bar labels</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxBarGauge/Configuration/tooltip/">tooltips</a></li> 
       </ul> 
       <p><strong>RangeSelector</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxRangeSelector/Configuration/scale/label/">scale labels</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxRangeSelector/Configuration/scale/marker/">scale markers</a> </li> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxRangeSelector/Configuration/sliderMarker/">slider markers</a> </li> 
       </ul> 
       <p><strong>Bullet</strong> and <strong>Sparkline</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxBullet/Configuration/tooltip/">tooltips</a></li> 
       </ul> 
       <p><strong>VectorMap</strong></p> 
       <ul> 
        <li><a href="/Documentation/16_1/ApiReference/Data_Visualization_Widgets/dxVectorMap/Configuration/tooltip/">tooltips</a></li> 
       </ul> 
       <p>For this purpose, the widgets provide a uniform set of options. No matter where you need to customize text, you always have the <strong>format</strong> and <strong>customizeText</strong> (or <strong>customizeTooltip</strong> for tooltips) options.</p> 
       <p>Also, DevExtreme widgets provide a set of predefined formats that cover most cases of applying a format. These formats can be divided into three groups.</p> 
       <ul> 
        <li><strong>Numeric Formats</strong></li> 
        <li><strong>Date-Time Formats</strong></li> 
        <li><strong>Currency Formats</strong></li> 
       </ul> 
       <p>Moreover, you can specify a custom format. See the following topics for detailed information on each case.</p> 
      </div> 
      <div class="documentationSection shiftedHeaders curr-light" style="margin-left: 0px"> 
       <div class="anchor article-without-gray-line" id="Numeric_Formats"></div> 
       <h2 class="article-section-header">Numeric Formats</h2> 
       <p><img src="/Content/images/doc/16_1/ChartJS/AvailableFormats_NumericFormats.png" alt="DevExtreme ChartJS AvailableFormats"></p> 
       <div class="note">
        <div class="note-header">
         NOTE
        </div> 
        <div class="note-text">
         Formats from the left column can be paired with formats from the right column using a space separator, e.g., 
         <em>'fixedPoint thousands'</em>.
        </div>
       </div> 
       <p>To set a numeric format, use the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#type">type</a> option. Also, you can apply a precision to numeric values using the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#precision">precision</a> option. For example, the following configuration makes <strong>Chart</strong>'s <a href="/Documentation/16_1/Guide/Data_Visualization/Charts/Chart_Elements/#Series_Point_Labels">series point labels</a> display values rounded up to the nearest hundredth.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var data = [
    { quarter: 'QI', quantity: 26.252 },
    { quarter: 'QII', quantity: 26.5 },
    { quarter: 'QIII', quantity: 26.375 },
    { quarter: 'QIV', quantity: 25.75 }
];

var chartOptions = {
    dataSource: data,
    series: {
        // ...
        label: {
            // ...
            format: {
                type: 'fixedPoint',
                precision: 2
            }
        }
    }
};</pre>
       </div> 
       <p><img src="/Content/images/doc/16_1/ChartJS/Format_Precision.png" alt="DevExpress HTML5 DataVisualizationWidgets Chart Format Precision"></p> 
       <p>The <strong>format</strong> option also accepts strings. Use this capability as a shortcut for specifying a predefined format.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: 'fixedPoint'
    }
};</pre>
       </div> 
       <p>Alternatively, you can use a native formatter of the Globalize library. In this case, the <strong>format</strong> option accepts an object whose fields are the same as the options passed to the <a href="https://github.com/jquery/globalize/blob/master/doc/api/number/number-formatter.md" target="_blank">numberFormatter</a>.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            // ...
            minimumIntegerDigits: 1
        }
    }
};</pre>
       </div> 
       <p>Note that you must not set the <strong>type</strong> option if you use a Globalize formatter. Also, this approach might require additional <a href="https://github.com/jquery/globalize/blob/master/README.md#2-cldr-content" target="_blank">CLDR modules</a> not shipped with the DevExtreme package.</p> 
      </div> 
      <div class="documentationSection shiftedHeaders curr-light" style="margin-left: 0px"> 
       <div class="anchor article-without-gray-line" id="Date-Time_Formats"></div> 
       <h2 class="article-section-header">Date-Time Formats</h2> 
       <p></p> 
       <ul> 
        <li><em>'longDate'</em> - "Thursday, January 01, 1970"</li> 
        <li><em>'longTime'</em> - "12:00:00 AM"</li> 
        <li><em>'longDateLongTime'</em> - "Thursday, January 01, 1970, 12:00:00 AM"</li> 
        <li><em>'monthAndDay'</em> - "January 01"</li> 
        <li><em>'monthAndYear'</em> - "1970 January"</li> 
        <li><em>'quarterAndYear'</em> - "QI 1970"</li> 
        <li><em>'shortDate'</em> - "1/25/1970"</li> 
        <li><em>'shortTime'</em> - "12:00 AM"</li> 
        <li><em>'shortDateShortTime'</em> - "1/25/1970, 12:00 AM"</li> 
        <li><em>'millisecond'</em> - "010"</li> 
        <li><em>'day'</em> - "01"</li> 
        <li><em>'month'</em> - "January"</li> 
        <li><em>'quarter'</em> - "QI"</li> 
        <li><em>'year'</em> - "1970"</li> 
        <li><em>'dayOfWeek'</em> - "Thursday"</li> 
        <li><em>'hour'</em> - "12"</li> 
        <li><em>'minute'</em> - "00"</li> 
       </ul> 
       <p>To set a date-time format, assign one of the values above to the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#type">type</a> option. For example, the following code applies the <em>'monthAndDay'</em> format to <strong>Chart</strong>'s <a href="/Documentation/16_1/Guide/Data_Visualization/Charts/Chart_Elements/#Axis_Labels">argument axis labels</a>.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var data = [
    { date: new Date(1994,2,3), quantity: 26.25 },
    { date: new Date(1994,2,4), quantity: 26.50 },
    { date: new Date(1994,2,5), quantity: 26.375 },
    { date: new Date(1994,2,6), quantity: 25.75 }
];

var chartOptions = {
    // ...
    dataSource: data,
    argumentAxis: {
        label: {
            // ...
            format: {
                type: 'monthAndDay'
            }
        }
    }
};</pre>
       </div> 
       <p><img src="/Content/images/doc/16_1/ChartJS/FormattingDateTime.png" alt="Date-Time Format"></p> 
       <p>The <strong>format</strong> option also accepts strings. Use this capability as a shortcut for specifying a predefined format.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: 'monthAndDay'
    }
};</pre>
       </div> 
       <p>Alternatively, you can use a native formatter of the Globalize library. In this case, the <strong>format</strong> option accepts an object whose fields are the same as the options passed to the <a href="https://github.com/jquery/globalize/blob/master/doc/api/date/date-formatter.md" target="_blank">dateFormatter</a>.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            skeleton: "GyMMMd"
        }
    }
};</pre>
       </div> 
       <p>Note that you must not set the <strong>type</strong> option if you use a Globalize formatter. Also, this approach might require additional <a href="https://github.com/jquery/globalize/blob/master/README.md#2-cldr-content" target="_blank">CLDR modules</a> not shipped with the DevExtreme package.</p> 
      </div> 
      <div class="documentationSection shiftedHeaders curr-light" style="margin-left: 0px"> 
       <div class="anchor article-without-gray-line" id="Currency_Formats"></div> 
       <h2 class="article-section-header">Currency Formats</h2> 
       <p>To set a currency format, assign <em>'currency'</em> to the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#type">type</a> option.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            type: 'currency'
        }
    }
};</pre>
       </div> 
       <p>This setting applies the global default currency - <em>'USD'</em>. It is specified by the <strong>defaultCurrency</strong> field of the object returned by the <strong>DevExpress.config()</strong> method. To change it, use the following code.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">DevExpress.config({ defaultCurrency: 'EUR' });</pre>
       </div> 
       <p>To use any currency that differs from USD, follow these steps.</p> 
       <ol> 
        <li>Get the <em>currencies.json</em> file that corresponds to your locale from one of the folders <a href="https://github.com/unicode-cldr/cldr-numbers-modern/tree/master/main" target="_blank">here</a>.</li> 
        <li>Load the contents of this file in your app using one of the methods described <a href="https://github.com/jquery/globalize/blob/master/doc/cldr.md#how-do-i-load-cldr-data-into-globalize" target="_blank">here</a>.</li> 
        <li><p>Assign the <a href="http://www.currency-iso.org/en/home/tables/table-a1.html" target="_blank">3-letter code</a> of the needed currency to the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#currency">currency</a> option.</p> 
         <div class="code-wrapper">
          <div class="code-type active" data-index="0">
           JavaScript
          </div>
          <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            type: 'currency',
            currency: 'EUR'
        }
    }
};</pre>
         </div></li> 
       </ol> 
       <p>Additionally, you can specify the precision of currency values using the <strong>format</strong> | <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#precision">precision</a> option.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            // ...
            type: 'currency',
            precision: 2
        }
    }
};</pre>
       </div> 
       <p>If you do not need to change neither the <strong>currency</strong> nor the <strong>precision</strong> field, assign <em>'currency'</em> straight to the <strong>format</strong> option. This setting will apply the global default currency.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: 'currency'
    }
};</pre>
       </div> 
       <p>Alternatively, you can use a native formatter of the Globalize library. In this case, the <strong>format</strong> option accepts an object whose fields are the same as the options passed to the <a href="https://github.com/jquery/globalize/blob/master/doc/api/currency/currency-formatter.md" target="_blank">currencyFormatter</a>.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            currency: 'EUR',
            style: 'name'
        }
    }
};</pre>
       </div> 
       <p>Note that you must not set the <strong>type</strong> option if you use a Globalize formatter. Also, this approach might require additional <a href="https://github.com/jquery/globalize/blob/master/README.md#2-cldr-content" target="_blank">CLDR modules</a> not shipped with the DevExtreme package.</p> 
      </div> 
      <div class="documentationSection shiftedHeaders curr-light" style="margin-left: 0px"> 
       <div class="anchor article-without-gray-line" id="Custom_Format"></div> 
       <h2 class="article-section-header">Custom Format</h2> 
       <p>If none of the predefined formats meet your requirements, specify a custom format using the <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#formatter">formatter</a> function. This function accepts the original value as the parameter and returns a formatted string.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            formatter: function (originalValue) {
                // ...
                return formattedString;
            }
        }
    }
};</pre>
       </div> 
       <p>In addition, you can implement the <a href="/Documentation/16_1/ApiReference/Common/Object_Structures/format/#parser">parser</a> function to parse numbers or dates presented as strings into the correct format. This function may be called internally by the widget at different points of your application's lifetime.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var widgetOptions = {
    // ...
    customizableElement: {
        format: {
            // ...
            parser: function (stringToParse) {
                // ...
                return valueWithCorrectFormat;
            }
        }
    }
};</pre>
       </div> 
       <p>If <strong>formatter</strong> is the only field that you need to specify in the <strong>format</strong> object, assign the function straight to the <strong>format</strong> option as shown below.</p> 
       <div class="code-tabs code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <div class="code-type" data-index="1">
         var widgetOptions = { // ... customizableElement: { format: function (originalValue) { // ... return formattedString; } } };
        </div>
        <pre class="prettyprint linenums active">JavaScript</pre>
        <pre class="prettyprint linenums"></pre>
       </div> 
      </div> 
      <div class="documentationSection shiftedHeaders curr-light" style="margin-left: 0px"> 
       <div class="anchor article-without-gray-line" id="Customize_Text"></div> 
       <h2 class="article-section-header">Customize Text</h2> 
       <p>When you need to customize text after (or instead of) applying a format, use the <strong>customizeText</strong> function. This function returns the final text. Its parameter contains data about the element whose text is being customized. The actual contents of this parameter depend on the element. For example, the following code customizes the text of the <strong>Chart</strong>'s <a href="/Documentation/16_1/Guide/Data_Visualization/Charts/Chart_Elements/#Axis_Labels">value axis labels</a> after applying a <em>'fixedPoint'</em> format.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var data = [
    { quarter: 'QI', quantity: 26.252 },
    { quarter: 'QII', quantity: 26.5 },
    { quarter: 'QIII', quantity: 26.375 },
    { quarter: 'QIV', quantity: 25.75 }
];

var chartOptions = {
    dataSource: data,
    // ...
    valueAxis: {
        // ...
        label: {
            format: {
                type: 'fixedPoint',
                precision: 2
            },
            customizeText: function (data) {
                return data.valueText + 'ml' ;
            }
        }
    }
};</pre>
       </div> 
       <p><img src="/Content/images/doc/16_1/ChartJS/FormattingCustomizeText.png" alt="Custom Text"></p> 
       <p>When providing a custom text for tooltips, use the <strong>customizeTooltip</strong> function instead of <strong>customizeText</strong>. This function must return an object wherein the required text is assigned to the <em>'text'</em> field. In the following code, the <strong>customizeTooltip</strong> function combines the formatted value and argument of a point.</p> 
       <div class="code-wrapper">
        <div class="code-type active" data-index="0">
         JavaScript
        </div>
        <pre class="prettyprint linenums active">var chartOptions = {
    // ...
    tooltip: {
        enabled: true,
        format: { 
            type: 'fixedPoint',
            precision: 2
        },
        argumentFormat: {
            skeleton: 'Md'
        },
        customizeTooltip: function (data) {
            return { 
                text: data.valueText + ' on ' + data.argumentText
            };
        }
    }
};</pre>
       </div> 
       <p><img src="/Content/images/doc/16_1/ChartJS/FormattingCustomizeTextWithArgumentNewTooltip.png" alt="Custom Text with Argument Value"></p>
