     <h1 class="title-page">Email</h1> 
     <div class="subtitle-page">
      Documentation of Meteor's email API.
     </div> 
     <div class="page-actions"> 
      <div class="actions-group"> 
       <a class="btn tertiary small round lowercase" href="https://github.com/meteor/docs/tree/master/source/api/email.md" target="_blank"><span class="icon-github"></span> <span>Edit on GitHub</span></a> 
      </div> 
     </div> 
    </div> 
    <div class="document-formatting"> 
     <p>The <code>email</code> package allows sending email from a Meteor app. To use it, add the package to your project by running in your terminal:</p> 
     <figure class="highlight bash">
      <table>
       <tbody>
        <tr>
         <td class="gutter"><pre>
           <div class="line">
            1
           </div></pre></td>
         <td class="code"><pre>
           <div class="line">
            meteor add email
           </div></pre></td>
        </tr>
       </tbody>
      </table>
     </figure> 
     <p>The server reads from the <code>MAIL_URL</code> environment variable to determine how to send mail. The <code>MAIL_URL</code> should reference an <a href="https://en.wikipedia.org/wiki/Simple_Mail_Transfer_Protocol" target="_blank" rel="external">SMTP</a> server and use the form <code>smtp://USERNAME:PASSWORD@HOST:PORT</code> or <code>smtps://USERNAME:PASSWORD@HOST:PORT</code>. The <code>smtps://</code> form (the <code>s</code> is for “secure”) should be used if the mail server requires TLS/SSL (and does not use <code>STARTTLS</code>) and is most common on port 465. Connections which start unencrypted prior to being upgraded to TLS/SSL (using <code>STARTTLS</code>) typically use port 587 (and <em>sometimes</em> 25) and should use <code>smtp://</code>. For more information see the <a href="https://nodemailer.com/smtp/" target="_blank" rel="external">Nodemailer docs</a></p> 
     <p>If <code>MAIL_URL</code> is not set, <code>Email.send</code> outputs the message to standard output instead.</p> 
     <div class="api  new-api-box"> 
      <div class="api-heading"> 
       <div class="locus">
         Server 
       </div> 
       <h2 title="<em>#</em>send" class="title-api selflink" id="Email-send"> <a href="#Email-send" class="link primary">Email.send(options)</a> </h2> 
       <div class="subtext-api"> 
        <div class="code">
         import { Email } from 'meteor/email'
        </div> 
        <a class="src-code link secondary" href="https://github.com/meteor/meteor/blob/master/packages/email/email.js#L145" target="_blank"> (email/email.js, line 145) </a> 
       </div> 
      </div> 
      <div class="api-body"> 
       <div class="desc"> 
        <p>Send an email. Throws an <code>Error</code> on failure to contact mail server or if mail server returns an error. All fields should match <a href="http://tools.ietf.org/html/rfc5322" target="_blank" rel="external">RFC5322</a> specification.</p> 
        <p>If the <code>MAIL_URL</code> environment variable is set, actually sends the email. Otherwise, prints the contents of the email to standard out.</p> 
        <p>Note that this package is based on <strong>mailcomposer 4</strong>, so make sure to refer to <a href="https://github.com/nodemailer/mailcomposer/blob/v4.0.1/README.md" target="_blank" rel="external">the documentation</a> for that version when using the <code>attachments</code> or <code>mailComposer</code> options.</p> 
       </div> 
       <h4 class="subheading">Options</h4> 
       <dl class="args"> 
        <dt> 
         <span class="name">from</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>"From:" address (required)</p> 
        </dd> 
        <dt> 
         <span class="name">to, cc, bcc, replyTo</span> 
         <span class="type">String or Array of Strings</span> 
        </dt> 
        <dd> 
         <p>"To:", "Cc:", "Bcc:", and "Reply-To:" addresses</p> 
        </dd> 
        <dt> 
         <span class="name">inReplyTo</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>Message-ID this message is replying to</p> 
        </dd> 
        <dt> 
         <span class="name">references</span> 
         <span class="type">String or Array of Strings</span> 
        </dt> 
        <dd> 
         <p>Array (or space-separated string) of Message-IDs to refer to</p> 
        </dd> 
        <dt> 
         <span class="name">messageId</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>Message-ID for this message; otherwise, will be set to a random value</p> 
        </dd> 
        <dt> 
         <span class="name">subject</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>"Subject:" line</p> 
        </dd> 
        <dt> 
         <span class="name">text, html</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>Mail body (in plain text and/or HTML)</p> 
        </dd> 
        <dt> 
         <span class="name">watchHtml</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>Mail body in HTML specific for Apple Watch</p> 
        </dd> 
        <dt> 
         <span class="name">icalEvent</span> 
         <span class="type">String</span> 
        </dt> 
        <dd> 
         <p>iCalendar event attachment</p> 
        </dd> 
        <dt> 
         <span class="name">headers</span> 
         <span class="type">Object</span> 
        </dt> 
        <dd> 
         <p>Dictionary of custom headers</p> 
        </dd> 
        <dt> 
         <span class="name">attachments</span> 
         <span class="type">Array of Objects</span> 
        </dt> 
        <dd> 
         <p>Array of attachment objects, as described in the <a href="https://github.com/nodemailer/mailcomposer/blob/v4.0.1/README.md#attachments" target="_blank" rel="external">mailcomposer documentation</a>.</p> 
        </dd> 
        <dt> 
         <span class="name">mailComposer</span> 
         <span class="type">MailComposer</span> 
        </dt> 
        <dd> 
         <p>A <a href="https://nodemailer.com/extras/mailcomposer/#e-mail-message-fields" target="_blank" rel="external">MailComposer</a> object representing the message to be sent. Overrides all other options. You can create a <code>MailComposer</code> object via <code>new EmailInternals.NpmModules.mailcomposer.module</code>.</p> 
        </dd> 
       </dl> 
      </div> 
     </div> 
     <p>You must provide the <code>from</code> option and at least one of <code>to</code>, <code>cc</code>, and <code>bcc</code>; all other options are optional.</p> 
     <p><code>Email.send</code> only works on the server. Here is an example of how a client could use a server method call to send an email. (In an actual application, you’d need to be careful to limit the emails that a client could send, to prevent your server from being used as a relay by spammers.)</p>

          var locationKey = location.pathname + location.hash;
          if (REDIRECTS[locationKey]) {
            location.replace(location.origin + '/' + REDIRECTS[locationKey]);

          if (REDIRECTS[hashKey]) {
            location.replace(location.origin + '/' + REDIRECTS[hashKey]);
