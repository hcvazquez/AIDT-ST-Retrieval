      <h1>5 JavaScript Libraries You'll Love</h1> 
      <div style="margin-bottom: 20px; overflow: hidden;">

      <p id="articleSummary">Explores 5 excellent JavaScript libraries that can take your JavaScript-powered websites and web apps to the next level.</p> 
      <img class="newstuff" src="/res/Image/newstuff/2011-09-09-5-javascript-libraries-youll-love.jpg" alt="5 JavaScript Libraries You'll Love"> 
      <p>When JavaScript first appeared on the scene way back in 1995, people were using it for fairly simple tasks such as form validation. These days, JavaScript has matured into a powerful language that, when coupled with HTML5 APIs like native video playback, drag-and-drop and the <code>canvas</code> element, allows you to build detailed, interactive user interfaces for your websites.</p> 
      <p>However, as you start to use JavaScript to build larger and larger web apps with complex data structures and rich user interfaces, it becomes clear that the JavaScript language alone is not really up to the task. Interacting directly with the DOM starts to get messy and tedious; JavaScript doesn't have much in the way of data manipulation functions; its object model can cause headaches; and its syntax can get overly fussy.</p> 
      <p>Fortunately, some great JavaScript libraries have sprung up in recent years to make your life as a web developer much easier. In this article I'm going to focus on five fairly recent libraries and technologies that are starting to gain traction:</p> 
      <ul> 
       <li><strong>Backbone</strong>, for adding structure to large JavaScript-based web apps.</li> 
       <li><strong>Underscore</strong>, a utility-belt library that adds functional programming features to JavaScript.</li> 
       <li><strong>Knockout</strong>, a great way to simplify your user interfaces.</li> 
       <li><strong>Sugar</strong>, which extends the JavaScript language with a wealth of useful methods.</li> 
       <li><strong>CoffeeScript</strong>, a language that compiles down to JavaScript and is really nice to work with.</li> 
      </ul> 
      <p>With each library, you'll get an overview of what it does and how it works, and I've also included some useful links for finding out more information.</p> 
      <p>Ready? Let's go!</p> 
      <h2 id="backbone"><a href="http://documentcloud.github.com/backbone/">Backbone</a>: Adds structure to big apps</h2> 
      <div class="articleImageWrap">
       <a href="http://documentcloud.github.com/backbone/"><img alt="Backbone logo" class="articleImage" style="max-width: 385px;" src="/res/Image/articles/development/javascript/5-javascript-libraries-youll-love/backbone.png"></a>
      </div> 
      <p>When you're building simple JavaScript-enabled web pages, you can often get away with writing your JavaScript code from scratch. Build your page using HTML markup; add <code>id</code> attributes to the elements you want to manipulate; then fetch the DOM elements using the JavaScript <code>getElementById()</code> method.</p> 
      <p>For more complex pages, you'll probably want to use a library such as jQuery to make it easier to grab an arbitrary bunch of elements in the DOM; manipulate those elements; and work with browser events such as page loads, clicks and keypresses.</p> 
      <p>Once you start writing <em>big</em> web apps, however, things can get messy as you manipulate DOM elements; manually track all the data in your app; link your data to your user interface; and deal with saving and reading the data on the server with Ajax calls and JSON.</p> 
      <p>At this point, you'll benefit from using a JavaScript library that can provide some structure for your web app's data, user interface, and functionality.</p> 
      <p><a href="http://documentcloud.github.com/backbone/">Backbone</a> is one such library. It provides a set of tools that cleanly separate your app's data and functionality from the user interface, and make it much easier to work with bunches of data and keep everything in sync with the web server.</p> 
      <p>Here are some of the key elements of Backbone:</p> 
      <ul> 
       <li><strong>Models</strong> store your app's data. Usually, a single Model object stores a single entity, such as a user, a tweet, or a forum post. With models, you can easily get and set data; validate the data automatically; save the data to a database on the web server; and retrieve a model's data from the database. By default, Backbone communicates with the server using RESTful JSON requests. You can easily specify the URL format that a model should use when saving and retrieving data on the server.</li> 
       <li><strong>Collections</strong> let you manage groups of data that are stored in Models. You can add and remove Models in a Collection; retrieve models from a Collection; specify the sort order of Models in a Collection; fetch an entire Collection from the server, and so on.</li> 
       <li><strong>Views</strong> control how your data appears in the page. You associate a view with a DOM element, then set up the view so that it responds to certain events, such as a Model's values being changed, and updates its DOM element accordingly. In this way, whenever any data in your app changes, the web page automatically updates to reflect the change. Nice!</li> 
       <li><strong>Routers</strong> map client-side URLs — created using either hashes like <code>#pageName</code>, or the new <a href="http://diveintohtml5.org/history.html">History API</a> — to actions and events in your app. This allows your app to create bookmarkable, shareable URLs. For example, you can create a router that maps the URL <code>/page/:pageName</code> to a <code>showPage</code> event, then create a <code>showPage</code> handler that accepts the value of <code>:pageName</code> and displays the appropriate page. So when the user visits the URL <code>/page/aboutUs</code>, your <code>showPage</code> handler is automatically called and passed the value <code>aboutUs</code>. It can then display the "About Us" page.</li> 
      </ul> 
      <p>Backbone is very lightweight and only has one hard dependency: <a href="#underscore">Underscore</a>, which you can use to manipulate Collections and create HTML templates that display data using Views. You can also include jQuery (or Zepto — see Honourable Mentions at the end of the article) to enable the RESTful persistence and history support, and to make it easier to access DOM elements in Views. Backbone also plays nicely with pretty much any other JavaScript library and templating system.</p> 
      <p>To learn more about Backbone, check out the <a href="http://documentcloud.github.com/backbone/">official docs</a> and take a look at this clearly-explained <a href="http://documentcloud.github.com/backbone/examples/todos/index.html">to-do list example</a>.</p> 
      <h2 id="underscore"><a href="http://documentcloud.github.com/underscore/">Underscore</a>: A utility-belt library</h2> 
      <div class="articleImageWrap">
       <a href="http://documentcloud.github.com/underscore/"><img alt="Underscore logo" class="articleImage shadow" style="max-width: 400px;" src="/res/Image/articles/development/javascript/5-javascript-libraries-youll-love/underscore.png"></a>
      </div> 
      <p><a href="http://documentcloud.github.com/underscore/">Underscore</a> is written by the same folks that bring you Backbone. Its main purpose is to add support for <a href="http://en.wikipedia.org/wiki/Functional_programming">functional programming</a> to JavaScript. It provides lots of useful functions for working with collections, arrays, functions and objects, as well as some additional utility functions for things like templating and creating unique IDs. Essentially, Underscore fills in a lot of the gaps in the JavaScript language and makes it much easier to work with data, functions and objects.</p> 
      <p>The Underscore library is very small — around 3kB when minified and compressed — and it plays nicely with other JavaScript libraries like jQuery and Backbone.</p> 
      <p>Here's just a sample of the functions that Underscore has to offer:</p> 
      <ul> 
       <li><strong>Collection functions.</strong> These include <code>map()</code> that runs each value in a list through a function, <code>reduce()</code> that uses a function to reduce a list to a single value, <code>select()</code> for finding all values in a list that pass a specified test, and <code>sortBy()</code> and <code>groupBy()</code> for sorting and grouping lists any way you like.</li> 
       <li><strong>Array functions.</strong> <code>first()</code>, <code>rest()</code> and <code>last()</code> let you easily extract slices from an array. <code>compact()</code> removes all "empty" values from an array; <code>flatten()</code> flattens nested arrays; and <code>without()</code> lets you prune a set of values from a list. Other useful array functions include <code>union()</code> and <code>intersection()</code> for combining arrays; <code>uniq()</code> for de-duping arrays; and <code>zip()</code> for merging data that's split across separate arrays.</li> 
       <li><strong>Function-related functions.</strong> These include <code>bind()</code> and <code>bindAll()</code> for binding functions to objects; <code>memoize()</code> for caching function results; and lots of useful ways to call functions only when specific conditions are met, including <code>defer()</code>, <code>throttle()</code>, <code>debounce()</code>, <code>once()</code>, and <code>after()</code>.</li> 
       <li><strong>Object-related functions.</strong> Underscore provides some functions that make it easier to work with objects, including <code>keys()</code>, <code>values()</code> and <code>functions()</code> for returning an object's components; <code>extend()</code> for inheriting objects; <code>clone()</code> for copying objects; and <code>tap()</code> for inserting arbitrary function calls within method chains. There's also a range of useful testing functions like <code>isEqual()</code>, <code>isEmpty()</code>, <code>isElement()</code>, <code>isArray()</code>, and more.</li> 
       <li><strong>Utility functions.</strong> The last group of functions provide some miscellaneous features, including <code>times()</code> for running a function a given number of times, and <code>uniqueId()</code> for generating GUIDs. A very useful function is <code>template()</code>, which takes a template string and compiles it into a JavaScript function. You can then apply the template by calling the returned function, passing in a list of values to use in the template.</li> 
      </ul> 
      <p>If you're familiar with <a href="http://www.prototypejs.org/">Prototype</a> at all then you'll no doubt recognize a lot of these functions from Prototype's <code>Array</code>, <code>Enumerable</code>, <code>Function</code> and <code>Object</code> modules. As such, Underscore isn't quite as useful if you're already using Prototype. However, it's perfect for augmenting other libraries such as jQuery. Unlike Prototype and <a href="#sugar">Sugar</a>, Underscore doesn't extend the built-in JavaScript objects, preferring instead to create a separate suite of functions under the <code>_</code> object (hence the name "Underscore").</p> 
      <p>You can call Underscore functions in two different ways. For example, here are two ways to call the <code>map()</code> function:</p> 
      <pre class="code"><code>
_.map( myArray, myFunction );
_( myArray ).map( myFunction );
</code></pre> 
      <p>The second approach allows you to <a href="http://en.wikipedia.org/wiki/Method_chaining">chain methods together</a>, which can help you write shorter, more elegant code.</p> 
      <p>The main <a href="http://documentcloud.github.com/underscore/">Underscore documentation</a> does a good job of explaining the library, and lists all the functions included in the library, along with plenty of examples. </p>
      <h2><a href="http://knockoutjs.com/">Knockout</a>: Simplifies your user interfaces</h2> 
      <div class="articleImageWrap">
       <a href="http://knockoutjs.com/"><img alt="Knockout logo" class="articleImage shadow" style="max-width: 531px;" src="/res/Image/articles/development/javascript/5-javascript-libraries-youll-love/knockout.png"></a>
      </div> 
      <p>Like <a href="#backbone">Backbone</a>, <a href="http://knockoutjs.com/">Knockout</a> is a library that helps you to structure your app and cleanly separate its data and behaviour (the model) from its user interface (the view). Knockout is simpler than Backbone and easier to get started with. It concentrates more on the link between the model and the view, whereas Backbone focuses more on manipulating the model and syncing the model with the server.</p> 
      <p>Knockout implements the Model-View-Viewmodel (MVVM) pattern, where a <dfn>viewmodel</dfn> is a JavaScript object that describes the data and behaviour of the user interface.</p> 
      <p>The essential idea is that you grab a data record from the server, load that data into a viewmodel, and add <code>data-bind</code> attributes to your markup to bind the viewmodel to elements in the DOM. Once you've done that, the viewmodel and the DOM are in sync — if you update a value in the viewmodel, the corresponding element in the page automatically changes to reflect the new value. Conversely, if the user updates an element — for example, they type a new value into a text field — the corresponding value in the viewmodel updates automatically.</p> 
      <p>You can bind practically any DOM element or elements this way, from a text field through to a select menu and even — if you use the <a href="http://api.jquery.com/category/plugins/templates/">jQuery Templates</a> that Knockout supports — an entire table of editable data.</p> 
      <p>Knockout is clever enough to track dependencies, too. For example, if you've created a text element in the page that takes its value from property <code>a</code> in your viewmodel, and property <code>a</code> is calculated using properties <code>b</code> and <code>c</code>, then whenever you update property <code>b</code> or <code>c</code>, Knockout updates both property <code>a</code> and the text element automatically.</p> 
      <p>There's a bit more to Knockout than this, but essentially it's a very simple, elegant way to bind your app's data to your view (web page), so that you never have to worry about manipulating the DOM directly. You just update the data in your viewmodel, and the web page updates automatically to reflect the changes.</p> 
      <p>The best way to learn Knockout is to use the excellent <a href="http://learn.knockoutjs.com/">set of interactive tutorials</a> on the Knockout site.</p> 
      <h2 id="sugar"><a href="http://sugarjs.com/">Sugar</a>: Makes JavaScript's native objects sweeter</h2> 
      <div class="articleImageWrap">
       <a href="http://sugarjs.com/"><img alt="Sugar logo" class="articleImage shadow" style="max-width: 480px;" src="/res/Image/articles/development/javascript/5-javascript-libraries-youll-love/sugar.jpg"></a>
      </div> 
      <p><a href="http://sugarjs.com/">Sugar</a> is, in some ways, similar to <a href="#underscore">Underscore</a>, in that it's a suite of tools that make working with JavaScript's built-in objects — such as <code>String</code>, <code>Array</code>, and <code>Object</code> — a more pleasant experience. You'll find many of Underscore's functions in Sugar too.</p> 
      <p>However, whereas Underscore focuses more on functional programming features, Sugar is more general-purpose, and also includes functions to manipulate strings and numbers; work with and format dates in very flexible ways; and make regular expressions a bit more programmer-friendly. It also fills in implementation gaps in certain browsers, allowing you to use modern JavaScript methods — such as <code>indexOf()</code>, <code>forEach()</code> and <code>reduce()</code> — safely across all browsers.</p> 
      <p>Another key difference is that Sugar — like the Prototype library — provides its functionality by <a href="http://sugarjs.com/native">extending the built-in JavaScript objects</a>. If this isn't to your taste, and you prefer to keep JavaScript "clean", then you'll likely be drawn to Underscore's more "hands-off" approach.</p> 
      <p>Here's a selection of methods that Sugar adds to native JavaScript objects:</p> 
      <ul> 
       <li><strong>String objects:</strong> <code>add()</code> for inserting characters into strings; <code>camelize()</code> to convert C-syntax strings to camel case; <code>chars()</code> for running a function on each character in a string; <code>compact()</code> to trim a string and remove all whitespace within it; <code>escapeHTML()</code> to convert HTML characters to entities; <code>split()</code> for splitting strings using any separator; and <code>stripTags()</code> to remove HTML tags.</li> 
       <li><strong>Number objects:</strong> <code>format()</code> to format numbers with thousands separators and decimal points; <code>hex()</code> to convert a number to hexadecimal; and <code>ordinalize()</code> to display an number's ordinal equivalent ("1st", "2nd", etc.).</li> 
       <li><strong>Array objects:</strong> <code>add()</code> for inserting elements anywhere in an array; <code>average()</code> and <code>sum()</code> for computing the average and total of a set of values; <code>compact()</code> for removing empty values; <code>each()</code> for running an iterator function over each element in an array; <code>intersect()</code> and <code>union()</code> for combining arrays; <code>map()</code> for mapping a set of array values to new values based on an iterator function; <code>randomize()</code> to mix up the elements in an array, and a lot more besides.</li> 
       <li><strong>Date objects:</strong> <code>create()</code> to create a Date object using practically any date format imaginable; <code>addDays()</code>, <code>addHours()</code>, <code>addMonths()</code> and so on for easy date arithmetic; <code>daysInMonth()</code> for finding the number of days in the date's month; <code>format()</code> for flexible date formatting; and <code>is()</code> for comparing two dates with varying precision.</li> 
       <li><strong>Object objects:</strong> <code>clone()</code> to copy an object (with optional deep cloning); <code>each()</code> to iterate a function over an object's properties; <code>equals()</code> for comparing objects; <code>fromQueryString()</code> to convert a URL query string to an object; <code>isArray()</code>, <code>isBoolean()</code> and so on for testing object types; and <code>merge()</code> for merging objects together.</li> 
       <li><strong>Function objects:</strong> <code>after()</code> for creating a function that only runs after it's been called a set number of times; <code>bind()</code> for setting the value of <code>this</code> within a function and for <a href="http://en.wikipedia.org/wiki/Currying">currying</a>; <code>debounce()</code> to stop a function being called too frequently due to a rapid series of events; and <code>delay()</code> and <code>wait()</code> to create non-blocking function calls.</li> 
       <li><strong>RegExp objects:</strong> <code>RegExp.escape()</code> escapes all regular expression tokens in a string (very useful), while <code>addFlag()</code>, <code>removeFlag()</code> and <code>setFlags()</code> let you manipulate a regular expression's flags after the regular expression has already been created.</li> 
      </ul> 
      <p>Sugar's <a href="http://sugarjs.com/features">features page</a> gives a good general overview of the library, while the easy-to-use <a href="http://sugarjs.com/api">API page</a> covers each method in turn, along with examples. </p>
      <h2><a href="http://jashkenas.github.com/coffee-script/">CoffeeScript</a>: A nicer way to write JavaScript</h2> 
      <div class="articleImageWrap">
       <a href="http://jashkenas.github.com/coffee-script/"><img alt="CoffeeScript logo" class="articleImage shadow" style="max-width: 380px;" src="/res/Image/articles/development/javascript/5-javascript-libraries-youll-love/coffeescript.jpg"></a>
      </div> 
      <p>I'm cheating a little here, because CoffeeScript isn't actually a library. Rather, it's a language (and compiler) that you can use as an alternative to writing JavaScript. You write your code in CoffeeScript, then run the compiler to turn the CoffeeScript code into actual JavaScript code that your browser can run.</p> 
      <p>Why would you want to write your JavaScript code in a different language? Well, CoffeeScript is a very nice language that irons out a lot of the ugliness and quirks of JavaScript. It can even make your code run a bit faster by optimizing the JavaScript that it generates.</p> 
      <p>Since it's essentially just JavaScript, CoffeeScript works seamlessly with all other JavaScript libraries like jQuery, Prototype, Dojo, Backbone and Knockout.</p> 
      <p>CoffeeScript borrows a lot of the syntactic sugar from languages like Ruby and Python, so if you're familiar with those languages then you'll be right at home with CoffeeScript. Here are some of the headline features of the CoffeeScript language:</p> 
      <ul> 
       <li><strong>A cleaner syntax.</strong> CoffeeScript does away with many of the parentheses and braces that tend to litter JavaScript code. For example, you can create <code>if/else</code> blocks, loops, functions and objects using indentation, rather than braces. You can also add newlines for readability when creating arrays and long strings.</li> 
       <li><strong>Functions</strong> are easy to create with the <code>-&gt;</code> operator. They implicitly return values, and you can supply default argument values easily, as well as splats (variable numbers of arguments).</li> 
       <li><strong>Comprehensions</strong> let you easily loop over arrays and objects. For example, to run the function <code>eat()</code> for each item in an array, you can simply write <code>eat food for food in ['toast', 'cheese', 'wine']</code>.</li> 
       <li><strong>Easy array slicing and splicing.</strong> You can use ranges to pull a slice from an array easily — for example, <code>mySlice = myArray[3..8]</code>. You can also <em>assign</em> values to an array range, letting you splice easily.</li> 
       <li><strong>The existential operator, <code>'?'</code>.</strong> This behaves like Ruby's <code>nil?</code> operator, giving you a nice way to check for a variable's existence.</li> 
       <li><strong>Classes.</strong> JavaScript's prototype-based object model is very powerful, but it can be tricky (and sometimes awkward) to work with. CoffeeScript makes life a lot simpler by adding the <code>class</code>, <code>extends</code> and <code>super</code> keywords, as well as constructors. This lets you build your objects using a class-based approach, and greatly simplifies the code you need to write.</li> 
       <li><strong>String interpolation and heredocs.</strong> CoffeeScript allows Ruby-style string interpolation, like this: <code>myString = "My variable's value is: #{ myVar }"</code>. No more endless concatenating of strings and variables with the <code>+</code> operator! You can also use heredoc syntax to insert large chunks of formatted text, much like you can in Perl, PHP and other languages.</li> 
      </ul> 
      <p>The standard way to install the CoffeeScript compiler is using <a href="http://nodejs.org/">Node.js</a> and npm. You'll find <a href="http://jashkenas.github.com/coffee-script/#installation">full instructions</a> in the CoffeeScript docs. Alternatively, if you just want to play around with CoffeeScript, you can click the <strong class="menu">Try CoffeeScript</strong> menu option on the <a href="http://jashkenas.github.com/coffee-script/">CoffeeScript site</a> and just start typing!</p> 
      <h2>Honourable mentions</h2> 
      <p>There are many other fantastic JavaScript libraries out there that, for one reason or another, didn't quite make it into this list. Here are some that you might like to check out:</p> 
      <ul> 
       <li><strong><a href="http://zeptojs.com/">Zepto</a></strong> is a very lightweight library for mobile sites and apps that aims to provide a jQuery-compatible API for a fraction of the download size. It does this by only supporting WebKit browsers (as used by iOS and Android), and skipping a lot of jQuery features that aren't needed on mobile devices, such as IE6 support. "jQuery Lite for mobiles", if you like.</li> 
       <li><strong><a href="http://jcscript.com/">jCanvaScript</a></strong> and <strong><a href="http://ocanvas.org/">oCanvas</a></strong> give you nice object-oriented interfaces for working with the <code>canvas</code> element, including drawing shapes and animation.</li> 
       <li><strong><a href="http://mapstraction.appspot.com/">Mapstraction</a></strong> is an abstraction library for a range of common mapping services, including Google Maps, OpenStreetMap, Yahoo! and Microsoft. This makes it easy to choose from, or switch between, mapping providers.</li> 
       <li><strong><a href="https://github.com/clojure/clojurescript/wiki">ClojureScript</a></strong> is similar to CoffeeScript, except that it's based on the <a href="http://clojure.org/">Clojure</a> dialect of the Lisp language.</li> 
       <li><strong><a href="http://phpjs.org/">php.js</a></strong> is a (brave!) effort to implement most of PHP's built-in functions in a JavaScript library. So far it's looking pretty good, with most commonly-used PHP functions ported.</li> 
       <li><strong><a href="http://cubiq.org/iscroll-4">iScroll</a></strong> is handy if you're developing mobile web apps. It gives you a way to have a scrollable content area in the page, along with a fixed header and/or footer — something that isn't currently possible to implement natively with most mobile browsers. It also provides some other nice features such as pull-to-refresh, pinch-to-zoom and custom scrollbars.</li> 
       <li><strong><a href="http://andreasgal.com/2011/06/15/pdf-js/">pdf.js</a></strong> aims to implement an entire PDF viewer/renderer in JavaScript. While it's still in early development, it's already pretty impressive, as you can see in <a href="http://andreasgal.github.com/pdf.js/web/viewer.html">this demo</a>.</li> 
       <h2>Summary</h2> 
       <p>In this article, you've looked at five recent JavaScript libraries and technologies that can help you write JavaScript apps more quickly and effectively:</p> 
       <ul> 
        <li><strong>Backbone</strong>, which provides a set of objects to help you structure your app's data in a manageable way, as well as cleanly separate your data from your user interface.</li> 
        <li><strong>Underscore</strong>, a nice library that adds a large number of functional programming features to JavaScript, making it much easier to work with data, functions and objects.</li> 
        <li><strong>Knockout</strong>, an implementation of the Model-View-Viewmodel (MVVM) pattern, which gives you a really clean way to link your app's data to its user interface, and puts an end to tedious DOM manipulation.</li> 
        <li><strong>Sugar</strong>, which extends the core String, Number, Array, Date, Object, Function and RegExp objects in order to fill some of the gaps in JavaScript, such as powerful string manipulation, number formatting and flexible date handling.</li> 
        <li><strong>CoffeeScript</strong>, a compact, easy-to-write language that compiles down to JavaScript and adds a lot of great features, such as comprehensions, array manipulation and some really nice object-oriented syntax.</li> 
       </ul> 
       <p>Many of these libraries work well together, and with other libraries — for example, you can happily use Backbone, Underscore and jQuery together on the same page, giving you a powerful set of libraries with which to build your websites and web apps.</p> 
       <p>You also looked at some other up-and-coming libraries that can make your JavaScript coding life easier.</p> 
       <p>I hope you enjoyed this article, and that it's inspired you to go off and learn a few of these fantastic technologies. Thanks for reading!</p> 
       <p>What's <em>your</em> favourite JavaScript library or technology, and why? Let us know in the comments below!</p>
