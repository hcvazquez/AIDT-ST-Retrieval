        <div class="post-body__content">
         <p>Using <a href="http://nodejs.org/">Node.js</a>, we can create web applications easily. Now, thanks to the <a href="https://github.com/rogerwang/node-webkit">node-webkit</a> we can also create desktop apps with it, using a unique combination of HTML5 and Node.</p> 
         <hr> 
         <h2>Introduction</h2> 
         <p>The library combines WebKit engine and Node.js in a unique way. Both WebKit and Node share the same context, allowing you to write your code like it's meant to be executed in a browser, but with the addition of all Node's features.</p> 
         <p>The list of uses is endless. You can create business apps, text and image editors, games, presentations, admin panels, etc. Just name the desktop app that you would like to create, and I can assure you that it will be possible with node-webkit.</p> 
         <p>In this tutorial, I will show you how to get started by showing you how to create a simple text editor.</p> 
         <hr> 
         <h2>Preparation</h2> 
         <p>First, you have to get the library. Download the version appropriate for your operating system (the application itself will run on all of them) from <a href="https://github.com/rogerwang/node-webkit#downloads">github</a> and unzip it wherever you'd like. Now, lets create a basic folder structure. We will have folders for HTML files (<code>.html</code>) and for JavaScript files (<code>.js</code>). Also, create the <code>package.json</code> file in the same directory that the <code>nw</code> executable is in and a <code>node_modules</code> folder to keep the modules we will create.</p> 
         <hr> 
         <h2>Package.json</h2> 
         <p>The first thing to do is to fill the required fields in the <code>package.json</code> file. In the case of node-webkit these are <code>name</code> and <code>main</code> (on <a href="https://github.com/rogerwang/node-webkit/wiki/Manifest-format">Github you can find the full list</a> of available <code>package.json</code> options). The first one is exactly the same as in the case of a plain Node.js app. The second one should be a (relative or absolute) path to the main HTML file that will be displayed upon the application starting. In the case of our example, the <code>package.json</code> file should look like this:</p> 
         <pre class="brush: noskimlinks noskimwords">{
	"name": "simple-text-editor",
	"main": "./html/index.html"
}</pre> 
         <p>Now if you run your app using the <code>nw</code> executable, you should see a blank screen like this:</p> 
         <figure> 
          <img width="600" alt="blank_app" src="https://cdn.tutsplus.com/net/uploads/2013/12/blank_app.png" height="427">
          <br> 
         </figure> 
         <hr> 
         <h2>The Main File</h2> 
         <p>Creating the user interface using node-webkit is no different than creating a webpage (with the slight difference that you know the rendering engine, so you don't have to provide any fallbacks for older browsers, or use libraries like jQuery - and actually you shouldn't, because they include fallback code that will slow down your app). Let's create an <code>index.html</code> file that we will use:</p> 
         <pre class="brush: html noskimlinks noskimwords">&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
	&lt;title&gt;Simple Text Editor&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;

&lt;/body&gt;
&lt;/html&gt;</pre> 
         <p>Let's also include (and create) the main JavaScript file that we will use, in the body, so it will be executed after the DOM is loaded:</p> 
         <pre class="brush: html noskimlinks noskimwords">&lt;script src="../js/main.js"&gt;&lt;/script&gt;</pre> 
         <p>Now the only thing that will change when you open the app will be the title.</p> 
         <hr> 
         <h2>Let's Use Some Node</h2> 
         <p>To show you how easily you can use Node.js in node-webkit, let's read the contents of <code>package.json</code> and print it on the screen. Create the <code>js/main.js</code> file and put the following code in it:</p> 
         <pre class="brush: noskimlinks noskimwords">var fs = require('fs');

fs.readFile('./package.json', 'utf-8', function (error, contents) {
	document.write(contents);
});</pre> 
         <p>As you can see, the code looks exactly the same as if you were writing it for Node. But then we use <code>document.write</code> to put the file contents on the page. No need to setup any local server.</p> 
         <p>Now open the app and you should see something like this:</p> 
         <figure> 
          <img width="600" alt="file_contents_printed" src="https://cdn.tutsplus.com/net/uploads/2013/12/file_contents_printed.png" height="427">
          <br> 
         </figure> 
         <hr> 
         <h2>Modules</h2> 
         <p>There is another nice thing about node-webkit; You don't have to include a series of <code>&lt;script&gt;</code> tags in your HTML if you want to split your code into modules. You can do it just like you do in Node.js - using <code>require</code>. Let's create a simple module to read file contents to a textarea or write it to a file. Name the file <code>file.js</code> and put it in the <code>node_modules</code> directory. Now put the overall module structure in it:</p> 
         <pre class="brush: noskimlinks noskimwords">var fs = require('fs');

function File() {
	function open(path, document) {
		
	}
	
	function save(path, document) {
		
	}
	
	this.open = open;
	this.save = save;
}

module.exports = new File;</pre> 
         <p>As you can see this will be a static class with two public methods - one for opening the files and one for saving them.</p> 
         <p>The <code>open</code> method will look like this:</p> 
         <pre class="brush: noskimlinks noskimwords">function open(path, document) {
	fs.readFile(path, 'utf-8', function (error, contents) {
		document.getElementById('editor').value = contents;
	});
}</pre> 
         <p>Pretty simple right? The function accepts the path to the file as the first parameter and puts the file's contents into the element with an id of "editor". We also need to pass the <code>document</code> object to the function because the script is called using Node's <code>require</code> and it doesn't have access to the WebKit stuff directly.</p> 
         <p>The <code>save</code> method is as simple as the previous one:</p> 
         <pre class="brush: noskimlinks noskimwords">function save(path, document) {
	var text = document.getElementById('editor').value;
	fs.writeFile(path, text);
}</pre> 
         <p>Now let's test if everything works. Change the contents of <code>js/main.js</code> to this:</p> 
         <pre class="brush: noskimlinks noskimwords">var file = require('file.js');

console.log(file.open, file.save);</pre> 
         <p>Now if you go to the developer tools console and click the <strong>developer refresh</strong> button in the right corner of the window you should see the two functions from this module logged. This is another useful thing in node-webkit - your <code>console.log</code> calls are being displayed in the developer tools console, so it's easier to debug your apps.</p> 
         <figure> 
          <img width="600" alt="dev_tools_log" src="https://cdn.tutsplus.com/net/uploads/2013/12/dev_tools_log.png" height="318">
          <br> 
         </figure> 
         <hr> 
         <h2>Node-Webkit's File Inputs</h2> 
         <p>Let's add two file inputs that we will need later:</p> 
         <pre class="brush: noskimlinks noskimwords">&lt;input id="open" type="file" style="display:none" accept="text/*"/&gt;
&lt;input id="save" type="file" nwsaveas style="display:none" accept="text/*"/&gt;</pre> 
         <p>Notice the <code>nwsaveas</code> attribute in the second input. It is a special type of input from node-webkit, that allows the user to select a non-existing file. Both of the inputs are hidden, because we will only access them from JavaScript. In node-webkit, file inputs are modified in a way that allows you to fire a <code>click</code> event on them, so you can open the file dialog without the user clicking on the input (with no need for the workarounds like invisible inputs above the button). Now we can move to the JavaScript.</p> 
         <p>First, remove the <code>console.log</code> call from the <code>js/main.js</code> file. Now put this code in there:</p> 
         <pre class="brush: noskimlinks noskimwords">function clickInput(id) {
	var event = document.createEvent('MouseEvents');
	event.initMouseEvent('click');
	document.getElementById(id).dispatchEvent(event);
}

document.addEventListener('keyup', function (e) {
	if (e.keyCode == 'O'.charCodeAt(0) &amp;&amp; e.ctrlKey) {
		clickInput('open');
	} else if (e.keyCode == 'S'.charCodeAt(0) &amp;&amp; e.ctrlKey) {
		clickInput('save');
	}
});</pre> 
         <p>This code is responsible for showing the <strong>Open</strong> and <strong>Save</strong> dialogs. The <code>clickInput</code> function does the main job here - it simulates the click event on the input element - in a normal browser, this would be impossible from security reasons, but here it's not a security threat at all. Later, there is a normal <code>keyup</code> event handler that checks if the proper key combination was pressed (<strong>Ctrl+O</strong> or <strong>Ctrl+S</strong>) and "clicks" on the inputs. Note that this would also be impossible to achieve in a browser - combinations like <strong>Ctrl+O</strong> and <strong>Ctrl+S</strong> are reserved for the browser's internal use and no event fires when they are pressed (only Firefox fires events for them).</p> 
         <p>Now press the <strong>developer refresh</strong> button and you should see the appropriate dialog when you press <strong>Ctrl+S</strong> or <strong>Ctrl+O</strong>. They of course do nothing for now.</p> 
         <hr> 
         <h2>Creating the Editor</h2> 
         <p>Now, since we will be creating a text editor, we need something to write on. Add the <code>textarea</code> to the HTML:</p> 
         <pre class="brush: noskimlinks noskimwords">&lt;textarea id="editor" style="position:fixed;top:0;bottom:0;left:0;right:0"&gt;&lt;/textarea&gt;</pre> 
         <p>Next we have to finish the opening/saving code. Let's create the <code>onchange</code> event listeners for the <code>open</code> and <code>save</code> inputs:</p> 
         <pre class="brush: noskimlinks noskimwords">document.getElementById('open').addEventListener('change', function (e) {
	file.open(this.value, document);
});

document.getElementById('save').addEventListener('change', function (e) {
	file.save(this.value, document);
});</pre> 
         <p>Thanks to the module created earlier, the code is really simple. This is also possible thanks to node-webkit, because in browsers (again for security reasons) the value of the file input element is set to a fake path - here it's the path which was selected. Now open the app (or press the <strong>developer refresh</strong> button, if you didn't closed it) and you should have a perfectly working text editor.</p> 
         <hr> 
         <h2>Further Enhancements</h2> 
         <p>We can also do a few other things to make the editor a bit more advanced and useful. For example, let's open a new window when a user presses <strong>Ctrl+N</strong>. First, add this <code>require</code> to the top of the script:</p> 
         <pre class="brush: noskimlinks noskimwords">var gui = require('nw.gui');</pre> 
         <p>The <code>nw.gui</code> module is a node-webkit's library related to the user interface (you can read more about this <a href="https://github.com/rogerwang/node-webkit/wiki/Native-UI-API-Manual">on Node-webkit's Github</a>). Next add this <code>else if</code> to the document's <code>keyup</code> listener:</p> 
         <pre class="brush: noskimlinks noskimwords">} else if (e.keyCode == 'N'.charCodeAt(0) &amp;&amp; e.ctrlKey) {
	gui.Window.open('index.html');
}</pre> 
         <p>And viola! If you refresh the app, now you can press <strong>Ctrl+N</strong> to open a new window. This function does differ from the normal <code>window.open</code> though. You can pass various window options as the second parameter. The list of what you can set there is available <a href="https://github.com/rogerwang/node-webkit/wiki/Manifest-format#window-subfields">in the documentation</a>.</p> 
         <p>Another thing that may be useful in a text editor is an application menu (the one that sits under the title bar in Windows/Linux and on the top of the screen on Mac). In node-webkit it's pretty simple. First, let's create the menu:</p> 
         <pre class="brush: noskimlinks noskimwords">var menu = new gui.Menu({ type: 'menubar' });</pre> 
         <p>The type <code>menubar</code> is reserved for the application menu. Now we can add items to it. Let it be a <code>File</code> menu:</p> 
         <pre class="brush: noskimlinks noskimwords">menu.append(new gui.MenuItem({
	label: 'File',
	submenu: new gui.Menu()
}));</pre> 
         <p>Now let's add some items to this menu:</p> 
         <pre class="brush: noskimlinks noskimwords">menu.items[0].submenu.append(new gui.MenuItem({
	label: 'New',
	click: function () {
		gui.Window.open('index.html');
	}
}));
menu.items[0].submenu.append(new gui.MenuItem({
	type: 'separator'
}));
menu.items[0].submenu.append(new gui.MenuItem({
	label: 'Close',
	click: function () {
		gui.Window.get().close();
	}
}));</pre> 
         <p><code>menu.items[0]</code> is the first item of our application menu (you can also assign it to a variable when creating it if you want to). We are appending new items to its <code>submenu</code>, and each item gets a <code>click</code> callback to handle the click on it. The <code>gui.Window.get</code> method gets the current window, so we can close it when the user selects the <strong>Close</strong> option in the <strong>File</strong> menu.</p> 
         <p>Finally, we can assign the menu to the window:</p> 
         <pre class="brush: noskimlinks noskimwords">gui.Window.get().menu = menu;</pre> 
         <p>Again, we are using <code>gui.Window.get</code> to get the current window, then we are assigning our menu to its <code>menu</code> property. Please note, that even though we can assign a different menu to every window, on OSX (Mac) one app can only have one menu (which is global for all windows), so if you want your app to be used on Macs, you should avoid using different menus on different windows.</p> 
         <p>Now if you open or refresh your app, you should see a <strong>system menu</strong> under the <strong>titlebar</strong>:</p> 
         <figure> 
          <img width="600" alt="system_menu" src="https://cdn.tutsplus.com/net/uploads/2013/12/system_menu.png" height="427">
          <br> 
         </figure> 
         <hr> 
         <h2>Packaging the App</h2> 
         <p>Now if you would like to share your app with other users, you can package it into one file, so the users can download the node-webkit's executable appropriate to their platform and launch your app with it. First, let's remove the <strong>toolbar</strong> that makes the window look like a browser - it's nice to have during development, but your users probably don't want to see it. We can achieve this by setting <code>window.toolbar</code> to <code>false</code> in <code>package.json</code>, so it looks like this:</p> 
         <pre class="brush: noskimlinks noskimwords">{
	"name": "example-app",
	"main": "./html/index.html",
	"window": {
		"toolbar": false
	}
}</pre> 
         <p>Now if you open the app (nothing will change if you just refresh it, <code>package.json</code> is loaded only on startup), you should see the final result:</p> 
         <figure> 
          <img width="600" alt="final_result" src="https://cdn.tutsplus.com/net/uploads/2013/12/final_result.png" height="427">
          <br> 
         </figure> 
         <p>Packaging the app is pretty simple. Just create a <code>.zip</code> archive with all your assets (all of the files you created, without the files supplied with node-webkit) and change its extension to <code>.nw</code>. That's all. If a user downloads node-webkit and your package, they only need to put it in the node-webkit's directory and run the <code>nw</code> executable. Detailed descriptions, with a few more tips, are available on the <a href="https://github.com/rogerwang/node-webkit/wiki/How-to-package-and-distribute-your-apps">node-webkit's github</a>.</p> 
         <p>Now your editor is ready to be shipped to your users.</p> 
         <hr> 
         <h2>Conclusion</h2> 
         <p>As you can see, node-webkit is a very promising and powerful library. With frequent updates and fixes, and being supported by Intel, there's almost no chance that it will be discontinued, which can sometimes happen with open source projects.</p> 
         <p>Share your thoughts about this project in the comments below. I personally think it's the best thing out there if you want to create a desktop application using Node.js and HTML.</p>
