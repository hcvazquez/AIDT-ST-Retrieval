          <p>It’s limitless what you can render in a browser today using Javascript. In a previous <a href="http://modeling-languages.com/web-based-modeling-tools/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://modeling-languages.com/web-based-modeling-tools/', 'post');" title="Our list of online modeling tools">post</a>&nbsp;we have presented a list of online modeling tools which allow creating diagrams directly in the browser. Most of these tools use Javascript to render shapes and interact with them. In the actual post we present a list of 10 Javascript &nbsp;libraries that could be used to create a diagramming tool.</p> 
          <div class="code-block code-block-4" style="margin: 8px 0;"> 
           <div id="bsap_1305055" class="bsarocks bsap_4f68faf8a714d972ed788779ad78d5b6"></div> 
          </div> 
          <p>&nbsp;</p> 
          <p>&nbsp;</p> 
          <p>Some of these libraries have high level support for advanced features (e.g., ready to use&nbsp;shapes, palette, drag/drop,…), while others present a low level drawing features. &nbsp;Both open source and commercial libraries are covered. A comparative table is presented at the end of the post.</p> 
          <h2><a name="_Toc414838057"></a> <strong><a href="http://www.jointjs.com/" target="_blank">JointJS</a>&nbsp;</strong></h2> 
          <p>JointJS is a JavaScript diagramming library. It can be used to create either static diagrams or fully interactive diagramming tools and application builders.</p> 
          <p>Here are some of its features:</p> 
          <ul> 
           <li>basic diagram elements (rectangle, circle, ellipse, text, image, path)</li> 
           <li>ready-to-use diagram elements of well-known diagrams (ERD, Org chart, FSA, UML, PN, DEVS, …)</li> 
           <li>custom diagram elements based on SVG or programmatically rendered</li> 
           <li>interactive elements and links</li> 
           <li>connecting diagram elements with links</li> 
           <li>customizable links, their arrowheads and labels</li> 
           <li>magnets (link connection points) can be placed basically anywhere</li> 
           <li>hierarchical diagrams</li> 
           <li>serialization/deserialization to/from JSON format</li> 
           <li>zoom in/out</li> 
           <li>touch support</li> 
          </ul> 
          <p id="lNnGvff"><img class=" size-full wp-image-4614  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d334dc6083.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d334dc6083.png 646w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d334dc6083-300x221.png 300w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d334dc6083-82x60.png 82w" sizes="(max-width: 646px) 100vw, 646px"></p> 
          <h2><a name="_Toc414838058"></a> <a href="http://www.jointjs.com/about-rappid" target="_blank"><strong>Rappid&nbsp;</strong></a></h2> 
          <p>Rappid is the commercial extension of JointJS. It’s a set of JointJS plugins and other components that gives more possibilities and ready to use features when creating a diagramming tool.</p> 
          <p>Here are some of its features:</p> 
          <ul> 
           <li>Real-time collaboration</li> 
           <li>Copy, cat and past with HTML 5 loacalStorage support</li> 
           <li>Manipulate more elements in one go.</li> 
           <li>validate your documents</li> 
           <li>local storage</li> 
           <li>Undo and Redo</li> 
           <li>Modal and non-modal dialog boxes</li> 
           <li>Inline Text editor for any SVG text</li> 
           <li>Auto-layout of directed graphs</li> 
           <li>Svg export</li> 
           <li>A set of ready to use shapes for ERD, UML, ORG, BPMN</li> 
          </ul> 
          <h2><a name="_Toc414838059"></a> <strong><a href="https://www.jgraph.com/" target="_blank">Mxgraph</a>&nbsp;</strong></h2> 
          <p>MxGraph is an interactive JavaScript HTML 5 diagramming library. You include it as a JavaScript link in your HTML file and you instantly have access to the most functional native browser diagramming component available. This library is used in Draw.io. The library is developed since 2005 and supports even IE 6.</p> 
          <p>The mxGraph package contains a client software, written in JavaScript, and a series of backends for various languages. The client software is a graph component with an optional application wrapper that is integrated into an existing web interface. The client requires a webserver to deliver the required files to the client or can be run from the local filesystem without a webserver. The backends may be used as is, or they may be embedded into an existing server application in one of the supported languages.</p> 
          <p id="tnAXBRC"><img class=" size-full wp-image-4615  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33b69d8db.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33b69d8db.png 646w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33b69d8db-300x225.png 300w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33b69d8db-65x50.png 65w" sizes="(max-width: 646px) 100vw, 646px"></p> 
          <h2><a name="_Toc414838060"></a> <strong><a href="http://gojs.net/latest/index.html" target="_blank">GoJS</a>&nbsp;</strong></h2> 
          <p>GoJS is a feature-rich JavaScript library for implementing interactive diagrams across modern browsers and platforms. GoJS makes constructing diagrams of complex Nodes, Links, and Groups easy with customizable templates and layouts.</p> 
          <p>GoJS offers many advanced features for user interactivity such as drag-and-drop, copy-and-paste, transactional state and undo management, palettes, overviews, data-bound models, event handlers, and an extensible tool system for custom operations.</p> 
          <p id="wxYLhWp"><img class=" size-full wp-image-4616  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33df0354a.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33df0354a.png 452w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d33df0354a-300x247.png 300w" sizes="(max-width: 452px) 100vw, 452px"></p> 
          <h2><a name="_Toc414838061"></a> <a href="http://raphaeljs.com/" target="_blank"><strong>Raphael</strong></a></h2> 
          <p>Raphaël is a small JavaScript library that should simplify your work with vector graphics on the web. If you want to create your own specific chart or image crop and rotate widget, for example, you can achieve it simply and easily with this library. Raphaël uses the SVG W3C Recommendation and VML as a base for creating graphics. This means every graphical object you create is also a DOM object, so you can attach JavaScript event handlers or modify them later. Raphaël’s goal is to provide an adapter that will make drawing vector art compatible cross-browser and easy.</p> 
          <h2><a name="_Toc414838062"></a> <a href="http://www.draw2d.org/" target="_blank"><strong>Draw2D</strong></a></h2> 
          <p>Create Visio like drawings, diagrams or an workflow editor with the Draw2D Javascript library. The User interface allows interactive drawing by using your standard browser. Draw2D uses Raphael.</p> 
          <p id="MAZwYFN"><img class=" size-full wp-image-4617  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d342d7ec09.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d342d7ec09.png 428w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d342d7ec09-300x230.png 300w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d342d7ec09-65x50.png 65w" sizes="(max-width: 428px) 100vw, 428px"></p> 
          <h2><a name="_Toc414838063"></a> <strong><a href="http://d3js.org/" target="_blank">D3</a>&nbsp;</strong></h2> 
          <p>D3.js is a JavaScript library for manipulating documents based on data. D3 helps you bring data to life using HTML, SVG, and CSS. D3’s emphasis on web standards gives you the full capabilities of modern browsers without tying yourself to a proprietary framework, combining powerful visualization components and a data-driven approach to DOM manipulation.</p> 
          <h2><a name="_Toc414838064"></a> <a href="http://fabricjs.com/" target="_blank"><strong>Fabricjs</strong></a></h2> 
          <p>Fabric is a javascript HTML canvas library. You can create and populate objects on canvas; objects like simple geometrical shapes or complex shapes consisting of several paths. Fabric also has SVG-to-canvas (and canvas-to-SVG) parser.</p> 
          <p id="WSqsZtO"><img class=" size-full wp-image-4618  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d3481b0981.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d3481b0981.png 369w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d3481b0981-300x295.png 300w" sizes="(max-width: 369px) 100vw, 369px"></p> 
          <h2><a name="_Toc414838065"></a><a href="http://paperjs.org/" target="_blank"><strong>Paperjs</strong></a></h2> 
          <p>Paper.js is an open source vector graphics scripting framework that runs on top of the HTML5 Canvas. It offers a clean Scene Graph / Document Object Model and a lot of powerful functionality to create and work with vector graphics and bezier curves, all neatly wrapped up in a well designed, consistent and clean programming interface.</p> 
          <h2><a name="_Toc414838066"></a><a href="https://jsplumbtoolkit.com/" target="_blank"> <strong>JsPlumb</strong></a></h2> 
          <p>jsPlumb provides a means for a developer to visually connect elements on their web pages. It uses SVG in modern browsers, and VML on IE 8 and below.</p> 
          <p id="SQvPJdA"><img class=" size-full wp-image-4619  aligncenter" src="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d34ebec5c3.png" alt="" srcset="https://modeling-languages.com/wp-content/uploads/2015/04/img_551d34ebec5c3.png 492w, https://modeling-languages.com/wp-content/uploads/2015/04/img_551d34ebec5c3-300x200.png 300w" sizes="(max-width: 492px) 100vw, 492px"></p>

          <h2>Bonus&nbsp;(04/07/2015)</h2> 
          <p>This post has received&nbsp;a considerable &nbsp;amount of attention in the last few days, specially as part of the discussion on <a href="https://news.ycombinator.com/item?id=9325024" target="_blank">Hacker News</a> (thanks for that!). We were glad to receive several feedbacks and testimonies from&nbsp; people using the cited libraries and to learn about some interesting new ones. Here is a bonus&nbsp;extension to the post based on the discussion so far.</p> 
          <ul> 
           <li>&nbsp;<a href="http://p5js.org/" target="_blank"><strong>p5.js</strong></a>:&nbsp;A JS client-side library for creating graphic and interactive experiences, based on the core principles of <a href="https://processing.org/" target="_blank">Processing</a>.</li> 
           <li><strong><a href="https://cytoscape.github.io/cytoscape.js/" target="_blank">Cytoscape.js</a></strong>: An open-source &nbsp;graph theory library that can be used for graph analysis and visualization.</li> 
           <li><a href="https://knsv.github.io/mermaid/index.html" target="_blank"><strong>mermaid</strong></a>: Fan of Markdown? &nbsp;mermed is a markdown-like script language for generating Flowcharts, Sequence Diagams, and Grantt Diagrams from text via javascript.</li> 
           <li><a href="https://github.com/cpettitt/dagre-d3" target="_blank"><strong>dagre-d3</strong></a>: A javascript library that acts as a front-end of <a href="https://github.com/cpettitt/dagre" target="_blank">dagre</a>&nbsp;(javascript library to lay out directed graphs on the client side), providing the actual rendering using <a href="http://d3js.org" target="_blank">D3</a>.</li>
