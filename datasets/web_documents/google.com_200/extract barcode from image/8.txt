       </svg></a>// Barcode Writer in Pure JavaScript</h1> 
     <p><a href="http://metafloor.github.io/bwip-js"><img alt="bwip-js" align="right" src="http://metafloor.github.io/bwip-js/images/bwip-js.png"></a> bwip-js is a translation to native JavaScript of the amazing code provided in <a href="https://github.com/bwipp/postscriptbarcode">Barcode Writer in Pure PostScript</a>. The translated code can run on any modern browser or JavaScript-based server framework.</p> 
     <p>The software has encoding modules for over 90 different barcode types and standards. All linear and two-dimensional barcodes in common use (and many uncommon ones) are available. An exhaustive list of supported barcode types can be found at the end of this document.</p> 
     <blockquote> 
      <p>Version 1.5 has a replacement for the emscripten-compiled FreeType library (which was causing issues with popular frameworks and pre-allocated too much memory for use in embedded servers). It <em>is</em> still possible to use FreeType with Node.js, but you must explicitly enable it. By default, you get the replacement. Browsers only use the replacement.</p> 
      <p>The new font library uses bitmapped fonts that were generated by freetype.js at the integral scale factors of 1x - 9x. There should be no change in how the fonts display when the default font sizes are used. If you use custom sizes <em>OR</em> use non-uniform scaling, there may be noticeable differences.</p> 
      <p>See <a href="https://github.com/metafloor/bwip-js/wiki/FreeType-Replacement">FreeType Replacement</a> for more details.</p> 
     </blockquote>

     <p>The <code>bwipjs()</code> method takes three parameters:</p> 
     <ul> 
      <li>The canvas on which to render the barcode. This can by an <code>id</code> string or the actual canvas element. The rendering will automatically resize the canvas to match the barcode image.</li> 
      <li>A bwip-js/BWIPP options object. See the Node.js Image Generator section below for a description of this object.</li> 
      <li>A callback to invoke once the canvas has been rendered. It uses the traditional two parameter callback convention. If <code>err</code> is set, <code>err</code> will contain an Error object or string. Otherwise, <code>err</code> will be falsy and <code>cvs</code> will contain the canvas. This will be the same value passed in originally, either the <code>id</code> string or element.</li> 
     </ul>

     <p>Only the first two options <code>bcid</code> and <code>text</code> are required. The other bwip-js specific options are:</p> 
     <ul> 
      <li> <p><code>scaleX</code> : The x-axis scaling factor. Must be an integer &gt; 0. Default is 2.</p> </li> 
      <li> <p><code>scaleY</code> : The y-axis scaling factor. Must be an integer &gt; 0. Default is <code>scaleX</code>.</p> </li> 
      <li> <p><code>scale</code> : Sets both the x-axis and y-axis scaling factors. Must be an integer &gt; 0.</p> </li> 
      <li> <p><code>rotate</code> : Allows rotating the image to one of the four orthogonal orientations. A string value. Must be one of:</p> 
       <ul> 
        <li><code>"N"</code> : Normal (not rotated). The default.</li> 
        <li><code>"R"</code> : Clockwise (right) 90 degree rotation.</li> 
        <li><code>"L"</code> : Counter-clockwise (left) 90 degree rotation.</li> 
        <li><code>"I"</code> : Inverted 180 degree rotation.</li> 
       </ul> </li> 
      <li> <p><code>paddingwidth</code> : Sets the left and right padding (in points/pixels) around the rendered barcode. Rotates and scales with the image.</p> </li> 
      <li> <p><code>paddingheight</code> : Sets the top and bottom padding (in points/pixels) around the rendered barcode. Rotates and scales with the image.</p> </li> 
      <li> <p><code>monochrome</code> : Sets the human-readable text to render in monochrome. Boolean <code>true</code> or <code>false</code>. Default is <code>false</code> which renders 256-level gray-scale anti-aliased text.</p> </li> 
     </ul> 
     <p>All other options are BWIPP specific. You will need to consult the <a href="https://github.com/bwipp/postscriptbarcode/wiki">BWIPP documentation</a> to determine what options are available for each barcode type.</p> 
     <p>Note that bwip-js normalizes the BWIPP <code>width</code> and <code>height</code> options to always be in millimeters. The resulting images are rendered at 72 dpi. To convert to pixels, use a factor of 2.835 px/mm (72 dpi / 25.4 mm/in). The bwip-js option <code>scale</code> multiplies both the <code>width</code> and <code>height</code> options. Likewise, the discrete scaling factors <code>scaleX</code> and <code>scaleY</code> multiply the <code>width</code> and <code>height</code> options, respectively.</p>

       </svg></a>Demo Usage</h2> 
     <p>To run the demo from your HTTP server, you should link the <code>bwip-js</code> directory to the server's public document directory and modify the server's configuration files, if necessary. Then navigate your browser to <code>bwip-js/demo.html</code>.</p> 
     <blockquote> 
      <p>You cannot run the demo using a <code>file://</code> URL. The freetype library and the freetype replacement library use XHR, which must run over HTTP.</p> 
     </blockquote> 
     <p>If you would like to implement your own interface to bwip-js, see <a href="https://github.com/metafloor/bwip-js/wiki/Integrating-With-Your-Code">Integrating With Your Code</a>. You should also look at the <code>node-bwipjs.js</code> module to see how it was done for Node.js.</p>
