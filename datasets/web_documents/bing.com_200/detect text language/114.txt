    <h2 id="regular-expressions"><a class="link" href="#regular-expressions">1. Regular Expressions</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="what-are-regular-expressions"><a class="link" href="#what-are-regular-expressions">1.1. What are regular expressions?</a></h3> 
      <div class="paragraph"> 
       <p>A <em>regular expression</em> defines a search pattern for strings. The abbreviation for regular expression is <em>regex</em>. The search pattern can be anything from a simple character, a fixed string or a complex expression containing special characters describing the pattern. The pattern defined by the regex may match one or several times or not at all for a given string.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Regular expressions can be used to search, edit and manipulate text.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The process of analyzing or modifying a text with a regex is called: <em>The regular expression is applied to the text/string</em>. The pattern defined by the regex is applied on the text from left to right. Once a source character has been used in a match, it cannot be reused. For example, the regex <code>aba</code> will match <em>ababababa</em> only two times (aba_aba__).</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="regex-examples"><a class="link" href="#regex-examples">1.2. Regex examples</a></h3> 
      <div class="paragraph"> 
       <p>A simple example for a regular expression is a (literal) string. For example, the <em>Hello World</em> regex matches the "Hello World" string. <code>.</code> (dot) is another example for a regular expression. A dot matches any single character; it would match, for example, "a" or "1".</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following tables lists several regular expressions and describes which pattern they would match.</p> 
      </div>

        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">this is text</p></td>

        </tr>

         <td class="tableblock halign-left valign-top"><p class="tableblock">Matches the word "this" followed by one or more whitespace characters followed by the word "is" followed by one or more whitespace characters followed by the word "text".</p></td>

         <td class="tableblock halign-left valign-top"><p class="tableblock">^ defines that the patter must start at beginning of a new line. \d+ matches one or several digits. The ? makes the statement in brackets optional. \. matches ".", parentheses are used for grouping. Matches for example "5", "1.5" and "2.21".</p></td>

     <div class="sect2"> 
      <h3 id="grouping-and-back-reference"><a class="link" href="#grouping-and-back-reference">3.4. Grouping and back reference</a></h3> 
      <div class="paragraph"> 
       <p>You can group parts of your regular expression. In your pattern you group elements with round brackets, e.g., <code>()</code>. This allows you to assign a repetition operator to a complete group.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In addition these groups also create a back reference to the part of the regular expression. This captures the group. A back reference stores the part of the <code>String</code> which matched the group. This allows you to use this part in the replacement.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Via the <code>$</code> you can refer to a group. <code>$1</code> is the first group, <code>$2</code> the second, etc.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Let’s, for example, assume you want to replace all whitespace between a letter followed by a point or a comma. This would involve that the point or the comma is part of the pattern. Still it should be included in the result.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// Removes whitespace between a word character and . or ,</span>
<span class="predefined-type">String</span> pattern = <span class="string"><span class="delimiter">"</span><span class="content">(</span><span class="char">\\</span><span class="content">w)(</span><span class="char">\\</span><span class="content">s+)([</span><span class="char">\\</span><span class="content">.,])</span><span class="delimiter">"</span></span>;
<span class="predefined-type">System</span>.out.println(EXAMPLE_TEST.replaceAll(pattern, <span class="string"><span class="delimiter">"</span><span class="content">$1$3</span><span class="delimiter">"</span></span>));</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>This example extracts the text between a title tag.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// Extract the text between the two title elements</span>
pattern = <span class="string"><span class="delimiter">"</span><span class="content">(?i)(&lt;title.*?&gt;)(.+?)()</span><span class="delimiter">"</span></span>;
<span class="predefined-type">String</span> updated = EXAMPLE_TEST.replaceAll(pattern, <span class="string"><span class="delimiter">"</span><span class="content">$2</span><span class="delimiter">"</span></span>);</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="negative-look-ahead"><a class="link" href="#negative-look-ahead">3.5. Negative look ahead</a></h3> 
      <div class="paragraph"> 
       <p>Negative look ahead provides the possibility to exclude a pattern. With this you can say that a string should not be followed by another string.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Negative look ahead are defined via <code>(?!pattern)</code>. For example, the following will match "a" if "a" is not followed by "b".</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java">a(?!b)</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="specifying-modes-inside-the-regular-expression"><a class="link" href="#specifying-modes-inside-the-regular-expression">3.6. Specifying modes inside the regular expression</a></h3> 
      <div class="paragraph"> 
       <p>You can add the mode modifiers to the start of the regex. To specify multiple modes, simply put them together as in (?ismx).</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>(?i) makes the regex case insensitive.</p> </li> 
        <li> <p>(?s) for "single line mode" makes the dot match all characters, including line breaks.</p> </li> 
        <li> <p>(?m) for "multi-line mode" makes the caret and dollar match at the start and end of each line in the subject string.</p> </li> 
       </ul> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="backslashes-in-java"><a class="link" href="#backslashes-in-java">3.7. Backslashes in Java</a></h3> 
      <div class="paragraph"> 
       <p>The backslash <code>\</code> is an escape character in Java Strings. That means backslash has a predefined meaning in Java. You have to use double backslash <code>\\</code> to define a single backslash. If you want to define <code>\w</code>, then you must be using <code>\\w</code> in your regex. If you want to use backslash as a literal, you have to type <code>\\\\</code> as <code>\</code> is also an escape character in regular expressions.</p> 
      </div> 
     </div> 
    </div> 
   </div> 
   <div class="sect1"> 
    <h2 id="using-regular-expressions-with-string-methods"><a class="link" href="#using-regular-expressions-with-string-methods">4. Using regular expressions with String methods</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="redefined-methods-on-string-for-processing-regular-expressions"><a class="link" href="#redefined-methods-on-string-for-processing-regular-expressions">4.1. Redefined methods on String for processing regular expressions</a></h3> 
      <div class="paragraph"> 
       <p><code>Strings</code> in Java have built-in support for regular expressions. <code>Strings</code> have four built-in methods for regular expressions, i.e., the <code>matches()</code>, <code>split())</code>, <code>replaceFirst()</code> and <code>replaceAll()</code> methods. The <code>replace()</code> method does NOT support regular expressions.</p> 
      </div>

        <col style="width: 50%;"> 
       </colgroup>

        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock"><code>s.split("regex")</code></p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Creates an array with substrings of <code>s</code> divided at occurrence of <code>"regex"</code>. <code>"regex"</code> is not included in the result.</p></td> 
        </tr> 
        <tr> 
         <td class="tableblock halign-left valign-top"><p class="tableblock"><code>s.replaceFirst("regex"), "replacement"</code></p></td> 
         <td class="tableblock halign-left valign-top"><p class="tableblock">Replaces first occurance of <code>"regex"</code> with <code>"replacement</code>.</p></td>

        </tr>

                <span class="predefined-type">String</span> link = matcher.group().replaceFirst(<span class="string"><span class="delimiter">"</span><span class="content">href=</span><span class="char">\"</span><span class="delimiter">"</span></span>, <span class="string"><span class="delimiter">"</span><span class="delimiter">"</span></span>)

                        .replaceFirst(<span class="string"><span class="delimiter">"</span><span class="char">\"</span><span class="content">[</span><span class="char">\\</span><span class="content">s]?target=</span><span class="char">\"</span><span class="content">[a-zA-Z_0-9]*</span><span class="delimiter">"</span></span>, <span class="string"><span class="delimiter">"</span><span class="delimiter">"</span></span>);
                <span class="keyword">if</span> (valid(link)) {
                    links.add(makeAbsolute(url, link));
                }
            }
        } <span class="keyword">catch</span> (<span class="exception">MalformedURLException</span> e) {
            e.printStackTrace();
        } <span class="keyword">catch</span> (<span class="exception">IOException</span> e) {
            e.printStackTrace();
        }
        <span class="keyword">return</span> links;
    }

    <span class="directive">private</span> <span class="type">boolean</span> valid(<span class="predefined-type">String</span> s) {
        <span class="keyword">if</span> (s.matches(<span class="string"><span class="delimiter">"</span><span class="content">javascript:.*|mailto:.*</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> <span class="predefined-constant">false</span>;
        }
        <span class="keyword">return</span> <span class="predefined-constant">true</span>;
    }

    <span class="directive">private</span> <span class="predefined-type">String</span> makeAbsolute(<span class="predefined-type">String</span> url, <span class="predefined-type">String</span> link) {
        <span class="keyword">if</span> (link.matches(<span class="string"><span class="delimiter">"</span><span class="content">http://.*</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> link;
        }
        <span class="keyword">if</span> (link.matches(<span class="string"><span class="delimiter">"</span><span class="content">/.*</span><span class="delimiter">"</span></span>) &amp;&amp; url.matches(<span class="string"><span class="delimiter">"</span><span class="content">.*$[^/]</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> url + <span class="string"><span class="delimiter">"</span><span class="content">/</span><span class="delimiter">"</span></span> + link;
        }
        <span class="keyword">if</span> (link.matches(<span class="string"><span class="delimiter">"</span><span class="content">[^/].*</span><span class="delimiter">"</span></span>) &amp;&amp; url.matches(<span class="string"><span class="delimiter">"</span><span class="content">.*[^/]</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> url + <span class="string"><span class="delimiter">"</span><span class="content">/</span><span class="delimiter">"</span></span> + link;
        }
        <span class="keyword">if</span> (link.matches(<span class="string"><span class="delimiter">"</span><span class="content">/.*</span><span class="delimiter">"</span></span>) &amp;&amp; url.matches(<span class="string"><span class="delimiter">"</span><span class="content">.*[/]</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> url + link;
        }
        <span class="keyword">if</span> (link.matches(<span class="string"><span class="delimiter">"</span><span class="content">/.*</span><span class="delimiter">"</span></span>) &amp;&amp; url.matches(<span class="string"><span class="delimiter">"</span><span class="content">.*[^/]</span><span class="delimiter">"</span></span>)) {
            <span class="keyword">return</span> url + link;
        }
        <span class="keyword">throw</span> <span class="keyword">new</span> <span class="exception">RuntimeException</span>(<span class="string"><span class="delimiter">"</span><span class="content">Cannot make the link absolute. Url: </span><span class="delimiter">"</span></span> + url
                + <span class="string"><span class="delimiter">"</span><span class="content"> Link </span><span class="delimiter">"</span></span> + link);
    }

     <div class="sect2"> 
      <h3 id="finding-duplicated-words"><a class="link" href="#finding-duplicated-words">6.5. Finding duplicated words</a></h3> 
      <div class="paragraph"> 
       <p>The following regular expression matches duplicated words.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="error">\</span>b(<span class="error">\</span>w+)<span class="error">\</span>s+<span class="error">\</span><span class="integer">1</span><span class="error">\</span>b</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p><code>\b</code> is a word boundary and <code>\1</code> references to the captured match of the first group, i.e., the first word.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The <code>(?!-in)\b(\w+) \1\b</code> finds duplicate words if they do not start with "-in".</p> 
      </div> 
      <div class="paragraph"> 
       <p>TIP:Add <code>(?s)</code> to search across multiple lines.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="finding-elements-which-start-in-a-new-line"><a class="link" href="#finding-elements-which-start-in-a-new-line">6.6. Finding elements which start in a new line</a></h3> 
      <div class="paragraph"> 
       <p>The following regular expression allows you to find the "title" word, in case it starts in a new line, potentially with leading spaces.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="text">(\n\s*)title</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="finding-non-javadoc-statements"><a class="link" href="#finding-non-javadoc-statements">6.7. Finding (Non-Javadoc) statements</a></h3> 
      <div class="paragraph"> 
       <p>Sometimes (Non-Javadoc) are used in Java source code to indicate that the method overrides a super method. As of Java 1.6 this can be done via the <code>@Override</code> annotation and it is possible to remove these statements from your code. The following regular expression can be used to identify these statements.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="text">(?s) /\* \(non-Javadoc\).*?\*/</code></pre> 
       </div> 
      </div> 
      <div class="sect3"> 
       <h4 id="replacing-the-docbook-table-statement-with-asciidoc"><a class="link" href="#replacing-the-docbook-table-statement-with-asciidoc">6.7.1. Replacing the DocBook table statement with Asciidoc</a></h4> 
       <div class="paragraph"> 
        <p>You can replace statements like the following:</p> 
       </div> 
       <div class="listingblock"> 
        <div class="content"> 
         <pre class="CodeRay highlight"><code data-lang="console">&lt;programlisting language="java"&gt;
    &lt;xi:include xmlns:xi="http://www.w3.org/2001/XInclude" parse="text" href="./examples/statements/MyClass.java" /&gt;
&lt;/programlisting&gt;</code></pre> 
        </div> 
       </div> 
       <div class="paragraph"> 
        <p>Corresponding regex:</p> 
       </div> 
       <div class="listingblock"> 
        <div class="content"> 
         <pre class="CodeRay highlight"><code data-lang="console">`\s+&lt;programlisting language="java"&gt;\R.\s+&lt;xi:include xmlns:xi="http://www\.w3\.org/2001/XInclude" parse="text" href="\./examples/(.*).\s+/&gt;\R.\s+&lt;/programlisting&gt;`</code></pre> 
        </div> 
       </div> 
       <div class="paragraph"> 
        <p>Target could be your example:</p> 
       </div> 
       <div class="listingblock"> 
        <div class="content"> 
         <pre class="CodeRay highlight"><code data-lang="console">`\R[source,java]\R----\R include::res/$1[]\R----</code></pre> 
        </div> 
       </div>
