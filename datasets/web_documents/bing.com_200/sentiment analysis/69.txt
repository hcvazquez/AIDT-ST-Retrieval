                    <p>As I can see, '<em>sentiment</em>' package is no longer available in the CRAN repository, but you may want to consider something similar, try '<em>syuzhet'</em>. Easy to use.</p> 
                    <p><a href="https://www.researchgate.net/deref/https%3A%2F%2Fcran.r-project.org%2Fweb%2Fpackages%2Fsyuzhet%2Findex.html" rel="noopener" target="_blank">https://cran.r-project.org/web/packages/syuzhet/index.html</a></p> 
                    <p>Regarding a measure of 'accuracy' for automated sentiment analysis, that's not a simple task, IMHO. Keeping in mind that 'opinion' text analysis is very dependent on the content, you barely can rely on literature or others' work regarding some package/dictionary 'general' performance and should always check it against your particular problem.<br><br>This is only possible by comparing the model's performance with something you 'know'. If no other resources available (some very-similar-to-yours-already-tagged-text-in-the-exactly-same-manner-the-model-do), you should prepare the text (apply some pre-processing) and manually tag it. Next, you should send the same (pre-processed) text to the model and hopefully find some way to have the text tagged in the same way, to be able to perform a 1:1 comparison with your work. Once you have the TPs, TNs, FPs and FNs counted, do whatever error measure you want.<br>(As shown in your example, 'sentiment' package allows you to have the text tagged at the sentence level, there are ways around to do this for other packages at different token level, not easy though.)<br><br>However, working only with tweets and only at the binary level (pos/neg) might be easier using the <em>bag-of-words</em> approach. Once you have the pre-processed and tagged text, you may randomly split it in training/testing set (<em>hold-out validation</em>), train a model using a classifier on the training set (eg <em>Logistic Regression</em> or <em>CART</em> or <em>RF</em> or whatever you like) and use '<em>predict</em>' function to have the model's prediction on the test set. Manually compute the confusion matrix, or have all done, in an elegant manner - including 'confusionMatrix' - with '<em>caret</em>'.</p> 
                    <p><a href="https://www.researchgate.net/deref/http%3A%2F%2Ftopepo.github.io%2Fcaret%2Findex.html" rel="noopener" target="_blank">http://topepo.github.io/caret/index.html</a></p> 
                    <p>Once you have a good model for your purposes just use it again and again on future tweets, just feeding it with the same type of data.</p> 
                    <p><br>If your problem is more complex than pos/neg tagging the tweets, apply the same strategy with the appropriate classifiers at the desired token level, but keep in mind that the accuracy measures will only have some validity on very similar corpora (maybe same phraseology, maybe same POS structure, maybe same author, maybe same domain - economical, medical etc. etc - maybe same same same, maybe all of the above - which should be learned from documentation and/or practice).<br><br>Sorry for the lengthy answer, I just hope that wasn't more confusing :/</p> 
                    <p>Regards,<br>Cristian</p>
