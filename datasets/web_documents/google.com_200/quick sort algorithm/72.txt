                    <h1 class="post-title entry-title" itemprop="name"> QuickSort Example in Java using Recursion - Sorting Algorithm Implementation </h1> 
                    <div class="post-header"> 
                     <div style="float: right; margin:5px 6px 0px 5px; position:relative; z-index:10 "> 
                      <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script> 
                      <ins class="adsbygoogle" data-ad-client="ca-pub-5808379456926196" data-ad-slot="7171924747" style="display:inline-block;width:336px;height:280px"></ins>

                     <div dir="ltr" style="text-align: left;" trbidi="on">
                       Quicksort is one of the very popular sorting algorithms in programming, often used to sort a large list of numbers. Though their are numerous algorithm available to sort list of objects, including integer, string and floating point number, quicksort is best for general purpose. It's a divide and conquer algorithm, where we divide the given array with respect to a particular element, known as 'pivot' such that the lower partition of the array are less than the pivot and upper partition elements of the array are higher than the pivot. Quicksort is also one of the best example of 
                      <a href="http://javarevisited.blogspot.sg/2012/12/recursion-in-java-with-example-programming.html">recursion</a>. It's naturally recursive, because it sort the large list by dividing into smaller sub-list and then applying same algorithm on those.
                      <br> 
                      <a name="more"></a>
                      <br> The base case of recursion is when list contains either one or zero element, in that case they are sorted. Quicksort is well ahead with primitive sorting algorithms e.g. insertion sort, selection sort and 
                      <a href="http://java67.blogspot.sg/2012/12/bubble-sort-in-java-program-to-sort-integer-array-example.html">bubble sort</a>. Average time complexity of quicksort is 
                      <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(nlogn)</span>, while in worst case it's performance is similar to bubble sort i.e. 
                      <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(n^2)</span>.
                      <br> 
                      <br> Apparently the worst case of quicksort is the best case of insertion sort, where they have to sort an already sorted list. In this article, we will learn 
                      <i>how to implement quicksort algorithm in Java using recursion</i>.
                      <br> 
                      <br> We will also learn how quicksort works, and how it sorts a large list of unsorted number. In the last section, we will see some important things about quicksort.
                      <br> 
                      <br> 
                      <div style="display: table; margin-left: auto; margin-right: auto "> 
                       <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>

                       <script>
  (adsbygoogle = window.adsbygoogle || []).push({});
</script>

                      <br> 
                      <br> 
                      <h2 style="font-size: 16pt; line-height: 1.5; text-align: left;"> <u>How QuickSort Algorithm Perform Sorting</u></h2> An old saying is, a picture is worth more than a thousand words. This is completely true in case of understanding 
                      <i>how sorting algorithm works</i>. In past, I have understood insertion sort, selection sort and quicksort much better by following a diagram rather then reading about it. That's why I am sharing this diagram which explains how quicksort algorithm works, how it 
                      <a href="http://java67.blogspot.sg/2012/10/how-to-sort-object-in-java-comparator-comparable-example.html">sort a list of integers</a>. It's similar to flowchart but doesn't use the notation flowchart uses, instead it practically shows how sorting happens. Once you go through this diagram, read the explanation, it will make more sense.
                      <br> 
                      <br> 
                      <div class="separator" style="clear: both; text-align: center;"> 
                       <a href="http://4.bp.blogspot.com/-_YjUeWSvDvg/U8U8vDKLq8I/AAAAAAAABsY/cWS2GcgMNYI/s1600/Quicksort+in+Java+with+Example.png" style="clear: left; float: left; margin-bottom: 1em; margin-right: 1em;"><img alt="How to implement QuickSort in Java - Working Example" border="0" src="https://4.bp.blogspot.com/-_YjUeWSvDvg/U8U8vDKLq8I/AAAAAAAABsY/cWS2GcgMNYI/s1600/Quicksort+in+Java+with+Example.png" title=""></a>
                      </div> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br>

                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> 
                      <br> As I told before 
                      <i>QuickSort is a recursive algorithm</i>, it divides the big list into smaller list around pivot until those lists are individually sorted. First step of Quicksort algorithm is to determine pivot, it's general practice to choose 
                      <a href="http://java67.blogspot.sg/2012/12/how-to-remove-element-from-array-in-java-example.html">middle element of array</a> as pivot, but you are free to choose any index. Now you have two list, next step is to ensure that left partition only contains numbers less than pivot and right partition only contains numbers greater than pivot. We start pointer from left and right of pivot, and as soon as left pointer encounter 4, it stops because 4 is greater than 3. Similarly, right pointer stops at 3 because all numbers on right side of the list is greater than 3. Now it's time to swap, so 3 takes place of 4 and vice-versa. Now, we move pointer to one more step, since 
                      <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">2 &gt; 3</span>, left pointer shifts but since 
                      <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">4 &gt; 3</span>, it stopped. Since left point is also past right pointer it stopped. Now if we repeat this process one more time, list will be sorted.
                      <br> 
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> <u>Concept of &nbsp;Pivot and Partition</u></h3> Though we often select a middle element of the array as a pivot, there is no such there is no such rule and pivot can be any element of the given array. You can even consider the first element as the pivot in every partition. It's experienced that choice of pivot effects the distribution of the elements in partitioning and affects the complexity of the quicksort algorithm. As per rule of partition, numbers in lower partition should be less than the pivot and upper partition numbers should be higher than the pivot. Running time of partition logic is linear.
                      <br> 
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> <u>Complexity of Quicksort Algorithm:</u></h3> On an average Quicksort Algorithm has the complexity of 
                      <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(nlogn)</span> and in the worst case, it has&nbsp;O(n²)&nbsp;when the elements of the input array are already sorted in ascending or descending order. The good&nbsp;thing about Quicksort is that it's an 
                      <i>in place algorithm</i>, which means it does not take any additional space, except those used by method stack. By the way, there are 
                      <b>some tricks to improve the performance of quicksort</b>, even in worst case. As suggested in one of the best algorithm design book, The Algorithm Design Manual, from Steven Skiena, you can apply the following the recommendation to improve your quicksort algorithm implementation.
                      <br> 
                      <br> 1) Randomization
                      <br> You can avoid worst-case performance of O(n²) when sorting nearly-sorted data by random permutation of keys. Though it incurs some cost of permutation but still gives better performance than O(n²)
                      <br> 
                      <br> 2) Leave small sub-arrays for Insertion sort
                      <br> finish Quicksort recursion and switch to insertion sort when fewer than 20 elements:
                      <br> 
                      <div> 
                       <br>
                      </div> By the way, there is a drawback of using recursion to implement quicksort algorithm, It will not scale because 
                      <a href="http://java67.blogspot.sg/2013/02/difference-between-jit-and-jvm-in-java.html">JVM </a>has no tail call optimization, it will simply grow the method call stack to something proportional to the array to sort, and it will fail for the very large array.

Logic of quicksort is encapsulated in method

                      <span style="color: #ff7800; font-family: &quot;consolas&quot; , &quot;lucida console&quot; , &quot;dejavu sans mono&quot; , &quot;monaco&quot; , &quot;courier new&quot; , monospace; font-size: 0.9333em; line-height: 1.5em;">int</span>

In order to hide implementation details, we have only exposed a&nbsp;</span></span><span style="line-height: 24px;">convenient</span><span style="font-family: inherit;"><span style="line-height: 1.5em;">&nbsp;static utility method called&nbsp;</span></span></span>

                      <span style="background-color: white; font-family: &quot;consolas&quot; , &quot;lucida console&quot; , &quot;dejavu sans mono&quot; , &quot;monaco&quot; , &quot;courier new&quot; , monospace; font-size: 0.9333em; line-height: 1.5em;">(</span>
                      <span style="color: #ff7800; font-family: &quot;consolas&quot; , &quot;lucida console&quot; , &quot;dejavu sans mono&quot; , &quot;monaco&quot; , &quot;courier new&quot; , monospace; font-size: 0.9333em; line-height: 1.5em;">int</span>
                      <span style="background-color: white; font-family: &quot;consolas&quot; , &quot;lucida console&quot; , &quot;dejavu sans mono&quot; , &quot;monaco&quot; , &quot;courier new&quot; , monospace; font-size: 0.9333em; line-height: 1.5em;">[] array), </span>
                      <span style="background-color: white; line-height: 1.5em;"><span style="font-family: inherit;">which takes an integer array and sort that in-place.</span></span>

                      <pre style="background: rgb(255, 255, 255); font-family: Consolas, 'Lucida Console', 'DejaVu Sans Mono', Monaco, 'Courier New', monospace; font-size: 0.9333em; line-height: 1.5em; padding: 4px;"><span style="color: #ff7800;">package</span> <span style="color: #ff7800;">test</span>;

<span style="color: #ff7800;">import</span> <span style="color: #ff7800;">java.util.Arrays</span>;


<span style="color: #8c868f;">/**
* Java program to Sort integer array using QuickSort algorithm using recursion.
* Recursive QuickSort algorithm, partitioned list into two parts by a pivot,
* and then recursively sorts each list.
* <span style="color: #ff7800;">@author</span> Javin Paul
*/</span>
<span style="color: #ff7800;">public</span> <span style="color: #ff7800;">class</span> <span style="color: #3b5bb5;">QuickSort</span>{

    <span style="color: #ff7800;">public</span> <span style="color: #ff7800;">static</span> <span style="color: #ff7800;">void</span> <span style="color: #3b5bb5;">main</span>(<span style="color: #ff7800;">String</span> args[]) {

        <span style="color: #ff7800;">int</span>[] input <span style="color: #ff7800;">=</span> { <span style="color: #3b5bb5;">23</span>, <span style="color: #3b5bb5;">31</span>, <span style="color: #3b5bb5;">1</span>, <span style="color: #3b5bb5;">21</span>, <span style="color: #3b5bb5;">36</span>, <span style="color: #3b5bb5;">72</span>};
        <span style="color: #ff7800;">System</span><span style="color: #ff7800;">.</span>out<span style="color: #ff7800;">.</span>println(<span style="color: #409b1c;">"Before sorting : "</span> <span style="color: #ff7800;">+</span> <span style="color: #ff7800;">Arrays</span><span style="color: #ff7800;">.</span>toString(input));
        quickSort(input); <span style="color: #8c868f;">// sort the integer array using quick sort algorithm</span>
        <span style="color: #ff7800;">System</span><span style="color: #ff7800;">.</span>out<span style="color: #ff7800;">.</span>println(<span style="color: #409b1c;">"After sorting : "</span> <span style="color: #ff7800;">+</span> <span style="color: #ff7800;">Arrays</span><span style="color: #ff7800;">.</span>toString(input));
     
        <span style="color: #8c868f;">// input with duplicates</span>
        <span style="color: #ff7800;">int</span>[] withDuplicates <span style="color: #ff7800;">=</span> { <span style="color: #3b5bb5;">11</span>, <span style="color: #3b5bb5;">14</span>, <span style="color: #3b5bb5;">16</span>, <span style="color: #3b5bb5;">12</span>, <span style="color: #3b5bb5;">11</span>, <span style="color: #3b5bb5;">15</span>};
        <span style="color: #ff7800;">System</span><span style="color: #ff7800;">.</span>out<span style="color: #ff7800;">.</span>println(<span style="color: #409b1c;">"Before sorting : "</span> <span style="color: #ff7800;">+</span> <span style="color: #ff7800;">Arrays</span><span style="color: #ff7800;">.</span>toString(withDuplicates));
        quickSort(withDuplicates); <span style="color: #8c868f;">// sort the integer array using quick sort algorithm</span>
        <span style="color: #ff7800;">System</span><span style="color: #ff7800;">.</span>out<span style="color: #ff7800;">.</span>println(<span style="color: #409b1c;">"After sorting : "</span> <span style="color: #ff7800;">+</span> <span style="color: #ff7800;">Arrays</span><span style="color: #ff7800;">.</span>toString(withDuplicates));
    }

    <span style="color: #8c868f;">/**
     * public method exposed to client, sorts given array using QuickSort
     * Algorithm in Java
     * <span style="color: #ff7800;">@param</span> array
     */</span>
    <span style="color: #ff7800;">public</span> <span style="color: #ff7800;">static</span> <span style="color: #ff7800;">void</span> <span style="color: #3b5bb5;">quickSort</span>(<span style="color: #ff7800;">int</span>[] array) {
        recursiveQuickSort(array, <span style="color: #3b5bb5;">0</span>, array<span style="color: #ff7800;">.</span>length <span style="color: #ff7800;">-</span> <span style="color: #3b5bb5;">1</span>);
    }

    <span style="color: #8c868f;">/**
     * Recursive quicksort logic
     *
     * <span style="color: #ff7800;">@param</span> array input array
     * <span style="color: #ff7800;">@param</span> startIdx start index of the array
     * <span style="color: #ff7800;">@param</span> endIdx end index of the array
     */</span>
    <span style="color: #ff7800;">public</span> <span style="color: #ff7800;">static</span> <span style="color: #ff7800;">void</span> <span style="color: #3b5bb5;">recursiveQuickSort</span>(<span style="color: #ff7800;">int</span>[] array, <span style="color: #ff7800;">int</span> startIdx, <span style="color: #ff7800;">int</span> endIdx) {
     
        <span style="color: #ff7800;">int</span> idx <span style="color: #ff7800;">=</span> partition(array, startIdx, endIdx);

        <span style="color: #8c868f;">// Recursively call quicksort with left part of the partitioned array</span>
        <span style="color: #ff7800;">if</span> (startIdx <span style="color: #ff7800;">&lt;</span> idx <span style="color: #ff7800;">-</span> <span style="color: #3b5bb5;">1</span>) {
            recursiveQuickSort(array, startIdx, idx <span style="color: #ff7800;">-</span> <span style="color: #3b5bb5;">1</span>);
        }

        <span style="color: #8c868f;">// Recursively call quick sort with right part of the partitioned array</span>
        <span style="color: #ff7800;">if</span> (endIdx <span style="color: #ff7800;">&gt;</span> idx) {
            recursiveQuickSort(array, idx, endIdx);
        }
    }

    <span style="color: #8c868f;">/**
     * Divides array from pivot, left side contains elements less than
     * Pivot while right side contains elements greater than pivot.
     *
     * <span style="color: #ff7800;">@param</span> array array to partitioned
     * <span style="color: #ff7800;">@param</span> left lower bound of the array
     * <span style="color: #ff7800;">@param</span> right upper bound of the array
     * <span style="color: #ff7800;">@return</span> the partition index
     */</span>
    <span style="color: #ff7800;">public</span> <span style="color: #ff7800;">static</span> <span style="color: #ff7800;">int</span> <span style="color: #3b5bb5;">partition</span>(<span style="color: #ff7800;">int</span>[] array, <span style="color: #ff7800;">int</span> left, <span style="color: #ff7800;">int</span> right) {
        <span style="color: #ff7800;">int</span> pivot <span style="color: #ff7800;">=</span> array[left]; <span style="color: #8c868f;">// taking first element as pivot</span>

        <span style="color: #ff7800;">while</span> (left <span style="color: #ff7800;">&lt;=</span> right) {
            <span style="color: #8c868f;">//searching number which is greater than pivot, bottom up</span>
            <span style="color: #ff7800;">while</span> (array[left] <span style="color: #ff7800;">&lt;</span> pivot) {
                left<span style="color: #ff7800;">++</span>;
            }
            <span style="color: #8c868f;">//searching number which is less than pivot, top down</span>
            <span style="color: #ff7800;">while</span> (array[right] <span style="color: #ff7800;">&gt;</span> pivot) {
                right<span style="color: #ff7800;">--</span>;
            }

            <span style="color: #8c868f;">// swap the values</span>
            <span style="color: #ff7800;">if</span> (left <span style="color: #ff7800;">&lt;=</span> right) {
                <span style="color: #ff7800;">int</span> tmp <span style="color: #ff7800;">=</span> array[left];
                array[left] <span style="color: #ff7800;">=</span> array[right];
                array[right] <span style="color: #ff7800;">=</span> tmp;

                <span style="color: #8c868f;">//increment left index and decrement right index</span>
                left<span style="color: #ff7800;">++</span>;
                right<span style="color: #ff7800;">--</span>;
            }
        }
        <span style="color: #ff7800;">return</span> left;
    }
}

                       <h3 style="text-align: left;"> <u>Things to know about QuickSort Algorithm in Java</u></h3> As I said, QuickSort is one of the most popular sorting algorithms between programmers, may be just next to Bubble sort, which is ironically worst algorithm to 
                       <a href="http://java67.blogspot.sg/2012/08/how-to-sort-arraylist-in-java-list.html">sort a large list of numbers</a>. But one thing is common between QuickSort and Bubble Sort, do you know what? In worst case both have the complexity of 
                       <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(n^2)</span>.
                       <br> 
                       <br> 1) QuickSort is a divide and conquer algorithm, which means it sort a large array of numbers by dividing them into a smaller array and then individually sorting them (conquer).
                       <br> 
                       <br> 2) Average case complexity of Quicksort is 
                       <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(n log(n))</span> and worst case complexity of Quicksort is&nbsp;
                       <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">O(n²)</span>.
                       <br> 
                       <br> 3)&nbsp;Quicksort is a comparison sort and, in efficient implementations, it's not a stable sort, which means equal numbers may not retain their original position after sorting.
                       <br> 
                       <br> 4) Quicksort algorithm can be implemented in-place, which means no additional space will be required. This makes it suitable to sort the large array of numbers.
                       <br> 
                       <br> 5) 
                       <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">Arrays.sort()</span> method in Java use quicksort to sort an array of primitives e.g. array of integers or float and uses 
                       <span style="font-family: &quot;courier new&quot; , &quot;courier&quot; , monospace;">Mergesort </span>to sot objects e.g. 
                       <a href="http://java67.blogspot.sg/2012/09/java-program-to-convert-string-arraylist-to-string-array.html">array of String</a>.
