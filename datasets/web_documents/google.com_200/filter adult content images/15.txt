    <div id="tgLic" style="display:none;color:#666;"> 
     <h2>Licensing</h2> 
     <ul> 
      <li>The license is issued on a per-domain basis (valid for a domain and its sub-domains). Intranet domains and IP domains should also be licensed.</li> 
      <li>A license can be used for one type of widget.</li> 
      <li>You can use the licensed widget multiple times within the website for which you've purchased a license.</li> 
      <li>Please refer to the license section in each product's main page for more details.</li> 
     </ul> 
     <p class="h2">FAQ about License</p> 
     <div class="litbox"> 
      <ol> 
       <li>How the license works? Does it go to another web service to verify? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipHowLicWorks',{position:0})">Answer</span></li> 
       <li>Do I need a license for my dev and testing domain that is different from the final client's domain? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipDevTst',{position: 0})">Answer</span></li> 
       <li>Will it work for sub-domains? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipSubdomain',{position: 0})">Answer</span></li> 
       <li>My website is on intranet. Do I need license for using your widgets in my intranet website? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipIntro',{position: 0})">Answer</span></li> 
       <li>Do you provide developer license that can be applied to multiple websites? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipLicForDvlper',{position:0})">Answer</span></li> 
       <li> I have multiple domain names pointing to the same website. Can I apply multiple licenses to the same script? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipLicForMulti',{position:0})">Answer</span> </li> 
       <li>Do I need to renew the license for future upgrades/releases of the widget? <span class="tooltip" onmouseover="tooltip.pop(this,'#tipNeedReview',{position:0})">Answer</span></li> 
      </ol> 
     </div> 
     <div style="display:none;"> 
      <div id="tipHowLicWorks">
       You need to set the acquired license key to the license property in the widget's JavaScript file. 
       <p>The JavaScript will use its own match pattern code to verify the license with the domain name on the browser's address bar. The JavaScript file will perform the validation by itself and no other services will be involved.</p> 
       <p>If the validation failed(the domain name doesn't match the license key), the UI component may periodically display a trial version alert, or even disable some functionalities.</p> 
      </div> 
      <div id="tipDevTst">
       <p>The code is fully functional even if the script is not licensed (though it may periodically show a trial version alert). So usually you don't need the license in your dev phase.</p> 
       <p>If you are developing under the domain <em>localhost</em>, either the trial version alert will be disabled, or we have provided a license for the <em>localhost</em> that you can find in the License section of the product's main page.</p> 
      </div> 
      <div id="tipSubdomain">
       Yes. If the license is valid for the main domain, it will also be valid for all its sub-domains(such as sub1.mydomain.com, sub2.mydomain.com).
      </div> 
      <div id="tipIntro">
       Yes license is required for intranet domains, and you can enter your intranet domain into the 
       <em>For website</em> box after clicking the Buy Now button.
      </div> 
      <div id="tipLicForDvlper">
       No. It is on a per-domain basis only.
      </div> 
      <div id="tipLicForMulti"> 
       <div>
        Yes, it is supported. The following links are the instructions:
       </div> 
       <p> <a href="/2115/Multiple-Licenses-for-the-same-slider">For JS Image Slider</a> &nbsp; &nbsp; <a href="/5832/License-question-for-the-website-that-has-multiple-domain-names">For Ninja Slider</a> &nbsp; &nbsp; <a href="/2143/License-for-a-website-that-has-multiple-domains-including-intranet">For ddmenu</a> &nbsp; &nbsp; <a href="/1744/License-for-two-domain-names">For JS Tooltip</a> </p> Note: 
       <ul style="width:500px;"> 
        <li>If your domain2.com will be redirected to your domain1.com, you only need a license for domain1.com. You don't need multiple licenses in this scenario.</li> 
        <li> If your multiple domains are obviously for the same site, such as menucool.com, menucool.net, menucool.com.us, you can input the regular price for the first domain, and input 1/2 of the price for the others. </li> 
       </ul> 
      </div> 
      <div id="tipNeedReview">
       <p>No. The license will be valid forever as long as your domain is not changed.</p> 
       <p>If your domain name is changed, you need to purchase a separate license for the new domain.</p>
      </div>
