        <div class="palsGenericContent ExLinks"> 
         <p>The conventional SharePoint 2013 or 2010 farm solution model accepts ImageFieldValue for parsing Publishing Image tag. But this approach does not work out for Sandboxed solution. Although usage of ImageFieldvalue does not throw any error during build and deployment ,it will throw and error on execution. To overcome this issue use the below code to extract the Image URL from PublishingPageImage field</p>
         <p><strong>Sandboxed Solution</strong></p>
         <pre class="brush: csharp; toolbar: false; auto-links: false; smart-tabs: false;">
&nbsp;private string GetPublishingPageImageURL(SPListItem item)
&nbsp;{
&nbsp;    try
&nbsp;    {
&nbsp;        string ImgTag = item.GetFormattedValue("PublishingPageImage");
&nbsp;        return String.IsNullOrEmpty(ImgTag) ? "#" : XElement.Parse(ImgTag + "&lt;/img&gt;").Attribute("src").Value;
&nbsp;    }
&nbsp;    catch
&nbsp;    {
&nbsp;        return "#";
&nbsp;    }
&nbsp;}
</pre>
         <p></p>
         <p><strong>Conventional Method by referring Microsoft.SharePoint.Publishing.Fields namespace</strong></p>
         <pre class="brush: csharp; toolbar: false; auto-links: false; smart-tabs: false;">
&nbsp;private string GetPublishingPageImageURL(SPListItem item)
&nbsp;{
&nbsp;    try
&nbsp;    {
&nbsp;        ImageFieldValue pageImage = item["PublishingPageImage"] as ImageFieldValue;
&nbsp;        return string.IsNullOrEmpty(pageImage.ImageUrl) ? "#" : pageImage.ImageUrl;
&nbsp;    }
&nbsp;    catch  
&nbsp;    {
&nbsp;        return "#";
&nbsp;    }
&nbsp;}
</pre>
