        <h2><a name="Overview" id="Overview"></a>Overview</h2> 
        <p><i>Org charts</i> are diagrams of a hierarchy of nodes, commonly used to portray superior/subordinate relationships in an organization. A family tree is a type of org chart.</p>

        <h2><a name="Data_Format" id="Data_Format"></a>Data Format</h2> 
        <p> A table with three string string columns, where each row represents a node in the orgchart. Here are the three columns:</p> 
        <ul> 
         <li>Column 0 - The node ID. It should be unique among all nodes, and can include any characters, including spaces. This is shown on the node. You can specify a formatted value to show on the chart instead, but the unformatted value is still used as the ID.</li> 
         <li>Column 1 - [<em>optional</em>] The ID of the parent node. This should be the unformatted value from column 0 of another row. Leave unspecified for a root node.</li> 
         <li>Column 2 - [<em>optional</em>] Tool-tip text to show, when a user hovers over this node.</li> 
        </ul> 
        <p>Each node can have zero or one parent node, and zero or more child nodes.</p>

           <td>collapse</td> 
           <td> <p> Event triggered when <code>allowCollapse</code> is set to <code>true</code> and the user double clicks on a node with children. </p> 
            <div> 
             <div>
              <b>Properties:</b>
             </div> 
             <div> 
              <code>collapsed</code> - A boolean indicating whether this is a 'collapse' or 'expand' event. 
             </div> 
             <div> 
              <code>row</code> - The zero-based index of the row in the data table, corresponding to the node being clicked. 
             </div> 
            </div> </td> 
          </tr> 
          <tr> 
           <td>onmouseover</td> 
           <td> <p>Triggered when the user hovers over a specific row.</p> 
            <div> 
             <div>
              <b>Properties:</b>
             </div> 
             <div> 
              <code>row</code> - The zero-based index of the row in the data table, corresponding to the node being moused over. 
             </div> 
            </div> </td> 
          </tr> 
          <tr> 
           <td>onmouseout</td> 
           <td> <p>Triggered when the user hovers out of a row.</p> 
            <div> 
             <div>
              <b>Properties:</b>
             </div> 
             <div> 
              <code>row</code> - The zero-based index of the row in the data table, corresponding to the node being moused out from. 
             </div> 
            </div> </td> 
          </tr> 
          <tr> 
           <td>select</td> 
           <td> <p>Standard select event</p> 
            <div> 
             <div>
              <b>Properties:</b>
             </div> 
             <div>
              None
             </div> 
            </div> </td> 
          </tr> 
          <tr> 
           <td>ready</td> 
           <td> <p> The chart is ready for external method calls. If you want to interact with the chart, and call methods after you draw it, you should set up a listener for this event <i>before</i> you call the <code>draw</code> method, and call the methods only after the event is fired. </p> 
            <div> 
             <div>
              <b>Properties:</b>
             </div> 
             <div>
              None
             </div>
