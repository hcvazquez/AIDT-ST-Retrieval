        <article class="markdown-body entry-content" itemprop="text">
         <h1><a id="user-content-exifjs" class="anchor" href="#exifjs" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Exif.js</h1> 
         <p>A JavaScript library for reading <a href="https://en.wikipedia.org/wiki/Exchangeable_image_file_format">EXIF meta data</a> from image files.</p> 
         <p>You can use it on images in the browser, either from an image or a file input element. Both EXIF and IPTC metadata are retrieved. This package can also be used in AMD or CommonJS environments.</p> 
         <p><strong>Note</strong>: The EXIF standard applies only to <code>.jpg</code> and <code>.tiff</code> images. EXIF logic in this package is based on the EXIF standard v2.2 (<a href="/exif-js/exif-js/blob/master/spec/Exif2-2.pdf">JEITA CP-3451, included in this repo</a>).</p> 
         <h2><a id="user-content-install" class="anchor" href="#install" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Install</h2> 
         <p>Install <code>exif-js</code> through <a href="https://www.npmjs.com/#getting-started">NPM</a>:</p> 
         <pre><code>npm install exif-js --save    
</code></pre> 
         <p>Or <a href="http://bower.io/">Bower</a>:</p> 
         <pre><code>bower install exif-js --save
</code></pre> 
         <p>Then add a <code>script</code> tag in your an HTML in the <a href="http://stackoverflow.com/questions/436411/where-is-the-best-place-to-put-script-tags-in-html-markup">best position</a> referencing your local file.</p> 
         <pre><code>&lt;script src="vendors/exif-js/exif-js"&gt;&lt;/script&gt;
</code></pre> 
         <p><strong>Note</strong>: This repo has no <code>.min.js</code>. Do your own <a href="https://en.wikipedia.org/wiki/Minification_(programming)">minification</a> if you want that.</p> 
         <p>If you prefer another package manager you will probably manage :D. Or you can clone this GIT repository or download it's ZIP file and extract <code>exif.js</code> to your project.</p> 
         <h2><a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Usage</h2> 
         <p>The package adds a global <code>EXIF</code> variable (or AMD or CommonJS equivalent).</p> 
         <p>Start with calling the <code>EXIF.getData</code> function. You pass it an image as a parameter:</p> 
         <ul> 
          <li>either an image from a <code>&lt;img src="image.jpg"&gt;</code></li> 
          <li>OR a user selected image in a <code>&lt;file type="input"&gt;</code> element on your page.</li> 
         </ul> 
         <p>As a second parameter you specify a callback function. In the callback function you should use <code>this</code> to access the image with the aforementioned metadata you can then use as you want. That image now has an extra <code>exifdata</code> property which is a Javascript object with the EXIF metadata. You can access it's properties to get data like the <em>image caption</em>, the <em>date a photo was taken</em> or it's <em>orientation</em>.</p> 
         <p>You can get all tages with <code>EXIF.getTag</code>. Or get a single tag with <code>EXIF.getTag</code>, where you specify the tag as the second parameter. The tag names to use are listed in <code>EXIF.Tags</code> in <code>exif.js</code>.</p> 
         <p><strong>Important</strong>: Note that you have to wait for the image to be completely loaded, before calling <code>getData</code> or any other function. It will silently fail otherwise. You can implement this wait, by running your exif-extracting logic on the <code>window.onLoad</code> function. Or on an image's own <code>onLoad</code> function. For jQuery users please note that you can NOT (reliably) use jQuery's <code>ready</code> event for this. Because it fires before images are loaded. You could use $(window).load() instead of $(document.ready() (please note that `exif-js has NO dependency on jQuery or any other external library).</p> 
         <p><strong>JavaScript</strong>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">window</span>.<span class="pl-smi">onload</span><span class="pl-k">=</span>getExif;

<span class="pl-k">function</span> <span class="pl-en">getExif</span>() {

         <p><strong>XMP</strong> Since issue #53 was merged also extracting of XMP data is supported. To not slow down this is optional, and you need to call <code>EXIF.enableXmp();</code> before using <code>..getDatat()</code>.</p>
