           <div class="text parbase section"> 
            <div class="text-component    "> 
             <p><em>This is the first in a series of posts that&nbsp;will teach you how to take advantage of your Web development knowledge in building hybrid applications for iOS and Android.</em></p>
             <p>It should come as no big surprise that users now spend&nbsp;far more time in the apps on their phones, rather than surfing the Web with their mobile devices. Given this, it's an opportune time to learn how to make your own apps.<em>&nbsp;</em>But where exactly do you start? Can you use some of your existing developer skills? Until recently, if you wanted to make an app for the two most popular mobile operating systems (iOS and Android) your only bet was to make the native application by using the&nbsp;respective&nbsp;SDKs. This, of course, meant that you needed to make two versions of your application; one for each OS. If youre a solo developer, you may not be proficient in both.</p>
             <p>For some time, developers were opting for either iOS or Android, whereas big firms had two developing departments, one for each platform. But now, with the Ionic Framework (and few others like <a href="http://phonegap.com/)," target="_blank">PhoneGap</a>, <a href="http://onsen.io/)," target="_blank">OnsenUI</a>, <a href="http://famous.org/" target="_blank">Famo.us</a>) you can create one application by using the skills you already have and then deploying this one codebase as an app to both iOS and Android stores.<br></p>
             <h3>Methods for making an app</h3>
             <br>There are three ways you can make an application for mobile devices:
             <br>
             <ul>
              <br> 
              <li>Native app</li>
              <br> 
              <li>Mobile website</li>
              <br> 
              <li>Hybrid app</li>
              <br>
             </ul>
             <br>
             <strong>Native app</strong>
             <p></p>
             <p>As mentioned, you can make an app specifically for iOS and Android by using specific SDKs. If you want to build a native application for iOS you must:<br></p>
             <ul>
              <br> 
              <li>Have a Mac computer. Sure, there are ways around it, but I don't recommend them. For starters, a cheap Mac Mini will do just fine.</li>
              <br> 
              <li>Download <a href="https://developer.apple.com/xcode/" target="_blank">Xcode</a>&nbsp;from the App Store.</li>
              <br> 
              <li>Buy the <a href="https://developer.apple.com/programs/" target="_blank">Apple Developer license</a> at $99 per year (if you want to publish to the App Store).</li>
              <br>
             </ul>
             <br>You can write the apps by using the 
             <a href="https://developer.apple.com/swift/">Swift</a> language or its predecessor,&nbsp;
             <a href="https://developer.apple.com/library/mac/documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC/Introduction/Introduction.html">ObjectiveC</a>. Swift is a great step up from the clunky ObjectiveC; If you ever decide to go native, make sure you go with Swift. You'll get to know your way around it sooner than you would with ObjectiveC, especially if you have a background in Web development.
             <p></p>
             <p>To build a native application for Android you must:<br></p>
             <ul>
              <br> 
              <li>Have any computer.</li>
              <br> 
              <li>Download the appropriate SDKs (we'll cover this in the next section).</li>
              <br> 
              <li>Buy the <a href="https://play.google.com/apps/publish/signup/" target="_blank">Google Developer license</a>&nbsp;at $25 one time fee (if you want to publish to the Play Store).</li>
              <br>
             </ul>
             <br>One benefit of native applications is the speed and direct access to a native API (you don't have to use any middleman wrappers, like in hybrid apps). On the downside, you need to build two or more applications, one for each desired platform.
             <p></p>
             <p><strong>Mobile website</strong></p>
             <p>Mobile website is a normal website that you visit with your phone's browser, designed specifically to adapt to your phone's screen. Developers once made specific sites just for mobile browsers but this proved difficult to maintain. A practice called <a href="https://developer.apple.com/library/mac/documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC/Introduction/Introduction.html" target="_blank">responsive website design</a> is now used, in which you have one HTML codebase, and you determine the look for specific devices (based on resolutions) by using the <a href="https://developer.mozilla.org/en-US/docs/Web/Guide/CSS/Media_queries" target="_blank">media queries</a>.&nbsp;<a href="https://jquerymobile.com/">jQuery mobile</a> is a great example of a mobile framework. Use it&nbsp;if you're only making a mobile version of your Web application. A definite advantage of mobile websites is that you can update them as needed, without waiting for approval from Apple or Google. Disadvantages of include lower engagement, and less features.</p>
             <p><strong>Hybrid app</strong></p>
             <p>A hybrid app is a basically a mobile application. Its written with the same languages that you use when building websites, with the addition that it contains an isolated browser instance called WebView, which runs a Web application inside of a native app. Hybrid apps can access the mobile device and use the additional phone features like camera or GPS.&nbsp;Hybrid apps have the advantage of allowing you to access additional phone features via plugins. You can also do all development with the same set of skills used when developing normal Web applications. While it's improving, the Web View has its limitations in terms of speed. So, it might not be best suited if you're on a quest to make the next best game with full blown 3D graphics.</p>
             <p><strong>What is Ionic (and why is it so great?)</strong></p>
             <p>Ionic is so much more than just a UI framework. Ionic allows you to:<br></p>
             <ul>
              <br> 
              <li>Generate icons and splash screens for all devices and device sizes with the single command <tt><code>ionic resources</code></tt></li>
              <br> 
              <li>Instantly update your apps with code changes, even when running directly on your device, with <tt><code>ionic run --livereload</code></tt></li>
              <br> 
              <li>Build and test iOS and Android versions side-by-side and see changes instantly with <tt><code>ionic serve --lab</code></tt></li>
              <br> 
              <li>Share your Ionic apps with clients, customers, and testers all around the world without ever going through the App Store, with <tt><code>ionic share</code></tt></li>
              <br> 
              <li>Easily access the full native functionality of the device using ngCordova.</li>
              <br>
             </ul>
             <br>Also, 
             <a href="http://drifty.com/" target="_blank">Drifty</a>&nbsp;(the team behind the Ionic framework) is building a full-stack back-end service and tools for your Ionic app like Deploy (for deploying a new version without going through Apples review process!). Ionic Command Line Interface (CLI) uses Cordova in the back-end and allows you to build apps for iOS and Android (just by doing 
             <tt><code>ionic build ios</code></tt> or 
             <tt><code>ionic build android</code></tt>). Ionic uses Angular as a front-end framework, so if youre familiar with it, it will come as a bonus. Theyre working closely with the 
             <a href="http://blog.ionic.io/angular-2-ionic/" target="_blank">Angular 2.0 team</a> too.
             <p></p>
             <p>Now, let's install all the prerequisites and start using Ionic!<br></p>
             <h3>Installing prerequisites for Mac and Windows</h3>
             <br>We need to have Node.js and Git installed in order to install both Ionic and Cordova. If you already have these tools installed, you can skip this section and go straight to installing Ionic.
             <p></p>
             <p><strong>Installing Node.js</strong></p>
             <p>In order to download Node.js, visit <a href="http://nodejs.org/download/" target="_blank">http://nodejs.org/download/</a>. Installation on Windows and Mac OS is simple, as you just have to download and run the appropriate installer and follow the familiar instructions. If you have <code>brew</code> on your Mac, you can also install Node.js with <tt><code>brew install node</code></tt></p>
             <p>In both cases, Node Package Manager (NPM) will be used to install other packages along with Node.js. To verify that you installed Node.js correctly on a Windows machine, run the following command in your Command prompt (or, even better, use <a href="http://www.nikola-breznjak.com/blog/quick-tips/customize-console-2-on-windows-machine">Console 2</a>):<br></p>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>
              <tbody>
               <tr>
                <td></td>
                <td><code>node -v</code></td>
               </tr>
              </tbody>
             </table>
             <br>You should get an output similar to:
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>
              <tbody>
               <tr>
                <td></td>
                <td><code>v0.12.7</code></td>
               </tr>
              </tbody>
             </table>
             <br>To verify that you installed Node.js correctly on your Mac, run the above command in your Terminal (or use 
             <a href="https://www.iterm2.com/">iTerm</a>) and you should get a similar output as above.
             <p></p>
             <p><strong>Installing Git</strong></p>
             <p>In order to install Git, head <a href="http://git-scm.com/download" target="_blank">here</a>. Installation on Windows and Mac OS is as simple as it is for Node.js. You just download and run the appropriate installer and follow the familiar instructions. To verify that you installed Git correctly on your Windows/Mac machine, run the following command in your Command prompt/Terminal:<br></p>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>
              <tbody>
               <tr>
                <td></td>
                <td><code>git</code></td>
               </tr>
              </tbody>
             </table>
             <br>You should get an output similar to:
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>
              <tbody>
               <tr>
                <td></td>
                <td><code>usage: git [--version] [--help] [-C &lt;path&gt;] [-c name=value]<br>[--exec-path[=&lt;path&gt;]] [--html-path] [--man-path] [--info-path]<br>[-p|--paginate|--no-pager] [--no-replace-objects] [--bare]<br>[--git-dir=&lt;path&gt;] [--work-tree=&lt;path&gt;] [--namespace=&lt;name&gt;]<br>&lt;command&gt; [&lt;args&gt;]</code></td>
               </tr>
              </tbody>
             </table>
             <br>The most commonly used git commands are:
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>

List, create, or delete branches<br><code>checkout</code>&nbsp;&nbsp; Checkout a branch or paths to the working tree<br><code>clone</code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Clone a repository into a new directory<br><code>commit</code>&nbsp;&nbsp;&nbsp;&nbsp;              <h3>Starting a project with Ionic by using the existing templates</h3>
             <br>Now that you've made it this far I promise you're going to see some code! Ionic CLI allows us to start and initialize your project by using the 
             <code>ionic start</code> command. If you take a look at the 
             <a href="http://ionicframework.com/docs/cli/start.html">official documentation</a> for the start command, youll see something like the following definition: ionic start appname [template].
             <p></p>
             <p>If you just run <code><tt>ionic start appname</tt></code>, the Ionic CLI will make a bootstrap application with all the needed parts in the appname folder, with the blank template. There are three named template starters including <strong>blank</strong>, <strong>sidemenu</strong> and <strong>tabs</strong>. Additionally, you can use Github repo starters and Codepen URL starters. For a comprehensive list of starter apps check out <a href="http://www.gajotres.net/a-comprehensive-list-of-ionic-starter-apps/">this post</a>. In our example, well use the sidemenu template, so execute the following command from your Terminal/Command prompt:<br></p>

             <br>
             <br>
             <br>
             <br>
             <br>
             <br>
             <table>

                <td></td>

               </tr>

             <br>
             <ul>
              <br> 
              <li>cd into your project: <code>$ cd Ionic_1stTutorial</code></li>
              <br> 
              <li>Set up this project to use Sass: <code>ionic setup sass</code></li>
              <br> 
              <li>Develop in the browser with live reload: <code>ionic serve</code></li>
              <br> 
              <li>Add a platform (ios or Android): <code>ionic platform add iOS [Android]</code></li>
              <br> 
              <li>Note: iOS development requires OS X</li>
              <br> 
              <li>See the Android Platform Guide for full :&nbsp;<a href="https://cordova.apache.org/docs/en/edge/guide_platforms_android_index.md.html" target="_blank">Android installation instructions</a></li>
              <br> 
              <li>Build your app: <code>ionic build </code></li>
              <br> 
              <li>Simulate your app: <code>ionic emulate </code></li>
              <br> 
              <li>Run your app on a device: <code>ionic run </code></li>
              <br> 
              <li>Package an app using Ionic package service: <code>ionic package </code></li>
              <br>
             </ul>

             <a href="http://ionicframework.com/docs/cli/start.html" target="_blank">Ionic docs</a>. You can also add push notifications to your Ionic app with 
             <a href="https://apps.ionic.io/signup" target="_blank">Ionic Push</a> (alpha).
