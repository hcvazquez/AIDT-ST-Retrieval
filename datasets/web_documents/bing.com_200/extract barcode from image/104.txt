                   <div id="rgw54_59becb9e9eb22" class="post-edit-content comment-body topic-post-text rich-text-styled">
                     Hi Numan.
                    <br> What Mr. van de Sande's code that you are using probably does is to densely sample SIFT features on a tight image grid. While the extraction itself should be fine, you probably want to have a more compressed representation of your image. Just plugging all extracted SIFT features into an SVM is just too much stuff with too less information (and a lot of redundancy in it).
                    <br> One smaller representation is achieved by using a bag of visual words, where your SIFT features are clustered and the occurrence in each cluster is just counted (and normalized). The number of features is then equal to the number of clusters. Moreover, you can use images of different sizes as well, without changing the dimensionality of your feature space. As reference, see e.g. 
                    <a href="https://www.researchgate.net/deref/http%3A%2F%2Fpeople.ee.ethz.ch%2F~bleibe%2Fteaching%2Ftutorial-aaai08%2Faaai2008-tutorial-part4_indexing_vocabs.pdf" rel="noopener" target="_blank">http://people.ee.ethz.ch/~bleibe/teaching/tutorial-aaai08/aaai2008-tutorial-part4_indexing_vocabs.pdf</a>.
                    <br> You can then use SVM or something else, preferably with a kernel that can sensibly use the peculiarities of histograms, such as the histogram intersection kernel. Due to some nice decomposition properties, this kernel enables a fast classification even if a terrible lot of images are in your database (if you manage to find an implementation using this trick). See 
                    <a href="https://www.researchgate.net/deref/http%3A%2F%2Fmplab.ucsd.edu%2Fwp-content%2Fuploads%2Fcvpr2008%2Fconference%2Fdata%2Fpapers%2F290.pdf" rel="noopener" target="_blank">http://mplab.ucsd.edu/wp-content/uploads/cvpr2008/conference/data/papers/290.pdf</a> for more details.
                    <br> If speed is your main concern, SURF features as suggested by Yasir may indeed be interesting for you. The main idea of above bag of words approach is not influenced by the choice of your base features. 
                   </div>
