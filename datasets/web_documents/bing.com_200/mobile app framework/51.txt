         <h1 class="highlight">Tabris, because we love code and excellent UIs</h1> 
        </div> 
       </div>

     <div class="slideshow"> 
      <article class="slide row-vert-center">

       <div class="col6"> 
        <h2>Native Performance. Multiple Platforms.</h2> 
        <p>Tabris is the first mobile framework that allows you to develop apps completely in Java while still leveraging the power of native UIs.</p> 
       </div> 
      </article> 
      <article class="slide row-vert-center"> 
       <div class="col6 center"> 
        <img src="/img/tabris/rocket.png" alt="Developer Guide"> 
       </div> 
       <div class="col6"> 
        <h2>Ready to jump in?</h2> 
        <p>The <a href="/tabris/docs/2.0">Tabris Developer Guide</a> is the best starting point to get familiar with Tabris.</p> 
       </div> 
      </article> 
      <article class="slide row-vert-center"> 
       <div class="col6 center"> 
        <img src="/img/tabris/tools.png" alt="Use your JavaEE skills and the tools of your choice"> 
       </div> 
       <div class="col6"> 
        <h2>Your tools, your way.</h2> 
        <p> Tabris apps can be developed using your favorite development environments such as: Eclipse, NetBeans or Idea, without requiring any extra plug-ins. This is possible because you can use standard JavaEE to develop apps. </p> 
       </div> 
      </article> 
      <article class="slide row-vert-center"> 
       <div class="col6 center"> 
        <img src="/img/tabris/stores.png" alt="Distribute to Apple App Store and Google Play"> 
       </div> 
       <div class="col6"> 
        <h2>Ready for Distribution.</h2> 
        <p> Tabris based applications can be found in both the Apple App Store and Google Play Store. </p> 
       </div> 
      </article> 
      <article class="slide row-vert-center"> 
       <div class="col6 center"> 
        <img src="/img/tabris/labs.png" alt="Free for open source and educational projects"> 
       </div> 
       <div class="col6"> 
        <h2>Free for...</h2> 
        <p> Tabris is free of charge if you use it within an Open Source or educational project. Please <a href="http://eclipsesource.com/en/about/contact-us/">contact us directly</a> and describe the project you want to use Tabris for. </p> 
       </div> 
      </article> 
      <article class="slide row-vert-center">

     <article id="how-does-it-work"> 
      <header class="row"> 
       <div class="col12"> 
        <h2 class="highlight">How does it work?</h2> 
       </div> 
      </header> 
      <section class="row"> 
       <div class="col5 center"> 
        <img src="/img/tabris/server.png" alt="Server"> 
       </div> 
       <div class="col7"> 
        <p> Tabris is a Java framework written by Java developers for Java developers to build mobile apps. It uses standard JavaEE technology to create an application on a server. Native clients act as a player for the applications's UI. They access the application by calling a URL that provides a <a href="http://json.org/">JSON</a> representation of the UI. The clients will use this JSON message to render the UI using native components. Native components are Cocoa Touch widgets in iOS, Java based widgets in Android and HTML5 in a browser. </p> 
        <div class="teaser" style="display: inline-block; margin:1.0em auto 2.5em 0; padding: 1.429em; text-align: center; font-size: small; border-radius: 0.357em; background: url('http://eclipsesource.com/fileadmin/templates/css/img/shaded-bg.png');"> 
         <p>Looking for tabris.js?<br><a href="https://tabrisjs.com" title="tabris.js website">https://tabrisjs.com</a> </p>
        </div> 
       </div> 
      </section> 
     </article> 
     <article id="secure-apps"> 
      <header class="row"> 
       <div class="col12"> 
        <h2 class="highlight">Secure Mobile Apps for Business.</h2> 
       </div> 
      </header> 
      <section class="row"> 
       <div class="col5 center"> 
        <img src="/img/tabris/tabris-banner-small.png" alt="Display"> 
       </div> 
       <div class="col7"> 
        <p> Mobile devices impose a security risk for sensitive data as they are easily lost or stolen. Storing data physically on the device is a risk. With Tabris there is no data stored on the device and the data sent to the client is encrypted using approved technologies like HTTPS. Your data remains safe on your servers. </p> 
        <p> Tabris's <a href="http://www.eclipse.org/rap/" target="_blank">Open Source Server Parts</a> use standard servlet technology and runs on any JEE servlet container. It integrates well with OSGi and other Eclipse technologies like <a href="http://eclipse.org/equinox/" target="_blank">Equinox</a>, <a href="http://eclipse.org/modeling/emf/" target="_blank">EMF</a>, <a href="http://eclipse.org/birt/phoenix/" target="_blank">Birt</a>, <a href="http://eclipse.org/cdo/" target="_blank">CDO</a>, <a href="http://eclipse.org/eclipselink/" target="_blank">EclipseLink</a>, <a href="http://eclipse.org/virgo/" target="_blank">Virgo</a> and others. If you have any business logic written in Java it is easy to integrate it into your mobile applications. </p> 
        <p> <a class="action-btn" href="/tabris/demos/"> Watch the Demos </a> </p> 
       </div> 
      </section> 
     </article> 
     <article id="powerful-ui"> 
      <header class="row"> 
       <div class="col12"> 
        <h2 class="highlight">Powerful User Interfaces.</h2> 
       </div> 
      </header> 
      <section class="row"> 
       <div class="col5 center"> 
        <img src="/img/tabris/devices.png" alt="Powerful UIs"> 
       </div> 
       <div class="col7"> 
        <p> Tabris provides a powerful widget toolkit that renders native iOS and Android widgets. It is based on a proven technology stack using <a href="http://wiki.eclipse.org/SWT" target="_blank">SWT</a>, <a href="http://wiki.eclipse.org/index.php/JFace" target="_blank">JFace</a> and <a href="http://en.wikipedia.org/wiki/OSGi" target="_blank">OSGi</a>. You can write your application entirely in Java, re-use existing code and benefit from <a href="http://www.eclipse.org/downloads/packages/eclipse-rcp-and-rap-developers/keplerr" target="_blank">first-class IDE tools</a> without the need for cross-compiling. </p> 
        <p> Using the <a href="http://wiki.eclipse.org/JFace" target="_blank">JFace</a> APIs makes accessing complex data models incredibly simple. <a href="http://wiki.eclipse.org/JFace_Data_Binding" target="_blank">JFace Data Binding</a> allows you to attach data to your widgets without tight coupling. Virtual Trees and Tables provide you with full control to push only the data that is needed to the client - just in time. </p> 
        <p> <a class="action-btn" href="/tabris/docs/"> Read the Documentation </a> </p> 
       </div> 
      </section>
