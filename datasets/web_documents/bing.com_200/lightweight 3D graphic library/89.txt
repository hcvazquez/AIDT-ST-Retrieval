    <p class="line862">This page lists libraries that may be useful when <a href="/moin/GameProgramming">GameProgramming</a> in Python. <span class="anchor" id="line-2"></span><span class="anchor" id="line-3"></span><span class="anchor" id="line-4"></span></p>

    <h2 id="PyGame_.2F_related">PyGame / related</h2> 
    <span class="anchor" id="line-5"></span>
    <span class="anchor" id="line-6"></span>
    <span class="anchor" id="line-7"></span>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-8"></span><p class="line862">Pygame (<a class="http" href="http://www.pygame.org/">http://www.pygame.org/</a>) is a set of Python modules designed for writing games. It is written on top of the excellent SDL library. This allows you to create fully featured games and multimedia programs in the python language. It is the most popular, and portable game library for python, with over 1000 free and open source projects that use pygame to look at. <span class="anchor" id="line-9"></span><span class="anchor" id="line-10"></span></p><p class="line862">The free books <a class="http" href="http://programarcadegames.com/">"Program Arcade Games with Python and Pygame"</a>, <span class="anchor" id="line-11"></span><a class="http" href="http://inventwithpython.com/pygame">"Making Games with Python &amp; Pygame"</a> <span class="anchor" id="line-12"></span>cover the basics of the Pygame library and offers the source code for several popular video game clones. <span class="anchor" id="line-13"></span><span class="anchor" id="line-14"></span><span class="anchor" id="line-15"></span><span class="anchor" id="line-16"></span></p></li>
    </ul>
    <p class="line867"><a class="http" href="http://inventwithpython.com/pyganim">http://inventwithpython.com/pyganim</a> <span class="anchor" id="line-17"></span><span class="anchor" id="line-18"></span><span class="anchor" id="line-19"></span></p>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-20"></span><p class="line891"><em>Pyganim</em> is a simple module for handling sprite animation. The <a class="http" href="http://inventwithpython.com/pyganim/tutorial.html">tutorial</a> covers how to use the module, and the site also provides a few demo programs. <span class="anchor" id="line-21"></span><span class="anchor" id="line-22"></span><span class="anchor" id="line-23"></span><span class="anchor" id="line-24"></span></p></li>
    </ul>
    <p class="line867"><a class="http" href="http://www.cosc.canterbury.ac.nz/greg.ewing/python/Albow/">http://www.cosc.canterbury.ac.nz/greg.ewing/python/Albow/</a> <span class="anchor" id="line-25"></span><span class="anchor" id="line-26"></span><span class="anchor" id="line-27"></span></p>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-28"></span><p class="line891"><em>Albow</em> (A Little Bit of Widgetry) is a graphical user interface toolkit for use in <a href="/moin/PyGame">PyGame</a>-based games. Widgets include labels, buttons (text or image), check boxes, radio buttons, text fields, palettes, tables and file load/save dialogs. Layout facilities include rows, columns and grids. A theme system allows fonts and colours to be customised easily. Also provides some facilities for locating and managing resources and playing and controlling music. Can be used in conjunction with OpenGL to incorporate 2D widgets into a 3D game. <span class="anchor" id="line-29"></span><span class="anchor" id="line-30"></span></p><p class="line862">Also available from the same site is <em>Humerus</em>, a game skeleton based on Albow. Humerus provides a framework for managing and loading levels, starting and resuming play, saving and restoring game state, and implementing an in-game level editor. A customisable main menu screen ties all these activities together. <span class="anchor" id="line-31"></span><span class="anchor" id="line-32"></span><span class="anchor" id="line-33"></span><span class="anchor" id="line-34"></span></p></li>
    </ul>
    <p class="line867"><a class="http" href="http://ocemp.sourceforge.net/gui.html">http://ocemp.sourceforge.net/gui.html</a> <span class="anchor" id="line-35"></span><span class="anchor" id="line-36"></span><span class="anchor" id="line-37"></span></p>
    <ul>
     <li style="list-style-type:none">OcempGUI is an abbreviation for Ocean Empire GUI. and denotes a toolkit, which contains a set of various user interface elements based on and for pygame. <span class="anchor" id="line-38"></span><span class="anchor" id="line-39"></span><span class="anchor" id="line-40"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://www.imitationpickles.org/pgu/">http://www.imitationpickles.org/pgu/</a> <span class="anchor" id="line-41"></span><span class="anchor" id="line-42"></span><span class="anchor" id="line-43"></span></p>
    <ul>
     <li style="list-style-type:none">pgu includes several scripts and libraries. The scripts are a tile editor and a level editor. The libraries include a state engine, a full featured gui, document layout, html rendering, text rendering, sprite and tile engine, and a timer. <span class="anchor" id="line-44"></span><span class="anchor" id="line-45"></span><span class="anchor" id="line-46"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://code.google.com/p/gameobjects/">http://code.google.com/p/gameobjects/</a> <span class="anchor" id="line-47"></span><span class="anchor" id="line-48"></span><span class="anchor" id="line-49"></span></p>
    <ul>
     <li style="list-style-type:none">Game Objects is a collection of classes to assist in the creation of games, and other real-time applications. (note, no changes for over a year 2008/07/21) <span class="anchor" id="line-50"></span><span class="anchor" id="line-51"></span><span class="anchor" id="line-52"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://aharrisbooks.net/pythonGame/">http://aharrisbooks.net/pythonGame/</a> <span class="anchor" id="line-53"></span><span class="anchor" id="line-54"></span><span class="anchor" id="line-55"></span></p>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-56"></span><p class="line862">Game Engine is a high-level wrapper on Pygame that greatly simplifies game development in Python / Pygame. It is described completely in the book "Game Programming - the L Line" but it is available for free even if you do not purchase the book. (Look for appendix B on the website to download <a class="nonexistent" href="/moin/GameEngine">GameEngine</a> and the documentation) <span class="anchor" id="line-57"></span><span class="anchor" id="line-58"></span><span class="anchor" id="line-59"></span><span class="anchor" id="line-60"></span></p></li>
    </ul>
    <p class="line867"><a class="http" href="http://www.renpy.org/wiki/renpy/Home_Page">http://www.renpy.org/wiki/renpy/Home_Page</a> <span class="anchor" id="line-61"></span><span class="anchor" id="line-62"></span><span class="anchor" id="line-63"></span></p>
    <ul>
     <li style="list-style-type:none">Ren'Py is an engine for visual novels and simulation games. It is written in Python and Pygame, and is extensible with Python. <span class="anchor" id="line-64"></span><span class="anchor" id="line-65"></span><span class="anchor" id="line-66"></span></li>
    </ul>
    <p class="line867"> </p>
    <h2 id="Pyglet_.2F_related">Pyglet / related</h2> 
    <span class="anchor" id="line-67"></span>
    <span class="anchor" id="line-68"></span>
    <span class="anchor" id="line-69"></span>
    <p class="line867"><a class="http" href="http://www.pyglet.org/">http://www.pyglet.org/</a> <span class="anchor" id="line-70"></span><span class="anchor" id="line-71"></span><span class="anchor" id="line-72"></span></p>
    <ul>
     <li style="list-style-type:none">pyglet is a cross-platform windowing and multimedia library for Python. <span class="anchor" id="line-73"></span><p class="line862">pyglet provides an object-oriented programming interface for developing games and other visually-rich applications for Windows, Mac OS X and Linux using for rendering <strong>OpenGL</strong>. Some of the features of pyglet are: <span class="anchor" id="line-74"></span><span class="anchor" id="line-75"></span></p>
      <ul>
       <li style="list-style-type:none">For most application and game requirements, pyglet needs nothing else besides Python, simplifying distribution and installation. Requires ctypes, and Opengl. Uses PIL, and AVBin for most format loading. Take advantage of multiple windows and multi-monitor desktops. pyglet allows you to use as many windows as you need, and is fully aware of multi-monitor setups for use with fullscreen games. Load images, sound, music and video in almost any format. pyglet can optionally use AVbin to play back audio formats such as MP3, OGG/Vorbis and WMA, and video formats such as DivX, MPEG-2, H.264, WMV and Xvid. <span class="anchor" id="line-76"></span><span class="anchor" id="line-77"></span><span class="anchor" id="line-78"></span></li>
      </ul></li>
    </ul>
    <p class="line867"><a class="http" href="http://pythonhosted.org/arcade/">http://pythonhosted.org/arcade/</a> <span class="anchor" id="line-79"></span><span class="anchor" id="line-80"></span></p>
    <ul>
     <li style="list-style-type:none">Arcade is an easy-to-learn Python library for creating 2D video games. It is <span class="anchor" id="line-81"></span>ideal for people learning to program, or developers that want to code a 2D game <span class="anchor" id="line-82"></span>without learning a complex framework. It is built on top of Pyglet and OpenGL. <span class="anchor" id="line-83"></span><span class="anchor" id="line-84"></span><span class="anchor" id="line-85"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://cocos2d.org/">http://cocos2d.org/</a> <span class="anchor" id="line-86"></span><span class="anchor" id="line-87"></span><span class="anchor" id="line-88"></span></p>
    <ul>
     <li style="list-style-type:none">cocos2d is a framework for building 2D games, demos, and other graphical/interactive applications. It is based on pyglet, and written in pure python, so it is cross-platform (but requires a decent/fast OpenGL platform). It provides scene flow control, sprites and actions, special effects and transitions. <span class="anchor" id="line-89"></span><span class="anchor" id="line-90"></span><span class="anchor" id="line-91"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://tdelusion.sourceforge.net">http://tdelusion.sourceforge.net</a> <span class="anchor" id="line-92"></span><span class="anchor" id="line-93"></span><span class="anchor" id="line-94"></span></p>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-95"></span><p class="line862">tdgl is a package of python modules built on top of pyglet, for constructing and animating scenes of 3D objects, moving the camera, controlling the lighting and so on. It was developed originally with a view to making an RPG, which has not yet happened, but it has been used in a number of <a href="/moin/PyWeek">PyWeek</a> entries. <span class="anchor" id="line-96"></span><span class="anchor" id="line-97"></span><span class="anchor" id="line-98"></span><span class="anchor" id="line-99"></span></p></li>
    </ul>
    <p class="line867"><a class="https" href="https://bitbucket.org/tartley/gloopy">https://bitbucket.org/tartley/gloopy</a> <span class="anchor" id="line-100"></span><span class="anchor" id="line-101"></span><span class="anchor" id="line-102"></span></p>
    <ul>
     <li style="list-style-type:none">Gloopy is a simple OpenGL render loop, in pure Python, based on pyglet. It also uses PyOpenGL bindings. It is at an early stage of development and is hardly documented, although there are a couple of example demo scripts. It provides utility functions and classes for populating a 3D world with items which are rendered as VBOs. There are utilities for moving a camera around in the world, and also facilities to programmatically create and manipulate simple polyhedra, and then convert the resulting shapes into VBOs for rendering. <span class="anchor" id="line-103"></span><span class="anchor" id="line-104"></span><span class="anchor" id="line-105"></span></li>
    </ul>
    <p class="line867"> </p>
    <h2 id="PyOpenGL_.2F_related">PyOpenGL / related</h2> 
    <span class="anchor" id="line-106"></span>
    <span class="anchor" id="line-107"></span>
    <span class="anchor" id="line-108"></span>
    <p class="line862">Generally PyOpenGL games will also make use of <a href="/moin/PyGame">PyGame</a>. <span class="anchor" id="line-109"></span><span class="anchor" id="line-110"></span><span class="anchor" id="line-111"></span></p>
    <p class="line867"><a class="http" href="http://pyopengl.sourceforge.net/">http://pyopengl.sourceforge.net/</a> <span class="anchor" id="line-112"></span><span class="anchor" id="line-113"></span><span class="anchor" id="line-114"></span></p>
    <ul>
     <li style="list-style-type:none">PyOpenGL includes support for OpenGL v1.1, GLU, GLUT v3.7, GLE 3, WGL 4, and Togl (Tk OpenGL widget) 1.6. It also includes support for dozens of extensions (where supported in the underlying implementation). OpenGL is an environment for developing high-performance 2D and 3D applications. <span class="anchor" id="line-115"></span><span class="anchor" id="line-116"></span><span class="anchor" id="line-117"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://glewpy.sourceforge.net/">http://glewpy.sourceforge.net/</a> <span class="anchor" id="line-118"></span><span class="anchor" id="line-119"></span><span class="anchor" id="line-120"></span></p>
    <ul>
     <li style="list-style-type:none">GLEWpy aims to bring advanced OpenGL extensions to Python. This will allow the Python OpenGL developer to use features such as fragment and vertex shaders and image processing on the GPU. It serves as a compliment to PyOpenGL and toolkits such as GLUT and SDL (pygame). <span class="anchor" id="line-121"></span><span class="anchor" id="line-122"></span><span class="anchor" id="line-123"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://slut.sourceforge.net/">http://slut.sourceforge.net/</a> <span class="anchor" id="line-124"></span><span class="anchor" id="line-125"></span><span class="anchor" id="line-126"></span></p>
    <ul>
     <li style="list-style-type:none"><span class="anchor" id="line-127"></span><p class="line862">Slut is a programming framework for generative, synthetic, interactive, network-enabled graphics. It is a layer above <a href="/moin/PyGame">PyGame</a>, PyOpenGL and Twisted. <span class="anchor" id="line-128"></span><span class="anchor" id="line-129"></span><span class="anchor" id="line-130"></span><span class="anchor" id="line-131"></span></p></li>
    </ul>
    <p class="line867"><a class="http" href="http://www.ixi-software.net/content/body_software_mirra.html">http://www.ixi-software.net/content/body_software_mirra.html</a> <span class="anchor" id="line-132"></span><span class="anchor" id="line-133"></span><span class="anchor" id="line-134"></span></p>
    <ul>
     <li style="list-style-type:none">Mirra is a 2D openGL python framework. <span class="anchor" id="line-135"></span><span class="anchor" id="line-136"></span><span class="anchor" id="line-137"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://pypi.python.org/pypi/QGL">http://pypi.python.org/pypi/QGL</a> <span class="anchor" id="line-138"></span><span class="anchor" id="line-139"></span><span class="anchor" id="line-140"></span></p>
    <ul>
     <li style="list-style-type:none">QGL is an intentionaly minimal scenegraph for rendering textured quads and text strings to an OpenGL display. It requires pygame, and is designed for building 2D games with hardware accelerated effects. It includes a pure Python and a Pyrex Renderer. <span class="anchor" id="line-141"></span><span class="anchor" id="line-142"></span><span class="anchor" id="line-143"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://rene.f0o.com/rdpyg/">http://rene.f0o.com/rdpyg/</a> <span class="anchor" id="line-144"></span><span class="anchor" id="line-145"></span><span class="anchor" id="line-146"></span></p>
    <ul>
     <li style="list-style-type:none">A collection of code for games using python. Mostly code for use with games made with pygame, and/or pyopengl. Although some of it can be used outside. <span class="anchor" id="line-147"></span><span class="anchor" id="line-148"></span><span class="anchor" id="line-149"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://pduel.sourceforge.net/spyre/">http://pduel.sourceforge.net/spyre/</a> <span class="anchor" id="line-150"></span><span class="anchor" id="line-151"></span><span class="anchor" id="line-152"></span></p>
    <ul>
     <li style="list-style-type:none">SPyRE is a lightweight OpenGL graphics engine, written entirely in Python. It includes a variety of cameras, several interfaces for user interaction, lighting controls and fog. <span class="anchor" id="line-153"></span><span class="anchor" id="line-154"></span><span class="anchor" id="line-155"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://opioid-interactive.com/~shang/projects/pygext/">http://opioid-interactive.com/~shang/projects/pygext/</a> <span class="anchor" id="line-156"></span><span class="anchor" id="line-157"></span><span class="anchor" id="line-158"></span></p>
    <ul>
     <li style="list-style-type:none">Pygame Extended (or pygext) contains additions to pygame.draw (e.g. rectangles with round corners), an opengl accelerated 2D vector graphics library and a full blown, event-based sprite/scene engine. <span class="anchor" id="line-159"></span><span class="anchor" id="line-160"></span><span class="anchor" id="line-161"></span></li>
    </ul>
    <p class="line867"><a class="http" href="http://arcticpaint.com/projects/rabbyt/">http://arcticpaint.com/projects/rabbyt/</a> <span class="anchor" id="line-162"></span><span class="anchor" id="line-163"></span><span class="anchor" id="line-164"></span></p>
    <ul>
     <li style="list-style-type:none">Rabbyt is a fast 2d sprite library for Python. Commonly used in combination with OpenGL context of the pyglet framework listed above. <span class="anchor" id="line-165"></span><span class="anchor" id="line-166"></span><span class="anchor" id="line-167"></span></li>
    </ul>

    <h2 id="Other">Other</h2> 
    <span class="anchor" id="line-168"></span>
