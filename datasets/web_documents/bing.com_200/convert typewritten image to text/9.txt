             <div class="post-text" itemprop="text"> 
              <p>Is there a command-line program (for Linux) that can convert an image to ascii art and then save that output as an image e.g. jpg or png?</p> 
              <p>To illustrate what I'm in need of, I want to convert the image below into ascii art <img src="https://i.stack.imgur.com/7CuSb.jpg" alt="enter image description here"></p> 
              <p>The output that I desire from a program should be similar to this one, which was created by taking a screenshot of the output of an image-to-ascii converter (the output was pure text)</p> 
              <p><img src="https://i.stack.imgur.com/NDRUZ.png" alt="enter image description here"></p> 
              <p>I'm aware that GIMP has the ability to save an image as ascii art and <a href="http://aa-project.sourceforge.net/aview/" rel="noreferrer">aview</a> can also convert an image to ascii. However, the former cannot be done via the command line and the latter only lets you view the output, not save it (I don't want to resort to taking a screenshot of the output).</p> 
              <p>I'm also aware that <a href="http://caca.zoy.org/wiki/libcaca" rel="noreferrer">libcaca</a> can save its output to an image, but it only outputs colourful ascii, like below.</p> 
              <p><img src="https://i.stack.imgur.com/xFxLm.png" alt="enter image description here"></p> 
              <p>Made by converting the source image to greyscale and then with the following command:</p> 
              <pre><code>img2txt -W 70 -f tga input.jpg &gt; output.tga
</code></pre> 
              <p>The reason for wanting a command-line program is to batch process a folder full of images.</p>

             <div class="post-text" itemprop="text"> 
              <p>In case you seem to be running Linux you may stitch the applications easily together to get the desired result. This can then also be incorporated in your batch processing script.</p> 
              <p>The example below makes use of <a href="http://csl.sublevel3.org/jp2a/">jp2a</a>, a powerfull JPG to ASCII-art converter with many options to achieve desired results (consult the manpage). We then further process the resulting ASCII text file to an image using <strong>convert</strong> from the <a href="http://www.imagemagick.org/script/convert.php">ImageMagick suite</a>. Again there are many options to fine tune the result. Both applications are likely to be available from your distribution's repositories.</p> 
              <p>Below sample script will produce an ASCII-art image (format depending on the extension we give, <code>.jpg</code> is for JPEG) from an input JPEG-image. The source is rendered to 160 character width at 6 pt. This will then just fit on the default page size as pre-defined from convert.</p> 
              <pre><code>#! /bin/bash
#Usage ./j2asc infile.jpg outfile.jpg

echo "converting $1 to $2"
jp2a --width=160 -i  $1 --output=tmp.txt
convert -font Courier -pointsize 6 tmp.txt $2
</code></pre>

             <div class="post-text" itemprop="text"> 
              <p>As long as I am aware that program doesn't have a command line interface, but does convert images to text and then to images. Source is available, so you can use it to write your own tool.</p> 
              <p>You are right, aalib doesn't output images, but it does output text. As well as many other programs that work on command line.</p> 
              <p>If you're using Linux, there is a command line program called asciitopgm that will convert ascii text to a bitmap. It's part of the netpbm package.</p> 
              <p>So you can use a combination of tools:</p> 
              <ol> 
               <li>aalib to draw an image using text</li> 
               <li>asciitopgm to create a pgm bitmap with the text file.</li> 
               <li>(optional) Use convert (From Imagemagick package) to convert the pgm to any other raster format you want.</li> 
              </ol> 
              <p>I think you can do what you wanted to this way.</p>
