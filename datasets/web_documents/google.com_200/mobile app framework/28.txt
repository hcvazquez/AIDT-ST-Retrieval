     <p>Are you using the Visual Studio 2017 RC release? If so, check out the <a href="../vs-taco-2017-first-app/">updated version of this article</a>.</p> 
     <p>Use Visual Studio Tools for Apache Cordova to build mobile apps using web technologies (HTML, CSS, JavaScript) that run on Android, iOS, and Windows devices. Since Cordova apps are built using web application code, you'll be able to share the majority of your code across device platforms; <a href="https://www.visualstudio.com/explore/cordova-vs">learn more</a>.</p> 
     <p>In this guide, we'll show how to build, test and debug a Cordova application using the Visual Studio Tools for Apache Cordova. Along the way, you'll build a simple mobile weather app that looks like this:</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/weather-app-overview.png" alt="Weather app"> </p> 
     <p>The application displays current weather conditions based on the current device location plus enables users to search for weather conditions using a US postal (zip) code. You can find the application's source code <a href="https://github.com/Microsoft/cordova-samples/tree/master/weather-app">here</a>.</p> 
     <blockquote> 
      <p><strong>Note:</strong> Before we begin, make sure you have completed the <a href="../install-vs-tools-apache-cordova/">Visual Studio Tools for Apache Cordova Installation</a>.</p>

     <h3><a id="the-folders-of-your-project" name="the-folders-of-your-project" href="#the-folders-of-your-project"></a>The folders of your project</h3> 
     <h4><a id="www" name="www" href="#www"></a>www</h4> 
     <p>The project's <code>www</code> folder contains the web application content files for the Cordova application, so most of your development will occur inside this folder. Here you'll find the web application content that is packaged into a native mobile application by the Cordova SDK. The HTML, JavaScript, and CSS files in this folder will render in the Cordova app the same way they will in a mobile browser, so be sure to optimize the content for the smaller and more limited smartphones and tablets. </p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/www.png" alt="www folder"> </p> 
     <p>The folder's <code>index.html</code> file is the main entry point for the web application, it's loaded by default when the Cordova application launches. The folder contains several subfolders that follow a typical web application folder structure:</p> 
     <ul> 
      <li><code>css</code> - Stores the web application's Cascading Style Sheet (CSS) files. The <code>index.css</code> shown in the folder is the standard .css file included with the default Cordova application template. </li> 
      <li><code>images</code> - Stores any image files used by the web application. The <code>cordova.png</code> file shown in the folder is the Cordova logo that displays in the center of the application screen in the default Cordova application template.</li> 
      <li><code>scripts</code> - Stores JavaScript files used by the web application. The project's <code>index.js</code> file contains the bootstrap code that initializes the Cordova application; it registers callback functions for the Cordova <code>deviceReady</code> event. The <code>platformOverrides.js</code> file works in conjunction with similar files located in the <code>merges</code> folder to enable developers to deliver different code per target platform. See <a href="#merges">merges Folder</a> for additional information.</li> 
     </ul> 
     <p>Cordova developers do most of their work in these folders, creating and updating web application content designed for mobile devices. The application content contained herein is packaged into a native mobile application by Visual Studio for deployment to mobile devices using the Cordova and device platform SDKs.</p> 
     <h4><a id="merges" name="merges" href="#merges"></a>merges</h4> 
     <p>Apache Cordova uses the <code>merges</code> folder to provide developers with the ability to deliver different content based on the target mobile platform. As you can see from the figure, there's a subfolder for each target mobile platform (Android, iOS and Windows). Any content in these folders will be copied to the native application project during the Cordova pre-build process (prepare), adding to or replacing content in the web application folder as needed.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/merges.png" alt="merges folder"> </p> 
     <p>For example, any content in the <code>merges\android</code> folder is copied to the Android project's web application folder after the base web application is copied. Content in the <code>merges\ios</code> folder is copied into the iOS project's web application folder. Finally, content in the <code>merges\windows</code> folder is copied to the Windows project's web application folder after the base web application is copied.</p> 
     <p>To help deliver a consistent web application experience in Cordova apps, the default project created by Visual Studio Tools for Apache Cordova includes <code>platformOverrides.js</code> files for Android (in <code>merges\android\scripts\platformOverrides.js</code>) and Windows (in <code>merges\windows\scripts\platformOverrides.js</code>) along with a platform-specific JavaScript library. On Android, it adds a polyfill for <code>bind()</code> on older Android devices, and on Windows it adds Microsoft's safeHTML polyfill library. The <code>merges\ios</code> folder doesn't include a <code>platformOverrides.js</code> file, so there's no platform-specific override, iOS applications get the generic version of the file located in <code>www\scripts\platformOverriders.js</code>.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/platform-overrides.png" alt="root files"> </p> 
     <p>For additional information on how merges works, see <a href="https://cordova.apache.org/docs/en/latest/guide/cli/#using-merges-to-customize-each-platform">Using merges to Customize Each Platform</a> in the Cordova documentation.</p> 
     <p>Use this same approach to merge CSS files, images and any other files in your application.</p> 
     <h4><a id="res" name="res" href="#res"></a>res</h4> 
     <p>Apache Cordova uses the contents of the <code>res</code> folder to store non-web application resources used by a native mobile application such as <a href="https://cordova.apache.org/docs/en/latest/config_ref/images.html">application icons</a>, <a href="https://cordova.apache.org/docs/en/latest/reference/cordova-plugin-splashscreen/index.html">splash screen</a> images, signing certificates, etc.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/res-folder.png" alt="res folder"> </p> 
     <h2><a id="run" name="run" href="#run"></a>Run your app</h2> 
     <ol> 
      <li><p>On the Standard toolbar, choose the <strong>Ripple  Nexus (Galaxy)</strong> button.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/ripple-start.png" alt="Ripple Emulator"> </p> <p>Visual Studio will build the application, then open a browser window running the application in the Apache Ripple simulator. New projects use the default Cordova template, so, since we've not added any code to the application, it will simply display the Cordova logo and a notification that the device is ready.</p> 
       <blockquote> 
        <p><strong>NOTE:</strong> There will already be some errors on the JavaScript console, such as failure to load favicon.ico, ripple.js and ajax-loader.gif. But these are not neccessary to continue developing in Ripple.</p>

     <p>This is a good time to quickly look at the <code>index.js</code> file. This file loads when the user runs the app. Why? Because the <code>index.html</code> page contains this reference to it:</p>

};

      + zipcode + ',us&amp;appid=' + OpenWeatherAppKey + '&amp;units=imperial';

      showWeatherData(results);

      }).fail(function (jqXHR) {
          $('#error-msg').show();
          $('#error-msg').text("Error retrieving data. " + jqXHR.statusText);
      });

      return false;

function showWeatherData(results) {

  if (results.weather.length) {

      $('#error-msg').hide();
      $('#weather-data').show();

      $('#title').text(results.name);
      $('#temperature').text(results.main.temp);
      $('#wind').text(results.wind.speed);

      var sunsetDate = new Date(results.sys.sunset * 1000);
      $('#sunset').text(sunsetDate.toLocaleTimeString());

  } else {
      $('#weather-data').hide();
      $('#error-msg').show();
      $('#error-msg').text("Error retrieving data. ");
  }
}

     <h2><a id="capability" name="capability" href="#capability"></a>Access a device capability</h2> 
     <p>Most likely you'll want your app to do more than just show web pages on a mobile device. You might want your app to store files, receive push notifications, or capture images by using the device's camera. This is where a <strong>plugin</strong> becomes handy. Apache Cordova uses plugins to provide access to native device capabilities that arent available to simple web apps.</p> 
     <p>You can find some of the more commonly used plugins in the <strong>Core</strong> page of the app's <a href="#settings">configuration designer</a>.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/core-plugins.png" alt="Configuration Designer"> </p> 
     <p>Use the <strong>Custom</strong> page to add plugins that are not one of the <a href="../use-cordova-plugins/">core plugins</a>. These could be standard Cordova plugins available on npm (Cordova 5.x or greater), or plugins that you or a colleague create for some specific purpose and then save to your local drive or to a GitHub repository. <a href="../manage-plugins/#Custom">This document</a> explains how to add a custom plugin.</p> 
     <p>If you need to use a specific version of a plugin, you can add a plugin by editing the <code>config.xml</code> file directly. You can find guidance about how to do that <a href="../manage-plugins/#AddOther">here</a>.</p> 
     <p>Let's add a plugin that gives us access to the device's geolocation system. That way, we can get the weather of a user's current location.</p> 
     <ol> 
      <li><p>In solution Explorer, double-click on the <code>config.xml</code> file to open the <a href="#settings">configuration designer</a>, and then choose the <strong>Plugins</strong> tab.</p></li> 
      <li><p>Choose the <strong>Geolocation</strong> plugin, and then choose the <strong>Add</strong> button.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/add-geolocation.png" alt="Geolocation plugin"> </p> <p>This adds the plugin to your project but you still have to know how to consume it in your app. To do that, open the <code>README.md</code> file. You'll find it in the <code>plugins\plugin-cordova-geolocation</code> folder as shown in the following image.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/plugin-readme.png" alt="Readme File"> </p> <p>Let's add some code that consumes this plugin.</p></li> 
      <li><p>In the <code>www\scripts\weather.js</code> file add the following code right after the <code>showWeatherData</code> function</p> <pre class="prettyprint copy-to-clipboard lang-javascript"> function getWeatherWithGeoLocation() {
   navigator.geolocation.getCurrentPosition(onGetLocationSuccess, onGetLocationError, { enableHighAccuracy: true });
  $('#error-msg').show();
  $('#error-msg').text('Determining your current location ...');
  $('#get-weather-btn').prop('disabled', true);
}

function onGetLocationSuccess(position) {
  var latitude = position.coords.latitude;
  var longitude = position.coords.longitude;

  var queryString = 'http://api.openweathermap.org/data/2.5/weather?lat='
    + latitude + '&amp;lon=' + longitude + '&amp;appid=' + OpenWeatherAppKey + '&amp;units=imperial';
  $('#get-weather-btn').prop('disabled', false);
  $.getJSON(queryString, function (results) {
    showWeatherData(results);
  }).fail(function (jqXHR) {
     $('#error-msg').show();
     $('#error-msg').text("Error retrieving data. " + jqXHR.statusText);
  });
}

function onGetLocationError(error) {
  $('#error-msg').text('Error getting location');
  $('#get-weather-btn').prop('disabled', false);
}  

</pre> <p>This code uses the device's geolocation capability to get the latitude and longitude of the device's location. It then gets the weather for that location.</p></li> 
      <li><p>Open the project's <code>www\scripts\index.js</code> file, and add the following code to the <code>onDeviceReady</code> function.</p> <pre class="prettyprint copy-to-clipboard lang-javascript">
  getWeatherWithGeoLocation();

</pre> <p>When you're done, the <code>onDeviceReady</code> function should look like this:</p> <pre class="prettyprint copy-to-clipboard lang-javascript">
 function onDeviceReady() {        
   $('#get-weather-btn').click(getWeatherWithZipCode);
   getWeatherWithGeoLocation();
};
</pre></li> 
      <li><p>Run the app. When the app starts, the label <em>Determining your current location ...</em> appears. After a brief period of time, the weather data for your location appears in the app.</p> 
       <blockquote> 
        <p><strong>Note</strong>: If you use the Apache Ripple emulator, youll have to configure it with your location.</p> 
        <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/change-ripple-location.png" alt="Change Apache Ripple emulator location"> </p> 
       </blockquote></li> 
     </ol> 
     <h2><a id="tailor" name="tailor" href="#tailor"></a>Tailor the app to a platform</h2> 
     <p>The code that you used to get the device's location works pretty well across all platforms, but what if it didn't? What if you had to write extra code to get the location of an iOS device?</p> 
     <p>This is where the <code>merges</code> folder becomes useful. We touched on that folder earlier in this article. Now let's try a few basic things.</p> 
     <h3><a id="tailor-the-behavior-of-your-app" name="tailor-the-behavior-of-your-app" href="#tailor-the-behavior-of-your-app"></a>Tailor the behavior of your app</h3> 
     <ol> 
      <li><p>In <strong>Solution Explorer</strong>, expand the project's <code>www</code> folder, and then the <code>scripts</code> folder.</p></li> 
      <li><p>Right-click the <code>weather.js</code> file, and choose <strong>Copy</strong>.</p></li> 
      <li><p>In <strong>Solution Explorer</strong>, expand the <code>merges\android</code> folder.</p></li> 
      <li><p>Right-click the <code>merges\android\scripts</code> folder, and then choose <strong>paste</strong>.</p> <p>You now have a copy of the <code>weather.js</code> file in the <code>merges\android</code> folder.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/android-js-file.png" alt="Platform-Specific Javascript File"> </p></li> 
      <li><p>In the <code>merges\android\scripts\weather.js</code> file, add the following code to the <code>getWeatherWithGeoLocation</code> method.</p> <pre class="prettyprint copy-to-clipboard lang-javascript">$('#app-title').text("Android Weather");
</pre></li> 
      <li><p>Run your app in the Apache Ripple Simulator.</p> <p>The app now has the title <strong>Android Weather</strong> because <code>merges\android\scripts\weather.js</code> file replaces <code>www\scripts\weather.js</code> file for the Android version of your app.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/tailor-js.png" alt="Tailor the behavior"> </p> <p>Making a copy of a file this large to change one line of code is an extreme example, but you can begin to imagine the possibilities, and you can always refactor your code in ways that leverage the <code>merges</code> folder more efficiently.</p> <p>The readme file of each plugin will tell you what sort of device-specific code you'll have to write.</p></li> 
     </ol> 
     <h3><a id="tailor-the-appearance-of-your-app" name="tailor-the-appearance-of-your-app" href="#tailor-the-appearance-of-your-app"></a>Tailor the appearance of your app</h3> 
     <p>In some cases, you'll have to tweak the layout of your pages so that they appear correctly for one type of device or another.</p> 
     <ol> 
      <li><p>In <strong>Solution Explorer</strong> return to the <code>merges\android</code> folder .</p></li> 
      <li><p>Right-click the <code>android</code> folder, choose <strong>Add</strong> -&gt; <strong>New Folder</strong>, and name the folder <code>css</code>.</p></li> 
      <li><p>Right-click on the new <code>css</code> folder, and then choose <strong>Add</strong> -&gt; <strong>New CSS file</strong>.</p></li> 
      <li><p>Name the file <code>index.css</code>, and then choose the <strong>Add</strong> button.</p> <p>You now have a file named <code>index.css</code> in both the <code>\merges\android\css</code> folder and the <code>www\css</code> folders.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/android-css-file.png" alt="Platform-Specific style sheet File"> </p></li> 
      <li><p>Replace the contents of the new <code>\merges\android\css\index.css</code> file with the following code.</p> <pre class="prettyprint copy-to-clipboard lang-css">.not-displayed {
    display: none;
}

form {
    border-bottom: 1px solid #ddd;
    padding-bottom: 5px;
}

#zip-code-input {
    font-size: 25px;
}

#get-weather-btn {
    font-size: 22px;
}

#title {
    font-size: 16px;
}

#summary {
    font-size: 35px;
}

#summary img {
    float: right;
}

#error-msg {
    text-align: center;
    margin-top:  50%;
    font-weight: bold;    
}

.ui-bar-inherit.ui-header.header {
    background-color: #3388cc;
    border-color: #3388cc;
    color: #ffffff;
    text-shadow: 0 1px 0 #005599;
}
</pre> <p>Note that the styles defined in this new CSS file are identical to the <code>www\css\index.css</code>, with the exception of these element:</p> <pre class="prettyprint copy-to-clipboard lang-css">.ui-bar-inherit.ui-header.header {
    background-color: #3388cc;
    border-color: #3388cc;
    color: #ffffff;
    text-shadow: 0 1px 0 #005599;
}
</pre> <p>This is a simple example of what you can do by using the <code>merges</code> folder. A more effective approach would be to use a separate CSS file for platform-specific styles.</p></li> 
      <li><p>Run your app in the Apache Ripple Simulator.</p> <p>The app header has a blue background because <code>merges\android\css\index.css</code> replaced <code>www\css\index.css</code> in the Android version of the app.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/get-started-first-mobile-app/20170602075435/tailor-css.png" alt="Tailor the appearance"> </p></li> 
     </ol> 
     <h2><a id="next-steps" name="next-steps" href="#next-steps"></a>Next steps</h2> 
     <p>Congratulations on building your first cross-platform mobile app. Although this topic gets you started, theres so much more to learn.</p> 
     <p>Here are a few ideas about what you can explore next on your journey to build mobile apps with HTML and JavaScript and Visual Studio.</p> 
     <p><strong>Connect to Existing Data</strong></p> 
     <p>Want to pull in data from sites like <a href="https://api.stackexchange.com/">StackExchange</a> or <a href="https://developer.github.com/v3/">Github</a>? How about your own database? </p> 
     <p>Just like any browser, your Cordova app supports HTTP requests and Ajax calls. However, Cordova adds additional security to filter which scripts, CSS files, map files and data files can be pulled into your app.</p> 
     <ul> 
      <li>The <a href="https://cordova.apache.org/docs/en/latest/reference/cordova-plugin-whitelist/">Cordova Whitelist Plugin</a> (which is automatically added into every project) controls which URLs the WebView itself can be navigated to, can open, or can make network requests from when loading the app. </li> 
      <li>A Content Security Policy contained in a tag on index.html 
       <meta http-equiv="Content-Security-Policy" ...=""> controls which network requests (images, XHRs, etc) are allowed to be made after your app has loaded, via webview directly.</li> 
     </ul> 
     <p>Additionally, your server--or servers that you pull from--may have additional security that restricts your app from being able to pull resources into your app.</p> 
     <ul> 
      <li>Cross-origin resource sharing (CORS) policies regulates which resources (e.g. fonts, JavaScript, etc.) from a web page can be requested from another domain outside the domain from which the resource originated. So, often times when your app is pulling data, fonts, images, and scripts from another web service, you need to make it in the form of a CORS request. <a href="~/en-us/docs/tips-and-workarounds-general-readme/#connection">See more information</a>.</li> 
     </ul> 
     <p><strong>Easy backend set up with Azure</strong></p> 
     <p>Get straight to building by hooking your app up to Azure. Azure offers a Mobile App service that utilizes Easy Tables to get your app connected to a SQL database with only a couple lines of javascript. <a href="https://azure.microsoft.com/en-us/documentation/articles/app-service-mobile-cordova-get-started/">This helpful guide</a> takes you through the process of creating a mobile app backend, and ends with the option of creating a quickstart Cordova app, or adding connections to an existing app. </p> 
     <p><strong>Explore available Cordova plugins</strong></p> 
     <p>search for available Cordova plugins on npm (Cordova 5.x or greater). The naming convention for these plugins is cordova-plugin-pluginname or phonegap-plugin-pluginname.</p> 
     <blockquote> 
      <p><strong>Tip:</strong> If find a plugin that doesn't appear in the <strong>Plugins</strong> tab of configuration designer, you can still use it. <a href="../manage-plugins/#Custom">Learn more</a>.</p> 
     </blockquote> 
     <p><strong>Try using Bower to add a package to your project</strong></p> 
     <p>see <a href="../tutorial-using-bower/">Add packages by using Bower</a>.</p> 
     <p><strong>Learn about other great ways to leverage the Visual Studio debugger</strong></p> 
     <p>see <a href="https://msdn.microsoft.com/library/k0k771bt.aspx">Debugger Basics</a>.</p> 
     <p><strong>Explore the many JavaScript frameworks out there</strong></p> 
     <p>Search online. They're everywhere! Here's a couple of them: <a href="https://angularjs.org/">AngularJS</a>, and <a href="http://ionicframework.com/">Ionic</a>.</p> 
     <p><strong>Get a Mac and then run your app in an iOS emulator, or on an iOS device</strong></p> 
     <p>See <a href="../ios-guide/">Setup guide: Target iOS mobile devices in a Visual Studio Tools for Apache Cordova project</a>.</p> 
     <p><strong>Tack on an Azure backend - it's free to try</strong></p> 
     <p>see <a href="../add-connected-services/">Connect a cloud service</a>.</p> 
     <p><strong>Take a look at TypeScript - it's fully supported in your project</strong></p> 
     <p>see <a href="../tutorial-typescript/">Use TypeScript</a>.</p>
