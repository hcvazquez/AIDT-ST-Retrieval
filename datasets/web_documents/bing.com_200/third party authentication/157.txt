         <h1 class="title topictitle1">HttpFS Authentication</h1> 
         <div class="body conbody"> 
          <div class="p">
           This section describes how to configure HttpFS CDH 5 with Kerberos security on a Hadoop cluster: 
           <ul class="ul" id="topic_7__ul_klc_z4t_4k"> 
            <li class="li"><a class="xref" href="cdh_sg_httpfs_security.html#topic_7_1">Configuring the HttpFS Server to Support Kerberos Security</a></li> 
            <li class="li"><a class="xref" href="cdh_sg_httpfs_security.html#topic_7_2">Using curl to access an URL Protected by Kerberos HTTP SPNEGO</a></li> 
           </ul> 
          </div> 
          <p class="p">For more information about HttpFS, see <a class="xref" href="https://archive.cloudera.com/cdh5/cdh/5/hadoop/hadoop-hdfs-httpfs/index.html" target="_blank">https://archive.cloudera.com/cdh5/cdh/5/hadoop/hadoop-hdfs-httpfs/index.html</a>.</p> 
          <aside class="note important">
           <span class="title">Important:</span> 
           <p class="p">To enable HttpFS to work with Kerberos security on your Hadoop cluster, make sure you perform the installation and configuration steps in <a class="xref" href="cdh_sg_cdh5_hadoop_security.html#topic_3">Configuring Hadoop Security in CDH 5</a>.</p> 
          </aside> 
          <aside class="note important">
           <span class="title">Important:</span> 
           <p class="p">If the NameNode, Secondary NameNode, DataNode, JobTracker, TaskTrackers, ResourceManager, NodeManagers, HttpFS, or Oozie services are configured to use Kerberos HTTP SPNEGO authentication, and two or more of these services are running on the same host, then all of the running services must use the same HTTP principal and keytab file used for their HTTP endpoints.</p> 
          </aside> 
         </div> 
         <section class="nested1" id="topic_7_1_section">
          <a id="topic_7_1" class="anchor"></a> 
          <h2 class="title topictitle2">Configuring the HttpFS Server to Support Kerberos Security</h2> 
          <div class="body conbody"> 
           <ol class="ol" id="topic_7_1__ol_fkn_bfy_3m"> 
            <li class="li">Create an HttpFS service user principal that is used to authenticate with the Hadoop cluster. The syntax of the principal is: <samp class="ph codeph">httpfs/&lt;fully.qualified.domain.name&gt;@&lt;YOUR-REALM&gt;</samp> where: <samp class="ph codeph">fully.qualified.domain.name</samp> is the host where the HttpFS server is running <samp class="ph codeph">YOUR-REALM</samp> is the name of your Kerberos realm <pre class="pre codeblock">kadmin: addprinc -randkey httpfs/fully.qualified.domain.name@YOUR-REALM.COM</pre></li> 
            <li class="li">Create a HTTP service user principal that is used to authenticate user requests coming to the HttpFS HTTP web-services. The syntax of the principal is: <samp class="ph codeph">HTTP/&lt;fully.qualified.domain.name&gt;@&lt;YOUR-REALM&gt;</samp> where: <samp class="ph codeph">'fully.qualified.domain.name'</samp> is the host where the HttpFS server is running <samp class="ph codeph">YOUR-REALM</samp> is the name of your Kerberos realm <pre class="pre codeblock">kadmin: addprinc -randkey HTTP/fully.qualified.domain.name@YOUR-REALM.COM</pre> 
             <aside class="note important">
              <span class="title">Important:</span> 
              <p class="p">The HTTP/ component of the HTTP service user principal must be upper case as shown in the syntax and example above.</p> 
             </aside> </li> 
            <li class="li">Create keytab files with both principals. <pre class="pre codeblock">$ kadmin
kadmin: xst -k httpfs.keytab httpfs/fully.qualified.domain.name
kadmin: xst -k http.keytab HTTP/fully.qualified.domain.name</pre></li> 
            <li class="li">Merge the two keytab files into a single keytab file: <pre class="pre codeblock">$ ktutil
ktutil: rkt httpfs.keytab
ktutil: rkt http.keytab
ktutil: wkt httpfs-http.keytab</pre></li> 
            <li class="li">Test that credentials in the merged keytab file work. For example: <pre class="pre codeblock">$ klist -e -k -t httpfs-http.keytab</pre></li> 
            <li class="li">Copy the <samp class="ph codeph">httpfs-http.keytab</samp> file to the HttpFS configuration directory. The owner of the <samp class="ph codeph">httpfs-http.keytab</samp> file should be the <samp class="ph codeph">httpfs</samp> user and the file should have owner-only read permissions.</li> 
            <li class="li">Edit the HttpFS server <samp class="ph codeph">httpfs-site.xml</samp> configuration file in the HttpFS configuration directory by setting the following properties: 
             <div class="tablenoborder"> 
              <table id="topic_7_1__table_ujb_43s_mj" class="table"> 
               <thead class="thead" align="left"> 
                <tr class="row"> 
                 <th class="entry" valign="top" width="50%" id="d3286944e169"> <p class="p"><strong class="ph b">Property</strong></p> </th> 
                 <th class="entry" valign="top" width="50%" id="d3286944e174"> <p class="p"><strong class="ph b">Value</strong></p> </th> 
                </tr> 
               </thead> 
               <tbody class="tbody"> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.authentication.type</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">kerberos</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.hadoop.authentication.type</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">kerberos</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.authentication.kerberos.principal</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">HTTP/&lt;HTTPFS-HOSTNAME&gt;@&lt;YOUR-REALM.COM&gt;</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.authentication.kerberos.keytab</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">/etc/hadoop-httpfs/conf/httpfs-http.keytab</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.hadoop.authentication.kerberos.principal</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">httpfs/&lt;HTTPFS-HOSTNAME&gt;@&lt;YOUR-REALM.COM&gt;</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.hadoop.authentication.kerberos.keytab</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">/etc/hadoop-httpfs/conf/httpfs-http.keytab</p> </td> 
                </tr> 
                <tr class="row"> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e169"> <p class="p">httpfs.authentication.kerberos.name.rules</p> </td> 
                 <td class="entry" valign="top" width="50%" headers="d3286944e174"> <p class="p">Use the value configured for 'hadoop.security.auth_to_local' in 'core-site.xml'</p> </td> 
                </tr> 
               </tbody> 
              </table> 
             </div> 
             <aside class="note important">
              <span class="title">Important:</span> 
              <p class="p">You must restart the HttpFS server to have the configuration changes take effect.</p>

         <section class="nested1" id="topic_7_2_section">
          <a id="topic_7_2" class="anchor"></a> 
          <h2 class="title topictitle2">Using curl to access an URL Protected by Kerberos HTTP SPNEGO</h2> 
          <div class="body conbody"> 
           <aside class="note important">
            <span class="title">Important:</span> 
            <p class="p">Your version of <samp class="ph codeph">curl</samp> must support GSS and be capable of running <samp class="ph codeph">curl -V</samp>.</p> 
           </aside> 
           <p class="p"><strong class="ph b">To configure curl to access an URL protected by Kerberos HTTP SPNEGO:</strong></p> 
           <ol class="ol" id="topic_7_2__ol_icd_cfy_3m"> 
            <li class="li">Run <samp class="ph codeph">curl -V</samp>: <pre class="pre codeblock">$ curl -V
curl 7.19.7 (universal-apple-darwin10.0) libcurl/7.19.7 OpenSSL/0.9.8l
zlib/1.2.3
Protocols: tftp ftp telnet dict ldap http file https ftps
