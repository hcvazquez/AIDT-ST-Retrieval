        <div class="article-header"> 
         <h1 class="h2">Sending iOS Push Notifications via APNS in JavaScript</h1>
         <ul class="list-inline"> 
          <li>By <a href="/blog/author/tomomi/" title="Posts by Tomomi Imura" rel="author">Tomomi Imura</a></li> 
          <li>Follow <a href="http://twitter.com/" target="_blank">@</a></li> 
          <li><time datetime="2014-12-22">December 22, 2014</time></li> 
         </ul>
        </div>
        <div class="blog-description">
         <div style="border: 4px; border-color: #d02129; border-style: solid; padding: 15px 15px 10px 15px; background-color: #eff1f1; margin-bottom: 15px;">
          <img style="display: block; margin: auto; padding-bottom: 10px;" src="/wp-content/uploads/2017/07/warning.png" width="50" height="50">
          <p style="text-align: center;"><strong>We’ve updated our SDKs, and this code is now deprecated.</strong></p>
          <p style="text-align: center;"><strong>Learn more in our <a href="/developers/tech/push-notifications/" target="_blank" rel="noopener">Push Notification Overview</a>, <a href="/docs/swift/mobile-gateway" target="_blank" rel="noopener">Swift Docs</a> or <a href="/docs/ios-objective-c/mobile-gateway" target="_blank" rel="noopener">Objective-C Docs</a>.</strong></p>
         </div>
         <p>Previously, I covered this same topic for <a href="/blog/2014-12-18-sending-android-push-notifications-via-gcm-javascript-using-phonegap/">Android push notifications using Google Cloud Messaging (GCM)</a>. Today, I am going to show you a step-by-step tutorial how to send and receive iOS push notifications for devices via APNS.</p>
         <h3>Prerequisites</h3>
         <p>You need to be familiar with <a href="http://phonegap.com/">PhoneGap</a> Command Line Interface, or its open-source version, <a href="http://cordova.apache.org/">Cordova</a>. For the first time PhoneGap developer, you should begin with the previous article, <a href="/blog/2014-10-30-converting-your-javascript-app-to-an-ios-app-w-phonegap/">Converting Your JavaScript App to an iOS App w/ PhoneGap</a>. You need an actual iDevices, like iPhone and iPad. Push notification is not supported on emulators.</p>
         <p>Also, I expect you to have a lot of patience. Setting up all the requirements from Apple may not be as smooth as you expect.</p>
         <h2>iOS Push Notifications Demo</h2>
         <p>I have created this demo, which simulates an <em>Internet of Things</em> desktop controller that lets you change the room temperature setting. When you set it over 80<strong>°</strong>F, it sends your iOS device a push notification.</p>
         <p>Unfortunately due to lack of openness, I cannot distribute ad hoc iOS apk easily. If you have an Android device, <a href="https://pubnub.github.io/iot-push-demo">try the demo on Android</a>.</p>
         <div align="center">
          <iframe src="https://vine.co/v/O5YLdlhqLBn/embed/simple" width="480" height="480" frameborder="0"></iframe>
         </div>
         <h3>Using Apple Push Notification Service for iOS Push Notifications</h3>
         <p>Apple Push NotificationService (APNs) is a service for iOS devices to send and receive push notification messages. The typical flow looks like:</p>
         <ol>
          <li>An App requests iOS for push notification</li>
          <li>iOS sends the request to APNs</li>
          <li>APNs sends back a <em>device token</em></li>
          <li>The client app sends the toke to the 3rd party server (in this case, you don’t need your own server because you are going to use PubNub network!)</li>
         </ol>
         <p>To send a push notification:</p>
         <ol start="5">
          <li>The server (well, PubNub!) sends a payload to APNs with the token</li>
          <li>APNs sends a push notification to the device</li>
         </ol>
         <div id="attachment_8987" style="width: 768px" class="wp-caption aligncenter">
          <img class="size-full wp-image-8987 " title="How APNs works" src="/wp-content/uploads/2014/11/apns.png" alt="How APNs works" width="758" height="372">
          <p class="wp-caption-text">How APNs works</p>
         </div>
         <h3>1. App ID Configurations and Certificates</h3>
         <p>To developer and distribute your iOS app, you need to create an SSL certificate associated to an App ID and a provisioning profile.</p>
         <p>This step is not going to be fun, but you must complete to be able to start developing for iOS. It is more complicated than I described in the <a href="/blog/2014-10-30-converting-your-javascript-app-to-an-ios-app-w-phonegap/">last article</a>, so I am going to use .gif animations to show you exactly what you are going to do.</p>
         <h4>1.1. Issuing a Certificate Signing Request</h4>
         <p>To create an SSL certificate for a push notifications, first you need to generate a Certificate Signing Request (CSR).</p>
         <ol>
          <li>On your Mac, open <strong>Keychain Access</strong></li>
          <li>From the top menu, select <strong>Keychain Access</strong> &gt; <strong>Certificate Assistant</strong> &gt; <strong>Request a Certificate From a Certificate Authority…</strong></li>
          <li>Fill out your email address, common name</li>
          <li>Check <strong>Save to disk</strong></li>
          <li>Press <strong>Continue</strong> and save the the <em>.certSigningRequest</em> file somewhere</li>
         </ol>
         <div id="attachment_9026" style="width: 780px" class="wp-caption aligncenter">
          <img class="size-full wp-image-9026" title="Issuing a Certificate Signing Request" src="/wp-content/uploads/2014/11/apple01-csr.gif" alt="Issuing a Certificate Signing Request" width="770" height="466">
          <p class="wp-caption-text">Issuing a Certificate Signing Request</p>
         </div>
         <h4>1.2. Creating an App ID</h4>
         <p>You are going to create a unique App ID.</p>
         <ol>
          <li>On web browser, <a href="https://developer.apple.com/account/ios/identifiers/bundle/bundleList.action">Apple Developer portal</a> and navigate to <strong>Identifiers</strong> where you should see a list of your apps</li>
          <li>Click <strong>+</strong> button at the top right to add a new app</li>
          <li>At <strong>Registering an App ID</strong>, enter the name of your app, enter an ID, such as <em>com.yourdomain.pushApp</em> (Do not select Wildcard App ID), check <strong>Push Notifications</strong></li>
          <li>Click <strong>Continue</strong> then <strong>Submit</strong></li>
         </ol>
         <div id="attachment_8989" style="width: 710px" class="wp-caption aligncenter">
          <img class="size-full wp-image-8989" title="Creating an App ID" src="/wp-content/uploads/2014/11/apple02-appid.gif" alt="Creating an App ID" width="700" height="496">
          <p class="wp-caption-text">Creating an App ID</p>
         </div>
         <h4>1.3. Configuring Push Notifications for the App ID</h4>
         <p>Once the APP ID registration is completed, you need to configure it to make to work with push notification.</p>
         <ol>
          <li>Select the App ID you just created, and click <strong>Edit</strong></li>
          <li>At <strong>Push Notifications</strong>, click <strong>Create Certificate</strong> then **Continue</li>
          <li>Click <strong>Choose File…</strong> and upload the <em>.certSigningRequest</em> file you have generated and saved on your Mac earlier. Click <strong>Generate</strong>.</li>
          <li>Once the certificate was generated, download it onto your Mac</li>
         </ol>
         <div id="attachment_8990" style="width: 710px" class="wp-caption aligncenter">
          <img class="size-full wp-image-8990" title="Configuring Push Notifications for the App ID" src="/wp-content/uploads/2014/11/apple03-edit.gif" alt="Configuring Push Notifications for the App ID" width="700" height="576">
          <p class="wp-caption-text">Configuring Push Notifications for the App ID</p>
         </div>
         <h4>1.4 Creating a Private Key</h4>
         <p>Now you are going to create a private key from the downloaded certificate.</p>
         <ol>
          <li>Double-click the downloaded file. It should opens in Keychain Access</li>
          <li>Select <strong>My Certificates</strong> at left. You should find the certificated you just added.</li>
          <li>Control-click the certificate, and select <strong>Export “Apple Development IOS Push Services:…”…</strong></li>
          <li>Save it as <em>..p12</em> file, by selecting <strong>File Format</strong> as <em>Personal Information Exchange (.p12)</em> in the Save As dialog</li>
         </ol>
         <div id="attachment_8991" style="width: 965px" class="wp-caption aligncenter">
          <img class="size-full wp-image-8991" title="Creating a Private Key" src="/wp-content/uploads/2014/11/apple04-ssl-p12.png" alt="Creating a Private Key" width="955" height="456">
          <p class="wp-caption-text">Creating a Private Key</p>
         </div>
         <h4>1.5 Creating a Provisioning Profile</h4>
         <ol>
          <li>On Apple Developer portal, go to <strong>Provisioning Profiles</strong></li>
          <li>Click <strong>+</strong> to create a new provisioning file</li>
          <li>Select <em>iOS App Development</em>, then <strong>Continue</strong></li>
          <li>Select a correct App ID from the dropdown list, then <strong>Continue</strong></li>
          <li>Select a correct certificate from the list, then <strong>Continue</strong></li>
          <li>Select devices you are going to develop with, the <strong>Continue</strong></li>
          <li>Enter a name for the provisioning profile, then <strong>Generate</strong></li>
          <li>Download the file</li>
          <li>Double-click open the file in Xcode. The profile now should be added to Xcode.</li>
         </ol>
         <div id="attachment_8992" style="width: 710px" class="wp-caption aligncenter">
          <img class="size-full wp-image-8992" title="Creating a Provisioning Profile" src="/wp-content/uploads/2014/11/apple05-provision.gif" alt="Creating a Provisioning Profile" width="700" height="574">
          <p class="wp-caption-text">Creating a Provisioning Profile</p>
         </div>
         <p>&nbsp;</p>
         <h3>2. Developing iOS App with Cordova Push Plugin</h3>
         <p>Phew, I hope you survived the process. You are now finally going to write some code.</p>
         <p>In this example, I am using PhoneGap’s open-sourced sibling, Cordova. First, create a project using Cordova CLI.</p>
         <p><code>$ cordova create push-demo com.mydomain.push PushDemo</code></p>
         <p><code>$ cd push-demo</code></p>
         <p><code>$ cordova platform add ios</code></p>
         <p>If you already have created the <code>push-demo</code> project at the previous tutorial for Android, skip the first two lines and just add an iOS platform to your Cordova project. Also, skip the next section.</p>
         <h4>2.1. Installing Cordova Push Plugin</h4>
         <p><a href="https://github.com/phonegap-build/PushPlugin">Cordova Push Plugin</a> allows your application to receive push notifications on Android, as well as Amazon Fire OS, iOS, Windows Phone and Windows8 devices.</p>
         <p>The plugin can be automatically installed via the Cordova CLI:</p>
         <p><code>$ cordova plugin add https://github.com/phonegap-build/PushPlugin.git</code></p>
         <p>This will install <strong>PushNotification.js</strong> file under <code>/plugins</code> directory, and you need to copy this to somewhere under <code>/www</code>, where you will be working.</p>
         <h4>2.2. Using APNS with Cordova</h4>
         <p>Include the <strong>PushNotification.js</strong> and <strong>pubnub.js</strong> in <code>/www/index.html</code></p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=cordova-01.html"></script>
         <p>In this tutorial, you will be writing an iOS app to register a device to obtain an unique <em>token ID</em>, so your JavaScript web app can send it push notifications from desktop.</p>
         <p>When you created a new project template, you should already have an <code>/www/index.js</code> with a few functions that PhoneGap uses have already filled in. Where the <code>addEventListener</code> method registers for <code>deviceready</code> event type, change the name of the listener to the starting point of your app, let’s say <code>init</code>. And in your <code>init</code> function, create a plugin instance, then register the device as follows:</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=cordova-02.js"></script>
         <p>To support both Android (GCM) and iOS (APNs), install a Cordova Device plugin:</p>
         <p><code>$ cordova plugin add https://git-wip-us.apache.org/repos/asf/cordova-plugin-device.git</code></p>
         <p>and separate each platform, using <code>device.platform</code> API method:</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=cordova-03.js"></script>
         <p>When <code>tokenHandler</code> callback is called, you should successfully be able to obtain the device token.</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=cordova-04.js"></script>
         <p>To test APNs push notifications, you must test on your actual device. This feature is not supported on emulators.</p>
         <p>When you are ready, try <code>alert(result)</code> instead of <code>console.log</code> to view the device’s token in an alert popup on device.</p>
         <p>The token should look something like: <em>24c99cd5c39b283c58b554509c999999937aadb7b4bxxx0043cb363af6…</em></p>
         <p>In the next step, I am using the static token value, however in reality, you probably want to store the value in PubNub and get the value from client to send a message to a targeted iOS device.</p>
         <h3>3. Sending Push Notifications from Web Site</h3>
         <p>Now your iOS device is ready to receive iOS push notifications via APNs, so let’s create a desktop web to send messages.</p>
         <p>This diagram below is the same as the diagram shown at the intro before Section 1, plus a user-flow from a desktop browser is included to describe the demo for this tutorial. The desktop is where a user changes room temperature, as publishing the information to PubNub service (step 5 in gray). If the room temperature setting is above 80F, PubNub connects to APNS (step 6) to send a push notification to the registered device (step 7).</p>
         <div id="attachment_9142" style="width: 780px" class="wp-caption aligncenter">
          <a href="/wp-content/uploads/2014/12/apns-app.png"><img class="size-full wp-image-9142 " title="PubNub JavaScript API to send push notification to an iOS device via APNS" src="/wp-content/uploads/2014/12/apns-app.png" alt="PubNub JavaScript API to send push notification to an iOS device via APNS" width="770" height="486"></a>
          <p class="wp-caption-text">PubNub JavaScript API to send push notification to an iOS device via APNS</p>
         </div>
         <p>Normally, to be able to send messages to APNs, you need to set up your server. However, with using <a href="/docs/web-javascript/mobile-gateway">PubNub Mobile Push Gateway</a>, you can can instantly deliver platform specific push notifications by just write front-end JavaScript and even run it on places like GitHub pages. It really eliminates the burden of developing an additional server side component!</p>
         <h4>3.1. Creating a Web Page</h4>
         <p>From now on, you no longer use Cordova. Just ceate a new web page with HTML, CSS and JavaScript. Don’t forget to include <code>pubnub.js</code> to your <code>index.html</code>, then initialize PubNub.</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=web-01.js"></script>
         <p>In the <a href="https://pubnub.github.io/iot-push-demo">demo app</a>, I am simulating an Internet of Things user interface with a fake temperature controller. Once a user sets the temperature too high, let’s say, 80<strong>°</strong>F, it sends your device a push notification via APNs. This is a simplified code to handle the user interaction:</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=web-02.js"></script>
         <h4>3.2. Using PubNub Mobile Push Gateway</h4>
         <p>To use the feature, you need to activate the feature at <a href="https://admin.pubnub.com/">Admin portal</a>.</p>
         <ol>
          <li>Previously at Section 1.4. you have created a private key <em>.p12</em> file. Convert it to *.pem. using terminal as follows:</li>
         </ol>
         <p><code>$ openssl pkcs12 -in CertificateName.p12 -out CertificateName.pem -nodes</code></p>
         <ol start="2">
          <li>On PubNub <a href="https://admin.pubnub.com/">Admin portal</a>, click <strong>Add</strong> to enable <em>Push Notifications</em></li>
          <li>Drag the <em>.pem</em> file into the box below <strong>APNS PEM Cert</strong>, and click <strong>OK</strong></li>
         </ol>
         <p style="text-align: center;"><img class="size-full wp-image-9143 aligncenter" title="APNS PEM Cert" src="/wp-content/uploads/2014/12/pem.png" alt="APNS PEM Cert" width="575" height="140"></p>
         <h4>3.3. Sending Messages to Your Phone Using PubNub APIs</h4>
         <p>There are two step to send a message via APNS:</p>
         <ol>
          <li>Register a device to PubNub channel</li>
          <li>Send a message to APNS service</li>
         </ol>
         <p>To register a device to a channel, use <code>mobile_gw_provision</code>&nbsp;API. The basic usage is as follows:</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=web-03.js"></script>
         <p><em>Note: In the code, I used a static value for the gcm_regid to simplify the code sample, however in a real scenario, you are handling multiple devices. You should take advantage of <a href="/docs/web-javascript/storage-and-history">PubNub Storage/Playback Service</a> – store a registration ID from Cordova app, and grab the ID from web!</em></p>
         <p>Once the device is registered to PubNub, it calls the callback that sends a message payload to APNs service. Use a PubNub message object, <code>PNmessage</code>, then use <code>apns</code> to set the GCM message object.</p>
         <script src="https://gist.github.com/61fc122b7df844563c5f.js?file=web-04.js"></script>
         <p>Now you should be able to send iOS push notifications to your iOS device from a web browser without setting up your own server! Awesome!</p>

         <p>I used a simplified code sample to keep the tutorial easier to follow, but if you wish for the entire source code of the web app demo, you can view it on&nbsp;<a href="https://github.com/pubnub/iot-push-demo">GitHub</a>, and the source&nbsp;code for&nbsp;<a href="https://gist.github.com/girliemac/61fc122b7df844563c5f#file-cordova-all-js">index.js in Cordova app is on Gist.</a></p>
