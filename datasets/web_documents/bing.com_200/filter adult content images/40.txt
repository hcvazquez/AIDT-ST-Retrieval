          </div> 
          <p>The <code><strong>import</strong></code> statement is used to import functions, objects, or primitives which are defined in and exported by an external module, script, or the like.</p> 
          <div class="note"> 
           <p>This feature is only <a href="https://jakearchibald.com/2017/es-modules-in-browsers/" class="external">just beginning to be implemented</a> in browsers natively at this time. It is implemented in many transpilers, such as the <a href="https://github.com/google/traceur-compiler" class="external">Traceur Compiler</a>, <a href="http://babeljs.io/" class="external">Babel</a>, <a href="https://github.com/rollup/rollup" class="external">Rollup</a>, and <a href="https://webpack.js.org/" class="external">Webpack</a>.</p> 
          </div> 
          <h2 id="Syntax">Syntax</h2> 
          <pre class="syntaxbox">import <em>defaultMember</em> from "<em>module-name</em>";
import * as <em>name</em> from "<em>module-name</em>";
import { <em>member </em>} from "<em>module-name</em>";
import { <em>member</em> as <em>alias </em>} from "<em>module-name</em>";
import { <em>member1 , member2</em> } from "<em>module-name</em>";
import { <em>member1 , member2</em> as <em>alias2</em> , <em>[...]</em> } from "<em>module-name</em>";
import <em>defaultMember</em>, { <em>member</em> [ , <em>[...]</em> ] } from "<em>module-name</em>";
import <em>defaultMember</em>, * as <em>name</em> from "<em>module-name</em>";
import "<em>module-name</em>";</pre> 
          <dl> 
           <dt>
            <code>defaultMember</code>
           </dt> 
           <dd>
            Name that will refer to the default export from the module.
           </dd> 
           <dt>
            <code>module-name</code>
           </dt> 
           <dd>
            The URL of the module to import; this should be a complete relative or absolute URL to the 
            <code>.js</code> file containing the module, including the 
            <code>.js</code> extension. Certain bundlers, along with 
            <a href="/en-US/docs/Glossary/Node.js" class="glossaryLink" title="Node.js: Node.js&nbsp;is a cross-platform JavaScript runtime environment that allows developers to build server-side and network applications with JavaScript.">Node.js</a>, don't require the use of the extension, but the best practice is usually to use it unless you're certain you dont need it.
           </dd> 
           <dt>
            <code>name</code>
           </dt> 
           <dd>
            Name of the object that will refer to the imports.
           </dd> 
          </dl> 
          <dl> 
           <dt>
            <code>member, memberN</code>
           </dt> 
           <dd>
            Name of the exported members to be imported.
           </dd> 
           <dt>
            <code>alias, aliasN</code>
           </dt> 
           <dd>
            Names that will refer to the named imports.
           </dd> 
          </dl> 
          <h2 id="Description">Description</h2> 
          <p>The <code>name</code> parameter is the name which will refer to the exported members. The <code>member</code> parameters specify individual members, while the <code>name</code> parameter imports all of them. Below are examples to clarify the syntax.</p> 
          <h3 id="Import_an_entire_module's_contents">Import an entire module's contents</h3> 
          <p>This inserts <code>myModule</code> into the current scope, containing all the exported bindings from the module in the file located in <code>/modules/my-module.js</code>.</p> 
          <pre class="brush: js">import * as <em>myModule</em> from '/modules/my-module.js';
</pre> 
          <p>Here, accessing the members means using the module name ("myModule" in this case) as a namespace. For example, if the module imported above includes a member function <code>doAllTheAmazingThings()</code>, you would call it like this:</p> 
          <pre class="brush: js">myModule.doAllTheAmazingThings();</pre> 
          <h3 id="Import_a_single_member_of_a_module">Import a single member of a module</h3> 
          <p>Given an object or value named <code>myMember</code> which has been exported from the module <code>my-module</code> either implicitly (because the entire module is exported) or explicitly (using the <a href="/en-US/docs/Web/JavaScript/Reference/Statements/export" title="The export statement is used when creating JavaScript modules to export functions, objects, or primitive values from the module so they can be used by other programs with the import statement."><code>export</code></a> statement), this inserts <code>myMember</code> into the current scope.</p> 
          <pre class="brush: js">import {myMember} from '/modules/my-module.js';</pre> 
          <h3 id="Import_multiple_members_of_a_module">Import multiple members of a module</h3> 
          <p>This inserts both <code>foo</code> and <code>bar</code> into the current scope.</p> 
          <pre class="brush: js">import {foo, bar} from '/modules/my-module.js';</pre> 
          <h3 id="Import_a_member_with_a_more_convenient_alias">Import a member with a more convenient alias</h3> 
          <p>You can rename a member when importing it. For example, this inserts <code>shortName</code> into the current scope.</p> 
          <pre class="brush: js">import {reallyReallyLongModuleMemberName as shortName}
&nbsp; from '/modules/my-module.js';</pre> 
          <h3 id="Rename_multiple_members_during_import">Rename multiple members during import</h3> 
          <p>Import multiple members of a module with convenient aliases.</p> 
          <pre class="brush: js">import {
&nbsp; reallyReallyLongModuleMemberName as shortName,
&nbsp; anotherLongModuleName as short
} from '/modules/my-module.js';</pre> 
          <h3 id="Import_a_module_for_its_side_effects_only">Import a module for its side effects only</h3> 
          <p>Import an entire module for side effects only, without importing any bindings. This runs the module's global code, but doesn't actually add any bindings to your project.</p> 
          <pre class="brush: js">import '/modules/my-module.js';
</pre> 
          <h3 id="Importing_defaults">Importing defaults</h3> 
          <p>It is possible to have a default <a href="/en-US/docs/Web/JavaScript/Reference/Statements/export" title="The export statement is used when creating JavaScript modules to export functions, objects, or primitive values from the module so they can be used by other programs with the import statement."><code>export</code></a> (whether it is an object, a function, a class, etc.). The &nbsp;<code>import</code>&nbsp;statement may then be used to import such defaults.</p> 
          <p>The simplest version directly imports the default:</p> 
          <pre class="brush: js">import myDefault from '/modules/my-module.js';</pre> 
          <p>It is also possible to use the default syntax with the ones seen above (namespace imports or named imports). In such cases, the default import will have to be declared first. For instance:</p> 
          <pre class="brush: js">import myDefault, * as myModule from '/modules/my-module.js';
// myModule used as a namespace</pre> 
          <p>or</p> 
          <pre class="brush: js">import myDefault, {foo, bar} from '/modules/my-module.js';
// specific, named imports
</pre> 
          <h2 id="Examples">Examples</h2> 
          <p>Importing a secondary file to assist in processing an AJAX JSON request.</p> 
          <h3 id="The_module_file.js">The module: file.js</h3> 
          <pre class="brush: js; highlight: [14]">function getJSON(url, callback) {
  let xhr = new XMLHttpRequest();
  xhr.onload = function () { 
    callback(this.responseText) 
  };
  xhr.open('GET', url, true);
  xhr.send();
}

export function getUsefulContents(url, callback) {
  getJSON(url, data =&gt; callback(JSON.parse(data)));
}</pre> 
          <h3 id="The_main_program_main.js">The main program: main.js</h3> 
          <pre class="brush: js">import { getUsefulContents } from '/modules/file.js';

getUsefulContents('http://www.example.com',
    data =&gt; { doSomethingUseful(data); });</pre>
