              <p><strong>SVOX pico2wave</strong></p> 
              <p>In addition to the other answers:</p> 
              <p>A very minimalistic TTS, a better sounding than espeak or mbrola (to my mind).</p> 
              <p>Some information:<br> <a href="https://launchpad.net/ubuntu/precise/+source/svox/">https://launchpad.net/ubuntu/precise/+source/svox/</a></p> 
              <p>I don't understand why pico2wave is, compared to espeak or mbrola, rarely discussed. It's small, but sounds really good (natural). Without modification you'll hear a natural sounding female voice.</p> 
              <p>AND ... compared to Mbrola, it recognise Units and speaks it the right way!<br> For example: </p> 
              <ul> 
               <li>2°C ? two degrees</li> 
               <li>2m ? two meters</li> 
               <li>2kg ? two kilograms</li> 
              </ul> 
              <p>After installation I use it in a script:</p> 
              <pre><code>#!/bin/bash
pico2wave -l=de-DE -w=/tmp/test.wav "$1"
aplay /tmp/test.wav
rm /tmp/test.wav
</code></pre> 
              <p>Then run it with the desired text:</p> 
              <pre><code>&lt;scriptname&gt;.sh "hello world"
</code></pre> 
              <p>or read the contents of an entire file:</p> 
              <pre><code>&lt;scriptname&gt;.sh "$(cat &lt;filename&gt;)"
</code></pre> 
              <p>That's all to have a lightweight, stable working TTS on Ubuntu.</p>

              <p>I believe Ive found the best TTS software for free using a Google Chrome extension called "SpeakIt". This only works in the Chrome browser for me on Ubuntu. It doesnt work with Chromium for some reason. SpeakIt comes with two female voices which both sound very realistic compared to everything else out there. There are at least four more male &amp; female voices listed s Chrome extensions if you search the Chrome Web Store using "TTS" as your query.</p> 
              <p><a href="https://chrome.google.com/webstore/detail/speakit/pgeolalilifpodheeocdmbhehgnkkbak?utm_source=chrome-ntp-icon">https://chrome.google.com/webstore/detail/speakit/pgeolalilifpodheeocdmbhehgnkkbak?utm_source=chrome-ntp-icon</a></p> 
              <p>For use on a website. you highlight the text you want to be read and either right click and "SpeakIt" or click the SpeakIt icon docked on the Chrome top bar.</p> 
              <p>Firefox users also have two options. Within Firefox addons, do a search for TTS and you should find "Click Speak" and also "Text to Voice". The voices are not as good as the Chrome SpeakIt voices, but are definitely usable.</p> 
              <p>The SpeakIt extension uses iSpeech technology and for a price of $20 a year, the site can convert text to MP3 audio files. You can input text, URLs, RSS feeds, as well as documents such as TXT, DOC, and PDF and output to MP3. You can make podcast, embed audio, etc. Here is a link...</p> 
              <p><a href="http://www.ispeech.org/free.text.to.speech.tts.software">http://www.ispeech.org/free.text.to.speech.tts.software</a></p> 
              <p>and a sample of their audio (dont know how long the link will last)... </p> 
              <p><a href="http://www.ispeech.org/view/681080/4429622">http://www.ispeech.org/view/681080/4429622</a></p>

             <div class="post-text" itemprop="text"> 
              <p><strong>Simple Google™ TTS</strong></p> 
              <p>Because of the lack of a better alternative I wrote a <a href="https://github.com/Glutanimate/simple-google-tts">bash script</a> that interfaces with <a href="https://gist.github.com/michalfapso/3110049">a perl script by Michal Fapso</a> to provide TTS via Google Translate. From the project description:</p> 
              <blockquote> 
               <p>The intention is to provide an easy to use interface to text-to-speech output via Google's speech synthesis system. A fallback option using pico2wave automatically provides TTS synthesis in case no Internet connection is found.</p> 
               <p>As it stands, the wrapper supports reading from standard input, plain text files and the X selection (highlighted text).</p> 
              </blockquote> 
              <p>The main features are:</p> 
              <ul> 
               <li>online TTS synthesis via Google translate </li> 
               <li>offline TTS synthesis via pico2wave</li> 
               <li>supports a variety of different languages</li> 
               <li>can read from CLI, text files and highlighted text</li> 
               <li>supports reading highlighted text with fixed formatting (e.g. PDF files)</li> 
              </ul> 
              <p>Installation and usage are documented on the <a href="https://github.com/Glutanimate/simple-google-tts">project page</a>.</p> 
              <p>I'd be glad if you gave it a try. <a href="https://github.com/Glutanimate/simple-google-tts/issues">Bug reports and any other feedback</a> are welcome!</p>

              <p>Combine SVOX tools (pico) with LibreOffice:</p> 
              <p>SVOX (pico) tools are easy to install and brings good quality voices in Ubuntu. Install it:</p> 
              <pre><code>sudo apt-get install libttspico0 libttspico-utils libttspico-data
</code></pre> 
              <p>You can use LibreOffice in combination with SVOX (pico) tools by install the <a href="http://extensions.libreoffice.org/extension-center/read-text" rel="noreferrer">"Read Text" extension</a> and you obtain a "GUI" for this excellent TTS software:</p> 
              <p><a href="http://sites.google.com/site/readtextextension/home/linux" rel="noreferrer">Set up Read Text Extension's options</a> with <em>Tools - Add-ons - Read selection....</em> Use <strong>/usr/bin/python</strong> as the external program. Select a command line option that includes the token <strong>(PICO_READ_TEXT_PY)</strong>, you may want to experiment some of them. </p> 
              <p>Now you only have to select some text in LO Writer, Calc, Impress or Draw and clic on the icon added as a tool bar (a happy face with a ballon).</p>

             <div class="post-text" itemprop="text"> 
              <p>Here is what I did to have <strong>pure natural speech</strong> for pdf and other text files(other solutions are not natural or they're just paid services). This is actually a work around using chromium or chrome but works fast and easy.</p> 
              <ol> 
               <li>Install <a href="https://chrome.google.com/webstore/detail/speakit/pgeolalilifpodheeocdmbhehgnkkbak?hl=en" rel="nofollow">SpeakIt!</a> extension on your chrome or chromium.</li> 
               <li>Install <a href="https://chrome.google.com/webstore/detail/pdf-viewer/oemmndcbldboiebfnladdacbdfmadadm/related" rel="nofollow">PDF Viewer</a> if you're using chromium(chrome already has a pdf viewer for free) and check 'Allow in incognito' and 'Allow access to file URLs' options in extensions settings of chromium.</li> 
               <li>Drag and drop your pdf to browser.</li> 
               <li>Now highlight some text and right click and select SpeakIt! so you can listen to pure natural text-to-speech.</li> 
              </ol> 
              <p>There's also ways to open other files like .doc and .txt in chrome and do the same. There's other extensions for chrome that view pdf files, check if it fits you better. Besides you can upload all kind of texts in Google Drive and use SpeakIt! to read it for you. Another extension called 'Speak text' works the same way and has natural speech.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Pico and espeak are fun and easy to get to work, but they're not all that good. The default Festival voices are also not that good. However, Festival is a scheme-based speech framework, where a number of researchers have built much better plug-in voices. You can easily surpass the pico2wave quality on stock Ubuntu, because one of those voices is available as a ready-made package.</p> 
              <p>To make Festival sound natural, here's what to do:</p> 
              <pre><code>sudo apt-get install festival
sudo apt-get install festvox-us-slt-hts
festival -i
festival&gt; (voice_cmu_us_slt_arctic_hts) 
festival&gt; (SayText "Don't hate me, I'm just doing my job!")
</code></pre> 
              <p>You can do it from the command line by using <code>-b</code> (or <code>--batch</code>) and putting each command into single quotes:</p> 
              <pre><code>festival -b '(voice_cmu_us_slt_arctic_hts)' \
    '(SayText "The temperature is 22 degrees centigrade and there is a slight breeze from the west.")'
</code></pre> 
              <p>You can get other quite good voices from the Nitech repository, but installing them is finicky, and the default paths changed so the file name references in the bundled scheme files may need to be manually edited to work on stock Ubuntu.</p>

              <p>when searching for a better tts engine to use with the new <strong>firefox 49 narrative mode</strong> I found <strong>pico tts</strong> (svox) - my favorite TTS engine.</p> 
              <pre><code>sudo apt install espeak libttspico0 libttspico-data libttspico-utils
</code></pre> 
              <p>How to change the default speech synthesis engine system wide? </p> 
              <p>People at arch linux brought me to the right path ( <a href="https://bbs.archlinux.org/viewtopic.php?id=217411" rel="nofollow noreferrer">https://bbs.archlinux.org/viewtopic.php?id=217411</a> ):</p> 
              <p>Uncomment the module you like and make it default in speech-dispatcher settings:</p> 
              <pre><code>#&gt; vim /etc/speech-dispatcher/speechd.conf

[...]
# -----OUTPUT MODULES CONFIGURATION-----
# Each AddModule line loads an output module.
#AddModule "espeak"       "sd_espeak"   "espeak.conf"
AddModule "pico-generic"  "sd_generic"   "pico-generic.conf"

[...]
#DefaultModule espeak
DefaultModule pico-generic
</code></pre> 
              <p>Restart the daemon:</p> 
              <pre><code>#&gt; sudo systemctl restart speech-dispatcher.service
</code></pre> 
              <p>BUT, when starting firefox again, nothing happens. According to the link above (arch forum post #10 and #16) works with festival (did not try), but the speech-dispatcher for pico does not list available voices. It won't run.</p> 
              <p>Any idea out there would be highly appreciated ;-)</p>
