      <h2 id="overview">Overview</h2> 
      <p>You can use the IBM Mobile Foundation framework to add multilingual translation in JavaScript (Cordova, Web) applications into other languages.<br> Items that can be translated are application strings and system messages.</p> 
      <h4 id="jump-to">Jump to:</h4> 
      <ul> 
       <li><a href="#translating-application-strings">Translating application strings</a></li> 
       <li><a href="#translating-system-messages">Translating system messages</a></li> 
       <li><a href="#multilanguage-translation">Multilanguage translation</a></li> 
       <li><a href="#detecting-the-device-locale-and-language">Detecting the device locale and language</a></li> 
       <li><a href="#sample-application">Sample application</a></li> 
      </ul> 
      <h2 id="translating-application-strings">Translating application strings</h2> 
      <p>Strings that are destined to be translated are stored in a <code class="highlighter-rouge">JSON</code> object called “Messages”.</p> 
      <ul> 
       <li>In Cordova applications that use the MobileFirst SDK, you can find it in the <strong>index.js</strong> file of the Cordova application: <strong>[cordova-project-root-directory]/www/js/index.js</strong>.</li> 
       <li>In Web applications, you need to add it.</li> 
      </ul> 
      <h3 id="json-object-structure-example">JSON object structure example</h3> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="kd">var</span> <span class="nx">Messages</span> <span class="o">=</span> <span class="p">{</span>
    <span class="na">headerText</span><span class="p">:</span> <span class="s2">"Default header"</span><span class="p">,</span>
    <span class="na">actionsLabel</span><span class="p">:</span> <span class="s2">"Default action label"</span><span class="p">,</span>
    <span class="na">sampleText</span><span class="p">:</span> <span class="s2">"Default sample text"</span><span class="p">,</span>
<span class="p">};</span>
</code></pre> 
      </div> 
      <p>Strings stored in the Messages <code class="highlighter-rouge">JSON</code> object can be referenced in two ways in the application logic:</p> 
      <p><strong>As a JavaScript object property:</strong></p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">Messages</span><span class="p">.</span><span class="nx">headerText</span>
</code></pre> 
      </div> 
      <p><strong>As an ID of an HTML element with <code class="highlighter-rouge">class="translate"</code>:</strong></p> 
      <div class="language-html highlighter-rouge">
       <pre class="highlight"><code><span class="nt">&lt;h1</span> <span class="na">id=</span><span class="s">"headerText"</span> <span class="na">class=</span><span class="s">"translate"</span><span class="nt">&gt;&lt;/h1&gt;</span>
</code></pre> 
      </div> 
      <h2 id="translating-system-messages">Translating system messages</h2> 
      <p>It is also possible to translate the system messages that the application displays, for example “Internet connection is not available” or “Invalid username or password”. System messages are stored in the <code class="highlighter-rouge">WL.ClientMessages</code> object.</p> 
      <p><strong>Note:</strong> Override system messages at a global JavaScript level because some parts of the code are executed only after the application has successfully initialized.</p> 
      <h3 id="web-applications">Web applications</h3> 
      <p>You can find a full list of system messages in the <code class="highlighter-rouge">messages.json</code> file, located in the <strong>[project root folder]\node_modules\ibm-mfp-web-sdk\lib\messages\ folder</strong>.</p> 
      <h3 id="cordova-applications">Cordova applications</h3> 
      <p>You can find a full list of system messages in the <code class="highlighter-rouge">messages.json</code> file, located inside the generated project.</p> 
      <ul> 
       <li>Android: <code class="highlighter-rouge">[Cordova-project]\platforms\android\assets\www\plugins\cordova-plugin-mfp\worklight\messages</code></li> 
       <li>iOS, Windows: <code class="highlighter-rouge">[Cordova-project]\platforms\[ios or windows]\www\plugins\cordova-plugin-mfp\worklight\messages</code></li> 
      </ul> 
      <p>To translate a system message, override it in the application code.</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">WL</span><span class="p">.</span><span class="nx">ClienMessages</span><span class="p">.</span><span class="nx">loading</span> <span class="o">=</span> <span class="s2">"Application HelloWorld is loading... please wait."</span><span class="p">;</span>
</code></pre> 
      </div> 
      <h2 id="multilanguage-translation">Multilanguage translation</h2> 
      <p>Using JavaScript, you can implement multilanguage translation for your application.<br> The below steps explain the implementation of this tutorial’s sample application.</p> 
      <ol> 
       <li> <p>Set up the default application strings in the <code class="highlighter-rouge">index.js</code> file.</p> 
        <div class="language-javascript highlighter-rouge">
         <pre class="highlight"><code><span class="kd">var</span> <span class="nx">Messages</span> <span class="o">=</span> <span class="p">{</span>
     <span class="na">headerText</span><span class="p">:</span> <span class="s2">"Default header"</span><span class="p">,</span>
     <span class="na">actionsLabel</span><span class="p">:</span> <span class="s2">"Default action label"</span><span class="p">,</span>
     <span class="na">sampleText</span><span class="p">:</span> <span class="s2">"Default sample text"</span><span class="p">,</span>

      <h2 id="detecting-the-device-locale-and-language">Detecting the device locale and language</h2> 
      <p>To detect the language used by the device or browser:</p> 
      <h3 id="web-applications-locale">Web applications</h3> 
      <p>Detect the browser language using <code class="highlighter-rouge">navigator.language</code> or any number of available frameworks and solutins.</p> 
      <h3 id="cordova-applications-locale">Cordova applications</h3> 
      <p>Detect the locale and the language of the device using the Cordova’s globalization plug-in: <code class="highlighter-rouge">cordova-plugin-globalization</code>.<br> The globalization plug-in is auto-installed when adding a platform to the Cordova application.</p> 
      <p>Use the <code class="highlighter-rouge">navigator.globalization.getLocaleName</code> and <code class="highlighter-rouge">navigator.globalization.getPreferredLanguage</code> functions to detect the locale and language respectively.</p>
