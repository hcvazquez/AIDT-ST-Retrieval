    <h1>DOM Manipulation</h1> 
    <div class="entry"> 
     <p>We previously <a href="/you-dont-need-jquery/selectors">learned how to easily select elements without relying on jQuery</a>. but what about changing elements? What about creating new elements? How about relocating elements elsewhere on the page? You may be glad to know that all of this, and more, is also possible without pulling in jQuery. The Web API provides us with all of the tools we need, once again.</p> 
     <p>You will see that some DOM manipulation is trivial without the aid of jQuery or any other library. However, others may be a bit tricky. This is the time to stress (again) that I am not trying to bash jQuery, nor will I assert that jQuery is useless or completely unnecessary. The intent of these posts is to inform you how to work with the browser without jQuery, if you so choose. You may find that, in many cases, a large library like jQuery goes mostly unused, and can be omitted.</p>

     <h2 id="creating-elements">Creating Elements</h2> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'&lt;div&gt;&lt;/div&gt;'</span><span class="p">);</span>
</code></pre>
     </div> 
     <h4 id="dom-api">DOM API</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>

     <p>Wow, that was pretty easy. jQuery saved us a few keystrokes, but that's hardly worth the bytes.</p> 
     <h2 id="inserting-elements-before-amp-after">Inserting Elements Before &amp; After</h2> 
     <p>Let's create an element and insert it after another specific element.</p> 
     <p>So, we start with:</p> 
     <div class="highlight">

     </div> 
     <p>... and we'd like to create a new element with an ID of '1.1' and insert it between the first two DIVs, giving us this:</p> 
     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;div</span> <span class="na">id=</span><span class="s">"1"</span><span class="nt">&gt;&lt;/div&gt;</span>

</code></pre>
     </div> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#1'</span><span class="p">).</span><span class="nx">after</span><span class="p">(</span><span class="s1">'&lt;div id="1.1"&gt;&lt;/div&gt;'</span><span class="p">);</span>

     </div> 
     <p>Ha! Take THAT jQuery! Pretty easy in every browser just relying on the tools built into the browser. </p>

     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;div</span> <span class="na">id=</span><span class="s">"0.9"</span><span class="nt">&gt;&lt;/div&gt;</span>

</code></pre>
     </div>

     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#1'</span><span class="p">).</span><span class="nx">before</span><span class="p">(</span><span class="s1">'&lt;div id="0.9"&gt;&lt;/div&gt;'</span><span class="p">);</span>

     </div> 
     <h4 id="dom-api">DOM API</h4>

     <h2 id="inserting-elements-as-children">Inserting Elements As Children</h2> 
     <p>Let's say we have this:</p> 
     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;div</span> <span class="na">id=</span><span class="s">"parent"</span><span class="nt">&gt;</span>

     <p>...and we want to create a new element and make it the first child of the parent, like so:</p> 
     <div class="highlight">

<span class="nt">&lt;/div&gt;</span>

     </div> 
     <h4 id="jquery">jQuery</h4>

      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#parent'</span><span class="p">).</span><span class="nx">prepend</span><span class="p">(</span><span class="s1">'&lt;div id="newChild"&gt;&lt;/div&gt;'</span><span class="p">);</span>

     <p>All of this looks a lot like the previous section that dealt with insertion of new elements. Again, it's pretty simple to do all of this, cross-browser, without any help from jQuery (or any other library). </p> 
     <h2 id="moving-elements">Moving Elements</h2>

     <div class="highlight">

     <h2 id="removing-elements">Removing Elements</h2> 
     <p>How can we remove an element from the DOM? Let's say we know an element with an ID of 'foobar' exists. Let's kill it!</p> 
     <h3 id="jquery">jQuery</h3> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#foobar'</span><span class="p">).</span><span class="nx">remove</span><span class="p">();</span>
</code></pre>
     </div> 
     <h3 id="dom-api">DOM API</h3> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foobar'</span><span class="p">).</span><span class="nx">parentNode</span>
    <span class="p">.</span><span class="nx">removeChild</span><span class="p">(</span><span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foobar'</span><span class="p">));</span>
</code></pre>
     </div> 
     <p>The DOM API approach is certainly a bit verbose and ugly, but it works! Note that we don't have to be directly aware of the parent element, which is nice.</p>

     <p>Let's remove that class now:</p> 
     <h4 id="jquery">jQuery</h4>

     <h4 id="dom-api">DOM API</h4>

<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">classList</span><span class="p">.</span><span class="nx">remove</span><span class="p">(</span><span class="s1">'bold'</span><span class="p">);</span>


<span class="c1">// All browsers</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">className</span> <span class="o">=</span> 
    <span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">className</span><span class="p">.</span><span class="nx">replace</span><span class="p">(</span><span class="sr">/^bold$/</span><span class="p">,</span> <span class="s1">''</span><span class="p">);</span>
</code></pre>
     </div> 
     <p>For IE 10 and up, the API is pretty much the same between jQuery and the Web API. </p> 
     <h2 id="adding-removing-changing-attributes">Adding/Removing/Changing Attributes</h2> 
     <p>Let's start out with a simple element, like this:</p> 
     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;div</span> <span class="na">id=</span><span class="s">"foo"</span><span class="nt">&gt;&lt;/div&gt;</span>
</code></pre>
     </div> 
     <p>Now, let's say this <code>&lt;div&gt;</code> actually functions as a button. We should attach the approriate <code>role</code> attribute to make this element more accessible. </p> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#foo'</span><span class="p">).</span><span class="nx">attr</span><span class="p">(</span><span class="s1">'role'</span><span class="p">,</span> <span class="s1">'button'</span><span class="p">);</span>
</code></pre>
     </div> 
     <h4 id="dom-api">DOM API</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">setAttribute</span><span class="p">(</span><span class="s1">'role'</span><span class="p">,</span> <span class="s1">'button'</span><span class="p">);</span>
</code></pre>
     </div> 
     <p>In both cases, a new attribute can be created, or an existing attribute can be updated using the same code. </p> 
     <p>What if the behavior of our <code>&lt;div&gt;</code> changes, and it no longer functions as a button. In fact, it's just a plain 'ole <code>&lt;div&gt;</code> now with some trivial text or markup. Let's remove that <code>role</code>...</p> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#foo'</span><span class="p">).</span><span class="nx">removeAttr</span><span class="p">(</span><span class="s1">'role'</span><span class="p">);</span>
</code></pre>
     </div> 
     <h4 id="dom-api">DOM API</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">removeAttribute</span><span class="p">(</span><span class="s1">'role'</span><span class="p">);</span>
</code></pre>
     </div> 
     <h2 id="adding-amp-changing-text-content">Adding &amp; Changing Text Content</h2> 
     <p>Now, we have the following markup:</p> 
     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;div</span> <span class="na">id=</span><span class="s">"foo"</span><span class="nt">&gt;</span>Hi there!<span class="nt">&lt;/div&gt;</span>
</code></pre>
     </div> 
     <p>...but we want to update the text to say "Goodbye!".</p> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#foo'</span><span class="p">).</span><span class="nx">text</span><span class="p">(</span><span class="s1">'Goodbye!'</span><span class="p">);</span>
</code></pre>
     </div> 
     <p>Note that you can also easily retrieve the current text of the element by calling <code>text</code> with no parameters.</p> 
     <h4 id="dom-api">DOM API</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">innerHTML</span> <span class="o">=</span> <span class="s1">'Goodbye!'</span><span class="p">;</span>

<span class="c1">// IE 5.5+ but NOT Firefox</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">innerText</span> <span class="o">=</span> <span class="s1">'GoodBye!'</span><span class="p">;</span>

<span class="c1">// IE 9+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'foo'</span><span class="p">).</span><span class="nx">textContent</span> <span class="o">=</span> <span class="s1">'Goodbye!'</span><span class="p">;</span>
</code></pre>
     </div> 
     <p>Both properties above will return the current element's HTML/text as well.</p> 
     <p>The advantage to using <code>innerText</code> or <code>textContent</code> is that any HTML is escaped, which is a great feature if the content is user-supplied and you only ever want to include text as content for the selected element.</p> 
     <h2 id="adding-updating-element-styles">Adding/Updating Element Styles</h2> 
     <p>Generally, adding styles inline or with JavaScript is a <a href="http://en.wikipedia.org/wiki/Code_smell">"code smell"</a>, but it may be needed in some unique instances. For those cases, I'll show you how that can be done with jQuery and the DOM API.</p> 
     <p>Given a simple element with some text:</p> 
     <div class="highlight">
      <pre><code class="language-html" data-lang="html"><span class="nt">&lt;span</span> <span class="na">id=</span><span class="s">"note"</span><span class="nt">&gt;</span>Attention!<span class="nt">&lt;/span&gt;</span>
</code></pre>
     </div> 
     <p>...we'd like to make it stand out a bit more, so let's make it appear bold.</p> 
     <h4 id="jquery">jQuery</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="nx">$</span><span class="p">(</span><span class="s1">'#note'</span><span class="p">).</span><span class="nx">css</span><span class="p">(</span><span class="s1">'fontWeight'</span><span class="p">,</span> <span class="s1">'bold'</span><span class="p">);</span>
</code></pre>
     </div> 
     <h4 id="dom-api">DOM API</h4> 
     <div class="highlight">
      <pre><code class="language-javascript" data-lang="javascript"><span class="c1">// IE 5.5+</span>
<span class="nb">document</span><span class="p">.</span><span class="nx">getElementById</span><span class="p">(</span><span class="s1">'note'</span><span class="p">).</span><span class="nx">style</span><span class="p">.</span><span class="nx">fontWeight</span> <span class="o">=</span> <span class="s1">'bold'</span><span class="p">;</span>
</code></pre>
     </div> 
     <p>I actually prefer the way the DOM API approach looks in this case. It seems much more intuitive than jQuery's <code>css</code> method.</p> 
     <h2 id="micro-libraries-for-more-help">Micro-Libraries For More Help</h2> 
     <p>So, guess what? You don't really need jQuery for cross-browser DOM manipulation either! I realize that jQuery may make some more complex manipulation a bit easier, bit if you're only really concerned with complex DOM manipulation tasks, then consider pulling in a smaller library that focuses mostly on this. A couple choices are <a href="https://github.com/kupriyanenko/jbone">jBone</a> and <a href="https://github.com/dkraczkowski/dom.js">dom.js</a>. There's nothing wrong with "rolling your own" library either. DOM manipulation isn't as difficult as you may think.</p>
