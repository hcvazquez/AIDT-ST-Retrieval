                    <h2><a name="TOC-Overview"></a>Overview</h2> 
                    <p>Character set detection is the process of determining the character set, or encoding, of character data in an unknown format. This is, at best, an imprecise operation using statistics and heuristics. Because of this, detection works best if you supply at least a few hundred bytes of character data that's mostly in a single language. In some cases, the language can be determined along with the encoding.</p> 
                    <p>Several different techniques are used for character set detection. For multi-byte encodings, the sequence of bytes is checked for legal patterns. The detected characters are also check against a list of frequently used characters in that encoding. For single byte encodings, the data is checked against a list of the most commonly occurring three letter groups for each language that can be written using that encoding. The detection process can be configured to optionally ignore html or xml style markup, which can interfere with the detection process by changing the statistics.</p> 
                    <p>The input data can either be a Java input stream, or an array of bytes. The output of the detection process is a list of possible character sets, with the most likely one first. For simplicity, you can also ask for a Java Reader that will read the data in the detected encoding.</p>
                    <p>There is another character set detection C++ library, the <a href="https://github.com/google/compact_enc_det">Compact Encoding Detector</a>, that may have a lower error rate, particularly when working with short samples of text.</p> 
                    <h2><a name="TOC-CharsetMatch"></a>CharsetMatch</h2> 
                    <p>The <font face="courier, monospaced">CharsetMatch</font> class holds the result of comparing the input data to a particular encoding. You can use an instance of this class to get the name of the character set, the language, and how good the match is. You can also use this class to decode the input data.</p> 
                    <p>To find out how good the match is, you use the getConfidence() method to get a <i>confidence value</i>. This is an integer from 0 to 100. The higher the value, the more confidence there is in the match For example:</p>

                    <p>To get the name of the character set, which can be used as an encoding name in Java, you use the <font face="courier, monospaced">getName()</font> method:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;byte characterData[] = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;String charsetName;<br>&nbsp;&nbsp;&nbsp;&nbsp;String unicodeData;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;charsetName = match.getName();<br>&nbsp;&nbsp;&nbsp;&nbsp;unicodeData = new String(characterData, charsetName);<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>To get the name of the character set in C :</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;const UCharsetMatch *ucm;<br>&nbsp;&nbsp;&nbsp;&nbsp;UErrorCode status = U_ZERO_ERROR;<br>&nbsp;&nbsp;&nbsp;&nbsp;const char *name = ucsdet_getName(ucm, &amp;status);</pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>To get the three letter ISO code for the detected language, you use the <font face="courier, monospaced">getLanguage()</font> method. If the language could not be determined, <font face="courier, monospaced">getLanguage()</font> will return <font face="courier, monospaced">null.</font><font face="arial, sans-serif">&nbsp;Note that language detection does not work with all charsets, and includes only a very small set of possible languages. It should not used if robust, reliable language detection is required.</font></p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;String languageCode;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;languageCode = match.getLanguage();<br>&nbsp;&nbsp;&nbsp;&nbsp;<br>&nbsp;&nbsp;&nbsp;&nbsp;if (languageCode != null) {<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// handle the language code...<br>&nbsp;&nbsp;&nbsp;&nbsp;}<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>The ucsdet_getLanguage(const UCharsetMatch *ucsm, UErrorCode *status) method can be used in C to get the language code. If the language could not be determined, the method will return an empty string.</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;const UCharsetMatch *ucm;<br>&nbsp;&nbsp;&nbsp;&nbsp;UErrorCode status = U_ZERO_ERROR;<br>&nbsp;&nbsp;&nbsp;&nbsp;const char *language = ucsdet_getLanguage(ucm, &amp;status);</pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you want to get a Java <font face="courier, monospaced">String</font> containing the converted data you can use the <font face="courier, monospaced">getString()</font> method:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;String unicodeData;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;unicodeData = match.getString();<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you want to limit the number of characters in the string, pass the maximum number of characters you want to the <font face="courier, monospaced">getString()</font> method:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;String unicodeData;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;unicodeData = match.getString(1024);<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>To get a <font face="courier, monospaced">java.io.Reader</font> to read the converted data, use the <font face="courier, monospaced">getReader()</font> method:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;Reader reader;<br>&nbsp;&nbsp;&nbsp;&nbsp;StringBuffer sb = new StringBuffer();<br>&nbsp;&nbsp;&nbsp;&nbsp;char[] buffer = new char[1024];<br>&nbsp;&nbsp;&nbsp;&nbsp;int bytesRead = 0;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;reader = match.getReader();<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<br>&nbsp;&nbsp;&nbsp;&nbsp;while ((bytesRead = reader.read(buffer, 0, 1024)) &gt;= 0) {<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;sb.append(buffer, 0, bytesRead);<br>&nbsp;&nbsp;&nbsp;&nbsp;}<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<br>&nbsp;&nbsp;&nbsp;&nbsp;reader.close();<br></pre> </td>

                    <h2><a name="TOC-CharsetDetector"></a>CharsetDetector</h2> 
                    <p>The <font face="courier, monospaced">CharsetDetector</font> class does the actual detection. It matches the input data against all character sets, and computes a list of <font face="courier, monospaced">CharsetMatch</font> objects to hold the results. The input data can be supplied as an array of bytes, or as a <font face="courier, monospaced">java.io.InputStream</font>.</p> 
                    <p>To use a <font face="courier, monospaced">CharsetDetector</font> object, first you construct it, and then you set the input data, using the <font face="courier, monospaced">setText()</font> method. Because setting the input data is separate from the construction, it is easy to reuse a <font face="courier, monospaced">CharsetDetector</font> object:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetDetector detector;<br>&nbsp;&nbsp;&nbsp;&nbsp;byte[] byteData = ...;<br>&nbsp;&nbsp;&nbsp;&nbsp;InputStream streamData = ...;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector = new CharsetDetector();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector.setText(byteData);<br>&nbsp;&nbsp;&nbsp;&nbsp;// use detector with byte data...<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector.setText(streamData);<br>&nbsp;&nbsp;&nbsp;&nbsp;// use detector with stream data...<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you want to know which character set matches your input data with the highest confidence, you can use the <font face="courier, monospaced">detect()</font> method, which will return a <font face="courier, monospaced">CharsetMatch</font> object for the match with the highest confidence:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetDetector detector;<br>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match;<br>&nbsp;&nbsp;&nbsp;&nbsp;byte[] byteData = ...;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector = new CharsetDetector();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector.setText(byteData);<br>&nbsp;&nbsp;&nbsp;&nbsp;match = detector.detect();<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you want to know which character set matches your input data in C, you can use the ucsdet_detect( UCharsetDetector *csd , UErrorCode *status) method.</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;UCharsetDetector* csd;<br>&nbsp;&nbsp;&nbsp;&nbsp;const UCharsetMatch *ucm;<br>&nbsp;&nbsp;&nbsp;&nbsp;static char buffer[BUFFER_SIZE] = {....};<br>&nbsp;&nbsp;&nbsp;&nbsp;int32_t inputLength = ... //length of the input text<br>&nbsp;&nbsp;&nbsp;&nbsp;UErrorCode status = U_ZERO_ERROR;<br>&nbsp;&nbsp;&nbsp;&nbsp;ucsdet_setText(csd, buffer, inputLength, &amp;status);<br>&nbsp;&nbsp;&nbsp;&nbsp;ucm = ucsdet_detect(csd, &amp;status);</pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you want to know all of the character sets that could match your input data with a non-zero confidence, you can use the <font face="courier, monospaced">detectAll()</font> method, which will return an array of <font face="courier, monospaced">CharsetMatch</font> objects sorted by confidence, from highest to lowest.:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetDetector detector;<br>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch matches[];<br>&nbsp;&nbsp;&nbsp;&nbsp;byte[] byteData = ...;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector = new CharsetDetector();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector.setText(byteData);<br>&nbsp;&nbsp;&nbsp;&nbsp;matches = detector.detectAll();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;for (int m = 0; m &lt; matches.length; m += 1) {<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// process this match...<br>&nbsp;&nbsp;&nbsp;&nbsp;}<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <table border="0" cellpadding="0" cellspacing="3"> 
                     <tbody> 
                      <tr> 
                       <td valign="top" width="20"> <img alt="Note" border="0" height="24" hspace="0" src="http://icu-project.org/userguide/images/note.gif" vspace="0" width="20"></td> 
                       <td valign="top"><i>Note:The ucsdet_detectALL( UCharsetDetector *csd , int32_t* matchesFound, UErrorCode* status) method can be used in C in order to detect all of the character sets where matchesFound is a pointer to a variable that will be set to the number of charsets identified that are consistent with the input data. </i></td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>The <font face="courier, monospaced">CharsetDetector</font> class also implements a crude <i>input filter</i> that can strip out html and xml style tags. If you want to enable the input filter, which is disabled when you construct a CharsetDetector, you use the enableInputFilter() method, which takes a boolean. Pass in true if you want to enable the input filter, and false if you want to disable it:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetDetector detector;<br>&nbsp;&nbsp;&nbsp;&nbsp;CharsetMatch match;<br>&nbsp;&nbsp;&nbsp;&nbsp;byte[] byteDataWithTags = ...;<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector = new CharsetDetector();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;detector.setText(byteDataWithTags);<br>&nbsp;&nbsp;&nbsp;&nbsp;detector.enableInputFilter(true);<br>&nbsp;&nbsp;&nbsp;&nbsp;match = detector.detect();<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>To enable an input filter in C, you can use ucsdet_enableInputFilter( UCharsetDetector* csd, UBool filter) function.</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;UCharsetDetector* csd;<br>&nbsp;&nbsp;&nbsp;&nbsp;const UCharsetMatch *ucm;<br>&nbsp;&nbsp;&nbsp;&nbsp;static char buffer[BUFFER_SIZE] = {....};<br>&nbsp;&nbsp;&nbsp;&nbsp;int32_t inputLength = ... //length of the input text<br>&nbsp;&nbsp;&nbsp;&nbsp;UErrorCode status = U_ZERO_ERROR;<br>&nbsp;&nbsp;&nbsp;&nbsp;ucsdet_setText(csd, buffer, inputLength, &amp;status);<br>&nbsp;&nbsp;&nbsp;&nbsp;ucsdet_enableInputFilter( csd, TRUE);<br>&nbsp;&nbsp;&nbsp;&nbsp;ucm = ucsdet_detect(csd, &amp;status);<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>If you have more detailed knowledge about the structure of the input data, it is better to filter the data yourself before you pass it to <font face="courier, monospaced">CharsetDetector</font>. For example, you might know that the data is from an html page that contains CSS styles, which will not be stripped by the input filter.</p> 
                    <p>You can use the inputFilterEnabled() method to see if the input filter is enabled:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr> 
                       <td bgcolor="#eeeeee" valign="top"> <pre>&nbsp;&nbsp;&nbsp;&nbsp;CharsetDetector detector;<br>&nbsp;&nbsp;&nbsp;&nbsp;<br>&nbsp;&nbsp;&nbsp;&nbsp;detector = new CharsetDetector();<br><br>&nbsp;&nbsp;&nbsp;&nbsp;// do a bunch of stuff with detector<br>&nbsp;&nbsp;&nbsp;&nbsp;// which may or may not enable the input filter...<br><br>&nbsp;&nbsp;&nbsp;&nbsp;if (detector.inputFilterEnabled()) {<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// handle enabled input filter<br>&nbsp;&nbsp;&nbsp;&nbsp;} else {<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;// handle disabled input filter<br>&nbsp;&nbsp;&nbsp;&nbsp;}<br></pre> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <table border="0" cellpadding="0" cellspacing="3"> 
                     <tbody> 
                      <tr> 
                       <td valign="top" width="20"> <img alt="Note" border="0" height="24" hspace="0" src="http://icu-project.org/userguide/images/note.gif" vspace="0" width="20"></td> 
                       <td valign="top"><i>Note:The ICU4C API provide uscdet_isInputFilterEnabled(const UCharsetDetector* csd) function to check whether the input filter is enabled.</i></td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>The <font face="courier, monospaced">CharsetDetector</font> class also has two convenience methods that let you detect and convert the input data in one step: the getReader() and getString() methods:</p> 
                    <table border="1" cellpadding="8" cellspacing="0"> 
                     <tbody> 
                      <tr>

                      </tr> 
                     </tbody> 
                    </table> 
                    <table border="0" cellpadding="0" cellspacing="3"> 
                     <tbody> 
                      <tr> 
                       <td valign="top" width="20"> <img alt="Note" border="0" height="24" hspace="0" src="http://icu-project.org/userguide/images/note.gif" vspace="0" width="20"></td> 
                       <td valign="top"><br> </td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <table border="0" cellpadding="0" cellspacing="3"> 
                     <tbody> 
                      <tr>

                       <td valign="top"><i>Note: the second argument to the getReader() and getString() methods is a String called declaredEncoding, which is not currently used. There is also a setDeclaredEncoding() method, which is also not currently used.</i></td> 
                      </tr> 
                     </tbody> 
                    </table> 
                    <p>The following code is equivalent to using the convenience methods:</p> 
                    <table border="1" cellpadding="8" cellspacing="0">
