Given a graph that has been laid out, edgepaint colors the edges in a manner to make it easier for the viewer to distinguish edges that intersect in an almost parallel fashion.</p>

In addition, we introduce <b>mingle</b>, a filter for edge bundling. </p><p>&nbsp;</p>

In addition, if the poppler library is available, Graphviz can now use PDF files as images. </p><h4>New features (July 2013)</h4> <p>Graphviz now supports the <strong>star</strong> node shape <img alt="" class="displayed" src="/News/star.png" style="display:block; margin-left:auto; margin-right:auto; margin-top:15px; width:80px"> and a variation of linear gradient fill with no gradient (thus, two solid colors) where the user can specify the proportion using a colorList type for fillcolor.</p> <p>&nbsp;</p>

If this works as planned, the user should see no differences. The advantage of the library is that it is much more flexible for the programmer, with a cleaner API and support for dynamic data structures. This latter, in addition to allowing easier mixing of algorithms, has the potential to greatly reduce memory use in Graphviz.</p> <p>One side-effect of the new library is a new level ranking algorithm for dot. This algorithm has fewer constraints, so it is now possible for a node to be in a cluster and be involved in a rank=same constraint.</p> <h4>New features (November 2012)</h4> <p>We have provided Graphviz with some additional graphics, which can be used to provide more semantic information in a graph or enhance its aesthetics. These include <a href="http://www.graphviz.org/content/attrs#kstyle">gradient fill</a> for nodes and graphs; <a href="http://www.graphviz.org/content/attrs#kstyle">striped and wedged fills</a>, so the amount of each color can be used to represent some percentage; <a href="http://www.graphviz.org/content/node-shapes">new node shapes</a> pertinent to the synthetic biology community (Thanks to Jenny Cheng); and curved edge routing.</p> <p>&nbsp;</p>

There was a recent <a href="http://www.lri.fr/~isenberg/publications/papers/Holten_2011_AEP.pdf">paper</a> by Holten et al about a study of the readability of various edge styles. We're working on some examples and documentation. Here are some examples. (We need a better example for xlabels, because they can float, and aren't stuck to the upper left of the node.)</p> <p><a href="http://www.graphviz.org/pub/tmp/images/tapered-edge.png" target="_blank"><img alt="inset of example diagram with tapered edges" src="/News//tapered-edge-inset.png"></a><img alt="example of Petersen graph diagrams with point nodes and floating node labels" src="/News/Petersen-xlabel.png" style="height:300px; width:330px"></p> <h4>New Release 2.28 (Summer 2011)</h4> <p>The 2.28 release of Graphviz incorporates&nbsp;<a href="http://www2.research.att.com/~yifanhu/MAPS/index.html" target="_blank">GvMap</a> (formerly GMap, or "graphs as maps"), an invention by Emden Gansner, Yifan&nbsp;Hu, and Stephen Kobourov. Try the cool interactive <a href="http://www2.research.att.com/~yifanhu/MAPS/imap.html" target="_blank">demos</a> showing maps of music, TV, books, and more.</p> <p>For people (like us) that care about such matters, we worked pretty hard to tweak the font rendering code, to&nbsp;<br> eliminate&nbsp;glitches causing off-center labels or the ransom-note effect.</p> <p>John Ellson finished porting Graphviz to <a href="http://www.opencsw.org/packages/CSWgraphviz/">Solaris OpenCSW</a>.</p> <p>We changed our license&nbsp;to the&nbsp;<a href="http://www.eclipse.org/legal/epl-v10.html" style="text-decoration: none; font-weight: bold; color: rgb(51, 153, 204);" target="_blank">EPL</a>&nbsp;(Eclipse Public License). (A&nbsp;<a href="http://en.wikipedia.org/wiki/Eclipse_Public_License" style="text-decoration: none; font-weight: bold; color: rgb(51, 153, 204);" target="_blank">wikipedia article</a>&nbsp;attempts to explain the differences, which we believe to be fairly limited but the change may help some commercial licensees.)</p> <h4>Graphviz in Google Charts</h4> <p>See <a href="http://code.google.com/apis/chart/image/docs/gallery/graphviz.html">here</a>.</p> <h4>Upgraded Bug-Tracking System</h4> <p>The <a href="http://www.graphviz.org/content/graphviz-issue-tracker">Graphviz bug tracking system</a> was moved to <a href="http://trac.edgewall.org/">Trac</a>. We are also moving the source code database to <a href="http://git-scm.com/">Git</a>, which seems more developer-friendly.</p> <h4>Edge Bundling to be presented at Pacific Vis 2011</h4> <p>Edge bundling is one of the most effective techniques available for reducing clutter in large graph layouts,<br> so there is considerable motivation to develop techniques that are effective and run fast. We have developed<br> methods to formally measure and reduce edge ink, that also allow GPU acceleration. This work will be presented at<a href="http://i.cs.hku.hk/~pvis2011/" target="_blank"> Pacific Vis 2011</a> in Hong Kong,&nbsp;&nbsp;March 1-4.</p> <h4>Graph Drawing 2010</h4> <p><em>On Maximum Differential Graph Coloring </em>was presented at GD2010 in Konstanz, Germany, Sept 21-24. This work is related to GMap.</p> <h4>GMap presented at Pacific Vis 2010</h4> <p><em>GMap: Visualizing graphs and clusters as maps </em>was presented at Pacific Visualization in Tapei, Taiwan, China, in February. A revised version of this work appeared in the Nov-Dec issue of IEEE Computer Graphics and Applications.</p> <h4>Graphs at CHI 2009</h4> <p>An interesting session on <a href="http://chi2009.org/Attending/AdvanceProgram/66.html" target="_blank"> <strong>Understanding Graphs</strong></a> was chaired by Martin Watternberg at CHI 2009 in Boston. One of the talks, <a href="http://www.moscovich.net/tomer/papers/moscovich_toponav_chi_2009.pdf" target="_blank"> Topology-Aware Navigation in Large Networks</a> presented novel techniques that were implemented and are available in the excellent <a href="http://zvtm.sf.net/zgrviewer.html" target="_blank"> ZGRViewer</a> zooming graph viewer Java program for Graphviz. (See their <a href="http://www.youtube.com/watch?v=hm2oFBqVM9o" target="_blank">video on youtube</a>.) Meanwhile, here's a <a href="News/new1.pdf">pdf</a> for your enjoyment (3.1M).</p> <h4>Software Visualization of the Graphviz Project</h4> <p><br> <a href="http://vimeo.com/2300505" target="_blank">Graphviz code_swarm</a> from <a href="http://vimeo.com/user925215" target="_blank">Pander Musubi</a> on <a href="http://vimeo.com" target="_blank">Vimeo</a>.</p> <h4>Curated Example Graphs (2008)</h4> <p>Donate your interesting example graphs! Please contact <a href="http://www2.research.att.com/~yifanhu/contact.html" target="_blank"> Yifan Hu </a></p> <h4>Graphviz for Windows (2007)</h4> <p>The 2.16 release occurred on 9 November. In general, in addition to various bug fixes, the release provides three new node shapes, "tab", "box3d" and "component", provided by Diomidis Spinellis, and we've added support for the "image" and "imagescale" attributes to simplify the use of images in nodes.</p> <p>More importantly for Windows users, the release provides many system features which were previously only available on Unix systems. These include:</p> 
             <ul> 
              <li>Rendering using the Cairo library. In particular, using this it is possible to get PostScript output using general Unicode input and non-Type 1 fonts.</li> 
              <li>Font naming using the fontconfig library.</li> 
              <li>Support for plug-ins. With this architectural change, users can add their own layout engines or renderers without having to make any changes to the Graphviz programs.</li> 
              <li>The gvpr tool, which enables a user to process graphs simply.</li>

SchemaSpy relies on Graphviz for visualization. Its description on SourceForge explains:</p> <p>SchemaSpy analyzes schema metadata, letting you click through the hierarchy of your tables' parent/child relationships either graphically or through HTML tables. It works with just about any RDBMS given an appropriate JDBC driver. SchemaSpy also identifies several common schema anomalies.</p> <h4>New scalable algorithm for directed graphs (Oct. 2005)</h4> <p>Graphviz now incorporates <em><a href="http://www.csse.monash.edu.au/~tdwyer/dig-cola_slides.ppt">DiG-COLA</a></em> (directed graph constrained layout) algorithm for which <a href="http://www.research.att.com/groups/infovis/people.html" target="_blank">Yehuda Koren</a> and <a href="http://www.csse.monash.edu.au/~tdwyer/" target="_blank">Tim Dwyer</a> (Monash University) won the IEEE Infovis 2005 Best Paper Award. (Try: neato -Gmode=hier). Neato already incorporates the <em>stress majorization</em> algorithm presented at Graph Drawing 2004, that supports an alternative model for graphs with nodes of high degree. (Try: neato -Gmodel=subset)</p> <h4>New plug-in architecture for layout, scripting languages and renderers (Oct. 2005)</h4> <p>Graphviz now has a more modular architecture that supports multiple layout engines, scripting languages, and renderers. About a dozen scripting languages, including perl, python, tcl and ruby are supported. An optional backend with high-quality antialiased graphics is written in <a href="http://cairographics.org" target="_blank">cairo</a>, and it also provides portable interactive graph canvas objects in <a href="http://www.gtk.org" target="_blank">GTK+</a>. This work was done by John Ellson.</p> <h4>&nbsp;</h4>
