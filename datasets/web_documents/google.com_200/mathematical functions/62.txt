          <div class="cell normal" data-type="normal"> 
           <div class="innerCell"> 
            <p>JavaScript is available as of&nbsp;<a href="/display/DOC/8.24">version 8.24.2</a>&nbsp;for scripts created as part of an operation (i.e. a project item). At this time, JavaScript is not available for use within transformation mappings or the script pad. See <a href="/display/DOC/Creating+a+Script">Creating a Script</a>&nbsp;for instructions on creating a new JavaScript and using the script builder in Jitterbit.</p>
            <h2 id="JavaScript-JavaScriptSupportinJitterbit">JavaScript Support in Jitterbit</h2>
            <p>Jitterbit's Javascript engine&nbsp;supports the ECMAScript 5.1 standard as specified here:&nbsp;<a rel="nofollow" href="https://www.ecma-international.org/ecma-262/5.1/" class="external-link">https://www.ecma-international.org/ecma-262/5.1/</a>. The standard JavaScript language has native support of JSON. JavaScript also has native support of definition/use of functions inside scripts.&nbsp;Jitterbit conforms to standard JavaScript object manipulation and behavior.</p>
            <div class="confluence-information-macro confluence-information-macro-warning conf-macro output-block" data-hasbody="true" data-macro-name="warning">
             <span class="aui-icon aui-icon-small aui-iconfont-error confluence-information-macro-icon"> </span>
             <div class="confluence-information-macro-body">
              <p><strong>NOTE:</strong> While Jitterbit supports JavaScript based on the standard, not all JavaScript functionality may be available. Jitterbit does <span><strong>not</strong></span> support the following:</p>
              <ul>
               <li>Document Object Model (DOM) web APIs</li>
               <li>Mozilla built-in functions and objects</li>
               <li>Some JavaScript types such as <code>Set</code> and <code>Map</code></li>
               <li>Access to Java objects</li>
              </ul>
             </div>
            </div>
            <p>Simple data types, as well as arrays&nbsp;and JSON objects, are fully supported. Jitterbit maps are also supported within JavaScript. JavaScript treats Jitterbit&nbsp;maps as JSON objects and Jitterbit scripts treat&nbsp;JSON objects as Jitterbit&nbsp;maps. JSON properties are accessed via map keys.</p>
            <p>For example, given this JSON object defined in JavaScript:</p>
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: js; gutter: false; theme: Confluence" data-theme="Confluence">var $myObj = {
 "name":"John",
 "age":30,
 "cars": {
   "car1":"Ford",
   "car2":"BMW",
   "car3":"Fiat"
   }
 };</pre> 
             </div>
            </div>
            <p>In Jitterbit script, the object would be accessed by a map. Access the "car3" property like this:</p>
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: java; gutter: false; theme: Confluence" data-theme="Confluence">$mycar = $myObj["cars"]["car3"];</pre> 
             </div>
            </div>
            <h2 id="JavaScript-CreatingaNewJavaScript">Creating a New JavaScript</h2>
            <p>Within your project within Jitterbit Studio, there are several ways to create a new JavaScript as a project item:</p>
            <ul>
             <li>In the tree on the left, right-click on the&nbsp;<strong>Scripts</strong>&nbsp;section and select<strong>&nbsp;New JavaScript</strong>.</li>
            </ul>
            <ul>
             <li>Within an existing operation, right-click on any insertion point&nbsp;<span class="confluence-embedded-file-wrapper confluence-embedded-manual-size"><img class="confluence-embedded-image confluence-thumbnail" height="12" src="/download/thumbnails/65699902/image2017-6-14_11-53-41.png?version=1&amp;modificationDate=1497462821588&amp;api=v2" data-image-src="/download/attachments/65699902/image2017-6-14_11-53-41.png?version=1&amp;modificationDate=1497462821588&amp;api=v2" data-unresolved-comment-count="0" data-linked-resource-id="66848242" data-linked-resource-version="1" data-linked-resource-type="attachment" data-linked-resource-default-alias="image2017-6-14_11-53-41.png" data-base-url="https://success.jitterbit.com" data-linked-resource-content-type="image/png" data-linked-resource-container-id="65699902" data-linked-resource-container-version="37"></span>&nbsp;and choose&nbsp;<strong>Insert</strong>&nbsp;&gt;&nbsp;<strong>Script</strong>. Or, right-click&nbsp;on a source or target and choose&nbsp;<strong>Insert Before This</strong>&nbsp;&gt;&nbsp;<strong>Script</strong>&nbsp;or&nbsp;<strong>Insert After This</strong>&nbsp;&gt;&nbsp;<strong>Script</strong>. A script icon will appear within the operation graph. Then do one of the following:
              <ul>
               <li>Right-click on the script icon and choose&nbsp;<strong>Create New JavaScript</strong>.</li>
              </ul>
              <ul>
               <li>Double-click on the script icon&nbsp;and click the button&nbsp;<strong>Create New JavaScript</strong>.</li>
              </ul></li>
            </ul>
            <h2 id="JavaScript-UsingScriptBuilder">Using Script Builder</h2>
            <p>After you have created a new JavaScript in Jitterbit, you can enter your JavaScript directly within the script area.</p>
            <div class="confluence-information-macro confluence-information-macro-information conf-macro output-block" data-hasbody="true" data-macro-name="info">
             <span class="aui-icon aui-icon-small aui-iconfont-info confluence-information-macro-icon"> </span>
             <div class="confluence-information-macro-body">
              <strong>NOTE:</strong> Unlike a Jitterbit script, which requires that the script be wrapped within 
              <code>&lt;trans&gt; ... &lt;/trans&gt;</code> tags, there are no tags needed for scripts in JavaScript. Instead, simply enter your JavaScript directly within the script area.
             </div>
            </div>
            <p><span class="confluence-embedded-file-wrapper"><img class="confluence-embedded-image" src="/download/attachments/65699902/image2017-4-26%2014%3A37%3A1.png?version=1&amp;modificationDate=1493239021109&amp;api=v2" data-image-src="/download/attachments/65699902/image2017-4-26%2014%3A37%3A1.png?version=1&amp;modificationDate=1493239021109&amp;api=v2" data-unresolved-comment-count="0" data-linked-resource-id="65700014" data-linked-resource-version="1" data-linked-resource-type="attachment" data-linked-resource-default-alias="image2017-4-26 14:37:1.png" data-base-url="https://success.jitterbit.com" data-linked-resource-content-type="image/png" data-linked-resource-container-id="65699902" data-linked-resource-container-version="37"></span></p>
            <h3 id="JavaScript-Functions">Functions</h3>
            <p>The functions available to use in a JavaScript are available under four categories.<span class="conf-macro output-inline" data-hasbody="true" data-macro-name="multiexcerpt"></span></p>
            <p><span class="confluence-embedded-file-wrapper"><img class="confluence-embedded-image" src="/download/attachments/65699902/image2017-5-31_12-27-36.png?version=1&amp;modificationDate=1496255256247&amp;api=v2" data-image-src="/download/attachments/65699902/image2017-5-31_12-27-36.png?version=1&amp;modificationDate=1496255256247&amp;api=v2" data-unresolved-comment-count="0" data-linked-resource-id="66847621" data-linked-resource-version="1" data-linked-resource-type="attachment" data-linked-resource-default-alias="image2017-5-31_12-27-36.png" data-base-url="https://success.jitterbit.com" data-linked-resource-content-type="image/png" data-linked-resource-container-id="65699902" data-linked-resource-container-version="37"></span></p>
            <h4 id="JavaScript-Jitterbit">Jitterbit</h4>
            <p>This category contains a list of functions specific to Jitterbit. These include a limited number of standard Jitterbit functions, as well as some Javascript functions specific to Jitterbit.</p>
            <p>Currently, a limited number of Jitterbit functions are directly available for use in JavaScript. To access a Jitterbit function in your JavaScript, simply prefix the function with "Jitterbit." The following Jitterbit functions are available for use in JavaScript in Jitterbit:</p>
            <ul>
             <li><p class="auto-cursor-target"><code>Jitterbit.ReadFile(string source_id[, string file_name])<br></code>Reads a file from the specified source as described in&nbsp;<a href="/display/DOC/JavaScript">Formula Builder File Functions</a>. The returned value is a string. For example, in JavaScript, using this Jitterbit function to read a file might look like this:</p>
              <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">
               <div class="codeContent panelContent pdl"> 
                <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: js; gutter: false; theme: Confluence" data-theme="Confluence">var MyData = Jitterbit.ReadFile("&lt;TAG&gt;Sources/myfile&lt;/TAG&gt;");</pre> 
               </div>
              </div></li>
             <li><code>Jitterbit.WriteFile(string target_id, type file_contents[, string file_name])<br></code>Writes to a file and automatically flushes file. There is no returned value. See more at&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder++File+Functions#FormulaBuilderFileFunctions-writefileFunction:WriteFile" rel="nofollow">Function: WriteFile</a>&nbsp;under&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder++File+Functions" rel="nofollow">Formula Builder File Functions</a>.</li>
             <li><code>Jitterbit.DbExecute(string database_id, string sql_str, ...)<br></code>Executes a SQL statement on a database and returns the results. The returned value is a string. See more at&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder+Database+Functions#FormulaBuilderDatabaseFunctions-dbexecuteFunction:DbExecute" rel="nofollow">Function: DbExecute</a>&nbsp;under&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder+Database+Functions" rel="nofollow">Formula Builder Database Functions</a>.</li>
             <li><code>Jitterbit.DbLookup(string database_id, string sql_str)<br></code>Executes a SQL statement on a database and returns the first result matching the specified criteria. The returned value is a string. See more at&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder+Database+Functions#FormulaBuilderDatabaseFunctions-dblookupFunction:DBLookup" rel="nofollow">Function: DBLookup</a>&nbsp;under&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder+Database+Functions" rel="nofollow">Formula Builder Database Functions</a>.<code><br></code></li>
             <li><code>Jitterbit.GetVar(string name)<br></code>Gets a previously assigned Jitterbit global variable (prefixed with "Jitterbit"). The returned value is the appropriate data type. See&nbsp;<a href="https://success.jitterbit.com/display/DOC/Using+Global+Variables" rel="nofollow">Using Global Variables</a>&nbsp;for more information.&nbsp;An example is provided below using both&nbsp;<code>Jitterbit.GetVar</code>&nbsp;and&nbsp;<code>Jitterbit.SetVar</code>.&nbsp;</li>
             <li><p><code>Jitterbit.SetVar(string name, string value)<br></code>Assigns a value to a Jitterbit global variable (prefixed with "Jitterbit") (see&nbsp;<a href="https://success.jitterbit.com/display/DOC/Using+Global+Variables" rel="nofollow">Using Global Variables</a>). The returned value&nbsp;is a string. The following example uses a custom API that runs an operation containing a JavaScript using&nbsp;<code>Jitterbit.GetVar</code>&nbsp;and&nbsp;<code>Jitterbit.SetVar</code>:</p>
              <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">
               <div class="codeContent panelContent pdl"> 
                <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: js; gutter: false; theme: Confluence" data-theme="Confluence">var response="Hello World";

response+="\r\nProject name: " + Jitterbit.GetVar("$jitterbit.operation.project_name");
response+="\r\nOperation Name: " + Jitterbit.GetVar("$jitterbit.operation.name");
response+="\r\ntest_var: " + Jitterbit.GetVar("$jitterbit.api.request.parameters.test_var");

Jitterbit.SetVar("$jitterbit.api.response.headers.content_type","text/plain");
Jitterbit.SetVar("$jitterbit.api.response.headers.test_header", "This is a header test");
Jitterbit.SetVar("$jitterbit.api.response",response);</pre> 
               </div>
              </div>
              <div class="confluence-information-macro confluence-information-macro-information conf-macro output-block" data-hasbody="true" data-macro-name="info">
               <span class="aui-icon aui-icon-small aui-iconfont-info confluence-information-macro-icon"> </span>
               <div class="confluence-information-macro-body">
                <p><strong>NOTE:</strong></p>
                <ul>
                 <li>As of&nbsp;<a href="https://success.jitterbit.com/display/DOC/8.26" rel="nofollow">version 8.26</a>, JavaScript variables must now start with a dollar sign in order to be global to the Jitterbit engine. This fixes the issue in version 8.25 where&nbsp;where global variables are not able to be assigned or referenced in a&nbsp;JavaScript.&nbsp;</li>
                 <li>As of&nbsp;<a href="https://success.jitterbit.com/display/DOC/8.26" rel="nofollow">version 8.26</a>, JavaScripts can now access Jitterbit variables whose names contain a period by making use of the Jitterbit.GetVar and Jitterbit.SetVar functions. In previous versions 8.24.2 or 8.25, periods in&nbsp;a JavaScript variable name were replaced with underscores.</li>
                </ul>
               </div>
              </div></li>
            </ul>
            <p>Also included in this category are several functions adapted for use with JavaScript in Jitterbit. These are not prefixed with "Jitterbit."</p>
            <ul>
             <li><code>WriteToOperationLog<br></code>Sends output to the current Jitterbit operation log. Built for use with JavaScript in Jitterbit, but works similarly to the Jitterbit function described in&nbsp;<a href="https://success.jitterbit.com/display/DOC/Formula+Builder+Logging+and+Error+Functions" rel="nofollow">Formula Builder Logging and Error Functions</a>.</li>
             <li><p class="auto-cursor-target"><code>SetScriptOutput</code>&nbsp;&amp;&nbsp;<code>SetScriptResult<br></code>Sets the output for the script. These functions are interchangeable depending on user preference. Unlike Jitterbit scripts, which automatically return the value of the last line, you can use either of these functions in your JavaScript to return the script result. For example:</p>
              <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">
               <div class="codeContent panelContent pdl"> 
                <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: js; gutter: false; theme: Confluence" data-theme="Confluence">var MyVariable = "Hello World";
WriteToOperationLog(MyVariable);
SetScriptResult(MyVariable);</pre> 
               </div>
              </div></li>

            <h3 id="JavaScript-Plugins">Plugins</h3>
            <p>Currently, using plugins in a JavaScript is not supported. Instead,&nbsp;use a Jitterbit script.</p>
            <h3 id="JavaScript-Test">Test</h3>
            <p>In a JavaScript, you can see or validate the values of variables just as you would for a Jitterbit script. However, the&nbsp;<strong>Disable breakpoints</strong>&nbsp;option is not applicable since debugging is not available in JavaScript.</p>

/////////////////

function getRandomNumbers(howMany,min,max) {
  var output = "";

  for (var i=0; i&lt;howMany; i++) {
    output = output + getRandomNumber(min,max) + " \n";
  }  

  return output;
}

function getRandomNumber(min,max) {
  return Math.floor((Math.random() * max) + min);
