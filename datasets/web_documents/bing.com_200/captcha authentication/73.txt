        <div class="section-title"> 
         <h3>Situation</h3> 
        </div> 
        <div class="article-authored">
         <p>Java Authentication fails when attempting to import a package.</p>
        </div> 
        <div class="section-title"> 
         <h3>Cause</h3> 
        </div> 
        <div class="article-authored">
         <p>Java runtime issue with IIS Integrated Windows Authentication mode.</p>
        </div> 
        <div class="section-title"> 
         <h3>Solution</h3> 
        </div> 
        <div class="article-authored">   
         <p>For Symantec Management Platform 7.0 "IIS 6"<br> - Open IIS manager, expand Altiris virtual directory &gt; Software Management, right click on "Dialogs" properties, Directory security tab, Edit Authentication and access control, untick "Integrated Windows authentication", then restart IIS.</p> 
         <p>For Symantec Management Platform 7.1 "IIS 7"<br> -Open IIS manager, expand Altiris virtual directory &gt; Software Management &gt; Dialogs, on the right pane open "Authentication", right click on "Windows authentication" Disable, then restart IIS.</p> 
         <p>Before applying you might need to restart your Internet explorer, and be sure Iexplorer.exe process is not loaded before restarting.</p> 
         <p>NB: Basic authentication transmits user names and passwords across the network in an unencrypted form. You can use SSL encryption in combination with Basic authentication to help secure user account information transmitted across the Internet or a corporate network.</p> 
         <p>&nbsp;</p> 
         <br> 
         <p><b>Applies To</b><br></p> 
         <p>Symantec Management Platform 7.0 / 7.1</p>  
        </div>
