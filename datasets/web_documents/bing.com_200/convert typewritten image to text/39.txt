             <div id="ctl00_description" class="summary">
              Parsing PDF files in .NET using PDFBox and IKVM.NET (managed code).
             </div>
             <span id="ctl00_thumbnailUrl" class="date" content="https://www.codeproject.com/script/Articles/Images/article100x80.png"></span> 
            </div> 
            <div id="contentdiv" class="text"> 
             <ul class="download"> 
              <li><a href="/KB/string/pdf2text/Pdf2Text.Small.20150420.zip">Download source files - 82 kB</a> [codeproject.com]</li> 
              <li><a href="http://www.squarepdf.net/how-to-convert-pdf-to-text-in-net">Download full project including all dependencies</a> [squarepdf.net]</li> 
             </ul> 
             <h2>Update</h2> 
             <p><em>April 20, 2015:</em>&nbsp;The article and the Visual Studio project are updated and work with the latest PDFBox version (1.8.9). It's also possible to download the&nbsp;<a href="http://www.squarepdf.net/how-to-convert-pdf-to-text-in-net-sample-project">project with all dependencies</a>&nbsp;(resolving the dependencies proved to be a bit tricky).</p> 
             <p><em>February 27, 2014:</em> This article originally described parsing PDF files using PDFBox. It has been extended to include samples for IFilter and iTextSharp.</p> 
             <p><span style="color: rgb(255, 153, 0); font-size: 30px">How to Parse PDF Files</span></p> 
             <p>There are several main methods for extracting text from PDF files in .NET:</p> 
             <ul> 
              <li>Microsoft IFilter interface and Adobe IFilter implementation.</li> 
              <li>iTextSharp</li> 
              <li>PDFBox</li> 
             </ul> 
             <p>None of these PDF parsing solutions is perfect. We will discuss all these methods below.</p> 
             <h2>1. Parsing PDF using Adobe PDF IFilter</h2> 
             <p>In order to parse PDF files using IFilter interface you need the following:</p> 
             <ul> 
              <li>Windows 2000 or later</li> 
              <li>Adobe Acrobat or Reader 7.0.5+ (or the standalone <a href="http://www.adobe.com/support/downloads/detail.jsp?ftpID=2611" target="_blank">Adobe PDF IFilter</a> [adobe.com])</li> 
              <li><a href="http://www.dotlucene.net/78726/how-to-extract-text-from-microsoft-office-documents-doc-xls">IFilter COM wrapper class</a> [dotlucene.net]</li> 
             </ul> 
             <p>Sample code:</p> 
             <pre><span class="code-keyword">using</span> IFilter;

<span class="code-comment">//</span><span class="code-comment"> ...</span>

<span class="code-keyword">public</span> <span class="code-keyword">static</span> <span class="code-keyword">string</span> ExtractTextFromPdf(<span class="code-keyword">string</span> path) {
  <span class="code-keyword">return</span> DefaultParser.Extract(path); 
} </pre> 
             <p>Download a sample project:</p> 
             <ul> 
              <li><a href="http://www.squarepdf.net/parsing-pdf-files-using-ifilter-c-net">Parsing PDF Files using IFilter</a> [squarepdf.net]</li> 
             </ul> 
             <p>If you are using the PDF IFilter that comes with Adobe Acrobat Reader you will need to rename the process to "<span style="color: rgb(0, 0, 0); font-family: &quot;Open Sans&quot;, Tahoma, Geneva, Arial, sans-serif; font-size: 13px">filtdump.exe" otherwise the IFilter interface will return </span>E_NOTIMPL error code. See more at <a href="http://www.squarepdf.net/parsing-pdf-files-using-ifilter-c-net">Parsing PDF Files using IFilter</a> [squarepdf.net].</p> 
             <p>Disadvantages:</p> 
             <ol> 
              <li>Using unreliable COM interop that handles IFilter interface (and the combination of IFilter COM and Adobe PDF IFilter is especially troublesome).</li> 
              <li>A separate installation of Adobe IFilter on the target system. This can be painful if you need to distribute your indexing solution to someone else.</li> 
              <li>You have to use "filtdump.exe" file name for your application with the latest PDF IFilter implementation that comes with Acrobat Reader.</li> 
             </ol> 
             <h2>2. Parsing PDF using iTextSharp</h2> 
             <p><a href="http://sourceforge.net/projects/itextsharp/" target="_blank">iTextSharp</a> is a .NET port of <a href="http://itextpdf.com/" target="_blank">iText</a>, a PDF manipulation library for Java. It is primarily focused on creating and not reading PDFs but it supports extracting text from PDF as well.</p> 
             <p>Sample code:</p> 
             <pre><span class="code-keyword">using</span> iTextSharp.text.pdf;
<span class="code-keyword">using</span> iTextSharp.text.pdf.parser;

<span class="code-comment">//</span><span class="code-comment"> ...</span>
 
<span class="code-keyword">public</span> <span class="code-keyword">static</span> <span class="code-keyword">string</span> ExtractTextFromPdf(<span class="code-keyword">string</span> path)
{
  <span class="code-keyword">using</span> (PdfReader reader = <span class="code-keyword">new</span> PdfReader(path))
  {
    StringBuilder text = <span class="code-keyword">new</span> StringBuilder();

    <span class="code-keyword">for</span> (<span class="code-keyword">int</span> i = <span class="code-digit">1</span>; i <span class="code-keyword">&lt;</span>= reader.NumberOfPages; i++)
    {
        text.Append(PdfTextExtractor.GetTextFromPage(reader, i));
    }

    <span class="code-keyword">return</span> text.ToString();
  }
} </pre> 
             <p>Credit: <a href="http://www.codeproject.com/Messages/4767278/iTextSharp.aspx">Member 10364982</a></p>

             <pre><span class="code-keyword">public</span> <span class="code-keyword">static</span> <span class="code-keyword">string</span> ExtractTextFromPdf(<span class="code-keyword">string</span> path)
{
  ITextExtractionStrategy its = <span class="code-keyword">new</span> iTextSharp.text.pdf.parser.LocationTextExtractionStrategy();
  
  <span class="code-keyword">using</span> (PdfReader reader = <span class="code-keyword">new</span> PdfReader(path))
  {
      StringBuilder text = <span class="code-keyword">new</span> StringBuilder();

      <span class="code-keyword">for</span> (<span class="code-keyword">int</span> i = <span class="code-digit">1</span>; i <span class="code-keyword">&lt;</span>= reader.NumberOfPages; i++)
      {
          <span class="code-keyword">string</span> thePage = PdfTextExtractor.GetTextFromPage(reader, i, its);

          {
              text.AppendLine(theLine);
          }
      }
      <span class="code-keyword">return</span> text.ToString();
  }
}  </pre>

             <h2>3. Parsing PDF using PDFBox</h2> 
             <p><a href="http://pdfbox.apache.org/" target="_blank">PDFBox</a> is another Java PDF library. It is also ready to be used with the original Java Lucene (see <a href="http://pdfbox.apache.org/apidocs/org/apache/pdfbox/searchengine/lucene/LucenePDFDocument.html" target="_blank">LucenePDFDocument</a>).</p> 
             <p>Fortunately, there is a .NET version of PDFBox that is created using <a href="http://www.ikvm.net/" target="_blank">IKVM.NET</a> (just <a href="http://www.squarepdf.net/pdfbox-in-net/" target="_blank">download the PDFBox package</a>).</p> 
             <p>Using PDFBox in .NET requires adding references to:</p> 
             <ul> 
              <li>IKVM.OpenJDK.Core.dll</li> 
              <li>IKVM.OpenJDK.SwingAWT.dll</li> 
              <li>pdfbox-1.8.9.dll</li> 
             </ul> 
             <p>and copying the following files the <em>bin</em> directory:</p> 
             <ul> 
              <li>commons-logging.dll</li> 
              <li>fontbox-1.8.9.dll</li> 
              <li>IKVM.OpenJDK.Text.dll</li> 
              <li>IKVM.OpenJDK.Util.dll</li> 
              <li>IKVM.Runtime.dll</li> 
             </ul> 
             <p>Using the PDFBox to parse PDFs is fairly easy:</p> 
             <pre lang="cs"><span class="code-keyword">using</span> org.apache.pdfbox.pdmodel;
<span class="code-keyword">using</span> org.apache.pdfbox.util;

<span class="code-comment">//</span><span class="code-comment"> ...</span>

<span class="code-keyword">private</span> <span class="code-keyword">static</span> <span class="code-keyword">string</span> ExtractTextFromPdf(<span class="code-keyword">string</span> path)
{
  PDDocument doc = <span class="code-keyword">null</span>;
  <span class="code-keyword">try</span> {
    doc = PDDocument.load(path)
    PDFTextStripper stripper = <span class="code-keyword">new</span> PDFTextStripper();
    <span class="code-keyword">return</span> stripper.getText(doc);
  }
  <span class="code-keyword">finally</span> {
    <span class="code-keyword">if</span> (doc != <span class="code-keyword">null</span>) {
      doc.close();
    }
  }
}  </pre>
