      <p>Welcome to Clearbit’s API! You can use this API to access all our API endpoints, such as the <a href="#person-api">Person API</a> to look up email addresses, or the <a href="#company-api">Company API</a> to look up company information related to a domain name.</p> 
      <p>The API is organized around <a href="http://en.wikipedia.org/wiki/Representational_State_Transfer">REST</a>. All requests should be made over SSL. All request and response bodies, including errors, are encoded in JSON.</p> 
      <p>We also have some specific language bindings to make integration easier. You can switch the programming language of the examples with the tabs in the top right.</p> 
      <p>Currently we support the following official client bindings:</p> 
      <ul> 
       <li><a href="https://github.com/clearbit/clearbit-ruby">Ruby</a></li> 
       <li><a href="https://github.com/clearbit/clearbit-node">Node</a></li>

       <li><a href="https://github.com/clearbit/clearbit-go">Go</a></li> 
       <li><a href="https://github.com/clearbit/clearbit-java">Java</a></li> 
      </ul>

      <h1 id="authentication">Authentication</h1> 
      <blockquote class="shell">
        Authentication using HTTP basic auth. 
      </blockquote> 
      <pre class="highlight shell"><code>curl <span class="s1">'https://person.clearbit.com/v1/people/email/alex@alexmaccaw.com'</span> <span class="se">\</span>
        -u <span class="o">{</span>key<span class="o">}</span>:
</code></pre> 
      <pre class="highlight ruby"><code><span class="nb">require</span> <span class="s1">'clearbit'</span>

<span class="no">Clearbit</span><span class="p">.</span><span class="nf">key</span> <span class="o">=</span> <span class="s1">'{key}'</span>
</code></pre> 
      <pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">clearbit</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'clearbit'</span><span class="p">)(</span><span class="s1">'{key}'</span><span class="p">);</span>

      <blockquote class="shell">
        Alternatively pass a Bearer token in an Authorization header. 
      </blockquote> 
      <pre class="highlight shell"><code>curl <span class="s1">'https://person.clearbit.com/v1/people/email/alex@alexmaccaw.com'</span> <span class="se">\</span>
        -H <span class="s1">'Authorization: Bearer {key}'</span>
</code></pre> 
      <p>Authentication is done via the API key which you can find in your <a href="/profile">account settings</a>.</p> 
      <p>Requests are authenticated using <a href="http://en.wikipedia.org/wiki/Basic_access_authentication">HTTP Basic Auth</a>. Provide your API key as the basic auth username. You do not need to provide a password.</p> 
      <p>Alternatively you can also pass your API key as a bearer token in an <code class="prettyprint">Authorization</code> header.</p> 
      <p>You can see your account’s API keys, and roll them if necessary, in the <a href="/keys">dashboard</a>.</p>

      <h1 id="rate-limiting">Rate limiting</h1> 
      <blockquote> 
       <p>Check to see how many requests we have left:</p> 
      </blockquote> 
      <pre class="highlight plaintext"><code>$ curl -i https://person.clearbit.com

HTTP/1.1 200 OK
Date: Mon, 01 Jul 2014 21:20:00 GMT
Status: 200 OK
X-RateLimit-Limit: 5000
X-RateLimit-Remaining: 4050
X-RateLimit-Reset: 3300
</code></pre> 
      <p>You can make 600 requests per minute to each API, unless you are using Streaming. Check the returned HTTP headers of any API request to see your current rate limit status. If you’re running into this error or think you’ll need a higher rate limit, drop us a line at <a href="mailto:support@clearbit.com">support@clearbit.com</a>. The Streaming version of each API is limited to 5 concurrent connections rather than being limited by the number of requests per minute.</p> 
      <blockquote> 
       <p>Example rate limit error response.</p> 
      </blockquote> 
      <pre class="highlight plaintext"><code>HTTP/1.1 429 Too Many Requests
X-RateLimit-Limit: 500
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1402010983
Retry-After: 50

Content-Type: application/json
  {
    "error": {
      "type": "rate_limit",
      "message": "Rate limit exceeded. Limit is 600 requests per minute. Rate limit will be reset in 50 seconds."
    }
  }
</code></pre> 
      <table>
       <thead> 
        <tr> 
         <th>Header</th> 
         <th>Name</th> 
        </tr> 
       </thead>
       <tbody> 
        <tr> 
         <td><code class="prettyprint">X-RateLimit-Limit</code></td> 
         <td>The maximum number of requests that the consumer is permitted to make per minute.</td> 
        </tr> 
        <tr> 
         <td><code class="prettyprint">X-RateLimit-Remaining</code></td> 
         <td>The number of requests remaining in the current rate limit window.</td> 
        </tr> 
        <tr> 
         <td><code class="prettyprint">X-RateLimit-Reset</code></td> 
         <td>The time at which the current rate limit window resets in UTC epoch seconds.</td> 
        </tr> 
        <tr> 
         <td><code class="prettyprint">Retry-After</code></td> 
         <td>The number of seconds to wait until the rate limit window resets. Only sent when the rate limit is reached.</td> 
        </tr> 
       </tbody>
      </table> 
      <p>Once you go over the rate limit you will receive a <code class="prettyprint">rate_limit</code> error response.</p> 
      <h1 id="versioning">Versioning</h1> 
      <blockquote> 
       <p>You can pass a specific API version like so:</p> 
      </blockquote> 
      <pre class="highlight shell"><code>curl <span class="s1">'https://person.clearbit.com/v1/people/email/alex@clearbit.com'</span> <span class="se">\</span>
        -H <span class="s1">'API-Version: 2016-08-31'</span> <span class="se">\</span>

      <pre class="highlight ruby"><code><span class="no">Clearbit</span><span class="o">::</span><span class="no">Person</span><span class="p">.</span><span class="nf">version</span> <span class="o">=</span> <span class="s1">'2016-08-31'</span>

      <pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">clearbit</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'clearbit'</span><span class="p">)(</span><span class="s1">'{key}'</span><span class="p">);</span>

      <p>When we make backwards-incompatible changes to any of our APIs, we release new dated versions. Each API we provide has a separate version (listed below).</p> 
      <p>All requests will use your <a href="/keys">account API settings</a>, unless you override the API version by sending a <code class="prettyprint">API-Version</code> header identifying the API version to use.       <p>Any webhooks triggered will also use a request specified version before falling back to your account’s version.</p> 
      <p>You can visit <a href="/keys">your dashboard</a> to upgrade your API version. You can also see a full API version <a href="http://support.clearbit.com/article/110-api-version-change-logs">changelog here</a>.</p>

  <span class="k">when</span> <span class="s1">'person'</span>
    <span class="n">person</span> <span class="o">=</span> <span class="no">Person</span><span class="p">[</span><span class="n">webhook</span><span class="p">.</span><span class="nf">id</span><span class="p">]</span>
    <span class="n">person</span><span class="p">.</span><span class="nf">clearbit</span> <span class="o">=</span> <span class="n">webhook</span><span class="p">.</span><span class="nf">body</span>

  <span class="k">end</span>

  <span class="mi">200</span>
<span class="k">end</span>
</code></pre> 
      <p>Certain requests, such as looking up an email address, may have an asynchronous response (since there’s some complex background processing involved). For these requests you can either poll the endpoint, or give us a webhook URL you want to be called when the request has finished processing.</p> 
      <p>You can set a webhook URL endpoint in your <a href="/api">account settings</a> to be used with all requests. Alternatively, you can pass along a <code class="prettyprint">webhook_url</code> query parameter along with requests.</p> 
      <p>To link up your requests to webhooks you can pass a <code class="prettyprint">webhook_id</code> parameter when making these calls, containing a custom defined webhook identifier. We’ll pass this identifier back to you with the webhook request.</p> 
      <p>If you return anything other than a <code class="prettyprint">HTTP 200</code> status to the webhook POST then we’ll try to deliver the webhook up to 5 times with an exponential backoff.</p>

      <p>The only difference between making calls to the normal and streaming API is the subdomain - <code class="prettyprint">person-stream.clearbit.com</code> rather than <code class="prettyprint">person.clearbit.com</code>.</p>

      <p>If we can’t find any information associated with the email address, we’ll return a 404 status.</p>

      <p>While we do our best to always return accurate data, we’re only 
       <strike>
        lizards
       </strike> human and sometimes we make mistakes. This is especially true when the lookup is a fuzzy match rather than an exact one.</p> 
      <p>This endpoint allows you to let us know if a person doesn’t seem quite right. You can flag a person as incorrect and also optionally pass through a correct set of attributes.</p> 
      <p>We look into all the flagging we receive and incorporate fixes into future results.</p>

      <h2 id="queries">Queries</h2> 
      <blockquote> 
       <p>Queries are space separated key colon values. For example to find all companies with an Alexa rank between 1 &amp; 1000, 30 employees or more, and based in SF the query would look like this:</p> 
      </blockquote> 
      <pre class="highlight plaintext"><code>  alexa_rank:0~1000 employees:30~ city:"San Francisco"
</code></pre> 
      <blockquote> 
       <p>As you can see, you can escape values by using quotes. You can also include sub queries by using parenthesis:</p> 
      </blockquote> 
      <pre class="highlight plaintext"><code>  and:(tech:stripe raised:10000~)
</code></pre> 
      <blockquote class="ruby">
        Alternatively, rather than using a string search query, you can also pass an array of hashes: 
      </blockquote> 
      <pre class="highlight ruby"><code><span class="no">Clearbit</span><span class="o">::</span><span class="no">Discovery</span><span class="p">.</span><span class="nf">search</span><span class="p">(</span>

      <h1 id="risk-api">Risk API</h1> 
      <p>Our Risk API takes an email and IP and calculates an associated risk score. This is especially useful for figuring out whether incoming signups to your service are spam or legitimate, or whether a payment has a high chargeback risk.</p> 
      <p>We’ll indicate if a particular email is disposable, undeliverable, or looks fake. We’ll also show you whether an IP address is blacklisted or is using a proxy. Lastly we’ll give you an overall risk score which you can use to take action, such as making suspicious signups complete additional steps like filling in a captcha or confirming their phone number.</p> 
      <p>Our <a href="/integrate/risk">integration docs</a> will take you step by step through using the API, and we’ve also put together <a href="https://github.com/clearbit/risk-examples">some example apps</a> in Ruby, Node and Python. Otherwise the full API docs below will get you started.</p> 
      <h2 id="javascript-library">JavaScript library</h2> 
      <blockquote> 
       <p>Place the following JavaScript snippet on your checkout and/or signup pages:</p> 
      </blockquote> 
      <pre class="highlight plaintext"><code>&lt;script&gt;
  !function(){
    var script = document.createElement('script');
    script.async = true;
    script.src = 'https://risk.clearbit.com/v1/risk.js';
    var parent = document.getElementsByTagName('script')[0];
    parent.parentNode.insertBefore(script, parent);
  }();
&lt;/script&gt;
</code></pre> 
      <p>There are two components to the Risk API: a HTTP REST API and a JavaScript library. Using the JS library in conjunction with the REST API <strong>is required</strong> to generate risk scores.</p> 
      <p>Place the piece of JavaScript referenced to the right on your checkout or signup page (before you make a request to our REST API).</p> 
      <h2 id="calculate-scores">Calculate scores</h2> 
      <p>You can ping the Risk REST API with an email &amp; IP address to calculate a risk score. We’ll also return some information about the parameters, for example indicating if the email is disposable or the IP belongs to a proxy.</p> 
      <p>For testing, you can provide the loopback IP address <code class="prettyprint">127.0.0.1</code>. That’ll switch off some of the things we do that don’t make sense in a testing environment like velocity checks.</p>

       <div class="flash">

      <h1 id="reveal-api">Reveal API</h1> 
      <p>Our Reveal API takes an IP address, and returns the company associated with that IP. This is especially useful for de-anonymizing traffic on your website, analytics, and customizing landing pages for specific company verticals.</p> 
      <p>Powering Reveal is a unique dataset that combines both public and proprietary sources to give you an insight into the companies visiting your website whatever their size, even if they don’t have their own public ASN block.</p> 
      <div class="wrap"> 
       <p class="flash error"> The Reveal API is currently under limited release. <br> To get access to the API, please <a href="mailto:success@clearbit.com">contact us</a>. </p> 
      </div> 
      <h2 id="lookup-ip-addresses">Lookup IP addresses</h2>

      <p>This method of calling the API has the advantage of blocking page load, so is useful for customizing the page on the fly.</p>

      <h2 id="default-images">Default images</h2> 
      <p>Sometimes we can’t find a logo and image requests will return with a <code class="prettyprint">404</code> response code. Rather than having a broken image show up you can use the <code class="prettyprint">onerror</code> event to replace the image with a default of your choosing.</p> 
      <p>We’ve put together <a href="https://jsfiddle.net/maccman/2kxxgjk8/3/">an example</a> of how to do this with both jQuery and pure JavaScript.</p> 
      <h2 id="specifying-your-own-logo">Specifying your own logo</h2> 
      <blockquote> 
       <p>Specify your own logo with the <code class="prettyprint">og:logo</code> Open Graph tag.</p> 
      </blockquote> 
      <pre class="highlight html"><code><span class="nt">&lt;meta</span> <span class="na">property=</span><span class="s">"og:logo"</span> <span class="na">content=</span><span class="s">"http://yourdomain.com/logo.png"</span> <span class="nt">/&gt;</span>
</code></pre> 
      <blockquote> 
       <p>Or to specify different sizes add multiple <code class="prettyprint">meta</code> tags with a <code class="prettyprint">size</code> attribute.</p> 
      </blockquote> 
      <pre class="highlight html"><code><span class="nt">&lt;meta</span> <span class="na">property=</span><span class="s">"og:logo"</span> <span class="na">content=</span><span class="s">"/logo.png"</span> <span class="na">size=</span><span class="s">"150x150"</span> <span class="nt">/&gt;</span>
<span class="nt">&lt;meta</span> <span class="na">property=</span><span class="s">"og:logo"</span> <span class="na">content=</span><span class="s">"/logo.png"</span> <span class="na">size=</span><span class="s">"250x250"</span> <span class="nt">/&gt;</span>
<span class="nt">&lt;meta</span> <span class="na">property=</span><span class="s">"og:logo"</span> <span class="na">content=</span><span class="s">"/logo.png"</span> <span class="na">size=</span><span class="s">"500x500"</span> <span class="nt">/&gt;</span>
</code></pre> 
      <p>While we normally try and detect a company’s logo using signals like their Twitter or Facebook account, if you control a domain you can indicate which logo you’d like used.</p> 
      <p>Just add a <code class="prettyprint">og:logo</code> Open Graph meta tag to your index page under the root domain, and point its <code class="prettyprint">content</code> attribute to your logo of choice. We support <code class="prettyprint">png</code> and <code class="prettyprint">jpg</code> formats.</p> 
      <p><em>* Commercial use of the Logo API requires a link attribution on any page in which the logo is displayed - i.e “<a href="http://clearbit.com">Logos provided by Clearbit</a>”. See <a href="http://clearbit.com/terms">our terms</a> for more info.</em></p>
