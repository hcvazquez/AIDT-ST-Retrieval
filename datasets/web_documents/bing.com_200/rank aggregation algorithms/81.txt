In Java, when you think of a class having that has member of different type, then there an association can be formed. In a Java class, where there lies an entity reference, it becomes an aggression. Let us consider a case where we have used two classes, student class and another id class along with staffs and teacher; in order to maintain Student id, teachers’ id, and staff id, programmers do not need to use the same code again and again.                       <h2 style="box-sizing: border-box; color: #111111; font-family: Roboto, sans-serif; font-size: 26px; font-weight: 400; line-height: 1.3em; margin: 30px 0px 10px; text-shadow: rgba(0, 0, 0, 0.129412) 1px 1px 2px; user-select: text !important;"> What is Method Overriding in Java?</h2> 
                      <div style="box-sizing: border-box; color: #111111; font-family: Roboto, sans-serif; font-size: 17px; line-height: 1.7em; margin-bottom: 10px; user-select: text !important;">
                        Declaring a method in subclass which already exists there in parent class is known as method overriding. When a class is inheriting a method from a super class of its own, then there is a option of overriding the method provided it is not declared as final. The advantage of using overriding is ability to classify a behavior that’s specific to the child class and the child class can implement a parent class method based on its necessity. There are certain rules that a programmer should follow in order to implement over riding. These are:
                      </div> 
                      <ul style="box-sizing: border-box; color: #111111; font-family: Roboto, sans-serif; font-size: 17px; margin-bottom: 10px; margin-left: 20px; margin-top: 0px; padding-left: 10px; user-select: text !important;"> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">In java, a method can only be written in child class and not in same class.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">Argument list should be exactly the same as that of the overridden method of that class.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">Instance methods can also be overridden if they are inherited by the child class.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">A constructor cannot be overridden.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">Final – declared methods cannot be overridden.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">Any method that is static cannot be used to override.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">The return type must have to be the same or a subtype of the return type declared in the original overridden method in the parent class.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">If a method cannot be inherited then it cannot be overridden.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">A child class within the same package as the instance’s parent class can override any parent class method that is not declared private or final.</li> 
                       <li style="box-sizing: border-box; list-style-type: none; margin-bottom: 10px; overflow: hidden; user-select: text !important;">A child class in a different package can only override the non-final methods declared as public or protected.</li> 
                      </ul>
