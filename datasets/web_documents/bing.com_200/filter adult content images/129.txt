           <h3 id="N10054" class="ibm-h3">Purpose</h3>
           <p>This guide demonstrates a method of using transformer's relative time categories within Report Studio Prompts. The guide shows how to create prompts that default to a relative time category value whilst also allowing the user to use any other time category. The methods shown do not use any Java scripts.</p>
           <h3 id="N1005A" class="ibm-h3">Applicability</h3>
           <p>The guide is applicable to IBM Cognos Report Studio versions 8.4 and above.</p>
           <h3 id="N10060" class="ibm-h3">Exclusions and exemptions</h3>
           <p>This technique is only suitable for OLAP sources that have relative time categories.</p>
           <h2 id="N10066" class="ibm-h2">Scenario</h2>
           <p>Users are able to use Relative time categories when creating self authored reports with Analysis Studio or Query Studio. Professional Report authors often want to provide similar functionality in reports that they create within Report Studio, giving users the ability to run a report for any selected time category or for a relative time category.</p>
           <p>By providing access to relative time categories users have a better experience using professionally authored reports:</p>
           <ul class="ibm-bullet-list">
            <li>When scheduling reports.<br> For example prompts answers do not have to be constantly changed to take into account a month change.</li>
            <li>When running reports.<br> Prompts can be set to default to the relative time category – for example defaulting to the current month.</li>
           </ul>
           <p>The techniques demonstrated by this paper show how to provide this functionality without the use of Java script. The techniques will use a typical powercube that contains a current month relative time category. Whilst the technique demonstrates current month the technique will work for any relative time category or for multiple relative time categories within the same prompt.</p>
           <p>Two techniques are shown. The first is a simple report where the relative time is either used to filter or slice the report. The second is where the relative time category is used with additional relative time functions within the report.</p>
           <h2 id="N1007B" class="ibm-h2">Using Relative Time Categories to Slice or Filter</h2>
           <p>This section shows how to provide a prompt that will allow the user to filter on the current month or any other month from the time dimension.</p>
           <p>Throughout this example we will use the Great Outdoors Sales (cube) sample package which is based on the Great Outdoor Sales cube.</p>

           <p>We now need to slice our main data query by the chosen month value.</p>
           <p>Within Query 1 add a Slicer Member Set to the slicer pane and the expression editor will open. Normally you would use a slicer definition similar to:</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">[great_outdoors_sales_en].[Years].[Years].[Month]-&gt; ?ChosenMonth?</pre></span>
           <p>However we are not going to use this syntax. This syntax assume we know the level that we will want to filter. We want our report to be able to filter on two different levels. Whilst for normal months we want to filter on the level [great_outdoors_sales_en].[Years].[Years].[Month], for the relative time filter we will want to filter on the level [great_outdoors_sales_en].[Years].[Current Month].[Current Month]. Clearly the conventional filter syntax does not allow this.</p>
           <p>Instead we will use a macro. In the Expression editor type:</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">#prompt( 'ChosenMonth', 'MUN', '[great_outdoors_sales_en].[Years].
[Current Month].[Current Month]-&gt;:[PC].[@MEMBER].[Current Month]' ) #</pre></span>

            <li>ChosenMonth is the name of the parameter.</li>
            <li>MUN is the datatype of the prompt.</li>
            <li>[great_outdoors_sales_en].[Years].[Current Month].[Current Month]-&gt;:[PC].[@MEMBER].[Current Month] is the MUN – member unique name – of the default value of the prompt. For our report this is the MUN of the Current Month relative time category.</li>
           </ul>

           <h2 id="N1015F" class="ibm-h2">Prompting for a Relative Time Member or Ordinary Member and Using Relative Time functions</h2>
           <p>The previously demonstrated technique works when the report author want to either slice or filter on a month. The technique will not work when the report author then needs to perform additional relative time functions on the chosen month.</p>
           <p>For example let us consider the following crosstab report with Revenue as rows and both a chosen month and previous month on the columns.</p>
           <p>Here the user wants to choose a month and show the previous month. This is a typical month on month comparison report and is a common requirement. If we repeat the first technique here we will find that it will not work. Why is this? In order to calculate the 'PreviousMonth' the PrevMember function is used:</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">prevMember( [ChosenMonth] )</pre></span>
           <p>Whilst this function will work fine if the user has chosen a prompt answer from the level member [great_outdoors_sales_en].[Years].[Years].[Month] the function will not work if the user has chosen the member [great_outdoors_sales_en].[Years].[Current Month].[Current Month]-&gt;:[PC].[@MEMBER].[Current Month]. The relative time member has no previous member so the function will not return the expected output.</p>
           <p>Creating this type of report with a relative month prompt is a little more tricky.</p>
           <p>The first problem to tackle is how to turn the Current Month member into a member of the [great_outdoors_sales_en].[Years].[Years].[Month] level so that relative time functions will work.</p>
           <p>If we examine the time dimension we can see that the Current Month Member has a child of the month. In Great Outdoors Sales (cube) package we can see that the Current Month member has a child of 2007/Jul. Using the dimensional function FirstChild on the Current Month member will return us the 2007/Jul member.</p>
           <p> However this is not the same member as the 2007/Jul found in the Months level. Let us compare the two MUNs to show this:</p>
           <p>First 2007/Jul from the Current Month hierarchy:</p>
           <p>[great_outdoors_sales_en].[Years].[CurrentMonth].[Month]-&gt;:[PC].[@MEMBER].[20070701-20070731]</p>
           <p>and now the 2007/Jul from the Years hierarchy:</p>
           <p>[great_outdoors_sales_en].[Years].[Years].[Month]-&gt;:[PC].[@MEMBER].[20070701-20070731]</p>
           <p>We can also see in the time dimension that the 2007/Jul found underneath Current Month has no siblings so the PrevMember function would not return the desired result.</p>
           <p>Handily there is a dimension function that allows us to easily return the corresponding member of a member in a different hierarchy – this is the linkMember function.</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">linkmember(firstchild([great_outdoors_sales_en].[Years].
[Current Month].[Current Month]-&gt;:[PC].[@MEMBER].[Current Month] ) ,
 [great_outdoors_sales_en].[Years].[Years].[Month] )</pre></span>
           <p>Linking the PrevMember and LinkMember functions together we can turn the relative Current Month member into a real month and then perform our relative time functions.</p>
           <p>From the first example our prompt either returns the Current Month member or a member from the months level. The function we have created above will not work on a member from the month level due to the FirstChild.</p>
           <p>These functions will only work on the Current Month member. If we try and perform these same functions on any other value from the prompt- that is values from the Month level – then we will not get the expected results due to the FirstLevel function.</p>
           <p>We will use an If-then-else to determine if our user has chosen Current Month. If they have we will invoke our linkMember based function. If they haven't then we will simply use the month our user has chosen. The If statement will look similar to:</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">if (chosenMonth = 'Current Month')
then ( linkmember based function)
else (chosenMonth)</pre></span>
           <p> This If statement does have an issue though. Both results from an If statement must come from the same hierarchy. At first glance it may appear that this is true for this If statement. It is not. If the user chooses Current Month the link member is a member from the Years hierarchy whilst the chosenMonth is a member from the Current Month hierarchy. We resolve this issue by having the prompt not return a member!</p>
           <p>Let us start putting all these pieces together into an example report.</p>
           <ol>
            <li>Create a new crosstab report within Report Studio based on the Great Outdoors Sales (cube) package. We will build our prompt first.</li>
            <li>Delete the Block containing the title text from the Page Header.</li>
            <li>Drag a Value Prompt from the Insertable Objects toolbox into the page header. The prompt wizard for the Value prompt will start. Create a new parameter called 'ChosenMonthEx2'. <h5 id="N1019F" class="ibm-h5">Figure 12: Prompt Wizard: Create a new parameter</h5><img src="page586_images/page586_figure12.jpg" class="ibm-downsize" alt="Figure 12: Prompt Wizard: Create a new parameter" height="182" width="221"><br> Press <strong>Next</strong>. </li>
            <li>Untick '<strong>Create a paramterized filter</strong>' and click <strong>next</strong>.</li>
            <li>Tick '<strong>Create a new query</strong>'. Call the query 'Prompt Month Ex2'. <h5 id="N101B9" class="ibm-h5">Figure 13: Prompt Wizard: Create new query</h5><img src="page586_images/page586_figure13.jpg" class="ibm-downsize" alt="Figure 13: Prompt Wizard: Create new query" height="157" width="273"><br> Click <strong>Finish</strong>. We will now refine the prompt query further.</li>
            <li>From the Query Explorer access the Prompt Month Ex2 query. Add a data item to the query data items. For the expression definition use: <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">caption ([great_outdoors_sales_en].[Years].[Years].[Month] )</pre></span><br> Give the data item a name of Month Caption.</li>
            <li>Add another data item to the query data items. For the expression definition use: <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">roleValue ('_memberUniqueName', [great_outdoors_sales_en].[Years].[Years].[Month] )</pre></span><br> Give the data item the name Month MUN.</li>
           </ol>
           <p>Return to the report page to complete the definition of the Value prompt. Select the Value prompt.</p>
           <ol>
            <li>In the <strong>Use value</strong> select 'Month MUN'.</li>
            <li>In the <strong>Display value</strong> select 'Month Caption'.</li>
            <li>Set the <strong>Auto-Submit</strong> property to 'Yes'.</li>
            <li>Enter the <strong>Static Choices</strong>. <h5 id="N101EB" class="ibm-h5">Figure 14: Value prompt properties: static choices box</h5><img src="page586_images/page586_figure14.jpg" class="ibm-downsize" alt="Figure 14: Value prompt properties: static choices box" height="216" width="189"><br> Enter a static choice of 'Current'.</li>
            <li>Enter the Default Selection of '<strong>Current</strong>'. <h5 id="N101FA" class="ibm-h5">Figure 15: Value prompt properties: default selection box</h5><img src="page586_images/page586_figure15.jpg" class="ibm-downsize" alt="Figure 15: Value prompt properties: default selection box" height="146" width="278"></li>
           </ol>
           <p>We will now work in the Query rather than directly on the crosstab. Go into the query that the crosstab is based upon.</p>
           <ol>
            <li>Add the Revenue measure to the query.</li>
            <li>Add a data item to the query. The expression definition is: <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">linkmember(  firstchild(  [great_outdoors_sales_en].[Years].
[Current Month].[Current Month]-&gt;:[PC].[@MEMBER].[Current Month] ),
[great_outdoors_sales_en].[Years].[Years].[Month])</pre></span></li>
            <li>And the Data Item is to be called 'Current Month as Real Month'. This gives us a data item for the Current Month as a member from the Years hierarchy.</li>
            <li>Add another data item to the query. The data item name is 'ChosenMonth'. The expression definition for the data item is: <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">if ( ?ChosenMonthEx2?= 'Current'  ) 
 then (  [Current Month as Real Month] )
 else (   #substitute('Current',
 '[great_outdoors_sales_en].[Years].[Years].[Month]-&gt;:
     [PC].[@MEMBER].[20040101-20040131]', 
 prompt( 'ChosenMonthEx2','token','Current') ) # )</pre></span><br> This appears to be complicated statement and the Else part of the clause is worthy on an explanation. Again we are using the IBM Cognos macro syntax. The ChosenMonth prompt will return a string. <br> If the string is 'Current' then the substitute function will replace that string with <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">'[great_outdoors_sales_en].[Years].[Years].[Month]-&gt;:
 [PC].[@MEMBER].[20040101-20040131]'</pre></span><br> This is just a member from the Months level of the Year hierarchy. However we know that if the ChosenMonth is 'Current Month' the first part of the If clause is used so this member will never actually be used.<br> If the ChosenMonthEx2 prompt is not 'Current' then the macro will simply return the prompt – which happens to be the MUN of the chosen month.<br> This complicated macro ensures that no matter what the user chooses – either Current or a real month – a member from the Month level of the Year hierarchy is returned. We now have an If condition which always returns a member from the same hierarchy level. </li>
            <li>Add another data item to the query. The data item name is PreviousMonth. The expression definition for the data item is: <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">prevMember([ChosenMonth])</pre></span></li>
           </ol>
           <p>Return to the Crosstab page.</p>
           <p>From the Insertable data items tab:</p>
           <ol>
            <li>Drag the Revenue data item to the rows.</li>
            <li>Drag the ChosenMonth data item to the columns.</li>
            <li>Drag the PreviousMonth data item into the columns alongside the ChosenMonth.</li>
           </ol>
           <p>If we run the report now we can see that it has defaulted the prompt to the Current Month. The crosstab on the report is showing us the chosen month 2007/Jul and the Previous Month. If we select another month from the prompt the report still works.</p>
