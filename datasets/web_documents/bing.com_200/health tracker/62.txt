        <h2>Tracking Guide</h2>
        <p class="lead">This guide will teach you the two ways to track packages with EasyPost.</p>
        <p>There's two methods of tracking packages with EasyPost:</p>
        <ul>
         <li>Provide your existing tracking number and carrier.</li>
         <li>Purchase the shipping label with EasyPost, which includes a <strong><em>free</em></strong> <a href="/docs/api.html#trackers">Tracker</a>.</li>
        </ul>
        <p>Our tracking service uses webhooks to give you updates about your shipments. Refer to our <a href="/webhooks-guide">webhooks guide</a> to learn how we use webhooks to send you tracking updates.</p>
        <div class="bg-gray-50 p-5 mb-4">
         <p class="lead demi-bold">Before You Start</p>
         <ol class="circled-number-ol">
          <li><a href="/signup">Sign up</a> for an EasyPost account or <a href="/signin">log in</a> to an existing account.</li>
          <li><a href="/account/webhooks-and-events">Setup your webhook URLs</a> for Test Mode and Production Mode.</li>
          <li>Grab one of our <a href="/docs/libraries">official client libraries</a>.</li>
          <li> Choose the "Step 1" that is applicable to you:
           <ul>
            <li><a href="#step1a">I have a tracking number.</a></li>
            <li><a href="#step1b">I want to buy a shipping label with a tracker.</a></li>
           </ul></li>
         </ol>
         <p class="mb-0 mt-3">If you haven't run through our <a href="/getting-started">Getting Started Guide</a>, definitely do that before moving on to this one.</p>

        <h4 id="step1a">Step 1: Create a Tracker using your tracking code &amp; carrier</h4>
        <p>To track a shipment not created with EasyPost, you simply need to create a Tracker Object. You need to pass the “tracking_code" and “carrier” to the API. The “carrier” attribute is optional, though it is best to pass it if you have it available. When the “carrier” attribute is not passed, we will auto-detect the carrier for you. If we are unable to match the tracking code to a specific carrier, we will return an error.</p>
        <p> Here is an example of how to make a tracker object:</p>

        <h4 id="step2">Step 2: Process Tracking Event Webhooks</h4>
        <p>After you purchase a shipping label or create a tracker, we will automatically start sending tracking update <a href="/docs/api#events">Events</a>. <a href="/docs/api#events">Event Objects</a> are sent to the webhook URLs you’ve configured. Updates will only be sent when there is a status change for the package (e.g. a package changes from "Out for Delivery" to "Delivered"). We check the status of each package more frequently once it is Out for Delivery.</p>
        <p>The Tracker Object statuses are listed in our <a href="/docs/api#trackers">API Documentation</a>.</p>
        <div class="alert alert-warning">
         <p>When you purchase a label in either Test or Production mode, you will immediately get a tracking event after purchase. Test and Production mode behave slightly differently:</p>
         <ul>
          <li>In Test Mode, you will get only one tracking event in one of the supported statuses (e.g. "in_transit", "delivered", etc.). After this update, you receive no more events in Test mode.</li>
          <li>In Production Mode, you will get your first of multiple tracking events for your package. The first tracking event is status "unknown". The package stays in "unknown" until it is scanned by the carrier.</li>
         </ul>
        </div>
        <p>You’ll know it is a tracking update event because the object "type" is "Event" and the "description" is "tracker.updated". The "result" value of Event will also contain a Tracker object that has the information about the current progress of the package. When building application logic, the “status” of Tracker object is most useful and reliable to use.</p>
        <p>The Tracker object also contains additional information from the carrier in “tracking_details” attribute. For example, along with “in_transit” updates you may receive information that it reached a particular location (eg “Processed through Sort Facility June 01 2013 4:53 pm BELL GARDENS CA 9020”). The “tracking_details” will be an array containing both details about the current status and all previous statuses. The oldest status is the first element of the array and we append newer statuses as they come in. Order is not perfectly reliable as carrier data is occasionally incorrect. We recommend using the “status” on the Tracker object for any key business logic.</p>
