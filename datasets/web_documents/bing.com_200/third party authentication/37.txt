              <h1>Context and Existing Code</h1> 
              <p>I am trying to "promisify" a third-party authentication library (auth0-js) written in JavaScript. Right now it uses callback functions and there are no plans to expose an <code>async/await</code> friendly API. So, I'm writing a wrapper, to avoid <a href="http://callbackhell.com/" rel="nofollow noreferrer">callback hell</a> in my own code.</p> 
              <p>Here's a callback factory function <code>cb</code> which is actively used for creating Promises.</p> 
              <pre><code>import * as a0 from "auth0-js";

export function cb&lt;TResult&gt;(
  resolve: (reason: TResult) =&gt; void,
  reject: (reason: a0.Auth0Error) =&gt; void,
): a0.Auth0Callback&lt;TResult&gt; {
  return (error, result) =&gt; error ? reject(error) : resolve(result);
}
</code></pre> 
              <p>And here's an example of a wrapper class.</p> 
              <pre><code>import * as a0 from "auth0-js";
import { cb } from "./cb";

export class Authentication {

  /**
   * Wraps the Auth0-js' `Authentication` object and exposes Promise-based methods.
   * @param wa Wrapped `Authentication` object.
   */
  constructor(private wa: a0.Authentication) { }

  public get dbConnection(): DbConnection {
    return new DBConnection(this.wa.dbConnection);
  }

  public buildAuthorizeUrl(options: any): string {
    return this.wa.buildAuthorizeUrl(options);
  }

  public loginWithDefaultDirectory(options: a0.DefaultDirectoryLoginOptions): Promise&lt;any&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.loginWithDefaultDirectory(options, cb(resolve, reject)));
  }

  public login(options: a0.DefaultLoginOptions): Promise&lt;any&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.login(options, cb(resolve, reject)));
  }

  public oauthToken(options: any): Promise&lt;any&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.oauthToken(options, cb(resolve, reject)));
  }

  public loginWithResourceOwner(options: a0.ResourceOwnerLoginOptions): Promise&lt;any&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.loginWithResourceOwner(options, cb(resolve, reject)));
  }

  public getSSOData(withActiveDirectories: boolean, options: a0.DelegationOptions): Promise&lt;any&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.getSSOData(withActiveDirectories, cb(resolve, reject)));
  }

  public userInfo(accessToken: string): Promise&lt;a0.Auth0UserProfile&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.userInfo(accessToken, cb(resolve, reject)));
  }

  public delegation(options: a0.DelegationOptions): Promise&lt;a0.Auth0DelegationToken&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.delegation(options, cb(resolve, reject)));
  }

  public getUserCountry(): Promise&lt;{ countryCode: string; }&gt; {
    return new Promise((resolve, reject) =&gt; this.wa.getUserCountry(cb&lt;{ countryCode: string; }&gt;(resolve, reject)));
  }
}
</code></pre> 
              <h1>Question</h1> 
              <p>While I'm always open to any constructive feedback, there are a few specific aspects I'll be particularly thankful for.</p> 
              <ol> 
               <li>I'm a C# developer and still know too little about JavaScript/TypeScript. <strong>If you know how to make this code look more <em>JS-idiomatic</em>, I'm all ears.</strong> Or I can paraphrase the question as "am I going the right/common route?"</li> 
               <li>You can see that there is a lot of structural repetition in the code that creates Promises from callbacks (e.g. <code>loginWithDefaultDirectory(...)</code>, <code>login(...)</code>, <code>oauthToken(...)</code> functions). Not nice. <strong>Is there a way to apply functional programming ideas here? I have a feeling that functional composition may get handy here, but I don't know how to apply it (a sign, I haven't grasp the concept yet)</strong></li> 
               <li><strong>Resolved</strong> 
                <strike>
                 It's also easy to notice that 
                 <code>buildAuthorizeUrl(...)</code> is a 
                 <em>pass-through</em>. No callbacks here. There must be an easier way to bind one object's function to another object's function, which I don't know how to do. Very similarly, 
                 <code>get dbConnection()</code> accessor is a supposed to be a 
                 <em>pass-through</em>.
                </strike></li> 
              </ol> 
              <h1>Update 1</h1> 
              <p>I figured the pass-through members can be coded very easily (one line per member):</p> 
              <pre><code> public buildAuthorizeUrl = this.wa.buildAuthorizeUrl;
</code></pre> 
              <p>So, point 3 is now out of question.</p>
