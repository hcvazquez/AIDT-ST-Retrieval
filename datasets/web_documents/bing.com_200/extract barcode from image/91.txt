            <h2 id="introduction">Introduction</h2> 
            <p class="tipbox"><strong>Universal Extractor Feedback</strong> <br>If you have any questions about, or find any problems with, Universal Extractor, please post your comments to the <a href="http://www.msfn.org/board/Universal-Extractor-f159.html">UniExtract subforum on the MSFN message board</a>.&nbsp; This has become the main point of discussion for this project.</p> 
            <p>Universal Extractor is a program designed to decompress and extract files from any type of archive or installer, such as ZIP or RAR files, self-extracting EXE files, application installers, etc.&nbsp; The full list of <a href="#formats">supported formats</a> can be found in the table below.&nbsp; It's able to support so many varied file formats by utilizing the many backend utilities listed in the <a href="#credits">credits</a> at the bottom of the page.</p> 
            <p>Please note that Universal Extractor is not intended to be a general purpose archiving program.&nbsp; It cannot (and never will) create archives, and therefore cannot fully replace archivers such as <a href="http://www.7-zip.org/">7-Zip</a> or <a href="http://www.rarlab.com/">WinRAR</a>.&nbsp; What it will do, however, is allow you to extract files from virtually any type of archive, regardless of source, file format, compression method, etc.</p> 
            <p>The original motivation behind this project was to create an easy, convenient way to extract files from various types of installation packages without the need to remember arcane command line switches or track down separate utilities to handle the unpacking.&nbsp; Over time, and with the encouragement of its many users and the fine folks over on the <a href="http://www.msfn.org/board/forum/159-universal-extractor/">MSFN forum</a>, it has evolved into a mature and very capable unarchiving utility that many, including myself, find indispensable.</p>

            <h3>Language Files</h3> 
            <p>As of version 1.4, Universal Extractor supports internationalization.&nbsp; If you're interested in translating Universal Extractor into another language, please download the source archive and follow the instructions in English.ini.&nbsp; Please send me your completed translation and I'll include it in the next version of Universal Extractor.&nbsp; Thanks to all of the contributors!</p>
            <p> </p>
            <p>To use these language files, simply download and copy into the <code>lang\</code> directory of your Universal Extractor installation.&nbsp; The new language will appear as an option the next time you run UniExtract.</p> 
            <p><em>Note:</em> The (alt) language files are alternate translations that are not included with the official release.&nbsp; Sometimes I get multiple submissions for a single language, and I like to give everyone credit, but I generally stick with the first one that's submitted out of fairness.&nbsp; Some of the alternate translations may be better than the included ones (or, they may not be), so feel free to give them a try.&nbsp; If you find that an alternate version is better than the included version, please let me know; I may switch them out for the next release.</p>

            <h2 id="usage">Installation and Usage </h2> 
            <p>To install UniExtract, download the installer and run it on your system.&nbsp; You will have a few options during installation:</p> 
            <p><strong>Select Components</strong> - These options specify which program components are installed to disk.&nbsp; These options are only really useful if you need to reduce the installation size.</p> 
            <ul> 
             <li><code>Documentation and licensing information</code> - This includes all files in the <code>docs\</code>subdirectory</li> 
             <li><code>Language files for internationalization support</code> - This includes all files in the <code>lang\</code> subdirectory</li> 
             <li><code>Support for ACE files</code> - Binaries and/or libraries required for ACE archives</li> 
             <li><code>Support for KGB files</code> - Binaries and/or libraries required for KGB archives</li> 
             <li><code>Support for Pea files</code> - Binaries and/or libraries required for Pea archives</li> 
             <li><code>Support for StuffIt files</code> - Binaries and/or libraries required for StuffIt (.sit) archives</li> 
            </ul> 
            <p><strong>Set Program Options</strong> - These options adjust the default Universal Extractor behavior.&nbsp; They can be changed after installation through the Preferences menu.</p> 
            <ul> 
             <li><code>Default Language</code> - specifies the language used by UniExtract</li> 
             <li><code>Debug File Location</code> - specifies the directory to which UniExtract should write its debug file</li> 
             <li><code>Append missing file extensions</code> - attempt to determine and append the correct extension for improperly named files using <a href="http://mark0.net/soft-trid-e.html">TrID</a></li> 
             <li><code>Maintain archive history</code> - maintain a list of the ten most recent files that were extracted, as well as the destination directories</li> 
             <li><code>Remove duplicate files</code> - attempt to detect and remove duplicate that may have been created during extraction</li> 
             <li><code>Remove temporary files</code> - automatically remove temporary files that may be created during extraction</li> 
             <li><code>Warn before executing files</code> - some .exe archives must actually be executed in order to perform the extraction.&nbsp; Since this can be a security risk (executing unknown/untrusted code is always a bad idea), UniExtract will display a warning before extracting these types of files.</li> 
            </ul>

            <p>For manual installation with the binary archive, simply extract the files to the destination folder you would like.&nbsp; By default, this puts UniExtract in "portable mode".&nbsp; This means that the <code>globalprefs</code> option is enabled (see <a href="/files/software/uniextract_changelog.txt">changelog</a> for details) and the <code>history</code> option is disabled (see Program Options above).&nbsp; The end result is UniExtract can be easily run from portable devices like USB flash drives without leaving any trace of use.&nbsp; Of course, it'll also work just as well if copied to a real hard drive, and all preferences can be modified at any time by editing <code>UniExtract.ini</code>, or by running <code>UniExtract.exe /prefs</code>.</p> 
            <p>Once UniExtract is installed, usage is simple.&nbsp; The easiest way, if you used the installer and enabled context menu integration, is to right-click on any archive and select click one of the UniExtract options.&nbsp; <code>UniExtract <u>H</u>ere</code> will extract files to the current directory, <code>UniExtract to <u>S</u>ubdir</code> will extract files to a subdirectory named after the file, and <code>UniExtract <u>F</u>iles</code> will prompt you for the destination directory.</p> 
            <p>If you did not enable context menu integration, you may run the Universal Extractor start menu shortcut, or double-click directly on UniExtract.exe.&nbsp; This will launch the UniExtract GUI.&nbsp; Enter (or use the file browser to select) the file you wish to extract and the destination directory, then click OK.&nbsp; It is also possible to drag-and-drop the file you wish to extract onto the icon or main application window to easily extract the dropped file.</p> 
            <p>UniExtract can also be run from the command line.&nbsp; Run <code>UniExtract.exe /?</code> for detailed command line parameters.</p> 
            <p>You can use Add/Remove Programs to uninstall UniExtract, or just delete the directory if you used the binary archive.</p>

            <h2 id="technical">Technical Details</h2> 
            <p>When a file is passed to UniExtract, it begins by scanning the file with <a href="http://mark0.net/soft-trid-e.html">TrID</a>.&nbsp; If the file format matches one of the supported non-executable formats listed below, it immediately begins extraction from the archive.&nbsp; If it's an executable file, it calls <a href="http://www.peid.info/">PEiD</a> to analyze the file signature.&nbsp; If the signature indicates that it's a supported format, then it attempts to begin extraction.&nbsp; If it is not a recognized signature, it will try running it through 7-Zip and UnZip as default cases.&nbsp; If 7-Zip or UnZip recognizes it, it will be extracted; otherwise, UniExtract displays an error message and exits.</p> 
            <p><em>Note: Filetype scanning with TrID is new as of v1.5.&nbsp; Older versions relied on the file extension to identify the filetype.&nbsp; This behavior is still used if the TrID scan cannot determine the filetype.</em></p> 
            <p>If the filetype is supported, UniExtract begins by checking for a valid destination directory and recording the directory size.&nbsp; If necessary, a new directory will be created (with a directory size of zero).&nbsp; UniExtract then spawns a shell and calls the supporting binary to begin extracting files from the selected archive to the destination directory.&nbsp; After completion, UniExtract checks the size and timestamp of the subdirectory to determine if extraction was successful; if the size or timestamp is greater than the initial values, UniExtract assumes success and exits.&nbsp; If extraction fails, UniExtract notifies the user and leaves a log file (<code>uniextract.log</code>) that (should) contain any error messages.</p> 
            <p>The <code>uniextract.log</code> file is created during the extraction process by the tee program.&nbsp; Essentially, all output from the running binary is redirected to tee, which in turn writes the output to the log file as well as echo it to the screen to give the user current progress.&nbsp; Upon successful execution this file is deleted, but if an error is detected it is left alone for the user to review.&nbsp; <em>Note: As of v1.4, the location of uniextract.log can be changed to any user-specified directory.</em></p> 
            <p class="return"><a href="#introduction">Return to top</a></p>
