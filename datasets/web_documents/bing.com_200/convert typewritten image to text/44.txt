             <div class="post-text" itemprop="text"> 
              <p>I need to convert my LaTeX documents into PNG. The problem is, I also need the resulting image to be as short as possible (height-wise). I've tried <code>latex</code> followed by <code>dvipng</code>, but the result is always the size of a page. For instance, take a <code>.tex</code> file containing:</p> 
              <pre><code>\documentclass{article}  
\begin{document}  
Hello. This is a test.
\begin{equation}
  L = 2                                                                     
\end{equation}  
\end{document}
</code></pre> 
              <p>If I compile it with <code>latex</code>, and then run <code>dvipng</code>, I get a PNG file that's the size of a full page. What I need is for the PNG file to be only as tall as needed for everything to fit. So the image would end immediately after the equation. The image still needs to have full width (because of the equation numbering).</p> 
              <p>Is there a way to achieve that?</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>You can use the <code>standalone</code> class for this. It loads the <code>preview</code> package automatically to crop the resulting PDF to the content. This makes the usage of <code>pdfcrop</code> unnecessary.</p> 
              <p>Simply exchange the <code>article</code> class with <code>standalone</code>. (It uses <code>article</code> internally but another class can be specified using the <code>class</code> option.) Note that since v1.0 the default option has been changed from <code>preview</code> to <code>crop</code>. The latter is better for pictures etc. but doesn't support line breaks. Either select <code>preview</code> manually or use the <code>varwidth</code> option.</p> 
              <pre><code>\documentclass[preview]{standalone}
\begin{document}
Hello. This is a test.
\begin{equation}
L = 2
\end{equation}
\end{document}
</code></pre> 
              <p>There is a <code>border</code> class option which sets the border around the content (the default is 0.5bp). This option excepts either one (border for all sides), two (left/right, top/bottom) or four values (left, bottom, right, top).</p> 
              <p>To convert it to a PNG I recommend to use the <code>convert</code> command of <a href="http://www.imagemagick.org/">Image Magick</a>:</p> 
              <pre><code>pdflatex file
convert -density 300 file.pdf -quality 90 file.png
</code></pre> 
              <p>Here the density is 300 DPI which can be adapted to your needs. The quality setting selects the compression level and other things and 90 is AFAIK the optimum.</p> 
              <p>You can also select the DPI resolution for X and Y separately and also resize the resulting image, e.g.:</p> 
              <pre><code>convert -density 600x600 file.pdf -quality 90 -resize 1080x800 file.png
</code></pre> 
              <hr> 
              <h3>Update 2011/12/21:</h3> 
              <p>The new version 1.0 <code>standalone</code> now has the ability to call the above command line (and others) automatically, e.g.:</p> 
              <pre><code>\documentclass[convert={density=300,size=1080x800,outext=.png}]{standalone}
</code></pre> 
              <p>or simply (using default setting 300dpi, no resizing, PNG):</p> 
              <pre><code>\documentclass[convert]{standalone}
</code></pre> 
              <p>This needs the <code>-shell-escape</code> compiler option to allow the execution of the conversion program from within the LaTeX document.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>One thing that's easy to miss is page numbers. The page number restricts the height of the final image so it's best to leave it out. An easy way to do that is to use the <code>empty</code> page style.</p> 
              <p>What I do when doing images for this place is to have a document a bit like:</p> 
              <pre><code>\documentclass{article}
\thispagestyle{empty}
\begin{document}
Hello. This is a test.
\begin{equation}
L = 2
\end{equation}
\end{document}
</code></pre> 
              <p>Then run <code>pdflatex</code> on it to get a PDF; next run <code>pdfcrop</code> (comes with TeXLive) to make it as small as possible; finally convert it to PNG using the NetPBM library tools. (This is on a Unix machine.) So my workflow is:</p> 
              <pre><code>pdflatex document.tex
pdfcrop document.pdf
pdftoppm document-crop.pdf|pnmtopng &gt; document.png
</code></pre> 
              <p>et voila:</p> 
              <p><img src="https://i.stack.imgur.com/6bEPk.png" alt="small png"></p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>I had a few issues using the standalone package and ImageMagick. Running the </p> 
              <pre><code>pdflatex -shell-escape file.tex
</code></pre> 
              <p>command did not generate the png file. I fixed this in two steps: </p> 
              <ol> 
               <li>Renaming the "convert.exe" file to "imgconvert.exe" in the ImageMagick folder (C:\Program Files\ImageMagick-6.9.0-Q16).</li> 
               <li>Installing a newer version of Ghostview (direct link: <a href="http://sourceforge.net/projects/ghostscript/files/GPL%20Ghostscript/9.01/gs901w32.exe/download" rel="nofollow noreferrer">http://sourceforge.net/projects/ghostscript/files/GPL%20Ghostscript/9.01/gs901w32.exe/download</a>)</li> 
              </ol> 
              <p>Before I did this I got errors saying that no conversion had happened:</p> 
              <pre><code>Class standalone Warning: Conversion unsuccessful!
(standalone)              There might be something wrong with your
(standalone)              conversation software or the file permissions!
</code></pre> 
              <p>Earlier in the compilation was a line saying imgconvert was not a recognized command so that suggests that the standalone package expects the convert.exe file to be renamed (found that solution here:<a href="https://tex.stackexchange.com/questions/93891/conversion-problem-using-standalone-and-imagemagick">Conversion problem using standalone and imagemagick</a>).</p> 
              <p>Renaming the file connected the standalone package and ImageMagick, but the Ghostview file "gswin32c.exe" stopped working every time I attempted a conversion. Installing a newer version solved that problem, as described here: <a href="http://freewarestyle.blogspot.com/2011/04/gswin32cexe-gswin32cexe-error-cutepdf.html" rel="nofollow noreferrer">http://freewarestyle.blogspot.com/2011/04/gswin32cexe-gswin32cexe-error-cutepdf.html</a></p> 
              <p>Just thought this could be useful for somebody else.</p> 
              <p><strong>Edit</strong> If I may, I add that it might not be sufficient/necessary for the first step to rename the "convert.exe" file, you alternatively can add the directory within your editor search path where your ImageMagick is installed.</p> 
              <p>For example for TexStudio users, this is done by clicking on "Options -&gt; Configure TexStudio" then click on tab "Build" and add the directory to the "Commands($PATH)".</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>i use this technique to prepare examples for posting to tex.sx, but it could presumably be used for any situation in which a small <code>.png</code> file is needed.</p> 
              <ul> 
               <li>create a dedicated input file, limiting the input to exactly what should be in the example. be sure to specify <code>\thispagestyle{empty}</code> (<em>after</em> <code>\begin{document}</code>). <br> the input file i use is just a cut-down version of whatever document the example is extracted from; i haven't had good luck with <code>standalone</code> or other "minimal" classes.</li> 
               <li>run pdflatex.</li> 
               <li><code>pdfcrop -margin 3 in-name.pdf out-name.pdf</code></li> 
               <li><code>convert -trim -density 200 out-name.pdf out-name.png</code> <br> (this requires imagemagick).</li> 
              </ul> 
              <p>this requires two "external" programs, and can probably be scripted.</p>

             <div class="post-text" itemprop="text"> 
              <p>The answers here are pretty great <em>if you just have a <strong>single</strong> equation</em> you'd like to export. What if you have a list of equations in some file that you'd like to export?</p> 
              <pre><code>% equation 1 %
\[ y = mx + b \]

% equation 2 %
e = \lim_{n \rightarrow \infty} \left ( 1 + \frac{1}{n} \right )^n
</code></pre> 
              <p>You'd like to automatically export each of these equations to a separate image file. I would suggest one of the following two approaches.</p> 
              <p><strong>Approach #1 (not so good, but easy)</strong> - Use <a href="http://matplotlib.org/" rel="nofollow noreferrer">matplotlib</a>, which can render basic equations using the <a href="http://matplotlib.org/api/pyplot_api.html#matplotlib.pyplot.text" rel="nofollow noreferrer"><code>plt.text()</code></a> command. Here is a snippet of code that handles some nice formatting (centering and padding for the equation: <a href="https://gist.github.com/ahwillia/c7e54f875913ebc3de3852e9f51ccc69" rel="nofollow noreferrer">https://gist.github.com/ahwillia/c7e54f875913ebc3de3852e9f51ccc69</a></p> 
              <p>The main formatting trick is to use the <code>get_window_extent()</code> function as suggested in this <a href="https://stackoverflow.com/questions/5320205/matplotlib-text-dimensions">StackOverflow discussion</a>, and then set the size of the figure appropriately.</p> 
              <p><strong>Approach #2 (more powerful)</strong> - Use a python script to make a bunch of temporary <code>.tex</code> files following the various templates suggested in this thread. And the use the <a href="https://docs.python.org/2/library/subprocess.html" rel="nofollow noreferrer"><code>subprocess</code></a> module to compile the latex, rename to an appropriate filename and delete <code>.log</code> and <code>.aux</code> files.</p> 
              <p>I have a basic script that works for my needs here: <a href="https://gist.github.com/ahwillia/ce9a842f122757518c65d0bd545f28c1#file-equations-tex-L2" rel="nofollow noreferrer">https://gist.github.com/ahwillia/ce9a842f122757518c65d0bd545f28c1#file-equations-tex-L2</a></p> 
              <p>There is also a more polished package called <a href="https://github.com/johnjosephhorton/texscrap" rel="nofollow noreferrer">texscrap</a>, though I haven't tried it extensively yet. It accepts a text file of one-liner equations as input, but I'm not sure if there are edge cases it fails on. Seem very promising though.</p>

             <div class="post-text" itemprop="text"> 
              <p>Without using <code>standalone</code>, I used to transform my <code>.pdf</code> output in a <code>.png</code> file by an online converter (I don't give the link because I don't know if it is reliable, however, you can easily find it googling around) but since I installed <code>arara</code> package I've done this conversion more quickly.</p> 
              <p>I have Windows 10 but I think this works with every operating system, you only need to have ImageMagick (it's free) and <code>arara</code> package installed.</p> 
              <p>Modifying <a href="https://tex.stackexchange.com/a/117972/101651">the <code>animate</code> rule</a> provided by cmhughes, which creates a <code>.gif</code> file, I have created a <code>convert.yaml</code> rule and put in <code>...\MiKTeX 2.9\scripts\arara\rules</code> (the path varies according to which operating system and TeX distribution you are using, I have MiKTeX):</p> 
              <pre><code>!config
# Convert .pdf to any format file allowed by ImageMagick convert command (the default is png)
# author: CarLaTeX
# last edited by: CarLaTeX, Dicember 26th 2016
# requires arara 3.0+
#
# Sample usage: 
# - these both create a .png file
# % arara: convert
# % arara: convert: {format: png}
#
# - this creates a .gif file with red background
# % arara: convert: {format: gif, background: red}
#
# - this creates a .png file with a trimmed image 
#   (use the parameter "otheroptions" to add any option not already explicitly considered by the rule, 
#   that is any option different from -background, -alpha, -density and -quality}
# % arara: convert: {otheroptions: -trim +repage}
#
#
# This rule is really just a shortcut for commands like the following:
#
#  convert -density 300 myfile.pdf myfile.png
#
# which will output myfile.png
#
identifier: convert
name: convert
commands: 
- &lt;arara&gt; @{ isWindows( "cmd /c convert", "convert" ) } -background @{background} -alpha @{alpha} -density @{density} @{otheroptions} "@{ getBasename(file) }.pdf" -quality @{quality} "@{ getBasename(file) }.@{format}"
arguments:
- identifier: density
  flag: &lt;arara&gt; @{parameters.density}
  default: 150
- identifier: otheroptions
  flag: &lt;arara&gt; @{parameters.otheroptions}
- identifier: quality
  flag: &lt;arara&gt; @{parameters.quality}
  default: 100
- identifier: background
  flag: &lt;arara&gt; @{parameters.background}
  default: white
- identifier: alpha
  flag: &lt;arara&gt; @{parameters.alpha}
  default: remove
- identifier: format
  flag: &lt;arara&gt; @{parameters.format}
  default: png
</code></pre> 
              <p>(Take into account that I'm not an expert, maybe this could be done better).</p> 
              <p>Of course, you have to do this once for all, then it is sufficient to put:</p> 
              <pre><code>% arara: pdflatex               (or any other command you are using to compile)
% arara: ...                    (possible other commands)
% arara: convert
</code></pre> 
              <p>at the beginning of your document and compile it with <code>arara</code>.</p> 
              <p>For example, if you have this <code>myfile.tex</code>:</p> 
              <pre><code>% arara: pdflatex
% arara: convert
\documentclass{article}
\begin{document}
    Quack!
\end{document}
</code></pre> 
              <p>and you run <code>arara myfile.tex</code>, you will get a <code>myfile.pdf</code> and a <code>myfile.png</code>.</p> 
              <p>For an analogous solution using Ghostscript, see <a href="https://tex.stackexchange.com/a/339388/101651">this answer of mine</a>.</p>

              <p>If you only want to use the PNG image to include it into MS Word you can choose among several programs that will greatly simplify your task. By hand you have to go through the whole process every time you modify the LaTeX part of your document. </p> 
              <p>As far as I know there is: </p> 
              <ul> 
               <li>TexPoint <a href="http://texpoint.necula.org/" rel="nofollow">link</a></li> 
               <li>Tex4PPT <a href="http://users.ecs.soton.ac.uk/srg/softwaretools/presentation/TeX4PPT/" rel="nofollow">link</a></li> 
               <li>Latex-to-Word from grindeq <a href="http://www.grindeq.com/index.php?p=latex2word" rel="nofollow">link</a> </li> 
               <li>MyTexPoint <a href="http://thd.pnpi.spb.ru/~gromov/mytexpoint.html" rel="nofollow">link</a></li> 
               <li>IguanaTeX <a href="http://www.technion.ac.il/~zvikabh/software/iguanatex/" rel="nofollow">link</a></li> 
              </ul> 
              <p>They all have different properties, some are freeware, shareware or opensource but it improves your workflow quite a bit if you can simply change the figure in Word directly and use LaTeX in the background for nice typesetting. Internally they all use some VBA or .NET code to connect the LaTeX, dvips, convert toolchain to MS Word. </p>

             <div class="post-text" itemprop="text"> 
              <p>Another approach with <code>preview</code> package directly.</p> 
              <pre><code>% host.tex
\documentclass[preview,border=12pt,12pt]{standalone}
\usepackage{filecontents}

\begin{filecontents*}{main.tex}
% this is main.tex
\documentclass[12pt]{article}
\usepackage[active,tightpage]{preview}
\PreviewBorder=12pt\relax
\begin{document}
\preview
Hello. This is a test.
\begin{equation}
L = 2
\end{equation}
\endpreview
\end{document}
\end{filecontents*}

\usepackage{graphicx}
\immediate\write18{pdflatex main.tex}
\immediate\write18{convert -density 100  main.pdf main.png}
\begin{document}
The following  is a PNG image.\newline
\fbox{\includegraphics{main.png}}
\end{document}
</code></pre> 
              <p><img src="https://i.stack.imgur.com/Kk2YO.png" alt="enter image description here"></p> 
              <h2>Notes</h2> 
              <ul> 
               <li>ImageMagick must be installed on your machine.</li> 
               <li>Compile the <code>host.tex</code> with <code>pdflatex -shell-escape host.tex</code>.</li> 
              </ul>
