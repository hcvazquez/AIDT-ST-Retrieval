                     <div class="lia-message-body-content"> 
                      <p>We recently released a <a href="http://pages.alteryx.com/Microsoft-Power-BI-Kit-W.html?sc=Web%20Direct&amp;scd=resources" target="_blank" rel="nofollow noopener noreferrer">Microsoft Kit</a> that included a <a href="https://gallery.alteryx.com/#!app/AzureMLTextAnalytics/561687b4aa690a14c493c998" target="_blank" rel="nofollow noopener noreferrer">text analytics tool</a>. This tool uses a Cortana Analytics Gallery text analytics API to provide sentiment analysis and key phrase extraction. The tool has received positive <a href="/t5/Advanced-Analytics/Natural-Language-Toolkit-and-Naive-Bayes/m-p/6230#M321" target="_blank">feedback</a>&nbsp;but is limited to 10,000 records per month before you have to pay a monthly fee.&nbsp;Given this backdrop,&nbsp;I wanted to compare the Microsoft sentiment analysis capability to a couple open source algorithms available.</p> 
                      <p>&nbsp;</p> 
                      <h1 id="toc-hId-474168458">The Sentiment Tool</h1> 
                      <p>&nbsp;</p> 
                      <p>The first open source package I identified to try out was the R package "sentiment". The package has long been archived on CRAN but is still available for <a href="https://cran.r-project.org/src/contrib/Archive/sentiment/" target="_blank" rel="nofollow noopener noreferrer">download</a>. It was not too difficult to leverage this package inside of Alteryx - a few lines of code in the R tool was all that was needed.</p> 
                      <p>&nbsp;</p> 
                      <p>The second package came to my attention via a Microsoft blog <a href="http://blogs.technet.com/b/machinelearning/archive/2015/04/08/introducing-text-analytics-in-the-azure-ml-marketplace.aspx" target="_blank" rel="nofollow noopener noreferrer">post</a>. The <a href="http://stanfordnlp.github.io/CoreNLP/" target="_blank" rel="nofollow noopener noreferrer">Stanford CoreNLP</a>&nbsp;project is an expansive "<span>set of natural language analysis tools", even though (for now) I'm only interested in sentiment analysis. I was able to utilize this package in Alteryx via the <a href="/t5/Engine-Works-Blog/I-Am-The-Developer-Toolset-And-So-Can-You/ba-p/11577" target="_blank">Run Command</a> tool. Whereas the "sentiment" package gives a total score for an entire block of text, the Stanford package parses each sentence and gives a separate score for each. For the sake of this tool, I averaged these scores together to give a single score for the entire text.</span></p> 
                      <p>&nbsp;</p> 
                      <p><span>To use the tool you'll need to download each of these packages and point to where you've downloaded them as per the instructions laid&nbsp;out in the tool's interface. (I'm not a lawyer, but I think by forcing you to download the packages yourself it absolves me of all liability of you violating the packages' license terms.) Here's how it looks after I've configured mine:</span></p> 
                      <p>&nbsp;</p> 
                      <p><span><img src="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2421iCB29D6EBFA2FC813/image-size/original?v=mpbl-1&amp;px=-1" border="0" alt="Sentiment.PNG" title="Sentiment.PNG" li-image-url="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2421iCB29D6EBFA2FC813?v=1.0" li-image-display-id="'2421iCB29D6EBFA2FC813'" li-message-uid="'12084'" li-messages-message-image="true" li-bindable="" class="lia-media-image"></span></p> 
                      <p>&nbsp;</p> 
                      <h1 id="toc-hId-1361672139"><span>The Analysis</span></h1> 
                      <p>&nbsp;</p> 
                      <p><span>I'm using <a href="http://help.sentiment140.com/for-students" target="_blank" rel="nofollow noopener noreferrer">Sentiment140</a>&nbsp;data for the analysis. Basically it's twitter data that's been pre-scored according to emoticons - if a tweet contained a smiley, it's positive; a frowny, negative. (Careful if you want to use this data as it's Twitter, so probably NSFW.) In order to do this for free (see Microsoft API record limit above), I'm limiting to a random subset of 10,000 tweets.</span></p> 
                      <p>&nbsp;</p> 
                      <p>&nbsp;<img src="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2428i856D446B18E9CEA0/image-size/original?v=mpbl-1&amp;px=-1" border="0" alt="accuracy.png" title="accuracy.png" li-image-url="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2428i856D446B18E9CEA0?v=1.0" li-image-display-id="'2428i856D446B18E9CEA0'" li-message-uid="'12084'" li-messages-message-image="true" li-bindable="" class="lia-media-image"></p> 
                      <p>The Microsoft algorithm came out on top for accuracy. Interestingly, it was also the fastest, even though it was leveraging an API over the web.</p> 
                      <p><img src="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2429iAEA45479E11CEFF6/image-size/original?v=mpbl-1&amp;px=-1" border="0" alt="speed.png" title="speed.png" li-image-url="https://pvsmt99345.i.lithium.com/t5/image/serverpage/image-id/2429iAEA45479E11CEFF6?v=1.0" li-image-display-id="'2429iAEA45479E11CEFF6'" li-message-uid="'12084'" li-messages-message-image="true" li-bindable="" class="lia-media-image"></p> 
                      <p>I've attached the Sentiment tool - feel free to tweak it to see if you can improve the accuracy or performance. I also encourage you to try to replicate my results, either with the Sentiment140 data or some other data source. I'll attach my analysis in the comments upon request.</p>
