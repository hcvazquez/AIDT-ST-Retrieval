      <h1> Getting Started with jQuery Mobile and Visualforce</h1> 
      <p> <a href="http://jquerymobile.com/" target="_blank">jQuery Mobile</a> is a touch-optimized web framework that lets you develop mobile web applications using HTML5, JavaScript and CSS3 for a wide variety of smartphones and tablet computers. In this quick start tutorial, you will learn how to combine jQuery Mobile with the Salesforce web framework, <a href="http://wiki.developerforce.com/page/User_Interface" target="_blank">Visualforce</a>, natively on Force.com.</p> 
      <p> This quick start creates a contact management app that provides basic CRUD access to the <a href="http://www.salesforce.com/us/developer/docs/api/Content/sforce_api_objects_contact.htm" target="_blank">Contact standard object</a>. Using this app, users can view, update, add and delete Contact records from any mobile device with a modern browser. Here are screenshots of the completed application.</p> 
      <p class="img_group"> <img alt="" src="http://res.cloudinary.com/hy4kyit2a/image/upload/w_300,h_563,c_limit/v1365288810/jf0wzambzmlsai0dkwne.png"> <img alt="" src="http://res.cloudinary.com/hy4kyit2a/image/upload/w_300,c_limit/v1365288783/kkki0nbbs9babqocezya.png"></p>

       <h2> Prerequisites</h2> 
       <ol start="1"> 
        <li> <a href="https://developer.salesforce.com/signup?d=70130000000EjHb" target="_blank">Sign up</a> for a free Salesforce Platform Developer Edition (DE), if you don’t already have one. This will act as your own isolated development and test environment. You should not use this tutorial with a trial, sandbox, or production org.</li>

      <h2> Install the Mobile Pack for jQuery Mobile into your DE Org</h2> 
      <ol start="1"> 
       <li> <a href="https://login.salesforce.com/" target="_blank">Log into your DE</a>.</li> 
       <li> Paste this install URL into the address bar (or click the link) to start installing the package: <a href="https://login.salesforce.com/packaging/installPackage.apexp?p0=04ti0000000JEkf" target="_blank">https://login.salesforce.com/packaging/installPackage.apexp?p0=04ti0000000JEkf</a></li> 
       <li> Click <strong>‘Continue’</strong> on the Package Installation Detail page</li> 
       <li> Click <strong>‘Next’</strong> for Step 1. Approve Package API Access</li> 
       <li> Choose the access level you want end users to have and then click <strong>‘Next’</strong> for Step 2. Choose security level (‘Grant access to all’ is recommended for a personal DE test environment)</li> 
       <li> Click <strong>‘Install’</strong> for Step 3. Install Package</li> 
      </ol> 
      <h2> What is included in this resource package?</h2> 
      <p> <strong>MobileSample_jQueryMobile.page / jQuery Moble Tab </strong>: This Visualforce page forms the heart of the application and contains both the list view and the update/add/delete view shown above. You can navigate to your page by manually redirecting in the URL or by using the included jQuery Moble tab. </p> 
      <p> <strong>MobileSample_Resources_jQueryMobile</strong> (<em>Static Resource zipfile</em>): This package contains all the CSS files, images and JS libraries used in the quick start. </p> 
      <p> <strong>RemoteTK VF component, Apex class, test class</strong>: Unlike the other mobile packs, this mobile pack uses RemoteTK which provides an abstraction very similar to the REST API, implemented via @RemoteAction methods in the controller; however, it does not consume any API calls. Check out the <a target="_blank" href="https://github.com/developerforce/Force.com-JavaScript-REST-Toolkit">Force.com JavaScript REST Toolkit README</a> to learn more about the custom component works. </p> 
      <h2> Test Your Application</h2> 
      <p> Navigate to the <strong>MobileSample_jQueryMobile</strong> Visualforce page by appending the page name to the end of your instance url - <a href="https://login.salesforce.com/apex/MobileSample_jQueryMobile" target="_blank">https://login.salesforce.com/apex/MobileSample_jQueryMobile</a>. You should see the application shown in the earlier screenshots. Ideally, you should test the page from a mobile device (after logging in with your Salesforce credentials). To modify the page or view the code, you can find it in the Setup tree under <strong>App Setup &gt; Develop &gt; Pages</strong></p>

       <li> Back in the command line where you created your Heroku application, create an <a href="https://devcenter.heroku.com/articles/config-vars" target="_blank">environment variable</a> for the application URL (for example, <code>https://(yourapplicationname).herokuapp.com</code>) and client ID (the consumer key) using fields from the connected app you just created: 
        <ol start="1"> 
         <li> Application URL: <code>heroku config:add app_url=[APP URL]</code></li> 
         <li> Client ID: <code>heroku config:add client_id=[CONSUMER KEY]</code></li> 
        </ol> </li> 
       <li> <a href="https://devcenter.heroku.com/articles/git" target="_blank">Deploy the application to Heroku</a> by running the following command: <code>git push heroku master</code></li> 
      </ol> 
      <h2> Test your application</h2> 
      <p> To test your app using the desktop browser, use the command <code>heroku open</code>. To test your app on a mobile device, navigate to the app with the application URL: <code>https://(yourapplicationname).herokuapp.com</code>.</p> 
      <div class="prereqs"> 
       <h3> I'm getting a "Client ID Invalid" error</h3> 
       <p> When a Connected App is first defined in Salesforce, it might take a few minutes before the client ID is publicly available. It shouldn't take more than 10 or 15 minutes, and usually takes less than 5. If you see this error on the first try, take a quick breather and try again in a few minutes.<br> If the error persists, you can check the client_id value against the Connected Apps screen by entering <code>heroku config</code> in the command line. If they don't match (i.e. characters that got left off when copied) - you can re-enter the <code>heroku config:add</code> command to change the environment variable.</p>
