         <p>The Javascript library and jQuery plugin that suggests a right domain when your users misspell it in an email address.</p>

           </svg></a>What does it do?</h2> 
         <p>When your user types in "<a href="mailto:user@gmil.con">user@gmil.con</a>", Mailcheck will suggest "<a href="mailto:user@gmail.com">user@gmail.com</a>".</p> 
         <p>Mailcheck will offer up suggestions for second and top level domains too. For example, when a user types in "<a href="mailto:user@hotmail.cmo">user@hotmail.cmo</a>", "hotmail.com" will be suggested. Similarly, if only the second level domain is misspelled, it will be corrected independently of the top level domain.</p> 
         <p><a href="https://raw.githubusercontent.com/mailcheck/mailcheck/master/doc/example.png" target="_blank"><img src="https://raw.githubusercontent.com/mailcheck/mailcheck/master/doc/example.png" alt="diagram" style="max-width:100%;"></a></p> 
         <p>See it live in action <a href="https://www.kickstarter.com/signup">here</a>.</p>

         <p>First, include jQuery and Mailcheck into the page.</p> 
         <div class="highlight highlight-text-html-basic">
          <pre>&lt;<span class="pl-ent">script</span> <span class="pl-e">src</span>=<span class="pl-s"><span class="pl-pds">"</span>jquery.min.js<span class="pl-pds">"</span></span>&gt;&lt;/<span class="pl-ent">script</span>&gt;
&lt;<span class="pl-ent">script</span> <span class="pl-e">src</span>=<span class="pl-s"><span class="pl-pds">"</span>mailcheck.min.js<span class="pl-pds">"</span></span>&gt;&lt;/<span class="pl-ent">script</span>&gt;</pre>
         </div> 
         <p>Have a text field.</p> 
         <div class="highlight highlight-text-html-basic">
          <pre>&lt;<span class="pl-ent">input</span> <span class="pl-e">id</span>=<span class="pl-s"><span class="pl-pds">"</span>email<span class="pl-pds">"</span></span> <span class="pl-e">name</span>=<span class="pl-s"><span class="pl-pds">"</span>email<span class="pl-pds">"</span></span> <span class="pl-e">type</span>=<span class="pl-s"><span class="pl-pds">"</span>email<span class="pl-pds">"</span></span> /&gt;</pre>
         </div> 
         <p>Now, attach Mailcheck to the text field. You can declare an array of domains, second level domains and top level domains you want to check against.</p> 
         <div class="highlight highlight-text-html-basic">
          <pre>&lt;<span class="pl-ent">script</span>&gt;<span class="pl-s1"></span>
<span class="pl-s1"><span class="pl-k">var</span> domains <span class="pl-k">=</span> [<span class="pl-s"><span class="pl-pds">'</span>gmail.com<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>aol.com<span class="pl-pds">'</span></span>];</span>
<span class="pl-s1"><span class="pl-k">var</span> secondLevelDomains <span class="pl-k">=</span> [<span class="pl-s"><span class="pl-pds">'</span>hotmail<span class="pl-pds">'</span></span>]</span>
<span class="pl-s1"><span class="pl-k">var</span> topLevelDomains <span class="pl-k">=</span> [<span class="pl-s"><span class="pl-pds">"</span>com<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>net<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>org<span class="pl-pds">"</span></span>];</span>
<span class="pl-s1"></span>

<span class="pl-s1">  <span class="pl-c"><span class="pl-c">//</span> a string distance algorithm of your choosing</span></span>
<span class="pl-s1">}</span>

<span class="pl-s1">  <span class="pl-en">$</span>(<span class="pl-c1">this</span>).<span class="pl-en">mailcheck</span>({</span>
<span class="pl-s1">    domains<span class="pl-k">:</span> domains,                       <span class="pl-c"><span class="pl-c">//</span> optional</span></span>
<span class="pl-s1">    secondLevelDomains<span class="pl-k">:</span> secondLevelDomains, <span class="pl-c"><span class="pl-c">//</span> optional</span></span>
<span class="pl-s1">    topLevelDomains<span class="pl-k">:</span> topLevelDomains,       <span class="pl-c"><span class="pl-c">//</span> optional</span></span>
<span class="pl-s1">    distanceFunction<span class="pl-k">:</span> superStringDistance,  <span class="pl-c"><span class="pl-c">//</span> optional</span></span>
<span class="pl-s1">    <span class="pl-en">suggested</span><span class="pl-k">:</span> <span class="pl-k">function</span>(<span class="pl-smi">element</span>, <span class="pl-smi">suggestion</span>) {</span>
<span class="pl-s1">      <span class="pl-c"><span class="pl-c">//</span> callback code</span></span>
<span class="pl-s1">    },</span>
<span class="pl-s1">    <span class="pl-en">empty</span><span class="pl-k">:</span> <span class="pl-k">function</span>(<span class="pl-smi">element</span>) {</span>
<span class="pl-s1">      <span class="pl-c"><span class="pl-c">//</span> callback code</span></span>
<span class="pl-s1">    }</span>
<span class="pl-s1">  });</span>
<span class="pl-s1">});</span>
<span class="pl-s1"></span><span class="pl-s1">&lt;</span>/<span class="pl-ent">script</span>&gt;</pre>
         </div> 
         <p>Mailcheck takes in two callbacks, <code>suggested</code> and <code>empty</code>. We recommend you supply both.</p> 
         <p><code>suggested</code> is called when there's a suggestion. Mailcheck passes in the target element and the suggestion. The suggestion is an object with the following members:</p> 
         <div class="highlight highlight-source-js">
          <pre>{
  address<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>test<span class="pl-pds">'</span></span>,          <span class="pl-c"><span class="pl-c">//</span> the address; part before the @ sign</span>
  domain<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>gmail.com<span class="pl-pds">'</span></span>,    <span class="pl-c"><span class="pl-c">//</span> the suggested domain</span>
  full<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>test@gmail.com<span class="pl-pds">'</span></span>  <span class="pl-c"><span class="pl-c">//</span> the full suggested email</span>
}</pre>
         </div> 
         <p>Mailcheck does not want to get in the way of how you can show suggestions. Use the suggestion object to display suggestions in your preferred manner.</p> 
         <p><code>empty</code> is called when there's no suggestion. Mailcheck just passes in the target element. It is a good idea to use this callback to clear an existing suggestion.</p>

         <p>Mailcheck has inbuilt defaults if the <code>domains</code>, <code>secondLevelDomains</code> or <code>topLevelDomains</code> options aren't provided. We still recommend supplying your own domains based on the distribution of your users.</p> 
         <h4><a id="user-content-adding-your-own-domains" class="anchor" href="#adding-your-own-domains" aria-hidden="true">

           </svg></a>Adding your own Domains</h4> 
         <p>You can replace Mailcheck's default domain/TLD suggestions by supplying replacements to <code>mailcheck.run</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">Mailcheck</span>.<span class="pl-en">run</span>({
  domains<span class="pl-k">:</span> [<span class="pl-s"><span class="pl-pds">'</span>customdomain.com<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>anotherdomain.net<span class="pl-pds">'</span></span>], <span class="pl-c"><span class="pl-c">//</span> replaces existing domains</span>
  secondLevelDomains<span class="pl-k">:</span> [<span class="pl-s"><span class="pl-pds">'</span>domain<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>yetanotherdomain<span class="pl-pds">'</span></span>], <span class="pl-c"><span class="pl-c">//</span> replaces existing SLDs</span>

         <h2><a id="user-content-customization" class="anchor" href="#customization" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Customization</h2> 
         <p>The Mailcheck jQuery plugin wraps Mailcheck. The prime candidates for customization are the methods <code>Mailcheck.findClosestDomain</code> and <code>Mailcheck.stringDistance</code>.</p> 
         <p>Mailcheck currently uses the <a href="http://siderite.blogspot.com/2007/04/super-fast-and-accurate-string-distance.html">sift3</a> string similarity algorithm by <a href="http://siderite.blogspot.com/">Siderite</a>. You can modify the inbuilt string distance function, or pass in your own when calling Mailcheck.</p> 
         <p>Since Mailcheck runs client side, keep in mind file size, memory usage and performance.</p>

           </svg></a>Tests</h2> 
         <p>Mailcheck is tested with <a href="https://jasmine.github.io/">Jasmine</a>. Run <code>npm test</code> from the command line to run the test suite. Alternatively, you can Load <code>spec/spec_runner.html</code> in your browser.</p>

           </svg></a>Contributing</h2> 
         <p>Let's make Mailcheck awesome. We're on the lookout for maintainers and <a href="https://github.com/mailcheck/mailcheck/contributors">contributors</a>.</p> 
         <p>And do send in those pull requests! To get them accepted, please:</p> 
         <ul> 
          <li>Test your code. Add test cases to <code>spec/mailcheckSpec.js</code>, and run it across browsers (yes, including IE).</li> 
          <li>Minify the plugin by running <code>grunt</code> in the Mailcheck directory (npm install should have installed a git pre-commit hook that takes care of this for you).</li> 
         </ul> 
         <p>Upcoming features, bugs and feature requests are managed in <a href="https://github.com/mailcheck/mailcheck/issues">Issues</a>.</p>
