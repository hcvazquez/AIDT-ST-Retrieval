   <div id="pre_intro_section" class="pre_intro_section">
    In this tutorial, you use Oracle JDeveloper and Oracle MAF to build and deploy a mobile application. 
    <p>The application uses a simple Java class to store the data and displays a list of employees on one page and a graph of salaries on another. As you build the application, you will add MAF layout components to further control and customize the behavior of the application.</p> 
    <p>In Part 2 of the tutorial, you add specific device integration to your application. In Part 3, you add a call to a public stock quote web service.<br> </p>

   <div class="part_header">
    <a id="part1"></a>Part 1: Creating a Mobile Application Framework application 
   </div> 
   <p>When you work in JDeveloper, you organize your work in projects within an application. JDeveloper provides several templates that you can use to create an application and projects. The templates are pre-configured with a basic set of technologies that are needed for developing various types of applications, and you create a working environment by selecting the template that fits your needs. You can then configure it to add any other technologies you plan to use.</p> 
   <p>In this first section, you create an application by using the MAF application template. You then create a couple of mobile features that will hold the application you are be building. <br> </p> 
   <div class="step_header">
    <a id="step1"></a>Step 1: Create an Application Workspace 
   </div>

    <li> <p>In the Create Mobile Application Framework Application wizard, name the application <span class="gui-object">Employees</span>, and set the Application Package Prefix to <span class="gui-object">mycomp. </span>Click <span class="gui-object">Next</span>. </p> </li>

   <div class="step_header">
    <a id="step2"></a>Step 2: Add Features to the Application 
   </div> 
   <p>When you deploy an Oracle MAF application, the application will be a single application on the mobile device. The application may consist of several parts. For example, there may be a search component, a document (like a help page), a data entry form, or even a list. Each of these parts is defined as a feature within the application. In this section of the tutorial, you create the features that you will deploy as part of your application. </p> 
   <ol> 
    <li> <p>The features of a MAF application are defined in the <span class="gui-object">maf-feature.xml</span> file. If the file is not already open, double-click the file to open it in the editor. </p> </li> 
    <img src="images/t010201.png" alt="adfmf-feature.xml"> 
    <li> <p>In maf-features.xml, click the green plus sign <img src="images/greenplussign.jpg" alt="Green Plus Sign" width="22" height="22"> in the Features table to create a new feature.</p> </li> 
    <img src="images/t010202.png" alt="features table"> 
    <li> <p>Name the new feature <span class="gui-object">Employees</span> and click <span class="gui-object">OK</span>. Ensure the Add a corresponding feature reference to maf-application.xml checkbox is selected. </p> </li> 
    <img src="images/t010203.png" alt="feature dialog"> 
    <li> <p> Click the green plus sign <img src="images/greenplussign.jpg" alt="Green Plus Sign" width="22" height="22"> again, in the Features table to create another new feature. Set the Feature Name to <span class="gui-object">Help</span>, then click <span class="gui-object">OK</span>. </p> </li> 
    <img src="images/t010204.png" alt="feature dialog"> 
    <li> <p>In the Features table, select the <span class="gui-object">Help</span> feature. Under the Features table, click the <span class="gui-object">Content</span> tab. Set the Type to <span class="gui-object">Local HTML</span>. </p> </li> 
    <img src="images/t010205.png" alt="feature table"> 
    <li> <p>Next to the URL property, click the <span class="gui-object">Create</span> (green plus sign <img src="images/greenplussign.jpg" alt="Green Plus Sign" width="22" height="22"> ) icon to create the local HTML file.</p> </li> 
    <img src="images/t010206.png" alt="feature table"> 
    <li> <p>Set the File Name to <span class="gui-object">help.html</span> and click <span class="gui-object">OK</span>. </p> </li> 
    <img src="images/t010207.png" alt="create html file"> 
    <li> <p>JDeveloper creates the file and opens it in the editor. Enter "This document will be the Help System" in the help.html file.<br> </p> </li> 
    <img src="images/t010208.png" alt="help.html page"> 
    <li> <p>Save your work. </p> </li> 
   </ol> 
   <div class="step_header">
    <a id="step3"></a>Step 3: Create an MAF Task Flow 
   </div> 
   <p>In this section you create a MAF task flow that will be consumed by the Employees feature. A MAF task flow contains multiple pages that interact with one another. The task flow provides structure to the interaction. </p> 
   <ol> 
    <li> <p>If it is not already open, double-click <span class="gui-object">maf-feature.xml</span> in the Application Navigator to open it in the editor. If it is already open, you can click the tab named <span class="gui-object">maf-feature.xml</span>.</p> </li> 
    <img src="images/t010301.png" alt="application navigator"> 
    <li> <p>In the editor, select the <span class="gui-object">Employees</span> feature, and click the <span class="gui-object">Content</span> tab. </p> </li> 
    <img src="images/t010302.png" alt="feature table"> 
    <li> <p>On the Content tab for the Employees Feature, use the <span class="gui-object">Type</span> dropdown to select <span class="gui-object">MAF Task Flow</span>.</p> </li> 
    <img src="images/t010303.png" alt="add feature"> 
    <li> <p>Click the <span class="gui-object">Add</span> icon <img src="images/greenplussign.jpg" alt="Green Plus Sign" width="22" height="22"> next to the File field.</p> </li> 
    <img src="images/t010304.png" alt="add feature"> 
    <li> <p>In the Create dialog box, set the <span class="gui-object">File Name</span> property to <span class="gui-object">EmpsTaskFlow.xml</span> and click <span class="gui-object">OK</span>.</p> </li> 
    <img src="images/t010305a.png" alt="Create task flow"> 
    <p> JDeveloper creates the task flow, and opens it in the editor.</p> 
    <img src="images/t010305b.png" alt="editor"> 
    <p>You can now add content to the task flow. The content will consist of view activities (pointers to pages), and control flow cases (navigation controls).</p> 
    <li> <p>Notice the Component Palette located in the upper-right corner of JDeveloper. From that palette, drag a <span class="gui-object">view </span> activity to the task flow.</p> </li> 
    <img src="images/t010306.png" alt="editor"> 
    <li> <p>Change the name the view activity to <span class="gui-object">empList</span>.</p> </li> 
    <img src="images/t010307.png" alt="task flow"> 
    <li> <p>Drag a second <span class="gui-object">view activity</span> to the task flow and name this one <span class="gui-object">graph</span>. </p> </li> 
    <img src="images/t010308.png" alt="task flow"> 
    <p>You now have two view activities on the task flow. The next step is to create navigation rules, or control flow cases, between the view activities. </p>
    <li> <p>Click <span class="gui-object">Control Flow Case</span> on the Component Window. </p> </li> 
    <img src="images/t010309.png" alt="task flow"> 
    <li> <p>After you have selected the control flow case, click the <span class="gui-object">empList</span> view activity and then click the <span class="gui-object">graph</span> view activity.</p> </li> 
    <img src="images/t010310.png" al10t="task flow"> 
    <li> <p>Set the name of the control flow case to <span class="gui-object">showGraph.</span></p> </li> 
    <img src="images/t010311.png" alt="task flow"> 
    <li> <p>Select the <span class="gui-object">showGraph</span> control flow case and in the Property Window under Behavior, set the <span class="gui-object">Transition</span> property to <span class="gui-object">flipRight</span>. </p> <img src="images/t010312a.png" alt="property inspector"> </li> 
    <li> <p>Save your work.</p> <p>The EmpsTaskFlow should look something like the following:</p> <p><img src="images/t010312b.png" alt="task flow"> </p> <p>Remember that the view activities you have created are merely pointers to pages. You create the underlying pages later in this tutorial. </p> </li> 
   </ol> 
   <div class="step_header">
    <a id="step4"></a>Step 4: Create Java Classes and Data Controls 
   </div> 
   <p>MAF provides an application layer that manages data so that the UI can consume the data without concern about where or how the data is stored. This layer is called the data control layer. </p>
   <p>This tutorial uses POJOs (Plain Old Java Objects) for the application data source. In this section, you learn how to create Java classes and generate data controls that are based on the classes. The user interface portion of the application uses the data controls to present data to the user. </p>
   <ol> 
    <li> <p>Right-click the <span class="gui-object">ViewController</span> project and select <span class="gui-object">New &gt; From Gallery </span>. </p> </li>

    <img src="images/t010402.png" alt="new gallery"> 
    <li> <p>Name the class <span class="gui-object">Emp</span> and click <span class="gui-object">OK</span> to accept the other default values. </p> </li> 
    <img src="images/t010403.png" alt="create java class">

Choose <span class="gui-object">java.util.Date</span> for the import. </p> <p class="code">private String name;<br> private String email;<br> private int salary;<br> private Date hireDate;</p> </li>

   <div class="step_header">
    <a id="step5"></a>Step 5: Create and Test the UI

   <p>In this section, you create the pages that are associated with the view activities you created on the task flow. You have already created the navigation rules, so you will now create the pages and add buttons for the application navigation. </p>
   <ol> 
    <li> <p>Open <span class="gui-object">EmpsTaskFlow.xml</span> in the editor. (HINT: The tab should still be open, but if not, double-click EmpsTaskFlow.xml in the Application Navigator.) </p> </li>

    <li> <p>Double-click <span class="summary_header">empList</span> and select all of the page facets, except the Footer. Click <span class="gui-object">OK</span> to create the page. </p> </li> 
    <img src="images/t010502.png" alt="create page"> 
    <li> <p>Use the drag bar to split the editor into two sections. The drag bar is the small vertical bar to the right of the bottom scroll bar in the editor. </p> </li> 
    <img src="images/t010503.png" alt="editor"> 
    <li> <p>In the editor, click the <span class="gui-object">Preview</span> tab at the bottom of the right pane to preview the UI. Set the device to <span class="gui-object">Android High</span> and the Zoom factor to <span class="gui-object">75%</span>. Note that this is for the preview pane only and does not affect your code or application.</p> </li> 
    <img src="images/t010504.png" alt="editor"> 
    <li> <p>In the code section, within the header facet, click the <span class="summary_header">outputText</span> component and set the <span class="gui-object">value</span> property to <span class="gui-object">Emp List</span>.</p> </li> 
    <img src="images/t010505.png" alt="alt text"> 
    <li> <p>In the source editor, select the button in the primary facet and in the Property Inspector, set the <span class="gui-object">Text</span> property to <span class="gui-object">Graph</span>. When the user clicks this button, the application will display a graph of employee data. </p> </li> 
    <img src="images/t010506.png" alt="alt text"> 
    <li> <p>Set the <span class="gui-object">Action</span> property to <span class="gui-object">showGraph</span>, which is the navigation case to navigate to the Graph page. </p> </li> 
    <img src="images/t010507.png" alt="alt text"> 
    <p>In the next few steps, you add data components to the page from the data controls you created earlier. </p> 
    <li> <p>Expand the <span class="gui-object">Data Controls</span> palette to see the Emps data control. <br> <br> If needed, click the blue circled arrow icon to refresh the data controls. </p> </li> 
    <img src="images/t010508.png" alt="alt text"> 
    <li> <p>Drag the <span class="gui-object">emps</span> collection from within the Emps data control to the <span class="gui-object">Panel Page</span> node in the Structure window and drop the collection as a <span class="gui-object">MAF List View</span>. The Structure window provides an easy to read hierarchical view of the current page. The hierarchical view makes it easier for you to select components and move them around on the page. Because the structure window and editor operate on the same code, the are always in sync. <br> <br> The data controls you created include collections and attributes. The emps collection is the one with the red table icon. Make sure you've selected the AMX page, otherwise you will not see anything in the Structure window. </p> </li> 
    <img src="images/t010509.png" alt="alt text"> 
    <p></p> 
    <li> <p>Under List Formats, select <span class="gui-object">Simple</span>, then under Variations, select the variation with text items grouped by dividers. Under Styles, verify that the first style in the list is selected and click <span class="gui-object">OK</span>. </p> </li> 
    <img src="images/t010510.png" alt="alt text"> 
    <li> <p>In the Edit List View dialog box, set the <span class="gui-object">Divider Mode</span> to <span class="gui-object">First Letter</span> and click <span class="gui-object">OK</span>.</p> </li> 
    <img src="images/t010511.png" alt="alt text"> 
    <p>That finishes the first page for now. Next, you create the page that will house the graph.</p> 
    <li> <p>Save your work. </p> </li> 
    <li> <p>Just as you did before, double-click the <span class="gui-object">graph</span> view activity on the <span class="gui-object">EmpsTaskFlow.</span></p> </li> 
    <img src="images/t010513.png" alt="alt text"> 
    <li> <p>Make sure that all the page facets are selected, except the Footer, and click <span class="gui-object">OK</span>. </p> </li> 
    <img src="images/t010514.png" alt="alt text"> 
    <li> <p>Use the drag bar and split the editor window to show two views. Remember, the drag bar is the small vertical bar to the right of the bottom scroll bar in the editor. </p> </li> 
    <img src="images/t010515.png" alt="alt text"> 
    <li> <p>In editor, click the <span class="gui-object">Preview</span> tab at the bottom of the right pane to preview the UI. Set the Device to <span class="gui-object">Android High</span> and the Zoom factor to <span class="gui-object">75%</span>.</p> </li> 
    <img src="images/t010516.png" alt="alt text">

    <img src="images/t010518.png" alt="alt text"> 
    <li> <p>Just as you did for the empList page, drag the <span class="gui-object">emps</span> collection from within the Emps data control to the <span class="gui-object">Panel Page</span> node in the Structure window, but this time, drop it as a <span class="gui-object">MAF Chart. </span><br> <br> Make sure you've selected the AMX page, otherwise you will not see anything in the Structure window.</p> </li> 
    <img src="images/t010519.png" alt="alt text">

Click <span class="gui-object">OK</span>. </p> </li> 
    <img src="images/t010521.png" alt="alt text">

   <div class="step_header">
    <a id="step6"></a>Step 6: Deploy and Test the Application
   </div> 
   <p>When you deploy a MAF application, you can improve performance by using Release mode. This mode causes the application to be deployed with an optimized JVM and minimal JavaScript libraries, thereby significantly improving the performance of the application.</p> 
   <p>For <span class="gui-object">iOS</span>, you select Release mode when you create the deployment profile, which you will do shortly. If you are using an iOS environment, you can skip the first 7 instructions and continue at instruction <a onclick="show_show('step8');">8. From the Application menu, select Deploy &gt; New Deployment Profile. </a></p> 
   <p>For <span class="gui-object">Android</span>, you have to create a local keystore before you can use Release mode.</p> 
   <p> The next few steps set up the keystore for Android deployment. </p> 
   <p>For iOS deployment you can skip to instruction 18.</p> 
   <ol> 
    <li> <p>Open a terminal or command window. Navigate to the JDK\bin directory used by JDeveloper. If you are not sure where to find this directory, use <span class="gui-object">Help &gt; About &gt; Properties &gt; java.home</span> to see the JDK or JRE that JDeveloper is using. For Windows 7 make sure you open the command window as Administrator. To open the command window as Administrator Click <span class="gui-object">Start</span>, in the Start Search box, type <span class="gui-object">cmd</span> and then press <span class="gui-object">CTRL + SHIFT + ENTER</span>. </p> </li> 
    <li> <p>Enter the following command but substitute a name of your choosing for the <span class="gui-object">Keystore Name</span> and <span class="gui-object">Alias Name</span>. In the screen shot below, we used <span class="gui-object">employees</span> for both. After you type the command, press <span class="gui-object">enter</span>. <span class="gui-object">NOTE</span>: You will need to type the command into the a command window instead of using copy and paste. NOTE: If you are on a mac, you may need to check permissions.</p> <p class="gui-object">keytool –genkey –v –keystore &lt;Keystore Name&gt;.keystore –alias &lt;Alias Name&gt; -keyalg RSA –keysize 2048 –validity 10000</p> <p><img src="images/t010602.png" alt="alt text"> </p> </li> 
    <li> <p>You will be prompted for several values including a password, your name, city, State, and so on. Enter values for those variables.</p> <p><img src="images/t010603.png" alt="alt text"></p> </li> 
    <li> <p> Confirm the information you entered, then enter a <span class="gui-object">password</span> as the last prompt:</p> <p><img src="images/t010604.png" alt="alt text"> </p> </li> 
    <p>When the command finishes, the keystore is ready to use. Next you configure JDeveloper to use the keystore.</p> 
    <li> <p>In the JDeveloper menu, select <span class="gui-object">Tools &gt; Preferences</span> (on Windows) or <span class="gui-object">JDeveloper &gt; Preferences</span> (on iOS).</p> </li> 
    <li> <p>Expand <span class="gui-object">Mobile Application Framework </span>and select <span class="gui-object">Android Platforms</span>. In the <span class="gui-object">Signing Credentials</span> section, click the <span class="gui-object">Release</span> tab.</p> </li> 
    <li> <p>In the <span class="gui-object">Keystore Location</span> property, enter the <span class="gui-object">Keystore Location</span> of the keystore you just created and then enter the <span class="gui-object">Keystore Password</span>, <span class="gui-object">Key Alias</span>,and <span class="gui-object">Key Password</span>. Click <span class="gui-object">OK</span>.</p> <p><img src="images/t010607.png" alt="alt text"> </p> </li> 
    <p>Now that JDeveloper knows about the keystore, you can use Release mode in the deployment profiles.</p> 
    <p>In the next few steps, you create a deployment profile.</p> 
    <p>NOTE: Remember that before you can deploy an Android application, you need to have started the emulator. Instructions are in the the tutorial <a href="http://docs.oracle.com/cd/E18941_01/tutorials/MobileTutorial/jdtut_11r2_54_2.html">Set Up and Configure an Android Environment</a>.</p> 
    <li><a></a> <p>From the Application Navigator dropdown menu, select <span class="gui-object">Deploy &gt; New Deployment Profile</span>.<span class="step_header"><a id="step8"></a></span></p> <img src="images/t010608.png" alt="alt text"> </li> 
    <li> <p>Select <span class="gui-object">MAF for Android</span> and click <span class="gui-object">OK</span>.<br> </p> </li> 
    <img src="images/t010609.png" alt="alt text"> 
    <br> 
    <li> <p>Click <span class="gui-object">Android Options</span> and select <span class="gui-object">Release</span> for the Build Mode. Then click <span class="gui-object">OK</span>. </p> <img src="images/t010610.png" alt="alt text"> </li>
    <li> <p>To deploy the application, from the Application menu, select <span class="gui-object">Deploy &gt; Android2</span>. </p> <img src="images/t010611.png" alt="alt text"> </li>
    <li> <p>Select <span class="gui-object">Deploy application to emulator</span> and click <span class="gui-object">Next</span>. </p> </li> 
    <img src="images/t010612.png" alt="alt text"> 
    <li> <p>Click <span class="gui-object">Finish</span> to begin the deployment. Make sure your Android emulator is up and running.</p> </li> 
    <img src="images/t010613.png" alt="alt text"> 
    <li> <p>In the Log window, click the <span class="gui-object">Deployment tab</span> to see the deployment messages. </p> </li> 
    <img src="images/t010614.png" alt="alt text" "> 
    <li> <p>After the deployment is complete, switch windows to the <span class="gui-object">Android emulator</span>. and select the home icon, then find the Employees icon. Start the application.<br> If prompted, click OK for sending notifications. </p> </li> 
    <img src="images/t010615.png" alt="alt text"> 
    <br> 
    <li> <p>Notice the Emp List which shows all of the employees from your data source. NOTE: The emulator may be slower than an actual device and it may take a while for the application to load. </p> </li> 
    <img src="images/t010616.png" alt="alt text"> 
    <li> <p>Click the <span class="gui-object">Graph</span> button to navigate to the Graph page. </p> <img src="images/t010617.png" alt="alt text"> <p>The following steps are for <span class="gui-object">iOS</span> deployment. </p></li> 
    <li> <p>From the Application Navigator dropdown menu, select <span class="gui-object">Deploy &gt; New Deployment Profile</span>.<span class="step_header"><a id="iOSDeploy"></a></span></p> <img src="images/t010618.png" alt="alt text"> </li> 
    <li> <p>Select <span class="gui-object">MAF for iOS</span> as the Profile Type and click <span class="gui-object">OK</span>.</p> <img src="images/t010619.png" alt="alt text"> </li> 
    <li> <p>In the Deployment Profile Properties, select <span class="gui-object">iOS options</span>.</p> <img src="images/t010620.png" alt="alt text"> </li> 
    <li> <p>For <span class="gui-object">Simulator</span>, select<span class="gui-object"> iPad 2 (iOS 8.1 - iPad 2)</span> and click <span class="gui-object">OK</span>.</p> <img src="images/t010621.png" alt="alt text"> </li> 
    <li> <p>From the Application Navigator dropdown menu, select <span class="gui-object">Deploy &gt; iOS2.</span></p> <img src="images/t010622.png" alt="alt text"> </li> 
    <li> <p>In the Deployment dialog, select <span class="gui-object">Deploy application to simulator</span> and click <span class="gui-object">Finish</span>.</p> <img src="images/t010623.png" alt="alt text"> <p>When you deploy to the iOS Simulator, the application is deployed and opened on the simulator. If prompted, click <span class="gui-object">OK</span> for sending notifications.</p> </li> 
    <li> <p>Notice the Emp List which shows all of the employees from your data source. Click <span class="gui-object">Graph</span> to see the graph page.</p> <img src="images/t010624.png" alt="alt text"> </li> 
    <li> <p>You now see the graph page. In iOS, you navigate back to the home page using the Simulator menu options <span class="gui-object">Hardware</span> &gt; <span class="gui-object">Home</span>.</p> <img src="images/t010625.png" alt="alt text"> </li> 
   </ol> 
   <p>You now have a working MAF application that you have deployed to either an Android emulator or the iOS simulator. Now that the deployment profiles are complete, you should be able to deploy your application to either environment by using just a few quick steps. You will be doing just that throughout the remainder of this tutorial. </p> 
   <div class="step_header">
    <a id="step7"></a>Step 7: Refine the UI 
   </div> 
   <p>Now that you have a working application, you refine the user interface to take advantage of some of the MAF layout components. </p> 
   <p>In the next few steps, you move the list of employees to a navigator pane. You also add a read-only form that shows employee details that you did not show in the employee list. As you recall, the employee list shows only the employee name. You also split the page in to two parts: an employee list, and an employee form. </p> 
   <ol> 
    <li> <p>In JDeveloper open the <span class="gui-object">empList.amx</span> page. (HINT: Either find the page in the Application Navigator and double-click it, or use the EmpsTaskFlow and double-click the empList view component) </p> </li> 
    <img src="images/t010701.png" alt="alt text"> 
    <p>There are several ways to add components to a page. You can drag a component from the Component Palette to either the source code of the page, or to the Structure window. You can also right-click the component in the Structure window and select Insert Inside from the context menu, then select the new component.</p> 
    <p>You should try the different techniques to see which one you find the easiest.</p> 
    <li> <p>In the Structure window, right-click <span class="gui-object">Panel Page</span> and select <span class="gui-object">Insert Inside Panel Page &gt; MAF AMX</span>.</p> <img src="images/t010702.png" alt="alt text"> </li> 
    <li> <p>Select <span class="gui-object">Panel Splitter</span> and click <span class="gui-object">OK</span>.</p> <img src="images/t010703.png" alt="alt text"> </li> 
    <li> <p>In the Structure window, expand the <span class="gui-object">Panel Splitter</span> to expose the <span class="gui-object">Facet - navigator</span> and the <span class="gui-object">Panel Item</span> component. </p> </li> 
    <img src="images/t010704.png" alt="alt text"> 
    <li> <p>You want the list to display as part of the new Panel Splitter so within the Structure window, drag the <span class="gui-object">List View</span> component down to the <span class="gui-object">Facet - navigator. </span></p> </li> 
    <img src="images/t010705a.png" alt="alt text"> 
    <br> 
    <br> 
    <img src="images/t010705b.png" alt="alt text"> 
    <li> <p>From the Data Controls palette, drag the <span class="gui-object">emps</span> data control to the the Panel Item in the Structure window and drop it as an <span class="gui-object">MAF Read Only Form</span>. </p> </li> 
    <img src="images/t010706a.png" alt="alt text"> 
    <br> 
    <br> 
    <img src="images/t010706b.png" alt="alt text"> 
    <li> <p>Click <span class="gui-object">OK</span> to accept the default labels and components for the form. </p> </li> 
    <img src="images/t010707.png" alt="alt text"> 
    <p>Now you have two sections on the page: one to show the employee list and another to display an individual employee. </p>
    <li> <p>Re-deploy the application. In the following steps, you will use the iOS Simulator. If you are using the Android emulator, the steps are similar. If you need a refresher, go to the deployment steps above.<br> From the Application menu in JDeveloper, select <span class="gui-object">Deploy &gt; iOS2 Deploy</span> and select the simulator as the target.</p> </li> 
    <img src="images/t010708.png" alt="alt text"> 
    <li> <p>After the application is deployed, run it as you did before and notice that the page shows employee details and that there is an arrow on the left side of the page. </p> </li> 
    <img src="images/t010709.png" alt="alt text"> 
    <li> <p>Click the <span class="gui-object">arrow</span> to expand the splitter and show the employee list. </p> </li> 
    <img src="images/t010710.png" alt="alt text"> 
    <li> <p>Click any other employee to show that employee's details. </p> </li> 
    <img src="images/t010711.png" alt="alt text"> 
    <li> <p>Click the arrow to hide the employee list and just show the employee details. </p> </li> 
    <p>You have now completed the first part of this tutorial. In the next part, you add a feature that uses two built in device features.</p>
