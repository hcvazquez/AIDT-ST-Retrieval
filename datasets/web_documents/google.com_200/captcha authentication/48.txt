         <h2><a id="user-content-application-default-credentials" class="anchor" href="#application-default-credentials" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Application Default Credentials</h2> 
         <p>This library provides an implementation of <a href="https://developers.google.com/identity/protocols/application-default-credentials#callingnode">Application Default Credentials</a> for Node.js.</p> 
         <p>The <a href="https://developers.google.com/identity/protocols/application-default-credentials#callingnode">Application Default Credentials</a> provide a simple way to get authorization credentials for use in calling Google APIs.</p> 
         <p>They are best suited for cases when the call needs to have the same identity and authorization level for the application independent of the user. This is the recommended approach to authorize calls to Cloud APIs, particularly when you're building an application that uses Google Compute Engine.</p> 
         <h4><a id="user-content-download-your-service-account-credentials-json-file" class="anchor" href="#download-your-service-account-credentials-json-file" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Download your Service Account Credentials JSON file</h4> 
         <p>To use <code>Application Default Credentials</code>, You first need to download a set of JSON credentials for your project. Go to <strong>APIs &amp; Auth</strong> &gt; <strong>Credentials</strong> in the <a href="https://console.developer.google.com">Google Developers Console</a> and select <strong>Service account</strong> from the <strong>Add credentials</strong> dropdown.</p> 
         <blockquote> 
          <p>This file is your <em>only copy</em> of these credentials. It should never be committed with your source code, and should be stored securely.</p> 
         </blockquote> 
         <p>Once downloaded, store the path to this file in the <code>GOOGLE_APPLICATION_CREDENTIALS</code> environment variable.</p> 
         <h4><a id="user-content-enable-the-api-you-want-to-use" class="anchor" href="#enable-the-api-you-want-to-use" aria-hidden="true">

           </svg></a>Enable the API you want to use</h4> 
         <p>Before making your API call, you must be sure the API you're calling has been enabled. Go to <strong>APIs &amp; Auth</strong> &gt; <strong>APIs</strong> in the <a href="https://console.developer.google.com">Google Developers Console</a> and enable the APIs you'd like to call. For the example below, you must enable the <code>DNS API</code>.</p>
