         <h1><a id="user-content-dateformat" class="anchor" href="#dateformat" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>dateformat</h1> 
         <p>A node.js package for Steven Levithan's excellent <a href="http://blog.stevenlevithan.com/archives/date-time-format">dateFormat()</a> function.</p> 
         <p><a href="https://travis-ci.org/felixge/node-dateformat"><img src="https://camo.githubusercontent.com/b2f09f72759a61dfbe9ba56337c15a5c40ac5462/68747470733a2f2f7472617669732d63692e6f72672f66656c697867652f6e6f64652d64617465666f726d61742e737667" alt="Build Status" data-canonical-src="https://travis-ci.org/felixge/node-dateformat.svg" style="max-width:100%;"></a></p> 
         <h2><a id="user-content-modifications" class="anchor" href="#modifications" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Modifications</h2> 
         <ul> 
          <li>Removed the <code>Date.prototype.format</code> method. Sorry folks, but extending native prototypes is for suckers.</li> 
          <li>Added a <code>module.exports = dateFormat;</code> statement at the bottom</li> 
          <li>Added the placeholder <code>N</code> to get the ISO 8601 numeric representation of the day of the week</li> 
         </ul> 
         <h2><a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">

         <h2><a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Usage</h2> 
         <p>As taken from Steven's post, modified to match the Modifications listed above:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> dateFormat <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>dateformat<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> now <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">Date</span>();

<span class="pl-c"><span class="pl-c">//</span> Basic usage</span>
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>dddd, mmmm dS, yyyy, h:MM:ss TT<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> Saturday, June 9th, 2007, 5:46:21 PM</span>

<span class="pl-c"><span class="pl-c">//</span> You can use one of several named masks</span>
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>isoDateTime<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 2007-06-09T17:46:21</span>

<span class="pl-c"><span class="pl-c">//</span> ...Or add your own</span>
<span class="pl-smi">dateFormat</span>.<span class="pl-smi">masks</span>.<span class="pl-smi">hammerTime</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>HH:MM! "Can<span class="pl-cce">\'</span>t touch this!"<span class="pl-pds">'</span></span>;
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>hammerTime<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 17:46! Can't touch this!</span>

<span class="pl-c"><span class="pl-c">//</span> You can also provide the date as a string</span>
<span class="pl-en">dateFormat</span>(<span class="pl-s"><span class="pl-pds">"</span>Jun 9 2007<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>fullDate<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> Saturday, June 9, 2007</span>

<span class="pl-c"><span class="pl-c">//</span> Note that if you don't include the mask argument,</span>
<span class="pl-c"><span class="pl-c">//</span> dateFormat.masks.default is used</span>
<span class="pl-en">dateFormat</span>(now);
<span class="pl-c"><span class="pl-c">//</span> Sat Jun 09 2007 17:46:21</span>

<span class="pl-c"><span class="pl-c">//</span> And if you don't include the date argument,</span>
<span class="pl-c"><span class="pl-c">//</span> the current date and time is used</span>
<span class="pl-en">dateFormat</span>();
<span class="pl-c"><span class="pl-c">//</span> Sat Jun 09 2007 17:46:22</span>

<span class="pl-c"><span class="pl-c">//</span> You can also skip the date argument (as long as your mask doesn't</span>
<span class="pl-c"><span class="pl-c">//</span> contain any numbers), in which case the current date/time is used</span>
<span class="pl-en">dateFormat</span>(<span class="pl-s"><span class="pl-pds">"</span>longTime<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 5:46:22 PM EST</span>

<span class="pl-c"><span class="pl-c">//</span> And finally, you can convert local time to UTC time. Simply pass in</span>
<span class="pl-c"><span class="pl-c">//</span> true as an additional argument (no argument skipping allowed in this case):</span>
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>longTime<span class="pl-pds">"</span></span>, <span class="pl-c1">true</span>);
<span class="pl-c"><span class="pl-c">//</span> 10:46:21 PM UTC</span>

<span class="pl-c"><span class="pl-c">//</span> ...Or add the prefix "UTC:" or "GMT:" to your mask.</span>
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>UTC:h:MM:ss TT Z<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 10:46:21 PM UTC</span>

<span class="pl-c"><span class="pl-c">//</span> You can also get the ISO 8601 week of the year:</span>
<span class="pl-en">dateFormat</span>(now, <span class="pl-s"><span class="pl-pds">"</span>W<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 42</span>

<span class="pl-c"><span class="pl-c">//</span> and also get the ISO 8601 numeric representation of the day of the week:</span>
<span class="pl-en">dateFormat</span>(now,<span class="pl-s"><span class="pl-pds">"</span>N<span class="pl-pds">"</span></span>);
<span class="pl-c"><span class="pl-c">//</span> 6</span></pre>
