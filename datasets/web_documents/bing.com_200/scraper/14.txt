           <div class="user-html">
            <p>A highly flexible and efficient Laravel 5.x scraper package. Easily scrape data from anywhere on the web and build content.</p> 
            <h3 id="item-description__top-features">Top Features</h3> 
            <p>Scavenger provides the following features and more out of the box.</p> 
            <ul> 
             <li>Ease of use 
              <ul> 
               <li>Scavenger is super-easy to configure. Simple publish the config file and set your targets.</li> 
              </ul> </li> 
             <li>Scrape data from multiple sources at once.</li> 
             <li>Convert scraped data into useable Laravel model objects. 
              <ul> 
               <li>eg. You may scrape an article and have it converted into an object of your choice and saved in your database. Immediately available to your viewers.</li> 
              </ul> </li> 
             <li>You can easily perform one or more operations to each property of any scraped entity. 
              <ul> 
               <li>eg. You may call a paraphrase service from a model or package of your choice on data attributes before saving them to your database.</li> 
              </ul> </li> 
             <li>Data integrity constraints 
              <ul> 
               <li>Scavanger uses a hashing algorithm of your choice to maintain data integrity. This hash is used to ensure that one scrap (source article) is not converted to multiple output objects (model duplicates).</li> 
              </ul> </li> 
             <li>Console Command 
              <ul> 
               <li>Once scavenger is configured, a simple artisan command launches the seeker. Since this is a console command it is more efficient and timeouts are less likely to occur.</li> 
               <li>Artisan command: <code>php artisan scavenger:seek</code> </li> 
              </ul> </li> 
             <li>Schedule ready 
              <ul> 
               <li>Scavenger can easily be set to scrape on a schedule. Hence, creating a someone autonomous website is super easy!</li> 
              </ul> </li> 
            </ul> 
            <h3 id="item-description__documentation">Documentation</h3> 
            <p>Full documentation can be found at <a href="http://scavenger.reliqarts.com" rel="nofollow">scavenger.reliqarts.com</a>.</p>
