              Active directory - LDAP integration Issue with authenticating users in the multiple domain with one way Trust.
             <br> 
             <br> 
             <b>Say for example: </b>
             <br> 
             <br> 
             <div class="bullet">
               DomainA.Lan, DomainB.Net and DomainC.Com. 
             </div> 
             <div class="bullet">
               The DomainB.Net and &nbsp;DomainC.Com are oneway trusted to the main DomainA.Lan. 
             </div> 
             <div class="bullet">
               Other domain users i.e. DomainB.Net and DomainC.Com users were created in the DomainA.Lan as disabled linked users. 
             </div> 
             <div class="bullet">
               LDAP Directory value has been given as DomainA.Lan URL and also expected to authenticate users of DomainB.Net and DomainC.Com. up to now we can only able to authenticate users of DomainA.Lan. 
             </div> 
             <br> 
             <b>Problem: How to authenticate users in all the above specified domains.</b>
             <br> 
             <br> Solution to the above problem would be greatly appreciated.

                   Trying to simplify the issue with two different forest (only) having the need to establish the external trust (or one way trust). These articles shared points to note in the design and architecture aspects [4] which I see overall as establishing external trusts to enable user authentication to an external domain. Also all external trusts are one-way non-transitive trusts.
                  <br> 
                  <br> Assuming Windows Server, using the New Trust Wizard [1], it can be either a Realm trust (between an Active Directory domain and a Kerberos V5 realm) or Forest trust (between one forest root domain and another forest root domain). A non transitive trust would be good enough. &nbsp;It [2] listed having an account federation server (internal) and resource federation server (external) for the trust link transaction as well. The trust need to be establish from external to internal so as to allow access But before that, the trust need to be establish [3].
                  <br> 
                  <br> I am suspecting whether SID filtering would be the culprit, and rightfully should not be creating other domain (such as B and C) users in internal domain (such as A). The trust is established at Root DC, which will pass the "referral" of its child DC to the requestor (from external child DC). It may be any comms break of this "referral" that cause failure. Of course, we are assuming the firewall ports are open as needed [1]. 
                  <br> 
                  <br> I am not an expert in LDAP but hope to share insights as much 
                  <br> 
                  <br> 1 - Kerberos-Based Processing of Authentication Requests Over Forest Trusts
                  <br> @ 
                  <a href="http://technet.microsoft.com/en-us/library/cc773178%28WS.10%29.aspx" target="_blank">http://technet.microsoft.com/en-us/library/cc773178%28WS.10%29.aspx</a>
                  <br>
