       <h1 itemprop="name" class="devsite-page-title"> Using the Java Support Library UX Package </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p>This page describes how to use features of the Support Library UX package for Java. For more information about the features themselves, see our guide on <a href="https://developers.google.com/tango/ux/ux-framework">Tango UX</a>.</p> 
        <p>You should also take a look at our <a href="https://developers.google.com/tango/apis/java/ux/reference/index">Support Library UX reference pages</a>.</p> 
        <h2 id="installation_instructions">Installation instructions</h2> 
        <p>Follow the <a href="https://developers.google.com/tango/apis/java/index#add_to_your_own_project">instructions</a> to add the Tango Support Library for Java to your project. Make sure to include <strong>support-base</strong> as a dependency.</p> 
        <p>If you use your own build system, <a href="https://developers.google.com/tango/downloads">download</a> and include the Java Support Library package in your project.</p> 
        <h2 id="ux_package_tutorial">UX package tutorial</h2> 
        <p>These code snippets show how to use the UX package, and how to implement common tasks for Java applications.</p> 
        <h3 id="initialization">Initialization</h3> 
        <p>Make sure to handle any issues that come up during Tango startup.</p> 
        <pre class="prettyprint">
mTango = new Tango(MainActivity.this, new Runnable() {
    @Override
    public void run() {
        synchronized (MainActivity.this) {
            <strong>try {
                mConfig = setupTangoConfig(mTango);
                mTango.connect(mConfig);
                startupTango();
            } catch (TangoOutOfDateException e) {
                Log.e(TAG, getString(R.string.exception_out_of_date), e);
            } catch (TangoErrorException e) {
                Log.e(TAG, getString(R.string.exception_tango_error), e);
            } catch (TangoInvalidException e) {
                Log.e(TAG, getString(R.string.exception_tango_invalid), e);
            }</strong>
        }
    }
});
</pre> 
        <h4 id="starting_the_ux_framework">Starting the UX Framework</h4> 
        <p>To use the UX package in Java, add the <a href="https://developers.google.com/tango/apis/java/ux/reference/TangoUx">TangoUx</a> object. The <code>TangoUx</code> class manages all of the UX package functionality.</p> 
        <pre class="prettyprint notranslate" translate="no"><code>mTangoUx = new TangoUx(context);
</code></pre> 
        <p>Now that you've created a <code>TangoUx</code> object, you need to <code>start()</code> it:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>mTangoUx.start();
mTango.connect();
</code></pre> 
        <p>When you are disconnecting from the Tango service, make sure to also stop <code>TangoUx</code>:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>mTango.disconnect();
mTangoUx.stop();
</code></pre> 
        <h4 id="waiting_for_valid_data">Waiting for valid data</h4> 
        <p>As mentioned in the UX overview, you should wait for valid data before rendering an interactive UI. This code snippet only updates the camera when we get a valid pose. You could also show a loading or splash screen until we get the first valid pose.</p> 
        <pre class="prettyprint notranslate" translate="no"><code>TangoPoseData lastFramePose = TangoSupport.getPoseAtTime(0,
       TangoPoseData.COORDINATE_FRAME_START_OF_SERVICE,
       TangoPoseData.COORDINATE_FRAME_DEVICE,
       TangoSupport.TANGO_SUPPORT_ENGINE_OPENGL,
       TangoSupport.TANGO_SUPPORT_ENGINE_OPENGL,
       mDisplayRotation);
if (lastFramePose.statusCode == TangoPoseData.POSE_VALID) {
   mRenderer.updateCameraPose(lastFramePose);
}
</code></pre> 
        <h4 id="exception_event_listener">Exception Event Listener</h4> 
        <p>We'll cover this in more detail in the <a href="#exception_handling">Exception Handling</a> section.</p> 
        <h4 id="forwarding_the_sensor_data">Forwarding the sensor data</h4> 
        <p>You need to forward the Tango sensor data received by <a href="https://developers.google.com/tango/apis/java/reference/Tango.OnTangoUpdateListener">OnTangoUpdateListener</a> callbacks to the UX library:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>OnTangoUpdateListener listener = new
    OnTangoUpdateListener() {
        @Override
        public void onPoseAvailable(final TangoPoseData pose) {
            if (mTangoUx != null) {
                mTangoUx.updatePoseStatus(pose.statusCode);
            }
        }

        @Override
        public void onPointCloudAvailable(TangoPointCloudData pointCloud) {
          if (mTangoUx != null) {
            mTangoUx.updatePointCloud(pointCloud);
          }
        }

        @Override
        public void onTangoEvent(TangoEvent event) {
            if (mTangoUx != null) {
                mTangoUx.updateTangoEvent(event);
            }
        }

        @Override
        public void onFrameAvailable(int cameraId) {
            // ...
        }
    };
</code></pre> 
        <h3 id="ux_errors_and_exceptions">UX errors and exceptions</h3> 
        <p>The application can get notifications per event type by using the <a href="https://developers.google.com/tango/apis/java/ux/reference/UxExceptionEventListener">UxExceptionEventListener</a>. In this example, we're creating an exception event listener that will set a boolean value when the framework detects that the device is lying on a surface:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>private UxExceptionEventListener mUxExceptionEventListener = new UxExceptionEventListener() {
    @Override
    public void onUxExceptionEvent(UxExceptionEvent event) {
        if (event.getType() == UxExceptionEvent.TYPE_LYING_ON_SURFACE) {
            boolean isDeviceLyingOnSurface = event.getStatus() == UxExceptionEvent.STATUS_DETECTED;
        }
    }
};

mTangoUx.setUxExceptionEventListener(mUxExceptionEventListener);
</code></pre> 
        <p>Note that in the last line, we set an existing <code>TangoUx</code> to use the <code>UxExceptionEventListener</code> we just created.</p> 
        <p>For a list of the possible exception event types returned by <code>UxExceptionEvent.getType()</code>, see the Java reference page for <a href="https://developers.google.com/tango/apis/java/ux/reference/UxExceptionEvent">UxExceptionEvent</a>.</p> 
        <h4 id="hold_posture_exceptions">Hold Posture exceptions</h4> 
        <p>The Hold Posture is a special exception which specifys how the user should hold the device while using your application.</p> 
        <p>The available postures are:</p> 
        <ul> 
         <li><code>TYPE_HOLD_POSTURE_NONE</code>: All directions are allowed, exception never shown.</li> 
         <li><code>TYPE_HOLD_POSTURE_FORWARD</code>: The device should point forward.</li> 
         <li><code>TYPE_HOLD_POSTURE_UP</code>: The device should point up, towards the sky.</li> 
         <li><code>TYPE_HOLD_POSTURE_DOWN</code>: The device should point down, towards the ground.</li> 
        </ul> 
        <p>The default posture is <code>TYPE_HOLD_POSTURE_NONE</code>.</p> 
        <p>Holding device in an incorrect way will result in a <code>UxExceptionEvent</code> of type <code>TYPE_HOLD_POSTURE</code>.</p> 
        <p>You can set a different posture anytime during the app lifecycle by calling <code>mTangoUx.setHoldPosture(TYPE);</code>, where <code>TYPE</code> is one of the four posture options. You should change the hold posture whenever necessary. For example, imagine a game has a setup phase where you point the tablet down to find the floor, then a play phase where you can roam around freely. The hold posture should be set to <code>TYPE_HOLD_POSTURE_DOWN</code> during the setup phase and <code>TYPE_HOLD_POSTURE_NONE</code> during the play phase.</p>
