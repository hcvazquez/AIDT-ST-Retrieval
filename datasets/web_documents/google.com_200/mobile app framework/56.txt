        <div class="post-body__content">
         <p>Vue is a&nbsp;popular JavaScript framework for web apps with easy binding between data in memory and the user interface. Now Weex allows us to code native mobile apps using the Vue framework!<br></p>
         <p>Why do we use Vue and other data binding frameworks like Angular and React? What's so special about them? &nbsp;We use them to ensure that application data which is stored in memory stays in sync with the user interface and vice versa.&nbsp;</p>
         <p>We also use these frameworks to enable us to build applications very quickly, and in a way that is optimized for performance.</p>
         <p>In this tutorial I'll show you how to use the Vue framework, in particular how to understand its concepts of data binding and templates. Then, I'll go on to introduce the Weex platform, for coding native mobile apps using Vue!</p>
         <h2>Data Binding</h2>
         <p>Let's look at a simple example to understand how these frameworks can actually save us time. What we want is a simple input field that stays in sync with our application data. Our app itself can change the data programmatically, and the user can change it via user input, so we need to be watching both the UI and application data.</p>
         <figure class="post_image">
          <img alt="Two way data binding" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Data-Binding-Diagram.png">
         </figure>
         <p>Writing code that would support this data binding would be verbose. We would need to create event listeners and proxy objects and observables&nbsp;to capture any changes in the application data. And these complexities just grow and grow as more types of data and inputs are added. Vue and other data binding frameworks prevent us having to write all of that binding code.&nbsp;</p>
         <p>With Vue, if our user changes some input data, it will sync back to the application data like so:</p>
         <figure class="post_image">
          <img alt="Changes flowing from the UI to the model" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-name-change.png">
         </figure>
         <p>Or if the application changes the data, it will update the user interface like this:</p>
         <figure class="post_image">
          <img alt="Changes flowing from the model to the UI" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-app-data-change.png">
         </figure>
         <p>When we keep the&nbsp;user interface and the app data together in sync, it means our app does exactly what we expect it&nbsp;to do. Vue will manage all of this and allow other powerful data binding processes to occur.</p>
         <h2>Getting Set Up</h2>
         <p>Now that we know why we use these frameworks, let's set up a simple Vue application to&nbsp;run some examples. Create a new HTML file anywhere on your computer and paste the following code into it:</p>
         <pre class="brush: html noskimlinks noskimwords">&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
  &lt;title&gt;Tuts+ Vue&lt;/title&gt;
  &lt;script src="https://unpkg.com/vue/dist/vue.min.js"&gt;&lt;/script&gt;
&lt;/head&gt;
&lt;body&gt;

  &lt;div id="app"&gt;
    
  &lt;/div&gt;

  &lt;script&gt;
    var app = new Vue({ el: '#app' });
  &lt;/script&gt;

&lt;/body&gt;
&lt;/html&gt;</pre>
         <p>This is&nbsp;a simple HTML file that links to the Vue JS&nbsp;library. It contains a <code class="inline">div</code> element with the ID of <code class="inline">app</code>. Inside the script tags we have a variable called <code class="inline">app</code> that we use to&nbsp;point to our viewI'll explain this part later.</p>
         <p>Then we create a new Vue instance, or "view". This constructor tells the framework which element will contain our application UI and keep it in sync with the application data.&nbsp;</p>
         <p>Now double click on the HTML file to open it in the browser, and open the browser console window.</p>
         <figure class="post_image">
          <img alt="The browser console in Chrome" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-blank.png">
         </figure>
         <h2>Text Interpolation</h2>
         <p>Text interpolation lets us&nbsp;embed expressions in our HTML code that will be interpreted when the page is rendered. The expressions are live, so if the data they depend on is changed, the page will be updated in real time. This is known as declarative rendering, which allows us to place expressions anywhere within an element's content. Let's review with a simple example.</p>
         <h4>JS</h4>
         <pre class="brush: javascript noskimlinks noskimwords">&lt;script&gt;
    var app = new Vue({
        el: '#app',
        data:
        {
            message: "Hello world"
        }
    });
&lt;/script&gt;</pre>
         <p>Our JavaScript code now contains a <code class="inline">data</code> object that will store all of our application data for this view. Within it, I've created the <code class="inline">message</code> property with the string <code class="inline">"Hello world"</code>.&nbsp;</p>
         <p>Next, let's display this property in the HTML.</p>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;
  &lt;pre style="font-family: 'helvetica'; "&gt;
      {{ message }}
      
      {{ message.split('').reverse().join('') }}
    &lt;/pre&gt;
&lt;/div&gt;</pre>
         <p>The expressions are indicated with double braces syntax. Vue JS manages the dependencies of each expression and updates the page live if they change. In our case, we have two expressions, with&nbsp;the <code class="inline">message</code> property as a dependency of each one. The first expression <code class="inline">{{ message }}</code>&nbsp;just displays the message property. The second <code class="inline">{{ message.split('').reverse().join('') }}</code>&nbsp;expression takes the message property, and then:</p>
         <ul> 
          <li>Splits the string into an array:<br><code class="inline">[ "h","e","l","l","o"," ","w","o","r","l","d"&nbsp;]</code> </li> 
          <li>Reverses the order of the array:<br><code class="inline">[ "d","l","r","o","w"," ","o","l","l","e","h"&nbsp;]</code> </li> 
          <li>Joins the array together to get&nbsp;<code class="inline">"dlrow olleh"</code>, which is the same message but printed backwards.</li> 
         </ul>
         <p>Now open the browser and refresh the HTML page. Then open the browser's console window and assign <code class="inline">app.message</code>&nbsp;a new value and see how the output is updated in the browser.</p>
         <figure class="post_image">
          <img alt="Simple Vue template example" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-prop-message.png">
         </figure>
         <p>You can also use math in template expressions, or structured data in objects. Actually, as you might have guessed, Vue expressions are just JavaScript. Access to global variables is sandboxed, though, so you can access <code class="inline">Math</code>, but not <code class="inline">window</code>. This helps keep your apps safeand your users too.&nbsp;</p>
         <p>By using Vue&nbsp;with just a few lines of code, we can have advanced two-way data binding. This ensures that the user interface and the application data are in sync with relative ease.&nbsp;</p>
         <h2>Directives</h2>
         <p>Directives are another way&nbsp;of binding data in our views. We can use them to&nbsp;bind application data to attributes, events, input&nbsp;fields, and repeatable data. Let's go through each directive and discover how they work.</p>
         <ul> 
          <li> <code class="inline">v-bind</code>: bind an attribute value</li> 
          <li> <code class="inline">v-model</code>: bind app data to an input element like a <code class="inline">textarea</code> </li> 
          <li> <code class="inline">v-on</code>: define an event handler</li> 
          <li> <code class="inline">v-for</code>: bind to an array or object</li> 
         </ul>
         <h3> <code class="inline">v-bind</code> Directive</h3>
         <p>The <code class="inline">v-bind</code>&nbsp;directive is used for attribute data binding. It allows us to bind application data to an attribute's value. Think of it like text interpolation but within an&nbsp;attribute. You can bind&nbsp;<code class="inline">class</code>,&nbsp;<code class="inline">id</code>,&nbsp;<code class="inline">value</code>&nbsp;or any other attribute with the&nbsp;<code class="inline">v-bind</code>&nbsp;directive.&nbsp;</p>
         <p>For this&nbsp;example, I would like to bind our application data to the <code class="inline">data-attr</code> attribute using the <code class="inline">v-bind</code> directive <code class="inline">v-bind:data-attr</code>.<br></p>
         <h4>JS</h4>
         <pre class="brush: javascript noskimlinks noskimwords">&lt;script&gt;
    var app = new Vue({
        el: '#app',
        data:
        {
            number: 20,
            object:
            {
                type: "Full Name Object",
                names: ["Lawrence","Anothy","Turton"]
            }
        }
    });
&lt;/script&gt;</pre>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;

  &lt;div v-bind:data-attr=" object.type "&gt;
    {{ object.type }} 
  &lt;/div&gt;

  &lt;div v-bind:data-attr=" object.names "&gt;
    {{ object.names }}  
  &lt;/div&gt;

  &lt;div v-bind:data-attr=" number * 20 / 2 "&gt;
    {{ number * 20 / 2 }}
  &lt;/div&gt;

  &lt;div v-bind:data-attr=" [ object.type, object.names, number * 20 / 2 ] "&gt;
    {{ [ object.type, object.names, number * 20 / 2 ] }}
  &lt;/div&gt;

&lt;/div&gt;</pre>
         <p>In the example above, we bound an attribute called <code class="inline">data-attr</code> to a series of different data sources. First, we bound it to the <code class="inline">type</code>&nbsp;and <code class="inline">names</code> properties of <code class="inline">object</code>. Then we bound it to a math expression, and then joined all the bindings together into a single array attribute.</p>
         <p>Take a look at the output in the browser: you can click on the <strong>Elements</strong> tab to see the output of each attribute's value.</p>
         <figure class="post_image">
          <img alt="Binding attribute values" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-attributes.png">
         </figure>
         <p>Again, you can change the application data in the console to see the data binding process.</p>
         <figure class="post_image">
          <img alt="Changes to the model are rendered automatically" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-change-v-bindtt.png">
         </figure>
         <p>Think&nbsp;of the <code class="inline">v-bind</code> directive as allowing expressions in attribute values. This is incredibly powerful and easy to establish with the Vue library.<br></p>
         <h3> <code class="inline">v-model</code> Directive</h3>
         <p>This directive is used specifically to keep the application data in sync with input fields, text areas, and&nbsp;selection elements. Here's an example:</p>
         <h4>JS</h4>
         <pre class="brush: javascript noskimlinks noskimwords">&lt;script&gt;
    var app = new Vue({
        el: '#app',
        data:
        {
          message: "message string",
          selected:"",
          checkedNames: []
        }
    });
&lt;/script&gt;</pre>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;

  &lt;input type="text" v-model="message"&gt;

  &lt;textarea v-model="message"&gt;&lt;/textarea&gt;

  &lt;hr&gt;

  &lt;select v-model="selected"&gt;
    &lt;option disabled value=""&gt;Select a name&lt;/option&gt;
    &lt;option&gt;Jack&lt;/option&gt;
    &lt;option&gt;John&lt;/option&gt;
    &lt;option&gt;Mike&lt;/option&gt;
  &lt;/select&gt;
  &lt;span&gt;Selected: {{ selected }}&lt;/span&gt;

  &lt;hr&gt;

  &lt;input type="checkbox" id="jack" value="Jack" v-model="checkedNames"&gt;
  &lt;label for="jack"&gt;Jack&lt;/label&gt;
  &lt;input type="checkbox" id="john" value="John" v-model="checkedNames"&gt;
  &lt;label for="john"&gt;John&lt;/label&gt;
  &lt;input type="checkbox" id="mike" value="Mike" v-model="checkedNames"&gt;
  &lt;label for="mike"&gt;Mike&lt;/label&gt;

  &lt;select v-model="checkedNames" multiple&gt;
    &lt;option&gt;Jack&lt;/option&gt;
    &lt;option&gt;John&lt;/option&gt;
    &lt;option&gt;Mike&lt;/option&gt;
  &lt;/select&gt;
  &lt;br&gt;
  &lt;span&gt;Selected Names: {{ checkedNames }}&lt;/span&gt;

&lt;/div&gt;</pre>
         <p>For the first section of this page, we have a text input field and text area, both of which have the <code class="inline">v-model</code> directive that keeps these input fields in sync with the <code class="inline">message</code>&nbsp;property, using&nbsp;<code class="inline">v-model="message"</code>. That way, if you change either one, the model will be updated and then the other input field will be updated. Go ahead and try it out!</p>
         <figure class="post_image">
          <img alt="Binding to input elements" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-v-model-1.png">
         </figure>
         <p>In the next section we have a drop-down synced to the&nbsp;<code class="inline">selected</code> property with&nbsp;&nbsp;<code class="inline">v-model="selected"</code>. That way, if the selection is changed, our model will be updated.</p>
         <figure class="post_image">
          <img alt="" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-v-model-2.png">
         </figure>
         <p>Finally, we have a multiple selection. I've shown how to do this in two ways: with multiple checkboxes and with a&nbsp;multiple select input. As we can have multiple values, I've created the&nbsp;<code class="inline">checkedNames</code> array that can store multiple values. The multiple select input and the checkboxes each have&nbsp;a <code class="inline">v-model</code> directive targeting the&nbsp;<code class="inline">checkNames</code>&nbsp;property. Give it a try, and they should stay in sync as you make selections.</p>
         <p><img alt="Multiple selection binding" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-v-model-3.png"></p>
         <h3>The <code class="inline">v-on</code>&nbsp;Directive<br> </h3>
         <p>This directive allows us to attach events to HTML elements. This directive can&nbsp;be used on certain elements that can have input events attached to them like hovering, clicks, text input,&nbsp;key presses, and more. For this example, we'll create&nbsp;a simple button with a click event.</p>
         <h4>JS</h4>
         <pre class="brush: javascript noskimlinks noskimwords">&lt;script&gt;
    var app = new Vue({
        el: '#app',
        data:
        {
          clicked: ""
        }
    });
&lt;/script&gt;</pre>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;

  &lt;button v-on:click=" [ alert('hello'), clicked = 'hello' ] "&gt;Click me!&lt;/button&gt;

  {{ clicked }}

&lt;/div&gt;</pre>
         <p>In the&nbsp;<code class="inline">v-on</code>&nbsp;directive, after the colon, we specify the event we want to attach. In this example,&nbsp;when the&nbsp;<code class="inline">click</code> event is fired, we'll&nbsp;run some expressions.&nbsp;First, we open an alert dialog box, and then we change the value of the <code class="inline">clicked</code> property. Note that you can call functions in <code class="inline">v-on</code> expressions.</p>
         <figure class="post_image">
          <img alt="A button with a click event" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-v-on-1.png">
         </figure>
         <h3> <code class="inline">v-for</code> Directive</h3>
         <p>This directive is one of the most powerful of them all. We can watch any object or array for changes and render&nbsp;repeatedly a part of our template code&nbsp;for each property or element found in that object or&nbsp;array. For example, to render an array of names as a list:</p>
         <h4><strong>JS</strong></h4>
         <pre class="brush: javascript noskimlinks noskimwords">&lt;script&gt;
    var app = new Vue({
        el: '#app',
        data:
        {
          names: [ "Lawrence", "John", "Mike", "Justin" ]
        }
    });
&lt;/script&gt;</pre>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;

  &lt;ul&gt;
    &lt;li v-for="name in names"&gt;{{ name }}&lt;/li&gt;
  &lt;/ul&gt;

&lt;/div&gt;</pre>
         <p>In this example, we first attach the <code class="inline">v-for</code> directive onto the element that you want to repeat. The value of the directive specifies the object or array we want to iterate over (<code class="inline">names</code>) as well as a variable that will contain the value for each iteration (<code class="inline">name</code>).&nbsp;Now, within this repeated element, we can use the&nbsp;<code class="inline">name</code> variable in expressions.</p>
         <p>If we then modify the array, for example with a method like&nbsp;<code class="inline">push</code>, <code class="inline">sort</code>&nbsp;or <code class="inline">reverse</code>, the template data will automatically be updated. Try running the example and modify the array in the console window.</p>
         <figure class="post_image">
          <img alt="Rendering a list of elements with the v-for directive" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-v-for.png">
         </figure>
         <h2>Components</h2>
         <p>The data binding process happens in views. Components are just reusable views that we can repeat throughout our application. Each component must have some template data to be rendered as part of the user interface and some application data. You can optionally include some styling if you want.</p>
         <p>Let's use the <code class="inline">v-for</code> directive again to output list items. But this time, we'll make each item a component. We'll give this component a name so that we can target it within our template code:&nbsp;<code class="inline">list-item</code>. Components can also have "registered properties", to which we can assign a value to be&nbsp;assigned when we create the component. In my case, the <code class="inline">list-item</code> component will expect to receive a value for the <code class="inline">name</code>&nbsp;property, defined with&nbsp;<code class="inline">props:["name"]</code>.&nbsp;</p>
         <p>Let's now&nbsp;create this component. Please add the following code into your script tags:</p>
         <h4>JS</h4>
         <pre class="brush: javascript noskimlinks noskimwords">Vue.component('list-item', {
  template: '&lt;li&gt;{{ name }}&lt;/li&gt;',
  props: [ "name" ]
});</pre>
         <p>Also make sure you have the&nbsp;following application data:</p>
         <pre class="brush: javascript noskimlinks noskimwords">var app = new Vue({
    el: '#app',
    data:
    {
        names: [ "Lawrence", "John", "Mike", "Justin" ]
    }
});</pre>
         <h4>HTML</h4>
         <pre class="brush: html noskimlinks noskimwords">&lt;div id="app"&gt;
  &lt;ul&gt;
    &lt;list-item v-for="name in names" v-bind:name="name"&gt;&lt;/list-item&gt;
  &lt;/ul&gt;
&lt;/div&gt;</pre>
         <p>Here you can see the&nbsp;<code class="inline">list-item</code> component will be repeated for each name found in the array. Each name will be assigned to the name variable which is defined in the <code class="inline">v-for</code> directive. We're passing the <code class="inline">name</code> property to the component using <code class="inline">v-bind</code>.</p>
         <p>Each instance is just a&nbsp;copy of the component. We can modify each copy or instance independently of one another. So think of&nbsp;components as the blueprint and the instances as copies made from the blueprint. Here is the end result:</p>
         <figure class="post_image">
          <img alt="List rendered with components" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-component.png">
         </figure>
         <h2>Compilation and the Vue CLI</h2>
         <p>We've been working with a very simple HTML file and letting Vue interpret everything at run-time. However, Vue does come with a command-line tool. You may be asking yourself why. One reason is that pre-compiled page templates will perform better&nbsp;than templates that Vue interprets at run-time. Another reason is that, if we were to build a large-scale application and&nbsp;try to fit it into one HTML file, it would quickly become unmanageable.&nbsp;</p>
         <p>So we need modularity to break down a large application into smaller pieces.&nbsp;</p>
         <h3>Installing the Vue CLI</h3>
         <p>To use the command line, you'll need to open up the console window for your operating system.</p>
         <ul> 
          <li>For Mac, press <strong>Command-Space</strong>&nbsp;and then type <strong>terminal</strong> and hit <strong>Return</strong>.</li> 
          <li>For Windows, search for the command prompt in the startup menu, make sure to&nbsp;right click and "<strong>open as administrator"</strong>.</li> 
          <li>For Linux, press <strong>Control-Alt-T</strong>.</li> 
         </ul>
         <p>Before proceeding, make sure you have the latest version of Node.js installed. Then we need to install Webpack, which&nbsp;will compress our project's file size, making it faster in our browser. Then we can install the Vue CLI and run the relevant&nbsp;commands for your platform:</p>
         <h4>Mac &amp; Linux</h4>
         <ul> 
          <li><code class="inline">sudo npm install -g webpack</code></li> 
          <li><code class="inline">sudo npm install -g vue-cli</code></li> 
         </ul>
         <h4>Windows (make sure you&nbsp;run the console as administrator)</h4>
         <ul> 
          <li><code class="inline">npm install -g webpack</code></li> 
          <li><code class="inline">npm install -g vue-cli</code></li> 
         </ul>
         <p>That's it! We're&nbsp;now ready to start setting up our advanced Webpack project. Firstly, navigate to the directory where we want to create our project, in my case the desktop,&nbsp;and then create the project. You can replace <strong>myapp</strong> with any project name you'd like.</p>
         <ul>
          <li> <code class="inline">vue init webpack </code><strong><code class="inline">myapp</code></strong><br> </li>
         </ul>
         <p>You will be led through a series of questions about your project, prompting you to fill in data like title and description and asking you whether to install third-party plugins. For this example, you can answer no to all of the optional extras. Once it's created, we need to install the dependencies, so navigate to the project directory, and we'll run the install command.</p>
         <ul> 
          <li> <code class="inline">cd </code><strong><code class="inline">myapp</code></strong> </li> 
          <li><code class="inline">npm install</code></li> 
         </ul>
         <p>Let NPM install all the dependency packages, and then we're ready to carry on.&nbsp;</p>
         <h3>Creating a Vue Project</h3>
         <p>Once the packages have been installed, we can then run the development server by typing <code class="inline">npm run dev</code>. Your browser window should open, displaying the&nbsp;following page.</p>
         <figure class="post_image">
          <img alt="The Vue starter project welcome screen" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Browser-vue-browser.png">
         </figure>
         <p>We won't go through the structure of the entire project, but when you open up the <strong>myapp</strong>&nbsp;directory you will see the&nbsp;<strong>src</strong> directory. It contains an <strong>App.vue </strong>file, a&nbsp;<strong>main.js</strong>&nbsp;file and, in the <strong>components</strong> directory, the&nbsp;<strong>Hello.vue</strong> file.</p>
         <p>The Vue files are components. The <strong>main.js</strong> file configures the initial view and potentially other configurations as well.&nbsp;Let's take a&nbsp;look at the <code class="inline">App.vue</code> and <code class="inline">Hello.vue</code> files.</p>
         <figure class="post_image">
          <img alt="The vue files from the Vue starter project" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Vue-file-1.png">
         </figure>
         <p>Here you can see each Vue component is broken down into three parts:</p>
         <ul> 
          <li> <code class="inline">&lt;template&gt;</code>: the HTML markup that makes up part of the UI.</li> 
          <li> <code class="inline">&lt;script&gt;</code>: application data, filters, methods, computed properties, watchers, and methods.</li> 
          <li> <code class="inline">&lt;style&gt;</code>: the CSS or Sass styling of our components.</li> 
         </ul>
         <h3>Compilation</h3>
         <p>Compiling all the components together will produce a large-scale application. This means when we're developing, we work on small modular pieces of code, called components, instead of the entire&nbsp;application. In Vue we have single-file components that contain the JavaScript, HTML, and&nbsp;CSS. It'll even take care of transpiling ES6 to ES5, Sass to CSS and Jade to HTML if you choose to use those languages.</p>
         <p>You'll notice in the <strong>App.vue</strong> file, I've highlighted where it imports the <strong>Hello.vue</strong> component. Components can be nested inside other components!&nbsp;</p>
         <p>When compiling, our project is built with the following process:</p>
         <figure class="post_image">
          <img alt="Vue build process" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Compilation.png">
         </figure>
         <p>We're still using the same syntax as in the earlier examples. However, we are now working with smaller Vue files and&nbsp;compiling them together. We also have some more advanced features like the preprocessors and Jade, Sass, ES6 and Webpack compression.</p>
         <h2>Weex</h2>
         <p>Now that we understand the Vue library, let's take a quick look to see how we can take our web app and turn it into a native mobile app, installable&nbsp;on iOS or Android.&nbsp;</p>
         <p>Weex&nbsp;is a hybrid framework, meaning it allows multiple technologies, just as a hybrid car can use both electric and petrol. In our case, we&nbsp;use JavaScript code from our web app, but we render&nbsp;to a native user interface. Also, from JavaScript we can access the&nbsp;device's native&nbsp;APIs to access hardware like the camera, sensors, and file system.</p>
         <figure class="post_image">
          <img alt="Parts of a Weex app" src="https://cms-assets.tutsplus.com/uploads/users/1757/posts/28782/image/Weex-1.png">
         </figure>
         <p>With Weex, instead of rendering our application to HTML, we use XML. Actually, HTML and XML are very similar, so the syntax will look familiar. This XML will then be converted to native components. Now our&nbsp;JavaScript will be able to talk to those&nbsp;native components just like it talks to the DOM in Vue with HTML. Not only that, but native components&nbsp;can be styled and positioned&nbsp;with CSS including&nbsp;animations, transitions and more, and they&nbsp;can be&nbsp;integrated to the&nbsp;native layer.</p>
         <p>That's just a teaser of what Weex can do. In the next tutorials, I'll take you more in depth with Weex, and we'll see how to build some practical cross-platform native apps.</p>
         <h4><strong>Pros</strong></h4>
         <p>Vue has a simple&nbsp;syntax and is&nbsp;increasing in popularity very quickly. Weex allows us to build mobile apps using&nbsp;JavaScript and Vue, but still deliver a native app experience.&nbsp;</p>
         <h4>Cons</h4>
         <p>Vue itself is stable, but Weex is still in early developmentit currently resides in the Apache incubator. But don't worry, Weex will be production ready soon, and it's backed by the tech giant Alibaba. So, if building on a developer preview concerns you, you might&nbsp;wait until Weex reaches full release.&nbsp;</p>
         <h2>Conclusion</h2>
         <p>You've now seen how data binding is a key reason for using Vue and similar frameworks. They save us development time and provide a standard way to build applications. You've also seen how to use text interpolation, directives, and events in Vue, and how these features work together to keep app data in sync with the user interface.</p>
         <p>We started out with a basic project with only one view. But then we created a more advanced project that had smaller, more modular views called components. Components let us break our project down so it is easier to code and maintain. After, we saw how to use the Vue compiler to allow transpilation of&nbsp;ES6, Jade and Sass into the standard languages, while compressing file sizes to the minimum.</p>
         <p>All of this knowledge will help you when you start coding with Weex. Weex will allow us to take our Vue web app and turn it into a mobile app. Weex is a bit differentwe use XML with special components, instead of HTMLbut we can still use other web technologies, like CSS and JavaScript. That means we don't have to change our application logic from web app to mobile app.</p>
         <p>Stay tuned for my next post on Weex development, and in the meantime, check out some of our other posts on mobile app development with web technologies.</p>
