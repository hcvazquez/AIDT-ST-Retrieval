           <p><a href="https://www.twilio.com/blog/wp-content/uploads/2012/04/lock1.png"><img class="alignright size-full wp-image-9304" title="lock" alt="" src="https://www.twilio.com/blog/wp-content/uploads/2012/04/lock1.png" width="151" height="193"></a></p> 
           <p>Two-factor authentication is becoming the go-to method for increased login security on the web, as it ties a physical device to your virtual credentials. It is being adopted by many of the major players in the industry, including <a href="http://www.twilio.com/gallery/customers/intuit" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://www.twilio.com/gallery/customers/intuit', 'Intuit');">Intuit</a>,&nbsp;<a href="http://support.google.com/accounts/bin/answer.py?hl=en&amp;answer=180744" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://support.google.com/accounts/bin/answer.py?hl=en&amp;answer=180744', 'Google');">Google</a>&nbsp;and&nbsp;<a href="https://www.facebook.com/note.php?note_id=10150172618258920" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://www.facebook.com/note.php?note_id=10150172618258920', 'Facebook');">Facebook</a>. While user security is paramount, modern web developers must also consider the UX problems that can arise from increased security measures.</p> 
           <p>Since UX is becoming increasingly important, I set out to design a more interactive and more modern method of two-factor authentication. In this post, I’ll outline how you can implement an interactive two-factor authentication solution using&nbsp;<a href="http://nodejs.org/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://nodejs.org/', 'Node.js');">Node.js</a>,&nbsp;<a href="http://socket.io/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://socket.io/', 'Socket.io');">Socket.io</a>,&nbsp;<a href="http://expressjs.com/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://expressjs.com/', 'Express');">Express</a>, and&nbsp;<a href="https://github.com/sjwalter/node-twilio" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/sjwalter/node-twilio', 'Node-Twilio');">Node-Twilio</a>&nbsp;(3rd-party). If you are interested in a solution using PHP and jQuery, please check out&nbsp;<a href="https://www.twilio.com/blog/2010/05/simple-phone-verification-with-twilio-php-mysql-and-jquery.html">this past blog post</a>. I will highlight the most important parts of implementing an interactive solution in this blog post, and if you’re interested in playing with a fully functional demo please fork my repository on GitHub!</p> 
           <p><strong>Steps</strong></p> 
           <ol> 
            <li>User attempts to log in to your website</li> 
            <li>A random verification code is generated and the application initiates a call to the user’s verified phone number</li> 
            <li>The user is prompted to enter the code, and given real-time feedback</li> 
            <li>If the code is entered successfully, user is forwarded to their account dashboard</li> 
            <li>If the code is entered incorrectly, the user must log in again</li> 
           </ol> 
           <p><strong>Step One: Getting Started</strong></p> 
           <p>To get started, you will need a public web server with Node.js installed. If you are new to Node.js, I recommend reading our article on <a href="https://www.twilio.com/blog/2011/05/getting-started-with-twilio-and-node-js.html">Getting Started with Twilio and Node.js</a>, as it explains how to get running for the first time. Once you have that all set up, you will need to install a couple of additional packages by running the following commands in your Terminal:</p>

           <p><strong>Step Two: Frontend</strong></p> 
           <p>Now that your environment is all set up, we can start writing our app. The first step is to write the <a href="http://jade-lang.com/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://jade-lang.com/', 'Jade');">Jade</a> template code that will generate the HTML for your login page. Create a folder called ‘views’ and a file named index.jade within it. The following code goes in that file:<br> <script type="text/javascript" src="https://gist.github.com/2378931.js?file=index-pt1.jade"></script>This code may look somewhat daunting if you are not familiar with Jade, but in reality most of the tags correspond directly to standard HTML tags, with shorthand for setting IDs and Classes for CSS styles. For example, div.myclass#myid is essentially the same as writing ‘&lt;div id=”myid” class=”myclass”&gt;&lt;/div&gt;’ You may also notice that I am including the JavaScript and CSS files for&nbsp;<a href="http://twitter.github.com/bootstrap/" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://twitter.github.com/bootstrap/', 'Twitter Bootstrap');">Twitter Bootstrap</a>. This is&nbsp;purely for the purpose of making this demo look more visually appealing. If you wish to include this portion of the application, you can <a href="http://twitter.github.com/bootstrap/assets/bootstrap.zip" onclick="__gaTracker('send', 'event', 'download', 'http://twitter.github.com/bootstrap/assets/bootstrap.zip');">download the bootstrap files</a>&nbsp;and extract them into the ‘bootstrap’ directory within your main application directory.</p> 
           <p>Now that we have defined the look and feel of our hypothetical login form, we can now add the frontend JavaScript that will make it feel interactive to the end user. I’m going to split the JavaScript code into multiple parts and explain each snippet individually so that it will be clear what is going on.<br> <script type="text/javascript" src="https://gist.github.com/2378931.js?file=frontend-pt1.js"></script><br> This code handles the submission of the form (when the user clicks the Log In button). It first prevents the actual submission event, which by default makes a POST request to the address specified in the ‘action’ parameter of the &lt;form&gt; tag. We then emit three socket events, each with a specific action. The first and second send the username and password data along to the server and the third sends along the user’s phone number. If this example were running in production you would likely have the user’s phone number saved in the database along with their account information, but for the purposes of this demo we will specify it in the login form.</p> 
           <p><script type="text/javascript" src="https://gist.github.com/2378931.js?file=frontend-pt2.js"></script>This next snippet of code handles the ‘newcode’ event, which is sent from the server to the client upon generation of a new verification code. This handles the display of the code on the frontend and the initialization of all CSS classes associated with it so that the user can get started with the authentication process. And this last, larger snippet of JavaScript code handles the events associated with the actual authentication process. When each digit is entered, the server checks to see if the code is valid and then emits the proper socket event so that the user can see a real-time visual response. Now that the frontend portion of this application is complete, I recommend checking out the entire <a href="https://gist.github.com/raw/2378931/6f64ccb0ddedb4df9344a0d40a229d5d3371ced0/index.jade" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://gist.github.com/raw/2378931/6f64ccb0ddedb4df9344a0d40a229d5d3371ced0/index.jade', 'index.jade file');">index.jade file</a> so that you can see how all of the pieces fit together.</p> 
           <p><strong>Step Three: Backend</strong></p> 
           <p><strong></strong>Now, in your main application directory, create a file called app.js – this is the primary Node.js application file. The first portion of this file deals with configuration of Express, User Sessions, and Node-Twilio. You will need to make changes here and include your Twilio account credentials, your server hostname, and the outgoing phone number that you will be using for verification. You can also change the session secret and key as well as the application port, though it is not necessary in order to run the demo.</p> 
           <p><script type="text/javascript" src="https://gist.github.com/2378931.js?file=app-pt1.js"></script>This next snippet of code handles the events that are emitted when the user clicks on the Log In button on the frontend. It demonstrates how you can save the username and password into session data. In a production application, I would recommend encrypting the password on the frontend and NOT storing it in session information. However, to show how session data is stored and retrieved I have included it in this tutorial.&nbsp;<script type="text/javascript" src="https://gist.github.com/2378931.js?file=app-pt2.js"></script>And now for the fun part! This session event ‘init phone’ is emitted when the user logs in, and it handles the initiation of the phone call to the end user so that they can authenticate themselves. The first important thing to notice are that you can change the length of the verification code merely by modifying a variable, ‘codelength’. The second thing to notice is that rather than defining a callback URL that renders TwiML (such as in the PHP, Ruby, or Python helper libraries), in Node.js this URL is generated and passed along dynamically. This creates a unique event-based flow where we can simply say that when a call is ‘answered’ we want to execute the following code. Within this event handler we also Gather the first Digit that the user enters for their verification code.</p> 
           <p><script type="text/javascript" src="https://gist.github.com/2378931.js?file=app-pt3.js"></script>The last portion of our Node.js application is a function that handles each Digit that the user enters. It is a recursive function that verifies each Digit as it is entered and emits the proper events so that the user sees real-time feedback on the frontend. Since we are Gathering a single Digit at a time, the user experience feels as if the user is interacting with the login form in real time.</p> 
           <p>Now that we have gone over each portion of our app.js file in detail, I recommend checking out the <a href="https://raw.github.com/gist/2378931/825581402071e5bfa02ccdc64248c9b42b3b3fd3/app.js" onclick="__gaTracker('send', 'event', 'download', 'https://raw.github.com/gist/2378931/825581402071e5bfa02ccdc64248c9b42b3b3fd3/app.js');">final file</a> so that you can see how all of the pieces fit together.</p>
