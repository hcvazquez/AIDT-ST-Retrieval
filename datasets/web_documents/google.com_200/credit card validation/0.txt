           </svg></a>Credit Card Validator <a href="https://travis-ci.org/braintree/card-validator"><img src="https://camo.githubusercontent.com/3e51537e21c9b4d429d4920be4976d48766cdeed/68747470733a2f2f7472617669732d63692e6f72672f627261696e747265652f636172642d76616c696461746f722e737667" alt="Build Status" data-canonical-src="https://travis-ci.org/braintree/card-validator.svg" style="max-width:100%;"></a> <a href="http://badge.fury.io/js/card-validator"><img src="https://camo.githubusercontent.com/78effc14c0c54a01feece9fd117e5e333da56bd5/68747470733a2f2f62616467652e667572792e696f2f6a732f636172642d76616c696461746f722e737667" alt="npm version" data-canonical-src="https://badge.fury.io/js/card-validator.svg" style="max-width:100%;"></a></h1> 
         <p>Credit Card Validator provides validation utilities for credit card data inputs. It is designed as a CommonJS module for use in Node.js, io.js, or the <a href="http://browserify.org/">browser</a>. It includes first class support for 'potential' validity so you can use it to present appropriate UI to your user as they type.</p> 
         <p>A typical use case in a credit card form is to notify the user if the data they are entering is invalid. In a credit card field, entering “411” is not necessarily valid for submission, but it is still potentially valid. Conversely, if a user enters “41x” that value can no longer pass strict validation and you can provide a response immediately.</p> 
         <p>Credit Card Validator will also provide a determined card type (using <a href="https://github.com/braintree/credit-card-type">credit-card-type</a>). This is useful for scenarios in which you wish to render an accompanying payment method icon (Visa, MasterCard, etc.). Additionally, by having access to the current card type, you can better manage the state of your credit card form as a whole. For example, if you detect a user is entering (or has entered) an American Express card number, you can update the <code>maxlength</code> attribute of your <code>CVV</code> input element from 3 to 4 and even update the corresponding <code>label</code> from 'CVV' to 'CID'.</p>

         <h2><a id="user-content-design-decisions" class="anchor" href="#design-decisions" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Design decisions</h2> 
         <ul> 
          <li>The maximum expiration year is 19 years from now. (<a href="/braintree/card-validator/blob/master/src/expiration-year.js">view in source</a>)</li> 
          <li><code>valid.expirationDate</code> will only return <code>month:</code> and <code>year:</code> as strings if the two are valid, otherwise they will be <code>null</code>.</li> 
          <li>Since non-US postal codes are alpha-numeric, the <code>postalCode</code> will allow non-number characters to be used in validation.</li> 
         </ul>
