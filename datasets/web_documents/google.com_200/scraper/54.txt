           <h1 name="d2cc" id="d2cc" class="graf graf--h3 graf--leading graf--title">Building a Web&nbsp;Scraper</h1>
           <p name="5fef" id="5fef" class="graf graf--p graf-after--h3">We briefly touched on how to build a web scraper in our last post on web crawling. In this post, I’ll go into more detail about how to do this. When I use the term “web scraper,” I’m referring to a very specific type of web crawler?—?one that looks at a specific website and extracts data from it. We do a lot of similar scraping for <a href="https://www.datafiniti.net" data-href="https://www.datafiniti.net" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Datafiniti</a>.</p>
           <p name="ccaf" id="ccaf" class="graf graf--p graf-after--p">This post is going to cover a lot of ground, including:</p>
           <ol class="postList">
            <li name="2e04" id="2e04" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Document Object Model (DOM):</strong> An object representation of HTML</li>
            <li name="1608" id="1608" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Jquery:</strong> A Javascript library that will help you manipulate the DOM</li>
            <li name="b418" id="b418" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Setting up our environment</strong></li>
            <li name="3ecd" id="3ecd" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Building the Scraper:</strong> Building out the scraper attribute-by-attribute</li>
            <li name="3ba2" id="3ba2" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Running the Scraper:</strong> Using 80legs to run the scraper</li>
           </ol>
           <h4 name="fce2" id="fce2" class="graf graf--h4 graf-after--li">The Document Object&nbsp;Model</h4>
           <p name="cbf0" id="cbf0" class="graf graf--p graf-after--h4">Before we dive into building a scraper, you’ll need to understand a very important concept?—?the Document Object Model, aka the DOM. The DOM is how all modern web browsers look at the HTML makes up a web page. The HTML is read in by the browser and converted to a more formalized data structure that helps the browser render the content to what you actually see on the site. You can think of the DOM as a nested collection of HTML data, and can even see this in your browser. In Chrome, you get this by right-clicking and choosing “Inspect Element”:</p>
           <h4 name="cd20" id="cd20" class="graf graf--h4 graf-after--p">JQuery</h4>
           <p name="1bdb" id="1bdb" class="graf graf--p graf-after--h4">Because the DOM is such an accepted, standardized way of working with HTML, there are a lot of tools available for manipulating it. One of the most widely used tools is JQuery, a library that enhances Javascript by giving it a ton of DOM-manipulation functionality.</p>
           <p name="128a" id="128a" class="graf graf--p graf-after--p">As an example, let’s say we wanted to capture all the most-nested elements in this HTML list (item-1, item-2, and item-3):</p>
           <pre name="7488" id="7488" class="graf graf--pre graf-after--p">&lt;ul class="level-1"&gt;<br> &lt;li class="item-i"&gt;I&lt;/li&gt;<br> &lt;li class="item-ii"&gt;II<br>  &lt;ul class="level-2"&gt;<br>   &lt;li class="item-a"&gt;A&lt;/li&gt;<br>   &lt;li class="item-b"&gt;B<br>    &lt;ul class="level-3"&gt;<br>     &lt;li class="item-1"&gt;1&lt;/li&gt;<br>     &lt;li class="item-2"&gt;2&lt;/li&gt;<br>     &lt;li class="item-3"&gt;3&lt;/li&gt;<br>    &lt;/ul&gt;<br>   &lt;/li&gt;<br>   &lt;li class="item-c"&gt;C&lt;/li&gt;<br>  &lt;/ul&gt;<br> &lt;/li&gt;<br> &lt;li class="item-iii"&gt;III&lt;/li&gt;<br>&lt;/ul&gt;</pre>
           <p name="4789" id="4789" class="graf graf--p graf-after--pre">With JQuery, we would just need to do something like this:</p>
           <pre name="52ed" id="52ed" class="graf graf--pre graf-after--p">var innerList = $html.find(‘ul.level-3 li’);</pre>
           <p name="8384" id="8384" class="graf graf--p graf-after--pre">As you’ll see, using JQuery with the DOM greatly simplifies the web scraping process.</p>
           <h4 name="b187" id="b187" class="graf graf--h4 graf-after--p">Setting Up Our Development Environment</h4>
           <p name="9c9e" id="9c9e" class="graf graf--p graf-after--h4">Now that we understand some of the basic concepts, we’re almost ready to start building our scraper. Before we can get to the fun stuff, however, we need to setup a development environment. If you do this, you’ll be able to follow along and build the scraper as you read the article. Here are the steps you need to take:</p>
           <ol class="postList">
            <li name="b0ba" id="b0ba" class="graf graf--li graf-after--p">Install <a href="http://git-scm.com/" data-href="http://git-scm.com/" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">Git</a>.</li>
            <li name="3d70" id="3d70" class="graf graf--li graf-after--li">Clone the <a href="https://github.com/datafiniti/EightyApps" data-href="https://github.com/datafiniti/EightyApps" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">EightyApps repo</a>.</li>
            <li name="36bf" id="36bf" class="graf graf--li graf-after--li">Install the EightyApp tester for Chrome. Instructions are on the EightyApps rep page.</li>
            <li name="d0f9" id="d0f9" class="graf graf--li graf-after--li"><a href="https://portal.80legs.com/signup?plan=basic" data-href="https://portal.80legs.com/signup?plan=basic" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">Register</a> on <a href="http://www.80legs.com" data-href="http://www.80legs.com" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">80legs</a>.</li>
           </ol>
           <h4 name="f00d" id="f00d" class="graf graf--h4 graf-after--li">Building the Web&nbsp;Scraper</h4>
           <p name="ed13" id="ed13" class="graf graf--p graf-after--h4">Now we’re ready to get started! Open the BlankScraper.js file, which should be in the repo you just cloned, in a text editor. In your browser, open <a href="http://www.citysearch.com/profile/10192700/lockhart_tx/black_s_barbecue.html" data-href="http://www.citysearch.com/profile/10192700/lockhart_tx/black_s_barbecue.html" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">http://www.citysearch.com/profile/10192700/lockhart_tx/black_s_barbecue.html</a>, which we’ll use as an example.</p>
           <p name="5234" id="5234" class="graf graf--p graf-after--p">For the purposes of this tutorial, we’ll say we’re interested in collecting the following attributes:</p>
           <ul class="postList">
            <li name="cb91" id="cb91" class="graf graf--li graf-after--p">Name</li>
            <li name="34eb" id="34eb" class="graf graf--li graf-after--li">Address</li>
            <li name="36fb" id="36fb" class="graf graf--li graf-after--li">City</li>
            <li name="5069" id="5069" class="graf graf--li graf-after--li">State</li>
            <li name="523e" id="523e" class="graf graf--li graf-after--li">Postal code</li>
           </ul>
           <p name="fc6a" id="fc6a" class="graf graf--p graf-after--li">Let’s start with address. If you right-click on the web page in your browser and select “View Source”, you’ll see the full HTML for the page. Find where the address (“215 N Main St”) is displayed in HTML. You can quickly do this by also clicking on the magnifying glass in the upper left corner of the inspect elements box and then clicking on where you actually see the address displayed on the web page.</p>
           <p name="8c2a" id="8c2a" class="graf graf--p graf-after--p">Note that the address value is stored within a span tag, which has an <em class="markup--em markup--p-em">itemprop</em> value of <em class="markup--em markup--p-em">“streetAddress”</em>. In JQuery, we can easily capture this value with this:</p>
           <pre name="fb7c" id="fb7c" class="graf graf--pre graf-after--p">object.address = $html.find(‘span[itemprop=”streetAddress”]).text();</pre>
           <p name="6445" id="6445" class="graf graf--p graf-after--pre">We can do similar things for city, state, and zip code:</p>
           <pre name="38d7" id="38d7" class="graf graf--pre graf-after--p">object.city = $html.find(‘span[itemprop=”addressLocality”]).text();<br>object.state = $html.find(‘span[itemprop=”addressRegion”]).text();<br>object.postalcode = $html.find(‘span[itemprop=”postalCode”]).text();</pre>
           <p name="d23c" id="d23c" class="graf graf--p graf-after--pre">Once we’ve built everything out, here’s what the code for the scraper looks like:</p>
           <p name="c2dc" id="c2dc" class="graf graf--p graf-after--p"><a href="https://gist.github.com/shiondev/c569e72fec0c3d8bcf34" data-href="https://gist.github.com/shiondev/c569e72fec0c3d8bcf34" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">https://gist.github.com/shiondev/c569e72fec0c3d8bcf34</a></p>

           <p name="1a00" id="1a00" class="graf graf--p graf-after--figure">You can use the <a href="http://80apptester.80legs.com/" data-href="http://80apptester.80legs.com/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">http://80apptester.80legs.com/</a> to test the code you’ve written. Just copy and paste the code in, and paste in different URLs to test what it grabs from these specific URLs.</p>
           <p name="33ce" id="33ce" class="graf graf--p graf-after--p">You may be wondering where the rest of the web crawling logic is. Because we’re going to use 80legs to run this scraper, we don’t need worry about anything except processDocument and parseLinks. 80legs will handle the rest for us. We just handle what to do on each single URL the crawl hits. This really simplifies the amount of code we have to write for the web scraper.</p>
           <h4 name="9dab" id="9dab" class="graf graf--h4 graf-after--p">Running the&nbsp;Scraper</h4>
           <p name="8fba" id="8fba" class="graf graf--p graf-after--h4">With our scraping code complete, we head over to 80legs, login, and <a href="https://portal.80legs.com/apps" data-href="https://portal.80legs.com/apps" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">upload the code</a>. We’ll also want to <a href="https://portal.80legs.com/urllists/new" data-href="https://portal.80legs.com/urllists/new" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">upload a URL list</a> so our crawler has at least 1 URL to start from. For this example, <a href="http://www.citysearch.com/listings/lockhart-tx/barbecue_restaurants/72035_1724" data-href="http://www.citysearch.com/listings/lockhart-tx/barbecue_restaurants/72035_1724" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">http://www.citysearch.com/listings/lockhart-tx/barbecue_restaurants/72035_1724</a> is a good start.</p>
           <p name="c94b" id="c94b" class="graf graf--p graf-after--p">With our code and URL list available in our 80legs account, all that’s left is to run the crawl. We can use <a href="https://portal.80legs.com/crawls/new" data-href="https://portal.80legs.com/crawls/new" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Create a Crawl</a> form to select all the necessary settings, and we’re off!</p>
           <p name="6e32" id="6e32" class="graf graf--p graf-after--p">The crawl will take some time to run. Once it’s done, we’ll get one or more result files containing our scraped data. And that’s it!</p>
