            <h2 style="text-align: left;" id="GuidetoAPIAuthentication-Introduction">Introduction</h2>
            <p>cPanel &amp; WHM supports several API authentication methods.&nbsp;Whenever code calls an API function, it <strong>must</strong> first authenticate to the server. The authentication process accesses the server in order to run code as a specific cPanel or WHM user with a specific set of permissions.</p>
            <div class="confluence-information-macro confluence-information-macro-warning conf-macro output-block" data-hasbody="true" data-macro-name="warning">
             <p class="title">Important:</p>
             <span class="aui-icon aui-icon-small aui-iconfont-error confluence-information-macro-icon"> </span>
             <div class="confluence-information-macro-body">
              <ul>
               <li>API calls that use a method that includes a URL&nbsp;<strong>must</strong>&nbsp;use the correct port:
                <ul>
                 <li><code>2082</code>&nbsp;— Unsecure calls to cPanel's APIs.</li>
                 <li><code>2083</code>&nbsp;— Secure calls to cPanel's APIs.</li>
                 <li><code>2086</code>&nbsp;— Unsecure calls to WHM's APIs, or to cPanel's APIs via the WHM API.</li>
                 <li><code>2087</code>&nbsp;— Secure calls to WHM's APIs, or to cPanel's APIs via the WHM API.</li>
                 <li><code>2095</code>&nbsp;— Unsecure calls to cPanel's APIs via a Webmail session.</li>
                 <li><code>2096</code>&nbsp;— Secure calls to cPanel's APIs via a Webmail session.</li>
                </ul>Otherwise-correct calls will return <code>Permission denied</code> or <code>Function not found</code> errors if they use an incorrect port number.&nbsp;</li>
               <li>This guide <strong>only</strong> includes cPanel &amp; WHM authentication methods. For Manage2 authentication information, read our <a href="/display/SDK/Guide+to+the+Manage2+API">Guide to the Manage2 API</a> documentation.</li>
              </ul>
             </div>
            </div>
            <h2 id="GuidetoAPIAuthentication-Authenticationmethods">Authentication methods</h2>
            <p>You can currently use the following methods to authenticate with cPanel &amp; WHM in your custom code:</p>
            <ul>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Access+Hash+Authentication">Access Hash Authentication</a>&nbsp;—&nbsp;<span class="conf-macro output-inline" data-hasbody="false" data-macro-name="excerpt-include">Access hashes allow you to authenticate with the server as the&nbsp;<code>root</code>&nbsp;user.</span></li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+API+Tokens">API Tokens</a>&nbsp;—&nbsp;Use API tokens to authenticate to execute WHM API 1 functions over HTTPS.</li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Browser-Based+Authentication">Browser-Based Authentication</a>&nbsp;—&nbsp;Browser-based authentication accesses cPanel &amp; WHM's functionality via the browser.</li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Secure+Remote+Logins">Secure Remote Logins</a>&nbsp;—&nbsp;<span class="conf-macro output-inline" data-hasbody="false" data-macro-name="excerpt-include">cPanel &amp; WHM supports secure remote logins via the <code>Cpanel::LogMeIn</code> module.</span></li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Single+Sign+On">Single Sign On</a>&nbsp;—&nbsp;<span class="conf-macro output-inline" data-hasbody="false" data-macro-name="excerpt-include">The&nbsp;<em>Single Sign On</em>&nbsp;feature generates a temporary session to authenticate with cPanel &amp; WHM.</span></li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Two-Factor+Authentication">Two-Factor Authentication</a>&nbsp;—&nbsp;<span class="conf-macro output-inline" data-hasbody="false" data-macro-name="excerpt-include"><a href="/display/ALD/Two-Factor+Authentication+for+WHM">Two-Factor authentication</a>&nbsp;(2FA) requires an additional security code to log in to cPanel &amp; WHM.</span></li>
             <li><a href="/display/SDK/Guide+to+API+Authentication+-+Username+and+Password+Authentication">Username and Password Authentication</a>&nbsp;—&nbsp;Scripts can authenticate via a username and password in an HTTP header.</li>
            </ul>
