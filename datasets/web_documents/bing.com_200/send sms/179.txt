        <p>4. To send data</p> 
        <p>Successfully initialized, you can send and receive the normal data. We are here to do shows the way to use PDU, Text one way or the phone do not support a lot of Modem, 2 is also simpler, in the final to mention.</p> 
        <p>First, we can send the data to be sent into the buffer list, each one sending the data. Because the Modem does not support asynchronous (my own experience), we can only send one completed after the second relapse. Sent send initialization commands with the same principle, but the point to note is to send the contents of the first PDU to be coded to send the form to:</p> 
        <p>AT + CMGS = &lt;Length&gt; &lt;CR&gt; PDU DATA &lt;CTRL-z&gt;</p> 
        <p>Length of them is encoded in the PDU data length after subtracting 18 and then by 2 to get, as to why this calculation, can refer to the manual AT command. Some Modem support direct to the above as a complete packet sent in the past (for example: FALCOM series), some of the very stupid (such as: Siemens Tc35i), it requires to start in front of the At command, and so received Responses to "&gt;" symbol after the re-send the PDU DATA, very troublesome, we point to the developers for the sake of not: (, I just eat the loss in this. in accordance with past practices, the results are unable to pronounce how to go.</p> 
        <p>Steps like this:</p> 
        <p>Out after the first PDU encoded data, calculate its length, generating At + cmgs = &lt;length&gt; &lt;cr&gt;</p> 
        <p>Then sent</p> 
        <p>Then listen COM port, waiting to receive "&gt;" character</p> 
        <p>Receive "&gt;" after, then send the rest</p> 
        <p>And then listening, receiving responses to see if sent successfully. If the sending is successful, usually receive this data:</p> 
        <p>The data itself: PDU DATA &lt;CTRL-z&gt;</p> 
        <p>Send number: + CMGS: 22</p> 
        <p>Symbol of success: OK</p> 
        <p>Sometimes also receive OK, but not receive + CMGS: 22 kinds of things, or not sent successfully, the two must have, truly delivered.</p> 
        <p>5. Receive Data</p>

        <p>PDU Data</p> 
        <p>"\ N \ r"</p> 
        <p>By the end of the last carriage return line feed</p>
