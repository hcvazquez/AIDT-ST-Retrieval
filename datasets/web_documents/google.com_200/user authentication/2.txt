        <div class="post-body__content">
         <p>Implementing robust authentication strategies for any application can be a daunting task and Node.js applications are no exception to this.&nbsp;</p>
         <p>In this tutorial, we will&nbsp;develop a Node.js application from scratch and use a relatively new but very popular authentication middleware - <a href="http://passportjs.org/" rel="external" target="_blank">Passport</a>&nbsp;to take care of our&nbsp;authentication concerns.&nbsp;<br></p>
         <blockquote>
          Passport's documentation describes it as a "simple, unobtrusive authentication middleware for Node" and rightly so.&nbsp;
         </blockquote>
         <p>By providing itself as a middleware, Passport does an excellent job at separating the other concerns of a web application from its authentication needs. It allows&nbsp;Passport to be easily configured into any <a href="http://expressjs.com/" rel="external" target="_blank">Express</a>-based web application, just like we configure other Express middleware such as <a href="https://github.com/expressjs/morgan" rel="external" target="_blank">logging</a>, <a href="https://github.com/expressjs/body-parser" rel="external" target="_blank">body-parsing</a>, <a href="https://github.com/expressjs/cookie-parser" rel="external" target="_blank">cookie-parsing</a>, session-handling, etc.&nbsp;</p>
         <p>This tutorial assumes a basic understanding of <a href="http://nodejs.org/" rel="external" target="_blank">Node.js</a> and <a href="http://expressjs.com/" rel="external" target="_blank">Express</a> framework and try to keep focus on authentication, although we do create a sample Express app from scratch and progress via adding routes to it and authenticating some of those routes.&nbsp;</p>
         <p>If you need help with anything from debugging to new features, try working with&nbsp;some of the experienced&nbsp;<a href="https://studio.envato.com/search?utf8=%E2%9C%93&amp;search%5Bquery%5D=javascript" target="_self">JavaScript developers</a> on Envato Studio.</p>
         <figure class="post_image">
          <a href="https://studio.envato.com/search?utf8=%E2%9C%93&amp;search%5Bquery%5D=javascript" target="_self"><img alt="JavaScript developers on Envato Studio" src="https://cms-assets.tutsplus.com/uploads/users/48/posts/21619/image/JavaScript-studio.jpg"></a>
          <figcaption>
           <a href="https://studio.envato.com/search?utf8=%E2%9C%93&amp;search%5Bquery%5D=javascript" target="_self">JavaScript developers on Envato Studio</a>
          </figcaption>
         </figure>
         <h2>Authentication Strategies</h2>
         <p>Passport provides us with&nbsp;140+ authentication mechanisms to choose from. You can authenticate against a local/remote database instance or use the single sign-on using OAuth providers for <a href="https://github.com/jaredhanson/passport-facebook" rel="external" target="_blank">Facebook</a>, <a href="https://github.com/jaredhanson/passport-twitter" rel="external" target="_blank">Twitter</a>, <a href="https://github.com/jaredhanson/passport-google" rel="external" target="_blank">Google</a>,&nbsp;etc. to authenticate with your social media accounts, or you can choose from an extensive list of <a href="http://passportjs.org/guide/providers/" target="_self">providers</a> which support authentication with Passport and provide a node module for that.</p>
         <p>But worry not: You do not need to include any strategy/mechanism that your application does not need. All these strategies are independent of each other and packaged as separate node modules which&nbsp;are not included by default when you install Passport's middleware: &nbsp;<code class="inline">npm install passport</code></p>
         <p>In this tutorial, we will use the Local Authentication Strategy of Passport and authenticate the users against a locally configured Mongo DB instance, storing the user details in the database. For using the Local Authentication Strategy, we need to install the <a href="https://github.com/jaredhanson/passport-local" target="_self">passport-local</a>&nbsp;module: <code class="inline">npm install passport-local</code></p>
         <p>But wait: Before you fire up your terminal and start executing these commands, let's start by building a Express app from scratch and add some routes to it (for&nbsp;login, registration&nbsp;and home) and then try to add our authentication middleware to it. Note that we will be using Express 4 for the purposes of this tutorial, but&nbsp;with some minor&nbsp;<a href="https://github.com/strongloop/express/wiki/Migrating-from-3.x-to-4.x" target="_self">differences</a>&nbsp;Passport works equally well with Express 3, as well.&nbsp;</p>
         <h2>Setting Up the Application</h2>
         <p>If you haven't already, then go ahead and install <a href="http://expressjs.com/" target="_self">Express</a>&nbsp;&amp;&nbsp;<a href="https://github.com/expressjs/generator" rel="external" target="_blank">express-generator</a> to generate a boilerplate application by simply executing <code class="inline">express passport-mongo</code> on the terminal. The generated application structure should look like this:</p>
         <figure class="post_image">
          <img alt="Initial Application Structure" src="https://cms-assets.tutsplus.com/uploads/users/388/posts/21619/image/passportjs-initial-app-structure.png">
         </figure>
         <p>Let's remove some of the default functionality that we won't be making use of - go ahead and delete the <code class="inline">users.js</code> route&nbsp;and remove its references from the <code class="inline">app.js</code> file.&nbsp;</p>
         <h2>Adding Project Dependencies</h2>
         <p>Open up <code class="inline">package.json</code> and add the dependencies for <code class="inline">passport</code> and <code class="inline">passport-local</code> module.</p>
         <pre class="brush: bash noskimlinks noskimwords">"passport": "~0.2.0",
"passport-local": "~1.0.0"</pre>
         <p>Since we will be saving the user details in MongoDB, we will use Mongoose as our object data modeling&nbsp;tool. Another way to install and save the dependency to <code class="inline">package.json</code> is by entering:</p>
         <pre class="brush: bash noskimlinks noskimwords">npm install mongoose --save </pre>
         <p><code class="inline">package.json</code> should look like this:</p>
         <figure class="post_image">
          <img alt="Added Mongoose Dependencies" src="https://cms-assets.tutsplus.com/uploads/users/388/posts/21619/image/passportjs-packagejson-final.png">
         </figure>
         <p>Now, install all the dependencies and run the boilerplate application by executing&nbsp;<code class="inline">npm install &amp;&amp; npm start</code>. It will now download and install all of the dependencies and will start the node server. You can check the basic Express app at <a href="http://localhost:3000/">http://localhost:3000/</a>&nbsp;but there is nothing much to see.&nbsp;</p>
         <p>Very soon, we are going to change that by creating a full-fledged express app that asks for&nbsp;shows a registration page for a new user,&nbsp;the login of a registered user, and authenticates the registered user by using Passport.&nbsp;</p>
         <h2>Creating Mongoose Model</h2>
         <p>Since we will be saving the user details in Mongo, let's create a User Model in <a href="http://mongoosejs.com/index.html" target="_self">Mongoose</a> and save that in <code class="inline">models/user.js</code> in our app.&nbsp;</p>
         <pre class="brush: javascript noskimlinks noskimwords">var mongoose = require('mongoose');

module.exports = mongoose.model('User',{
        username: String,
	password: String,
	email: String,
	gender: String,
	address: String
});</pre>
         <p>Basically, we are creating a Mongoose <a href="http://mongoosejs.com/docs/models.html" target="_self">model</a> using which we can perform CRUD operations on the underlying&nbsp;database.</p>
         <h2>Configuring Mongo&nbsp;</h2>
         <p>If you do not have Mongo installed locally then we recommend that you use cloud database services such as&nbsp;<a href="https://modulus.io/" rel="external" target="_blank">Modulus</a>&nbsp;or&nbsp;<a href="https://mongolab.com/welcome/" rel="external" target="_blank">MongoLab</a>. Creating a working MongoDB instance using these is not only&nbsp;free but is&nbsp;just a matter of few&nbsp;clicks.&nbsp;</p>
         <p>After you create a database on one of these services, it will give you a database URI like<code class="inline">mongodb://&lt;dbuser&gt;:&lt;dbpassword&gt;@novus.modulusmongo.net:27017/&lt;dbName&gt;</code>&nbsp;which can be used to perform CRUD operations on the database.&nbsp;It's a good idea to keep the database configuration in a separate file which can be pull up as and when needed. As such, we create a node module&nbsp;<code class="inline">db.js</code>&nbsp;which looks like:</p>
         <pre class="brush: javascript noskimlinks noskimwords">module.exports = {
  'url' : 'mongodb://&lt;dbuser&gt;:&lt;dbpassword&gt;@novus.modulusmongo.net:27017/&lt;dbName&gt;'
}</pre>
         <p>If you're&nbsp;like me,&nbsp;you are using a local Mongo instance then it's time to start the <code class="inline">mongod</code> daemon and the <code class="inline">db.js</code> should look like</p>
         <pre class="brush: javascript noskimlinks noskimwords">module.exports = {
  'url' : 'mongodb://localhost/passport'
}</pre>
         <p>&nbsp;Now we use this configuration in&nbsp;<code class="inline">app.js</code>&nbsp;and connect to it using Mongoose APIs:&nbsp;&nbsp;</p>
         <pre class="brush: javascript noskimlinks noskimwords">var dbConfig = require('./db.js');
var mongoose = require('mongoose');
mongoose.connect(dbConfig.url);</pre>
         <h2>Configuring Passport</h2>
         <p>Passport just provides the mechanism to handle&nbsp;authentication leaving the onus of implementing session-handling ourselves and for that we will be using <a href="https://github.com/expressjs/session" target="_self">express-session</a>. Open up&nbsp;<code class="inline">app.js</code>&nbsp;and paste the code below before configuring the routes:</p>
         <pre class="brush: javascript noskimlinks noskimwords">// Configuring Passport
var passport = require('passport');
var expressSession = require('express-session');
app.use(expressSession({secret: 'mySecretKey'}));
app.use(passport.initialize());
app.use(passport.session());</pre>
         <div>
          <br>
         </div>
         <p>This is needed as we want our user sessions to be persistent in nature. Before running the app, we need to install&nbsp;<a href="https://github.com/expressjs/session" target="_self">express-session</a>&nbsp;and add it to our dependency list in <code class="inline">package.json</code>. To do that type&nbsp;<code class="inline">npm install --save express-session</code></p>
         <h2>Serializing and Deserializing User Instances</h2>
         <p>Passport also needs to serialize and deserialize user instance from a session store in order to support login sessions, so that every subsequent request will not contain the user credentials. It provides two methods&nbsp;<code class="inline">serializeUser</code>&nbsp;and&nbsp;<code class="inline">deserializeUser</code> for this purpose:</p>
         <pre class="brush: javascript noskimlinks noskimwords">passport.serializeUser(function(user, done) {
  done(null, user._id);
});

passport.deserializeUser(function(id, done) {
  User.findById(id, function(err, user) {
    done(err, user);
  });
});</pre>
         <h2>Using Passport Strategies</h2>
         <p>&nbsp;We will now define Passport's strategies for handling <b>login</b> and <b>signup</b>. Each of them would be an instance of&nbsp;&nbsp;the&nbsp;<b>Local Authentication Strategy</b>&nbsp;of Passport and would be created using the <code class="inline">passport.use()</code> function.&nbsp;We use&nbsp;<a href="https://www.npmjs.org/package/connect-flash" target="_self">connect-flash</a>&nbsp;to help us with&nbsp;error handling by providing flash messages which can be displayed to user on error.</p>
         <h3>Login Strategy</h3>
         <p>The login strategy looks like this:</p>
         <pre class="brush: javascript noskimlinks noskimwords">// passport/login.js
passport.use('login', new LocalStrategy({
    passReqToCallback : true
  },
  function(req, username, password, done) { 
    // check in mongo if a user with username exists or not
    User.findOne({ 'username' :  username }, 
      function(err, user) {
        // In case of any error, return using the done method
        if (err)
          return done(err);
        // Username does not exist, log error &amp; redirect back
        if (!user){
          console.log('User Not Found with username '+username);
          return done(null, false, 
                req.flash('message', 'User Not found.'));                 
        }
        // User exists but wrong password, log the error 
        if (!isValidPassword(user, password)){
          console.log('Invalid Password');
          return done(null, false, 
              req.flash('message', 'Invalid Password'));
        }
        // User and password both match, return user from 
        // done method which will be treated like success
        return done(null, user);
      }
    );
}));</pre>
         <p>The first parameter to&nbsp;<code class="inline">passport.use()</code> is the <b>name</b> of the strategy which will be&nbsp;used to identify this strategy when applied later. The second parameter is the&nbsp;<b>type</b> of strategy that you want to create, here we use the <a href="http://passportjs.org/guide/username-password/" target="_self">username-password</a> or the LocalStrategy.&nbsp;It is to be noted that by default the LocalStrategy expects to find the user credentials in <code class="inline">username</code> &amp; <code class="inline">password</code> parameters, but it allows us to use any other named parameters as well. The&nbsp;<code class="inline">passReqToCallback</code> config variable allows us to access the <code class="inline">request</code> object in the callback, thereby enabling us to use any parameter associated with the request.<br></p>
         <p>Next, we use the <a href="http://mongoosejs.com/docs/api.html#model_Model.findOne" target="_self">Mongoose API</a> to find the User in our underlying collection of Users to check if the user is a valid user or not. The last parameter in our callback : <code class="inline">done</code> denotes a useful method using which we could signal success or failure to Passport module. To specify failure either the first parameter should contain the error, or the second parameter should evaluate to <code class="inline">false</code>. To signify success the first parameter should be <code class="inline">null</code> and the second parameter should evaluate to a <code class="inline">truthy</code> value, in which case it will be made available on the <code class="inline">request</code> object</p>
         <p>Since passwords are inherently weak in nature, we should always <a href="http://codahale.com/how-to-safely-store-a-password/" target="_self">encrypt</a> them before saving them to the database. For this, we use <a href="https://www.npmjs.org/package/bcrypt-nodejs" target="_self">bcrypt-nodejs</a> to help us out with encryption and decryption of passwords.&nbsp;</p>
         <pre class="brush: javascript noskimlinks noskimwords">var isValidPassword = function(user, password){
  return bCrypt.compareSync(password, user.password);
}</pre>
         <p>If you are feeling uneasy with the code snippets and prefer to see the complete code in action, feel free to browse <a href="https://github.com/tutsplus/passport-mongo" rel="external" target="_blank">the code here</a>.</p>
         <h3>Registration Strategy</h3>
         <p>Now, we define the next strategy which will handle registration of a new user and creates his or&nbsp;her entry in our underlying Mongo DB:</p>
         <pre class="brush: javascript noskimlinks noskimwords">passport.use('signup', new LocalStrategy({
    passReqToCallback : true 
  },
  function(req, username, password, done) {
    findOrCreateUser = function(){
      // find a user in Mongo with provided username
      User.findOne({'username':username},function(err, user) {
        // In case of any error return
        if (err){
          console.log('Error in SignUp: '+err);
          return done(err);
        }
        // already exists
        if (user) {
          console.log('User already exists');
          return done(null, false, 
             req.flash('message','User Already Exists'));
        } else {
          // if there is no user with that email
          // create the user
          var newUser = new User();
          // set the user's local credentials
          newUser.username = username;
          newUser.password = createHash(password);
          newUser.email = req.param('email');
          newUser.firstName = req.param('firstName');
          newUser.lastName = req.param('lastName');

          // save the user
          newUser.save(function(err) {
            if (err){
              console.log('Error in Saving user: '+err);  
              throw err;  
            }
            console.log('User Registration succesful');    
            return done(null, newUser);
          });
        }
      });
    };
    
    // Delay the execution of findOrCreateUser and execute 
    // the method in the next tick of the event loop
    process.nextTick(findOrCreateUser);
  });
);</pre>
         <p>Here, we again use the Mongoose API&nbsp;to find if any user with the given username already exists or not. If not, then create a new user and saves the user information in Mongo. Else return the error using the <code class="inline">done</code> callback and flash messages.&nbsp;Note that we use <code class="inline">bcrypt-nodejs</code> for creating the hash of the password before saving it:</p>
         <pre class="brush: javascript noskimlinks noskimwords">// Generates hash using bCrypt
var createHash = function(password){
 return bCrypt.hashSync(password, bCrypt.genSaltSync(10), null);
}</pre>
         <h2>Creating Routes</h2>
         <p>If we were to see a&nbsp;birds eye view of our application, it would look like:</p>
         <figure class="post_image">
          <img alt="Birds Eye View of Our Application" src="https://cms-assets.tutsplus.com/uploads/users/388/posts/21619/image/Bird_Eye_View_of_PassportJS_App.png">
         </figure>
         <p>We now define our routes for the application in the following module which takes the instance of Passport created in <code class="inline">app.js</code> above. Save this module in <code class="inline">routes/index.js</code></p>
         <pre class="brush: javascript noskimlinks noskimwords">module.exports = function(passport){

  /* GET login page. */
  router.get('/', function(req, res) {
    // Display the Login page with any flash message, if any
    res.render('index', { message: req.flash('message') });
  });

  /* Handle Login POST */
  router.post('/login', passport.authenticate('login', {
	successRedirect: '/home',
	failureRedirect: '/',
	failureFlash : true  
  }));

  /* GET Registration Page */
  router.get('/signup', function(req, res){
	res.render('register',{message: req.flash('message')});
  });

  /* Handle Registration POST */
  router.post('/signup', passport.authenticate('signup', {
	successRedirect: '/home',
	failureRedirect: '/signup',
	failureFlash : true  
  }));

  return router;
}</pre>
         <p>The most important part of the above code snippet&nbsp;is the use of <code class="inline">passport.authenticate()</code> to delegate the authentication to <code class="inline">login</code> and <code class="inline">signup</code> strategies when a HTTP&nbsp;<code class="inline">POST</code> is made to <code class="inline">/login</code> and <code class="inline">/signup</code> routes respectively. Note that it is not mandatory to name the strategies on the route path and it can be named anything.</p>
         <h2>Creating Jade Views</h2>
         <p>Next, we create the following two views for our application:</p>
         <ol> 
          <li> <code class="inline">layout.jade</code>&nbsp;contains the basic layout &amp; styling information &nbsp;</li> 
          <li> <code class="inline">index.jade</code> contains the login page containing the login form and giving option to create a new account</li> 
         </ol>
         <pre class="brush: html noskimlinks noskimwords">extends layout

block content
  div.container
	div.row
	  div.col-sm-6.col-md-4.col-md-offset-4
		h1.text-center.login-title Sign in to our Passport app
		  div.account-wall
			img(class='profile-img', src='https://lh5.googleusercontent.com/-b0-k99FZlyE/AAAAAAAAAAI/AAAAAAAAAAA/eu7opA4byxI/photo.jpg?sz=120')
			form(class='form-signin', action='/login', method='POST')
			  input(type='text', name='username' class='form-control', placeholder='Email',required, autofocus)
			  input(type='password', name='password' class='form-control', placeholder='Password', required)
			  button(class='btn btn-lg btn-primary btn-block', type='submit') Sign in
			  span.clearfix
		  a(href='/signup', class='text-center new-account') Create an account
		  #message
		  if message
			h1.text-center.error-message #{message}</pre>
         <p>Thanks to Bootstrap, our Login page now looks like&nbsp;</p>
         <figure class="post_image">
          <img alt="Login Page for Our Passport App" src="https://cms-assets.tutsplus.com/uploads/users/388/posts/21619/image/passportjs-login-page.png">
         </figure>
         <p>We need two more views for registration details and for the home page of the application:</p>
         <ol> 
          <li> <code class="inline">register.jade</code>&nbsp;contains the registration form</li> 
          <li> <code class="inline">home.jade</code>&nbsp;says hello and shows logged in user's details</li> 
         </ol>
         <p>If you are unfamiliar with&nbsp;Jade, check out the <a href="http://jade-lang.com/reference/" target="_self">documentation</a>.</p>
         <h2>Implementing Logout Functionality</h2>
         <p>Passport, being a middleware, is permitted to add certain properties and&nbsp;methods on request and response objects and it makes proper use of it by adding a very handy&nbsp;<code class="inline">request.logout()</code>&nbsp;method which invalidates the user session apart from other properties.</p>
         <pre class="brush: javascript noskimlinks noskimwords">/* Handle Logout */
router.get('/signout', function(req, res) {
  req.logout();
  res.redirect('/');
});</pre>
         <h2>Protecting Routes</h2>
         <p>Passport also gives the ability to protect access to a route which is deemed unfit for an anonymous user. This means that if some user tries to access <a href="http://localhost:3000/home">http://localhost:3000/home</a>&nbsp;without authenticating in the application, he will be redirected to home page by doing</p>
         <pre class="brush: javascript noskimlinks noskimwords">/* GET Home Page */
router.get('/home', isAuthenticated, function(req, res){
  res.render('home', { user: req.user });
});

// As with any middleware it is quintessential to call next()
// if the user is authenticated
var isAuthenticated = function (req, res, next) {
  if (req.isAuthenticated())
	return next();
  res.redirect('/');
}</pre>
         <h2>Conclusion</h2>
         <p>Passport is not the only player in this arena when its comes to authenticating Node.js applications and there exists alternatives like <a href="https://github.com/bnoguchi/everyauth/" target="_self">EveryAuth</a> but the modularity, flexibility, community support and the fact that its just a middleware makes&nbsp;Passport definitely a much better choice.&nbsp;</p>
         <p>For a detailed comparison between the two, <a href="http://stackoverflow.com/questions/11974947/everyauth-vs-passport-js" target="_self">here</a> is an interesting and informative&nbsp;perspective from the developer of Passport himself.</p>
