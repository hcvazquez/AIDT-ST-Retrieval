       <h1>yFiles for Java </h1> 
      </header> 
      <div class="col-sm-12"> 
       <div style="margin-top:10px" data-title="yFiles for Java" data-services="[&quot;facebook&quot;,&quot;googleplus&quot;,&quot;twitter&quot;]" data-lang="en" data-theme="white" data-backend-url="/resources/social-media/counts-yfilesjava.json" class="shariff"></div> 
       <div class="col-sm-6 left"> 
        <div itemscope itemtype="http://schema.org/SoftwareApplication"> 
         <link itemprop="applicationCategory" href="http://schema.org/DeveloperApplication">

         <div itemprop="description"> 
          <p><b itemprop="name">yFiles for Java</b> is an extensive Java™ class library for the Java Swing world that provides algorithms and components enabling the analysis, visualization, and the automatic layout of graphs, diagrams, and networks. </p> 
         </div> 
         <p> The 2.x line of yFiles for Java supports Java 7 and earlier. If your Java Swing project requires Java 8, we recommend the <a href="/products/yfiles-for-java">next-generation yFiles for Java</a>. </p> 
         <p>Looking for a <b>JavaFX</b>™ diagramming library instead? Choose&nbsp;<a href="/products/yfiles-for-javafx">yFiles for JavaFX</a>!</p>

       <h1>Next-generation yFiles for Java </h1> 
      </header> 
      <div class="col-sm-12">
       <a name="yFilesJava3.x"></a> 
       <p> </p>
       <div style="width:100px; float:left; display:inline-block; border:1px solid #036; border-radius:4px; margin-right:10px; padding:6px 10px; text-align:center">
        <span style="font-size:28px">Java 8</span>
        <br>
        <span style="font-size:16px">or higher</span>
       </div> 
       <p><a href="/products/yfiles-for-java">Beginning with version 3.0</a>, the yFiles for Java diagramming library features a <b>brand-new, modern API</b> which requires Java 8. This API is mature and provides many new features that are not available out-of-the-box in the 2.x line. </p> 
       <p><b>We recommend</b> to start development with the <a href="/products/yfiles-for-java">next-generation yFiles for Java</a> in new Java Swing projects.<br>Especially in projects that don't need to support legacy Java versions before Java 8, this is the recommended way to realize your application's diagram visualization part. </p>

      <div class="col-sm-12"> 
       <div class="col-sm-12"> 
        <p>The <strong>yFiles for Java</strong> library enables you to create impressive cross-platform Java diagramming applications. It features all essential functionality from the <a href="/products/yfiles/">yFiles product family</a>.</p> 
       </div> 
       <div class="col-sm-4"> 
        <p>yFiles for Java functionality is divided into three parts - Basic, Layout, and Viewer component.</p> 
        <p class="box">Learn more about the <a href="/products/yfiles-for-java-2.x/packages">yFiles for Java product types</a>, all available combinations of Basic, Layout, and Viewer components.</p> 
       </div> 
       <div class="col-sm-8"> 
        <p style="float:left; width:130px"> </p>
        <h2 style="font-weight: 700; font-size: 28px">Basic</h2> 
        <p></p> 
        <p>The Basic component contains essential classes and data types for graph analysis. From highly efficient implementations of <a href="http://docs.yworks.com/yfiles/doc/api-json/#/api/y.base.Graph" target="_blank">Graph</a>, Priority Queue, and other advanced data types, to a wide variety of <a href="http://docs.yworks.com/yfiles/doc/api-json/#/api/y.algo" target="_blank">graph and network algorithms</a>, The Basic component provides an indispensable toolkit for a range of network analysis tasks. </p> 
        <div class="clearfix"></div> 
        <p style="float:left; width:130px"> </p>
        <h2 style="font-weight: 700; font-size: 28px">Layout</h2> 
        <p></p> 
        <p>The Layout component builds upon the Basic component to offer <a href="http://docs.yworks.com/yfiles/doc/api-json/#/dguide/major_layouters" target="_blank">a perfect suite of graph layout algorithms</a>. Diverse layout styles including hierarchic, orthogonal, and circular are provided as easy-to-integrate components that can be configured programmatically to suit most layout demands. Even <a href="http://docs.yworks.com/yfiles/doc/api-json/#/dguide/incremental_layout" target="_blank">incremental layout</a> is made simple, and our <a href="http://docs.yworks.com/yfiles/doc/api-json/#/dguide/major_routers" target="_blank">edge routing algorithms</a> make it easy to route edges into existing diagrams. </p> 
        <div class="clearfix"></div> 
        <p style="float:left; width:130px"> </p>
        <h2 style="font-weight: 700; font-size: 28px">Viewer</h2> 
        <p></p> 
        <p>The Viewer component also builds upon the Basic component. It features the powerful graph viewer UI component showcased in the <a href="/products/yed">yEd graph editor</a>, along with several other useful Swing-based GUI elements. The Viewer component also supports diverse graph and image file formats (<a href="http://docs.yworks.com/yfiles/doc/api-json/#/dguide/graphml" target="_blank">GraphML</a>, JPG, GIF) and has excellent <a href="http://docs.yworks.com/yfiles/doc/api-json/#/dguide/printing" target="_blank">printing capabilities</a>. </p> 
       </div> 
       <div class="clearfix"></div> 
       <div class="col-sm-12"> 
        <h3 style="margin: 10px 0 20px">Extension Packages</h3> 
        <p>Additional functionality such as <a href="/products/yfiles-for-java-2.x/ysvg">SVG output</a> or <a href="/products/yfiles-for-java-2.x/yexport">PDF output</a> (among others), and support for the <a href="/products/yfiles-for-java-2.x/ybpmn">Business Process Model and Notation (BPMN)</a> is provided through diverse <a href="/products/yfiles-for-java-2.x/extensionpackages">yFiles for Java extension packages</a>.</p> 
       </div>
