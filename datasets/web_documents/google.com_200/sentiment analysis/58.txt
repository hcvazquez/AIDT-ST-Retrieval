         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Using a subset of a Twitter stream was the perfect choice to use in this demonstration, since it had everything we needed: an endless and continuous data source that was ready to be explored.</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">SPARK STREAMING, MINIMIZED</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;"><a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=https%3A%2F%2Fmapr.com%2Fproducts%2Fproduct-overview%2Fapache-spark-streaming" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank">Spark Streaming</a> is very well explained <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=http%3A%2F%2Fspark.apache.org%2Fdocs%2Flatest%2Fstreaming-programming-guide.html" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank"><em>here</em></a> and in chapter 6 of the ebook <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=https%3A%2F%2Fmapr.com%2Febooks%2Fspark%2F06-apache-spark-streaming-framework.html" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank">"Getting Started with Apache Spark,"</a> so we are going to skip some of the details about the Streaming API and move on to setting up our app.</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">SETTING UP OUR APP</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Lets see how to prepare our app before doing anything else.</p>
         <pre class="language-none line-numbers"><code>val config = new SparkConf().setAppName("twitter-stream-sentiment") val sc = new SparkContext(config) sc.setLogLevel("WARN") val ssc = new StreamingContext(sc, Seconds(5))&nbsp; System.setProperty("twitter4j.oauth.consumerKey", "consumerKey")&nbsp; System.setProperty("twitter4j.oauth.consumerSecret", "consumerSecret")&nbsp; System.setProperty("twitter4j.oauth.accessToken", accessToken)&nbsp; System.setProperty("twitter4j.oauth.accessTokenSecret", "accessTokenSecret") val stream = TwitterUtils.createStream(ssc, None)<br><span class="line-numbers-rows"><span></span><span></span></span></code></pre>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Here we have created the Spark Context <span style="font-weight: bold;"><strong><em>sc</em></strong></span>, and set the log level to <em>WARN</em> to eliminate the noisy log Spark generates. We also created a Streaming Context <span style="font-weight: bold;"><strong><em>ssc</em></strong></span> using <span style="font-weight: bold;"><strong><em>sc</em></strong></span>. Then we set up our Twitter credentials (before doing this we needed to follow <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=http%3A%2F%2Fiag.me%2Fsocialmedia%2Fhow-to-create-a-twitter-app-in-8-easy-steps%2F" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank">these steps</a>) that we got from the Twitter website. <em>Now the real fun starts</em>.</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">WHAT IS TRENDING RIGHT NOW ON TWITTER?</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">It is easy to find out what is trending on Twitter at any given moment; it is just a matter of counting the appearances of each tag on the stream. Lets see how Spark allows us to do this operation.</p>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <pre class="language-none line-numbers"><code>val config = new SparkConf().setAppName("twitter-stream-sentiment") val sc = new SparkContext(config) sc.setLogLevel("WARN") val ssc = new StreamingContext(sc, Seconds(5))&nbsp; System.setProperty("twitter4j.oauth.consumerKey", "consumerKey")&nbsp; System.setProperty("twitter4j.oauth.consumerSecret", "consumerSecret")&nbsp; System.setProperty("twitter4j.oauth.accessToken", accessToken)&nbsp; System.setProperty("twitter4j.oauth.accessTokenSecret", "accessTokenSecret") val stream = TwitterUtils.createStream(ssc, None)<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">First, we got the tags from the Tweets, counted how many times it (a tag) appeared, and sorted them by the count. After that, we persisted the result in order to point Splunk (or any other tool for that matter) to it. We could build some interesting dashboards using this information in order to track the most trending hashtags. Based on this information, my coworker could create campaigns and use these popular tags to attract a bigger audience.</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">ANALYZING TWEETS</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Now we want to add functionality to get an overall opinion of what people think about a set of topics. For the sake of this example, lets say that we want to know the <span style="font-weight: bold;"><strong><em>sentiment</em></strong></span> of Tweets about <span style="font-weight: bold;"><strong>Big Data</strong></span> and <span style="font-weight: bold;"><strong>Food</strong></span>, two very unrelated topics.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">There are several APIs for analyzing sentiments from Tweets, but we are going to use an interesting library from <span style="font-weight: bold;"><strong>The Stanford Natural Language Processing Group</strong></span> in order extract the corresponding <span style="font-weight: bold;"><strong><em>sentiments</em></strong></span>.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">In our <span style="font-weight: bold;"><strong><em>build.sbt</em></strong></span> file we need to add the corresponding dependencies.</p>
         <pre class="language-none line-numbers"><code>libraryDependencies += "edu.stanford.nlp" % "stanford-corenlp" % "3.5.1" libraryDependencies += "edu.stanford.nlp" % "stanford-corenlp" % "3.5.1"classifier "models"<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Now, we need to select only those Tweets we really care about by filtering the <span style="font-weight: bold;"><strong><em>stream</em></strong></span> using certain <em>hashtag (#)</em>. This filtering is quite easy, thanks to a unified Spark API.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Lets see how.</p>
         <pre class="language-none line-numbers"><code>val tweets = stream.filter {t =&gt; val tags = t.getText.split(" ").filter(_.startsWith("#")).map(_.toLowerCase) tags.contains("#bigdata") &amp;&amp; tags.contains("#food") }<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Here, we get all tags in each Tweet, checking that it has been tagged with<span style="font-weight: bold;"><strong><em>#bigdata</em></strong></span> and <span style="font-weight: bold;"><strong><em>#food</em></strong></span>.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Once we have our Tweets, extracting the corresponding sentiment is quite easy. Lets define a function that extracts the sentiment from the Tweets content so we can plug it in in our pipeline.</p>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <pre class="language-none line-numbers"><code>def detectSentiment(message: String): SENTIMENT_TYPE<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">We are going to use this function, assuming it does what it should, and we will put its implementation at the end, since it's not the focus of this post. In order to get an idea of how it works, let's build some tests around it.</p>
         <pre class="language-none line-numbers"><code>it("should detect not understood sentiment") { detectSentiment("")should equal (NOT_UNDERSTOOD)&nbsp; } it("should detect a negative sentiment") { detectSentiment("I am feeling very sad and frustrated.")should equal (NEGATIVE)&nbsp; } it("should detect a neutral sentiment") { detectSentiment("I'm watching a movie")should equal (NEUTRAL)&nbsp; } it("should detect a positive sentiment") { detectSentiment("It was a nice experience.")should equal (POSITIVE)&nbsp; } it("should detect a very positive sentiment") { detectSentiment("It was a very nice experience.")should equal (VERY_POSITIVE) }<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">These tests should be enough to show how <span style="font-weight: bold;"><strong><em>detectSentiment</em></strong></span> works.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Lets see an example.</p>
         <pre class="language-none line-numbers"><code>val data = tweets.map {status =&gt; val sentiment = SentimentAnalysisUtils.detectSentiment(status.getText) val tags = status.getHashtagEntities.map(_.getText.toLowerCase) (status.getText, sentiment.toString, tags) }<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;"><span style="font-weight: bold;"><strong><em>data</em></strong></span> represents a <em>DStream</em> of Tweets we want, the associated sentiment, and the hashtags within the Tweet (here we should find the tags we used to filter).</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">SQL INTEROPERABILITY</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Now we want to cross reference the sentiment data with an external dataset that we can query using SQL. For my coworker, it makes a lot of sense to be able to <span style="font-weight: bold;"><strong><em>join</em></strong></span> the Twitter stream with his other dataset.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Lets take a look at how we could achieve this.</p>
         <pre class="language-none line-numbers"><code>val sqlContext = new SQLContext(sc)&nbsp; import sqlContext.implicits._ <br>data.foreachRDD {rdd =&gt; rdd.toDF().registerTempTable("sentiments") }<span class="line-numbers-rows"><span></span><span></span></span></code></pre>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">We have transformed our stream into a different representation (a <span style="font-weight: bold;"><strong><em>DataFrame</em></strong></span>), which is also backed by all Spark concepts (resilient, distributed, very fast) and exposed it as a table so my coworker can use his beloved SQL to query different sources.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">The table <em>sentiment</em> (that we defined from our DataFrame) will be queried as any other table in his system. Another possibility is that we could query other data sources (Cassandra, Xmls, or our own binary formatted files) using Spark SQL and cross them with the stream.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">You can find out more information about this topic <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=https%3A%2F%2Fmedium.com%2F%40anicolaspp%2Fapache-spark-as-a-distributed-sql-engine-4373e254e0f9%23.55n08p6w4" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank"><em>here</em></a> and <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=https%3A%2F%2Fmedium.com%2F%40anicolaspp%2Fextending-our-spark-sql-query-engine-5f4a088de986%23.9jm66wp3o" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank"><em>here</em></a><em>.</em></p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">An example of querying a DataFrame is shown next.</p>
         <pre class="language-none line-numbers"><code>sqlContext.sql("select * from sentiments").show()<span class="line-numbers-rows"><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">WINDOWED OPERATIONS</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Spark Streaming has the ability to look back in the stream, a functionality most streaming engines lack (if they do have this functionality, it's very hard to implement).</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">In order to implement a windowed operation, you'll need to <em>checkpoint</em> the stream, but this is an easy task. You'll find more information about this <a class="jive-link-external-small" href="https://community.mapr.com/external-link.jspa?url=http%3A%2F%2Fspark.apache.org%2Fdocs%2Flatest%2Fstreaming-programming-guide.html%23checkpointing" rel="nofollow" style="color: #c8102e; background-color: transparent; font-weight: 400; font-size: 17px;" target="_blank"><em>here</em></a>.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Here's a small example of this kind of operation:</p>
         <pre class="language-none line-numbers"><code>tags&nbsp; <br>&nbsp;&nbsp; .window(Minutes(1)) . (...)<span class="line-numbers-rows"><span></span><span></span></span></code></pre>
         <p style="min-height: 8pt; padding: 0px; color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">&nbsp;</p>
         <h2 style="color: #000000; background-color: #ffffff; font-weight: 800; font-size: 30px; margin: 2rem auto;">CONCLUSION</h2>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Even though our examples are quite simple, we were able to solve a real life problem using Spark. We now have the ability to identify trending topics on Twitter, which helps us both target and increase our audience. At the same time, we are able to access different data sets using a single set of tools such as SQL.</p>
         <p style="color: #000000; background-color: #ffffff; font-size: 17px; margin: 2rem auto;">Very interesting results came back from <span style="font-weight: bold;"><strong><em>#bigdata</em></strong></span> and <span style="font-weight: bold;"><strong><em>#food</em></strong></span> at the same time. Perhaps people Tweet about big data at lunch timewho knows?</p>
