           <p>OpenWhisk is a cloud-first distributed event-based programming service that is integrated with <span data-widget="tooltip" title="IBM's Platform as a Service (PaaS) environment powered by
                        Cloud Foundry">IBM Bluemix</span>®. OpenWhisk uses a <a href="https://developer.ibm.com/openwhisk/what-is-serverless-computing/">serverless deployment and operations model</a> that hides infrastructure complexity and lets you focus on the code you want to execute.</p>
           <p>Even though OpenWhisk is designed for internal processing, it offers several benefits for user-facing applications. When you combine it with a static file, you can use it to serve applications that are relatively easy to debug. And because OpenWhisk applications are a lot less computationally intensive than running a server process on a PaaS platform, they are considerably cheaper, as well.</p>
           <p>Another benefit of using OpenWhisk is its modular architecture, which lets you implement your application as a collection of microservices. You can easily modify parts of the user interface without changing the rest of it, or reuse panels in multiple applications.</p>
           <p>In this tutorial, we'll start by writing a trivial "Hello OpenWhisk" application. Then we'll expand that application into the start of a more useful one, a multi-language phrase book.</p>
           <p class="ibm-button-link"><a class="ibm-btn-pri dw-article-btn-pri" href="https://github.com/qbzzt/bluemix/tree/master/openwhisk/201706">Get the code on GitHub</a></p>

           <h2 id="N100A0" class="ibm-h2">A trivial app: "Hello OpenWhisk"</h2>

           <h3 id="N101A6" class="ibm-h3">Browser-side code</h3>
           <p>In the past, it was necessary to use a Node.js application as a conduit to OpenWhisk. But now that APIs can have CORS (cross-origin resource sharing) enabled, all you need is an index.html file that can come from anywhere.</p>
           <p>All you need to do is download the file from GitHub, modify the value of the owUrl variable in line 21 to the "Route" value, and open the file in a browser. It will call the trivial OpenWhisk action, display the HTML in the response, and run the JavaScript</p>
           <p>The browser-side code uses the <a href="https://angularjs.org/">Angular library</a> (the "A" in MEAN stack). That library comes with a service called <code>$http</code> that allows the JavaScript code in the browser to act as a web client. </p>
           <p>The function that performs most of the work is <code>$scope.getHtml</code>. This function appends the name of the action to the route URL and attempts to get it.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true;">$http.get(owUrl + "/" + action)</pre>
           <p>If the request is successful, the result will be in <code>response.data</code>, already parsed (correctly, as JSON). This code uses jQuery. <code>$("# " + tag)</code> searches for an HTML tag whose id attribute is <code>tag</code>. Then, the <code>.html(&lt;string&gt;)</code> method sets that tag's HTML to the string, in this case <code>response.data.html</code>, the result from OpenWhisk. Note that this is normally a risky operation. But it's acceptable here as long as we make sure the HTML we send from OpenWhisk is safe.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">          		.then(function(response) {
          			$("# " + tag).html(response.data.html);</pre>
           <p>The <code>eval</code> function gets the JavaScript code in <code>response.data.js</code>, and executes it.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true;">        			eval(response.data.js);</pre>
           <h2 id="N101E9" class="ibm-h2">A more useful app: Phrase translator</h2>
           <p>Now that we have a trivial "Hello OpenWhisk" application running, let's turn it into a useful application. This application lets you select a word in English, and gives you that same word in multiple languages. The application's list of words is hard wired for the sake of simplicity. If you want to use a database, you do it the same way you would with a Node.js application.</p>
           <h3 id="N101F2" class="ibm-h3">Create two panels</h3>
           <p>In this application we use two panels—a side panel for navigation and a main panel to show information. These two panels are specified in the <code>panels</code> action in OpenWhisk. They are placed side-by-side using the <a href="http://getbootstrap.com/">Bootstrap framework</a>.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">function main(params) {// Store  the HTML to send in this variable
	var html = "";

	// The two main panels
	html = '&lt;div class="col-md-3" id="side"&gt;&lt;/div&gt;' +
		'&lt;div class="col-md-9" id="main"&gt;&lt;/div&gt;';
		
	// The row-fluid
	html = '&lt;div class="row-fluid"&gt;' + html + "&lt;/div&gt;";
	
	// The container-fluid
	html = '&lt;div class="container-fluid"&gt;' + html + "&lt;/div&gt;";</pre>
           <p>Note that the two panels have <code>id</code> attributes. The JavaScript code returned by the action calls <code>getHtml</code> to fill those two panels.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">  return {
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"html": html,
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;"js": '$scope.getHtml("side", "sidepanel"); $scope.getHtml("main", "mainpanel");'
&nbsp;&nbsp;&nbsp;&nbsp;};
  }</pre>
           <p>After you create the action and make it live, add to the API a new operation with the path <code>/panels</code> to call it.</p>
           <img src="Figure4.png" class="ibm-downsize" alt="Add a new operation with the /panels path to the API" height="697" width="975">

           <h3 id="N1021B" class="ibm-h3">Create the side panel</h3>
           <p>The side panel displays a list of words as buttons. When any of them is pressed, it uses the <code>getHtml</code> function to fill the main panel with information about that word. This is the action:</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">function main(params) {

	var words = ["Hello", "Goodbye", "Thanks", "You're welcome"];
	var html = "";
	
	for (var i=0; i&lt;words.length; i++) {</pre>
           <p>This code is outside the Angular scope, so it cannot call <code>$scope.getHtml</code>. To solve this, the stub creates a global variable called <code>scope</code> and assigns it the value of <code>$scope</code>. That function itself is within the Angular scope, so it can use <code>$http</code>. The word itself is escaped to allow for expressions that contain apostrophes, such as "You're welcome."</p>
           <p>The query is used to as part of the action as an easy way to transfer a parameter to the main panel action. It appears in the input, as you will see in the next section.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">		var onClick = "scope.getHtml('main', 'mainpanel?word=" + escape(words[i]) + "')";
		html += '&lt;p&gt;&lt;button type="button" class="btn btn-info" onClick="'
			+ onClick + '"&gt;' + words[i] + '&lt;/button&gt;&lt;/p&gt;';
	}</pre>
           <p>The return value does not contain JavaScript, because there is no JavaScript we need to run when the HTML is displayed.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">	return { "html": html };
}</pre>
           <p>Add this action to the API as <code>/sidepanel</code>.</p>
           <h3 id="N10245" class="ibm-h3">Create the main panel</h3>
           <p>The main panel shows a word and all of the translations known to the program. It uses the parameter passed to <code>main</code> to get the word, and builds up the HTML based on a hash table. Again, it does not need to tell the browser to run any JavaScript.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">function main(params) {
	var words = {
		"Hello" : {
			Hebrew: "Shalom",
			Spanish: "Hola",
			Arabic: "Marchaba"
		}
		// The full list of words is in GitHub
	};
	
	var html = "";
	
	// Heading
	html += "&lt;h2&gt;" + params.word + "&lt;/h2&gt;";</pre>
           <p>This parameter comes from the query part of the URL that's used by the side panel.</p>
           <pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">	var list = words[params.word];
	
	if (list == undefined)
		html += "I am not familiar with this word, sorry";
	else {
		var langs = Object.keys(list);
		
		html += '&lt;table class="table table-striped"&gt;';
		for (var i=0; i&lt;langs.length; i++)
			html += '&lt;tr&gt;&lt;th&gt;' + langs[i] + '&lt;/th&gt;&lt;td&gt;' + list[langs[i]] + '&lt;/td&gt;&lt;/tr&gt;';
		html += "&lt;/table&gt;";	
	}

	return { "html": html };
}</pre>
           <p>Add this action to the API as <code>/mainpanel</code>.</p>
           <h3 id="N1025C" class="ibm-h3">Modify index.html</h3>
           <p>Finally, change line 41 in index.html to get <code>panels</code> instead of <code>trivial</code>.</p>
           <h2 id="N1026A" class="ibm-h2">Conclusion</h2>
           <p>I hope these simple applications have given you a feel for the power of OpenWhisk. The OpenWhisk platform allows you to develop and execute logic in response to events in applications without having to worry about infrastructure.</p>
           <p>OpenWhisk is available as a <a href="https://new-console.ng.bluemix.net/openwhisk/">Bluemix service</a> and as an <a href="https://github.com/openwhisk">open source project on GitHub</a> to encourage developers to accelerate its development and to generate a powerful ecosystem of event providers and consumers.</p>
