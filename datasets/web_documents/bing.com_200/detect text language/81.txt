                <a name="ApiLoading"></a> 
                <h2><span style="font-size:85%;">Including the AJAX Language API on Your Page</span></h2> 
                <p>To include the AJAX Language API in your page, you will utilize the <a href="http://code.google.com/apis/ajax/documentation/">Google AJAX API loader</a>. The common loader allows you to load all of the AJAX apis that you need, including the language API. You need to both include the Google AJAX APIs script tag and call <code>google.load("language", "1")</code>:</p> 
                <pre class="prettyprint"><span class="pun">&lt;</span><span class="tag">script</span><span class="pln"> </span><span class="atn">type</span><span class="pun">=</span><span class="atv">"text/javascript"</span><span class="pln"> </span><span class="atn">src</span><span class="pun">=</span><span class="atv">"http://www.google.com/jsapi"</span><span class="pun">&gt;
                  <!--</span--><span class="tag">script</span><span class="pun">&gt;</span><span class="pln"><br></span><span class="pun">&lt;</span><span class="tag">script</span><span class="pln"> </span><span class="atn">type</span><span class="pun">=</span><span class="atv">"text/javascript"</span><span class="pun">&gt;</span><span class="pln"><br>google</span><span class="pun">.</span><span class="pln">load</span><span class="pun">(</span><span class="str">"language"</span><span class="pun">,</span><span class="pln"> </span><span class="str">"1"</span><span class="pun">);</span><span class="pln"><br></span><span class="pun">
                   <!--</span--><span class="tag">script</span><span class="pun">&gt;</span></span></span></pre> 
                <p>The first script tag loads the <code>google.load</code> function, which lets you load individual Google APIs. <code>google.load("language", "1")</code> loads <i>Version 1</i> of the <i>Language</i> API. Currently the AJAX Language API is in Version 1, but new versions may be available in the future. See the <a href="http://code.google.com/apis/ajaxlanguage/documentation/#Versioning">versioning discussion</a> below for more information.</p> 
                <a name="Versioning"></a> 
                <h2><span style="font-size:85%;">API Updates</span></h2> 
                <p>The second argument to <code>google.load</code> is the <i>version</i> of the AJAX Language API you are using. Currently the AJAX Language API is in version <b>1</b>, but new versions may be available in the future.</p> 
                <p>If we do a significant update to the API in the future, we will change the version number and post a notice on <a href="http://code.google.com/">Google Code</a> and the <a href="http://googleajaxsearchapi.blogspot.com/">AJAX APIs discussion group</a>. When that happens, we expect to support both versions for at least a month in order to allow you to migrate your code.</p> 
                <p>The AJAX Language API team periodically updates the API with the most recent bug fixes and performance enhancements. These bug fixes should only improve performance and fix bugs, but we may inadvertently break some API clients. Please use the <a href="http://googleajaxsearchapi.blogspot.com/">AJAX APIs discussion group</a> to report such issues.</p>

                <h1><span style="font-size:85%;">Flash and other Non-Javascript Environments<sup class="new">New!</sup></span></h1> 
                <p> For Flash developers, and those developers that have a need to access the AJAX Language API from other Non-Javascript environments, the API exposes a simple RESTful interface. In all cases, the method supported is <code>GET</code> and the response format is a <a href="http://www.json.org/">JSON</a> encoded result with embedded status codes. Applications that use this interface must abide by all existing <a href="http://code.google.com/apis/ajaxlanguage/terms.html">terms of use</a>. An area to pay special attention to relates to correctly identifying yourself in your requests. Applications MUST always include a valid and accurate <a href="http://en.wikipedia.org/wiki/HTTP_referer">http referer header</a> in their requests. In addition, we ask, but do not require, that each request contains a valid API Key. By providing a key, your application provides us with a secondary identification mechanism that is useful should we need to contact you in order to correct any problems. </p> 
                <p> The easiest way to start learning about this interface is to try it out... Using the command line tool <code>curl</code> or <code>wget</code> execute the following command: </p> 
                <pre class="code">curl -e http://www.my-ajax-site.com \<br>    'http://ajax.googleapis.com/ajax/services/language/translate?v=1.0&amp;q=hello%20world&amp;langpair=en%7Cit'<br></pre> 
                <p> This command performs a Language Translation(<code>/ajax/services/language/translate</code>), for <em>Hello World</em> (<code>q=hello%20world</code>) from English to Italian (<code>langpair=en%7Cit</code>). The response has a <code>Content-Type</code> of <code>text/javascript; charset=utf-8</code>. You can see from the response below that the <code>responseData</code> is identical to the properties described in the <a href="http://code.google.com/apis/ajaxlanguage/documentation/reference.html#translateResult">Result Objects</a> documentation. </p> 
                <pre class="prettyprint"><span class="pun">{</span><span class="str">"responseData"</span><span class="pun">:</span><span class="pln"> </span><span class="pun">{</span><span class="pln"><br> </span><span class="str">"translatedText"</span><span class="pun">:</span><span class="str">"Ciao mondo"</span><span class="pln"><br></span><span class="pun">},</span><span class="pln"><br></span><span class="str">"responseDetails"</span><span class="pun">:</span><span class="pln"> </span><span class="kwd">null</span><span class="pun">,</span><span class="pln"> </span><span class="str">"responseStatus"</span><span class="pun">:</span><span class="pln"> </span><span class="lit">200</span><span class="pun">}</span></pre> 
                <p> In addition to this response format, the protocol also supports a classic JSON-P style callback which is triggered by specifying a <code>callback</code> argument. When this argument is present the json object is delivered as an argument to the specified callback. </p> 
                <pre class="prettyprint"><span class="pln">callbackFunction</span><span class="pun">(</span><span class="pln"><br></span><span class="pun">{</span><span class="str">"responseData"</span><span class="pun">:</span><span class="pln"> </span><span class="pun">{</span><span class="pln"><br></span><span class="str">"translatedText"</span><span class="pun">:</span><span class="str">"Ciao mondo"</span><span class="pln"><br></span><span class="pun">},</span><span class="pln"><br></span><span class="str">"responseDetails"</span><span class="pun">:</span><span class="pln"> </span><span class="kwd">null</span><span class="pun">,</span><span class="pln"> </span><span class="str">"responseStatus"</span><span class="pun">:</span><span class="pln"> </span><span class="lit">200</span><span class="pun">})</span></pre> 
                <p> And finally, the protocol supports a <code>callback</code> and <code>context</code> argument. When these url arguments are specified, the response is encoded as a direct Javascript call with a signature of: <code>callback(context, result, status, details, unused)</code>. Note the slight difference in the following command and response. </p> 
                <pre class="code">curl -e http://www.my-ajax-site.com \<br>'http://ajax.googleapis.com/ajax/services/language/translate?v=1.0&amp;q=hello%20world&amp;langpair=en%7Cit&amp;callback=foo&amp;context=bar'<br></pre> 
                <p> This command performs a Language Translate and is identical to the previous call, BUT has been altered to pass both <code>callback</code> and <code>context</code>. With these arguments in place, instead of a JSON object being returned, a Javascript call is returned in the response and the JSON object is passed via the <code>result</code> parameter. </p> 
                <pre class="prettyprint"><span class="pln">foo</span><span class="pun">(</span><span class="str">'bar'</span><span class="pun">,</span><span class="pln"> </span><span class="pun">{</span><span class="str">"translatedText"</span><span class="pun">:</span><span class="str">"Ciao mondo"</span><span class="pun">},</span><span class="pln"> </span><span class="lit">200</span><span class="pun">,</span><span class="pln"> </span><span class="kwd">null</span><span class="pun">)<br><br>curtsy: http://code.google.com<br></span></pre> 
                <div style="clear: both;"></div>
