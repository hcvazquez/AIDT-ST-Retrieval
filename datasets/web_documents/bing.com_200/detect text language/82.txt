        <p class="date">9:09 pm <em>in <a href="http://scug.be/sccm/category/app-v/" rel="category tag">App-V</a>, <a href="http://scug.be/sccm/category/app-v-5-0/" rel="category tag">App-V 5.0</a>, <a href="http://scug.be/sccm/category/application-model/" rel="category tag">Application Model</a>, <a href="http://scug.be/sccm/category/appv/" rel="category tag">AppV</a>, <a href="http://scug.be/sccm/category/cm12/" rel="category tag">CM12</a>, <a href="http://scug.be/sccm/category/configmgr-2012-sp1/" rel="category tag">ConfigMgr 2012 SP1</a>, <a href="http://scug.be/sccm/category/deployment/" rel="category tag">Deployment</a>, <a href="http://scug.be/sccm/category/deployment-types/" rel="category tag">deployment types</a>, <a href="http://scug.be/sccm/category/detection-methods/" rel="category tag">detection methods</a>, <a href="http://scug.be/sccm/category/operating-system-deployment/" rel="category tag">Operating System Deployment</a>, <a href="http://scug.be/sccm/category/osd/" rel="category tag">OSD</a>, <a href="http://scug.be/sccm/category/sccm/" rel="category tag">sccm</a>, <a href="http://scug.be/sccm/category/sccm-2012-sp1/" rel="category tag">SCCM 2012 SP1</a>, <a href="http://scug.be/sccm/category/vbscript/" rel="category tag">vbscript</a> by <a href="http://scug.be/members/kenny/" title="Kenny Buntinx [MVP]">Kenny Buntinx [MVP]</a></em></p> 
        <div class="entry"> 
         <p>&nbsp;</p> 
         <p>In this blog post we will show you how to create the application for deploying the App-V 5.0 client which is part of the MDOP ( Microsoft Desktop Optimization Pack )suite.</p> 
         <p>App-V 5.0 client is supported on the following platforms (As you can see , there is NO XP support) : </p> 
         <p><a href="http://scug.be/sccm/files/2013/01/image5.png"><img title="image" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="image" src="http://scug.be/sccm/files/2013/01/image_thumb5.png" width="500" height="78"></a></p> 
         <p>First we need to have a look on the exact prerequisites needed to run the App-V 5.0 client . You will find that kind of information here : <a title="http://technet.microsoft.com/en-us/library/jj713458.aspx" href="http://technet.microsoft.com/en-us/library/jj713458.aspx">http://technet.microsoft.com/en-us/library/jj713458.aspx</a></p> 
         <p>However the information on what version you need on the Microsoft Visual C++ is vague. We will clarify that below :</p> 
         <ol> 
          <li>Microsoft Windows .NET Framework 4 (Full Package) (<a href="http://www.microsoft.com/en-us/download/details.aspx?id=17718)">http://www.microsoft.com/en-us/download/details.aspx?id=17718)</a> </li> 
          <li>Windows PowerShell 3.0 (<a href="http://www.microsoft.com/en-us/download/details.aspx?id=34595)">http://www.microsoft.com/en-us/download/details.aspx?id=34595)</a> </li> 
          <li>Download and install KB2533623 (<a href="http://www.microsoft.com/en-us/download/details.aspx?id=26746)">http://www.microsoft.com/en-us/download/details.aspx?id=26746)</a> </li> 
          <li>The Microsoft Visual C++ 2010 SP1 x64 Redistributable –&gt; v10.0.40219 (<a href="http://go.microsoft.com/fwlink/?LinkId=268896)">http://go.microsoft.com/fwlink/?LinkId=268896)</a> </li> 
         </ol> 
         <p><a href="http://scug.be/sccm/files/2013/01/image6.png"><img title="image" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; margin: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="image" src="http://scug.be/sccm/files/2013/01/image_thumb6.png" width="244" height="102"></a></p> 
         <ol> 
          <li>The Microsoft Visual C++ 2005 SP1 x86 Redistributable –&gt; v8.0.61001 <a title="http://www.microsoft.com/en-us/download/details.aspx?id=26347" href="http://www.microsoft.com/en-us/download/details.aspx?id=26347">http://www.microsoft.com/en-us/download/details.aspx?id=26347</a> </li> 
         </ol> 
         <p><a href="http://scug.be/sccm/files/2013/01/image7.png"><img title="image" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; margin: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="image" src="http://scug.be/sccm/files/2013/01/image_thumb7.png" width="244" height="102"></a></p> 
         <p>Now we need to create for each component an application . Lets start :</p> 
         <p><strong><u>Windows .NET Framework 4 (Full Package)</u></strong></p> 
         <p><a href="http://scug.be/sccm/files/2013/01/Knipsel.png"><img title="Knipsel" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Knipsel" src="http://scug.be/sccm/files/2013/01/Knipsel_thumb.png" width="329" height="146"></a></p> 
         <p><a href="http://scug.be/sccm/files/2013/01/Knipsel2.png"><img title="Knipsel2" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Knipsel2" src="http://scug.be/sccm/files/2013/01/Knipsel2_thumb.png" width="331" height="144"></a></p> 
         <p><a href="http://scug.be/sccm/files/2013/01/Knipsel3.png"><img title="Knipsel3" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Knipsel3" src="http://scug.be/sccm/files/2013/01/Knipsel3_thumb.png" width="332" height="284"></a></p> 
         <p>Full installation code : <strong>dotNetFx40_Full_x86_x64.exe /q /norestart /ChainingPackage ADMINDEPLOYMENT</strong></p> 
         <p><a href="http://scug.be/sccm/files/2013/01/Knipsel4.png"><img title="Knipsel4" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Knipsel4" src="http://scug.be/sccm/files/2013/01/Knipsel4_thumb.png" width="334" height="286"></a></p> 
         <p>Create 2 detection methods as shown below</p> 
         <p><a href="http://scug.be/sccm/files/2013/01/Knipsel5.png"><img title="Knipsel5" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="Knipsel5" src="http://scug.be/sccm/files/2013/01/Knipsel5_thumb.png" width="331" height="323"></a></p>

         <p><a href="http://scug.be/sccm/files/2013/01/311.png"><img title="31" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="31" src="http://scug.be/sccm/files/2013/01/31_thumb.png" width="339" height="290"></a></p>

         <p>In this post I’m going to outline how to handle multiple languages in a Windows 7 <strong>Enterprise</strong> deployment. Windows 7 <strong>Enterprise </strong>comes natively as a MultiLanguage User Interface type OS to which you can add on Language Packs. A language pack allows user to change the interface display language of Windows’ dialog boxes, menus and other text to the selected language. You can download/install them in a few different ways.</p> 
         <p>They take a long time to install and in our scenario it’s better to integrate them into your hybrid base .WIM image.</p>

         <p><strong><u>Scenario 1 :</u></strong></p> 
         <p>So let’s start with the first example , being certificate deployment as an application in a&nbsp; task sequence :</p> 
         <p>Let’s ask ourselves first this question : Why would we use an application to deploy certificates during an OSD deployment if we have Group Policy Object to do it for us ? Well , the answer is :&nbsp; This Microsoft document at: <a href="http://technet.microsoft.com/en-us/library/bb693951.aspx">http://technet.microsoft.com/en-us/library/bb693951.aspx</a> states that "The <strong>Setup Windows and ConfigMgr”</strong> task sequence action is responsible for running Group Policy on the newly installed computer. At which point during the task sequence action that Group Policy is applied depends on the operating system being deployed. On Windows XP and Windows Server 2003, Group Policy runs after the task sequence is finished, the task sequence GINA has been unloaded and then replaced with the GINA on Windows. On Windows Vista and Windows Server 2008, Group Policy runs after the <strong>Setup Windows and ConfigMgr</strong> task sequence action completes. "</p> 
         <p>Let’s build our Application now :</p>

         <p>In my case : HKLM\SOFTWARE\Microsoft\SystemCertificates\<em><font color="#ff0000"><strong><u>TrustedPublisher</u></strong>\</font></em>Certificates\6BFF5439A57586FEF61B8D8E2194A96DD459B511 and HKLM\SOFTWARE\Microsoft\SystemCertificates\<font color="#ff0000"><strong><em>AuthRoot</em></strong>\</font>Certificates\6BFF5439A57586FEF61B8D8E2194A96DD459B511</p>

         <p>Where does the “6BFF5439A57586FEF61B8D8E2194A96DD459B511” value comes from ? It will come from the properties of your exported certificate.</p> 
         <p>Take the properties of your certificate and take a look at the value “ Thumbprint” as shown below :</p> 
         <p><a href="http://scug.be/sccm/files/2012/12/SNAGHTML10644ee2.png"><img title="SNAGHTML10644ee2" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="SNAGHTML10644ee2" src="http://scug.be/sccm/files/2012/12/SNAGHTML10644ee2_thumb.png" width="233" height="290"></a></p> 
         <p>6. Define your requirement here directly or with a global condition :</p> 
         <p><a href="http://scug.be/sccm/files/2012/12/SNAGHTML10661df4.png"><img title="SNAGHTML10661df4" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="SNAGHTML10661df4" src="http://scug.be/sccm/files/2012/12/SNAGHTML10661df4_thumb.png" width="397" height="340"></a></p> 
         <p>7. Your done ! </p> 
         <p><strong><u>Scenario 2 :</u></strong> <strong>Internet explorer 9 Post install hotfix</strong></p>

         <p><a href="http://scug.be/sccm/files/2012/12/SNAGHTML1074c859.png"><img title="SNAGHTML1074c859" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="SNAGHTML1074c859" src="http://scug.be/sccm/files/2012/12/SNAGHTML1074c859_thumb.png" width="428" height="366"></a></p> 
         <p>5. Now we will use a script as detection method . If the exit code is a non-zero value, then the script has failed and the application detection status is unknown. If the exit code is zero and STDOUT contains data then the application detection state is installed. –&gt; see <a title="http://technet.microsoft.com/en-us/library/b2483e0f-3b9b-4551-ba5e-19fe0f5be3be#BKMK_Step4" href="http://technet.microsoft.com/en-us/library/b2483e0f-3b9b-4551-ba5e-19fe0f5be3be#BKMK_Step4">http://technet.microsoft.com/en-us/library/b2483e0f-3b9b-4551-ba5e-19fe0f5be3be#BKMK_Step4</a></p> 
         <p>From my experience, if you use a vbscript method of detection; any returned value from wscript.echo, anything at all, means that the detection passed.&nbsp; Even if you wscript.echo "FALSE", it doesn’t matter; something was returned, the script passed.</p> 
         <p>If you don’t want it to pass detection, simply don’t echo anything.</p>
