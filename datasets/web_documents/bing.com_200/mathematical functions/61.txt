           </svg></a>Description</h2> 
         <p>Parses and evaluates mathematical expressions. It's a safer and more math-oriented alternative to using JavaScript’s <code>eval</code> function for mathematical expressions.</p> 
         <p>It has built-in support for common math operators and functions. Additionally, you can add your own JavaScript functions. Expressions can be evaluated directly, or compiled into native JavaScript functions.</p>

         <p>For example, the following will create a <code>Parser</code> that does not allow comparison or logical operators:</p>

         <p>Evaluate the expression, with variables bound to the values in {variables}. Each variable in the expression is bound to the corresponding member of the <code>variables</code> object. If there are unbound variables, <code>evaluate</code> will throw an exception.</p> 
         <pre><code>js&gt; expr = Parser.parse("2 ^ x");
(2^x)
js&gt; expr.evaluate({ x: 3 });

</code></pre>

           </svg></a>simplify(variables: object)</h4> 
         <p>Simplify constant sub-expressions and replace variable references with literal values. This is basically a partial evaluation, that does as much of the calculation as it can with the provided variables. Function calls are not evaluated (except the built-in operator functions), since they may not be deterministic.</p> 
         <p>Simplify is pretty simple. For example, it doesn’t know that addition and multiplication are associative, so <code>((2*(4*x))+1)</code> from the previous example cannot be simplified unless you provide a value for x. <code>2*4*x+1</code> can however, because it’s parsed as <code>(((2*4)*x)+1)</code>, so the <code>(2*4)</code> sub-expression will be replaced with "8", resulting in <code>((8*x)+1)</code>.</p> 
         <pre><code>js&gt; expr = Parser.parse("x * (y * atan(1))").simplify({ y: 4 });
(x*3.141592653589793)
js&gt; expr.evaluate({ x: 2 });
6.283185307179586
</code></pre> 
         <h4><a href="#variablesoptions-object" aria-hidden="true" class="anchor" id="user-content-variablesoptions-object">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>variables(options?: object)</h4> 
         <p>Get an array of the unbound variables in the expression.</p> 
         <pre><code>js&gt; expr = Parser.parse("x * (y * atan(1))");
(x*(y*atan(1)))
js&gt; expr.variables();
x,y
js&gt; expr.simplify({ y: 4 }).variables();
x
</code></pre> 
         <p>By default, <code>variables</code> will return "top-level" objects, so for example, <code>Parser.parse(x.y.z).variables()</code> returns <code>['x']</code>. If you want to get the whole chain of object members, you can call it with <code>{ withMembers: true }</code>. So <code>Parser.parse(x.y.z).variables({ withMembers: true })</code> would return <code>['x.y.z']</code>.</p> 
         <h4><a href="#symbolsoptions-object" aria-hidden="true" class="anchor" id="user-content-symbolsoptions-object">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>symbols(options?: object)</h4> 
         <p>Get an array of variables, including any built-in functions used in the expression.</p> 
         <pre><code>js&gt; expr = Parser.parse("min(x, y, z)");
(min(x, y, z))
js&gt; expr.variables();
min,x,y,z
js&gt; expr.simplify({ y: 4, z: 5 }).variables();
min,x
</code></pre> 
         <p>Like <code>variables</code>, <code>symbols</code> accepts an option argument <code>{ withMembers: true }</code> to include object members.</p> 
         <h4><a href="#tostring" aria-hidden="true" class="anchor" id="user-content-tostring">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>toString()</h4> 
         <p>Convert the expression to a string. <code>toString()</code> surrounds every sub-expression with parentheses (except literal values, variables, and function calls), so it’s useful for debugging precedence errors.</p> 
         <h4><a href="#tojsfunctionparameters-array--string-variables-object" aria-hidden="true" class="anchor" id="user-content-tojsfunctionparameters-array--string-variables-object">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>toJSFunction(parameters: array | string, variables?: object)</h4> 
         <p>Convert an <code>Expression</code> object into a callable JavaScript function. <code>parameters</code> is an array of parameter names, or a string, with the names separated by commas.</p> 
         <p>If the optional <code>variables</code> argument is provided, the expression will be simplified with variables bound to the supplied values.</p> 
         <pre><code>js&gt; expr = Parser.parse("x + y + z");
((x + y) + z)
js&gt; f = expr.toJSFunction("x,y,z");
[Function] // function (x, y, z) { return x + y + z; };
js&gt; f(1, 2, 3)
6
js&gt; f = expr.toJSFunction("y,z", { x: 100 });
[Function] // function (y, z) { return 100 + y + z; };
js&gt; f(2, 3)
105
</code></pre>

           </svg></a>Unary operators</h4> 
         <p>The parser has several built-in "functions" that are actually unary operators. The primary difference between these and functions are that they can only accept exactly one argument, and parentheses are optional. With parentheses, they have the same precedence as function calls, but without parentheses, they keep their normal precedence (just below <code>^</code>). For example, <code>sin(x)^2</code> is equivalent to <code>(sin x)^2</code>, and <code>sin x^2</code> is equivalent to <code>sin(x^2)</code>.</p> 
         <p>The unary <code>+</code> and <code>-</code> operators are an exception, and always have their normal precedence.</p>
