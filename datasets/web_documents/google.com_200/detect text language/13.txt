              <p>I just wrote this tiny library called <a href="https://github.com/dhilipsiva/isRTL.coffee" rel="nofollow">isRTL.coffee</a> to determine the direction of the text. Is there any better way of doing this?</p> 
              <pre><code>rtlChars = '\u0600-\u06FF' # Arabic - Range

rtlChars += '\u0750-\u077F' # Arabic Supplement - Range

rtlChars += '\uFB50-\uFDFF' # Arabic Presentation Forms-A - Range

rtlChars += '\uFE70-\uFEFF' # Arabic Presentation Forms-B - Range

reRTL = new RegExp "^[#{rtlChars}]"

window.isRTL = (value) -&gt;
    if value.match reRTL
        true
    else
        false
</code></pre>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Yep, it's possible to improve. You can simply do:</p> 
              <pre><code>window.isRTL = (value) -&gt;
    reRTL.test value
</code></pre> 
              <p>Another way to improve is the way you declare your special characters. Right now, it's confusing between the <code>=</code> and <code>+=</code>. Here is another way:</p> 
              <pre><code>rtlChars = [
    '\u0600-\u06FF' # Arabic - Range
    '\u0750-\u077F' # Arabic Supplement - Range
    '\uFB50-\uFDFF' # Arabic Presentation Forms-A - Range
    '\uFE70-\uFEFF' # Arabic Presentation Forms-B - Range
].join("")

reRTL = new RegExp "^[#{rtlChars}]"
</code></pre> 
              <p>This way, adding a new character is simply adding a line in an array.</p>

             <div class="post-text" itemprop="text"> 
              <p>The real usefulness of such a library can only come from completeness. You should try to support all other RTL languages as well. Anybody can come up with a simple function to detect Arabic chars.</p> 
              <p>Also, what if the text contains a mix of languages, some RTL while others LTR. Your library would report any text containing at least one arabic character as RTL.</p> 
              <ul> 
               <li>One approach would be to report number of RTL chars v/s LTR chars.</li> 
               <li>Another take would be to parse the text into sections, each labeled as RTL or LTR.</li> 
              </ul>
