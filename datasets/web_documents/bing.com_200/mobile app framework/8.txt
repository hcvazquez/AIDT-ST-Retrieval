             <h1>Intro</h1>
            </div>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> For those who don’t know, 
            <strong>Hybrid App</strong> is a combination of native app and mobile web app concepts. Typically the native code is provided by a wrapper framework (like Phonegap) which bridges a gap between native side and a JavaScript API so that the app’s JavaScript code can perform native functions such as taking a picture with the camera. 
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> Unlike classic mobile web apps, which work with server side code, hybrid mobile apps will wrap 
            <strong>HTML</strong>, 
            <strong>JavaScript</strong>, and 
            <strong>CSS</strong> files locally (while in same cases it can still be used from a server side) so no Internet access is required. 
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> I will try to be as objective as I can. I used to be a 
            <strong>jQuery Mobile</strong> and 
            <strong>Sencha Touch</strong> developer, now I’m working with 
            <strong>OnsenUI</strong> and 
            <strong>Ionic Framework</strong>. I have also spent a lot of time playing with these frameworks. This, of course, doesn’t mean other frameworks are bad or inadequate, as you will see, my decision was hard, and it took time.

             <h1>1. Famo.us</h1>
            </div>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> Famo.us is a free, open-source JavaScript framework that helps you create smooth, sophisticated mobile applications. Like Ionic, Famo.us is made for hybrid mobile development, though, up to the point, it can be used for desktop web applications development (unfortunately, at this time, Chrome, Firefox, and Chrome usage is still buggy). But, and here’s one large BUT, this framework is nothing you have seen so far. Famo.us development looks more like an animation or a game system than like conventional mobile app design. 
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> This is because Famo.us is the only JavaScript framework that includes an open source 3D layout engine fully integrated with a 3D physics animation engine that can render to DOM, Canvas, or WebGL (this statement is only partly true because there are some other frameworks that can offer similar functionalities, like threejs.org).
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> 
            <span><a href="http://www.gajotres.net/best-html5-mobile-app-frameworks-famo-us-review" class="btn-primary2 add-shadow" target="" rel=""> <span class="icon-speccc icon ion-link"></span><span class="btn-primary-text-container">Detailed Review</span> </a></span> 
            <span><a href="https://famo.us/" class="btn-primary2 add-shadow" target="_blank" rel=""> <span class="icon-speccc icon ion-link"></span><span class="btn-primary-text-container">Link</span> </a></span> 
            <span><a href="https://github.com/Famous/famous" class="btn-primary2 add-shadow" target="_blank" rel=""> <span class="icon-speccc icon ion-link"></span><span class="btn-primary-text-container">GitHub</span> </a></span>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> 
            <h3><strong>Good</strong></h3>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> 
            <ul> 
             <li>Open-source and free</li> 
             <li>Usable with AngularJS</li> 
             <li>iOS and Android support (including buggy desktop support)</li> 
             <li>3D layout engine</li> 
             <li>3D physics based animation library</li> 
             <li>60 fps animations (though I would like to see this on lower end devices)</li> 
             <li>Can be rendered to DOM (Canvas, or WebGL are in a roadmap)</li> 
             <li>Supports <strong>Cordova</strong>, <strong>PhoneGap</strong>, <strong>Trigger.io</strong> or <strong>Crosswalk</strong> plus its own native wrapper</li> 
             <li>Help is available throug the IRC <a href="http://webchat.freenode.net/?channels=famous"><strong>chanell</strong></a></li> 
            </ul>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> 
            <h3><strong>Bad</strong></h3>
            <div style="width:100%;height: 20px;">
             &nbsp;
            </div> 
            <ul> 
             <li>It’s not a animation/tweening library nor it is a <strong>jQuery</strong> animation replacement tool</li> 
             <li>You can’t use it for classic web development</li> 
             <li>Famo.us is very different from many Javascript frameworks since it endorses zero touches to the DOM.</li> 
             <li>If you like HTML5 markup based frameworks then this is not a framework for you.</li> 
            </ul>

                   <p>It’s hard to predict if hybrid apps will replace native apps in foreseeable future.</p> 
                   <p>I can, with relative accuracy, state that it will not happen shortly. </p> 
                   <p>The reason, hybrid mobile frameworks are still not on the same performance level as native apps. It’s all about today smartphone’s processing power. They simply can’t fluidly handle all required animations, plus, in a given time, they can process only 10% of JavaScript actions compared to desktop computers; which is still not enough.</p> 
                   <p>From my experience, I know of only one framework that works on the (or almost near) same level as native applications, and that’s OnsenUI 2. But you should understand that I have several years of experience with hybrid mobile applications, I know what to do to make them as fast as possible. Plus even that knowledge will fail if I create a complex app. </p> 
                   <p>To conclude, currently, hybrid mobile apps are excellent for smaller and simple apps. I don’t see it been usable for anything else, maybe corporation apps that don’t have a large user base. Nothing will kill your app better than people expectations, and they want fluid experience.</p>
