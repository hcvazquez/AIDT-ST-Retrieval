       <li> To learn more about what filter and action hooks are, see <a href="/Plugin_API" title="Plugin API">Plugin API</a>. </li> 
       <li> To learn about writing plugins in general, see <a href="/Writing_a_Plugin" title="Writing a Plugin">Writing a Plugin</a>. </li> 
       <li> For a reference list of action hooks, see <a href="/Plugin_API/Action_Reference" title="Plugin API/Action Reference">Plugin API/Action Reference</a>. </li> 
       <li> For information about filter and action hooks in previous versions of WordPress, see <a href="/Plugin_API/Hooks_2.0.x" title="Plugin API/Hooks 2.0.x">Plugin API/Hooks 2.0.x</a>. </li>

      </ul> 
      <p>Note: If you want to add to or clarify this documentation, please follow the style of the existing entries. Describe what data the filter is applied to, and if the filter function takes additional arguments, describe the argument list. </p>

      <h2><span class="mw-headline" id="Post.2C_Page.2C_and_Attachment_.28Upload.29_Filters">Post, Page, and Attachment (Upload) Filters</span></h2> 
      <p>See also <a href="#Category_and_Term_Filters">#Category and Term Filters</a>, <a href="#Author_and_User_Filters">#Author and User Filters</a>, <a href="#Link_Filters">#Link Filters</a>, <a href="#Date_and_Time_Filters">#Date and Time Filters</a>, and <a href="#Administrative_Filters">#Administrative Filters</a> below. </p> 
      <h3><span class="mw-headline" id="Database_Reads">Database Reads</span></h3> 
      <p>Filters in this section are applied to information read from the database, prior to displaying on a page or editing screen. </p> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/attachment_fields_to_edit" title="Plugin API/Filter Reference/attachment fields to edit">attachment_fields_to_edit</a>&nbsp;
       </dt> 
       <dd>
         applied to the form fields to be displayed when editing an attachment. Called in the 
        <tt>get_attachment_fields_to_edit</tt> function. Filter function arguments: an array of form fields, the post object. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/attachment_icon" title="Plugin API/Filter Reference/attachment icon">attachment_icon</a>&nbsp;
       </dt> 
       <dd>
         applied to the icon for an attachment in the 
        <tt>get_attachment_icon</tt> function. Filter function arguments: icon file as an HTML IMG tag, attachment ID. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/attachment_innerHTML&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/attachment innerHTML (page does not exist)">attachment_innerHTML</a>&nbsp;
       </dt> 
       <dd>
         applied to the title to be used for an attachment if there is no icon, in the 
        <tt>get_attachment_innerHTML</tt> function. Filter function arguments: inner HTML (defaults to the title), attachment ID. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/author_edit_pre" title="Plugin API/Filter Reference/author edit pre">author_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post author prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/body_class" title="Plugin API/Filter Reference/body class">body_class</a>&nbsp;
       </dt> 
       <dd>
         applied to the classes for the HTML 
        <code>&lt;body&gt;</code> element. Called in the 
        <code><a href="/Function_Reference/get_body_class" title="Function Reference/get body class">get_body_class</a></code> function. Filter function arguments: an array of class names and an array of additional class names that were added to the first array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/content_edit_pre" title="Plugin API/Filter Reference/content edit pre">content_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post content prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/content_filtered_edit_pre" title="Plugin API/Filter Reference/content filtered edit pre">content_filtered_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post content filtered prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/excerpt_edit_pre" title="Plugin API/Filter Reference/excerpt edit pre">excerpt_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post excerpt prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/date_edit_pre" title="Plugin API/Filter Reference/date edit pre">date_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post date prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/date_gmt_edit_pre" title="Plugin API/Filter Reference/date gmt edit pre">date_gmt_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post date prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_attached_file&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get attached file (page does not exist)">get_attached_file</a>&nbsp;
       </dt> 
       <dd>
         applied to the attached file information retrieved by the 
        <tt>get_attached_file</tt> function. Filter function arguments: file information, attachment ID. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_enclosed&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get enclosed (page does not exist)">get_enclosed</a>&nbsp;
       </dt> 
       <dd>
         applied to the enclosures list for a post by the 
        <tt>get_enclosed</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_pages&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get pages (page does not exist)">get_pages</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of pages returned by the 
        <a href="/Function_Reference/get_pages" title="Function Reference/get pages"><tt>get_pages</tt> function</a>. Filter function arguments: list of pages (each item of which contains a page data array), 
        <tt>get_pages</tt> function argument list (telling which pages were requested). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/get_pung" title="Plugin API/Filter Reference/get pung">get_pung</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of pinged URLs for a post by the 
        <tt>get_pung</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_the_archive_title&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get the archive title (page does not exist)">get_the_archive_title</a>&nbsp;
       </dt> 
       <dd>
         applied to the archive's title in the 
        <tt>get_the_archive_title</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/get_the_excerpt" title="Plugin API/Filter Reference/get the excerpt">get_the_excerpt</a>&nbsp;
       </dt> 
       <dd>
         applied to the post's excerpt in the 
        <tt>get_the_excerpt</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/get_the_guid" title="Plugin API/Filter Reference/get the guid">get_the_guid</a>&nbsp;
       </dt> 
       <dd>
         applied to the post's GUID in the 
        <tt>get_the_guid</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/get_to_ping" title="Plugin API/Filter Reference/get to ping">get_to_ping</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of URLs to ping for a post by the 
        <tt>get_to_ping</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/icon_dir&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/icon dir (page does not exist)">icon_dir</a>&nbsp;
       </dt> 
       <dd>
         applied to the template's image directory in several functions. Basically allows a plugin to specify that icons for MIME types should come from a different location. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/icon_dir_uri&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/icon dir uri (page does not exist)">icon_dir_uri</a>&nbsp;
       </dt> 
       <dd>
         applied to the template's image directory URI in several functions. Basically allows a plugin to specify that icons for MIME types should come from a different location. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/image_size_names_choose" title="Plugin API/Filter Reference/image size names choose">image_size_names_choose</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of image sizes selectable in the Media Library. Commonly used to make 
        <a href="/Function_Reference/add_image_size" title="Function Reference/add image size">custom image sizes</a> selectable. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/mime_type_edit_pre" title="Plugin API/Filter Reference/mime type edit pre">mime_type_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post mime type prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/modified_edit_pre" title="Plugin API/Filter Reference/modified edit pre">modified_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post modification date prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/modified_gmt_edit_pre" title="Plugin API/Filter Reference/modified gmt edit pre">modified_gmt_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post modification gmt date prior to display for editing. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/no_texturize_shortcodes" title="Plugin API/Filter Reference/no texturize shortcodes">no_texturize_shortcodes</a>&nbsp;
       </dt> 
       <dd>
         applied to registered shortcodes. Can be used to exempt shortcodes from the automatic texturize function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/parent_edit_pre" title="Plugin API/Filter Reference/parent edit pre">parent_edit_pre</a>&nbsp;
       </dt> 
       <dd>
         applied to post parent id prior to display for editing. 
       </dd> 
      </dl>

      <h2><span class="mw-headline" id="WP_Query_Filters">WP_Query Filters</span></h2> 
      <p>These are filters run by the <a href="/Class_Reference/WP_Query" title="Class Reference/WP Query">WP_Query object</a> in the course of building and executing a query to retrieve posts. See also <a href="#Advanced_WordPress_Filters">#Advanced WordPress Filters</a> for queries relating to users, meta values, and more generic queries. </p> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/found_posts" title="Plugin API/Filter Reference/found posts">found_posts</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of posts, just after querying from the database. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/found_posts_query&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/found posts query (page does not exist)">found_posts_query</a>&nbsp;
       </dt> 
       <dd>
         after the list of posts to display is queried from the database, WordPress selects rows in the query results. This filter allows you to do something other than 
        <tt>SELECT FOUND_ROWS()</tt> at that step. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/post_limits" title="Plugin API/Filter Reference/post limits">post_limits</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>LIMIT</tt> clause of the query that returns the post array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_clauses" title="Plugin API/Filter Reference/posts clauses">posts_clauses</a>&nbsp;
       </dt> 
       <dd>
         applied to the entire SQL query, divided into a keyed array for each clause type, that returns the post array. Can be easier to work with than 
        <tt>posts_request</tt>. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_distinct" title="Plugin API/Filter Reference/posts distinct">posts_distinct</a>&nbsp;
       </dt> 
       <dd>
         allows a plugin to add a 
        <tt>DISTINCTROW</tt> clause to the query that returns the post array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/posts_fields&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/posts fields (page does not exist)">posts_fields</a>&nbsp;
       </dt> 
       <dd>
         applied to the field list for the query that returns the post array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_groupby" title="Plugin API/Filter Reference/posts groupby">posts_groupby</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>GROUP BY</tt> clause of the query that returns the post array (normally empty). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_join" title="Plugin API/Filter Reference/posts join">posts_join</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>JOIN</tt> clause of the query that returns the post array. This is typically used to add a table to the 
        <tt>JOIN</tt>, in combination with the 
        <tt>posts_where</tt> filter. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_join_paged" title="Plugin API/Filter Reference/posts join paged">posts_join_paged</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>JOIN</tt> clause of the query that returns the post array, after the paging is calculated (though paging does not affect the JOIN, so this is actually equivalent to 
        <tt>posts_join</tt>). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_orderby" title="Plugin API/Filter Reference/posts orderby">posts_orderby</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>ORDER BY</tt> clause of the query that returns the post array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_request" title="Plugin API/Filter Reference/posts request">posts_request</a>&nbsp;
       </dt> 
       <dd>
         applied to the entire SQL query that returns the post array, just prior to running the query. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_results" title="Plugin API/Filter Reference/posts results">posts_results</a>&nbsp;
       </dt> 
       <dd>
         allows you to manipulate the resulting array returned from the query. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_search" title="Plugin API/Filter Reference/posts search">posts_search</a>&nbsp;
       </dt> 
       <dd>
         applied to the search SQL that is used in the 
        <tt>WHERE</tt> clause of WP_Query. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/posts_where" title="Plugin API/Filter Reference/posts where">posts_where</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>WHERE</tt> clause of the query that returns the post array. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/posts_where_paged&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/posts where paged (page does not exist)">posts_where_paged</a>&nbsp;
       </dt> 
       <dd>
         applied to the 
        <tt>WHERE</tt> clause of the query that returns the post array, after the paging is calculated (though paging does not affect the WHERE, so this is actually equivalent to 
        <tt>posts_where</tt>). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/the_posts&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/the posts (page does not exist)">the_posts</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of posts queried from the database after minimal processing for permissions and draft status on single-post pages. 
       </dd> 
      </dl>

       </dd>

       <dt>

      <dl>

        <a href="/index.php?title=Plugin_API/Filter_Reference/get_next_post_join&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get next post join (page does not exist)">get_next_post_join</a>&nbsp;
       </dt> 
       <dd>
         in function 
        <tt>get_next_post</tt> (which finds the post after the currently-displayed post), applied to the SQL JOIN clause (which normally joins to the category table if user is viewing a category archive). Filter function arguments: JOIN clause, stay in same category (true/false), list of excluded categories. 
       </dd> 
      </dl> 
      <dl> 
       <dt>

       </dt> 
       <dd>
         in function 
        <tt>get_next_post</tt> (which finds the post after the currently-displayed post), applied to the SQL ORDER BY clause (which normally orders by post date in ascending order with a limit of 1 post). Filter function arguments: ORDER BY clause. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_next_post_where&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get next post where (page does not exist)">get_next_post_where</a>&nbsp;
       </dt> 
       <dd>
         in function 
        <tt>get_next_post</tt> (which finds the post after the currently-displayed post), applied to the SQL WHERE clause (which normally looks for the next dated published post). Filter function arguments: WHERE clause, stay in same category (true/false), list of excluded categories. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_previous_post_join&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get previous post join (page does not exist)">get_previous_post_join</a>&nbsp;
       </dt>

Filter function arguments: join clause, stay in same category (true/false), list of excluded categories. 
       </dd> 
      </dl> 
      <dl> 
       <dt>

       </dt>

        <a href="/Plugin_API/Filter_Reference/gettext" title="Plugin API/Filter Reference/gettext">gettext</a>&nbsp;

        <a href="/index.php?title=Function_Reference/translation&amp;action=edit&amp;redlink=1" class="new" title="Function Reference/translation (page does not exist)"><tt>translation()</tt></a> function (which is called by functions like the 
        <a href="/Function_Reference/_2" title="Function Reference/ 2"><tt>__()</tt></a> and 
        <a href="/Function_Reference/_e" title="Function Reference/ e"><tt>_e()</tt></a> internationalization functions ). Filter function arguments: translated text, untranslated text and the text domain. Gets applied even if internationalization is not in effect or if the text domain has not been loaded. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/override_load_textdomain&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/override load textdomain (page does not exist)">override_load_textdomain</a> 
       </dt> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_meta_sql&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get meta sql (page does not exist)">get_meta_sql</a>&nbsp;
       </dt> 
       <dd>
         in function 
        <tt>WP_Meta_Query::get_sql</tt> (which generates SQL clauses to be appended to a main query for advanced meta queries.), applied to the SQL JOIN and WHERE clause generated by the advanced meta query. Filter function arguments: array( compact( 'join', 'where' ), $this-&gt;queries, $type, $primary_table, $primary_id_column, $context ) 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_others_drafts&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get others drafts (page does not exist)">get_others_drafts</a>&nbsp;
       </dt> 
       <dd>
         applied to the query that selects the other users' drafts for display in the admin menus. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/get_users_drafts&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/get users drafts (page does not exist)">get_users_drafts</a>&nbsp;
       </dt> 
       <dd>
         applied to the query that selects the users' drafts for display in the admin menus. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/locale" title="Plugin API/Filter Reference/locale">locale</a>&nbsp;
       </dt> 
       <dd>
         applied to the locale by the 
        <tt>get_locale</tt> function. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/query&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/query (page does not exist)">query</a>&nbsp;
       </dt> 
       <dd>
         applied to all queries (at least all queries run after plugins are loaded). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/query_string&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/query string (page does not exist)">query_string</a>&nbsp;
       </dt> 
       <dd>
         deprecated - use 
        <tt>query_vars</tt> or 
        <tt>request</tt> instead. 
       </dd> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/query_vars" title="Plugin API/Filter Reference/query vars">query_vars</a>&nbsp;
       </dt> 
       <dd>
         applied to the list of public WordPress query variables before the SQL query is formed. Useful for removing extra permalink information the plugin has dealt with in some other manner. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/request" title="Plugin API/Filter Reference/request">request</a>&nbsp;
       </dt> 
       <dd>
         like 
        <tt>query_vars</tt>, but applied after "extra" and private query variables have been added. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/excerpt_length" title="Plugin API/Filter Reference/excerpt length">excerpt_length</a>&nbsp;
       </dt> 
       <dd>
         Defines the length of a single-post excerpt. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/excerpt_more" title="Plugin API/Filter Reference/excerpt more">excerpt_more</a>&nbsp;
       </dt> 
       <dd>
         Defines the more string at the end of the excerpt. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/post_edit_form_tag" title="Plugin API/Filter Reference/post edit form tag">post_edit_form_tag</a>&nbsp;
       </dt> 
       <dd>
         Allows you to append code to the form tag in the default post/page editor. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/update_user_query&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/update user query (page does not exist)">update_user_query</a>&nbsp;
       </dt> 
       <dd>
         applied to the update query used to update user information, prior to running the query. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/uploading_iframe_src&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/uploading iframe src (page does not exist)">uploading_iframe_src</a> (removed since WP 2.5)
       </dt> 
       <dd>
         applied to the HTML src tag for the uploading iframe on the post and page editing screens. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/index.php?title=Plugin_API/Filter_Reference/xmlrpc_methods&amp;action=edit&amp;redlink=1" class="new" title="Plugin API/Filter Reference/xmlrpc methods (page does not exist)">xmlrpc_methods</a>&nbsp;
       </dt> 
       <dd>
         applied to list of defined XMLRPC methods for the XMLRPC server. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/wp_mail_from" title="Plugin API/Filter Reference/wp mail from">wp_mail_from</a>&nbsp;
       </dt> 
       <dd>
         applied before any mail is sent by the wp_mail function. Supplied value is the calculated from address which is wordpress at the current hostname (set by $_SERVER['SERVER_NAME']). The filter should return an email address or name/email combo in the form "user@example.com" or "Name &lt;user@example.com&gt;" (without the quotes!). 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/wp_mail_from_name" title="Plugin API/Filter Reference/wp mail from name">wp_mail_from_name</a>&nbsp;
       </dt> 
       <dd>
         applied before any mail is sent by the wp_mail function. The filter should return a name string to be used as the email from name. 
       </dd> 
      </dl> 
      <dl> 
       <dt>
        <a href="/Plugin_API/Filter_Reference/update_(meta_type)_metadata" title="Plugin API/Filter Reference/update (meta type) metadata">update_(meta_type)_metadata</a>&nbsp;
       </dt> 
       <dd>
         applied before a metadata gets updated. For example if a user metadata gets updated the hook would be 'update_user_metadata' 
       </dd> 
      </dl>

        <a href="/Plugin_API/Filter_Reference/widget_tag_cloud_args" title="Plugin API/Filter Reference/widget tag cloud args">widget_tag_cloud_args</a>&nbsp;
