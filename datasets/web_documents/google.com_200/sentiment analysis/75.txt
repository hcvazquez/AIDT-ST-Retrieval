          <section class="section-article"> 
           <!-- .entry-header --> 
           <div class="entry-content post-content"> 
            <div> 
             <p><span style="font-size: 14pt"><strong>You want it you got it!</strong></span></p> 
             <p>&nbsp;</p> 
             <p>Finally we are publishing all HANA apps that all of us, B1 Solution Architects, developed to demonstrate how far you can go using SAP HANA!</p> 
             <p>&nbsp;</p> 
             <p>If you are seeing this application for the first time, you didn’t go to one of the Business One Innovation Summits 2015 (Bangkok, Barcelona or Miami). Shame on you and you have to be there next year.</p> 
             <p>&nbsp;</p> 
             <p>This application implements 2 interesting HANA functionalities:</p> 
             <p>&nbsp;</p> 
             <h2>1 – SAP HANA Text Analysis</h2> 
             <p>&nbsp;</p> 
             <p>This functionality allows us to handle “Unstructured Data” (in our case tweets). This feature classifies terms of a given input in several categories (People, companies, locations etc). Its is also possible to extract the whole meaning of an unstructured text and classify in a Positive or Negative Sentiment.</p> 
             <h2> 2 – SAP HANA Text Mining (Available on SPS09 only)</h2> 
             <p>&nbsp;</p> 
             <p>This feature works with a concept called <a class="jive-link-external-small" href="https://en.wikipedia.org/wiki/Bag-of-words_model">Bag of words</a> not only acting at a document level (tweet) but a whole set of documents (an entire table of tweets for example).</p> 
             <p>&nbsp;</p> 
             <p>With those features we can classify documents relevancy for a given input. E.g. give me the tweets that are more relevant for the term “Steve Jobs” or suggest me terms that are related to it (“apple”).</p> 
             <p>&nbsp;</p> 
             <p>Lets see the app running in this conceptual demonstration:</p> 
             <p>&nbsp;</p> 
             <p><iframe width="625" height="352" src="https://www.youtube.com/embed/q6dCTP9AkbA?feature=oembed" frameborder="0" allowfullscreen></iframe></p> 
             <p>&nbsp;</p> 
             <h2>App Installation (SPS08 or higher required)</h2> 
             <p>&nbsp;</p> 
             <p>As Sally’s just explained on the video. This solution in composed by 2 applications:</p> 
             <p>&nbsp;</p> 
             <p>You can find all installation files and a detailed presentation <a class="jive-link-external-small" href="https://www.dropbox.com/sh/cmboo7lgaut1qhd/AABA6GmoU8joxzghh-QzgTUca/Examples/Twitter%20Analysis%20App?dl=0">on my Dropbox</a>.</p> 
             <p>Or clone the repository on <a class="jive-link-external-small" href="https://github.com/Ralphive/StructurerOne">my GitHub</a></p> 
             <p>&nbsp;</p> 
             <h3>1 – Structurer One (The HANA App)</h3> 
             <p>&nbsp;</p> 
             <p>You just have to import the Delivery Unit (file .tgz) on your HANA System to have all the App structure set up. If you don’t know how to work with DUs, use this <a class="jive-link-external-small" href="http://help.sap.com/openSAP/HANA1/openSAP_HANA1_Week_01_How_to_Import_Delivery_Units.pdf">simple example of how to import it</a> as a guide.</p> 
             <p>&nbsp;</p> 
             <p>After the import, on the development perspective of HANA Studio, check the software repositories and execute all the SQL commands that are listed on the file <strong>Summit15 &gt; SQL &gt; CreateIndexes.sql</strong></p> 
             <p>&nbsp;</p> 
             <p><a href="http://i.imgur.com/iV3o1pF.png"><img style="margin-left: auto;margin-right: auto" src="http://i.imgur.com/iV3o1pF.png" alt="http://i.imgur.com/iV3o1pF.png" width="297" height="238"></a></p> 
             <p>&nbsp;</p> 
             <p>Those commands will Activate Text Analysis features on the app.</p> 
             <p>&nbsp;</p> 
             <p>The app will be at:</p> 
             <p align="center"><a class="jive-link-external-small" href="http://%3Cyourserver%3E:8000/Summit15/UI/index.html"><strong>http://&lt;yourserver&gt;:8000/Summit15/UI/index.html</strong></a></p> 
             <p align="center"> </p>
             <p align="center"><a href="http://i.imgur.com/67zI9ou.png"><img src="http://i.imgur.com/67zI9ou.png" alt="http://i.imgur.com/67zI9ou.png" width="659" height="333"></a></p> 
             <p align="center"> </p>
             <h3>2 – Tweets Retriever</h3> 
             <p>&nbsp;</p> 
             <p>To the app works completely, we need data. Of course you can input it manually, but the whole idea is to perform Real Time Analysis.</p> 
             <p>So, to gather tweets in real time we have a script written in Server Side JavaScript and runs on a framework called <a class="jive-link-external-small" href="http://nodejs.org/">Node.JS</a>.</p> 
             <p>You can install this framework in any OS and run the tweet retriever from your laptop for example.</p> 
             <p>&nbsp;</p> 
             <p>This is a small script is responsible to listen the <a class="jive-link-external-small" href="https://dev.twitter.com/">Twitter API</a> and every time a new tweet comes up, it will be stored on a HANA Server.</p> 
             <p>&nbsp;</p> 
             <p><strong>To run it:</strong></p> 
             <ul> 
              <li>Download and Install <a class="jive-link-external-small" href="https://nodejs.org/download/">Node.JS</a></li> 
              <li>From terminal (cmd, shell, etc) download the script dependencies by running these 3 commands: 
               <ul> 
                <li>npm install util</li> 
                <li>npm install hdb</li> 
                <li>npm install twitter</li> 
               </ul> </li> 
              <li><a class="jive-link-external-small" href="https://apps.twitter.com/app/new">Register a new application on Twitter Developers</a> to have your own twitter keys</li> 
              <li>Open the node scripts (brands.js, sapb1.js and twitterSummit.js) and fill your twitter keys and HANA server information.</li> 
             </ul> 
             <p>&nbsp;</p> 
             <p><img src="http://i.imgur.com/tejWii7.png"></p> 
             <p><strong>&nbsp;</strong></p> 
             <p><strong>Run the Tweets Retriever scripts with the commands</strong></p> 
             <ul> 
              <li>node brands.js</li> 
              <li>node twitterSummit.js</li> 
              <li>node brands.js</li> 
             </ul> 
             <p>&nbsp;</p> 
             <p>And it should work like this:</p> 
             <p>&nbsp;</p> 
             <p><iframe width="625" height="469" src="https://www.youtube.com/embed/vzYfL6nMQ-Q?feature=oembed" frameborder="0" allowfullscreen></iframe></p> 
             <h2>Final Considerations</h2> 
             <p>&nbsp;</p> 
             <p>This app is, off course, for demonstration purposes. It is 100% open source and can be enhanced or modified accordingly. It was developed under a didactical perspective, in a matter that you could have a comprehensive experience understanding each step.</p> 
             <p>&nbsp;</p> 
             <p>One point to be emphasized it’s the amount of free third party resources that I used here to show you that, once working on HANA, we are free to work with a infinite of libraries and resources.<br> <strong>Examples:</strong></p> 
             <ul> 
              <li>The UI is 100% build with <a class="jive-link-external-small" href="http://getbootstrap.com/2.3.2/">Twitter Bootstrap</a></li> 
              <li>Dashboards are made with <a class="jive-link-external-small" href="http://morrisjs.github.io/morris.js/">Morris.JS</a></li> 
              <li>Maps are on <a class="jive-link-external-small" href="http://jvectormap.com/">jVectorMap</a></li> 
              <li><span style="font-size: 10pt">Rest calls with <a class="jive-link-external-small" href="http://jquery.com/">jQuery</a></span></li> 
              <li>And others…</li> 
             </ul> 
             <p>&nbsp;</p> 
             <p>SAP embraces and support (a lot) open source community. You can see several examples on <a class="jive-link-external-small" href="https://github.com/sap">SAP GitHub Repository</a>. Here we are using the HANA Node driver that is a example of it and I bet you will <a class="jive-link-external-small" href="http://www.linuxfoundation.org/news-media/announcements/2015/06/nodejs-foundation-advances-community-collaboration-announces-new">hear more and more about this in a near the future</a>.</p> 
             <p>&nbsp;</p> 
             <p>Let me know what you think and don’t forget to follow me on twitter! <a class="jive-link-external-small" href="https://twitter.com/Ralphive">Follow @Ralphive</a></p> 
             <p>&nbsp;</p> 
             <p>Cheers!</p> 
             <p>&nbsp;</p> 
             <p>Ralph</p>
