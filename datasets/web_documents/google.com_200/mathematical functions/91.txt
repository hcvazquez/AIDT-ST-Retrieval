       <h1 itemprop="name" class="devsite-page-title"> Built-in Google Services </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p>Google Apps Script provides more than 30 built-in services for interacting with user data, other Google systems, and external systems. These services are provided as global objects akin to JavaScript's standard <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math"><code>Math</code></a> object. For example, just as <code>Math</code> offers methods like <code>random()</code> and constants like <code>PI</code>, Apps Script's <a href="https://developers.google.com/apps-script/reference/spreadsheet/">Spreadsheet service</a> offers methods like <a href="https://developers.google.com/apps-script/reference/spreadsheet/spreadsheet-app#openById(String)"><code>openById(id)</code></a>, classes (child objects) like <a href="https://developers.google.com/apps-script/reference/spreadsheet/range"><code>Range</code></a>, and enums like <a href="https://developers.google.com/apps-script/reference/spreadsheet/data-validation-criteria"><code>DataValidationCriteria</code></a>.</p> 
        <p>The reference documentation for services that control G Suite products are collected in the "G Suite Services" section under the "Reference" header in the sidebar of this site. Utility services (for things like creating user interfaces, parsing XML, or writing log data) are collected in the "Script Services" section.</p> 
        <h2 id="using_autocomplete">Using autocomplete</h2> 
        <p>The script editor provides a "content assist" feature, more commonly called "autocomplete," which reveals the global objects as well as methods and enums that are valid in the script's current context. To show autocomplete suggestions, select the menu item <strong>Edit &gt; Content assist</strong> or press <strong>Ctrl+Space</strong>. Autocomplete suggestions also appear automatically whenever you type a period after a global object, enum, or method call that returns an Apps Script class. For example:</p> 
        <ul> 
         <li>If you click on a blank line in the script editor and activate autocomplete, you will see a list of the global objects.</li> 
         <li>If you type the full name of a global object or select one from autocomplete, then type <code>.</code> (a period), you will see all methods and enums for that class.</li> 
         <li>If you type a few characters and activate autocomplete, you will see all valid suggestions that begin with those characters.</li> 
        </ul> 
        <h2 id="understanding_global_objects">Understanding global objects</h2> 
        <p>Each service provides at least one global (top-level) object; for example, the <a href="https://developers.google.com/apps-script/reference/gmail">Gmail service</a> is accessed solely from the <a href="https://developers.google.com/apps-script/reference/gmail/gmail-app"><code>GmailApp</code></a> object. Some services provide multiple global objects; for example, the <a href="https://developers.google.com/apps-script/reference/base">Base service</a> includes four global objects: <a href="https://developers.google.com/apps-script/reference/base/browser"><code>Browser</code></a>, <a href="https://developers.google.com/apps-script/reference/base/logger"><code>Logger</code></a>, <a href="https://developers.google.com/apps-script/reference/base/mime-type"><code>MimeType</code></a>, and <a href="https://developers.google.com/apps-script/reference/base/session"><code>Session</code></a>.</p> 
        <h2 id="calling_methods">Calling methods</h2> 
        <p>The global objects of nearly all built-in or <a href="https://developers.google.com/apps-script/guides/services/advanced">advanced services</a> include methods that return data or an Apps Script class. Scripts make method calls in this format:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>GlobalObjectName.methodName(argument1, argument2, ..., argumentN);
</code></pre> 
        <p>For example, a script can send an email by calling the <a href="https://developers.google.com/apps-script/reference/gmail/gmail-app#sendEmail(String,String,String)"><code>sendEmail(recipient, subject, body)</code></a> method of the Gmail service like so:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>GmailApp.sendEmail('claire@example.com', 'Subject line', 'This is the body.');
</code></pre> 
        <p>If a method returns another Apps Script class, you can chain method calls on one line. (Return types are shown both in autocomplete and in a method's reference documentation.) For example, the method <a href="https://developers.google.com/apps-script/reference/document/document-app#create(String)"><code>DocumentApp.create()</code></a> returns a <a href="https://developers.google.com/apps-script/reference/document/document"><code>Document</code></a>; thus, the following two sections of code are equivalent:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>var doc = DocumentApp.create('New document');
var body = doc.getBody();
body.appendParagraph('New paragraph.');

// Same result as above.
DocumentApp.create('New document').getBody().appendParagraph('New paragraph.');
</code></pre> 
        <h2 id="accessing_child_classes">Accessing child classes</h2> 
        <p>Every service includes one or more child classes that cannot be accessed from the top level as a global object can. You cannot use the <code>new</code> keyword to construct these classes, as you can with standard JavaScript classes like <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date"><code>Date</code></a>; you can only access a child class by calling a method that returns it. If you're not sure how to access a certain class, visit the root page for the service's reference documentation and look for a method that returns the class you want.</p> 
        <h2 id="dealing_with_interfaces">Dealing with interfaces</h2> 
        <p>A handful of services include special classes that are labeled as "interfaces" in the reference documentation. These are generic classes used as return types for methods that cannot determine the precise type in advance; for example, the <a href="https://developers.google.com/apps-script/reference/document/">Document service</a> method <a href="https://developers.google.com/apps-script/reference/document/body#getChild(Integer)"><code>Body.getChild(childIndex)</code></a> returns a generic <a href="https://developers.google.com/apps-script/reference/document/element"><code>Element</code></a> object. <code>Element</code> is an interface that represents some other class, possibly a <a href="https://developers.google.com/apps-script/reference/document/paragraph"><code>Paragraph</code></a> or <a href="https://developers.google.com/apps-script/reference/document/table"><code>Table</code></a>. Interface objects are rarely useful on their own; instead, you usually want to call a method like <a href="https://developers.google.com/apps-script/reference/document/element#asParagraph()"><code>Element.asParagraph()</code></a> to cast the object back to a precise class.</p> 
        <h2 id="working_with_enums">Working with enums</h2> 
        <p>Most services include a few enums (enumerated types) of named values. For example, the <a href="https://developers.google.com/apps-script/reference/drive/">Drive service</a> uses the enums <a href="https://developers.google.com/apps-script/reference/drive/access"><code>Access</code></a> and <a href="https://developers.google.com/apps-script/reference/drive/permission"><code>Permission</code></a> to determine which users have access to a file or folder. In almost all cases, you access these enums from the global object. For example, a call to the method <a href="https://developers.google.com/apps-script/reference/drive/folder#setSharing(Access,Permission)"><code>Folder.setSharing(accessType, permissionType)</code></a> looks like this:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>// Creates a folder that anyone on the Internet can read from and write to. (Domain administrators can
// prohibit this setting for G Suite users.)
var folder = DriveApp.createFolder('Shared Folder');
folder.setSharing(DriveApp.Access.ANYONE, DriveApp.Permission.EDIT);
</code></pre> 
        <h2 id="basic_javascript_features">Basic JavaScript features</h2> 
        <p>Apps Script is based on <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/New_in_JavaScript/1.6">JavaScript 1.6</a>, plus a few features from <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/New_in_JavaScript/1.7">1.7</a> and <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/New_in_JavaScript/1.8">1.8</a>. Many basic JavaScript features are thus available in addition to the built-in and <a href="https://developers.google.com/apps-script/guides/services/advanced">advanced Google services</a>: you can use common objects like <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array"><code>Array</code></a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date"><code>Date</code></a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp"><code>RegExp</code></a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference">and so forth</a>, as well as the <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math"><code>Math</code></a> and <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object"><code>Object</code></a> global objects. However, because Apps Script code runs on Google's servers (not client-side, except for <a href="https://developers.google.com/apps-script/guides/html/">HTML-service</a> pages), browser-based features like DOM manipulation or the <a href="https://developer.mozilla.org/en-US/docs/Web/API/Window"><code>Window</code></a> API are not available.</p>
