        <h2 class="text-center">Getting started with Optical Character Recognition (OCR) with Tesseract in Node.js</h2>

        <article id="article-content" itemscope itemtype="http://schema.org/BlogPosting">
         <p>Let's suppose that you need to digitize a page of a book or a printed document, you will use a scanner to create an image of the real page. However although you have the rights to edit the content of the scanned document, you can't edit it in your computer because it's an image, and you can't simply edit an image as if it were a digital document. Yeah, the user can use programs that creates PDF with selectable text and then they can do what they want, however as a developer, you can offer your user the possibility of extract the text from images using the Optical Character Recognition technology. To achieve our goal of converting images to text, we are going to use Tesseract written in C++ installing it in the system and then using the command line with the Node.js&nbsp;wrapper.</p> 
         <p>In this article you will learn how to extract the text from an image with the help of Tesseract using Javascript in Node.js.</p> 
         <h2><strong>1.</strong> Install Tesseract in your system</h2> 
         <p>In order to use the optical character recognition API, as mentioned in the article, we are going to use Tesseract. <a href="https://github.com/tesseract-ocr/tesseract" target="_blank" rel="nofollow">Tesseract</a>&nbsp;is an open source Optical Character Recognition (OCR) Engine, available under the Apache 2.0 license. It can be used directly using an API to extract typed, handwritten or printed text from images. It supports a wide variety of languages (that needs to be installed).&nbsp;Tesseract supports various output formats: plain-text, hocr(html) and pdf.</p> 
         <p>The installation process of Tesseract in your system will vary according to the Operative System that you use:</p> 
         <h3>Windows</h3> 
         <p>The installation of Tesseract in Windows is pretty simple, we recommend you <a href="https://github.com/UB-Mannheim/tesseract/wiki" target="_blank" rel="nofollow">to use the unnofficial installer mentioned in the wiki here (tesseract-ocr-setup-&lt;version&gt;.exe)</a>. You can get <a href="http://digi.bib.uni-mannheim.de/tesseract/" target="_blank" rel="nofollow">a list of all the available setups in the official website of tesseract here</a>&nbsp;(download always the most recent version).</p> 
         <p>The installation process is very straightforward, just follow the wizard. However we recommend you to install directly all the languages that you need for tesseract in the setup (only the ones you need, otherwise the download process will take long) and register tesseract in the PATH:</p> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-58691b8e58cab.jpg" alt="Tesseract Windows Installation Setup" class="img img-thumbnail img-responsive center-block"></p> 
         <p>Wait till the installation finishes and you're ready to go. You can test if it was correctly installed executing in a new command prompt window <code>tesseract -v</code> (that should output the installed version).</p> 
         <h3>Ubuntu</h3> 
         <p>Install Tesseract using the following command:</p> 
         <pre><code class="lang-batch">sudo apt-get install tesseract-ocr</code></pre> 
         <p>Then, install the languages that you need to recognize (e.g . -deu, -fra , -eng , -spa english required):</p> 
         <pre><code class="lang-batch">sudo apt-get install tesseract-ocr-eng</code></pre> 
         <p>Then tesseract should be available on any terminal and therefore accesible by our PHP scripts later.</p> 
         <h3>MacOS</h3> 
         <p>If you're using Mac OS X, you can install tesseract using either&nbsp;<a href="https://www.macports.org/" target="_blank" rel="nofollow">MacPorts</a><span> or </span><a href="http://brew.sh/" target="_blank" rel="nofollow">Homebrew</a>:</p> 
         <h4>MacPorts</h4> 
         <p>To install Tesseract run this command:</p> 
         <pre><code class="lang-batch">sudo port install tesseract</code></pre> 
         <p>To install any language data, execute:</p> 
         <pre><code class="lang-batch">sudo port install tesseract-&lt;langcode&gt;</code></pre> 
         <p>A complete list <span>of available langcodes can be found on </span><a href="https://www.macports.org/ports.php?by=name&amp;substr=tesseract-" target="_blank" rel="nofollow">MacPorts tesseract page</a><span>.</span></p> 
         <h4>Homebrew</h4> 
         <p>To install Tesseract run this command:</p> 
         <pre><code class="lang-batch">brew install tesseract</code></pre> 
         <p>In case you need more information or your operative system isn't listed, please refer to the <a href="https://github.com/tesseract-ocr/tesseract/wiki#installation" target="_blank" rel="nofollow">Installation wiki of the Tesseract repository in Github here</a>.</p> 
         <h2><strong>2.</strong> Install the Tesseract Node.js wrapper</h2> 
         <p>To handle Tesseract with Node.js, we are going to use the most known Wrapper of Tesseract&nbsp;written by <a href="https://github.com/desmondmorris" target="_blank" rel="nofollow">@desmondmorris</a>. The node-tesseract module is&nbsp;a very&nbsp;simple wrapper for the Tesseract OCR package for node.js, it requires Tesseract 3.01 or higher.</p> 
         <p>To install the <code>node-tesseract</code> module in your Node.js project execute the following command:</p> 
         <pre><code class="lang-batch">npm install node-tesseract</code></pre> 
         <p>Then you'll be able to require the module using <code>require('node-tesseract')</code>.</p> 
         <h2><strong>3.</strong> Processing image</h2> 
         <p>The usage of the wrapper in Node.js is pretty easy, it consists of a simple method named <code>process</code>. This method expects as first parameter the absolute or relative filepath of the image to process, as second parameter an object with the configuration (optional to use the default settings) and as third parameter the callback triggered when the command ends.</p> 
         <h3>Default initialization</h3> 
         <p>To process an image, require the node-tesseract module and call the process function with the 2 required arguments (filename and callback):</p> 
         <pre><code class="lang-js">var tesseract = require('node-tesseract');

tesseract.process('image.jpeg', (err, text) =&gt; {
    if(err){
        return console.log("An error occured: ", err);
    }

    console.log("Recognized text:");
    // the text variable contains the recognized text
    console.log(text);
});</code></pre> 
         <h3>Custom options</h3> 
         <p>All the properties in the options object are the same arguments that you use with tesseract via command line e.g the command&nbsp;<code>tesseract image.jpeg output_filename -l eng+deu -psm 6</code>&nbsp;has 4 arguments, however the first argument is set by your code and the second by the library, that means that the <code>-l</code> and <code>-psm</code> parameters need to be given in the options object:</p> 
         <pre><code class="lang-js">var tesseract = require('node-tesseract');

var options = {
    // Use the english and german languages
    l: 'eng+deu',
    // Use the segmentation mode #6 that assumes a single uniform block of text.
    psm: 6
};

tesseract.process('image.jpeg', options , (err, text) =&gt; {
    if(err){
        return console.log("An error occured: ", err);
    }

    console.log("Recognized text:");
    console.log(text);
});</code></pre> 
         <h2>Known issues</h2> 
         <p>When using different languages (and maybe huge files), you may catch the following exception at the execution of your script:</p> 
         <pre><code class="lang-js">Error :An error occured:  { Error: stderr maxBuffer exceeded
    at Socket.&lt;anonymous&gt; (child_process.js:278:14)
    at emitOne (events.js:96:13)
    at Socket.emit (events.js:188:7)
    at readableAddChunk (_stream_readable.js:176:18)
    at Socket.Readable.push (_stream_readable.js:134:10)
    at Pipe.onread (net.js:548:20)&lt;/p&gt;</code></pre> 
         <p>This error "stderr maxBuffer exceeded" is caused because the&nbsp;child_process.exec used to handle tesseract with node.js kills the process because the allowed amount of data of the standart output isn't enough (200KB as default value). To prevent this error, you need to use and set the <code>maxBuffer</code>&nbsp;property in&nbsp;the process options. node-tesseract allow you to achieve this by providing an object in the <code>env</code> property:</p> 
         <pre><code class="lang-js">var tesseract = require('node-tesseract');

var options = {
    // Use the english and german languages
    l: 'deu',
    // Use the segmentation mode #6 that assumes a single uniform block of text.
    psm: 6,
    // Increase the allowed amount of data in stdout to 16MB (A little exaggerated)
    env: {
        maxBuffer: 4096 * 4096
    }
};

tesseract.process('german.jpeg', options , (err, text) =&gt; {
    if(err){
        return console.log("An error occured: ", err);
    }

    console.log("Recognized text:");
    console.log(text);
});</code></pre> 
         <div class="note note-info note-left-striped"> 
          <p><strong>Note:</strong> according to your needs, increase the maxBuffer value.</p> 
         </div> 
         <p>Finally, we recommend you to&nbsp;read <a href="https://github.com/tesseract-ocr/tesseract/wiki/Command-Line-Usage" target="_blank" rel="nofollow">the documentation of Tesseract usage with the command line</a> to see more configuration options.</p> 
         <p>Happy coding <i class="fa fa-heart"></i>!</p>

        $(function () {
            var _search_api = {
                getSuggestions: function (searchTerm) {
                    return $.ajax({
                        url: '/ajax-endpoint/search',
                        data: {
                            q: searchTerm
                        },
                        type:"GET",
                        dataType: 'JSON'
                    });
                },
                process :  function(title,data,noCase){
                    var html = '<h4>'+title+' <small>('+data.length+')</small></h4>';

                    if(!data.length){
                        return html += '<p>'+noCase+'</p>';
                    }

                    html += '<ul class="list-material">';

                    for(var c = 0;c < data.length;c++){
                        var row = data[c];

                        html += '<li data-toggle="tooltip" data-placement="top" data-original-title="'+row.name+'">';
                        html +=     '<a href="'+row.path+'" class="visible" style="padding: 0px;">';
                        html +=         '<div class="list-content">';
                        html +=             '<span class="title">'+row.name+'</span>';
                        html +=             '<span class="caption">'+row.date+'</span>';
                        html +=         '</div>';
                        html +=     '</a>';
                        html += '</li>';
                    }

                    html += '</ul>';


                    return html;
                }
            };

            $('#search-input-ourcodeworld').bind("keyup change", OurCodeWorld.debounce(function () {
                var value = $(this).val();

                if (!value) {
                    return;
                }

                _search_api.getSuggestions(value).then(function (data) {
                    var articles = data.data;
                    var group =  {
                        left: articles.splice(0, Math.ceil(articles.length/2)),
                        right: articles
                    };
                    
                    $('#search-layout-zone-articles').html(_search_api.process("Articles", group.left ,"No articles were found"));
                    $('#search-layout-zone-news').html(_search_api.process("Articles", group.right ,"No articles were found"));
                }, function (err) {
                    console.log(err);
                });
            }, 300));

            OurCodeWorld.init();
            Layout.init();
        });
    </script>
