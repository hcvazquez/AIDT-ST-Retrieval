      <div class="article__content content column"> 
       <p>There is a time for each project when internationalization becomes absolutely needed. Sometimes it's because of regional specific customers or when the application has to be shown to people in many countries. Usually, when its architecture is not ready for that- it starts becoming a really painful process.</p> 
       <p>In the AngularJS world, there's the awesome <a href="https://github.com/angular-translate/angular-translate">angular-translate</a> (it's used for handling language translation stuff) and the <a href="https://github.com/lgalfaso/angular-dynamic-locale">angular-dynamic-locale</a> (it's used for changing angular $locale- which means formatting dates, numbers, currencies, etc.) libraries.</p> 
       <p>On the other hand we have the really popular <a href="https://github.com/yeoman/generator-angular">Yeoman AngularJS generator</a>, which is widely used for scaffolding AngularJS applications.</p> 
       <p>Let's provide asynchronous translate (without page reload) to Yeoman's AngularJS application.</p> 
       <p><strong>Be sure to checkout the <a href="http://blog.gospodarets.com/angular-translate-yeoman/dist/#/">demo</a></strong>.</p> 
       <div class="text-center"> 
        <a href="http://blog.gospodarets.com/angular-translate-yeoman/dist/#/"><img src="https://cask.scotch.io/2015/02/i18n-angular-applications-864x500.jpeg" alt="i18n-angular-applications" width="864" height="500" class="aligncenter size-large wp-image-7356 img-responsive"></a> 
       </div> 
       <h2>Creating an AngularJS App with Yeoman</h2> 
       <p>In <a href="https://github.com/yeoman/generator-angular">Github</a>, you can find detailed instructions on how to create applications using <code>generator-angular</code>.</p> 
       <p>In our case just run:</p> 
       <pre><code class="language-javascript">npm install -g generator-angular

mkdir angular-translate-yeoman &amp;&amp; cd $_

yo angular translate
</code></pre> 
       <p>Answer "Yes" to all questions regarding the decision to use Sass and Sass Bootstrap versions. Leave the default list of AngularJS modules:</p> 
       <div class="text-center"> 
        <img src="https://cask.scotch.io/2015/03/yeoman-angular-install.png" alt="yeoman-angular-install" width="1112" height="238" class="aligncenter size-full wp-image-7360 img-responsive"> 
       </div> 
       <p>After a while you will have the Yeoman-based Angular application with <a href="https://github.com/malyw/angular-translate-yeoman/tree/yeoman-angular-clean">some pre-generated file structure</a> and pre-defined controllers, views, and even some tests.</p> 
       <p>To run it in you browser with the live-reload, on any changes, just run:</p> 
       <pre><code class="language-javascript">grunt serve
</code></pre> 
       <p>It might look similar to <a href="https://malyw.github.io/angular-translate-yeoman/dist-yeoman-angular-clean/">that</a>. </p> 
       <h2>Adding i18n</h2> 
       <ol> 
        <li>We have to download libraries and include them in the project.</li> 
        <li>Then we'll need to provide translation files and set them working.</li> 
        <li>Next, we will change angular $locale after the language change.</li> 
        <li>Once step 4 is complete, we need to have the drop-down with the list of languages to select from.</li> 
        <li>Lastly, the selected language should be stored and applied after the page reloads. </li> 
       </ol> 
       <h2>Adding Bower Dependencies</h2> 
       <p>Run in the shell:</p> 
       <pre><code class="language-javascript">### ANGULAR-TRANSLATE STUFF
# adds angular-translate library
bower install --save angular-translate

# util for asynchronous loading translations files
bower install --save angular-translate-loader-static-files
# util to save selected language preferences to localStorage
bower install --save angular-translate-storage-local
# util to track missed IDs in translation files
bower install --save angular-translate-handler-log

# ANGULAR-DYNAMIC-LOCALE STUFF
# adds angular-dynamic-locale library
bower install --save angular-dynamic-locale

# list of predefined settings for each locale
bower install --save angular-i18n
</code></pre> 
       <h2>Applying translations in the templates</h2> 
       <p>Let's add a translation for <code>Splendid!</code> text on the green button. We can do it by applying to this text translate filter in <code>app/views/main.html</code>:</p> 
       <pre><code class="language-markup">{{"views.main.Splendid!" | translate}}
</code></pre> 
       <p>I also provided a prefix for translations to quickly figure out where they are used:</p> 
       <p><code>views.main.Splendid!</code> -&gt; <code>views</code> folder, file <code>main.html</code></p> 
       <p>Using this, in this way, you can provide translations for all your templates. </p> 
       <h2>Providing Translations Files</h2> 
       <p>Create files <code>app/resources/locale-{{locale}}.json</code> where <code>{{locale}}</code> - needed locales to handle. In the current example I'll provide the following locales:</p> 
       <p><code>app/resources/locale-en_US.json</code> for English (United States):</p> 
       <pre><code class="language-javascript">{
    "views.main.Splendid!": "Splendid!",
    "directives.language-select.Language": "Language"
}
</code></pre> 
       <p><code>app/resources/locale-ru_RU.json</code> for Russian (Russia):</p> 
       <pre><code class="language-javascript">{
    "views.main.Splendid!": "???????!",
    "directives.language-select.Language": "????"
}
</code></pre> 
       <p><code>directives.language-select.Language</code> will be used in languages dropdown. </p> 
       <h3>Including angular-translate and dynamic loading in AngularJS applications</h3> 
       <p>In <code>appscriptsapp.js</code> file:</p> 
       <p>Add dependencies for the main app:</p> 
       <pre><code class="language-javascript">angular.module('translateApp', [
 ...
 'pascalprecht.translate',// angular-translate
 'tmh.dynamicLocale'// angular-dynamic-locale
])
</code></pre> 
       <p>Provide info about locales and the preferred locale which are used in your app (key values will be used in languages dropdown):</p> 
       <pre><code class="language-javascript">.constant('LOCALES', {
    'locales': {
        'ru_RU': '???????',
        'en_US': 'English'
    },
    'preferredLocale': 'en_US'
})
</code></pre> 
       <p>To get warnings in the developer console, regarding forgotten IDs in translations, just add:</p> 
       <pre><code class="language-javascript">.config(function ($translateProvider) {
    $translateProvider.useMissingTranslationHandlerLog();
})
</code></pre> 
       <p>Next step is about adding asynchronous loading for the translations:</p> 
       <pre><code class="language-javascript">.config(function ($translateProvider) {
    $translateProvider.useStaticFilesLoader({
        prefix: 'resources/locale-',// path to translations files
        suffix: '.json'// suffix, currently- extension of the translations
    });
    $translateProvider.preferredLanguage('en_US');// is applied on first load
    $translateProvider.useLocalStorage();// saves selected language to localStorage
})
</code></pre> 
       <p>And, finally, provide the config with direction of where to load the $locale settings files for <code>angular-dynamic-locale</code>:</p> 
       <pre><code class="language-javascript">.config(function (tmhDynamicLocaleProvider) {
    tmhDynamicLocaleProvider.localeLocationPattern('bower_components/angular-i18n/angular-locale_{{locale}}.js');
})
</code></pre> 
       <h2>AngularJS service for getting / setting current locale</h2> 
       <p>We need to have some Services to change language and apply some additional logic (e.g. change AngularJS $locale). This will be used for creation and interaction with the languages drop down later.</p> 
       <p>Create <code>app/scripts/services/LocaleService.js</code> file with the following content:</p> 
       <pre><code class="language-javascript">angular.module('translateApp') .service('LocaleService', function ($translate, LOCALES, $rootScope, tmhDynamicLocale) {
    'use strict';
    // PREPARING LOCALES INFO
    var localesObj = LOCALES.locales;

    // locales and locales display names
    var _LOCALES = Object.keys(localesObj);
    if (!_LOCALES || _LOCALES.length === 0) {
      console.error('There are no _LOCALES provided');
    }
    var _LOCALES_DISPLAY_NAMES = [];
    _LOCALES.forEach(function (locale) {
      _LOCALES_DISPLAY_NAMES.push(localesObj[locale]);
    });

    // STORING CURRENT LOCALE
    var currentLocale = $translate.proposedLanguage();// because of async loading

    // METHODS
    var checkLocaleIsValid = function (locale) {
      return _LOCALES.indexOf(locale) !== -1;
    };

    var setLocale = function (locale) {
      if (!checkLocaleIsValid(locale)) {
        console.error('Locale name "' + locale + '" is invalid');
        return;
      }
      currentLocale = locale;// updating current locale

      // asking angular-translate to load and apply proper translations
      $translate.use(locale);
    };

    // EVENTS
    // on successful applying translations by angular-translate
    $rootScope.$on('$translateChangeSuccess', function (event, data) {
      document.documentElement.setAttribute('lang', data.language);// sets "lang" attribute to html

       // asking angular-dynamic-locale to load and apply proper AngularJS $locale setting
      tmhDynamicLocale.set(data.language.toLowerCase().replace(/_/g, '-'));
    });

    return {
      getLocaleDisplayName: function () {
        return localesObj[currentLocale];
      },
      setLocaleByDisplayName: function (localeDisplayName) {
        setLocale(
          _LOCALES[
            _LOCALES_DISPLAY_NAMES.indexOf(localeDisplayName)// get locale index
            ]
        );
      },
      getLocalesDisplayNames: function () {
        return _LOCALES_DISPLAY_NAMES;
      }
    };
});
</code></pre> 
       <h2>Language Dropdown</h2> 
       <p>In this part we will add the language drop down and set actions to changing the language in it. This select element has to be shown only when there are more than 1 languages provided.</p> 
       <p>Let's create an AngularJS directive <code>app/scripts/directives/LanguageSelectDirective.js</code>:</p> 
       <pre><code class="language-javascript">angular.module('translateApp') .directive('ngTranslateLanguageSelect', function (LocaleService) { 'use strict';

        return {
            restrict: 'A',
            replace: true,
            template: ''+
            '&lt;div class="language-select" ng-if="visible"&gt;'+
                '&lt;label&gt;'+
                    '{{"directives.language-select.Language" | translate}}:'+
                    '&lt;select ng-model="currentLocaleDisplayName"'+
                        'ng-options="localesDisplayName for localesDisplayName in localesDisplayNames"'+
                        'ng-change="changeLanguage(currentLocaleDisplayName)"&gt;'+
                    '&lt;/select&gt;'+
                '&lt;/label&gt;'+
            '&lt;/div&gt;'+
            '',
            controller: function ($scope) {
                $scope.currentLocaleDisplayName = LocaleService.getLocaleDisplayName();
                $scope.localesDisplayNames = LocaleService.getLocalesDisplayNames();
                $scope.visible = $scope.localesDisplayNames &amp;&amp;
                $scope.localesDisplayNames.length &gt; 1;

                $scope.changeLanguage = function (locale) {
                    LocaleService.setLocaleByDisplayName(locale);
                };
            }
        };
    });
</code></pre> 
       <h2>Including locale service and language select</h2> 
       <p>And don't forget to include these scripts and language selects in the <code>app/index.html</code>:</p> 
       <pre><code class="language-markup">...
&lt;div ng-translate-language-select&gt;&lt;/div&gt;
...

&lt;!-- build:js({.tmp,app}) scripts/scripts.js --&gt;
...

&lt;script src="scripts/directives/LanguageSelectDirective.js"&gt;&lt;/script&gt;
&lt;!-- endbuild --&gt;
</code></pre>

       <h2>Tips and tricks</h2> 
       <p>In this section we will review some examples of providing language-specific content.</p> 
       <h3>Templates</h3> 
       <h4>Text</h4> 
       <p>As above, we can apply translations for templates using e.g. translate filter:</p> 
       <pre><code class="language-markup">{{"views.main.Splendid!" | translate}}
</code></pre> 
       <p>There are also a <a href="https://github.com/angular-translate/angular-translate/wiki/Getting-Started">couple of other techniques</a>.</p> 
       <h4>Title and attributes</h4> 
       <p>To apply a changing page title and a <code>meta[name="description"]</code> attribute (which is used to provide text for sharing in social networks), you can use angular <code>ng-bind</code> and <code>ng-attr-content</code> directives (see how it's done in the <a href="http://blog.gospodarets.com/angular-translate-yeoman/dist/">demo app</a>):</p> 
       <pre><code class="language-markup">&lt;title ng-bind="pageTitle"&gt;i18n for your AngularJS applications&lt;/title&gt;
&lt;meta name="description" ng-attr-content="{{pageContent}}" content="How to translate your AngularJS applications without page reload with angular-translate"&gt;
</code></pre> 
       <p>and to provide an update to these fields in controller:</p> 
       <pre><code class="language-javascript">$translate(pageTitleTranslationId, pageContentTranslationId)// ON INIT
 .then(function (translatedPageTitle, translatedPageContent) {
  $rootScope.pageTitle = translatedPageTitle;
  $rootScope.pageContent = translatedPageContent;
});

$rootScope.$on('$translateChangeSuccess', function (event, data) {// ON LANGUAGE CHANGED
 $rootScope.pageTitle = $translate.instant(pageTitleTranslationId);
 $rootScope.pageContent = $translate.instant(pageContentTranslationId);
});
</code></pre> 
       <h2>Images</h2> 
       <p>You can replace your images when you switch the language providing <code>{{locale}}</code> part to <code>ng-src</code> attribute in your views:</p> 
       <pre><code class="language-markup">&lt;img ng-src="images/no-filerev/yeoman-{{locale}}.png" /&gt;
</code></pre> 
       <p>And in the controller:</p> 
       <pre><code class="language-javascript">$scope.locale = $translate.use();// ON INIT

$rootScope.$on('$translateChangeSuccess', function (event, data) {// ON LANGUAGE CHANGED
 $scope.locale = data.language;
});
</code></pre> 
       <p>You, also, can check it out on the <a href="http://blog.gospodarets.com/angular-translate-yeoman/dist/#/">Home page</a> (Yeoman image).</p> 
       <h2>CSS</h2> 
       <p>If you want to apply some specific css, depending on current locale, you can do it, because in our example we provided the <code>lang</code> attribute for the <code>&lt;html/&gt;</code> tag with current locale value. E.g.:</p> 
       <pre><code class="language-css">[lang="ru_RU"]{
    /*some special styles*/
}
</code></pre> 
       <p>You can see that by switching the language on the <a href="http://blog.gospodarets.com/angular-translate-yeoman/dist/#/about">About page</a> - it will, then, change the page layout. </p>
