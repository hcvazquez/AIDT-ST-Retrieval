             <div class="mosimage" style=" float: left;" align="center">
              <img src="http://oracle.anilpassi.com/images/stories/OAF/preface.gif" width="245" height="74" hspace="6" alt="Capthca in Java/Oracle" title="Capthca in Java/Oracle" border="0">
              <div class="mosimage_caption">
               Capthca in Java/Oracle
              </div>
             </div>In this article, I am explaining a step by step approach to implement a Captcha Web Service from Oracle's Java based OA Framework. In this OA Framework tutorial, we will display a captcha image within a OA Framework page. Also, the user will be able to enter the response to Captcha Image. The response will be validated against the Captcha Webservice.<br> <br><span style="font-weight: bold">Will I be using BPEL for this exercise?</span><br>Not really, however you can implement something similar in BPEL too.<br>I prefer using BPEL when there are multiple Web Services involved. <br><br><br><span style="font-weight: bold">Why use a Webservice, when you can get Open Source software like jCaptcha</span>.<br>This is your personal choice. Both options are equally feasible and usefull depending upon your need.<br>However in my case, I preferred Captcha Webservice for below reasons:-<br>1. All functionality enhancements are available to you by Webservice Provider.<br>2. No maintenance on our system, to do bug-fixes[if any] to captcha generation logic.<br><br><br><span style="font-weight: bold">Which Webservice will I be using?</span><br>I will be using free WebService from Textdisguise.com<br>Their WSDL is&nbsp; http: //www.textdisguise.com/TextDisguise/CaptchaService/CaptchaService.asmx?WSDL<br>You must register on their website, and they will email you username and password.<br>I quite liked Web Service from Capchas.net too, however you can decide to use whatever suits your need.<span style="font-weight: bold"><br><br><br>Before we go into details, what are the steps in brief?</span><br> Firstly, you may like to read this design approach <a href="integrate-oa-framework-with-web-service.html" target="blank_" title="on link Web Service Design approach in OA Framework">on link Web Service Design approach in OA Framework</a>. <br><span style="font-weight: bold">1.</span> Generate the stubs[java api for remote WebService] using WSDL URL. <br>This will generate some java files which internally make calls to captcha Web Service<br><span style="font-weight: bold">2.</span> Create a field of type image, into which Captcha Image will be displayed.<br><span style="font-weight: bold">3.</span> In controller class processRequest[executed when page renders], we make a call to Webservice API [ stubs generated in Step-1].<br>The webservice will return the URL of the image on remote machine<br><span style="font-weight: bold">4.</span> Assign the URL to "image field" defined in step 2<br><span style="font-weight: bold">5.</span> When user submits the page, call webservice again to validate users response.<br><br><br>In jDeveloper, right click on project to create a Webservice Stub from your WSDL.<br> <img style="height: 209px; width: 401px" src="http://docs.google.com/File?id=dkg3r3j_144fvdzdw" alt=""><br> <br> <br> <br> <br> Enter the URL of Webservice [or call it a WSDL as they say].I am creating these in package named xxfreecaptcha01.<br>However, you can create these classes in any package as you wish.<br> <img style="height: 309px; width: 475px" src="http://docs.google.com/File?id=dkg3r3j_145f49t6x" alt=""><br> <br><br> <br> <br>The IDE of jDeveloper will create the classes based on definition of WebService in WSDL Document. <br>All complexities related to SOAP etc are taken care of by these Stubs.<br>For us, usage of Captcha is like calling any ordinary java Program.<br><span style="font-weight: bold;
