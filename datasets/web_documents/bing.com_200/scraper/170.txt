         <div id="windows-installer">
          <h3>Why Windows Installer?</h3>
          <ul>
           <li>Best Windows integration</li>
           <li>Zero-overhead installation packages</li>
           <li>100% native code</li>
          </ul>
          <p>Using Windows Installer gives you the best possible integration of your application in the Windows Platform. Windows Installer is the standard Windows install technology and it is required by Microsoft for any application intending to meet the Windows logo certification.</p>
          <p>Because the Windows Installer engine is included in the latest Microsoft operating systems, distributed with the latest products (like Office) and also freely downloadable on the Internet, you don't have to include it in your installation package (unlike a proprietary install engine).</p>
          <p>Advanced Installer and the packages it creates contain 100% Windows native code and technologies. This ensures the smallest footprint and the tightest platform integration for your install packages.</p>
         </div>
         <h3>What can Windows Installer do for your Java programs</h3>
         <ul>
          <li>Install/uninstall files, registry keys, environment variables</li>
          <li>Create shortcuts in the Start menu or on the desktop</li>
          <li>File associations</li>
          <li><a href="jre-install.html" title="JRE Installation: Advanced Installer can create an installer for applications requiring the Java Runtime Environment">Bundle, or download and install a JRE</a></li>
          <li>Unattended installs</li>
          <li>Operating System version check</li>
          <li>Digitally signing your install packages</li>
         </ul>
         <p>Read more <a href="features.html" title="Software Installation Tool Features">Advanced Installer Features</a>. </p>
         <div id="java-native-launcher">
          <h3>Native Java launcher</h3>
          <ul>
           <li>Your own process name, file icon and version</li>
           <li>Instant-on splash screen</li>
           <li>JRE auto detection and version check</li>
           <li>Runs as a GUI/console application or a Win32 service</li>
          </ul>
          <p>Advanced Installer provides a <b>native launcher for your Java applications</b>. Using this launcher your users will see and use your Java programs just like regular Windows native executables.</p>
          <p>The native launcher provides a fast and friendly start up process for your <i>Java applications</i>. It will start by instantly displaying a splash screen, after which it will look for and detect a compatible JRE. Finally, it will load the JRE and launch your application. Any unexpected error will be presented to the user in a friendly, easy to understand manner, together with a remedial suggestion.</p>
         </div>
         <div id="mac-os-x">
          <h3>Mac OS X package</h3>
          <ul>
           <li>Build standard, native bundles to install your Java programs on Mac OS X</li>
          </ul>
          <p>Advanced Installer has the ability to create <b>standalone Java applications for installing on Mac OS X</b>. They respect the industry guidelines, supporting all the platform specific features - from folder layout to document types file associations and more. The packages are strictly native, without the overhead of Java installers.</p>
          <p>The cross-platform installation elements are included, while the Windows-specific ones are ignored. The <i>Mac OS X build</i> is fully customizable with its own file layout, icons, descriptions and more.</p>
          <p class="alert alert-important"><span class="icon-placeholder" title="Important"><img width="36" height="36" src="img/all/ico/important.png" alt="Important"></span>Includes Java 7 applications support for deployment on Mac OS X.</p>
         </div>
         <h3>Windows Installer vs. Cross Platform installers</h3>
         <p>The main advantage of a cross platform installer is its ability to generate a single install package that can run on various platforms. However this ability comes often at the expense of the <i>seamless Operating System integration</i> that a native installer offers.</p>
         <p>Cross platform installers come with their own proprietary, non-standard installation engines. This engine must be included in the installation, increasing its size sometimes by as much as 1 Megabyte. <b>Windows Installer</b> is a standard, well documented technology that already comes bundled with the latest versions of Windows and most popular applications.</p>
         <p>Most cross platform installers are themselves written in Java. Because of that, they require that a JRE be installed before the installer is run. Otherwise, incomprehensible error messages are displayed. Furthermore, they sport a non-standard look and feel, with a sluggish response time.</p>
