      <div class="entry-content"> 
       <p>Today <a href="http://www.hanselman.com/blog/">Scott Hanselman announced</a> the Release Candidate of Visual Studio 2015, Team Foundation Server 2015 and Visual Studio 2013 Update 5. We hope that all //build attendees are enjoying the show, and we are following up on the news of the Visual Studio 2015 Release Candidate to show you some of the cool new features delivered in this version for web developers. With the Visual Studio 2015 release candidate we are delivering updates for ASP.NET 4.6 and our next generation ASP.NET 5 framework.&nbsp; You can read more about ASP.NET 5 online at <a href="http://www.asp.net/vnext">http://www.asp.net/vnext</a> and review the new open source documentation for ASP.NET 5 at <a href="http://docs.asp.net">http://docs.asp.net</a> Let’s take a look at the new features delivered in the Visual Studio 2015 release candidate for ASP.NET 4.6 and the ASP.NET 5 beta4 preview:</p> 
       <h2>The Big Rename</h2> 
       <p>If you’ve been following the previous releases of Visual Studio 2015 and the nightly builds of ASP.NET, you have no doubt been familiar with the Project K executables: k.exe, klr.exe, kpm.exe, and kvm.exe&nbsp; In order to drop the Project K codename and move to a more market-friendly name of the ASP.NET tooling, these command-line tools and the K-Runtime were renamed as follows:</p> 
       <ul> 
        <li>k and klr are now DNX. DNX stands for .NET Execution Environment.</li> 
        <li>kvm is now dnvm or the .NET Version Manager</li> 
        <li>kpm is migrating some functionality of its functionality to dnu, the .NET Development Utility and some functionality will migrate to be core NuGet functionality</li> 
        <li>The Aspnet50 and aspnetcore50 framework monikers in project.json files are being replaced with dnx451 and dnxcore50.</li> 
       </ul> 
       <div>
        The ASP.NET 5 Community Standup has a 
        <a href="https://www.youtube.com/watch?v=-qt0POsiAF8" target="_blank">video discussing the rename decision</a> in depth.&nbsp; That video is from some time ago that refers to nightly builds or “daily bits”, those changes have been completed and are in this release.
       </div> 
       <h2>New Project Templates and Project Types</h2> 
       <p>In the prior versions of Visual Studio 2015, we made project types available for Project K class library and Project K console applications.&nbsp; These project type names don’t make sense in our new world and we have renamed them appropriately:</p> 
       <p style="text-align: center;"><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/7652.projectTypes.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/7652.projectTypes.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/7652.projectTypes.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/7652.projectTypes.png" alt="" border="0"></a></p> 
       <p>Class Library (Package) is a class library project that is configured to allow you to build reference libraries that compile directly into NuGet packages.&nbsp; Console application is a similar project that allows console applications to be built in the same model as an ASP.NET 5 web application, referencing NuGet packages and configuring multiple frameworks to build against.&nbsp; Both of these project types allow the developer to configure everything with a project.json file and target the .NET Core framework.</p> 
       <p>We’ve also simplified and updated the ASP.NET web project templates in this release. There are clear dividing lines between the ASP.NET 4.6 and ASP.NET 5 projects, as well as updated templates for a Single Page Application and Azure Mobile Service:</p> 
       <p style="text-align: center;"><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/5025.templates.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/5025.templates.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/5025.templates.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/5025.templates.png" alt="" border="0"></a></p> 
       <p>The ASP.NET 5 templates are updated as well, with additional security and JavaScript features configured to make them an easier starting point for building applications.&nbsp; You can learn more about how to start a new ASP.NET 5 project online at: <a href="http://www.asp.net/vnext/overview/aspnet-vnext/aspnet-5-overview">http://www.asp.net/vnext/overview/aspnet-vnext/aspnet-5-overview</a></p> 
       <h2>Add Missing NuGet Packages in ASP.NET 5</h2> 
       <p>Our favorite new feature for ASP.NET 5 developers is the cool new feature to help you add the correct NuGet packages to your project for the types that you are referencing. In the ASP.NET 5 environment, all references are NuGet packages, and how will you know which ones to add when you just want to light-up that XDocument type to work with an XML document?&nbsp; In an ASP.NET 5 web project or class library project, you can just press CTRL + . (period) to activate a helper in the editor that will offer to add the missing package to your project:</p> 
       <p style="text-align: center;"><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/0003.addMissingPackage.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/0003.addMissingPackage.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/0003.addMissingPackage.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/0003.addMissingPackage.png" alt="" border="0"></a></p> 
       <p>The editor will suggest the package to add and show you the changes that it is going to make to the Using statements at the top of the code file in order to allow your code to function properly.</p> 
       <h2>Performance Updates for ASP.NET 5</h2> 
       <p>We focused a significant amount of time on improving the performance of the Visual Studio developer experience. In this release, you should see a noticeable speed increase when opening existing solutions with ASP.NET 5 projects as well as launching the application. There are a number of other less important performance tweaks under the hood for ASP.NET 5 projects as well.</p> 
       <h2>Editor Improvements to Support TagHelpers</h2> 
       <div> 
        <div> 
         <p style="text-align: center;"><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/5123.tagHelper.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/5123.tagHelper.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/5123.tagHelper.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/5123.tagHelper.png" alt="" border="0"></a></p> 
         <p>TagHelpers for ASP.NET MVC were introduced in an earlier preview, but in this release they really come into their own.&nbsp; TagHelpers provide a simple tag experience in the razor editor that allow you to execute code on the server and inject content in the output.&nbsp; <a href="http://www.hanselman.com/blog/ASPNET5VNextWorkInProgressExploringTagHelpers.aspx">Scott Hanselman</a> and <a href="http://www.jeffreyfritz.com/2014/11/asp-net-tag-helpers-the-basics/">Jeff Fritz</a> have posts that introduce and demonstrate how to start writing your first TagHelper.&nbsp; The ASP.NET Community standup <a href="https://www.youtube.com/watch?v=Tv9pv1FYP1A">discussed TagHelpers in March 2015</a>.</p> 
         <p>We have added editor support so that you can easily see where your .NET code will take over rendering your HTML.&nbsp; With bold tag names and intellisense support for attributes, coding HTML with these complex server-side interactions has become easier than ever.</p> 
         <h2>Add Reference to Project</h2> 
        </div> 
        <div style="text-align: center;">
         <a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/2185.addRefrenceToProject.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/2185.addRefrenceToProject.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/2185.addRefrenceToProject.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/2185.addRefrenceToProject.png" alt="" border="0"></a>
        </div> 
        <div style="text-align: left;"> 
         <p>We’ve re-introduced the ability to add references to an ASP.NET 5 project from standard class library projects. In the image above, the ClassLibrary1 project is a standard class library, while the TagHelperSample.Helpers project is an ASP.NET 5 specific class library. Both library types are supported interchangeably now.</p> 
         <div> 
          <h2>Cleaned Up Solution Explorer with Automatic File Nesting</h2> 
          <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/8054.autoFileNesting.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/8054.autoFileNesting.png"><img style="float: right;" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/8054.autoFileNesting.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/8054.autoFileNesting.png" alt="" width="174" height="150"></a>With ASP.NET 5, we’ve promoted the fact that ASP.NET developers should start to use more standard web practices for acquiring, minifying, and otherwise processing your static resources before they are served to your web application’s visitors.</p> 
          <p>To help make the display of minified resources easier to review in your solution explorer, any files with an extension of min.css or min.js will be nested automatically under the un-processed version of that file. We hope that you find this to be a useful addition to the solution explorer.</p> 
         </div> 
         <h2>Local IIS Express Custom Configuration</h2> 
         <p>If you were finding that you were constantly updating your applicationHost.config file on your machine each time you started a new project, then this feature will be a welcome change. With the new ASP.NET tooling, your custom IIS web server configurations are stored in the solution’s .vs folder in an applicationHost.config file. This should help isolate your web server configurations for each of your projects without conflicting references to a single configuration file.</p> 
         <h2>Security Updates for ASP.NET</h2> 
         <p>There are a number of security-focused updates in this release for both ASP.NET 4.6 and ASP.NET 5:</p> 
         <ul> 
          <li>ASP.NET 4.6 
           <ul> 
            <li>Updates to templates using “Work And School Account” for Authentication. This also includes fixes to Domain picker used to select your tenant.</li> 
            <li>Use Roslyn Code Dom Compilers. The packages are pre-installed in Web Forms Template. They will be added to all 4.6 templates by RTM. You can link to your post about VB or <a href="http://blogs.msdn.com/b/webdev/archive/2014/05/12/enabling-the-net-compiler-platform-roslyn-in-asp-net-applications.aspx">http://blogs.msdn.com/b/webdev/archive/2014/05/12/enabling-the-net-compiler-platform-roslyn-in-asp-net-applications.aspx</a> ?</li> 
           </ul> </li> 
          <li>ASP.NET 5 
           <ul>

       <h2>Custom Launch Settings are saved as Project Properties</h2> 
       <p>If you want to lock in the version of the framework that your project for compilation and debugging, those settings are now saved with your project in the new launchSettings.json file in your project properties folder. This means that all of your team members will have the same debugging configuration when they synchronize source code.</p> 
       <p>Additionally, we have simplified the format of this configuration file to make it easier to read and work with:</p> 
       <p style="text-align: center;"><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/2543.launchSEttings.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/2543.launchSEttings.png"><img src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/2543.launchSEttings.png" original-url="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-00-63-56/2543.launchSEttings.png" alt="" border="0"></a></p>

       <h2>ASP.NET 4.6 Improvements</h2> 
       <p>We have a number of updates for ASP.NET 4.6 which includes Web Forms, MVC 5 and Web API 2. These include support for .NET Compiler Platform (Roslyn), hosting changes in IIS to support HTTP/2 and more. Read this <a href="http://blogs.msdn.com/b/webdev/archive/2015/04/30/updates-for-asp-net-4-6-web-forms-mvc-5-web-api-2.aspx">post</a> for more details.</p> 
       <h2>Summary</h2> 
       <p>We are working on evolving the ASP.NET environment and tooling. This might be a release candidate build for Visual Studio, but we are looking at the big picture for ASP.NET. We will continue to develop at the same pace in the open at <a href="http://github.com/aspnet/home">http://github.com/aspnet/home</a>. We are building and releasing our libraries and updates for Visual Studio with the updates delivered on <a href="http://www.nuget.org/">NuGet</a> and the <a href="https://visualstudiogallery.msdn.microsoft.com/">Visual Studio Extension Gallery</a>. Keep an eye on our blog to hear about our next set of features.</p>
