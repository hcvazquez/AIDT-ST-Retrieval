           <h3 name="2b39" id="2b39" class="graf graf--h3 graf-after--figure">Introduction</h3>
           <p name="452d" id="452d" class="graf graf--p graf-after--h3">Lately I’ve been exploring <a href="https://oauth.net/2/" data-href="https://oauth.net/2/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">OAuth</a>, but how to handle functional testing? Here I’ll demonstrate an approach for testing an application that uses OAuth via the popular <a href="http://passportjs.org/" data-href="http://passportjs.org/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">passport middleware</a> for express. When we’re done, we can test routes that require authentication without making any HTTPS calls to the OAuth provider.</p>
           <p name="e2bf" id="e2bf" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Note:</strong> I’m using <a href="http://koajs.com/" data-href="http://koajs.com/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Koa</a> (and thus the <a href="https://github.com/rkusa/koa-passport" data-href="https://github.com/rkusa/koa-passport" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">koa-passport</a> wrapper module), but this approach will work with <a href="https://expressjs.com/" data-href="https://expressjs.com/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Express</a> with modification.</p>
           <p name="4443" id="4443" class="graf graf--p graf-after--p">The relevant files are shown below:</p>
           <pre name="44da" id="44da" class="graf graf--pre graf-after--p">app.js<br>index.js<br>mockProfile.js<br>package.json</pre>
           <pre name="4e70" id="4e70" class="graf graf--pre graf-after--pre">node_modules/:<br>…</pre>
           <pre name="56f7" id="56f7" class="graf graf--pre graf-after--pre">test/:<br>index.js</pre>
           <pre name="7151" id="7151" class="graf graf--pre graf-after--pre">util/:<br>auth.js<br>mock-strategy.js</pre>
           <p name="245d" id="245d" class="graf graf--p graf-after--pre">Each file serves a particular purpose:</p>
           <ul class="postList">
            <li name="2adc" id="2adc" class="graf graf--li graf-after--p"><code class="markup--code markup--li-code">app.js</code>: Wrapper to bootstrap application</li>
            <li name="460f" id="460f" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">index.js</code>: The application</li>
            <li name="c4dd" id="c4dd" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">mockProfile.js</code>: My Github user profile data (substitute your own profile data here)</li>
            <li name="5c71" id="5c71" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">package.json</code>: Our dependencies and scripts</li>
            <li name="7f87" id="7f87" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">node_modules/</code>: third-party modules</li>
            <li name="3e67" id="3e67" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">test/index.js</code>: Our tests</li>
            <li name="9f12" id="9f12" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">util/auth.js</code>: Our authentication code</li>
            <li name="ce4f" id="ce4f" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">util/mock-strategy</code>: Our mocked passport authentication strategy</li>
           </ul>
           <p name="ffd7" id="ffd7" class="graf graf--p graf-after--li">And our dependancies:</p>
           <pre name="9c9e" id="9c9e" class="graf graf--pre graf-after--p">"dependencies": {<br>    "koa": "^2.3.0",<br>    "koa-passport": "^3.0.0",<br>    "koa-route": "^3.2.0",<br>    "koa-session": "^5.4.0",<br>    "passport-github2": "^0.1.10"<br>  },<br>  "devDependencies": {<br>    "supertest": "^3.0.0",<br>    "tape": "^4.8.0"<br>  }</pre>
           <p name="fa4d" id="fa4d" class="graf graf--p graf-after--pre">In the discussion that follows, I will omit some code for brevity, particularly the code that mocks a data store. Calls to <code class="markup--code markup--p-code">userStore.fetchUser()</code> and <code class="markup--code markup--p-code">userStore.saveUser()</code> reference functions defined in <code class="markup--code markup--p-code">util/auth.js</code>.</p>
           <h3 name="d0f9" id="d0f9" class="graf graf--h3 graf-after--p">Our routes</h3>
           <p name="9abc" id="9abc" class="graf graf--p graf-after--h3">The routes for this example are shown below. When we’re done, we will be able to access <code class="markup--code markup--p-code">/app</code> from within our tests.</p>
           <pre name="8552" id="8552" class="graf graf--pre graf-after--p">const route = require('koa-route');<br><br>app.use(route.get('/auth/github',<br>  passport.authenticate('github');<br>));<br><br>// Classic redirect behavior<br>app.use(route.get('/auth/github/callback',<br>  passport.authenticate('github', {<br>    successRedirect: '/app',<br>    failureRedirect: '/'<br>  })<br>));</pre>
           <pre name="587a" id="587a" class="graf graf--pre graf-after--pre">// Require authentication for now<br>app.use(async function(ctx, next) {<br>  if (ctx.isAuthenticated()) {<br>    return next();<br>  } else {<br>    ctx.throw(403);<br>  }<br>});<br><br>app.use(route.get('/app', function(ctx) {<br>  ctx.body = ctx.state.user;<br>}));</pre>
           <h3 name="f07d" id="f07d" class="graf graf--h3 graf-after--pre">Choose a strategy and provide a callback&nbsp;function</h3>
           <p name="37fc" id="37fc" class="graf graf--p graf-after--h3">Inside<code class="markup--code markup--p-code">util/auth.js</code>, I define a function that returns the appropriate strategy based on the <code class="markup--code markup--p-code">NODE_ENV</code> environment variable.</p>
           <pre name="24e8" id="24e8" class="graf graf--pre graf-after--p">const passport = require(‘koa-passport’);</pre>
           <pre name="446a" id="446a" class="graf graf--pre graf-after--pre">const GitHubStrategy = require(‘passport-github2’).Strategy;<br>const MockStrategy = require(‘./mock-strategy’).Strategy;</pre>
           <pre name="0e98" id="0e98" class="graf graf--pre graf-after--pre">function strategyForEnvironment() {<br>  let strategy;<br>  switch(process.env.NODE_ENV) {<br>    case ‘production’:<br>      strategy = new GitHubStrategy({<br>      clientID: process.env.CLIENT_ID,<br>      clientSecret: process.env.CLIENT_SECRET,<br>      callbackURL: process.env.CALLBACK_URL<br>    }, strategyCallback);<br>    break;<br>    default:<br>    strategy = new MockStrategy(‘github’, strategyCallback);<br>  }<br>  return strategy;<br>}</pre>
           <p name="72c1" id="72c1" class="graf graf--p graf-after--pre">The callback function shown below, <code class="markup--code markup--p-code">strategyCallback()</code>, is where we receive the result of the OAuth2 flow. Usually this is where you’ll match the OAuth profile ID to a user in your database or create one if the user does not exist. This function is part of the normal configuration for passport and it is passed to both the <code class="markup--code markup--p-code">GitHubStrategy</code> and the <code class="markup--code markup--p-code">MockStrategy</code>.</p>
           <pre name="03b6" id="03b6" class="graf graf--pre graf-after--p">function strategyCallback(accessToken, refreshToken, profile, done) {<br>  // Possibly User.findOrCreate({…}) or similar<br>  let u = {<br>    id: 1,<br>    oauthId: profile.id,<br>    oauthProvider: profile.provider,<br>    email: profile.emails[0].value,<br>    username: profile.username,<br>    avatarUrl: profile._json.avatar_url<br>  };<br>  // synchronous in this example<br>  userStore.saveUser(u);<br>  done(null, u);<br>}</pre>
           <p name="4cb9" id="4cb9" class="graf graf--p graf-after--pre">At the end of <code class="markup--code markup--p-code">util/auth.js</code>, we choose our desired strategy by calling the function <code class="markup--code markup--p-code">strategyForEnvironment()</code> that we defined earlier.</p>
           <pre name="104b" id="104b" class="graf graf--pre graf-after--p">passport.use(strategyForEnvironment());</pre>
           <h3 name="ab3d" id="ab3d" class="graf graf--h3 graf-after--pre">Define mock&nbsp;strategy</h3>
           <p name="15ac" id="15ac" class="graf graf--p graf-after--h3">The mock strategy is defined using the strategy interface provided by passport. This code lives in <code class="markup--code markup--p-code">util/mock-strategy.js</code>:</p>
           <pre name="ef85" id="ef85" class="graf graf--pre graf-after--p">// <a href="https://github.com/marcosnils/passport-dev/blob/master/lib/strategy.js" data-href="https://github.com/marcosnils/passport-dev/blob/master/lib/strategy.js" class="markup--anchor markup--pre-anchor" rel="noopener nofollow" target="_blank">https://github.com/marcosnils/passport-dev/blob/master/lib/strategy.js</a><br>const passport = require(‘passport-strategy’);<br>const util = require(‘util’);</pre>
           <pre name="cfb8" id="cfb8" class="graf graf--pre graf-after--pre">// The reply from Github OAuth2<br>const user = require(‘../mockProfile’);</pre>
           <pre name="78a3" id="78a3" class="graf graf--pre graf-after--pre">function Strategy(name, strategyCallback) {<br>  if (!name || name.length === 0) { throw new TypeError(‘DevStrategy requires a Strategy name’) ; }</pre>
           <pre name="499b" id="499b" class="graf graf--pre graf-after--pre">  passport.Strategy.call(this);</pre>
           <pre name="6e1d" id="6e1d" class="graf graf--pre graf-after--pre">  this.name = name;<br>  this._user = user;<br>  // Callback supplied to OAuth2 strategies handling verification<br>  this._cb = strategyCallback;<br>}</pre>
           <pre name="70e2" id="70e2" class="graf graf--pre graf-after--pre">util.inherits(Strategy, passport.Strategy);</pre>
           <pre name="48c0" id="48c0" class="graf graf--pre graf-after--pre">Strategy.prototype.authenticate = function() {<br>  this._cb(null, null, this._user, (error, user) =&gt; {<br>    this.success(user);<br>  });<br>}</pre>
           <pre name="6bd8" id="6bd8" class="graf graf--pre graf-after--pre">module.exports = {<br>  Strategy<br>};</pre>
           <p name="4aa3" id="4aa3" class="graf graf--p graf-after--pre">The code above has two important purposes: It defines the constructor function for our mock strategy which accepts our <code class="markup--code markup--p-code">strategyCallback()</code> function and it defines the <code class="markup--code markup--p-code">passport.authenticate()</code> method used in our route for <code class="markup--code markup--p-code">/auth/github/callback</code> which provides our user object.</p>
           <p name="37bb" id="37bb" class="graf graf--p graf-after--p">The flow for this looks like this:</p>
           <ol class="postList">
            <li name="8201" id="8201" class="graf graf--li graf-after--p">A <code class="markup--code markup--li-code">GET</code> request is made to <code class="markup--code markup--li-code">/auth/github/callback</code></li>
            <li name="39c1" id="39c1" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">passport.authenticate()</code> is called</li>
            <li name="a7ed" id="a7ed" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">passport.authenticate()</code> calls our <code class="markup--code markup--li-code">strategyCallback()</code> with a faked OAuth profile and an anonymous <code class="markup--code markup--li-code">done</code> callback</li>
            <li name="3329" id="3329" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">strategyCallback()</code> retrieves the correct user from a data store</li>
            <li name="9913" id="9913" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">strategyCallback()</code> calls the <code class="markup--code markup--li-code">done</code> callback with the user object</li>
            <li name="d744" id="d744" class="graf graf--li graf-after--li">The <code class="markup--code markup--li-code">done</code> callback supplied to <code class="markup--code markup--li-code">strategyCallback()</code> in <code class="markup--code markup--li-code">passport.authenticate()</code> calls <code class="markup--code markup--li-code">this.success()</code> with our user object</li>
            <li name="d5b2" id="d5b2" class="graf graf--li graf-after--li"><code class="markup--code markup--li-code">this.success()</code> performs the necessary work to populate the user session</li>
           </ol>
           <p name="0044" id="0044" class="graf graf--p graf-after--li">Our <code class="markup--code markup--p-code">strategyCallback()</code> function defined earlier in <code class="markup--code markup--p-code">util/auth.js</code> is saved to <code class="markup--code markup--p-code">this._cb</code> in the <code class="markup--code markup--p-code">Strategy()</code> constructor function.</p>
           <p name="de5c" id="de5c" class="graf graf--p graf-after--p">The <code class="markup--code markup--p-code">passport.authenticate()</code> method is called by our route handling function for the <code class="markup--code markup--p-code">/auth/github/callback</code> route. Here we call <code class="markup--code markup--p-code">this._cb</code> (containing a reference to our <code class="markup--code markup--p-code">strategyCallback()</code>). As you’ll recall, the function signature is <code class="markup--code markup--p-code">function strategyCallback(accessToken, refreshToken, profile, done)</code>. For our mock, we don’t care about <code class="markup--code markup--p-code">accessToken</code> or <code class="markup--code markup--p-code">refreshToken</code>, so we pass <code class="markup--code markup--p-code">null</code> for both.</p>
           <p name="19d2" id="19d2" class="graf graf--p graf-after--p">We do care about <code class="markup--code markup--p-code">profile</code> and <code class="markup--code markup--p-code">done</code>. We mock the <code class="markup--code markup--p-code">profile</code> object, in this example using my own Github profile data. We supply a callback function for <code class="markup--code markup--p-code">done</code> within which we call <code class="markup--code markup--p-code">this.success(user)</code>. Our <code class="markup--code markup--p-code">done</code> callback is how <code class="markup--code markup--p-code">strategyCallback()</code> provides our final user object, the one that is serialized into the <code class="markup--code markup--p-code">ctx.state.user</code> in our example Koa app.</p>
           <p name="877b" id="877b" class="graf graf--p graf-after--p">Whoa, that’s a lot to think about!</p>
           <h3 name="36c4" id="36c4" class="graf graf--h3 graf-after--p">Test with&nbsp;sessions</h3>
           <p name="9e38" id="9e38" class="graf graf--p graf-after--h3">We can now test application routes that require authentication! Let’s define a few helper methods for our tests. The first helper function wires up the server and configures <code class="markup--code markup--p-code">supertest</code> to work with it. We can use this function to set up a new instance of the server for each test.</p>
           <p name="7e98" id="7e98" class="graf graf--p graf-after--p">I am using <code class="markup--code markup--p-code">tape</code> as the test framework, which requires any outstanding file handles to be explicitly closed. This is the purpose behind calls to <code class="markup--code markup--p-code">httpServer.close()</code> that appear in the examples below.</p>
           <pre name="8894" id="8894" class="graf graf--pre graf-after--p">const prepare = () =&gt; {<br>  const httpServer = app.listen();<br>  const request = agent(app.callback());<br>  return {<br>    request,<br>    httpServer<br>  };<br>}</pre>
           <p name="75ff" id="75ff" class="graf graf--p graf-after--pre">The next helper, <code class="markup--code markup--p-code">createAuthenticatedUser()</code>, allows us to chain requests using <code class="markup--code markup--p-code">supertest</code>. It works by saving the request object to <code class="markup--code markup--p-code">authenticatedUser</code>, which persists our request state for future use. First, the function authenticates the user by accessing the <code class="markup--code markup--p-code">/auth/github/callback</code> route defined earlier. Then, it passes <code class="markup--code markup--p-code">authenticatedUser</code> to a callback function we supply to make future requests with.</p>
           <pre name="e4ab" id="e4ab" class="graf graf--pre graf-after--p">// <a href="https://medium.com/@juha.a.hytonen/testing-authenticated-requests-with-supertest-325ccf47c2bb" data-href="https://medium.com/@juha.a.hytonen/testing-authenticated-requests-with-supertest-325ccf47c2bb" class="markup--anchor markup--pre-anchor" target="_blank">https://medium.com/@juha.a.hytonen/testing-authenticated-requests-with-supertest-325ccf47c2bb</a><br>const createAuthenticatedUser = (done) =&gt; {<br>  const httpServer = app.listen();<br>  const authenticatedUser = agent(app.callback());<br>  authenticatedUser<br>    .get(‘/auth/github/callback’)<br>    .end((error, resp) =&gt; {<br>      done(authenticatedUser);<br>      httpServer.close();<br>  });<br>}</pre>
           <p name="341c" id="341c" class="graf graf--p graf-after--pre">Now you can test routes that require authentication. The test below calls <code class="markup--code markup--p-code">createAuthenticatedUser</code> and passes in a callback function that receives a persisted <code class="markup--code markup--p-code">supertest</code> object as its only parameter. Performing a request using this object allows the request to succeed.</p>
           <pre name="f614" id="f614" class="graf graf--pre graf-after--p">test(‘it should work’, t =&gt; {<br>  createAuthenticatedUser(request =&gt; {<br>    request<br>    .get(‘/app’)<br>    .expect(200)<br>    .end((err, res) =&gt; {<br>      console.log(res.body);<br>      t.end(err);<br>    });<br>  });<br>});</pre>
           <p name="5b45" id="5b45" class="graf graf--p graf-after--pre">The same request without an authenticated request object will fail:</p>
           <pre name="87a6" id="87a6" class="graf graf--pre graf-after--p">test(‘it should deny access to /app’, t =&gt; {<br> const {httpServer, request} = prepare();</pre>
           <pre name="8e7d" id="8e7d" class="graf graf--pre graf-after--pre">request<br>  .get(‘/app’)<br>  .expect(403)<br>  .end((err, res) =&gt; {<br>    httpServer.close();<br>    t.end(err);<br>  });<br>});</pre>
           <p name="c9b2" id="c9b2" class="graf graf--p graf-after--pre">The entire repository (<a href="https://github.com/jboxman/koa-passport-oauth2-testing" data-href="https://github.com/jboxman/koa-passport-oauth2-testing" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">koa-passport-oauth2-testing</a>), with working tests, is available.</p>
           <p name="b5b4" id="b5b4" class="graf graf--p graf-after--p graf--trailing">Happy testing!</p>
