         <div class="devsite-landing-row-header-text">
          <h2 id="easily-access-google-apis-from-java">Easily access Google APIs from Java</h2>
         </div>
        </header>
        <div class="devsite-landing-row-group">
         <div class="devsite-landing-row-item 
  
  
  

  ">
          <figure class="devsite-landing-row-item-image
            ">
           <img src="https://lh3.googleusercontent.com/FRQQdZAvn7lk1UdlnnTNe_QvOc5EdMiHYB_Dy2xdP7Ut28X6TkDJ5RwrxejKHdJ_8wbgx2fd7SK4mCDSQLh0JnN4lOwUvQY=s888" srcset="https://lh3.googleusercontent.com/FRQQdZAvn7lk1UdlnnTNe_QvOc5EdMiHYB_Dy2xdP7Ut28X6TkDJ5RwrxejKHdJ_8wbgx2fd7SK4mCDSQLh0JnN4lOwUvQY=s888 1x,
                https://lh3.googleusercontent.com/FRQQdZAvn7lk1UdlnnTNe_QvOc5EdMiHYB_Dy2xdP7Ut28X6TkDJ5RwrxejKHdJ_8wbgx2fd7SK4mCDSQLh0JnN4lOwUvQY=s888 2x">
          </figure>
          <div class="devsite-landing-row-item-description
                ">
           <div class="devsite-landing-row-item-description-content">
             The Google API Client Library for Java provides functionality common to all Google APIs, for example HTTP transport, error handling, authentication, JSON parsing, media download/upload, and batching. The library includes a powerful 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/oauth2">OAuth 2.0 library</a> with a consistent interface; lightweight, efficient XML and JSON data models that support any data schema; and support for 
            <a href="https://developers.google.com/protocol-buffers/">protocol buffers</a>. 
            <br>
            <br> To call a Google API using Google's client libraries for Java, you need the generated Java library for the Google API you are accessing. These 
            <a href="https://developers.google.com/api-client-library/java/apis/">generated libraries</a> include the core google-api-java-client library along with API-specific information such as the root URL. They also include classes that represent entities in the context of the API, and that are useful for making conversions between JSON objects and Java objects. 
           </div>
          </div>
         </div>
        </div>

          <h2 id="highlights-of-the-google-api-client-library-for-java">Highlights of the Google API Client Library for Java</h2>
         </div>
        </header>
        <div class="devsite-landing-row-group">
         <div class="devsite-landing-row-item 
  
  
  

  
            devsite-landing-row-item-no-image
  ">
          <div class="devsite-landing-row-item-description
                ">
           <h3 id="its-simple-to-call-google-apis">It's simple to call Google APIs</h3>
           <div class="devsite-landing-row-item-description-content">
             You can call Google APIs using Google service-specific generated libraries with the Google API Client Library for Java. (To find the generated client library for a Google API, visit the list of 
            <a href="https://developers.google.com/api-client-library/java/apis/">supported Google APIs</a>.) Here's an example that uses the Calendar API Client Library for Java to make a call to the Google Calendar API:
            <br>
            <pre class="prettyprint"> // Show events on user's calendar.
 View.header("Show Calendars");
 CalendarList feed = client.calendarList().list().execute();
 View.display(feed);</pre>
            <br>
            <h3>The library makes batching and media upload/download easier</h3> The library offers helper classes for 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/batch">batching</a>, 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/media-upload">media upload</a>, and 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/media-download">media download</a>. 
           </div>
          </div>
         </div>
         <div class="devsite-landing-row-item 
  
  
  

  
            devsite-landing-row-item-no-image
  ">
          <div class="devsite-landing-row-item-description
                ">
           <h3 id="the-library-makes-auth-easier">The library makes auth easier</h3>
           <div class="devsite-landing-row-item-description-content">
             The library includes a powerful 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/oauth2">authentication library</a> that can reduce the amount of code you need to handle OAuth 2.0. Sometimes a few lines is all you need. For example:
            <br>
            <pre class="prettyprint"> /** Authorizes the installed application to access user's protected data. */
 private static Credential authorize() throws Exception {
   // load client secrets
   GoogleClientSecrets clientSecrets = GoogleClientSecrets.load(JSON_FACTORY,
       new InputStreamReader(CalendarSample.class.getResourceAsStream("/client_secrets.json")));
   // set up authorization code flow
   GoogleAuthorizationCodeFlow flow = new GoogleAuthorizationCodeFlow.Builder(
       httpTransport, JSON_FACTORY, clientSecrets,
       Collections.singleton(CalendarScopes.CALENDAR)).setDataStoreFactory(dataStoreFactory)
      .build();
   // authorize
   return new AuthorizationCodeInstalledApp(flow, new LocalServerReceiver()).authorize("user");
}</pre>
           </div>
          </div>
         </div>
        </div>

            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/app-engine">App Engine-specific helpers</a> make quick work of authenticated calls to APIs, and you do not need to worry about exchanging code for tokens.

 protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
   AppIdentityCredential credential =

   Urlshortener shortener =
       new Urlshortener.Builder(new UrlFetchTransport(), new JacksonFactory(), credential)
       .build();
   UrlHistory history = shortener.URL().list().execute();
   ...
 }</pre>

           <h3 id="the-library-runs-on-android-15-or-higher-beta">The library runs on Android 1.5 or higher (@Beta).</h3>
           <div class="devsite-landing-row-item-description-content">
             The Google Client Library for Java's Android-specific helper classes are well-integrated with 
            <a href="http://developer.android.com/reference/android/accounts/AccountManager.html">Android AccountManager</a>. For example:
            <br>
            <pre class="prettyprint"> @Override
 public void onCreate(Bundle savedInstanceState) {
   super.onCreate(savedInstanceState);
   // Google Accounts
   credential =
       GoogleAccountCredential.usingOAuth2(this, Collections.singleton(TasksScopes.TASKS));
   SharedPreferences settings = getPreferences(Context.MODE_PRIVATE);
   credential.setSelectedAccountName(settings.getString(PREF_ACCOUNT_NAME, null));
   // Tasks client
   service =
       new com.google.api.services.tasks.Tasks.Builder(httpTransport, jsonFactory, credential)
           .setApplicationName("Google-TasksAndroidSample/1.0").build();
 }</pre>

           <h3 id="installation-is-easy">Installation is easy</h3>
           <div class="devsite-landing-row-item-description-content">
             If you are not using a 
            <a href="https://developers.google.com/api-client-library/java/apis/">generated library</a>, you can download the binary for the Google API Client Library for Java directly from the 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/download">downloads page</a>, or you can use Maven or Gradle. To use Maven, add the following lines to your pom.xml file:
            <br>
            <pre class="prettyprint"> &lt;project&gt;
  &lt;dependencies&gt;
   &lt;dependency&gt;
     &lt;groupId&gt;com.google.api-client&lt;/groupId&gt;
     &lt;artifactId&gt;google-api-client&lt;/artifactId&gt;
     &lt;version&gt;1.22.0&lt;/version&gt;
   &lt;/dependency&gt;
  &lt;/dependencies&gt;
 &lt;/project&gt;</pre>
            <br> To use Gradle, add the following lines to your build.gradle file:
            <br>
            <pre class="prettyprint"> repositories {
      mavenCentral()
  }
  dependencies {
      compile 'com.google.api-client:google-api-client:1.22.0'
  }</pre> For more details about installing and setting up the Google API Client Library for Java, see the 
            <a href="https://developers.google.com/api-client-library/java/google-api-java-client/setup">download and setup instructions</a>. 
           </div>
          </div>

          <div class="devsite-landing-row-item-description
                ">
           <h3 id="supported-environments">Supported environments</h3>
           <div class="devsite-landing-row-item-description-content">
             The Google API Client Library for Java supports these Java environments:
            <ul>
             <li>Java 5 or higher, standard (SE) and enterprise (EE).</li>
             <li><a href="https://cloud.google.com/appengine/docs">Google App Engine</a>.</li>
             <li>Android 1.5 or higher–but if a <a href="https://developer.android.com/google/play-services/index.html">Google Play Services</a> library is available for the Google service you need, use that library instead of this one. The Google Play library will give you the best possible performance and experience.</li>
            </ul> Not supported: Google Web Toolkit (GWT), Java mobile (ME), and Java 1.4 (or earlier). 
            <h3>Dependencies</h3> The Google API Client Library for Java (google-api-java-client) is built on top of two common libraries, also built by Google, and also designed to work with any HTTP service on the web:
            <ul>
             <li><a href="https://developers.google.com/api-client-library/java/google-http-java-client/">Google HTTP Client Library for Java</a></li>
             <li><a href="https://developers.google.com/api-client-library/java/google-oauth-java-client/">Google OAuth Client Library for Java</a></li>
            </ul>
           </div>
