      <p>The <b>Java Speech API</b> (JSAPI) is an <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> for cross-platform support of command and control recognizers, dictation systems, and <a href="/wiki/Speech_synthesizers" class="mw-redirect" title="Speech synthesizers">speech synthesizers</a>. Although JSAPI defines an interface only there are several implementations created by third parties, for example <a href="/wiki/FreeTTS" title="FreeTTS">FreeTTS</a>.</p>

      <h2><span class="mw-headline" id="Core_technologies">Core technologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Speech_API&amp;action=edit&amp;section=1" title="Edit section: Core technologies">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Two core speech technologies are supported through the Java Speech API: <i><a href="/wiki/Speech_synthesis" title="Speech synthesis">speech synthesis</a></i> and <i><a href="/wiki/Speech_recognition" title="Speech recognition">speech recognition</a></i>.</p> 
      <h3><span class="mw-headline" id="Speech_synthesis">Speech synthesis</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Speech_API&amp;action=edit&amp;section=2" title="Edit section: Speech synthesis">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Speech synthesis provides the reverse process of producing synthetic speech from text generated by an application, an applet, or a user. It is often referred to as text-to-speech technology.</p> 
      <p>The major steps in producing speech from text are as follows:</p> 
      <ul> 
       <li>Structure analysis: Processes the input text to determine where paragraphs, sentences, and other structures start and end. For most languages, punctuation and formatting data are used in this stage.</li> 
       <li>Text pre-processing: Analyzes the input text for special constructs of the language. In English, special treatment is required for abbreviations, acronyms, dates, times, numbers, currency amounts, e-mail addresses, and many other forms. Other languages need special processing for these forms, and most languages have other specialized requirements.</li> 
      </ul> 
      <p>The result of these first two steps is a spoken form of the written text. Here are examples of the differences between written and spoken text:</p> 
      <pre>
St. Matthew's hospital is on Main St.
-&gt; “Saint Matthew's hospital is on Main Street”

Add $20 to account 55374.
-&gt; “Add twenty dollars to account five five, three seven four.”
</pre> 
      <p>The remaining steps convert the spoken text to speech:</p> 
      <ul> 
       <li>Text-to-phoneme conversion: Converts each word to phonemes. A phoneme is a basic unit of sound in a language.</li> 
       <li>Prosody analysis: Processes the sentence structure, words, and phonemes to determine the appropriate prosody for the sentence.</li> 
       <li>Waveform production: Uses the phonemes and prosody information to produce the audio waveform for each sentence.</li> 
      </ul> 
      <p>Speech synthesizers can make errors in any of the processing steps described above. Human ears are well-tuned to detecting these errors, but careful work by developers can minimize errors and improve the speech output quality. While the Java Speech API 1 relied on the <a href="/wiki/Java_Speech_Markup_Language" title="Java Speech Markup Language">Java Speech API Markup Language</a> (JSML), the newer release utilizes <a href="/wiki/Speech_Synthesis_Markup_Language" title="Speech Synthesis Markup Language">SSML</a> to provide many ways for you to improve the output quality of a speech synthesizer.</p> 
      <h3><span class="mw-headline" id="Speech_recognition">Speech recognition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Speech_API&amp;action=edit&amp;section=3" title="Edit section: Speech recognition">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Speech recognition provides computers with the ability to listen to spoken language and determine what has been said. In other words, it processes audio input containing speech by converting it to text.</p> 
      <p>The major steps of a typical speech recognizer are as follows:</p> 
      <ul> 
       <li>Grammar design: Defines the words that may be spoken by a user and the patterns in which they may be spoken.</li> 
       <li>Signal processing: Analyzes the spectrum (i.e., the frequency) characteristics of the incoming audio.</li> 
       <li>Phoneme recognition: Compares the spectrum patterns to the patterns of the phonemes of the language being recognized.</li> 
       <li>Word recognition: Compares the sequence of likely phonemes against the words and patterns of words specified by the active grammars.</li> 
       <li>Result generation: Provides the application with information about the words the recognizer has detected in the incoming audio.</li> 
      </ul> 
      <p>A <i>grammar</i> is an object in the Java Speech API that indicates what words a user is expected to say and in what patterns those words may occur. Grammars are important to speech recognizers because they constrain the recognition process. These constraints make recognition faster and more accurate because the recognizer does not have to check for bizarre sentences.</p> 
      <p>The Java Speech API 1 supports two basic grammar types: rule grammars and dictation grammars. These types differ in various ways, including how applications set up the grammars; the types of sentences they allow; how results are provided; the amount of computational resources required; and how they are used in application design. Rule grammars are defined in JSAPI 1 by <a href="/wiki/JSGF" title="JSGF">JSGF</a>, the Java Speech Grammar Format. The newer JSAPI 2 supports the more recent <a href="/wiki/SRGS" class="mw-redirect" title="SRGS">SRGS</a> format. JSAPI 2 does not offer support for dictation.</p> 
      <h2><span id="The_Java_Speech_API’s_classes_and_interfaces"></span><span class="mw-headline" id="The_Java_Speech_API.E2.80.99s_classes_and_interfaces">The Java Speech API’s classes and interfaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Speech_API&amp;action=edit&amp;section=4" title="Edit section: The Java Speech API’s classes and interfaces">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The different classes and interfaces that form the Java Speech API are grouped into the following three packages:</p> 
      <ul> 
       <li>javax.speech: Contains classes and interfaces for a generic speech engine</li> 
       <li>javax.speech.synthesis: Contains classes and interfaces for speech synthesis.</li> 
       <li>javax.speech.recognition: Contains classes and interfaces for speech recognition.</li> 
      </ul> 
      <p>The EngineManager class is like a factory class that all Java Speech API applications use. It provides static methods to enable the access of speech synthesis and speech recognition engines. The Engine interface encapsulates the generic operations that a Java Speech API-compliant speech engine should provide for speech applications.</p> 
      <p>Speech applications can primarily use methods to perform actions such as retrieving the properties and state of the speech engine and allocating and deallocating resources for a speech engine. In addition, the Engine interface exposes mechanisms to pause and resume the audio stream generated or processed by the speech engine. Streams can be manipulated by the AudioManager. The Engine interface is subclassed by the Synthesizer and Recognizer interfaces, which define additional speech synthesis and speech recognition functionality. The Synthesizer interface encapsulates the operations that a Java Speech API-compliant speech synthesis engine should provide for speech applications.</p> 
      <p>The Java Speech API is based on event-handling. Events generated by the speech engine can be identified and handled as required. Speech events can be handled through the EngineListener interface, and more specifically through the RecognizerListener and the SynthesizerListener.</p> 
      <h2><span class="mw-headline" id="Related_Specifications">Related Specifications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Speech_API&amp;action=edit&amp;section=5" title="Edit section: Related Specifications">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The Java Speech API was written before the <a href="/wiki/Java_Community_Process" title="Java Community Process">Java Community Process</a> (JCP) and targeted the <a href="/wiki/Java_Platform,_Standard_Edition" title="Java Platform, Standard Edition">Java Platform, Standard Edition</a> (Java SE). Subsequently, the Java Speech API 2 (JSAPI2) was created as <a rel="nofollow" class="external text" href="http://www.jcp.org/en/jsr/detail?id=113">JSR 113</a> under the JCP. This API targets the <a href="/wiki/Java_Platform,_Micro_Edition" title="Java Platform, Micro Edition">Java Platform, Micro Edition</a> (Java ME), but also complies with Java SE.</p>
