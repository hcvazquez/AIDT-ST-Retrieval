      <h1>Form Validation with JavaScript</h1> 
      <div style="margin-bottom: 20px; overflow: hidden;">

      <p id="articleSummary">Shows you how to write a script that ensures your form is filled in correctly before it's sent to your server. Great for "required fields" such as email addresses!</p> 
      <p>This tutorial will show you how to create a JavaScript-enabled form that checks whether a user has filled in the form correctly before it's sent to the server. This is called <dfn>form validation</dfn>. First we'll explain why form validation is a useful thing, and then build up a simple example form, explaining things as we go along. At the end, there's a little exercise to keep you busy too!</p> 
      <h2>What is form validation?</h2> 
      <p>Form validation is the process of checking that a form has been filled in correctly before it is processed. For example, if your form has a box for the user to type their email address, you might want your form handler to check that they've filled in their address before you deal with the rest of the form.</p> 
      <p>There are two main methods for validating forms: <dfn>server-side</dfn> (using CGI scripts, ASP, etc), and <dfn>client-side</dfn> (usually done using JavaScript). Server-side validation is more secure but often more tricky to code, whereas client-side (JavaScript) validation is easier to do and quicker too (the browser doesn't have to connect to the server to validate the form, so the user finds out instantly if they've missed out that required field!).</p> 
      <div class="articleImageWrap">
       <img alt="Client-side validation" class="articleImage" style="max-width: 376px;" src="/res/Image/articles/development/javascript/form-validation-with-javascript/client_side.gif">
      </div> 
      <div class="articleImageCaption">
       Client-side form validation (usually with JavaScript embedded in the Web page)
      </div> 
      <div class="articleImageWrap">
       <img alt="Server-side validation" class="articleImage" style="max-width: 376px;" src="/res/Image/articles/development/javascript/form-validation-with-javascript/server_side.gif">
      </div> 
      <div class="articleImageCaption">
       Server-side form validation (usually performed by a CGI or ASP script)
      </div> 
      <p>In this tutorial we'll build a simple form with client-side JavaScript validation. You can then adapt this form to your own requirements.</p> 
      <h2>A simple form with validation</h2> 
      <p>Let's build a simple form with a validation script. The form will include one text field called "Your Name", and a <code>submit</code> button. Our validation script will ensure that the user enters their name before the form is sent to the server.</p> 
      <p>Open <a href="/res/File/articles/development/javascript/form-validation-with-javascript/simple_form.html">this page</a> to see it in action. Try pressing the <strong class="menu">Send Details</strong> button without filling anything in the "Your Name" field.</p> 
      <p>You might like to open the <a href="/res/File/articles/development/javascript/form-validation-with-javascript/simple_form_source.html">source code</a> for this form in a separate window, so that you can refer to it throughout the tutorial.</p> 
      <p>You can see that the page consists of a JavaScript function called <code>validate_form()</code> that performs the form validation, followed by the form itself. Let's look at the form first.</p> 
      <h3>The form</h3> 
      <p>The first part of the form is the <code>form</code> tag:</p> 
      <pre class="code"><code>
&lt;form name="contact_form" method="post"
action="/cgi-bin/articles/development/
  javascript/form-validation-with-javascript/contact_simple.cgi"
onsubmit="return validate_form ( );"&gt;
</code></pre> 
      <p>The form is given a <code>name</code> of <code>"contact_form"</code>. This is so that we can reference the form by name from our JavaScript validation function.</p> 
      <p>The form uses the <code>post</code> method to send the data off to a dummy CGI script on ELATED.com's server that thanks the user. In reality, you would of course send the data to your own CGI script, ASP page, etc. (e.g. a form mailer).</p> 
      <p>Finally, the <code>form</code> tag includes an <code>onsubmit</code> attribute to call our JavaScript validation function, <code>validate_form()</code>, when the "Send Details" button is pressed. The <code>return</code> allows us to return the value <code>true</code> or <code>false</code> from our function to the browser, where <code>true</code> means "carry on and send the form to the server", and <code>false</code> means "don't send the form". This means that we can prevent the form from being sent if the user hasn't filled it in properly.</p> 
      <p>The rest of the form prompts the user to enter their name into a form field called <code>contact_name</code>, and adds a "Send Details" submit button:</p> 
      <pre class="code"><code>
&lt;h1&gt;Please Enter Your Name&lt;/h1&gt;

&lt;p&gt;Your Name: &lt;input type="text" name="contact_name"&gt;&lt;/p&gt;
&lt;p&gt;&lt;input type="submit" name="send" value="Send Details"&gt;&lt;/p&gt;

&lt;/form&gt;
</code></pre> 
      <p>Now let's take a look at the JavaScript form validation function that does the actual work of checking our form.</p> 
      <h3>The <code>validate_form()</code> function</h3> 
      <p>The form validation function, <code>validate_form()</code>, is embedded in the <code>head</code> element near the top of the page:</p> 
      <pre class="code"><code>
&lt;script type="text/javascript"&gt;
&lt;!--

function validate_form ( )
{
    valid = true;

    if ( document.contact_form.contact_name.value == "" )
    {
        alert ( "Please fill in the 'Your Name' box." );
        valid = false;
    }

    return valid;
}

//--&gt;
&lt;/script&gt;
</code></pre> 
      <p>The first line (<code>&lt;script type="text/javascript"&gt;</code>) tells the browser that we are writing some JavaScript, and the HTML comment (<code>&lt;!--</code>) in the second line hides the script from older browsers that don't understand JavaScript.</p> 
      <p>Next we start our <code>validate_form()</code> function, then set a variable called <code>valid</code> to the value <code>true</code>:</p> 
      <pre class="code"><code>
function validate_form ( )
{
    valid = true;
</code></pre> 
      <p>We use this <code>valid</code> variable to keep track of whether our form has been filled out correctly. If one of our checks fails, we'll set <code>valid</code> to <code>false</code> so that the form won't be sent.</p> 
      <p>The next 5 lines check the value of our <code>contact_name</code> field to make sure it has been filled in:</p> 
      <pre class="code"><code>
    if ( document.contact_form.contact_name.value == "" )
    {
        alert ( "Please fill in the 'Your Name' box." );
        valid = false;
    }
</code></pre> 
      <p>If the field is empty, the user is warned with an alert box, and the variable <code>valid</code> is set to <code>false</code>.</p> 
      <p>Next, we return the value of our <code>valid</code> variable to the <code>onSubmit</code> event handler (described above). If the value is <code>true</code> then the form will be sent to the server; if it's <code>false</code> then the form will not be sent:</p> 
      <pre class="code"><code>
    return valid;
</code></pre> 
      <p>Finally, we finish our <code>validate_form()</code> function with a closing brace, and end our HTML comment and <code>script</code> element:</p> 
      <pre class="code"><code>
}

//--&gt;

&lt;/script&gt;
</code></pre> 
      <p class="highlightBox">That's all there is to simple JavaScript form validation! Our example is very simple as it only checks one field. Let's expand this example with a more complex function that checks lots of form fields. We'll also look at how to check other types of fields, such as checkboxes, radio buttons and drop-down lists.</p> 
      <h2>A more complex form</h2> 
      <p>Let's look at a more complex validated form with some different types of form fields.</p> 
      <p>Open <a href="/res/File/articles/development/javascript/form-validation-with-javascript/complex_form.html">this page</a> to see it in action. Try pressing the <strong class="menu">Send Details</strong> button without filling in the form and see what happens.</p> 
      <p>Again, you might like to open the <a href="/res/File/articles/development/javascript/form-validation-with-javascript/complex_form_source.html">source code</a> for this form in a separate window, so that you can refer to it as we talk you through.</p> 
      <p>Like our previous example, this page has a form called <code>contact_form</code> and a function called <code>validate_form()</code>. In addition to the previous text field, the form has radio buttons, a drop-down list and a checkbox.</p> 
      <p>The <code>validate_form()</code> function now has 3 extra checks, one for each of our new fields.</p> 
      <h3>Validating radio buttons</h3> 
      <p>After the <code>contact_name</code> text box has been checked, the <code>gender</code> radio buttons are validated:</p> 
      <pre class="code"><code>
    if ( ( document.contact_form.gender[0].checked == false )
    &amp;&amp; ( document.contact_form.gender[1].checked == false ) )
    {
        alert ( "Please choose your Gender: Male or Female" );
        valid = false;
    }
</code></pre> 
      <p>This code checks to see whether either of the radio buttons ("Male" or "Female") have been clicked. If neither have been clicked (<code>checked == false</code>), the user is alerted and <code>valid</code> is set to <code>false</code>.</p> 
      <h3>Validating drop-down lists</h3> 
      <p>Next the "Age" drop-down list is checked to see if the user has selected an option. In the form, we named the first option in the drop-down list <code>"Please Select an Option"</code>. Our JavaScript can then check which option was selected when the user submitted the form. If the first option is selected, we know the user has not selected a "real" option and can alert them:</p> 
      <pre class="code"><code>
    if ( document.contact_form.age.selectedIndex == 0 )
    {
        alert ( "Please select your Age." );
        valid = false;
    }
</code></pre> 
      <p>Note that the values for <code>selectedIndex</code> start at zero (for the first option).</p> 
      <h3>Validating checkboxes</h3> 
      <p>Finally, the "Terms and Conditions" checkbox is validated. We want to the user to agree to our imaginary Terms and Conditions before they send the form, so we'll check to make sure they've clicked the checkbox:</p> 
      <pre class="code"><code>
    if ( document.contact_form.terms.checked == false )
    {
        alert ( "Please check the Terms &amp; Conditions box." );
        valid = false;
    }

</code></pre> 
      <p>Because we set our <code>valid</code> variable to <code>false</code> in any one of the above cases, if one or more of our checks fail, the form will not be sent to the server. If the user has not completed more than one field, then they will see an alert box appear for each field that is missing.</p> 
      <p>Now you know how to write a form validation script that can handle multiple form fields, including text boxes, radio buttons, drop-down lists and check boxes!</p> 
      <p>One point to note about JavaScript validation is that it can always be circumvented by the user disabling JavaScript in their browser, so for secure validation you'll need to write your validating code in your server-side scripts. However, for day-to-day use JavaScript is a quick and easy way to check over your forms before they're sent to your server.</p> 
      <h2>An exercise: "one field at a time" validation</h2> 
      <p>Our example script works by validating all the form fields at once. This can be a bit confusing for the user, especially if they've missed out more than one field, as they will get lots of alert boxes appearing and they might forget which fields they need to fill in!</p> 
      <p>As an exercise, try modifying the script to only prompt the user one field at a time. For example, if they miss out the "Name" and "Gender" fields and press "Send Details", it will only prompt them for the "Name" field initially. Then, after they fill in the "Name" field and press "Send Details" again, it will prompt them for the "Gender" field.</p> 
      <p>As a finishing touch, try making the script move the cursor to the field that needs filling in each time (<em>Hint:</em> use the <code>focus()</code> method to do this).</p> 
      <p>Good luck!</p>
