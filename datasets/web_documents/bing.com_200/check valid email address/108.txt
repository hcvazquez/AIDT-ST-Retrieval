      <h1 class="title">Validating Form Fields Using JavaScript in FrontPage</h1> 
      <div class="lw_vs"> 
       <div id="curversion"> 
        <strong> Office 2003 </strong> 
       </div> 
      </div> 
      <div style="clear:both;"></div> 
      <a name="odc_fpvalidatingformcontrolsusingjavascript"> </a> 
      <div id="nstext" valign="bottom">
       &nbsp; 
       <p>Lisa Wollin<br> Microsoft Corporation</p> 
       <p>May 2004</p> 
       <p>Applies to:<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft® Office FrontPage® 2003<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft FrontPage 2002<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft FrontPage 2000</p> 
       <p><b>Summary:</b> Learn how validate HTML form fields using JavaScript in Microsoft Office FrontPage 2003. This article assumes a general understanding of HTML and JavaScript. (16 printed pages)</p> 
       <h4 class="dtH1">Contents</h4> 
       <p><a href="#odc_fpvalidatingformcontrolsusingjavascript_introduction">Introduction</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_gettingstarted">Getting Started</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_writingthevalidationscript">Writing the Validation Script</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_connectingtheformtothescript">Connecting the Form to the Script</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_associatingformfieldswithcustomscript">Associating Form Fields with Custom Script</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_codelisting">Code Listing</a><br> <a href="#odc_fpvalidatingformcontrolsusingjavascript_conclusion">Conclusion</a></p> 
       <h2 class="dtH1"><a name="odc_fpvalidatingformcontrolsusingjavascript_introduction"></a>Introduction</h2> 
       <p>When you create forms, providing form validation is useful to ensure that your customers enter valid and complete data. For example, you may want to ensure that someone inserts a valid e-mail address into a text box, or perhaps you want to ensure that someone fills in certain fields.</p> 
       <p>Microsoft® Office FrontPage® 2003 allows you to create some custom validation within the form feature. (For more information on how to create form validation using the FrontPage form feature, see <a href="http://go.microsoft.com/fwlink/?linkid=28399">About creating forms in FrontPage 2003</a>.) However, if the form validation provided in FrontPage does not suit your needs or the site on which you are working is hosted on a server that does not have FrontPage Server Extensions from Microsoft installed, you may need to create custom form validation.</p> 
       <blockquote class="dtBlock"> 
        <b class="le">Note</b>&nbsp;&nbsp;&nbsp;FrontPage Server Extensions are a set of programs that provide access to special FrontPage features, such as database processing, form processing, hit counters, and other built-in FrontPage components that require server-side processing.
       </blockquote> 
       <p>You can provide custom validation for your forms in two ways: server-side validation and client-side validation.</p> 
       <p>Server-side validation incorporates code validation into a form handler. For example, you may have an ASP or ASP.NET application on the server that provides the functionality that your form needs for processing after a customer has submitted it. Because the code is stored on the server, server-side validation requires a roundtrip to the server. Sending data back and forth between the client and server can cause some delays. However, for some form functions, server-side validation is necessary. For example, if you have a form that populates a drop-down list based on the value of another form field, server-side processing may be needed to pull data from a database and populate the drop-down list.</p> 
       <p>Client-side validation provides validation within the browser on client computers through JavaScript. Because the code is stored within the page or within a linked file, it is downloaded into the browser when a user accesses the page and, therefore, doesn't require a roundtrip to the server. For this reason, client form validation can be faster than server-side validation. However, client-side validation may not work in all instances. A user may have a browser that doesn't support client-side scripting or may have scripting disabled in the browser. Knowing the limits of client-side scripting helps you decide whether to use client-side form validation.</p> 
       <p>This article explains how to provide client-side validation for forms that you create either by hand or by using the form tools within FrontPage. In addition, this article assumes a general understanding of HTML and JavaScript, although a thorough understanding is not required. For more information about HTML and Web scripting, see one or more of the following references. </p>
       <ul type="disc"> 
        <li><a href="https://msdn.microsoft.com/en-us/library/office/ms533045.aspx">HTML and Dynamic HTML documentation</a></li> 
        <li><a href="http://go.microsoft.com/fwlink/?linkid=28139">Windows Script</a></li> 
        <li><a href="https://msdn.microsoft.com/en-us/library/office/67defydd.aspx">JScript Users Guide</a></li>

       <h3 class="dtH1">Determining Whether Data Is Valid</h3> 
       <p>In some cases, you may want to verify that the data is valid based on a specified format. For example, you know that all e-mail addresses contain an at symbol (@) and at least one period (.). For the e-mail address, the <b>if</b> statement needs to be slightly more detailed to check for the needed values. In addition, because you need to check for multiple values, you need multiple <b>if</b> statements. In this case, there are three <b>if</b> statements: one to check whether the e-mail field contains a value, the second to check whether the e-mail string contains an @ symbol, and the third to check whether the e-mail string contains a period.</p> 
       <p>The second and third <b>if</b> statements use the <b>indexOf</b> method to determine whether a string occurs within another string. For the e-mail address, you need to determine whether an @ symbol or a period occurs within the e-mail form field. If the value returned from the <b>indexOf</b> method is less than zero (or –1), the e-mail address is invalid, the validate function returns false, and focus is returned to the e-mail form field.</p> 
       <p>The following code shows validation for the Email field.</p>

       <p>Multiple <b>if</b> statements have a cascading effect. If the first <b>if</b> statement indicates that the e-mail address contains a value, the second <b>if</b> statement runs, and if the second <b>if</b> statement indicates that the e-mail address contains an @ symbol, the third <b>if</b> statement runs. If any of the three <b>if</b> statements indicate an invalid e-mail address, a message is displayed to the user, the validation function returns false, and the user is returned to the form with the Insertion Point in the Email field.</p> 
       <p>You can also create a similar <b>if</b> statement for telephone numbers. If you specify the format for telephone numbers, you can use the <b>indexOf</b> method to determine whether a specified string (which in some cases is a hyphen [-]) occurs within the telephone form field. If the <b>indexOf</b> method returns a –1, the telephone number does not contain a hyphen.</p> 
       <h3 class="dtH1">Returning a Value from a Function</h3> 
       <p>You can return a value from a custom function by using the <b>return</b> statement. As mentioned previously, each of the <b>if</b> statements shown previously contains a line that returns a <b>Boolean</b> value of <b>false</b><code class="ce">.</code> When the validation function returns to the form, the return value tells the form whether to continue processing. A value of <b>false</b> tells the browser to stop processing the form. Therefore, at the end of the validation function, you also need a statement to return a value of <b>true</b>. This means that all form fields have validated and the form can continue processing.</p>

function ValidateContactForm()
{
    var name = document.ContactForm.Name;
    var email = document.ContactForm.Email;
    var phone = document.ContactForm.Telephone;
    var nocall = document.ContactForm.DoNotCall;
    var what = document.ContactForm.Subject;
    var comment = document.ContactForm.Comment;

    if (name.value == "")
    {
        window.alert("Please enter your name.");
        name.focus();
        return false;
    }
    
    if (email.value == "")
    {

        return false;
    }
    if (email.value.indexOf("@", 0) &lt; 0)

        email.focus();
        return false;
    }
    if (email.value.indexOf(".", 0) &lt; 0)
    {
        window.alert("Please enter a valid e-mail address.");
        email.focus();
        return false;
    }

    if ((nocall.checked == false) &amp;&amp; (phone.value == ""))
    {
        window.alert("Please enter your telephone number.");
        phone.focus();
        return false;
    }

    if (what.selectedIndex &lt; 1)

        what.focus();
        return false;
    }

    if (comment.value == "")

       <h2 class="dtH1"><a name="odc_fpvalidatingformcontrolsusingjavascript_associatingformfieldswithcustomscript"></a>Associating Form Fields with Custom Script</h2> 
       <p>Sometimes you may want to respond when users select certain fields. For example, when a user clicks a button or selects a check box, you may want to enable or disable another field, display a secondary form page, or just display a message. Events allow you to respond to such situations.</p> 
       <p>An event is something that occurs within the browser or something that occurs as a result of something that a user does. For example, the <b>onload</b> event occurs every time a page loads and the <b>onclick</b> event occurs every time a user clicks something contained within the page. For more information, see <a href="https://msdn.microsoft.com/en-us/library/office/cc197055.aspx">onload event</a> and <a href="https://msdn.microsoft.com/en-us/library/office/ms536913.aspx">onclick event</a>. Events enable the browser to perform a certain action when a specific action occurs.</p> 
       <p>You can respond to user events in two ways. </p>
       <ul type="disc"> 
        <li>Add the event script inline within the opening tag for the element</li> 
        <li>Add the event to the element with a reference to a custom function</li> 
       </ul> 
       <p>In the contact form is a check box named DoNotCall. When a user selects this check box, the browser disables the Telephone field so that the user cannot enter a value. The following sections show how to disable the Telephone field using both methods.</p>
