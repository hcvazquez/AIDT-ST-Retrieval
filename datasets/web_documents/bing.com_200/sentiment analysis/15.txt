                      </div>Is the code open source?</h3>
                     <p style="margin-top:0px;margin-right:0px;margin-bottom:0px;margin-left:0px;direction:inherit;color:rgb(0,0,0);font-family:Arial;line-height:normal">Sentiment140 isn't open source, but there are resources with open source code with a similar implementation:</p>
                     <ul>
                      <li><a href="http://streamhacker.com/2010/05/10/text-classification-sentiment-analysis-naive-bayes-classifier/" style="background-color:transparent;font-size:10pt">Text Classification for Sentiment Analysis</a><span style="background-color:transparent;font-size:10pt"> by Jacob Perkins</span></li>
                      <li><a href="http://www.gbsheli.com/2009/03/twitgraph-en.html" style="background-color:transparent;font-size:10pt">TwitGraph</a><span style="background-color:transparent;font-size:10pt"> by Ran Tavory</span></li>
                      <li><a href="http://www.laurentluce.com/posts/twitter-sentiment-analysis-using-python-and-nltk/" style="background-color:transparent;font-size:10pt">Twitter sentiment analysis using Python and NLTK</a><span style="background-color:transparent;font-size:10pt"> by Laurent Luce</span></li>
                      <li><a href="http://www.sananalytics.com/lab/twitter-sentiment/" style="background-color:transparent;font-size:10pt">Twitter Sentiment Corpus</a><span style="background-color:transparent;font-size:10pt"> by Niek Sanders</span></li>
                     </ul>
                     <div>
                      <h3><a name="TOC-What-algorithm-are-you-using-"></a><span style="font-size:large">What algorithm are you using?</span></h3>
                      <p style="margin:0px;direction:inherit;line-height:normal">We are using a Maximum Entropy classifier. <a href="http://cs.stanford.edu/people/alecmgo/papers/TwitterDistantSupervision09.pdf">Read about our machine learning approach.</a></p>
                      <p style="margin:0px;direction:inherit;line-height:normal"><br><span style="color:rgb(68,68,68);font-family:Arial,Verdana,sans-serif;font-size:1.4em;background-color:transparent">Where is the training data?</span></p>
                      <h2 style="color:rgb(0,0,0);font-size:14pt;font-family:Arial;line-height:normal"><a name="Where_is_the_Tweet_corpus_8553"></a></h2>You can download our training data:
                     </div>
                     <div>
                      <ul>
                       <li><a href="http://cs.stanford.edu/people/alecmgo/trainingandtestdata.zip">Stanford link</a></li>
                       <li><a href="https://docs.google.com/file/d/0B04GJPshIjmPRnZManQwWEdTZjg/edit">Google Drive link</a></li>
                      </ul>
                     </div>
                     <h3><a name="TOC-What-is-the-format-of-the-training-data-"></a>What is the format of the training data?</h3>
                     <div>
                      <div>
                       The data is a CSV with emoticons removed. Data file format has 6 fields:
                       <br>0 - the polarity of the tweet (0 = negative, 2 = neutral, 4 = positive)
                       <br>1 - the id of the tweet (2087)
                       <br>2 - the date of the tweet (Sat May 16 23:58:44 UTC 2009)
                       <br>3 - the query (lyx). If there is no query, then this value is NO_QUERY.
                       <br>4 - the user that tweeted (robotickilldozr)
                       <br>5 - the text of the tweet (Lyx is cool)
                       <br>
                       <br>If you use this data, please cite Sentiment140 as your source.
                      </div>

                      <h3><a name="TOC-How-was-your-data-collected-and-annotated-"></a>How was your data collected and annotated?</h3>
                      <div>
                       <div style="line-height:normal">
                        <p>Our approach was unique because our training data was automatically created, as opposed to having humans manual annotate tweets. In our approach, we assume that any tweet with positive emoticons, like :), were positive, and tweets with negative emoticons, like :(, were negative. We used the Twitter Search API to collect these tweets by using keyword search. This is described in our <a href="http://cs.stanford.edu/people/alecmgo/papers/TwitterDistantSupervision09.pdf">paper</a>.<br></p>
                        <span style="font-size:1.4em;background-color:transparent">Where is the tweet corpus for Spanish?</span>
                       </div>
                      </div>
                      <div>
                       <div>
                        Unfortunately, we do not provide the Spanish data set.
                        <br>
                        <br>
                        <span style="font-size:1.4em;background-color:transparent">What did you use to build this?</span>
                       </div>
                       <h2 style="color:rgb(0,0,0);font-size:14pt;font-family:Arial;line-height:normal"><a name="What_did_you_use_to_build_this_2824957212433219"></a></h2>We built this using the following technologies:
                       <br>
                       <ul>
                        <li>Twitter API</li>
                        <li>Amazon EC2 (for the backend)</li>
                        <li>Google Closure (for the JavaScript library)</li>
                        <li>Google Visualization API (for the annotated timeline)</li>
                        <li>Google Charts API (for the pie and bar charts)</li>
                        <li>Google Sites (for this documentation)</li>
                        <li>Google Spreadsheets (for our feedback form)</li>
                        <li>Google Analytics</li>
                       </ul>
                      </div>
                     </div>
                     <div>
                      <h3><a name="TOC-Want-to-discuss-ideas-"></a>Want to discuss ideas?</h3>
                     </div>
                     <div>
                      You're welcome to discuss ideas on the 
                      <a href="https://groups.google.com/forum/#!forum/sentiment140">Sentiment140 forum</a>.
                      <br>
                     </div>
                     <h3><a name="TOC-Do-you-have-any-project-ideas-"></a>Do you have any project ideas?</h3>
                    </div>
                    <div>
                     If you are new to the field of sentiment analysis, we recommend reading the following by Pang and Lee:
                    </div>
                    <div>
                     <a href="http://www.cs.cornell.edu/home/llee/opinion-mining-sentiment-analysis-survey.html">Opinion mining and sentiment analysis</a>
                    </div>
                    <div>
                     <br>
                    </div>
                    <div>
                     There are still many unsolved problems in sentiment analysis.&nbsp;If you're interested, you can help us by working on one of the problems below.
                    </div>
                    <div>
                     <ul>
                      <li><b>Building a classifier for subjective vs. objective tweets.&nbsp;</b>We've focused mostly on classifying positive vs. negative correctly. We haven't looked at classifying tweets with sentiment vs. no sentiment very closely.</li>
                      <li><b>Handling negation. </b>Words like no, not, and never are difficult to handle properly.</li>
                      <ul>
                       <li>Relevant papers:</li>
                       <ul>
                        <li>Isaac G. Councill, Ryan McDonald, and Leonid Velikovich. 2010. What's great and what's not: learning to classify the scope of negation for improved sentiment analysis. [<a href="http://research.google.com/pubs/archive/36744.pdf">pdf</a>]</li>
                        <li>Potts, Christopher. 2010. On the negativity of negation. [<a href="http://www.stanford.edu/%7Ecgpotts/papers/potts-salt20-negation.pdf">pdf</a>]</li>
                       </ul>
                      </ul>
                      <li><b>Handling comparisons. </b>Our bag of words model doesn't handle comparisons very well. For example, in the phrase "Stanford is better than Berkeley", the tweet would be&nbsp;considered&nbsp;positive for both Stanford and Berkeley using our bag of words model because it doesn't take into account the relation towards "better".</li>
                      <li><b>The "aboutness" problem.</b>&nbsp;Given a tweet, automatically detect if the sentiment is towards an entity.</li>
                      <ul>
                       <li>Example:</li>
                       <ul>
                        <li>about the term [Google]: "I love Google."</li>
                        <li>not about the term [Google]: "You should Google that."</li>
                       </ul>
                       <li>Relevant papers:</li>
                       <ul>
                        <li>Target-dependent Twitter Sentiment Classification [<a href="http://www.aclweb.org/anthology/P/P11/P11-1016.pdf">pdf</a>]<a href="http://www.aclweb.org/anthology/P/P11/P11-1016.pdf" target="_blank"><br></a></li>
                       </ul>
                      </ul>
                      <li><b>Determine context switches.</b>&nbsp;Sometimes tweets contain two different ideas. It would be good to be able to segment these two different ideas out. Here's an example: "Just chomped my way through a massive apple, was pretty tasty. Now for work. Business revision."</li>
                      <li><b>Building an accurate parser for tweets. </b>Dependency parsers, like the Stanford Parser, doesn't handle ungrammatical text very well because they were trained on corpuses like the Wall Street Journal . It would be great to develop a parser that can handle informal text better.</li>
                      <li><b>Sarcasm detection.</b></li>
                      <li><b>Topic classification for tweets.</b></li>
                      <li><b>Tag clouds.</b>&nbsp;&nbsp;Given a list of positive and negative tweets, what are the most meaningful words to put in a tag cloud?</li>
                      <li><b>Applying sentiment analysis to Facebook messages. </b>Facebook messages don't have the same character limitations as Twitter, so it's unclear if our methodology would work on Facebook messages.</li>
                      <li><b>Internationalization.&nbsp;</b>We focus only on English sentences,&nbsp;but Twitter has many international users. It should be possible to use our approach to classify sentiment in other languages.</li>
                     </ul>
                    </div>
