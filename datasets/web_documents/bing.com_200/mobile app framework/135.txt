                    <span class="rendered_qtext"><p class="qtext_para">Its pretty hard to say which single JavaScript framework is preferred and it definitely depends on who you talk to. But there are a couple that stand out among the rather crowded field. As of now, these are the two I would look at:</p>
                     <ul>
                      <li><span class="qlink_container"><a href="http://bit.ly/2ptfspk" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 45095295);" class="external_link" data-qt-tooltip="bit.ly">Ionic</a></span> - is free and open source and youll only need HTML, CSS, and JavaScript to build really great hybrid apps (some Angular knowledge would come in handy though). Built on Cordova/PhoneGap, this is an excellent option for hybrid app development.</li>
                      <li><span class="qlink_container"><a href="http://bit.ly/2q5ZBOf" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 45095295);" class="external_link" data-qt-tooltip="bit.ly">React Native</a></span> - This one has been gaining popularity as of late. And even though you write everything in JavaScript, you still make a real mobile app that's indistinguishable from an app built using Objective-C or Java.</li>
                     </ul><p class="qtext_para">Check them out <span class="qlink_container"><a href="http://bit.ly/2qE3YCW" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 45095295);" class="external_link" data-qt-tooltip="bit.ly">side-by-side</a></span>:</p>
                     <div class="qtext_image_wrapper">
                      <canvas class="qtext_image_placeholder landscape qtext_image zoomable_in zoomable_in_feed" width="602.0" height="308.0546875"></canvas>
                      <img class="landscape qtext_image zoomable_in zoomable_in_feed lazy_loaded lazy_loading" src="" data-src="https://qph.ec.quoracdn.net/main-qimg-483976c289253307a883c6ec46c9fdd1.webp" master_src="https://qph.ec.quoracdn.net/main-qimg-dc56a7bf6b26aa115c180749dc61eea8-c" master_w="1024" master_h="524">
                     </div><p class="qtext_para">With both you can really make some great apps in little time. The nature of your app and whether you want to go with hybrid or native will influence your decision, but either way I think you cant go wrong with either of these frameworks.</p></span>

                    <span class="rendered_qtext"><p class="qtext_para">Coredova (PhoneGap),&nbsp; Sencha and Appcelerator (Titanium) are some of the commonly used frameworks.</p><p class="qtext_para">PhoneGap is used quite extensively because of short learning curve involved. Anybody who is a web developer can get started with Phonegap development. It basically makes use of JavaScript libraries to access phone hardware specific functionality. In case&nbsp; your requirement is very much unique you can build your own JavaScript plugin.</p><p class="qtext_para">Sench a typically used with PhoneGap to provide native platform specific UI look and feel. It can be used independently as well. Many of the mobile web sites built using Sench. This is once again JavaScript but the syntax is complicated and takes some time to get around quickly.</p><p class="qtext_para">Titanium is similar to PhoneGap only the difference is they claim that apps with Titanium are more performant than PhoneGap (which is once again open for debate).</p><p class="qtext_para">Thanks for A2A.</p><p class="qtext_para">----------------------Late Edit-------------------------</p><p class="qtext_para">After being asked similar questions quite often, I have created a video series on my YouTube video channel.</p><p class="qtext_para"><span class="qlink_container"><a href="https://www.youtube.com/playlist?list=PLfuE3hOAeWhaKeTcGr2ZlnxzzepuskTRT" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 5574141);" class="external_link" data-qt-tooltip="youtube.com">MAD for Dummies - YouTube</a></span> </p></span>
