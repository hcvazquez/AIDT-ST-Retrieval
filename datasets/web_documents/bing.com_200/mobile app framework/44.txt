           <p class="u-text-teaser" itemprop="description"> Entwickler, die nativen Code pro Endgerät schreiben, lieben natürlich ihren Code. Aber was ist, wenn die App auch auf anderen Plattformen laufen soll? Richtig, die selbe App muss dann nochmal für zwei bis drei Systeme neu geschrieben werden. Das kostet nicht nur Zeit, sondern auch Geld. Hybride <a title="Weitere News zu Webentwicklung" href="http://t3n.de/tag/webentwicklung">Entwicklung</a> kann das ändern. </p>

            <h2>Hybride App-Entwicklung: Ein Code für alle Geräte</h2> 
            <p>Vorteil der hybriden Entwicklung: Applikationen müssen nur einmal mit einem hybriden Framework entwickelt werden und voilá: ein Code für alle Geräte. Dabei nutzt ihr HTML5, CSS und JavaScript –&nbsp;also Technologien, die ihr schon beherrscht – und könnt damit <a title="Weitere News zu Apps" href="http://t3n.de/tag/apps">Apps</a> für verschiedene Plattformen schreiben.</p> 
            <p>Der Zugriff auf native Hardware (wie zum Beispiel die Kamera) passiert über native JavaScript-APIs, wie sie bei Cordova angeboten werden. Deine App wird dann in native Pakete kompiliert, die auf mobilen Geräten oder in den App Store geladen werden können. Im Folgenden ein kurzer Einblick in die bekanntesten Frameworks.</p> 
            <div id="Ads_BA_CAD3" class="u-gap-bottom u-gap-top">
             <script type="text/javascript">Ads_BA_AD("CAD3");</script>

            <h2>1. <a href="http://ionicframework.com">Ionic</a></h2> 
            <figure id="attachment_617705" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-02/" rel="attachment wp-att-617705"><img class="wp-image-617705 size-large" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-02-595x382.jpg" alt="Bildschirmfoto 2015-06-19 um 12.04.31" width="595" height="382"></a>
             <figcaption class="tg-noadgoal">
              Hybride App-Entwicklung: Ionic ist Spitzenklasse. (Screenshot: Ionic)
             </figcaption>
            </figure> 
            <p>Dieses Framework hat sich in den letzten Jahren zum unangefochtenen Framework der Spitzenklasse gemausert. Das Framework wird regelmäßig mit Updates versorgt, sei es um Features der Konkurrenz nachzulegen oder um eigene neue Features präsentieren zu können.</p> 
            <p>Das Fundament bildet das Angular-Framework und stützt sich zusätzlich (wie die meisten anderen Frameworks) auf Cordova, um nativen Code für iOS-, Android- und Windows-Plattformen zur Verfügung zu stellen.</p> 
            <p>Der Vorteil bei Ionic ist, dass es einfach zu schreiben (SASS, HTML und JS), skalierbar und leicht zu warten ist. Darüber hinaus werden Tools für Gesten angeboten, um die Entwicklung auf mobilen Geräten noch zu vereinfachen. Ein Tutorial zum Thema<a href="http://t3n.de/magazin/entwicklung-ionic-237246/"> „Hybride Entwicklung“&nbsp;findet ihr hier auf t3n</a>.</p> 
            <h2>2. <a href="https://software.intel.com/en-us/intel-xdk">Intel XDK</a></h2> 
            <figure id="attachment_617703" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-04/" rel="attachment wp-att-617703"><img class="size-large wp-image-617703" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-04-595x344.jpg" alt="Hybride Entwicklung mit Intel: Intel XDK ist einen Blick wert. (Screenshot: Intel)" width="595" height="344"></a>
             <figcaption class="tg-noadgoal">
              Hybride Entwicklung mit Intel: Intel XDK ist einen Blick wert. (Screenshot: Intel)
             </figcaption>
            </figure> 
            <p>Noch relativ jung ist das Framework von <a title="Weitere News zu Intel" href="http://t3n.de/tag/intel">Intel</a>, die Features sind aber schon beachtlich: Tools für Entwicklung und Emulation sowie Testing und Debugging sind vorhanden, dDazu gibt es „Ready-to-use“-App-Templates, die auf dem <a href="http://„http://app-framework-software.intel.com“">Open-Source-JavaScript-Framework von Intel</a>&nbsp;basieren. Damit können iOS-, Blackberry-, Windows-, Tizen- oder Android-Apps realisiert werden.</p> 
            <p>Darüber hinaus werden <a href="http://t3n.de/news/bootstrap-3-ist-da-488417/">Bootstrap 3</a>, jQuery mobile und das Topcoat-UI-Framework unterstüzt. Mit Intel XDK können (ergänzend zu den oben genannten) auch Applikationen für Firefox OS, Amazon, Chrome und Facebook entwickelt werden.</p> 
            <h2>3. <a href="http://onsen.io">Onsen UI</a></h2> 
            <figure id="attachment_617704" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-03/" rel="attachment wp-att-617704"><img class="size-large wp-image-617704" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-03-595x416.jpg" alt="Hybride Entwicklung auch für Anfänger: Onsen UI. (Screenshot: Monaca) " width="595" height="416"></a>
             <figcaption class="tg-noadgoal">
              Hybride Entwicklung auch für Anfänger: Onsen UI. (Screenshot: Monaca)
             </figcaption>
            </figure> 
            <p>Auch Onsen UI ist ein vergleichsweise junges Framework. Das Open-Source-Framework untersteht der Apache-Lizenz und nutzt ebenfalls Angular und das Topcoat-UI-Framework.</p> 
            <p>Gerade der Einsatz von jQuery-Komponenten macht es auch zu einem idealen Framework für Einsteiger. Ihr könnt auf Angular verzichten und stattdessen auf sehr viele fertige Komponenten setzen. Nativ wird deine App mithilfe von Phonegap und Cordova.</p> 
            <h2>4. <a href="http://www.telerik.com/kendo-ui">Kendo UI</a></h2> 
            <figure id="attachment_617700" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-07/" rel="attachment wp-att-617700"><img class="size-large wp-image-617700" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-07-595x350.jpg" alt="Kendo UI: Hybride Entwicklung mit HTML5 und JavaScript. (Screenshot: Telerik) " width="595" height="350"></a>
             <figcaption class="tg-noadgoal">
              Kendo UI: Hybride Entwicklung mit HTML5 und JavaScript. (Screenshot: Telerik)
             </figcaption>
            </figure> 
            <p>Dieses auf jQuery basierende Framework geht mit knapp 80 fertigen jQuery-Widgets an den Start. Der größte Vorteil liegt jedoch&nbsp;im&nbsp;Theme-Builder, der neben <a href="http://t3n.de/news/material-design-beispiele-581707/">Material Design</a> auch das Erstellen von völlig eigenen Themes zulässt.</p> 
            <p>Das Kendo-UI-Framework unterstützt Angular sowie das Bootstrap-UI-Famework. Ähnlich wie <a title="Weitere News zu Sencha Touch" href="http://t3n.de/tag/sencha-touch">Sencha Touch</a>, hat Kendo UI gerade im Enterprise-Segment einen guten Ruf. Neben einer kommerziellen gibt es auch eine&nbsp;kostenlose Version.</p> 
            <h2>5. <a href="https://www.sencha.com/store/">Sencha Touch</a></h2> 
            <figure id="attachment_617706" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-01/" rel="attachment wp-att-617706"><img class="size-large wp-image-617706" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-01-595x297.jpg" alt="Hybride Entwicklung für Enterprises: Sencha. ( Screenshot Sencha) " width="595" height="297"></a>
             <figcaption class="tg-noadgoal">
              Hybride Entwicklung für Enterprises: Sencha. ( Screenshot Sencha)
             </figcaption>
            </figure> 
            <p>Wenn es um Enterprise-Entwicklung geht, hat Sencha Touch die Nase vorne – allerdings auch zu einem beachtlichen Preis. Sencha setzt auf ExtJS –&nbsp;eines der bekanntesten JavaScript-Frameworks –, das für hoch performante Einsatzmöglichkeiten bekannt ist.</p> 
            <p>Wie auch andere Anbieter, liefert Sencha vorgefertigte Widgets und Themes für die iOS-, Windows-, und Blackberry-Systeme. Das besondere am Application-Builder ist definitiv das Drag-&amp;-Drop-Interface.</p> 
            <h2>6. <a href="http://mobileangularui.com">Mobile Angular UI</a></h2> 
            <figure id="attachment_617699" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung/" rel="attachment wp-att-617699"><img class="size-large wp-image-617699" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-595x359.jpg" alt="Bootstrap und AngularJS: Mobile Angular UI. (Screenshot: Mobile Angular) " width="595" height="359"></a>
             <figcaption class="tg-noadgoal">
              Bootstrap und AngularJS: Mobile Angular UI. (Screenshot: Mobile Angular)
             </figcaption>
            </figure> 
            <p>Wer Bootstrap und Angular mag, wird Mobile Angular UI lieben. Kein Framework ergänzt sich so gut mit Bootstrap 3 wie dieses – ohne die Abhängigkeiten zu den JavaScript-Komponenten von Bootstrap oder jQuery.</p> 
            <p>Konkret ergänzt das Framework Bootstrap um „Switches“, „Overlays“, „Sidebars“, scrollbare Bereiche und absolut positionierte Navigationsleisten. Um die Performance auch auf mobilen Geräten gewährleisten zu können, setzt Mobile Angular UI auf <code>overthrow.js</code>&nbsp;und&nbsp;<code>fastclick.js</code>.</p> 
            <h2>7. <a href="http://famous.org">Famous</a></h2> 
            <figure id="attachment_617701" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-06/" rel="attachment wp-att-617701"><img class="size-large wp-image-617701" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-06-595x403.jpg" alt="Für schwere Grafik-Applikationen: Famous. (Screenshot: Famous) " width="595" height="403"></a>
             <figcaption class="tg-noadgoal">
              Für schwere Grafik-Applikationen: Famous. (Screenshot: Famous)
             </figcaption>
            </figure> 
            <p>Wenn ihr sehr grafikintensive Apps entwickeln wollt, solltet ihr euch famo.us ansehen. Im Gegensatz zu den anderen hybriden Frameworks liegt hier der Fokus auf 2D- und 3D-Renderings, was sie ideal für die Spielentwicklung macht.</p> 
            <h2>7,5. <a href="http://www.idangero.us/framework7/#.VYM1T2AYz-N">Framework 7</a></h2> 
            <figure id="attachment_617702" class="caption alignnone">
             <a href="http://t3n.de/news/hybride-app-entwicklung-frameworks-617199/hybride-app-entwicklung-05/" rel="attachment wp-att-617702"><img class="size-large wp-image-617702" src="http://t3n.de/news/wp-content/uploads/2015/06/hybride-app-entwicklung-05-595x364.jpg" alt="Für iOS-Entwicklung: Framework 7 (Screenshot: Framework 7) " width="595" height="364"></a>
             <figcaption class="tg-noadgoal">
              Für iOS-Entwicklung: Framework 7 (Screenshot: Framework 7)
             </figcaption>
            </figure> 
            <p>Der Hauptvorteil von Framework 7 ist definitiv der Support von iOS-Apps –&nbsp;nur mit JavaScript, HTML und CSS. Das Framework unterstützt sogar Features wie „Swiping“, und UI-Elementen wie Media-Listen, oder Side-Panels ohne, dass sich Entwickler mit JavaScript beschäftigen müssen.</p> 
            <p>Der Nachteil: Framework 7 unterstützt keine anderen Plattformen –&nbsp;aber es ist sicher eines der besten Frameworks, wenn es um iOS-Entwicklung geht.</p> 
            <h2>Fazit: Hybride App-Entwicklung</h2> 
            <p>Für kleine Agenturen oder Freelancer ist Ionic wahrscheinlich die erste Wahl, wer auf Enterprise setzt, sollte Sencha und Kendo UI unter die Lupe nehmen.</p> 
            <p>Grundlegend ist aber zu sagen, dass hybride Entwicklung nicht unbedingt die selbe Performance erreichen kann wie nativer Code, der für eine Plattform entwickelt wurde. Wer also hochperformante Apps entwickeln will, wird nicht um den Einsatz von nativem Code herumkommen. Hybride Frameworks sind aber durchaus eine 80-/20-Lösung die ihr nicht außer Acht lassen solltet.</p> 
            <p>Darüber hinaus gibt es auch andere Alternativen <a href="http://xamarin.com">für die hybride Entwicklung wie zum Beispiel Xamarin</a>. Damit&nbsp;werden Applikationen in C# geschrieben und dann in nativen Code kompiliert. Allerdings ist die Einstiegshürde bei C# ungemein höher als bei CSS/JS/HTML, was die Entwicklung, beziehungsweise den Zugang zur hybriden App-Entwicklung erschwert.</p>
