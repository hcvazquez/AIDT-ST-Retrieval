       <h1 itemprop="name" class="devsite-page-title"> Usage Limits and Billing </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <link href="https://developers.google.com/maps/documentation/local_extensions.css" rel="stylesheet" type="text/css"> 
        <link href="https://developers.google.com/appengine/css/local_extensions.css" rel="stylesheet" type="text/css"> 
        <link href="https://developers.google.com/places/styles/local_extensions.css" rel="stylesheet" type="text/css"> 
        <link rel="stylesheet" href="https://developers.google.com/maps/styles/tos-cards.css">
        <p></p> 
        <aside class="note">
         <strong>Note:</strong>
         <span> If you are developing a web based application that only needs to search for places, and does not submit new places, you can use the Places Library of the Google Maps Javascript API rather than the Google Places API Web Service. The same usage limits apply when using the Places Library in the Google Maps JavaScript API. The daily usage is calculated as the sum of client-side and server-side queries combined.</span>
        </aside> 
        <aside class="note">
         <strong>Note:</strong>
         <span> If you are using Google Places API Web Service in your Android or iOS app, we recommend that you use the Google Places API for Android or the Google Places API for iOS instead. Note that the usage limits are different on <a href="https://developers.google.com/places/android-api/usage">Android</a> and <a href="https://developers.google.com/places/ios-api/usage">iOS</a> and that the Google Places API for Android and the Google Places API for iOS are not included with the Google Maps APIs Premium Plan.</span>
        </aside> 
        <h2 id="usage_limits">Usage limits</h2> 
        <p>The Google Places API Web Service enforces a default limit of <strong>1,000 free requests per 24 hour period</strong>, calculated as the sum of <a href="https://developers.google.com/maps/documentation/javascript/places">client-side</a> and server-side requests. If your app exceeds the initial limit, the app will start failing. You can increase this limit free of charge, up to 150,000 requests per 24 hour period, by enabling billing on the Google API Console to <a href="#enable_billing">verify your identity</a>. A credit card is required for verification. We ask for your credit card purely to validate your identity. Your card will not be charged for use of the Google Places API Web Service.</p> 
        <p>The maximum limit for free usage is <strong>150,000 requests per 24 hour period</strong>. If your app exceeds the limit, the app will start failing again. <a href="#purchase-m4w">Purchase a Google Maps APIs Premium Plan license to get more than 150,000 requests per 24 hour period</a>.</p> 
        <p>Please take steps to increase your limit early if you expect to exceed the default number of requests allowed.</p> 
        <p>Learn how to <a href="https://support.google.com/googleapi/answer/7036164">Monitor, filter, and cap API usage</a>.</p> 
        <p><strong>Note:</strong> The Text Search service is subject to a 10-times multiplier. That is, each Text Search request that you make will count as 10 requests against your quota. If you've purchased the Google Places API Web Service as part of your Google Maps APIs Premium Plan contract, the multiplier may be different. For details, see the <a href="https://developers.google.com/maps/premium/usage-limits">Google Maps APIs Premium Plan documentation</a>.</p>

        <p><a id="enable_billing"></a></p> 
        <h2 id="verify_your_identity_by_enabling_billing">Verify your identity by enabling billing</h2> 
        <p>Verify your identity to get 150,000 free requests per 24 hour period, by enabling billing on the Google API Console. A credit card is required for verification. <strong>We ask for your credit card purely to validate your identity. Your card will not be charged for use of the Google Places API Web Service.</strong></p> 
        <p><button class="button-blue big devsite-dialog-button" data-modal-dialog-id="enable-billing-modal-dialog">Enable Billing</button></p> 
        <div id="enable-billing-modal-dialog" class="devsite-dialog devsite-enable-billing-dialog"> 
         <div class="devsite-dialog-contents"> 
          <h3>Enabling billing</h3> 
          <p>To enable billing and unlock higher quotas for the Google Maps APIs, we'll guide you through the Google Developers Console to complete a few tasks:</p> 
          <ol> 
           <li>Create or choose a project</li> 
           <li>Set up or choose your billing profile</li> 
           <li>Enable billing on your project</li> 
          </ol> 
         </div> 
         <div class="devsite-dialog-buttons"> 
          <button class="devsite-dialog-close button-white" data-category="getKeyModalCancel" data-action="linkClick" data-label="modal">Cancel</button> 
          <a class="button button-white gc-analytics-event" data-category="enableBilling" data-action="buttonClick" data-label="body" href="https://console.developers.google.com/project/_/billing/enable?redirect=https://developers.google.com/places/web-service/usage?dialogOnLoad%3Dbilling-enabled">Continue</a> 
         </div> 
        </div> 
        <p>Click the 'Enable Billing' button to enable billing for your project. If this is the first time you've enabled billing, you will be prompted to set up a billing profile, including specifying a payment method. After you enable billing, your quota will be automatically increased to 150,000 requests per 24 hour period.</p> 
        <p>Note that billing is enabled at the project level, not the API level. You will not be charged for use of the Google Places API Web Service. However, when you enable billing for a given project, all activated, billable APIs in that project will be billed based on your project's usage.</p> 
        <p>You can disable billing for a project at any time on the <a class="gc-analytics-event" data-category="billingPane" data-action="linkClick" data-label="body" href="https://console.developers.google.com/project/_/settings">Billing pane</a>. When you cancel billing, your project returns to an unbilled state, and all activated APIs will be restricted to the courtesy usage limit.</p> 
        <p><a id="purchase-m4w"></a></p> 
        <h2 id="purchase_a_product_maps_api_for_work_license">Purchase a Google Maps APIs Premium Plan license</h2> 
        <p>If you need to use more than 150,000 requests per 24 hour period for the Google Places API Web Service, you must <a href="https://developers.google.com/maps/premium/usage-limits">purchase a Google Maps APIs Premium Plan license</a>. With this license, you can also access <a href="https://developers.google.com/places/web-service/premium-data">additional features of the Google Places API Web Service</a>, such as enhanced user reviews and request filters.</p>
