       <h1 itemprop="name" class="devsite-page-title"> CSS Package for Add-ons </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <style>
  .framebox { margin: 0; }
</style> 
        <p>To help your <a href="https://developers.google.com/apps-script/add-ons">add-on</a> look and feel like Google Sheets, Docs, or Forms, link in the CSS package below to apply Google styling to fonts, buttons, and form elements. For a sample of the CSS package in use, see the <a href="https://developers.google.com/apps-script/quickstart/docs">add-ons quickstart</a>. To use the CSS package, just include the following at the top of each HTML file:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>&lt;link rel="stylesheet" href="https://ssl.gstatic.com/docs/script/css/add-ons1.css"&gt;
</code></pre> 
        <p>Note that the style for form elements cannot be completely controlled in all browsers. In particular, <code>&lt;select&gt;</code> elements display some visual artifacts in Firefox and Internet Explorer, although they still work properly. To see what the styles look like in a given browser, simply load this page in that browser.</p>

        <h2 id="sidebars">Sidebars</h2> 
        <p>Sidebars can be tricky to style because while the height is variable, many add-ons need to include a branding area that does not scroll. Below is a simplified copy of the sidebar from the <a href="https://developers.google.com/apps-script/quickstart/docs">add-on for Google Docs quickstart</a>. If you drag the bottom-right corner of the text area to make the content taller than the sidebar, you'll see that the content area automatically scrolls, but the branding at the bottom does not.</p> 
        <p>As you'll see, the example uses the <code>sidebar</code> class to apply the correct padding and the <code>bottom</code> class to force the branding area to the bottom. A local class, <code>branding-below</code>, then defines the area that the main area of the sidebar should leave clear from the bottom.</p>
