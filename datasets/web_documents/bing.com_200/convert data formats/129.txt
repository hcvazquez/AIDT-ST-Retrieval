             <div id="ctl00_description" class="summary">
              A fast VB-like date format function in JavaScript (requires JScript 5.5+).
             </div>
             <span id="ctl00_thumbnailUrl" class="date" content="https://www.codeproject.com/script/Articles/Images/article100x80.png"></span> 
            </div> 
            <div id="contentdiv" class="text"> 
             <h2>Introduction</h2> 
             <p>I needed a JavaScript date format function such as the Visual Basic <code>Format</code> function, in which you can pass a format string; my first approach was to issue a series of consecutive and "destructive" replace calls, but upon discovering that the 5.5 (or higher) version of JScript supported the use of a function as the <code>replaceText</code> argument of the <code>replace</code> method, I got creative.</p> 
             <p>Here's an example call of what I wanted:</p>
             <pre lang="jscript">SomeDiv.innerText = (<span class="code-keyword">new</span> Date()).format(<span class="code-string">'</span><span class="code-string">dddd, mmmm dd, yyyy.'</span>);</pre> 
             <p>This would display:</p>
             <pre lang="text">Saturday, July 16, 2005</pre> 
             <p>So in my first approach, I globally and case-insensitively replaced <em>dddd</em> with the corresponding string, which "destroyed" every occurrence, so that later in the code I could replace <em>dd</em> with the date number.</p> 
             <p>This worked just fine, but I knew that by inspecting the format specifier for a match, I could skip the search of every format specifier; say I only want the month and the date; well, by switching upon the format specifier (or rather "datepart" specifier), the year replacement will never be issued. Get it?</p> 
             <p>The fun part relies in the use of a function in the <code>replaceText</code> argument of the <code>replace</code> method; this way the <code>$1</code> property as a function argument always represents the last match.</p> 
             <p>Other considerations include the format or "datepart" specifiers: none other than <code>yyyy</code> will be parsed as the year; months and days have the usual three flavors of fullname (<code>mmmm</code>), three-letter (<code>mmm</code>) or numeric (<code>mm</code>); hours (<code>hh</code>) can be rectified to the 12-hour format with the <code>a/p</code> specifier, and minutes (<code>nn</code>) and seconds (<code>ss</code>) may also be specified.</p> 
             <h2>Implementation</h2> 
             <p>WOFA, (Without Further Adou):</p>
             <pre lang="jscript"><span class="code-comment">//</span><span class="code-comment"> a global month names array</span>
<span class="code-keyword">var</span> gsMonthNames = <span class="code-keyword">new</span> Array(
<span class="code-string">'</span><span class="code-string">January'</span>,
<span class="code-string">'</span><span class="code-string">February'</span>,
<span class="code-string">'</span><span class="code-string">March'</span>,
<span class="code-string">'</span><span class="code-string">April'</span>,
<span class="code-string">'</span><span class="code-string">May'</span>,
<span class="code-string">'</span><span class="code-string">June'</span>,
<span class="code-string">'</span><span class="code-string">July'</span>,
<span class="code-string">'</span><span class="code-string">August'</span>,
<span class="code-string">'</span><span class="code-string">September'</span>,
<span class="code-string">'</span><span class="code-string">October'</span>,
<span class="code-string">'</span><span class="code-string">November'</span>,
<span class="code-string">'</span><span class="code-string">December'</span>
);
<span class="code-comment">//</span><span class="code-comment"> a global day names array</span>
<span class="code-keyword">var</span> gsDayNames = <span class="code-keyword">new</span> Array(
<span class="code-string">'</span><span class="code-string">Sunday'</span>,
<span class="code-string">'</span><span class="code-string">Monday'</span>,
<span class="code-string">'</span><span class="code-string">Tuesday'</span>,
<span class="code-string">'</span><span class="code-string">Wednesday'</span>,
<span class="code-string">'</span><span class="code-string">Thursday'</span>,
<span class="code-string">'</span><span class="code-string">Friday'</span>,
<span class="code-string">'</span><span class="code-string">Saturday'</span>
);
<span class="code-comment">//</span><span class="code-comment"> the date format prototype</span>
Date.prototype.format = <span class="code-keyword">function</span>(f)
{
    <span class="code-keyword">if</span> (!this.valueOf())
        <span class="code-keyword">return</span> <span class="code-string">'</span><span class="code-string">&amp;nbsp;'</span>;

    <span class="code-keyword">var</span> d = <span class="code-keyword">this</span>;

    <span class="code-keyword">return</span> f.replace(/(yyyy|mmmm|mmm|mm|dddd|ddd|dd|hh|nn|ss|a\/p)/gi,
        <span class="code-keyword">function</span>($1)
        {
            <span class="code-keyword">switch</span> ($1.toLowerCase())
            {
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">yyyy'</span>: <span class="code-keyword">return</span> d.getFullYear();
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">mmmm'</span>: <span class="code-keyword">return</span> gsMonthNames[d.getMonth()];
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">mmm'</span>:  <span class="code-keyword">return</span> gsMonthNames[d.getMonth()].substr(<span class="code-digit">0</span>, <span class="code-digit">3</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">mm'</span>:   <span class="code-keyword">return</span> (d.getMonth() + <span class="code-digit">1</span>).zf(<span class="code-digit">2</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">dddd'</span>: <span class="code-keyword">return</span> gsDayNames[d.getDay()];
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">ddd'</span>:  <span class="code-keyword">return</span> gsDayNames[d.getDay()].substr(<span class="code-digit">0</span>, <span class="code-digit">3</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">dd'</span>:   <span class="code-keyword">return</span> d.getDate().zf(<span class="code-digit">2</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">hh'</span>:   <span class="code-keyword">return</span> ((h = d.getHours() % <span class="code-digit">12</span>) ? h : <span class="code-digit">12</span>).zf(<span class="code-digit">2</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">nn'</span>:   <span class="code-keyword">return</span> d.getMinutes().zf(<span class="code-digit">2</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">ss'</span>:   <span class="code-keyword">return</span> d.getSeconds().zf(<span class="code-digit">2</span>);
            <span class="code-keyword">case</span> <span class="code-string">'</span><span class="code-string">a/p'</span>:  <span class="code-keyword">return</span> d.getHours() &lt; <span class="code-digit">12</span> ? <span class="code-string">'</span><span class="code-string">a'</span> : <span class="code-string">'</span><span class="code-string">p'</span>;
            }
        }
    );
}</pre> 
             <h2>Notes</h2> 
             <ul> 
              <li>A date with a value of 0 returns a non-breaking space. </li>
              <li>Notice how the <code>d</code> variable is available to the replacement function (but the <code lang="jscript">this</code> object is not). </li>
              <li>The <code>zf</code> number prototype&nbsp;can be found in&nbsp;<a href="http://www.codeproject.com/Articles/16010/Extending-JavaScript-Intrinsic-Objects-with-Protot">Extending JavaScript Objects with Prototypes</a>, where it is called 'zp' for zero padding. It&nbsp;pads a number with zeroes the specified number of times, up to the number's character length, i.e. 2 turns into 02, but 16 remains 16. </li>
              <li>The regular expression looks for any of the bracketed pattern characters in a two+ sequence, or for the very specific a/p match. </li>
              <li>Defining names globally helps and serves other purposes, i.e. listing days in a calendar. </li>
             </ul>

                               </div>Ok, I changed the 'sup' format specifier with 's/p' and replaced the switch with a math operation:<br> <br> <code>case 's/p': var i = d.getDate() % 10; return 'thstndrd'.substr(2 * (i &lt; 4) * i, 2);</code><br> <br> You see, <code>i</code> (date mod 10) ranges from 0 to 9, but only 1, 2 and 3 have specific ordinal suffixes, so with a little math, and in a fashion similar to a choose statement in VB, I got rid of the switch statement for (hopefully) faster execution.<br> <br> The math goes as follows: <code>(i &lt; 4)</code> yields true for 0, 1, 2, 3 and false for everything else; fortunately for us, booleans yield 1 or 0 when used in arithmetic operations (in VB, 'true' actually equals -1). So this simple expression filters out the 'uninteresting' cases.<br> <br> Next, we need the actual <code>i</code> value to pinpoint each case, that's why we multiply. This is also why we calculate and store <code>i</code> ahead of time, for we'll use it twice in the expression.<br> <br> The rest involves doubling the value to get a 2-char index into the strange string we sub into, which is nothing but the ordinal suffixes, th st nd rd.<br> <br> Incidentally, I realized you would need another date specifier to avoid presenting '06th', that is, skip the zero fill.
