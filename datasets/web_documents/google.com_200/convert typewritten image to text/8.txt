        <h3>Introduction</h3> 
        <p>As developers, we consume our time with code editors (or IDEs). Therefore there is a kind of inescapable emotional commitment. Just like some people we spend too much time with, code editors deserve attention, too.</p> 
        <p>I quite enjoy to be a developer with its all aspects, including trying to solve a problem, thinking how to write in an easier way, seeing a working example on browser, even finding right keywords to reach a specific solution on search engines. Most of these things can be made with code editors. For this reason, we need to make them realize what we demand and what we need. Thus, effective and faster code writing relies heavily on both developers and code editors' understanding each other very well.</p> 
        <p>I believe speed is the important thing for a developer if i set knowledge aside for a while. I regardlessly spend at least an hour with discovering and developing to automatize small works which do not take more than ten seconds but had to make them five times on a regular development day. For example, I spend half an hour for finding the right plugin, writing regular expressions, recording a macro, assigning a keyboard shortcut to it in order to make comment-out javascript console logs, which are not needed in a specific code block or function. it will not be a surprise to see this workflow as a waste of time by some of you. However, this would mean the person who does so is underestimating to make these small works over and over again which make someone lose focus. In addition, once you automatize such a series of work, the time you spent will be a profit of yours for the rest of your life.</p> 
        <h2>Best Sublime Text Packages</h2> 
        <p>We need plugins to fully customize Sublime Text rather than config customizations and tweaks. I am currently using Sublime Text version three but going to give alternatives for version two, too.</p> 
        <p>There are two different ways to install packages to Sublime Text.</p> 
        <p>First, you can download package repository zip file from package's github page then extract folder to Sublime Text Packages folder. (For Windows: <strong><em>{{ User name }}</em>\AppData\Roaming\Subllime Text <em>{{ version }}</em>\Packages\ </strong>)</p> 
        <p>Second which is easier way, you can install through Sublime Text official <a href="https://packagecontrol.io/installation" target="_blank" title="Sublime Text Package Control">Package Control</a> manager. Navigate to page by clicking the link and grab the code that belongs to which Sublime version you have. Then, go back to Sublime Text window and paste copied code to<strong>&nbsp;</strong>Sublime Text Console&nbsp;(<code>ctrl+"</code>). You need to restart Sublime Text after installation complete. And lastly, open commands window, (<code>ctrl + shit + p</code>) then type <strong>Package Control: Install Package<strong> </strong></strong>to install a package.</p> 
        <p>After you install a package, try to avoid making changes on package's default configuration file. Most packages will be installed as a single file on Sublime Text 3 and it does not let you modify default package configuration file. However,&nbsp; all of the packages will be extracted to a folder on Sublime Text 2 and it's open for user's editing. Make changes as long as possible in packages' user config file. In this way, when packages&nbsp;or Sublime Text itself are upgraded, you will not lose any changes you made.</p> 
        <h3><a href="https://packagecontrol.io/packages/AdvancedNewFile" target="_blank" title="Sublime Text - AdvancedNewFile">AdvancedNewFile</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-advanced-new-file.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>As it can be understood from its name, AdvancedNewFile makes possible to create new files and folders in Sublime Text. Without any need for Windows or Mac Navigator, you can create files or folders at any level of your project folder. In addition to its <strong>absolute</strong> and <strong>relative</strong> path option, you can also set default extension for newly created files.</p> 
        <h3><a href="https://packagecontrol.io/packages/Alignment" target="_blank" title="Sublime Text- Alignment">Alignment</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-alignment.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>We align code snippets with Alignment package.</p> 
        <h3><a href="https://packagecontrol.io/packages/AngularJS" target="_blank" title="Sublime Text - AngularJS">AngularJS</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-angularjs.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>If you use Angular Js framework on your projects, this package would be what you need with SublimeCodeIntel package. SublimeCodeIntel cannot autocomplete Angular JS controllers' and modules' names in ng-*** attributes. With this package, you can autocomplete modules', controllers', factories' and services' variables by <strong>Rebuilt Search Index</strong> command. Package also makes possible to autocomplete all Angular JS attributes which start with ng-*** and snippets of Angular JS built-in functions like $http, $filter.&nbsp;</p> 
        <h3><a href="https://packagecontrol.io/packages/Auto%20Semi-Colon" target="_blank" title="Sublime Text - Auto Semi-Colon">Auto Semi-Colon</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-auto-semi-colon.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>AutoSemiColon automatically ends line of CSS and Javascript files with <code><strong>;</strong></code> semi-colon. If you do not want to get error while compressing js files, and more importantly to write a clean code, you should not miss semi-colon.</p> 
        <h3><a href="https://packagecontrol.io/packages/AutoFileName" target="_blank" title="Sublime Text - AutoFileName">AutoFileName</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-auto-file-name.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>AutoFileName is one of the top packages that I use mostly. It completes file names automatically. Generally, it autocompletes file names when cursor is between quotes, but I changed the configuration so that autocomplete triggers only with a key combination to avoid file names and variables mixed in the autocomplete pop-up list.&nbsp;</p> 
        <h3><a href="https://packagecontrol.io/packages/ApacheConf.tmLanguage" target="_blank" title="Sublime Text - ApacheConf.tmLanguage">ApacheConf.tmLanguage</a></h3> 
        <figure>
         <img alt="ApacheConf.tmLanguage" class="img-responsive" src="/site/assets/files/1080/apacheconf_tmlanguage.jpg"> 
         <figcaption>
          ApacheConf.tmLanguage
         </figcaption> 
        </figure> 
        <p>Syntax highlight for Apache Configuration files. (.htaccess)</p> 
        <h3><a href="https://packagecontrol.io/packages/BracketHighlighter" target="_blank" title="Sublime Text - BracketHighlighter">BracketHighlighter</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-bracket-highlighter.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>It's one of my favourite packages of Sublime Text. It simply highlights your brackets. When cursor is between quotes, curly braces, parenthesis, it marks the beginning and the end of this part. While writing a complicated mathematical series or nested functions, this package helps you to see in which block the cursor is. It also covers advanced selection functions.</p> 
        <h3><a href="https://packagecontrol.io/packages/Case%20Conversion" target="_blank" title="Sublime Text - Case Conversion">Case Conversion</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-case-conversion.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>We all have our own method of naming variables. If we write more than a couple languages, then it means we use more than a method of naming variables, like <strong>PascalCaseMethod</strong>, <strong>camelCaseMethod</strong>, <strong>snake_case_method</strong> ve <strong>kabab-case-method</strong>. With Case Conversion package, it is possible to pass through different naming methods with a single hotkey.</p> 
        <h3><a href="https://packagecontrol.io/packages/Color%20Highlighter" target="_blank" title="Sublime Text - Color Highlighter">Color Highlighter</a></h3> 
        <figure>
         <img alt="ColorHighlighter" class="img-responsive" src="/site/assets/files/1080/color_highlighter.jpg"> 
         <figcaption>
          ColorHighlighter
         </figcaption> 
        </figure> 
        <p>ColorHighlighter highlights your color values. It supports different kinds of color values like hex, rgb, rgba, hsl, hsla. It also highlights less and sass color variables.</p> 
        <h3><a href="https://packagecontrol.io/packages/Console%20Wrap%20for%20js" target="_blank" title="Sublime Text - Console Wrap for JS">Console Wrap for JS</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-console-wrap-for-js.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>It is one of the most underestimated packages for Javascript Developers. This package adds your variables to the second line in console logs.</p> 
        <h3><a href="https://packagecontrol.io/packages/CSS%20Extended%20Completions" target="_blank" title="Sublime Text - CSS Extended Completions">CSS Extended Completions</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-css-extended-completions.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>This is my favourite package while writing css styles wtih css preprocessors. CSS Extended Completions package indexes your <strong>.less</strong> and <strong>.scss</strong> (or <strong>.sass</strong>) and caches your mixins, variables, class or id names and autocompletes both on html and css. It autocompletes your less and sass mixins with mixin arguments. It also supports emmet completions and gets your css classes on popup window. If you use common css patterns or mixins just like me, this package can index predefined folder path content on Sublime Text launch. Addition to these features, it searches stylesheet link tags on html files and indexes them, too. This feature is disabled on default. You have to make few adjustment on package configuration file to enable it.</p> 
        <h3><a href="https://packagecontrol.io/packages/DocBlockr" target="_blank" title="Sublime Text - DocBlockr">DocBlockr</a></h3> 
        <figure>
         <img alt="DocBlockr" class="img-responsive" src="/site/assets/files/1080/docblockr.jpg"> 
         <figcaption>
          DocBlockr
         </figcaption> 
        </figure> 
        <p>DocBlockr documents your code blocks and functions. It automatically makes entries for function arguments, variables' types and descriptions.</p> 
        <h3><a href="https://packagecontrol.io/packages/Emmet" target="_blank" title="Sublime Text - Emmet">Emmet</a></h3> 
        <p><a href="http://docs.emmet.io/" target="_blank" title="Emmet Official Documentation">Emmet Official Documentation</a> says:</p> 
        <blockquote> 
         <p>Basically, most text editors out there allow you to store and re-use commonly used code chunks, called <em>“snippets”</em>. While snippets are a good way to boost your productivity, all implementations have common pitfalls: you have to define the snippet first and you can’t extend them in runtime.</p> 
         <p>Emmet takes the snippets idea to a whole new level: you can type <em>CSS-like</em> expressions that can be dynamically parsed, and produce output depending on what you type in the abbreviation. Emmet is developed and optimised for web-developers whose workflow depends on HTML/XML and CSS, but can be used with programming languages too.</p> 
        </blockquote> 
        <p>Emmet is the most downloaded package of Sublime Text with 2,2 million downloads on Sublime Text Package Community. It replaced ZenCoding a long time ago and still is the most popular package. There is no need to explain it, just take a look at <a href="http://docs.emmet.io/" target="_blank" title="Emmet Official Documentation">Official Documentation</a>.&nbsp;</p> 
        <h3><a href="https://packagecontrol.io/packages/Gist" target="_blank" title="Sublime Text - Gist">Gist</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-gist.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Gist is a part of Github which you can share your snippets or code blocks with both public and private. This extension makes possible for you to reach your gists directly via Sublime Text. You can create, modify, delete or view all your Gists only with a few commands.</p> 
        <h3><a href="https://packagecontrol.io/packages/Grunt" target="_blank" title="Sublime Text - Grunt">Grunt</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-grunt.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p><a href="http://gruntjs.com/" target="_blank" title="Grunt">Grunt</a> is node based task runner system just like Gulp. This package allows you to run grunt commands within Sublime Text without touching terminal window.</p> 
        <h3><a href="https://packagecontrol.io/packages/HTML%20Nest%20Comments" target="_blank" title="Sublime Text - HTML Nest Comments">HTML Nest Comments</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-html-nest-comments.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Sublime text cannot recognize already commented out html codes in selection. Moreover, it tries to enter the html comment markup only at the beginning and the end of the selection. This package comments out html code blocks which has been already commented out html code.&nbsp;&nbsp;</p> 
        <h3><a href="https://packagecontrol.io/packages/HTML5" target="_blank" title="Sublime Text - HTML5">HTML5</a></h3> 
        <p>This package adds new HTML5 tag completion to Sublime Text, like <code>section</code>, <code>main</code>, <code>header</code>, <code>footer</code>, <code>figure</code>.</p> 
        <h3><a href="https://packagecontrol.io/packages/Inc-Dec-Value" target="_blank" title="Sublime Text - Inc-Dec-Value">Inc-Dec-Value</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-inc-dec-value.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>This Sublime Text package&nbsp;is not known well by developers. Just like Emmet does, Inc-Dec-Value makes possible to modify numbers incremented and decremented by 0.1, 1, 10, 100. It also converts color formats from one to another, like hex, rgb, rgba, hsl, hsla. The most important feature of Inc-Dec Value is to pass through between pre-defined word arrays. For example, I defined all javascript and CSS opposite values in arrays, like <code>fadeIn / fadeOut</code>, <code>true / false</code>, <code>show / hide</code>, <code>static / relative / absolute / fixed</code> and <code>next / prev</code>.</p> 
        <h3><a href="https://packagecontrol.io/packages/jQuery" target="_blank" title="Sublime Text - jQuery">jQuery</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-jquery.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>jQuery is the most popular javascript library and this package adds jQuery functions' completion feature to Sublime Text with <strong>callbacks</strong> options.</p> 
        <h3><a href="https://packagecontrol.io/packages/LESS" title="Sublime Text - LESS">LESS</a></h3> 
        <figure>
         <img alt="LESS" class="img-responsive" src="/site/assets/files/1080/less.jpg"> 
         <figcaption>
          LESS
         </figcaption> 
        </figure> 
        <p>If you use <a href="http://lesscss.org/" target="_blank" title="LESS - CSS Processor">less.js</a> as CSS Preprocessor, this package adds syntax highlight feature to <strong>.less</strong> files. It also completes built-in LESS functions, like <code>fadein</code>, <code>darken</code>, <code>desaturate</code>, <code>data-uri</code>.</p> 
        <h3><a href="https://packagecontrol.io/packages/Local%20History" target="_blank" title="Sublime Text - Local History">Local History</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-local-history.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>It is one of my favorites, too. You can think Local History as a local version system just like Git. It backs up currently editing file on a folder every time you hit ctrl + s. It also compares backed up files with the current one just like Git diff command.</p> 
        <h3><a href="https://packagecontrol.io/packages/Monokai%20Extended" target="_blank" title="Sublime Text - Monokai Extended">Monokai Extended</a></h3> 
        <p>Monoka Extended is an alternate and advanced version of Sublime Text's default color scheme Monokai.</p> 
        <h3><a href="https://packagecontrol.io/packages/Quick%20File%20Open" target="_blank" title="Sublime Text - Quick File Open">Quick File Open</a></h3> 
        <figure>
         <img alt="Quick File Open" class="img-responsive" src="/site/assets/files/1080/quick_file_open.jpg"> 
         <figcaption>
          Quick File Open
         </figcaption> 
        </figure> 
        <p>This package makes possible to reach pre-defined files with a single hotkey.</p> 
        <h3><a href="https://packagecontrol.io/packages/Random%20Everything" target="_blank" title="Sublime Text - Random Everything">Random Everything</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-random-everything.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Random Everything is not much known package of Sublime Text by developers. This package generates random data including date (DD/MM/YYYY formatted), integers on a specific range, random letters, random paragraph, UUID, url, random word, e-mail address, human name, human surname, human name and surname, hex formatted color, IPv4 address, IPv6 address and random value that both includes letters and numbers. It is a very handy package and its usage area depends on your imagination. You can generate html tables with logical values within few seconds.&nbsp;</p> 
        <h3><a href="https://packagecontrol.io/packages/RegReplace" target="_blank" title="Sublime Text - RegReplace">RegReplace</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-reg-replace.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>RegReplace package handles predefined regex replaces for you. It includes few regex replace functions already like removing comments on html file, trimming white spaces (Sublime Text has a built in method for this purpose on save) and removing HTML5 deprecated tags. For example; i created a regexp replace function to comment-out console logs and vice versa on javascript files. There is no limit that you can do with it.</p> 
        <h3><a href="https://packagecontrol.io/packages/Sass" target="_blank" title="Sublime Text - SASS">SASS</a></h3> 
        <figure>
         <img alt="SASS" class="img-responsive" src="/site/assets/files/1080/sublime-text-sass.jpg"> 
         <figcaption>
          SASS
         </figcaption> 
        </figure> 
        <p>If you use <a href="http://sass-lang.com/" target="_blank" title="SASS">SASS</a> as CSS Preprocessor, this package adds syntax highlight feature to <strong>.scss</strong> and <strong>.sass</strong> files. It also completes built-in SASS functions, like <code>@mixin</code>, <code>@include</code> and <code>@if @else</code> statements.</p> 
        <h3><a href="https://packagecontrol.io/packages/SFTP" target="_blank" title="Sublime Text - SFTP">SFTP</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-sftp.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Filezilla is a very diffucult and long way to reach remote files. I often see that developers launch Filezilla to upload files to local server. SFTP Package makes possible to edit, delete, browse, sync, upload and download files on remote server. It is a one of the handy packages of Sublime Text.</p> 
        <h3><a href="https://packagecontrol.io/packages/SideBarEnhancements" target="_blank" title="Sublime Text - SideBarEnhancements">SideBarEnhancements</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-sidebar-enhancments.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Sublime Text Sidebar commands are quite limited and only handles simple actions. SideBarEnhancements package is one of the must installed packages of Sublime Text. It gains your Sidebar context menu a lot of enhancments. You can set "<strong>open with</strong>" command to specific extensions, like opening image files with Photoshop or open your html files with different browsers and assign a specific hotkey to these commands. It supports copy, cut or move between files and folder without touching file explorer. It also makes possible to copy file path, copy file name, search on a specific folder or show selected file on Windows Explorer.</p> 
        <h3><a href="https://packagecontrol.io/packages/StyleToken" target="_blank" title="Sublime Text - StyleToken">StyleToken</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-style-token.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>StyleToken is one of the unknown of Sublime Text packages, too. It makes possible to focus different parts of code or word definition with different highlight styles.</p> 
        <h3><a href="https://packagecontrol.io/packages/SublimeCodeIntel" target="_blank" title="Sublime Text - SublimeCodeIntel">SublimeCodeIntel</a></h3> 
        <p>SublimeCodeIntel package turns your text editor to fully featured IDE. You have definitely install it for extended and advanced code completion on Sublime Text. With a new release of SublimeCodeIntel, you will notice the code completion logic reached higher level compare to previous version.</p> 
        <p>This package makes possible to autocomplete every single word definition on your project. It supports JavaScript, SCSS, Python, HTML, Ruby, Python3, XML, Sass, HTML5, Perl, CSS, Twig, Less, Node.js, TemplateToolkit and PHP completion predefined on config file.</p> 
        <h3><a href="https://packagecontrol.io/packages/SublimeLinter" target="_blank" title="Sublime Text - SublimeLinter">SublimeLinter</a></h3> 
        <figure>
         <img alt="SublimeLinter" class="img-responsive" src="/site/assets/files/1080/sublime_linter.jpg"> 
         <figcaption>
          SublimeLinter
         </figcaption> 
        </figure> 
        <p>SublimeLinter package simply lints your code. It supports&nbsp;PHP, Javascript, Java, CSS, HTML, HAML, Python, Ruby, Json, CoffeeScript, Google Closure, XML, React.js, Slim, Markdown, Go, Perl, C, C++, SQL, Twig, Bootstrap, Jade, LESS, SASS for now. If you want to notice your syntax errors on Sublime Text before browser, or more importantly write cleaner code, you must install SublimeLinter package to achieve your goal. SublimeLinter is a core package for linting, you must install additional packages for each language or framework to make linting process work.</p> 
        <h3><a href="https://packagecontrol.io/packages/SwapStrings" target="_blank" title="Sublime Text - SwapStrings">SwapStrings</a></h3> 
        <figure class="embed-responsive embed-responsive-16by9"> 
         <video class="embed-responsive-item" controls="" preload="none">
          <source src="/site/assets/files/1080/sublime-text-swap-strings.mp4" type="video/mp4">
         </video> 
        </figure> 
        <p>Let's assume that you have mixed two variable a file. For example, you accidently typed <strong>up </strong>instead of <strong>down</strong>, and typed <strong>down</strong> where you must type <strong>up</strong>. What is the fastest way to get over this problem? First, you replace <strong>up</strong> word with quite unique value, like&nbsp;<strong>hfjdhj34gj5h3.</strong> Then replace <strong>down</strong> word with <strong>up</strong>. And lastly, you replace&nbsp;<strong>hfjdhj34gj5h3</strong> value with <strong>up</strong>. This package allows you to replace two variable or word with each other. It also change double quotes to single and single quotes to double within a selected code block or whole document.</p> 
        <h3><a href="https://packagecontrol.io/packages/Terminal" target="_blank" title="Sublime Text - Terminal">Terminal</a></h3> 
        <figure>
         <img alt="Terminal" class="img-responsive" src="/site/assets/files/1080/terminal.jpg"> 
         <figcaption>
          Terminal
         </figcaption> 
        </figure> 
        <p>As a developer, we often use terminal to automatize our jobs through Bower, Grunt, Gulp, npm or node. Beside Sublime Text, some of the editors have built-in Terminal support just like Visual Studio and WebStorm. This package makes possible to open terminal window with a single hotkey and sets project root folder as base folder. It also sets currently opened file path as base folder, too.</p>
