        <h1 class="entry-title" itemprop="headline">Validate Email Address using Java Regular Expression</h1>
        <p class="entry-meta"><time class="entry-modified-time" itemprop="dateModified" datetime="2016-07-05T16:36:47+00:00">July 5, 2016</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="https://www.journaldev.com/author/pankaj" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">Pankaj</span></a></span> <span class="entry-comments-link"><a href="https://www.journaldev.com/638/validate-email-address-using-java-regular-expression#comments">4 Comments</a></span></p>
       </header>
       <div class="entry-content" itemprop="text">
        <div style="float:none;margin:10px 0 10px 0;text-align:center;">
         <ins class="adsbygoogle" style="display:block" data-ad-client="ca-pub-5143693538742787" data-ad-slot="7174295726" data-ad-format="auto"></ins> 
         <script>(adsbygoogle=window.adsbygoogle||[]).push({});</script> 
        </div>
        <p>In the last post, I explained about <strong><a href="https://www.journaldev.com/634/regular-expression-in-java-regex-example">java regular expression</a></strong> in detail with some examples. In this real life example, I am trying to validate email addresses using java regular expression.</p>
        <p>If you look at the <a href="https://en.wikipedia.org/wiki/Email_address#Valid_email_addresses">Wikipedia article for valid email address</a> it supports a lot of special characters but to cover most of the valid email addresses, I am assuming an email address to consist of characters, digits and special characters +, – , . and _. Also every email address must contain only one @ symbol.</p>
        <p>For the domain name, we can have subdomains also. So a subdomain can contain characters, digits and special characters – and _. Again subdomain can be of multiple levels like <em>xyz.abc.com</em>. So middle level TLDs must start with . (DOT) and it can contain characters, digits and special characters – and _. The last TLD should be of minimum length 2 and can contain words only.</p>
        <div style="float:none;margin:10px 0 10px 0;text-align:center;">
         <ins class="adsbygoogle" style="display:block" data-ad-client="ca-pub-5143693538742787" data-ad-slot="5697562523" data-ad-format="auto"></ins> 
         <script>(adsbygoogle=window.adsbygoogle||[]).push({});</script> 
        </div>
        <p>One more important thing to note is that email addresses are case insensitive. So we will use case insensitive flag to create the pattern object.</p>
        <p>Taking all these into account some valid email addresses are:</p>
        <ul>
         <li>journaldev@yahoo.com, journaldev-100@yahoo.com, journaldev.100@yahoo.com</li>
         <li>journaldev111@journaldev.com, journaldev-100@journaldev.net,journaldev.100@journaldev.com.au</li>
         <li>journaldev@1.com,journaldev@gmail.com.com, journaldev+100@gmail.com</li>
         <li>journaldev-100@yahoo-test.com, journaldev_100@yahoo-test.ABC.CoM</li>
        </ul>
        <p>Now let’s see some invalid email addresses are reason for them to be invalid.</p>
        <div style="float:none;margin:10px 0 10px 0;text-align:center;">
         <div data-type="ad" data-publisher="journaldev.com" data-format="300x250" data-zone="jd_post_mid_300x250"></div>
        </div>
        <ul>
         <li>journaldev – No @ symbole</li>
         <li>journaldev@.com.my – Dot after @ symbol</li>
         <li>journaldev123@gmail.a – last TLD length is less than 2</li>
         <li>journaldev123@@.com.com – Two @ symbols</li>
         <li>.journaldev@journaldev.com – ID can’t start with .</li>
         <li>journaldev()*@gmail.com – invalid special characters in the ID</li>
         <li>journaldev@%*.com – invalid special characters in the TLD</li>
         <li>journaldev..2002@gmail.com – ID can’t have two dots</li>
         <li>journaldev.@gmail.com – ID can’t end with dot</li>
         <li>journaldev@journaldev@gmail.com – Two @ symbols</li>
         <li>journaldev@gmail.com.1a – last TLD can have characters only</li>
        </ul>
        <p>So using all these information, here is the EmailValidator class that can be used to validate email address using regular expression.</p>
        <pre class="brush: java; title: ; notranslate" title="">
package com.journaldev.util;

import java.util.regex.Matcher;
import java.util.regex.Pattern;

public class EmailValidator {
	// Regex pattern to valid email address
	private static final String EMAIL_REGEX=&amp;quot;^[\\w-\\+]+(\\.[\\w]+)*@[\\w-]+(\\.[\\w]+)*(\\.[a-z]{2,})$&amp;quot;;
	//static Pattern object, since pattern is fixed
	private static Pattern pattern;
	//non-static Matcher object because it's created from the input String
	private Matcher matcher;
	
	public EmailValidator(){
		//initialize the Pattern object
		pattern = Pattern.compile(EMAIL_REGEX, Pattern.CASE_INSENSITIVE);
	}
	/**
	 * This method validates the input email address with EMAIL_REGEX pattern
	 * @param email
	 * @return boolean
	 */
	public boolean validateEmail(String email){
		matcher = pattern.matcher(email);
		return matcher.matches();
	}
}
</pre>

package com.journaldev.util;

public class EmailValidatorTest {
	
	// list of valid email addresses
	private static final String[] validEmailIds = new String[] { &amp;quot;journaldev@yahoo.com&amp;quot;,
		&amp;quot;journaldev-100@yahoo.com&amp;quot;, &amp;quot;journaldev.100@yahoo.com&amp;quot;,
		&amp;quot;journaldev111@journaldev.com&amp;quot;, &amp;quot;journaldev-100@journaldev.net&amp;quot;,
		&amp;quot;journaldev.100@journaldev.com.au&amp;quot;, &amp;quot;journaldev@1.com&amp;quot;,
		&amp;quot;journaldev@gmail.com.com&amp;quot;, &amp;quot;journaldev+100@gmail.com&amp;quot;,
		&amp;quot;journaldev-100@yahoo-test.com&amp;quot;, &amp;quot;journaldev_100@yahoo-test.ABC.CoM&amp;quot;  };
	
	// list of invalid email addresses
	private static final String[] invalidEmailIds = new String[] { &amp;quot;journaldev&amp;quot;, &amp;quot;journaldev@.com.my&amp;quot;,
		&amp;quot;journaldev123@gmail.a&amp;quot;, &amp;quot;journaldev123@.com&amp;quot;, &amp;quot;journaldev123@.com.com&amp;quot;,
		&amp;quot;.journaldev@journaldev.com&amp;quot;, &amp;quot;journaldev()*@gmail.com&amp;quot;, &amp;quot;journaldev@%*.com&amp;quot;,
		&amp;quot;journaldev..2002@gmail.com&amp;quot;, &amp;quot;journaldev.@gmail.com&amp;quot;,
		&amp;quot;journaldev@journaldev@gmail.com&amp;quot;, &amp;quot;journaldev@gmail.com.1a&amp;quot; };
	
	private static EmailValidator emailValidator;

	public static void main(String args[]){
		emailValidator = new EmailValidator();
		for (String temp : validEmailIds) {
			boolean valid = emailValidator.validateEmail(temp);
			System.out.println(&amp;quot;Email ID &amp;quot;+temp+&amp;quot; is valid? &amp;quot; + valid);
		}
		System.out.println(&amp;quot;\n\n&amp;quot;);
		for (String temp : invalidEmailIds) {
			boolean valid = emailValidator.validateEmail(temp);
			System.out.println(&amp;quot;Email ID &amp;quot;+temp+&amp;quot; is valid? &amp;quot; + valid);
		}
	}
}
</pre>
