     <p>reCAPTCHA for .NET allows a developer to easily integrate Google's reCAPTCHA service (the most popular captcha control used by millions of sites) in an ASP.NET Web Forms or ASP.NET MVC web application.<br><br>The primary features of the library are:<br>* Renders recaptcha control with appropriate options.<br>* Verifies user's answer to reCAPTCHA's challenge.<br>* Supports ASP.NET 4.0, 4.5, and 4.5.1.<br>* Supports ASP.NET MVC 4 and 5. <br>* Seamless support for reCAPTCHA version 1 and version 2.<br><br>reCAPTCHA for .NET has the best and most up-to-date documentation among all the other competing libraries.</p>

     <h2>Release Notes</h2> 
     <p>v2.1<br>1. Supports data type property (API v2 only)<br>2. Supports data size property (API v2 only)<br>3. Bug fixes <br><br>v2.0<br>1. Support for reCAPTCHA v2 APIs.<br>2. Bug fixes<br><br>v1.8<br>1. Fixed the SSL issue by adding an enum called SslBehavior and made it the type of the UseSsl property. UseSsl property is no longer a boolean property.<br><br>v1.7<br>1. Migrated code from CodePlex to GitHub.<br>2. Bug fixes<br>3. Optimized code<br>4. Changed the license to Apache license v2.0<br><br>v1.6.0<br>1. Bug fixes<br>2. Optimized code<br><br>v1.5.1<br>1. Added support for HTTPS.<br>2. Signed the assemblies<br><br>v1.5<br>1. Added legacy support for .NET 4.0 &amp; ASP.NET MVC 4.<br>2. Added support for .NET 4.5.1.<br><br>v1.4<br>1. Minor bug fixes<br>2. Replaced the direct reference to ASP.NET MVC assembly with a dependency on the ASP.NET MVC Nuget package.<br><br>v1.3<br>1. You can now verify a user's response to recpatcha challenge asynchronously using the new "await" keyword.<br>2. By default, recpacha control uses the current UI culture's language. With this release, you can now set it manually if you want.<br>3. The package adds the RecpatchaNet.ReadMe.txt file to your project to help you get started as quickly as possible.</p>
