       <p><a target="_blank" href="https://cordova.apache.org/">Cordova</a> is a framework that lets you create mobile apps using web technologies like HTML, Javascript and CSS. This allows you to target multiple platforms like Android and iOS using one code base. Although you still need platform-specific technologies like the Android SDK and XCode to build an app, you can create apps without having to write any Android or iOS code.</p> 
       <p>Since you can write code in HTML and Javascript, it's very easy to use front-end Javascript libraries like <a target="_blank" href="https://vuejs.org/">Vue.js</a> with Cordova.</p> 
       <p>This tutorial will show you how to create a simple mobile app which generates random words by using Cordova and Vue.js.</p> 
       <h1 id="pre-requisites">Pre-requisites</h1> 
       <ul> 
        <li>Download <a target="_blank" href="https://nodejs.org/en/">Node.js</a></li> 
        <li>Install Cordova: <code>npm install -g cordova</code></li> 
        <li><a target="_blank" href="https://coligo.io/vuejs-the-basics/">Vue.js Basics</a></li> 
       </ul> 
       <h1 id="setting-up-a-cordova-project">Setting Up a Cordova Project</h1> 
       <p>Create a Cordova project called RandomWord:</p> 
       <pre><code class="hljs bash">cordova create RandomWord
<span class="hljs-built_in">cd</span> RandomWord
</code></pre> 
       <p>This will create the directory structure of a Cordova project:</p> 
       <p><img class="ui image" src="directory-structure.png" alt="Cordova Vue.js Directory Structure"></p> 
       <ul> 
        <li><strong>config.xml</strong> - contains info about the app, plugins it uses and platforms it targets</li> 
        <li><strong>platforms</strong> - contains the Cordova libraries for the targeted platforms like Android and iOS that the app will run on</li> 
        <li><strong>plugins</strong> - contains the Cordova libraries for plugins used by the app which allow the app to access things related to the device like Camera and Battery Status</li> 
        <li><strong>www</strong> - contains source code for the app like HTML, Javascript and CSS files</li> 
        <li><strong>hooks</strong> - contains scripts used to customize the build system for the app</li> 
       </ul> 
       <p>Add the Android platform:</p> 
       <pre><code class="hljs bash">cordova <span class="hljs-built_in">platform</span> <span class="hljs-built_in">add</span> android <span class="hljs-comment">--save</span>
</code></pre> 
       <p>This will add the Android platform library to the platforms directory (platforms/android).</p> 
       <p>It will also add the whitelist plugin which is used to specify which URLs the app can link to or open in the browser. The random word generator app will not need this functionality but you can read more about the whitelist plugin <a target="_blank" href="https://cordova.apache.org/docs/en/latest/reference/cordova-plugin-whitelist/">here</a>.</p> 
       <p>The <code>--save</code> flag adds the platform engine to config.xml which is used by the <a target="_blank" href="https://cordova.apache.org/docs/en/latest/reference/cordova-cli/#cordova-prepare-command">cordova prepare</a> command when initializing a Cordova project from a <strong>config.xml</strong> file.</p> 
       <pre><code class="hljs xml">...

       <p>Check if you have the requirements for building/running Android apps through Cordova:</p> 
       <pre><code class="hljs bash">cordova <span class="hljs-keyword">requirements</span>

       <p>To allow the app to access the Vue.js library, we also need to add the following to the end of the Content Security Policy (CSP) <code>meta</code> tag in www/index.html:</p> 
       <pre><code class="hljs html">; script-src <span class="hljs-string">'self'</span> <span class="hljs-symbol">http:</span>/<span class="hljs-regexp">/cdn.jsdelivr.net/vue</span><span class="hljs-regexp">/1.0.16/vue</span>.js <span class="hljs-string">'unsafe-eval'</span>
</code></pre> 
       <p>The Content Security Policy of a webpage allows you to create a whitelist of sources of trusted content and instructs the browser to only execute or render resources from those sources. This is different from the whitelist plugin mentioned above since the whitelist plugin is used mainly to define which links the app is allowed to open whereas the CSP is used to define which scripts the app can execute and which urls the app can make http requests to.</p> 
       <p>The <code>script-src</code> part of the CSP <code>meta</code> tag defines which scripts can be executed by the app.</p> 
       <ul> 
        <li>’self’ - allows scripts from the same origin like www/js/index.js</li> 
        <li><a target="_blank" href="http://cdn.jsdelivr.net/vue/1.0.16/vue.js">http://cdn.jsdelivr.net/vue/1.0.16/vue.js</a> - allows the Vue.js library</li> 
        <li>’unsafe-eval’ - allows unsafe dynamic code evaluation since parts of the Vue.js library code use strings to generate functions</li> 
       </ul> 
       <p>The CSP meta tag should look like this:</p> 
       <pre><code class="hljs html">&lt;meta http-equiv="Content-Security-Policy" content="<span class="hljs-keyword">default-src</span> <span class="hljs-string">'self'</span> data: gap: https://ssl.gstatic.com <span class="hljs-string">'unsafe-eval'</span>; <span class="hljs-keyword">style-src</span> <span class="hljs-string">'self'</span> <span class="hljs-string">'unsafe-inline'</span>; <span class="hljs-keyword">media-src</span> *; <span class="hljs-keyword">script-src</span> <span class="hljs-string">'self'</span> http://cdn.jsdelivr.net/vue/1.0.16/vue.js <span class="hljs-string">'unsafe-eval'</span>"&gt;
</code></pre> 
       <p>For more info about CSP, see <a target="_blank" href="http://www.html5rocks.com/en/tutorials/security/content-security-policy/">html5rocks</a> and the <a target="_blank" href="https://github.com/apache/cordova-plugin-whitelist/blob/master/README.md#content-security-policy">Cordova Docs</a>.</p>

       <h1 id="making-http-requests-with-vue-resource">Making HTTP Requests with vue-resource</h1> 
       <p>Instead of picking a random word from a hard-coded list of words, the app can do a GET request to an API that generates random words like the <a target="_blank" href="http://developer.wordnik.com/docs.html#!/words/getRandomWord_get_4">Wordnik Random Word API</a></p> 
       <p>To allow the app to make an http request to the random word API, add the following to the end of the CSP meta tag:</p> 
       <pre><code class="hljs html">; <span class="hljs-built_in">connect</span>-src http:<span class="hljs-comment">//api.wordnik.com:80/v4/words.json/randomWord</span>
</code></pre> 
       <p>The <code>connect-src</code> part of the CSP meta tag defines which origins the app can make http requests to.</p> 
       <p>The app will use the <a target="_blank" href="https://github.com/vuejs/vue-resource">vue-resource library</a> to make an HTTP request so we have to add vue-resource to the <code>script-src</code> part of the CSP meta tag and add the vue-resource CDN.</p> 
       <p><strong>index.html</strong> will look like:</p>

       <pre><code class="hljs javascript">...

       <h1 id="using-vue-components">Using Vue Components</h1> 
       <p><a target="_blank" href="https://github.com/vuejs/vueify">Vueify</a> is a library for Vue.js that lets you break down the UI into individual components with their own HTML, JavaScript and CSS. This will make your app more modular and it allows you to define components in a hierarchical manner.</p> 
       <p>Using Vue components will add an additional step to your build system to bundle all the components together. Cordova makes this really easy by using <a target="_blank" href="https://cordova.apache.org/docs/en/latest/guide/appdev/hooks/">hooks</a> which let you specify additional scripts to be run at different parts of your build system.</p> 
       <p>This is what your directory will look like after adding the Vue Component:</p> 
       <p><img class="ui image" src="directory-structure-2.png" alt="Cordova Vue.js Directory Structure"></p>

       <pre><code class="hljs html"><span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">template</span>&gt;</span> <span class="hljs-tag">&lt;<span class="hljs-name">div</span> <span class="hljs-attr">class</span>=<span class="hljs-string">"app"</span>&gt;</span> <span class="hljs-tag">&lt;<span class="hljs-name">h1</span>&gt;</span>Random Word<span class="hljs-tag">&lt;/<span class="hljs-name">h1</span>&gt;</span> <span class="hljs-tag">&lt;<span class="hljs-name">button</span> <span class="hljs-attr">id</span>=<span class="hljs-string">"btn-get-random-word"</span> @<span class="hljs-attr">click</span>=<span class="hljs-string">"getRandomWord"</span>&gt;</span>Get Random Word<span class="hljs-tag">&lt;/<span class="hljs-name">button</span>&gt;</span> <span class="hljs-tag">&lt;<span class="hljs-name">p</span>&gt;</span></span><span class="hljs-template-variable">{{randomWord}}</span><span class="xml"><span class="hljs-tag">&lt;/<span class="hljs-name">p</span>&gt;</span> <span class="hljs-tag">&lt;/<span class="hljs-name">div</span>&gt;</span> <span class="hljs-tag">&lt;/<span class="hljs-name">template</span>&gt;</span> <span class="hljs-tag">&lt;<span class="hljs-name">script</span>&gt;</span><span class="javascript"> <span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> { data () { <span class="hljs-keyword">return</span> { randomWord: <span class="hljs-string">''</span> } }, methods: { getRandomWord: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{ <span class="hljs-keyword">this</span>.randomWord = <span class="hljs-string">'...'</span>;        <pre><code class="hljs javascript"><span class="hljs-keyword">var</span> Vue = <span class="hljs-keyword">require</span>(<span class="hljs-string">'vue'</span>);

       <h1 id="summary">Summary</h1> 
       <p>All Done.</p> 
       <p>Cordova makes developing mobile apps with web technologies super simple. Connecting Cordova and Vue.js is also very easy and lets you take advantage of all the cool things about Vue.js (2-way data binding, components, ...) in mobile apps. Now you can make an app in HTML, JavaScript and CSS that targets multiple platforms with 1 code base.</p> 
       <p>This tutorial went over:</p> 
       <ul> 
        <li>Creating a Cordova project</li> 
        <li>Connecting Cordova and Vue.js </li> 
        <li>Making HTTP Requests in a Cordova app by updating it's Content Security Policy </li>

       </ul>
