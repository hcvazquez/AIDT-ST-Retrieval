             <h2>Introduction</h2> 
             <p>SSIS is a great ETL tool and has been there now for a long time.But not always we get all the desired component that we need for transformation.For that we go ahead with Script component. But the disadvantage being that it is application specific.If we need to do the same kind of operation in other SSIS packages, then script component won't help.Instead we can go ahead with our own custom transformation component and even that can be reusable.This article will focus on building, deploying and using a custom script transformation component in a step by step manner</p> 
             <h2>Background</h2> 
             <p>SSIS transformation components helps us to transform the source data to some desirable format. Though SSIS provides many transformation components, but, sometime we may need tomperform certain actions for which there is no components available.In such cases custom components comes very handy.In this article we will look into how we can make a custom SSIS transformation component.The custom component is a simple email validator that will read the emails from some source and will let us know if it is a valid email or not. We will use C# as the language of choice.</p> 
             <h2>Using the code</h2> 
             <p><strong>Step 1:</strong></p> 
             <p>To begin with, let us create a Class Library project (say <strong>DFCEmailValidator</strong>). </p> 
             <p><strong>Step 2:</strong></p> 
             <p>Once the project is created, let us add the following dll references in our project.</p> 
             <ol> 
              <li>Microsoft.SqlServer.Dts.Design </li> 
              <li>Microsoft.SqlServer.DTSPipelineWrap </li> 
              <li>Microsoft.SqlServer.DTSRuntimeWrap </li> 
              <li>Microsoft.SqlServer.ManagedDTS </li> 
              <li>Microsoft.SqlServer.PipelineHost </li> 
             </ol> 
             <p><strong>Step 3:</strong></p> 
             <p>Let us create a class by the name <strong>CustomEmailValidator.cs </strong>.Inherit the class from the <strong>PipelineComponent</strong> class and decorate it with <strong>DtsPipelineComponent attribute</strong> as under</p> 
             <pre><span class="code-keyword">using</span> System;
<span class="code-keyword">using</span> System.Text.RegularExpressions;
<span class="code-keyword">using</span> Microsoft.SqlServer.Dts.Pipeline;
<span class="code-keyword">using</span> Microsoft.SqlServer.Dts.Pipeline.Wrapper;
<span class="code-keyword">using</span> Microsoft.SqlServer.Dts.Runtime.Wrapper;

<span class="code-keyword">namespace</span> DFCEmailValidator
{
    [DtsPipelineComponent(DisplayName = <span class="code-string">"</span><span class="code-string">CustomEmailValidator"</span>
                        , Description = <span class="code-string">"</span><span class="code-string">Custom Email Validator of Data Flow 
                                         Component-Niladri Biswas"</span>
                        , ComponentType = ComponentType.Transform)]
    <span class="code-keyword">public</span> <span class="code-keyword">class</span> CustomEmailValidator : PipelineComponent
    {
    }
}</pre> 
             <p>As we are preparing a custom Data Flow Component henceforth it needs to inherit PipelineComponent Base class. The presence of the DTSPipelineComponentAttribute makes this class as a Transformation type Data Flow Component. Pipeline components are the backbone of SSIS components and works in conjunction with Connection Manager for moving and transforming the data.</p> 
             <p><strong>Step 4: Override the ProvideComponentProperties method</strong></p> 
             <p>Component initialization happens in the ProvideComponentProperties method. This method is called when a component is first added to the data flow task, to initialize the Microsoft.SqlServer.Dts.Pipeline.PipelineComponent.ComponentMetaData of the component. We will perform the below action in this method</p> 
             <ol> 
              <li>Set the component information </li> 
              <li>Add an input object </li> 
              <li>Add an output object </li> 
              <li>Add an error object </li> 
             </ol>

             <p>Lastly add two flat file destination component one of whose source will be the success path of the Conditional path and the other being the Failure. The destination files are named respectively as ValidEmail.txt and InvalidEmail.txt.</p>
