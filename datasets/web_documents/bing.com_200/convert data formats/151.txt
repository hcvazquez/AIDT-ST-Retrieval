               <h2>You Asked </h2> 
               <div class="ate-Question-youAsked">
                 how should i get following format by combining 2 varchar2 columns named test1,test2. 
                <br> 
                <br>in the databse these fields data is stored in this following format 
                <br> 
                <br>test1 column format - 07/23/2001 
                <br>test2 column format - 02:20:55 
                <br> 
                <br>i want's to display in the following format 
                <br> 
                <br>'YYYY-MM-DDHH24.MI.SS' 
                <br> 
                <br>i tried several ways of doing with to_date function still doesn't workout. 
                <br> 
                <br>How can i use same scenario in a stored procedure, if i want's to insert above same format by using above two varchar2 datatype columns into date datatype column called test3. 
                <br> 
                <br>Thanks in advance 
                <br> 
                <br>Partha 
                <br> 
                <br> 
               </div> 
               <h2>and we said...</h2> 
               <div class="ate-Question-weSaid">
                 ops$tkyte@ORA817.US.ORACLE.COM&gt; drop table t; 
                <br> 
                <br>Table dropped. 
                <br> 
                <br>ops$tkyte@ORA817.US.ORACLE.COM&gt; create table t ( test1 varchar2(10), test2 varchar2(10) ); 
                <br> 
                <br>Table created. 
                <br> 
                <br>ops$tkyte@ORA817.US.ORACLE.COM&gt; insert into t values ( '07/23/2001', '02:20:55' ); 
                <br> 
                <br>1 row created. 
                <br> 
                <br> 1 select to_char( 
                <br> 2 to_date( test1 || test2, 'mm/dd/yyyyHH24:mi:ss' ), 
                <br> 3 'yyyy-mm-ddhh24.mi.ss' ) 
                <br> 4* from t 
                <br>ops$tkyte@ORA817.US.ORACLE.COM&gt; / 
                <br> 
                <br>TO_CHAR(TO_DATE(TE 
                <br>------------------ 
                <br>2001-07-2302.20.55 
                <br> 
                <br> 
                <br> 
                <br>Use to_date( test1 || test2, 'mm/dd/yyyyHH24:mi:ss' ) on the insert in PLSQL 
                <br> 
                <br>

               <pre>answer backwards...

you store the number of seconds since some base date (like unix).  Ok, they can store dates back to 1970.  That would not work in real life as people want to store things back to 1900 or 47BC or even 4712BC.  We need to realistically support a broad range of dates.

Also, there is the issue of performance and ease of doing things.  Trunc(date,'mm') becomes a very simple "just consider the fire 3 bytes when comparing", not the first 2.75 bytes or anything like that.

Could you pack it smaller?  Probably.
Would it perform better?  most likely not even close.
Would it be a lot harder to deal with cross platform and such?  probably.


seconds - 0..59, 6 bits
minutes - 0..59, 6 bits
hours   - 0..23, 5 bits
days    - 1..31, 5 bits
and so on

so you mash those all together -- ugh, bit field from heck to intrepret.  trunc becomes hard, everything becomes hard.


back to the seconds since something -- that is computationally the most expensive way to do anything.  

people ask "give me everything between 01-jan-2005 and 06-jan-2005"

must now figure out how many 'seconds' since some date 01-jan-2005 is.  That is *really hard*

                Hi, 
               <br> 
               <br>I've had to create a function that could be used in a SQL statement to convert varchar to date. Maybe it is useful to someone else too. 
               <br> 
               <br>(Validation was intended to be used with table journaling column creationdate and last modification date. But can also be ommited by specifying fixed dates.) 
               <br> 
               <br>Kind regards, 
               <br>Maurice 
               <br> 
               <br> 
               <br>function DATECONVERSION ( 
               <br> p_enteredtext varchar2 
               <br> ,p_firstdate date 
               <br> ,p_lastdate date 
               <br> ) 
               <br>RETURN date 
               <br>AS 
               <br> 
               <br> /* 
               <br> * Function to convert varchar to date with validation check. 
               <br> * 
               <br> * Inputs: varchar2 - text to convert to date 
               <br> * date - lowerlimit of date 
               <br> * date - upperlimit of date 
               <br> * 
               <br> * Output: date - validated date or null when no conversion was posible 
               <br> * 
               <br> * Author: Maurice Kremer 
               <br> * 
               <br> * Date Version Author Comment 
               <br> * 05-Aug-05 0.1 MKr New 
               <br> * 
               <br> */ 
               <br> 
               <br> type t_varchartable 
               <br> is table of varchar2(50) 
               <br> index by binary_integer; -- table type definition 
               <br> 
               <br> v_dateformats t_varchartable; -- table of dateformats 
               <br> 
               <br> v_converted date; -- varchar date to date 
               <br> v_validdate date; -- validated date 
               <br> 
               <br>BEGIN 
               <br> 
               <br> /* setup dateformats to check 
               <br> * can be expanded freely 
               <br> */ 
               <br> v_dateformats(1) := 'DD-Mon-YY HH24:MI:SS'; 
               <br> v_dateformats(2) := 'MM/DD/YY HH24:MI:SS'; 
               <br> v_dateformats(3) := 'DD/MM/YY HH24:MI:SS'; 
               <br> v_dateformats(4) := 'YY/DD/MM HH24:MI:SS'; 
               <br> v_dateformats(5) := 'YY/MM/DD HH24:MI:SS'; 
               <br> v_dateformats(6) := 'MM/YY/DD HH24:MI:SS'; 
               <br> v_dateformats(7) := 'SS/YY/MM HH24:MI:SS'; 
               <br> 
               <br> v_validdate := null; 
               <br> 
               <br> for v_index in 1..v_dateformats.count loop 
               <br> 
               <br> begin 
               <br> 
               <br> v_converted := null; -- clear field of previous conversion 
               <br> 
               <br> /* 
               <br> * try to convert date using specific date format, on error 
               <br> * loop will continue without raising error 
               <br> */ 
               <br> v_converted := to_date( 
               <br> p_enteredtext 
               <br> ,v_dateformats(v_index) 
               <br> ); 
               <br> exception 
               <br> when others then 
               <br> null; -- conversion failed, just continue 
               <br> end; 
               <br> 
               <br> if not v_converted is null -- is date converted 
               <br> and v_converted &lt; p_lastdate -- validate against p_lastdate 
               <br> and v_converted &gt; p_firstdate then -- validate against p_firstdate 
               <br> 
               <br> if nvl(v_validdate,v_converted) = v_converted then 
               <br> 
               <br> /* date is valid */ 
               <br> v_validdate := v_converted; 
               <br> 
               <br> else 
               <br> 
               <br> /* a different second valid date has been found 
               <br> * routine should quit 
               <br> */ 
               <br> return null; 
               <br> end if; 
               <br> 
               <br> end if; 
               <br> 
               <br> end loop; 
               <br> 
               <br> return v_validdate; 
               <br> 
               <br>END;
