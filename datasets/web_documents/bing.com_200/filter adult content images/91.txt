    <p>Slim can be setup using only HTML. See the attribute overview below for an outline of Slims configuration options.</p>

       </tr>

       <tr> 
        <th>data-force-size</th> 
        <td> <p>Determine the forced output size of the resulting image.</p> <p>For example <code>"320,320"</code> will scale the image to fit those dimensions.</p> <p>Force size will also set the aspect ratio to fit the supplied size, so with force-size set to <code>"320,320"</code> this would be <code>"1:1"</code>.</p> <p class="default-value">By default Slim does not resize the output image.</p> </td>

       <tr>

        <td> <p>Slim can sharpen images after they've been resized, this can be used to highlight colors and high contrast edges. This can lead to a slight improvement in image clarity.        </tr> 
       <tr> 
        <th>data-service</th>

    <span class="hljs-attr">"status"</span>:<span class="hljs-string">"success"</span>,
    <span class="hljs-attr">"name"</span>:<span class="hljs-string">"uid_filename.jpg"</span>,
    <span class="hljs-attr">"path"</span>:<span class="hljs-string">"path/uid_filename.jpg"</span>
}</code></pre> <p>You can also set a function reference, in that case Slim will call the function passing the following parameters.</p> <pre><code><span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">formdata, progress, success, failure</span>) </span>{

    <span class="hljs-comment">// "formdata" is a FormData object ready to be send to the server</span>
    <span class="hljs-comment">// more on working with formdata can be found here:</span>
    <span class="hljs-comment">// https://developer.mozilla.org/en-US/docs/Web/API/FormData</span>

    <span class="hljs-comment">// "progress(current, total)" is a function you can call to update the progress indicator</span>
    <span class="hljs-comment">// it expects the uploaded amount of bytes and the total bytes as parameters</span>

    failure(<span class="hljs-string">"something went wrong"</span>);

       </tr> 
       <tr> 
        <th>data-service-format</th> 
        <td> <p>If the <code>data-service</code> attribute is set to a function you can use <code>data-service-format</code> to define how Slim will pass the data to the function.</p> <p>By default Slim will send data as a (FormData Object)[<a href="https://developer.mozilla.org/en-US/docs/Web/API/FormData/Using_FormData_Objects">https://developer.mozilla.org/en-US/docs/Web/API/FormData/Using_FormData_Objects</a>]. Set this attribute to <code>"file"</code> and Slim will send an array of files instead. If only output data is requested then the array will only contain one file, if input and output data is requested it will contain the input file first and then the output file.</p> </td> 
       </tr> 
       <tr> 
        <th>data-fetcher</th> 
        <td> <p>By default Slim will not load dropped URLs as it needs a server script to do so.</p> <p>The <code>data-fetcher</code> attribute can be given a url to the server script that loads remote images. <span class="hljs-tag">&lt;/<span class="hljs-name">div</span>&gt;</span></code></pre> <p class="default-value">By default no remote fetch service is defined.</p> </td> 
       </tr> 
       <tr> 
        <th>data-upload-base64</th> 
        <td> <p>Determines if the asynchronous upload method (data-service) sends base64 data or sends a file object.</p> <p class="default-value">"false"</p> </td> 
       </tr> 
       <tr> 
        <th>data-meta-*</th> 
        <td> <p>Use this attribute to send additional data to the server.</p> <p>For example setting <code>data-meta-user-id</code> to <code>"1234"</code> wil send along that user id to the server which can then be accessed on the image object as shown below.</p> <pre><code><span class="hljs-comment">// get the image data</span>

       </tr> 
       <tr> 
        <th>data-push</th> 
        <td> <p>When set to <code>"true"</code> will hide the upload button and will automatically upload cropped images to the server.</p> <p class="default-value">Default is "false"</p> </td> 
       </tr> 
       <tr> 
        <th>data-post</th> 
        <td> <p>What information to send to the server.</p> <p>The original input image, the cropped output image and/or the actions (crop position and size).</p> <p>Can be set as a comma separated list <code>"input, output, actions"</code>.</p> <p>Note that when sending the input image along the resulting upload size can potentially get two times as big as the original image (input + output).</p> <p>Input and output metadata like filename, size, width and height will always be sent.</p> <p class="default-value">By default only the "output" and the user "actions" are sent.</p> </td> 
       </tr>

                function averagePixelColor(image) {

                    var context = image.getContext('2d');
                    var data = context.getImageData(0, 0, image.width, image.height).data;
                    var length = data.length;
                    var block = 5;
                    var i = -4;
                    var count = 0;
                    var rgb = {r:0, g:0, b:0};

                    while ((i += block * 4) < length) {
                        ++count;
                        rgb.r += data[i];
                        rgb.g += data[i+1];
                        rgb.b += data[i+2];
                    }

                    rgb.r /= count;
                    rgb.g /= count;
                    rgb.b /= count;

                    return rgb;
                }

                function rgbToHsl(color){
                    var r = color.r;
                    var g = color.g;
                    var b = color.b;
                    r /= 255, g /= 255, b /= 255;
                    var max = Math.max(r, g, b);
                    var min = Math.min(r, g, b);
                    var h, s, l = (max + min) / 2;

                    if(max == min){
                        h = s = 0;
                    } else {
                        var d = max - min;
                        s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
                        switch(max){
                            case r: h = (g - b) / d + (g < b ? 6 : 0); break;
                            case g: h = (b - r) / d + 2; break;
                            case b: h = (r - g) / d + 4; break;
                        }
                        h /= 6;
                    }

                    return {
                        h: h * 360,
                        s: s * 100,
                        l: l * 100
                    }
                }

    // get the drawing context for the output image
    var ctx = data.output.image.getContext('2d');

    // draw our watermark on the center of the image
    var size = data.output.width / 20

    var x = data.output.width * .5;
    var y = data.output.height * .5;
    var text = ctx.measureText('Slim is Awesome');

        y + (size * .35)

function addMask(data, ready) {

    // get the drawing context for the output image
    var ctx = data.output.image.getContext('2d');

    // only draw image where mask is
    ctx.globalCompositeOperation = 'destination-in';

    ctx.fillStyle = 'black';
    ctx.beginPath();
    ctx.arc(
        data.output.width * .5,
        data.output.height * .5,
        data.output.width * .5,
        0, 2 * Math.PI);
    ctx.fill();

    // restore to default composite operation (is draw over current image)
    ctx.globalCompositeOperation = 'source-over';

    // continue saving the data
    ready(data);

}

    <h2 id="php-api">PHP API</h2> 
    <p>Slim ships with a static PHP Class which you can use to access the posted data.</p> 
    <p>Use the <code>getImages()</code> method to quickly get the posted image(s).</p>

    <pre><code>$images = Slim::getImages();

    <p>Store the image on your server with the <code>saveFile()</code> method (ready to use examples of this are included in the package). All the information of the original image can be accessed through the image object returned by the <code>getImages()</code> method.</p>

    <h2 id="form-post-or-ajax">Form Post or AJAX Upload</h2> 
    <p>If you're planning to send the cropped data to your server using AJAX, add the <code>data-service</code> attribute and supply it with the url to the API endpoint.</p> 
    <p>If you don't set the <code>data-service</code> attribute Slim will store the data in a automatically created hidden input field named "slim[]". If you add a hidden input field yourself, Slim will use that field. When using AJAX Slim will store the server response in the hidden input field so after a form submit you know which file was uploaded last.</p> 
    <p>You can change the default name of the automatically created input using the <code>defaultInputName</code> property or the <code>data-default-input-name</code> attribute.</p> 
    <p>In both situations Slim will prevent the original file input from posting so data is not sent to the server twice.</p>
