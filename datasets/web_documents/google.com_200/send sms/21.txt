         <h3><a id="user-content-textbelt-open-source" class="anchor" href="#textbelt-open-source" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>TextBelt Open Source</h3> 
         <p>TextBelt Open Source is a REST API that sends outgoing SMS. It uses a free mechanism for sending texts, different from the more reliable paid version available at <a href="https://textbelt.com">https://textbelt.com</a>.</p> 
         <p>This project uses carrier-specific gateways to deliver your text messages for free, and without ads. The service is fairly reliable when configured on a private server and has sent over 1 million texts.</p> 
         <p>Send a text with a simple POST request:</p> 
         <div class="highlight highlight-source-shell">
          <pre>$ curl -X POST http://textbelt.com/text \
   -d number=5551234567 \
   -d <span class="pl-s"><span class="pl-pds">"</span>message=I sent this message for free with textbelt.com<span class="pl-pds">"</span></span></pre>
         </div> 
         <p><code>number</code> and <code>message</code> parameters are required.</p> 
         <h3><a id="user-content-success-and-failure" class="anchor" href="#success-and-failure" aria-hidden="true">

           </svg></a>Success and Failure</h3> 
         <p>Sample success:</p> 
         <div class="highlight highlight-source-json">
          <pre>{<span class="pl-s"><span class="pl-pds">"</span>success<span class="pl-pds">"</span></span>:<span class="pl-c1">true</span>}</pre>
         </div> 
         <p>Note that success means that the message was sent to Textbelt's list of providers. We can't guarantee delivery on your network.</p>

         <h3><a id="user-content-usage-as-a-module" class="anchor" href="#usage-as-a-module" aria-hidden="true">

           </svg></a>Usage as a module</h3> 
         <p>Though this repository contains an express server so you may run your own instance of the web app, you may also use it to send text messages in your project. The only requirement is unix/posix <code>sendmail</code>, used to forward the message.</p> 
         <p>For example, to send a text using the default settings:</p>

         <h3><a id="user-content-usage-as-a-standalone-server" class="anchor" href="#usage-as-a-standalone-server" aria-hidden="true">

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Usage as a standalone server</h3> 
         <p>Textbelt can be run as a standalone server with: <code>node server/app.js</code>. Be sure to install dependencies first with <code>npm install</code>. This project also relies on redis. To install redis locally, please see the <a href="http://redis.io/topics/quickstart">redis documentation</a>. Before launching the app, ensure redis is running on port 6379 with <code>redis-server</code>.</p> 
         <p>Textbelt also depends on a local installation of <code>mutt</code>, the email client (<a href="http://www.mutt.org/">http://www.mutt.org/</a>). On Linux (Ubuntu), you can <code>sudo apt-get install mutt</code>. On macs, you can <code>brew install mutt</code>.</p> 
         <p>By default, the server listens on port 9090 and is configured to accept traffic from a reverse proxy or load balancer such as nginx. To enable accurate IP rate limiting, the reverse proxy should be configured to set the <code>X-Real-IP</code> header.</p> 
         <p>Don't forget to set <code>fromAddress</code> in <code>lib/config.js</code> to the email address you want to send from.</p>

           </svg></a>Canadian and International endpoints</h3> 
         <p>The /text endpoint supports U.S. phone numbers (and parts of Canada).</p> 
         <p>For Canadian texts, curl <code>http://textbelt.com/canada</code>.</p> 
         <p>For international texts, curl <code>http://textbelt.com/intl</code>.</p> 
         <p>Canadian and international support may not be complete. Refer to the list of supported carriers.</p>

           </svg></a>Notes and Limitations</h3>
