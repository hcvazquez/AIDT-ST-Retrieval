         <h2 class="helpHead2" id="Tdxn4tBK-heading2"><span class="ph" id="topic-title"><a name="topic-title"></a>Using Mobile Frameworks</span></h2> 
         <div class="body">
          <div class="shortdesc">
           A mobile web framework makes it easy to support the single-page application pattern when writing Visualforce pages for Salesforce1.
          </div> 
          <p class="p">Getting apps to render perfectly on different screen sizes can be tedious. Styling every user interface component to provide an optimized mobile experience is hard as well. Use existing CSS frameworks that have already done the hard work for you. To build mobile-optimized user interfaces, we recommend that you use the Lightning Design System (SLDS).</p> 
          <p class="p">The Lightning Design System is a CSS framework that gives you access to the icons, color palettes, and font that our developers use to create this new Salesforce look and feel.</p> 
          <p class="p">There are other CSS frameworks out there, such as Bootstrap, Ionic, and Ratchet. However, if your goal is to match the style of Salesforce1, the Lightning Design System is the way to go. It’s continuously updated, so you’re always consistent with Lightning Experience. You can even use other CSS frameworks with SLDS for extra customization. </p> 
          <p dir="ltr" class="p">Writing single-page apps from scratch is complex. Fortunately, you can use existing JavaScript frameworks that abstract and manage most of that complexity. To build Visualforce pages using the single-page application architecture, you can use JavaScript frameworks, such as <a class="xref" href="https://angularjs.org/" target="_blank" title="html (New Window)">AngularJS</a> and <a class="xref" href="http://facebook.github.io/react/" target="_blank" title="html (New Window)">React</a>.</p> 
          <p class="p">These frameworks are just a couple of the options available. Take a look at a few different options and decide which one matches the skills and preferences of your organization. To get you started we’ll take a brief look at two of them. We don’t provide a recommendation—we’re simply showing you the first few steps along a path. You’re in charge of your journey! Use one of the options presented here, or blaze your own trail. The principles you learn here will prove useful along the way.</p> 
          <div class="btb callout box guidedTour">
           <div class="inner">
            <div class="bd">
             <div class="media">
              <div class="mediaBd">
               <h4 class="mbs">Beyond the Basics</h4>
               <p>One other option you should at least consider is the Lightning Component framework. The Lightning Component framework is a modern UI framework for developing dynamic single-page applications. This is a complete digression from Visualforce, the topic at hand, so we won’t cover the details here. See the unit in the <a class="xref" href="https://trailhead.salesforce.com/module/lex_dev_overview" target="_blank" title="html (New Window)">Lightning Experience Development</a> module for an in-depth comparison and points of consideration.</p>

         <h2 class="helpHead2" id="Tdxn4tBK-heading5"><span class="ph" id="topic-title"><a name="topic-title"></a>Use AngularJS and SLDS to Create a Mobile-Friendly Visualforce Page</span></h2> 
         <div class="body taskbody">
          <div class="shortdesc">
           Let’s use AngularJS, a JavaScript framework, and the Salesforce Lightning Design System (SLDS), a Salesforce CSS framework, to create an interactive business card template for mobile.
          </div> 
          <ol class="ol steps enumList"> 
           <li class="li step"> <span class="ph cmd">Open the Developer Console and click <span class="ph menucascade"><strong class="ph uicontrol">File</strong> | <strong class="ph uicontrol">New</strong> | <strong class="ph uicontrol">Visualforce Page</strong></span>. Enter <kbd class="ph userinput">MobileFrameworks</kbd> for the page name.</span> </li> 
           <li class="li step"> <span class="ph cmd">In the editor, replace the contents with the following.</span> 
            <div class="itemgroup info"> 
             <div class="codeblock">
              <pre>&lt;apex:page showHeader="false" standardStylesheets="false" sidebar="false"
      applyHtmlTag="false" applyBodyTag="false" docType="html-5.0"&gt;

  &lt;html xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" ng-app="MobileFrameworksApp"&gt;
    &lt;head&gt;
      &lt;meta charset="utf-8" /&gt;
      &lt;meta http-equiv="x-ua-compatible" content="ie=edge" /&gt;
      &lt;title&gt;SLDS Visualforce Page in Salesforce1&lt;/title&gt;
      &lt;meta name="viewport" content="width=device-width, initial-scale=1" /&gt;

      &lt;script src= "https://ajax.googleapis.com/ajax/libs/angularjs/1.3.14/angular.min.js"&gt;&lt;/script&gt;

      &lt;!-- Import the Design System style sheet --&gt;
        &lt;apex:slds /&gt;
    &lt;/head&gt;

    &lt;body&gt;
    &lt;!-- REQUIRED SLDS WRAPPER --&gt;
      &lt;div class="slds-scope"&gt;
                            
      &lt;!-- PRIMARY CONTENT WRAPPER --&gt;

      &lt;!-- / PRIMARY CONTENT WRAPPER --&gt;

    &lt;/div&gt;
    &lt;!-- / REQUIRED SLDS WRAPPER --&gt;

    &lt;!-- JAVASCRIPT --&gt;
          &lt;!-- / JAVASCRIPT --&gt;
    &lt;/body&gt;
  &lt;/html&gt;
&lt;/apex:page&gt;</pre>
             </div> 
            </div> 
            <div class="itemgroup stepresult">
             This code is a simple example of using SLDS and AngularJS. A lot of the SLDS markup is similar to what we’ve written in previous units. Let’s take a closer look at the new AngularJS elements.
             <ul class="ul bulletList"> 
              <li class="li">The <samp>&lt;script&gt; src</samp> attribute points to the external AngularJS file.</li> 
              <li class="li">The <samp>ng-app</samp> directive tells AngularJS that “MobileFrameworksApp” is the root element of the AngularJS application. All AngularJS applications need a root element.</li> 
             </ul> 
            </div> </li> 
           <li class="li step"> <span class="ph cmd">Now let’s add the code that creates the input fields and the virtual business card under the <samp>&lt;!-- PRIMARY CONTENT WRAPPER --&gt;</samp> tag.</span> 
            <div class="itemgroup info"> 
             <div class="codeblock">
              <pre>  &lt;div class="myapp"&gt;
&lt;!-- HEADING --&gt;
  &lt;div class="slds-text-heading_small slds-p-top_medium slds-p-bottom_medium"&gt;Virtual Business Card Template&lt;/div&gt;
&lt;!-- / HEADING --&gt;

&lt;!-- INPUT FIELDS --&gt;
  &lt;div class="slds-col slds-size--1-of-2 slds-small-size--1-of-6 slds-medium-size--1-of-6" &gt;
    &lt;input type="text" ng-model="yourName" placeholder="Name:" class="slds-input"/&gt;
    &lt;input type="text" ng-model="yourCompany" placeholder="Company Name:" class="slds-input"/&gt;
    &lt;input type="tel" ng-model="yourNumber" placeholder="Phone Number:" class="slds-input"/&gt;
    &lt;input type="email" ng-model="yourEmail" placeholder="Email:" class="slds-input"/&gt;
  &lt;/div&gt;
&lt;!-- / INPUT FIELDS --&gt;

&lt;!-- BUSINESS CARD --&gt;
  &lt;div class="slds-col slds-size--1-of-1 slds-small-size--1-of-3 slds-medium-size--1-of-3" &gt;
    &lt;div class="slds-box slds-m-top_x-large"&gt;

      &lt;div class="slds-text-heading_large slds-text-align_center"&gt;{{ yourName }}&lt;/div&gt;
      &lt;div class="slds-text-heading--medium slds-text-align_center"&gt;{{ yourCompany }}&lt;/div&gt;

      &lt;!-- ICON --&gt;
        &lt;div class="slds-align_absolute-center"&gt;
          &lt;span class="slds-icon_container slds-icon-standard-avatar
                       slds-icon-align_center slds-m-bottom_small slds-m-top_small"&gt;
            &lt;svg aria-hidden="true" class="slds-icon"&gt;
              &lt;use xlink:href="{!URLFOR($Asset.SLDS, 'assets/icons/standard-sprite/svg/symbols.svg#avatar')}"&gt;&lt;/use&gt;
            &lt;/svg&gt;
          &lt;/span&gt;
        &lt;/div&gt;                    
      &lt;!-- / ICON --&gt;

      &lt;div class="slds-text-body_regular slds-text-align_center"&gt;{{ yourNumber }}&lt;/div&gt;
      &lt;div class="slds-text-body_regular slds-text-align_center"&gt;{{ yourEmail }}&lt;/div&gt;
    &lt;/div&gt;
  &lt;/div&gt;

&lt;!-- / BUSINESS CARD --&gt;
&lt;/div&gt;    </pre>
             </div> 
            </div> 
            <div class="itemgroup stepresult"> 
             <ul class="ul bulletList"> 
              <li class="li">The <samp>ng-model</samp> directive lets you bind the value of your input field to a variable.</li> 
              <li class="li">AngularJS displays your content from the model using the <samp>{{ yourInput }}</samp> expression, where “Input” is your name, company, phone number, and email.</li> 
             </ul> 
            </div> </li> 
           <li class="li step"> <span class="ph cmd">Finally, add the following JavaScript code.</span> 
            <div class="itemgroup info"> 
             <div class="codeblock">
              <pre>&lt;script&gt;
  angular.module('MobileFrameworksApp', []);
&lt;/script&gt;</pre>
             </div> 
            </div> 
            <div class="itemgroup stepresult"> 
             <ul class="ul bulletList"> 
              <li class="li">The <samp>angular.module('MobileFrameworksApp', []);</samp> call in the JavaScript section of the markup defines the module which contains our AngularJS application. The AngularJS module is a container for functions that run when the application starts.</li> 
             </ul> 
            </div> </li> 
          </ol> 
          <div class="section result">
           <div class="p">
            Preview your Visualforce page. You’ll see that the business card is dynamically updated as you type. Now view your Visualforce page in the Salesforce1 app. Where would you add this page to Salesforce1? How would you add it? Take five minutes right now and test your memory. Can you add this page to Salesforce1 without referring to a prior unit?
