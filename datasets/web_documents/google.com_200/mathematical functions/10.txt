     <div class="post" itemprop="articleBody"> 
      <p><strong>About the atmosphere to NPM switch.</strong></p> 
      <p>We tried this too and it works upto some extent.<br>Once example is <a href="https://github.com/kadirahq/react-komposer/">react-komposer</a>. It uses <a href="https://github.com/kadirahq/react-komposer/blob/master/lib/index.js#L147">Tracker</a> and published to NPM.</p> 
      <p>I'm trying to FlowRouter's SSR API in <a href="https://github.com/kadirahq/react-mounter">react-mounter</a>.</p> 
      <p>This is just about using the global variables. We can something better. This is how I think about it.</p> 
      <ul> 
       <li>Publish a NPM package called <code>meteor-core</code> which let us access whole Meteor core API and get access to packages. It should also support stuff like <code>Meteor.startup</code>.</li> 
       <li>Meteor should support getting multiple source files for different architectures via some <code>package.json</code> configurations.</li> 
       <li>Meteor should support CSS files just like above or build a JavaScript utility to get them. (Simply allow us to import .css / .scss / .less files and Meteor will pick them)</li> 
      </ul> 
      <p>Until above these things are getting completed, I don't think it's 100% safe to use NPM as a Atmosphere replacement.</p> 
      <p><strong>But for libraries, NPM should be the way to go.</strong></p>
