     <h1 id="introduction">Introduction</h1> 
     <div class="annotation-block"> 
      <div class="annotation-block--left"> 
       <p> API Reference pages provide an easy way for you to see the methods that are provided by a service and how to call them. This column provides general information and explains the parameters that are required by methods when called from different languages. Its content changes based on the selected langauge. On screens that are wide enough, the right column provides selectable tabs that show how to make sample method calls in REST (via the cURL command), Node.js, and Java, and includes example responses to those calls. On narrow screens, these examples are inlined, and you can access the page navigation and switch between REST, Node.js, and Java examples from the menu at the left. </p> 
      </div> 
      <div class="annotation-block--right"> 
       <div class="method-curl"> 
        <p class="code-header">API Endpoint</p> 
        <pre><code class="nohighlight">https://gateway.watsonplatform.net/language-translator/api</code></pre> 
        <p class="code-header">Important: If you have Bluemix Dedicated, this may not be your service endpoint. Double check your endpoint URL on the Service Credentials page in your instance of the Language Translator service on Bluemix.</p> 
       </div> 
       <div class="method-node"> 
        <p class="code-header">The code examples on this tab use the client-side library that is provided for Node.js, rather than issuing REST calls.</p>

     <h1 id="apiexplorer">API explorer</h1> 
     <div class="annotation-block"> 
      <div class="annotation-block--left"> 
       <p> To interact with this API, use the Language Translator Service <a target="_blank" href="https://watson-api-explorer.mybluemix.net/apis/language-translator-v2">API explorer</a>. Use the explorer to test your calls to the API, and to view live responses from the server. </p>

     <h1 id="authentication">Authentication</h1> 
     <div class="annotation-block"> 
      <div class="annotation-block--left"> 
       <p> You authenticate to the Language Translator API by providing the username and password that are provided in the service credentials for the service instance that you want to use. The API uses Basic Authentication. </p> 
       <p> After creating an instance of the Language Translator service, select <b>Service Credentials</b> from the left navigation for its dashboard to see the username and password that are associated with that instance. </p> 
       <aside class="notice"> 
        <b>Note</b>: Service credentials ("username" and "password") are different from your Bluemix account username and password. 
       </aside> 
      </div> 
      <div class="annotation-block--right"> 
       <p class="code-header">Replace <code>{username}</code> and <code>{password}</code> with your credentials</p> 
       <pre class="method-curl"><code class="lang-bash">
curl -u "{username}":"{password}" \
"https://gateway.watsonplatform.net/language-translator/api/v2"
</code></pre> 
       <pre class="method-node"><code class="lang-javascript">
var watson = require('watson-developer-cloud');
var language_translator = watson.language_translator({
  username: '{username}',
  password: '{password}',
  version: 'v2'
});
</code></pre> 
       <pre class="method-java"><code class="lang-java">
LanguageTranslator service = new LanguageTranslator();
service.setUsernameAndPassword("{username}","{password}");
</code></pre> 
       <pre class="method-python"><code class="lang-python">
from watson_developer_cloud import LanguageTranslatorV2 as LanguageTranslator

language_translator = LanguageTranslator(
    username='{username}',
    password='{password}')
</code></pre> 
      </div>

     <h1 id="methods">Methods</h1> 
     <h2 id="translate">Translate</h2> 
     <p>Translates input text from the source language to the target language.</p> 
     <p class="method-curl">Use <code class="rest-verb">GET /v2/translate</code> for smaller request data sizes and for URL query parameters, and use <code class="rest-verb">POST /v2/translate</code> for larger request data sizes and for JSON requests.</p> 
     <pre class="method-curl"><code class="nohighlight">
GET  /v2/translate
POST  /v2/translate
</code></pre> 
     <h3 id="request-translate">Request</h3> 
     <!-- request curl --> 
     <div class="annotation-block method-curl"> 
      <div class="annotation-block--left"> 
       <table class="align-right-1 unwrap-1 unwrap-2"> 
        <tbody>
         <tr> 
          <th>Parameter</th> 
          <th>Type</th> 
          <th>Description</th> 
         </tr> 
         <tr> 
          <td>model_id</td> 
          <td>query</td> 
          <td>The unique model_id of the translation model used to translate text. The model_id inherently specifies source, target language, and domain. If the model_id is specified, there is no need for the source and target parameters, and the values will be ignored.</td> 
         </tr> 
         <tr> 
          <td>source</td> 
          <td>query</td> 
          <td>Used in combination with target as an alternative way to select the model for translation. When target and source are set, and model_id is not set, the system choose a default model with the right language pair to translate (usually the model based on the news domain).</td> 
         </tr> 
         <tr> 
          <td>target</td> 
          <td>query</td> 
          <td>Translation target language in 2 or 5 letter language code. Should use 2 letter codes except for when clarifying between multiple supported languages. When model_id is used directly, it will override the source-target language combination. Also, when a 2 letter language code is used, and no suitable default is found (such as “zh”), it returns an error.</td> 
         </tr> 
         <tr> 
          <td>text</td> 
          <td>query</td> 
          <td>(<strong>Required</strong>) Input text in UTF-8 encoding. Multiple text query parameters indicate multiple input paragraphs, and a single string is valid input. </td> 
         </tr> 
         <tr> 
          <td>accept</td> 
          <td>header</td> 
          <td>Describes the format of the return values. Valid values are “text/plain”, or “application/json”. When left undefined, empty, or '*/*', the default is "text/plain".</td> 
         </tr> 
        </tbody>
       </table> 
      </div> 
      <!--request code--> 
      <div class="annotation-block--right"> 
       <p class="code-header">Example GET request</p> 
       <pre><code class="lang-bash">
curl -u "{username}":"{password}" \
"https://gateway.watsonplatform.net/language-translator/api/v2/translate?source=en&amp;target=es&amp;text=hello"
</code></pre> 
       <p class="code-header">Example POST request</p> 
       <pre><code class="lang-bash">
curl -u "{username}":"{password}" \
-X POST \
-H "Content-Type: application/json" \
-H "Accept: application/json" \
-d '{"text":"Hello","source":"en","target":"es"}' \
"https://gateway.watsonplatform.net/language-translator/api/v2/translate"
</code></pre> 
      </div> 
     </div>

     <h2 id="create-model">Create model</h2> 
     <p>Uploads a TMX glossary file on top of a domain to customize a translation model.</p> 
     <p>Depending on the size of the file, training can range from minutes for a glossary to several hours for a large parallel corpus. Glossary files must be less than 10 MB. The cumulative file size of all uploaded glossary and corpus files is limited to 250 MB.</p> 
     <pre class="method-curl"><code class="nohighlight">
POST  /v2/models
</code></pre> 
     <h3 id="request-create-model">Request</h3> 
     <!-- request curl --> 
     <div class="annotation-block">

          <td>query</td> 
          <td>(<strong>Required</strong>). Specifies the domain model that is used as the base for the training. <span class="method-curl"> To see current supported domain models, use the GET /v2/models method.</span></td> 
         </tr>

          <td>name</td> 
          <td>query</td> 
          <td>The model name. Valid characters are letters, numbers, -, and _. No spaces.</td>

          <td>file</td>

          <td>monolingual_corpus<br></td>
