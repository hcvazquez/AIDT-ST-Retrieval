   <h2 class="ui dividing header">The versioning system</h2> 
   <p> <strong class="ui font red big">X</strong>.x.x Major and incompatible changes<br> <small>Upgrading version will most likely break your current installation. Read the version notes to know what have changed between the 2 major versions to upgrade your project.</small> </p> 
   <p> x.<strong class="ui font red big">X</strong>.x Plugin upgrades and additional functionalities<br> <small>Upgrading version will not require upgrades on your forms as it contains compatibility functions to handle the previous behaviors.</small> </p> 
   <p> x.x.<strong class="ui font red big">X</strong> - Minor changes, patches and hotfixes <strong>Keeping up to date this version is strongly recommended</strong><br> <small>Upgrading version will most likely resolve your current issues.</small> </p> 
   <!-- Version 1.5.3 --> 
   <h2 class="ui dividing header">Version 1.5.3 <a class="ui salmon tiny label">Fixes</a> <time datetime="2015-06-14">June 14, 2015</time></h2> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li> Fixed possible <code>TypeError: Converting circular structure to JSON</code> error when used inside Angular or Meteor projects </li> 
    <li> Fixed Error display if there was a list inside the form and display set to "block" </li> 
   </ul> 
   <p>Notes</p> 
   <ul class="ui list"> 
    <li>onSubmit callback now return formData as the second param</li> 
   </ul> 
   <!-- Version 1.5.2 --> 
   <h2 class="ui dividing header">Version 1.5.2 <a class="ui salmon tiny label">Fixes</a> <time datetime="2015-02-18">February 18, 2015</time></h2> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li> Fixed the <code>NOTEMPTY</code> rule to contain at least a non-space character </li> 
    <li> Added <code>NAME</code> rule to match first and last names, <a href="/jqueryvalidation/documentation/#rule-name">see documentation</a>. </li> 
    <li> Inputs with the attribute <code>[readonly]</code> are not validated anymore since the value cannot be changed </li> 
   </ul> 
   <p>Notes</p> 
   <ul class="ui list"> 
    <li>Updated <a href="/jqueryvalidation/demo/#form-contact_v1">Contact V1</a> form to use NAME Validation rule</li> 
   </ul> 
   <!-- Version 1.5.0 --> 
   <h2 class="ui dividing header">Version 1.5.0 <a class="ui green tiny label">Features</a> <time datetime="2015-02-05">February 5, 2015</time></h2> 
   <p>New Features</p> 
   <ul class="ui list"> 
    <li> Added $.alterValidationRules method to modify the Global Validation rules - <a href="/jqueryvalidation/documentation/#api-altervalidationrules">Documentation</a> - <a href="/jqueryvalidation/demo/#form-signin_v4">Demo</a> </li> 
    <li>Modified NUMERIC rule (to match ex: 1,000.00 or 1000)</li> 
    <li>Added INTEGER rule</li> 
    <li>Added data-validation-regex-reverse attribute to reverse the data-validation-regex.</li> 
   </ul> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li> <code>data-validation-regex</code> is not reversed anymore by default, an error will be triggered if the input value does not match the regex. </li> 
    <li> <code>options.submit.settings.button</code> default value is changed to <code>[type="submit"]</code> </li> 
    <li> Fixed <code>NOSPACE</code> regex </li> 
   </ul> 
   <p>Notes</p> 
   <ul class="ui list"> 
    <li>Make sure the node is a <code>&lt;form&gt;</code> or the Validation is dropped</li> 
    <li>Removed <code>window.Validation.message</code> configuration, use <code>$.alterValidationRules</code> instead</li> 
   </ul> 
   <!-- Version 1.4.0 --> 
   <h2 class="ui dividing header">Version 1.4.0 <a class="ui green tiny label">Features</a> <time datetime="2014-09-28">September 28, 2014</time></h2> 
   <p>New Features</p> 
   <ul class="ui list"> 
    <li> Added <code>destroy</code> method. - <a href="/jqueryvalidation/documentation/#api-destroy">Documentation</a> - <a href="/jqueryvalidation/demo/#form-signup_v3">Demo</a> </li> 
   </ul> 
   <p>Note</p> 
   <ul class="ui list"> 
    <li> <code>node</code> now refers to a jQuery object instead of the DOM object. </li> 
   </ul>

    <li> Added /demo/ files inside the project </li> 
   </ul> 
   <!-- Version 1.3.3 --> 
   <h2 class="ui dividing header">Version 1.3.3 <a class="ui salmon tiny label">Fixes</a> <time datetime="2014-08-16">August 16, 2014</time></h2> 
   <ul class="ui list"> 
    <li> Fixed issue where the input name was forced into the message. You are now able to override the displayed name with <code>data-validation-label</code> or <code>window.Validation.labels</code>. - <a href="/jqueryvalidation/documentation/#validation-labels">Configuration Documentation</a> - <a href="/jqueryvalidation/documentation/#data-validation-label">Attribute Documentation</a> - <a href="/jqueryvalidation/demo/#form-contact_v1">Demo</a> </li> 
   </ul> 
   <!-- Version 1.3.2 --> 
   <h2 class="ui dividing header">Version 1.3.2 <a class="ui salmon tiny label">Fixes</a> <time datetime="2014-08-11">August 11, 2014</time></h2> 
   <ul class="ui list"> 
    <li> Fixed some debug cases. </li> 
    <li> Removed the <code>Debug</code> code from the minified javascript file <code>jquery.validation.min</code> to reduce it's size by ~25% for production environment. </li> 
   </ul> 
   <!-- Version 1.3.1 --> 
   <h2 class="ui dividing header">Version 1.3.1 <a class="ui salmon tiny label">Fixes</a> <time datetime="2014-07-31">July 31, 2014</time></h2> 
   <ul class="ui list"> 
    <li> Replaced the API <code>window.debug</code> to <code>window.Debug</code> to unify the debug process. </li> 
   </ul> 
   <!-- Version 1.3.0 --> 
   <h2 class="ui dividing header">Version 1.3.0 <a class="ui green tiny label">Features</a> <time datetime="2014-07-25">July 25, 2014</time></h2> 
   <p>New Features</p> 
   <ul class="ui list"> 
    <li> Added <code>debug</code> configuration. Set debug to <strong>true</strong> in development to drill down issues. - <a href="/jqueryvalidation/documentation/#debug">Documentation</a> - <a href="/jqueryvalidation/demo/#form-debug_v1">Demo</a> </li> 
   </ul> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li> Fixed issue when Array was prototyped (outside of this project), it would cause the form to never be able to submit. </li> 
    <li> Fixed JavaScript error when the comparison rule using "V" was used on a string value. Ex: <code>input data-validation="[V&gt;=10]" value="This is a string"</code> </li> 
    <li> Fixed issue when using a dynamic form sometimes when submitting 2 <code>error-list</code> would appear. </li> 
   </ul> 
   <!-- Version 1.2.0 --> 
   <h2 class="ui dividing header">Version 1.2.0 <a class="ui green tiny label">Features</a> <time datetime="2014-05-08">May 08, 2014</time></h2> 
   <p>New Features</p> 
   <ul class="ui list"> 
    <li> Added <code>keyCode</code> parameter inside the dynamic callback. The key that was pressed is now returned as the 3rd parameter. - <a href="/jqueryvalidation/documentation/#dynamic-callbacks">Documentation</a> - <a href="/jqueryvalidation/demo/#form-signup_v3">Demo</a> </li> 
    <li> <code>dynamic.settings.trigger</code> now have jQuery eventType <code>change</code> and <code>paste</code> added by default if the trigger is not <code>focusout</code>. This solves the issue of onchange selects or pasted data inside inputs not triggering the validation. - <a href="/jqueryvalidation/documentation/#dynamic-settings-trigger">Documentation</a> </li> 
    <li> Added the possibility to <code>$.addError</code> and <code>$.removeError</code> on desired input(s). - <a href="/jqueryvalidation/documentation/#adderror">Documentation</a> - <a href="/jqueryvalidation/demo/#form-signin_v2">Demo</a> </li> 
   </ul> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li> Namespaced the jQuery event binding with <code>.vd</code> and <code>.vr</code> so it will not conflicts / override other bindings anymore. </li> 
    <li> Fixed some rare case where an error would be displayed twice for one inputName. </li> 
   </ul> 
   <p>Notes</p> 
   <ul class="ui list"> 
    <li>Added <code>window.Validation</code> object definition.</li> 
   </ul> 
   <!-- Version 1.1.1 --> 
   <h2 class="ui dividing header">Version 1.1.1 <a class="ui salmon tiny label">Fixes</a> <time datetime="2014-04-28">April 28, 2014</time></h2> 
   <ul class="ui list"> 
    <li> Removed the dependency over <code>$.scrollTo</code> plugin for <code>scrollToError</code> option. The option now accepts an object of <strong>offset</strong> and <strong>duration</strong>. - <a href="/jqueryvalidation/demo/#form-signin_v2">Demo</a> </li> 
   </ul> 
   <!-- Version 1.1.0 --> 
   <h2 class="ui dividing header">Version 1.1.0 <a class="ui green tiny label">Features</a> <time datetime="2014-04-24">April 24, 2014</time></h2> 
   <p>New Features</p> 
   <ul class="ui list"> 
    <li> Added support for <code>$.creditCardValidator</code> plugin to validate credit card input - <a href="https://github.com/flesler/jquery.scrollTo" target="_blank">Download</a> - <a href="http://jquerycreditcardvalidator.com/" target="_blank">Documentation</a> - <a href="/jqueryvalidation/demo/#form-purchase_v1">Demo</a> </li> 
    <li> <code>submit.settings.scrollToError</code> option has been added to scroll to the first error, even on page reload. <strong>Requires</strong> <code>$.ScrollTo</code> - <a href="https://github.com/flesler/jquery.scrollTo" target="_blank">Download</a> - <a href="/jqueryvalidation/demo/#form-signin_v2">Demo</a> </li> 
    <li> Ability to override the global Validation <code>_messages</code> or only override the <code></code> instance - <a href="/jqueryvalidation/demo/#form-contact_v1">Demo</a> </li> 
   </ul> 
   <p>Fixes</p> 
   <ul class="ui list"> 
    <li>Fixed issue when <code>dynamic.settings.trigger: keypress</code> would get conflicted and stop working on specific cases</li> 
    <li>Fixed issue when <code>[disabled]</code> fields wouldn't get validated if the attribute was removed.</li> 
   </ul> 
   <p>Notes</p> 
   <ul class="ui list"> 
    <li><code>submit.settings.display: block</code> doesn't scroll to error by default anymore, use the new option <code>submit.setting.scrollToError</code> instead</li> 
   </ul>
