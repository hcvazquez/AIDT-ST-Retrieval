         <h1 class="h2">Converting Your JavaScript App to an Android App w/ PhoneGap</h1>
         <ul class="list-inline"> 
          <li>By <a href="/blog/author/tomomi/" title="Posts by Tomomi Imura" rel="author">Tomomi Imura</a></li> 
          <li>Follow <a href="http://twitter.com/" target="_blank">@</a></li>

        <div class="blog-description">
         <p>So you’re here because you want to turn your JavaScript application into a mobile Android application. Luckily for you, we’ve written this tutorial just for that, using the PhoneGap framework. In addition to our PhoneGap tutorial, we’ll also walk you through a&nbsp;manual way using Cordova CLI with an Android SDK.</p>
         <p><a href="/blog/2014-10-30-converting-your-javascript-app-to-an-ios-app-w-phonegap/" target="_blank"><em>If you want to convert an iOS app, we have a tutorial for that as well.</em></a></p>
         <p>We’ve built numerous chat demos and tutorials with JavaScript,&nbsp;including&nbsp;<a href="/developers/demos/10chat/"><i>10Chat</i></a>, a simple chat app written with only 10 lines of JavaScript. These demos do work on mobile, but are not exclusively Android applications. On that note, we’ll change that.</p>
         <p style="text-align: center;"><img class="aligncenter" alt="Android PhoneGap app" src="/wp-content/uploads/2014/10/phonegap.jpg" width="1168" height="561"></p>
         <h3>Closing the Gap Between Web and Native with PhoneGap</h3>
         <p><a href="http://phonegap.com/" target="_blank">PhoneGap</a> is an open source mobile framework that enables you to create cross-platform apps that run on various mobile devices including iOS and Android. You write your web app in HTML5, JavaScript and CSS, and PhoneGap helps you to turn it into native apps, most likely Android or iOS.</p>
         <p>You can create mobile, or responsive web apps with any UI frameworks and libraries of your choice- <a href="http://jquerymobile.com/" target="_blank">jQuery Mobile</a>, <a href="http://www.sencha.com/products/touch/" target="_blank">Sencha Touch</a>, <a href="http://enyojs.com/" target="_blank">Enyo</a>, <a href="http://www.telerik.com/kendo-ui" target="_blank">Kendo UI</a>, <a href="http://onsen.io/" target="_blank">Onsen UI</a>… you name it. Or, of course, VanillaJS sans any frameworks can be your choice. The sample code used for this tutorial is <a title="sample source code" href="https://github.com/pubnub/super-simple-chat" target="_blank">available on Github</a>, and this demo has very clean interface with minimal CSS and no images, and using no extra UI libraries.</p>
         <p>There are two ways to make your web app into an Android app using PhoneGap. One is a no-pain ultra easy way, and another way is a manual way using Cordova CLI with an Android SDK.</p>
         <h3>Option 1: Seamless Android Apps with PhoneGap Build</h3>
         <p>Using PhoneGap Build is absolutely painless. All you need to do is zip up your web app, upload it to Adobe PhoneGap Build cloud, and the service takes care everything for you. You don’t even need to download and set up SDK or emulators.</p>
         <ol>
          <li>Write a web app</li>
          <li>Zip it up</li>
          <li>Upload the zip file to <a href="https://build.phonegap.com/apps" target="_blank">PhoneGap Build</a> cloud. Click <em>Ready to Build</em>. Edit the name of the app and upload an app icon too.</li>
          <li>When ready, scan the QR code with your Android phone, and follow the link</li>
          <li>Download the <em>apk</em> and install it on your phone</li>
          <li>Woo-hoo!</li>
          <li>(…Profit!)</li>
         </ol>
         <p>The overview of the whole process in 6 seconds!</p>
         <div align="center">
          <iframe src="https://vine.co/v/OAr6lrT90hP/embed/simple" height="480" width="480" frameborder="0"></iframe>
         </div>
         <p>&nbsp;</p>
         <h3>Option 2: Full Development with PhoneGap Cordova CLI</h3>
         <p>Right, that was easy. Or, you can install Cordova command-line interface (CLI) tools and Android SDKs for full development with taking advantage of the plugins that enable for you to use hardware APIs, push notifications and more, and debugging.</p>
         <h4>Step 1: Install Android Eclipse Bundle</h4>
         <p>First, you need to download and install <a href="http://developer.android.com/sdk/installing/index.html?pkg=adt" target="_blank">Eclipse ADT Bundle</a>. The bundle includes the Android SDK tools, Eclipse IDE with built-in Android Developer Tools.</p>
         <h4>Step 2: Set Up the Android SDK</h4>
         <p>Once everything you need is installed, open Eclipse and go to <b>Window &gt; Android SDK Manager</b>. In the dialog, select SDK tools. You probably need:</p>
         <ul>
          <li>Android SDK Tools</li>
          <li>Android SDK Platform-tools</li>
          <li>Android SDK Build-tools (API 19 to work with Cordova, as of Oct. 2014)</li>
          <li>SDK Platform</li>
          <li>ARM EABI v7a System Image</li>
         </ul>
         <img class="aligncenter size-full wp-image-8323" alt="Android-sdk-mgr" src="/wp-content/uploads/2014/10/screenshot-android-sdk-mgr-.png" width="814" height="614">
         <p>For more information on adding SDK packages, please refer, <a href="http://developer.android.com/sdk/installing/adding-packages.html" target="_blank">http://developer.android.com/sdk/installing/adding-packages.html</a> site.</p>
         <h4>Step 3: Set Up Android Emulator</h4>
         <p>Now, you need to create an Android Virtual Device (AVD). On Eclipse, go to <b>Window &gt; Android Virtual Device Manager</b>. In the AVD Manager window, at Device Definitions tab, pick an emulator and click <i>Create AVD…</i>.</p>
         <img class="aligncenter size-full wp-image-8324" alt="avd manager" src="/wp-content/uploads/2014/10/screenshot-avd-mgr-1.png" width="929" height="674">
         <img class="aligncenter size-full wp-image-8325" alt="avd manager" src="/wp-content/uploads/2014/10/screenshot-avd-mgr-2.png" width="1109" height="827">
         <h4>Step 4: Install Cordova CLI</h4>
         <p>Installation of the command line interface requires node.js. If you have not, you must <a href="http://nodejs.org/">install it</a> first.</p>
         <p>Install cordova with npm:</p>
         <p><code>$ sudo npm install -g cordova</code></p>
         <h4>Step 5: Create a New Project in the SDK</h4>
         <p>Create a new project by typing these commands:</p>
         <p><code>$ cordova create simple-chat com.mydomain.chat SimpleChat</code><br><code>$ cd simple-chat </code><br><code>$ cordova platform add android</code></p>
         <p>Try building it and deploying the app to an emulator, or an device:</p>
         <p><code>$ cordova build</code><br><code>$ cordova run android</code></p>
         <p>It may take a while to launch the emulator, but be patient. Once the emulator is up and running, you should see this Cordova splash screen!</p>
         <img class="aligncenter size-full wp-image-8326" alt="screenshot-cordova-android" src="/wp-content/uploads/2014/10/screenshot-cordova-android.png" width="400" height="573">
         <h4>Step 6: Port Your Web App</h4>
         <p>In your project folder, you should find a <code>/www/</code> directory. This is where your web app goes.</p>
         <img class="aligncenter size-full wp-image-8328" alt="screenshot-cordova-dir" src="/wp-content/uploads/2014/10/screenshot-cordova-dir1.png" width="877" height="369">
         <p>Your <code>index.html</code> body content goes into the body of <code>/www/index.html</code>. Do not delete or replace head section unless you know what you are doing.</p>
         <p>And <code>/js/index.js</code> includes functions that are required for PhoneGap. You just need to call your app functions in <code>onDeviceReady()</code>.</p>
         <script src="https://gist.github.com/3ae18a6a95359cbe0152.js?file=index.js"></script>
         <p>Also, you include all your css files and images under <code>/www/</code>.</p>
         <p>The application icons can be anywhere, you just need to specify the location in your config.xml file.</p>
         <script src="https://gist.github.com/3ae18a6a95359cbe0152.js?file=config.xml"></script>
         <p style="text-align: center;"><img class="aligncenter" alt="screenshot-android-icon" src="/wp-content/uploads/2014/10/screenshot-android-icon.png" width="400" height="246"></p>
         <p>Deploy to an emulator or device.</p>
         <p><code>$ cordova run android</code></p>
         <p>Now you have your own chat app for Android! Super!</p>
         <h3>Remote Debugging with Chrome</h3>
         <p>When you are working on Android apps, you can easily remote-debug on Chrome.</p>
         <ol>
          <li>Enable USB debugging mode on your Android device by going to <b>Settings &gt; Developer Options</b>, then turn on <b>USB debugging option</b>.</li>
          <li>On your desktop Chrome browser, go to <b>chrome://inspect</b>.</li>
          <li>Connect the device and your computer with a USB cable</li>
          <li>You should see your device name (in this screenshot below, <em>XT1032</em> is the name of Moto G that I am testing with), also your app on Chrome screen</li>
          <li>Click “inspect”</li>
         </ol>
         <img class="aligncenter size-full wp-image-8342" alt="screenshot-remote-debug" src="/wp-content/uploads/2014/10/screenshot-remote-debug.png" width="716" height="509">
         <p>Once you click “inspect”, &nbsp;you should see the familiar DevTools window for debugging! If you have a trouble with the tools, go to <a href="https://developer.chrome.com/devtools/docs/remote-debugging" target="_blank">Google Chrome Developer</a> for more info.</p>
         <p>I hope you enjoyed this tutorial. In a week or two, we’ll be releasing the this tutorial for iOS and PhoneGap!</p>
