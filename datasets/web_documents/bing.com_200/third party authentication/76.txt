     <a name="End_of_life"></a>
     <h2>End of life</h2> 
     <p> The Commons HttpClient project is now end of life, and is no longer being developed. It has been replaced by the <a href="../">Apache HttpComponents</a> project in its <a href="../httpcomponents-client-ga">HttpClient</a> and <a href="../httpcomponents-core-ga/">HttpCore</a> modules, which offer better performance and more flexibility. </p> 
    </div>
    <div class="section">
     <a name="Introduction"></a>
     <h2>Introduction</h2> 
     <p> The Hyper-Text Transfer Protocol (HTTP) is perhaps the most significant protocol used on the Internet today. Web services, network-enabled appliances and the growth of network computing continue to expand the role of the HTTP protocol beyond user-driven web browsers, while increasing the number of applications that require HTTP support. </p> 
     <p> Although the <tt>java.net</tt> package provides basic functionality for accessing resources via HTTP, it doesn't provide the full flexibility or functionality needed by many applications. The Jakarta Commons <em>HttpClient</em> component seeks to fill this void by providing an efficient, up-to-date, and feature-rich package implementing the client side of the most recent HTTP standards and recommendations. See the <a href="features.html">Features</a> page for more details on standards compliance and capabilities. </p> 
     <p> Designed for extension while providing robust support for the base HTTP protocol, the <em>HttpClient</em> component may be of interest to anyone building HTTP-aware client applications such as web browsers, web service clients, or systems that leverage or extend the HTTP protocol for distributed communication. </p> 
     <p> There are many projects that use <em>HttpClient</em> to provide the core HTTP functionality. Some of these are open source with project pages you can find on the web while others are closed source that you would never see or hear about. The Apache Source License provides maximum flexibility for source and binary reuse. Please see the <a href="http://wiki.apache.org/jakarta-httpclient/HttpClientPowered" class="externalLink" title="External Link"> Applications</a> page for projects using <em>HttpClient</em>. </p> 
    </div>
    <div class="section">
     <a name="History"></a>
     <h2>History</h2> 
     <p> <em>HttpClient</em> was started in 2001 as a subproject of the Jakarta Commons, based on code developed by the <a href="http://jakarta.apache.org/slide/" class="externalLink" title="External Link">Jakarta Slide</a> project. It was promoted out of the Commons in 2004, graduating to a separate Jakarta project. In 2005, the HttpComponents project at Jakarta was created, with the task of developing a successor to <em>HttpClient 3.x</em> and to maintain the existing codebase until the new one is ready to take over. The <a href="http://commons.apache.org/" class="externalLink" title="External Link">Commons</a> project, cradle of <em>HttpClient</em>, left <a href="http://jakarta.apache.org/" class="externalLink" title="External Link">Jakarta</a> in 2007 to become an independent Top Level Project. Later in the same year, the <a href="http://httpcomponents.apache.org/" class="externalLink" title="External Link">HttpComponents</a> project also left Jakarta to become an independent Top Level Project, taking the responsibility for maintaining <em>HttpClient</em> with it. </p>
