        <article id="article-content" itemscope itemtype="http://schema.org/BlogPosting">
         <p>Useful for those who love to code, you may need a plugin like this to highlight any code example on your website or blog. The following collection of syntax highlighter plugins contains only those that offers the best features, easy initialization&nbsp;and&nbsp;themables. None of them requires jQuery to work as they are independent and uses only VanillaJS.</p> 
         <h2>5. <a href="http://shjs.sourceforge.net/" target="_blank" rel="noopener noreferrer">SHJS</a></h2> 
         <p>SHJS is a JavaScript script&nbsp;which highlights source code passages in HTML documents. Documents using SHJS are highlighted on the client side by the web browser. SHJS uses language definitions from GNU Source-highlight. This gives SHJS the ability to highlight source code written in many different languages.&nbsp;SHJS should work in any browser that conforms to the HTML 4, ECMA-262, and DOM Level 2 standards (I know, too old but provides a fallback for too old browsers and that can come in handy).</p> 
         <p>To use SHJS, you need at least 3 files:</p> 
         <ul> 
          <li>The main script or the minified version</li> 
          <li>A language definition file</li> 
          <li>A stylesheet</li> 
         </ul> 
         <p>This plugin is of the old school, therefore you need to load a .js file with the language definition that you want to highlight. However, you can wrap them all in just one file.</p> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-57303672f13e1.png" alt="" class="img img-thumbnail img-responsive center-block"></p> 
         <h2>4. <a href="http://alexgorbatchev.com/SyntaxHighlighter/" target="_blank" rel="noopener noreferrer">SyntaxHighlighter</a></h2> 
         <p>SyntaxHighlighter is a fully functional self-contained,&nbsp;open source client side code syntax highlighter developed in JavaScript.</p> 
         <p>To get SyntaxHighlighter to work on you page, you need to do the following:</p> 
         <ul> 
          <li>Following the Building instructions to assemble your own syntaxhighlighter.js</li> 
          <li>Drop it on the page using a <code>&lt;script src="syntaxhighlighter.js" /&gt;</code> tag or follow the CommonJS usage instructions</li> 
         </ul> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-573039af6aa6c.png" alt="" class="img img-thumbnail img-responsive center-block"></p> 
         <p>To initialize javascript code, you can use :</p> 
         <pre class="language-html"><code>&lt;script type="text/javascript" src="syntaxhighlighter.js"&gt;&lt;/script&gt;

&lt;pre class="brush: js"&gt;
function foo()
{
}
&lt;/pre&gt;</code></pre> 
         <p>Unfortunately SyntaxHighlighter doesn't support any standard, therefore your markup will be highlighted by this plugin and is not so maintainable.</p> 
         <h2>3. <a href="https://craig.is/making/rainbows" target="_blank" rel="noopener noreferrer">Rainbow</a></h2> 
         <p>Rainbow is a code syntax highlighting library written in Javascript. It was designed to be lightweight (1.4kb), easy to use, and extendable. It is completely themable via CSS.&nbsp;Rainbow on its own is very simple. It goes through code blocks, processes regex patterns, and wraps matching patterns in <code>&lt;span&gt;</code> tags. All the theming is left up to CSS.&nbsp;</p> 
         <p>In your markup the data-language attribute is used to specify what language to use for highlighting:</p> 
         <pre><code class="lang-html">&lt;pre&gt;&lt;code data-language="javascript"&gt;var testing = true;&lt;/code&gt;&lt;/pre&gt;</code></pre> 
         <h2><img src="https://ourcodeworld.com/public-media/gallery/gallery-57303bc73d2c9.png" alt="" class="img img-thumbnail img-responsive center-block"></h2> 
         <p>Use the Rainbow.color method to highlight the code :</p> 
         <pre><code class="lang-javascript">Rainbow.color();</code></pre> 
         <h2>2. <a href="https://highlightjs.org/" target="_blank" rel="noopener noreferrer">highlight.js</a></h2> 
         <p>Highlight is a super DOM code highlighter. It tries to detect the language automatically. If automatic detection doesn’t work for you, you can specify the language in the class attribute:</p> 
         <ul id="features" class="ui"> 
          <li><a href="https://highlightjs.org/static/demo/">152 languages and 72 styles</a></li> 
          <li>automatic language detection</li> 
          <li>multi-language code highlighting</li> 
          <li>available for node.js</li> 
          <li>works with any markup</li> 
          <li>compatible with any js framework</li> 
         </ul> 
         <p>To use highlight.js in your project, just include the css (1) and js (1) files and initialize with the&nbsp;<code>initHighlightingOnLoad()</code> method:</p> 
         <pre><code class="lang-html">&lt;!-- Include library --&gt;
&lt;link rel="stylesheet" href="/path/to/styles/default.css"&gt;
&lt;script src="/path/to/highlight.pack.js"&gt;&lt;/script&gt;
&lt;!-- Initialize highlight --&gt;
&lt;script&gt;hljs.initHighlightingOnLoad();&lt;/script&gt;</code></pre> 
         <p>The required markup should look like :</p> 
         <pre><code class="lang-html">&lt;pre&gt;&lt;code class="html"&gt;Some escaped code to highlight here&lt;/code&gt;&lt;/pre&gt;</code></pre> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-573032b9e9538.png" alt="" class="img img-thumbnail img-responsive center-block"></p> 
         <h2>1. <a href="http://prismjs.com/" target="_blank" rel="nofollow noopener noreferrer">Prism.js</a></h2> 
         <p>Prism is a lightweight, extensible syntax highlighter, built with modern web standards in mind. It’s used in thousands of websites, including some of those you visit daily.</p> 
         <p>All styling is done through CSS, with sensible class names like <code>.comment</code>, <code>.string</code>, <code>.property</code> etc. You can choose one of the seven available themes. Prism doesn't support&nbsp;IE 6-8.</p> 
         <ul> 
          <li>Only 2KB minified &amp; gzipped (core). Each language definition adds roughly 300-500 bytes.</li> 
          <li>Encourages good author practices. Other highlighters encourage or even force you to use elements that are semantically wrong, like &lt;pre&gt; (on its own) or &lt;script&gt;. Prism forces you to use the correct element for marking up code: <code>&lt;code&gt;</code>. On its own for inline code, or inside a <code>&lt;pre&gt;</code> for blocks of code. In addition, the language is defined through the way recommended in the HTML5 draft: through a language-xxxx class.</li> 
          <li>The language definition is inherited. This means that if multiple code snippets have the same language, you can just define it once, in one of their common ancestors.</li> 
          <li>You can use <a href="http://prismjs.com/#plugins" target="_blank" rel="nofollow noopener noreferrer">plugins</a>&nbsp;like line highlight or line numbers (all optional).</li> 
          <li>Responsive</li> 
         </ul> 
         <p>To install just include the prism.css (choose the theme that you want) file and select the languages that you need to highlight in the plugin homepage (or select all) and include them in your document.</p> 
         <p>According to the <a href="http://www.w3.org/TR/html5/text-level-semantics.html#the-code-element" target="_blank" rel="nofollow noopener noreferrer">HTML5 spec</a>, the recommended way to define a code language is a language-xxxx class, which is what Prism uses. To make the things easier the highlight will be automatically triggered if the code element has already a class with an specified language i.e:</p> 
         <pre><code class="lang-html">&lt;!-- Highligh in C# mode --&gt;
&lt;pre&gt;&lt;code class="lang-csharp"&gt;Some code&lt;/code&gt;&lt;/pre&gt;

&lt;!-- This block will be not highlighted --&gt;
&lt;pre&gt;&lt;code&gt;Some code&lt;/code&gt;&lt;/pre&gt;</code></pre> 
         <p>Our Code World uses prism to highlight the code examples <i class="fa fa-heart"></i></p> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-573031e6dcbcc.png" alt="" class="img img-thumbnail img-responsive center-block"></p> 
         <h2>Extra</h2> 
         <h3><a href="https://github.com/asvd/microlight" target="_blank" rel="nofollow noopener noreferrer">Microlight.js</a></h3> 
         <p><a href="https://github.com/asvd/microlight" target="_blank" rel="nofollow noopener noreferrer">Github</a> | <a href="http://asvd.github.io/microlight/" target="_blank" rel="nofollow noopener noreferrer">Demo</a></p> 
         <p>microlight.js is an open-source micro-library which improves readability of code snippets by highlighting, for any programming language, without attaching additional language-packages or styles.</p> 
         <p>Unlike other code-highlighting solutions, <span class="logo">microlight.js</span> does not keep a set of rules for many languages. Instead it uses a general highlighting strategy providing a reasonable highlight for most of programming languages. As result:</p> 
         <ul> 
          <li>The library size is extremely compact 2.2k.</li> 
          <li>You don't need to specify which language is that and you don't need to bother users with this as well.</li> 
         </ul> 
         <p>In fact, microlight.js does not care about what language the code is written in, nor does it care about the code structure. It just goes through the code and highlights it.</p> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-574e8597d8e81.png" alt="Microlight example" class="img img-thumbnail img-responsive center-block"></p> 
         <h3><a href="http://enlighterjs.org/" target="_blank" rel="nofollow noopener noreferrer">EnlighterJS</a></h3> 
         <p>EnlighterJS is a free, easy-to-use, syntax highlighting plugin developed for MooTools. Using it can be as simple as adding a single script and style to your website, choosing the elements you wish to highlight, and EnlighterJS takes care of the rest.&nbsp;Just add the <code>data-enlighter-language</code> attribute to specify the programming language :</p> 
         <pre class="language-html"><code>&lt;pre data-enlighter-language="js"&gt;
Element.implement({
...some js code to highlight..
});
&lt;/pre&gt;</code></pre> 
         <p>EnlighterJS requires MooTools.Core &gt;= 1.4.5.</p> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-576bbb5710d2a.png" alt="EnlighterJS" class="img img-thumbnail img-responsive center-block"></p> 
         <h3><a href="https://github.com/google/code-prettify" target="_blank" rel="nofollow noopener noreferrer">Google Prettyprint</a></h3> 
         <p>Google Prettyprint is an embeddable script that makes source-code snippets in HTML prettier.</p> 
         <ul> 
          <li>Works on HTML pages.</li> 
          <li>Works even if code contains embedded links, line numbers, etc.</li> 
          <li>Simple API: include some JS &amp; CSS and add an onload handler.</li> 
          <li>Lightweights: small download and does not block page from loading while running.</li> 
          <li>Customizable styles via CSS. See the<span>&nbsp;</span><a href="https://rawgit.com/google/code-prettify/master/styles/index.html" target="_blank" rel="nofollow noopener noreferrer">themes gallery</a>.</li> 
          <li>Supports all C-like, Bash-like, and XML-like languages. No need to specify the language.</li> 
          <li>Extensible language handlers for other languages. You can specify the language.</li> 
         </ul> 
         <p>Put code snippets in <code>&lt;pre class="prettyprint"&gt;...&lt;/pre&gt;</code> or <code>&lt;code class="prettyprint"&gt;...&lt;/code&gt;</code> and it will automatically be pretty-printed:</p> 
         <pre class="language-html"><code>&lt;pre class="prettyprint"&gt;class Voila {
public:
  // Voila
  static const string VOILA = "Voila";

  // will not interfere with embedded &lt;a href="#voila2"&gt;tags&lt;/a&gt;.
}&lt;/pre&gt;</code></pre> 
         <p><img src="https://ourcodeworld.com/public-media/gallery/gallery-598ef1b198c3d.png" alt="Google Pretty Print Syntax Highlighter" class="img img-thumbnail img-responsive center-block"></p> 
         <p>If you know another awesome syntax highlight plugin, please share it with us in the comment box. Have fun <i class="fa fa-heart"></i></p>

        $(function () {
            var _search_api = {
                getSuggestions: function (searchTerm) {
                    return $.ajax({
                        url: '/ajax-endpoint/search',
                        data: {
                            q: searchTerm
                        },
                        type:"GET",
                        dataType: 'JSON'
                    });
                },
                process :  function(title,data,noCase){
                    var html = '<h4>'+title+' <small>('+data.length+')</small></h4>';

                    if(!data.length){
                        return html += '<p>'+noCase+'</p>';
                    }

                    html += '<ul class="list-material">';

                    for(var c = 0;c < data.length;c++){
                        var row = data[c];

                        html += '<li data-toggle="tooltip" data-placement="top" data-original-title="'+row.name+'">';
                        html +=     '<a href="'+row.path+'" class="visible" style="padding: 0px;">';
                        html +=         '<div class="list-content">';
                        html +=             '<span class="title">'+row.name+'</span>';
                        html +=             '<span class="caption">'+row.date+'</span>';
                        html +=         '</div>';
                        html +=     '</a>';
                        html += '</li>';
                    }

                    html += '</ul>';


                    return html;
                }
            };

            $('#search-input-ourcodeworld').bind("keyup change", OurCodeWorld.debounce(function () {
                var value = $(this).val();

                if (!value) {
                    return;
                }

                _search_api.getSuggestions(value).then(function (data) {
                    var articles = data.data;
                    var group =  {
                        left: articles.splice(0, Math.ceil(articles.length/2)),
                        right: articles
                    };
                    
                    $('#search-layout-zone-articles').html(_search_api.process("Articles", group.left ,"No articles were found"));
                    $('#search-layout-zone-news').html(_search_api.process("Articles", group.right ,"No articles were found"));
                }, function (err) {
                    console.log(err);
                });
            }, 300));

            OurCodeWorld.init();
            Layout.init();
        });
    </script>
