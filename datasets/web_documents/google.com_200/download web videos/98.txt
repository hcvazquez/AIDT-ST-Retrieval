                <h1 class="mce-heading-1">Understanding the JavaScript API for Kaltura Media Players</h1> 
                <p class="Sub-Heading"><strong>What is the Kaltura Media Player JavaScript API and why use it?</strong></p> 
                <p>Kaltura’s flexible media players – the <a href="http://html5video.org/wiki/Getting_Started_with_Kaltura_HTML5">Kaltura HTML5 player</a> and the Flash-based <a href="http://www.kaltura.org/project/Video_Player_Playlist_Widget">Kaltura Dynamic Player version 3 (KDP3)</a> – provide media online publishing solutions that are easy to use and embed. Kaltura enables you to reach the widest possible end-user audience by using HTML5 in conjunction with KDP3. To learn more about how to embed Kaltura players in your web site and support cross-platform media playback, refer to <a href="http://knowledge.kaltura.com/embedding-kaltura-media-players-your-site">Embedding Kaltura Media Players in Your Site</a>.</p> 
                <p>Embedding a Kaltura player is the first step in ensuring a rich user experience. Kaltura's powerful media player JavaScript API enables you to design flexible, multifaceted interaction with the player.</p> 
                <p>The JavaScript API is a two-way communication channel that lets the player tell you what it is doing and lets you tell the player to do things.</p> 
                <p>You can use the JavaScript API to:</p> 
                <ul>
                 <li>Enable your web page to interact seamlessly with the player</li> 
                 <li>Create user interactions with the player</li> 
                 <li>Customize aspects of the player</li> 
                </ul>
                <p class="Sub-Heading"><strong>The Basics of the Base Methods</strong></p> 
                <p>Using the JavaScript API's powerful base methods, you can:</p> 
                <ul>
                 <li>Tell the player to initiate an action or react to something a user does. For example, when a user selects a thumbnail in a media gallery, tell the player to play the selection. See <a href="#sendNotification">Invoking player actions (sendNotification)</a>.</li> 
                 <li>Find out something about the player. For example, get information about the current video to show to the user. See <a href="#evaluate">Retrieving information in runtime (evaluate)</a>.</li> 
                 <li>Change something about the player without reloading the web page. For example, make the player begin playing automatically without the user clicking a Play button. See <a href="#setKDPAttribute">Changing player attributes in runtime (setKDPAttribute)</a>.</li> 
                </ul>
                <p class="Sub-Heading"><strong>Advanced Uses</strong></p> 
                <p>There are more advanced things that you can do with the JavaScript API.</p> 
                <ul>
                 <li>If you want multiple players on the same web page, you need to configure the <a href="http://www.kaltura.org/demos/kdp3/docs.html#jscallbackready">jsCallbackReady</a> JavaScript API function. See <a href="#ManagingMultiplePlayersontheSamePage">Managing Multiple Players on the Same Page</a>.</li> 
                 <li>You may enrich the user experience and help monetize your site with calls to action and by displaying supplementary material or advertisements at specific points in the video. To support these features, you need to handle the relevant player JavaScript events. See <a href="#HandlingPlayerJavaScriptEvents">Handling Player JavaScript Events</a>.</li> 
                </ul>
                <p class="Sub-Heading"><strong>In this document you will learn about:</strong></p> 
                <ul>
                 <li><a href="#UnderstandingtheJavaScriptAPIWorkflow">The JavaScript API Workflow</a>, which includes: 
                  <ul>
                   <li><a href="#EnablingtheJavascriptAPI">Enabling the JavaScript API</a></li> 
                   <li><a href="#NotificationPlayerAPIIsReady">Receiving Notification that the Player API Is Ready (jsCallbackReady)</a></li> 
                   <li><a href="#UsingtheJavaScriptAPsBaseMethods">Using the JavaScript API's Base Methods</a>: 
                    <ul>
                     <li><a href="#UnderstandingEventsNotificationsCommands">Understanding Events, Notifications, and Commands</a></li> 
                     <li><a href="#addJsListenerremoveJsListener">Listening and responding to player events (addJsListener/removeJsListener)</a></li> 
                     <li><a href="#sendNotification">Invoking player actions (sendNotification)</a></li> 
                     <li><a href="#evaluate">Retrieving information in runtime (evaluate)</a></li> 
                     <li><a href="#setKDPAttribute">Changing player attributes in runtime (setKDPAttribute)</a></li> 
                    </ul></li> 
                  </ul></li> 
                 <li> <p><a href="#UnderstandingJSAPIDiffs4FlashHTML5">Understanding JavaScript API Support Differences for Flash and HTML5 Players</a></p> </li> 
                 <li> <p><a href="#ManagingMultiplePlayersontheSamePage">Managing Multiple Players on the Same Page</a></p> </li> 
                 <li> <p><a href="#HandlingPlayerJavaScriptEvents">Handling Player JavaScript Events</a>:</p> </li> 
                 <ul>
                  <li> <p><a href="#CreatingCallstoActionCustomButtons">Creating Calls to Action Using Custom Buttons</a></p> </li> 
                  <li> <p><a href="#HandlingCodeandAdCuePointsUsingJS">Handling Player Cue Points Using JavaScript</a></p> </li> 
                 </ul>
                 <li> <p><a href="#BestPractices">Best Practices</a> for coding</p> </li> 
                </ul>
                <h1 class="mce-heading-1"><a name="UnderstandingtheJavaScriptAPIWorkflow" id="UnderstandingtheJavaScriptAPIWorkflow"></a>Understanding the JavaScript API&nbsp;Workflow</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>How do you work with the JavaScript API?</p> 
                <p class="Sub-Heading"><strong>JavaScript API Workflow</strong></p> 
                <p>The workflow for the Kaltura media player JavaScript API is:</p> 
                <ol>
                 <li><a href="#EnablingtheJavascriptAPI">Enable the JavaScript API</a>.</li> 
                 <li><a href="#NotificationPlayerAPIIsReady">Receive notification that the player API is ready (jsCallbackReady)</a>.</li> 
                 <li><a href="#UsingtheJavaScriptAPsBaseMethods">Use the JavaScript API's base methods</a>.</li> 
                </ol>
                <p>For API documentation, refer to <a href="http://www.kaltura.org/demos/kdp3/docs.html#jsapi">Kaltura Dynamic Player version 3, JavaScript APIs</a>.</p> 
                <h1 class="mce-heading-1"><a name="EnablingtheJavascriptAPI" id="EnablingtheJavascriptAPI"></a>Enabling the&nbsp;JavaScript&nbsp;API</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>How do you enable the JavaScript API to work with the player?</p> 
                <p class="Sub-Heading"><strong>Enabling the externalInterfaceDisabled Parameter</strong></p> 
                <p>For a Kaltura player to communicate with the JavaScript API, you enable the Flash's external interface <em>externalInterfaceDisabled</em> parameter. By default, the parameter is disabled.</p> 
                <p class="Sub-Heading mce-procedure">To enable the externalInterfaceDisabled parameter:</p> 
                <p>Include the following flashVar in the player embed code: <em>externalInterfaceDisabled=false</em></p> 
                <h1><a name="NotificationPlayerAPIIsReady" id="NotificationPlayerAPIIsReady"></a>Receiving Notification that the Player API Is Ready (jsCallbackReady)</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>How do you know that the player is ready to work with the JavaScript API?</p> 
                <p class="Sub-Heading"><strong>Notification of Player Readiness</strong></p> 
                <p>Before you can <a href="#UsingtheJavaScriptAPsBaseMethods">use the JavaScript API's base methods</a>, the player has to reach the point in its internal loading sequence when it is ready to interact with your code. The player lets you know that it is ready by calling the <a href="http://www.kaltura.org/demos/kdp3/docs.html#jscallbackready">jsCallbackReady</a> JavaScript function on the page.</p> 
                <p><em>jsCallbackReady</em> is the player's first callback.</p> 
                <p>The player passes <em>jsCallbackReady</em> an <em>objectId</em> parameter that represents the identifier of the player that is embedded on the page:</p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">function jsCallbackReady(objectId) {
	window.kdp = document.getElementById(objectId);
}	
</pre>
                <p>Kaltura recommends that you place <em>jsCallbackReady</em> in the global scope. This allows easily finding this critical function in the JavaScript code.</p> 
                <h1 class="mce-heading-1"><a name="UsingtheJavaScriptAPsBaseMethods" id="UsingtheJavaScriptAPsBaseMethods"></a>Using the JavaScript API's Base Methods</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>How do you use the JavaScript API?</p> 
                <p class="Sub-Heading"><strong>Accessing the JavaScript API Base Methods</strong></p> 
                <p>You can access the entire JavaScript API through methods that enable:</p> 
                <ul>
                 <li><a href="#addJsListenerremoveJsListener">Listening and responding to player events (addJsListener/removeJsListener)</a>:<br> React to internal player events, such as beginning play and pausing.</li> 
                 <li><a href="#sendNotification">Invoking player actions (sendNotification)</a>:<br> Tell the player to do something, such as play or pause.</li> 
                 <li><a href="#evaluate">Retrieving information in runtime (evaluate)</a>:<br> Find out something about a player, such as the media that is loaded in the player and flashVars that the player passes.</li> 
                 <li><a href="#setKDPAttribute">Changing player attributes in runtime (setKDPAttribute)</a>:<br> Modify player attributes, such as a label on a player UI.</li> 
                </ul>
                <h2 class="mce-heading-2"><a name="UnderstandingEventsNotificationsCommands" id="UnderstandingEventsNotificationsCommands"></a>Understanding Events, Notifications, and Commands</h2> 
                <p>KDP3 is based on the PureMVC framework. Significant occurrences in KDP3 are passed as <em>notifications</em>. KDP3 externalizes Flash notifications to be available for JavaScript.</p> 
                <p>KDP3 fires notifications of passive events. A JavaScript listener (<a href="#addJsListenerremoveJsListener">JsListener</a>) detects KDP3 JavaScript notifications. This lets you detect specific events and react to them.</p> 
                <p>Notifications also are active events that you can use with the <a href="http://www.kaltura.org/demos/kdp3/docs.html#sendnotification">sendNotification</a> method to command the player to do something.</p> 
                <p>Refer to <a href="http://www.kaltura.org/kalorg/kdp/trunk/kdp3Lib/docs/notifications%20-%20KDP%20events.txt">KDP Events</a> for descriptions of passive and active events.</p> 
                <h2 class="mce-heading-2"><a name="addJsListenerremoveJsListener" id="addJsListenerremoveJsListener"></a>Listening and Responding to Player Events (addJsListener/removeJsListener)</h2> 
                <p class="Note mce-note-graphic">In KDP versions 3.4.10.1 and earlier, JsListeners are executed first and precede the player’s actions. In KDP versions 3.4.10.2 and later, player actions (such as calling the core and plugin handlers) always precede calling the JsListeners.</p> 
                <h3 class="mce-heading-3"><a name="UsingaddJsListener" id="UsingaddJsListener"></a>Using addJsListener</h3> 
                <p>Use the <a href="http://www.kaltura.org/demos/kdp3/docs.html#addlistener">addJsListener</a> method to listen for a specific notification that something happened in the player, such as the video is playing or is paused.</p> 
                <p class="Sub-Heading"><strong>addJsListener Method Syntax</strong></p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">addJsListener("event", "functionName")</pre>
                <ul>
                 <li><em>event</em> is a string that represents the name of the event notification. Enclose the notification name in single or double quotation marks.</li> 
                 <li><em>functionName</em> is a string that represents the name of the listener function to call when the event occurs. Enclose the listener function name in single or double quotation marks, without parentheses.</li> 
                </ul>
                <p class="Sub-Heading"><strong><a name="addJsListenerSampleCode" id="addJsListenerSampleCode"></a>Sample Code</strong></p> 
                <p>You may want to enrich the user experience by displaying a video's elapsed time as it plays. Implementing this option requires listening for the video entry's updated time.</p> 
                <p>To listen for a player entry time being updated:</p> 
                <ol>
                 <li>Pass <em>playerUpdatePlayhead</em> on the notification name parameter in <em>addJsListener</em>.</li> 
                 <li>On your web page, define a JavaScript function that is called when the <em>playerUpdatePlayhead</em> notification is fired in the player (for example, <em>playerUpdatePlayheadHandler</em>).</li> 
                 <li>Pass the JavaScript function name as the listener <em>functionName</em> parameter in <em>addJsListener</em>.</li> 
                </ol>
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">kdp.addJsListener(“playerUpdatePlayhead”, “playerUpdatePlayheadHandler”)
function playerUpdatePlayheadHandler(data, id) {
	// data = the player's progress time in seconds
	// id = the ID of the player that fired the notification
}</pre>
                <h3 class="mce-heading-3">Using removeJsListener</h3> 
                <p>Use the <a href="http://www.kaltura.org/demos/kdp3/docs.html#removelistener">removeJsListener</a> method to remove a listener that is no longer needed.</p> 
                <p class="Sub-Heading"><strong>Why Remove a JsListener?</strong></p> 
                <p>KDP3 accumulates JsListeners. If you add a JsListener for a notification and then add another JsListener for the same notification, the new JsListener does not override the previous one. Both JsListeners are executed in the order in which they are added. To prevent unexpected behavior in your application, Kaltura recommends that you remove unnecessary JsListeners.</p> 
                <p>When you remove a listener, you must specify the associated function name.</p> 
                <p class="Sub-Heading"><strong>removeJsListener Method Syntax</strong></p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">removeJsListener("event", "functionName")</pre>
                <ul>
                 <li><em>event</em> is a string that represents the name of the event notification. Enclose the notification name in single or double quotation marks.</li> 
                 <li><em>functionName</em> is a string that represents the name of the listener function called when the event occurs. Enclose the listener function name in single or double quotation marks, without parentheses.</li> 
                </ul>
                <p class="Sub-Heading"><strong>Sample Code</strong></p> 
                <p>To limit listening to a player update event to a single instance:</p> 
                <ol>
                 <li>Insert the <em>addJsListener</em> method to listen for a player update event. For details, see <a href="#addJsListenerSampleCode">addJsListener sample code</a>.</li> 
                 <li>Insert the <em>removeJsListener</em> method:</li> 
                </ol>
                <blockquote>
                 <ul>
                  <li>Pass "<em>playerUpdatePlayhead</em>" on the notification name parameter.</li> 
                  <li>Pass the JavaScript function name on the listener <em>functionName</em> parameter.</li> 
                 </ul>
                </blockquote> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">kdp.addJsListener(“playerUpdatePlayhead”, “playerUpdatePlayheadHandler”)
function playerUpdatePlayheadHandler(data, id) {
	// data = the player's progress time in seconds
	// id = the ID of the player that fired the notification;
}
kdp.removeJsListener(“playerUpdatePlayhead”, “playerUpdatePlayheadHandler”)</pre>
                <h2 class="mce-heading-2"><a name="sendNotification" id="sendNotification"></a>Invoking Player Actions (sendNotification)</h2> 
                <p>Use the <a href="http://www.kaltura.org/demos/kdp3/docs.html#sendnotification">sendNotification</a> method to create custom notifications that tell the player to do something, such as play, seek, or pause.</p> 
                <p class="Sub-Heading"><strong>sendNotification Method Syntax</strong></p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">sendNotification("command", { optional : data })</pre>
                <ul>
                 <li><em>command</em> is a string that represents the player command. Enclose the command in single or double quotation marks.</li> 
                 <li><em>{ optional : data }</em> is an optional &nbsp;JavaScript Object Notation (JSON) object.</li> 
                </ul>
                <p class="Sub-Heading"><strong>Sample Code</strong></p> 
                <p>To play a video from a specific point, call the <em>jumpToTime</em> function in the script area of your web page:</p> 
                <ol>
                 <li>Pass the <em>doPlay</em> command on the <em>command</em> parameter in <em>sendNotification</em>.<br>Note: <em>doPlay</em> does not require a parameter. You can either use null or omit it.</li> 
                 <li>Pass the <em>doSeek</em> command on the <em>command</em> parameter in <em>sendNotification</em>. <br> Specify a <em>timesec</em> parameter in milliseconds from the beginning of the video.</li> 
                </ol>
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">&lt;script language="JavaScript"&gt;
	var kdp;
	function jumpToTime(timesec)
	{
		kdp.sendNotification("doPlay");
		// kdp.sendNotification('doPlay', null); // Null parameters are optional

	// Moves to a specific point, defined in seconds from the start of the video
		kdp.sendNotification("doSeek", timesec); 
	}
&lt;/script&gt;</pre>
                <h2 class="mce-heading-2"><a name="evaluate" id="evaluate"></a>Retrieving Information in Runtime (evaluate)</h2> 
                <p class="Sub-Heading"><strong>Understanding the evaluate Method</strong></p> 
                <p>Use the <a href="http://www.kaltura.org/demos/kdp3/docs.html#evaluate">evaluate</a> method to find out something about a player by extracting data from player components.</p> 
                <p>The <em>evaluate</em> method is valid for any object that KDP3 externalizes for JavaScript, including:</p> 
                <ul>
                 <li>uiConf components 
                  <ul>
                   <li>Buttons and labels are examples of uiConf components.<br>Note: Use the component ID to access a uiConf component.</li> 
                  </ul></li> 
                </ul>
                <ul>
                 <li>Modifiable KDP3 data, such as flashvars and media information 
                  <ul>
                   <li>Media information is managed as proxy data, known as Value Objects (VO). Refer to&nbsp;<a href="http://knowledge.kaltura.com/faq/how-create-kdp-plugins#proxies" target="_blank">KDP-Data-Proxies</a>.</li> 
                  </ul></li> 
                 <li> <p>The Kaltura Entry Object</p> </li> 
                 <ul>
                  <li> <p>The Kaltura Entry Object stores all of the textual data related to an entry.</p> </li> 
                 </ul>
                </ul>
                <p class="Sub-Heading"><strong>evaluate Method Syntax</strong></p> 
                <pre class="brush: as3;fontsize: 100; first-line: 1; ">evaluate("{object.property.property}")</pre>
                <ul>
                 <li><em>object.property.property</em>is the reference to the component object with data that you want to extract. Enclose the reference in curly braces within single or double quotation marks. 
                  <ul>
                   <li><em>property</em> references are optional sub-objects and/or properties. <em>property</em> can be an object.</li> 
                  </ul></li> 
                </ul>
                <p class="Sub-Heading"><strong>Sample Code</strong></p> 
                <p>To get the name of an entry from a player so that you can display it:</p> 
                <ol>
                 <li>On your web page, define a function to get the entry name, for example, <em>getName</em>.</li> 
                 <li>Pass the component object reference on the <em>object.property.property</em> parameter in <em>evaluate</em>. Specify the object and property (<em>entry</em> and <em>name</em>) to extract.</li> 
                 <li>Define an alert that displays the extracted data.</li> 
                </ol>
                <pre class="brush: as3;fontsize: 100; first-line: 1; ">function getName() {	
var entry_name = kdp.evaluate('{mediaProxy.entry.name}');
	alert('Entry name: '+entry_name);
}</pre>
                <h2 class="mce-heading-2"><a name="setKDPAttribute" id="setKDPAttribute"></a>Changing Player Attributes in Runtime (setKDPAttribute)</h2> 
                <p>Use the <em>setKDPAttribute</em> method to change something about a player by setting player attribute values.</p> 
                <p class="Note mce-note-graphic">For KDP versions 3.2 and later, use setKDPAttribute. For earlier KDP versions, use <a href="http://www.kaltura.org/demos/kdp3/docs.html#setattribute">setAttribute</a>.</p> 
                <p class="Sub-Heading"><strong>setKDPAttribute Method Syntax</strong></p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">setKDPAttribute("object","property","value")</pre>
                <ul>
                 <li><em>object</em> is a string that represents the object you want to modify. Use standard dot notation to specify sub-objects, for example, <em>configProxy.flashvars</em>.</li> 
                 <li><em>property</em> is the player property that you want to modify.</li> 
                 <li><em>value</em> is the new value that you want to set for the player property.</li> 
                 <li>Enclose each parameter – <em>object</em>, <em>property</em>, and <em>value</em> – in single or double quotation marks.<br>Note: Do not enclose the parameters in curly braces . (Compare to <a href="#evaluate">Retrieving Information in Runtime (evaluate)</a>).</li> 
                </ul>
                <p class="Sub-Heading"><strong>Sample Code</strong></p> 
                <p>To make a player begin playing automatically, pass the following parameters in <em>setKDPAttribute</em>:</p> 
                <ol>
                 <li>Pass <em>configProxy.flashvars</em> on the <em>object</em> parameter. Define this object using standard dot notation. <em>configProxy</em> holds parameters related to general player configuration.</li> 
                 <li>Pass <em>autoPlay</em> on the <em>property</em> parameter.</li> 
                 <li>Pass <em>true</em> on the <em>value</em> parameter.</li> 
                </ol>
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">kdp.setKDPAttribute("configProxy.flashvars","autoPlay","true")</pre>

                <h1 class="mce-heading-1"><a name="ManagingMultiplePlayersontheSamePage" id="ManagingMultiplePlayersontheSamePage"></a>Managing Multiple Players on the Same Page</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>What do you need to do with the JavaScript API to manage multiple players on the same web page?</p> 
                <p class="Sub-Heading"><strong>JavaScript API Considerations in Managing Multiple Players</strong></p> 
                <p class="Note mce-note-graphic">Kaltura does not recommend using multiple players on the same page. The best practice is to use a single player and display additional videos as thumbnails. When a user selects a thumbnail, the selection loads and plays in the single player.</p> 
                <p>You can manage multiple players on the same page using either of the following options:</p> 
                <p class="Note mce-note-graphic">Assign a unique object identifier to each player.</p> 
                <ul>
                 <li>In the <a href="http://www.kaltura.org/demos/kdp3/docs.html#jscallbackready">jsCallbackReady</a> JavaScript API function, add a conditional <em>if</em> statement for each player identifier.</li> 
                 <li>For each player, use unique <em>jsCallbackReadyFunc</em> flashVars to call a different <em>jsCallbackReady</em> function for each player. <em>jsCallbackReadyFunc</em> specifies the name of the JavaScript function to call when the player informs the JavaScript interface that it is loaded and ready for interaction.</li> 
                </ul>
                <blockquote>
                 <ol>
                  <li>Define a <em>jsCallbackReady</em> function for each player object identifier.</li> 
                  <li>Pass flashVars in the format <em>jsCallbackReadyFunc = {jsCallbackReadyFuncName}</em> for each player.</li> 
                 </ol>
                </blockquote> 
                <ul>
                 <li>Replace <em>{jsCallbackReadyFuncName}</em>, including the curly braces, with each unique <em>jsCallbackReady</em> function name.<br>Note: Kaltura recommends that you use <em>jsCallbackReady</em> as part of each function name. &nbsp;</li> 
                </ul>
                <p>For more information on <em>jsCallbackReady</em>, see <a href="#NotificationPlayerAPIIsReady">Receive notification that the player API is ready (jsCallbackReady)</a>.</p>

                <h2 class="mce-heading-2"><a name="CreatingCallstoActionCustomButtons" id="CreatingCallstoActionCustomButtons"></a>Creating Calls to Action Using Custom Buttons</h2> 
                <p class="Sub-Heading"><strong>Understanding Calls to Action and Their Benefits</strong></p> 
                <p>A call to action is a web page element that prompts the user to do something, usually by clicking a button.</p> 
                <p>Calls to action can help you:</p> 
                <ul>
                 <li>Increase engagement in your site</li> 
                 <li>Improve your site's conversion rate</li> 
                 <li>Promote user interaction</li> 
                 <li>Encourage users to return to your site – even when your player is embedded on a different site</li> 
                </ul>
                <p class="Sub-Heading"><strong>Understanding How to Create Call to Action Custom Buttons</strong></p> 
                <p>You customize call to action buttons for Kaltura players by:</p> 
                <ol>
                 <li><a href="#DesigningaCallToActionButton">Designing a call to action button</a> in the Kaltura Management Console (KMC)</li> 
                 <li><a href="#Definingcalltoactionbutton">Defining what happens when a user clicks a call to action button</a> by writing a JavaScript command that you assign to the button</li> 
                </ol>
                <p>You can add up to five custom buttons.</p> 
                <p>Each button calls a JavaScript function that you define.</p> 
                <p>For each button, you can decide where and when the button is displayed, and define a label and tooltip.</p> 
                <p>The icon for a custom button is a star. You cannot modify the icon.</p> 
                <h3 class="mce-heading-3"><a name="DesigningaCallToActionButton" id="DesigningaCallToActionButton"></a>Designing a Call To Action Button</h3> 
                <p class="Note mce-note-graphic">To learn more about how to configure the player features, refer to the KMC User Manual.</p> 
                <p class="Sub-Heading mce-procedure">To design a call to action button:</p> 
                <ol>
                 <li>Open the KMC and go to <a href="http://www.kaltura.com/index.php/kmc/kmc4#studio%7CplayersList">Studio</a>.</li> 
                 <li>Edit an existing player or select a new player.<br><img alt="" class="media-image" height="186" width="640" typeof="foaf:Image" src="/sites/default/files/styles/large/public/Studio%20edit_new%20player_0.png"></li> 
                 <li>In the Edit Player or Create New Player window, go to the Features tab and open the Custom Buttons menu.<br><img alt="" class="media-image" height="264" width="438" typeof="foaf:Image" src="/sites/default/files/styles/large/public/Studio%20Custom%20Buttons.png"></li> 
                 <li>Select one or more custom buttons and click <strong>Options</strong> to design the button. You can display your changes in the Preview pane.<br><img alt="" class="media-image" height="440" width="640" typeof="foaf:Image" src="/sites/default/files/styles/large/public/Studio%20Select%20Custom%20Button%20Options.png"></li> 
                 <li>Under Options, you can modify the following:<br><img alt="" class="media-image" height="424" width="234" typeof="foaf:Image" src="/sites/default/files/styles/large/public/Studio%20Custom%20Button%20Options.png"></li> 
                </ol>
                <ul>
                 <li>Location and Playing States 
                  <ul>
                   <li>Display the button in the main video area, in the video controls area at the bottom of the player, or both.</li> 
                   <li>Define when during the play cycle to display the button in the main video area.</li> 
                   <li>Define whether to display the button's icon or label in the video controls area.</li> 
                  </ul></li> 
                 <li>Label – Define a textual label for the button.</li> 
                 <li>Tooltip – Define a tooltip that is displayed when the user hovers over the button in the main video area or the video controls area.</li> 
                </ul>
                <li>Click <strong>Apply</strong> to review your modifications, and then click <strong>Save Changes</strong>.</li> 
                <p>For example, you can design a player with a Rent Video button.</p> 
                <p><img alt="" class="media-image" height="333" width="405" typeof="foaf:Image" src="/sites/default/files/styles/large/public/Rent%20Video%20Button.png"></p> 
                <p>Continue with <a href="#Definingcalltoactionbutton">defining what happens when a user clicks the button</a>.</p> 
                <h3 class="mce-heading-3"><a name="Definingcalltoactionbutton" id="Definingcalltoactionbutton"></a>Defining What Happens When a User Clicks a Call to Action Button</h3> 
                <p class="Sub-Heading mce-procedure">To define the action that a user triggers when clicking a custom button:</p> 
                <p>In the script area of the web page where you embed the player with the custom button:</p> 
                <ol>
                 <li>Insert the following JavaScript function. <pre class="brush: jscript;fontsize: 100; first-line: 1; ">&lt;script&gt; function customFuncx (entryId){ // Add your custom code here. } &lt;/script&gt;</pre></li> 
                 <li>&nbsp;Replace <em>x</em> in <em>customFunc<strong>x</strong></em> with the custom button digit. For example, use <em>customFunc1</em> for Custom&nbsp;Button&nbsp;1. See <a href="#DesigningaCallToActionButton">Designing a Call To Action Button</a>.</li> 
                 <li>Where indicated, add JavaScript code that defines an action. For example, create an alert that Custom Button 1 was clicked:</li> 
                </ol>
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">function customFunc1(entryId)
{
     alert('Custom button was clicked. Played entryId is '+ entryId );

                <h2 class="mce-heading-2"><a name="HandlingCodeandAdCuePointsUsingJS" id="HandlingCodeandAdCuePointsUsingJS"></a>Handling Player Cue Points Using JavaScript</h2> 
                <p class="Sub-Heading"><strong>Understanding Cue Points and Their Benefits</strong></p> 
                <p>A cue point is a marker at a precise point of time in a video or audio entry. You may assign metadata to a cue point using the <a href="http://www.kaltura.org/kaltura-api-and-testme-console-introduction">Kaltura API</a>.</p> 
                <p>You use a cue point to trigger a time-based event. You can use <a href="#HandlingCodeCuePoints">code cue points</a> and ad cue points.</p> 
                <p>You use code cue points to display a file – such as a document, image, or comment – that relates to the content. Code cue points can help you:</p> 
                <ul>
                 <li>Enrich the user experience</li> 
                 <li>Coordinate related material with specific media content</li> 
                 <li>Enable users to skip to specific chapters</li> 
                 <li>Share user-generated content at precise points of the playback</li> 
                </ul>
                <p>You use ad cue points to insert advertisements at a specified point in the video.</p> 
                <p class="Note"><span class="mce-note-graphic">To learn how to insert advertisements, refer to the KMC User Manual.</span></p> 
                <p>Ad cue points can help you:</p> 
                <ul>
                 <li>Generate revenue using midroll advertisements</li> 
                 <li>Time ads for specific intervals</li> 
                 <li>Coordinate ads with the media content</li> 
                </ul>
                <p>In a single video, you may define different types of cue point events and include multiple cue points of each type.</p> 
                <p>In the <a href="http://demo.kaltura.com/showcase/">Kaltura Demo Apps</a>, try out the Code Cue Points Demo by selecting the demo from the drop-down menu at the top of the page. The demo lets you create code cue points in a Kaltura video entry and display related files that you tailor for each code cue point.</p>

                <h1 class="mce-heading-1"><a name="BestPractices" id="BestPractices"></a>Best Practices</h1> 
                <p class="Sub-Heading"><strong>Questions Answered</strong></p> 
                <p>What best practices does Kaltura recommend for working with the JavaScript API?</p> 
                <p><span class="mce-heading-2">Caching the Player Object in a Global Variable</span></p> 
                <p>Cache a reference to the player object inside a global variable using the following syntax:</p> 
                <pre class="brush: jscript;fontsize: 100; first-line: 1; ">window.kdp = document.getElementById(player_id);</pre>
                <p>When you place a reference to the player object inside a variable, you do not need to query the document object model (DOM) each time you do something with the player, such as adding or removing event listeners.</p> 
                <p>Kaltura recommends prefixing <em>window.</em> to the variable name to indicate that a global variable is being created.</p>
