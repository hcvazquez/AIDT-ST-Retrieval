       <p class="markdown-p">This method expects two arguments: the kind of webhook notification to be generated, and an ID of the object which triggered it.</p> 
       <div class="message message--note">
        <span class="message__title message__title--note">note</span>
        <p class="markdown-p">The notification returned will contain a dummy object of the type specified, but this object is not a complete item from your gateway. It will not necessarily have everything you expect from a production webhook.</p> 
       </div> 
       <p class="markdown-p">The webhook kind can be specified as any one of the following:</p> 
       <h4 class="markdown-h4"> Braintree Auth <a class="icon icon--link hash-link" href="#braintree-auth"></a> <a class="hash-link-target" name="braintree-auth"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.CONNECTED_MERCHANT_STATUS_TRANSITIONED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.CONNECTED_MERCHANT_PAY_PAL_STATUS_CHANGED</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Disbursement <a class="icon icon--link hash-link" href="#disbursement"></a> <a class="hash-link-target" name="disbursement"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.DISBURSEMENT</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.DISBURSEMENT_EXCEPTION</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.TRANSACTION_DISBURSED</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Dispute <a class="icon icon--link hash-link" href="#dispute"></a> <a class="hash-link-target" name="dispute"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.DISPUTE_LOST</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.DISPUTE_OPENED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.DISPUTE_WON</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Partner Merchant Account <a class="icon icon--link hash-link" href="#partner-merchant-account"></a> <a class="hash-link-target" name="partner-merchant-account"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.PARTNER_MERCHANT_CONNECTED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.PARTNER_MERCHANT_DECLINED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.PARTNER_MERCHANT_DISCONNECTED</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Sub-merchant Account <a class="icon icon--link hash-link" href="#sub-merchant-account"></a> <a class="hash-link-target" name="sub-merchant-account"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUB_MERCHANT_ACCOUNT_APPROVED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUB_MERCHANT_ACCOUNT_DECLINED</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Subscription <a class="icon icon--link hash-link" href="#subscription"></a> <a class="hash-link-target" name="subscription"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_CANCELED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_CHARGED_SUCCESSFULLY</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_CHARGED_UNSUCCESSFULLY</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_EXPIRED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_TRIAL_ENDED</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_WENT_ACTIVE</code></li> 
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.SUBSCRIPTION_WENT_PAST_DUE</code></li> 
       </ul> 
       <h4 class="markdown-h4"> Test <a class="icon icon--link hash-link" href="#test"></a> <a class="hash-link-target" name="test"></a> </h4> 
       <ul class="list list--disc">
        <li><code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.CHECK</code></li> 
       </ul> 
       <h2 class="markdown-h2"> Trigger a test notification from Braintree <a class="icon icon--link hash-link" href="#trigger-a-test-notification-from-braintree"></a> <a class="hash-link-target" name="trigger-a-test-notification-from-braintree"></a> </h2> 
       <p class="markdown-p">Once you've created a webhook, you can use the Control Panel to fire a test notification to the webhook's destination URL. This notification's kind is <code class="syntax-inline syntax-inline--theme">WebhookNotification.Kind.CHECK</code>.</p> 
       <p class="markdown-p">The <strong>Check URL</strong> button that fires the webhook is located on the primary webhooks page in the Control Panel, next to each configured webhook. Be careful when using this button in production- if your webhook handling code doesnt look for the <a href="/reference/general/webhooks/overview#notification-kinds">kind</a> of webhooks it receives, this could lead to unexpected behavior.</p> 
       <p class="markdown-p">For example, if your integration assumes it will only receive <em>Subscription Canceled</em> webhooks and you test the URL, it might cause an exception when your code tries to access the non-existent subscription object on the webhook.</p>

       <h2 class="markdown-h2"> Purge sandbox data <a class="icon icon--link hash-link" href="#purge-sandbox-data"></a> <a class="hash-link-target" name="purge-sandbox-data"></a> </h2> 
       <p class="markdown-p">You can purge all of the following data from your sandbox by going to <strong>Settings</strong> &gt; <strong>Purge Sandbox Data</strong> in the <a href="https://sandbox.braintreegateway.com/login">Control Panel</a>:</p> 
       <ul class="list list--disc">
        <li>Transactions</li> 
        <li>Customers</li> 
        <li>Payment methods</li> 
        <li>Addresses</li> 
        <li>Subscriptions</li> 
        <li>Sub-merchant accounts (if you are using <a href="/guides/braintree-marketplace/overview">Braintree Marketplace</a>)</li> 
       </ul> 
       <div class="message message--note">
        <span class="message__title message__title--note">note</span>
        <p class="markdown-p">This will log you out, lock your account, and block API operations until the purge is complete.</p> 
       </div> 
       <p class="markdown-p">Purging this data will not affect regular merchant accounts, recurring billing plans, webhooks, or other account settings. While you can <a href="https://articles.braintreepayments.com/guides/recurring-billing/plans#deleting-a-plan">delete individual plans</a> and webhooks, you cannot delete merchant accounts.</p>
