             <div class="post-text" itemprop="text"> 
              <p>I need to use the features of the <a href="http://library.wolfram.com/infocenter/MathSource/3336/" rel="nofollow">Summa package</a> but it doesn't work in <em>Mathematica</em> 9. It gives the error</p> 
              <pre><code>Cannot open Utilities`FilterOptions`
</code></pre> 
              <p>My question is: </p> 
              <p>Is there an alternative (more modern) package with similar features, or a way to make <code>Summa.m</code> work in <em>Mathematica</em> 9?</p> 
              <hr> 
              <p>Following @Nasser I solved it. The problem was that the input of <a href="http://reference.wolfram.com/mathematica/ref/FilterRules.html" rel="nofollow"><code>FilterRules</code></a> is in opposite order, e.g.</p> 
              <pre><code>FilterOptions[Expand,{opts}]
</code></pre> 
              <p>is changed to</p> 
              <pre><code>Sequence@@FilterRules[{opts},Options@Expand]
</code></pre>

             <div class="post-text" itemprop="text"> 
              <p>As of version 9 this has changed <a href="http://reference.wolfram.com/mathematica/Compatibility/tutorial/Utilities/FilterOptions.html" rel="nofollow noreferrer">http://reference.wolfram.com/mathematica/Compatibility/tutorial/Utilities/FilterOptions.html</a></p> 
              <p><img src="https://i.stack.imgur.com/CbiUP.png" alt="Mathematica graphics"></p> 
              <p>So this package needs to use <code>FilterRules</code>. I Downloaded this package, and changed <code>Summa.m</code> according to the above, and now it loads ok. </p> 
              <p>Changed every place it said <code>FilterOptions</code> to <code>Sequence@@FilterRules</code>. Run few tests from the notebook given. Example of change:</p> 
              <p>old:</p> 
              <pre><code>inrule1[opts___Rule]:= c_ Summa[el1_,it:{k_,__}]:&gt;
        Summa[If[$Explode,
                Expand[c el1,FilterOptions[Expand,{opts}]],
                c el1]  ,it] /; FreeQ[c,k];
</code></pre> 
              <p>new:</p> 
              <pre><code>inrule1[opts___Rule]:= c_ Summa[el1_,it:{k_,__}]:&gt;
        Summa[If[$Explode,
            Expand[c el1,Sequence@@FilterRules[{opts},Expand]],
                c el1]  ,it] /; FreeQ[c,k];
</code></pre> 
              <p>now</p> 
              <pre><code>Get["Summa.m"]
</code></pre> 
              <p><img src="https://i.stack.imgur.com/Mdnrz.png" alt="Mathematica graphics"></p> 
              <p>I put the updated <code>Summa.m</code> in this folder <a href="http://12000.org/tmp/summa/" rel="nofollow noreferrer">here</a> , replace your <code>Summa.m</code> with this version. If you find more issues with this, try to contact the author, they might have better solution or newer version. </p>

   ,FilterOptions[a_,b___] := Sequence @@ FilterRules[{b}, Options[a]]
   ];

~/.Mathematica/Kernel/init.m                                   is the right place on Unix/Linux.
