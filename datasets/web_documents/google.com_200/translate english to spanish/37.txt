             <div class="post-text" itemprop="text"> 
              <p>So I was finally able to figure out the problem.</p> 
              <p>Seems that the JS template translation are read from <code>js-translation.json</code> which is generated during <code>setup:static-content:deploy</code> execution. In order to populate data in this file a new <strong>language package</strong> has to be created for the project.</p> 
              <p>So instead of adding the CSV at the theme level like <code>app/design/&lt;area&gt;/&lt;vendor&gt;/&lt;theme-name&gt;/i18n/xx_XX.csv</code> we need to add it in the language package.</p> 
              <p>To create a new <strong>Language Package</strong> first from <code>project document root</code> we will need to create the following directories:</p> 
              <pre><code>mkdir -p app/i18n/&lt;project-name&gt;/&lt;xx_xx&gt;
</code></pre> 
              <blockquote> 
               <p><strong>Important</strong>: USE lowercase DIRECTORY NAMES ONLY camcelcased folder names will not work</p> 
              </blockquote> 
              <p>Then change directory to the newly created folders:</p> 
              <pre><code>cd app/i18n/&lt;project-name&gt;/&lt;xx_xx&gt;
</code></pre> 
              <p>Now you can create a <code>composer.json</code> (optional) file with the following content:</p> 
              <pre><code>{                                                     
    "name": "&lt;project-name&gt;/&lt;xx_xx&gt;",                             
    "description": "&lt;sample description&gt;", //Ex:English (United States) language
    "version": "&lt;version-number&gt;", //100.0.1                             
    "license": [                                      
        "OSL-3.0",                                    
        "AFL-3.0"                                     
    ],                                                
    "require": {                                      
        "magento/framework": "100.0.*"                
    },                                                
    "type": "magento2-language",                      
    "autoload": {                                     
        "files": [                                    
            "registration.php"                        
        ]                                             
    }                                                 
}                                                     
</code></pre> 
              <p>Next create we need a <code>language.xml</code> file with the following contents:</p> 
              <pre><code>&lt;language xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:framework:App/Language/package.xsd"&gt;
    &lt;code&gt;xx_XX&lt;/code&gt; &lt;!-- example: &lt;code&gt;en_US&lt;/code&gt; --&gt;
    &lt;vendor&gt;&lt;project-name&gt;&lt;/vendor&gt;
    &lt;package&gt;&lt;xx_xx&gt;&lt;/package&gt; &lt;!-- example: &lt;package&gt;en_us&lt;/package&gt; --&gt;
&lt;/language&gt;
</code></pre> 
              <p>After than <code>registration.php</code> containing the following content is needed:</p> 
              <pre><code>\Magento\Framework\Component\ComponentRegistrar::register(
    \Magento\Framework\Component\ComponentRegistrar::LANGUAGE,
    '&lt;project-name&gt;_&lt;xx_xx&gt;',
    __DIR__
);
</code></pre> 
              <p>Now we can create a our translation CSV. If you already have one inside the theme folder something like <code>app/design/&lt;area&gt;/&lt;vendor&gt;/&lt;theme-name&gt;/i18n/xx_XX.csv</code> you can just move it to <code>app/i18n/&lt;project-name&gt;/&lt;xx_xx&gt;/xx_XX.csv</code></p> 
              <p>Now from the <code>project document root</code> we need to run these commands:</p> 
              <pre><code>find pub/static -name js-translation.json -exec rm -rf {} \;
</code></pre> 
              <blockquote> 
               <p>We need to delete the <code>js-translation.json</code> which has been already created before running the <code>setup:static-content:deploy</code></p> 
              </blockquote> 
              <p>Now we run static content deploy:</p> 
              <pre><code>php bin/magento setup:static-content:deploy &lt;xx_XX&gt;
</code></pre> 
              <p>Once thats done we clear the cache:</p> 
              <pre><code>php bin/magento cache:clean
php bin/magento cache:flush
</code></pre> 
              <p>We can verify if the translation files for JS template has been generated by finding all the <code>js-translation.json</code> inside the pub/static folder.</p> 
              <pre><code>find pub/static -name js-translation.json
</code></pre> 
              <p>This will provide the list of all the translation files generated for JS templates.</p> 
              <blockquote> 
               <p>Reference:</p> 
               <ol> 
                <li><a href="http://devdocs.magento.com/guides/v2.0/config-guide/cli/config-cli-subcommands-i18n.html#m2devgde-xlate-severalpacks" rel="noreferrer">Magento DevDocs</a></li> 
                <li><a href="https://github.com/magento/magento2/issues/2056" rel="noreferrer">Related Github Issue</a></li> 
               </ol> 
              </blockquote>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>As I described in the Github issue (<a href="https://github.com/magento/magento2/issues/7525#issuecomment-316690100" rel="nofollow noreferrer">https://github.com/magento/magento2/issues/7525#issuecomment-316690100</a>) there is another (easier) solution to that problem.</p> 
              <p>You just need to make sure to deploy every language separate.</p> 
              <p>So instead of using:</p> 
              <pre><code>php bin/magento setup:static-content:deploy en_US de_DE it_IT fr_FR es_ES
</code></pre> 
              <p>use</p> 
              <pre><code>php bin/magento setup:static-content:deploy en_US
php bin/magento setup:static-content:deploy de_DE
php bin/magento setup:static-content:deploy it_IT
php bin/magento setup:static-content:deploy fr_FR
php bin/magento setup:static-content:deploy es_ES
</code></pre> 
              <p>This is probably caused by the fact that the <code>static-content:deploy</code> only compiles the files once and copies them afterwards to other languages.</p>
