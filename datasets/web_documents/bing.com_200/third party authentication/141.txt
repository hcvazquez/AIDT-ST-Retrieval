      <div class="span12 widget-span widget-type-cell sonatype-section blog-body" style="" data-widget-type="cell" data-x="0" data-w="12"> 
       <div class="row-fluid-wrapper row-depth-1 row-number-7 ">

         <div class="span8 widget-span widget-type-cell " style="" data-widget-type="cell" data-x="0" data-w="8">

            <div class="span12 widget-span widget-type-blog_content " style="" data-widget-type="blog_content" data-x="0" data-w="12">

                <div class="blog-post-wrapper cell-wrapper"> 
                 <div class="section post-header"> 
                  <div class="text30px-bold darkgrey-text">
                   <span id="hs_cos_wrapper_name" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="text">Setting up a Docker Private Registry with Authentication Using Nexus and Nginx</span>
                  </div> 
                  <div id="hubspot-author_data" class="hubspot-editable" data-hubspot-form-id="author_data" data-hubspot-name="Blog Author"> 
                   <span class="hs-author-label">by</span> 
                   <a class="author-link" href="http://blog.sonatype.com/author/stefan-prodan">Stefan Prodan</a> on March 15, 2017 
                  </div> 
                 </div> 
                 <div class="section post-body"> 
                  <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text">
                   <div id="ts-fab-below" class="ts-fab-wrapper"> 
                    <p>This article shows how you can set up a Docker Private Registry with authentication and SSL using Nexus Repository OSS.</p> 
                    <p>Nexus Repository OSS is a universal repository manager with support for all major package formats and types. It’s a free solution for storing and sharing Docker images and other components like NuGet or NPM packages across the deployment pipeline while keeping your proprietary and third-party images private and secure.
                     <!--more--></p> 
                    <p><img class="alignnone size-full wp-image-64170" src="http://www.sonatype.org/nexus/content/uploads/2017/03/stefan1.png" alt="stefan1" width="520" height="258"></p> 
                    <p>I am using an Ubuntu Server 16.04 and Docker 1.12 to host the Nexus Repository and NGINX containers.</p> 
                    <h3 id="running-nexus-repository-container">Running Nexus Repository container</h3> 
                    <p>First you have to build your own Nexus 3 docker image and expose port 8081 and 5000. Nexus management UI will run on 8081 while Docker Registry will run on 5000. This Docker image can be found on Docker Hub at <a href="https://hub.docker.com/r/stefanprodan/nexus/">stefanprodan/nexus</a>.</p> 
                    <p>Create a directory named nexus and add a Dockerfile with the following content:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>FROM alpine:3.4

ENV <span class="nv">NEXUS_VERSION</span><span class="o">=</span><span class="s2">"3.0.2-02"</span> <span class="se">\</span>
    <span class="nv">NEXUS_DATA</span><span class="o">=</span><span class="s2">"/nexus-data"</span> <span class="se">\</span>
    <span class="nv">JAVA_MIN_MEM</span><span class="o">=</span><span class="s2">"1200M"</span> <span class="se">\</span>
    <span class="nv">JAVA_MAX_MEM</span><span class="o">=</span><span class="s2">"1200M"</span> <span class="se">\</span>
    <span class="nv">JKS_PASSWORD</span><span class="o">=</span><span class="s2">"changeit"</span>

RUN <span class="nb">set</span> -x <span class="se">\</span>
    <span class="o">&amp;&amp;</span> apk --no-cache add <span class="se">\</span>
        openjdk8-jre-base <span class="se">\</span>
        openssl <span class="se">\</span>
        su-exec <span class="se">\</span>
    <span class="o">&amp;&amp;</span> mkdir <span class="s2">"/opt"</span> <span class="se">\</span>
    <span class="o">&amp;&amp;</span> wget -qO - <span class="s2">"https://download.sonatype.com/nexus/3/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">-unix.tar.gz"</span> <span class="se">\</span>
    | tar -zxC <span class="s2">"/opt"</span> <span class="se">\</span>
    <span class="o">&amp;&amp;</span> adduser -S -h <span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span> nexus <span class="se">\</span>
	<span class="o">&amp;&amp;</span> sed <span class="se">\</span>
		-e <span class="s2">"s|-Xms1200M|-Xms</span><span class="k">${</span><span class="nv">JAVA_MIN_MEM</span><span class="k">}</span><span class="s2">|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|-Xmx1200M|-Xmx</span><span class="k">${</span><span class="nv">JAVA_MAX_MEM</span><span class="k">}</span><span class="s2">|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|karaf.home=.|karaf.home=/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|karaf.base=.|karaf.base=/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|karaf.etc=etc|karaf.etc=/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">/etc|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|java.util.logging.config.file=etc|java.util.logging.config.file=/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">/etc|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|karaf.data=data|karaf.data=</span><span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span><span class="s2">|g"</span> <span class="se">\</span>
		-e <span class="s2">"s|java.io.tmpdir=data/tmp|java.io.tmpdir=</span><span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span><span class="s2">/tmp|g"</span> <span class="se">\</span>
		-i <span class="s2">"/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">/bin/nexus.vmoptions"</span> <span class="se">\</span>
	<span class="o">&amp;&amp;</span> mkdir -p <span class="s2">"</span><span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span><span class="s2">"</span> <span class="se">\</span>
	<span class="o">&amp;&amp;</span> chown -R nexus <span class="s2">"</span><span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span><span class="s2">"</span>

EXPOSE 8081 5000

WORKDIR <span class="s2">"/opt/nexus-</span><span class="k">${</span><span class="nv">NEXUS_VERSION</span><span class="k">}</span><span class="s2">"</span>

VOLUME <span class="k">${</span><span class="nv">NEXUS_DATA</span><span class="k">}</span>

CMD <span class="o">[</span><span class="s2">"su-exec"</span>, <span class="s2">"nexus"</span>, <span class="s2">"bin/nexus"</span>, <span class="s2">"run"</span><span class="o">]</span>
</code></pre> 
                    </div> 
                    <p>Next you need to create a dedicated docker network for your registry:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>docker network create intranet
</code></pre> 
                    </div> 
                    <p>Now you can build the nexus image and run the nexus container:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>docker build -t nexus-img .

docker run -d --name nexus <span class="se">\</span>
    -v /path/to/nexus-data:/nexus-data <span class="se">\</span>
    --restart unless-stopped <span class="se">\</span>
    --network intranet nexus-img
</code></pre> 
                    </div> 
                    <p>Replace <code class="highlighter-rouge">/path/to/nexus-data</code> with your own location.</p> 
                    <h3 id="running-nginx-as-reverse-proxy-for-nexus">Running NGINX as reverse proxy for Nexus</h3> 
                    <p>Create a directory named nginx and add a Dockerfile with the following content:</p> 
                    <div class="highlighter-rouge"> 
                     <pre class="highlight"><code>FROM nginx
COPY ./nginx.conf /etc/nginx/nginx.conf
</code></pre> 
                    </div> 
                    <p>In the same directory create the nginx.conf file with the following content:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>worker_processes 2;

events <span class="o">{</span> 
	worker_connections 1024; 
<span class="o">}</span>

http <span class="o">{</span>
	error_log /var/log/nginx/error.log warn;
	access_log  /dev/null;
	proxy_intercept_errors off;
	proxy_send_timeout 120;
	proxy_read_timeout 300;
	
	upstream nexus <span class="o">{</span>
        server nexus:8081;
	<span class="o">}</span>

	upstream registry <span class="o">{</span>
        server nexus:5000;
	<span class="o">}</span>

	server <span class="o">{</span>
        listen 80;
        server_name nexus.demo.com;

        keepalive_timeout  5 5;
        proxy_buffering    off;

        <span class="c"># allow large uploads</span>
        client_max_body_size 1G;

        location / <span class="o">{</span>
		<span class="c"># redirect to docker registry</span>
		<span class="k">if</span> <span class="o">(</span><span class="nv">$http_user_agent</span> ~ docker <span class="o">)</span> <span class="o">{</span>
			proxy_pass http://registry;
		<span class="o">}</span>
		proxy_pass http://nexus;
		proxy_set_header Host <span class="nv">$host</span>;
		proxy_set_header X-Real-IP <span class="nv">$remote_addr</span>;
		proxy_set_header X-Forwarded-For <span class="nv">$proxy_add_x_forwarded_for</span>;
        <span class="o">}</span>
    <span class="o">}</span>
<span class="o">}</span>

</code></pre> 
                    </div> 
                    <p>Replace <code class="highlighter-rouge">nexus.demo.com</code> with your own domain. The NGINX server detects if a call is made by the docker client, based on user agent, and redirects that call to the Docker Registry.</p> 
                    <p>Build and run the NGINX container:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>docker build -t nginx-img .

docker run -d --publish 80:80 --name nginx <span class="se">\</span>
    --restart unless-stopped <span class="se">\</span>
    --network intranet <span class="se">\</span>
    nginx-img
</code></pre> 
                    </div> 
                    <p>Now you can access the Nexus UI by navigation to your nexus sub-domain. The default credentials are <code class="highlighter-rouge">admin</code> / <code class="highlighter-rouge">admin123</code>, you should change them before proceeding with the setup. Nexus can be configured to support static or dynamic user and group definitions and can authenticate users against LDAP or Active Directory.</p> 
                    <p>Navigate to the repository administration page and create a new repository by selecting the <strong><em>docker (hosted)</em></strong> recipe. In the repository connectors section, check <strong><em>Create an HTTP connector at specified port</em></strong> and insert <strong><em>5000</em></strong> as the port value. For a detailed walkthrough check the nexus documentation on <a href="https://books.sonatype.com/nexus-book/3.0/reference/docker.html?__hstc=239247836.7e789409bfa516187dbb0cc3fb843cee.1487784788853.1489409847685.1489412772483.7&amp;__hssc=239247836.4.1489412772483&amp;__hsfp=2635578825">Docker Registry</a>.</p> 
                    <p>At this point, the Docker Registry is up and running, but you can’t access it from a docker client because Docker requires the registry to run on SSL.</p> 
                    <p>You can use letsencrypt <a href="https://github.com/certbot/certbot">certbot</a> to generate a certificate for nexus sub-domain or you can use CloudFlare to manage your domain and enable the free Flexible SSL option. Since certbot NGINX plug-in is still experimental I opted for the CloudFlare certificate.</p> 
                    <p>Once you’ve configured the certificate you can start using the Docker Private Registry by logging in with your nexus credentials:</p> 
                    <div class="language-sh highlighter-rouge"> 
                     <pre class="highlight"><code>docker login nexus.demo.com
</code></pre> 
                    </div> 
                    <p><a href="https://www.sonatype.com/nexus-repository-oss?__hstc=239247836.7e789409bfa516187dbb0cc3fb843cee.1487784788853.1489409847685.1489412772483.7&amp;__hssc=239247836.4.1489412772483&amp;__hsfp=2635578825">Nexus Repository OSS</a> is used by more than 100,000 development teams, if you need to run a self-hosted Docker Registry you should consider using Nexus.</p>
