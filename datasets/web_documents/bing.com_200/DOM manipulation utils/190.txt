         <td height="39" width="25%"><font size="2"><b><font face="Arial, Helvetica, sans-serif" color="#003366">Project Status?</font></b></font></td> 
         <td height="39" width="75%"><font face="Arial, Helvetica, sans-serif" size="2"><b>JDOM 2.0.6 is available!</b> This is a recommended upgrade for anyone still using JDOM 1.1.3 or earlier, and using Java5 or later. JDOM 2.0.6 contains a few fixes: 
           <ul> 
            <li>Fixes Issue 120 - Incorrect cloning of JaxenCompile ending up with variables shared between clones. </li>
            <li>Fixes Issue 127 - When Outputting content, getting a PrettyFormat Format, then setting it to "Preserve" does not preserve properly </li>
            <li>Fixes Issue 126 - While namespaces prefixes like "Xml" are discouraged, they are actually allowed (except "xml" and "xmlns" - case specific). </li>
            <li>Fixes Issue 128 - Third party Security audit revealed a few places which were vulnerable. </li>
            <li>Fixes Issue 129 - Java 8 introduces public sort method on List interface. This conflicts with protected sort on ContentList </li>
            <li>Fixes Issue 131 - JavaDoc fix </li>
            <li>Fixes Issue 134 - Upgrade Xalan to 2.7.2 </li>
            <li>Fixes Issue 143 - Maven third party dependency versions </li>
            <li>Fixes Issue 136 - XMLReaders enum initializes all three members on first use (Non-validating, DTD, and XSD). This can be slow when not needed. </li>
            <li>Fixes Issue 138 - Non-Namespace Aware DOM constructs get Parsed OK by JDOM 1.x but fail on JDOM 2.x. Make JDOM 2.x look harder for undeclared prefixes. </li>
           </ul> <p></p> <a href="downloads/">Get JDOM Here!</a> <p></p> JDOM 2.x versions are a significant reworking of the JDOM API to enhance it with Generics, and other Java Language features introduced with Java 5. JDOM 2.x is fully supported for use with Java 6 or Java 7, and is also compatible with Java 5 with a few restrictions. See the full <a href="https://github.com/hunterhacker/jdom/wiki/JDOM2-Dependencies-and-Supported-Versions"> Statement on Supported Versions and Dependencies</a> <p></p> To facilitate the deployment of JDOM 2.x, the entire JDOM package has been renamed. If you are migrating to JDOM 2.x from previous JDOM versions you will need to change your package imports from org.jdom.* to org.jdom2.* <p></p> A full <a href="https://github.com/hunterhacker/jdom/wiki/JDOM2-Migration-Issues"> JDOM 1.x to JDOM 2.x migration guide</a> is available. <p></p> JDOM 1.x versions are operating in maintenance mode only and only bug fixes will be considered for future 1.x releases. The only reason to continue using JDOM 1.x is for compatibility with older Java versions (back to Java 1.2). <p></p> New features are being considered for JDOM2 though. If you want to be involved, or have suggestions or questions, please write in to the <a href="involved/lists.html">mailing list</a> and check out the <a href="https://github.com/hunterhacker/jdom">GitHub Repository</a>. <p></p> <a href="downloads/">Get JDOM Here!</a></font> </td>
