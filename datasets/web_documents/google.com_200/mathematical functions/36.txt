       <span id="math-support-in-sphinx"></span>
       <span id="math-support"></span>
       <h1>Math support in Sphinx<a class="headerlink" href="#module-sphinx.ext.mathbase" title="Permalink to this headline">¶</a></h1> 
       <div class="versionadded"> 
        <p><span class="versionmodified">New in version 0.5.</span></p> 
       </div> 
       <p>Since mathematical notation isn’t natively supported by HTML in any way, Sphinx supports math in documentation with several extensions.</p> 
       <p>The basic math support is contained in <a class="reference internal" href="#module-sphinx.ext.mathbase" title="sphinx.ext.mathbase: Common math support for imgmath and mathjax / jsmath."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.mathbase</span></code></a>. Other math support extensions should, if possible, reuse that support too.</p> 
       <div class="admonition note"> 
        <p class="first admonition-title">Note</p> 
        <p class="last"><a class="reference internal" href="#module-sphinx.ext.mathbase" title="sphinx.ext.mathbase: Common math support for imgmath and mathjax / jsmath."><code class="xref py py-mod docutils literal"><span class="pre">mathbase</span></code></a> is not meant to be added to the <a class="reference internal" href="../config.html#confval-extensions"><code class="xref std std-confval docutils literal"><span class="pre">extensions</span></code></a> config value, instead, use either <a class="reference internal" href="#module-sphinx.ext.imgmath" title="sphinx.ext.imgmath: Render math as PNG or SVG images."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.imgmath</span></code></a> or <a class="reference internal" href="#module-sphinx.ext.mathjax" title="sphinx.ext.mathjax: Render math using JavaScript via MathJax."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.mathjax</span></code></a> as described below.</p> 
       </div> 
       <p>The input language for mathematics is LaTeX markup. This is the de-facto standard for plain-text math notation and has the added advantage that no further translation is necessary when building LaTeX output.</p> 
       <p>Keep in mind that when you put math markup in <strong>Python docstrings</strong> read by <a class="reference internal" href="autodoc.html#module-sphinx.ext.autodoc" title="sphinx.ext.autodoc: Include documentation from docstrings."><code class="xref py py-mod docutils literal"><span class="pre">autodoc</span></code></a>, you either have to double all backslashes, or use Python raw strings (<code class="docutils literal"><span class="pre">r"raw"</span></code>).</p> 
       <p><a class="reference internal" href="#module-sphinx.ext.mathbase" title="sphinx.ext.mathbase: Common math support for imgmath and mathjax / jsmath."><code class="xref py py-mod docutils literal"><span class="pre">mathbase</span></code></a> provides the following config values:</p> 
       <dl class="confval"> 
        <dt id="confval-math_number_all"> 
         <code class="descname">math_number_all</code>
         <a class="headerlink" href="#confval-math_number_all" title="Permalink to this definition">¶</a>
        </dt> 
        <dd>
         <p>Set this option to <code class="docutils literal"><span class="pre">True</span></code> if you want all displayed math to be numbered. The default is <code class="docutils literal"><span class="pre">False</span></code>.</p> 
        </dd>
       </dl> 
       <p><a class="reference internal" href="#module-sphinx.ext.mathbase" title="sphinx.ext.mathbase: Common math support for imgmath and mathjax / jsmath."><code class="xref py py-mod docutils literal"><span class="pre">mathbase</span></code></a> defines these new markup elements:</p> 
       <dl class="role"> 
        <dt id="role-math"> 
         <code class="descname">:math:</code>
         <a class="headerlink" href="#role-math" title="Permalink to this definition">¶</a>
        </dt> 
        <dd>
         <p>Role for inline math. Use like this:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span>Since Pythagoras, we know that <span class="na">:math:</span><span class="nv">`a^2 + b^2 = c^2`</span>.
</pre>
          </div> 
         </div> 
        </dd>
       </dl> 
       <dl class="directive"> 
        <dt id="directive-math"> 
         <code class="descname">.. math::</code>
         <a class="headerlink" href="#directive-math" title="Permalink to this definition">¶</a>
        </dt> 
        <dd>
         <p>Directive for displayed math (math that takes the whole line for itself).</p> 
         <p>The directive supports multiple equations, which should be separated by a blank line:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span><span class="p">..</span> <span class="ow">math</span><span class="p">::</span>

   (a + b)^2 = a^2 + 2ab + b^2

   (a - b)^2 = a^2 - 2ab + b^2
</pre>
          </div> 
         </div> 
         <p>In addition, each single equation is set within a <code class="docutils literal"><span class="pre">split</span></code> environment, which means that you can have multiple aligned lines in an equation, aligned at <code class="docutils literal"><span class="pre">&amp;</span></code> and separated by <code class="docutils literal"><span class="pre">\\</span></code>:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span><span class="p">..</span> <span class="ow">math</span><span class="p">::</span>

   (a + b)^2  &amp;=  (a + b)(a + b) \\
              &amp;=  a^2 + 2ab + b^2
</pre>
          </div> 
         </div> 
         <p>For more details, look into the documentation of the <a class="reference external" href="http://www.ams.org/publications/authors/tex/amslatex">AmSMath LaTeX package</a>.</p> 
         <p>When the math is only one line of text, it can also be given as a directive argument:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span><span class="p">..</span> <span class="ow">math</span><span class="p">::</span> (a + b)^2 = a^2 + 2ab + b^2
</pre>
          </div> 
         </div> 
         <p>Normally, equations are not numbered. If you want your equation to get a number, use the <code class="docutils literal"><span class="pre">label</span></code> option. When given, it selects an internal label for the equation, by which it can be cross-referenced, and causes an equation number to be issued. See <code class="xref rst rst-role docutils literal"><span class="pre">eqref</span></code> for an example. The numbering style depends on the output format.</p> 
         <p>There is also an option <code class="docutils literal"><span class="pre">nowrap</span></code> that prevents any wrapping of the given math in a math environment. When you give this option, you must make sure yourself that the math is properly set up. For example:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span><span class="p">..</span> <span class="ow">math</span><span class="p">::</span>
   <span class="nc">:nowrap:</span>

   \begin{eqnarray}
      y    &amp; = &amp; ax^2 + bx + c \\
      f(x) &amp; = &amp; x^2 + 2xy + y^2
   \end{eqnarray}
</pre>
          </div> 
         </div> 
        </dd>
       </dl> 
       <dl class="role"> 
        <dt id="role-eq"> 
         <code class="descname">:eq:</code>
         <a class="headerlink" href="#role-eq" title="Permalink to this definition">¶</a>
        </dt> 
        <dd>
         <p>Role for cross-referencing equations via their label. This currently works only within the same document. Example:</p> 
         <div class="highlight-rest">
          <div class="highlight">
           <pre><span></span><span class="p">..</span> <span class="ow">math</span><span class="p">::</span> e^{i\pi} + 1 = 0
   <span class="nc">:label:</span> <span class="nf">euler</span>

Euler's identity, equation <span class="na">:eq:</span><span class="nv">`euler`</span>, was elected one of the most
beautiful mathematical formulas.
</pre>
          </div> 
         </div> 
        </dd>
       </dl> 
       <div class="section" id="module-sphinx.ext.imgmath"> 
        <span id="sphinx-ext-imgmath-render-math-as-images"></span>
        <h2><a class="reference internal" href="#module-sphinx.ext.imgmath" title="sphinx.ext.imgmath: Render math as PNG or SVG images."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.imgmath</span></code></a> – Render math as images<a class="headerlink" href="#module-sphinx.ext.imgmath" title="Permalink to this headline">¶</a></h2> 
        <div class="versionadded"> 
         <p><span class="versionmodified">New in version 1.4.</span></p> 
        </div> 
        <p>This extension renders math via LaTeX and <a class="reference external" href="http://savannah.nongnu.org/projects/dvipng/">dvipng</a> or <a class="reference external" href="http://dvisvgm.bplaced.net/">dvisvgm</a> into PNG or SVG images. This of course means that the computer where the docs are built must have both programs available.</p> 
        <p>There are various config values you can set to influence how the images are built:</p> 
        <dl class="confval"> 
         <dt id="confval-imgmath_image_format"> 
          <code class="descname">imgmath_image_format</code>
          <a class="headerlink" href="#confval-imgmath_image_format" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The output image format. The default is <code class="docutils literal"><span class="pre">'png'</span></code>. It should be either <code class="docutils literal"><span class="pre">'png'</span></code> or <code class="docutils literal"><span class="pre">'svg'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_latex"> 
          <code class="descname">imgmath_latex</code>
          <a class="headerlink" href="#confval-imgmath_latex" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The command name with which to invoke LaTeX. The default is <code class="docutils literal"><span class="pre">'latex'</span></code>; you may need to set this to a full path if <code class="docutils literal"><span class="pre">latex</span></code> is not in the executable search path.</p> 
          <p>Since this setting is not portable from system to system, it is normally not useful to set it in <code class="docutils literal"><span class="pre">conf.py</span></code>; rather, giving it on the <strong class="program">sphinx-build</strong> command line via the <a class="reference internal" href="../invocation.html#id2"><code class="xref std std-option docutils literal"><span class="pre">-D</span></code></a> option should be preferable, like this:</p> 
          <div class="highlight-rest">
           <div class="highlight">
            <pre><span></span>sphinx-build -b html -D imgmath_latex=C:\tex\latex.exe . _build/html
</pre>
           </div> 
          </div> 
          <p>This value should only contain the path to the latex executable, not further arguments; use <a class="reference internal" href="#confval-imgmath_latex_args"><code class="xref std std-confval docutils literal"><span class="pre">imgmath_latex_args</span></code></a> for that purpose.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_dvipng"> 
          <code class="descname">imgmath_dvipng</code>
          <a class="headerlink" href="#confval-imgmath_dvipng" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The command name with which to invoke <code class="docutils literal"><span class="pre">dvipng</span></code>. The default is <code class="docutils literal"><span class="pre">'dvipng'</span></code>; you may need to set this to a full path if <code class="docutils literal"><span class="pre">dvipng</span></code> is not in the executable search path. This option is only used when <code class="docutils literal"><span class="pre">imgmath_image_format</span></code> is set to <code class="docutils literal"><span class="pre">'png'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_dvisvgm"> 
          <code class="descname">imgmath_dvisvgm</code>
          <a class="headerlink" href="#confval-imgmath_dvisvgm" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The command name with which to invoke <code class="docutils literal"><span class="pre">dvisvgm</span></code>. The default is <code class="docutils literal"><span class="pre">'dvisvgm'</span></code>; you may need to set this to a full path if <code class="docutils literal"><span class="pre">dvisvgm</span></code> is not in the executable search path. This option is only used when <code class="docutils literal"><span class="pre">imgmath_image_format</span></code> is <code class="docutils literal"><span class="pre">'svg'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_latex_args"> 
          <code class="descname">imgmath_latex_args</code>
          <a class="headerlink" href="#confval-imgmath_latex_args" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>Additional arguments to give to latex, as a list. The default is an empty list.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_latex_preamble"> 
          <code class="descname">imgmath_latex_preamble</code>
          <a class="headerlink" href="#confval-imgmath_latex_preamble" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>Additional LaTeX code to put into the preamble of the short LaTeX files that are used to translate the math snippets. This is empty by default. Use it e.g. to add more packages whose commands you want to use in the math.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_dvipng_args"> 
          <code class="descname">imgmath_dvipng_args</code>
          <a class="headerlink" href="#confval-imgmath_dvipng_args" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>Additional arguments to give to dvipng, as a list. The default value is <code class="docutils literal"><span class="pre">['-gamma',</span> <span class="pre">'1.5',</span> <span class="pre">'-D',</span> <span class="pre">'110',</span> <span class="pre">'-bg',</span> <span class="pre">'Transparent']</span></code> which makes the image a bit darker and larger then it is by default, and produces PNGs with a transparent background. This option is used only when <code class="docutils literal"><span class="pre">imgmath_image_format</span></code> is <code class="docutils literal"><span class="pre">'png'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_dvisvgm_args"> 
          <code class="descname">imgmath_dvisvgm_args</code>
          <a class="headerlink" href="#confval-imgmath_dvisvgm_args" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>Additional arguments to give to dvisvgm, as a list. The default value is <code class="docutils literal"><span class="pre">['--no-fonts']</span></code>. This option is used only when <code class="docutils literal"><span class="pre">imgmath_image_format</span></code> is <code class="docutils literal"><span class="pre">'svg'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_use_preview"> 
          <code class="descname">imgmath_use_preview</code>
          <a class="headerlink" href="#confval-imgmath_use_preview" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p><code class="docutils literal"><span class="pre">dvipng</span></code> has the ability to determine the “depth” of the rendered text: for example, when typesetting a fraction inline, the baseline of surrounding text should not be flush with the bottom of the image, rather the image should extend a bit below the baseline. This is what TeX calls “depth”. When this is enabled, the images put into the HTML document will get a <code class="docutils literal"><span class="pre">vertical-align</span></code> style that correctly aligns the baselines.</p> 
          <p>Unfortunately, this only works when the <a class="reference external" href="http://www.gnu.org/software/auctex/preview-latex.html">preview-latex package</a> is installed. Therefore, the default for this option is <code class="docutils literal"><span class="pre">False</span></code>.</p> 
          <p>Currently this option is only used when <code class="docutils literal"><span class="pre">imgmath_image_format</span></code> is <code class="docutils literal"><span class="pre">'png'</span></code>.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_add_tooltips"> 
          <code class="descname">imgmath_add_tooltips</code>
          <a class="headerlink" href="#confval-imgmath_add_tooltips" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>Default: <code class="docutils literal"><span class="pre">True</span></code>. If false, do not add the LaTeX code as an “alt” attribute for math images.</p> 
         </dd>
        </dl> 
        <dl class="confval"> 
         <dt id="confval-imgmath_font_size"> 
          <code class="descname">imgmath_font_size</code>
          <a class="headerlink" href="#confval-imgmath_font_size" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The font size (in <code class="docutils literal"><span class="pre">pt</span></code>) of the displayed math. The default value is <code class="docutils literal"><span class="pre">12</span></code>. It must be a positive integer.</p> 
         </dd>
        </dl> 
       </div> 
       <div class="section" id="module-sphinx.ext.mathjax"> 
        <span id="sphinx-ext-mathjax-render-math-via-javascript"></span>
        <h2><a class="reference internal" href="#module-sphinx.ext.mathjax" title="sphinx.ext.mathjax: Render math using JavaScript via MathJax."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.mathjax</span></code></a> – Render math via JavaScript<a class="headerlink" href="#module-sphinx.ext.mathjax" title="Permalink to this headline">¶</a></h2> 
        <div class="versionadded"> 
         <p><span class="versionmodified">New in version 1.1.</span></p> 
        </div> 
        <p>This extension puts math as-is into the HTML files. The JavaScript package <a class="reference external" href="https://www.mathjax.org/">MathJax</a> is then loaded and transforms the LaTeX markup to readable math live in the browser.</p> 
        <p>Because MathJax (and the necessary fonts) is very large, it is not included in Sphinx.</p> 
        <dl class="confval"> 
         <dt id="confval-mathjax_path"> 
          <code class="descname">mathjax_path</code>
          <a class="headerlink" href="#confval-mathjax_path" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The path to the JavaScript file to include in the HTML files in order to load MathJax.</p> 
          <p>The default is the <code class="docutils literal"><span class="pre">https://</span></code> URL that loads the JS files from the <a class="reference external" href="https://cdjns.com">cdnjs</a> Content Delivery Network. See the <a class="reference external" href="http://docs.mathjax.org/en/latest/start.html">MathJax Getting Started page</a> for details. If you want MathJax to be available offline, you have to download it and set this value to a different path.</p> 
          <p>The path can be absolute or relative; if it is relative, it is relative to the <code class="docutils literal"><span class="pre">_static</span></code> directory of the built docs.</p> 
          <p>For example, if you put MathJax into the static path of the Sphinx docs, this value would be <code class="docutils literal"><span class="pre">MathJax/MathJax.js</span></code>. If you host more than one Sphinx documentation set on one server, it is advisable to install MathJax in a shared location.</p> 
          <p>You can also give a full <code class="docutils literal"><span class="pre">http://</span></code> URL different from the CDN URL.</p> 
         </dd>
        </dl> 
       </div> 
       <div class="section" id="module-sphinx.ext.jsmath"> 
        <span id="sphinx-ext-jsmath-render-math-via-javascript"></span>
        <h2><a class="reference internal" href="#module-sphinx.ext.jsmath" title="sphinx.ext.jsmath: Render math using JavaScript via JSMath."><code class="xref py py-mod docutils literal"><span class="pre">sphinx.ext.jsmath</span></code></a> – Render math via JavaScript<a class="headerlink" href="#module-sphinx.ext.jsmath" title="Permalink to this headline">¶</a></h2> 
        <p>This extension works just as the MathJax extension does, but uses the older package <a class="reference external" href="http://www.math.union.edu/~dpvc/jsmath/">jsMath</a>. It provides this config value:</p> 
        <dl class="confval"> 
         <dt id="confval-jsmath_path"> 
          <code class="descname">jsmath_path</code>
          <a class="headerlink" href="#confval-jsmath_path" title="Permalink to this definition">¶</a>
         </dt> 
         <dd>
          <p>The path to the JavaScript file to include in the HTML files in order to load JSMath. There is no default.</p> 
          <p>The path can be absolute or relative; if it is relative, it is relative to the <code class="docutils literal"><span class="pre">_static</span></code> directory of the built docs.</p> 
          <p>For example, if you put JSMath into the static path of the Sphinx docs, this value would be <code class="docutils literal"><span class="pre">jsMath/easy/load.js</span></code>. If you host more than one Sphinx documentation set on one server, it is advisable to install jsMath in a shared location.</p> 
         </dd>
        </dl>
