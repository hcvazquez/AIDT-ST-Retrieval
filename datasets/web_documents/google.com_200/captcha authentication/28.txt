     <h3 id="howto_include_session">A Note On PHP Session Persistence</h3> 
     <p>The <a target="_blank" href="https://packagist.org/packages/captcha-com/cakephp-captcha">CakePHP CAPTCHA Plugin</a> requires persistent per-user server storage to work properly. By default, it uses CakePHP Sessions, which is convenient in most use cases.</p> 
     <p>If your environment requires a different approach to handle Session data, you can be easily adding a custom session driver as <a target="_blank" href="http://book.cakephp.org/3.0/en/development/sessions.html#creating-a-custom-session-handler">described here.</a></p> 
     <p>Note that a Session cleanup right after creating a Captcha class instance may cause an error since BotDetect will not find its Session data.</p>

     <pre>
<span class="hl opt">&lt;</span>?php <span class="hl kwa">namespace</span> App\Controller<span class="hl opt">;</span>

    parent<span class="hl opt">::</span><span class="hl kwd">initialize</span><span class="hl opt">();</span>

<strong>    <span class="hl slc">// load the Captcha component and set its parameter</span>
    <span class="hl kwc">$this</span><span class="hl opt">-&gt;</span><span class="hl kwd">loadComponent</span><span class="hl opt">(</span><span class="hl str">'CakeCaptcha.Captcha'</span><span class="hl opt">, [</span>

     <p>The BotDetect MVC Captcha component is available in CakePHP code under the <code>CakeCaptcha</code> plugin namespace. It's loaded via <code>loadComponent()</code> in your Controller’s initialize() method (or you're able to load it via <code>$components</code> array), and then assigns it a captcha configuration key defined in <code>config/captcha.php</code> file. Each individual Controller action including Captcha validation should use a unique <code>captchaConfig</code> value to avoid persistence name clashes.</p>

     <h2 id="howto_validate">3. Validate the CAPTCHA In Your CakePHP Controller</h2> 
     <p>After your <a href="#howto_display">Captcha is displayed</a> alongside a form input field, you will want to validate user submitted Captcha code in a Controller action. This should always be done before any sensitive processing to confirm the submitted data is actually entered by a person and not a bot.</p> 
     <h3 id="howto_validate_input">Validate the Submitted CAPTCHA Code User Input</h3> 
     <p>The validation of Captcha code input should be performed in Controler-&gt;Action that is on the receiving end of the form submission (where the form action points to).</p> 
     <p>Usually, the submitted form data is available inside the CakePHP request object, by the same name you gave the Captcha code input field on the form where it was displayed. For example: <code>$this-&gt;request-&gt;data['CaptchaCode']</code>.</p> 
     <p>It is then a matter of calling the <code>captcha_validate()</code> helper function and passing it the user input.</p> 
     <pre>
<span class="hl slc">// validate the user-entered Captcha code</span>
<span class="hl kwc">$isHuman</span> <span class="hl opt">=</span> <span class="hl kwd">captcha_validate</span><span class="hl opt">(</span><span class="hl kwc">$this</span><span class="hl opt">-&gt;</span>request<span class="hl opt">-&gt;</span>data<span class="hl opt">[</span><span class="hl str">'CaptchaCode'</span><span class="hl opt">]);</span>

<span class="hl slc">// clear previous user input, since each Captcha code can only be validated once</span>
<span class="hl kwa">unset</span><span class="hl opt">(</span><span class="hl kwc">$this</span><span class="hl opt">-&gt;</span>request<span class="hl opt">-&gt;</span>data<span class="hl opt">[</span><span class="hl str">'CaptchaCode'</span><span class="hl opt">]);</span>
</pre> 
     <p>After Captcha validation, it is best to clear the previous user input automatically; since each Captcha code <a href="../captcha-faq.html#validation-singleattempt">can only be validated once</a> regardless of the validation result, keeping previous user input makes no sense.</p> 
     <h3 id="howto_validate_feedback">Handle CAPTCHA Validation Failure / Success</h3> 
     <p>You should then write some code to handle both Captcha validation success and failure. If it failed, you should abort further processing and display an error message; if it succeeded, you can proceed with the protected form action:</p> 
     <pre><span class="hl kwa">if</span> <span class="hl opt">(</span><span class="hl kwc">$isHuman</span><span class="hl opt">) {</span>
  <span class="hl slc">// TODO: Captcha validation passed:</span>
  <span class="hl slc">// continue with form processing, knowing the submission was made by a human</span>
<span class="hl opt">}</span> <span class="hl kwa">else</span> <span class="hl opt">{</span>
  <span class="hl slc">// TODO: Captcha validation failed:</span>
  <span class="hl slc">// abort sensitive action, return an error message</span>
<span class="hl opt">}</span></pre> 
     <p>This will ensure that the sensitive action can not be automated and is only performed when a human fills out the form.</p>
