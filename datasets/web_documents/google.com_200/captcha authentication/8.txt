       </svg></a>inkrato community edition</h1> 
     <p>Inkrato community edition is open source base for community discussion and collaborative issue tracking software.</p> 
     <p>It supports login via email, Facebook, Twitter, Google+ and GitHub. Members can post, reply to posts and up or downvote posts. What's being discussed or tracked (e.g. issues, feedback, ideas) is fully configurable, as are the discussion topics and labels and the UI theme. There is also a REST based API.</p> 
     <p>You are free to use and modify this software for both non-commercial and commercial purposes. Please bear in mind this is an early release.</p> 
     <p>For a working demo see <a href="https://discussion.inkrato.com">discussion.inkrato.com</a>.</p>

     <h3><a id="user-content-deploying-to-heroku" class="deep-link" href="#deploying-to-heroku">

       </svg></a>Deploying to Heroku</h3> 
     <p>The easiest way to gets started is the button below which will clone this project and deploy a working instance to Heroku:</p> 
     <p><a href="https://heroku.com/deploy?template=https://github.com/inkrato/inkrato"><img src="https://www.herokucdn.com/deploy/button.png" alt="Deploy"></a></p> 
     <p>You can set <strong>Config Variables</strong> by clicking the <strong>Manage App</strong> button that Heroku dashboard displays after deploying your app for the first time, then selecting <strong>Settings</strong>.</p> 
     <p><em>NB: If only developing and testing locally specify <code>http://localhost:3000</code> in place of <code>http://YOURAPPNAME.herokuapp.com/</code> in the configuration steps below.</em></p> 
     <p>By default inkrato is it's own mail server (for things like sending password reset emails and API activation keys) but you can find details of how to configure additional servics in <strong>docs/Authentication.md</strong>.</p>

     <p>Just clone the respostory, run <code>npm install</code> to install required dependancies and <code>npm start</code> to start the application.</p>

       </svg></a>Authentication</h4> 
     <p>Authentication details for login services like Facebook and Twitter can be added in <code>config/secrets.js</code>. See <strong>docs/Authentication.md</strong> for details.</p> 
     <p><em>IMPORTANT! Don't commit a configured <code>secrets.js</code> to a public repository (this would expose private authentication details!).</em></p> 
     <p>You can also pass options at runtime via environment variables.</p> 
     <p>e.g. <code>PORT 80 npm start</code></p> 
     <p>Any login options you provide configuration details for (Twitter, Facebook, Google+, GitHub) will automatically appear on the sign-in screen.</p> 
     <h4><a id="user-content-email" class="deep-link" href="#email">

     <p>You can optionally specify a SendGrid account to use for sending emails more faster and more reliably than relying on inkrato to act as it's own mail server. They offer a free tier for low-volume usage and the service provide integrated email list management.</p>

       </svg></a>Appearance</h4> 
     <p>To customize the user interface theme edit the variables in <code>public/css/theme.less</code> and reload the page (new CSS will be auto-generated).</p> 
     <h2><a id="user-content-security-considerations" class="deep-link" href="#security-considerations">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>Security considerations</h2> 
     <p>There is currently no email address verification requirement, rate-limiting, CAPTCHA support or ability to moderate users or spam, meaning there is currently little protection against users with malicious intent.</p> 
     <p>If there are no user accounts in the system, the first user to login is granted ADMIN access (and can edit any post), but subsequent users have normal privileges and while they can create new posts they can only edit their own posts.</p> 
     <p>If you wish to grant admin access to additional users you can update the Users collection in monogodb and change their role to 'ADMIN'.</p> 
     <p>The API is disabled by default. If enabled allows read+write access via a REST interface. See <strong>docs/API.md</strong> for API documentation.</p>

       </svg></a>About the inkrato platform</h2> 
     <p>The inkrato platform has been used as a discussion platform for a wide range of uses - by activists (for the Labour Digital Government Review), charitable organizations (Friends of the Earth) for video game communities (PlanetSide Tracker), by private companies and by local community groups.</p> 
     <p>The release of this next generation version as open source software allows anyone to take advantage of it host a community site of their own, for free.</p> 
     <p>Commercial instances and bespoke versions are available on request. Commercial enquires should be sent to <a href="mailto:feedback@inkrato.com">feedback@inkrato.com</a></p>
