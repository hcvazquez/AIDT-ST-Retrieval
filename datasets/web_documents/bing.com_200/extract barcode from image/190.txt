    <h2 class="boxed">Jargon</h2> 
    <div class="section"> 
     <dl> 
      <dt> 
       <strong>Algorithm</strong> 
      </dt> 
      <dd>
       A step-by-step problem-solving procedure, especially an established, recursive computational procedure for solving a problem in a finite number of steps.
      </dd> 
      <dt> 
       <strong>Archive</strong> 
      </dt> 
      <dd>
       A file that contains a group of files and possibly directories which must be extracted in order to use them. An archive may optionally be compressed which would require it to be decompressed before the files can be extracted.
      </dd> 
      <dt> 
       <strong>Compression</strong> 
      </dt> 
      <dd>
       The temporary coding of data in a way that saves storage space or transmission time. Most text files can be compressed to about half their normal size. Graphics can be compressed to 10 percent of their original size.
      </dd> 
      <dt> 
       <strong>Digital Signing</strong> 
      </dt> 
      <dd>
       An attempt to mimic the offline act of a person applying their signature to a paper document.
      </dd> 
      <dt> 
       <strong>Enumeration</strong> 
      </dt> 
      <dd>
       An object that assists in iterating over a set of objects.
      </dd> 
      <dt> 
       <strong>Portability</strong> 
      </dt> 
      <dd>
       A measure of system independence; portable programs can be moved to a new system by recompiling without having to make any other changes.
      </dd> 
      <dt> 
       <strong>Sealed</strong> 
      </dt> 
      <dd>
       A Sealed Jar guarantees that all classes in a package come from the same code source.
      </dd> 
      <dt> 
       <strong>Security</strong> 
      </dt> 
      <dd>
       Techniques and practices that preserve the integrity of computer systems, and digital library services and collections.
      </dd> 
      <dt> 
       <strong>Stream</strong> 
      </dt> 
      <dd>
       A continuous flow of data, usually digitally encoded, designed to be processed sequentially. Also called a bitstream.
      </dd> 
      <dt> 
       <strong>Versioning</strong> 
      </dt> 
      <dd>

      </dd>

    <a name="N1005D"></a>
    <a name="Introduction"></a> 
    <h2 class="boxed">Introduction</h2> 
    <div class="section"> 
     <p>Jar files provide a way to package software written in Java and to easily distribute that software. In this chapter we will look at what Jar files are, how to use them and how to use the Jar API to access Jar files directly from within our programs.</p> 
    </div> 
    <a name="N10067"></a>
    <a name="What+Are+Jar+Files%3F"></a> 
    <h2 class="boxed">What Are Jar Files?</h2> 
    <div class="section"> 
     <a name="N1006D"></a>
     <a name="A+Compressed+Java+Archive+File"></a> 
     <h3 class="boxed">A Compressed Java Archive File</h3> 
     <p>A Java Archive File, commonly referred to as a Jar File is nothing more than a compressed archive file. The file has two important characteristics:</p> 
     <ul> 
      <li>The archive does not have to be compressed but if it is, the Zip compression algorithm is used.</li> 
      <li>The archive's filename must have the <span class="codefrag">.jar</span> extension.</li> 
     </ul> 
     <p>Archives are used to bundle a number of files and directories into one single file, as such Java Archives or Jar Files are no different. Typically, Jar files are used to distribute Java software, these Jar files contains the actual class files of the programs archived in them.</p> 
     <p>Some benefits of using Jar files are:</p> 
     <ul> 
      <li> <em>Security</em>, A jar file can be digitally signed enabling users to verify the signature and then grant the program security privileges.</li> 
      <li> <em>Decreased download time</em>, since the archive is compressed it takes less time to download than it would to download each individual file.</li> 
      <li> <em>Package Versioning</em>, A Jar file may contain vendor and version information about the files it contains.</li> 
      <li> <em>Portability</em>, all Java Runtime Environments know how to handle Jar files.</li> 
     </ul> 
     <a name="N100A0"></a>
     <a name="Creating+and+Viewing+a+Jar+File"></a> 
     <h3 class="boxed">Creating and Viewing a Jar File</h3> 
     <p>The Jar Tool comes bundled with the Java Development Kit, so if you have a JDK installed you will have the Jar tool. The Jar tool is run via the command line, this means you will use an MS Dos Prompt in Windows and a Shell or Console in Linux / Unix. The command for running the Jar tool is <span class="codefrag">jar</span>, you simply issue this command in the command line.</p> 
     <p>The basic command for creating a Jar file is:</p> 
     <pre class="code">
jar cf <em>jar-filename</em>  <em> input-files</em>
                
</pre> 
     <p>The <span class="codefrag">c</span> option indicates that you wish to create a jar and the <span class="codefrag">f</span> option indicates that output should be sent to a file, <span class="codefrag">jar-filename</span> is the name of the resulting Jar file and <span class="codefrag">input-files</span> are the files you wish to include.</p> 
     <p>As an example, consider a program in the <span class="codefrag">project</span> directory. The program consists of a class file and some images in an <span class="codefrag">images</span> directory:</p> 
     <p> <img alt="example" src="images/chapter05_01.gif"></p> 
     <p>You would then run the following command to create the jar file from within the <span class="codefrag">project</span> directory.</p> 
     <pre class="code">
jar cvf project.jar MyClass.class images
                </pre> 
     <p>The <span class="codefrag">v</span> option is for verbose output. The output would be as follows:</p> 
     <pre class="code">
added manifest
adding: MyClass.class(in = 462) (out= 291)(deflated 37%)
adding: images/(in = 0) (out= 0)(stored 0%)
adding: images/logo.gif(in = 4178) (out= 4001)(deflated 4%)
                </pre> 
     <p>The command to view the contents of a jar file is as follows:</p> 
     <pre class="code">
jar tf <em>jar-filename</em>
                
</pre> 
     <p>The <span class="codefrag">t</span> option is to print a table of contents and the <span class="codefrag">f</span> option indicates that your are reading from a file, the <span class="codefrag">jar-filename</span> is the filename of the jar you wish to use. If we use the jar file we created earlier:</p> 
     <pre class="code">
jar tf project.jar                
                </pre> 
     <p>The output is as follows:</p> 
     <pre class="code">
META-INF/
META-INF/MANIFEST.MF
MyClass.class
images/
images/logo.gif
                </pre> 
     <p>The output indicates that out class file and the images directory were included in the jar file. You will also notice that the Jar tool, automatically created and added the manifest file for us. We will find out more about this file in the next section.</p> 
     <a name="N1010F"></a>
     <a name="Extracting+Files+from+a+Jar+File"></a> 
     <h3 class="boxed">Extracting Files from a Jar File</h3> 
     <p>If we wish to extract a file from the Jar file, we use the following command:</p> 
     <pre class="code">
jar xf <em>jar-filename</em> <em> archived-files</em>
                
</pre> 
     <p>The <span class="codefrag">x</span> option indicates we wish to extract files, the <span class="codefrag">f</span> option indicates from a jar file, the <span class="codefrag">jar-filename</span> is the filename of the jar file to extract from and the <span class="codefrag">archived-files</span> are the files to extract. In our example we can do the following:</p> 
     <pre class="code">
jar xf project.jar MyClass.class images/logo.gif
            </pre> 
     <p>This will produce no output on the command line, but will extract the class file and the image file. We can also extract all files by simply not specifying which files to extract:</p> 
     <pre class="code">
jar xf project.jar
            </pre> 
     <p>If you look in the <span class="codefrag">project</span> directory you will notice a directory named <span class="codefrag">META-INF</span> which contains a file named <span class="codefrag">MANIFEST.MF</span>.</p> 
    </div> 
    <a name="N1014A"></a>
    <a name="Packaging+Software+in+a+Jar+File"></a> 
    <h2 class="boxed">Packaging Software in a Jar File</h2> 
    <div class="section"> 
     <a name="N10150"></a>
     <a name="The+Manifest+File"></a> 
     <h3 class="boxed">The Manifest File</h3> 
     <p>We said earlier that the Jar file automatically adds a manifest file to the jar file, the file is always in a directory named <span class="codefrag">META-INF</span> and is named <span class="codefrag">MANIFEST.MF</span>; these are, like Java, case sensitive. This file contins information about the jar file and the files it contains. If you open the manifest file that was extracted in the previous section you will see it contains the following:</p> 
     <pre class="code">
Manifest-Version: 1.0
Created-By: 1.5.0 (Sun Microsystems Inc.)
                </pre> 
     <p>Jar files are used in a wide variety of situations and therefore need to support a wide range of functionality like digital signing, version control, package sealing, and others. By tailoring the meta information contained in the manifest file, this Jar file is enabled to be used in these situations.</p> 
     <p>An example of modifying the manifest file is package sealing, which means that all classes defined in a package must be archived in the same Jar file. To do this you need to add a Name and Sealed header to the manifest file, the name contains the package name and the sealed header indicates whether it is sealed or not.</p> 
     <pre class="code">
Name: MyCompany/MyProject/MyPackage/
Sealed: true
                </pre> 
     <p>You would add this information to a text file, suppose it is named seal.txt, you can add this information to the manifest using the <span class="codefrag">m</span> option.</p> 
     <pre class="code">
jar cmf seal.txt myJar.jar MyCompany
                </pre> 
     <p>Two other important manifest headers are the Main-Class and Class-Path, which are used to bundle software into a Jar file so that the software can be run from within the Jar file. This is known as an executable Jar file.</p> 
     <a name="N1017B"></a>
     <a name="Packaging+the+Program"></a> 
     <h3 class="boxed">Packaging the Program</h3> 
     <p>Assume your program is name <span class="codefrag">Main.java</span> which compiles to <span class="codefrag">Main.class</span>, and that it requires a jar file named <span class="codefrag">jutil.jar</span> in the <span class="codefrag">lib</span> directory. You can get the source code <a href="chapter5.zip">here</a>. You will then add the following to a text file named <span class="codefrag">headers.txt</span>:</p> 
     <pre class="code">
Main-Class: Main
Class-Path: ./lib/jutil.jar
                </pre> 
     <p>Make sure the headers.txt has a blank new line at the end of it. You can use the following command to create the Jar file.</p> 
     <pre class="code">
jar cmf headers.txt project.jar Main.class
                </pre> 
     <a name="N101A3"></a>
     <a name="Running+the+Program"></a> 
     <h3 class="boxed">Running the Program</h3> 
     <p>You will use the Java tool to run the program packaged inside the Jar file. Simply issue the command:</p> 
     <pre class="code">
java -jar project.jar
                </pre> 
     <p>This should run the program which will display a blank Swing Window.</p> 
     <p> <img alt="Swing Window" src="images/chapter05_02.jpg"></p> 
    </div> 
    <a name="N101BA"></a>
    <a name="Using+The+Jar+API"></a> 
    <h2 class="boxed">Using The Jar API</h2> 
    <div class="section"> 
     <p>Java comes with an API for working with Zip archives as well as Jar files. In this section we'll look at how to use these APIs.</p> 
     <a name="N101C3"></a>
     <a name="Printing+the+Contents+of+a+Jar+File"></a> 
     <h3 class="boxed">Printing the Contents of a Jar File</h3> 
     <p>The first thing that we'll want to do is to open the jar file. This is easily done by creating a new <span class="codefrag">java.util.jar.JarFile</span> instance.</p> 
     <pre class="code">
try {
    JarFile jarFile = new JarFile("test.jar");
} catch (Exception e) {
    e.printStackTrace();
}
                </pre> 
     <p>The next thing is to enumerate through the jar entries. We get an <span class="codefrag">Enumeration</span> that contains <span class="codefrag">java.util.zip.ZipEntry</span> instances. We then simply print out the name of each entry.</p> 
     <pre class="code">
Enumeration entries = jarFile.entries();
while (entries.hasMoreElements()) {
    ZipEntry entry = (ZipEntry) entries.nextElement();
	System.out.println(entry.getName());
}
                </pre> 
     <p>The last thing we need to do is to close the Jar file.</p> 
     <pre class="code">
jarFile.close();
                </pre> 
     <p>Running this program will print out all the entries in the jar file named <span class="codefrag">test.jar</span>.</p> 
     <a name="N101EE"></a>
     <a name="Extracting+a+File+from+a+Jar+File"></a> 
     <h3 class="boxed">Extracting a File from a Jar File</h3> 
     <p>Extracting a file from a Jar file is a little more tricky since we have to use some IO classes to do the dirty work for use. We can create the Jar file as we did in the previous section and we can get the entry by its name. We will use the jar entry to get an input stream to it as follows:</p> 
     <pre class="code">
JarFile jarFile = new JarFile("test.jar");
InputStream in = jarFile.getInputStream(
    jarFile.getEntry("META-INF/MANIFEST.MF"));
                </pre> 
     <p>We then create an output stream to a file.</p> 
     <pre class="code">
OutputStream out = new FileOutputStream("manifest.mf");
                </pre> 
     <p>We then copy the data from the input stream into the output stream.</p> 
     <pre class="code">
int c;
while ((c = in.read()) != -1)
    out.write(c);
                </pre> 
     <p>Finally we close the streams and the Jar file.</p> 
     <pre class="code">
in.close();
out.close();
jarFile.close();
                </pre> 
     <p>Running this program will output the Jar file's manifest to a file named <span class="codefrag">manifest.mf</span>.</p> 
    </div> 
    <a name="N10218"></a>
    <a name="Summary"></a> 
    <h2 class="boxed">Summary</h2> 
    <div class="section"> 
     <p>A Java Archive File, commonly referred to as a Jar File is nothing more than a compressed archive using the Zip compression algorithm. Jar files are used to package Java programs and makes them easy to distribute. We can also access Jar files from within our Java programs.</p> 
    </div> 
    <a name="N10222"></a>
    <a name="Exercise+Set"></a> 
    <h2 class="boxed">Exercise Set</h2> 
    <div class="section"> 
     <a name="N10228"></a>
     <a name="Multiple+Choice"></a> 
     <h3 class="boxed">Multiple Choice</h3> 
     <ol> 
      <li> <p>A Jar File is a compressed archive file.</p> 
       <ol> 
        <li>True</li> 
        <li>False</li> 
       </ol> </li> 
      <li> <p>The Java Archive must be compressed.</p> 
       <ol> 
        <li>True</li> 
        <li>False</li> 
       </ol> </li> 
      <li> <p>Benefits of using Jar Files are:</p> 
       <ol> 
        <li>Security</li> 
        <li>Decreased download time</li> 
        <li>Package Versioning</li> 
        <li>Portability</li> 
        <li>All the Above</li> 
       </ol> </li> 
      <li> <p>Which command is used to create a jar file?</p> 
       <ol> 
        <li>java -jar project.jar</li> 
        <li>jar tf jar-filename</li>
