       <h1 class="at_top">Realtime Email Verification API</h1> 
       <p>To capture good leads, you need to verify email address information right at the point of entry. Integrating our Realtime Email Address Validation API directly into the sign-up forms on your website allows your website visitors to correct errors instantly and only valid email addresses will be accepted. </p> 
       <p>Just register for a <b>free API key</b> and start testing our service.</p> 
       <p><a class="btn btn-default btn-lg" href="/email-validation-online-api-get-free-trial.html" role="button">Get Your FREE API Key</a></p> 
       <h2>How Does the Email Validation API Work?</h2> 
       <p>Integrating our <a href="#" id="info-email-validation-checks"><b>Real-Time Online Email Validation API</b></a> into the Javascript, PHP, Java, C#, VB.NET code of your website or application or into content management systems (CMS) like Drupal, Typo3, WordPress or Joomla is very easy - all you have to do is to send a simple HTTP or HTTPS (SSL) request to our servers! The request returns a three digit result code and a description (list of <a href="/email-verification-results.html">email address verification API result codes</a>).</p> 
       <div id="email-validation-checks" style="display: none" title="Realtime Online Email Validation"> 
        <p> Our <b>realtime online email verification</b> process includes the following tests:</p> 
        <ul> 
         <li><b>Syntax verification</b> (IETF/RFC standard conformance)</li> 
         <li><b>DNS validation</b>, including MX record lookup</li> 
         <li><b><a href="https://en.wikipedia.org/wiki/Disposable_email_address">Disposable email address</a> detection</b></li> 
         <li><b>Misspelled domain detection to prevent <a href="https://en.wikipedia.org/wiki/Typosquatting">Typosquatting</a></b></li> 
         <li><b>SMTP connection and availability checking</b></li> 
         <li><b>Temporary unavailability detection</b></li> 
         <li><b>Mailbox existence checking</b></li> 
         <li><b><a href="http://en.wikipedia.org/wiki/Catch-all">Catch-All</a> testing</b></li> 
         <li><b><a href="http://en.wikipedia.org/wiki/Greylisting">Greylisting</a> detection</b></li> 
        </ul> 
        <p>During the entire process, we <b>never send any email</b> to the recipient address.</p>
