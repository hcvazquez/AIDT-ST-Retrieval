       <h1 class="post-title entry-title"><a href="http://scraping.pro/javascript-protected-content-scrape/" title="Scraping JavaScript protected content">Scraping JavaScript protected content</a></h1> 
       <!-- /post-title --> 
       <!-- post-meta --> 
       <p class="post-meta"> <time datetime="2015-04-09" class="post-date" pubdate><span class="updated">Apr 9, 2015</span></time> <span class="post-author"><em>By</em><span class="vcard author"><span class="fn"><a href="https://plus.google.com/112270279973392897175?rel=author" title="Visit Igor Savinkin’s website" rel="author external">Igor Savinkin</a></span></span></span> <span class="post-category"><em>in</em> <a href="http://scraping.pro/category/misc/" rel="category tag">Miscellaneous</a></span> <span class="post-comment"><a href="http://scraping.pro/javascript-protected-content-scrape/#comments">7 Comments</a></span> </p>

        <p>Here we come to one&nbsp;new milestone: the&nbsp;JavaScript-driven web sites scrape.</p> 
        <p>Recently a friend of mine got stumped as he was trying to get content of a website using PHP <em>simplehtmldom</em> library. He was failing to do it and finally found out the site was being saturated with JavaScript&nbsp;code. The anti-scrape JavaScript insertions do a tricky check to see if the page is requested and processed by a real browser and only if that&nbsp;is true, will it render&nbsp;the rest of&nbsp;page’s HTML&nbsp;code.<span id="more-7370"></span></p> 
        <h2><b>How to find out if site is JS&nbsp;stuffed?</b></h2> 
        <p>When you&nbsp;approach a target page – you won’t necessarily be able to tell whether or not it is JS-scrape-proof locked. It might take you&nbsp;some time and a few&nbsp;unsuccessful trials, before you begin to suspect something is wrong; especially since there’s no essential output at scraper’s end. So, prior to starting web scraping it’s wise to&nbsp;use a <a title="Web Sniffers Review" href="http://scraping.pro/web-sniffers-review/" target="_blank">web sniffer</a> (also called network analyzer) to watch the network activity in the target site’s page load. Nowadays, every decent browser has built-in developer tool that includes a web sniffer. In this example I used&nbsp;<a href="https://chrome.google.com/webstore/detail/web-sniffer/ndfgffclcpdbgghfgkmooklaendohaef">Chrome ext. web sniffer</a>.&nbsp;In the following&nbsp;picture you can&nbsp;see the multiple JS scripts vs only one mainframe load (in red box by me) on&nbsp;every page load (taken by Google ext. sniffer).<img class="aligncenter wp-image-7373 size-full" src="http://scraping.pro/wp-content/uploads/scripts-vs-mainframe-in-JS-stuffed-sites-marked.png" alt="scripts vs mainframe in JS-stuffed sites" width="585" height="730"></p> 
        <h2><b>JS-protected content scrape</b></h2> 
        <h3><b>JS protection logic</b></h3> 
        <p>First of all, let’s consider what JS logic is using&nbsp;to prevent scraping.</p> 
        <p>When your browser requests the HTML content, the JS code should arrive first (or at least simultaneously). This code calculates a value which it returns to the target server. Based on this calculation,&nbsp;the HTML&nbsp;code might be not restored from obfuscating&nbsp;or might not even be sent to the requester, thus leaving scraping efforts thwarted.</p> 
        <h3>The ways to break thru&nbsp;anti-scrape JS logic</h3> 
        <ol> 
         <li><b>Simulate </b><b>real browser</b> for scrape. The most used&nbsp;tools for that are Selenium and iMacros. There is a&nbsp;recently emerged web IDE called&nbsp;<a title="Writing next generation scraping scripts with Web Robots IDE" href="http://scraping.pro/writing-next-generation-scraping-scripts-with-web-robots-ide/" target="_blank">WebRobots</a> that drives Chrome browser thru JaveScript robots. Read how to <a href="/use-headless-firefox-scraping-linux/" target="_blank">leverage Selenium</a> with headless FF browser.</li> 
         <li>Use <b>JS plugins</b> to execute JS&nbsp;logic and return needed value to the target server to unlock content for scrape. I found <a href="http://php.net/manual/en/class.v8js.php">v8js</a>&nbsp;PHP&nbsp;plugin and its Python analog – <a href="https://code.google.com/p/pyv8/">PyV8</a>.</li> 
         <li>Use the specific libraries (toolkits) as add-ons for the scripting languages. Below is an&nbsp;example of using a library with python in web scraping.</li> 
        </ol> 
        <h3>Example use of <em>dryscrape</em> library</h3> 
        <p>I wanna show you the code example of how to leverage <a href="http://dryscrape.readthedocs.org/en/latest/installation.html" target="_blank">dryscrape library</a>&nbsp;to evaluate scraped JS for JavaScript protected content.</p>
