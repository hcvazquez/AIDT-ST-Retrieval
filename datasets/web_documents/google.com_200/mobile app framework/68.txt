          <p class="wp-caption-text">React Native Tutorial: Build native iOS applications with JavaScript.</p>

         <p><em>Update note:</em> This tutorial has been updated to React Native 0.46, Xcode 9, and iOS 11 by Christine Abernathy. The previous tutorial was written by iOS Team member Tom Elliot.</p> 
         <p>In this React Native tutorial you’ll learn how to build native apps based on the hugely popular <a href="https://facebook.github.io/react/">React</a> JavaScript library.</p> 
         <p>What makes React Native different from other frameworks such as <a href="http://phonegap.com">PhoneGap (Apache Cordova)</a> or <a href="http://www.appcelerator.com/mobile-app-development-products/">Appcelerator Titanium</a>, that use JavaScript to create iOS apps?</p> 
         <ol> 
          <li>(Unlike PhoneGap) with React Native your code may be written in JavaScript but the app’s UI is fully native. It doesn’t have the drawbacks typically associated with a hybrid HTML5 app.</li> 
          <li>Additionally (unlike Titanium), React introduces a novel, radical and highly functional approach to constructing user interfaces. Your UI is simply a function of the current app state.</li> 
         </ol> 
         <p>React Native brings the <a href="http://facebook.github.io/react/">React</a> paradigm to mobile app development. It’s goal isn’t to write the code once and run it on any platform. The goal is to learn-once (the React way) and write-anywhere. An important distinction to make. This tutorial only covers iOS, but once you’ve learned the concepts here you could port that knowledge into creating an Android app very quickly.</p> 
         <p>If you have only ever written applications in Swift, you might not be particularly excited about the prospect of using JavaScript. However, the functional programming aspect should pique your interest!</p> 
         <p>Swift has introduced more functional ways to encode algorithms, as well as techniques that encourage transformation and immutability. However, the way in which you construct your UI is very much the same as it was when developing with Objective-C: it’s still UIKit-based and imperative.</p> 
         <p>The community has even added tools such as <a href="https://expo.io/">Expo</a> and <a href="https://github.com/react-community/create-react-native-app">Create React Native App</a> to help you quickly build React Native apps without having to touch Xcode or Android Studio!</p> 
         <p>This React Native tutorial takes you through the process of building an iOS app for searching UK property listings:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Flow-2.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Flow-2-650x382.png" alt="React Native tutorial finished app" width="650" height="382" class="aligncenter size-large wp-image-166376" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Flow-2-650x382.png 650w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Flow-2-480x282.png 480w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Flow-2.png 1248w" sizes="(max-width: 650px) 100vw, 650px"></a></p> 
         <p>Don’t worry if you’ve never written any JavaScript or used the CSS-like properties you’ll see. This tutorial will guide you through every step and provide resources where you can learn more.</p> 
         <p>Ready to get going? Read on!</p> 
         <h2>Getting Started</h2> 
         <p>React Native uses <a href="https://nodejs.org/">Node.js</a>, a JavaScript runtime, to build your JavaScript code. If you don’t already have Node.js installed, it’s time to get it!</p> 
         <p>First install Homebrew using the instructions on the <a href="http://brew.sh/">Homebrew website</a>. Then install Node.js by executing the following in a Terminal window:</p> 
         <pre lang="console">
brew install node
</pre> 
         <p>Next, use <code>homebrew</code> to install <a href="https://facebook.github.io/watchman/">watchman</a>, a file watcher from Facebook:</p> 
         <pre lang="console">
brew install watchman
</pre> 
         <p>This is used by React Native to figure out when your code changes and rebuild accordingly. It’s like having Xcode do a build each time you save your file.</p> 
         <p>Next use npm to install the React Native Command Line Interface (CLI) tool:</p> 
         <pre lang="console">
npm install -g react-native-cli
</pre> 
         <p>This uses the <a href="https://www.npmjs.com/">Node Package Manager</a> to fetch the CLI tool and install it globally; npm is similar in function to <a href="https://cocoapods.org/">CocoaPods</a> or <a href="https://github.com/Carthage/Carthage">Carthage</a> and is packaged with Node.js.</p> 
         <p>Navigate to the folder where you would like to develop your app and run the following in Terminal:</p> 
         <pre lang="console">
react-native init PropertyFinder
</pre> 
         <p>This uses the CLI tool to create a starter project containing everything you need to build and run a React Native app.</p> 
         <div class="note">
          If you get complaints about the version of node, make sure the one installed by brew is the one in use. Run 
          <code>brew link --overwrite node</code> in the terminal.
         </div> 
         <p>In Terminal, run:</p> 
         <pre lang="console">
cd PropertyFinder
</pre> 
         <p>In the created folders and files you will find a few items of note:</p> 
         <ul> 
          <li><em>node_modules</em> is a folder which contains the React Native framework</li> 
          <li><em>index.ios.js</em> is the skeletal app created by the CLI tool</li> 
          <li><em>ios</em> is a folder containing an Xcode project and the code required to <em>bootstrap</em><br> your application</li> 
          <li>Android counterparts to the above, although you won’t be touching those in this tutorial.</li> 
         </ul> 
         <p>Open the <em>PropertyFinder.xcodeproj</em> in the <em>ios</em> folder with Xcode, then build and run. The simulator will start and display the following:</p>

          <p>Many of the warnings may be related to unused parameters. You may also see null pointer logic errors in the <em>yoga</em> library that are actually not errors.</p>

         <p>You might also have noticed that a terminal window has popped up, displaying the following:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/RNPackager.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/RNPackager-480x235.png" alt="React Native packager terminal window" width="480" height="235" class="bordered aligncenter size-medium wp-image-166400" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/RNPackager-480x235.png 480w, https://koenig-media.raywenderlich.com/uploads/2017/07/RNPackager.png 564w" sizes="(max-width: 480px) 100vw, 480px"></a></p>

         <h2>React Native Basics</h2> 
         <p>In this section, you’ll be introduced to React Native basics as you begin working on PropertyFinder.</p> 
         <p>Open <em>index.ios.js</em> in your text editor of choice and take a look at the structure of the code in the file:</p> 
         <pre lang="javascript">
import React, { Component } from 'react'; // 1
import {
  AppRegistry,
  StyleSheet,
  Text,
  View
} from 'react-native';

export default class PropertyFinder extends Component { ... } // 2

const styles = StyleSheet.create({ ... }); // 3

AppRegistry.registerComponent('PropertyFinder', () =&gt; PropertyFinder); // 4
</pre> 
         <p>Let’s go through the code step-by-step:</p> 
         <ol> 
          <li>Import the required modules.</li> 
          <li>Defines the component that represents the UI.</li> 
          <li>Creates a style object that controls the component’s layout and appearance.</li> 
          <li>Registers the component that handles the app’s entry point.</li> 
         </ol> 
         <p>Take a closer look at this import statement:</p> 
         <pre lang="javascript">
import React, { Component } from 'react';
</pre> 
         <p>This uses the ECMAScript 6 (ES6) import syntax to load the <code>react</code> module and assign it to a variable called <code>React</code>. This is roughly equivalent to linking and importing libraries in Swift. It also uses what’s called a <em>destructuring assignment</em> to bring in the <code>Component</code> object. Destructuring lets you extract multiple object properties and assign them to variables using a single statement.</p> 
         <div class="note">
          <em>Note:</em> For more information about ES6 modules I’d recommend reading 
          <a href="http://2ality.com/2014/09/es6-modules-final.html">this blog post by Dr. Axel Rauschmayer</a>. 
         </div> 
         <p>ES6 is a much nicer way to write JavaScript, supporting features like default parameters, classes, arrow functions, and destructuring assignments. Apple has supported ES6 since iOS 10, but older browsers may not be compatible with it. React Native uses a tool called <a href="https://babeljs.io">Babel</a> to automatically translate modern JavaScript into compatible legacy JavaScript where necessary.</p> 
         <p>Back to <em>index.ios.js</em>, check out the class definition:</p> 
         <pre lang="javascript">
export default class PropertyFinder extends Component
</pre> 
         <p>This defines a class which extends a React <code>Component</code>. The <code>export default</code> class modifier makes the class “public”, allowing it to be used in other files.</p> 
         <p>It’s time to start building your app.</p> 
         <p>In <em>index.ios.js</em>, add the following at the top of the file, just before the import statements:</p> 
         <pre lang="javascript">
'use strict';
</pre> 
         <p>This enables <em>Strict Mode</em>, which adds improved error handling and disables some less-than-ideal JavaScript language features. In simple terms, it makes JavaScript better!</p> 
         <p>Inside the <code>PropertyFinder</code> class replace <code>render()</code> with the following:</p> 
         <pre lang="javascript">
render() {
  return React.createElement(Text, {style: styles.description}, "Search for houses to buy!");
}
</pre> 
         <p><code>PropertyFinder</code> extends <code>React.Component</code>, the basic building block of the React UI. Components contain immutable properties, mutable state variables and expose a method for rendering. Your current application is quite simple and only requires a render method.</p> 
         <p>React Native components are not UIKit classes; instead they are a lightweight equivalent. The framework takes care of transforming the tree of React components into the required native UI.</p> 
         <p>Next, replace the <code>const styles</code> statement with the following:</p> 
         <pre lang="javascript">
const styles = StyleSheet.create({
  description: {
    fontSize: 18,
    textAlign: 'center',
    color: '#656565',
    marginTop: 65,
  },
});
</pre> 
         <p>This defines a single style that you’ve applied to the description text. If you’ve done any web development before, you’ll probably recognize those property names. The React Native StyleSheet class used to style the application UI is similar to the <a href="https://developer.mozilla.org/en-US/docs/Web/CSS">Cascading Style Sheets (CSS)</a> used on the Web.</p> 
         <p>Save your changes to <em>index.ios.js</em> and return to the simulator. Press <em>Cmd+R</em>, and you’ll see your fledgling property search app starting to take shape:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-281x500.png" alt="First signs of the React Native tutorial app" width="281" height="500" class="bordered aligncenter size-large wp-image-166275 bordered" srcset="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics.png 1242w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <p>That’s a JavaScript application running in the simulator, rendering a native UI, without a browser in sight!</p> 
         <p>Still don’t trust me? :] Verify it for yourself: within Xcode, select <em>Debug\View Debugging\Capture View Hierarchy</em> and take a look at the native view hierarchy. You will see no <code>UIWebView</code> instances anywhere! Your text is being displayed in a view called <code>RCTText</code>. But what is that? Back in Xcode select <em>File\Open Quickly…</em> and type in <code>RCTView.h</code>. Notice <code>RCTView</code> inherits directly from <code>UIView</code>. Neat!</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/06/ViewDebugging-1.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/06/ViewDebugging-1-480x236.png" alt="React Native tutorial app's view hierarchy" width="480" height="236" class="bordered aligncenter size-medium wp-image-166276" srcset="https://koenig-media.raywenderlich.com/uploads/2017/06/ViewDebugging-1-480x236.png 480w, https://koenig-media.raywenderlich.com/uploads/2017/06/ViewDebugging-1-650x319.png 650w, https://koenig-media.raywenderlich.com/uploads/2017/06/ViewDebugging-1.png 698w" sizes="(max-width: 480px) 100vw, 480px"></a></p> 
         <p>Curious as to how it all works? In Xcode open <em>AppDelegate.m</em> and locate <code>application:didFinishLaunchingWithOptions:</code>. This method constructs a <code>RCTRootView</code>, which loads the JavaScript application from the index.ios JavaScript file and renders the resultant view.</p> 
         <p>The Terminal window that was opened when you ran this application started a packager and server that allows your JavaScript code to be fetched, by default on port 8081. For example:</p> 
         <pre>
http://localhost:8081/index.ios.bundle
</pre> 
         <p>Open this URL in Safari; you’ll see the JavaScript code for your app. You can find your “Search for houses to buy!” description code embedded among the React Native framework.</p> 
         <p>When your app starts, this code is loaded and executed by the JavaScriptCore framework. In the case of your application, it loads the <code>PropertyFinder</code> component, then constructs the native UIKit view.</p> 
         <h2>Using JSX</h2> 
         <p>Your current application uses <code>React.createElement</code> to construct the simple UI for your application, which React turns into the native equivalent. While your JavaScript code is perfectly readable in its present form, a more complex UI with nested elements would rapidly become quite a mess.</p> 
         <p>Make sure the app is still running, then return to your text editor to edit <em>index.ios.js</em>. Modify the body of <code>render</code> to be the following:</p> 
         <pre lang="javascript">
return &lt;Text style={styles.description}&gt;Search for houses to buy! (Again)&lt;/Text&gt;;
</pre> 
         <p>This is <a href="http://facebook.github.io/react/docs/jsx-in-depth.html">JSX</a>, or JavaScript syntax extension, which mixes HTML-like syntax directly in your JavaScript code; if you’re already a web developer, this should feel rather familiar. You’ll use JSX throughout this article.</p> 
         <p>Save your changes to <em>index.ios.js</em> and return to the simulator. Press <em>Cmd+R</em>, and you’ll see your application refresh to display the updated message:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/Simulator-Screen-Shot-15-Jul-2017-17.46.06.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/Simulator-Screen-Shot-15-Jul-2017-17.46.06-281x500.png" alt="React Native tutorial app with updated message" width="281" height="500" class="aligncenter size-large wp-image-166786 bordered" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/Simulator-Screen-Shot-15-Jul-2017-17.46.06-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/07/Simulator-Screen-Shot-15-Jul-2017-17.46.06-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/07/Simulator-Screen-Shot-15-Jul-2017-17.46.06.png 1242w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <p>Re-running a React Native application is really as simple as refreshing a web browser! :] Note that this will only reflect changes made to your JavaScript files – native code or resource changes will require a rebuild in Xcode. </p> 
         <p>You can even skip having to refresh the app by enabling live reload. Press <em>Cmd+D</em> in the simulator then select <em>Enable Live Reload</em>:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/06/EnableLiveReload.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/06/EnableLiveReload-281x500.png" alt="Enabling live reload in a React Native iOS app" width="281" height="500" class="bordered aligncenter size-large wp-image-166278" srcset="https://koenig-media.raywenderlich.com/uploads/2017/06/EnableLiveReload-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/06/EnableLiveReload-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/06/EnableLiveReload.png 875w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <p>In <em>index.ios.js</em>, modify the <code>render</code> method’s body to the following:</p> 
         <pre lang="javascript">
return &lt;Text style={styles.description}&gt;Search for houses to buy!&lt;/Text&gt;;
</pre> 
         <p>Save your changes. Note that the simulator automatically refreshes to reflect your changes:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-281x500.png" alt="Live reload in a React Native iOS app" width="281" height="500" class="bordered aligncenter size-large wp-image-166275" srcset="https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/06/PropertyFinder-Basics.png 1242w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <h2>Adding Navigation</h2> 
         <p>PropertyFinder uses the standard stack-based navigation experience provided by UIKit’s navigation controller. It’s time to add this behavior.</p> 
         <p>In <em>index.ios.js</em>, find the import statements near the top and add a comma following the <code>View</code> destructuring assignment. Then add the following below it:</p> 
         <pre lang="javascript">
NavigatorIOS,
</pre> 
         <p>This brings in <code>NavigatorIOS</code> that you’ll use for navigation.</p> 
         <p>Next, replace the <code>PropertyFinder</code> class definition with the following:</p> 
         <pre lang="javascript">
class SearchPage extends Component {
</pre> 
         <p>You’ve just renamed the class. You’ll see an error as a result, but will clean that up next. </p> 
         <p>Add the following class below the <code>SearchPage</code> component:</p> 
         <pre lang="javascript">
class PropertyFinder extends Component {
  render() {
    return (
      &lt;NavigatorIOS
        style={styles.container}
        initialRoute={{
          title: 'Property Finder',
          component: SearchPage,
        }}/&gt;
    );
  }
}
</pre> 
         <p>This constructs a navigation controller, applies a style and sets the initial route to the <code>SearchPage</code> component. In web development, <em>routing</em> is a technique for defining the navigation structure of an application, where pages — or <i>routes</i> — are mapped to URLs.</p> 
         <p>Within the same file, add the following to the styles list under the closing brace of the <code>description</code> style:</p> 
         <pre lang="javascript">
container: {
  flex: 1,
},
</pre> 
         <p>This tells the component using this style to fill all available space. This ensures that the component’s children are visible.</p>

         <h2>Styling with Flexbox</h2> 
         <p>So far, you’ve seen basic CSS properties that deal with margins, paddings and color. However, you might not be familiar with <em>Flexbox</em>, a more recent addition to the CSS specification that’s useful for handling complex layout across different screen sizes.</p> 
         <p>React Native uses the <a href="https://github.com/facebook/yoga">Yoga</a> library under the hood to drive layout. Yoga is a C implementation of Flexbox and it includes bindings for Swift, Objective-C, Java (for Android), and C# (for .NET).</p> 
         <p>Generally you use a combination of <code>flexDirection</code>, <code>alignItems</code>, and <code>justifyContent</code> Yoga properties to manage your layout. </p> 
         <p>So far, your layout has a container with two children arranged vertically:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/FlexStack-2.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/FlexStack-2-480x214.png" alt="Flexbox elements arranged vertically and centered horizontally" width="480" height="214" class="bordered aligncenter size-medium wp-image-166368" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/FlexStack-2-480x214.png 480w, https://koenig-media.raywenderlich.com/uploads/2017/07/FlexStack-2-650x289.png 650w, https://koenig-media.raywenderlich.com/uploads/2017/07/FlexStack-2.png 921w" sizes="(max-width: 480px) 100vw, 480px"></a></p> 
         <p>This is due to the default <code>flexDirection</code> value of <code>column</code> being active. <code>flexDirection</code> helps define the <em>main axis</em> and <em>cross axis</em>. Your container’s main axis is vertical. It’s cross axis is therefore horizontal.</p> 
         <p><code>alignItems</code> determines the placement of children in the cross axis. Your app has set this value to <code>center</code>. This means the children are center-aligned.</p>

         <h2>Adding Component State</h2> 
         <p>A React component can manage its internal state through an object called, you guessed it, <code>state</code>. Whenever a component’s state changes, <code>render()</code> is called.</p> 
         <p>Within <em>SearchPage.js</em>, add the following code just before <code>render()</code>:</p> 
         <pre lang="javascript">
constructor(props) {
  super(props);
  this.state = {
    searchString: 'london'
  };
}
</pre> 
         <p>Your component now has a <code>state</code> variable, with <code>searchString</code> set to an initial value of <code>london</code>.</p> 
         <p>Within <code>render()</code>, change <code>TextInput</code> to the following:</p> 
         <pre lang="javascript">
&lt;TextInput
  style={styles.searchInput}
  value={this.state.searchString}
  placeholder='Search via name or postcode'/&gt;
</pre> 
         <p>This sets the <code>TextInput</code> value property — that is, the text displayed to the user — to the current value of the <code>searchString</code> state variable. This takes care of setting the initial state, but what happens when the user edits this text?</p> 
         <p>The first step is to create a method that acts as an event handler. Within the <code>SearchPage</code> class add the following method below the <code>constructor</code>:</p> 
         <pre lang="javascript">
_onSearchTextChanged = (event) =&gt; {
  console.log('_onSearchTextChanged');
  this.setState({ searchString: event.nativeEvent.text });
  console.log('Current: '+this.state.searchString+', Next: '+event.nativeEvent.text);
};
</pre> 
         <p>This defines a function using the <code>=&gt;</code> syntax. This is an arrow function, another <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions">recent addition to the JavaScript language</a> that provides a succinct syntax for creating anonymous functions.</p> 
         <p>The function takes the value from the native browser event’s <code>text</code> property and uses it to update the component’s state. It also adds some logging code that will make sense shortly.</p> 
         <div class="note"> 
          <p><em>Note:</em> JavaScript classes do not have access modifiers, so they have no concept of <code>private</code>. As a result you often see developers prefixing methods with an underscore to indicate that they should be considered private.</p>
         </div> 
         <p>To wire up this method so it gets called when the text changes, return to the <code>TextInput</code> field within the <code>render</code> method and add an <code>onChange</code> property so the tag looks like the following:</p> 
         <pre lang="javascript">
&lt;TextInput
  style={styles.searchInput}
  value={this.state.searchString}
  onChange={this._onSearchTextChanged}
  placeholder='Search via name or postcode'/&gt;
</pre> 
         <p>Whenever the user changes the text, you invoke the function supplied to <code>onChange</code>; in this case, it’s <code>_onSearchTextChanged</code>.</p> 
         <p>There’s one final step before you refresh your app again: add the following logging statement to the top of <code>render()</code>, just before <code>return</code>:</p> 
         <pre lang="javascript">
console.log('SearchPage.render');
</pre>

         <p>A React component state change triggers a UI update. This de-couples the rendering logic from state changes affecting the UI. Most other UI frameworks put the onus on you to update the UI based on state changes. Alternatively, the updates are done through an implicit link between the state and UI. For an example of the latter, see this article on implementing the <a href="http://www.raywenderlich.com/74106/mvvm-tutorial-with-reactivecocoa-part-1">MVVM pattern with ReactiveCocoa</a>.</p> 
         <p>At this point you’ve probably spotted a fundamental flaw in this concept. Yes, that’s right — performance!</p> 
         <p>Surely you can’t just throw away your entire UI and re-build it every time something changes? This is where React gets really smart. </p> 
         <p>Each time the UI renders itself, it takes the view tree returned by your render methods, and reconciles — or diffs — it with the current UIKit view. The output of this reconciliation process is a simple list of updates that React needs to apply to the current view. That means only the things that have actually changed will re-render!</p> 
         <p>You can wrap your head around all that later; you still have some work to do in the app.</p>

function urlForQueryAndPage(key, value, pageNumber) {
  const data = {
      country: 'uk',
      pretty: '1',
      encoding: 'json',
      listing_type: 'buy',
      action: 'search_listings',
      page: pageNumber,
  };
  data[key] = value;

  const querystring = Object.keys(data)
    .map(key =&gt; key + '=' + encodeURIComponent(data[key]))
    .join('&amp;');

  return 'https://api.nestoria.co.uk/api?' + querystring;
}
</pre>

It first creates the query string based on the parameters in <code>data</code>. Then it transforms the data into <code>name=value</code> pairs separated by ampersands. Finally, it calls the <a href="http://www.nestoria.co.uk/help/api">Nestoria API</a> to return the property listings.</p>

         <h2>Performing an API Request</h2> 
         <p>Still within <em>SearchPage.js</em>, update the initial state in the class constructor to add a <code>message</code> variable to the end of the list:</p> 
         <pre lang="javascript">
message: '',
</pre> 
         <p>Within <code>render</code>, add the following to the bottom of your UI, right after the spinner:</p> 
         <pre lang="javascript">
&lt;Text style={styles.description}&gt;{this.state.message}&lt;/Text&gt;
</pre> 
         <p>You’ll use this to display a range of messages to the user.</p> 
         <p>Add the following code to the end of <code>_executeQuery</code>:</p> 
         <pre lang="javascript">
fetch(query)
  .then(response =&gt; response.json())
  .then(json =&gt; this._handleResponse(json.response))
  .catch(error =&gt;
     this.setState({
      isLoading: false,
      message: 'Something bad happened ' + error
   }));
</pre> 
         <p>This makes use of the <code>fetch</code> function, which is <a href="https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API">part of the Fetch API</a>. The asynchronous response is returned as a <a href="http://www.html5rocks.com/en/tutorials/es6/promises/">Promise</a>. The success path calls <code>_handleResponse</code> which you’ll define next, to parse the JSON response.</p> 
         <p>Add the following function to <code>SearchPage</code>:</p> 
         <pre lang="javascript">
_handleResponse = (response) =&gt; {
  this.setState({ isLoading: false , message: '' });
  if (response.application_response_code.substr(0, 1) === '1') {
    console.log('Properties found: ' + response.listings.length);
  } else {
    this.setState({ message: 'Location not recognized; please try again.'});
  }
};
</pre> 
         <p>This clears <code>isLoading</code> and logs the number of properties found if the query was successful.</p> 
         <div class="note"> 
          <p><em>Note:</em> Nestoria has <a href="http://www.nestoria.co.uk/help/api-return-codes">a number of non-1** response codes</a> that are potentially useful. For example, 202 and 200 return a list of best-guess locations.</p>
         </div> 
         <p>Save your changes, head back to the simulator and press <em>Go</em>. You should see an Xcode console log message saying that 20 properties (the default result size) were found:</p> 
         <pre lang="console">
2017-07-04 00:34:12.425 [info][tid:com.facebook.React.JavaScript] Properties found: 20
</pre> 
         <p>Also note that when this message is logged, the spinner goes away.</p> 
         <p>It’s time to see what those 20 properties actually look like!</p> 
         <h2>Displaying the Results</h2> 
         <p>Create a new file <em>SearchResults.js</em>, and add the following:</p> 
         <pre lang="javascript">
'use strict';

import React, { Component } from 'react'
import {
  StyleSheet,
  Image,
  View,
  TouchableHighlight,
  FlatList,
  Text,
} from 'react-native';
</pre> 
         <p>This imports the relevant modules you’ll use.</p> 
         <p>Next, add the component:</p> 
         <pre lang="javascript">
export default class SearchResults extends Component {
  _keyExtractor = (item, index) =&gt; index;

  _renderItem = ({item}) =&gt; {
    return (
      &lt;TouchableHighlight
        underlayColor='#dddddd'&gt;
        &lt;View&gt;
          &lt;Text&gt;{item.title}&lt;/Text&gt;
        &lt;/View&gt;
      &lt;/TouchableHighlight&gt;
    );
    
  };

  render() {
    return (
      &lt;FlatList
        data={this.props.listings}
        keyExtractor={this._keyExtractor}
        renderItem={this._renderItem}
      /&gt;
    );
  }
}
</pre> 
         <p>The above code makes use of a more specialized component — <code>FlatList</code> — which displays rows of data within a scrolling container, similar to <code>UITableView</code>. Here’s a look at the <code>FlatList</code> properties:</p> 
         <ul> 
          <li><code>data</code> provides the data to display</li> 
          <li><code>keyExtractor</code> provides a unique key that React uses for efficient list item management</li> 
          <li><code>renderItem</code> specifies how the UI is rendered for each row</li> 
         </ul> 
         <p>Save your new file.</p> 
         <p>Add the following to <em>SearchPage.js</em> just beneath the <code>import</code> statements:</p> 
         <pre lang="javascript">
import SearchResults from './SearchResults';
</pre> 
         <p>This brings in the newly added <code>SearchResults</code> class.</p> 
         <p>In <code>_handleResponse</code>, replace the <code>console.log</code> statement with the following:</p> 
         <pre lang="javascript">
this.props.navigator.push({
  title: 'Results',
  component: SearchResults,
  passProps: {listings: response.listings}
});
</pre> 
         <p>This navigates to your newly added <code>SearchResults</code> component and passes in the listings from the API request. Using the <code>push</code> method ensures the search results are pushed onto the navigation stack, which means you’ll get a <em>Back</em> button to return to the root.</p> 
         <p>Save your changes, head back to the simulator and press <em>Go</em>. You’ll be greeted by a list of properties:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Raw.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Raw-281x500.png" alt="React Native tutorial app results screen" width="281" height="500" class="bordered aligncenter size-large wp-image-166308" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Raw-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Raw-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Raw.png 1242w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <p>It’s great to see the property listings, but that list is a little drab. Time to liven things up a bit.</p> 
         <h2>A Touch of Style</h2> 
         <p>Add the following style definition at the end of <em>SearchResults.js</em>:</p> 
         <pre lang="javascript">
const styles = StyleSheet.create({
  thumb: {
    width: 80,
    height: 80,
    marginRight: 10
  },
  textContainer: {
    flex: 1
  },
  separator: {
    height: 1,
    backgroundColor: '#dddddd'
  },
  price: {
    fontSize: 25,
    fontWeight: 'bold',
    color: '#48BBEC'
  },
  title: {
    fontSize: 20,
    color: '#656565'
  },
  rowContainer: {
    flexDirection: 'row',
    padding: 10
  },
});
</pre> 
         <p>This defines all the styles that you are going to use to render each row.</p> 
         <p>Add a new component representing a row by adding the following just under the import statements:</p> 
         <pre lang="javascript">
class ListItem extends React.PureComponent {
  _onPress = () =&gt; {
    this.props.onPressItem(this.props.index);
  }

  render() {
    const item = this.props.item;
    const price = item.price_formatted.split(' ')[0];
    return (
      &lt;TouchableHighlight
        onPress={this._onPress}
        underlayColor='#dddddd'&gt;
        &lt;View&gt;
          &lt;View style={styles.rowContainer}&gt;
            &lt;Image style={styles.thumb} source={{ uri: item.img_url }} /&gt;
            &lt;View style={styles.textContainer}&gt;
              &lt;Text style={styles.price}&gt;{price}&lt;/Text&gt;
              &lt;Text style={styles.title}
                numberOfLines={1}&gt;{item.title}&lt;/Text&gt;
            &lt;/View&gt;
          &lt;/View&gt;
          &lt;View style={styles.separator}/&gt;
        &lt;/View&gt;
      &lt;/TouchableHighlight&gt;
    );
  }
}
</pre> 
         <p>This manipulates the returned price, which is in the format <em>300,000 GBP</em>, to remove the GBP suffix. Then it renders the row UI using techniques that you are by now quite familiar with. Of note, an <code>Image</code> is added to the row and is loaded from a returned URL (<code>item.img_url</code>) which React Native decodes off the main thread.</p> 
         <div class="note">
          You may have noticed that this component extends 
          <code>React.PureComponent</code>. React re-renders a 
          <code>Component</code> if its props or state changes. React only re-renders a 
          <code>PureComponent</code> if a shallow compare of the state and props shows changes. Used under the right conditions, this can give your app a performance boost.
         </div> 
         <p>Now replace <code>_renderItem</code> with the following:</p> 
         <pre lang="javascript">
_renderItem = ({item, index}) =&gt; (
  &lt;ListItem
    item={item}
    index={index}
    onPressItem={this._onPressItem}
  /&gt;
);

_onPressItem = (index) =&gt; {
  console.log("Pressed row: "+index);
};
</pre> 
         <p><code>_onPressItem</code> is passed into <code>ListItem</code> to handle a row selection. This design pattern is equivalent to a callback. In this callback, the index for the selected row is logged.</p> 
         <p>Save your work, head back to the simulator, press <em>Go</em>, and check out your results:</p> 
         <p><a href="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Styled.png"><img src="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Styled-281x500.png" alt="React Native tutorial finished results screen" width="281" height="500" class="bordered aligncenter size-large wp-image-166310" srcset="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Styled-281x500.png 281w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Styled-180x320.png 180w, https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-Results-Styled.png 1242w" sizes="(max-width: 281px) 100vw, 281px"></a></p> 
         <p>That looks a lot better — although it’s a wonder anyone can afford to live in London!</p> 
         <p>Tap the first row and check that the Xcode console reflects the selection:</p> 
         <pre lang="console">
2017-07-04 01:29:19.799 [info][tid:com.facebook.React.JavaScript] Pressed row: 0
</pre> 
         <p>Try tapping other listings or searching other locations in the UK.</p> 
         <h2>Where To Go From Here?</h2> 
         <p>Congratulations on completing this React Native tutorial! You can <a href="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-RN46-final.zip">find the complete project here</a> if you want to compare notes. :]</p> 
         <p>As a challenge, try showing a property’s details when the user selects one from the search list. You can check out <a href="https://koenig-media.raywenderlich.com/uploads/2017/07/PropertyFinder-RN46-challenge.zip">the challenge solution</a> if you get stuck.</p> 
         <div class="note">
          Before opening the finished project or the challenge solution, first run 
          <code>npm install</code> via the Terminal in the root folder of the project.
         </div> 
         <p>Check out the <a href="http://facebook.github.io/react-native/">React Native’s source code</a> if you’re curious. I suggest taking a look at this <a href="https://babeljs.io/learn-es2015/">ES6 resource</a> to continue brushing up on modern JavaScript.</p> 
         <p>If you’re a web developer, you’ve seen how to use JavaScript to easily create a native app. If you’re a native app developer, you’ve gained some appreciation for React Native’s fast iteration cycle. Whether you decide to use React Native in a future app or simply stick with Swift, I hope you’ve learned some interesting principles to apply to your next project.</p>
