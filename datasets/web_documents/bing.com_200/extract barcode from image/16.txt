     <h1><a id="user-content-node-pdf" class="deep-link" href="#node-pdf">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>Node PDF</h1> 
     <p class="package-description-redundant">Node PDF is a set of tools that takes in PDF files and converts them to usable formats for data processing. The library supports both extracting text from searchable pdf files as well as performing OCR on pdfs which are just scanned images of text</p> 
     <p class="badge-only"><a href="https://travis-ci.org/nisaacson/pdf-extract"><img src="https://travis-ci.org/nisaacson/pdf-extract.png" alt="Build Status" class="badge"></a></p> 
     <h2><a id="user-content-installation" class="deep-link" href="#installation">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>Installation</h2> 
     <p>To begin install the module.</p> 
     <p class="badge-only"><code>npm install pdf-extract</code></p> 
     <p>After the library is installed you will need the following binaries accessible on your path to process pdfs.</p> 
     <ul> 
      <li>pdftk 
       <ul> 
        <li>pdftk splits multi-page pdf into single pages.</li> 
       </ul> </li> 
      <li>pdftotext 
       <ul> 
        <li>pdftotext is used to extract text out of searchable pdf documents</li> 
       </ul> </li> 
      <li>ghostscript 
       <ul> 
        <li>ghostscript is an ocr preprocessor which convert pdfs to tif files for input into tesseract</li> 
       </ul> </li> 
      <li>tesseract 
       <ul> 
        <li>tesseract performs the actual ocr on your scanned images</li> 
       </ul> </li> 
     </ul> 
     <h3><a id="user-content-osx" class="deep-link" href="#osx">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>OSX</h3> 
     <p>To begin on OSX, first make sure you have the homebrew package manager installed.</p> 
     <p><strong>pdftk</strong> is not available in Homebrew. However a gui install is available here. <a href="http://www.pdflabs.com/docs/install-pdftk/">http://www.pdflabs.com/docs/install-pdftk/</a></p> 
     <p><strong>pdftotext</strong> is included as part of the <strong>poppler</strong> utilities library. <strong>poppler</strong> can be installed via homebrew</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>brew&nbsp;install&nbsp;poppler</span></span>
       </div></pre>
     </div> 
     <p><strong>ghostscript</strong> can be install via homebrew</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>brew&nbsp;install&nbsp;gs</span></span>
       </div></pre>
     </div> 
     <p><strong>tesseract</strong> can be installed via homebrew as well</p> 
     <p class="badge-only"><code>brew install tesseract</code></p> 
     <p>After tesseract is installed you need to install the alphanumeric config and an updated trained data file</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>cd&nbsp;&lt;root&nbsp;of&nbsp;this&nbsp;module&gt;</span></span>
       </div>
       <div class="line">
        <span class="text"><span>cp&nbsp;"./share/eng.traineddata"&nbsp;"/usr/local/Cellar/tesseract/3.02.02_3/share/tessdata/eng.traineddata"</span></span>
       </div>
       <div class="line">
        <span class="text"><span>cp&nbsp;"./share/dia.traineddata"&nbsp;"/usr/local/Cellar/tesseract/3.02.02_3/share/tessdata/dia.traineddata"</span></span>
       </div>
       <div class="line">
        <span class="text"><span>cp&nbsp;"./share/configs/alphanumeric"&nbsp;"/usr/local/Cellar/tesseract/3.02.02_3/share/tessdata/configs/alphanumeric"</span></span>
       </div></pre>
     </div> 
     <h3><a id="user-content-ubuntu" class="deep-link" href="#ubuntu">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>Ubuntu</h3> 
     <p><strong>pdftk</strong> can be installed directly via apt-get</p> 
     <div class="highlight bash">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>apt-get&nbsp;install&nbsp;pdftk</span></span>
       </div></pre>
     </div> 
     <p><strong>pdftotext</strong> is included in the <strong>poppler-utils</strong> library. To installer poppler-utils execute</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>apt-get&nbsp;install&nbsp;poppler-utils</span></span>
       </div></pre>
     </div> 
     <p><strong>ghostscript</strong> can be install via apt-get</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>apt-get&nbsp;install&nbsp;ghostscript</span></span>
       </div></pre>
     </div> 
     <p><strong>tesseract</strong> can be installed via apt-get. Note that unlike the osx install the package is called <strong>tesseract-ocr</strong> on Ubuntu, not <strong>tesseract</strong></p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>apt-get&nbsp;install&nbsp;tesseract-ocr</span></span>
       </div></pre>
     </div> 
     <p>For the OCR to work, you need to have the tesseract-ocr binaries available on your path. If you only need to handle ASCII characters, the accuracy of the OCR process can be increased by limiting the tesseract output. To do this copy the <em>alphanumeric</em> file included with this pdf-extract module into the <em>tess-data</em> folder on your system. Also the eng.traineddata included with the standard tesseract-ocr package is out of date. This pdf-extract module provides an up-to-date version which you should copy into the appropriate location on your system</p> 
     <div class="highlight">
      <pre class="editor editor-colors">
       <div class="line">
        <span class="text"><span>cd&nbsp;&lt;root&nbsp;of&nbsp;this&nbsp;module&gt;</span></span>

     <p>Important! You will have to add some variables to the PATH of your machine. You do this by right clicking your computer in file explorer, select Properties, select Advanced System Settings, Environment Variables. You can then add <strong>the folder that contains the executables</strong> to the path variable.</p> 
     <p><strong>pdftk</strong> can be installed using the PDFtk Server installer found here: <a href="https://www.pdflabs.com/tools/pdftk-server/">https://www.pdflabs.com/tools/pdftk-server/</a> It should autmatically add itself to the PATH, if not, the default install location is <em>"C:\Program Files (x86)\PDFtk Server\bin"</em></p> 
     <p><strong>pdftotext</strong> can be installed using the recompiled poppler utils for windows, which have been collected and bundled here: <a href="http://manifestwebdesign.com/2013/01/09/xpdf-and-poppler-utils-on-windows/">http://manifestwebdesign.com/2013/01/09/xpdf-and-poppler-utils-on-windows/</a> Unpack these in a folder, (example: <em>"C:\poppler-utils"</em>) and add this to the PATH.</p> 
     <p><strong>ghostscript</strong> for Windows can be found at: <a href="http://www.ghostscript.com/download/gsdnld.html">http://www.ghostscript.com/download/gsdnld.html</a> Make sure you download the General Public License and the correct version (32/64bit). Install it and go to the installation folder (default: <em>"C:\Program Files\gs\gs9.19"</em>) and go into the <strong>bin</strong> folder. Rename the <em>gswin64c</em> to <em>gs</em>, and add the bin folder to your PATH.</p> 
     <p><strong>tesseract</strong> can be build, but you can also download an older version which seems to work fine. Downloads at: <a href="https://sourceforge.net/projects/tesseract-ocr-alt/files/">https://sourceforge.net/projects/tesseract-ocr-alt/files/</a> Version tested is <em>tesseract-ocr-setup-3.02.02.exe</em>, the default install location is <em>"C:\Program Files (x86)\Tesseract-OCR"</em> and is also added to the PATH. Note, this is only when you've checked that it will install for everyone on the machine.</p> 
     <p>Everything should work after all this! If not, try restarting to make sure the PATH variables are correctly used. <strong>This setup was tested on a Windows 10 Pro N 64bit machine.</strong></p>
