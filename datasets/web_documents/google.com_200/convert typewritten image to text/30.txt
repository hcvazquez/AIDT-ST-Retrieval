         <h1 class="entry-title post-title">An Introduction To JavaScript Blobs and File Interface</h1> 
         <article>
          <div class="st-format-image-holder">
           <a href="http://i0.wp.com/qnimate.com/wp-content/uploads/2014/06/html5-file-api.jpg?fit=1030%2C9999" title="An Introduction To JavaScript Blobs and File Interface"><img src="http://i0.wp.com/qnimate.com/wp-content/uploads/2014/06/html5-file-api.jpg?fit=679%2C9999" class="size-original wp-post-image" alt="html5-file-api" data-hidpi="http://i0.wp.com/qnimate.com/wp-content/uploads/2014/06/html5-file-api.jpg?fit=1358%2C19998"></a>
          </div>

           <p></p>
           <div class="qnimate-post-series-post-content">
            <div>
             This post is a part 18 of 
             <a href="http://qnimate.com/post-series/advanced-javascript-apis/">Advanced JavaScript APIs</a> post series.
            </div>
           </div>
           <p></p>
           <p>In this article you will be learning all about JavaScript blobs. Every API in this post comes under HTML5 specification.</p>
           <hr>
           <h3><span id="What_is_a_Blob">What is a Blob?</span></h3>
           <p> A blob object represents a chuck of bytes that holds data of a file. But a blob is not a reference to a actual file, it may seem like it is.</p>
           <p>A blob has its size and MIME type just like a file has. Blob data is stored in the memory or filesystem depending on the browser and blob size. A blob can be used like a file wherever we use files.</p>
           <p>Most APIs for working with blobs are asynchronous. But synchronous versions of APIs are also available so that they can be used in <a href="http://qnimate.com/multithreading-in-html5-using-web-workers/">Web Workers</a>.</p>
           <p>Content of a blob can be read as <a href="http://qnimate.com/javascript-arraybuffers-and-typed-arrays">ArrayBuffer</a> and therefore it makes blobs very handy to store binary data.</p>
           <h3><span id="Creating_a_Blob">Creating a Blob</span></h3>
           <p> A blob can be created using <a href="https://developer.mozilla.org/en-US/docs/Web/API/Blob" class="external" rel="nofollow" target="_blank">Blob</a> class.</p>
           <div class="codecolorer-container javascript default" style="overflow:auto;white-space:nowrap;">
            <div class="javascript codecolorer">
             <span class="co1">//first arguement must be an regular array. The array can be of any javascript objects. Array can contain array to make it multi dimensional </span>
             <br> 
             <span class="co1">//second parameter must be a BlogPropertyBag object containing MIME property</span>
             <br> 
             <span class="kw2">var</span> myBlob 
             <span class="sy0">=</span> 
             <span class="kw2">new</span> Blob
             <span class="br0">(</span>
             <span class="br0">[</span>
             <span class="st0">"This is my blob content"</span>
             <span class="br0">]</span>
             <span class="sy0">,</span> 
             <span class="br0">{</span>type 
             <span class="sy0">:</span> 
             <span class="st0">"text/plain"</span>
             <span class="br0">}</span>
             <span class="br0">)</span>
             <span class="sy0">;</span>
            </div>
           </div>
           <p>In the above code we saw how we can insert data to a blob. We can read data from a blob using <a href="https://developer.mozilla.org/en-US/docs/Web/API/FileReader" class="external" rel="nofollow" target="_blank">FileReader</a> Class.</p>
           <p><iframe width="100%" height="300" src="http://jsfiddle.net/qnimate/jk3QR/10/embedded/js,html,result" allowfullscreen frameborder="0"></iframe></p>
           <h3><span id="Blob_URLs">Blob URLs</span></h3>
           <p> As we have file:// URLs, referencing to a real file in local filesystem. Similarly we have blob:// URLs referencing to an blob. blob:// URLs can be used almost wherever we use regular URLs.</p>
           <p>A blob:// URL to a blob can be obtained using the <b>createObjectURL</b> object.</p>
           <p><iframe width="100%" height="300" src="http://jsfiddle.net/qnimate/Va9KB/6/embedded/js,result" allowfullscreen frameborder="0"></iframe></p>
           <h3><span id="Remote_data_as_Blobs">Remote data as Blobs</span></h3>
           <p> We can retrieve remote files using AJAX and and store the file data inside a blob. AJAX API provides us a method to download and store remote files in form of blobs.</p>
           <p><iframe width="100%" height="300" src="http://jsfiddle.net/qnimate/5NvsD/4/embedded/js,result" allowfullscreen frameborder="0"></iframe></p>
           <p>We can get the blob content in an ArrayBuffer and then analyze the ArrayBuffer as binary data. This can be done using <b>FileReader.readAsArrayBuffer() method</b>.</p>
           <p><iframe width="100%" height="300" src="http://jsfiddle.net/qnimate/ZTHJB/2/embedded/js" allowfullscreen frameborder="0"></iframe></p>
           <h3><span id="File_Interface">File Interface</span></h3>
           <p> A <a href="https://developer.mozilla.org/en-US/docs/Web/API/File" class="external" rel="nofollow" target="_blank">File</a> object in JavaScript references an actual file in the local filesystem. This File object inherits all properties and methods from the Blob class. Although the File objects and Blob objects are different, they expose same methods and properties.</p>
           <p>There is no way to create a File object, some JavaScript API return references File objects.</p>
           <p>File object can be retrieved from a <a href="https://developer.mozilla.org/en-US/docs/Web/API/FileList" class="external" rel="nofollow" target="_blank">FileList</a> object returned as a result of a user selecting files using the &lt;input&gt; element or from a drag and drop operation’s <a href="http://qnimate.com/html5-drag-and-drop-api/">DataTransfer</a> object.</p>
           <p><iframe width="100%" height="300" src="http://jsfiddle.net/qnimate/wQuP9/3/embedded/js,result" allowfullscreen frameborder="0"></iframe></p>
           <hr>
           <p><strong>Conclusion</strong></p>
           <p> Blobs are very useful while working with binary remote files. A blob can be very large i.e., can contain audio and video data too. They can be created dynamically and using blob URLs they can be used as files. You can use them in many different ways to make them more useful. Thanks for reading.</p>
