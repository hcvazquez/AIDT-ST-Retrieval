           <p>In the past few years i have seen many Developers asking the question that “<strong>how they can send sms using JAVA/PHP/C#/Python and any other language ?</strong>“. I n the past i developed API for sending sms using some free API’s like way2sms, but that’s not the case anymore because they made it more secure. So i decided to find out the alternatives and i got a few ways which can help you but these solutions depends on your exact requirements. Let me ask you few questions first.</p> 
           <ol> 
            <li>&nbsp;Are you a student and working on Project ?</li> 
            <li>Are a Professional developer and working on some startup project ?</li> 
            <li>Are you working on a company Projects ?</li> 
           </ol> 
           <p>I asked these question because each of them have a different solutions.</p> 
           <h2>How to Send SMS completely free only&nbsp; applies for students</h2> 
           <p>This is easy but the only limitation imposes here is you have to verify each number on providers site, which will definitely work for a student because they need is this just for testing project and demoing their projects in the exam.</p> 
           <h3>Steps to follow:</h3> 
           <ol> 
            <li>Create your free <a href="https://www.twilio.com/" target="_blank">Twilio </a>Account</li> 
            <li>Get the number for your account, its completely free</li> 
            <li>Go to <a href="https://www.twilio.com/user/account/developer-tools/api-explorer/message-create" target="_blank">Api Explorer</a></li> 
            <li>Test the number using Api Explorer</li> 
            <li><a href="https://www.twilio.com/user/account/phone-numbers/verified" target="_blank">Verify Few Numbers</a> For testing</li> 
            <li>Copy the exact code for your need. They have code samples for Java, PHP, C#, Python, Ruby</li> 
           </ol>

           <h2>How To Send SMS using paid API’s</h2> 
           <p>if you are a professional developer or working for any company then you can afford to pay a very small amount for testing or deployment purpose. you can choose either the Paid version of&nbsp; <a href="https://www.twilio.com/" target="_blank">twilio</a> and follow the same process&nbsp; or you can use any other alternative provider’s API.</p> 
           <p>All the provider mostly use direct URL based API’s where you have to send some authentication code along with sms details and it goes pretty easy . you can achieve that Using CURL.</p> 
           <p>Please Share it, if you found this post helpful in your development process.</p>
