        <h2 id="audience">Audience</h2> 
        <p>This tutorial is designed to let you quickly start exploring and developing applications with the Google Cloud Natural Language API. It is designed for people familiar with basic programming, though even without much programming knowledge, you should be able to follow along. Having walked through this tutorial, you should be able to use the <a href="https://cloud.google.com/natural-language/docs/reference/rest/">Reference documentation</a> to create your own basic applications.</p> 
        <p>This tutorial steps through a Natural Language API application using Python code. The purpose here is not to explain the Python client libraries, but to explain how to make calls to the Natural Language API. Applications in Java and Node.js are essentially similar. Consult the Natural Language API <a href="https://cloud.google.com/natural-language/docs/samples">Samples</a> for samples in other languages (including this sample within the tutorial).</p> 
        <h2 id="prerequisites">Prerequisites</h2> 
        <p>This tutorial has several prerequisites:</p> 
        <ul> 
         <li>You've <a href="https://cloud.google.com/natural-language/docs/getting-started#set_up_a_project" target="_blank">set up a Cloud Natural Language API project</a> in the Google Cloud Platform Console.</li> 
         <li>You've set up your environment using <a href="https://cloud.google.com/natural-language/docs/common/auth#adc">Application Default Credentials</a>.</li> 
         <li>You have basic familiarity with <a href="https://www.python.org/">Python</a> programming.</li> 
         <li>You've installed <a href="https://pip.pypa.io/en/latest/installing/" target="_blank">pip</a>.</li> 
         <li>You've installed the <a href="https://cloud.google.com/natural-language/docs/reference/libraries" target="_blank" track-name="naturalLanguageClientLibraries" type-type="navigateTo">Google Cloud Client Library for Python</a></li> 
        </ul> 
        <h2 id="analyzing_document_sentiment">Analyzing document sentiment</h2> 
        <p>This tutorial walks you through a basic Natural Language API application, using an <code>analyzeSentiment</code> request, which performs sentiment analysis on text. Sentiment analysis attempts to determine the overall attitude (positive or negative) and is represented by numerical <code>score</code> and <code>magnitude</code> values. (For more information on these concepts, consult <a href="https://cloud.google.com/natural-language/docs/basics#sentiment-analysis-values">Natural Language Basics</a>.)</p> 
        <p>We'll show the entire code first. (Note that we have removed most comments from this code in order to show you how brief it is. We'll provide more comments as we walk through the code.)</p> 
        <aside class="note">
         <strong>Note:</strong>
         <span> To copy the code to your clipboard, click the copy widget that appears in the top-right of the code snippet when you mouseover the code snippet. Copy this code into a <code>sentiment-analysis.py</code> file within your development directory.</span>
        </aside>

          <pre class="prettyprint lang-py" track-metadata-position="GoogleCloudPlatform/python-docs-samples/language/sentiment/sentiment_analysis.py/master/sentiment_tutorial">"""Demonstrates how to make a simple call to the Natural Language API."""

import argparse

from google.cloud import language
from google.cloud.language import enums
from google.cloud.language import types


def print_result(annotations):
    score = annotations.document_sentiment.score
    magnitude = annotations.document_sentiment.magnitude

    for index, sentence in enumerate(annotations.sentences):
        sentence_sentiment = sentence.sentiment.score
        print('Sentence {} has a sentiment score of {}'.format(
            index, sentence_sentiment))

    print('Overall Sentiment: score of {} with magnitude of {}'.format(
        score, magnitude))
    return 0


def analyze(movie_review_filename):
    """Run a sentiment analysis request on text within a passed filename."""
    client = language.LanguageServiceClient()

    with open(movie_review_filename, 'r') as review_file:
        # Instantiates a plain text document.
        content = review_file.read()

    document = types.Document(
        content=content,
        type=enums.Document.Type.PLAIN_TEXT)
    annotations = client.analyze_sentiment(document=document)

    # Print the results
    print_result(annotations)


if __name__ == '__main__':
    parser = argparse.ArgumentParser(
        description=__doc__,
        formatter_class=argparse.RawDescriptionHelpFormatter)
    parser.add_argument(
        'movie_review_filename',
        help='The filename of the movie review you\'d like to analyze.')
    args = parser.parse_args()

    analyze(args.movie_review_filename)</pre> 
         </div> 
        </section>

         <li>Imports the libraries necessary to run the application</li> 
         <li>Takes a text file and passes it to the <code>main()</code> function</li> 
         <li>Reads the text file and makes a request to the service</li> 
         <li>Parses the response from the service and displays it to the user</li> 
        </ul>

        <h2 id="authenticating_to_the_api">Authenticating to the API</h2> 
        <p>Before communicating with the Natural Language API service, you need to authenticate your service using previously acquired credentials. Within an application, the simplest way to obtain credentials is to use <a href="https://cloud.google.com/natural-language/docs/common/auth#adc" target="_blank">Application Default Credentials</a> (ADC). By default, ADC will attempt to obtain credentials from the <code>GOOGLE_APPLICATION_CREDENTIALS</code> environment file, which should be set to point to your service account's JSON key file. (You should have set up your service account and environment to use ADC in the <a href="https://cloud.google.com/natural-language/docs/getting-started">Quickstart</a>. See <a href="https://cloud.google.com/natural-language/docs/common/auth#set_up_a_service_account" target="_blank">Setting Up a Service Account</a> for more information.)</p> 
        <p>The Google Cloud Client Library for Python automatically uses the application default credentials.</p>

        <h2 id="run_the_sample">Run the sample</h2> 
        <p>To run our sample, we'll test it on a set of (fake) movie reviews for the movie "Bladerunner."</p> 
        <ol> 
         <li> <p>Download the samples from Google Cloud Storage:</p> <p></p><pre class="lang-sh">
gsutil cp gs://cloud-samples-tests/natural-language/sentiment-samples.tgz .
</pre><p></p> </li> 
         <li> <p>Unzip those samples, which will create a "<strong>reviews</strong>" folder:</p> <p></p><pre class="lang-sh">
gunzip sentiment-samples.tgz
tar -xvf sentiment-samples.tar
</pre><p></p> </li> 
         <li> <p>Run our sentiment analysis on one of the specified files:</p> <p></p><pre class="lang-sh">
python sentiment-analysis.py reviews/bladerunner-pos.txt
Sentence 0 has a sentiment score of 0.8
Sentence 1 has a sentiment score of 0.9
Sentence 2 has a sentiment score of 0.8
Sentence 3 has a sentiment score of 0.2
Sentence 4 has a sentiment score of 0.1
Sentence 5 has a sentiment score of 0.4
Sentence 6 has a sentiment score of 0.3
Sentence 7 has a sentiment score of 0.4
Sentence 8 has a sentiment score of 0.2
Sentence 9 has a sentiment score of 0.9
Overall Sentiment: score of 0.5 with magnitude of 5.5
</pre><p></p> </li> 
        </ol> 
        <p>The above example would indicate a review that was relatively positive (score of <code>0.5</code>), and relatively emotional (magnitude of <code>5.5</code>).</p> 
        <p>Running analysis on the other examples should produce values similar to those shown below:</p> 
        <pre class="lang-sh">
python sentiment-analysis.py reviews/bladerunner-neg.txt
...
Overall Sentiment: score of -0.6 with magnitude of 3.3

python sentiment-analysis.py reviews/bladerunner-mixed.txt
...
Overall Sentiment: score of 0 with magnitude of 4.7

python sentiment-analysis.py reviews/bladerunner-neutral.txt
...
Overall Sentiment: score of -0.1 with magnitude of 1.8
</pre> 
        <p>Note that the magnitudes are all similar (indicating a relative equal amount of emotionally significant sentiment) except for the "neutral" case, which indicates a review with not very much emotional sentiment, either positive or negative. (For more information on sentiment scores and magnitude, and how to interpret these values, see <a href="https://cloud.google.com/natural-language/docs/basics#interpreting_sentiment_analysis_values">Interpreting Sentiment Analysis Values</a>.)</p> 
        <p>If you wish to explore sentiment analysis with more data, Stanford provides a dataset of <a class="external" href="http://www.imdb.com">IMDB</a> movie reviews. To retrieve these movie reviews:</p> 
        <ol> 
         <li>Download the <a href="http://ai.stanford.edu/~amaas/data/sentiment/aclImdb_v1.tar.gz">Large Movie Review dataset</a>.</li> 
         <li>Unzip the file into your working directory. The movie reviews are divided into <code>pos</code> and <code>neg</code> directories within <code>train</code> and <code>test</code> data directories, with each text file containing one movie review.</li> 
         <li>Run the <code>sentiment-analysis.py</code> tool on any of the movie review text files.</li> 
        </ol>
