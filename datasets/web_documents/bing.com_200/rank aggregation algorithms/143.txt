        <p align="justify"></p>
        <p><strong>Introduction<br> <br> </strong>Aggregation functions are very important for QlikView applications. These type of functions can only be used within a field list for loadiing and used with group by clause statements. These functions can be used to formulate a calculation.<br> <br> <img width="550" height="329" alt="" src="http://csharpcorner.mindcrackerinc.netdna-cdn.com/UploadFile/35fef0/aggregation-functions-in-qlikview/Images/fun.jpg" border="0"><br> <strong><br> Types of aggregation functions</strong></p> 
        <ul> 
         <li>Basic aggregation functions</li>
         <li>String&nbsp; aggregation functions</li>
         <li>Counter&nbsp; aggregation functions</li>
         <li>Advanced&nbsp; aggregation functions </li> 
        </ul> 
        <p><strong>Basic aggregation functions</strong><br> <strong><br> </strong>There are the following types of basic aggregation functions:</p> 
        <ul> 
         <li>sum()</li>
         <li>min()</li>
         <li>max()</li>
         <li>only()</li>
         <li>mode()</li>
         <li>firstsortedvalue() </li> 
        </ul> 
        <p><strong>Sum()</strong></p> 
        <p>This function returns a sum of expressions over a number of records as defined by a group by clause. If the <strong>distinct </strong>keyword occurs before the expression then all the duplicates will be discarded.<strong><br> <br> Syntax<br> </strong><br> sum([distinct]expression)<br> <br> <strong>Example<br> <br> </strong>Load Month, sum(Sales) as SalesPerMonth<br> from abc.xls group by month;</p> 
        <p><strong>Min()<br> <br> </strong>This function returns the minimum value of an expression encountered over a number of records as defined by a group by clause. By default the rank is 1, which is corresponds to the lowest value. If the rank is 3 then<strong> </strong>the second lowest value will be returned. If the rank is 3 then the third lowest value will be returned and so on.<br> <strong><br> Syntax<br> <br> </strong>min( expression[, rank] )<br> <br> <strong>Example</strong><br> <br> Load Month, min(Sales) as SmallestSalePerMonth from abc.xls group by Month;<br> Load Month, min(Sales, 2) as SecondSmallestSalePerMonth from abc.xls group by Month;<br> <br> <strong>Max()<br> <br> </strong>This function returns the maximum value of an expression encountered over a number of records as defined by a group by clause. By default the rank is 1, which corresponds to the highest value. If the rank is 3 then<strong> </strong>the second highest value will be returned. If the rank is 3 then the third highest value will be returned and so on.<br> <br> <strong>Syntax<br> <br> </strong>max( expression[, rank] )<br> <br> <strong>Example</strong><br> <br> Load Month, max(Sales) as HighestSalePerMonth from abc.xls group by Month;<br> Load Month, max(Sales, 2) as SecondHighestSalePerMonth from abc.xls group by Month;<br> <br> <strong>Only()<br> <br> </strong>If the expression is over a number of records and it is defined by a group by clause then it contains only one numeric value then only that value is returned otherwise<strong> </strong>&nbsp;something else is returned.<br> <br> <strong>Syntax<br> <br> </strong>only(expression )<br> <br> <strong>Example<br> <br> </strong>Load Month, only(Price) as OnlyPriceSoldFor from abc.xls group by Month;<br> <br> <strong>Mode()<br> <br> </strong>This functions returns the mode value, in other words the most common value of the expression over a number of records in the data as defined by a group by clause. If more than one value that is equally common occurs then the value is returned. The Mode function returns numeric values as well as text values.</p> 
        <p><strong>Syntax<br> <br> </strong>mode(expression )<strong><br> <br> Example</strong></p> 
        <p>Load Month, mode( ErrorNumber ) as MostCommonErrorNumber from abc.xls group by Month;<br> Load Month, mode( Product ) as ProductMostOftenSold from abc.xls group by Month;<br> <br> <strong>Firstsortedvalue()<br> <br> </strong>This function returns the first value of the expression sorted by corresponding sort-weight when the expression is executed over a number of records as defined by a group by clause. Sort-weight returns a numeric value where the lowest value will render the corresponding value of the expression to be sorted first.<br> <br> <strong>Syntax<br> <br> </strong>firstsortedvalue ([ distinct ] expression [, sort-weight [, n ]])<br> <br> <strong>Example</strong><br> <br> Load Customer,&nbsp;firstsortedvalue(PurchasedArticle, OrderDate) as FirstProductBought from abc.xls group by Customer;</p> 
        <p><strong>String Aggregation Functions<br> <br> </strong>There are the following types of string aggregation functions:</p> 
        <ul> 
         <li>MinString</li>
         <li>MaxString</li>
         <li>FirstValue</li>
         <li>LastValue</li>
         <li>Concat() </li> 
        </ul> 
        <p><strong>MinString()</strong><br> <br> This string function returns the first text value of an expression over a number of records in business data as defined by a group by clause. If no text value is found, then value is returned.<br> <br> <strong>Syntax<br> <br> </strong>MinString(expression )<br> <br> <strong>Example<br> </strong><br> Load Month, MinString(Month) as FirstSalesMonth from abc.xls group by year;<br> <br> <strong>MaxString<br> <br> </strong>This string function returns the last text value of an expression over a number of records in business data as defined by a group by clause. If no text value is found, then value is returned.<br> <br> <strong>Syntax<br> <br> </strong>MaxString(expression )<br> <br> <strong>Example<br> </strong><br> Load Month, MaxString(Month) as LastSalesMonth from abc.xls group by year;<br> <br> <strong>FirstValue()<br> <br> </strong>This script function returns the first value in the load order of an expression over a number of records in business data as defined by a group by clause. If no text value is found, then value is returned. This function is only available as a script function.<strong><br> <br> Syntax<br> <br> </strong>FirstValue(expression)<strong><br> <br> Example<br> <br> </strong>Load City, FirstValue(Name), as FirstName from abc.xls group by City;<br> <br> <strong>LastValue()<br> <br> </strong>This script function returns the last value in the load order of an expression over a number of records in business data as defined by a group by clause. If no text value is found, then value is returned. This function is only available as a script function.<strong><br> <br> Syntax<br> <br> </strong>LastValue(expression)<strong><br> <br> Example<br> <br> </strong>Load City, LastValue(Name), as LastName from abc.xls group by City;<br> <br> <strong>Concat()<br> <br> </strong>This script functions returns the aggregated string concatenation of all values of an expression iterated over a number of records in business data as defined by a group by clause statement. In this script function each value is separated by the string found in a delimiter. The order of concatenation is by sort-weight. Sort-weight returns a numeric value where the lowest value is sorted first. If distinct occurs before the expression, all duplicates will be discarded.<strong><br> <br> Syntax<br> <br> </strong>concat ([ distinct ] expression [, delimiter [, sort-weight]])<strong><br> <br> Example<br> <br> </strong>Load Department, concat(Name,';') as NameList from abc.xls group by Department;<br> <strong><br> Counter Aggregation Functions<br> </strong><br> There are the following types of counter aggregation functions:</p> 
        <ul> 
         <li>Count()</li>
         <li>NumericCount()</li>
         <li>TextCount()</li>
         <li>NullCount()</li>
         <li>MissingCount() </li> 
        </ul> 
        <p><strong>Count()</strong><br> <br> This function returns the count of an expression over a number of records in business data as defined by a group by clause. If the distinct occurs before the expression, all duplicates will be discarded.<br> <strong><br> Syntax<br> <br> </strong>count([distinct ] expression | * )<br> <strong><br> Example<br> </strong><br> Load Month, count(Sales) as NumberOfSalesPerMonth from abc.xls group by Month;<br> Load Month, count(distinct Customer) as CustomerBuyingPerMonth from abc.xls group by Month;<br> LoadMonth, count(*) as NumberOfRecordsPerMonth from abc.xls group per Month;<br> <br> <strong>NumericCount()<br> <br> </strong>This function returns the numeric count of an expression over a number of records as defined by a group by clause. If the distinct occurs before the expression, all duplicates will be discarded.<br> <strong><br> Syntax<br> <br> </strong>NumericCount([ distinct ] expression )<br> <strong><br> Example<br> <br> </strong>Load Month, NumericCount(Item) as NumberOfNumericItems from abc.xls group by Month;<br> <br> <strong>TextCount()<br> <br> </strong>This function returns the text count of an expression over a number of records as defined by a group by clause. If the distinct occurs before the expression, all duplicates will be discarded.<br> <br> <strong>Syntax<br> <br> </strong>TextCount([ distinct ] expression )<br> <strong><br> Example<br> <br> </strong>Load Month, TextCount(Item) as NumberOfTextItems from abc.xls group by Month;<br> <br> <strong>NullCount()<br> <br> </strong>This function returns the null count of an expression over a number of records as defined by a group by clause. If the distinct occurs before the expression, all duplicates will be discarded.<br> <br> <strong>Syntax<br> <br> </strong>NullCount([ distinct ] expression )<br> <strong><br> Example<br> <br> </strong>Load Month, NullCount(Item) as NumberOfNullItems from abc.xls group by Month;<br> <strong><br> MissingCount()<br> <br> </strong>This function returns the missing count of an expression over a number of records as defined by a group by clause. If the distinct occurs before the expression, all duplicates will be discarded.<br> <br> <strong>Syntax<br> <br> </strong>MissingCount([ distinct ] expression )<br> <strong><br> Example<br> <br> </strong>Load Month, MissingCount(Item) as NumberOfMissingItems from abc.xls group by Month;<br> <br> <strong>Advanced Aggregation function<br> <br> </strong>The aggr function is known as an advanced aggregation function. This function returns a set of values expression calculated over dimensions. The result can be compared to the expression column of a chart, where the aggr function resides. Each&nbsp; chart has a dimension. The dimension is a single field. The dimension is not considered as an expression. The expression is a calculated dimension.<br> <br> <strong>Syntax</strong><br> <br> aggr ([ distinct | nodistinct ] [{set_expression}]expression {,<br> dimension})<br> <br> <strong>Examples</strong><br> <br> aggr( sum(Sales), Country )<br> aggr( nodistinct sum(Sales), Country )<br> aggr( sum(Sales), Country, Region )<br> count( aggr( sum(Sales), Country ))<br> <br> <strong>Summary<br> <br> </strong>This article provides an introduction to aggregation functions in QlikView and also describes various types of aggregation functions. I hope this is beneficial for you when you calculate dimensions in a chart as an expression in QlikView.</p>
