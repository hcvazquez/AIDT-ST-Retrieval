       <h1>SublimeCodeIntel</h1> 
       <p>Code intelligence plugin ported from <a href="http://www.openkomodo.com/">Open Komodo Editor</a> to <a href="http://www.sublimetext.com/">Sublime Text</a>.</p> 
       <p>Supports all the languages Komodo Editor supports for Code Intelligence (CIX, CodeIntel2):</p> JavaScript, Mason, XBL, XUL, RHTML, SCSS, Python, HTML, Ruby, Python3, XML, Sass, XSLT, Django, HTML5, Perl, CSS, Twig, Less, Smarty, Node.js, Tcl, TemplateToolkit, PHP.
       <p>Provides the following features:</p> 
       <ul>
        <li>Jump to Symbol Definition - Jump to the file and line of the definition of a symbol.</li> 
        <li>Imports autocomplete - Shows autocomplete with the available modules/symbols in real time.</li> 
        <li>Function Call tooltips - Displays information in the status bar about the working function.</li> 
       </ul>
       <p>Plugin should work in all three platforms (MacOS X, Windows and Linux).</p> 
       <a href="https://www.paypal.com/cgi-bin/webscr?cmd=_s-xclick&amp;hosted_button_id=VVX4Q9H3924LE"><img alt="Click here to lend your support to SublimeCodeIntel and make a donation!" src="/readmes/img/d0696fa5aefd283edcd758f8d1c59e7bc6616a2c.gif"></a> 
       <h2>Installing</h2> 
       <p><strong>With the Package Control plugin:</strong> The easiest way to install SublimeCodeIntel is through Package Control, which can be found at this site: <a href="http://wbond.net/sublime_packages/package_control">http://wbond.net/sublime_packages/package_control</a></p> 
       <p>Once you install Package Control, restart Sublime Text and bring up the Command Palette (Command+Shift+P on OS X, Control+Shift+P on Linux/Windows). Select "Package Control: Install Package", wait while Package Control fetches the latest package list, then select SublimeCodeIntel when the list appears. The advantage of using this method is that Package Control will automatically keep SublimeCodeIntel up to date with the latest version.</p> 
       <p><strong>Without Git:</strong> Download the latest source from <a href="http://github.com/SublimeCodeIntel/SublimeCodeIntel">GitHub</a> and copy the whole directory into the Packages directory.</p> 
       <p><strong>With Git:</strong> Clone the repository in your Sublime Text Packages directory, located somewhere in user's "Home" directory:</p> 
       <pre>
git clone git://github.com/SublimeCodeIntel/SublimeCodeIntel.git
</pre> 
       <p>The "Packages" packages directory is located differently in different platforms. To access the directory use:</p> 
       <ul>
        <li><p>OS X:</p> <pre>
Sublime Text -&gt; Preferences -&gt; Browse Packages...
</pre> </li> 
        <li><p>Linux:</p> <pre>
Preferences -&gt; Browse Packages...
</pre> </li> 
        <li><p>Windows:</p> <pre>
Preferences -&gt; Browse Packages...
</pre> </li> 
       </ul>
       <h2>Using</h2> 
       <ul>
        <li><p>Start typing code as usual, autocomplete will pop up whenever it's available. SublimeCodeIntel will also allow you to jump around symbol definitions even across files with just a click ..and back.</p> 
         <dl>
          <dt>
           For Mac OS X:
          </dt> 
          <dd>
           <ul>
            <li>Jump to definition = Control+Click</li> 
            <li>Jump to definition = Control+Command+Alt+Up</li> 
            <li>Go back = Control+Command+Alt+Left</li> 
            <li>Manual Code Intelligence = Control+Shift+space</li> 
           </ul>
          </dd> 
          <dt>
           For Linux:
          </dt> 
          <dd>
           <ul>
            <li>Jump to definition = Super+Click</li> 
            <li>Jump to definition = Control+Super+Alt+Up</li> 
            <li>Go back = Control+Super+Alt+Left</li> 
            <li>Manual Code Intelligence = Control+Shift+space</li> 
           </ul>
          </dd> 
          <dt>
           For Windows:
          </dt> 
          <dd>
           <ul>
            <li>Jump to definition = Alt+Click</li> 
            <li>Jump to definition = Control+Windows+Alt+Up</li> 
            <li>Go back = Control+Windows+Alt+Left</li> 
            <li>Manual Code Intelligence = Control+Shift+space</li> 
           </ul>
          </dd> 
         </dl></li> 
       </ul>
       <p>Don't despair! The first time you use it it needs to build some indexes and it can take more than a few seconds.</p> 
       <p>It just works!</p> 
       <h2>Configuring</h2> 
       <p>For adding additional library paths (django and extra libs paths for Python or extra paths to look for .js files for JavaScript for example), either add those paths as folders to your project, or create an optional codeintel configuration file in your home or in your project's root.</p> 
       <p>Configuration files (<span class="pre">~/.codeintel/config</span> or <span class="pre">project_root/.codeintel/config</span>). All configurations are optional. Example:</p> 
       <pre>
{
    "PHP": {
        "php": '/usr/bin/php',
        "phpExtraPaths": [],
        "phpConfigFile": 'php.ini'
    },
    "JavaScript": {
        "javascriptExtraPaths": []
    },
    "Perl": {
        "perl": "/usr/bin/perl",
        "perlExtraPaths": []
    },
    "Ruby": {
        "ruby": "/usr/bin/ruby",
        "rubyExtraPaths": []
    },
    "Python": {
        "python": '/usr/bin/python',
        "pythonExtraPaths": []
    },
    "Python3": {
        "python": '/usr/bin/python3',
        "pythonExtraPaths": []
    }
}
</pre> 
       <p>Additional settings can be configured in the User File Settings:</p> 
       <p>Do NOT edit the default SublimeCodeIntel settings. Your changes will be lost when SublimeCodeIntel is updated. ALWAYS edit the user SublimeCodeIntel settings by selecting "Preferences-&gt;Package Settings-&gt;SublimeCodeIntel-&gt;Settings - User". Note that individual settings you include in your user settings will <strong>completely</strong> replace the corresponding default setting, so you must provide that setting in its entirety.</p> 
       <p>Available settings:</p> 
       <ul>
        <li>A list of disabled languages can be set using "codeintel_disabled_languages". Ex. "codeintel_disabled_languages": ['css']</li> 
        <li>Live autocomplete can be disabled by setting "codeintel_live" to false.</li> 
        <li>Live autocompletion can be disabled in a per-language basis, using "codeintel_live_disabled_languages". Ex. "codeintel_live_disabled_languages": ['css']</li> 
        <li>Information for more settings is available in the <span class="pre">SublimeCodeIntel.sublime-settings</span> file in the package.</li> 
       </ul>
       <h2>Troubleshooting</h2> 
       <p>If you are having problems or SublimeCodeIntel seems slow/unresponsive/non-working, there are some things you should try:</p> 
       <ol>
        <li>Make sure the SublimeCodeIntel is not being treated as an ignored package.</li> 
        <li>Regenerate indexing database.</li> 
        <li>Open a new issue.</li> 
       </ol>
       <h3>Ignored package?</h3> 
       <p>To make sure Sublime Text didn't add SublimeCodeIntel to the list of ignored packages (this happens some times when packages are being installed/upgraded):</p> 
       <ul>
        <li>Select the Preferences/Settings - Default menu item</li> 
        <li>Find the ignored_packages setting and remove SublimeCodeIntel from the list.</li> 
       </ul>
       <h3>Regenerate index</h3> 
       <p>To force re-indexation of the code intelligence database you need to follow these steps:</p> 
       <ul>
        <li>Close Sublime Text</li> 
        <li>Open a terminal or navigate through your directories to find the directory <span class="pre">~/.codeintel</span> that contains codeintel.log, VERSION and the directory db. In Windows, this should be at <span class="pre">%userprofile%\.codeintel</span>.</li> 
        <li>Delete the whole directory <span class="pre">~/.codeintel</span> and all of its content. Particularly, if you want to delete only the indexes, the code intelligence database indexes are located inside <span class="pre">~/.codeintel/db</span>.</li> 
        <li>Start Sublime Text</li> 
        <li>Try doing some code intelligence with the files in your project and enjoy a clean re-indexing! (Remember that the database is being regenerated with this procedure, so you'll see it takes some time to autocomplete the first few times, you'll have to wait a bit for things to be fast again)</li> 
       </ul>
       <h3>Opening an issue</h3> 
       <p>If the problems persists after trying the above solutions, open a new issue in <a href="https://github.com/SublimeCodeIntel/SublimeCodeIntel/issues/">https://github.com/SublimeCodeIntel/SublimeCodeIntel/issues/</a></p> 
       <p>When opening new issues, please check no other issues exist which report the same problem and provide all the messages from the Sublime Text console (the console is accessed via the ctrl+` shortcut or the View &gt; Show Console menu) and the <span class="pre">~/.codeintel/codeintel.log</span> file (<span class="pre">%userprofile%\.codeintel\codeintel.log</span> in Windows) as well as mentioning the Sublime Text version, the platform you are using and the languages you are using the code intelligence on.</p> 
       <h2>Building</h2> 
       <p>Building process is no longer distributed with this repository. You need to get SublimeCodeIntel/<a href="https://github.com/SublimeCodeIntel/CodeIntelSources/">CodeIntelSources</a> to run build.sh.</p> 
       <p>More information in SublimeCodeIntel/CodeIntelSources/<a href="https://github.com/SublimeCodeIntel/CodeIntelSources/tree/master/src">src</a>.</p> 
       <h2>Notes</h2> 
       <p>For the next major version of SublimeCodeIntel (v3.0.0) you will have to ensure that CodeIntel package (<a href="https://pypi.python.org/pypi/CodeIntel">https://pypi.python.org/pypi/CodeIntel</a>) is installed on your system usually by using pip or easy_install. The Code intelligence will be handled by that package and the command codeintel it will install.</p> 
       <p>Please start trying to install the CodeIntel package as soon as possible to make sure you are ready for the upcoming version of SublimeCodeIntel:</p> 
       <ol>
        <li>Install Python 2 and pip (<a href="http://www.pip-installer.org/en/latest/installing.html">http://www.pip-installer.org/en/latest/installing.html</a>).</li> 
        <li>Install (or upgrade) CodeIntel by typing the following in a terminal: pip install <span class="pre">-U</span> codeintel</li> 
       </ol>
