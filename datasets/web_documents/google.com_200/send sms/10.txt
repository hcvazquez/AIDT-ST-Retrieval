            <h2 data-reactid="33"><a href="/reactjs-twilio-example-tutorial/" data-reactid="34">How To Use Twilio With ReactJS</a></h2>
            <div class="post-info" data-reactid="35">

           <div class="post-content" data-reactid="38">
            <div data-reactid="39">
             <p>Twilio can be a lot of fun. It has a robust API that lets you send text messages. 
              <!-- more --> I worked on a side project that involved scheduling text messages in a ReactJS app. I'm not a developer by trade, just an enthusiast, so I ran into some challenges using Twilio's API to send messages from a React component. Many of these problems would be trivial for a seasoned developer, but if you're a beginner like me, you might find this information helpful. </p> 
             <p>In this tutorial, I'll show you what a basic Twilio-ReactJS integration looks like. The difficulty level of this tutorial is beginner to intermediate.</p> 
             <p>What we'll be using: </p> 
             <ul> 
              <li>ReactJS (naturally)</li> 
              <li>Express </li> 
              <li>Your typical combination of NPM and Webpack</li> 
             </ul> 
             <p>We'll need the node package <code>fetch</code> on the client-side, and the packages <code>body-parser</code> and <code>twilio</code> for the backend. </p> 
             <p>Before we get started, here's a link to the Github repository for a <a href="https://github.com/focuswish/react-twilio-example">barebones Twilio/React integration</a>. </p> 
             <h2 id="overall-strategy">Overall Strategy</h2> 
             <p>We don't want to make client-side HTTP requests to Twilio because that would expose our Twilio API keys.</p> 
             <p>Therefore, we pass information from the client to the server, and then make the HTTP requests to Twilio server-side.</p> 
             <p>For this example, we'll use <code>Express</code> to make the final API request to Twilio from our ReactJS app. </p> 
             <p>Express describes itself as:</p> 
             <blockquote> 
              <p>A minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications.</p> 
             </blockquote> 
             <p>So Express is like a layer that sits on top of Node.js. </p> 
             <h3 id="sending-data-from-our-react-component-to-the-server-express-">Sending Data from Our React Component to The Server (Express)</h3> 
             <p>We'll use fetch (<code>npm install whatwg-fetch --save</code>) to make web requests within our ReactJS components. Here's an example to get you started:</p> 
             <pre><code class="lang-language-javascript">
<span class="hljs-selector-tag">fetch</span>(<span class="hljs-string">'/sendsms'</span>, {
      <span class="hljs-attribute">method</span>: <span class="hljs-string">'POST'</span>,
      <span class="hljs-attribute">headers</span>: {
        <span class="hljs-attribute">Accept</span>: <span class="hljs-string">'application/JSON'</span>,
        <span class="hljs-string">'Content-Type'</span>: <span class="hljs-string">'application/JSON'</span>
      },
      <span class="hljs-attribute">body</span>: JSON.stringify({<span class="hljs-string">"data"</span>: this.state.data})
});
</code></pre> 
             <p>If you want to see this in context, here's the <a href="https://github.com/focuswish/react-twilio-example/blob/master/src/client.js">file on github</a>. </p> 
             <p>As you can see, with <code>fetch</code>, the pattern is fairly straightforward: <code>fetch(url, {method, headers, body, etc});</code> </p> 
             <p>Fetch is invoked inside a ReactJS component. For this example, I invoked it inside a function <code>sendSms()</code> that was called when a button was pressed: </p> 
             <pre><code class="lang-language-javascript">&lt;<span class="hljs-selector-tag">button</span> onClick={this<span class="hljs-selector-class">.sendSms</span><span class="hljs-selector-class">.bind</span>(this)}&gt;Send SMS&lt;/button&gt;
</code></pre> 
             <p>In this case, fetch posts data to the relative endpoint <code>/sendsms</code>. When we configure our server with Express, we'll define this endpoint and use Twilio's node package (<code>npm install twilio --save</code>) to make the final request to Twilio.</p> 
             <p>To pass data via <code>fetch</code>, use <code>this.state.myData</code>. Don't forget to send the data stored in state as a JSON by calling <code>JSON.stringify({...})</code></p> 
             <h2 id="setting-up-express">Setting Up Express</h2> 
             <p>Here's what our full express configuration file <code>server.js</code> looks like:</p> 
             <pre><code class="lang-language-javascript"><span class="hljs-keyword">import</span> express <span class="hljs-keyword">from</span> <span class="hljs-string">'express'</span>;
<span class="hljs-keyword">import</span> path <span class="hljs-keyword">from</span> <span class="hljs-string">'path'</span>;
<span class="hljs-keyword">import</span> keys <span class="hljs-keyword">from</span> <span class="hljs-string">'./twiliokeys'</span>;
<span class="hljs-keyword">import</span> bodyParser <span class="hljs-keyword">from</span> <span class="hljs-string">'body-parser'</span>;

<span class="hljs-keyword">const</span> app = express();

app.use(<span class="hljs-string">'/'</span>, express.static(<span class="hljs-string">'public'</span>));

app.post(<span class="hljs-string">'/sendsms'</span>, bodyParser.json(), <span class="hljs-function">(<span class="hljs-params">req, res</span>) =&gt;</span> {
  <span class="hljs-keyword">var</span> client = <span class="hljs-built_in">require</span>(<span class="hljs-string">'twilio'</span>)(keys.sid, keys.token);
  client.sendMessage({
    to: req.body.data,
    <span class="hljs-keyword">from</span>: <span class="hljs-string">'+12223334444'</span>,
    body: <span class="hljs-string">'word to your mother.'</span>
  }, <span class="hljs-function"><span class="hljs-keyword">function</span> (<span class="hljs-params">err, responseData</span>) </span>{
    <span class="hljs-keyword">if</span> (!err) {
      res.json({<span class="hljs-string">"From"</span>: responseData.from, <span class="hljs-string">"Body"</span>: responseData.body});
    }
  });
});

app.listen(process.env.PORT || <span class="hljs-number">3000</span>);
</code></pre> 
             <p>Here's a link to the <a href="https://github.com/focuswish/react-twilio-example/blob/master/server.babel.js">file on Github</a>. </p> 
             <ul> 
              <li><code>import keys from './twilioKeys';</code>. I stored my twilio API keys in a separate file called <code>twilioKeys.js</code></li> 
              <li><code>app.use('/', express.static('public'));</code> - This tells Express what to do with requests to localhost:3000/. I set this to 'public' because that's the location of my built ReactJs application. </li> 
              <li><code>app.post('/sendsms', bodyParser.json(), (req, res) =&gt; {}); </code> This tells the server how to handle post requests to the /sendsms endpoint. This was the endpoint we told fetch to send data to in our ReactJs component. </li> 
              <li><code>var client = require('twilio')(keys.sid, keys.token);</code> Here we require the Twilio node package. <code>require('twilio')</code> actually returns a function that takes the twilio SID and Auth Token as arguments. </li> 
              <li><code>client.sendMessage({to, from, body})</code> This is where you'd include the sender and recipient's phone numbers as well as the body of the text message.</li> 
              <li><code>req.body.data</code> What does this mean? When we send a JSON using fetch from our react component, we need req.body refers to the body of the POST request. So if we had a JSON like <code>{"food": "apple"}</code>, then we could write <code>req.body.food</code> to return "apple." </li> 
             </ul>
