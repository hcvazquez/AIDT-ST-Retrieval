       <div class="heading-block center"> 
        <h2>What's so special about Codename One?</h2> 
        <span>We are the only ones that provide...</span> 
       </div> 
       <div class="row"> 
        <div class="col_half"> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Write Once Run Anywhere, no special hardware &amp; 100% code reuse
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             Most other solutions don't offer write once run anywhere and usually require a Mac for iOS development or a Windows machine for Windows development!
            <br> Codename One solves these problems by providing a unique cloud build system. We have Mac servers in the cloud running xcode where your binary JAR is statically translated to native C code and compiled with xcode using our open source VM. It's signed using an official Apple certificate and the binary is delivered back to you for installation on your phone or upload to the store.
            <br> You retain full ownership of your application and your source code remains secure on your local machine, only the compiled bytecode reaches the build server. 
            <br> Pretty much every other solution needs some native code or at least native API usage to bind the lifecycle of the application. 
           </div> 
          </div> 
         </div> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Compile Java(tm) or Kotlin to native iOS, Windows (UWP), Android, Desktop &amp; JavaScript
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             Codename One is ridiculously portable. It is based on work started by our founders at Sun Microsystems back in 2006 and leverages decades of cross platform mobile development experience. In 
            <a href="https://www.youtube.com/watch?v=EMRmo6ZRnGw" target="_blank">this video</a> Shai Almog explains the basics of Codename One and its portability. You can also read some of the low level explanations in 
            <a href="https://stackoverflow.com/questions/10639766/how-does-codename-one-work/10646336" target="_blank">this stackoverflow answer</a>.
            <br> The gist of it is that Codename One translates code to native OS's. Since it's based on Java Codename One is native to Android. For iOS the bytecode is translated to C using our 
            <a href="https://github.com/codenameone/CodenameOne/tree/master/vm" target="_blank">open source VM</a>. For Windows UWP (Universal Windows Platform) we use iKVM which is a bytecode to .net translator. For JavaScript we use TeaVM which translates bytecode to JavaScript and even supports threads! 
           </div> 
          </div> 
         </div> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Easy to use with 100% portable Drag &amp; Drop GUI builder
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             Easy to use cross platform tools are usually "weak" in terms of capabilities. Tools that are as powerful as Codename One is usually don't offer a portable GUI builder due to the difficulty of cross platform component layout and positioning. 
            <br> Our second generation GUI builder uses a unique approach for layout that is powerful, portable &amp; intuitive. 
           </div> 
          </div> 
         </div> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>You can control every pixel and draw anywhere
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             In Codename One you can override the paint method or styling of any component and just change the way they draw themselves and it will work!
            <br> You can override the glass pane or layered pane behavior and draw over any region in the screen without a hassle or native issues. This is possible since Codename One is a lightweight framework, a unique architecture among mobile frameworks. But it takes that even further with heavyweight mixing.
            <br> Check out 
            <a href="https://stackoverflow.com/questions/10639766/how-codename-one-works/10646336">this stackoverflow answer</a> to understand the architecture of Codename One. You can also check out 
            <a href="https://www.youtube.com/watch?v=EMRmo6ZRnGw" target="_blank">this video</a>. 
           </div> 
          </div> 
         </div> 
        </div> 
        <div class="col_half col_last"> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Full access to native OS using the native language (e.g. Obj-C, C# etc.)
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             You can and usually should write code that is 100% portable with Codename One. But you might reach a point where you need something from the native OS. For that we have 
            <a href="/how-do-i---access-native-device-functionality-invoke-native-interfaces.html">native interfaces</a>.
            <br> Native Interfaces let you write native OS code if you need to, they let you package it in a library so your code can be completely abstracted. There are no limits on the type of code you can write with Codename One.
            <br> Some other tools require usage of their language when working in native, so if you find a sample on Apples website you need to translate it to your language of choice first. 
            <br> With native interfaces you can just paste the code into the native interface and it will work when running on iOS or the respective platform. 
           </div> 
          </div> 
         </div> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Use native widgets &amp; mix them with our components in the hierarchy (heavyweight/lightweight mixing)
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             One of the popular features of Codename One is native Google Maps support. It works by literally embedding the native Google maps widget into a Codename One application. You can place Codename One components on top of this map &amp; interact with both...
            <br> This is very powerful and as far as we know a unique ability of Codename One within the lightweight frameworks. 
           </div> 
          </div> 
         </div> 
         <div class="toggle toggle-bg"> 
          <div class="togglet">
           <i class="toggle-closed icon-ok-circle"></i>
           <i class="toggle-open icon-remove-circle"></i>Open Source &amp; Free for commercial use with enterprise grade commercial offering
          </div> 
          <div style="display: none;" class="togglec"> 
           <div class="row">
             Most tools have an open source project. Codename One originated as an open source project fork of a project started by our founders at Sun Microsystems.
            <br> However, many open source projects of this type don't have a commercial backing or long term sustainable business model. A few exist simply to serve corporate goals of companies with a completely different business model... 
            <br> In those cases official paid support is impossible. The pairing of open source and a sustainable commercial offering is surprisingly rare in the cross platform niche. Furthermore, many of the open source options use things like nag screens or licenses that prohibit commercial use. With Codename One this isn't the case.
