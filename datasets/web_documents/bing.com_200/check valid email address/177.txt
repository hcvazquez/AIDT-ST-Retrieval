       <h1 class="entry-title" itemprop="name">V2 Mail Send</h1> 
      </header> 
      <div class="content"> 
       <p>This endpoint allows you to send email over SendGrids Web API v2.</p> 
       <div class="callout callout-warning"> 
        <p>For information about the latest mail send endpoint, please visit our <a href="//sendgrid.com/docs/API_Reference/Web_API_v3/Mail/index.html">Web API v3 Mail Send documentation</a>. To current v2 users: We encourage <a href="//sendgrid.com/docs/Classroom/Send/v3_Mail_Send/how_to_migrate_from_v2_to_v3_mail_send.html">migrating</a> to the new endpoint to take advantage of upcoming features.</p> 
       </div> 
       <div class="callout callout-info"> 
        <p>There are a number of <a href="//sendgrid.com/docs/Integrate/libraries.html">official SendGrid libraries</a> to allow for email sending through the Web API v2. We recommend using these if possible.</p> 
        <p><a href="//sendgrid.com/docs/Integrate/libraries.html">View List </a></p> 
       </div> 
       <div class="callout callout-info"> 
        <p>When you make a request to the Web API v2 to send a message, we will accept the request and then process the email, meaning that a successful API request does not mean an email was successfully sent. Please see our <a href="//sendgrid.com/docs/API_Reference/Webhooks/event.html">Event Webhook</a> if you want to be notified of email send status.</p> 
       </div> 
       <h2 class="anchor-wrap"><a name="-send" class="anchor" href="#-send"><span class="anchor-icon"><i class="icon-link"></i></span></a>send</h2> 
       <p>SendGrids <a href="//sendgrid.com/docs/User_Guide/Account/api_keys.html">API Keys</a> should be used when sending email over the API.</p> 
       <div class="callout callout-info"> 
        <p>You can have up to 10,000 recipients per Web API v2 request. While we do not enforce rate limits on sending mail via the v2 mail send endpoint, we do recommend that users avoid making more than 3,000 API requests per second.</p>
