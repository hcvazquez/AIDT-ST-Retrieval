         <p>Want to learn how to write a complete mobile application using PhoneGap, but don't know how to get started? Follow this two-part tutorial series and I'll show you how to build a mobile app that tracks, records, and maps your exercise route using the GPS features on your phone!<br> 
          <!--more--></p> 
         <hr> 
         <h2>Project Overview</h2> 
         <p>This section is a broad overview of ExerciseTracker, the application we will be building.</p> 
         <div class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/home.png" data-original-url="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/home.png" alt="Home" title=""> 
          <p>The home page checks that the phone has an active network connection (we need this to load<br> Google Maps to plot the exercise route). It also has two buttons that are useful for testing the app, but they can be removed easily later if desired.</p> 
         </div> 
         <div class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/trackworkout.png" data-original-url="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/trackworkout.png" alt="Track Workout" title=""> 
          <p>The Track Workout page has a single text entry field for the user to enter a name or ID for their workout. When they click "Start Tracking", the phone will start recording their GPS location. When they click "Stop Tracking" the GPS data is saved on the phone.</p> 
         </div> 
         <p> </p>
         <div class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/history.png" data-original-url="https://mobiletuts.s3.amazonaws.com/PhoneGap_Exercise-Tracker/1/history.png" alt="History" title=""> 
          <p>The History page lists all of the tracked workouts the user has completed.</p> 
         </div> 
         <div class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/trackinfo.png" data-original-url="https://mobiletuts.s3.amazonaws.com/PhoneGap_Exercise-Tracker/1/trackinfo.png" alt="Track Info" title=""> 
          <p>The Track Info page presents workout specific information. It displays the total distance travelled in kilometers and the total time taken for the workout, and it then plots the route the user travelled on a Google Map.</p> 
         </div> 
         <hr> 
         <h2>Technologies Implemented</h2> 
         <p>Our application, ExerciseTracker, is going to run on the <a href="http://phonegap.com/">PhoneGap</a> mobile platform. PhoneGap allows us to easily create mobile applications using familiar web technologies such as HTML 5, CSS, and JavaScript, and then seamlessly deploys them as native apps on platforms such as iOS, Android, and Windows Phone.</p> 
         <p>The exercise tracker app we will be building utilizes the following technologies:</p> 
         <ul> 
          <li>PhoneGap Location API</li> 
          <li>jQueryMobile</li> 
          <li>Google Maps JavaScript API</li> 
          <li>HTML5 &amp; Local Storage</li> 
         </ul> 
         <p>In this two-part series of tutorials, you will learn how to use the above to build a complete PhoneGap application.</p> 
         <hr> 
         <h2>Beginning Development</h2> 
         <h3>PhoneGap Setup</h3> 
         <p>The first thing we need to do is set up the PhoneGap development environment. PhoneGap provides a great <a href="http://phonegap.com/start">Start Page</a> which walks you through setting up the PhoneGap development environment for each of the different mobile platforms. Choose the platform you want to develop for and follow the instructions to set up the basic PhoneGap application. In this tutorial, I will be developing on an Android setup (Android 4.0.3 API v15), however the code in this tutorial should work on all platforms.</p> 
         <p>You should now have a minimal <em>index.html</em> file which you can view on your device emulator.</p> 
         <h3>Dependencies and Scripts</h3> 
         <p>Let's start development of our app by downloading and including all of the required JavaScript and CSS files.</p> 
         <ul> 
          <li><a href="http://code.jquery.com/jquery-1.7.2.min.js">jquery-1.7.2.min.js</a></li> 
          <li> <a href="http://code.jquery.com/mobile/1.1.0/jquery.mobile-1.1.0.zip">jquery.mobile-1.1.0.zip</a> 
           <ul> 
            <li>jquery.mobile.1.1.0.min.css</li> 
            <li>jquery.mobile-1.1.0.min.js</li> 
            <li>images/</li> 
           </ul> </li> 
          <li><a href="https://raw.github.com/douglascrockford/JSON-js/master/json2.js">json2.js</a></li> 
          <li>exercisetracker.js (create this, our JS will go here)</li> 
         </ul> 
         <p>Download and copy each of these files into the same directory as the <em>index.html</em> and <em>cordova-1.7.0.js</em> files.</p> 
         <p>At this stage, the directory structure of my Android app looks like this:</p> 
         <div class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/dirstructure.png" data-original-url="https://mobiletuts.s3.amazonaws.com/PhoneGap_Exercise-Tracker/1/dirstructure.png" alt="Android dir structure" title=""> 
          <p>Your directory structure should look like this.</p> 
         </div> 
         <h3>Google Maps API Key</h3> 
         <p>In order for our application to use Google Maps, we must have an API key. API keys are free to get and support up to 25,000 queries per day. Follow the <a href="https://developers.google.com/maps/documentation/javascript/tutorial#api_key">guide</a> that Google provides to register your API key. Note it down, as we will need it in the next step!</p> 
         <h3>Skeleton index.html</h3> 
         <p>Let's write the first version of <em>index.html</em>. This will include all of the JavaScript and CSS dependencies ExerciseTracker will have.</p> 
         <pre class="brush: html noskimlinks noskimwords">

         <p>You should be able to launch this in your emulator and recieve no errors.</p>

         <h2>Creating the Home Page</h2> 
         <p>Now lets create the Home page. jQueryMobile allows us to define our user interface using standard HTML tags. Pages in our app are <code>&lt;div&gt;</code> elements, buttons are <code>&lt;button&gt;</code> or <code>&lt;a&gt;</code> elements and lists are made up of <code>&lt;ul&gt;</code> and <code>&lt;li&gt;</code> elements. We describe semantic information about each of these elements by defining data attributes. For example, the <code>data-role</code> attribute tells jQueryMobile the role of the HTML element and the <code>data-icon</code> attribute specifies what icon will be shown inside the element. You can see below how the home page is composed of HTML elements with <code>data-role</code> attributes.</p> 
         <div class="tutorial_image"> 
          <p><img src="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/home.png" data-original-url="https://cdn.tutsplus.com/mobile/uploads/legacy/PhoneGap_Exercise-Tracker/1/home.png" alt="Home" title=""> </p> 
         </div>

         <h2>Checking for Network Access</h2> 
         <p>We now need to check to see if the user has access to the internet. If they don't, then the Google Map will not load and the app will be cripppled. </p> 
         <p>We start by listening for the <code>deviceready</code> event which is fired when PhoneGap has finished loading. We then use the PhoneGap <a href="http://docs.phonegap.com/en/1.7.0/cordova_connection_connection.md.html#Connection">Connection API</a> to query if we have access to the Internet. If we don't have Internet access, then we need to update the button on the Home page. We update the text, set a new icon, and call the <code>button('refresh')</code> method, which forces jQuery Mobile to redraw the button.</p> 
         <pre class="brush: javascript noskimlinks noskimwords">
document.addEventListener("deviceready", function(){
  
  if(navigator.network.connection.type == Connection.NONE){
    $("#home_network_button").text('No Internet Access')
                 .attr("data-icon", "delete")
                 .button('refresh');
  }

});</pre>

         <h2>Tracking the User's Movements</h2> 
         <p>We can monitor the user's location using the PhoneGap <a href="http://docs.phonegap.com/en/1.7.0/cordova_geolocation_geolocation.md.html#Geolocation">Geolocation API</a>. It provides two main functions: <code>getCurrentPosition()</code> and <code>watchPosition()</code>. The <code>getCurrentPosition()</code> function would be used when you want to get a one-off fix on the user's location (such as saving the location of the user when they take a photo). Our app will use <code>watchPosition()</code> to do this, which regularly polls the GPS feature of the phone and recieves constant updates on the user's location.</p> 
         <p>The <code>watchPosition()</code> accepts three arguments. The first is a function which is called every time the phone successfully returns a GPS location, the second is a function which is called when there is a GPS error, and the third is an object of <a href="http://docs.phonegap.com/en/1.7.0/cordova_geolocation_geolocation.md.html#geolocationOptions">settings</a>.</p> 
         <p>Every time the "success" function is called, it is passed a <a href="http://docs.phonegap.com/en/1.7.0/cordova_geolocation_geolocation.md.html#Position">Position</a> object. This contains information about the user's GPS position, such as their latitude, longitude, and altitude. Whenever the "success" function is called in ExerciseTracker, we append the latest Position object to the array <code>tracking_data</code>.</p> 
         <pre class="brush: javascript noskimlinks noskimwords">
var track_id = '';      // Name/ID of the exercise
var watch_id = null;    // ID of the geolocation
var tracking_data = []; // Array containing GPS position objects

$("#startTracking_start").live('click', function(){
    
    // Start tracking the User
    watch_id = navigator.geolocation.watchPosition(
    
        // Success
        function(position){
            tracking_data.push(position);
        },
        
        // Error
        function(error){
            console.log(error);
        },
        
        // Settings
        { frequency: 3000, enableHighAccuracy: true });
    
    // Tidy up the UI
    track_id = $("#track_id").val();
    
    $("#track_id").hide();
    
    $("#startTracking_status").html("Tracking workout: &lt;strong&gt;" + track_id + "&lt;/strong&gt;");
});</pre>

In the next tutorial, we will complete the Track Workout page and finish the ExerciseTracker app by allowing the user to view saved workouts plotted on a Google Map.</p>
