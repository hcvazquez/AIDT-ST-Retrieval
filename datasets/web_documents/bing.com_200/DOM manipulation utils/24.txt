      <h1 class="post-title"><a href="https://blogs.windows.com/buildingapps/2013/07/10/jquery-and-winjs-working-together-in-windows-store-apps/">jQuery and WinJS working together in Windows Store apps</a></h1> 
      <div class="author">
        By 
       <a href="https://blogs.windows.com/buildingapps/author/windowsappsteam/" title="Posts by Windows Apps Team" class="author url fn" rel="author">Windows Apps Team</a> 
      </div> 
      <div id="addthis-2482" class="addthis_toolbox addthis_default_style expanded" addthis:url="https://blogs.windows.com/buildingapps/2013/07/10/jquery-and-winjs-working-together-in-windows-store-apps/" addthis:title="jQuery and WinJS working together in Windows Store apps"> 
       <a class="addthis_button_facebook"><i class="icon icon-facebook"></i> <span>Share</span></a> 
       <a class="addthis_button_twitter"><i class="icon icon-twitter"></i> <span>Tweet</span></a> 
       <a class="addthis_button_reddit"><i class="icon icon-reddit"></i> <span>Share</span></a> 
       <a class="addthis_button_linkedin"><i class="icon icon-linkedin"></i> <span>Share</span></a> 
       <a class="addthis_button_skype" href="#"><i class="icon icon-skype"></i> <span>Skype</span></a> 
      </div> 
     </header> 
     <p>The <a href="http://msdn.microsoft.com/en-us/library/windows/apps/hh465493.aspx" target="_blank">Windows Library for JavaScript (WinJS)</a> and <a href="http://msdn.microsoft.com/en-us/library/windows/apps/hh758331.aspx" target="_blank">Visual Studio templates</a> for Windows Store apps give developers great flexibility when using third-party JavaScript libraries, including the popular <a href="http://jquery.com/" target="_blank">jQuery</a>. Many modern web developers use jQuery for quick, easy, and powerful DOM access and manipulation. </p> 
     <p><a href="http://blogs.msdn.com/b/windowsappdev/archive/2013/04/01/windows-store-app-support-in-jquery-version-2-0.aspx" target="_blank">As we announced in April</a>, you can use jQuery release 2.0 in a Windows Store app without getting errors related to the JavaScript runtime or dynamic content. In this post, I’ll show how you can apply jQuery to your Windows Store app. In addition, I’ll also look at some areas where WinJS provides compelling alternatives to specific jQuery APIs.</p> 
     <p>(If you want a refresher on the jQuery APIs, take a look at the <a href="http://api.jquery.com/" target="_blank">jQuery documentation</a>. You might also take a minute to remind yourself about how to use <a href="http://msdn.microsoft.com/en-us/library/ie/hh772056(v=vs.85).aspx" target="_blank">CSS selectors</a>.) </p> 
     <h2>Using jQuery in Windows Store apps</h2> 
     <p>You need a local version of jQuery in your Window Store app, so you’ll need to <a href="http://jquery.com/download/" target="_blank">download a copy of jQuery release 2.0 (or higher)</a>. After you have a copy of the jQuery library, add it to your JavaScript project in Visual Studio 2012 (right-click the ‘js’ folder in the project, click <b>Add &gt;</b> <b>Existing Item</b>, and then select the jQuery library from where you saved it.) </p> 
     <p>(You can also get jQuery directly into your Visual Studio solution using <a href="http://nuget.org/" target="_blank">NuGet</a>.)</p> 
     <p>Next, you’ll need to reference the jQuery library in the HTML files for your app. Note that you’ll need to include a reference to jQuery on any HTML page that references a JavaScript file that uses jQuery. I recommend placing the reference in the starting page for the app (usually ‘default.html’), which ensures that you can use jQuery from any page loaded into (navigated from) that page.</p>

To use Sizzle, download a local copy of the library, add it to your project, and include a reference to the library in your HTML page (just like jQuery).</p>

      </div> 
     </blockquote> 
     <h3>Promises</h3> 
     <p>Both jQuery and WinJS provide their own implementation of the <a href="http://blogs.msdn.com/b/windowsappdev/archive/2013/06/11/all-about-promises-for-windows-store-apps-written-in-javascript.aspx" target="_blank">Promises pattern</a>. Some jQuery functions return a promise object by default (functions like <b>animate</b> and <b>get</b> – more on those later). Other functions can be wrapped in a promise by calling the <b>promise</b> function on the results of the function. When you do this, the promise returns immediately with a resolved promise.</p> 
     <blockquote> 
      <div id="codeSnippetWrapper"> 
       <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// The code in the done method that follows executes immediately.<br>$("div").promise().done(function (result) { });<br></pre> 
      </div> 
     </blockquote> 
     <p>In some circumstances, you might want to chain WinJS and jQuery promises together. Luckily, the two implementations are interoperable. jQuery promises implement the same <b>then</b> and <b>done</b> methods that WinJS uses. </p> 
     <p>In this next (somewhat silly) example, I’ve mixed jQuery and WinJS promises together using both promise chaining and the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br211774.aspx" target="_blank">WinJS.Promise.join method</a>. This code gets data stored in the app using the jQuery <b>get</b> function, creates a temporary file, writes the data from the app to the file, reads the file, and then displays the data from the file on the screen.</p> 
     <blockquote> 
      <div id="codeSnippetWrapper"> 
       <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">var tempFolder = Windows.Storage.ApplicationData.current.temporaryFolder,<br>    promiseArray = [],<br>    filename = "tempfile.txt";<br><br>promiseArray.push(tempFolder.createFileAsync(filename,<br>  Windows.Storage.CreationCollisionOption.replaceExisting));<br>promiseArray.push($.get("/data/data.txt"));<br><br>WinJS.Promise.join(promiseArray).<br>  then(function (response) {<br><br>    var file = response[0],<br>        data = response[1],<br>        memoryStream =<br>          new Windows.Storage.Streams.InMemoryRandomAccessStream(),<br>        dataWriter =<br>          new Windows.Storage.Streams.DataWriter(memoryStream);<br><br>    dataWriter.writeString(data);<br>    var buffer = dataWriter.detachBuffer();<br>    dataWriter.close();<br><br>    return Windows.Storage.FileIO.writeBufferAsync(file, buffer);<br>  }).<br>  then(function () {<br>    return tempFolder.getFileAsync(filename);<br>  }).<br>  then(function (file) {<br>    return Windows.Storage.FileIO.readTextAsync(file);<br>  }).<br>  then(function (data) {<br>    return $("#scenario").text(data).promise();<br>  }).<br>  done(function () {<br>    $("#scenario").append("<span style="color: rgb(0, 0, 255);">&lt;</span><span style="color: rgb(128, 0, 0);">br</span><span style="color: rgb(0, 0, 255);">/&gt;&lt;</span><span style="color: rgb(128, 0, 0);">br</span><span style="color: rgb(0, 0, 255);">/&gt;</span>Finished!");<br>  });<br></pre> 
      </div> 
     </blockquote> 
     <h2>Use the tools best suited for the job</h2> 
     <p>Although jQuery adds a powerful complement to the APIs in Windows, it’s good to keep in mind that WinJS already provides a wealth of tools you can use. The WinJS APIs are optimized for use with Windows and are designed around app-specific scenarios (whereas jQuery is platform agnostic). If you find yourself considering whether to use jQuery or WinJS for a particular task, it’s a good idea to defer to the WinJS APIs.</p> 
     <p>Here are a handful of scenarios where WinJS already supplies the best ‘tools’ for a job.</p> 
     <h3>Animating elements</h3> 
     <p>jQuery includes a group of functions for animating DOM content, many of which functions are based on its <b>animate</b> function. However, the animations in WinJS are specially designed to make efficient use of machine resources by <a href="http://blogs.msdn.com/b/windowsappdev/archive/2012/05/01/fast-and-fluid-animations-in-your-metro-style-app.aspx" target="_blank">offloading the animation workload from the CPU to the GPU</a> (for animations that affect transform and opacity). As a result, the WinJS animations perform well and don’t block the UI thread.</p> 
     <p>WinJS includes animations that are equivalent to the animations provided by jQuery, animations that match the design and look of Windows. Take the jQuery <b>fadeIn</b> and <b>fadeOut</b> functions, which make an HTML element fade into / out of view and change the display value of the element’s style. WinJS has the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br212673.aspx" target="_blank">WinJS.UI.Animation.fadeIn</a> and <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br212674.aspx" target="_blank">WinJS.UI.Animation.fadeOut</a> functions, which also smoothly fade an HTML element into and out of view, but without changing the display value. </p> 
     <blockquote> 
      <div id="codeSnippetWrapper"> 
       <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// Animating content into view with jQuery.<br>$("#content").fadeIn("slow");<br><br>// Animating content into view with WinJS and jQuery.<br>WinJS.UI.Animation.fadeIn($("#content")[0]);<br><br>// Animating content out of view with jQuery.<br>$("#content").fadeOut("fast");<br><br>// Animating content out of view with WinJS and jQuery.<br>WinJS.UI.Animation.fadeOut($("#content")[0]);<br></pre> 
      </div> 
     </blockquote> 
     <p>In the above sample, note that the WinJS<b> </b>effects only change the opacity of the affected elements. The jQuery animations affect the opacity and the display value, which lead to a re-layout of the page.</p>

      <p>Plus, many animations built into Windows Store apps can be applied using CSS3. The following code combines jQuery with the new CSS3 animation styles to create a “flip” effect on the affected element.</p> 
      <blockquote> 
       <div id="codeSnippetWrapper">

      <h3>Checking that the DOM has loaded</h3> 
      <p>When building a normal web application, you need to know when the DOM has completed loading before you can begin manipulating any elements that it contains. This means using the <b>ready</b> function in jQuery to add a handler to the event raised when the DOM is fully loaded.</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// Check to see if the webpage has finished loading and is ready.<br>$(document).ready(function() {<br>  // Add set up code here.<br>})<br></pre> 
       </div> 
      </blockquote> 
      <p>However, the jQuery <b>ready </b>function is largely unnecessary in a Windows Store app given the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br229844.aspx" target="_blank">WinJS.Application.onready event</a> and <a href="http://msdn.microsoft.com/en-us/library/windows/apps/hh770590.aspx" target="_blank">IPageControl.ready method</a>. The <b>IPageControl.ready </b>method is raised when the DOM for a page / element has been fully loaded into the app. You then add your initialization code for the page to the handler function for the <b>IPageControl.ready</b> method.</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// When the page is ready to be used, run setup code.<br>WinJS.UI.Pages.define("pageone.html", {<br>  ready: function (element, options) { <br>    // Add set up code here.<br>  }<br>});<br></pre> 
       </div> 
      </blockquote> 
      <h3>Accessing web resources</h3> 
      <p>You may be familiar with using the <b>load, get,</b> or <b>ajax</b> functions in jQuery to access remote resources on the web. However, you’ll find that the best way to get web resources from a Windows Store app is the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br229787.aspx" target="_blank">WinJS.xhr</a> function. The WinJS.xhr wraps the standard XMLHttpRequest object in a Promise object. </p> 
      <p>(Note that in Windows 8.1 there’s a new <b>Windows.Web.Http.HttpClient</b> API that’s simple to use and yet more powerful than <b>WinJS.xhr</b>. It’s the recommended API for HTTP requests for apps targeting Windows 8.1.)</p> 
      <p>For example, the <b>WinJS.xhr</b> function can easily retrieve the RSS feed from this blog:</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">var url = "http://blogs.msdn.com/b/windowsappdev/rss.aspx";<br><br>// Request the RSS feed from the blog.<br>WinJS.xhr({ url: url }).done(<br>  function (result) {<br>    $("#output").text(result.responseText);<br>});<br></pre> 
       </div> 
      </blockquote> 
      <p>With the <b>WinJS.xhr</b> function, we can parse the response from the web resource just like a normal <b>XMLHttpRequest</b>. </p> 
      <h3>Navigating between pages</h3> 
      <p>Navigating from one page to another is a key design feature for most apps. With jQuery, one might create a multi-page navigation in using the <b>load</b> function. For instance, you can inject the HTML from the <b>body</b> element of one page into a host element contained in another page using the following code:</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// Display the page navigated to in the content host.<br>$("#content-host").load("/html/pageone.html");<br></pre> 
       </div> 
      </blockquote> 
      <p>The WinJS library contains a <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br229778.aspx" target="_blank">Navigation</a> namespace that, when combined with the APIs included in the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/hh770584.aspx" target="_blank">WinJS.UI.Pages</a><b></b><b> </b>namespace, provides robust multi-page navigation. In the following example, a new page is displayed inside of an element with the ID value ‘content-host’. </p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// Navigate to a new page.<br>WinJS.Navigation.navigate("/html/pageone.html");<br><br>// Other code …<br><br>// After the app has navigated to a new page, display the contents<br>// of the new page in the 'content-host' div.<br>WinJS.Navigation.addEventListener("navigated", function (eventObject) {<br>  var url = eventObject.detail.location,<br>      host = $("#content-host")[0];<br><br>  host.winControl &amp;&amp; host.winControl.unload &amp;&amp; host.winControl.unload();<br>  WinJS.Utilities.empty(host);<br>  eventObject.detail.setPromise(<br>    WinJS.UI.Pages.render(url, host, eventObject.detail.state).then(function () {<br>      WinJS.Application.sessionState.lastUrl = url;<br>      WinJS.UI.Animation.enterPage(host);<br>      WinJS.UI.processAll();<br>  }));<br>});<br></pre> 
       </div> 
      </blockquote> 
      <p>The code sample calls <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br229837.aspx" target="_blank">WinJS.Navigation.navigate</a> function to navigate to the new URL. It then adds a handler to the ‘navigated’ event to display the opened page within the content host. Before the new page is displayed, the <b>IPageControl.unload</b> function is called on the host and the element is emptied. Note how the <b>sessionState</b> for the app is updated after the navigation and a nice little animation is run when the new page enters.</p> 
      <p>(Some of the templates in Visual Studio also include the navigator.js file, which provides additional navigation utilities for multi-page apps.)</p> 
      <h3>Selecting single elements</h3> 
      <p>In Windows Store apps, you can reference an HTML element that has a unique ID attribute by its ID. Thus, you don’t particularly need the selector function in jQuery in situations when you need to select a single element with a known ID. (You see similar behavior in the code-behind files of Windows Store apps built with C# and XAML.) </p> 
      <p>For example, take a simple HTML <b>span</b> element with a valid ID attribute on a Windows Store app page:</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);"><span style="color: rgb(0, 0, 255);">&lt;</span><span style="color: rgb(128, 0, 0);">span</span> <span style="color: rgb(255, 0, 0);">id</span><span style="color: rgb(0, 0, 255);">='mySpan'</span><span style="color: rgb(0, 0, 255);">&gt;&lt;/</span><span style="color: rgb(128, 0, 0);">span</span><span style="color: rgb(0, 0, 255);">&gt;</span></pre> 
       </div> 
      </blockquote> 
      <p>In a JavaScript file referenced by the app page, the <b>span</b> can be referenced directly by its ID, without needing to declare a new variable:</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">// We can reference the span directly using its ID as a variable name.<br>// The following statement evaluates to 'true'.<br>mySpan.tagName == 'SPAN';<br></pre> 
       </div> 
      </blockquote> 
      <p>You don’t need to capture a reference to an element with a unique ID because it’s already provided to you. This works only if the ID value of the element is allowed as a variable name in JavaScript: ‘mySpan’ works but ‘my-span’ doesn’t.</p> 
      <p>Moreover, you can use this trait of Windows Store apps in conjunction with jQuery. Because the variable references the element as an <a href="http://msdn.microsoft.com/en-us/library/windows/apps/hh868706.aspx" target="_blank">HTMLElement</a> object, you can pass in the variable directly to the jQuery selector function without using a CSS selector expression.</p> 
      <blockquote> 
       <div id="codeSnippetWrapper"> 
        <pre id="codeSnippet" style="margin: 0em; padding: 0px; width: 100%; text-align: left; color: black; line-height: 12pt; overflow: visible; font-family: &quot;Courier New&quot;, courier, monospace; font-size: 8pt; direction: ltr; background-color: rgb(244, 244, 244);">$(mySpan).css('background-color', 'rgba(40, 200, 40, 1)');</pre> 
       </div> 
      </blockquote> 
      <h2>Better together</h2> 
      <p>As we’ve seen, jQuery complements the tools available in WinJS very well. With jQuery, you can select multiple HTML elements from your app’s markup with greater ease and finer control. The functions in jQuery can turbocharge the WinJS code you use to manipulate your app. With the two, you can build a dynamic and elegant Windows Store app with less code, meaning less time to develop (and less time to publish to the Store). </p> 
      <p>Thank you for reading and keep on developing apps!</p>
