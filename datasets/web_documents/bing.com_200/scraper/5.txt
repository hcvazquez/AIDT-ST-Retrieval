           </svg></a>Getting started</h1> 
         <p>Scraperjs exposes two different scrapers,</p> 
         <ul> 
          <li>a <strong>StaticScraper</strong>, that is light fast and with a low footprint, however it doesn't allow for more complex situations, like scraping dynamic content.</li> 
          <li>a <strong>DynamicScraper</strong>, that is a bit more heavy, but allows you to scrape dynamic content, like in the browser console. both scrapers expose a <em>very</em> similar API, with some minor differences when it comes to scraping.</li> 
         </ul>

         <p>Again, the <code>scrape</code> promise receives a function to scrape the page, the only difference is that, because we're using a dynamic scraper, the scraping function is <a href="https://github.com/sgentle/phantomjs-node/wiki#evaluating-pages">sandboxed</a> only with the page scope, so <strong>no closures!</strong> This means that in <em>this</em> (and only in this) scraper you can't call a function that has not been defined inside the scraping function. Also, the result of the scraping function must be <a href="https://github.com/sgentle/phantomjs-node/wiki#evaluating-pages">JSON-serializable</a>. We use <a href="https://github.com/sgentle/phantomjs-node">phantom</a> and <a href="https://github.com/ariya/phantomjs">phantomjs</a> to make it happen, we also inject jQuery for you.</p> 
         <p>However, it's possible to <a href="/ruipgil/scraperjs/blob/master/**Example**">pass JSON-serializable data</a> to <em>any</em> scraper.</p> 
         <p><em>The <code>$</code> varible received by the scraping function is, only for the dynamic scraper, hardcoded.</em></p>

         <p>Scraperjs uses promises whenever possible.</p> 
         <h4><a id="user-content-staticscraper-dynamicscraper-and-scraperpromise" class="anchor" href="#staticscraper-dynamicscraper-and-scraperpromise" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>StaticScraper, DynamicScraper and ScraperPromise</h4> 
         <p>So, the scrapers should be used with the ScraperPromise. By creating a scraper</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> scraperPromise <span class="pl-k">=</span> <span class="pl-smi">scraperjs</span>.<span class="pl-smi">StaticScraper</span>.<span class="pl-en">create</span>() <span class="pl-c"><span class="pl-c">//</span> or DynamicScraper</span></pre>
         </div> 
         <p>The following promises can be made over it, they all return a scraper promise,</p> 
         <ul> 
          <li><code>onStatusCode(code:number, callback:function(utils:Object))</code>, executes the callback when the status code is equal to the code,</li> 
          <li><code>onStatusCode(callback:function(code:number, utils:Object))</code>, executes the callback when receives the status code. The callback receives the current status code,</li> 
          <li><code>delay(time:number, callback:function(last:?, utils:Object))</code>, delays the execution of the chain by time (in milliseconds),</li> 
          <li><code>timeout(time:number, callback:function(last:?, utils:Object))</code>, executes the callback function after time (in milliseconds),</li> 
          <li><code>then(lastResult:?, callback:function(last:?, utils:Object))</code>, executes the callback after the last promise,</li> 
          <li><code>async(callback:function(last:?, done:function(result:?, err:?), utils))</code>, executes the callback, stopping the promise chain, resuming it when the <code>done</code> function is called. You can provide a result to be passed down the promise chain, or an error to trigger the catch promise,</li> 
          <li><code>catch(callback:function(error:Error, utils:Object))</code>, executes the callback when there was an error, errors block the execution of the chain even if the promise was not defined,</li> 
          <li><code>done(callback:function(last:?, utils:Object))</code>, executes the callback at the end of the promise chain, this is always executed, even if there was an error,</li> 
          <li><code>get(url:string)</code>, makes a simple HTTP GET request to the url. This promise should be used only once per scraper.</li> 
          <li><code>request(options:Object)</code>, makes a (possibly) more complex HTTP request, scraperjs uses the <a href="https://github.com/mikeal/request">request</a> module, and this method is a simple wrapper of <code>request.request()</code>. This promise should be used only once per scraper.</li> 
          <li><code>scrape(scrapeFn:function(...?), callback:function(result:?, utils:Object)=, ...?)</code>, scrapes the page. It executes the scrapeFn and passes it's result to the callback. When using the StaticScraper, the scrapeFn receives a jQuery function that is used to scrape the page. When using the DynamicScraper, the scrapeFn doesn't receive anything and can only return a <a href="https://github.com/sgentle/phantomjs-node/wiki#evaluating-pages">JSON-serializable</a> type. Optionally an arbitrary number of arguments can be passed to the scraping function. The callback may be omitted, if so, the result of the scraping may be accessed with the <code>then</code> promise or <code>utils.lastReturn</code> in the next promise.</li> 
         </ul> 
         <p>All callback functions receive as their last parameter a utils object, with it the parameters of an url from a router can be accessed. Also the chain can be stopped.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">DynamicScraper</span>.<span class="pl-en">create</span>()
	.<span class="pl-c1">get</span>(<span class="pl-s"><span class="pl-pds">"</span>http://news.ycombinator.com<span class="pl-pds">"</span></span>)
	.<span class="pl-en">then</span>(<span class="pl-k">function</span>(<span class="pl-smi">_</span>, <span class="pl-smi">utils</span>) {
		<span class="pl-smi">utils</span>.<span class="pl-c1">stop</span>();
		<span class="pl-c"><span class="pl-c">//</span> utils.params.paramName</span>
	});</pre>
         </div> 
         <p>The promise chain is fired with the same sequence it was declared, with the exception of the promises get and request that fire the chain when they've received a valid response, and the promises <code>done</code> and <code>catch</code>, which were explained above.</p> 
         <p>You can also waterfall values between promises by returning them (with the exception of the promise <code>timeout</code>, that will always return <code>undefined</code>) and it can be access through <code>utils.lastReturn</code>.</p> 
         <h5><a id="user-content-the-utils-object" class="anchor" href="#the-utils-object" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>The <code>utils</code> object</h5> 
         <p>You've seen the <code>utils</code> object that is passed to promises, it provides useful information and methods to your promises. Here's what you can do with it:</p> 
         <ul> 
          <li><code>.lastResult</code>, value returned in the last promise</li> 
          <li><code>.stop()</code>, function to stop the promise chain,</li> 
          <li><code>.url</code>, url provided to do the scraper,</li> 
          <li><code>.params</code>, object with the parameters defined in the router matching pattern.</li> 
         </ul> 
         <h5><a id="user-content-a-more-powerful-dynamicscraper" class="anchor" href="#a-more-powerful-dynamicscraper" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>A more powerful DynamicScraper.</h5> 
         <p>When lots of instances of DynamicScraper are needed, it's creation gets really heavy on resources and takes a lot of time. To make this more lighter you can use a <em>factory</em>, that will create only one PhantomJS instance, and every DynamicScraper will request a page to work with. To use it you must start the factory before any DynamicSrcaper is created, <code>scraperjs.DynamicScraper.startFactory()</code> and then close the factory after the execution of your program, <code>scraperjs.DynamicScraper.closeFactory()</code>. To make the scraping function more robust you can inject code into the page,</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> ds <span class="pl-k">=</span> <span class="pl-smi">scraperjs</span>.<span class="pl-smi">DynamicScraper</span>
	.<span class="pl-en">create</span>(<span class="pl-s"><span class="pl-pds">'</span>http://news.ycombinator.com<span class="pl-pds">'</span></span>)
	.<span class="pl-en">async</span>(<span class="pl-k">function</span>(<span class="pl-smi">_</span>, <span class="pl-smi">done</span>, <span class="pl-smi">utils</span>) {
		<span class="pl-smi">utils</span>.<span class="pl-smi">scraper</span>.<span class="pl-en">inject</span>(<span class="pl-c1">__dirname</span><span class="pl-k">+</span><span class="pl-s"><span class="pl-pds">'</span>/path/to/code.js<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">err</span>) {
			<span class="pl-c"><span class="pl-c">//</span> in this case if there was an error won't fire catch promise.</span>
			<span class="pl-k">if</span>(err) {
				<span class="pl-en">done</span>(err);
			} <span class="pl-k">else</span> {
				<span class="pl-en">done</span>();
			}
		});
	})
	.<span class="pl-en">scrape</span>(<span class="pl-k">function</span>() {
		<span class="pl-k">return</span> <span class="pl-en">functionInTheCodeInjected</span>();
	})
	.<span class="pl-en">then</span>(<span class="pl-k">function</span>(<span class="pl-smi">result</span>) {
		<span class="pl-en">console</span>.<span class="pl-c1">log</span>(result);
	});</pre>
         </div> 
         <h4><a id="user-content-router" class="anchor" href="#router" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Router</h4> 
         <p>The router should be initialized like a class</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> router <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">scraperjs.Router</span>(options);</pre>
         </div> 
         <p>The options object is optional, and these are the options:</p> 
         <ul> 
          <li><code>firstMatch</code>, a boolean, if true the routing will stop once the first path is matched, the default is false.</li> 
         </ul> 
         <p>The following promises can be made over it,</p> 
         <ul> 
          <li><code>on(path:string|RegExp|function(url:string))</code>, makes the promise for the match url or regular expression, alternatively you can use a function to accept or not a passed url. The promises <code>get</code> or <code>request</code> and <code>createStatic</code> or <code>createDynamic</code> are expected after the on promise.</li> 
          <li><code>get()</code>, makes so that the page matched will be requested with a simple HTTP request,</li> 
          <li><code>request(options:Object)</code>, makes so that the page matched will be requested with a possible more complex HTTP request, , scraperjs uses the <a href="https://github.com/mikeal/request">request</a> module, and this method is a simple wrapper of <a href="https://github.com/mikeal/request#requestoptions-callback">request.request()</a>,</li> 
          <li><code>createStatic()</code>, associates a static scraper to use to scrape the matched page, this returns ScraperPromise, so any promise made from now on will be made over a ScraperPromise of a StaticScraper. Also the <code>done</code> promise of the scraper will not be available.</li> 
          <li><code>createDynamic()</code>, associates a dynamic scraper to use to scrape the matched page, this returns ScraperPromise, so any promise made from now on will be made over a ScraperPromise of a DynamicScraper. Also the <code>done</code> promise of the scraper will not be available.</li> 
          <li><code>route(url:string, callback:function(boolean))</code>, routes an url through all matched paths, calls the callback when it's executed, true is passed if the route was successful, false otherwise.</li> 
          <li><code>use(scraperInstance:ScraperPromise)</code>, uses a ScraperPromise already instantiated.</li> 
          <li><code>otherwise(callback:function(url:string))</code>, executes the callback function if the routing url didn't match any path.</li> 
          <li><code>catch(callback:function(url:string, error:Error))</code>, executes the callback when an error occurred on the routing scope, not on any scraper, for that situations you should use the <code>catch</code> promise of the scraper.</li> 
         </ul> 
         <h4><a id="user-content-notes" class="anchor" href="#notes" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Notes</h4> 
         <ul> 
          <li>Scraperjs <strong>always</strong> fetches the document with <code>request</code>, and then when using a DynamicScraper, leverages phantom's <code>setContent()</code> to set the body of the page object. This will result in subtly different processing of web pages compared to directly loading a URL in PhantomJS.</li> 
         </ul> 
         <h4><a id="user-content-more" class="anchor" href="#more" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>More</h4> 
         <p>Check the <a href="/ruipgil/scraperjs/blob/master/doc/examples">examples</a>, the <a href="/ruipgil/scraperjs/blob/master/test">tests</a> or just dig into the code, it's well documented and it's simple to understand.</p> 
         <h1><a id="user-content-dependencies" class="anchor" href="#dependencies" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Dependencies</h1> 
         <p>As mentioned above, scraperjs is uses some dependencies to do the the heavy work, such as</p> 
         <ul> 
          <li><a href="https://github.com/caolan/async"><code>async</code></a>, for flow control</li> 
          <li><a href="https://github.com/mikeal/request"><code>request</code></a>, to make HTTP requests, again, if you want more complex requests see it's <a href="https://github.com/mikeal/request#requestoptions-callback">documentation</a></li> 
          <li><a href="https://github.com/sgentle/phantomjs-node"><code>phantom</code></a> + <a href="https://github.com/ariya/phantomjs"><code>phantomjs</code></a>, phantom is an awesome module that links node to phantom, used in the DynamicScraper</li> 
          <li><a href="https://github.com/cheeriojs/cheerio"><code>cheerio</code></a>, light and fast DOM manipulation, used to implement the StaticScraper</li> 
          <li><a href="https://github.com/jquery/jquery"><code>jquery</code></a>, to include jquery in the DynamicScraper</li> 
          <li>although <a href="https://github.com/aaronblohowiak/routes.js"><code>Routes.js</code></a> is great, scraperjs doesn't use it to maintain it's "interface layout", but the code to transform the path given on the on promise to regular expressions is from them</li> 
         </ul>
