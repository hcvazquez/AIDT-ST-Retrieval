                 <!-- migxf:done -->Hi Hakan,
                 <br> 
                 <br> In Miscrosoft Excel, by default, numbers are used to represent dates. Excel uses the following format to store numbers:
                 <br> 
                 <br> 1st Jan 1900 is stored as 1
                 <br> 2nd Jan 1900 is stored as 2
                 <br> and so on.... Every subsequent number represents the next date.
                 <br> 
                 <br> If you extend this, you will see that 37894 represents 30 Sep 2003.
                 <br> 
                 <br> So as Ino said, it is an internal Excel representation. Other software like Oracle Database do not accept this particular format, but will have their own representation. So there is no direct way to convert numbers to dates inside non-Excel software. You can convert Excel to convert all these dates and store them back in the database as dates.
                 <br>
