    <p>General </p>
    <p>As a JAVA solution, Mobildev JAVA API enables software developers to integrate SMS technology into their solutions and helps them develop platform independent applications in a cost-free way. <br> <br>JAVA API is accessing <a href="http://gateway.mobilus.net/com.mobilus%20"><strong><font color="#0066ff">http://gateway.mobilus.net/com.mobilus</font></strong></a> address over HTTP port 80 and thus accessing Mobildev SMS Gateway. If your network is behind a firewall, the computer that uses Mobildev JAVA API must be enabled accessing this address with data transmission in both ways.<br><br></p>

    <p><a name="sistem"></a>System Requirements</p>
    <p>Following is the minimum system requirements list in order to use Mobildev JAVA SMS API solution.</p>
    <ul> 
     <li> <p>Windows 98 SE, ME, 2000, XP, 2003, Linux, UNIX or MAC OS</p></li> 
     <li> <p>Internet Explorer 6 SP1 (for Windows Platform)</p></li> 
     <li> <p>JVM - JAVA Virtual Machine (for each operating system)<br> </p> </li>
    </ul>
    <p><a name="kullanici"></a><br>User Information Control</p>
    <p><em>Before starting with sending SMS, SMS user information control must be carried out. If you don’t have a current SMS account please <a href="http://www.mobildev.com/basvuru.asp?bid=2"><font color="#0000FF">CLICK HERE</font></a> to obtain a test account from Mobildev Customer Services. <br> <br> </em>Mobildev JAVA API offers a method called CreditReporter for user information control. In order to have the user information checked by the Gateway, you will need to call SendMessage () method by inputting your user name (username-company code) to SetUsername(), your password to SetPassword() as parameters. The returning values will be the credit balance and the originator information. The following example shows how CreditReporter method is used. <br><br></p>

    <p>After submitting the CreditReporter method to the Gateway as in the example, the below values will be returned:</p>

    <br>
    <p>After checking the information sent to Gateway, 01 Error Code will be returned if the information is not correct. Error Code 01 means that one or all of the following is incorrect: user name – password – company code. In this case, please contact with Mobildev Technical Support Team and check your login details.</p>
    <p><br> <a name="karakter"></a>Notice Regarding Characters To Be Used</p>
    <p>During sending, the below is not allowed:</p>
    <ul> 
     <li> <p>Turkish characters (ç,?,ü,ö,?,Ç,Ü,?,?,Ö,?) .</p></li> 
     <li> <p>~, ^^, &amp;, &lt;, &gt;, €, £, ã, Ã, ‰, €, ©, ®, ?, ° signs</p></li> 
     <li> <p>LineFeed, Enter, Tab commands</p> </li>
    </ul>
    <p>Within a message, only the following characters are allowed: ( ) [ ] { } = * ? \ / % + # ! ” : ; _ |<br></p>
    <p><a name="gsm"></a><br>Notice Regarding GSM Number Format</p>
    <p>GSM numbers must be formatted as <em>05321234567</em>, <em>5321234567</em>, <em>905321234567</em> and written within <em>Numbers</em> parameter node. If the GSM numbers are not written in this format, 04 Error Code will be returned. Error Code 04 means that that GSM number format is not correct. In this case, please consider checking the Notice Regarding GSM Number Format section.</p>

    <p><br> <a name="toplusms"></a>Notice Regarding Bulk SMS Sending Method</p>
    <p>Mobildev JAVA API provides 2 methods, namely:</p> 
    <p></p>
    <ol> 
     <li> SmsToMany for sending the same message to different numbers, </li> 
     <li>SmsMultiSender for sending different messages to different people.</li> 
    </ol>

    <p><br> <strong>SmsToMany Method</strong></p>
    <p>SmsToMany method is used for <u><strong>sending the same message to different GSM numbers</strong></u>. In order to do this, after getting a sample SmsToMany method, </p>
    <ul> 
     <li> <p>Input user name to SetUsername() method,</p></li> 
     <li> <p>Input password to SetPassword() method,</p></li> 
     <li> <p>Input originator information to SetOriginator() method,</p></li> 
     <li> <p>Input message text to SetMessage() method (text to be no longer than 160 characters, not to be containing any Turkish characters and the following signs: &amp;, &lt;, &gt;, -, ( ), [ ], /),</p></li> 
     <li> <p>Input DayMonthYearHourMinute (ddmmyyyyhhmm) to SetDate() method, if you want to send messages at a later specified time,</p></li> 
     <li> <p>Input GSM Numbers to SetNumbers() method, after this, if you call SendMessage() method, the SMS package will be submitted to the Gateway.</p></li> 
     <li> <p>Once the package is processed by the Gateway, an appropriate return value will be generated. You can use the return value by transferring SendMessage() method to a variable. </p> </li>
    </ul>
    <table border="0" cellspacing="2" cellpadding="2" width="100%">

    <p>If you submit the SMS package with the SendMessage() method, the Gateway will return the following values:</p>
    <p><strong>1. If successful:</strong> TimerID information in “ID: 1234567” format will be returned. TimerID is the automatically assigned number to the SMS packages. </p>
    <p><strong>2. If an error occurred, </strong>please consider checking Error Codes Section.<br></p>
    <p><strong><br>SmsMultiSender Method </strong></p>
    <p>SmsMultiSender method is used for <u>sending different messages to different GSM numbers</u>. In order to do this, after getting a sample SmsToMany method, </p> 
    <p></p>
    <ul> 
     <li> Input user name to SetUsername() method,</li> 
     <li>Input password to SetPassword() method,</li> 
     <li>Input originator information to SetOriginator() method,</li> 
     <li>Input message text to SetMessage() method (text to be no longer than 160 characters, not to be containing any Turkish characters and the following signs: &amp;, &lt;, &gt;, -, ( ), [ ], /),</li> 
     <li>Input DayMonthYearHourMinute (ddmmyyyyhhmm) to SetDate() method, if you want to send messages at a later specified time,</li> 
     <li>Once the package is processed by the Gateway, an appropriate return value will be generated. You can use the return value by transferring SendMessage() method to a variable. </li> 
    </ul> 
    <strong>SetUsername</strong> parameter refers to user name associated with your Mobildev SMS account. 
    <br> 
    <strong>SetPassword</strong> parameter, refers to password for your SMS user name.
    <br> 
    <strong>SetOriginator</strong> parameter, refers to &nbsp;user name that will be displayed on recipient’s GSM number.
    <br>
    <strong>SetDate</strong> parameter refers to the sending time of the message. 
    <br>
    <br>
