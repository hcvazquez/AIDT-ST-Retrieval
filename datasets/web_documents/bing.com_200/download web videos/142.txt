       <h2><a id="purpose">
         <!----></a>Purpose</h2> 
       <p>Microsoft Windows Installer is an installation and configuration service provided with Windows. The installer service enables customers to provide better corporate deployment and provides a standard format for component management. The installer also enables the advertisement of applications and features according to the operating system. For more information, see <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa370835(v=vs.85).aspx">Platform Support of Advertisement</a>.</p> 
       <p>This documentation describes Windows Installer&nbsp;5.0 and earlier versions. Not all the capabilities available in later Windows Installer versions are available in earlier versions. This documentation does not describe versions earlier than Windows Installer&nbsp;2.0. Installation packages and patches that are created for Windows Installer&nbsp;2.0 can still be installed by using Windows Installer&nbsp;3.0 and later.</p> 
       <p>Windows Installer&nbsp;3.0 and later, can install multiple patches with a single transaction that integrates installation progress, rollback, and reboots. The installer can apply patches in a specified order regardless of the order that the patches are provided to the system. Patching using Windows Installer&nbsp;3.0 only updates files affected by the patch and can be significantly faster than earlier installer versions. Patches installed with Windows Installer 3.0 or later can be uninstalled in any order to leave the state of the product the same as if the patch was never installed. Accounts with administrator privileges can use the API of Windows Installer&nbsp;3.0 and later to query and inventory product, feature, component, and patch information. The installer can be used to read, edit, and replace source lists for network, URL, and media sources. Administrators can enumerate across user and install contexts, and manage source lists from an external process.</p> 
       <p>Windows Installer 4.5 and later can install multiple installation packages using <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa372093(v=vs.85).aspx#_msi_transaction_processing_gly"><em xmlns="http://www.w3.org/1999/xhtml">transaction processing</em></a>. If all the packages in the transaction cannot be installed successfully, or if the user cancels the installation, the Windows Installer can roll back changes and restore the computer to its original state. The installer ensures that all the packages belonging to a multiple-package transaction are installed or none of the packages are installed. </p> 
       <p>Beginning with Windows Installer 5.0, a package can be authored to secure new accounts, Windows <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/ms685141(v=vs.85).aspx">Services</a>, files, folders, and registry keys. The package can specify a security descriptor that denies permissions, specifies inheritance of permissions from a parent resource, or specifies the permissions of a new account. For information, see <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dd408053(v=vs.85).aspx">Securing Resources</a>. The Windows Installer 5.0 service can enumerate all components installed on the computer and obtain the key path for the component. For more information, see <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dd407934(v=vs.85).aspx">Enumerating Components</a>. By <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dd408113(v=vs.85).aspx">Using Services Configuration</a>, Windows Installer 5.0 packages can customize the services on a computer. Setup developers can use Windows Installer 5.0 and <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dd408068(v=vs.85).aspx">Single Package Authoring</a> to develop single installation packages capable of installing an application in either the per-machine or per-user <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dd765197(v=vs.85).aspx">installation context</a>. </p> 
       <h2><a id="where_applicable">
         <!----></a>Where applicable</h2> 
       <p>Windows Installer enables the efficient installation and configuration of your products and applications running on Windows. The installer provides new capabilities to advertise features without installing them, to install products on demand, and to add user customizations.</p> 
       <p>Windows Installer 5.0 running on Windows Server&nbsp;2012 or Windows&nbsp;8 supports the installation of approved apps on Windows&nbsp;RT. A Windows Installer package, patch, or transform that has not been signed by Microsoft cannot be installed on Windows&nbsp;RT. The <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa372070(v=vs.85).aspx"><strong xmlns="http://www.w3.org/1999/xhtml">Template Summary</strong></a> property indicates the platform that is compatible with an installation database and in this case should include the value for Windows&nbsp;RT. </p> 
       <p>Windows Installer is intended for the development of desktop style applications.</p> 
       <h2><a id="developer_audience_heading">
         <!----></a>Developer audience</h2> 
       <p>This documentation is intended for software developers who want to make applications that use Windows Installer. It provides general background information about installation packages and the installer service. It contains complete descriptions of the application programming interface and elements of the installer database. This documentation also contains supplemental information for developers who want to use a table editor or a package creation tool to make or maintain an installation.</p> 
       <h2><a id="run_time_requirements_heading">
         <!----></a>Run-time requirements</h2> 
       <p>Windows Installer&nbsp;5.0 will be released with, and require Windows Server&nbsp;2012, Windows&nbsp;8, Windows Server&nbsp;2008&nbsp;R2 or Windows&nbsp;7. Versions earlier than Windows Installer 5.0 were released with Windows Server&nbsp;2008, Windows&nbsp;Vista, Windows Server&nbsp;2003, Windows&nbsp;XP, and Windows&nbsp;2000.</p> 
       <p>Windows Installer&nbsp;4.5 requires Windows Server&nbsp;2008, Windows&nbsp;Vista, Windows&nbsp;XP with Service Pack&nbsp;2 (SP2) and later, and Windows Server&nbsp;2003 with Service Pack&nbsp;1 (SP1) and later.</p> 
       <p>Windows Installer&nbsp;4.0 requires Windows&nbsp;Vista or Windows Server&nbsp;2008. There is no redistributable for installing Windows Installer&nbsp;4.0 on other operating systems. An updated version of Windows Installer&nbsp;4.0, which does not add any new features, is available in Windows&nbsp;Vista with Service Pack&nbsp;1 (SP1) and Windows Server&nbsp;2008.</p> 
       <p>Windows Installer&nbsp;3.1 requires Windows Server&nbsp;2003, Windows&nbsp;XP, or Windows&nbsp;2000 with Service Pack&nbsp;3 (SP3).</p> 
       <p>Windows Installer&nbsp;3.0 requires Windows Server&nbsp;2003, Windows&nbsp;XP, or Windows&nbsp;2000 with SP3. Windows Installer&nbsp;3.0 is included in Windows&nbsp;XP with Service Pack&nbsp;2 (SP2). It is available as a redistributable for Windows&nbsp;2000 Server with Service Pack&nbsp;3 (SP3) and Windows&nbsp;2000 Server with Service Pack&nbsp;4 (SP4), Windows&nbsp;XP RTM and Windows&nbsp;XP with Service Pack&nbsp;1 (SP1), and Windows Server&nbsp;2003 RTM.</p> 
       <p> <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa372856(v=vs.85).aspx">Windows Installer Redistributables</a> are available for Windows Installer 4.5 and some earlier versions. There is no redistributable available for Windows Installer 4.0.</p> 
       <p>Windows Installer&nbsp;2.0 is contained in Windows Server&nbsp;2003 and Windows&nbsp;XP.</p> 
       <p>Windows Installer&nbsp;2.0 is available as a package for installing or upgrading to Windows Installer&nbsp;2.0 on Windows&nbsp;2000. This package should not be used to install or upgrade Windows Installer&nbsp;2.0 on Windows Server&nbsp;2003 and Windows&nbsp;XP.</p> 
       <p>You can find all the available Windows Installer redistributables at the <a href="http://go.microsoft.com/fwlink/p/?LinkID=83468">Microsoft Download Center</a>. </p>
