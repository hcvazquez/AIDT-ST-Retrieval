        <h1 class="toc-linked"><a href="#validate-forms-like-you-39-ve-never-validated-before" id="validate-forms-like-you-39-ve-never-validated-before" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Validate forms like you've never validated before!</h1> 
        <p><strong>"But doesn't jQuery make it easy to write your own validation plugin?"</strong><br> Sure, but there are still a lot of subtleties to take care of: You need a standard library of validation methods (such as emails, URLs, credit card numbers). You need to place error messages in the DOM and show and hide them when appropriate. You want to react to more than just a submit event, like keyup and blur.<br> You may need different ways to specify validation rules according to the server-side enviroment you are using on different projects. And after all, you don't want to reinvent the wheel, do you?</p> 
        <p><strong>"But aren't there already a ton of validation plugins out there?"</strong><br> Right, there are a lot of non-jQuery-based solutions (which you'd avoid since you found jQuery) and some jQuery-based solutions. This particular one is one of the oldest jQuery plugins (started in July 2006) and has proved itself in projects all around the world. There is also an <a href="http://bassistance.de/2007/07/04/about-client-side-form-validation-and-frameworks/">article</a> discussing how this plugin fits the bill of the should-be validation solution.</p> 
        <p>Not convinced? <a href="https://jqueryvalidation.org/files/demo/">Have a look at this example</a>:</p>

        <h3 class="toc-linked"><a href="#isn-39-t-that-nice-and-easy" id="isn-39-t-that-nice-and-easy" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Isn't that nice and easy?</h3> 
        <p>A single line of jQuery to select the form and apply the validation plugin, plus a few annotations on each element to specify the validation rules.</p> 
        <p>Of course that isn't the only way to specify rules. You also don't have to rely on those default messages, but they come in handy when starting to setup validation for a form.</p> 
        <h3 class="toc-linked"><a href="#a-few-things-to-look-out-for-when-playing-around-with-the-demo" id="a-few-things-to-look-out-for-when-playing-around-with-the-demo" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> A few things to look out for when playing around with the demo</h3> 
        <ul> 
         <li>After trying to submit an invalid form, the first invalid element is focused, allowing the user to correct the field. If another invalid field – that wasn't the first one – was focused before submit, that field is focused instead, allowing the user to start at the bottom if he or she prefers.</li> 
         <li>Before a field is marked as invalid, the validation is lazy: Before submitting the form for the first time, the user can tab through fields without getting annoying messages – they won't get bugged before having the chance to actually enter a correct value</li> 
         <li>Once a field is marked invalid, it is eagerly validated: As soon as the user has entered the necessary value, the error message is removed</li> 
         <li>If the user enters something in a non-marked field, and tabs/clicks away from it (blur the field), it is validated – obviously the user had the intention to enter something, but failed to enter the correct value</li> 
        </ul> 
        <p>That behaviour can be irritating when clicking through demos of the validation plugin – it is designed for an unobtrusive user experience, annoying the user as little as possible with unnecessary error messages. So when you try out other demos, try to react like one of your users would, and see if the behaviour is better then. If not, please let me know about any ideas you may have for improvements!</p> 
        <h1 class="toc-linked"><a href="#api-documentation" id="api-documentation" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> API Documentation</h1> 
        <p>You're probably looking for</p> 
        <h3 class="toc-linked"><a href="#options-for-the-validate-method" id="options-for-the-validate-method" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="/validate">Options for the validate() method</a></h3> 
        <p>If not, read on.</p> 
        <p>Throughout the documentation, two terms are used very often, so it's important that you know their meaning in the context of the validation plugin:</p> 
        <ul> 
         <li><strong>method</strong>: A validation method implements the logic to validate an element, like an email method that checks for the right format of a text input's value. A set of standard methods is available, and it is easy to write your own.</li> 
         <li><strong>rule</strong>: A validation rule associates an element with a validation method, like "validate input with name "primary-mail" with methods "required" and "email".</li> 
        </ul> 
        <h2 class="toc-linked"><a href="#plugin-methods" id="plugin-methods" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Plugin methods</h2> 
        <p>This library adds three jQuery plugin methods, the main entry point being the <code>validate</code> method:</p> 
        <ul> 
         <li><a href="/validate"><code>validate()</code> – Validates the selected form.</a></li> 
         <li><a href="/valid"><code>valid()</code> – Checks whether the selected form or selected elements are valid.</a></li> 
         <li><a href="/rules"><code>rules()</code> – Read, add and remove rules for an element.</a></li> 
        </ul> 
        <h2 class="toc-linked"><a href="#custom-selectors" id="custom-selectors" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Custom selectors</h2> 
        <p>This library also extends jQuery with three custom selectors:</p> 
        <ul> 
         <li><a href="/blank-selector"><code>:blank</code> – Selects all elements with a blank value.</a></li> 
         <li><a href="/filled-selector"><code>:filled</code> – Selects all elements with a filled value.</a></li> 
         <li><a href="/unchecked-selector"><code>:unchecked</code> – Selects all elements that are unchecked.</a></li> 
        </ul> 
        <h2 class="toc-linked"><a href="#validator" id="validator" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Validator</h2> 
        <p>The validate method returns a Validator object that has a few public methods that you can use to trigger validation programmatically or change the contents of the form. The validator object has more methods, but only those documented here are intended for usage.</p> 
        <ul> 
         <li><a href="/Validator.form"><code>Validator.form()</code> – Validates the form.</a></li> 
         <li><a href="/Validator.element"><code>Validator.element()</code> – Validates a single element.</a></li> 
         <li><a href="/Validator.resetForm"><code>Validator.resetForm()</code> – Resets the controlled form.</a></li> 
         <li><a href="/Validator.showErrors"><code>Validator.showErrors()</code> – Show the specified messages.</a></li> 
         <li><a href="/Validator.numberOfInvalids"><code>Validator.numberOfInvalids()</code> – Returns the number of invalid fields.</a></li> 
         <li><a href="/Validator.destroy"><code>Validator.destroy()</code> – Destroys this instance of validator.</a></li> 
        </ul> 
        <p>There are a few static methods on the validator object:</p> 
        <ul> 
         <li><a href="/jQuery.validator.addMethod"><code>jQuery.validator.addMethod()</code> – Add a custom validation method.</a></li> 
         <li><a href="/jQuery.validator.format"><code>jQuery.validator.format()</code> – Replaces {n} placeholders with arguments.</a></li> 
         <li><a href="/jQuery.validator.setDefaults"><code>jQuery.validator.setDefaults()</code> – Modify default settings for validation.</a></li> 
         <li><a href="/jQuery.validator.addClassRules"><code>jQuery.validator.addClassRules()</code> – Add a compound class method.</a></li> 
        </ul> 
        <h2 class="toc-linked"><a href="#list-of-built-in-validation-methods" id="list-of-built-in-validation-methods" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> List of built-in Validation methods</h2> 
        <p>A set of standard validation methods is provided:</p> 
        <ul> 
         <li><a href="/required-method"><code>required</code> – Makes the element required.</a></li> 
         <li><a href="/remote-method"><code>remote</code> – Requests a resource to check the element for validity.</a></li> 
         <li><a href="/minlength-method"><code>minlength</code> – Makes the element require a given minimum length.</a></li> 
         <li><a href="/maxlength-method"><code>maxlength</code> – Makes the element require a given maximum length.</a></li> 
         <li><a href="/rangelength-method"><code>rangelength</code> – Makes the element require a given value range.</a></li> 
         <li><a href="/min-method"><code>min</code> – Makes the element require a given minimum.</a></li> 
         <li><a href="/max-method"><code>max</code> – Makes the element require a given maximum.</a></li> 
         <li><a href="/range-method"><code>range</code> – Makes the element require a given value range.</a></li> 
         <li><a href="/step-method"><code>step</code> – Makes the element require a given step.</a></li> 
         <li><a href="/email-method"><code>email</code> – Makes the element require a valid email</a></li> 
         <li><a href="/url-method"><code>url</code> – Makes the element require a valid url</a></li> 
         <li><a href="/date-method"><code>date</code> – Makes the element require a date.</a></li> 
         <li><a href="/dateISO-method"><code>dateISO</code> – Makes the element require an ISO date.</a></li> 
         <li><a href="/number-method"><code>number</code> – Makes the element require a decimal number.</a></li> 
         <li><a href="/digits-method"><code>digits</code> – Makes the element require digits only.</a></li> 
         <li><a href="/equalTo-method"><code>equalTo</code> – Requires the element to be the same as another one</a></li> 
        </ul> 
        <p>Some more methods are provided as add-ons, and are currently included in additional-methods.js in the download package. Not all of them are documented here:</p> 
        <ul> 
         <li><a href="/accept-method"><code>accept</code> – Makes a file upload accept only specified mime-types.</a></li> 
         <li><a href="/creditcard-method"><code>creditcard</code> – Makes the element require a credit card number.</a></li> 
         <li><a href="/extension-method"><code>extension</code> – Makes the element require a certain file extension.</a></li> 
         <li><a href="/phoneUS-method"><code>phoneUS</code> – Validate for valid US phone number.</a></li> 
         <li><a href="/require_from_group-method"><code>require_from_group</code> – Ensures a given number of fields in a group are complete.</a></li> 
        </ul> 
        <p>You can find the <a href="https://github.com/jzaefferer/jquery-validation/tree/master/src/additional">source code for all additional methods in the GitHub repository</a>.</p> 
        <p>It is possible to re-define the implementation of the built-in rules using the <a href="/jQuery.validator.methods/"><code>$.validator.methods</code> property</a></p> 
        <h1 class="toc-linked"><a href="#general-guidelines" id="general-guidelines" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="/reference">General Guidelines</a></h1> 
        <p>The General Guidelines section provides detailed discussion of the design and ideas behind the plugin, explaining why certain things are as they are. It covers the features in more detail than the API documentation, which just briefly explains the various methods and options available.</p> 
        <p>If you've decided to use the validation plugin in your application and want to get to know it better, it is recommended that you read the guidelines.</p> 
        <h2 class="toc-linked"><a href="#fields-with-complex-names-brackets-dots" id="fields-with-complex-names-brackets-dots" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> Fields with complex names (brackets, dots)</h2> 
        <p>When you have a name attribute like user[name], make sure to put the name in quotes. More details in the <a href="/reference">General Guidelines</a>.</p>

        <h2 class="toc-linked"><a href="#the-marketo-sign-up-form" id="the-marketo-sign-up-form" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="https://jqueryvalidation.org/files/demo/marketo/">The Marketo sign-up form</a></h2> 
        <h3 class="toc-linked"><a href="#the-marketo-sign-up-form-step-2" id="the-marketo-sign-up-form-step-2" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="https://jqueryvalidation.org/files/demo/marketo/step2.htm">The Marketo sign-up form, step 2</a></h3> 
        <p>Based on an old version of the marketo.com sign-up form. The custom validation was once replaced with this plugin. Thanks to Glen Lipka for contributing it!</p> 
        <p><em>Notable features of the demo:</em></p> 
        <ul> 
         <li>Customized message display: No messages displayed for the required method, only for typing-errors (like wrong email format); A summary is displayed at the top ("You missed 12 fields. They have been highlighted below.")</li> 
         <li>Remote validation of email field. Try to enter eg. <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="70171c151e301d11021b15041f5e131f1d">[email&nbsp;protected]</a></li> 
         <li>Integration with masked-input plugin, see Zip and Phone fields and Credit Card Number on step 2</li> 
         <li>A custom method for making the billing address on step 2 optional when "Same as Company Address" is checked</li> 
         <li>A custom method for checking the password: Checks that the password contains at least one number and one character and that it is at least 6 characters long. If the user blurs the field with an invalid value, the input is emptied and gets focus again.</li> 
        </ul> 
        <h2 class="toc-linked"><a href="#the-remember-the-milk-sign-up-form" id="the-remember-the-milk-sign-up-form" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="https://jqueryvalidation.org/files/demo/milk/">The Remember The Milk sign-up form</a></h2> 
        <p>The sign-up form from rememberthemilk.com (based on an older version). The custom validation was replaced using this plugin. Thanks to RTM for contributing!</p> 
        <p><em>Notable features of the demo:</em></p> 
        <ul> 
         <li>Custom message display, based on the original table layout, using success option to display a checkmark for valid fields</li> 
         <li>Remote validation of username, to check if it is already taken (try "Peter", "asdf" or "George")</li> 
        </ul> 
        <h2 class="toc-linked"><a href="#a-multipart-quot-buy-amp-sell-a-house-quot-form" id="a-multipart-quot-buy-amp-sell-a-house-quot-form" class="icon-link toc-link"><span class="visuallyhidden">link</span></a> <a href="https://jqueryvalidation.org/files/demo/multipart/">A multipart "buy&amp;sell a house" form</a></h2> 
        <p>Contributed by Michael Evangelista, showing a multipart form for buying and selling houses.</p> 
        <p><em>Notable features of the demo:</em></p>

         <li>Multipart, implemented using the jQuery UI accordion and a custom method to check if an element is on the current page when validated</li> 
         <li>Integration with masked-input plugin, see Phone and Zip fields</li> 
        </ul>
