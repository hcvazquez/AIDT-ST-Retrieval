       <h1 class="entry-title">Four Serious Math Libraries for JavaScript</h1> 
       <div class="post-info">
        <span class="date published time" title="2013-10-23T09:11:25+00:00">October 23, 2013</span> by 
        <a href="https://blog.smartbear.com/author/john-mueller/" title="Posts by John Mueller" class="author url fn" rel="author">John Mueller</a> 
        <span class="post-comments"><a href="https://blog.smartbear.com/testing/four-serious-math-libraries-for-javascript/#comments">10 Comments</a></span> 
       </div>
       <div class="entry-content">
        <div> 
         <p>&nbsp;<a href="http://blog.smartbear.com/wp-content/uploads/2013/10/122403023.jpg" class="gallery_colorbox"><img class="aligncenter" alt="math libraries" src="https://blog.smartbear.com/wp-content/uploads/2013/10/122403023.jpg" width="414" height="414"></a></p> 
         <p><i>JavaScript performs all its math tasks using floating point math. Using floating point numbers works fine up to a point, but then you encounter rounding errors. If you regularly need high-precision numeric output (such as performing financial or scientific calculations), you need one or more of these libraries.</i></p> 
        </div> 
        <p>Some math tasks require extreme precision. Imagine being just a little off when you calculate the trajectory to Mars, causing the spaceship to miss the planet entirely. Or think about how you’d feel if the interest calculation for your mortgage was imprecise (in the bank’s favor) or the ingredients in your prescription were proportioned slightly different from the doctor’s intent (enough to kill you).</p> 
        <p>Precise math calculations is a requirement for many software applications, but JavaScript and other languages can’t provide such precision without help due to the nature of math and the physics of computer hardware. If you need precise output, then you also need a library that helps you go beyond the usual definition of “precise enough.”</p> 
        <p>Of course, the basic question that most people rightfully ask about now is why such libraries are necessary at all. The main culprit is rounding errors. Some numbers can’t be represented in some bases. For example, 1/3 presents a problem when working in base 10 because it’s a repeating decimal and there isn’t any way to represent it precisely. There are also irrational numbers such as ? (pi—a value of approximately 3.1415926535897932384626433832795 – and that “approximately” is the first intimation of problems for some scientific calculations).</p> 
        <p>Computer rounding errors are compounded because there’s a difference in base (the range of numbers used to perform a calculation). All computer applications perform math using <a title="Base 2 (Binary)" href="http://www.jimloy.com/math/base2.htm" target="_blank">base 2</a> (with values of 0 and 1), even though you see the output as a base 10 (with values between 0 and 9) representation. Interestingly enough, even though you can represent 1/10 as 0.1 in base 10, it becomes a repeating number in base 2 (0.000110011…). Plus, JavaScript must also contend with rounding errors introduced by representing all numbers in <a title="JavaScript Number Object" href="http://www.w3schools.com/js/js_obj_number.asp" target="_blank">64-bit</a> <a title="Floating Point and JavaScript" href="http://javascript.about.com/od/hintsandtips/a/Floating-Point-And-Javascript.htm" target="_blank">floating point</a> form. Consequently, the act of performing precision math that appears in human readable form is a difficult undertaking.</p> 
        <p>The approximations you get are quite usable for most purposes. Obtaining the precision output is time consuming, and making an application run a lot slower in order to obtain precision you don’t need doesn’t make sense.</p> 
        <p>But when it does matter, it <i>really</i> matters. However, finding a true, hard core, JavaScript math library can be a bit vexing. Many offerings don’t actually offer math functionality, even though they’re billed as math libraries. And the math libraries vary greatly in their functionality.</p> 
        <p>The following libraries help you perform specific kinds of precision math. Perhaps I can save you time in comparing what’s available and appropriate for your own JavaScript applications.</p> 
        <h1><a title="sjkaliski / numbers.js GitHub" href="https://github.com/sjkaliski/numbers.js/" target="_blank">numbers.js</a></h1> 
        <p>The numbers.js library provides substantial math functionality for server-side use. You can use it to perform tasks in these categories:</p> 
        <ul> 
         <li>Basic calculations (such as adding two numbers)</li> 
         <li>Complex numbers</li> 
         <li>Calculus</li> 
         <li>Digital Signal Processing (DSP)</li> 
         <li>Matrix operations</li> 
         <li>Computing prime numbers</li> 
         <li>Statistics</li> 
         <li>Generation calculations (such as Fibonacci)</li> 
        </ul> 
        <p>In order to use numbers.js, you must install it on your server (actually in a Web server folder) and ensure that you have <a title="nodejs" href="http://nodejs.org/download/" target="_blank">node.js</a> installed on your system. You use node.js to create server-side, not client-side, applications. After you install <span style="text-decoration: underline;">node.js</span>, you open the node.js command prompt (part of the installed <span style="text-decoration: underline;">node.js</span> functionality), change directories to wherever you want to access <span style="text-decoration: underline;">numbers.js</span> (such as a directory on your server), type <span style="text-decoration: underline;">npm install numbers</span>, and press Enter. You now have the <span style="text-decoration: underline;">numbers.js</span> library installed on your server for use in server-side JavaScript applications. (There isn’t space in this article to show you how to use node.js in a server application, but there are many excellent resources such as <a title="The Node Beginner Book" href="http://www.nodebeginner.org/" target="_blank">The Node Beginner Book</a>, which has a wonderful <a title="&quot;Hello World&quot; -- The Node Beginner Book" href="http://www.nodebeginner.org/#hello-world" target="_blank">Hello World</a> example.)</p> 
        <p>The documentation with number.js is terrible, which is unfortunate because the library does provide a wealth of functionality. The <a title="Advanced Mathematics Library for Node.js and JavaScript" href="https://github.com/sjkaliski/numbers.js/" target="_blank">download page</a> isn’t much help either because it doesn’t provide you with a full list of functions. Follow these steps to see a full list of functions in the current version of numbers.js:</p> 
        <ol> 
         <li>Type <span style="text-decoration: underline;">node</span> and press Enter at the node.js command prompt. You see a greater than sign (&gt;) appear on screen.</li> 
         <li>Type <span style="text-decoration: underline;">var Test = require(‘numbers’);</span> and press Enter. This action loads the library so you can work with it.</li> 
         <li>Type <span style="text-decoration: underline;">console.log(Test);</span> and press Enter. You now see a full hierarchical list of functions as shown in Figure 1.</li> 
        </ol> 
        <p><a href="http://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-01.png" class="gallery_colorbox"><img class="aligncenter size-large wp-image-14435" alt="Four Serious Math Libraries for JavaScript 01" src="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-01-600x303.png" width="600" height="303" srcset="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-01-600x303.png 600w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-01-300x151.png 300w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-01.png 686w" sizes="(max-width: 600px) 100vw, 600px"></a></p> 
        <p>At this point, you can try out some of the functions. For example, let’s say you want to determine whether 5 is a prime number. In this case, you’d type <span style="text-decoration: underline;">console.log(Test.prime.simple(5));</span> and press Enter. The result is true, much as you’d expect. To test a specific function, you start with the name of the variable containing the library reference (Test), the name of the calculation module (prime), followed by the name of the function (simple), and end with any required input variables.</p> 
        <p><b>Demo:</b> <a title="Advanced Mathematics Library for Node.js and JavaScript" href="https://github.com/sjkaliski/numbers.js/" target="_blank">https://github.com/sjkaliski/numbers.js/</a></p> 
        <p><b>Download:</b> Performed through node.js</p> 
        <p><b>Documentation:</b> Pretty much non-existent</p> 
        <h1><a title="Numerical analysis in Javascript" href="http://www.numericjs.com/" target="_blank">Numeric Javascript</a></h1> 
        <p>The Numeric Javascript library focuses on these categories of calculation:</p> 
        <ul> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;Linear algebra</li> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;Complex numbers</li> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;Splines</li> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;ODE solver</li> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;Unconstrained optimization</li> 
         <li>&nbsp;&nbsp;&nbsp;&nbsp;PDE and sparse linear algebra</li> 
        </ul> 
        <p>Math libraries vary in where they run. Numeric Javascript runs on the client, rather than on the server as numbers.js does. The advantage is speed in performing small calculations; a client-side library typically runs small calculations faster because it doesn’t have to transfer data over the network. However, the disadvantage is that a client-side library doesn’t have easy access to server data when performing large, complex calculations. If your calculation involves heavy usage of server data, you might actually find a server-side library is better.</p> 
        <p>This is a hosted library, which means that you don’t download anything. In addition, the documentation for this library is unique. You click on one of the math categories and see a page of all the functions that Numeric Javascript provides. You can type values into those functions and see them change right before your eyes. Using this approach helps you create code with precisely the right values. Figure 2 shows an example of the functions on the complex numbers page.</p> 
        <p><a href="http://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-02.png" class="gallery_colorbox"><img class="aligncenter size-large wp-image-14436" alt="Four Serious Math Libraries for JavaScript 02" src="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-02-600x417.png" width="600" height="417" srcset="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-02-600x417.png 600w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-02-300x208.png 300w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-02.png 1031w" sizes="(max-width: 600px) 100vw, 600px"></a></p> 
        <p>When looking at this page, you can change the values in the <span style="text-decoration: underline;">z = numeric.t(3,4);</span> entry and see the new output immediately on screen. In fact, you can modify the equation. For example, changing it to <span style="text-decoration: underline;">z = numeric.t(3,4).x;</span> displays just the x output. More important, the page shows precisely how you should use the function in your own code.</p> 
        <p>To access the hosted site in your own application, you add the requisite script entry to your page. The full version (the one with whitespace to make it human readable) is found at:&nbsp;<a title="Numerics.js Full version" href="http://www.numericjs.com/lib/numeric-1.2.6.js" target="_blank">http://www.numericjs.com/lib/numeric-1.2.6.js</a></p> 
        <p>Or you can use the minimum version (the one lacking whitespace and comments) to make things faster. This version is found at: &nbsp;<a title="Numerics.js minimum version" href="http://www.numericjs.com/lib/numeric-1.2.6.min.js" target="_blank">http://www.numericjs.com/lib/numeric-1.2.6.min.js</a></p> 
        <p><b>Demo:</b> <a title="Numerical analysis in Javascript" href="http://www.numericjs.com/" target="_blank">http://www.numericjs.com/</a></p> 
        <p><b>Download:</b> Hosted, none required</p> 
        <p><b>Documentation:</b> <a title="Numerical analysis in Javascript" href="http://www.numericjs.com/" target="_blank">http://www.numericjs.com/</a></p> 
        <h1><a title="tangle: explorable explanations made easy" href="http://worrydream.com/Tangle/" target="_blank">Tangle</a></h1> 
        <p>The math library category doesn’t just include those that perform calculations. Part of the problem with creating precision math in JavaScript is the way in which the user interacts with the page that supports the math operation.</p> 
        <p>Precision already costs you a premium in the form of calculation time. However, adding to that cost by transferring whole page elements back and forth between the client and server makes things worse. The Tangle library helps you create reactive documents—ones that interact with the user in a contextual manner with extremely fast response times. The reactive nature of the page makes it possible for the user to interact with the math functionality in ways that would normally be possible only with a desktop application where everything is localized.</p> 
        <p>The unique aspect of Tangle is that it provides a simpler way to obtain input from the user. For example, a user might see a numeric value underlined on a page. When the mouse hovers over the value, a small balloon tip saying “drag” appears on screen. The user drags and finds that the value increases or decreases. A more interesting input method displays a graph containing the results of an equation. Moving the line on the graph automatically changes the equation content to match. Figure 3 shows a typical graph application. Notice the circle and associated balloon help where the mouse is hovered, which shows that you can drag the mouse to change the appearance of the graph (and the resulting equation).</p> 
        <p><a href="http://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-03.png" class="gallery_colorbox"><img class="aligncenter size-large wp-image-14437" alt="Four Serious Math Libraries for JavaScript 03" src="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-03-600x544.png" width="600" height="544" srcset="https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-03-600x544.png 600w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-03-300x272.png 300w, https://blog.smartbear.com/wp-content/uploads/2013/10/Four-Serious-Math-Libraries-for-JavaScript-03.png 803w" sizes="(max-width: 600px) 100vw, 600px"></a></p> 
        <p>To use Tangle, you either download the kit or create a series of links to the hosted site. Using the hosted site has the advantage of letting you get updates and fixes immediately and automatically; the downloaded version gives you speed of access and stability. Here are the links for the hosted version:</p> 
        <p><a title="Tangle.js" href="http://worrydream.com/Tangle/Tangle.js" target="_blank">http://worrydream.com/Tangle/Tangle.js</a><br> <a title="mootools.js" href="http://worrydream.com/Tangle/TangleKit/mootools.js" target="_blank">http://worrydream.com/Tangle/TangleKit/mootools.js</a><br> <a title="sprintf.js" href="http://worrydream.com/Tangle/TangleKit/sprintf.js" target="_blank">http://worrydream.com/Tangle/TangleKit/sprintf.js</a><br> <a title="BVTouchable.js" href="http://worrydream.com/Tangle/TangleKit/BVTouchable.js" target="_blank">http://worrydream.com/Tangle/TangleKit/BVTouchable.js</a><br> <a title="TangleKit.js" href="http://worrydream.com/Tangle/TangleKit/TangleKit.js" target="_blank">http://worrydream.com/Tangle/TangleKit/TangleKit.js</a></p> 
        <p>You also need to add styles to define which entries on a page can change. The Tangle CSS file helps you accomplish this task:</p> 
        <p><span style="color: #993300;">&lt;link rel=”stylesheet” href=”http://worrydream.com/Tangle/TangleKit/TangleKit.css” /&gt;</span></p> 
        <p>The HTML for this example isn’t attached to a particular element. For example, you could easily put variables in a list, should you want to do so. However, you do need to identify how particular elements interact with Tangle, as shown here.</p> 
        <p><span style="color: #993300;">&lt;p id=”DoAddition”&gt;</span><br> <span style="color: #993300;"> When adding numbers,</span><br> <span style="color: #993300;"> &lt;span</span><br> <span style="color: #993300;"> data-var=”Value1?</span><br> <span style="color: #993300;"> data-min=”-20?</span><br> <span style="color: #993300;"> data-max=”20?&gt;&lt;/span&gt; +</span><br> <span style="color: #993300;"> &lt;span</span><br> <span style="color: #993300;"> data-var=”Value2?</span><br> <span style="color: #993300;"> data-min=”-20?</span><br> <span style="color: #993300;"> data-max=”20?&gt;&lt;/span&gt; =</span><br> <span style="color: #993300;"> &lt;span data-var=”Result”&gt;&lt;/span&gt;.</span><br> <span style="color: #993300;"> &lt;/p&gt;</span></p> 
        <p>The container element, <span style="text-decoration: underline;">&lt;p&gt;</span>, must have an id attribute so that Tangle can locate it on the page. Each variable element requires a data-var attribute with a unique name to identify it as well. When you want to make a variable user changeable, you assign it the <span style="text-decoration: underline;">TKAdjustableNumber</span> class. The data-min and data-max attributes control the range of numbers that the user can select.</p> 
        <p>To make everything work, you need to create a script that defines the initial variable values, the actions that take place when a user changes a value, and the container element used to hold the variables. Here’s a simple script that completes the addition of two numbers scenario.</p> 
        <p><span style="color: #993300;">window.addEvent(‘domready’, function ()</span><br> <span style="color: #993300;"> {</span><br> <span style="color: #993300;"> var model =</span><br> <span style="color: #993300;"> {</span><br> <span style="color: #993300;"> initialize: function ()</span><br> <span style="color: #993300;"> {</span><br> <span style="color: #993300;"> this.Value1 = 2;</span><br> <span style="color: #993300;"> this.Value2 = 3;</span><br> <span style="color: #993300;"> this.Result = 5;</span><br> <span style="color: #993300;"> },</span><br> <span style="color: #993300;"> update: function ()</span><br> <span style="color: #993300;"> {</span><br> <span style="color: #993300;"> this.Result = this.Value1 + this.Value2;</span><br> <span style="color: #993300;"> }</span><br> <span style="color: #993300;"> };</span><br> <span style="color: #993300;"> new Tangle(document.getElementById(“DoAddition”), model);</span><br> <span style="color: #993300;"> });</span></p> 
        <p>The example begins by ensuring the page is actually loaded. When the page has completed loading, the code creates a model that defines the initial variable values and what happens during an update. This is a simple example; you could combine Tangle with any of the other math libraries to create incredibly complex update functions, and the updates can occur in multiple stages. The final step is to create the actual Tangle object and to assign the model you’ve defined to the container holding the variables.</p>

        <h1><a href="http://josscrowcroft.github.io/accounting.js/">accounting.js</a></h1> 
        <p>A number of math libraries accept input of a specific kind or create output to make numbers human-readable. The accounting.js library is a prime example. It helps a developer create pleasing numeric output and it also accepts formatted input. For example, if someone tries to input $100,000.00 to JavaScript, the software complains because it can’t parse such input as a number. Using accounting.js makes it possible for a user to input this sort of information and still have JavaScript accept it as a number.</p> 
        <p>To use this library, you can either download the code from the site or you can use the hosted version. Here’s the link for the hosted version: &nbsp;<a title="accounting.js" href="http://josscrowcroft.github.io/accounting.js/accounting.js" target="_blank">http://josscrowcroft.github.io/accounting.js/accounting.js</a></p> 
        <p>This library is solely used to format output and unformat input. As a consequence, it’s really lightweight. The documentation for this library is quite good and using the library is relatively simple. Here’s a script that takes a numeric value and outputs it as a monetary value using the local symbols and formatting methodology.</p> 
        <p><span style="color: #993300;">&lt;p&gt;</span><br> <span style="color: #993300;"> This number:</span><br> <span style="color: #993300;"> &lt;script&gt;</span><br> <span style="color: #993300;"> document.write(accounting.formatMoney(1234567.89));</span><br> <span style="color: #993300;"> &lt;/script&gt;</span><br> <span style="color: #993300;"> should appear as a monetary amount.</span><br> <span style="color: #993300;"> &lt;/p&gt;</span></p> 
        <p>As you can see, there are no objects to create, no weird scripts to provide. All you do is call the requisite function to format the output as needed. Simple doesn’t mean lack of flexibly, however. This library does give you full control over the appearance of the output through the use of optional parameters.</p> 
        <p><b>Demo:</b> <a title="Demo / Try it out -- Accounting.js" href="http://josscrowcroft.github.io/accounting.js/#demo" target="_blank">http://josscrowcroft.github.io/accounting.js/#demo</a></p> 
        <p><b>Download:</b> <a title="Download accounting.js" href="http://josscrowcroft.github.io/accounting.js/#download" target="_blank">http://josscrowcroft.github.io/accounting.js/#download</a></p> 
        <p><b>Documentation:</b> <a title="accounting.js #23documentation" href="http://josscrowcroft.github.io/accounting.js/#documentation" target="_blank">http://josscrowcroft.github.io/accounting.js/#documentation</a></p> 
        <h1>Bottom Line</h1> 
        <p>A single article can’t provide you with a complete list of all the math libraries available. You can probably find a library for most needs. A good place to start such a search is the <a title="JSter" href="http://jster.net/category/math-libraries" target="_blank">JSter site</a> or <a title="JS Wiki of math libraries" href="https://github.com/bebraw/jswiki/wiki/Math-libraries" target="_blank">the JS Wiki of math libraries</a> which provide individualized summaries. The two things you need to consider as highest priority are getting the right result and getting that result easily. It doesn’t matter how nice the library is if you don’t get the right result and making things hard only increases the chances of human error.</p>

         <div class="comment-content"> 
          <p>Actually, the physics of computer hardware are completely irrelevant to the precision problem of double precision (64-bit) floating point variables. The new 64-bit x86-derivative architectures actually have support for quadruple precision (128-bit) floating point data too, so they can do the most common math operations on 128-bit floating point data in one CPU clock cycle with vastly greater precision. That platform-specific functionality just isn’t exposed by the language for platform independence reasons. With regards to the physics of computer hardware, the use of math extension libraries makes no difference whatsoever. Adding a JavaScript library to your code is not going to magically enable the computer hardware that you are running this on to become immune to the boundaries of physics and get that computer to suddenly become capable of doing quantum computations or anything else of the kind. It will do nothing more than get the computer to switch from working with IEEE-standard floating point formats to working with arbitrarily long arrays of integers strung together. You should get details like this straight to avoid very badly confusing and misleading impressionable newbies. These kinds of subtle details tie newbies into knots later on. I’ve seen it happen.</p>
