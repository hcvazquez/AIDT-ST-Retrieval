   <h1>2.1 &nbsp; Static Methods</h1> 
   <p><br> The Java construct for implementing functions is known as the <em>static method</em>. </p>
   <p></p>
   <h2>Using and defining static methods.</h2> The use of static methods is easy to understand. For example, when you write 
   <tt>Math.abs(a-b)</tt> in a program, the effect is as if you were to replace that code with the 
   <em>return value</em> that is produced by Java's 
   <tt>Math.abs()</tt> method method when passed the expression 
   <tt>a-b</tt> as an 
   <em>argument</em>. 
   <!--
If you think about what the 
system has to do to create this effect, you will realize that it involves 
changing a program's <em>flow of control</em>.
--> 
   <p> </p>
   <ul> 
    <a name="2.1.1"></a> 
    <li><em>Flow-of-control.</em> <a href="Harmonic.java.html">Harmonic.java</a> comprises two static methods: <tt>harmonic()</tt> to compute harmonic numbers and and <tt>main()</tt> to interact with user.

     <blockquote> 
      <img src="images/function-call.png" width="275" align="top" alt="harmonic numbers"> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
      <img src="images/function-trace.png" width="150" align="top" alt="function-call trace"> 
     </blockquote> </li>
    <li><em>Function-call trace.</em> One simple approach to following the flow of control through function calls is to imagine that each function prints its name and argument value when it is called and its return value just before returning, with indentation added on calls and subtracted on returns. </li>
    <li><em>Static method definition.</em> The first line of a static method definition, known as the <em>signature</em>, gives a name to the method and to each <em>parameter variable</em>. It also specifies the <em>type</em> of each parameter variable and the <em>return type</em> of the method. Following the signature is the <em>body</em> of the method, enclosed in curly braces. The body consists of the kinds of statements we discussed in Chapter 1. It also can contain a <em>return statement</em>, which transfers control back to the point where the static method was called and returns the result of the computation or <em>return value</em>. The body may declare <em>local variables</em>, which are variables that are available only inside the method in which they are declared. 
     <blockquote> 
      <img src="../11cheatsheet/images/function.png" width="300" alt="anatomy of a static method"> 
     </blockquote> </li>
    <li><em>Function calls.</em> A static method call is nothing more than the method name followed by its arguments, separated by commas and enclosed in parentheses. A method call is an expression, so you can use it to build up more complicated expressions. Similarly, an argument is an expression—Java evaluates the expression and passes the resulting value to the method. So, you can write code like <tt>Math.exp(-x*x/2) / Math.sqrt(2*Math.PI)</tt> and Java knows what you mean. 
     <blockquote> 
      <img src="images/function-call-anatomy.png" width="250" alt="anatomy of a static method call"> 
     </blockquote> </li>
   </ul> 
   <p></p>
   <h2>Properties of static methods.</h2> 
   <ul> 
    <p></p>
    <li><em>Multiple arguments.</em> Like a mathematical function, a Java static method can take on more than one argument, and therefore can have more than one parameter variable. <p></p></li>
    <li><em>Multiple methods.</em> You can define as many static methods as you want in a <tt>.java</tt> file. These methods are independent and can appear in any order in the file. A static method can call any other static method in the same file or any static method in a Java library such as <tt>Math</tt>. <p></p></li>
    <li><em>Overloading.</em> Static methods with different signatures are different methods. whose signatures differ are different static methods. Using the same name for two static methods whose signatures differ is known as <em>overloading</em>. <p></p></li>
    <li><em>Multiple return statements.</em> You can put <tt>return</tt> statements in a method wherever you need them: control goes back to the calling program as soon as the first <tt>return</tt> statement is reached. <p></p></li>
    <li><em>Single return value.</em> A Java method provides only one return value to the caller, of the type declared in the method signature. <p></p></li>
    <li><em>Scope.</em> The <em>scope</em> of a variable is the part of the program that can refer to that variable by name. The general rule in Java is that the scope of the variables declared in a block of statements is limited to the statements in that block. In particular, the scope of a variable declared in a static method is limited to that method's body. Therefore, you cannot refer to a variable in one static method that is declared in another. 
     <blockquote> 
      <img src="images/function-scope.png" width="425" alt="scope"> 
     </blockquote> </li>
    <li><em>Side effects.</em> A <em>pure function</em> is a function that, given the same arguments, always returns the same value, without producing any observable <em>side effects</em>, such as consuming input, producing output, or otherwise changing the state of the system. The function <tt>harmonic()</tt> is an example of a pure function. However, in computer programming, we often define functions whose <em>only</em> purpose is to produce side effects. In Java, a static method may use the keyword <tt>void</tt> as its return type, to indicate that it has no return value. </li>
   </ul> 
   <a href="FunctionExamples.java.html">FunctionExamples.java</a> gives a number of examples.

   <h2>Implementing mathematical functions.</h2> 
   <img SRC="images/gaussian.png" width="200" align="right" hspace="25" alt="Gaussian distribution function and cumulative distribution function"> We now consider two important functions that play an important role in science, engineering, and finance. The 
   <em>Gaussian (normal) distribution function</em> is characterized by the familiar bell-shaped curve and defined by the formula: 
   <blockquote>
     $$ \phi(x) \;=\; \frac{1}{\sqrt{2 \pi}} e^{-x^2/2} $$ 
   </blockquote> and the 
   <em>cumulative Gaussian distribution function</em> \(\Phi(z)\) is defined to be the area under the curve defined by \(\phi(x)\) above the 
   <em>x</em>-axis and to the left of the vertical line 
   <em>x</em> = 
   <em>z</em>. 
   <!--
For example, &Phi;(1.96) = 0.975. This means that 97.5% of the area
under the standard Gaussian curve is to the left of <em>z</em> = 1.96.
--> 
   <ul> 
    <li><em>Closed form.</em> In the simplest situation, we have a closed-form mathematical equation defining our function in terms of functions that are implemented in the <tt>Math</tt> library. This is the case for \(\phi(x)\). </li>
    <li><em>No closed form.</em> Otherwise, we may need a more complicated algorithm to compute function values. This situation is the case for \(\Phi(z)\), for which no closed-form expression exists. For small (respectively large) <em>z</em>, the value is extremely close to 0 (respectively 1); so the code directly returns 0 (respectively 1); otherwise the following Taylor series approximation is an effective basis for evaluating the function: 
     <blockquote>
       $$ \begin{eqnarray*} \Phi(z) &amp;= &amp; \int_{-\infty}^{z} \phi(x) dx \\ &amp; = &amp; \frac{1}{2} \;+\; \phi(z) \; \left(z \;+\; \frac{z^3}{3} \;+\; \frac{z^5}{3 \cdot 5} \;+\; \frac{z^7}{3 \cdot 5 \cdot 7} \;+\; \ldots \;\; \right) \end{eqnarray*} $$ 
     </blockquote> </li>
   </ul> 
   <a href="Gaussian.java.html">Gaussian.java</a> implements both of these static methods. 
   <a name="2.1.3"></a> 
   <p></p>
   <h2>Using static methods to organize code.</h2> With the ability to define functions, we can better organize our programs by defining functions within them when appropriate. For example, 
   <a href="Coupon.java.html">Coupon.java</a> is a version of 
   <a href="../14array/CouponCollector.java.html">CouponCollector.java</a> that better separates the individual components of the computation. 
   <ul> 
    <li> Given <em>n</em>, compute a random coupon value. </li>
    <li> Given <em>n</em>, do the coupon collection experiment. </li>
    <li> Get <em>n</em> from the command line, and then compute and print the result. </li>
   </ul> 
   <em>Whenever you can clearly separate tasks in programs, you should do so.</em> 
   <p></p>
   <h2>Passing arguments and returning values.</h2> Next, we examine the specifics of Java's mechanisms for passing arguments to and returning values from functions. 
   <ul> 
    <li><em>Pass by value.</em> You can use parameter variables anywhere in the code in the body of the function in the same way you use local variables. The only difference between a parameter variable and a local variable is that Java evaluates the argument provided by the calling code and initializes the parameter variable with the resulting value. This approach is known as <em>pass by value</em>. </li>
    <li><em>Arrays as arguments.</em> When a static method takes an array as an argument, it implements a function that operates on an arbitrary number of values of the same type. </li>
    <li><em>Side effects with arrays.</em> It is often the case that the purpose of a static method that takes an array as argument is to produce a side effect (change values of array elements). </li>
    <li><em>Arrays as return values.</em> A static method can also provide an array as a return value. </li>
   </ul> 
   <a href="ArrayFunctionExamples.java.html">ArrayFunctionExamples.java</a> gives a number of examples of arrays as arguments to and return values from functions. 
   <a name="2.1.4"></a> 
   <p></p>
   <h2>Superposition of sound waves.</h2> Notes like concert A have a pure sound that is not very musical, because the sounds that you are accustomed to hearing have many other components. Most musical instruments produce 
   <em>harmonics</em> (the same note in different octaves and not as loud), or you might play chords (multiple notes at the same time). To combine multiple sounds, we use 
   <em>superposition</em>: simply add the waves together and rescale to make sure that all values stay between ?1 and 1. 
   <blockquote> 
    <img src="images/superposition.png" width="250" alt="Superposition of sound waves"> 
   </blockquote> 
   <a href="PlayThatTuneDeluxe.java.html">PlayThatTuneDeluxe.java</a> is a version of

<H4>Q + A</H4>

<p><b>Q.</b> Why do I need to use the return type <tt>void</tt>? Why not
omit the return type?
<p>A. Java requires it; we have to include it. Second-guessing a decision
made by a programming-language designer is the first step to becoming one.

<p><b>Q.</b> Can I return from a <tt>void</tt> function by using <tt>return</tt>. If
so, what return value should I use?
<p><b>A.</b> Yes. use the statement <tt>return;</tt> with no return value.

<p><b>Q.</b> What happens if I leave out the keyword <tt>static</tt>?
<p><b>A.</b> As usual, the best way to answer a question like this is to try it yourself 
and see what happens. Non-static methods are different from static methods.
You will learn about them in <a href = "../30oop">Chapter 3</a>.

<p><b>Q.</b> What happens if I write code after a <tt>return</tt> statement?
<p><b>A.</b> Once a <tt>return</tt> statement is reached, control immediately
returns to the caller, so any code after a <tt>return</tt> statement 
is useless. The Java compiler identifies this situation as an error, reporting
<tt>unreachable code</tt>.

<p><b>Q.</b> What happens if I do not include a <tt>return</tt> statement?
<p><b>A.</b> No problem, if the return type is <tt>void</tt>. In this case,
control will return to the caller after the last statement. When the return
type is not <tt>void</tt>, the compiler will report a <tt>missing return statement</tt>
error if there is <em>any</em> path through the code that does not end in a 
<tt>return</tt>.

<p><b>Q.</b> The issue with side effects and arrays passed as arguments is confusing.
Is it really all that important?
<p><b>A.</b>
Yes. Properly controlling side effects is one of a programmers most important 
tasks in large systems. Taking the time to be sure that you understand the difference 
between passing a value (when arguments are of a primitive type) and passing a 
reference (when arguments are arrays) will certainly be worthwhile. The very same 
mechanism is used for all other types of data, as you will learn in Chapter 3.

<p><b>Q.</b>
So why not just eliminate the possibility of side effects by making all arguments 
pass-by-value, including arrays? 
<p><b>A.</b>
Think of a huge array with, say, millions of elements. Does it make sense to 
copy all of those values for a static method that is just going to exchange two of 
them? For this reason, most programming languages support passing an array to a 
function without creating a copy of the array elements&mdash;Matlab is a notable
exception.
