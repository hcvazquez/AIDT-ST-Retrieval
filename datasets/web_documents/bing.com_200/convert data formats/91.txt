      <h3>Importing Dates</h3> 
      <div>
       Dates can be imported from character, numeric, POSIXlt, and POSIXct formats using the 
       <i>as.Date</i>&nbsp;function from the 
       <b>base</b>&nbsp;package.
       <p></p> 
       <p>If your data were exported from Excel, they will possibly be in numeric format. Otherwise, they will most likely be stored in character format.</p> 
       <h4>Importing Dates from Character Format</h4> 
       <div>
        If your dates are stored as characters, you simply need to provide 
        <i>as.Date</i>&nbsp;with your vector of dates and the format they are currently stored in. The possible date segment formats are listed in a table below.&nbsp;
       </div> 
       <div></div> 
       <div>
        For example,
       </div> 
       <div>
        “05/27/84” is in the format %m/%d/%y, while “May 27 1984” is in the format %B %d %Y.
       </div> 
       <div></div> 
       <div>
        To import those dates, you would simply provide your dates and their format (if not specified, it tries %Y-%m-%d and then %Y/%m/%d):
       </div> 
       <div>
        <!-- HTML generated using hilite.me --> 
        <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
         <pre style="line-height: 125%; margin: 0;"></pre> 
         <pre style="line-height: 125%; margin: 0;">dates <span style="color: #555555;">&lt;-</span> c(<span style="color: #cc3300;">"05/27/84"</span>, <span style="color: #cc3300;">"07/07/05"</span>)<br>betterDates <span style="color: #555555;">&lt;-</span> as.Date(dates,<br>  format <span style="color: #555555;">=</span> <span style="color: #cc3300;">"%m/%d/%y"</span>)</pre> 
         <pre style="line-height: 125%; margin: 0;"><span style="color: #555555;">&gt;</span> betterDates<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span><br></pre> 
        </div> 
        <p></p>
       </div> 
       <div>
        Or:
       </div> 
       <p>
        <!-- HTML generated using hilite.me --> </p>
       <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
        <pre style="line-height: 125%; margin: 0;">dates <span style="color: #555555;">&lt;-</span> c(<span style="color: #cc3300;">"May 27 1984"</span>, <span style="color: #cc3300;">"July 7 2005"</span>)<br>betterDates <span style="color: #555555;">&lt;-</span> as.Date(dates,<br>  format <span style="color: #555555;">=</span> <span style="color: #cc3300;">"%B %d %Y"</span>)<br><br><span style="color: #555555;">&gt;</span> betterDates<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span><br></pre> 
       </div> 
       <div>
        <br class="Apple-interchange-newline">This outputs the dates in the ISO 8601 international standard format %Y-%m-%d. If you would like to use dates in a different format, read “Changing Date Formats” below.
        <br> 
        <h4>Importing Dates from Numeric Format</h4> 
        <div>
         If you are importing data from Excel, you may have dates that are in a numeric format. We can still use 
         <i>as.Date</i>&nbsp;to import these, we simply need to know the origin date that Excel starts counting from, and provide that to 
         <i>as.Date</i>.
        </div> 
        <div></div> 
        <div>
         For Excel on Windows, the origin date is December 30, 1899 for dates after 1900. (Excel’s designer thought 1900 was a leap year, but it was not.) For Excel on Mac, the origin date is January 1, 1904.
        </div> 
       </div> 
       <p>
        <!-- HTML generated using hilite.me --> </p>
       <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
        <pre style="line-height: 125%; margin: 0;"><span style="color: #0099ff; font-style: italic;"># from Windows Excel:</span><br>  dates <span style="color: #555555;">&lt;-</span> c(<span style="color: #ff6600;">30829</span>, <span style="color: #ff6600;">38540</span>)<br>  betterDates <span style="color: #555555;">&lt;-</span> as.Date(dates,<br>    origin <span style="color: #555555;">=</span> <span style="color: #cc3300;">"1899-12-30"</span>)<br><br><span style="color: #555555;">&gt;</span>   betterDates<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span><br><br><span style="color: #0099ff; font-style: italic;"># from Mac Excel:</span><br>  dates <span style="color: #555555;">&lt;-</span> c(<span style="color: #ff6600;">29367</span>, <span style="color: #ff6600;">37078</span>)<br>  betterDates <span style="color: #555555;">&lt;-</span> as.Date(dates,<br>    origin <span style="color: #555555;">=</span> <span style="color: #cc3300;">"1904-01-01"</span>)<br><br><span style="color: #555555;">&gt;</span>   betterDates<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span><br></pre> 
       </div> 
       <div>
        This outputs the dates in the ISO 8601 international standard format %Y-%m-%d. If you would like to use dates in a different format, read the next step:
       </div> 
      </div> 
      <ul> 
       <ul></ul> 
      </ul> 
      <h3>Changing Date Formats</h3> 
      <div>
       If you would like to use dates in a format other than the standard %Y-%m-%d, you can do that using the 
       <i>format</i>&nbsp;function from the 
       <b>base</b>&nbsp;package.
       <p></p> 
       <p>For example,<br>
        <!-- HTML generated using hilite.me --> </p>
       <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
        <pre style="line-height: 125%; margin: 0;">format(betterDates,<br>  <span style="color: #cc3300;">"%a %b %d"</span>)<br><br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"Sun May 27"</span> <span style="color: #cc3300;">"Thu Jul 07"</span><br></pre> 
       </div> 
       <p></p>
      </div> 
      <h3>Correct Centuries</h3> 
      <div>
       If you are importing data with only two digits for the years, you will find that it assumes that years 69 to 99 are 1969-1999, while years 00 to 68 are 2000–2068 (subject to change in future versions of R).
      </div> 
      <div></div> 
      <div>
       Often, this is not what you intend to have happen. 
       <a href="http://stackoverflow.com/questions/9508747/r-adding-century-to-year" rel="nofollow" target="_blank">This page</a> gives a good explanation of several ways to fix this depending on your preference of centuries. One solution it provides is to assume all dates R is placing in the future are really from the previous century. That solution is as follows:
      </div> 
      <div>
       <!-- HTML generated using hilite.me --> 
       <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
        <pre style="line-height: 125%; margin: 0;">dates <span style="color: #555555;">&lt;-</span> c(<span style="color: #cc3300;">"05/27/84"</span>, <span style="color: #cc3300;">"07/07/05"</span>, <span style="color: #cc3300;">"08/17/20"</span>)<br>betterDates <span style="color: #555555;">&lt;-</span> as.Date(dates, <span style="color: #cc3300;">"%m/%d/%y"</span>)<br><br><span style="color: #555555;">&gt;</span> betterDates<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span> <span style="color: #cc3300;">"2020-08-17"</span><br><br>correctCentury <span style="color: #555555;">&lt;-</span> as.Date(ifelse(betterDates <span style="color: #555555;">&gt;</span> Sys.Date(), <br>  format(betterDates, <span style="color: #cc3300;">"19%y-%m-%d"</span>), <br>  format(betterDates)))<br><br><span style="color: #555555;">&gt;</span> correctCentury<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span> <span style="color: #cc3300;">"2005-07-07"</span> <span style="color: #cc3300;">"1920-08-17"</span><br></pre> 
       </div> 
       <p></p>
      </div> 
      <div> 
       <h3>Purpose of Proper Formatting</h3> 
       <p>Having your dates in the proper format allows R to know that they are dates, and as such knows what calculations it should and should not perform on them. For one example, see my <a href="http://www.mollietaylor.com/2013/08/plot-weekly-or-monthly-totals-in-r.html" rel="nofollow" target="_blank">post on plotting weekly or monthly totals</a>. Here are a few more examples:<br>
        <!-- HTML generated using hilite.me --> </p>
       <div style="background: #f0f3f3; border-width: .1em .1em .1em .8em; border: solid gray; overflow: auto; padding: .2em .6em; width: auto;"> 
        <pre style="line-height: 125%; margin: 0;"><span style="color: #555555;">&gt;</span>   mean(betterDates)<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1994-12-16"</span><br><br><span style="color: #555555;">&gt;</span>   max(betterDates)<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"2005-07-07"</span><br><br><span style="color: #555555;">&gt;</span>   min(betterDates)<br>[<span style="color: #ff6600;">1</span>] <span style="color: #cc3300;">"1984-05-27"</span><br></pre> 
       </div> 
       <p></p>
      </div> 
      <p>The code is available in a <a href="https://gist.github.com/mollietaylor/6258459" rel="nofollow" target="_blank">gist</a>.<br> </p>

function createCookie(name,value,days) {
    var expires = "";
    if (days) {
        var date = new Date();
        date.setTime(date.getTime() + (days*24*60*60*1000));
        expires = "; expires=" + date.toUTCString();
    }
    document.cookie = name + "=" + value + expires + "; path=/";
}

function readCookie(name) {
    var nameEQ = name + "=";
    var ca = document.cookie.split(';');
    for(var i=0;i < ca.length;i++) {
        var c = ca[i];
        while (c.charAt(0)==' ') c = c.substring(1,c.length);
        if (c.indexOf(nameEQ) == 0) return c.substring(nameEQ.length,c.length);
    }
    return null;
}

function eraseCookie(name) {
    createCookie(name,"",-1);
}


function readTextFile(file)
{
	// Helps people browse between pages without the need to keep downloading the same 
	// ads txt page everytime. This way, it allows them to use their browser's cache.

		var random_number = Math.floor(Math.random()*100*(new Date().getTime()/1000));
		createCookie("ad_random_number_cookie",random_number,1)
	}
	
    file += '?t='+random_number;
    var rawFile = new XMLHttpRequest();
    rawFile.onreadystatechange = function ()
    {
        if(rawFile.readyState === 4)
        {
            if(rawFile.status === 200 || rawFile.status == 0)
            {
                // var allText = rawFile.responseText;
                // document.write(allText);
                document.write(rawFile.responseText);
            }
        }
    }
    rawFile.open("GET", file, false);
    rawFile.send(null);
}
