           </svg></a>Philosophy</h2> 
         <p>Shoestring is a lightweight, modular DOM framework intended for developers who need their code to download and run as fast as possible. It is intended to be "just enough" of a tool to efficently write maintainable cross-browser JavaScript. The API is modeled after jQuery but we intentionally implement a tiny subset of the API to cover only the methods we commonly need in our projects. Each feature is built as an optional extension to a minimal core so each can be removed from the production build (dependencies between extensions are rare). The selector engine delegates to modern browsers' native <code>document.querySelectorAll</code> (IE8) and <code>addEventListener</code> (IE9), which means it requires browsers that support those features. For projects that require deeper compatibility or a richer set of features, it is simple to swap in jQuery instead. As a rule, anything that works with Shoestring should work with jQuery, but not the converse.</p> 
         <p>Our goal is to strike a good balance of code weight, runtime speed, browser support, and developer convenience.</p> 
         <h2><a id="user-content-features" class="anchor" href="#features" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Features</h2> 
         <p>Shoestring's API is inspired by jQuery.</p> 
         <p>Technically, shoestring.js is a very small, extendable core function. That core function doesn't come with much more than a means of finding and/or generating HTML elements, a DOM-ready handler, and a few essential element-traversal methods like <code>each</code>, <code>find</code>, <code>children</code>. Using its <code>shoestring.fn</code> API, its core is easy to extend further, and many extensions are available for you to include in your build – the default build includes all extensions, but it's still very small (~3kb compressed).</p> 
         <p>If you are concerned about compatibility issues/pitfalls consider using the development build releases (<code>-dev</code> in the <a href="https://github.com/filamentgroup/shoestring/releases">releases</a>). We've endeavored to throw exceptions where a particular invocation pattern or feature isn't supported as a means to document the disparity. We recommend that you use the development version in development and the regular non-<code>-dev</code> version in production.</p> 
         <p>There are three sets of extensions to the Shoestring core: DOM manipulation, events, and ajax.</p> 
         <h3><a id="user-content-dom" class="anchor" href="#dom" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>DOM</h3> 
         <p>If you've used jQuery, the structure and behavior of the DOM manipulation methods will be immediately familiar:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">shoestring</span>( <span class="pl-s"><span class="pl-pds">"</span>.foo<span class="pl-pds">"</span></span> ).<span class="pl-en">addClass</span>( <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span> ).<span class="pl-en">attr</span>( <span class="pl-s"><span class="pl-pds">"</span>data-baz<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>bak<span class="pl-pds">"</span></span> );</pre>
         </div> 
         <p>That is, construct a sequence of elements from the DOM and invoke each method on all the elements of the sequence in turn. You can find a full list of the supported DOM methods and their arguments in the <a href="http://filamentgroup.github.io/shoestring/dist/docs/">API docs</a> under the <code>dom/*</code> subdirectory.</p> 
         <h3><a id="user-content-events" class="anchor" href="#events" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Events</h3> 
         <p>Shoestring supports most of the jQuery events API including: custom events that bubble, event namespaces, a normalized event object, event arguments, DOM Ready, a <code>.one</code> method, comprehensive <code>.unbind</code>, etc. The major difference here is that Shoestring does not support the event delegation helpers built into jQuery, like <code>.delegate</code> or <code>.on</code> with a selector argument.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">shoestring</span>( <span class="pl-s"><span class="pl-pds">"</span>.foo<span class="pl-pds">"</span></span> ).<span class="pl-en">bind</span>( <span class="pl-s"><span class="pl-pds">"</span>click<span class="pl-pds">"</span></span>, <span class="pl-k">function</span>( <span class="pl-c1">event</span> ) { <span class="pl-k">...</span> });</pre>
         </div> 
         <p>Or with a custom event triggered on a child element:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">shoestring</span>( <span class="pl-s"><span class="pl-pds">"</span>.foo<span class="pl-pds">"</span></span> ).<span class="pl-en">bind</span>( <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>, <span class="pl-k">function</span>( <span class="pl-c1">event</span>, <span class="pl-smi">arg</span> ) {
  <span class="pl-k">...</span>
  <span class="pl-k">if</span>(arg <span class="pl-k">==</span> <span class="pl-c1">1</span>) { <span class="pl-k">...</span> }
  <span class="pl-k">...</span>
});

<span class="pl-c"><span class="pl-c">//</span> ...</span>

<span class="pl-en">shoestring</span>( <span class="pl-s"><span class="pl-pds">"</span>.foo<span class="pl-pds">"</span></span> ).<span class="pl-en">children</span>().<span class="pl-en">first</span>().<span class="pl-en">trigger</span>( <span class="pl-s"><span class="pl-pds">"</span>bar<span class="pl-pds">"</span></span>, <span class="pl-c1">1</span> );</pre>
         </div> 
         <p>You can find a full list of the supported event methods and their arguments in the <a href="http://filamentgroup.github.io/shoestring/dist/docs/">API docs</a> under the <code>events/*</code> subdirectory.</p> 
         <h3><a id="user-content-ajax" class="anchor" href="#ajax" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Ajax</h3> 
         <p>Shoestring supports a full <code>shoestring.ajax</code> method as well as some shorthand helpers like <code>shoestring.get</code> and <code>shoestring.post</code>.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">shoestring</span>.<span class="pl-en">ajax</span>( <span class="pl-s"><span class="pl-pds">"</span>/foo<span class="pl-pds">"</span></span>, {
  <span class="pl-en">success</span><span class="pl-k">:</span> <span class="pl-k">function</span>(){ <span class="pl-k">...</span> },
  method<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">"</span>GET<span class="pl-pds">"</span></span>,
  <span class="pl-k">...</span>
});</pre>
         </div> 
         <p>Which could also be accomplished using <code>shoestring.get</code></p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">shoestring</span>.<span class="pl-c1">get</span>( <span class="pl-s"><span class="pl-pds">"</span>/foo<span class="pl-pds">"</span></span>, <span class="pl-k">function</span>(){ <span class="pl-k">...</span> });</pre>
         </div> 
         <p>You can find a full list of the supported ajax methods and their arguments in the <a href="http://filamentgroup.github.io/shoestring/dist/docs/">API docs</a> under the <code>ajax/*</code> subdirectory.</p> 
         <h2><a id="user-content-extensions" class="anchor" href="#extensions" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Extensions</h2> 
         <p>Extending Shoestring is done in nearly the same fashion as jQuery. There is an object on which you can define properties using functions and those functions will have access to the Shoestring DOM element sequence during invocation using <code>this</code>. As an example the <code>remove</code> method:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c"><span class="pl-c">/**</span></span>
<span class="pl-c"> * Remove the current set of elements from the DOM.</span>
<span class="pl-c"> *</span>
<span class="pl-c"> * <span class="pl-k">@return</span> shoestring</span>
<span class="pl-c"> * <span class="pl-k">@this</span> <span class="pl-en">shoestring</span></span>
<span class="pl-c"> <span class="pl-c">*/</span></span>
<span class="pl-smi">shoestring</span>.<span class="pl-smi">fn</span>.<span class="pl-en">remove</span> <span class="pl-k">=</span> <span class="pl-k">function</span>(){
  <span class="pl-k">return</span> <span class="pl-c1">this</span>.<span class="pl-en">each</span>(<span class="pl-k">function</span>(){
    <span class="pl-k">if</span>( <span class="pl-c1">this</span>.<span class="pl-c1">parentNode</span> ) {
      <span class="pl-c1">this</span>.<span class="pl-c1">parentNode</span>.<span class="pl-c1">removeChild</span>( <span class="pl-c1">this</span> );
    }
  });
};</pre>
         </div> 
         <p>It uses the <code>each</code> method to handle the DOM elements in the current sequence in turn.</p> 
         <p><strong>NOTE</strong> these definitions must be made before a Shoestring object that depends on them is constructed. This is in contrast with jQuery where each object has access to new methods through the prototype chain.</p> 
         <h3><a id="user-content-modules" class="anchor" href="#modules" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Modules</h3> 
         <p>Each extension to Shoestring included in the repository is defined as an AMD module, but only for build purposes. We don't support, or plan to support, loading the library as modules in the browser.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c"><span class="pl-c">//</span>&gt;&gt;excludeStart("exclude", pragmas.exclude);</span>
<span class="pl-en">define</span>([ <span class="pl-s"><span class="pl-pds">"</span>shoestring<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>dom/remove<span class="pl-pds">"</span></span> ], <span class="pl-k">function</span>(){
<span class="pl-c"><span class="pl-c">//</span>&gt;&gt;excludeEnd("exclude");</span>

<span class="pl-smi">shoestring</span>.<span class="pl-smi">fn</span>.<span class="pl-en">foo</span> <span class="pl-k">=</span> <span class="pl-k">function</span>(){ <span class="pl-k">...</span> };

<span class="pl-c"><span class="pl-c">//</span>&gt;&gt;excludeStart("exclude", pragmas.exclude);</span>
});
<span class="pl-c"><span class="pl-c">//</span>&gt;&gt;excludeEnd("exclude");</span></pre>
         </div> 
         <p>Note that the AMD wrapper is removed during the process of the build and that the dependencies are defined from the <code>src</code> subdirectory. More on custom builds below.</p> 
         <h3><a id="user-content-dependencies" class="anchor" href="#dependencies" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Dependencies</h3> 
         <p>Browsing the modules in Shoestring you'll notice that very few have explicit dependencies in their module definitions. This is by design. We are interested in being able to select the minimum number of methods necessary for a given project to reduce load and parse times.</p> 
         <h2><a id="user-content-builds" class="anchor" href="#builds" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Builds</h2> 
         <p>Shoestring releases include two different builds, one for development and one for production. The development build is larger. It is intended to help with jQuery compatibility issues and includes other development utilities like the method tracker. The production build is meant to be shipped in production and does not include the extra dev-time helpers.</p> 
         <h3><a id="user-content-custom" class="anchor" href="#custom" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Custom</h3> 
         <p>This repository supports custom builds through creating a meta-module in the <code>build/custom/</code> directory and running the default Grunt task. To get started building a custom production build, do the following:</p> 
         <ol> 
          <li>make sure the project dependencies are installed with <code>npm install</code></li> 
          <li>copy <code>build/development.js</code> to <code>build/custom/foo.js</code></li> 
          <li>run <code>grunt</code> or <code>node node_modules/.bin/grunt</code></li> 
          <li>use <code>dist/foo.js</code></li> 
         </ol> 
         <h3><a id="user-content-tracker" class="anchor" href="#tracker" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Tracker</h3> 
         <p>Included in the development build is a method tracker. It works by proxying all calls to <code>shoestring.fn</code> methods through a corresponding method that records the invocation in local storage. <strong>NOTE</strong> this does not include methods defined on <code>shoestring</code>. Then the methods being used across pages by your application can be inspected.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">JSON</span>.<span class="pl-c1">parse</span>( <span class="pl-c1">window</span>.<span class="pl-smi">localStorage</span>.<span class="pl-c1">getItem</span>(<span class="pl-smi">shoestring</span>.<span class="pl-smi">trackedMethodsKey</span>) );</pre>
         </div> 
         <p>If the method tracker is included during a significant portion of development this list can be used to remove unused functions from your Shoestring build with a custom meta-module.</p>

           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
