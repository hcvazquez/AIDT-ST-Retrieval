                   <td class="sites-layout-tile sites-tile-name-content-1">
                    <div dir="ltr">
                     <div></div> This page shows an example on text mining of Twitter data with R packages 
                     <i>twitteR</i>, 
                     <i>tm</i> and 
                     <i>wordcloud</i>. Package 
                     <i>twitteR</i> provides access to Twitter data, 
                     <i>tm</i> provides functions for text mining, and 
                     <i>wordcloud</i> visualizes the result with a word cloud.
                     <br>
                     <br>If you have no access to Twitter, the tweets data can be downloaded as file "rdmTweets.RData" at 
                     <a href="http://www.rdatamining.com/data">the Data page</a>, and then you can skip the first step below.
                     <br> 
                     <h3><a name="TOC-Retrieving-Text-from-Twitter"></a>Retrieving Text from Twitter</h3> Twitter API requires authentication since March 2013. Please follow instructions in "Section 3 - Authentication with OAuth" in 
                     <a href="http://cran.r-project.org/web/packages/twitteR/vignettes/twitteR.pdf" rel="nofollow" target="_blank">the twitteR vignettes on CRAN</a> or 
                     <a href="http://geoffjentry.hexdump.org/twitteR.pdf" target="_blank">this link</a> to complete authentication before running the code below.
                     <br>
                     <br>
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; library(twitteR)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # retrieve the first 100 tweets (or all tweets if fewer than 100)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # from the user timeline of @rdatammining</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; rdmTweets &lt;- userTimeline("rdatamining", n=100)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; n &lt;- length(rdmTweets)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; rdmTweets[1:3]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[1]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>Text Mining Tutorial http://t.co/jPHHLEGm</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[2]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>R cookbook with examples http://t.co/aVtIaSEg</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[3]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>Access large amounts of Twitter data for data mining and other tasks within </code><br><code>R via the twitteR package. http://t.co/ApbAbnxs</code></span>
                     </div>
                     <br> 
                     <h3><a name="TOC-Transforming-Text"></a>Transforming Text</h3> The tweets are first converted to a data frame and then to a corpus.
                     <br>
                     <br> 
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; df &lt;- do.call("rbind", lapply(rdmTweets, as.data.frame))</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; dim(df)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[1] 79 10</code></span>
                      <br> 
                      <br> 
                      <span style="font-family:courier new,monospace"><code>&gt; library(tm)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # build a corpus, which is a collection of text documents</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # VectorSource specifies that the source is character vectors.</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- Corpus(VectorSource(df$text))</code></span>
                     </div> 
                     <br> After that, the corpus needs a couple of transformations, including changing letters to lower case, removing punctuations/numbers and removing stop words. The general English stop-word list is tailored by adding "available" and "via" and removing "r".
                     <br>
                     <br> 
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, tolower)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # remove punctuation</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, removePunctuation)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # remove numbers</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, removeNumbers)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # remove stopwords</code></span>
                      <div>
                       <font face="'courier new', monospace"><code>&gt; # keep "r" by removing it from stopwords</code></font>
                       <br style="font-family:courier new,monospace"> 
                       <span style="font-family:courier new,monospace"><code>&gt; myStopwords &lt;- c(stopwords('english'), "available", "via")</code></span>
                       <br style="font-family:courier new,monospace"> 
                       <span style="font-family:courier new,monospace"><code>&gt; idx &lt;- which(myStopwords == "r")</code></span>
                       <br style="font-family:courier new,monospace"> 
                       <span style="font-family:courier new,monospace"><code>&gt; myStopwords &lt;- myStopwords[-idx]</code></span>
                       <br style="font-family:courier new,monospace"> 
                       <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, removeWords, myStopwords)</code></span>
                      </div>
                     </div>
                     <br> 
                     <h3><a name="TOC-Stemming-Words"></a>Stemming Words</h3> In many cases, words need to be stemmed to retrieve their radicals. For instance, "example" and "examples" are both stemmed to "exampl". However, after that, one may want to complete the stems to their original forms, so that the words would look "normal".
                     <br> 
                     <br> 
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; dictCorpus &lt;- myCorpus</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # stem words in a text document with the snowball stemmers,</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # which requires packages Snowball, RWeka, rJava, RWekajars</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, stemDocument)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # inspect the first three ``documents"</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; inspect(myCorpus[1:3])</code></span>
                      <br style="font-family:courier new,monospace">
                      <code> (Some details are removed to make it short. Same applies to inspect() below.)</code>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[1]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>text mine tutori</code></span> 
                      <span style="font-family:courier new,monospace"><code>httptcojphhlegm</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[2]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>r cookbook exampl httptcoavtiaseg</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[3]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>access amount twitter data data mine task r twitter packag httptcoapbabnx</code></span>
                      <br> 
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # stem completion</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myCorpus &lt;- tm_map(myCorpus, stemCompletion, dictionary=dictCorpus)</code></span>
                     </div> 
                     <br> Print the first three documents in the built corpus.
                     <br>
                     <br> 
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; inspect(myCorpus[1:3])</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[1]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>text miners tutorial httptcojphhlegm</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[2]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>r cookbook examples httptcoavtiaseg</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>[[3]]</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>access amounts twitter data data miners task r twitter package httptcoapbabnxs</code></span>
                     </div> 
                     <br> Something unexpected in the above stemming and stem completion is that, word "mining" is first stemmed to "mine", and then is completed to "miners", instead of "mining", although there are many instances of "mining" in the tweets, compared to only one instance of "miners".
                     <br>

                     <h3><a name="TOC-Word-Cloud"></a>Word Cloud</h3> After building a document-term matrix, we can show the importance of words with a word cloud (also kown as a tag cloud) . In the code below, word "miners" are changed back to "mining".
                     <br>
                     <br> 
                     <div class="sites-codeblock sites-codesnippet-block">
                      <span style="font-family:courier new,monospace"><code>&gt; library(wordcloud)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; m &lt;- as.matrix(myDtm)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; # calculate the frequency of words</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; v &lt;- sort(rowSums(m), decreasing=TRUE)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myNames &lt;- names(v)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; k &lt;- which(names(v)=="miners")</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; myNames[k] &lt;- "mining"</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; d &lt;- data.frame(word=myNames, freq=v)</code></span>
                      <br style="font-family:courier new,monospace"> 
                      <span style="font-family:courier new,monospace"><code>&gt; wordcloud(d$word, d$freq, min.freq=3)</code></span>
                     </div> 
                     <br> 
                     <br> 
                     <div style="display:block;text-align:left"> 
                      <div style="display:block;text-align:left">
                       <a href="http://www.rdatamining.com/examples/text-mining/wordcloud.png?attredirects=0" imageanchor="1"><img border="0" src="http://www.rdatamining.com/_/rsrc/1421498623901/examples/text-mining/wordcloud.png" title="Word cloud of @RDataMining tweets"></a>
                      </div> 
                      <br> The above word cloud clearly shows that "r", "data" and "mining" are the three most important words, which validates that the @RDataMining tweets present information on R and data mining. The other important words are "analysis", "examples", "slides", "tutorial" and "package", which shows that it focuses on documents and examples on analysis and R packages.
