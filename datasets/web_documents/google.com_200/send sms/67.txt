        <p>Twilio is powering the future of business communications, enabling developers to embed voice, VoIP, and messaging into applications. They virtualize all infrastructure needed in a cloud-based, global environment, exposing it through the Twilio communications API platform. Applications are simple to build and scalable. Enjoy flexibility with pay-as-you go pricing, and benefit from cloud reliability.</p> 
        <p><strong>Twilio Voice</strong> enables your application to make and receive phone calls. <strong>Twilio SMS</strong> enables your application to send and receive text messages. <strong>Twilio Client </strong>allows you to make VoIP calls from any phone, tablet, or browser and supports WebRTC.</p> 
        <h2 id="pricing">Pricing</h2> 
        <p>Google App Engine customers receive <a href="http://ahoy.twilio.com/googlecloudplatform">complimentary credit</a> for SMS messages or inbound minutes when you upgrade.</p> 
        <p>Twilio is a pay-as-you-go service. There are no set-up fees and you can close your account at any time. You can find more details at <a href="http://www.twilio.com/pricing">Twilio Pricing</a>.</p> 
        <h2 id="platform">Platform</h2> 
        <p>The Twilio platform consists of the Twilio Markup Language (TwiML), a RESTful API and VoIP SDKs for web browsers, Android and iOS. Helper libraries are available in multiple languages. You can find the full list at <a href="http://www.twilio.com/docs/libraries">Twilio Helper Libraries</a>.</p> 
        <h3 id="twiml">TwiML</h3> 
        <p>TwiML is a set of instructions that you can use to tell Twilio what to do when you receive an incoming call or SMS. When someone makes a call or sends an SMS to one of your Twilio numbers, Twilio will look up the URL associated with that phone number and make a request to that URL. Twilio will read TwiML instructions at that URL to determine what to do:</p> 
        <ul> 
         <li><code>&lt;Say&gt;</code> - text to speech</li> 
         <li><code>&lt;Record&gt;</code> - record the call</li> 
         <li><code>&lt;Play&gt;</code> - play a message for the caller</li> 
         <li><code>&lt;Gather&gt;</code> - prompt the caller to press digits on their keypad</li> 
         <li><code>&lt;Message&gt;</code> - send a message</li> 
        </ul> 
        <p>Learn about the other verbs and capabilities from the <a href="http://www.twilio.com/docs/api/twiml">Twilio Markup Language documentation</a>.</p> 
        <h3 id="rest_api">REST API</h3> 
        <p>The Twilio REST API allows you to query metadata about your account, phone numbers, calls, text messages, and recordings. You can also do some fancy things like initiate outbound calls and send text messages.</p> 
        <p>Since the API is based on REST principles, it's very easy to write and test applications. You can use your browser to access URLs, and you can use pretty much any HTTP client in any programming language to interact with the API. For more information, see the <a href="http://www.twilio.com/docs/api/rest">Twilio REST API</a>.</p>

        <h2 id="setup">Setup</h2>   
        <p>To use Twilio to receive an incoming call, send an SMS, or receive an SMS:</p>

         <li> <p>In Twilio, <a href="https://www.twilio.com/user/account/phone-numbers/getting-started">create a number</a>. Configure the URLs for your Twilio number as needed for your app. For example, for the <a href="#copying_the_sample_code">sample code</a>:</p> 
          <ul> 
           <li>Configure the voice request URL to be <code>https://[your-app-id].appspot.com/call/receive</code></li> 
           <li>Configure the SMS request URL to be <code>https://[your-app-id].appspot.com/sms/receive</code></li> 
          </ul> </li> 
         <li> <p>Configure your Twilio settings in the environment variables section in <code>app.yaml</code>. For example:

  const to = req.query.to;
  if (!to) {
    res.status(400).send('Please provide an number in the "to" query string parameter.');
    return;
  }

  twilio.sendMessage({
    to: to,
    from: TWILIO_NUMBER,
    body: 'Hello from Google App Engine'
  }, (err) =&gt; {
    if (err) {
      next(err);
      return;
    }
    res.status(200).send('Message sent.');
  });
});</pre>
