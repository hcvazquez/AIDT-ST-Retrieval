      <h2>js pivot <small>open-source jQuery pivottable.js plugin</small></h2> 
      <ul> 
       <li>Lightweight, free and open source javascript pivot table: <a href="https://github.com/nicolaskruchten/pivottable" target="_blank">Pivottable.js</a> (pivot.js) </li> 
       <li>user-friendly drag&amp;drop UI for creating pivot table (crosstab, summary table)</li> 
       <li>handles up to 100,000 rows, all calculations are performed by js code</li> 
       <li>supports typical aggregate functions: count, count unique values, list unique values, sum, average, min, max, % of totals; custom function can be easily added</li> 
       <li>additional renderers: heatmaps, pivot charts (Google Charts / <a href="http://c3js.org/" target="_blank">C3.js</a>)</li> 
      </ul> 
      <h2>nreco extensions <small>advanced features for pivottable.js</small></h2> 
      <ul> 
       <li>fixed headers (freeze headers for pivot table rows &amp; columns)<sup class="text-warning">new</sup></li> 
       <li>sort pivot table by values, totals or labels</li> 
       <li>drill down handler (click on cell and see aggregated data details)</li> 
       <li>access to calculated pivot table data (for pivot table exports)</li> 
       <li>Bootstrap 3 mobile-friendly styles</li> 
       <li> <a target="_blank" class="webpivotdemo" href="http://webpivot.nrecosite.com/">ASP.NET WebPivot Report Builder </a> package includes: 
        <ul> 
         <li>PivotTable.js NReco Extensions (commercial version)</li> 
         <li>Data query builder js components (jQuery plugins)</li> 
         <li>NReco WebPivot Editor (integrates pivottable.js + NReco extensions with data query builder)</li> 
         <li>Server-side API for loading data by used-defined query from SQL database (based on <a href="https://github.com/nreco/data" target="_blank">NReco.Data</a> library)</li> 
         <li>Export pivot table components (CSV, Excel, PDF)</li> 
         <li>Integrated ASP.NET example (MVC app): 
          <ul> 
           <li>use SQL database as data source for pivot table</li> 
           <li>server-side data filter: reduce amount of data for javascript pivot table</li> 
           <li>save / load pivot table report state (user-defined reports)</li> 
           <li>export pivot table to CSV/Excel/PDF formats</li> 
          </ul> </li> 
         <li>1 year of email support and free package code updates</li> 
        </ul> </li> 
      </ul>

      <h2>how to use</h2> 
      <ol> 
       <li>Include jQuery, pivot.js and nrecopivottableext.js files</li> 
       <li>Initialize PivotTable jQuery plugin and apply NReco Extensions with the following js code: <pre>var nrecoPivotExt = new NRecoPivotTableExtensions({
	drillDownHandler: function (dataFilter) { 
		console.log(dataFilter); 
	} });
var sampleData = [{"A":1,"B":1,V:1}, ... ];
var pvtOpts = {
	renderer: nrecoPivotExt.wrapTableRenderer($.pivotUtilities.renderers["Table"]),
	rendererOptions: { sort: { direction : "desc", column_key : [ "1" ]} },
	vals: ["V"], rows: ["A"], cols: ["B"],
	aggregator : $.pivotUtilities.aggregators["Sum"](["V"]),
}	
$('#pivot').pivot(sampleData, pvtOpts);
</pre> </li> 
      </ol> Have a question?

    <h2>plugin options</h2> 
    <table class="table table-bordered table-condensed api-doc"> 
     <tbody>
      <tr> 
       <th>Option</th> 
       <th width="35%">Description</th> 
       <th width="45%">Example</th> 
      </tr> 
      <tr> 
       <td><code>fixedHeaders</code></td> 
       <td>Default: true. If enabled and pivot table has an overflow (container element limited in width and/or height) fixed headers are used.</td> 
       <td><pre>{ fixedHeaders: true }</pre></td> 
      </tr> 
      <tr> 
       <td><code>sortByLabelEnabled</code></td> 
       <td>Default: true. If enabled user can click on dimension name to order table by dimension labels (keys).</td> 
       <td><pre>{ sortByLabelEnabled: true }</pre></td> 
      </tr> 
      <tr> 
       <td><code>sortByColumnsEnabled</code></td> 
       <td>Default: true. If enabled user can click on column label to order table by column values.</td> 
       <td><pre>{ sortByColumnsEnabled: true }</pre></td> 
      </tr> 
      <tr> 
       <td><code>sortByRowsEnabled</code></td> 
       <td>Default: true. If enabled user can click on row label to order table by row values.</td> 
       <td><pre>{ sortByRowsEnabled: true }</pre></td> 
      </tr> 
      <tr> 
       <td><code>wrapWith</code></td> 
       <td>Default: null. Wraps &lt;table&gt; with specified element. Useful when fixedHeaders=true to limit pivot table by height and/or width.</td> 
       <td><pre>{ wrapWith: '&lt;div class="pvtTableRendererHolder"&gt;&gt;/div&gt;', }</pre></td> 
      </tr> 
      <tr> 
       <td><code>drillDownHandler</code></td> 
       <td>Default: null (function). This function is called when user clicks on table value cell. Argument is a (dimension name)?(dimension key) map for the clicked row x column intersection.</td> 
       <td><pre>{
  drillDownHandler: function (attrFilter) {
    console.log(attrFilter); 
  }
}</pre></td> 
      </tr> 
      <tr> 
       <td><code>onSortHandler</code></td> 
       <td>Default: null (function). This function is called when user changes order of the pivot table. Argument is a structure that describes sort state (rendererOptions.sort entry).</td> 
       <td><pre>{
  onSortHandler:function (sortOpts) {console.log(sortOpts);}
}</pre></td> 
      </tr> 
     </tbody>
    </table>

      <h2 title="FAQ">frequently asked questions</h2> 
      <div class="accordion" id="faq"> 
       <div class="accordion-group"> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq1"> 1. Server-side pivot table rendering </a> 
        </div> 
        <div id="faq1" class="accordion-body collapse in"> 
         <div class="accordion-inner">
           In some cases pivot table should be generated on server side (for example, report generation on schedule). It is possible to render PivotTable.js on the server side with one of the following tools: 
          <ul> 
           <li>PhantomJS (headless WebKit browser): <a href="http://www.nrecosite.com/phantomjs_wrapper_net.aspx">PhantomJS .NET Wrapper</a></li> 
           <li>WkHtmlToPdf/WkHtmlToImage (also based on WebKit browser engine): <a href="http://www.nrecosite.com/pdf_generator_net.aspx">PDF Generator for .NET</a> / <a href="http://www.nrecosite.com/html_to_image_generator_net.aspx">Image Generator for .NET</a></li> 
          </ul> As alternative, consider using another .NET component for calculating pivot data and rendering pivot table on the server: 
          <a href="pivot_data_library_net.aspx">PivotData Toolkit for .NET</a>. 
         </div> 
        </div> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq2"> 2. I have more than 100,000 rows. Can I build a pivot table? </a> 
        </div> 
        <div id="faq2" class="accordion-body collapse"> 
         <div class="accordion-inner"> 
          <div>
            Pivottable.js plugin is not suitable in this case: it performs all calculations in a web browser (dataset is aggregated by javascript code). Depending on concrete environment (web browser, available memory) it can load and handle a bit more than 100k rows but in most cases this leads to weird user experience (slowness, freezes, browser crush).
          </div> 
          <br> For large datasets you may use components that perform pivot table calculations on the server-side: 
          <ul> 
           <li><a href="pivot_data_library_net.aspx">PivotData Toolkit</a>: .NET library for ASP.NET projects</li> 
           <li> <a href="pivotdata_service.aspx">PivotData REST Service</a>: .NET Core backend, can be hosted on Linux or Windows and integrated with any web application (PHP, Ruby, Python, NodeJS, Java). </li> 
          </ul> PivotData packages include embeddable 
          <a href="http://pivottable.nrecosite.com/">web pivot tables &amp; charts builder</a> (js frontend) that gives similar experience to pivottable.js and offers more features (several measures, group subtotals, TOP N limits, exports etc). 
         </div> 
        </div>
