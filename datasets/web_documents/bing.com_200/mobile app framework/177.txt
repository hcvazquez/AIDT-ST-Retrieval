             <h2>Introduction</h2> 
             <p>Suppose you have been asked to migrate an existing multi-tier application to .NET where the business layer is written in Java. Normally you would have no option but to recode and port the entire application to any .NET language (e.g. C#). &nbsp;However this is where IKVM.NET comes to the rescue. </p> 
             <p>IKVM.NET is an open source implementation of Java for <a href="http://www.go-mono.org/">Mono</a> /<a href="http://msdn.microsoft.com/netframework/">Microsoft .NET Framework</a> and makes it possible both to develop .NET applications in Java, and to use existing Java API's and libraries in applications written in any .NET language. It is written in C# and the executables, documentation and source code can be downloaded from <a href="http://sourceforge.net/project/showfiles.php?group_id=69637">here</a>.</p> 
             <p>IKVM.NET consists of the following&nbsp;three main parts:</p> 
             <ol> 
              <li>A Java Virtual Machine implemented in .NET </li> 
              <li>A .NET implementation of the Java class libraries </li> 
              <li>Tools that enable Java and .NET interoperability </li> 
             </ol> 
             <p>However before we get any further into this topic, let’s discuss about few of the main components of the IKVM.NET package which we would be using later in this article.</p> 
             <ul> 
              <li><em>IKVM.Runtime.dll</em>: The VM runtime and all supporting code containing the byte code JIT compiler/verifier, object model remapping infrastructure and the managed .NET re-implementations of the native methods in Classpath.&nbsp; </li> 
              <li><em>IKVM.GNU.Classpath.dll</em>: Compiled version of GNU Classpath, the Free Software Foundation's implementation of the Java class libraries, plus some additional IKVM.NET specific code.&nbsp; </li> 
              <li><em>ikvm.exe</em>: Starter executable, comparable to <em>java.exe</em> ("dynamic mode"). </li> 
              <li><em>ikvmc.exe</em>: Static compiler. Used to compile Java classes and jars into a .NET assembly ("static mode"). </li> 
             </ul> 
             <p>Now back to our problem of migrating the existing Java business classes so that they can be accessed by the newly proposed .NET application. We would also like to use the various existing Java API and libraries in our .NET application. &nbsp;Let’s start by doing just that.</p> 
             <h2>Setting Up IKVM.NET</h2> 
             <p>Download the binary distribution from the sourceforge site and unzip the contents to <em>C:\ikvm</em> (or <em>X:\ikvm</em> where X is your drive). You would find the ikvm executables and DLLs in the <em>C:\ikvm\bin</em> directory.&nbsp; Open a command or shell window, cd to <em>C:\ikvm\bin</em>, and type ‘ikvm’. </p> 
             <p>If your system is operating correctly, you should see the following output:</p> 
             <p>usage: ikvm [-options] &lt;class&gt; [args...] (to execute a class) or ikvm -jar [-options] &lt;jarfile&gt; [args...] (to execute a jar file)</p> 
             <p>For Linux based systems, the setup is similar as above. This is all you need to do for running the demo application.</p> 
             <h4>Our Demo Java Business Class (JavaToNet.java) </h4> 
             <pre lang="Java"><span class="code-keyword">public</span> <span class="code-keyword">class</span> JavaToNet 
{
    <span class="code-keyword">public</span> <span class="code-keyword">static</span> <span class="code-keyword">void</span> main(<span class="code-sdkkeyword">String</span>[] args)

Our objective is to access these functions through our C# application. Compile the above Java file to get the <em>JavaToNet.class</em>. We will use this Java class file to generate the .NET DLL to be referenced in our C# program.</p> 
             <h2>Using IKVM.NET to Convert Java Class to .NET DLL</h2> 
             <p>Copy the above Java class file (<em>JavaToNet.class</em>) to the <em>C:\ikvm\bin</em> directory. Now run the following command:</p> 
             <img height="63" hspace="0" src="/KB/dotnet/csharpikvm/csharpikvm001.jpg" width="576" border="0"> 
             <p>This would create the <em>JavaToNet.dll</em> from the <em>JavaToNet.class</em> file. There are other command line option for <em>ikvmc.exe</em>. For example: ‘<code>ikvmc –target:exe javaToNet.class</code>’ would create an&nbsp;EXE and not a DLL. You can get all the options by typing ‘ikvmc’ in the command line.</p>

             <h2>Conclusion</h2> 
             <p>Since these methods had originally been written in Java, IKVM.NET provides us an easy and viable way to access those classes and methods from a .NET application. Similarly as shown above in the ‘Time Zone’ example, you can access most of the existing Java packages (e.g. <em>java.io</em>, <em>java.util</em>, etc.) and use them in your application.</p> 
             <p>However there are certain drawbacks. IKVM.NET, while still being actively developed, has limited support for AWT classes and hence porting Java GUI can be ruled out at present. Also some of the default Java classes are still being ported so you might not get all the functionalities you require. Also if your application depends on the exact Java class loading semantics, you might have to modify it to suit your needs.</p>
