           <h1 name="7c96" id="7c96" class="graf graf--h3 graf--leading graf--title">Modular javascript web and mobile applications using Raincatcher framework</h1>
           <p name="b1c3" id="b1c3" class="graf graf--p graf-after--h3"><a href="https://github.com/feedhenry-raincatcher" data-href="https://github.com/feedhenry-raincatcher" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Raincatcher</a> is solution that provides reusable plugins to build mobile and web applications. Target of Raincatcher is to provide default implementations for various functionalities like Maps with GPS tracking, Photo collections with storage, tasks managment and many more.</p>
           <p name="95a9" id="95a9" class="graf graf--p graf-after--p">Each of this areas are implemented as separate plugin (npm module) that can be customized by application developers by extending and modifying module API. All modules are organized and used as part of demo application which provide reference how to use them as part of external system.</p>
           <figure name="2ff8" id="2ff8" class="graf graf--figure graf-after--p">
            <div class="aspectRatioPlaceholder is-locked" style="max-width: 694px; max-height: 391px;">

              <canvas class="progressiveMedia-canvas js-progressiveMedia-canvas"></canvas>
              <img class="progressiveMedia-image js-progressiveMedia-image" data-src="https://cdn-images-1.medium.com/max/1600/1*wsHXHa8sgqu0u6ZZMUMwWQ.png">

           <p name="8cd8" id="8cd8" class="graf graf--p graf-after--figure">Modules can provide various types of implementations:</p>
           <ul class="postList">
            <li name="f1e0" id="f1e0" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">client (business logic)</strong></li>
            <li name="1289" id="1289" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">server side (API)</strong></li>
            <li name="6706" id="6706" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">frontend (UI like angular)</strong></li>
            <li name="22ca" id="22ca" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">utility modules (services)</strong></li>
           </ul>
           <p name="f62e" id="f62e" class="graf graf--p graf-after--li">Client implementations contain vanilla javascript code for common operations. Typical client code may contain some algorithms, business logic or call <em class="markup--em markup--p-em">Apache Cordova</em> plugins on mobile. Server side modules are responsible for implementing Raincatcher API using node and<em class="markup--em markup--p-em"> Express.js </em>framework<strong class="markup--strong markup--p-strong">. </strong>UI modules currently contain <em class="markup--em markup--p-em">Angular.js </em>directives routes and other javascript code that are responsible to render html views. Utility modules are created to aggregate common, low level functionalities like storage, data-sync, application cache etc.</p>
           <h3 name="66d7" id="66d7" class="graf graf--h3 graf-after--p">Challenges of plugin based architecture</h3>
           <p name="0dd2" id="0dd2" class="graf graf--p graf-after--h3">Each module can communicate with other module thanks to exposed interface and communication module called mediator. Plugins are implemented as <em class="markup--em markup--p-em">common.js </em>modules supported by node for cloud and <em class="markup--em markup--p-em">browserify</em> for browser. Raincatcher allows to swap functionality by providing customized version of default modules that will implement the same or extended API. Customization process looks as bellow:</p>
           <figure name="3546" id="3546" class="graf graf--figure graf-after--p">
            <div class="aspectRatioPlaceholder is-locked" style="max-width: 690px; max-height: 141px;">

           <p name="bd2f" id="bd2f" class="graf graf--p graf-after--figure">A developer need to clone plugin, provide custom implementation, publish new module to npm or use npm link to swap functionality. Custom plugin can be replaced with default package in package.json file. Thanks to predefined interface in each plugin our end user application will still work properly without any additional changes.</p>
           <p name="69c8" id="69c8" class="graf graf--p graf-after--p">To make this process possible plugins are indirectly linked between each other. Every plugin is listed as required component in top level app. Modules can communicate registering their interface and exposing it to global javascript scope.</p>
