              <p>I have been challenged by @nhgrif to write a Fraction class to better my understanding of OOP. I have found an interesting challenge <a href="https://stackoverflow.com/questions/5690663/c-fractions-class">here</a> (I apologize that the source is not mentioned, I was not able to find it either).</p> 
              <blockquote> 
               <p><strong>Problem statement</strong>: Create a class named Fraction having two integer data members named for a fraction's numerator and denominator. The class' default constructor should provide both data members with default values of 1 if no explicit initialization is provided. The constructor must also prohibit a 0 denominator value. Include member functions for displaying an object's data values and mathematical functions capable of adding, subtracting, multiplying, and dividing two Fraction objects.</p> 
              </blockquote> 
              <p>I have written this <a href="http://ideone.com/lBf83A" rel="nofollow noreferrer">on Ideone.com</a> hence explaining why everything is in one file. I have made a slight change to the default values given no parameters, as I have been told a <code>0/1</code> default fraction makes more sense than <code>1/1</code>.</p> 
              <p>I have also added a simple but potentially useful method to cast a fraction into its decimal value. </p> 
              <p>I named my fraction variables using a pattern like <code>fractionN_D</code> where N is numerator and D is denominator. It seemed short enough to be practical (as compared to <code>fractionThreeFourths</code>) while still carrying meaning. Any suggestion on that is of course welcome.</p> 
              <pre><code> class Fraction {
    private final int numerator;
    private final int denominator;

    Fraction() {
        numerator = 0;
        denominator = 1;
    }

    Fraction (int numerator, int denominator) {
        this.numerator = numerator;
        if (denominator == 0) {
            throw new IllegalArgumentException("Denominator cannot be zero.");
        } else {
            this.denominator = denominator;
        }
    }

    public String getFraction() {
        return numerator + "/" + denominator;
    }
    public double getFractionDecimal() {
        return ((double) numerator) / denominator;
    }

    // Mathematical functions

    public String addFraction(Fraction otherFraction) {
        // Sum formula: a/b + c/d = (ad + cb)/bd
        int a = this.numerator;
        int b = this.denominator;
        int c = otherFraction.numerator;
        int d = otherFraction.denominator;
        return ((a*d) + (c*b)) + "/" + (b*d);
    }

    public String subtractFraction(Fraction otherFraction) {
        // Subtraction formula: a/b - c/d = (ad - cb)/bd
        int a = this.numerator;
        int b = this.denominator;
        int c = otherFraction.numerator;
        int d = otherFraction.denominator;
        return ((a*d) - (c*b)) + "/" + (b*d);
    }

    public String multiplyByFraction(Fraction otherFraction) {
        // Multiplication formula: a/b * c/d = ac/bd
        int a = this.numerator;
        int b = this.denominator;
        int c = otherFraction.numerator;
        int d = otherFraction.denominator;
        return (a*c) + "/" + (b*d);
    }

    public String divideByFraction(Fraction otherFraction) {
        // Division formula: (a/b) / (c/d) = ad/bc
        int a = this.numerator;
        int b = this.denominator;
        int c = otherFraction.numerator;
        int d = otherFraction.denominator;
        return (a*d) + "/" + (b*c);
    }
}

class TestFraction {
    public static void main(String[] args) {
        Fraction fraction3_4 = new Fraction(3,4);
        Fraction fraction2_3 = new Fraction(2,3);

        System.out.println("F1: " + fraction3_4.getFraction() + " Dec: " + fraction3_4.getFractionDecimal());
        System.out.println("F2: " + fraction2_3.getFraction() + " Dec: " + fraction2_3.getFractionDecimal() + "\n");
        System.out.println("F1 + F2: " + fraction3_4.addFraction(fraction2_3));
        System.out.println("F2 + F1: " + fraction2_3.addFraction(fraction3_4) + "\n");
        System.out.println("F1 - F2: " + fraction3_4.subtractFraction(fraction2_3));
        System.out.println("F2 - F1: " + fraction2_3.subtractFraction(fraction3_4) + "\n");
        System.out.println("F1 * F2: " + fraction3_4.multiplyByFraction(fraction2_3));
        System.out.println("F2 * F1: " + fraction2_3.multiplyByFraction(fraction3_4) + "\n");
        System.out.println("F1 / F2: " + fraction3_4.divideByFraction(fraction2_3));
        System.out.println("F2 / F1: " + fraction2_3.divideByFraction(fraction3_4));
    }
}

             <div class="post-text" itemprop="text"> 
              <h3>Public</h3> 
              <pre><code>class Fraction {
</code></pre> 
              <p>I get it that you're writing this in ideone so you cannot do this, but for the record, that class should be <code>public class</code></p> 
              <pre><code>Fraction() {
</code></pre> 
              <p>This constructor should be <code>public</code></p> 
              <pre><code>Fraction (int numerator, int denominator) {
</code></pre> 
              <p>This constructor should also be <code>public</code></p> 
              <h3>Constructor calls constructor</h3> 
              <p>As your <code>Fraction()</code> constructor is essentially initializing it with specific values you can call one constructor from another</p> 
              <pre><code>public Fraction() {
    this(0, 1);
}
</code></pre> 
              <h3>Throw or else</h3> 
              <pre><code>if (denominator == 0) {
    throw new IllegalArgumentException("Denominator cannot be zero.");
} else {
</code></pre> 
              <p>If you throw an exception inside the <code>if</code>, there's no need to write <code>else</code> explicitly. Just put the code that is currently in the <code>else</code> after the <code>if</code>-statement. Like this:</p> 
              <pre><code>public Fraction (int numerator, int denominator) {
    this.numerator = numerator;
    if (denominator == 0) {
        throw new IllegalArgumentException("Denominator cannot be zero.");
    }
    this.denominator = denominator;
}
</code></pre> 
              <p><em>(I do however agree about @Jeroen's answer about putting the denominator check first)</em></p> 
              <h3>Returning something useful</h3> 
              <pre><code>public String addFraction(Fraction otherFraction) {
</code></pre> 
              <p>What do you get when you add a <code>Fraction</code> with another <code>Fraction</code>? (Mathematically speaking)</p> 
              <p>You get... A STRING!</p> 
              <p>No you don't. You get a <code>Fraction</code></p> 
              <p>These mathematical methods you have written here should really return a new <code>Fraction</code> instance.</p> 
              <p>And you don't need to have <code>Fraction</code> in the name. Simply <code>add</code>, <code>subtract</code>, <code>multiply</code>, <code>divide</code> is enough.</p> 
              <h3>MOAR METHODS!</h3> 
              <p>I would recommend adding a <code>toString</code> method, in your case the implementation is simple:</p> 
              <pre><code>@Override
public String toString() {
    return getFraction();
}
</code></pre> 
              <p>I would also like to see a <code>normalize</code> method that returns a fraction that has been reduced as much as possible, for example:</p> 
              <ul> 
               <li>18/4 --&gt; 9/2</li> 
               <li>24/4 --&gt; 6/1</li> 
               <li>4/8 --&gt; 1/2</li> 
              </ul> 
              <p>I am also missing <code>public int getNumerator()</code> and <code>public int getDenominator()</code>, which would make your <code>Fraction</code> class much more useful.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>It's good that you made your class immutable. I'd make the class and its constructors <code>public</code> as well.</p> 
              <p>If you offer a default constructor at all, you should chain them:</p> 
              <pre><code>public Fraction() {
    this(0, 1);
}
</code></pre> 
              <p>… but consider offering constants <code>ZERO</code> and <code>ONE</code> instead.</p> 
              <p>Instead of <code>IllegalArgumentException</code>, consider throwing <code>ArithmeticException</code> — that's what happens when you try to divide by zero.</p> 
              <p>The <code>getFraction()</code> and <code>getFractionDecimal()</code> methods are confusingly named. I'd call them <code>toString()</code> and <code>toDouble()</code>, respectively.</p> 
              <p>Your arithmetic functions could have shorter names, e.g. <code>add(Fraction addend)</code>. They should each return another <code>Fraction</code>, not a <code>String</code>. (You could always call <code>toString()</code> on the result, but parsing a <code>String</code> back into a <code>Fraction</code> is relatively hard.)</p> 
              <p>The results should be in reduced form. Arguably, that should be the case for all <code>Fraction</code>s, and enforced in the constructor, so that <code>(new Fraction(2, 4)).toString()</code> produces <code>"1/2"</code>.</p> 
              <p>Note that dividing by another fraction has the possibility of dividing by zero.</p>

              <pre><code>Fraction (int numerator, int denominator) {
    if (denominator == 0) {
        throw new IllegalArgumentException("Denominator cannot be zero.");
    }

    this.denominator = denominator;
    this.numerator = numerator;       
}
</code></pre> 
              <hr> 
              <p>Instead of <code>getFraction()</code> and <code>getFractionDecimal()</code> I would use <code>toString()</code> for the string representation and <code>getDecimal()</code> for the decimal one.</p>

               <pre><code>public String divideByFraction(Fraction otherFraction) {
    // Division formula: (a/b) / (c/d) = ad/bc
    int a = this.numerator;
    int b = this.denominator;
    int c = otherFraction.numerator;
    int d = otherFraction.denominator;
    return (a*d) + "/" + (b*c);
}
</code></pre>

              <p>Taking in mind the comments made by <em>Simon</em>, <em>200_success</em>, and others, we need to fix this particular method up a bit.</p> 
              <p>If <code>this</code> is equal to zero (zero numerator) or <code>otherFraction</code> is equal to zero (zero numerator), the result will be some number over 0. If we were actually returning a Fraction, then we'd be constructing one in here, and the constructor (and thus, this method) will throw an <code>IllegalArgumentException</code>. Java probably has a <code>DivisionByZero</code> except, and this is actually what we want to use, so let's check for zero numerators/divisors. If <code>this</code> has a numerator of zero, we can just quickly return a zero fraction. If <code>otherFraction</code> has a numerator of zero, we need to throw a division by zero exception.</p> 
              <p>So, something like this:</p> 
              <pre><code>public Fraction divide(Fraction otherFraction) {
    if (otherFraction.numerator == 0) {
        throw new IllegalArgumentException("Division by zero.")
    }

    int numerator = this.numerator * otherFraction.denominator;
    int denominator = this.denominator * otherFraction.denominator;

    return new this(numerator, denominator);
}
</code></pre> 
              <hr> 
              <p>And fundamentally, we have to ask ourselves, should the math functions return a new <code>Fraction</code> or mutate the current one?</p> 
              <p>You <em>could</em> make the argument for something like this:</p> 
              <pre><code>public void divide(Fraction divisor) {
    if (divisor.numerator == 0) {
        throw new IllegalArgumentException("Division by zero.")
    }

    this.numerator *= divisor.denominator;
    this.denominator *= divisor.numerator;
    this.reduce();
}

static public Fraction divide(Fraction dividend, Fraction divisor) {
    int numerator = dividend.numerator * divisor.denominator;
    int denominator = dividen.denominator * divisor.numerator;
    return new this(numerator, denominator);
}
</code></pre> 
              <p>So, now usage would look like this:</p> 
              <pre><code>Fraction threeFifths = new Fraction(3,5);
Fraction oneHalf = new Fraction(1,2);
threeFifths.divide(oneHalf); // three fifths is now 6/5
Fraction twelveFifths = Fraction.divide(threeFifths,oneHalf);
</code></pre>

              <p>Firstly I would say the proper mathematical name for this class is <code>Rational</code> but perhaps that's just a matter of taste.</p> 
              <p>Secondly I would put the object in a canonical format. For example I would only allow the numerator to be negative. If the denominator is passed into a constructor as negative I would negate the numerator sign and use <code>java.lang.Math.abs(denominator)</code>.</p> 
              <p>I would also divide numerator and denominator by their greatest common divisor (gcd) to make sure objects have a unique and simple form.</p> 
              <p>In that event your <code>equals</code> method is trivial and you also stave off overflow. Without reduction to simplest terms you will encounter overflow very quickly.</p>

              <p>Let us take a look at the <code>addFraction()</code> method </p> 
              <blockquote> 
               <pre><code>public String addFraction(Fraction otherFraction) {
    // Sum formula: a/b + c/d = (ad + cb)/bd
    int a = this.numerator;
    int b = this.denominator;
    int c = otherFraction.numerator;
    int d = otherFraction.denominator;
    return ((a*d) + (c*b)) + "/" + (b*d);
}
</code></pre> 
              </blockquote> 
              <p>it would be very easy to change this to returning a <code>Fraction</code> instead of a <code>String</code> </p> 
              <pre><code>public Fraction add(Fraction otherFraction) {
    // Sum formula: a/b + c/d = (ad + cb)/bd
    int a = this.numerator;
    int b = this.denominator;
    int c = otherFraction.numerator;
    int d = otherFraction.denominator;
    if(b == d) {
        return new Fraction(a + c, b);
    }
    return new Fraction(a * d + c * b, b * d);
}  
</code></pre> 
              <p>and together with a overridden <code>toString()</code> method we don't need the formerly returned <code>String</code> anymore. </p> 
              <pre><code>@Override
public String toString() {
    if(denominator == 1) {
        return numerat.toString();
    }
    return numerator + "/" + denominator;
}  
</code></pre> 
              <p>which I would prefer over the former <code>getFraction()</code> method, because this isn't returning a <code>Fraction</code>. </p> 
              <p>This together would make e.g. this possible: </p> 
              <pre><code>Fraction first = new Fraction(12, 3);
Fraction second = new Fraction(6, 3);
Fraction sum = first.add(second);

System.out.print(first + " + " + second + " = " + sum);  
</code></pre> 
              <p>Which would output </p> 
              <pre><code>12/3 + 6/3 = 18/3  
</code></pre>
