      <h1 class="front-page">Form validation made easy</h1> 
      <p class="intro-txt"> jQuery Form Validator is a feature rich and <strong>multilingual</strong> jQuery plugin that makes it easy to validate user input while <strong>keeping your HTML markup clean from javascript</strong> code. Even though this plugin has a wide range of validation functions it's designed to require <strong>as little <img src="static/img/jquery.png" alt="jQuery" class="jquery-logo"> network traffic as possible</strong>. This is achieved by grouping together validation functions in "modules", making it possible to load only those functions that's needed to validate a particular form. </p> 
      <p> This plugin is available through <a href="#" class="click-download">cdnjs</a>, <a href="https://www.npmjs.com/package/jquery-form-validator" target="_blank">npm</a>, <a href="#" class="click-download">bower</a> and <a href="https://github.com/victorjonsson/jQuery-Form-Validator/" target="_blank">github</a>. </p> 
      <h3>Basic example</h3> 
      <pre class="prettyprint linenums">

       <div class="convert-code">

        <form action="/registration" method="POST"> 
         <p> User name (4 characters minimum, only alphanumeric characters): <input data-validation="length alphanumeric" data-validation-length="min4"> </p> 
         <p> Year (yyyy-mm-dd): <input data-validation="date" data-validation-format="yyyy-mm-dd"> </p> 
         <p> Website: <input data-validation="url"> </p> 
         <p> <input type="submit"> </p> 
        </form>

       </div>

      <h1>Default validators</h1> 
      <p>The default features of this plugin doesn't require you to load any module. Here you can try out a <a href="#complete-test-form">complete form</a> with all default validators present.</p> 
      <hr> 
      <h3 data-page-section="length:Length (required)">Answer length (required)</h3> 
      <pre class="prettyprint convert-code">

       <!-- Require an answer (can be applied to all types of inputs and select elements) -->
<input type="text" data-validation="required">
<input type="checkbox" name="agreement" data-validation="required">
<select name="answer" data-validation="required">
    <option value=""> - - Answer - - </option>
    <option>Yes</option>
    <option>No</option>
</select>


       <!-- Max 100 characters -->
<input type="text" data-validation="length" data-validation-length="max100">


       <!-- Minimum 20 characters -->
<input type="text" data-validation="length" data-validation-length="min20">


       <!-- No less than 50 characters and no more than 200 characters -->
<input type="text" data-validation="length" data-validation-length="50-200">


       <!-- Require that atleast 2 options gets choosen -->
<select multiple size="5" data-validation="length" data-validation-length="min2">
  <option>A</option>
  <option>B</option>
  <option>C</option>
  <option>D</option>
  <option>E</option>
</select>   
</pre> 
      <p><em>This plugin also supports the attributes "required" and "maxlength" by using the <a href="#configuration_html5">Html5 module</a>.</em></p> 
      <h3 data-page-section="numbers:Numbers">Numbers</h3> 
      <pre class="prettyprint convert-code">

       <!-- Any numerical value -->
<input type="text" data-validation="number">


       <!-- Only allowing float values -->
<input type="text" data-validation="number" data-validation-allowing="float">


       <!-- Allowing float values and negative values -->
<input type="text" data-validation="number" data-validation-allowing="float,negative">


       <!-- Validate float number with comma separated decimals -->
<input type="text" data-validation="number" data-validation-allowing="float" break="" data-validation-decimal-separator=",">


       <!-- Only allowing numbers from 1 to 100 -->
<input type="text" data-validation="number" data-validation-allowing="range[1;100]">


       <!-- Only allowing numbers from -50 to 30 -->
<input type="text" data-validation="number" data-validation-allowing="range[-50;30],negative">


       <!-- Only allowing numbers from 0.05 to 0.5 -->
<input type="text" data-validation="number" data-validation-allowing="range[0.05;0.5],float">
                </pre> 
      <p>You can also define the decimal separator when initializing the validation.</p>

&lt;/form&gt;

      <h3 data-page-section="checkboxgroup:Checkbox group">Checkboxes Group</h3> 
      <p>Validate qty of checkboxes in a group (same name) have been checked, using min, max or range. Only the first checkbox element in the group needs to have the validation attributes added. </p> 
      <pre class="prettyprint convert-code">

       <!-- Require checkboxes in this group, min1 -->
<input type="checkbox" name="newsletters[]" data-validation="checkbox_group" data-validation-qty="min1">
</pre> 
      <pre class="prettyprint convert-code">

       <!-- Require checkboxes in this group, max3 -->
<input type="checkbox" name="newsletters[]" data-validation="checkbox_group" data-validation-qty="max3">
</pre> 
      <pre class="prettyprint convert-code">

       <!-- Require checkboxes in this group, min1, max4 -->
<input type="checkbox" name="newsletters[]" data-validation="checkbox_group" data-validation-qty="1-4">
</pre> 
      <p> If your checkboxes group is generated by a server-side script and you don't want to add the validation attributes to each input element, you can use this javascript snippet before calling the validatorLoad() function </p> 
      <pre class="prettyprint">
&lt;!-- Add validation attributes to first input element in
 checkboxes group, before loading validator --&gt;
&lt;script&gt;
$("[name='newsletters[]']:eq(0)")
    .valAttr('','validate_checkbox_group')
    .valAttr('qty','1-2')
    .valAttr('error-msg','chose 1, max 2');
&lt;/script&gt;
</pre>

      <h3 data-page-section="help:Display help text">Display help text</h3> 
      <p>It is possible to display help information beside each input. The text will fade in when the input gets focus on and fade out when the input looses focus. The container for the help text will have the class <em>form-help</em>. If you don't want this feature you can read the <a href="#configuration">setup guide</a> on how to disable it.</p>

      <p>This validator can be used to validate that the values of two inputs are the same. The first input should have a name suffixed with <em>_confirmation</em> and the second should have the same name but without the suffix.</p>

      <h3 data-page-section="backend: Server-side validation">Validate on server side</h3> 
      <p> By using this validator you can validate the value given by the user on the server before the form gets submitted. The validation function will send a POST request to the URL declared in <code>data-validation-url</code>. The argument posted to the URL will have the same name as the input being validated. </p> 
      <p> The form will get the class <em>validating-server-side</em> while the server is being requested. </p> 
      <p> The response from the validation script must be a JSON formatted object, containing the properties "valid" and "message". </p> 
      <pre class="prettyprint">
{
    "valid" : true|false,
    "message" : "String with text that should be displayed as error message"
}</pre> 
      <div class="pre-title">
       <strong>Form</strong>
      </div> 
      <pre class="prettyprint">

       <div class="convert-code">
    
        <form action=""> 
         <p> <strong>User name:</strong> <input name="user" data-validation="server" data-validation-url="/validate-input.php"> </p> ... 
        </form>

       </div>
</pre> 
      <div class="pre-title">
       <strong>/validate-input.php</strong>
      </div> 
      <pre class="prettyprint">
&lt;?php
$response = array(
    'valid' =&gt; false,
    'message' =&gt; 'Post argument "user" is missing.'
);

    $userRepo = new UserRepository( DataStorage::instance() );
    $user = $userRepo-&gt;loadUser( $_POST['user'] );

    if( $user ) {
        // User name is registered on another account
        $response = array('valid' =&gt; false, 'message' =&gt; 'This user name is already registered.');
    } else {

      <h3 data-page-section="size:File size">File size</h3> 
      <p> This validation is only supported by Internet Explorer 10, Mozilla FireFox v &gt;= 3.6 and any of the later versions of webkit based browsers. </p> 
      <pre class="prettyprint convert-code">

       <!-- Validate that file isn't larger than 512 kilo bytes -->
<input type="file" data-validation="size" data-validation-max-size="512kb">


       <!-- Validate that file isn't larger than 3 mega bytes -->
<input type="file" data-validation="size" data-validation-max-size="3M">

      <h3 data-page-section="type:File type">File type</h3> 
      <p> This validation will fall back on checking the file extension in older browsers. In modern browsers the validation will check that any of the extensions in <code>data-validation-allowing</code> exists in the mime type declaration of the file. This means that <code>data-validation-allowing="pdf"</code> will work in both modern browsers (checking against "application/pdf") and older browsers (checking the file extension ".pdf"). </p> 
      <pre class="prettyprint convert-code">

       <!-- Validate that file is an image of type JPG, GIF or PNG and not larger than 2 mega bytes -->
<input type="file" data-validation="mime size" break0="" data-validation-allowing="jpg, png, gif" break="" data-validation-max-size="2M">


       <!-- Validate that a file is given and that it has .txt as extension -->
<input type="file" data-validation="required extension" data-validation-allowing="txt">
</pre> 
      <p>Validating multiple files (with separate error messages depending on failed validation):</p> 
      <pre class="prettyprint">
&lt;input type="file" multiple="multiple" name="images"

    data-validation-allowing="jpg, png, gif"
    data-validation-max-size="512kb"
    data-validation-error-msg-size="You can not upload images larger than 512kb"

      <p> By using the module sanitize you will be able to declare sanitation functions that will be triggered when inputs looses focus. You declare them using the attribute data-sanitize. The implemented rules are: </p> 
      <ul> 
       <li>trim</li> 
       <li>trimLeft</li> 
       <li>trimRight</li> 
       <li>upper (convert all letters to upper case)</li> 
       <li>lower (convert all letters to lower case)</li> 
       <li>capitalize (convert the first letter in all words to upper case)</li> 
       <li>insertRight (declare a text that should be inserted at the end of the value, attribute <code>data-sanitize-insert-right</code>)</li> 
       <li>insertLeft (declare a text that should be inserted at the beginning of the value, attribute <code>data-sanitize-insert-left</code>)</li> 
       <li>strip (comma separated list with words that gets automatically removed)</li> 
       <li>escape (convert &lt; &gt; &amp; ' " to html entities)</li> 
       <li>numberFormat (declare the attribute <code>data-sanitize-number-format</code> with any of the formats described on <a href="http://numeraljs.com/">http://numeraljs.com/</a>. Note that this rule requires that numeral.js is included in the page)</li> 
      </ul>

      <h3 data-page-section="modules:Loading modules">Loading modules</h3> 
      <p> Use the property <code>modules</code> to declare which modules you want to load. The property should contain a comma separated string with module names. </p>

      <h3 data-page-section="callbacks:Validation callbacks">Validation callbacks</h3> 
      <p> There are four different callbacks you can declare when setting up the validation. There are also two types of events triggered on elements when they get validated. </p> 
      <ul> 
       <li> <strong>onError</strong>  Called when the user submits the form and one or more inputs has an invalid value. </li> 
       <li> <strong>onSuccess</strong>  Called when the user submits the form and no input with an invalid value can be found. </li> 
       <li> <strong>onValidate</strong>  Called when the user submits the form and all inputs have become validated. This callback is called before onError/onSuccess. You can add error messages from this callback by returning an object containing the property "element" (referring to the input element having an invalid value) and "message". You can also return an array with error objects if you want to add several error messages. The validation procedure will go on as normally if the callback function doesn't return anything. </li> 
       <li> <strong>onElementValidate</strong>  Called after that an element have become validated. </li> 
       <li> <strong>beforeValidation</strong>  jQuery event triggered on elements right before they become validated. Use this event if you want to prevent validation in runtime by setting <code>data-validation-skipped="1"</code>. </li> 
       <li> <strong>validation</strong>  jQuery event triggered on elements after they have been validated. </li> 
      </ul> 
      <pre class="prettyprint">
    $.validate({
        form : '#registration-form',
        modules : 'security',
        onError : function($form) {
            alert('Validation of form '+$form.attr('id')+' failed!');
        },
        onSuccess : function($form) {
            alert('The form '+$form.attr('id')+' is valid!');
            return false; // Will stop the submission of the form
        },
        onValidate : function($form) {
            return {
                element : $('#some-input'),
                message : 'This input has an invalid value for some reason'
            }
        },
        onElementValidate : function(valid, $el, $form, errorMess) {
            console.log('Input ' +$el.attr('name')+ ' is ' + ( valid ? 'VALID':'NOT VALID') );
        }
    });

    // Validation event listeners
    $('input')
        .on('beforeValidation', function(value, lang, config) {
            console.log('Input "'+this.name+'" is about to become validated');
            // Call $(this).attr('data-validation-skipped', 1); to prevent validation
        })
        .on('validation', function(evt, valid) {
            console.log('Input "'+this.name+'" is ' + (valid ? 'VALID' : 'NOT VALID'));
        });
</pre>

$('#the-input').validate(function(valid, elem) {

      <h3 data-page-section="submit:Validate form without submit">Validate form without submit</h3> 
      <p>You can use the method <code>$.fn.isValid(lang, config, displayErrors)</code> if you want to validate a form programmatically. The function takes three arguments. The first argument being an object used to override the default <a href="#localization_object"> error dialogs</a> and the second argument being the configuration object. Use the third argument to tell the validation function if error dialogs should be displayed or not.</p> 
      <p>Here you can see an example code where we collect all the error dialogs during the validation and pass them on to a custom function displaying the error dialogs.</p> 
      <pre class="prettyprint">
// Array used as error collection
var errors = [],

// Validation configuration
conf = {
    onElementValidate : function(valid, $el, $form, errorMess) {
       if( !valid ) {
          // gather up the failed validations
          errors.push({el: $el, error: errorMess});
       }
    }
},

// Optional language object
lang = {
    ...
};

// Manually load the modules used in this form
$.formUtils.loadModules('security, date');

$('#check-form').on('click', function() {
   // reset error array
   errors = [];
   if( !$(this).isValid(lang, conf, false) ) {
        displayErrors( errors );
   } else {
     // The form is valid
   }
});
&lt;/script&gt;</pre> 
      <p> <strong>Notice!</strong> This method won't be able to tell if the form is valid if your'e using asynchronous validators, like the <a href="#security-validators_backend">server-side validator</a> and the <a href="#file-validators_dimension">image dimensions validator</a>. In such a case you must combine the method call with the <a href="#configuration_callbacks">onSuccess callback</a>. </p>

      <h3 data-page-section="jsonly:Setup validation using JavaScript">Setup validation using only JavaScript</h3> 
      <p> The whole point of this plugin is to not having to write JavaScript code when configuring form validations. If you feel the need to configure your form validation using JavaScript you probably should look into <a href="http://jqueryvalidation.org/" target="_blank">jquery-validation</a> or <a href="http://formvalidation.io/" target="_blank">formvalidation.io</a> or any of the other superb form validation solutions out there. </p> 
      <p> But there is of course a way to accomplish this using jQuery-Form-Validator as well, by using the module <code>jsconf</code>. </p> 
      <pre class="prettyprint">
&lt;script&gt;
    var config = {
      // ordinary validation config
      form : 'form'
      // reference to elements and the validation rules that should be applied
      validate : {
          'input-name' : {
            validation : 'length, alphanumeric',
            length : '5-12',
            '_data-sanitize': 'trim',
          },
          '.password-input-class' : {
            validation : 'length, strength'
            strength : '3',
            length : 'min10'
         },
         '#date-input-id' : {
            validation : 'date',
            format : 'dd/mm/yyyy'
         }
      }
    };

$.validate({
   modules : 'jsconf, security',
   onModulesLoaded : function() {
        $.setupValidation(config);
   }
});
&lt;/script&gt;
</pre> 
      <p>The first call to <code>$.validate</code> should only contain the module declaration (it must also contain jsconf) and a callback for the event <code>onModulesLoaded</code>. In the callback you invoke the function <code>$.setupValidation</code> with the configuration object as argument.</p> 
      <p>The configuration object supports all the ordinary configuration options. Use the property <code>validate</code> to tell the jsconf-module which elements that should have validation rules. Begin the property with <code>_</code> if you want to add an attribute without the data-validation prefix.</p> 
      <p> All the configuration options for each validation rule is also supported. When applying them via markup they're all prefixed with <code>data-validation</code>. When declaring them using the jsconf-module you simply exclude the prefix. </p> 
      <pre class="prettyprint">
    &lt;input type="file" multiple="multiple" name="images"
           data-validation="length mime size"
           data-validation-optional="true"
           data-validation-length="min2"
           data-validation-allowing="jpg, png, gif"
           data-validation-max-size="512kb"
           data-validation-error-msg="Error message..."
            /&gt;
</pre> 
      <p>Would be translated into:</p> 
      <pre class="prettyprint">
var conf = {
   ...
   validate : {
      images : {
             'validation' : 'length mime size',
             'optional' : true,
             'length' : 'min2'
             'allowing' : 'jpg, png, gif',
             'max-size' : '512kb',
             'error-msg' : 'Error message...'
      }
   }
};
</pre> 
      <h3 data-page-section="toggleDisabled:Toggle disabled form submit">Toggle disabled form submit</h3> 
      <p> You can use the module <code>toggleDisabled</code> if you want to have the submit button disabled until the form is completely filled out. </p> 
      <form action="#" class="form-horizontal well test-form toggle-disabled"> 
       <div class="control-group"> 
        <label class="control-label">E-mail:</label> 
        <div class="controls"> 
         <input type="text" data-validation="email" data-validation-error-msg="You did not enter a valid e-mail"> 
        </div> 
       </div> 
       <div class="control-group"> 
        <label class="control-label">Password:</label> 
        <div class="controls"> 
         <input type="password" data-validation="required" data-validation-error-msg="You did not enter a password"> 
        </div> 
       </div> 
       <div class="control-group"> 
        <div class="controls"> 
         <input type="submit" class="btn" value="Login"> 
        </div> 
       </div> 
      </form> 
      <pre class="prettyprint">
$.validate({
    modules : 'toggleDisabled',
    disabledFormFilter : 'form.toggle-disabled',
    showErrorDialogs : false
});
</pre> 
      <p> Use the property <code>disabledFormFilter</code> to target specific forms on your page. By omitting the property the disabled feature will become enabled on all forms. Use the property <code>showErrorDialogs</code> to tell the module whether or not to display error dialogs (defaults to true). </p>

      <h1>Writing a custom validator</h1> 
      <p> You can use the function <code>$.formUtils.addValidator()</code> to add your own validation function. Here's an example of a validator that checks if the input contains an even number. </p> 
      <pre class="prettyprint linenums">

       <div class="convert-code">
    
        <form action="" method="post"> 
         <p> <input type="text" data-validation="even_number"> </p> ... 
        </form>

       </div>
&lt;script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js"&gt;&lt;/script&gt;
&lt;script src="//cdnjs.cloudflare.com/ajax/libs/jquery-form-validator/2.3.26/jquery.form-validator.min.js"&gt;&lt;/script&gt;
&lt;script&gt;

// Add custom validation rule
$.formUtils.addValidator({
    name : 'even_number',
    validatorFunction : function(value, $el, config, language, $form) {
        return parseInt(value, 10) % 2 === 0;
    },
    errorMessage : 'You have to answer with an even number',
    errorMessageKey: 'badEvenNumber'
});

// Setup form validation
$.validate();

&lt;/script&gt;
</pre> 
      <div class="alert-info well"> 
       <strong>Note:</strong> you can also write 
       <a href="#default-validators_custom" style="text-decoration: underline">inline regexp</a> if wanting to do a custom validation 
      </div> 
      <h3>Required properties passed into $.formUtils.addValidator</h3> 
      <p><strong>name</strong> - The name of the validator, which is used in the validation attribute of the input element.</p> 
      <p><strong>validatorFunction</strong> - Callback function that validates the input. Should return a boolean telling if the value is considered valid or not.</p> 
      <p><strong>errorMessageKey</strong> - Name of <a href="#localization_object">language property</a> that is used in case the value of the input is invalid.</p> 
      <p><strong>errorMessage</strong> - An alternative error message that is used if errorMessageKey is left with an empty value or isn't defined in the language object. Note that you also can use <a href="#localization">inline error messages</a> in your form.</p> 
      <p>The validation function takes these five arguments:</p> 
      <ul> 
       <li>value  the value of the input thats being validated</li> 
       <li>$el  jQuery object referring to the input element being validated</li> 
       <li>config  Object containing the configuration of this form validation</li> 
       <li>language  Object with error dialogs</li> 
       <li>$form  jQuery object referring to the form element being validated</li> 
      </ul> 
      <hr> 
      <h2>Creating a custom module</h2> 
      <p>A "module" is basically a javascript file containing one or more calls to <code>$.formUtils.addValidator()</code>. The module file should either have the file extension <em>.js</em> (as an ordinary javascript file) or <em>.dev.js</em> </p> 
      <p>Using the file extension <em>.dev.js</em> will tell <code>$.formUtils.loadModules</code> to always append a timestamp to the end of the URL, so that the browser never caches the file. You should of course never use <em>.dev.js</em> on a production website.</p> 
      <h3>Loading your module</h3> 
      <pre class="prettyprint linenums">
&lt;script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js"&gt;&lt;/script&gt;
&lt;script src="//cdnjs.cloudflare.com/ajax/libs/jquery-form-validator/2.3.26/jquery.form-validator.min.js"&gt;&lt;/script&gt;
&lt;script&gt;
    // Load my custom module
    $.formUtils.loadModules('mymodule.dev', 'js/validation-modules/');

    // Setup validation on all forms
    $.validate();
&lt;/script&gt;
</pre> 
      <p>The first argument of <code>$.formUtils.loadModules</code> is a comma separated string with names of module files, without file extension (add .dev if the file name is for example <em>mymodule.dev.js</em>, this will ensure you that the browser never caches the javascript).</p> 
      <p>The second argument is the path where the module files are located. This argument is optional, if not given the module files has to be located in the same directory as the core modules shipped together with this jquery plugin.</p>
