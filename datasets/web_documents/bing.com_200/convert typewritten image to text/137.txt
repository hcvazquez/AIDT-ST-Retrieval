              <p>I have had success with the BSD-licensed <a href="https://launchpad.net/cuneiform-linux">Linux port of Cuneiform</a> OCR system. </p> 
              <p>No binary packages seem to be available, so you need to build it from source. Be sure to have the ImageMagick C++ libraries installed to have support for essentially any input image format (otherwise it will only accept BMP).</p> 
              <p>While it appears to be essentially undocumented apart from a brief README file, I've found the OCR results quite good. The nice thing about it is that it can output position information for the OCR text in <a href="http://code.google.com/p/hocr-tools/">hOCR</a> format, so that it becomes possible to put the text back in in the correct position in a hidden layer of a PDF file. This way you can create "searchable" PDFs from which you can copy text. </p> 
              <p>I have used <a href="http://www.exactcode.de/site/open_source/exactimage/hocr2pdf/">hocr2pdf</a> to recreate PDFs out of the original image-only PDFs and OCR results. Sadly, the program does not appear to support creating multi-page PDFs, so you might have to create a script to handle them:</p> 
              <pre><code>#!/bin/bash
# Run OCR on a multi-page PDF file and create a new pdf with the
# extracted text in hidden layer. Requires cuneiform, hocr2pdf, gs.
# Usage: ./dwim.sh input.pdf output.pdf

set -e

input="$1"
output="$2"

tmpdir="$(mktemp -d)"

# extract images of the pages (note: resolution hard-coded)
gs -SDEVICE=tiffg4 -r300x300 -sOutputFile="$tmpdir/page-%04d.tiff" -dNOPAUSE -dBATCH -- "$input"

# OCR each page individually and convert into PDF
for page in "$tmpdir"/page-*.tiff
do
    base="${page%.tiff}"
    cuneiform -f hocr -o "$base.html" "$page"
    hocr2pdf -i "$page" -o "$base.pdf" &lt; "$base.html"
done

# combine the pages into one PDF
gs -q -dNOPAUSE -dBATCH -sDEVICE=pdfwrite -sOutputFile="$output" "$tmpdir"/page-*.pdf

rm -rf -- "$tmpdir"
</code></pre> 
              <p>Please note that the above script is very rudimentary. For example, it does not retain any PDF metadata.</p>

              <p>See if <a href="http://en.wikipedia.org/wiki/Pdftotext">pdftotext</a> will work for you. If it's not on your machine, you'll have to install the poppler-utils package</p> 
              <pre><code>sudo apt-get install poppler-utils 
</code></pre> 
              <p>You might also find the <a href="http://en.wikipedia.org/wiki/Pdftk">pdf toolkit</a> of use. </p> 
              <p>A full list of pdf software <a href="http://en.wikipedia.org/wiki/List_of_PDF_software">here</a> on wikipedia.</p> 
              <p><strong>Edit:</strong> Since you <em>do need</em> OCR capabilities, I think you'll have to try a different tack. (i.e I couldn't find a linux pdf2text converter that does OCR).</p> 
              <ul> 
               <li>Convert the pdf to an image</li> 
               <li>Scan the image to text using OCR tools</li> 
              </ul> 
              <p><strong>Convert pdf to image</strong></p> 
              <ul> 
               <li><p>gs: The <a href="http://www.linuxforums.org/forum/art-imaging-linux/58248-pdf-tiff-any-other-image-type.html">below command</a> should convert multipage pdf to individual tiff files.</p> 
                <blockquote> 
                 <p>gs -SDEVICE=tiffg4 -r600x600 -sPAPERSIZE=letter -sOutputFile=filename_%04d.tif -dNOPAUSE -dBATCH -- filename</p> 
                </blockquote></li> 
               <li><p><a href="http://www.cyberciti.biz/faq/howto-convert-a-pdf-file-to-an-image/">ImageMagik utilities</a>: There are other questions on the SuperUser site about using ImageMagik that you might use to help you do the conversion.</p> 
                <blockquote> 
                 <p>convert foo.pdf foo.png</p> 
                </blockquote></li> 
              </ul> 
              <p><strong>Convert image to text with OCR</strong></p> 
              <ul> 
               <li><a href="http://jocr.sourceforge.net/">GOCR</a>: <a href="http://en.wikipedia.org/wiki/GOCR">Wikipedia page</a></li> 
               <li><a href="http://www.gnu.org/software/ocrad/">Ocrad</a>: <a href="http://en.wikipedia.org/wiki/Ocrad">Wikipedia page</a></li> 
               <li><a href="http://code.google.com/p/ocropus/">ocropus</a>: <a href="http://en.wikipedia.org/wiki/OCRopus">Wikipedia page</a></li> 
               <li><a href="http://code.google.com/p/tesseract-ocr/">tesseract-ocr</a>: <a href="http://en.wikipedia.org/wiki/Tesseract_%28software%29">Wikipedia page</a></li> 
              </ul> 
              <p>Taken from the Wikipedia's <a href="http://en.wikipedia.org/wiki/Optical_character_recognition">list of OCR software</a></p>

              <p><strong>Update</strong><br> 1. To try it out, upload a &lt;2MB pdf to google docs from a web browser.<br> 2. Right click on the uploaded document and click "Open with Google Docs".<br> ...Google Docs will convert to text and output to a new file with same name but Google Docs type in same folder.</p>

              <p><a href="https://rubygems.org/gems/pdfbeads" rel="nofollow noreferrer" title="PDFBeads">PDFBeads</a> works well for me. This thread “<a href="http://www.diybookscanner.org/forum/viewtopic.php?f=3&amp;t=683" rel="nofollow noreferrer" title="Convert Scanned Images to a Single PDF File">Convert Scanned Images to a Single PDF File</a>” got me up and running. For a b&amp;w book scan, you need to:</p> 
              <ol> 
               <li>Create an image for every page of the PDF; either of the gs examples above should work</li> 
               <li>Generate hOCR output for each page; I used tesseract (but note that Cuneiform seems to work better).</li> 
               <li>Move the images and the hOCR files to a new folder; the filenames must correspond, so file001.tif needs file001.html, file002.tif file002.html, etc.</li> 
               <li><p>In the new folder, run </p> <pre><code>pdfbeads * &gt; ../Output.pdf
</code></pre></li> 
              </ol> 
              <p>This will put the collated, OCR'd PDF in the parent directory.</p>

              <p>another script using tesseract :</p> 
              <pre><code>#!/bin/bash
# Run OCR on a multi-page PDF file and create a txt with the
# extracted text in hidden layer. Requires tesseract, gs.
# Usage: ./pdf2ocr.sh input.pdf output.txt

set -e

input="$1"
output="$2"

tmpdir="$(mktemp -d)"

# extract images of the pages (note: resolution hard-coded)
gs -SDEVICE=tiff24nc -r300x300 -sOutputFile="$tmpdir/page-%04d.tiff" -dNOPAUSE -dBATCH -- "$input"

# OCR each page individually and convert into PDF
for page in "$tmpdir"/page-*.tiff
do
    base="${page%.tiff}"
    tesseract "$base.tiff" $base
done

# combine the pages into one txt
cat "$tmpdir"/page-*.txt &gt; $output

rm -rf -- "$tmpdir"
</code></pre>
