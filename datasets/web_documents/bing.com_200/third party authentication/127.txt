       <h1 itemprop="name" class="devsite-page-title"> Get API Key </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p> To use the Google Maps JavaScript API, you must register your app project on the Google API Console and get a Google API key which you can add to your app. </p> 
        <h2>Quick guide to getting a key</h2> 
        <h3>Step 1: Get an API Key from the Google API Console</h3> 
        <p> Click the button below, which guides you through the process of registering a project in the Google API Console, activates the Google Maps JavaScript API and any related services automatically, and generates a generic, unrestricted API key. </p> 
        <a class="devsite-api-getstarted-widget gc-analytics-event button button-blue" data-category="getKey" data-action="buttonClick" data-label="body" data-api-id="maps_backend" data-henhouse-link-docs="/maps/documentation/javascript/tutorial" data-henhouse-link-support="/maps/documentation/javascript/support" data-henhouse-platforms-list="WEB_BROWSER" data-henhouse-extra-api-ids="geocoding_backend,directions_backend,distance_matrix_backend,elevation_backend,places_backend">Get a Key</a> 
        <p><strong>Notes:</strong></p> 
        <ul class="roman"> 
         <li> <strong>Tip:</strong> During development and testing, you can register a project for testing purposes in the Google API Console and use a generic, unrestricted API key. When you are ready to move your app or website into production, register a separate project for production, create a browser-restricted API key, and add the key to your application. </li> 
         <li> <strong>Premium Plan customers</strong>: For production-ready apps, you must use a browser-restricted API key that is set up in the Google Maps APIs Premium Plan project created for you when you purchased the Premium Plan. Alternatively, you can <a href="#client-id">use a client ID</a> in combination with URL registration (instead of an API key). </li> 
         <li> For more information, see the detailed guides below for <a href="#standard-auth">Standard Plan</a> and <a href="#premium-auth">Premium Plan</a> customers. </li> 
        </ul> 
        <h3>Step 2: Add the API key to your application</h3> 
        <p> When loading the Google Maps JavaScript API, substitute <code>YOUR_API_KEY</code> in the code below with the API key you got from the previous step. </p> 
        <pre class="prettyprint notranslate">
  &lt;script async defer src="https://maps.googleapis.com/maps/api/js?key=<var class="apiparam replaceable-credential">YOUR_API_KEY</var>&amp;callback=initMap"
  type="text/javascript"&gt;&lt;/script&gt;</pre> 
        <h2 id="key">More about API keys</h2> 
        <ul> 
         <li> The API key allows you to monitor your application's API usage in the Google API Console. See <a href="https://support.google.com/googleapi/#topic=7013279"> Google API Console help</a> for more information. </li> 
         <li> If you are a Standard Plan customer, with an API key you have access to generous <a href="https://developers.google.com/maps/documentation/javascript/usage"> free daily quota</a>, as well as the option to increase your daily quota by <a href="https://developers.google.com/maps/documentation/javascript/usage#increase_quota">enabling pay-as-you-go billing</a>. </li> 
         <li> If you are a Premium Plan customer, you must use an API key or your client ID to access all the custom features and benefits of your Premium Plan. </li> 
         <li> Registering for an API key ensures that Google can contact you about your application if necessary.</li> 
        </ul> 
        <h2 id="standard-auth">Detailed guide for users of the standard Google Maps JavaScript API</h2> 
        <p> Follow these steps to get an API key: </p> 
        <ol> 
         <li>Go to the <a class="gc-analytics-event" data-category="getKey" data-action="linkClick" data-label="body" href="https://console.developers.google.com/flows/enableapi?apiid=maps_backend,geocoding_backend,directions_backend,distance_matrix_backend,elevation_backend,places_backend&amp;reusekey=true" target="_blank"> Google API Console</a>.</li> 
         <li> Create or select a project. </li> 
         <li> Click <strong>Continue</strong> to enable the API and any related services. </li> 
         <li> On the <strong>Credentials</strong> page, get an <strong>API key</strong>. <br>Note: If you have an existing unrestricted API key, or a key with browser restrictions, you may use that key. </li> 
         <li> From the dialog displaying the API key, select <strong>Restrict key</strong> to set a browser restriction on the API key. </li> 
         <li> In the <strong>Key restriction</strong> section, select <strong>HTTP referrers (web sites)</strong>, then follow the on-screen instructions to set referrers. </li> 
         <li> (Optional) Enable billing. See <a href="https://developers.google.com/maps/documentation/javascript/usage">Usage Limits</a> for more information. </li> 
        </ol> 
        <p><br>In the Google API Console, you can also <a class="gc-analytics-event" data-category="lookUpExistingKey" data-action="linkClick" data-label="body" href="https://console.developers.google.com/project/_/apiui/credential" target="_blank"> look up an existing key</a> or <a class="gc-analytics-event" data-category="lookUpEnabledApis" data-action="linkClick" data-label="body" href="https://console.developers.google.com/project/_/apiui/enabled" target="_blank"> view a list of enabled APIs</a>. </p> 
        <p>For more information on using the Google API Console, see <a href="https://support.google.com/googleapi">API Console Help</a>.</p> 
        <h2 id="premium-auth">Detailed guide for users of the Google Maps APIs Premium Plan license</h2> 
        <p>When using the Google Maps JavaScript API with a <a href="https://developers.google.com/maps/premium/overview">Google Maps APIs Premium Plan license</a>, you must authenticate your application with either an API key or a client ID.<br> <strong>Note:</strong> If you have a <a href="#prev-license">previous Maps API for Business</a> license, you must use a client ID, not an API key.</p> 
        <h3 id="clientID-features">Choosing an authentication method for your application</h3> 
        <p> The section below provides a summary of the various tools and reports that are available to Premium Plan customers, based on the method you choose to authenticate your application. </p> 
        <ul> 
         <li><strong>Authentication using an API key</strong><br> (<strong>Note:</strong> Customers with a current Premium Plan license may use an API key, but customers holding a <a href="#prev-license">previous Maps API for Business</a> license must use a client ID.)<br> By using an API key to authenticate your applications, you can: 
          <ul> 
           <li>Manage all of your APIs in the <a href="https://console.developers.google.com/project/_/apis/enabled" target="_blank"> Google API Console</a>.</li> 
           <li>Access real-time usage data and 30 days of historical usage data for your application in the <a href="https://console.developers.google.com/project/_/apis/enabled" target="_blank"> Google API Console</a>.</li> 
           <li>View usage reports with more than 30 days of data in the <a href="https://google.secure.force.com/" target="_blank">Google Cloud Support Portal</a>.</li> 
          </ul> </li> 
         <li><strong>Authentication using a Client ID</strong><br> By using your client ID (instead of an API key) to authenticate your application, you can: 
          <ul> 
           <li>Add the <code>channel</code> parameter to requests so you can view more detailed usage reports.</li> 
           <li>View usage reports with more than 30 days of data in the <a href="https://google.secure.force.com/" target="_blank">Google Cloud Support Portal</a>.</li> 
           <li>Use Maps Analytics tools for the Maps JavaScript API.</li> 
          </ul> </li> 
        </ul> 
        <p> Get more information on <a href="https://developers.google.com/maps/premium/reports/">reports available to Premium Plan customers</a>. </p> 
        <p> <strong>Note:</strong> The information below on using an API key applies only to the Google Maps APIs Premium Plan, which became available on <b>January 6, 2016</b>. </p> 
        <div id="prev-license"></div> 
        <p class="note"><b>Have a <strong><em>previous</em></strong> Maps APIs for Work or Maps API for Business license?</b> See our <a href="https://developers.google.com/maps/premium/previous-licenses/">Maps APIs for Work Licenses guide</a>. To determine whether you have a previous license: In the <a href="https://google.secure.force.com/">Google Cloud Support Portal</a>, click <strong>Maps: Usage Report</strong> on the left. If the ID at the top of the report is in the following format, you have the new Premium Plan: <br><code>gme-[<em>company</em>] &amp; proj-[<em>number</em>] ([<em>type</em>])</code><br> Otherwise, you have a previous license. </p> 
        <h3 id="premium-key">Authenticating your application using an API key</h3> 
        <p> Follow these steps to get an API key: </p> 
        <ol> 
         <li>Go to the <a class="gc-analytics-event" data-category="getKey" data-action="premiumLinkClick" data-label="body" href="https://console.developers.google.com/flows/enableapi?apiid=maps_backend,geocoding_backend,directions_backend,distance_matrix_backend,elevation_backend,places_backend&amp;reusekey=true" target="_blank"> Google API Console</a>. </li> 
         <li>From the Project drop-down menu, select the project created for you when you purchased the Premium Plan. The project name starts with <b>Google Maps APIs for Business or Google Maps for Work or Google Maps</b>. <strong>Important:</strong> If you have a <a href="#prev-license">previous Maps API for Business</a> license, you must use a client ID, not an API key. </li> 
         <li> Click <b>Continue</b>. </li> 
         <li> On the<strong> Credentials</strong> page, get an <strong>API key</strong>. <br>Note: If you have an existing unrestricted API key, or a key with browser restrictions, you may use that key. </li> 
         <li> From the dialog displaying the API key, select <strong>Restrict key</strong> to set a browser restriction on the API key. </li> 
         <li> In the <strong>Key restriction</strong> section, select <strong>HTTP referrers (web sites)</strong>, then follow the on-screen instructions to set referrers. </li> 
        </ol> 
        <p><br>You can also <a class="gc-analytics-event" data-category="lookUpExistingKey" data-action="premiumLinkClick" data-label="body" href="https://console.developers.google.com/project/_/apiui/credential" target="_blank"> look up an existing key</a> in the Google API Console.</p> 
        <p>For more information on using the Google API Console, see <a href="https://support.google.com/googleapi">API Console Help</a>.</p> 
        <h3 id="client-id">Authenticating your application using a client ID</h3> 
        <p> Upon purchasing your <a href="https://developers.google.com/maps/premium/overview"> Google Maps APIs Premium Plan license</a>, you will receive a welcome email from Google that contains your <strong>client ID</strong>, which allows to access the <a href="#clientID-features"> various reports and tools of the Google Maps APIs Premium Plan</a>. </p> 
        <p> All client IDs begin with a <strong>gme-</strong> prefix. Below is the format of a typical client ID. </p> 
        <p> <strong>gme-[company] &amp; proj-[number] ([type])</strong> </p> 
        <p> You can authenticate your application with the Google Maps JavaScript API using a client ID in combination with <strong>URL registration</strong> (instead of an API key). </p> 
        <p> Note: This client ID is not a key, and it can only be used by URLs that you authorize. Read more about <a href="#registering-authorized-urls">registering authorized URLs</a>. </p> 
        <h3>Specifying a client ID when loading the API</h3> 
        <p> The code below shows you how to substitute YOUR_CLIENT_ID with your own client ID when loading the Google Maps JavaScript API. </p> 
        <pre class="prettyprint notranslate">
  &lt;script async defer
  src="https://maps.googleapis.com/maps/api/js?client=<var class="apiparam">YOUR_CLIENT_ID</var>&amp;v=3.29&amp;callback=initMap"&gt;&lt;/script&gt;
</pre> 
        <p> You must specify the <a href="https://developers.google.com/maps/documentation/javascript/versions">release version</a> (also referred to as the <strong>feature-stable</strong> version) or an earlier version, by appending a <code>v=3.29</code> parameter. Applications that use the experimental version are not covered under the Google Maps APIs Premium Plan SLA. </p> 
        <p class="note"><strong>Note</strong>: If you were previously using an API key for authentication and are switching to using a client ID, you must remove the <code>key</code> parameter before loading the API. <strong>The API will fail to load if both a client ID and an API key are included</strong>. </p> 
        <h2 id="key-restrictions">Restricting an API key</h2> 
        <p> Google Maps APIs are available for web browsers, Android or iOS apps, and via HTTP web services. APIs in any platform can use a generic (unrestricted) API key. You can optionally add a restriction (for example, HTTP referrer) to the API key. Once restricted, a key will only work on platforms that support that type of restriction. </p> 
        <p> <strong>Tip</strong>: Before moving your app or website to production, you should secure your API key. Keys for the Google Maps JavaScript API use the <strong>HTTP referrers (web sites)</strong> key restriction. <a href="https://developers.google.com/maps/faq#keysystem">Learn more about keys and credentials</a>. </p> 
        <p> To add web browser restrictions to an existing, generic API key, do the following: </p> 
        <ol> 
         <li>Go to the Credentials page of the <a class="gc-analytics-event" data-category="updateKeyRestrictions" data-action="linkClick" data-label="body" href="https://console.developers.google.com/project/_/apiui/credential" target="_blank"> Google API Console</a>.</li> 
         <li>Select the project that contains the API key you want to edit.</li> 
         <li>On the <strong>Credentials</strong> page, from the list of API keys, select the name of the API key to edit the details of the key.</li> 
         <li>In the <strong>Key restriction</strong> section of the page, select <strong>HTTP referrers (web sites)</strong>, follow the on-screen instructions to set referrers, then click <strong>Save</strong>.</li> 
        </ol> 
        <p> <strong>Note:</strong> file:// referers need a special representation to be added to the Key restriction. The "file:/" part should be replaced with "__file_url__" before being added to the Key restriction. For example, "file:///path/to/" should be formatted as "__file_url__//path/to/*". After enabling file:// referers, it is recommended you regularly check your usage, to make sure it matches your expectations. </p> 
        <h2 id="registering-authorized-urls">Registering authorized URLs</h2> 
        <p>To prevent a third party from using your client ID on their own website, the use of your client ID is restricted to a list of URLs that you specifically authorize.</p> 
        <p>To see the URLs you have already authorized or to authorize additional URLs:</p> 
        <ol> 
         <li>Log in to the <a href="https://google.secure.force.com/">Google Cloud Support Portal</a>.</li> 
         <li>In the left-hand menu, click <strong>Maps: Manage Client ID</strong>. </li> 
        </ol> 
        <p>You can add up to 100 URLs at a time, to a total of 3000 URLs. If you require higher limits, please contact <a href="https://developers.google.com/maps/premium/support">support</a>.</p> 
        <p>The following considerations apply regarding URLs that are authorized:</p> 
        <dl> 
         <dt>
          The domain name or IP address does not have to be publicly accessible.
         </dt> 
         <dd>
          For example, 
          <code>http://myintranet</code> and 
          <code>http://192.168.1.1</code> are valid entries.
         </dd> 
         <dt>
          All subdomains of a specified domain are also authorized.
         </dt> 
         <dd>
          For example, if 
          <code>http://example.com</code> is authorized, then 
          <code>http://www.example.com</code> is also authorized. The reverse is not true: if 
          <code>http://www.example.com</code> is authorized, 
          <code>http://example.com</code> is not necessarily authorized.
         </dd> 
         <dt>
          All subpaths of an authorized path are also authorized.
         </dt> 
         <dd>
          For example, if 
          <code>http://example.com</code> is authorized, then 
          <code>http://example.com/foo</code> is also authorized. In addition, because subdomains of a specified domain are also authorized, 
          <code>http://sub.example.com/bar</code> is authorized. 
         </dd> 
         <dt>
          Paths are case sensitive.
         </dt> 
         <dd>
          For example, 
          <code>http://www.example.com/ThisPath/</code> is not the same as 
          <code>http://www.example.com/thispath/</code>. 
         </dd> 
         <dt>
          You may restrict valid URLs to those using certain ports.
         </dt> 
         <dd>
          For example, if 
          <code>http://example.com:8080/foo</code> is specified, that doesn't authorize 
          <code>http://example.com</code>.
         </dd> 
         <dt>
          HTTP and HTTPS protocols are considered different URLs.
         </dt> 
         <dd>
          For example, if 
          <code>https://example.com</code> is authorized, 
          <code>http://example.com</code> is not necessarily authorized. If you'd like to authorize both at once, you may add a domain without using a protocol: 
          <code>example.com</code>/
         </dd> 
        </dl> 
        <p>All the rules above are applied to each authorization, so you should take care to plan your authorizations carefully. For example, because all subpaths of a specified path are authorized, and all subdomains, you may end up authorizing pages that you didn't intend to. For example:</p> 
        <p><code>http://example.com/</code> also authorizes <code>http://sub.example.com/path</code>.</p> 
        <p>For more information, see <a href="https://developers.google.com/maps/premium/troubleshooting-auth/">Troubleshooting Google Maps APIs Premium Plan Authorization</a>.</p>
