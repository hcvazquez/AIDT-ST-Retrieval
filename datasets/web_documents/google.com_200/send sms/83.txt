             <h1 id="auth0-js-v8-reference" class="anchor-heading"><span id="1" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Auth0.js v8 Reference</h1> 
             <p>Auth0.js is a client-side library for Auth0. Using auth0.js in your web apps makes it easier to do authentication and authorization with Auth0 in your web apps.</p> 
             <h2 id="ready-to-go-example" class="anchor-heading"><span id="2" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Ready-to-go example</h2> 
             <p>The <a href="https://github.com/auth0/auth0.js/tree/master/example">example directory</a> of the auth0.js library is a ready-to-go app that can help you to quickly and easily try out auth0.js. In order to run it, follow these quick steps:</p> 
             <ol> 
              <li>If you don't have <a href="http://nodejs.org/">node</a> installed, do that now</li> 
              <li>Download dependencies by running <code>npm install</code> from the root of this project</li> 
              <li>Finally, execute <code>npm start</code> from the root of this project, and then browse to your app running on the node server, presumably at <code>http://localhost:3000/example</code>.</li> 
             </ol> 
             <h2 id="setup-and-initialization" class="anchor-heading"><span id="3" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Setup and initialization</h2> 
             <p>Now, let's get started integrating auth0.js into your project. We'll cover <a href="#installation-options">methods of installation</a>, <a href="#initialization">how to initialize auth0.js</a>, <a href="#signup">signup</a>, <a href="#login">login</a>, <a href="#logout">logout</a>, and more!</p> 
             <h3 id="installation-options" class="anchor-heading"><span id="4" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Installation options</h3> 
             <p>You have a few options for using auth0.js in your project. Pick one of the below depending on your needs:</p> 
             <p>Install via <a href="https://npmjs.org">npm</a>:</p> 
             <pre><code class="language-sh">npm install auth0-js
</code></pre> 
             <p>Install via <a href="http://bower.io">bower</a>:</p> 
             <pre><code class="language-sh">bower install auth0.js
</code></pre> 
             <pre><code class="language-html">&lt;script src="bower_components/auth0.js/build/auth0.min.js"&gt;&lt;/script&gt;
</code></pre> 
             <p>Include via our CDN:</p> 
             <pre><code class="language-html">&lt;script src="https://cdn.auth0.com/js/auth0/8.9/auth0.min.js"&gt;&lt;/script&gt;
</code></pre> 
             <div class="alert alert-default alert-with-icon"> 
              <i class="icon-budicon-724 alert-icon"></i> 
              <div class="alert-content"> 
               <p>For production use, the latest patch release (for example, 8.0.0) is recommended, rather than the latest minor release indicated above.</p> 
              </div>
             </div> 
             <p>If you are using a bundler, you will want to install with <code>npm i auth0-js --production --save</code>.</p> 
             <h3 id="initialization" class="anchor-heading"><span id="5" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Initialization</h3> 
             <p>Initialize a new instance of the Auth0 client as follows:</p> 
             <pre><code class="language-html">&lt;script src="https://cdn.auth0.com/js/auth0/8.0.4/auth0.min.js"&gt;&lt;/script&gt;
&lt;script type="text/javascript"&gt;
  var webAuth = new auth0.WebAuth({
    domain:       'YOUR_AUTH0_DOMAIN',
    clientID:     'YOUR_CLIENT_ID'
  });
&lt;/script&gt;
</code></pre> 
             <h4 id="available-parameters" class="anchor-heading"><span id="6" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Available parameters</h4> 
             <p>There are two required parameters that must be passed in the <code>options</code> object when instantiating <code>webAuth</code>, and more that are optional.</p> 
             <table class="table">
              <thead> 
               <tr> 
                <th><strong>Parameter</strong></th> 
                <th><strong>Required</strong></th> 
                <th><strong>Description</strong></th> 
               </tr> 
              </thead> 
              <tbody> 
               <tr> 
                <td><code>domain</code></td> 
                <td>required</td> 
                <td>(String) Your Auth0 account domain (ex. myaccount.auth0.com)</td> 
               </tr> 
               <tr> 
                <td><code>clientID</code></td> 
                <td>required</td> 
                <td>(String) Your Auth0 client_id</td> 
               </tr> 
               <tr> 
                <td><code>redirectUri</code></td> 
                <td>optional</td> 
                <td>(String) The default <code>redirectUri</code> used. Defaults to an empty string (none).</td> 
               </tr> 
               <tr> 
                <td><code>scope</code></td> 
                <td>optional</td> 
                <td>(String) The default scope(s) used by the application. Using scopes can allow you to return specific claims for specific fields in your request. You should read our <a href="https://auth0.com/docs/scopes">documentation on scopes</a> for further details.</td> 
               </tr> 
               <tr> 
                <td><code>audience</code></td> 
                <td>optional</td> 
                <td>(String) The default audience to be used for requesting API access.</td> 
               </tr> 
               <tr> 
                <td><code>responseType</code></td>

             <h2 id="passwordless-login" class="anchor-heading"><span id="14" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Passwordless Login</h2> 
             <p>Passwordless authentication allows users to log in by receiving a one-time password via email or text message. The process will require you to start the Passwordless process, generating and dispatching a code to the user, (or a code within a link), followed by accepting their credentials via the verification method. That could happen in the form of a login screen which asks for their (email or phone number) and the code you just sent them. It could also be implemented in the form of a Passwordless link instead of a code sent to the user. They would simply click the link in their email or text and it would hit your endpoint and verify this data automatically using the same verification method (just without manual entry of a code by the user).</p> 
             <p>In order to use Passwordless, you will want to initialize Auth0.js with a <code>redirectUri</code> and to set the <code>responseType: 'token'</code>.</p> 
             <pre><code class="language-js">var webAuth = new auth0.WebAuth({
  clientID: 'YOUR_CLIENT_ID',
  domain: 'YOUR_AUTH0_DOMAIN',
  redirectUri: 'http://example.com',
  responseType: 'token'
});
</code></pre> 
             <h3 id="start-passwordless" class="anchor-heading"><span id="15" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Start Passwordless</h3> 
             <p>The first step in Passwordless authentication with Auth0.js is the <code>passwordlessStart</code> method, which has several parameters which can be passed within its <code>options</code> object:</p>

               </tr> 
               <tr> 
                <td><code>send</code></td> 
                <td>required</td> 
                <td>(String) Value must be either <code>code</code> or <code>link</code>. If <code>null</code>, a link will be sent.</td> 
               </tr> 
               <tr> 
                <td><code>phoneNumber</code></td>

             <h2 id="extract-the-authresult-and-get-user-info" class="anchor-heading"><span id="17" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Extract the authResult and Get User Info</h2> 
             <p>After authentication occurs, you can use the <code>parseHash</code> method to parse a URL hash fragment when the user is redirected back to your application in order to extract the result of an Auth0 authentication response. You may choose to handle this in a callback page that will then redirect to your main application, or in-page, as the situation dictates.</p> 
             <p>The <code>parseHash</code> method takes an <code>options</code> object that contains the following parameters:</p> 
             <table class="table">
              <thead> 
               <tr> 
                <th><strong>Parameter</strong></th> 
                <th><strong>Required</strong></th> 
                <th><strong>Description</strong></th> 
               </tr> 
              </thead> 
              <tbody> 
               <tr> 
                <td><code>state</code></td> 
                <td>optional</td> 
                <td>(String) An opaque value the client adds to the initial request that Auth0 includes when redirecting back to the client. This value must be used by the client to prevent CSRF attacks.</td> 
               </tr> 
               <tr> 
                <td><code>nonce</code></td> 
                <td>optional</td> 
                <td>(String) Used to verify the <code>id_token</code></td> 
               </tr> 
               <tr> 
                <td><code>hash</code></td> 
                <td>optional</td> 
                <td>(String) The URL hash (if not provided, <code>window.location.hash</code> will be used by default)</td> 
               </tr> 
              </tbody> 
             </table> 
             <div class="alert alert-default alert-with-icon"> 
              <i class="icon-budicon-724 alert-icon"></i> 
              <div class="alert-content"> 
               <p>This method requires that your tokens are signed with RS256 rather than HS256. For more information about this, check the <a href="https://auth0.com/docs/libraries/auth0js/migration-guide#the-parsehash-method">Auth0.js v8 Migration Guide</a>.</p> 
              </div>
             </div> 
             <p>The contents of the authResult object returned by <code>parseHash</code> depend upon which authentication parameters were used. It can include:</p> 
             <table class="table">
              <thead> 
               <tr> 
                <th><strong>Item</strong></th> 
                <th><strong>Description</strong></th> 
               </tr> 
              </thead> 
              <tbody> 
               <tr> 
                <td><code>accessToken</code></td> 
                <td>An access token for the API, specified by the <code>audience</code></td> 
               </tr> 
               <tr> 
                <td><code>expiresIn</code></td> 
                <td>A string containing the expiration time (in seconds) of the <code>accessToken</code></td> 
               </tr> 
               <tr> 
                <td><code>idToken</code></td> 
                <td>An id token JWT containing user profile information</td> 
               </tr> 
              </tbody> 
             </table> 
             <pre><code class="language-js">webAuth.parseHash({ hash: window.location.hash }, function(err, authResult) {
  if (err) {
    return console.log(err);
  }

  webAuth.client.userInfo(authResult.accessToken, function(err, user) {
    // Now you have the user's information
  });
});
</code></pre> 
             <p>As shown above, the <code>client.userInfo</code> method can be called passing the returned <code>accessToken</code>. It will make a request to the <code>/userinfo</code> endpoint and return the <code>user</code> object, which contains the user's information, formatted similarly to the below example.</p> 
             <pre><code class="language-json">{

    "email": "test@example.com",

    "name": "tester9@example.com",
    "picture": "https://gravatar.com/avatar/example.png",
    "user_id": "auth0|123456789012345678901234",
    "nickname": "tester9",
    "identities": [
        {
            "user_id": "123456789012345678901234",
            "provider": "auth0",
            "connection": "Username-Password-Authentication",
            "isSocial": "false"
        }
    ],
    "created_at": "2017-01-20T20:06:05.008Z",
    "sub": "auth0|123456789012345678901234"
}
</code></pre> 
             <p>You can now do something else with this information as your application needs, such as acquire the user's entire set of profile information with the Management API, as described below.</p> 
             <h2 id="using-nonce" class="anchor-heading"><span id="18" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Using <code>nonce</code></h2> 
             <p>By default (and if <code>responseType</code> contains <code>id_token</code>), <code>auth0.js</code> will generate a random <code>nonce</code> when you call <code>webAuth.authorize</code>, store it in local storage, and pull it out in <code>webAuth.parseHash</code>. The default behavior should work in most cases, but some use cases may require a developer to control the <code>nonce</code>. If you want to use a developer generated <code>nonce</code>, then you must provide it as an option to both <code>webAuth.authorize</code> and <code>webAuth.parseHash</code>.</p> 
             <pre><code class="language-js">webAuth.authorize({nonce: '1234', responseType: 'token id_token'});
webAuth.parseHash({nonce: '1234'}, callback);
</code></pre> 
             <p>If you're calling <code>webAuth.renewAuth</code> instead of <code>webAuth.authorize</code>, then you only have to specify your custom <code>nonce</code> as an option to <code>renewAuth</code>:</p> 
             <pre><code class="language-js">webAuth.renewAuth({
  audience: 'https://example.com/api/v2',
  scope: 'openid read:something write:otherthing',
  responseType: 'token id_token',
  nonce: '1234',
  usePostMessage: true
}, function (err, authResult) {
    ...
});
</code></pre> 
             <p>The <code>webAuth.renewAuth</code> method will automatically verify that the returned <code>id_token</code>'s <code>nonce</code> claim is the same as the option.</p>

        webAuth.signup({
            connection: 'Username-Password-Authentication',
            email: $('.signup-email').val(),
            password: $('.signup-password').val()
        }, function (err) {
            if (err) return alert('Something went wrong: ' + err.message);
            return alert('success signup without login!')
        });
    });
&lt;/script&gt;
</code></pre> 
             <h2 id="using-renewauth-to-acquire-new-tokens" class="anchor-heading"><span id="21" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Using renewAuth to acquire new tokens</h2> 
             <p>The <code>renewAuth</code> method allows you to acquire a new token from Auth0 for a user who is already authenticated against the <a href="https://auth0.com/docs/hosted-pages/login">hosted login page</a> for your domain. The method accepts any valid OAuth2 parameters that would normally be sent to <code>authorize</code>.</p> 
             <pre><code class="language-js">webAuth.renewAuth({
  audience: 'https://example.com/api/v2',
  scope: 'read:something write:otherthing',
  redirectUri: 'https://example.com/auth/silent-callback',
  usePostMessage: true
}, function (err, authResult) {
  // err if automatic parseHash fails
  ...
});
</code></pre> 
             <p>The actual redirect to <code>/authorize</code> happens inside an iframe, so it will not reload your application or redirect away from it. However, it is strongly recommended to have a dedicated callback page for silent authentication in order to avoid the delay of loading your entire application again inside an iframe.</p> 
             <p>This callback page should simply pass the local URL hash to the parent document via postMessage. The <code>webAuth.renewAuth</code> method will receive the hash string and automatically perform a <code>webAuth.parseHash</code>, passing the <code>err</code> or <code>authResult</code> to the callback function. The callback page should be something like the following one:</p> 
             <pre><code class="language-html">&lt;!DOCTYPE html&gt;
&lt;html&gt;
  &lt;head&gt;
    &lt;script type="text/javascript"&gt;
      parent.postMessage(window.location.hash, "https://example.com/");
    &lt;/script&gt;
  &lt;/head&gt;
  &lt;body&gt;&lt;/body&gt;
&lt;/html&gt;
</code></pre> 
             <p>Remember to add the URL of the silent authentication callback page that you create to the <strong>Allowed Callback URLs</strong> list of your Auth0 client in the <a href="https://manage.auth0.com">Auth0 Dashboard</a> under your client's <em>Settings</em>.</p> 
             <div class="alert alert-warning alert-with-icon"> 
              <i class="icon-budicon-354 alert-icon"></i> 
              <div class="alert-content"> 
               <p>If the connection is a social connection and you are using Auth0 dev keys, the <code>renewAuth</code> call will always return <code>login_required</code>.</p> 
              </div>
             </div> 
             <h2 id="password-reset-requests" class="anchor-heading"><span id="22" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Password reset requests</h2> 
             <p>If attempting to set up a password reset functionality, you'll use the <code>changePassword</code> method and pass in an "options" object, with a "connection" parameter and an "email" parameter.</p> 
             <pre><code class="language-js">  $('.change_password').click(function () {
    webAuth.changePassword({
      connection: 'db-conn',
      email:   'foo@bar.com'
    }, function (err, resp) {
      if(err){
        console.log(err.message);
      }else{
        console.log(resp);
      }
    });
  });
</code></pre> 
             <p>The user will then receive an email which will contain a link that they can follow to reset their password.</p> 
             <h2 id="user-management" class="anchor-heading"><span id="23" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>User management</h2> 
             <p>The Management API provides functionality that allows you to link and unlink separate user accounts from different providers, tying them to a single profile (Read more about <a href="https://auth0.com/docs/link-accounts">Linking Accounts</a> with Auth0). It also allows you to update user metadata.</p> 
             <p>To get started, create a new <code>auth0.Management</code> instance by passing it the account's Auth0 domain, and the token for the primary identity. In the case of linking users, this primary identity is the user profile that you want to "keep" the data for, and which you plan to link other identities to.</p> 
             <pre><code class="language-js">var auth0Manage = new auth0.Management({
  domain: 'YOUR_AUTH0_DOMAIN',
  token: "YOUR_PRIMARY_IDENTITY_TOKEN"
});
</code></pre> 
             <h3 id="getting-the-user-profile" class="anchor-heading"><span id="24" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Getting the user profile</h3> 
             <p>In order to get the user profile data, use the <code>getUser()</code> method, with the <code>userId</code> and a callback as parameters. The method returns the user profile. Note that the <code>userID</code> required here will be the same one fetched from the <code>client.userInfo</code> method.</p> 
             <pre><code class="language-js">auth0Manage.getUser(userId, cb);
</code></pre> 
             <h3 id="updating-the-user-profile" class="anchor-heading"><span id="25" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Updating the user profile</h3> 
             <p>When updating user metadata, you will need to first create a <code>userMetadata</code> object, and then call the <code>patchUserMetadata</code> method, passing it the user id and the <code>userMetadata</code> object you created. The values in this object will overwrite existing values with the same key, or add new ones for those that don't yet exist in the user metadata. Visit the <a href="https://auth0.com/docs/metadata">User Metadata</a> documentation for more details on user metadata.</p> 
             <pre><code class="language-js">auth0Manage.patchUserMetadata(userId, userMetadata, cb);
</code></pre> 
             <h3 id="linking-users" class="anchor-heading"><span id="26" class="anchor" href="#' + anchor + '"><i class="icon icon-budicon-345"></i></span>Linking users</h3> 
             <p>Linking user accounts will allow a user to authenticate from any of their accounts and no matter which one they use, still pull up the same profile upon login. Auth0 treats all of these accounts as separate profiles by default, so if you wish a user's accounts to be linked, this is the way to go.</p> 
             <p>The <code>linkUser</code> method accepts two parameters, the primary user id and the secondary user token (the token obtained after login with this identity). The user id in question is the unique identifier for this user account. If the id is in the format <code>facebook|1234567890</code>, the id required is the portion after the delimiting pipe. Visit the <a href="https://auth0.com/docs/link-accounts">Linking Accounts</a> documentation for more details on linking accounts.</p> 
             <pre><code class="language-js">auth0Manage.linkUser(userId, secondaryUserToken, cb);
</code></pre> 
             <div class="alert alert-default alert-with-icon"> 
              <i class="icon-budicon-724 alert-icon"></i> 
              <div class="alert-content"> 
               <p>Note that when accounts are linked, the secondary account's metadata is <strong>not</strong> merged with the primary account's metadata, and if they are ever unlinked, the secondary account will likewise not retain the primary account's metadata when it becomes separate again.</p> 
              </div>
