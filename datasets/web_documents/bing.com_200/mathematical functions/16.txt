      <p><b>Java Platform, Standard Edition</b> or <b>Java SE</b> is a <a href="/wiki/Computing_platform" title="Computing platform">computing platform</a> for development and deployment of <a href="/wiki/Porting" title="Porting">portable code</a> for <a href="/wiki/Desktop_computer" title="Desktop computer">desktop</a> and <a href="/wiki/Server_(computing)" title="Server (computing)">server environments</a>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> Java SE was formerly known as <b>Java 2 Platform, Standard Edition</b> or <b>J2SE</b>.</p> 
      <p>The platform uses <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> programming language and is part of the <a href="/wiki/Java_(software_platform)" title="Java (software platform)">Java software-platform</a> family. Java SE defines a range of general-purpose <a href="/wiki/API" class="mw-redirect" title="API">APIs</a>—such as <a href="/wiki/List_of_Java_APIs" title="List of Java APIs">Java APIs</a> for the <a href="/wiki/Java_Class_Library" title="Java Class Library">Java Class Library</a>—and also includes the <a href="/wiki/Java_Language_Specification" class="mw-redirect" title="Java Language Specification">Java Language Specification</a> and the <a href="/wiki/Java_Virtual_Machine_Specification" class="mw-redirect" title="Java Virtual Machine Specification">Java Virtual Machine Specification</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> One of the most well-known<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup> implementations of Java SE is <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle Corporation</a>'s <a href="/wiki/Java_Development_Kit" title="Java Development Kit">Java Development Kit</a> (JDK).<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></p>

      <h2><span id="Nomenclature,_standards_and_specifications"></span><span class="mw-headline" id="Nomenclature.2C_standards_and_specifications">Nomenclature, standards and specifications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=1" title="Edit section: Nomenclature, standards and specifications">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The platform was known as <i>Java 2 Platform, Standard Edition</i> or <i>J2SE</i> from version 1.2, until the name was changed to <i>Java Platform, Standard Edition</i> or <i>Java SE</i> in version 1.5. The "SE" is used to distinguish the base platform from the Enterprise Edition (<a href="/wiki/Java_EE" class="mw-redirect" title="Java EE">Java EE</a>) and Micro Edition (<a href="/wiki/Java_ME" class="mw-redirect" title="Java ME">Java ME</a>) platforms. The "2" was originally intended to emphasize the major changes introduced in version 1.2, but was removed in version 1.6. The naming convention has been changed several times over the <a href="/wiki/Java_version_history" title="Java version history">Java version history</a>. Starting with J2SE 1.4 (Merlin), Java SE has been developed under the <a href="/wiki/Java_Community_Process" title="Java Community Process">Java Community Process</a>, which produces descriptions of proposed and final specifications for the Java platform called <a href="/wiki/Java_Specification_Request" class="mw-redirect" title="Java Specification Request">Java Specification Requests (JSR)</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> JSR 59 was the umbrella specification for J2SE 1.4 and JSR 176 specified J2SE 5.0 (Tiger). Java SE 6 (Mustang) was released under JSR 270.</p> 
      <p><a href="/wiki/Java_Platform,_Enterprise_Edition" title="Java Platform, Enterprise Edition">Java Platform, Enterprise Edition</a> (Java EE) is a related specification that includes all the <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">classes</a> in Java SE, plus a number that are more useful to programs that run on <a href="/wiki/Server_(computing)" title="Server (computing)">servers</a> as opposed to <a href="/wiki/Workstation" title="Workstation">workstations</a>.</p> 
      <p><a href="/wiki/Java_Platform,_Micro_Edition" title="Java Platform, Micro Edition">Java Platform, Micro Edition</a> (Java ME) is a related specification intended to provide a certified collection of Java APIs for the development of software for small, resource-constrained devices such as <a href="/wiki/Cell_phone" class="mw-redirect" title="Cell phone">cell phones</a>, <a href="/wiki/Portable_digital_assistant" class="mw-redirect" title="Portable digital assistant">PDAs</a> and <a href="/wiki/Set-top_box" title="Set-top box">set-top boxes</a>.</p> 
      <p>The <a href="/wiki/Java_Virtual_Machine#Execution_environment" class="mw-redirect" title="Java Virtual Machine">Java Runtime Environment (JRE)</a> and <a href="/wiki/Java_Development_Kit" title="Java Development Kit">Java Development Kit (JDK)</a> are the actual files downloaded and installed on a computer to run or develop Java programs, respectively.</p> 
      <h2><span class="mw-headline" id="General_purpose_packages">General purpose packages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=2" title="Edit section: General purpose packages">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="java.lang">java.lang</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=3" title="Edit section: java.lang">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <a href="/wiki/Java_package" title="Java package">Java package</a> <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/package-summary.html">java.lang</a></code> contains fundamental classes and <a href="/wiki/Interface_(Java)" title="Interface (Java)">interfaces</a> closely tied to the language and <a href="/wiki/Run_time_system" class="mw-redirect" title="Run time system">runtime</a> system. This includes the root classes that form the <a href="/wiki/Class_hierarchy" title="Class hierarchy">class hierarchy</a>, types tied to the language definition, basic <a href="/wiki/Exception_handling" title="Exception handling">exceptions</a>, math functions, <a href="/wiki/Thread_(computer_science)" class="mw-redirect" title="Thread (computer science)">threading</a>, security functions, as well as some information on the underlying native system. This package contains 22 of 32 <code>Error</code> classes provided in JDK 6.</p> 
      <p>The main classes and interfaces in <code>java.lang</code> are:</p> 
      <ul> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Object.html">Object</a></code> – the class that is the root of every class hierarchy.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Enum.html">Enum</a></code> – the base class for <a href="/wiki/Enumerated_type" title="Enumerated type">enumeration classes</a> (as of J2SE 5.0).</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html">Class</a></code> – the class that is the root of the Java <a href="/wiki/Reflection_(computer_science)" class="mw-redirect" title="Reflection (computer science)">reflection</a> system.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Throwable.html">Throwable</a></code> – the class that is the base class of the exception class hierarchy.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Error.html">Error</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Exception.html">Exception</a></code>, and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/RuntimeException.html">RuntimeException</a></code> – the base classes for each exception type.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Thread.html">Thread</a></code> – the class that allows operations on threads.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/String.html">String</a></code> – the class for <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a> and <a href="/wiki/String_literal" title="String literal">string literals</a>.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/StringBuffer.html">StringBuffer</a></code> and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/StringBuilder.html">StringBuilder</a></code> – classes for performing <a href="/wiki/StringBuffer_and_StringBuilder" class="mw-redirect" title="StringBuffer and StringBuilder">string manipulation</a> (<code>StringBuilder</code> as of J2SE 5.0).</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Comparable.html">Comparable</a></code> – the interface that allows generic comparison and ordering of objects (as of J2SE 1.2).</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Iterable.html">Iterable</a></code> – the interface that allows generic iteration using the <a href="/wiki/Foreach#Java" class="mw-redirect" title="Foreach">enhanced <code>for</code> loop</a> (as of J2SE 5.0).</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ClassLoader.html">ClassLoader</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Process.html">Process</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Runtime.html">Runtime</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/SecurityManager.html">SecurityManager</a></code>, and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/System.html">System</a></code> – classes that provide "system operations" that manage the <a href="/wiki/Dynamically_loaded_library" class="mw-redirect" title="Dynamically loaded library">dynamic loading</a> of classes, creation of external <a href="/wiki/Process_(computing)" title="Process (computing)">processes</a>, host environment inquiries such as the time of day, and enforcement of <a href="/wiki/Security_policy" title="Security policy">security policies</a>.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Math.html">Math</a></code> and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/StrictMath.html">StrictMath</a></code> – classes that provide basic math functions such as <a href="/wiki/Sine" title="Sine">sine</a>, <a href="/wiki/Cosine" class="mw-redirect" title="Cosine">cosine</a>, and <a href="/wiki/Square_root" title="Square root">square root</a> (<code>StrictMath</code> as of J2SE 1.3).</li> 
       <li>The <a href="/wiki/Primitive_wrapper_class" title="Primitive wrapper class">primitive wrapper classes</a> that <a href="/wiki/Encapsulation_(computer_science)" class="mw-redirect" title="Encapsulation (computer science)">encapsulate</a> <a href="/wiki/Primitive_type" class="mw-redirect" title="Primitive type">primitive types</a> as <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a>.</li> 
       <li>The basic exception classes thrown for language-level and other common exceptions.</li> 
      </ul> 
      <p>Classes in <code>java.lang</code> are automatically imported into every <a href="/wiki/Source_file" class="mw-redirect" title="Source file">source file</a>.</p> 
      <h4><span class="mw-headline" id="java.lang.ref">java.lang.ref</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=4" title="Edit section: java.lang.ref">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/package-summary.html">java.lang.ref</a></code> package provides more flexible types of <a href="/wiki/Reference_(computer_science)" title="Reference (computer science)">references</a> than are otherwise available, permitting limited interaction between the application and the <a href="/wiki/Java_Virtual_Machine" class="mw-redirect" title="Java Virtual Machine">Java Virtual Machine</a> (JVM) <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collector</a>. It is an important package, central enough to the language for the language designers to give it a name that starts with "java.lang", but it is somewhat special-purpose and not used by a lot of developers. This package was added in J2SE 1.2.</p> 
      <p>Java has an expressive system of references and allows for special behavior for garbage collection. A normal reference in Java is known as a "strong reference." The <code>java.lang.ref</code> package defines three other types of references—soft, <a href="/wiki/Weak_reference" title="Weak reference">weak</a>, and phantom references. Each type of reference is designed for a specific use.</p> 
      <ul> 
       <li>A <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/SoftReference.html">SoftReference</a></code> can be used to implement a <a href="/wiki/Cache_(computing)" title="Cache (computing)">cache</a>. An object that is not reachable by a strong reference (that is, not strongly reachable), but is referenced by a soft reference is called "softly reachable." A softly reachable object may be garbage collected at the discretion of the garbage collector. This generally means that softly reachable objects are only garbage collected when free memory is low—but again, this is at the garbage collector's discretion. Semantically, a soft reference means, "Keep this object when nothing else references it, unless the memory is needed."</li> 
       <li>A <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/WeakReference.html">WeakReference</a></code> is used to implement weak maps. An object that is not strongly or softly reachable, but is referenced by a weak reference is called "<a href="/wiki/Weakly_reachable" class="mw-redirect" title="Weakly reachable">weakly reachable</a>". A weakly reachable object is garbage collected in the next collection cycle. This behavior is used in the class <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/util/WeakHashMap.html">java.util.WeakHashMap</a></code>. A weak map allows the programmer to put key/value pairs in the map and not worry about the objects taking up memory when the key is no longer reachable anywhere else. Another possible application of weak references is the <a href="/wiki/String_intern_pool" class="mw-redirect" title="String intern pool">string intern pool</a>. Semantically, a weak reference means "get rid of this object when nothing else references it at the next garbage collection."</li> 
       <li>A <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/PhantomReference.html">PhantomReference</a></code> is used to reference objects that have been marked for garbage collection and have been <a href="/wiki/Finalizer" title="Finalizer">finalized</a>, but have not yet been reclaimed. An object that is not strongly, softly or weakly reachable, but is referenced by a phantom reference is called "phantom reachable." This allows for more flexible cleanup than is possible with the finalization mechanism alone. Semantically, a phantom reference means "this object is no longer needed and has been finalized in preparation for being collected."</li> 
      </ul> 
      <p>Each of these reference types extends the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/Reference.html">Reference</a></code> class, which provides the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/Reference.html#get()">get()</a></code> <a href="/wiki/Method_(computer_science)" class="mw-redirect" title="Method (computer science)">method</a> to return a strong reference to the referent object (or <code>null</code> if the reference has been cleared or if the reference type is phantom), and the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/Reference.html#clear()">clear()</a></code> method to clear the reference.</p> 
      <p>The <code>java.lang.ref</code> also defines the class <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/ref/ReferenceQueue.html">ReferenceQueue</a></code>, which can be used in each of the applications discussed above to keep track of objects that have changed reference type. When a <code>Reference</code> is created it is optionally registered with a reference queue. The application polls the reference queue to get references that have changed reachability state.</p> 
      <h4><span class="mw-headline" id="java.lang.reflect">java.lang.reflect</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=5" title="Edit section: java.lang.reflect">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p><a href="/wiki/Reflection_(computer_science)" class="mw-redirect" title="Reflection (computer science)">Reflection</a> is a constituent of the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> API that lets Java code examine and "reflect" on Java components at runtime and use the reflected members. Classes in the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/package-summary.html">java.lang.reflect</a></code> package, along with <code>java.lang.Class</code> and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Package.html">java.lang.Package</a></code> accommodate applications such as <a href="/wiki/Debugger" title="Debugger">debuggers</a>, <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreters</a>, object inspectors, <a href="/wiki/Class_browser" title="Class browser">class browsers</a>, and services such as object <a href="/wiki/Serialization" title="Serialization">serialization</a> and <a href="/wiki/JavaBeans" title="JavaBeans">JavaBeans</a> that need access to either the public members of a target object (based on its runtime class) or the members declared by a given class. This package was added in JDK 1.1.</p> 
      <p>Reflection is used to instantiate classes and invoke methods using their names, a concept that allows for dynamic programming. Classes, interfaces, methods, <a href="/wiki/Field_(computer_science)" title="Field (computer science)">fields</a>, and <a href="/wiki/Constructor_(computer_science)" class="mw-redirect" title="Constructor (computer science)">constructors</a> can all be discovered and used at runtime. Reflection is supported by <a href="/wiki/Metadata" title="Metadata">metadata</a> that the JVM has about the program.</p> 
      <h5><span class="mw-headline" id="Techniques">Techniques</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=6" title="Edit section: Techniques">edit</a><span class="mw-editsection-bracket">]</span></span></h5> 
      <p>There are basic techniques involved in reflection:</p> 
      <ul> 
       <li>Discovery – this involves taking an object or class and discovering the members, superclasses, implemented interfaces, and then possibly using the discovered elements.</li> 
       <li>Use by name – involves starting with the symbolic name of an element and using the named element.</li> 
      </ul> 
      <h6><span class="mw-headline" id="Discovery">Discovery</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=7" title="Edit section: Discovery">edit</a><span class="mw-editsection-bracket">]</span></span></h6> 
      <p>Discovery typically starts with an object and calling the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Object.html#getClass()">Object.getClass()</a></code> method to get the object's <code>Class</code>. The <code>Class</code> object has several methods for discovering the contents of the class, for example:</p> 
      <ul> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getMethods()">getMethods()</a></code> – returns an array of <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Method.html">Method</a></code> objects representing all the public methods of the class or interface</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getConstructors()">getConstructors()</a></code> – returns an array of <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Constructor.html">Constructor</a></code> objects representing all the public constructors of the class</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getFields()">getFields()</a></code> – returns an array of <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Field.html">Field</a></code> objects representing all the public fields of the class or interface</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getClasses()">getClasses()</a></code> – returns an array of <code>Class</code> objects representing all the public classes and interfaces that are members (e.g. <a href="/wiki/Inner_class" title="Inner class">inner classes</a>) of the class or interface</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getSuperclass()">getSuperclass()</a></code> – returns the <code>Class</code> object representing the superclass of the class or interface (<code>null</code> is returned for interfaces)</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getInterfaces()">getInterfaces()</a></code> – returns an array of <code>Class</code> objects representing all the interfaces that are implemented by the class or interface</li> 
      </ul> 
      <h6><span class="mw-headline" id="Use_by_name">Use by name</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=8" title="Edit section: Use by name">edit</a><span class="mw-editsection-bracket">]</span></span></h6> 
      <p>The <code>Class</code> object can be obtained either through discovery, by using the <i>class literal</i> (e.g. <code>MyClass.class</code>) or by using the name of the class (e.g. <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#forName(java.lang.String)">Class.forName("mypackage.MyClass")</a></code>). With a <code>Class</code> object, member <code>Method</code>, <code>Constructor</code>, or <code>Field</code> objects can be obtained using the symbolic name of the member. For example:</p> 
      <ul> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getMethod(java.lang.String,java.lang.Class...)">getMethod("methodName", Class...)</a></code> – returns the <code>Method</code> object representing the public method with the name "methodName" of the class or interface that accepts the parameters specified by the <code>Class...</code> parameters.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getConstructor(java.lang.Class...)">getConstructor(Class...)</a></code> – returns the <code>Constructor</code> object representing the public constructor of the class that accepts the parameters specified by the <code>Class...</code> parameters.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#getField(java.lang.String)">getField("fieldName")</a></code> – returns the <code>Field</code> object representing the public field with the name "fieldName" of the class or interface.</li> 
      </ul> 
      <p><code>Method</code>, <code>Constructor</code>, and <code>Field</code> objects can be used to dynamically access the represented member of the class. For example:</p> 
      <ul> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Field.html#get(java.lang.Object)">Field.get(Object)</a></code> – returns an <code>Object</code> containing the value of the field from the instance of the object passed to <code>get()</code>. (If the <code>Field</code> object represents a static field then the <code>Object</code> parameter is ignored and may be <code>null</code>.)</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Method.html#invoke(java.lang.Object,%20java.lang.Object...)">Method.invoke(Object, Object...)</a></code> – returns an <code>Object</code> containing the result of invoking the method for the instance of the first <code>Object</code> parameter passed to <code>invoke()</code>. The remaining <code>Object...</code> parameters are passed to the method. (If the <code>Method</code> object represents a <a href="/wiki/Static_method" class="mw-redirect" title="Static method">static method</a> then the first <code>Object</code> parameter is ignored and may be <code>null</code>.)</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Constructor.html#newInstance(java.lang.Object...)">Constructor.newInstance(Object...)</a></code> – returns the new <code>Object</code> instance from invoking the constructor. The <code>Object...</code> parameters are passed to the constructor. (Note that the parameterless constructor for a class can also be invoked by calling <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/Class.html#newInstance()">newInstance()</a></code>.)</li> 
      </ul> 
      <h5><span class="mw-headline" id="Arrays_and_proxies">Arrays and proxies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=9" title="Edit section: Arrays and proxies">edit</a><span class="mw-editsection-bracket">]</span></span></h5> 
      <p>The <code>java.lang.reflect</code> package also provides an <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Array.html">Array</a></code> class that contains static methods for creating and manipulating array objects, and since J2SE 1.3, a <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/Proxy.html">Proxy</a></code> class that supports dynamic creation of proxy classes that implement specified interfaces.</p> 
      <p>The implementation of a <code>Proxy</code> class is provided by a supplied object that implements the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/InvocationHandler.html">InvocationHandler</a></code> interface. The <code>InvocationHandler</code>'s <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/lang/reflect/InvocationHandler.html#invoke(java.lang.Object,java.lang.reflect.Method,java.lang.Object">[]) invoke(Object, Method, Object[])</a></code> method is called for each method invoked on the proxy object—the first parameter is the proxy object, the second parameter is the <code>Method</code> object representing the method from the interface implemented by the proxy, and the third parameter is the array of parameters passed to the interface method. The <code>invoke()</code> method returns an <code>Object</code> result that contains the result returned to the code that called the proxy interface method.</p> 
      <h3><span class="mw-headline" id="java.io">java.io</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=10" title="Edit section: java.io">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/package-summary.html">java.io</a></code> package contains classes that support <a href="/wiki/Input/output" title="Input/output">input and output</a>. The classes in the package are primarily <a href="/wiki/Stream_(computing)" title="Stream (computing)">stream-oriented</a>; however, a class for <a href="/wiki/Random_access" title="Random access">random access</a> <a href="/wiki/Computer_file" title="Computer file">files</a> is also provided. The central classes in the package are <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/InputStream.html">InputStream</a></code> and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/OutputStream.html">OutputStream</a></code>, which are <a href="/wiki/Class_(computer_science)#Concrete_classes" class="mw-redirect" title="Class (computer science)">abstract</a> base classes for reading from and writing to <a href="/wiki/Byte_stream" class="mw-redirect" title="Byte stream">byte streams</a>, respectively. The related classes <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/Reader.html">Reader</a></code> and <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/Writer.html">Writer</a></code> are abstract base classes for reading from and writing to <a href="/wiki/Character_(computing)" title="Character (computing)">character</a> streams, respectively. The package also has a few miscellaneous classes to support interactions with the host <a href="/wiki/File_system" title="File system">file system</a>.</p> 
      <h4><span class="mw-headline" id="Streams">Streams</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=11" title="Edit section: Streams">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>The stream classes follow the <a href="/wiki/Decorator_pattern" title="Decorator pattern">decorator pattern</a> by extending the base subclass to add features to the stream classes. Subclasses of the base stream classes are typically named for one of the following attributes:</p> 
      <ul> 
       <li>the source/destination of the stream data</li> 
       <li>the type of data written to/read from the stream</li> 
       <li>additional processing or filtering performed on the stream data</li> 
      </ul> 
      <p>The stream subclasses are named using the naming <a href="/wiki/Pattern" title="Pattern">pattern</a> <code><i>XxxStreamType</i></code> where <code><i>Xxx</i></code> is the name describing the feature and <code><i>StreamType</i></code> is one of <code>InputStream</code>, <code>OutputStream</code>, <code>Reader</code>, or <code>Writer</code>.</p>

      <p>Other standard library packages provide stream implementations for other destinations, such as the <code>InputStream</code> returned by the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/net/Socket.html#getInputStream()">java.net.Socket.getInputStream()</a></code> method or the Java EE <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javaee/7/api/javax/servlet/ServletOutputStream.html">javax.servlet.ServletOutputStream</a></code> class.</p> 
      <p>Data type handling and processing or filtering of stream data is accomplished through stream <a href="/wiki/Filter_(software)" title="Filter (software)">filters</a>. The filter classes all accept another compatible stream object as a parameter to the constructor and <i>decorate</i> the enclosed stream with additional features. Filters are created by extending one of the base filter classes <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/FilterInputStream.html">FilterInputStream</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/FilterOutputStream.html">FilterOutputStream</a></code>, <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/FilterReader.html">FilterReader</a></code>, or <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/FilterWriter.html">FilterWriter</a></code>.</p> 
      <p>The <code>Reader</code> and <code>Writer</code> classes are really just byte streams with additional processing performed on the data stream to convert the bytes to characters. They use the default <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a> for the platform, which as of J2SE 5.0 is represented by the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/nio/charset/Charset.html">Charset</a></code> returned by the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/nio/charset/Charset.html#defaultCharset()">java.nio.charset.Charset.defaultCharset()</a></code> static method. The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/InputStreamReader.html">InputStreamReader</a></code> class converts an <code>InputStream</code> to a <code>Reader</code> and the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/OutputStreamWriter.html">OutputStreamWriter</a></code> class converts an <code>OutputStream</code> to a <code>Writer</code>. Both these classes have constructors that support specifying the character encoding to use. If no encoding is specified, the program uses the default encoding for the platform.</p>

      <h4><span class="mw-headline" id="Random_access">Random access</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=12" title="Edit section: Random access">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/RandomAccessFile.html">RandomAccessFile</a></code> class supports <i><a href="/wiki/Random_access" title="Random access">random access</a></i> reading and writing of files. The class uses a <i><a href="/wiki/Data_file#File_pointers_and_random_access" title="Data file">file pointer</a></i> that represents a byte-offset within the file for the next read or write operation. The file pointer is moved implicitly by reading or writing and explicitly by calling the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/RandomAccessFile.html#seek(long)">seek(long)</a></code> or <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/RandomAccessFile.html#skipBytes(int)">skipBytes(int)</a></code> methods. The current position of the file pointer is returned by the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/RandomAccessFile.html#getFilePointer()">getFilePointer()</a></code> method.</p> 
      <h4><span class="mw-headline" id="File_system">File system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=13" title="Edit section: File system">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/File.html">File</a></code> class represents a <a href="/wiki/Computer_file" title="Computer file">file</a> or <a href="/wiki/Directory_(file_systems)" class="mw-redirect" title="Directory (file systems)">directory</a> <a href="/wiki/Path_(computing)" title="Path (computing)">path</a> in a <a href="/wiki/File_system" title="File system">file system</a>. <code>File</code> objects support the creation, deletion and renaming of files and directories and the manipulation of <a href="/wiki/File_attribute" title="File attribute">file attributes</a> such as <i>read-only</i> and <i>last modified timestamp</i>. <code>File</code> objects that represent directories can be used to get a list of all the contained files and directories.</p> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/io/FileDescriptor.html">FileDescriptor</a></code> class is a <a href="/wiki/File_descriptor" title="File descriptor">file descriptor</a> that represents a source or sink (destination) of bytes. Typically this is a file, but can also be a <a href="/wiki/System_console" title="System console">console</a> or <a href="/wiki/Network_socket" title="Network socket">network socket</a>. <code>FileDescriptor</code> objects are used to create <code>File</code> streams. They are obtained from <code>File</code> streams and <code>java.net</code> sockets and datagram sockets.</p> 
      <h3><span class="mw-headline" id="java.nio">java.nio</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=14" title="Edit section: java.nio">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main article: 
       <a href="/wiki/New_I/O" class="mw-redirect" title="New I/O">New I/O</a>
      </div> 
      <p>In J2SE 1.4, the package <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/nio/package-summary.html">java.nio</a></code> (NIO or New I/O) was added to support <a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">memory-mapped I/O</a>, facilitating <a href="/wiki/Input/output" title="Input/output">I/O</a> operations closer to the underlying hardware with sometimes dramatically better performance. The <code>java.nio</code> package provides support for a number of buffer types. The subpackage <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/nio/charset/package-summary.html">java.nio.charset</a></code> provides support for different <a href="/wiki/Character_encoding" title="Character encoding">character encodings</a> for character data. The subpackage <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/nio/channels/package-summary.html">java.nio.channels</a></code> provides support for <i>channels,</i> which represent connections to entities that are capable of performing I/O operations, such as files and sockets. The <code>java.nio.channels</code> package also provides support for fine-grained locking of files.</p> 
      <h3><span class="mw-headline" id="java.math">java.math</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=15" title="Edit section: java.math">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/math/package-summary.html">java.math</a></code> package supports <a href="/wiki/Arbitrary-precision_arithmetic" title="Arbitrary-precision arithmetic">multiprecision arithmetic</a> (including modular arithmetic operations) and provides multiprecision prime number generators used for cryptographic key generation. The main classes of the package are:</p> 
      <ul> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/math/BigDecimal.html">BigDecimal</a></code> – provides arbitrary-precision signed decimal numbers. <code>BigDecimal</code> gives the user control over rounding behavior through <code>RoundingMode</code>.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/math/BigInteger.html">BigInteger</a></code> – provides arbitrary-precision integers. Operations on <code>BigInteger</code> do not <a href="/wiki/Arithmetic_overflow" class="mw-redirect" title="Arithmetic overflow">overflow</a> or lose precision. In addition to standard arithmetic operations, it provides <a href="/wiki/Modular_arithmetic" title="Modular arithmetic">modular arithmetic</a>, <a href="/wiki/Greatest_common_divisor" title="Greatest common divisor">GCD</a> calculation, <a href="/wiki/Primality_testing" class="mw-redirect" title="Primality testing">primality testing</a>, <a href="/wiki/Prime_number" title="Prime number">prime number</a> generation, <a href="/wiki/Bit" title="Bit">bit</a> manipulation, and other miscellaneous operations.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/math/MathContext.html">MathContext</a></code> – encapsulate the context settings that describe certain rules for numerical operators.</li> 
       <li><code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/math/RoundingMode.html">RoundingMode</a></code> – an enumeration that provides eight rounding behaviors.</li> 
      </ul>

Included in the package is the <a href="/wiki/Collections_API" class="mw-redirect" title="Collections API">Collections API</a>, an organized data structure hierarchy influenced heavily by the <a href="/wiki/Design_pattern_(computer_science)" class="mw-redirect" title="Design pattern (computer science)">design patterns</a> considerations.</p>

      <h3><span class="mw-headline" id="java.applet">java.applet</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=20" title="Edit section: java.applet">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

       Main article: 
       <a href="/wiki/Java_applet" title="Java applet">Java applet</a>
      </div> 
      <p>Created to support <a href="/wiki/Java_applet" title="Java applet">Java applet</a> creation, the <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/java/applet/package-summary.html">java.applet</a></code> package lets applications be downloaded over a network and run within a guarded sandbox. Security restrictions are easily imposed on the sandbox. It also provides several basic <a href="/wiki/Layout_manager" title="Layout manager">layout managers</a>, a datatransfer package for use with the <a href="/wiki/Clipboard_(software)" class="mw-redirect" title="Clipboard (software)">Clipboard</a> and <a href="/wiki/Drag_and_drop" title="Drag and drop">Drag and Drop</a>, the interface to <a href="/wiki/Input_device" title="Input device">input devices</a> such as <a href="/wiki/Mouse_(computing)" class="mw-redirect" title="Mouse (computing)">mice</a> and <a href="/wiki/Keyboard_(computing)" class="mw-redirect" title="Keyboard (computing)">keyboards</a>, as well as access to the <a href="/wiki/System_tray" class="mw-redirect" title="System tray">system tray</a> on supporting systems.       <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/javax/rmi/package-summary.html">javax.rmi</a></code> package provides the support for the remote communication between applications, using the RMI over IIOP protocol. This protocol combines RMI and CORBA features.</p> 
      <p><a rel="nofollow" class="external text" href="http://java.sun.com/javase/technologies/core/corba/index.jsp">Java SE Core Technologies - CORBA / RMI-IIOP</a></p> 
      <h3><span class="mw-headline" id="javax.swing">javax.swing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=27" title="Edit section: javax.swing">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main article: 
       <a href="/wiki/Swing_(Java)" title="Swing (Java)">Swing (Java)</a>
      </div> 
      <p><a href="/wiki/Swing_(Java)" title="Swing (Java)">Swing</a> is a collection of routines that build on <code>java.awt</code> to provide a platform independent <a href="/wiki/Widget_toolkit" title="Widget toolkit">widget toolkit</a>. <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/javax/swing/package-summary.html">javax.swing</a></code> uses the 2D drawing routines to render the user interface components instead of relying on the underlying native <a href="/wiki/Operating_system" title="Operating system">operating system</a> GUI support.</p> 
      <p>This package contains the largest number of classes (133 in all) in JDK 6. This package, along with <code>java.awt</code> also contains the largest number of enums (7 in all) in JDK 6. It supports pluggable looks and feels (PLAFs) so that widgets in the GUI can imitate those from the underlying native system. Design patterns permeate the system, especially a modification of the <a href="/wiki/Model-view-controller" class="mw-redirect" title="Model-view-controller">model-view-controller</a> pattern, which loosens the <a href="/wiki/Coupling_(computer_science)" class="mw-redirect" title="Coupling (computer science)">coupling</a> between function and appearance. One inconsistency is that (as of J2SE 1.3) fonts are drawn by the underlying native system, and not by Java, limiting text portability. Workarounds, such as using bitmap fonts, do exist. In general, "layouts" are used and keep elements within an aesthetically consistent GUI across platforms.</p> 
      <h3><span class="mw-headline" id="javax.swing.text.html.parser">javax.swing.text.html.parser</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=28" title="Edit section: javax.swing.text.html.parser">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/javax/swing/text/html/parser/package-summary.html">javax.swing.text.html.parser</a></code> package provides the error tolerant HTML parser that is used for writing various web browsers and web bots.</p> 
      <h3><span class="mw-headline" id="javax.xml.bind.annotation">javax.xml.bind.annotation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=29" title="Edit section: javax.xml.bind.annotation">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/javax/xml/bind/annotation/package-summary.html">javax.xml.bind.annotation</a></code> package contains the largest number of Annotation Types (30 in all) in JDK 6. It defines annotations for customizing Java program elements to XML Schema mapping.</p> 
      <h2><span class="mw-headline" id="OMG_packages">OMG packages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=30" title="Edit section: OMG packages">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="org.omg.CORBA">org.omg.CORBA</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=31" title="Edit section: org.omg.CORBA">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main article: 
       <a href="/wiki/CORBA" class="mw-redirect" title="CORBA">CORBA</a>
      </div> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/org/omg/CORBA/package-summary.html">org.omg.CORBA</a></code> package provides the support for the remote communication between applications using the <a href="/wiki/General_Inter-ORB_Protocol" title="General Inter-ORB Protocol">General Inter-ORB Protocol</a> and supports other features of the <a href="/wiki/CORBA" class="mw-redirect" title="CORBA">common object request broker architecture</a>. Same as <a href="/wiki/Java_remote_method_invocation" title="Java remote method invocation">RMI</a> and <a href="/wiki/RMI-IIOP" title="RMI-IIOP">RMI-IIOP</a>, this package is for calling remote methods of objects on other virtual machines (usually via network).</p> 
      <p>This package contains the largest number of <code>Exception</code> classes (45 in all) in JDK 6. From all communication possibilities CORBA is portable between various languages; however, with this comes more complexity.</p> 
      <h3><span class="mw-headline" id="org.omg.PortableInterceptor">org.omg.PortableInterceptor</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Java_Platform,_Standard_Edition&amp;action=edit&amp;section=32" title="Edit section: org.omg.PortableInterceptor">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>The <code><a rel="nofollow" class="external text" href="//docs.oracle.com/javase/9/docs/api/org/omg/PortableInterceptor/package-summary.html">org.omg.PortableInterceptor</a></code> package contains the largest number of interfaces (39 in all) in JDK 6. It provides a mechanism to register ORB hooks through which ORB services intercept the normal flow of execution of the ORB.</p>
