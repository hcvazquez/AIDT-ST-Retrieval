     <h1 id="getting-started">Getting Started</h1> 
     <p>This section describes how to configure the Syncfusion ASP.NET components into the ASP.NET applications. There are three ways for embedding our controls into ASP.NET application:</p> 
     <ol> 
      <li>Using Syncfusion Project Template</li> 
      <li>Through Syncfusion NuGet Packages</li> 
      <li>Through Manual Integration into the new/existing Application</li> 
     </ol> 
     <p>The procedure that are followed in manual integration process is entirely automated, when we create an application using Syncfusion Project template.</p> 
     <h2 id="using-syncfusion-project-template">Using Syncfusion Project Template</h2> 
     <p>Syncfusion provides the Visual Studio Project Template support for ASP.NET (EJWEB) components, which will automatically include all the Syncfusion ASP.NET control’s client and server side dependencies are preconfigured and copied to it appropriate folders. Below are listed the steps for you to follow when creating an ASP.NET web site.</p> 
     <p>Step 1 Select File &gt; New &gt; Web Site. <br><br> Step 2 Choose Templates &gt; Visual C# &gt; Syncfusion ASP.NET Web Site <br><br> Step 3 Set a name and location for the project.<br><br> Step 4 Click OK to create the project. <br></p> 
     <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img1.png"></p> 
     <p>Add your required controls in the ASPX Page. Let see the code for adding DatePicker control within the “Default.aspx” file as follows.</p>

     <h2 id="configuring-syncfusion-nuget-packages-in-visual-studio">Configuring Syncfusion NuGet Packages in Visual Studio</h2> 
     <p>This topic explains how to add the Syncfusion ASP.NET Web controls into the new ASP.NET Web Forms application by making use of Syncfusion NuGet Packages.</p> 
     <p>Syncfusion ASP.NET Web Forms NuGet packages are available <a href="http://nuget.syncfusion.com/package/aspnet">here</a>.</p> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <p>If you wish to use the ASP.NET NuGet packages then Essential Studio or ASP.NET platform installation not required to implement with Syncfusion ASP.NET controls.</p> 
     </blockquote> 
     <h3 id="nuget-configuration">NuGet Configuration</h3> 
     <p>Syncfusion NuGet Package feed links are configured in Visual Studio in the following ways,</p> 
     <ol> 
      <li>Syncfusion NuGet Package Manager</li> 
      <li>NuGet Package Manager settings</li> 
     </ol> 
     <h4 id="syncfusion-nuget-package-manager">Syncfusion NuGet Package Manager</h4> 
     <p>The steps to configure the Syncfusion ASP.NET Web Forms NuGet Packages in Visual Studio are as follows,</p> 
     <ol> 
      <li> <strong>Syncfusion NuGet Manager</strong> is allows you to <strong>add</strong> the Syncfusion NuGet sources (for available platforms) to NuGet Package Manager. Download the <a href="http://www.syncfusion.com/downloads/support/directtrac/general/ze/SyncfusionNugetManager-1610952973.zip">Syncfusion NuGet Manager</a> utility.</li> 
      <li>Extract zip file and run the SyncfusionNuGetManager.exe</li> 
      <li>Syncfusion NuGet Manager Window will be opened.</li> 
      <li> <p>Select the platform <strong>ASP.NET</strong> from <strong>“Select platforms to add”</strong> (Left side of the window) column and click <strong>Add»</strong> button. Then click <strong>Configure</strong> button.</p> <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Syncfusion_NuGet_Manager_img1.jpg"></p> </li> 
      <li> <p>Once Syncfusion NuGet Manager added the Syncfusion NuGet sources, the changes will be reflected in package sources of your Visual Studio.</p> <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Syncfusion_NuGet_Manager_img2.jpg"></p> </li> 
     </ol> 
     <p><strong>Note:</strong> Get the more details about Syncfusion NuGet Manager utility from <a href="https://help.syncfusion.com/extension/syncfusion-nuget-packages/syncfusion-nuget-manager">here</a>.</p> 
     <h4 id="nuget-package-manager-settings">NuGet Package Manager settings</h4> 
     <p>The steps to configure the Syncfusion ASP.NET Web Forms NuGet Packages in Visual Studio are as follows,</p> 
     <ol> 
      <li> <p>In Visual Studio, navigate to <code>Tools | NuGet Package Manager | Package Manager Settings</code>, the options dialog will appear on the screen as shows below,</p> <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/NuGetConfig1.jpeg"></p> </li> 
      <li> <p>Select <code>NuGet Package Manager | Package Sources</code> and click <code>Add</code> button to add the <code>Package Name</code> and <code>Package Source</code> of Syncfusion NuGet Packages.</p> <p><strong>Name</strong>: Name of the package that listed in Available package sources <br> <strong>Source</strong>: Syncfusion ASP.NET Web Forms NuGet Package feed URL <br> <a href="http://nuget.syncfusion.com/aspnet">http://nuget.syncfusion.com/aspnet</a></p> <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/NuGetConfig2.jpeg"></p> 
       <blockquote class="notes"> 
        <p class="info-header">NOTE</p> 
        <p>The <code>Source</code> text box in the above image denotes the location of the NuGet packages and the <code>Name</code> section, allows you to provide a unique name for NuGet Packages Source.</p> 
       </blockquote> </li> 
     </ol> 
     <blockquote class="important"> 
      <p class="info-header">IMPORTANT</p> 
      <p>Syncfusion other platforms NuGet packages feed links are available <a href="http://nuget.syncfusion.com/">here</a></p> 
     </blockquote> 
     <h3 id="creating-aspnet-web-application">Creating ASP.NET Web Application</h3> 
     <p>Start the Visual Studio. Create a new Web Forms application by using File -&gt; New -&gt; Project and run the application by pressing Ctrl+F5, which shows something similar to the following screenshot in your web browser,</p> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img9.png"><br> Empty Application when run on the web browser</p> 
     <h3 id="nuget-installation">NuGet Installation</h3> 
     <p>Syncfusion ASP.NET Web Forms NuGet can install once configured the package source. The NuGet installation steps as below,</p> 
     <ol> 
      <li> <p>Once configured the Package source with Syncfusion NuGet Packages, right click on project and choose <code>Manage NuGet Packages | Online | &lt;Package Source Name&gt;</code>.</p> <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/NuGetConfig3.jpeg"></p> </li> 
      <li> <p>The NuGet Packages are listed which are available in package source location. Install the required packages to your application by clicking <code>Install</code> button.</p> 
       <blockquote class="notes"> 
        <p class="info-header">NOTE</p> 
        <p>NuGet packages can be install directly through the <strong>command line</strong> (Package Manager Console). Further details click <a href="http://help.syncfusion.com/extension/syncfusion-nuget-packages/nuget-install-and-configuration#install-from-package-manager-console">here</a></p> 
       </blockquote> </li> 
     </ol>

     <h3 id="updating-a-nuget-package">Updating a NuGet Package</h3> 
     <p>Using the <code>Manage NuGet Packages</code> in Visual Studio, NuGet packages can be update.</p> 
     <ol>

     <h3 id="adding-syncfusion-aspnet-control-1">Adding Syncfusion ASP.NET control</h3> 
     <p>Here, we will see how to add the DatePicker control into the newly created Syncfusion ASP.NET Web Application. To add the control, we can use either of the following two ways</p> 
     <p>i. Drag and drop the control from the toolbox <br><br> ii. Manually using the control code. <br></p> 
     <h4 id="drag-and-drop-the-control-from-the-toolbox">Drag and drop the control from the toolbox</h4> 
     <ol> 
      <li>An easy way of adding control into the Default.aspx page is by making use of the Toolbox option present in the Visual Studio. When you install Essential Studio Package or ASP.NET setup in your machine, all the available ASP.NET controls are automatically configured into the Visual Studio Toolbox.</li> 
     </ol> 
     <p>You just need to drag the required control from the toolbox and then drop it into the Content section of the Default.aspx page, so that the required control code gets generated automatically and also the required dependent assembly get registered automatically.</p> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img23.png"><br> Toolbox</p> 
     <ol> 
      <li>When you drag and drop the DatePicker control into the Default.aspx page, the page looks as the image shown below with the automatic control code generated and the required assemblies registered,</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img24.png"><br> Default.aspx page with DatePicker control code and assembly registered for it at the top</p> 
     <h4 id="manually-using-the-control-code">Manually using the control code</h4> 
     <p>If you want to add the control code manually, follow the below steps.</p> 
     <h5 id="assembly-reference">Assembly Reference</h5> 
     <p>Refer the following assemblies in your newly created ASP.NET application, which allows you to use any of the Syncfusion ASP.NET controls within it.</p> 
     <ul> 
      <li>Syncfusion.EJ</li> 
      <li>Syncfusion.EJ.Web</li> 
     </ul> 
     <p>The reference to the Syncfusion assemblies can be added to your application in either of the following ways, <br> - Referring from GAC<br> - Referring from the installed location</p> 
     <h5 id="referring-from-gac">Referring from GAC</h5> 
     <ol> 
      <li>Once you have installed the Essential Studio package in your system, the Syncfusion assemblies are automatically registered in the GAC. You can easily add the reference assemblies to your project by choosing Add Reference option as shown below,</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img25.png"><br> Right click on the References –&gt; choose Add Reference option</p> 
     <ol> 
      <li>Now the Reference Manager pop-up will appear on the screen. In that pop-up, select the required assemblies from the Extensions tab as below, by choosing the appropriate versions (13.1450.0.21). The version to be chosen for the reference assemblies is based on the Framework used in the application.</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img26.png"><br> Reference Manager Pop-up</p> 
     <h5 id="referring-from-the-installed-location">Referring from the installed location</h5> 
     <ol> 
      <li>Add the reference assemblies to your project by choosing Add Reference option as shown below,</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img25.png"><br> Right click on the References –&gt; choose Add Reference option</p> 
     <ol> 
      <li>Now the Reference Manager pop-up will appear on the screen. Select the Browse tab in it and navigate to the installed location of the Syncfusion Essential Studio package in your system. (As depicted in the below image.)</li> 
     </ol> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <installed location="">
       \Syncfusion\Essential Studio\13.1.0.21\precompiledassemblies\13.1.0.21 
      </installed> 
     </blockquote> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <p>For example, If you have installed the Essential Studio package within C:\Program Files (x86), then navigate to the below location,</p> 
     </blockquote> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <p>C:\Program Files (x86)\Syncfusion\Essential Studio\13.1.0.21\precompiledassemblies\13.1.0.21</p> 
     </blockquote> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img27.png"><br> Reference Manager Pop-up with Browse button clicked</p> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <p>In the above image, the folders 3.5, 4.0, 4.5, 4.5.1 denotes the .NET Framework version. Based on the Framework version used in your application, you can choose assemblies from the appropriate folders. The Syncfusion.EJ.Web and other core assemblies like Syncfusion.Core, Syncfusion.EJ are available within these folders.</p> 
     </blockquote> 
     <ol> 
      <li>Add the Syncfusion.EJ, Syncfusion.EJ.Web and Syncfusion.Core assemblies to your application from the below specified location,</li> 
     </ol> 
     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <installed location="">
       \Syncfusion\Essential Studio\13.1.0.21\precompiledassemblies\13.1.0.21\4.5 
       <br> For example, If you have installed the Essential Studio package within C:\Program Files (x86), then navigate to the below location, 
       <br> C:\Program Files (x86)\Syncfusion\Essential Studio\13.1.0.21\precompiledassemblies\13.1.0.21\4.5 
      </installed> 
     </blockquote> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img28.png"><br> Reference Manager Pop-up with assemblies selected</p> 
     <ol> 
      <li>Once the assembly selection is done, click OK to add the selected references to your project. You can view the assembly references added to your application, in the solution explorer as shown below,</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img29.png"><br> Selected Assemblies added to the Project reference</p> 
     <h5 id="registering-syncfusion-assemblies-within-the-webconfig">Registering Syncfusion Assemblies within the Web.config</h5> 
     <p>In your application’s web.config file, add the below assembly information within the 
      <assemblies>
        tag.
      </assemblies></p> 
     <li role="presentation" class=""><a data-target="#cn01fbrkeiiqzil7l2xo36zwaqsimlcj-xml" aria-controls="home" role="tab" data-toggle="tab" data-original-lang="xml">xml</a></li>

       <pre><code class="language-xml" data-lang="xml"><span class="nt">&lt;system.web&gt;</span>
    <span class="nt">&lt;compilation</span> <span class="na">debug=</span><span class="s">"true"</span> <span class="na">targetFramework=</span><span class="s">"4.5"</span><span class="nt">&gt;</span>
          <span class="nt">&lt;assemblies&gt;</span>

            <span class="nt">&lt;add</span> <span class="na">assembly=</span><span class="s">"Syncfusion.EJ, Version=13.1450.0.21, Culture=neutral, PublicKeyToken=3d67ed1f87d44c89"</span> <span class="nt">/&gt;</span>

            <span class="nt">&lt;add</span> <span class="na">assembly=</span><span class="s">"Syncfusion.EJ.Web, Version=13.1450.0.21, Culture=neutral, PublicKeyToken=3d67ed1f87d44c89"</span> <span class="nt">/&gt;</span>

          <span class="nt">&lt;/assemblies&gt;</span>
    <span class="nt">&lt;/compilation&gt;</span>

   …
<span class="nt">&lt;/system.web&gt;</span></code></pre>
      </div> 
     </div>

      <li>Add the below DatePicker code within the Default.aspx file, as shown in the image following it,</li> 
     </ol> 
     <li role="presentation" class=""><a data-target="#x4r3wjua06hkdr1psc0gv5i3u5boyq26-html" aria-controls="home" role="tab" data-toggle="tab" data-original-lang="html">html</a></li>

     <blockquote class="notes"> 
      <p class="info-header">NOTE</p> 
      <p>Add the DatePicker code within the Content section, by removing the unwanted code within it.</p> 
     </blockquote> 
     <ol> 
      <li>Also register the required assemblies within the Default.aspx page at the top where you are using the control as shown below,</li> 
     </ol> 
     <p><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img30.png"></p> 
     <ol> 
      <li>Finally build and run the project by pressing F5, so that you can now see the output similar to the below screenshot in your web browser,</li> 
     </ol> 
     <p class="caption"><img src="https://cdn.syncfusion.com/documentation/images/default.png" alt="" class="lazy" data-original="Getting-Started_images/Getting-Started_img21.png"><br> DatePicker control displaying on the web browser</p> 
     <p>Thus the DatePicker control is rendered successfully with its default appearance. You can then use its various properties to set its value and also make use of its available events to trigger when necessary.</p>

      <p>The Script Manager is mandatory in order to place our control initialization script in the page.<br><br> Ensure whether the “ScriptManager” is added in the “Site.Master” or else add the “ScriptManager” to your web page. <br><br> You just need to drag the “ScriptManager” from the toolbox under the “AJAX Extension” and then drop it into the Content section of the Default.aspx page, so that the required scripts to initiate our component will be added to your web page.</p>
