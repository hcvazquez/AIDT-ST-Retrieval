   <h1>3.1 &nbsp; Using Data Types</h1> 
   <p><br> A <em>data type</em> is a set of values and a set of operations defined on those values. 
    <!--
For example, the values of the primitive data type <tt>int</tt> are integers
between &minus;2<sup>31</sup> and 2<sup>31</sup> &minus; 1; the operations of <tt>int</tt>
are the basic arithmetic and logical operations that we have been using 
since Chapter 1.--> 
    <!--
Programming in Java is largely based on doing so with data types
known as <em>reference types</em> that are designed to support
object-oriented programming.
--> The primitive data types that you have been using are supplemented in Java by extensive libraries of <em>reference types</em> that are tailored for a large variety of applications. In this section, we consider reference types for string processing and image processing. </p>
   <p> </p>
   <h2>Strings.</h2> You have already been using a data type that is not primitive—the 
   <tt>String</tt> data type, whose values are sequences of characters. We specify the behavior of a data type in an 
   <em>application programming interface</em> (API). Here is a partial API for Java’s 
   <a href="https://docs.oracle.com/javase/8/docs/api/java/lang/String.html">String</a> data type: 
   <!--
Several of the methods use integers to refer to a
character’s index within a string; as with arrays, these indices start at 0.
--> 
   <p> </p>
   <blockquote> 
    <img SRC="../11cheatsheet/images/string-api.png" width="525" alt="String API"> 
   </blockquote> The first entry, with the same name as the class and no return type, defines a special method known as a 
   <em>constructor</em>. The other entries define 
   <em>instance methods</em> that can take arguments and return values. 
   <blockquote> 
    <img SRC="../11cheatsheet/images/object.png" width="250" alt="using a reference data type"> 
   </blockquote> 
   <ul> 
    <li><em>Declaring variables.</em> You declare variables of a reference type in precisely the same way that you declare variables of a primitive type. A declaration statement does not create anything; it just says that we will use the variable name <tt>s</tt> to refer to a <tt>String</tt> object. </li>
    <li><em>Creating objects.</em> Each data-type value is stored in an <em>object</em>. When a client invokes a constructor, the Java system creates (or <em>instantiates</em>) an individual object (or <em>instance</em>). To invoke a constructor, use the keyword <tt>new</tt>; followed by the class name; followed by the constructor’s arguments, enclosed in parentheses and separated by commas. 
     <!--
You can create any number of objects from the same class;
each object has its own identity and may or may not store the same
value as another object of the same type.
--> </li>
    <li><em>Invoking instance methods.</em> The most important difference between a variable of a reference type and a variable of a primitive type is that you can use reference-type variables to invoke the <em>instance methods</em> that implement data-type operations (in contrast to the built-in syntax involving operators such as <tt>+</tt>* that we used with primitive types). </li>
   </ul> Now, we consider various string-processing examples.

     <blockquote> 
      <img SRC="images/string-examples.png" width="450" alt="String code fragments">

    <li><em>Genomics.</em> Biologists use a simple model to represent the building blocks of life, in which the letters A, C, G, and T represent the four bases in the DNA of living organisms. 
     <!--
In each living organism, these basic building blocks appear in a set of long sequences
(one for each chromosome) known as a <em>genome</em>.
--> A <em>gene</em> is a substring that represents a functional unit of critical importance in understanding life processes. <a href="PotentialGene.java.html">PotentialGene.java</a> takes a DNA string as an argument and determines whether it corresponds to a potential gene based on the following criteria: 
     <ul> 
      <li> It begins with the <em>start codon</em> ATG. </li>
      <li> Its length is a multiple of 3. </li>
      <li> It ends with one of the <em>stop codons</em> TAG, TAA, or TGA. </li>

   <a name="Picture"> <p></p><h2>Image processing.</h2></a> 
   <img SRC="images/picture-anatomy.png" width="150" align="right" hspace="25" alt="anatomy of a digital image"> A 
   <em>digital image</em> is a rectangular grid of 
   <em>pixels</em> (picture elements), where the color of each pixel is individually defined. Digital images are sometimes referred to as 
   <em>raster</em> or 
   <em>bitmapped</em> images. In contrast, the types of images that we produce with 
   <tt>StdDraw</tt> (which involved geometric objects) are referred to as 
   <em>vector</em> images. 
   <p> The <a href="http://introcs.cs.princeton.edu/java/stdlib/javadoc/Picture.html">Picture</a> data type allows you to manipulate digital images. The set of values is a two-dimensional matrix of <tt>Color</tt> values, and the operations are what you might expect: create an image (either blank or from a file), set the value of a pixel to a given color, and extract the color of a given pixel. The following API summarizes the available operations: </p>
   <p> </p>
   <blockquote> 
    <img SRC="../11cheatsheet/images/picture-api.png" width="500" alt="Picture API"> 
   </blockquote>
