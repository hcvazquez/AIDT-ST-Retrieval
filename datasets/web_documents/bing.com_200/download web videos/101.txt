         <p> </p> 
         <div class="introduction">
          <p>This topic explains how to deploy a web project by creating a deployment package in Visual Studio on the source computer and installing it manually on the destination server. If you want to configure a continuous integration system to create and install deployment packages automatically, see <a href="http://www.asp.net/web-forms/tutorials/deployment/deploying-web-applications-in-enterprise-scenarios/deploying-web-applications-in-enterprise-scenarios">Deploying Web Applications in Enterprise Scenarios</a>.</p>

             <tr>
              <td><p>This topic applies to Visual Studio 2012 and Visual Studio Express 2012 for Web. The topic covers features that are included in the latest <a href="http://go.microsoft.com/fwlink/?LinkID=208120" target="_blank">Visual Studio Web Publish Update</a> available as of June, 2013. Most of these features are also available in Visual Studio 2010 and Visual Web Developer 2010 Express when you install the Web Publish Update.</p></td>

           </table>
          </div>
          <p>To configure deployment for a web project in Visual Studio, you create one or more <span class="parameter">publish profiles</span> using the <span class="label">Publish Web</span> wizard. A publish profile specifies the databases to deploy and other deployment options. When you are ready to create a deployment package, you choose the profile you want to use and click the <span class="label">Publish</span> button in the wizard or in the <span class="label">Web One Click Publish</span> toolbar. You can then install the package on the destination server by using the command line or <span class="label">IIS Manager</span>.</p>
          <p>You might also have to perform other tasks for deployment besides creating and installing the deployment package. For more information, see <span><a href="https://msdn.microsoft.com/en-us/library/dd394698(v=vs.110).aspx">Web Deployment Overview for Visual Studio and ASP.NET</a></span>. </p>
         </div>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Using the Publish Web Wizard to Create a Deployment Package</span></a>
            <div id="Anchor_0" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/dd465323(v=vs.110).aspx#Anchor_0" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <a id="sectionToggle0"></a>
           <p></p>
           <p>Refer to <strong>Creating a Publish Profile</strong> in <span><a href="https://msdn.microsoft.com/en-us/library/dd465337(v=vs.110).aspx">How to: Deploy a Web Project Using One-Click Publish in Visual Studio</a></span> for information about how to create a new publish profile. This topic only covers what is different when you want to create a package rather than deploy directly to a server. </p>
           <p>You configure a publish profile to create a deployment package by selecting <span class="label">Web Deploy Package</span> as the publish method on the <span class="label">Connection</span> tab of the <span class="label">Publish Web</span> wizard.</p>
           <h3 class="procedureSubHeading">To configure settings on the Connection tab</h3>
           <div class="subSection">
            <ol>
             <li><p>In the <span class="label">Publish method</span> drop-down list, select <span class="label">Web Deploy Package</span>. </p><img id="ocp_Publish_Web_Connection_tab_for_Deployment_Package" alt="Publish_Web_Connection_tab_for_Deployment_Pack" src="https://i-msdn.sec.s-msft.com/dynimg/IC591683.png" title="Publish_Web_Connection_tab_for_Deployment_Pack" xmlns=""></li>
             <li><p>In the <span class="label">Package location</span> box, enter the path to where you want the package to be created, including the .zip file name; for example, <span class="code">c:\temp\</span><span class="parameter">&lt;projectname&gt;</span><span class="code">.zip</span>. As an alternative, you can click the ellipsis to open a <span class="label">Select Package Location</span> dialog box.</p><img id="ocp_Select_Package_Location_dialog" alt="Select_Package_Location_dialog" src="https://i-msdn.sec.s-msft.com/dynimg/IC591684.png" title="Select_Package_Location_dialog" xmlns=""><p>When you select a folder in the <span class="label">Select Package Location</span> dialog box, the package name is set to <span class="parameter">&lt;projectname&gt;</span><span class="code">.zip</span> by default.</p></li>
             <li><p>In the <span class="label">Site/application</span> box enter the name of the IIS site and application that you intend to install the package to. This value can be overridden when you install the package. </p><p>If you enter an application name without an IIS web site name, the package is installed in the Default Web Site location (unless you override this value when you install the package).</p><p>If the project that you are deploying is an IIS web project, do not specify the same web site and application name as the project's web site. If you do, some of the project files might be overwritten during deployment.</p></li>
             <li><p>Click <span class="label">Next</span>.</p></li>
            </ol>
           </div>
           <h3 class="procedureSubHeading">To configure the Package section of the Settings tab</h3>
           <div class="subSection">
            <ol>
             <li><p>In the <span class="label">Package</span> section of the <span class="label">Settings</span> tab, choose the build configuration to deploy.</p><p>You normally deploy a Release build. A Debug build is less efficient and less secure and is typically selected only when you are deploying to a test environment and you want to debug in that environment.</p><img id="ocp_Publish_Web_Settings_tab_for_Deployment_Package" alt="Publish_Web_Settings_tab_for_Deployment_Packag" src="https://i-msdn.sec.s-msft.com/dynimg/IC591685.png" title="Publish_Web_Settings_tab_for_Deployment_Packag" xmlns=""><p>(For the other publish methods there is also a <span class="label">Remove additional files at destination</span> check box on the <span class="label">Settings</span> tab. You specify this option when you install the package, not when you create it. Therefore, the check box is absent from this tab when you select <span class="label">Web Deploy Package</span> as the publish method.) </p></li>
             <li><p>Configure database settings by using the information in the <strong>Configuring Databases in the Settings Tab</strong> section in <span><a href="https://msdn.microsoft.com/en-us/library/dd465337(v=vs.110).aspx">How to: Deploy a Web Project Using One-Click Publish in Visual Studio</a></span>.</p></li>
            </ol>
           </div>
           <p>After you configure the <span class="label">Settings</span> tab, click <span class="label">Publish</span> to create the package at the path you specified on the <span class="label">Connection</span> tab. (The wizard advances to the <span class="label">Preview</span> tab if you click <span class="label">Next</span> on the <span class="label">Settings</span> tab, but previews are not provided for deployment packages.)</p>
           <p>The Output window reports successful package creation.</p>
           <img id="ocp_Output_window_after_package_creation" alt="Output_window_after_package_creation" src="https://i-msdn.sec.s-msft.com/dynimg/IC591686.png" title="Output_window_after_package_creation" xmlns="">
           <p>In addition to the .zip file that contains the deployment package, the folder in which the .zip file is created contains the following files:</p>
           <ul>
            <li><p><span class="parameter">projectname</span>.deploy.cmd. This is a helper command-line batch file that invokes Web Deploy in order to install the application on the destination server locally or remotely.</p></li>
            <li><p><span class="parameter">projectname</span>.SetParameters.xml. This file contains parameters that are passed to Web Deploy on the destination server. By default, this file contains the values that are specified during the packaging process. If you want to install the application on multiple servers with different settings for each server, you can modify the parameter values in this file each time that you install the package by using the <span class="parameter">projectname</span>.deploy.cmd command file.</p></li>
            <li><p><span class="parameter">projectname</span>.SourceManifest.xml. This file contains settings that Visual Studio used to create the deployment package. This file is only used to create the package. It is not used when the package is installed on the destination server. </p></li>
           </ul>
          </div>
         </div>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Installing the Package</span></a>
            <div id="Anchor_1" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/dd465323(v=vs.110).aspx#Anchor_1" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <a id="sectionToggle1"></a>
           <p>In order to install a package, Web Deploy must be installed on the destination server. In addition, the version of Web Deploy on the destination server must be compatible with the version on the development computer. (By default, Web Deploy is installed on the development computer when you install Visual Studio.) For information about how to install IIS&nbsp;7 and Web Deploy by using the Web Platform Installer, see <a href="http://go.microsoft.com/fwlink/?LinkId=153787" target="_blank">Microsoft Web Platform Installer</a> on the Microsoft Web site. For information about how to install IIS&nbsp;7 and Web Deploy individually, see <a href="http://go.microsoft.com/fwlink/?LinkId=182312" target="_blank">IIS 7 Installation and Deployment</a> and <a href="http://go.microsoft.com/fwlink/?LinkId=182314" target="_blank">Web Deployment Tool Installation</a> on the Microsoft TechNet Web site.</p>
           <p>You can install a package in the following ways:</p>
           <ul>
            <li><p>Use IIS Manager.</p><p>IIS&nbsp;7 must be installed on the destination server, and the package must be created as a .zip file. For more information, see <a href="http://go.microsoft.com/fwlink/?LinkId=148349" target="_blank">Import a Package through IIS&nbsp;Manager</a> on the IIS.net Web site.</p></li>
            <li><p>Use the <span class="parameter">&lt;projectname&gt;</span>.deploy.cmd file that Visual Studio creates with the package. </p><p>This file lets you use a simplified command-line syntax to create the much longer and more complex Web Deploy commands that install the package. For more information, see <span><a href="https://msdn.microsoft.com/en-us/library/ff356104(v=vs.110).aspx">How to: Install a Deployment Package Using the deploy.cmd File Created by Visual Studio</a></span>.</p></li>
            <li><p>Use Web Deploy commands directly from the command line or by executing PowerShell commands.</p><p>For more information about Web Deploy, see <a href="https://technet.microsoft.com/en-us/library/dd568996.aspx">Web Deployment Tool</a> on the TechNet web site and <a href="http://learn.iis.net/page.aspx/346/web-deploy/">Web Deploy</a> on the IIS.net web site.</p></li>
           </ul>
          </div>
         </div>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Advanced Deployment Package Creation Options</span></a>
            <div id="Anchor_2" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/dd465323(v=vs.110).aspx#Anchor_2" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <a id="sectionToggle2"></a>
           <p>You can specify some advanced options for deployment package creation on the <span class="label">Package/Publish Web</span> tab of the <span class="label">Project Properties</span> window.</p>
           <ul>
            <li><p>Specify the physical IIS path.</p></li>
            <li><p>Specify a password for IIS settings.</p></li>
           </ul>
           <p>For more information about these options, see <span><a href="https://msdn.microsoft.com/en-us/library/dd410108(v=vs.110).aspx">Package/Publish Web Tab, Project Properties</a></span>. </p>
          </div>
