         <h1><a id="user-content-natural" class="anchor" href="#natural" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>natural</h1> 
         <p><a href="https://www.npmjs.com/package/natural"><img src="https://camo.githubusercontent.com/331058fa9ea2c18070e6d7c547ed0e571621472a/68747470733a2f2f696d672e736869656c64732e696f2f6e706d2f762f6e61747572616c2e737667" alt="NPM version" data-canonical-src="https://img.shields.io/npm/v/natural.svg" style="max-width:100%;"></a> <a href="https://travis-ci.org/NaturalNode/natural"><img src="https://camo.githubusercontent.com/7d02b9ac10843467f3dfa735055677247d50dfe4/68747470733a2f2f7472617669732d63692e6f72672f4e61747572616c4e6f64652f6e61747572616c2e706e673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/NaturalNode/natural.png?branch=master" style="max-width:100%;"></a></p> 
         <p>"Natural" is a general natural language facility for nodejs. Tokenizing, stemming, classification, phonetics, tf-idf, WordNet, string similarity, and some inflections are currently supported.</p> 
         <p>It's still in the early stages, so we're very interested in bug reports, contributions and the like.</p> 
         <p>Note that many algorithms from Rob Ellis's <a href="https://github.com/NaturalNode/node-nltools">node-nltools</a> are being merged into this project and will be maintained from here onward.</p> 
         <p>At the moment, most of the algorithms are English-specific, but in the long-term, some diversity will be in order. Thanks to Polyakov Vladimir, Russian stemming has been added!, Thanks to David Przybilla, Spanish stemming has been added!.</p> 
         <p>Aside from this README, the only documentation is <a href="http://www.dzone.com/links/r/using_natural_a_nlp_module_for_nodejs.html">this DZone article</a>, <a href="https://egghead.io/courses/natural-language-processing-in-javascript-with-natural">this free course on Egghead.io</a>, and <a href="http://www.chrisumbel.com/article/node_js_natural_language_porter_stemmer_lancaster_bayes_naive_metaphone_soundex">here on my blog</a>, which is a bit older.</p>

         <h2><a id="user-content-tokenizers" class="anchor" href="#tokenizers" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Tokenizers</h2> 
         <p>Word, Regexp, and <a href="http://www.cis.upenn.edu/%7Etreebank/tokenization.html">Treebank tokenizers</a> are provided for breaking text up into arrays of tokens:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> tokenizer <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.WordTokenizer</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">tokenizer</span>.<span class="pl-en">tokenize</span>(<span class="pl-s"><span class="pl-pds">"</span>your dog has fleas.<span class="pl-pds">"</span></span>));
<span class="pl-c"><span class="pl-c">//</span> [ 'your', 'dog', 'has', 'fleas' ]</span></pre>
         </div> 
         <p>The other tokenizers follow a similar pattern:</p> 
         <div class="highlight highlight-source-js">
          <pre>tokenizer <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.TreebankWordTokenizer</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">tokenizer</span>.<span class="pl-en">tokenize</span>(<span class="pl-s"><span class="pl-pds">"</span>my dog hasn't any fleas.<span class="pl-pds">"</span></span>));
<span class="pl-c"><span class="pl-c">//</span> [ 'my', 'dog', 'has', 'n\'t', 'any', 'fleas', '.' ]</span>

tokenizer <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.RegexpTokenizer</span>({pattern<span class="pl-k">:</span><span class="pl-sr"> <span class="pl-pds">/</span><span class="pl-cce">\-</span><span class="pl-pds">/</span></span>});
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">tokenizer</span>.<span class="pl-en">tokenize</span>(<span class="pl-s"><span class="pl-pds">"</span>flea-dog<span class="pl-pds">"</span></span>));
<span class="pl-c"><span class="pl-c">//</span> [ 'flea', 'dog' ]</span>

tokenizer <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.WordPunctTokenizer</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">tokenizer</span>.<span class="pl-en">tokenize</span>(<span class="pl-s"><span class="pl-pds">"</span>my dog hasn't any fleas.<span class="pl-pds">"</span></span>));
<span class="pl-c"><span class="pl-c">//</span> [ 'my',  'dog',  'hasn',  '\'',  't',  'any',  'fleas',  '.' ]</span></pre>
         </div>

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>String Distance</h2> 
         <p>Natural provides an implementation of the <a href="http://en.wikipedia.org/wiki/Jaro%E2%80%93Winkler_distance">Jaro–Winkler</a> string distance measuring algorithm. This will return a number between 0 and 1 which tells how closely the strings match (0 = not at all, 1 = exact match):</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">JaroWinklerDistance</span>(<span class="pl-s"><span class="pl-pds">"</span>dixon<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>dicksonx<span class="pl-pds">"</span></span>))
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">JaroWinklerDistance</span>(<span class="pl-s"><span class="pl-pds">'</span>not<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>same<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>Output:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">0.7466666666666666</span>
<span class="pl-c1">0</span></pre>
         </div> 
         <p>Natural also offers support for Levenshtein distances:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">LevenshteinDistance</span>(<span class="pl-s"><span class="pl-pds">"</span>ones<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>onez<span class="pl-pds">"</span></span>));
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">LevenshteinDistance</span>(<span class="pl-s"><span class="pl-pds">'</span>one<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>one<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>Output:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">1</span>
<span class="pl-c1">0</span></pre>
         </div> 
         <p>The cost of the three edit operations are modifiable for Levenshtein:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">LevenshteinDistance</span>(<span class="pl-s"><span class="pl-pds">"</span>ones<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>onez<span class="pl-pds">"</span></span>, {
    insertion_cost<span class="pl-k">:</span> <span class="pl-c1">1</span>,
    deletion_cost<span class="pl-k">:</span> <span class="pl-c1">1</span>,
    substitution_cost<span class="pl-k">:</span> <span class="pl-c1">1</span>
}));</pre>
         </div> 
         <p>Output:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">1</span></pre>
         </div> 
         <p>And Dice's co-efficient:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">DiceCoefficient</span>(<span class="pl-s"><span class="pl-pds">'</span>thing<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>thing<span class="pl-pds">'</span></span>));
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">DiceCoefficient</span>(<span class="pl-s"><span class="pl-pds">'</span>not<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>same<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>Output:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-c1">1</span>
<span class="pl-c1">0</span></pre>
         </div>

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>

         <p>Currently matching is supported via the Levenshtein algorithm.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> source <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>The RainCoat BookStore<span class="pl-pds">'</span></span>;
<span class="pl-k">var</span> target <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>All the best books are here at the Rain Coats Book Store<span class="pl-pds">'</span></span>;

<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-en">LevenshteinDistance</span>(source, target, {search<span class="pl-k">:</span> <span class="pl-c1">true</span>}));</pre>
         </div> 
         <p>Output:</p> 
         <div class="highlight highlight-source-js">
          <pre>{ substring<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>the Rain Coats Book Store<span class="pl-pds">'</span></span>, distance<span class="pl-k">:</span> <span class="pl-c1">4</span> }</pre>
         </div> 
         <h2><a id="user-content-stemmers" class="anchor" href="#stemmers" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Stemmers</h2> 
         <p>Currently stemming is supported via the <a href="http://tartarus.org/martin/PorterStemmer/index.html">Porter</a> and <a href="http://www.comp.lancs.ac.uk/computing/research/stemming/">Lancaster</a> (Paice/Husk) algorithms.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);</pre>
         </div> 
         <p>This example uses a Porter stemmer. "word" is returned.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-smi">PorterStemmer</span>.<span class="pl-en">stem</span>(<span class="pl-s"><span class="pl-pds">"</span>words<span class="pl-pds">"</span></span>)); <span class="pl-c"><span class="pl-c">//</span> stem a single word</span></pre>
         </div> 
         <p>in Russian:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-smi">PorterStemmerRu</span>.<span class="pl-en">stem</span>(<span class="pl-s"><span class="pl-pds">"</span>??????<span class="pl-pds">"</span></span>));</pre>
         </div> 
         <p>in Spanish:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">natural</span>.<span class="pl-smi">PorterStemmerEs</span>.<span class="pl-en">stem</span>(<span class="pl-s"><span class="pl-pds">"</span>jugaría<span class="pl-pds">"</span></span>));</pre>
         </div> 
         <p><code>attach()</code> patches <code>stem()</code> and <code>tokenizeAndStem()</code> to String as a shortcut to <code>PorterStemmer.stem(token)</code>. <code>tokenizeAndStem()</code> breaks text up into single words and returns an array of stemmed tokens.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">natural</span>.<span class="pl-smi">PorterStemmer</span>.<span class="pl-en">attach</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>i am waking up to the sounds of chainsaws<span class="pl-pds">"</span></span>.<span class="pl-en">tokenizeAndStem</span>());
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>chainsaws<span class="pl-pds">"</span></span>.<span class="pl-en">stem</span>());</pre>
         </div> 
         <p>the same thing can be done with a Lancaster stemmer:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">natural</span>.<span class="pl-smi">LancasterStemmer</span>.<span class="pl-en">attach</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>i am waking up to the sounds of chainsaws<span class="pl-pds">"</span></span>.<span class="pl-en">tokenizeAndStem</span>());
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>chainsaws<span class="pl-pds">"</span></span>.<span class="pl-en">stem</span>());</pre>
         </div> 
         <h2><a id="user-content-classifiers" class="anchor" href="#classifiers" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Classifiers</h2> 
         <p>Two classifiers are currently supported, <a href="http://en.wikipedia.org/wiki/Naive_Bayes_classifier">Naive Bayes</a> and <a href="http://en.wikipedia.org/wiki/Logistic_regression">logistic regression</a>. The following examples use the BayesClassifier class, but the LogisticRegressionClassifier class could be substituted instead.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> classifier <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.BayesClassifier</span>();</pre>
         </div> 
         <p>You can train the classifier on sample text. It will use reasonable defaults to tokenize and stem the text.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>i am long qqqq<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>buy<span class="pl-pds">'</span></span>);
<span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>buy the q<span class="pl-cce">\'</span>s<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>buy<span class="pl-pds">'</span></span>);
<span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>short gold<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>);
<span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>sell gold<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>);

<span class="pl-smi">classifier</span>.<span class="pl-en">train</span>();</pre>
         </div> 
         <p>Outputs "sell"</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">classifier</span>.<span class="pl-en">classify</span>(<span class="pl-s"><span class="pl-pds">'</span>i am short silver<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>Outputs "buy"</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">classifier</span>.<span class="pl-en">classify</span>(<span class="pl-s"><span class="pl-pds">'</span>i am long copper<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>You have access to the set of matched classes and the associated value from the classifier.</p> 
         <p>Outputs:</p> 
         <div class="highlight highlight-source-js">
          <pre>[ { label<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>buy<span class="pl-pds">'</span></span>, value<span class="pl-k">:</span> <span class="pl-c1">0.39999999999999997</span> },
  { label<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>, value<span class="pl-k">:</span> <span class="pl-c1">0.19999999999999998</span> } ]</pre>
         </div> 
         <p>From this:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">classifier</span>.<span class="pl-en">getClassifications</span>(<span class="pl-s"><span class="pl-pds">'</span>i am long copper<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>The classifier can also be trained with and can classify arrays of tokens, strings, or any mixture of the two. Arrays let you use entirely custom data with your own tokenization/stemming, if you choose to implement it.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>([<span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>gold<span class="pl-pds">'</span></span>], <span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>);</pre>
         </div> 
         <p>The training process can be monitored by subscribing to the event <code>trainedWithDocument</code> that's emitted by the classifier, this event's emitted each time a document is finished being trained against:</p> 
         <pre><code>classifier.events.on('trainedWithDocument', function (obj) {
   console.log(obj);
   /* {
   *   total: 23 // There are 23 total documents being trained against
   *   index: 12 // The index/number of the document that's just been trained against
   *   doc: {...} // The document that has just been indexed
   *  }
   */
});
</code></pre> 
         <p>A classifier can also be persisted and recalled so you can reuse a training</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">classifier</span>.<span class="pl-en">save</span>(<span class="pl-s"><span class="pl-pds">'</span>classifier.json<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">err</span>, <span class="pl-smi">classifier</span>) {
    <span class="pl-c"><span class="pl-c">//</span> the classifier is saved to the classifier.json file!</span>
});</pre>
         </div> 
         <p>To recall from the classifier.json saved above:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">natural</span>.<span class="pl-smi">BayesClassifier</span>.<span class="pl-c1">load</span>(<span class="pl-s"><span class="pl-pds">'</span>classifier.json<span class="pl-pds">'</span></span>, <span class="pl-c1">null</span>, <span class="pl-k">function</span>(<span class="pl-smi">err</span>, <span class="pl-smi">classifier</span>) {
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">classifier</span>.<span class="pl-en">classify</span>(<span class="pl-s"><span class="pl-pds">'</span>long SUNW<span class="pl-pds">'</span></span>));
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">classifier</span>.<span class="pl-en">classify</span>(<span class="pl-s"><span class="pl-pds">'</span>short SUNW<span class="pl-pds">'</span></span>));
});</pre>
         </div> 
         <p>A classifier can also be serialized and deserialized like so:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> classifier <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.BayesClassifier</span>();
<span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>([<span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>gold<span class="pl-pds">'</span></span>], <span class="pl-s"><span class="pl-pds">'</span>sell<span class="pl-pds">'</span></span>);
<span class="pl-smi">classifier</span>.<span class="pl-en">addDocument</span>([<span class="pl-s"><span class="pl-pds">'</span>buy<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>silver<span class="pl-pds">'</span></span>], <span class="pl-s"><span class="pl-pds">'</span>buy<span class="pl-pds">'</span></span>);

<span class="pl-c"><span class="pl-c">//</span> serialize</span>
<span class="pl-k">var</span> raw <span class="pl-k">=</span> <span class="pl-c1">JSON</span>.<span class="pl-c1">stringify</span>(classifier);
<span class="pl-c"><span class="pl-c">//</span> deserialize</span>
<span class="pl-k">var</span> restoredClassifier <span class="pl-k">=</span> <span class="pl-smi">natural</span>.<span class="pl-smi">BayesClassifier</span>.<span class="pl-en">restore</span>(<span class="pl-c1">JSON</span>.<span class="pl-c1">parse</span>(raw));
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">restoredClassifier</span>.<span class="pl-en">classify</span>(<span class="pl-s"><span class="pl-pds">'</span>i should sell that<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p><strong>Note:</strong> if using the classifier for languages other than English you may need to pass in the stemmer to use. In fact, you can do this for any stemmer including alternate English stemmers. The default is the <code>PorterStemmer</code>.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">const</span> <span class="pl-c1">PorterStemmerRu</span> <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>./node_modules/natural/lib/natural/stemmers/porter_stemmer_ru<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> classifier <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.BayesClassifier</span>(PorterStemmerRu);</pre>
         </div> 
         <h2><a id="user-content-phonetics" class="anchor" href="#phonetics" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Phonetics</h2> 
         <p>Phonetic matching (sounds-like) matching can be done with the <a href="http://en.wikipedia.org/wiki/Soundex">SoundEx</a>, <a href="http://en.wikipedia.org/wiki/Metaphone">Metaphone</a> or <a href="http://en.wikipedia.org/wiki/Metaphone#Double_Metaphone">DoubleMetaphone</a> algorithms</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> metaphone <span class="pl-k">=</span> <span class="pl-smi">natural</span>.<span class="pl-smi">Metaphone</span>;
<span class="pl-k">var</span> soundEx <span class="pl-k">=</span> <span class="pl-smi">natural</span>.<span class="pl-smi">SoundEx</span>;

<span class="pl-k">var</span> wordA <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>phonetics<span class="pl-pds">'</span></span>;
<span class="pl-k">var</span> wordB <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>fonetix<span class="pl-pds">'</span></span>;</pre>
         </div> 
         <p>To test the two words to see if they sound alike:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">if</span>(<span class="pl-smi">metaphone</span>.<span class="pl-en">compare</span>(wordA, wordB))
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>they sound alike!<span class="pl-pds">'</span></span>);</pre>
         </div> 
         <p>The raw phonetics are obtained with <code>process()</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">metaphone</span>.<span class="pl-en">process</span>(<span class="pl-s"><span class="pl-pds">'</span>phonetics<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>A maximum code length can be supplied:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">metaphone</span>.<span class="pl-en">process</span>(<span class="pl-s"><span class="pl-pds">'</span>phonetics<span class="pl-pds">'</span></span>, <span class="pl-c1">3</span>));</pre>
         </div> 
         <p><code>DoubleMetaphone</code> deals with two encodings returned in an array. This feature is experimental and subject to change:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> dm <span class="pl-k">=</span> <span class="pl-smi">natural</span>.<span class="pl-smi">DoubleMetaphone</span>;

<span class="pl-k">var</span> encodings <span class="pl-k">=</span> <span class="pl-smi">dm</span>.<span class="pl-en">process</span>(<span class="pl-s"><span class="pl-pds">'</span>Matrix<span class="pl-pds">'</span></span>);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(encodings[<span class="pl-c1">0</span>]);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(encodings[<span class="pl-c1">1</span>]);</pre>
         </div> 
         <p>Attaching will patch String with useful methods:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">metaphone</span>.<span class="pl-en">attach</span>();</pre>
         </div> 
         <p><code>soundsLike</code> is essentially a shortcut to <code>Metaphone.compare</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">if</span>(<span class="pl-smi">wordA</span>.<span class="pl-en">soundsLike</span>(wordB))
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>they sound alike!<span class="pl-pds">'</span></span>);</pre>
         </div> 
         <p>The raw phonetics are obtained with <code>phonetics()</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>phonetics<span class="pl-pds">'</span></span>.<span class="pl-en">phonetics</span>());</pre>
         </div> 
         <p>Full text strings can be tokenized into arrays of phonetics (much like how tokenization-to-arrays works for stemmers):</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>phonetics rock<span class="pl-pds">'</span></span>.<span class="pl-en">tokenizeAndPhoneticize</span>());</pre>
         </div> 
         <p>Same module operations applied with <code>SoundEx</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">if</span>(<span class="pl-smi">soundEx</span>.<span class="pl-en">compare</span>(wordA, wordB))
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>they sound alike!<span class="pl-pds">'</span></span>);</pre>
         </div> 
         <p>The same String patches apply with <code>soundEx</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">soundEx</span>.<span class="pl-en">attach</span>();

<span class="pl-k">if</span>(<span class="pl-smi">wordA</span>.<span class="pl-en">soundsLike</span>(wordB))
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>they sound alike!<span class="pl-pds">'</span></span>);

<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>phonetics<span class="pl-pds">'</span></span>.<span class="pl-en">phonetics</span>());</pre>
         </div>

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Nouns</h3> 
         <p>Nouns can be pluralized/singularized with a <code>NounInflector</code>:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> nounInflector <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.NounInflector</span>();</pre>
         </div> 
         <p>To pluralize a word (outputs "radii"):</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">nounInflector</span>.<span class="pl-en">pluralize</span>(<span class="pl-s"><span class="pl-pds">'</span>radius<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>To singularize a word (outputs "beer"):</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">nounInflector</span>.<span class="pl-en">singularize</span>(<span class="pl-s"><span class="pl-pds">'</span>beers<span class="pl-pds">'</span></span>));</pre>
         </div> 
         <p>Like many of the other features, String can be patched to perform the operations directly. The "Noun" suffix on the methods is necessary, as verbs will be supported in the future.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">nounInflector</span>.<span class="pl-en">attach</span>();
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>radius<span class="pl-pds">'</span></span>.<span class="pl-en">pluralizeNoun</span>());
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>beers<span class="pl-pds">'</span></span>.<span class="pl-en">singularizeNoun</span>());</pre>
         </div>

         <h2><a id="user-content-tf-idf" class="anchor" href="#tf-idf" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>tf-idf</h2> 
         <p><a href="http://en.wikipedia.org/wiki/Tf%E2%80%93idf">Term Frequency–Inverse Document Frequency (tf-idf)</a> is implemented to determine how important a word (or words) is to a document relative to a corpus. The following example will add four documents to a corpus and determine the weight of the word "node" and then the weight of the word "ruby" in each document.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> TfIdf <span class="pl-k">=</span> <span class="pl-smi">natural</span>.<span class="pl-smi">TfIdf</span>;
<span class="pl-k">var</span> tfidf <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">TfIdf</span>();

<span class="pl-smi">tfidf</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>this document is about node.<span class="pl-pds">'</span></span>);
<span class="pl-smi">tfidf</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>this document is about ruby.<span class="pl-pds">'</span></span>);
<span class="pl-smi">tfidf</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>this document is about ruby and node.<span class="pl-pds">'</span></span>);
<span class="pl-smi">tfidf</span>.<span class="pl-en">addDocument</span>(<span class="pl-s"><span class="pl-pds">'</span>this document is about node. it has node examples<span class="pl-pds">'</span></span>);

<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>node --------------------------------<span class="pl-pds">'</span></span>);
<span class="pl-smi">tfidf</span>.<span class="pl-en">tfidfs</span>(<span class="pl-s"><span class="pl-pds">'</span>node<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">i</span>, <span class="pl-smi">measure</span>) {
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>document #<span class="pl-pds">'</span></span> <span class="pl-k">+</span> i <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">'</span> is <span class="pl-pds">'</span></span> <span class="pl-k">+</span> measure);
});

<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>ruby --------------------------------<span class="pl-pds">'</span></span>);
<span class="pl-smi">tfidf</span>.<span class="pl-en">tfidfs</span>(<span class="pl-s"><span class="pl-pds">'</span>ruby<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">i</span>, <span class="pl-smi">measure</span>) {
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">'</span>document #<span class="pl-pds">'</span></span> <span class="pl-k">+</span> i <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">'</span> is <span class="pl-pds">'</span></span> <span class="pl-k">+</span> measure);
});</pre>
         </div> 
         <p>The above outputs:</p> 
         <pre><code>node --------------------------------
document #0 is 1
document #1 is 0
document #2 is 1
document #3 is 2
ruby --------------------------------
document #0 is 0
document #1 is 1.2876820724517808
document #2 is 1.2876820724517808
document #3 is 0
</code></pre>

         <p>The following example measures the term "node" in the first and second documents.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">tfidf</span>.<span class="pl-en">tfidf</span>(<span class="pl-s"><span class="pl-pds">'</span>node<span class="pl-pds">'</span></span>, <span class="pl-c1">0</span>));

         </div>

<span class="pl-smi">tfidf</span>.<span class="pl-en">tfidfs</span>(<span class="pl-s"><span class="pl-pds">'</span>node ruby<span class="pl-pds">'</span></span>, <span class="pl-k">function</span>(<span class="pl-smi">i</span>, <span class="pl-smi">measure</span>) {

         <p>A probabilistic spellchecker based on <a href="http://norvig.com/spell-correct.html">http://norvig.com/spell-correct.html</a></p> 
         <p>This is best constructed with an array of tokens from a corpus, but a simple list of words from a dictionary will work.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> corpus <span class="pl-k">=</span> [<span class="pl-s"><span class="pl-pds">'</span>something<span class="pl-pds">'</span></span>, <span class="pl-s"><span class="pl-pds">'</span>soothing<span class="pl-pds">'</span></span>];

         <p>It uses the trie datastructure for fast boolean lookup of a word</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-smi">spellcheck</span>.<span class="pl-en">isCorrect</span>(<span class="pl-s"><span class="pl-pds">'</span>cat<span class="pl-pds">'</span></span>); <span class="pl-c"><span class="pl-c">//</span> false</span></pre>
         </div> 
         <p>It suggests corrections (sorted by probability in descending order) that are up to a maximum edit distance away from the input word. According to Norvig, a max distance of 1 will cover 80% to 95% of spelling mistakes. After a distance of 2, it becomes very slow.</p>

         <h3><a id="user-content-specifying-transformation-rules" class="anchor" href="#specifying-transformation-rules" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Specifying transformation rules</h3> 
         <p>Transformation rules are specified as follows:</p> 
         <pre><code>OLD_CAT NEW_CAT PREDICATE PARAMETER
</code></pre> 
         <p>This means that if the category of the current position is OLD_CAT and the predicate is true, the category is replaced by NEW_CAT. The predicate may use the parameter in different ways: sometimes the parameter is used for specifying the outcome of the predicate:</p> 
         <pre><code>NN CD CURRENT-WORD-IS-NUMBER YES
</code></pre> 
         <p>This means that if the outcome of predicate CURRENT-WORD-IS-NUMBER is YES, the category is replaced by <code>CD</code>. The parameter can also be used to check the category of a word in the sentence:</p> 
         <pre><code>VBD NN PREV-TAG DT
</code></pre> 
         <p>Here the category of the previous word must be <code>DT</code> for the rule to be applied.</p> 
         <h3><a id="user-content-algorithm" class="anchor" href="#algorithm" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Algorithm</h3> 
         <p>The tagger applies transformation rules that may change the category of words. The input sentence must be split into words which are assigned with categories. The tagged sentence is then processed from left to right. At each step all rules are applied once; rules are applied in the order in which they are specified. Algorithm:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">function</span>(<span class="pl-smi">sentence</span>) {
  <span class="pl-k">var</span> tagged_sentence <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">Array</span>(<span class="pl-smi">sentence</span>.<span class="pl-c1">length</span>);

  <span class="pl-c"><span class="pl-c">//</span> snip</span>

  <span class="pl-c"><span class="pl-c">//</span> Apply transformation rules</span>
  <span class="pl-k">for</span> (<span class="pl-k">var</span> i <span class="pl-k">=</span> <span class="pl-c1">0</span>, size <span class="pl-k">=</span> <span class="pl-smi">sentence</span>.<span class="pl-c1">length</span>; i <span class="pl-k">&lt;</span> size; i<span class="pl-k">++</span>) {
    <span class="pl-c1">this</span>.<span class="pl-smi">transformation_rules</span>.<span class="pl-c1">forEach</span>(<span class="pl-k">function</span>(<span class="pl-smi">rule</span>) {
      <span class="pl-smi">rule</span>.<span class="pl-c1">apply</span>(tagged_sentence, i);
    });
  }
  <span class="pl-k">return</span>(tagged_sentence);
}</pre>
         </div> 
         <h3><a id="user-content-adding-a-predicate" class="anchor" href="#adding-a-predicate" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Adding a predicate</h3> 
         <p>Predicates are defined in module <code>lib/RuleTemplates.js</code>. In that file predicate names are mapped to metadata for generaring transformation rules. The following properties must be supplied:</p> 
         <ul> 
          <li>Name of the predicate</li> 
          <li>A function that evaluates the predicate (should return a boolean)</li> 
          <li>A window <code>[i, j]</code> that defines the span of the predicate in the sentence relative to the current position</li> 
          <li>The number of parameter the predicate needs: 0, 1 or 2</li> 
          <li>If relevant, a function for parameter 1 that returns its possible values at the current position in the sentence (for generating rules in training)</li> 
          <li>If relevant, a function for parameter 2 that returns its possible values at the current position in the sentence (for training)</li> 
         </ul> 
         <p>A typical entry for a rule templates looks like this:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-s"><span class="pl-pds">"</span>NEXT-TAG<span class="pl-pds">"</span></span><span class="pl-k">:</span> {
    <span class="pl-c"><span class="pl-c">//</span> maps to the predicate function</span>
    <span class="pl-s"><span class="pl-pds">"</span>function<span class="pl-pds">"</span></span><span class="pl-k">:</span> next_tag_is,
    <span class="pl-c"><span class="pl-c">//</span> Minimum required window before or after current position to be a relevant predicate</span>
    <span class="pl-s"><span class="pl-pds">"</span>window<span class="pl-pds">"</span></span><span class="pl-k">:</span> [<span class="pl-c1">0</span>, <span class="pl-c1">1</span>],
    <span class="pl-c"><span class="pl-c">//</span> The number of parameters the predicate takes</span>
    <span class="pl-s"><span class="pl-pds">"</span>nrParameters<span class="pl-pds">"</span></span><span class="pl-k">:</span> <span class="pl-c1">1</span>,
    <span class="pl-c"><span class="pl-c">//</span> Function that returns relevant values for parameter 1</span>
    <span class="pl-s"><span class="pl-pds">"</span>parameter1Values<span class="pl-pds">"</span></span><span class="pl-k">:</span> nextTagParameterValues
  }</pre>
         </div> 
         <p>A predicate function accepts a tagged sentence, the current position in the sentence that should be tagged, and the outcome(s) of the predicate. An example of a predicate that checks the category of the current word:</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">function</span> <span class="pl-en">next_tag_is</span>(<span class="pl-smi">tagged_sentence</span>, <span class="pl-smi">i</span>, <span class="pl-smi">parameter</span>) {
  <span class="pl-k">if</span> (i <span class="pl-k">&lt;</span> <span class="pl-smi">tagged_sentence</span>.<span class="pl-c1">length</span> <span class="pl-k">-</span> <span class="pl-c1">1</span>) {
    <span class="pl-k">return</span>(tagged_sentence[i<span class="pl-k">+</span><span class="pl-c1">1</span>][<span class="pl-c1">1</span>] <span class="pl-k">===</span> parameter);
  }
  <span class="pl-k">else</span> {
    <span class="pl-k">return</span>(<span class="pl-c1">false</span>);
  }
}</pre>
         </div> 
         <p>A values function for a parameter returns an array all possible parameter values given a location in a tagged sentence.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">function</span> <span class="pl-en">nextTagParameterValues</span>(<span class="pl-smi">sentence</span>, <span class="pl-smi">i</span>) {
  <span class="pl-k">if</span> (i <span class="pl-k">&lt;</span> <span class="pl-smi">sentence</span>.<span class="pl-c1">length</span> <span class="pl-k">-</span> <span class="pl-c1">1</span>) {
    <span class="pl-k">return</span> [sentence[i <span class="pl-k">+</span> <span class="pl-c1">1</span>].<span class="pl-smi">tag</span>];
  }
  <span class="pl-k">else</span> {
    <span class="pl-k">return</span> [];
  }
}</pre>
         </div> 
         <p>Please note that these functions work with a different data type. Here, a sentence is an array of tokens and tokens are maps that have at least a token (word) and a tag.</p> 
         <h3><a id="user-content-training" class="anchor" href="#training" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Training</h3> 
         <p>The trainer allows to learn a new set of transformation rules from a corpus. It takes as input a tagged corpus and a set of rule templates. The algorithm generates positive rules (rules that apply at some location in the corpus) from the templates and iteratively extends and optimises the rule set.</p> 
         <p>First, a corpus should be loaded. Currently, the format of Brown corpus is supported. Then a lexicon can be created from the corpus. The lexicon is needed for tagging the sentences before the learning algorithm is applied.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-k">=</span> <span class="pl-c1">require</span>(natural);
<span class="pl-k">var</span> text <span class="pl-k">=</span> <span class="pl-smi">fs</span>.<span class="pl-en">readFileSync</span>(brownCorpusFile, <span class="pl-s"><span class="pl-pds">'</span>utf8<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> corpus <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.Corpus</span>(text, <span class="pl-c1">1</span>);
<span class="pl-k">var</span> lexicon <span class="pl-k">=</span> <span class="pl-smi">corpus</span>.<span class="pl-en">buildLexicon</span>();</pre>
         </div> 
         <p>The next step is to create a set of rule templates from which the learning algorithm can generate transformation rules. Rule templates are defined in <code>PredicateMapping.js</code>.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> templateNames <span class="pl-k">=</span> [
  <span class="pl-s"><span class="pl-pds">"</span>NEXT-TAG<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>NEXT-WORD-IS-CAP<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>PREV-1-OR-2-OR-3-TAG<span class="pl-pds">"</span></span>,
  <span class="pl-s"><span class="pl-pds">"</span>...<span class="pl-pds">"</span></span>,
];
<span class="pl-k">var</span> templates <span class="pl-k">=</span> <span class="pl-smi">templateNames</span>.<span class="pl-en">map</span>(<span class="pl-k">function</span>(<span class="pl-smi">name</span>) {
  <span class="pl-k">return</span> <span class="pl-k">new</span> <span class="pl-en">natural.RuleTemplate</span>(name);
});</pre>
         </div> 
         <p>Using lexicon and rule templates we can now start the trainer as follows.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> natural <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> Tester <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>natural.BrillPOSTrainer<span class="pl-pds">'</span></span>);
<span class="pl-k">var</span> trainer <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">Trainer</span>(<span class="pl-c"><span class="pl-c">/*</span> optional threshold <span class="pl-c">*/</span></span>);
<span class="pl-k">var</span> ruleSet <span class="pl-k">=</span> <span class="pl-smi">trainer</span>.<span class="pl-en">train</span>(corpus, templates, lexicon);</pre>
         </div> 
         <p>A threshold value can be passed to constructor. Transformation rules with a score below the threshold are removed after training. The train method returns a set of transformation rules that can be used to create a POS tagger as usual. Also you can output the rule set in the right format for later usage.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-smi">ruleSet</span>.<span class="pl-en">prettyPrint</span>());</pre>
         </div>

           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Testing</h3> 
         <p>Now we can apply the lexicon and rule set to a test set.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">var</span> tester <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.BrillPOSTester</span>();
<span class="pl-k">var</span> tagger <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">natural.BrillPOSTagger</span>(lexicon, ruleSet);
<span class="pl-k">var</span> scores <span class="pl-k">=</span> <span class="pl-smi">tester</span>.<span class="pl-c1">test</span>(corpora[<span class="pl-c1">1</span>], tagger);</pre>
         </div> 
         <p>The test method returns an array of two percentages: first percentage is the ratio of right tags after tagging with the lexicon; second percentage is the ratio of right tags after applying the transformation rules.</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>Test score lexicon <span class="pl-pds">"</span></span> <span class="pl-k">+</span> scores[<span class="pl-c1">0</span>] <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">"</span>%<span class="pl-pds">"</span></span>);
<span class="pl-en">console</span>.<span class="pl-c1">log</span>(<span class="pl-s"><span class="pl-pds">"</span>Test score after applying rules <span class="pl-pds">"</span></span> <span class="pl-k">+</span> scores[<span class="pl-c1">1</span>] <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">"</span>%<span class="pl-pds">"</span></span>);</pre>
         </div>
