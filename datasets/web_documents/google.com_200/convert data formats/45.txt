       </svg> </button> How To Work with JSON in JavaScript </h1> 
     <span class="meta-section timestamp"><span class="tutorial-date-text">Posted</span><span class="tutorial-date">December 9, 2016</span></span> 
     <span class="meta-section pageviews"><span class="icon icon-eye v-mid"></span><span class="views-count v-mid">46.9k</span><span class="sr-only"> views</span></span> 
     <span class="meta-section tags"> <a class="dev-tag" href="/community/tags/development?type=tutorials">Development</a> <a class="tag" href="/community/tags/javascript?type=tutorials">JavaScript</a> </span>

     <h3 id="introduction">Introduction</h3> 
     <p>Because <strong>JSON</strong> is derived from the JavaScript programming language, it is a natural choice to use as a data format in JavaScript. JSON, short for <strong>JavaScript Object Notation</strong>, is usually pronounced like the name “Jason.”</p> 
     <p>To learn more about JSON in general terms, read the “<a href="https://www.digitalocean.com/community/tutorials/an-introduction-to-json">An Introduction to JSON</a>” tutorial.</p> 
     <p>To begin thinking about where you may use JSON in your JavaScript programs, some general use cases of JSON include:</p> 
     <ul> 
      <li>Storing data</li> 
      <li>Generating data structures from user input</li> 
      <li>Transferring data from server to client, client to server, and server to server</li> 
      <li>Configuring and verifying data</li> 
     </ul> 
     <p>This tutorial will provide you with an introduction to working with JSON in JavaScript. To make the most use of this introduction, you should have some familiarity with the JavaScript programming language.</p> 
     <h2 id="json-format">JSON Format</h2> 
     <p>JSON’s format is derived from JavaScript object syntax, but it is entirely text-based. It is a key-value data format that is typically rendered in curly braces. </p> 
     <p>When you’re working with JSON, you’ll likely see JSON objects in a <code>.json</code> file, but they can also exist as a JSON object or string within the context of a program. Read more about the <a href="https://www.digitalocean.com/community/tutorials/an-introduction-to-json#syntax-and-structure">syntax and structure here</a>. </p> 
     <p>When you’re working with a <code>.json</code> file, it will look like this:</p> 
     <div class="code-label " title="sammy.json">
      sammy.json
     </div>
     <pre class="code-pre "><code class="code-highlight language-javascript">{ 
  "first_name"  :  "Sammy", 
  "last_name"   :  "Shark", 
  "online"      :  true 
}
</code></pre> 
     <p>If, instead, you have a JSON object in a <code>.js</code> or <code>.html</code> file, you’ll likely see it set to a variable:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var sammy = { 
  "first_name"  :  "Sammy", 
  "last_name"   :  "Shark", 
  "online"      :  true 
}
</code></pre> 
     <p>Additionally, you may see JSON as a string rather than an object within the context of a JavaScript program file or script. In this case, you may also see it all on one line:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var sammy = '{"first_name" : "Sammy", "last_name" : "Shark", "location" : "Ocean"}';
</code></pre> 
     <p>Converting JSON objects into strings can be particularly useful for transporting data in a quick manner. </p> 
     <p>We’ve gone over the general format of JSON and how you may expect to see it as a <code>.json</code> file, or within JavaScript as an object or a string.</p> 
     <h2 id="comparison-to-javascript-object">Comparison to JavaScript Object</h2> 
     <p>It is worth keeping in mind that JSON was developed to be used by any programming language, while JavaScript objects can only be worked with directly through the JavaScript programming language.</p> 
     <p>In terms of syntax, JavaScript objects are similar to JSON, but the keys in JavaScript objects are not strings in quotes. Also, JavaScript objects are less limited in terms of types passed to values, so they can use functions as values. </p> 
     <p>Let’s look at an example of a JavaScript object of the website user Sammy Shark who is currently online. </p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var user = {
    first_name: "Sammy",
    last_name : "Shark",
    online    : true,
    full_name : function() {
       return this.first_name + " " + this.last_name;
    }
};
</code></pre> 
     <p>This will look very familiar to you as a JSON object, but there are no quotes around any of the keys (<code>first_name</code>, <code>last_name</code>, <code>online</code>, or <code>full_name</code>), <strong>and</strong> there is a function value in the last line.</p> 
     <p>If we want to access the data in the JavaScript object above, we could use <a href="https://www.digitalocean.com/community/tutorials/how-to-work-with-json-in-javascript#accessing-json-data">dot notation</a> to call <code>user.first_name;</code> and get a string, but if we want to access the full name, we would need to do so by calling <code>user.full_name();</code> because it is a function.</p> 
     <p>JavaScript objects can only exist within the JavaScript language, so when you’re working with data that needs to be accessed by various languages, it is best to opt for JSON.</p> 
     <h2 id="accessing-json-data">Accessing JSON Data</h2> 
     <p>JSON data is normally accessed in Javascript through dot notation. To understand how this works, let’s consider the JSON object <code>sammy</code>:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var sammy = { 
  "first_name"  :  "Sammy", 
  "last_name"   :  "Shark", 
  "online"      :  true 
}
</code></pre> 
     <p>In order to access any of the values, we’ll be using dot notation that looks like this:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">sammy.first_name
sammy.last_name
sammy.online
</code></pre> 
     <p>The variable <code>sammy</code> is first, followed by a dot, followed by the key to be accessed. </p> 
     <p>To create a JavaScript alert that shows us the value associated with the key <code>first_name</code> in a pop-up, we can do so by calling the JavaScript <code>alert()</code> function:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">alert(sammy.first_name);
</code></pre>
     <pre class="code-pre "><code langs="">
       <div class="secondary-code-label " title="Output">
        Output
       </div>Sammy
</code></pre> 
     <p>Here, we’ve successfully called the value associated with the <code>first_name</code> key from the <code>sammy</code> JSON object.</p> 
     <p>We can also use square bracket syntax to access data from JSON. To do that, we would keep the key in double quotes within square brackets. For our <code>sammy</code> variable above, using square bracket syntax in an <code>alert()</code> function looks like this:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">alert(sammy["online"]);
</code></pre>
     <pre class="code-pre "><code langs="">
       <div class="secondary-code-label " title="Output">
        Output
       </div>true
</code></pre> 
     <p>When you’re working with <a href="https://www.digitalocean.com/community/tutorials/an-introduction-to-json#nested-arrays">nested array elements</a>, you should call the number of the item in your array. Let’s consider the JSON below:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var user_profile = { 
  "username" : "SammyShark",
  "social_media" : [
    {
      "description" : "twitter",
      "link" : "https://twitter.com/digitalocean"
    },
    {
      "description" : "facebook",
      "link" : "https://www.facebook.com/DigitalOceanCloudHosting"
    },
    {
      "description" : "github",
      "link" : "https://github.com/digitalocean"
    }
  ]
}
</code></pre> 
     <p>To access the string <code>facebook</code>, we can call that item in the array within the context of dot notation:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">alert(user_profile.social_media[1].description);
</code></pre>
     <pre class="code-pre "><code langs="">
       <div class="secondary-code-label " title="Output">
        Output
       </div>facebook
</code></pre> 
     <p>Notice that for each nested element we’ll use an additional dot.</p> 
     <p>Using dot notation or square bracket syntax allows us to access the data contained in JSON format. </p> 
     <h2 id="functions-for-working-with-json">Functions for Working with JSON</h2> 
     <p>This section will look at two methods for stringifying and parsing JSON. Being able to convert JSON from object to string and vice versa is useful for transferring and storing data. </p> 
     <h3 id="json-stringify">JSON.stringify()</h3> 
     <p>The <code>JSON.stringify()</code> function converts an object to a JSON string. </p> 
     <p>Strings are useful for transporting data from a client to a server through storing or passing information in a lightweight way. For example, you may gather a user’s settings on the client side and then send them to a server. Later, you can then read the information with the <code>JSON.parse()</code> method and work with the data as needed.</p> 
     <p>We’ll look at a JSON object that we assign to the variable <code>obj</code>, and then we’ll convert it using <code>JSON.stringify()</code> by passing <code>obj</code> to the function. We can assign this string to the variable <code>s</code>:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var obj = {"first_name" : "Sammy", "last_name" : "Shark", "location" : "Ocean"}

var s = JSON.stringify(obj)
</code></pre> 
     <p>Now, if we work with <code>s</code>, we’ll have the JSON available to us as a string rather than an object.</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">'{"first_name" : "Sammy", "last_name" : "Shark", "location" : "Ocean"}'
</code></pre> 
     <p>The <code>JSON.stringify()</code> function lets us convert objects to strings. To do the opposite, we’ll look at the <code>JSON.parse()</code> function.</p> 
     <h3 id="json-parse">JSON.parse()</h3> 
     <p>Strings are useful for transporting but you’ll want to be able to convert them back to a JSON object on the client and/or the server side. While you can convert text to an object with the <code>eval()</code> function, it is not very secure, so we’ll use the <code>JSON.parse()</code> function instead.</p> 
     <p>To convert the example in the <a href="https://www.digitalocean.com/community/tutorials/how-to-work-with-json-in-javascript/#functions-for-working-with-json">JSON.stringify()</a> section above, we would pass the string <code>s</code> to the function, and assign it to a new variable:</p> 
     <pre class="code-pre "><code class="code-highlight language-javascript">var o = JSON.parse(s)
</code></pre> 
     <p>Then, we would have the object <code>o</code> to work with, which would be identical to the object <code>obj</code>.</p> 
     <p>To take a deeper look, let’s consider an example of <code>JSON.parse()</code> within the context of an HTML file:</p> 
     <pre class="code-pre "><code class="code-highlight language-html">&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;body&gt;

&lt;p id="user"&gt;&lt;/p&gt;

&lt;script&gt;
var s = '{"first_name" : "Sammy", "last_name" : "Shark", "location" : "Ocean"}';

var obj = JSON.parse(s);

document.getElementById("user").innerHTML =
"Name: " + obj.first_name + " " + obj.last_name + "&lt;br&gt;" +
"Location: " + obj.location;
&lt;/script&gt;

&lt;/body&gt;
&lt;/html&gt;
</code></pre>
     <pre class="code-pre "><code langs="">
       <div class="secondary-code-label " title="Output">
        Output
       </div>Name: Sammy Shark
Location: Ocean
</code></pre> 
     <p>Within the context of an HTML file, we can see how the JSON string <code>s</code> is converted to an object that is retrievable on the final rendering of the page by <a href="https://www.digitalocean.com/community/tutorials/how-to-work-with-json-in-javascript#accessing-json-data">accessing the JSON</a> via dot notation.</p> 
     <p><code>JSON.parse()</code> is a secure function to parse JSON strings and convert them to objects.</p> 
     <h2 id="conclusion">Conclusion</h2> 
     <p>JSON is a natural format to use in JavaScript and has many implementations available for use in many popular programming languages. If you want to use the format in another progamming language, you can see full language support on the “<a href="http://www.json.org/">Introducing JSON</a>” site.</p> 
     <p>Because it is lightweight and is readily transferred between programming languages and systems, JSON has been experiencing increased support in APIs, including the <a href="https://www.digitalocean.com/community/tutorials/how-to-create-a-twitter-app">Twitter API</a>. </p> 
     <p>You likely won’t be creating your own <code>.json</code> files but procuring them from other sources. You can check out these <a href="https://www.digitalocean.com/community/tutorials/an-introduction-to-json#resources">resources</a> to learn about converting other data structures to JSON.</p>
