     <h1 class="title" id="page-title">Using LiveValidation JavaScript library</h1> 
     <div class="content"> 
      <article class="node-6023 node node-documentation view-mode-full no-comments clearfix"> 
       <header> 
        <p class="submitted"> PUBLISHED <time pubdate datetime="2015-07-27T19:47:35-07:00">27 Jul 2015</time> </p> 
       </header> 
       <div class="field field-name-body field-type-text-with-summary field-label-hidden">
        <div class="field-items">
         <div class="field-item even">
          <p>This article describes the LiveValidation JavaScript library and how to use it to validate forms filled out by users in real-time.</p> 
          <h2><span>Introduction</span></h2> 
          <p>LiveValidation is a lightweight JavaScript library for making real-time validations. It controls the form fields while the user is typing. The purpose of the library is to save the users' time and make the filling of forms' fields less daunting for them. A JavaScript validator doesn't replace the server-side validation that should be performed all the same.&nbsp;</p> 
          <p>The library makes validation easy, quick and powerful. Two versions of LiveValidation library are available: stand-alone (pure JavaScript) and Prototype. You can download them from <a href="https://github.com/alechill/livevalidation">this page</a>. In this article we'll focus on the stand-alone version.</p> 
          <p>The library's documentation is available <a href="http://livevalidation.com/documentation">HERE</a>. Some examples of its use can be found <a href="http://livevalidation.com/examples">HERE</a>.</p> 
          <h2><span>How to work with the LiveValidation library</span></h2> 
          <ol> 
           <li><span>First, we create an element that we want to validate:</span></li> 
          </ol> 
          <pre class="prettyprint lang-html " data-pbcklang="html">
&lt;!-- index.html file --&gt;
&lt;label for="nick-input"&gt;Nick:&lt;/label&gt;
&lt;input type="text" id="nick-input" value="" class="LV_valid_field" /&gt;</pre>
          <p>Add the "LV_valid_field" class to an &lt;input&gt; element. As a user type, the field is validated. If the field's content doesn't meet the defined requirements, the element's class changes on the “LV_invalid_field”. It's important to define a CSS style for these two classes, e.g.&nbsp;</p> 
          <pre class="prettyprint lang-css " data-pbcklang="css">
/* validation.css file */
.LV_valid_field {
    border: 1px solid rgba(0, 180, 0, 0.75);
}
.LV_invalid_field {
    border: 1px solid rgba(240, 0, 0, 0.75);
}</pre>
          <p><span>2. Next, we create a LiveValidation object instance for each element, e.g.</span></p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
var validatedObj = new LiveValidation('nick-input', { validMessage: "Valid!"});</pre>
          <p>The <a href="http://livevalidation.com/documentation#LiveValidationConstructor">LiveValidation constructor</a> takes as arguments: the string id of the element to validate and (optionally) an object containing options that can override <a href="http://livevalidation.com/documentation#LiveValidationConstructor">defaults</a>: validMessage, onValid, onInvalid, insertAfterWhatNode, onlyOnBlur, wait, onlyOnSubmit. The validMessage parameter means a message that is displayed if validation is successful (“Thankyou!” by default).</p> 
          <p>3. When we have the LiveValidation object, we can use the <a href="http://livevalidation.com/documentation#LiveValidationAdd">add()</a> function to apply a suitable validation method and object containing its parameters that can override defaults:</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.method, methodParam);</pre>
          <p>The Validate class contains methods responsible for performing different types of validation. The add() function returns “true” or throws a Validate.Error object - that contains a failure message that is displayed as shown below (“Not valid!” message):</p> 
          <p>We can change the style of this message by defining the LV_valid, LV_invalid and LV_validation_message. It's required to use these strings for class names.&nbsp;</p> 
          <pre class="prettyprint lang-css " data-pbcklang="css">
/* validation.css */
.LV_valid {
    color:rgba(0, 180, 0, 0.75);
}
.LV_invalid {
    color:rgba(240, 0, 0, 0.75);
}
.LV_validation_message{
    margin:0 0 0 2px;
}</pre>
          <p>Now, let’s describe the available validation methods. Their names are hyper-linked to the correct fragments of the documentation.</p> 
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidatePresence">Presence</a>&nbsp;</h3> 
          <p>The method checks if a validated field contains a value different from an empty string, null, undefined, etc.</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.Presence);</pre>
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateFormat">Format</a>&nbsp;</h3> 
          <p>The method can validate a value (typed by a user) against some regular expression.</p> 
          <p>Let us assume that we want the user's name to start with a letter, contain letters and numeric values and have a length of 8-16 signs. Using a regular expression we can write this as:</p> 
          <pre class="prettyprint lang-json " data-pbcklang="json">
/^[a-zA-Z][0-9a-zA-Z]{7,15}$/</pre>
          <p>Now, we can use this pattern as parameter for the Validate.Format() method:</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.Format, {pattern: /^[a-zA-Z][0-9a-zA-Z]{7,15}$/});</pre>
          <p>As you can see – validation works.</p> 
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateNumericality">Numericality</a>&nbsp;</h3> 
          <p>The method is used to check if a validated value is numeric and whether it is:</p> 
          <ul> 
           <li>an integer,</li> 
           <li>a specific number,</li> 
           <li>a number bigger/smaller than specified,</li> 
           <li>within a range of numbers,</li> 
          </ul> 
          <p>or a combination of the above.</p> 
          <p>We want to check if the user types an integer within a range (1, 30). We do it as follows:</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.Numericality, { minimum: 1, maximum: 30, onlyInteger: true });</pre>
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateLength">Length</a>&nbsp;</h3> 
          <p>The length of a typed value can also be validated using the Length() method. You can set the minimum, maximum or the range of a value's length (minimum AND maximum).</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.Length, {minimum: 3, maximum: 25});</pre>
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateAcceptance">Acceptance</a>&nbsp;</h3> 
          <p>The method checks if a value is equal to “true”. Very useful while checking if a user accepted some terms, etc.</p> 
          <pre class="prettyprint lang-html " data-pbcklang="html">
&lt;!-- index.html file --&gt;
&lt;label for="accept-input"&gt;Accept:&lt;/label&gt;
&lt;input type="checkbox" id="accept-input" value="1" class="checkbox" /&gt;</pre>
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
var validatedObj = new LiveValidation('accept-input', { validMessage: "OK!"});
validatedObj.add(Validate.Acceptance);</pre>
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateConfirmation">Confirmation</a>&nbsp;</h3> 
          <p>The method validates if a specified value matches the given one. It can be very useful e.g. in case of password confirmation. It's mandatory to set the match parameter of the method.</p> 
          <pre class="prettyprint lang-html " data-pbcklang="html">
&lt;label for="pass-input"&gt;Password:&lt;/label&gt;
&lt;input type="text" id="pass-input" value="" class="LV_valid_field" /&gt;
&lt;label for="confpass-input"&gt;Confirm password:&lt;/label&gt;
&lt;input type="text" id="confpass-input" value="" class="LV_valid_field" /&gt;</pre>
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
var validatedObj = new LiveValidation('confpass-input', { validMessage: "Valid!"});
validatedObj.add(Validate.Confirmation, { match: 'pass-input' });</pre>
          <p>&nbsp;</p> 
          <h3><a href="http://livevalidation.com/documentation#ValidateEmail">Email</a></h3> 
          <p>The method validates if an email address typed by the user is valid.</p> 
          <pre class="prettyprint lang-javascript " data-pbcklang="javascript">
validatedObj.add(Validate.Email);</pre>
          <p>&nbsp;</p> 
          <p>There are some other useful validation methods like <a href="http://livevalidation.com/documentation#ValidateCustom">Custom</a>, <a href="http://livevalidation.com/documentation#ValidateInclusion">Inclusion</a>, <a href="http://livevalidation.com/documentation#ValidateExclusion">Exclusion</a>, <a href="http://livevalidation.com/documentation#ValidateNow">now</a>.&nbsp;</p> 
          <h2>Sample application&nbsp;</h2> 
          <p>Our sample application uses the standalone version of the LiveValidation library.&nbsp;</p> 
          <p align="center"><img alt="" height="500" src="/sites/default/files/users/user-1451/lv001.png" width="286"></p> 
          <p align="center"><em>Figure 1 LiveValidation sample application screen shot</em></p> 
          <h2>Summary</h2> 
          <p>We hope this article has shown how you can use the LiveValidation library in your Tizen Web application. Use it to make your form's validation more easy, quick and powerful.</p>
