       <h1 class="title">Walkthrough: Service a Virtual Hard Disk Image Offline</h1> 
       <p>Published: October 22, 2009</p>
       <p>Updated: October 22, 2009</p>
       <p>Applies To: Windows 7, Windows Server 2008 R2</p>

         <p>A virtual hard disk (VHD) with an applied Windows®&nbsp;7 image can be serviced offline or <a href="#BKMK_serviceOnline">Service an Online VHD</a> with command-line tools such as Deployment Image Servicing and Management (DISM.exe). For more information about creating a VHD and applying a Windows&nbsp;7 image, see <a href="https://technet.microsoft.com/en-us/library/dd744338(v=ws.10).aspx">Walkthrough: Deploy a Virtual Hard Disk for Native Boot</a>.</p> 
         <div class="alert"> 
          <table summary="table"> 
           <tbody>
            <tr>

            </tr> 
            <tr> 
             <td> DISM operations performed on a VHD cannot be discarded by using the <strong>/unmount</strong> option, as they can with .wim files. It is recommended that you create a backup copy of the .vhd file before servicing it.<p></p></td>

         <h2 class="heading">Terms</h2> 
         <div id="sectionSection0" class="section"> 
          <p> <strong>attach</strong> </p> 
          <p>You can map the contents of a Windows&nbsp;7 image in a VHD to a directory by attaching a vdisk using the Windows&nbsp;7 DiskPart tool. Mapping the contents of an image to a directory is similar to mounting the image. The <strong>Attach vdisk</strong> command adds the .vhd file as a disk to the storage controller on the host.</p> 
          <p> <strong>offline</strong> </p> 
          <p>Offline servicing occurs when you modify or service a Windows® image entirely offline without booting it first.</p> 
          <p> <strong>vdisk</strong> </p> 
          <p>A virtual disk, as referred to in the DiskPart tool.</p> 
         </div>

         <div id="sectionSection1" class="section">

          <h3 class="subHeading">Add a local pack</h3>
