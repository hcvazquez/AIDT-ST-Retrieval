           <h1 class="css-1a9m1x6">Integrating with Other Libraries</h1>
          </header>
          <div class="css-124oy3v">
           <div class="css-1nw8hwm">
            <p>React can be used in any web application. It can be embedded in other applications and, with a little care, other applications can be embedded in React. This guide will examine some of the more common use cases, focusing on integration with <a href="https://jquery.com/">jQuery</a> and <a href="http://backbonejs.org/">Backbone</a>, but the same ideas can be applied to integrating components with any existing code.</p> 
            <h2 id="integrating-with-dom-manipulation-plugins"><a href="#integrating-with-dom-manipulation-plugins" aria-hidden="true" class="anchor">
              <svg aria-hidden="true" height="16" version="1.1" viewbox="0 0 16 16" width="16">
               <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
              </svg></a>Integrating with DOM Manipulation Plugins</h2> 
            <p>React is unaware of changes made to the DOM outside of React. It determines updates based on its own internal representation, and if the same DOM nodes are manipulated by another library, React gets confused and has no way to recover.</p> 
            <p>This does not mean it is impossible or even necessarily difficult to combine React with other ways of affecting the DOM, you just have to be mindful of what each are doing.</p> 
            <p>The easiest way to avoid conflicts is to prevent the React component from updating. You can do this by rendering elements that React has no reason to update, like an empty <code>&lt;div /&gt;</code>.</p> 
            <h3 id="how-to-approach-the-problem"><a href="#how-to-approach-the-problem" aria-hidden="true" class="anchor">
              <svg aria-hidden="true" height="16" version="1.1" viewbox="0 0 16 16" width="16">
               <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
              </svg></a>How to Approach the Problem</h3> 
            <p>To demonstrate this, lets sketch out a wrapper for a generic jQuery plugin.</p> 
            <p>We will attach a <a href="/docs/refs-and-the-dom.html">ref</a> to the root DOM element. Inside <code>componentDidMount</code>, we will get a reference to it so we can pass it to the jQuery plugin.</p> 
            <p>To prevent React from touching the DOM after mounting, we will return an empty <code>&lt;div /&gt;</code> from the <code>render()</code> method. The <code>&lt;div /&gt;</code> element has no properties or children, so React has no reason to update it, leaving the jQuery plugin free to manage that part of the DOM:</p> 
            <div class="gatsby-highlight"> 
             <pre class="gatsby-code-jsx"><code><span class="token keyword">class</span> <span class="token class-name">SomePlugin</span> <span class="token keyword">extends</span> <span class="token class-name">React<span class="token punctuation">.</span>Component</span> <span class="token punctuation">{</span>
  <span class="token function">componentDidMount</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span>
<span class="gatsby-highlight-code-line"> <span class="token keyword">this</span><span class="token punctuation">.</span>$el <span class="token operator">=</span> <span class="token function">$</span><span class="token punctuation">(</span><span class="token keyword">this</span><span class="token punctuation">.</span>el<span class="token punctuation">)</span><span class="token punctuation">;</span> </span><span class="gatsby-highlight-code-line"> <span class="token keyword">this</span><span class="token punctuation">.</span>$el<span class="token punctuation">.</span><span class="token function">somePlugin</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span> </span>  <span class="token punctuation">}</span>

  <span class="token function">render</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{</span>
<span class="gatsby-highlight-code-line"> <span class="token keyword">return</span> <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>div</span> <span class="token attr-name">ref</span><span class="token script language-javascript"><span class="token punctuation">=</span><span class="token punctuation">{</span>el <span class="token operator">=&gt;</span> <span class="token keyword">this</span><span class="token punctuation">.</span>el <span class="token operator">=</span> el<span class="token punctuation">}</span></span> <span class="token punctuation">/&gt;</span></span><span class="token punctuation">;</span> </span>  <span class="token punctuation">}</span>

            <p>Note that we defined both <code>componentDidMount</code> and <code>componentWillUnmount</code> <a href="/docs/react-component.html#the-component-lifecycle">lifecycle hooks</a>. Many jQuery plugins attach event listeners to the DOM so its important to detach them in <code>componentWillUnmount</code>. If the plugin does not provide a method for cleanup, you will probably have to provide your own, remembering to remove any event listeners the plugin registered to prevent memory leaks.</p>

            <p>React can be embedded into other applications thanks to the flexibility of <a href="/docs/react-dom.html#render"><code>ReactDOM.render()</code></a>.</p> 
            <p>Although React is commonly used at startup to load a single root React component into the DOM, <code>ReactDOM.render()</code> can also be called multiple times for independent parts of the UI which can be as small as a button, or as large as an app.</p> 
            <p>In fact, this is exactly how React is used at Facebook. This lets us write applications in React piece by piece, and combine it with our existing server-generated templates and other client-side code.</p>

            <p>The approach above requires your React components to be aware of the Backbone models and collections. If you later plan to migrate to another data management solution, you might want to concentrate the knowledge about Backbone in as few parts of the code as possible.</p> 
            <p>One solution to this is to extract the models attributes as plain data whenever it changes, and keep this logic in a single place. The following is <a href="/docs/higher-order-components.html">a higher-order component</a> that extracts all attributes of a Backbone model into state, passing the data to the wrapped component.</p> 
            <p>This way, only the higher-order component needs to know about Backbone model internals, and most components in the app can stay agnostic of Backbone.</p> 
            <p>In the example below, we will make a copy of the models attributes to form the initial state. We subscribe to the <code>change</code> event (and unsubscribe on unmounting), and when it happens, we update the state with the models current attributes. Finally, we make sure that if the <code>model</code> prop itself changes, we dont forget to unsubscribe from the old model, and subscribe to the new one.</p>
