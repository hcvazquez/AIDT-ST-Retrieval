        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Specifies the output format of image data. Binary output yields smaller files, but not all workflows can accommodate binary output.</p>

        <div class="variable-text">
         <p class="text"></p>
         <p>Specifies the level of PostScript compatibility. Use <span class="uicontrol">Language Level</span> 3 only if the target output device supports it. <span class="uicontrol">Language Level</span> 2 is suitable for EPS files that are placed in another document and color-separated as part of that document. Use <span class="uicontrol">Language Level</span> 2 for EPS files that you import into Microsoft applications.</p> 
         <p></p>
        </div>

        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Specifies the fonts to be included in the PostScript. Embedded fonts are taken from the PDF; the referenced fonts are taken from the computer in use.</p>

         <p></p>
        </div> 
       </div>

        <div class="variable-text">

In Acrobat, go to&nbsp;<span class="uicontrol">Tools&nbsp;</span>&gt;&nbsp;<span class="uicontrol">Export PDF</span> &gt; <span class="uicontrol">More Formats</span> &nbsp;and select&nbsp;<span class="uicontrol">Text (Acessible)</span> or <span class="uicontrol">Text (Plain)</span> or <span class="uicontrol">XML 1.0</span> as required.</p>

        <div class="header  "> 
         <h4 id="OutputFileSettings"> Output File Settings </h4> 
        </div> 
        <!-- for Cascading Toc --> 
       </div> 
       <div class="variable parbase section"> 
        <!-- Commented out by cmiqueo --> 
        <!-- <a name="Encoding" href="./"></a><div class="variable-title">Encoding</div> --> 
        <a name="Encoding"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">Encoding</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Refers to the binary values, based on international standards, used to represent the text characters. UTF-8 is a Unicode representation of characters using one or more 8-bit bytes per character; UTF-16 represents characters using 16-bit bytes. ISO-Latin-1 is an 8-bit representation of characters that is a superset of ASCII. UCS-4 is a <span class="uicontrol">Universal Character Set</span> coded in four octets. HTML/ASCII is a 7-bit representation of characters developed by ANSI.</p> 
         <p><span class="uicontrol"><strong>Use Mapping Table Default</strong></span> uses the default character encoding defined in mapping tables, which appear in the Plug-ins/SaveAsXML/MappingTables folder. These mapping tables specify many characteristics of how the data is output, including the following default character encodings: UTF-8 (Save as XML or HTML 4.0.1) and HTML/ASCII (Save as HTML 3.2).</p> 
         <p></p>
        </div> 
       </div> 
       <div class="variable parbase section"> 
        <!-- Commented out by cmiqueo --> 
        <!-- <a name="Generate Bookmarks" href="./"></a><div class="variable-title">Generate Bookmarks</div> --> 
        <a name="Generate Bookmarks"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">Generate Bookmarks</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Generates bookmark links to content for HTML or XML documents. Links are placed at the beginning of the resulting HTML or XML document.</p>
         <p></p>
        </div> 
       </div> 
       <div class="variable parbase section"> 
        <!-- Commented out by cmiqueo --> 
        <!-- <a name="Generate Tags For Untagged Files" href="./"></a><div class="variable-title">Generate Tags For Untagged Files</div> --> 
        <a name="Generate Tags For Untagged Files"></a> 
        <div class="help-variable"> 
         <p class="text"> <span class="help-variable-title">Generate Tags For Untagged Files</span></p>
        </div>
        <div class="variable-text">
         <p class="text"></p>
         <p>Generates tags for files that are not already tagged, such as PDFs created using Acrobat 4.0 or earlier. If this option is not selected, untagged files are not converted.</p> 
         <p></p>
        </div> 
       </div> 
       <div class="note parbase section"> 
        <div class="help-note"> 
         <p> <span class="help-note-title">Note:</span> </p>
         <p>Tags are applied only as part of the conversion process and are discarded after the conversion. This is not a method for creating tagged PDFs from legacy files</p> 
         <p></p> 
        </div>

        <div class="variable-text">
         <p class="text"></p>
         <p>Specifies the folder in which to store generated images. The default is Images.</p>
         <p></p>

         <p></p>
        </div> 
       </div>
