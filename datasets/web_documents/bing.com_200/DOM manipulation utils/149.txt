   <div class="sect1"> 
    <h2 id="overview"><a class="link" href="#overview">1. Overview</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="overview_logger"><a class="link" href="#overview_logger">1.1. Logging</a></h3> 
      <div class="paragraph"> 
       <p><em>Logging</em> is the process of writing log messages during the execution of a program to a central place. This logging allows you to report and persist error and warning messages as well as info messages (e.g., runtime statistics) so that the messages can later be retrieved and analyzed.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The object which performs the logging in applications is typically just called <em>Logger</em>.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="overview_loggerjava"><a class="link" href="#overview_loggerjava">1.2. Logging in Java</a></h3> 
      <div class="paragraph"> 
       <p>Java contains the <em>Java Logging API</em>. This logging API allows you to configure which message types are written. Individual classes can use this logger to write messages to the configured log files.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The <code>java.util.logging</code> package provides the logging capabilities via the <code>Logger</code> class.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="overview_create"><a class="link" href="#overview_create">1.3. Create a logger</a></h3> 
      <div class="paragraph"> 
       <p>To create a logger in your Java code, you can use the following snippet.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">import</span> <span class="include">java.util.logging.Logger</span>;

<span class="comment">// assumes the current class is called MyLogger</span>
<span class="directive">private</span> <span class="directive">final</span> <span class="directive">static</span> <span class="predefined-type">Logger</span> LOGGER = <span class="predefined-type">Logger</span>.getLogger(MyLogger.class.getName());</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>The <code>Logger</code> you create is actually a hierarchy of Loggers, and a <code>.</code> (dot) in the hierarchy indicates a level in the hierarchy. So if you get a Logger for the <code>com.example</code> key, this Logger is a child of the <code>com</code> Logger and the <code>com</code> Logger is child of the Logger for the empty <code>String</code>. You can configure the main logger and this affects all its children.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="overview_level"><a class="link" href="#overview_level">1.4. Level</a></h3> 
      <div class="paragraph"> 
       <p>The log levels define the severity of a message. The <code>Level</code> class is used to define which messages should be written to the log.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following lists the Log Levels in descending order:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>SEVERE (highest)</p> </li> 
        <li> <p>WARNING</p> </li> 
        <li> <p>INFO</p> </li> 
        <li> <p>CONFIG</p> </li> 
        <li> <p>FINE</p> </li> 
        <li> <p>FINER</p> </li> 
        <li> <p>FINEST</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>In addition to that you also have the levels <code>OFF</code> and <code>ALL</code> to turn the logging off or to log everything.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, the following code sets the logger to the <code>info</code> level, which means all messages with severe, warning and info will be logged.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java">LOGGER.setLevel(<span class="predefined-type">Level</span>.INFO);</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="general_handler"><a class="link" href="#general_handler">1.5. Handler</a></h3> 
      <div class="paragraph"> 
       <p>Each logger can have access to several handlers.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The handler receives the log message from the logger and exports it to a certain target.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A handler can be turned off with the <code>setLevel(Level.OFF)</code> method and turned on with <code>setLevel()</code> method.</p> 
      </div> 
      <div class="paragraph"> 
       <p>You have several standard handlers. The following list gives some examples.</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>ConsoleHandler: Write the log message to console</p> </li> 
        <li> <p>FileHandler: Writes the log message to file</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>Log levels <code>INFO</code> and higher will be automatically written to the console.</p> 
      </div> 
     </div> 
     <div class="sect2">

      </div> 
      <div class="paragraph"> 
       <p>Available formatter</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>SimpleFormatter: Generate all messages as text</p> </li> 
        <li> <p>XMLFormatter: Generates XML output for the log messages</p> </li> 
       </ul> 
      </div>

      <h3 id="general_manager"><a class="link" href="#general_manager">1.7. Log Manager</a></h3> 
      <div class="paragraph"> 
       <p>The log manager is responsible for creating and managing the logger and the maintenance of the configuration.</p> 
      </div> 
      <div class="paragraph"> 
       <p>We could set the logging level for a package, or even a set of packages, by calling the <code>LogManager.setLevel(String name, Level level)</code> method. So, for example, we could set the logging level of all loggers to <code>Level.FINE</code> by making this call:</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="predefined-type">LogManager</span>.getLogManager().getLogger(<span class="predefined-type">Logger</span>.GLOBAL_LOGGER_NAME).setLevel(<span class="predefined-type">Level</span>.FINE);</code></pre> 
       </div> 
      </div>

      <h3 id="chapter20s1a"><a class="link" href="#chapter20s1a">1.8. Best Practices</a></h3> 
      <div class="paragraph"> 
       <p>It is common practice to use the fully qualified name of each class whose activity is being logged as a message category, because this allows developers to fine-tune log settings for each class.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Using the fully qualified class name of your class as the name of your Logger is the approach recommended by the Logging API documentation.</p>
