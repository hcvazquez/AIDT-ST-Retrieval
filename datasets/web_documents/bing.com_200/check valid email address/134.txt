   <h1><span class="nomobile">JavaScript: </span>Password Validation using regular expressions and HTML5</h1> 
   <p class="social_counters"><a class="tweets" title="Share on Twitter" target="_blank" href="https://twitter.com/intent/tweet?text=Password+Validation+using+regular+expressions+and+HTML5:&amp;url=http%3A%2F%2Fwww.the-art-of-web.com%2Fjavascript%2Fvalidate-password%2F&amp;via=theartofweb"><span>&nbsp;<span>Tweet</span></span></a><a class="likes" title="Share on Facebook" target="_blank" href="https://www.facebook.com/sharer/sharer.php?u=http%3A%2F%2Fwww.the-art-of-web.com%2Fjavascript%2Fvalidate-password%2F"><span>68<span> Shares</span></span></a><a class="gplus" title="Recommend on Google+" target="_blank" href="https://plus.google.com/share?url=http%3A%2F%2Fwww.the-art-of-web.com%2Fjavascript%2Fvalidate-password%2F"><span>&nbsp;<span>Share</span></span></a><a class="reddit" title="Post to Reddit" target="_blank" href="http://www.reddit.com/submit?title=Password+Validation+using+regular+expressions+and+HTML5&amp;url=http%3A%2F%2Fwww.the-art-of-web.com%2Fjavascript%2Fvalidate-password%2F"><span>0<span> Tweets</span></span></a><a class="feedback" title="User Commentc" href="#user_comments"><span>17<span> Comments</span></span></a></p> 
   <p>A lot of websites now require registration, meaning that users need to be assigned a username and password. Here are some simple steps to make the process more secure.</p> 
   <p>If the purpose of registration is to confirm that the person exists, and that they have supplied a valid email address, then as part of the registration processe you a should either <b>email them a random password or a confirmation token</b> rather than letting them choose their own password and use it immediately.</p> 
   <p>The code presented below would then be used for letting the user change their password.</p> 
   <h2>Guidelines for Secure Password Input</h2> 
   <h3>Use the "password" input type</h3> 
   <p>Instead of &lt;input type="text"&gt;, use &lt;input type="password"&gt; as this lets the browser (and the user) know that the contents of that field need to be secured.</p> 
   <p>The password won't appear on the screen as you type and most browsers also won't 'remember' the values entered in <tt>password</tt> fields as they do with other form elements.</p> 
   <p>In some cases, such as on mobile devices, displaying the password may improve usability without compromising security. After all it's only the browser display being obfuscated and not the data transfer.</p> 
   <h3>Confirm password input</h3> 
   <p>Because the <tt>password</tt> input type obscures the text typed, you should let the user confirm that they haven't made a mistake. The simplest way to do this is to have the password entered twice, and then check that they are identical.</p> 
   <p>Another method is to display what they've entered as part of a 'confirmation page'. The problem here is that you're making the password visible in the browser, browser cache, proxy, etc. For security a password should <b>never</b> be displayed in HTML or sent by email.</p> 
   <h3>Enforce 'strong' passwords</h3> 
   <p>If you're concerned about security you should have some policy on what constitutes a valid password. Some common restrictions are:</p> 
   <ul> 
    <li>at least <i>n</i> characters</li> 
    <li>combination of upper- and lower-case characters</li> 
    <li>one or more digits</li> 
    <li>not related to other user data (name, address, username, ...)</li> 
    <li>not a dictionary word</li> 
   </ul> 
   <p>Leaving the last requirement for now, as it requires a server-side script, let's see what's possible using just client-side HTML and JavaScript.</p> 
   <h3>Server security</h3> 
   <p>While having a strong password is a good first step, it needs to be backed up by additional measures on the server that prevent brute-force attacks. One popular approach is to <a href="/system/fail2ban-howto/">install Fail2Ban</a> to monitor log files and lock out repeat offendors. Of course that only works if your login system reports failed login attempts to a system log file. Otherwise your application needs to provide this function.</p> 
   <p>Passwords need to be <a href="/php/blowfish-crypt/">stored encrypted</a> in the database or elsewhere and any backups should also be encrypted.</p> 
   <h2>Basic Demonstration</h2> 
   <p>The form below has three input fields: username, pwd1 and pwd2. When the form is submitted the <tt>checkForm</tt> script parses the input values and returns either <tt>true</tt> or <tt>false</tt>. If a false value is returned then the form submission is cancelled.</p> 
   <p>This code will work for browsers as far back as Netscape 4 (circa 1997).</p>

   <h2>Advanced regular expressions</h2> 
   <p>In most browsers - those that support JavaScript 1.5 (Firefox, Chrome, Safari, Opera 7 and Internet Explorer 8 and higher) - you can use more powerful regular expressions. <b>Older browsers will not recognise these patterns</b>.</p> 
   <p>The code presented above is fine in that it checks everything that we wanted to check, but uses a lot of code to test each requirement individually and present different error messages. We're going to show you now how to apply the password tests using a single regular expression.</p> 
   <p>Consider the following:</p> 
   <code>&lt;script type="text/javascript"&gt; function checkPassword(str) { <i>// at least one number, one lowercase and one uppercase letter // at least six characters</i> var re = /(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}/; return re.test(str); } &lt;/script&gt;</code> 
   <p>The type of expression used here is called a 'look-ahead' which tries to match the contained regexp against the 'future' part of the string.</p> 
   <p><b>Translation:</b></p> 
   <ul> 
    <li>matches a string of six or more characters;</li> 
    <li>that contains at least one digit (\d is shorthand for [0-9]);</li> 
    <li>at least one lowercase character; and</li> 
    <li>at least one uppercase character:</li> 
   </ul>

   <code class="final">&lt;script type="text/javascript"&gt; function checkPassword(str) { <i>// at least one number, one lowercase and one uppercase letter // at least six characters that are letters, numbers or the underscore</i> var re = /^(?=.*\d)(?=.*[a-z])(?=.*[A-Z])<span>\w</span>{6,}$/; return re.test(str); } &lt;/script&gt;</code> 
   <p>The <tt>\w</tt> is shorthand for 'any letter, number or the underscore character'.</p> 
   <script type="text/javascript">function checkPassword2(str){var re=/^(?=.*\d)(?=.*[a-z])(?=.*[A-Z])\w{6,}$/;if(re.test(str)){alert(str+" is a VALID password!");}else{alert(str+" is NOT a valid password!");}}</script>

   <p><b>Restricting which characters can be used is not good practice as punctuation and other symbols provide extra security.</b></p> 
   <h2>Sample HTML and JavaScript code</h2> 
   <p>You might implement this code on your own website as follows:</p> 
   <code>&lt;script type="text/javascript"&gt; function <span>checkPassword(str)</span> { var re = /^(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}$/; return re.test(str); } function <span>checkForm(form)</span> { if(form.username.value == "") { alert("Error: Username cannot be blank!"); form.username.focus(); return false; } re = /^\w+$/; if(!re.test(form.username.value)) { alert("Error: Username must contain only letters, numbers and underscores!"); form.username.focus(); return false; } if(form.pwd1.value != "" &amp;&amp; form.pwd1.value == form.pwd2.value) { if(!<span>checkPassword(form.pwd1.value)</span>) { alert("The password you have entered is not valid!"); form.pwd1.focus(); return false; } } else { alert("Error: Please check that you've entered and confirmed your password!"); form.pwd1.focus(); return false; } return true; } &lt;/script&gt; &lt;form method="POST" action="<i>...</i>" onsubmit="<span>return checkForm(this);</span>"&gt; &lt;p&gt;Username: &lt;input type="text" name="username"&gt;&lt;/p&gt; &lt;p&gt;Password: &lt;input type="password" name="pwd1"&gt;&lt;/p&gt; &lt;p&gt;Confirm Password: &lt;input type="password" name="pwd2"&gt;&lt;/p&gt; &lt;p&gt;&lt;input type="submit"&gt;&lt;/p&gt; &lt;/form&gt;</code> 
   <p>As you can see, it's well worth learning the intricacies of regular expressions. They can be used not just in JavaScript, but also PHP, Perl, Java and many other languages. Some text editors (not just vi) also allow them when searching for or replacing text.</p> 
   <h2>HTML5 Form Validation</h2> 
   <p>We earlier mentioned HTML5 form validation. This is a new technique available in modern browsers and definitely the way of the future. A few simple form attributes can have the same effect as reams of JavaScript code libraries.</p> 
   <p>Here we have an enhanced version of the above code where we've added HTML5 <tt>required</tt> and <tt>pattern</tt> elements to apply regular expression tests within the form itself in supporting browsers. Helpfully the regular expression syntax is identical with just the <tt>/^</tt> and <tt>$/</tt> removed.</p> 
   <p>We've also added a tricky little <tt>onchange</tt> handler to the first password field which updates the pattern required by the second password field - in effect forcing them to be identical:</p> 
   <script type="text/javascript">function checkPassword3(str){var re=/^(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}$/;return re.test(str);}function checkForm3(form){if(form.username.value==""){alert("Error: Username cannot be blank!");form.username.focus();return false;}re=/^\w+$/;if(!re.test(form.username.value)){alert("Error: Username must contain only letters, numbers and underscores!");form.username.focus();return false;}if(form.pwd1.value!=""&&form.pwd1.value==form.pwd2.value){if(!checkPassword3(form.pwd1.value)){alert("The password you have entered is not valid!");form.pwd1.focus();return false;}}else{alert("Error: Please check that you've entered and confirmed your password!");form.pwd1.focus();return false;}alert("Both username and password are VALID!");return true;}if(!RegExp.escape){RegExp.escape=function(s){return String(s).replace(/[\\^$*+?.()|[\]{}]/g,'\\$&');};}</script>

     <label for="field_pwd2">Confirm Password</label>

   <p>In this example it should be clear to the user that the form can only be submitted once all three green ticks appear. In any case browsers such as Firefox and Opera will enforce the HTML5 validation rules and present messages as shown here:</p>

   <p>Presumably the browser messages will change according to the users language - something that would never be possible using only JavaScript.</p> 
   <p>All we have changed from the previous example is to add some extra attributes to the form input fields. The rest of the HTML and JavaScript remains unaltered:</p> 
   <code>... &lt;p&gt;Username: &lt;input type="text" <span>required</span> <span>pattern="<i>\w+</i>"</span> name="username"&gt;&lt;/p&gt; &lt;p&gt;Password: &lt;input type="password" <span>required</span> <span>pattern="<i>(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}</i>"</span> name="pwd1" <span>onchange="form.pwd2.pattern = RegExp.escape(this.value);"</span>&gt;&lt;/p&gt; &lt;p&gt;Confirm Password: &lt;input type="password" <span>required</span> name="pwd2"&gt;&lt;/p&gt; ...</code> 
   <p><strong>Note that you will need the <tt>RegExp.escape()</tt> polyfill code which can be found further down the page.</strong></p> 
   <p>The best thing about HTML5 attributes is that they have no effect whatsoever on unsupported browsers, so older versions of Internet Explorer will act as if they are not present and just run the JavaScript validation as before.</p> 
   <p>At this stage both Firefox and Opera enforce HTML5 validation attributes in the browser while Safari only lets you use them with JavaScript or for CSS effects.</p> 
   <h2>Customised HTML5 browser alerts</h2> 
   <p>As you can see from the screenshot above the alert message in Firefox for when the input doesn't match the <tt>pattern</tt> attribute is simply "Please match the requested format.". Not entirely helpful in this case where we have a number of different requirements.</p> 
   <p>Fortunately it is possible to customise this message using just a touch of JavaScript.</p> 
   <form method="POST" action="/javascript/validate-password/" onsubmit="return checkForm3(this) &amp;&amp; false;">

   </form> 
   <p>The only change between this and the previous example is that we've modified the <tt>onchange</tt> handler for the Password input and added one for Confirm Password:</p> 
   <code>&lt;form method="POST" action="<i>...</i>" onsubmit="return checkForm(this);"&gt; &lt;fieldset&gt; &lt;legend&gt;Change Password&lt;/legend&gt; &lt;p&gt;Username: &lt;input title="<i>Enter your username</i>" type="text" required pattern="\w+" name="username"&gt;&lt;/p&gt; &lt;p&gt;Password: &lt;input title="<i>Password must contain at least 6 characters, including UPPER/lowercase and numbers</i>" type="password" required pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}" name="pwd1" onchange=" <span> this.setCustomValidity(this.validity.patternMismatch ? this.title : ''); if(this.checkValidity()) form.pwd2.pattern = RegExp.escape(this.value);</span> "&gt;&lt;/p&gt; &lt;p&gt;Confirm Password: &lt;input title="<i>Please enter the same Password as above</i>" type="password" required pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}" name="pwd2" onchange=" <span> this.setCustomValidity(this.validity.patternMismatch ? this.title : '');</span> "&gt;&lt;/p&gt; &lt;p&gt;&lt;input type="submit" value="Save Changes"&gt;&lt;/p&gt; &lt;/fieldset&gt; &lt;/form&gt;</code> 
   <p><b>We use the <tt>title</tt> attribute to store the validation messages as some browsers will automatically display that text in case of a missing value or pattern mismatch.</b></p> 
   <p><strong>Note that you will need the <tt>RegExp.escape()</tt> polyfill code which can be found further down the page.</strong></p> 
   <p>When the Password input is changed we check its <tt>validity.patternMismatch</tt> flag to see whether it matches the <tt>pattern</tt> attribute. If it doesn't match we set a custom error message to appear when the form is submitted.</p> 
   <p>Whenever <tt>setCustomValidity()</tt> has been used to set a custom message the field in question will be regarded as invalid and prevent the form from submitting (at least in Firefox and Opera). To reverse this it needs to be set to blank, which we do when the input matches our regex rules.</p> 
   <p>The custom message we have set appears in Firefox as shown here:</p> 
   <p style="text-align: center;"><img src="http://cdn.the-art-of-web.com/images/html5-custom-alert1.jpg" width="456" height="190" alt=""></p> 
   <p style="text-align: center;"><img src="http://cdn.the-art-of-web.com/images/html5-custom-alert2.jpg" width="456" height="190" alt=""></p> 
   <p style="text-align: center;"><img src="http://cdn.the-art-of-web.com/images/html5-custom-alert3.jpg" width="456" height="190" alt=""></p> 
   <p>You can see from the code that we have applied a similar technique to the Confirm Password field so it will now display "Please enter the same Password as above" if there is a mismatch between the two fields. Basically what we've achieved here is to replicate our JavaScript validation script using HTML5.</p> 
   <p>Because we are only checking for <tt>patternMismatch</tt> we are not affecting the other default validation options - namely the <tt>required</tt> attribute - so submitting the form with blank fields will still display the generic "Please fill out this field" alert message. To override those errors you would need to check the <tt>validity.valueMissing</tt> flag.</p> 
   <p>For more details on the <a href="http://www.whatwg.org/specs/web-apps/current-work/multipage/association-of-controls-and-forms.html" target="_blank">constraint validation API</a> follow the links under References below.</p> 
   <p><b>Different browsers will show both the <tt>title</tt> text and the <tt>CustomValidity</tt> text in the same tooltip; just the <tt>CustomValidity</tt> message; or neither.</b></p> 
   <h2>Separating form and function</h2> 
   <p>You may have noticed the code in the above examples getting progressivley 'messier' with a mixture of HTML and JavaScript. A more professional approach these days is to totally separate the JavaScript from the HTML.</p> 
   <p>This makes the code easier to maintain, especially when working in groups, as well as easier to read and understand. It does, however, lead to using more code as you can see below.</p> 
   <p>First, the HTML:</p> 
   <code class="final">&lt;form id="<span>myForm</span>" method="POST" action="<i>...</i>"&gt; &lt;p&gt;Username: &lt;input id="<span>field_username</span>" title="Username must not be blank and contain only letters, numbers and underscores." type="text" required pattern="\w+" name="username"&gt;&lt;/p&gt; &lt;p&gt;Password: &lt;input id="<span>field_pwd1</span>" title="Password must contain at least 6 characters, including UPPER/lowercase and numbers." type="password" required pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}" name="pwd1"&gt;&lt;/p&gt; &lt;p&gt;Confirm Password: &lt;input id="<span>field_pwd2</span>" title="Please enter the same Password as above." type="password" required pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}" name="pwd2"&gt;&lt;/p&gt; &lt;p&gt;&lt;input type="submit" value="Submit"&gt;&lt;/p&gt; &lt;/form&gt;</code> 
   <p>and then the JavaScript:</p> 
   <code id="box2" class="final" style="height: 400px;">&lt;script type="text/javascript"&gt; document.addEventListener("DOMContentLoaded", function() { <i>// JavaScript form validation</i> var checkPassword = function(str) { var re = /^(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{6,}$/; return re.test(str); }; var checkForm = function(e) { if(this.username.value == "") { alert("Error: Username cannot be blank!"); this.username.focus(); e.preventDefault(); <i>// equivalent to return false</i> return; } re = /^\w+$/; if(!re.test(this.username.value)) { alert("Error: Username must contain only letters, numbers and underscores!"); this.username.focus(); e.preventDefault(); return; } if(this.pwd1.value != "" &amp;&amp; this.pwd1.value == this.pwd2.value) { if(!checkPassword(this.pwd1.value)) { alert("The password you have entered is not valid!"); this.pwd1.focus(); e.preventDefault(); return; } } else { alert("Error: Please check that you've entered and confirmed your password!"); this.pwd1.focus(); e.preventDefault(); return; } alert("Both username and password are VALID!"); }; var myForm = document.getElementById("<span>myForm</span>"); myForm.addEventListener("submit", checkForm, true); <i>// HTML5 form validation</i> var supports_input_validity = function() { var i = document.createElement("input"); return "setCustomValidity" in i; } if(supports_input_validity()) { var usernameInput = document.getElementById("<span>field_username</span>"); usernameInput.setCustomValidity(usernameInput.title); var pwd1Input = document.getElementById("<span>field_pwd1</span>"); pwd1Input.setCustomValidity(pwd1Input.title); var pwd2Input = document.getElementById("<span>field_pwd2</span>"); <i>// input key handlers</i> usernameInput.addEventListener("keyup", function(e) { usernameInput.setCustomValidity(this.validity.patternMismatch ? pwd2Input.title : ""); }, false); } }, false); &lt;/script&gt;</code> 
   <p style="margin-top: 0; text-align: right;"><small><a href="#box2" onclick="document.getElementById('box2').style['height'] = 'auto'; this.parentNode.style['display'] = 'none'; return true;">expand code box</a></small></p> 
   <p>The only hooks between the HTML and JavaScript are the <tt>id</tt> values of the form and input fields which you can see highlighted in the code.</p> 
   <p><b>This example will work in Internet Explorer 9 and higher. To get it working in IE8 requires a polyfill for the event handling functions. You can find code for this <a href="https://developer.mozilla.org/en-US/docs/Web/API/EventTarget.addEventListener#Older_way_to_register_event_listeners" target="_blank">here</a>.</b></p> 
   <p><strong>Note that you will also need the <tt>RegExp.escape()</tt> polyfill code which can be found further down the page.</strong></p>

   <p>To break it down, our form inputs are now being validated by:</p> 
   <ul> 
    <li>immediate visual feedback using CSS styles;</li> 
    <li>inline prompts using custom HTML5 messages;</li> 
    <li>JavaScript when the form is submitted; and</li> 
    <li>(presumably) by a server-side script when the data is received.</li>

   <h2>RegExp.escape polyfill</h2> 
   <p>This function comes from a <a href="https://github.com/benjamingr/RegExp.escape" target="_blank">proposed ECMAScript addition</a>:</p> 
   <code class="final">&lt;script type="text/javascript"&gt; <i>// polyfill for RegExp.escape</i> if(!RegExp.escape) { RegExp.escape = function(s) { return String(s).replace(/[\\^$*+?.()|[\]{}]/g, '\\$&amp;'); }; } &lt;/script&gt;</code> 
   <p>Without this, if the contents of <tt>pwd1</tt> contain a '*' or other special regular expression characters, the pattern for <tt>pwd2</tt> can become invalid.</p> 
   <h2>Conclusions</h2> 
   <p>As you can see there's a lot involved in providing a rich user experience even for something as simple as changing a password. To summarise what we've covered:</p> 
   <ul> 
    <li>Always start by emailing new users a random password or unique activation link;</li> 
    <li>Use the <tt>password</tt> input type to prevent passwords appearing on screen;</li> 
    <li>Decide what constitutes a 'strong' password for your system and enforce it:
     <ul> 
      <li>server-side for spambots and users with JavaScript disabled;</li> 
      <li>using JavaScript for browsers that don't support HTML5 validation;</li> 
      <li>using HTML5 for a more user-friendly experience;</li> 
     </ul></li> 
    <li>Use CSS rules to highlight valid/invalid input for browsers that don't have alerts built-in; and</li> 
    <li>Customise the HTML5 error messages where appropriate for improved usability;</li> 
   </ul> 
   <p>Most of all don't feel you have to install massive JavaScript or jQuery libraries just to validate a form. By taking advantage of new browser standards as show in this article you can save time and resources and at the same time provide a better user experience.</p> 
   <p><b>HTML5 form validation is supported by Firefox, Chrome, Opera and Internet Explorer 10. In Safari you will see the CSS styles, but the browser will not prevent the form from submitting or display the inline error messages for invalid or missing values.</b></p>
