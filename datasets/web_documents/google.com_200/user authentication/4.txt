     <h2><a id="user-content-configurations" class="deep-link" href="#configurations" aria-hidden="true">
       <svg aria-hidden="true" class="deep-link-icon" height="16" version="1.1" viewbox="0 0 16 16" width="16">
        <path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
       </svg></a>Configurations</h2> 
     <ul> 
      <li><code>realm</code> - Authentication realm, by default it is <strong>Users</strong>.</li> 
      <li><code>file</code> - File where user details are stored. 
       <ul> 
        <li>Line format is <strong>{user:pass}</strong> or <strong>{user:passHash}</strong> for basic access.</li> 
        <li>Line format is <strong>{user:realm:passHash}</strong> for digest access.</li> 
       </ul> </li> 
      <li><code>algorithm</code> - Algorithm that will be used only for <strong>digest</strong> access authentication. 
       <ul> 
        <li><strong>MD5</strong> by default.</li> 
        <li><strong>MD5-sess</strong> can be set.</li> 
       </ul> </li> 
      <li><code>qop</code> - Quality of protection that is used only for <strong>digest</strong> access authentication. 
       <ul> 
        <li><strong>auth</strong> is set by default.</li> 
        <li><strong>none</strong> this option is disabling protection.</li> 
       </ul> </li> 
      <li><code>msg401</code> - Message for failed authentication 401 page.</li> 
      <li><code>msg407</code> - Message for failed authentication 407 page.</li> 
      <li><code>contentType</code> - Content type for failed authentication page.</li> 
      <li><code>skipUser</code> - Set this to <strong>true</strong>, if you don't want req.user to be filled with authentication info.</li> 
     </ul>
