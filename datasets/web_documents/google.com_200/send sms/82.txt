             <a class="a2a_button_linkedin" href="http://www.addtoany.com/add_to/linkedin?linkurl=http%3A%2F%2Fnodexperts.com%2Fblog%2Fotp-authentication-sms-twilio-meteor%2F&amp;linkname=OTP%20Authentication%20with%20SMS%20using%20Twilio%20in%20Meteor" title="LinkedIn" rel="nofollow" target="_blank"></a>
            </div>
           </div>
           <p>Using OTP or SMS authentication is a very useful feature to have in an app. It provides two-factor security and verification in addition to the authentication layer. Twilio provides a very useful SMS API that can be used to implement such a feature. Here I will take you through the general process to implement Twilio SMS authentication in Meteor user signup and registration.</p> 
           <h3 style="margin-bottom: 0px;"><b>Packages</b></h3> 
           <p>First, you need to install one of the following packages:</p> 
           <ul> 
            <li><strong><a href="https://atmospherejs.com/meteor/http">http</a></strong> — <em>(To call the Twilio API directly from your methods) or,</em></li> 
            <li><strong><a href="https://atmospherejs.com/accolver/twilio-meteor">accolver:twilio-meteor</a> </strong>— <em>(You can try using the services this package provides off of the official <a href="https://www.twilio.com/docs/libraries/node">Twilio-Node Helper Library</a>)</em></li> 
           </ul> 
           <p>Next, you should also install the <a href="https://atmospherejs.com/okland/accounts-phone">okland:accounts-phone</a> package to help enable login via phone number. Their <a href="https://github.com/okland/accounts-phone/">GitHub</a> provides easy to follow instructions on how to set it up.</p> 
           <h3 style="margin-bottom: 0px;"><b>Password</b></h3> 
           <p>It is strongly recommended to create user accounts with a password, along with the phone number. It is a good security feature to have and is also required by default on Meteor Accounts package.</p> 
           <h3 style="margin-bottom: 0px;"><b>Verification Process</b></h3> 
           <p>So here I will be giving an example using <strong><i>server-side Meteor methods</i></strong>. For frontend you can write your Blaze, React or AngularJS handlers accordingly.</p> 
           <p>This example will be using the HTTP package. If you wish, you can modify it to include other wrapper packages like twilio-meteor in your code.</p> 
           <p><img class="aligncenter wp-image-529 size-full" src="http://nodexperts.com/blog/wp-content/uploads/2017/06/otp-meteor-twilio.png" alt="Flow Chart of OTP Authentication with Twilio in Meteor" width="700" height="569" srcset="http://nodexperts.com/blog/wp-content/uploads/2017/06/otp-meteor-twilio.png 700w, http://nodexperts.com/blog/wp-content/uploads/2017/06/otp-meteor-twilio-300x244.png 300w" sizes="(max-width: 709px) 85vw, (max-width: 909px) 67vw, (max-width: 984px) 61vw, (max-width: 1362px) 45vw, 600px"></p> 
           <h4 style="margin-bottom: 0px;"><em><strong>Step 1: Register your user and send OTP verification SMS.</strong></em></h4> 
           <p><strong><i>createNewUser method:</i></strong></p> 
           <pre style="overflow: auto; padding: 0.5px 9.5px; margin: 0 0 10px; font-size: 13px; line-height: 1.42857143; color: #333; background-color: #f5f5f5; border: 1px solid #ccc;"><code style="background: transparent; padding: 0.5em;">
'createNewUser': (password, phoneNumber) =&gt; {
&nbsp; const min = 10000,
&nbsp;   max = 99999,
&nbsp;   OTP = Math.floor(Math.random() * (max - min + 1)) + min,
&nbsp;   dbUser = Meteor.users.findOne({ username: phoneNumber });
&nbsp; 
&nbsp; if (dbUser) {
&nbsp; &nbsp;&nbsp; &nbsp;// user already exists
&nbsp; &nbsp;&nbsp; &nbsp;return throw new Meteor.Error('error',
        `User already exists for ${phoneNumber}`);
&nbsp; }
&nbsp; 
&nbsp; // user does not exist &nbsp;
&nbsp; const user = {
&nbsp;&nbsp;&nbsp; username: phoneNumber,
&nbsp;&nbsp;&nbsp; password,
&nbsp;&nbsp;&nbsp; profile: { 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; OTP, 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; isMobileVerified: false, 
&nbsp;&nbsp;&nbsp; },
&nbsp; };
&nbsp; 
&nbsp; // send verification code
&nbsp; Meteor.call('sendSMS', OTP, phoneNumber, (error) =&gt; {
&nbsp; &nbsp;&nbsp; &nbsp;if(error) {
&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;return throw new Meteor.Error('error', error);
&nbsp;&nbsp;&nbsp; }
&nbsp;&nbsp; &nbsp;
&nbsp;&nbsp;&nbsp; // create user
&nbsp;&nbsp;&nbsp; return Accounts.createUser(user);
&nbsp; });
}</code></pre> 
           <p><strong><i>sendSMS method:</i></strong></p> 
           <pre style="overflow: auto; padding: 0.5px 9.5px; margin: 0 0 10px; font-size: 13px; line-height: 1.42857143; color: #333; background-color: #f5f5f5; border: 1px solid #ccc;"><code style="background: transparent; padding: 0.5em;">
'sendSMS': (code, mobile) =&gt; {
&nbsp; // call twilio API
&nbsp; HTTP.call(
&nbsp;&nbsp;&nbsp; 'POST',
&nbsp;&nbsp;&nbsp; 'https://api.twilio.com/{yyyy-dd-mm}/Accounts/' + 
&nbsp;&nbsp;&nbsp; '{TWILIO_APPKEY}' + '/SMS/Messages.json', {
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; params: {
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; From: '{YOUR_NUMBER}',
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; To: mobile,
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Body: `Greetings! Your OTP is ${code}`,
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; },
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; auth: '{TWILIO_APPKEY}' + ':' + '{TWILIO_PASSWORD}'
&nbsp;&nbsp;&nbsp; }, function (error) {
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (error) {
&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;console.log('API request failed.');
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return throw new Meteor.Error('error', error);
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; console.log('SMS sent successfully.');
&nbsp;&nbsp;&nbsp; };
&nbsp; );
}</code></pre> 
           <h4 style="margin-bottom: 0px;"><em><strong>Step 2: Ask user for verification code and check code input by user</strong></em></h4> 
           <p><strong><i>verifySMS method:</i></strong></p> 
           <pre style="overflow: auto; padding: 0.5px 9.5px; margin: 0 0 10px; font-size: 13px; line-height: 1.42857143; color: #333; background-color: #f5f5f5; border: 1px solid #ccc;"><code style="background: transparent; padding: 0.5em;">
'verifySMS': (code, phoneNumber) =&gt; {
&nbsp; const user = Meteor.users.findOne({ username: phoneNumber });
&nbsp; const { OTP } = user.profile;
&nbsp; 
&nbsp; if(!OTP || OTP !== code) {
&nbsp; &nbsp;&nbsp; &nbsp;return throw new Meteor.Error('error', 'Invalid OTP.');
&nbsp; }
&nbsp; return Meteor.users.update({ username: phoneNumber }, {
&nbsp;&nbsp;&nbsp; $set: { 'profile.isMobileVerified': true },
&nbsp;&nbsp;&nbsp; $unset: { 'profile.OTP': 0 },
&nbsp; });
}</code></pre> 
           <p>&nbsp;</p> 
           <p><strong>Lastly, you need to use the above methods in client side.&nbsp;</strong>From your client-side code handling, you can make a request to the Meteor method. Then if the input code matches, approve the user. Else display appropriate error message.</p> 
           <p>Voila! This is all you need to implement OTP SMS based authentication for user signup in Meteor.</p>
