      <div id="contentTop">
       In this series of Sorting Algorithms, we have talked about Selection Sort, Bubble Sort and Insertion Sort. We have seen than these algorithms are not so fast; all of them have O(n2) time complexity in worst and average case. In this particular article we will see What is Merge Sort, How to implement Merge sort algorithm in Java and Complexity analysis of Merge Sort Algorithm. Merge sort is pretty much faster and have a time complexity of O(nlogn) in worst case.
      </div> 
      <div id="contentBelow">
       <br> 
       <h2>Merge sort algorithms</h2> Merge sort is a Divide and Conquer algorithm and one of the most commonly used sorting algorithm for large data sets. As the name implies unsorted array is first divided into two possibly similar array based on a selected middle element. These sub arrays are than sorted individually and merged back into a whole array in sorted order. Sorting happens at the time of merging that's why the name merge sort. 
       <br>
       <br> Lets say we have an unsorted array having some integers like this: 
       <pre class="java" name="code">

A {2,4,4,1,6,8,5,3,7}
</pre> Our approach to sort this array using merge sort will be entirely different than the previous three algorithms, here we are going to break this problem into sub problems, we will divide this array into two possibly equal half. We will find a middle element where all the elements before that element will belong to first array and on and after this element all elements will belong to second array. 
       <pre class="java" name="code">

A {2,4,1,6,8,5,3,7}

L {2,4,1,6}	R {8,5,3,7}
</pre> So if somehow we are able to sort L and R separately and than merge them into A in sorted array the whole array will be sorted. 
       <pre class="java" name="code">

L{1,2,4,6}	R {3,5,7,8} // after sorting sub arrays separately
</pre> Now we can merge them into one by putting smaller element among the to into a: 
       <pre class="java" name="code">

L{1,2,4,6}	R {3,5,7,8}
L [0] = 1 and R [0] = 3 // L[0] is smaller so we will put this into array.

A{1,_,_,_,_,_,_}

L{1,2,4,6}	R {3,5,7,8}
L [1] = 2 and R [0] = 3 // L[1] is smaller so we will put this into array.

A{1,2,_,_,_,_,_}


L{1,2,4,6}	R {3,5,7,8}
L [2] = 4 and R [0] = 3 // R[0] is smaller so we will put this into array.

A{1,2,3,_,_,_,_,_}

L{1,2,4,6}	R {3,5,7,8}
L [2] = 4 and R [1] = 5 // L[2] is smaller so we will put this into array.

A{1,2,3,4,_,_,_,_}

L{1,2,4,6}	R {3,5,7,8}
L [4] = 6 and R [1] = 5 //R[1] is smaller so we will put this into array.

A{1,2,3,4,5,_,_,_}

L{1,2,4,6}	R {3,5,7,8}
L [4] = 6 and R [2] = 7 //L[3] is smaller so we will put this into array.

A{1,2,3,4,5,6,_,_}
</pre> 
       <br> Now L is completely exhausted and we will copy all remaining elements from R into A. 
       <pre class="java" name="code">


A{1,2,3,4,5,6,7,8}
</pre> This way we can get a sorted array back from two individually sorted arrays. This is merging part of merge sort, as the name suggest sorting is done using merge functionality. 
       <br>
       <br> Now we need to think about functionality of sorting sub arrays, so if we divide sub arrays further until there are single elements left than we will have as much arrays as the size of list and because of having single element at that positions all sub arrays at the end will be automatically sorted. 
       <br>
       <br> Then those smallest arrays can be merger into smaller arrays in sorted order and at the end to the big array and the array will be sorted. For our example this will happen something like this: 
       <br> 
       <br> 
       <h2>Pseudo Code for Merge Sort Algorithm</h2> Now lets write a pseudo code for Merge Sort Algorithm, it will contain two different function one for sorting and second for merge there two functions will do the task using recursion. 
       <pre class="java" name="code">

Merge(L,R,A)
{
   nL &#x2190; length(L)
   nR &#x2190; length(R)

   I &#x2190; 0,j &#x2190; 0,k &#x2190; 0; // Here I will mark the smallest index in L, j in R and k 		    	    // marks the index of big array A  

   While(i&lt;nL &amp;&amp; jlt;nR)
	{  
		   if(L[i]lt;=R[j])
		   {
			 A[k] &#x2190; L[i]
	     		 k &#x2190; k+1
	             i &#x2190; i+1
		   }
		   else
		   {
			 A[k] &#x2190; R[j]
	     		 k &#x2190; k+1
	             j &#x2190; j+1
		   }
		
	}
  
	 while(i &lt; nL)
	    {
		A[k] &#x2190; L[i]
	     	k &#x2190; k+1
	      i &#x2190; i+1  
	     }
	 while(j &lt; nR)
	    {
		 A[k] &#x2190; L[i]
	     	 k &#x2190; k+1
	       j &#x2190; j+1
		   
	     }
}


MergeSort(A)
{
	n &#x2190; length(A)
	if(n&lt;2)
	  return;

	mid &#x2190; n/2;
	left &#x2190; array of size(mid)
	right &#x2190; array of size(n-mid)

	for i&#x2190; 0 to mid-1
	  left[i] &#x2190; A[i]
	for i&#x2190;mid to n-1
	  right[i-mid] &#x2190; A[i]
	
	MergeSort(left)
	MergeSort(right)
	Merge(left, right, A)
	   
}
</pre> 
       <br> 
       <h2>Complexity Analysis of Merge Sort</h2> Lets calculate time complexity of the algorithm by calculating time consumed by each step: 
       <pre class="java" name="code">

1)  	n &#x2190; length(A)
	if(n&lt;2)
	  return;

	mid &#x2190; n/2;
	left &#x2190; array of size(mid)
	right &#x2190; array of size(n-mid)
</pre> This block contains simple statements and will be executed in constant time say C1. 
       <pre class="java" name="code">

2) for i&lt;- 0 to mid-1
	  left[i] &#x2190; A[i]
   for i&lt;-mid to n-1
	  right[i-mid] &#x2190; A[i]
</pre> The cost of above statement will be approx, C2.n 
       <pre class="java" name="code">

3) MergeSort(left)
   MergeSort(right)
</pre> These two recursive calls are just half of the input array, and hence will take T(n/2) and T(n/s) time respectively.
       <br>
       <br> 4) The merge function will take approx constant time of c3.n+c4. Now if we calculate total time we will get: 
       <pre class="java" name="code">

T(n) = 16T(n/16) + 4C'n

T(n) = O(nlogn)
</pre> 
       <br> 
       <h2>Merge Sort Algorithm Implementation in Java</h2> We have seen how Insertion Sort Algorithm works using an extra array and in memory, we have also seen pseudo code for the same, now we will implement Insertion sort in Java: 
       <pre class="java" name="code">

package com.techburps.bsort;

import java.util.Arrays;

public class MSort {
	public static void main(String args[]) {
		int A[] = { 1, 4, 2, 6, 8, 5, 3, 7 };
		System.out.println("Unsorted Array: " + Arrays.toString(A));
		MSort ms = new MSort();
		ms.mergeSort(A);
		System.out.println("Sorted Array: " + Arrays.toString(A));
	}

	public void merge(int L[], int R[], int A[]) {
		int nL = L.length;
		int nR = R.length;
		int i = 0, j = 0, k = 0;

		while (i &lt; nL &amp;&amp; j &lt; nR) {
			if (L[i] &lt;= R[j]) {
				A[k] = L[i];
				k = k + 1;
				i = i + 1;
			} else {
				A[k] = R[j];
				k = k + 1;
				j = j + 1;
			}
		}

		while (i &lt; nL) {
			A[k] = L[i];
			k = k + 1;
			i = i + 1;
		}

		while (j &lt; nR) {
			A[k] = R[j];
			k = k + 1;
			j = j + 1;
		}
	}

	public void mergeSort(int A[]) {
		int n = A.length;

		if (n &lt; 2)
			return;

		int mid = n / 2;

		int L[] = new int[mid];
		int R[] = new int[n - mid];

		for (int i = 0; i &lt; mid; i++) {
			L[i] = A[i];

		}
		for (int i = mid; i &lt; n; i++) {
			R[i - mid] = A[i];

		}
		mergeSort(L);
		mergeSort(R);
		merge(L, R, A);
	}
}
</pre>

       <div class="console">
         Unsorted Array: [1, 4, 2, 6, 8, 5, 3, 7]
        <br> Sorted Array: [1, 2, 3, 4, 5, 6, 7, 8]

       <h2>Properties of Merge Sort</h2> 
       <b>1)</b> A divide and conquer algorithm : Merge Sort algorithm is a divide and conquer algorithm, in this category we divides a problem into sub problems and then we find out solutions for sub problem and from the solutions of sub problems we construct solution of actual problem. 
       <br>
       <br> 
       <b>2)</b> Recursive : Merge sort finds the solution using recursion, Recursion reduces a problem in self similar manner. 
       <br>
       <br> 
       <b>3)</b> Stable: Merge sort is a stable sorting algorithm, the meaning of stable is, we can sort any for of data using merge sort: 
       <br> If we have some complex data to sort like (1,2), (3,4), (2,7), (3,9) and we want to sort this on the basis of first coordinate, then there can be two solutions. 
       <br> (1,2), (2,7), (3,9), (3,4) 
       <br> or 
       <br> (1,2), (2,7), (3,4), (3,9) 
       <br> But using merge sort we will get 2st one always, that means it preserves the relative order of records with same key. 
       <br>
       <br> 
       <b>4)</b> Not in place algorithm, that means the space taken by merge sort is directly propositional to size of list. The space complexity of Merge sort is O(n). 
       <br>
       <br> 
       <b>5)</b> Time complexity for merge sort algorithm is O(nlogn) in worst case. 
       <br>
       <br> O(nlogn) in worst case is definitely a good time taken, Merge sort is a fast algorithm and hence can be used to Sort large data sets. 
       <br>
