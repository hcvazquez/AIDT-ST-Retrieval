         <h1>Really simple inline Javascript validation</h1> 
         <div class="meta"> 
          <span>Posted by <a href="http://culttt.com/author/philipbrown/" title="Posts by Philip Brown" rel="author">Philip Brown</a> on August 8th, 2012</span> 
         </div> 
         <div class="post"> 
          <p><img src="http://i0.wp.com/culttt.com/wp-content/uploads/2012/08/Really-simple-inline-Javascript-validation.jpg" alt="Really simple inline Javascript validation" title="Really simple inline Javascript validation" data-recalc-dims="1"><br> Just about any website or web application will require user input in one form or another. Whether it’s filling in a contact form or entering profile details into a new application, handling input errors can be the difference between a good or a bad user experience.</p> 
          <p>Have you ever filled in a huge form online, only to submit it and find that there was an error so you have to start again? Frustrating, isn’t it? It is these kind of fundamental errors that stop people returning to your website.</p> 
          <p>A much better way of solving this problem is to include inline validation that checks the user’s input as she progresses. This alerts the user to any errors and allows her to correct her errors before proceeding and losing her progress.</p> 
          <p>Inline validation powered by Javascript can be incredibly easy to implement, so there really is no excuse for not making your user’s lives as easy as possible.</p> 
          <p>Here is a tutorial for building your own Javascript validation that you can use on any input on your website or web application. I’ve intentionally kept it as easy, simple and lightweight as possible.</p> 
          <h2>Creating your page</h2> 
          <p>For this tutorial I’m just going to include all of the HTML and Javascript in one file to keep things simple.</p> 
          <p>So to start off, create yourself a little HTML file, like this.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;!doctype html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
	&lt;meta charset="utf-8"&gt;
	&lt;title&gt;Validation tutorial&lt;/title&gt;
	&lt;script&gt;
		// JS
	&lt;/script&gt;
&lt;/head&gt;
&lt;body&gt;

&lt;/body&gt;
&lt;/html&gt;
</pre> 
          <h2>Create the form</h2> 
          <p>Next we need to a create an input field to validate. Create a form and an input field to allow you to enter your name.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;form action=""&gt;
	&lt;fieldset&gt;
		&lt;label for="name"&gt;Name&lt;/label&gt;
		&lt;input type="text" name="name" id="name" /&gt;
	&lt;/fieldset&gt;
&lt;/form&gt;
</pre> 
          <p>Next we need to add the <code>onblur</code> Javascript event to the input field. The <code>onblur</code> event allows you to run Javascript code when the user clicks away from the field. In this case, we want to be able to validate what the user has entered into the field, but we only want to do that when she has finished typing and has moved on.</p> 
          <p>Update your input field so it looks like this.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;input type="text" name="name" id="name" onblur="validateName(name)" /&gt;
</pre> 
          <p>As you can see, I’ve already named the Javascript function that we are going to make in a minute and I’ve passed in the name of the the field, in this case “name”.</p> 
          <h2>Introducing the Javascript</h2> 
          <p>Next we need to create the Javascript function.</p> 
          <p>In between the <code><script></script></code> tags at the top of the page, type the following code.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
function validateName(x){
	alert(document.getElementById(x).value);
}	
</pre> 
          <p>Now save your file and open it in the browser. Enter some text into the input field and click away from the field. An alert box should appear with the text you entered.</p> 
          <p>So as you can see…</p> 
          <ul> 
           <li>We created an input field</li> 
           <li>Added an onblur event which is triggered when the user clicks away</li> 
           <li>We then used a Javascript function to display the contents of the input in an alert box.</li> 
          </ul> 
          <p>So hopefully you can see how the Javascript is run when it is triggered, and how we are able to get whatever the user has typed into the input box and do something with it.</p> 
          <p>Next I’ll show you how to validate the input.</p> 
          <p>Delete the following line from your Javascript.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
alert(document.getElementById(x).value);
</pre> 
          <h2>Javascript Regular Expressions</h2> 
          <p>In order to validate the input, we have to test it against a set of rules that define a pass or fail. For example, you don’t want your users to enter &amp; or @ or % into your name field because these characters don’t belong there.</p> 
          <p>In order to create a set of rules we use a <a href="http://en.wikipedia.org/wiki/Regular_expression" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://en.wikipedia.org/wiki/Regular_expression', 'Regular Expression']);">Regular Expression</a> (Regex). A Regular Expression is a way of matching an input to a set of rules. If the input matches the pattern it will return true, but if it does not, it will return false. So in the above example, if the input contained an @ when the pattern does not include an @ the test would return false and the validation would fail.</p> 
          <p>To create a validation for your name field, type the following line into your <code>validateName()</code> function.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
var re = /[A-Za-z -']$/;
</pre> 
          <p>In the above line of code, we create a variable called <code>re</code> and assign it the regex of any capital letter (A-Z), any lower case letter (a-z), a space, a hyphen or an apostrophe.</p> 
          <p>With this regex rule saved, we can then run a test on our input to see if what was entered matches our defined rule.</p> 
          <p>Type the following code after the previous line.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
if(re.test(document.getElementById(x).value)){
	document.getElementById(x).style.background ='#ccffcc';
	return true;
}else{
	document.getElementById(x).style.background ='#e35152';
	return false;	
}
</pre> 
          <p>Here’s how the above code breaks down.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
if(re.test(document.getElementById(x).value)){
</pre> 
          <p>We test to see if the input value matches our rule.</p> 
          <p>If it does, we change the input box’s background to green.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
document.getElementById(x).style.background ='#ccffcc';
return true;
</pre> 
          <p>But if it doesn’t, we turn the input box red.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
}else{
	document.getElementById(x).style.background ='#e35152';
	return false;	
}
</pre> 
          <p>Save this out and refresh the page in your browser. You should now have a swanky input box that validates and gives a visual feedback to the user to tell them if their input was accepted or not.</p> 
          <h2>Including a prompt</h2> 
          <p>We’ve added a huge amount of usability to our form already, but there is still more we can do.</p> 
          <p>Not every input field will have obvious rules. For example, you might have a username field where you have a very precise set of allowed characters. In this instance, it is important to give your user some kind of message to help them complete the form. Without this prompt, you are likely going to annoy your user if they are repeatedly failing your validation test.</p> 
          <p>To add a validation prompt, add the following line of code to your form, just under your input field.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;span id="nameError" style="display: none;"&gt;You can only use alphabetic characters, hyphens and apostrophes&lt;/span&gt;
</pre> 
          <p>Next add the following line to the <code>else</code> branch of your if else statement in your Javascript.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
document.getElementById(x + 'Error').style.display = "block";
</pre> 
          <p>Save your file and fresh your browser. You should now be given a prompt when you enter invalid characters.</p> 
          <p>Finally add the following line to the if branch of your if else statement.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
document.getElementById(x + 'Error').style.display = "none";
</pre> 
          <p>This will clean up the prompt message once the user corrects their mistake.</p> 
          <h2>Validation on form submission</h2> 
          <p>The validation you’ve added so far is great because it allows the user to correct their mistakes as they progress. The next step is to prevent the form from being submitted if any of the validation rules are broken.</p> 
          <p>This is handy because we don’t really want to allow our user to submit a form with invalid data. Once the form is validated on the server side, it should spit it back out with an error message. However, we can stop it from ever getting this far by validating on the client side too.</p> 
          <h3>Client side validation</h3> 
          <p>So the first thing we need to do is to create a button to allow our form to be submitted.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;input type="submit" id="submit" name="submit" value="Submit" /&gt;
</pre> 
          <p>Next we need to add an <code>onsubmit</code> event. This is very similar to the <code>onblur</code> event we created earlier, the only difference is it is run only when the form is submitted.</p> 
          <p>Update your form tag so it looks like this.</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;form action="" onsubmit="return validateForm()"&gt;
</pre> 
          <h3>More Javascript</h3> 
          <p>Now we need to create the <code>validateForm()</code> function that will check the entire form for valid date.</p> 
          <p>In between the script tags, write the following.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
function validateForm(){

}
</pre> 
          <p>Next we need to create an error count.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
// Set error catcher
var error = 0;
</pre> 
          <p>Next we run our <code>validateName()</code> function.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
// Check name
if(!validateName('name')){
	document.getElementById('nameError').style.display = "block";
	error++;
}
</pre> 
          <p>This runs the <code>validateName()</code> function on the name input field. If it returns false we know that the input field did not validate and so we display our error prompt and increase the <code>error</code> counter by 1.</p> 
          <p>Finally we create a little catch so if there are errors the form will not submit.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
// Don't submit form if there are errors
if(error &gt; 0){
	return false;
}
</pre> 
          <p>Now when you save out your form you shouldn’t be able to submit it if there are errors. To successfully submit the form you need to correct all the errors so the fields turn green.</p> 
          <h2>Extending our Javascript validation</h2> 
          <p>You know have a full working example of inline validation and client side validation that you can implement on your website’s forms.</p> 
          <p>Hopefully this has given you enough of a start to extend the code to meet your requirements.</p> 
          <p>Here are some further Javascript validation functions that you can add to you code to extend it further and help you get started making some really awesome inline validation.</p> 
          <h3>Validate email addresses</h3> 
          <p>HTML</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
