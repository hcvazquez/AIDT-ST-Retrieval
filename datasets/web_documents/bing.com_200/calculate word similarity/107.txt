             <div class="post-text" itemprop="text"> 
              <p>I'm using Graphlab's LDA functionality, and I get two matrices as the result: The document topic matrix (i.e. <code>P(Topic|Document)</code> for each document) and the the topic-word matrix (i.e. <code>P(Word|Topic)</code> for each topic).</p> 
              <p>What I want to accomplish is calculate the <em>document-specific</em> similarity between an arbitrary pair of words in a document, probably using Jensen Shannon divergence (though in principal, most any similarity measure could be used here). </p> 
              <p>Now, I can't just use rows of the word-topic matrix, since those really represent <code>P(Word|Topic)</code>, and to reasonably compute the similarity between two words I need <code>P(Topic|Word)</code> (i.e. a probability distribution over topics for a given word). This is doable, I believe, with some renormalization of the matrices (basically by converting the <code>P(Word|Topic)</code> values in the word-topic matrix to actual token frequencies, then normalizing word-wise instead of topic-wise). I'm pretty confident this works, but can add more detail if ther are questions. In any case, this captures the global (i.e. across the full corpus) similarity between any two words.</p> 
              <p>But now here's the kicker: <strong>Is there a principled way to calculate a <em>document-specific</em> topic distribution for a term that I could use for calculating the similarity between two terms within a particular document?</strong> To put it another way, instead of <code>P(Topic|Word)</code>, I need <code>P(Topic|Word,Document)</code>.</p> 
              <p>The intuition this should capture is that if a document is focused on a particular subset of topics, there should be more meaningful distinctions between words that are strongly associated with those topics, whereas for words strongly associated with topics that are <em>not</em> important to the document, distinctions should be less strong. Here's a concrete (toy) example where our model has only 4 topics:</p> 
              <p>Let's say we have two words, a and b, with the following topic distributions (obtained using the method I described above):</p> 
              <pre><code>In [44]: a
Out[44]: array([ 0.9 ,  0.05,  0.03,  0.02])

In [45]: b
Out[45]: array([ 0.6 ,  0.25,  0.1 ,  0.05])
</code></pre> 
              <p>The distance (Jensen Shannon Divergence) between these would be:</p> 
              <pre><code>In [46]: JSD(a,b)
Out[46]: 0.064751544117297333
</code></pre> 
              <p>Now clearly both these words are skewed towards the first two topics, and let's now imagine a document that predominantly focuses on these topics, with the following topic distribution:</p> 
              <pre><code>In [47]: doc_x
Out[47]: array([ 0.7 ,  0.2 ,  0.05,  0.05])
</code></pre> 
              <p>and a second document that favors the other two topics:</p> 
              <pre><code>In [58]: doc_y
Out[58]: array([ 0.1,  0.1,  0.5,  0.5])
</code></pre> 
              <p>So based on all this, we should expect the distance between <code>a</code> and <code>b</code> to be <em>increased</em> in <code>doc_x</code> as compared to the global JSD, because that document focuses more on the first two topics, whereas in doc_y the distance should be <em>decreased</em> as compared to the global JSD. I believe this result should come out naturally if I use distributions of <code>P(Topic|Word,Document)</code> instead of <code>P(Topic|Word)</code>.</p> 
              <p>Now, if I had the topic assignments for every token in the corpus (which in my case, is on the the order of 4 billion), this might be simpler, but I don't - just the two matrices above. Any ideas on how to go about this?</p>
