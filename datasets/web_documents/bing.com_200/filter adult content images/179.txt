        <h2><a name="Overview" id="Overview"></a>Overview</h2> 
        <p> A table that can be sorted and paged. Table cells can be formatted using format strings, or by directly inserting HTML as cell values. Numeric values are right-aligned; boolean values are displayed as check marks. Users can select single rows either with the keyboard or the mouse. Users can sort rows by clicking on column headers. The header row remains fixed as the user scrolls. The table fires a number of events corresponding to user interaction.</p>

        <h2><a name="Data_Format" id="Data_Format"></a>Data Format</h2> 
        <p> The DataTable is converted into a corresponding HTML table, with each row/column in the DataTable converted into a row/column in the HTML table. Each column must be of the same data type, and all standard visualization data types are supported (string, boolean, number, etc). </p> 
        <h4><a name="customproperties" id="customproperties"></a>Custom Properties</h4> 
        <p> You can assign the following custom properties to data table elements, using the <code><a href="https://developers.google.com/chart/interactive/docs/reference#DataTable_setProperty">setProperty()</a></code> method of <a href="https://developers.google.com/chart/interactive/docs/reference">DataTable</a>. </p> 
        <table> 
         <tbody>
          <tr> 
           <th>Property Name</th> 
           <th>Applies To</th> 
           <th>Description</th> 
          </tr> 
          <tr> 
           <td>className</td> 
           <td>Cell</td> 
           <td>A string class name to assign to an individual cell. Use this to assign CSS styling to individual cells.</td> 
          </tr> 
          <tr> 
           <td>style</td> 
           <td>Cell</td> 
           <td>A style string to assign inline to the cell. This will override CSS class styles applied to that cell. You must set the property allowHtml=true for this to work. Example: <code>'border: 1px solid green;'</code>.</td> 
          </tr> 
         </tbody>
        </table>

           <td>cssClassNames</td> 
           <td> <p> An object in which each property name describes a table element, and the property value is a string, defining a class to assign to that table element. Use this property to assign custom CSS to specific elements of your table. To use this property, assign an object, where the property name specifies the table element, and the property value is a string, specifying a class name to assign to that element. You must then define a CSS style for that class on your page. The following property names are supported: </p> 
            <ul> 
             <li> <code>headerRow</code> - Assigns a class name to the table header row (<code>&lt;tr&gt;</code> element). </li> 
             <li> <code>tableRow</code> - Assigns a class name to the non-header rows (<code>&lt;tr&gt;</code> elements). </li> 
             <li> <code>oddTableRow</code> - Assigns a class name to odd table rows (<code>&lt;tr&gt;</code> elements). <strong>Note:</strong> the alternatingRowStyle option must be set to true. </li> 
             <li> <code>selectedTableRow</code> - Assigns a class name to the selected table row (<code>&lt;tr&gt;</code> element). </li> 
             <li> <code>hoverTableRow</code> - Assigns a class name to the hovered table row (<code>&lt;tr&gt;</code> element). </li> 
             <li> <code>headerCell</code> - Assigns a class name to all cells in the header row (<code>&lt;td&gt;</code> element). </li> 
             <li> <code>tableCell</code> - Assigns a class name to all non-header table cells (<code>&lt;td&gt;</code> element). </li> 
             <li> <code>rowNumberCell</code> - Assigns a class name to the cells in the row number column (<code>&lt;td&gt;</code> element). <strong>Note:</strong> the showRowNumber option must be set to true. </li> 
            </ul> <p> <strong>Example:</strong> <code> var cssClassNames = {headerRow: 'bigAndBoldClass',<br>hoverTableRow: 'highlightClass'}; </code> </p> <p class="note"> <b>Note:</b> In CSS, some elements override others. For instance, if you specify a background color for a <code>&lt;tr&gt;</code> element and a <code>&lt;td&gt;</code> element, the latter takes precedence over the former. Be sure to specify all relevant CSS styles in <code>cssClassNames</code> to avoid conflicts. </p>

            </div>

            </div> </td>

           <td>rtlTable</td> 
           <td> <p> Adds basic support for right-to-left languages (such as Arabic or Hebrew) by reversing the column order of the table, so that column zero is the rightmost column, and the last column is the leftmost column. This does not affect the column index in the underlying data, only the order of display. Full bi-directional (BiDi) language display is not supported by the table visualization even with this option. This option will be ignored if you enable paging (using the page option), or if the table has scroll bars because you have specified height and width options smaller than the required table size. </p> 
            <div>
             <b>Type:</b> boolean
            </div> 
            <div>
             <b>Default:</b> false
            </div> </td> 
          </tr> 
          <tr> 
           <td>scrollLeftStartPosition</td> 
           <td> <p> Sets the horizontal scrolling position, in pixels, if the table has horizontal scroll bars because you have set the width property. The table will open scrolled that many pixels past the leftmost column. </p> 
            <div>
             <b>Type:</b> number
            </div> 
            <div>
             <b>Default:</b> 0
            </div> </td> 
          </tr> 
          <tr> 
           <td>showRowNumber</td> 
           <td> <p>If set to true, shows the row number as the first column of the table.</p> 
            <div>
             <b>Type:</b> boolean
            </div> 
            <div>
             <b>Default:</b> false
            </div> </td> 
          </tr> 
          <tr> 
           <td>sort</td> 
           <td> <p> If and how to sort columns when the user clicks a column heading. If sorting is enabled, consider setting the sortAscending and sortColumn properties as well. Choose one of the following string values: </p> 
            <ul> 
             <li> 'enable' - [<em>Default</em>] Users can click on column headers to sort by the clicked column. When users click on the column header, the rows will be automatically sorted, and a 'sort' event will be triggered. </li> 
             <li> 'event' - When users click on the column header, a 'sort' event will be triggered, but the rows will not be automatically sorted. This option should be used when the page implements its own sort. See the <a href="https://developers.google.com/chart/interactive/docs/examples#tablequerywrapper">TableQueryWrapper example</a> for an example of how to handle sorting events manually. </li> 
             <li>'disable' - Clicking a column header has no effect. </li> 
            </ul> 
            <div>
             <b>Type:</b> string
            </div> 
            <div>
             <b>Default:</b> 'enable'
            </div> </td> 
          </tr> 
          <tr> 
           <td>sortAscending</td> 
           <td> <p> The order in which the initial sort column is sorted. True for ascending, false for descending. Ignored if <code>sortColumn</code> is not specified. </p> 
            <div>
             <b>Type:</b> boolean
            </div> 
            <div>
             <b>Default:</b> true
            </div> </td> 
          </tr> 
          <tr> 
           <td>sortColumn</td> 
           <td> <p> An index of a column in the data table, by which the table is initially sorted. The column will be marked with a small arrow indicating the sort order. </p> 
            <div>
             <b>Type:</b> number
            </div> 
            <div>
             <b>Default:</b> -1
            </div> </td> 
          </tr> 
          <tr> 
           <td>startPage</td> 
           <td> <p> The first table page to display. Used only if <code>page</code> is in mode enable/event. </p> 
            <div>
             <b>Type:</b> number
            </div> 
            <div>
             <b>Default:</b> 0
            </div> </td> 
          </tr> 
          <tr> 
           <td>width</td> 
           <td> <p> Sets the width of the visualization's container element. You can use standard HTML units (for example, '100px', '80em', '60'). If no units are specified the number is assumed to be pixels. If not specified, the browser will adjust the width automatically to fit the table, shrinking as much as possible in the process; if set smaller than the width required, the table will add a horizontal scroll bar. If set to '100%', the table will expand as much as possible into the container element. </p> 
            <div>
             <b>Type:</b> string
            </div> 
            <div>
             <b>Default:</b> automatic
            </div> </td> 
          </tr> 
         </tbody>
        </table>
