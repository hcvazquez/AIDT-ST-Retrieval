      <h2 title="FAQ">frequently asked questions</h2> 
      <div class="accordion" id="faq"> 
       <div class="accordion-group"> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq1"> 1. Can I use ImageGenerator in commercial project for FREE? </a> 
        </div> 
        <div id="faq1" class="accordion-body collapse in"> 
         <div class="accordion-inner">
           ImageGenerator can be used for FREE in single-deployment projects (websites, intranet/extranet) or applications for company's internal business purposes (redistributed only internally inside the company). 
          <br>
          <br> Commercial license is required for: 
          <ul> 
           <li>Applications for external redistribution (ISV)</li> 
           <li>SaaS deployments</li> 
          </ul> Note that support is NOT available for free users. 
         </div> 
        </div> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq2"> 2. What about images and external resources (like css files)? </a> 
        </div> 
        <div id="faq2" class="accordion-body collapse"> 
         <div class="accordion-inner">
           It is possible to include images by specifying their FULL URL or path (if they're on local filesystem). External CSS and javascript files are also supported. 
         </div> 
        </div> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq3"> 3. Technical limitations, LT builds for Mono/.NET Core </a> 
        </div> 
        <div id="faq3" class="accordion-body collapse"> 
         <div class="accordion-inner">
           ImageGenerator embeds WkHtmlToImage x86 binaries and invokes it in separate process with System.Diagnostics.Process. In case of ASP.NET applications this might be prohibited in partial trust environments (shared ASP.NET hostings, Azure Websites). Also this means that by default non-Windows environments are NOT supported. 
          <br>
          <br> 
          <ul> 
           <li>Both .NET 2.0 and 4.0 runtimes are supported.</li> 
           <li>ImageGenerator works fine with Azure VM instances / ServiceFabric. Azure Apps (shared plans) are not supported.</li> 
           <li>Mono and .NET Core are supported by special ImageGenerator.LT build that doesn't embed wkhtmltoimage binaries (they should be deployed with your app). These builds are available only for commercial users.</li> 
          </ul> 
         </div> 
        </div> 
        <div class="accordion-heading"> 
         <a class="accordion-toggle" data-toggle="collapse" data-parent="#faq" href="#faq4"> 4. How to handle "Exit with code 1 due to network error: ..." errors </a> 
        </div> 
        <div id="faq4" class="accordion-body collapse"> 
         <div class="accordion-inner">
           In some cases image generation fails with one of the following errors: 
          <ul> 
           <li>Exit with code 1 due to network error: ContentNotFoundError</li> 
           <li>Exit with code 1 due to network error: ProtocolUnknownError</li> 
           <li>Exit with code 1 due to network error: HostNotFoundError</li> 
          </ul> This means that HTML template has external reference (js/css/image URL) that cannot be loaded by WkHtmlToImage. In most cases image can be generated anyway by specifying additional WkHtmlToImage parameter: 
          <pre>var htmlToImage = new HtmlToImageConverter();
htmlToImage.CustomArgs = " --load-media-error-handling ignore ";</pre> 
         </div>
