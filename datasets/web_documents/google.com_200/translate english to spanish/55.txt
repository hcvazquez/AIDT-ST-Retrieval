         <h2>INTRODUCTION</h2> 
         <p>There may be different reasons why you need to translate your WordPress website into other languages:</p> 
         <ul> 
          <li>You are designing a website in a<strong> country that uses multiple&nbsp;languages</strong> (in Canada, English and French; in Switzerland, French, German and Italian; etc.)</li> 
          <li>You want to make a Spanish version to <strong>sell your products or services in other parts of the world</strong> or a Russian version for russian customers.</li> 
          <li>Or just want to have your web page in several languages to <strong>reach a wider audience</strong>.</li> 
         </ul> 
         <p>Choosing the most suitable translation plugin for your needs will take some time. If you have a look at&nbsp;the WordPress Plugin Directory for a <a href="http://wordpress.org/plugins/search.php?q=multilingual" target="_blank">list of multilingual Plugins</a> you’ll find many options.</p> 
         <p>WordPress does not offer a simple solution for building multilingual websites. There are several ways to make a multilingual site. They can be divided into 4 groups:</p> 
         <ul> 
          <li><strong>Option 1</strong>: Using&nbsp;a translation plugin in a single WordPress environment (the most common way)</li> 
          <li><strong>Option 2</strong>: Using WordPress Multisite environment&nbsp;and&nbsp;a multilingual network plugin</li> 
          <li><strong>Option 3:&nbsp;</strong>Using a WordPress theme with integrated multilingual system</li> 
          <li><strong>Option 4</strong>: WordPress localization with translation&nbsp;proxy</li> 
         </ul> 
         <p>In this article, I’d like to guide you through the different&nbsp;options available to have a multilanguage WordPress site. Which is the best option? It depends on many factors:</p> 
         <ul> 
          <li><strong>Translation</strong> – Do you want to use machine translation or human translation?</li> 
          <li><strong>Cost</strong> – What’s your budget for the multilingual project?</li> 
          <li><strong>Support</strong> – Do you want to have technical support?</li> 
          <li><strong>Speed</strong> – How can you provide a good user experience without decreasing the speed of the application?</li> 
          <li><strong>Size</strong> – How large is your website?</li> 
          <li><strong>Linking</strong> – Does each post or page always have a translation and do they need to be linked to each other?</li> 
          <li><strong>User profile</strong> – Are you a website owner, a freelance translator or a language service provider?</li> 
         </ul> 
         <p><strong>SOME TIPS</strong>:</p> 
         <ul> 
          <li>Since WordPress 4.1 you can change your site language (and install new languages) from the WordPress dashboard. You don’t need to modify WPLANG in wp-config.php file (which has disappeared). Go to <strong>Settings &gt; General &gt; Site Language</strong>, and select a new language (or&nbsp;install a new one). More info: <a href="http://wplang.org/changing-wordpress-language/" target="_blank">WordPress 100% In My Language</a></li> 
          <li>Before starting a multilingual WordPress site is highly recommended choosing a&nbsp;<strong>multilingual theme</strong>&nbsp;already translated into other languages. More info:&nbsp;<a href="http://wplang.org/wordpress-multilingual-themes/" target="_blank">A Guide to Choosing a Multilingual WordPress Theme</a></li> 
          <li>You can choose <strong>different URL formats</strong> when building a multilingual website. More info in this article: <a href="http://wplang.org/domain-subdomain-subdirectory/" target="_blank">Domain, Subdomain, Subdirectory, Languages and WordPress</a></li> 
          <li>Also you should implement a <strong>multilingual SEO strategy</strong> for search engines (Google, Bing, Yandex –Russia- or Baidu –China-). More info in this post: <a href="http://wplang.org/multilanguage-seo-wordpress/" target="_blank">Multilingual SEO for WordPress Sites</a></li> 
         </ul>

         <p><a href="http://wplang.org/plugins/gtranslate" target="_blank"><img class="alignright size-full wp-image-5931" src="http://wplang-1109.kxcdn.com/wp-content/uploads/2015/12/Translate-WordPress-with-GTranslate-—-WordPress-Plugins-ratings.jpg" alt="Translate WordPress with GTranslate — WordPress Plugins - ratings" width="228" height="341" srcset="http://wplang-1109.kxcdn.com/wp-content/uploads/2015/12/Translate-WordPress-with-GTranslate-—-WordPress-Plugins-ratings.jpg 228w, http://wplang-1109.kxcdn.com/wp-content/uploads/2015/12/Translate-WordPress-with-GTranslate-—-WordPress-Plugins-ratings-201x300.jpg 201w" sizes="(max-width: 228px) 100vw, 228px">GTranslate</a>&nbsp;is a plugin that can be used for automatic&nbsp;machine&nbsp;translations. This <strong>free plugin</strong> allows you to insert the Google Language Translator tool onto your website using a widget.</p> 
         <p><strong>Advantages</strong></p> 
         <ul> 
          <li>It’s the cheapest option. You don’t need to perform the costly task of translating your website or hire a translator.</li> 
         </ul> 
         <p><strong>Disadvantages</strong></p> 
         <ul> 
          <li>Translations often don’t make sense. Search engines could consider them as spam. What to do in these cases is prevent these translations from being indexed using the robots.txt file and allowing users to request the translation via a widget.</li> 
         </ul> 
         <p>The plugin also offers a paid version with more advanced features: edit translations, URL translation, metadata translation, etc.</p> 
         <p><strong>Conclusion</strong>: the best option to use automatic machine translations.</p>

Also you can hire world-class translators from within the WordPress admin dashboard, get affordable rates (about $0,09 / word) and enjoy a simple translation workflow (<a href="http://www.icanlocalize.com/my/invite/57633" target="_blank">ICanLocalize</a> translation services).</li>

They will refund 100% of your payment – no questions asked (<a href="http://wplang.org/purchase/refunds-policy" target="_blank" rel="nofollow">Refunds Policy</a>).          <h2>OPTION 2. WordPress Multisite: One website per language</h2> 
         <p><a name="2.1"></a></p> 
         <h3 style="padding-left: 30px;">2.1. WordPress Multisite (no need to use network plugins)</h3> 
         <p><img class="aligncenter wp-image-58" src="http://wplang-1109.kxcdn.com/wp-content/uploads/2014/06/Wordpress-Multisite-Languages.jpg" alt="Wordpress Multisite Languages" width="414" height="111" srcset="http://wplang-1109.kxcdn.com/wp-content/uploads/2014/06/Wordpress-Multisite-Languages.jpg 690w, http://wplang-1109.kxcdn.com/wp-content/uploads/2014/06/Wordpress-Multisite-Languages-300x80.jpg 300w" sizes="(max-width: 414px) 100vw, 414px"></p> 
         <p>Since WordPress 3.0 it is possible to build&nbsp;a <a href="http://codex.wordpress.org/Create_A_Network" target="_blank">WordPress Multisite installation</a>. It is a collection of sites that share the same WordPress installation. They can share themes and plugins. The individual sites are virtual sites in the sense that they do not have their own directories on your server, although they do have separate directories for media uploads, and they do have separate tables in the database.</p> 
         <p>This will allow you to create <strong>one website per language</strong>. The main advantage is that it’s native, using WordPress core functionality, so it’s safe and free to use. This way you can create each website in a different language within your network.</p> 
         <p><strong>WordPress Multisite Advantages</strong></p> 
         <ul> 
          <li>The main advantage over the WPML plugin is that WordPress will be native in every language. This means you can have a primary address (mydomain.com), a subdomain or subdirectory for the Spanish version (es.mydomain.com or mydomain.com/es), one for the French version (fr.mydomain.com or mydomain.com/fr), etc.</li> 
          <li>If you need to set up multiple sites across multiple domains (mydomain.com, mydomain.es, mydomain.fr, etc), you can use the <a href="https://wordpress.org/plugins/wordpress-mu-domain-mapping/" target="_blank">WordPress MU Domain Mapping</a> plugin as well – as long as the domains are all hosted on the same server.</li> 
          <li>It’s an excellent option for large and complex sites, offering minor compatibility issues without additional costs.</li> 
         </ul> 
         <p><strong>WordPress Multisite Disadvantages</strong></p> 
         <ul> 
          <li>It is more difficult to configure and manage: 
           <ul> 
            <li>First, you have to create a network of sites by using the multisite feature.</li> 
            <li>Once installed, every time you make an adjustment (themes, plugins, menus, widgets, etc) you must port it to all the websites. If there are just two languages that’s not a&nbsp;big deal,&nbsp;but&nbsp;with three or more it can be a big problem.</li> 
            <li>Also, keep in mind that some plugins an themes do not work properly on Multisite installations.</li> 
           </ul> </li> 
          <li>WordPress Multisite was not originally intended for creating multilingual website.</li> 
          <li>Managing translation of content is more laborious because it’s easy to lose track of the contents which have been translated or no.</li> 
         </ul> 
         <p>In this option,&nbsp;<strong>you only need to insert a language switcher into the header</strong> to being redirected to the homepage when switching from one language to another. Many multilingual websites are very different in each language so it’s not&nbsp;always&nbsp;necessary link contents (posts, pages, categories, tags, etc.).</p> 
         <p><strong>Conclusion</strong>: The best option&nbsp;when each post or page don’t need to be linked to each other</p>

         <h2>CONCLUSIONS</h2> 
         <p>As you can see, there is no a better or worse solution to have your blog or WordPress site in two or more languages. <strong>It depends on many&nbsp;factors</strong>: functionality, price, machine or human translations, ease to manage, technical support, SEO, etc.</p> 
         <ul> 
          <li>If you don’t want to spend your <strong>money</strong> with plugins, I recommend you to use Polylang or Zanto / Multisite Language Switcher (WordPress Multisite).&nbsp;If you want support and updates and money it’s not a problem, use WPML (single WordPress), Multilingual Press (WordPress Multisite) or&nbsp;a WordPress premium theme with integrated multilingual system such as AitThemes.Club.</li> 
          <li>If you’re a WordPress <strong>beginner</strong> user you can use option 1: WPML or Polylang.</li> 
          <li>If you have a <strong>small or medium site</strong>, use WPML. If you have a <strong>large site</strong>, use Multisite (Multilingual Press, Zanto or&nbsp;Multisite Language Switcher).</li> 
          <li>If you don’t need to <strong>link posts and&nbsp;pages</strong>, then go Multisite (no need to use plugins). Otherwise you can use WPML or Multisite + Network plugin.</li> 
          <li>If <strong>Search Engine Optimization</strong> (SEO) is important for you, don’t use automated translation. However&nbsp;if you want to use automatic translations, Ajax Revolution Translator&nbsp;is the&nbsp;best choice since allows you also make your own translations.</li> 
          <li>If you’re a <strong>language service provider</strong> you might want to use Easyling.</li> 
         </ul> 
         <p><strong>In summary</strong>, if you want to create a professional, easy to manage, multilingual WordPress website, I&nbsp;recommend you use a professional payment solution with good support such as <a href="http://wplang.org/plugins/wpml" target="_blank" rel="nofollow">WPML</a>&nbsp;(single WordPress),&nbsp;<a href="http://marketpress.com/product/multilingual-press-pro/" target="_blank">Multilingual Press</a>&nbsp;(WordPress&nbsp;Multisite) or <a href="http://wplang.org/wordpress/ait-themes" target="_blank">AitThemes</a> (multilingual WordPress themes right out of the box). In other cases it may be enough to use a free plugin such as <a href="http://wordpress.org/plugins/polylang/" target="_blank">Polylang</a>&nbsp;(single WordPress)&nbsp;or&nbsp;<a href="https://wordpress.org/plugins/multisite-language-switcher/" target="_blank">Multisite Language Switcher</a>&nbsp;(WordPress Multisite).</p> 
         <p>&nbsp;</p> 
         <p style="text-align: center;"><em><strong>I hope this article&nbsp;will help you&nbsp;to choose the best option for you.</strong></em><br> <a name="6"></a></p> 
         <p>&nbsp;</p>
