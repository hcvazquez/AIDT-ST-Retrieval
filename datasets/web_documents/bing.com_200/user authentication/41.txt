           <p> A previous tutorial, "<a href="http://ibm.com/developerworks/library/se-use-ldap-authentication-authorization-node.js-bluemix-application/index.html">Use LDAP for authentication and authorization in your Node.js Bluemix application,</a>" explained how to use your internal LDAP server for authentication and authorization in a Node.js Bluemix® application. But what if your organization uses Microsoft™ Active Directory? Active Directory follows the LDAP standard, for the most part, but it does have a few important differences.</p>

           <h2 id="14ActiveDirectoryvsLDAPoutline" class="ibm-h2">Active Directory versus LDAP</h2>
           <p>Microsoft Active Directory has an LDAP interface. However, as in many other areas, Microsoft has a different interpretation of the standard. Here are the differences that are relevant to logging users in and checking them for group membership:</p>
           <table border="0" cellpadding="0" cellspacing="0" class="ibm-data-table" data-widget="datatable">
            <thead xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
             <tr>
              <th class="ibm-background-neutral-white-30"></th>
              <th class="ibm-background-neutral-white-30"> LDAP </th>
              <th class="ibm-background-neutral-white-30"> Microsoft Active Directory </th>
             </tr>
            </thead>
            <tbody xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
             <tr>
              <td> User identifier </td>
              <td><code>uid</code></td>
              <td> Either <code>sAMAccountName</code> (user identifier only) or <code>userPrincipalName</code> (&lt;user identifier&gt;@&lt;domain&gt;) </td>
             </tr>
             <tr>
              <td> Bind identifier </td>
              <td> Distinguished name </td>
              <td> You can use the distinguished name if you know it, but you can also use <code>userPrincipalName</code></td>
             </tr>
             <tr>
              <td> Suffix </td>
              <td> Many options </td>
              <td> Always the domain name as dc (domain component) attributes. For example, for ibm.com, the suffix would be <code>dc=ibm,dc=com</code></td>
             </tr>
             <tr>
              <td> Group membership </td>
              <td> Encoded in the group object as a <code>member</code> attribute </td>
              <td> Encoded in the user object as a <code>memberOf</code> attribute </td>
             </tr>
            </tbody>
           </table>
           <h2 id="15Thelogonflowoutline" class="ibm-h2">The login flow</h2>
           <p>You do not need the user's distinguished name, so a single LDAP connection is sufficient. Just create a client with the correct LDAP URL, and attempt to bind with the user's <code>userPrincipalName</code>, which would typically be a valid e-mail address and password. If the information is wrong, the bind would ordinarily fail. However, there is one exception. If the password is empty, then any bind attempt would succeed. It is easy to block this in the code that processes the login form:</p>
           <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">// Process the login form for Active Directory
app.post("/adlogin", function(req, res) {
	var url = "ldap://" + req.body.ad;
	var userPrincipalName = req.body.uid + "@" + req.body.domain;
	var passwd = req.body.passwd;

	if (passwd === "") {
		res.send("The empty password trick does not work here.");
		return ;
	}

	// Bind as the user
	var adClient = ldap.createClient({ url: url });
	adClient.bind(userPrincipalName, passwd, function(err) {

		if (err != null) {
			if (err.name === "InvalidCredentialsError")
				res.send("Credential error");
			else
				res.send("Unknown error: " + JSON.stringify(err));
		} else
			res.send("Hello " + req.body.uid);
	});
});</pre></span>
           <p>Once logged in, you can manage sessions as you would with an LDAP login.</p>
           <h2 id="16Usinggroupsforauthorizationoutline" class="ibm-h2">Use groups for authorization</h2>
           <p>The group membership is part of the user object, so the simplest way to get it is to read that object:</p>
           <ol type="1">
            <li>Get the suffix for use as the base of the search. The suffix in Active Directory is made up of the domain by making each dot-separated value into a domain component. For example, the domain <code>austin.ibm.com</code> yields the suffix <code>dc=austin,dc=ibm,dc=com</code>. <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">	var suffix = "dc=" + req.body.domain.replace(/\./g, ",dc=");</pre></span></li>
            <li>Search for the user who uses the <code>userPrincipalName</code> attribute, which is the same as the value used to log in to the LDAP. <span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">	adClient.search(suffix,
		{
			scope: "sub",
			filter: "(userPrincipalName=" + userPrincipalName + ")"
		},</pre></span></li>
            <li>There will be a result, because you just logged in with that user. However, the <code>memberOf</code> attribute in the result could have one of three values: 
             <ul class="ibm-bullet-list">
              <li>Undefined: There is no such attribute because the user is not a member of any group.</li>
              <li>A string: The user is a member of exactly one group, and the string contains the distinguished name (DN) of that group.</li>
              <li>An array: The user is a member of two or more groups. Each member of the array is a string with the DN of a group.</li>
             </ul><p>The following code handles all three possibilities:</p><span class="dw-code-nohighlight"><pre data-widget="syntaxhighlighter" class="brush: js; html-script: true; gutter: true;">// When we get a result, that is the user data	
ldapResult.on('searchEntry', function(entry) {
		var groups = entry.object.memberOf;
		
		// We want to handle a string (single value) as an array
		if (typeof groups === "string")
			groups = [groups];
						
		var html = "";
		html += "&lt;h2&gt;Hello " + entry.object.cn + "&lt;/h2&gt;";
		if (groups != undefined) {
			html += "&lt;h4&gt;Groups:&lt;/h4&gt;&lt;ol&gt;";
			for (var i=0; i&lt;groups.length; i++)
				html += "&lt;li&gt;" + groups[i]  + "&lt;/li&gt;";
				html += "&lt;/ol&gt;";
		} else
			html += "No group membership detected.";

						
		res.send(html);
	});</pre></span></li>
           </ol>
           <h3 id="1.7.Conclusion|outline" class="ibm-h3">Conclusion</h3>
           <p>You should now be able to use Microsoft Active Directory as the user repository for your Bluemix Node.js applications.</p>
