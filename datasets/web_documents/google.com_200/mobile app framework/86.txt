       <p><strong>Update (September 30, 2016)</strong>: Update article to recent ionic RC.0 release</p> 
       <blockquote> 
        <p>This article has been updated to use Ionic 2 RC.0 Release</p>
        <div class="notification m-b-md">
         Related Course: 
         <a href="https://scotch.io/courses/getting-started-with-angular-2">Getting Started with Angular v2+</a>
        </div> 
       </blockquote> 
       <p><a href="https://ionicframework.com/">The Ionic Framework</a> enables the creation of cross platform mobile applications with HTML, CSS and JavaScript(Angular). Ionic 1 was built with <em>Angular 1.*</em>, and with <a href="https://angular.io/">Angular 2</a> recently released, the second major version of Ionic is also imminent. </p> 
       <p><img data-original="https://cdn.scotch.io/1/zDr051u7RBakE035eK8Y_4YxQWTz.png" alt="Ionic Home Page"></p> 
       <p><a href="https://ionicframework.com/docs/v2/">Ionic 2</a> has just hit <a href="https://github.com/driftyco/ionic/milestones">Release Candidate</a> stage, and if you are looking to build cross-platform apps quickly, and you already know Angluar 2 or JavaScript, this guide will get you up to speed.</p> 
       <h2>Why Choose Ionic?</h2> 
       <h3>You already know HTML/CSS/JS</h3> 
       <p>You can leverage the skills you already have from developing web applications with HTML, JavaScript and CSS to build cross-platform mobile apps. If you work with Angular 2, this will be a seamless transitions.</p> 
       <h3>Take advantage of Progressive Web Apps</h3> 
       <p>Google has been talking about <a href="http://blog.ionic.io/what-is-a-progressive-web-app/">Progressive Web Apps</a>. These are simply web apps that give an app-like user experience to users, and are built with Web Technologies and <a href="https://ionicframework.com/docs/v2/resources/progressive-web-apps/">Ionic 2 is at the fore front</a> of implementing this.</p> 
       <p>In fact, in the last Angular Connect conference, some members of the Ionic team spoke about ionic and progressive web apps.</p> 
       <p>Examples of progressive web apps made with ionic 2 can be found in <a href="http://pwa.ionic.io">pwa.ionic.io</a>. Just visit any of the apps in your chrome browser on your phone, click on menu and tap <strong>Add to Home Screen</strong>. You'll then be able to load it as an app from your home screen.</p> 
       <h3>Target all major mobile platforms</h3> 
       <p>If you need to quickly build an app for all major mobile platforms (Android, iOS and Windows Phone), having one codebase may be the fastest way to do it, and Ionic is perfect for such a scenario. Updating the app, or rolling uout updates is just as easy as editing one code base. </p> 
       <h3>Native Functionality is easy to implement</h3> 
       <p>There are <a href="https://cordova.apache.org/plugins/">a lot of cordova plugins</a> that allow you to include native functionality of the platform you are building for. Ionic provides a feature called <strong>ionic-native</strong> that enables you use most of the cordova plugins as providers in your ionic app. The Ionic 2 documentation has <a href="https://ionicframework.com/docs/v2/native/">implimentation examples of some of the plugins</a>.</p> 
       <h2>What We'll be Building</h2> 
       <p>We will build a simple app that consumes the <a href="https://api.github.com">github api</a>. Our app will list github users, offer a search box for searching users, and be able to view the number of followers, repos, and gists the user has.</p> 
       <p>Below is a short video of the finished app.</p> 
       <iframe width="420" height="500" src="https://www.youtube.com/embed/J2tnT-WFpyg" frameborder="0"></iframe> 
       <h2>Install Ionic 2</h2> 
       <p>Ionic 2 is built on top of <a href="https://cordova.apache.org/">Cordova</a>, a framework that enables development for cross patform apps with HTML, CSS and JavaScript. Make sure you have <a href="https://nodejs.org/en/">nodejs</a> installed.</p> 
       <p>Install Cordova</p> 
       <pre><code class="language-bash">$ npm install -g cordova</code></pre> 
       <p>For Mac/Linux/Unix users, if you see errors with <code>EACCESS</code>, you can run the command as root user with <code>sudo npm install -g cordova</code>, or <a href="https://docs.npmjs.com/getting-started/fixing-npm-permissions">just fix your npm persmissions.</a>.</p> 
       <p>Install Ionic CLI</p> 
       <pre><code class="language-bash">$ npm install -g ionic</code></pre> 
       <p><em>Note: If you see errors you don't understand, you may want to uninstall previous versions of ionic-cli with <code>npm uninstall -g ionic</code>, clear your cache with <code>npm cache clean</code> and then now install ionic-cli with <code>npm install -g ionic</code></em></p> 
       <p>This installs the Ionic CLI into your workstation, and now we are good to go.</p> 
       <p><strong>NOTE</strong> To check the details of your installed ionic, simply run</p> 
       <pre><code class="language-bash">$ ionic info</code></pre> 
       <p>You should see results almost similar to this <img data-original="https://cdn.scotch.io/303/AJFBV3wfSP2fMsT0BSQ9_Screen%20Shot%202016-09-28%20at%2012.16.08.png" alt=""></p> 
       <p>You can ignore the warnings for now. Most of them are only necessary when we want to deploy or build to either ios or android.</p> 
       <p>Finally, we need to have <a href="https://www.typescriptlang.org/">TypeScript</a>. TypeScript is a superset of JavaScript, and works really well with Angular 2, you can go through the articles <a href="https://pub.scotch.io/@pelekes">by this author</a> to quickly get up to speed with TypeScript.</p> 
       <pre><code class="language-bash">$ npm install -g typescript</code></pre> 
       <h2>Hello Ionic 2</h2> 
       <p>Ionic 2 provides a simple starter template to quickly scaffold a simple app. Our app will be called githubIonic, and so we'll quickly scaffold it with the <a href="https://ionicframework.com/docs/v2/cli/">ionic cli</a>.</p> 
       <p>In your terminal:</p> 
       <pre><code class="language-bash">$ ionic start githubIonic tutorial --v2</code></pre> 
       <p>The above command scaffolds an ionic project based on a tutorial template ionic provides. It then downloads the necessary npm packages. We'll build the application using TypeScript, and that's why we have a <code>--ts</code> flag. </p> 
       <p>Go to the <strong>githubIonic</strong> directory with <code>cd githubIonic</code>. We'll serve the app first, then go through the folder structure. In your terminal, run:</p> 
       <pre><code class="language-bash">$ ionic serve</code></pre> 
       <p>You may be asked to select how you would like to serve the ionic app, select <em>localhost:8100</em> option. Open <code>http://localhost:8100</code> in your browser, it should looks something similar to this.</p> 
       <p><img data-original="https://cdn.scotch.io/303/pRoXQzjxQ692Bj1IO6Kx_Screen%20Shot%202016-09-30%20at%2010.36.56.png" alt=""></p> 
       <p>Notice that I have opened my <em>Chrome Dev Tools</em>, and on the top left part, clicked on the devices Icon so that my app is displayed on a device screen. In this case, the Galaxy S5.</p> 
       <p>To see the app rendered in all mobile platforms, you can run </p> 
       <pre><code class="language-bash">$ ionic serve -l</code></pre> 
       <p>It should open your browser, but if not, visit <code>http://localhost:8100/ionic-lab</code></p> 
       <p><img data-original="https://cdn.scotch.io/303/KGEuESr8SWa7X0mqYoJu_Screen%20Shot%202016-09-30%20at%2010.41.07.png" alt=""></p> 
       <p>You can play around with the app.</p> 
       <h2>Ionic 2 Project Structure</h2> 
       <p>When you first generate an ionic 2 app, This is the generated project structure </p> 
       <pre><code class="language-markdown">??? config.xml
??? hooks
??? ionic.config.json
??? node_modules
??? package.json
??? platforms
??? plugins
??? resources
??? src
??? tsconfig.json
??? tslint.json</code></pre> 
       <p>We will almost always spend 90% of the time in the <strong>src</strong> folder, as that's where the application logic is. The whole app however is usually loaded from a simple <strong>src/index.html</strong> file, which during build is copied to a new folder <strong>www</strong></p> 
       <p><strong>src/index.html</strong></p>

@Injectable()

  constructor(public http: Http) { }

  load(): Observable&lt;User[]&gt; {

       <h3>Get correct user details.</h3> 
       <p>Now that we have the user in our details vew, we need to get his specific details. To do this we need to edit the <strong>GithubUsers</strong> provider to handle the request. The request will be made to <code>https://api.github.com/users/{login}</code>, where we pass in the username/login as the last parameter.</p> 
       <p><strong>src/providers/github-users.ts</strong></p> 
       <pre><code class="language-typescript">// Imports

@Injectable()
export class GithubUsers {
  githubApiUrl = 'https://api.github.com';

  constructor(public http: Http) { }

  // Load all github users
  load(): Observable&lt;User[]&gt; {
   // Load useres
  }

  // Get github user by providing login(username)
  loadDetails(login: string): Observable&lt;User&gt; {
    return this.http.get(`${this.githubApiUrl}/users/${login}`)
      .map(res =&gt; &lt;User&gt;(res.json()))
  }
}</code></pre> 
       <p>We've added an almost identical method to the initail <code>load()</code> called <code>loadDetails()</code>. It takes in a string <em>login</em> as a parameter and returns an Observable of User, that we can subscribe to, to get the results of the request. It casts the response to the User model with <code>&lt;User&gt;res.json()</code>.</p> 
       <p>The <code>http.get</code> request is sent to <code>${this.githubApiUrl}/users/${login}</code>. This is ES6 template strings, also available in typescript. It Involves using backticks, the key below the escape key on the keyboard, and passing any values with the <code>${var}</code> syntax, and it will reolve to a valid string.</p>

@Injectable()
export class GithubUsers {
  githubApiUrl = 'https://api.github.com';

  constructor(public http: Http) { }

  // Load all github users
  load(): Observable&lt;User[]&gt; {
    Load Users
  }

  // Get github user by providing login(username)
  loadDetails(login: string): Observable&lt;User&gt; {
    // Load Details
  }

   // Search for github users  
  searchUsers(searchParam: string): Observable&lt;User[]&gt; {
    return this.http.get(`${this.githubApiUrl}/search/users?q=${searchParam}`) 
      .map(res =&gt; &lt;User[]&gt;(res.json().items))

       <h3>Simple Search</h3> 
       <p>We'll only search when three or more characters have been typed.</p> 
       <p>Let's capture the value typed in from the user in the searchbar.</p> 
       <p><strong>src/pages/users/user.html</strong></p> 
       <pre><code class="language-html">&lt;!-- HTML commented out for brevity --&gt;

&lt;ion-content padding &gt;
  &lt;ion-searchbar (input)="search($event)"&gt;&lt;/ion-searchbar&gt;
  &lt;ion-list&gt;
    &lt;!-- HTML commented out for brevity --&gt;
  &lt;/ion-list&gt;
&lt;/ion-content&gt;</code></pre> 
       <p>We've added an <code>(input)="search($event)"</code> in the <code>ion-searchbar</code> to capture the input event for the searchbar. Angular 2 binds to events through <code>(event)</code> syntax. </p> 
       <p><strong>src/pages/users/users.ts</strong></p> 
       <pre><code class="language-typescript">// Imports commented out for brevity

@Component({
  selector: 'page-users',
  templateUrl: 'users.html'
})
export class UsersPage {
  users: User[]
  originalUsers: User[];

  constructor(public navCtrl: NavController, private githubUsers: GithubUsers) {
    githubUsers.load().subscribe(users =&gt; {
      this.users = users;
      this.originalUsers = users;
    })
  }

  goToDetails(login: string) {
    this.navCtrl.push(UserDetailsPage, {login});
  }

  search(searchEvent) {
    let term = searchEvent.target.value
    // We will only perform the search if we have 3 or more characters
    if (term.trim() === '' || term.trim().length &lt; 3) {
      // Load cached users
      this.users = this.originalUsers;
    } else {
      // Get the searched users from github
      this.githubUsers.searchUsers(term).subscribe(users =&gt; {
        this.users = users
      });
    }
  }
}
</code></pre> 
       <p>We've added an <code>originalUsers</code> class property, to cache the original results of the github users fetch.</p> 
       <p>Our search function is structured to only work when the search parameter is of character length that is grater than 3. We reset the results if this condition is not met. (This is not robust however, because the user can enter non-alphanumeric characters, but for learning purposes, will do).</p> 
       <p>Go to <code>http://localhost:8100</code>, and try searching for a term, or a username you know. <img data-original="https://cdn.scotch.io/303/sFQUAfz0S7AkWFZiOCOU_Screen%20Shot%202016-06-08%20at%2000.53.19.png" alt=""> Clicking any of the result should take you to that user's profile.</p> 
       <h2>Build/Run the application.</h2> 
       <h3>Android</h3> 
       <p>You need to have the android sdk set up. The installation instructions for your specific platform can be <a href="https://ionicframework.com/docs/guide/installation.html">found here</a>.</p> 
       <p>Add android platform to your application</p> 
       <pre><code class="language-bash"> $ ionic platform add android</code></pre> 
       <p>This adds a folder <strong>android</strong> within the <strong>platforms</strong> folder in your app and adds the neccesarry resources needed to build an android application.</p> 
       <p>Build the android app</p> 
       <pre><code class="language-bash"> $ ionic build android</code></pre> 
       <p>This builds the application, and puts the built apk in the <strong>platforms/android/build/outputs</strong> folder.</p> 
       <p>Run the android app</p> 
       <pre><code class="language-bash"> $ ionic run android</code></pre> 
       <p>This will run the app in either your default emulator, or a phone connected to your computer with usb debugging enabled.</p> 
       <h3>IOS</h3> 
       <p>To build for ios, you need to be using MacOs and have <a href="https://developer.apple.com/xcode/">xCode</a> installed. You also need two node modules installed globally. They just help in building and deploying the app.</p> 
       <pre><code class="language-bash">$ npm install -g ios-deploy
$ npm install -g ios-sim version</code></pre> 
       <p>Add ios platform to your project.</p> 
       <pre><code class="language-bash"> $ ionic platform add ios</code></pre> 
       <p>This adds a folder <strong>ios</strong> within the <strong>platforms</strong> folder in your app and adds the necesarry resources needed to build an ios application.</p> 
       <p>Build your app for ios</p> 
       <pre><code class="language-bash"> $ ionic build ios</code></pre> 
       <p>This builds the application, and puts the build output in the <strong>platforms/ios/build/emulator</strong> folder.</p> 
       <p>Then run the application in an ios simulator that comes installed with xCode</p> 
       <pre><code class="language-bash"> $ ionic run ios</code></pre> 
       <h2>Conclusion</h2> 
       <p>Building Ionic 2 Apps is pretty intuitive and cleaner once you have an understanding of how Angular 2 works. Ionic handles its views as components. You can build an application really quickly, especially if you have your api already defined somewhere.</p> 
       <p>I think Ionic plays a major role, especially in the development of <a href="https://developers.google.com/web/progressive-web-apps/">Progressive Web Apps</a>. If anyone is looking to invest some time in them, they should definitely give Ionic a shot.</p>
