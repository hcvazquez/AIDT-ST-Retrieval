    <span class="na">birthday</span><span class="p">:</span> <span class="s1">'date'</span><span class="p">,</span>
    <span class="na">cars</span><span class="p">:</span>     <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'list'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">},</span>
    <span class="na">picture</span><span class="p">:</span>  <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'data'</span><span class="p">,</span> <span class="na">optional</span><span class="p">:</span> <span class="kc">true</span><span class="p">},</span> <span class="c1">// optional property</span>
  <span class="p">}</span>
<span class="p">};</span>

<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">CarSchema</span><span class="p">,</span> <span class="nx">PersonSchema</span><span class="p">]})</span>

    <span class="c1">// Create Realm objects and write to local storage</span>
    <span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
      <span class="kr">const</span> <span class="nx">myCar</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">,</span> <span class="p">{</span>
        <span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span>
        <span class="na">model</span><span class="p">:</span> <span class="s1">'Civic'</span><span class="p">,</span>
        <span class="na">miles</span><span class="p">:</span> <span class="mi">1000</span><span class="p">,</span>
      <span class="p">});</span>
      <span class="nx">myCar</span><span class="p">.</span><span class="nx">miles</span> <span class="o">+=</span> <span class="mi">20</span><span class="p">;</span> <span class="c1">// Update a property value</span>
    <span class="p">});</span>

    <span class="c1">// Query Realm for all cars with a high mileage</span>
    <span class="kr">const</span> <span class="nx">cars</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">).</span><span class="nx">filtered</span><span class="p">(</span><span class="s1">'miles &gt; 1000'</span><span class="p">);</span>

    <span class="c1">// Will return a Results object with our 1 car</span>
    <span class="nx">cars</span><span class="p">.</span><span class="nx">length</span> <span class="c1">// =&gt; 1</span>

    <span class="c1">// Add another car</span>
    <span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
      <span class="kr">const</span> <span class="nx">myCar</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">,</span> <span class="p">{</span>
        <span class="na">make</span><span class="p">:</span> <span class="s1">'Ford'</span><span class="p">,</span>
        <span class="na">model</span><span class="p">:</span> <span class="s1">'Focus'</span><span class="p">,</span>
        <span class="na">miles</span><span class="p">:</span> <span class="mi">2000</span><span class="p">,</span>
      <span class="p">});</span>
    <span class="p">});</span>

    <span class="c1">// Query results are updated in realtime</span>
    <span class="nx">cars</span><span class="p">.</span><span class="nx">length</span> <span class="c1">// =&gt; 2</span>
  <span class="p">});</span></code></pre>

      <h2 id="models">Models</h2> 
      <p>Realm data models are defined by the schema information passed into a Realm during initialization. The schema for an object consists of the object’s <code>name</code> and a set of properties each of which has a <code>name</code> and <code>type</code> as well as the <code>objectType</code> for object and list properties. You can also designate each property to be <code>optional</code> or to have a <code>default</code> value.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">Realm</span> <span class="o">=</span> <span class="nx">require</span><span class="p">(</span><span class="s1">'realm'</span><span class="p">);</span>

<span class="kr">const</span> <span class="nx">CarSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">make</span><span class="p">:</span>  <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">model</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">miles</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'int'</span><span class="p">,</span> <span class="na">default</span><span class="p">:</span> <span class="mi">0</span><span class="p">},</span>
  <span class="p">}</span>
<span class="p">};</span>
<span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span>     <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">birthday</span><span class="p">:</span> <span class="s1">'date'</span><span class="p">,</span>
    <span class="na">cars</span><span class="p">:</span>     <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'list'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">},</span>
    <span class="na">picture</span><span class="p">:</span>  <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'data'</span><span class="p">,</span> <span class="na">optional</span><span class="p">:</span> <span class="kc">true</span><span class="p">},</span> <span class="c1">// optional property</span>
  <span class="p">}</span>
<span class="p">};</span>

<span class="c1">// Initialize a Realm with Car and Person models</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">CarSchema</span><span class="p">,</span> <span class="nx">PersonSchema</span><span class="p">]})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// ... use the realm instance to read and modify data</span>
  <span class="p">})</span></code></pre> 
      <h3 id="classes">Classes</h3> 
      <div class="alert alert-warning"> 
       <p>At this point, support for defining models via classes is limited. It works in React Native but not in Node.</p> 
      </div> 
      <p>If you want to use ES2015 classes (and maybe want to inherit existing features), you just need to define the schema on the constructor:</p> 
      <pre class="highlight javascript"><code><span class="kr">class</span> <span class="nx">Person</span> <span class="p">{</span>
  <span class="nx">get</span> <span class="nx">fullName</span><span class="p">()</span> <span class="p">{</span>
    <span class="k">return</span> <span class="k">this</span><span class="p">.</span><span class="nx">firstName</span> <span class="o">+</span> <span class="s1">' '</span> <span class="o">+</span> <span class="k">this</span><span class="p">.</span><span class="nx">lastName</span><span class="p">;</span>
  <span class="p">}</span>
<span class="p">}</span>

<span class="nx">Person</span><span class="p">.</span><span class="nx">schema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">firstName</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>
    <span class="na">lastName</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">}</span>
  <span class="p">}</span>
<span class="p">};</span></code></pre> 
      <p>You can now pass the class itself to the <code>schema</code> property of the open configuration:</p> 
      <pre class="highlight javascript"><code><span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">Person</span><span class="p">]})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span> <span class="cm">/* ... */</span> <span class="p">);</span></code></pre> 
      <p>You access properties as always:</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="kr">const</span> <span class="nx">john</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">,</span> <span class="p">{</span>
    <span class="na">firstName</span><span class="p">:</span> <span class="s1">'John'</span><span class="p">,</span>
    <span class="na">lastName</span><span class="p">:</span> <span class="s1">'Smith'</span>
  <span class="p">});</span>
  <span class="nx">john</span><span class="p">.</span><span class="nx">lastName</span> <span class="o">=</span> <span class="s1">'Peterson'</span><span class="p">;</span>
  <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">john</span><span class="p">.</span><span class="nx">fullName</span><span class="p">);</span> <span class="c1">// -&gt; 'John Peterson'</span>
<span class="p">});</span></code></pre> 
      <h3 id="supported-types">Supported types</h3> 
      <p>Realm supports the following basic types: <code>bool</code>, <code>int</code>, <code>float</code>, <code>double</code>, <code>string</code>, <code>data</code>, and <code>date</code>.</p> 
      <ul> 
       <li><code>bool</code> properties map to JavaScript <code>Boolean</code> objects</li> 
       <li><code>int</code>, <code>float</code>, and <code>double</code> properties map to JavaScript <code>Number</code> objects. Internally ‘int’ and ‘double’ are stored as 64 bits while <code>float</code> is stored with 32 bits.</li> 
       <li><code>string</code> properties map to <code>String</code></li> 
       <li><code>data</code> properties map to <code>ArrayBuffer</code></li> 
       <li><code>date</code> properties map to <code>Date</code></li> 
      </ul> 
      <p>When specifying basic properties as a shorthand you may specify only the type rather than having to specify a dictionary with a single entry:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">CarSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="c1">// The following property types are equivalent</span>
    <span class="na">make</span><span class="p">:</span>   <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>
    <span class="na">model</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <h3 id="relationships">Relationships</h3> 
      <h4 id="to-one-relationships">To-One Relationships</h4> 
      <p>For to-one relationships you specify the <code>name</code> property of the object schema you are referencing as the property’s type:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="c1">// All of the following property definitions are equivalent</span>
    <span class="na">car</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">},</span>
    <span class="na">van</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">,</span>
  <span class="p">}</span>
<span class="p">};</span></code></pre> 
      <p>When using object properties you need to make sure all referenced types are present in the schema used to open the Realm:</p> 
      <pre class="highlight javascript"><code><span class="c1">// CarSchema is needed since PersonSchema contains properties of type 'Car'</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">CarSchema</span><span class="p">,</span> <span class="nx">PersonSchema</span><span class="p">]})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="cm">/* ... */</span><span class="p">);</span></code></pre> 
      <p>When accessing object properties, you can access nested properties using normal property syntax:</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="kr">const</span> <span class="nx">nameString</span> <span class="o">=</span> <span class="nx">person</span><span class="p">.</span><span class="nx">car</span><span class="p">.</span><span class="nx">name</span><span class="p">;</span>
  <span class="nx">person</span><span class="p">.</span><span class="nx">car</span><span class="p">.</span><span class="nx">miles</span> <span class="o">=</span> <span class="mi">1100</span><span class="p">;</span>

  <span class="c1">// create a new Car by setting the property to valid JSON</span>
  <span class="nx">person</span><span class="p">.</span><span class="nx">van</span> <span class="o">=</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Ford'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Transit'</span><span class="p">};</span>

  <span class="c1">// set both properties to the same car instance</span>
  <span class="nx">person</span><span class="p">.</span><span class="nx">car</span> <span class="o">=</span> <span class="nx">person</span><span class="p">.</span><span class="nx">van</span><span class="p">;</span>
<span class="p">});</span></code></pre> 
      <h4 id="to-many-relationships">To-Many Relationships</h4> 
      <p>For to-many relationships you must specify the property type as <code>list</code> as well as the <code>objectType</code>:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">cars</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'list'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">},</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <p>When accessing list properties a <code>List</code> object is returned. <code>List</code> has methods very similar to a regular JavaScript array. The big difference is that any changes made to a <code>List</code> are automatically persisted to the underlying Realm. Additionally, <code>List</code>s belong to the underlying object they were acquired from - you can only get <code>List</code> instances by accessing a property from an owning object and they cannot be manually created.</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">carList</span> <span class="o">=</span> <span class="nx">person</span><span class="p">.</span><span class="nx">cars</span><span class="p">;</span>

<span class="c1">// Add new cars to the list</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nx">carList</span><span class="p">.</span><span class="nx">push</span><span class="p">({</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Accord'</span><span class="p">,</span> <span class="na">miles</span><span class="p">:</span> <span class="mi">100</span><span class="p">});</span>
  <span class="nx">carList</span><span class="p">.</span><span class="nx">push</span><span class="p">({</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Toyota'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Prius'</span><span class="p">,</span> <span class="na">miles</span><span class="p">:</span> <span class="mi">200</span><span class="p">});</span>
<span class="p">});</span>

<span class="kd">let</span> <span class="nx">secondCar</span> <span class="o">=</span> <span class="nx">carList</span><span class="p">[</span><span class="mi">1</span><span class="p">].</span><span class="nx">model</span><span class="p">;</span>  <span class="c1">// access using an array index</span></code></pre> 
      <h4 id="inverse-relationships">Inverse Relationships</h4> 
      <p>Links are unidirectional. So if a to-many property <code>Person.dogs</code> links to <code>Dog</code> instances and a to-one property <code>Dog.owner</code> links to <code>Person</code>, these links are independent from one another. Appending a <code>Dog</code> to a <code>Person</code> instance’s <code>dogs</code> property doesn’t automatically set the dog’s <code>owner</code> property to this <code>Person</code>. Because manually synchronizing pairs of relationships is error prone, complex and duplicates information, Realm provides linking objects properties to represent these inverse relationships.</p> 
      <p>With linking objects properties, you can obtain all objects that link to a given object from a specific property. For example, a <code>Dog</code> object can have a property named <code>owners</code> that contains all of the <code>Person</code> objects that have this exact <code>Dog</code> object in their <code>dogs</code> property. This is done by making the <code>owners</code> property of type <code>linkingObjects</code> and then specifying the relationship that it has with the <code>Person</code> object.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">dogs</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'list'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Dog'</span><span class="p">},</span>
  <span class="p">}</span>
<span class="p">}</span>

<span class="kr">const</span> <span class="nx">DogSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span><span class="s1">'Dog'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">owners</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'linkingObjects'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span> <span class="na">property</span><span class="p">:</span> <span class="s1">'dogs'</span><span class="p">}</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <p>A <code>linkingObjects</code> property can point either to a <code>List</code> property (to-many relationship) or an <code>Object</code> property (to-one relationship):</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">ShipSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Ship'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">captain</span><span class="p">:</span> <span class="s1">'Captain'</span>
  <span class="p">}</span>
<span class="p">}</span>

<span class="kr">const</span> <span class="nx">CaptainSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Captain'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">ships</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'linkingObjects'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Ship'</span><span class="p">,</span> <span class="na">property</span><span class="p">:</span> <span class="s1">'captain'</span><span class="p">}</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <p>When accessing <code>linkingObjects</code> properties, a <code>Results</code> object is returned, so further <a href="#queries">querying and sorting</a> are fully supported. <code>linkingObject</code> properties belong to the object they were acquired from and can not be set or manipulated directly. They are updated automatically when a transaction is committed.</p> 
      <p><em>Accessing <code>linkingObjects</code> without a schema:</em> if you have opened a Realm file without specifying a schema, e.g. in a <a href="/docs/realm-object-server/#realm-functions">Realm Functions</a> callback, you can get a <code>linkingObjects</code> property by calling <code>linkingObjects(objectType, property)</code> on an <code>Object</code> instance:</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">captain</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objectForPrimaryKey</span><span class="p">(</span><span class="s1">'Captain'</span><span class="p">,</span> <span class="mi">1</span><span class="p">);</span>
<span class="kd">let</span> <span class="nx">ships</span> <span class="o">=</span> <span class="nx">captain</span><span class="p">.</span><span class="nx">linkingObjects</span><span class="p">(</span><span class="s1">'Ship'</span><span class="p">,</span> <span class="s1">'captain'</span><span class="p">);</span></code></pre> 
      <h3 id="optional-properties">Optional Properties</h3> 
      <p>Properties can be declared as optional or non-optional by specifying the <code>optional</code> designator in your property definition:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span>     <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>               <span class="c1">// required property</span>
    <span class="na">birthday</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'date'</span><span class="p">,</span> <span class="na">optional</span><span class="p">:</span> <span class="kc">true</span><span class="p">},</span> <span class="c1">// optional property</span>

    <span class="c1">// object properties are always optional</span>
    <span class="na">car</span><span class="p">:</span>      <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">},</span>
  <span class="p">}</span>
<span class="p">};</span>

<span class="kd">let</span> <span class="nx">realm</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Realm</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">PersonSchema</span><span class="p">,</span> <span class="nx">CarSchema</span><span class="p">]});</span>

<span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="c1">// optional properties can be set to null or undefined at creation</span>
  <span class="kd">let</span> <span class="nx">charlie</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">,</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span> <span class="s1">'Charlie'</span><span class="p">,</span>
    <span class="na">birthday</span><span class="p">:</span> <span class="k">new</span> <span class="nb">Date</span><span class="p">(</span><span class="mi">1995</span><span class="p">,</span> <span class="mi">11</span><span class="p">,</span> <span class="mi">25</span><span class="p">),</span>
    <span class="na">car</span><span class="p">:</span> <span class="kc">null</span><span class="p">,</span>
  <span class="p">});</span>

  <span class="c1">// optional properties can be set to `null`, `undefined`,</span>
  <span class="c1">// or to a new non-null value</span>
  <span class="nx">charlie</span><span class="p">.</span><span class="nx">birthday</span> <span class="o">=</span> <span class="kc">undefined</span><span class="p">;</span>
  <span class="nx">charlie</span><span class="p">.</span><span class="nx">car</span> <span class="o">=</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Accord'</span><span class="p">,</span> <span class="na">miles</span><span class="p">:</span> <span class="mi">10000</span><span class="p">};</span>
<span class="p">});</span></code></pre> 
      <p>As seen above object properties are always optional and do not need an optional designation. List properties cannot be declared as optional or set to null. You can set or initialize a list with an empty array to clear it.</p> 
      <h3 id="default-property-values">Default Property Values</h3> 
      <p>Default property values can be specified by setting the <code>default</code> designator in the property definition. To use a default value, leave the property unspecified during object creation.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">CarSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Car'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">make</span><span class="p">:</span>  <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>
    <span class="na">model</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>
    <span class="na">drive</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span> <span class="na">default</span><span class="p">:</span> <span class="s1">'fwd'</span><span class="p">},</span>
    <span class="na">miles</span><span class="p">:</span> <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'int'</span><span class="p">,</span>    <span class="na">default</span><span class="p">:</span> <span class="mi">0</span><span class="p">}</span>
  <span class="p">}</span>
<span class="p">};</span>

<span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="c1">// Since `miles` is left out it defaults to `0`, and since</span>
  <span class="c1">// `drive` is specified, it overrides the default value</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">,</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Accord'</span><span class="p">,</span> <span class="na">drive</span><span class="p">:</span> <span class="s1">'awd'</span><span class="p">});</span>
<span class="p">});</span></code></pre> 
      <h3 id="indexed-properties">Indexed Properties</h3> 
      <p>You can add an <code>indexed</code> designator to a property definition to cause that property to be indexed. This is supported for <code>int</code>, <code>string</code>, and <code>bool</code> property types:</p> 
      <pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">BookSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Book'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span> <span class="p">{</span> <span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span> <span class="na">indexed</span><span class="p">:</span> <span class="kc">true</span> <span class="p">},</span>
    <span class="na">price</span><span class="p">:</span> <span class="s1">'float'</span>
  <span class="p">}</span>
<span class="p">};</span></code></pre> 
      <p>Indexing a property will greatly speed up queries where the property is compared for equality at the cost of slower insertions.</p> 
      <h3 id="primary-keys">Primary Keys</h3> 
      <p>You can specify the <code>primaryKey</code> property in an object model for <code>string</code> and <code>int</code> properties. Declaring a primary key allows objects to be looked up and updated efficiently and enforces uniqueness for each value. Once an object with a primary key has been added to a Realm the primary key cannot be changed.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">BookSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Book'</span><span class="p">,</span>
  <span class="na">primaryKey</span><span class="p">:</span> <span class="s1">'id'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">id</span><span class="p">:</span>    <span class="s1">'int'</span><span class="p">,</span>    <span class="c1">// primary key</span>
    <span class="na">title</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">price</span><span class="p">:</span> <span class="s1">'float'</span>
  <span class="p">}</span>
<span class="p">};</span></code></pre> 
      <p>Primary key properties are automatically indexed.</p> 
      <h2 id="writes">Writes</h2> 
      <p>Changes to objects in a Realm—creating, updating and deleting—must take place within a <code>write()</code> transaction block. Note that write transactions have a non-negligible overhead; you should try to minimize the number of <code>write</code> blocks within your code.</p> 
      <h3 id="creating-objects">Creating Objects</h3> 
      <p>Objects are created using the <code>create</code> method:</p> 
      <pre class="highlight javascript"><code><span class="k">try</span> <span class="p">{</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">,</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Accord'</span><span class="p">,</span> <span class="na">drive</span><span class="p">:</span> <span class="s1">'awd'</span><span class="p">});</span>
  <span class="p">});</span>
<span class="p">}</span> <span class="k">catch</span> <span class="p">(</span><span class="nx">e</span><span class="p">)</span> <span class="p">{</span>
  <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="s2">"Error on creation"</span><span class="p">);</span>
<span class="p">}</span></code></pre> 
      <p>Note that any exceptions thrown in <code>write()</code> will cancel the transaction. The <code>try/catch</code> block won’t be shown in all examples, but it’s good practice.</p> 
      <h3 id="nested-objects">Nested Objects</h3> 
      <p>If an object has object properties, values for those properties can be created recursively by specifying JSON values for each child property:</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">,</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span> <span class="s1">'Joe'</span><span class="p">,</span>
    <span class="c1">// nested objects are created recursively</span>
    <span class="na">car</span><span class="p">:</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'Accord'</span><span class="p">,</span> <span class="na">drive</span><span class="p">:</span> <span class="s1">'awd'</span><span class="p">},</span>
  <span class="p">});</span>
<span class="p">});</span></code></pre> 
      <h3 id="updating-objects">Updating Objects</h3> 
      <h4 id="typed-updates">Typed Updates</h4> 
      <p>You can update any object by setting its properties within a write transaction.</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nx">car</span><span class="p">.</span><span class="nx">miles</span> <span class="o">=</span> <span class="mi">1100</span><span class="p">;</span>
<span class="p">});</span></code></pre> 
      <h4 id="creating-and-updating-objects-with-primary-keys">Creating and Updating Objects With Primary Keys</h4> 
      <p>If your model class includes a <a href="#primary-keys">primary key</a>, you can have Realm intelligently update or add objects based off of their primary key values. This is done by passing <code>true</code> as the third argument to the <code>create</code> method:</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="c1">// Create a book object</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Book'</span><span class="p">,</span> <span class="p">{</span><span class="na">id</span><span class="p">:</span> <span class="mi">1</span><span class="p">,</span> <span class="na">title</span><span class="p">:</span> <span class="s1">'Recipes'</span><span class="p">,</span> <span class="na">price</span><span class="p">:</span> <span class="mi">35</span><span class="p">});</span>

  <span class="c1">// Update book with new price keyed off the id</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Book'</span><span class="p">,</span> <span class="p">{</span><span class="na">id</span><span class="p">:</span> <span class="mi">1</span><span class="p">,</span> <span class="na">price</span><span class="p">:</span> <span class="mi">55</span><span class="p">},</span> <span class="kc">true</span><span class="p">);</span>
<span class="p">});</span></code></pre> 
      <p>In the example above, since an object already exists with the <code>id</code> value of <code>1</code> and we have passed in <code>true</code> for the third argument, the price property is updated rather than trying to create a new object. Since the <code>name</code> property is omitted the object retains the original value for this property. Note that when creating or updating objects with primary key properties the primary key must be specified.</p> 
      <h3 id="deleting-objects">Deleting Objects</h3> 
      <p>Objects can be deleted by calling the <code>delete</code> method within a write transaction.</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="c1">// Create a book object</span>
  <span class="kd">let</span> <span class="nx">book</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Book'</span><span class="p">,</span> <span class="p">{</span><span class="na">id</span><span class="p">:</span> <span class="mi">1</span><span class="p">,</span> <span class="na">title</span><span class="p">:</span> <span class="s1">'Recipes'</span><span class="p">,</span> <span class="na">price</span><span class="p">:</span> <span class="mi">35</span><span class="p">});</span>

  <span class="c1">// Delete the book</span>
  <span class="nx">realm</span><span class="p">.</span><span class="k">delete</span><span class="p">(</span><span class="nx">book</span><span class="p">);</span>

  <span class="c1">// Delete multiple books by passing in a `Results`, `List`,</span>
  <span class="c1">// or JavaScript `Array`</span>
  <span class="kd">let</span> <span class="nx">allBooks</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Book'</span><span class="p">);</span>
  <span class="nx">realm</span><span class="p">.</span><span class="k">delete</span><span class="p">(</span><span class="nx">allBooks</span><span class="p">);</span> <span class="c1">// Deletes all books</span>
<span class="p">});</span></code></pre> 
      <h2 id="queries">Queries</h2> 
      <p>Queries allow you to get objects of a single type from a Realm, with the option of filtering and sorting those results. All queries (including queries and property access) are lazy in Realm. Data is only read when objects and properties are accessed. This allows you to represent large sets of data in a performant way.</p> 
      <p>When performing queries you are returned a <code>Results</code> object. Results are simply a view of your data and are not mutable.</p> 
      <p>The most basic method for retrieving objects from a Realm is using the <code>objects</code> method on a <code>Realm</code> to get all objects of a given type:</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">dogs</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Dog'</span><span class="p">);</span> <span class="c1">// retrieves all Dogs from the Realm</span></code></pre> 
      <h3 id="filtering">Filtering</h3> 
      <p>You can get a filtered <code>Results</code> by calling the <code>filtered</code> method with a query string.</p> 
      <p>For example, the following would change our earlier example to retrieve all dogs with the color tan and names beginning with ‘B’:</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">dogs</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Dog'</span><span class="p">);</span>
<span class="kd">let</span> <span class="nx">tanDogs</span> <span class="o">=</span> <span class="nx">dogs</span><span class="p">.</span><span class="nx">filtered</span><span class="p">(</span><span class="s1">'color = "tan" AND name BEGINSWITH "B"'</span><span class="p">);</span></code></pre> 
      <p>At the moment only a subset of the NSPredicate syntax is supported in the query language. Basic comparison operators <code>==</code>, <code>!=</code>, <code>&gt;</code>, <code>&gt;=</code>, <code>&lt;</code>, and <code>&lt;=</code> are supported for numeric properties. <code>==</code>, <code>BEGINSWITH</code>, <code>ENDSWITH</code>, and <code>CONTAINS</code> are supported for string properties. String comparisons can be made case insensitive by appending <code>[c]</code> to the operator: <code>==[c]</code>, <code>BEGINSWITH[c]</code> etc. Filtering by properties on linked or child objects can by done by specifying a keypath in the query eg <code>car.color == 'blue'</code>.</p> 
      <h3 id="sorting">Sorting</h3> 
      <p><code>Results</code> allows you to specify a sort criteria and order based on a single or multiple properties. For example, the following call sorts the returned cars from the example above numerically by miles:</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">hondas</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">).</span><span class="nx">filtered</span><span class="p">(</span><span class="s1">'make = "Honda"'</span><span class="p">);</span>

<span class="c1">// Sort Hondas by mileage</span>
<span class="kd">let</span> <span class="nx">sortedHondas</span> <span class="o">=</span> <span class="nx">hondas</span><span class="p">.</span><span class="nx">sorted</span><span class="p">(</span><span class="s1">'miles'</span><span class="p">);</span></code></pre> 
      <div class="alert alert-warning"> 
       <p>Note that the order of <code>Results</code> is only guaranteed to stay consistent when the query is sorted. For performance reasons, insertion order is not guaranteed to be preserved.</p> 
      </div> 
      <h3 id="auto-updating-results">Auto-Updating Results</h3> 
      <p><code>Results</code> instances are live, auto-updating views into the underlying data, which means results never have to be re-fetched. Modifying objects that affect the query will be reflected in the results immediately.</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">hondas</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">).</span><span class="nx">filtered</span><span class="p">(</span><span class="s1">'make = "Honda"'</span><span class="p">);</span>
<span class="c1">// hondas.length == 0</span>

<span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">,</span> <span class="p">{</span><span class="na">make</span><span class="p">:</span> <span class="s1">'Honda'</span><span class="p">,</span> <span class="na">model</span><span class="p">:</span> <span class="s1">'RSX'</span><span class="p">});</span>
<span class="p">});</span>
<span class="c1">// hondas.length == 1</span></code></pre> 
      <p>This applies to all <code>Results</code> instances, included those returned by the <code>objects</code>, <code>filtered</code>, and <code>sorted</code> methods.</p> 
      <p>This property of <code>Results</code> not only keeps Realm fast and efficient, it allows your code to be simpler and more reactive. For example, if your view relies on the results of a query, you can store the <code>Results</code> in a property and access it without having to make sure to refresh its data prior to each access.</p> 
      <p>You can subscribe to <a href="#notifications">notifications</a> to know when Realm data is updated, indicating when your app’s UI should be refreshed for example, without having to re-fetch your <code>Results</code>.</p> 
      <h3 id="limiting-results">Limiting Results</h3> 
      <p>Most other database technologies provide the ability to ‘paginate’ results from queries (such as the ‘LIMIT’ keyword in SQLite). This is often done out of necessity to avoid reading too much from disk, or pulling too many results into memory at once.</p> 
      <p>Since queries in Realm are lazy, performing this sort of paginating behavior isn’t necessary at all, as Realm will only load objects from the results of the query once they are explicitly accessed.</p> 
      <p>If for UI-related or other implementation reasons you require a specific subset of objects from a query, it’s as simple as taking the <code>Results</code> object, and reading out only the objects you need.</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">cars</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">);</span>

<span class="c1">// get first 5 Car objects</span>
<span class="kd">let</span> <span class="nx">firstCars</span> <span class="o">=</span> <span class="nx">cars</span><span class="p">.</span><span class="nx">slice</span><span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="mi">5</span><span class="p">);</span></code></pre> 
      <h2 id="realms">Realms</h2> 
      <h3 id="opening-realms">Opening Realms</h3> 
      <p>Opening a Realm is simply performed by calling the static <code>open</code> method on the <code>Realm</code> class. Pass a configuration object. We’ve seen this used already in examples with a configuration object that includes the <code>schema</code> key:</p> 
      <pre class="highlight javascript"><code><span class="c1">// Get the default Realm with support for our objects</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">Car</span><span class="p">,</span> <span class="nx">Person</span><span class="p">]})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// ...use the realm instance here</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="k">catch</span><span class="p">(</span><span class="nx">error</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// Handle the error here if something went wrong</span>
  <span class="p">});</span></code></pre> 
      <p>For full details about the configuration object, see the API Reference for <a href="https://realm.io/docs/javascript/latest/api/Realm.html#~Configuration">configuration</a>. Some of the more common keys for the object, beyond <code>schema</code>, include:</p> 
      <ul> 
       <li><code>path</code>: specify a path to <a href="#other-realms">another Realm</a></li> 
       <li><code>migration</code>: a <a href="#migrations">migration function</a></li> 
       <li><code>sync</code>: a <a href="#sync">sync object</a>, to open a Realm synchronized with the Realm Object Server</li> 
       <li><code>inMemory</code>: the Realm will be opened in-memory, and objects are not persisted; once the last Realm instance is closed, all objects vanish</li> 
      </ul> 
      <h3 id="the-default-realm">The Default Realm</h3> 
      <p>You may have noticed in all previous examples that the path argument has been omitted. In this case the default Realm path is used. You can access and change the default Realm path using the <code>Realm.defaultPath</code> global property.</p> 
      <h3 id="other-realms">Other Realms</h3> 
      <p>It’s sometimes useful to have multiple Realms persisted at different locations. For example, you may want to bundle some data with your application in a Realm file, in addition to your main Realm. You can do this by specifying the <code>path</code> argument when initializing your realm. All paths are relative to the writable documents directory for your application:</p> 
      <pre class="highlight javascript"><code><span class="c1">// Open a realm at another path</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span>
  <span class="na">path</span><span class="p">:</span> <span class="s1">'anotherRealm.realm'</span><span class="p">,</span>
  <span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">CarSchema</span><span class="p">]</span>
<span class="p">}).</span><span class="nx">then</span><span class="p">(</span><span class="cm">/* ... */</span><span class="p">);</span></code></pre> 
      <h3 id="schema-version">Schema Version</h3> 
      <p>Another option available when opening a Realm is the <code>schemaVersion</code> property. When omitted, the <code>schemaVersion</code> property defaults to <code>0</code>. You are required to specify the <code>schemaVersion</code> when initializing an existing Realm with a schema that contains objects that differ from their previous specification. If the schema was updated and the <code>schemaVersion</code> was not, an exception will be thrown.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span> <span class="s1">'string'</span>
  <span class="p">}</span>
<span class="p">};</span>

<span class="c1">// schemaVersion defaults to 0</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">PersonSchema</span><span class="p">]});</span></code></pre> 
      <p>If you then later do something like this: ```js const UpdatedPersonSchema = { // The schema name is the same, so previous <code>Person</code> object // in the Realm will be updated name: ‘Person’, properties: { name: ‘string’, dog: ‘Dog’ // new property } };</p> 
      <p>// this will throw because the schema has changed // and <code>schemaVersion</code> is not specified Realm.open({schema: [UpdatedPersonSchema]});</p> 
      <p>// this will succeed and update the Realm to the new schema Realm.open({schema: [UpdatedPersonSchema], schemaVersion: 1}); ```</p> 
      <p>If you wish to retrieve the current schema version of a Realm, you may do so with the <code>Realm.schemaVersion</code> method.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">currentVersion</span> <span class="o">=</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">schemaVersion</span><span class="p">(</span><span class="nx">Realm</span><span class="p">.</span><span class="nx">defaultPath</span><span class="p">);</span></code></pre> 
      <h3 id="synchronously-opening-realms">Synchronously Opening Realms</h3> 
      <p>You can create a realm instance by simply invoking the constructor and passing a configuration object to it. This is typically not recommended as it blocks and could potentially be a time consuming operation, especially if there are <a href="#migrations">migrations</a> to run or if the realm is <a href="#sync">synchronized</a> and you don’t want to risk modifying data before it’s been completely downloaded.</p> 
      <p>If you still want to do this, the pattern is simple:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">realm</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Realm</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">PersonSchema</span><span class="p">]});</span>

<span class="c1">// You can now access the realm instance.</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(</span><span class="cm">/* ... */</span><span class="p">);</span></code></pre> 
      <div class="alert alert-warning"> 
       <p>If a Realm has read-only <a href="#access-control" title="Access Control">permissions</a>, then you <em>must</em> use the asynchronous api’s to open it. Opening a read-only Realm with the above pattern will cause an error.</p> 
      </div> 
      <h2 id="migrations">Migrations</h2> 
      <p>When working with a database your data model will most likely change over time. For example, suppose we have the following <code>Person</code> model:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">firstName</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">lastName</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">age</span><span class="p">:</span> <span class="s1">'int'</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <p>We want to update the data model to require a <code>name</code> property, rather than separate first and last names. To do this, we simply change the schema to the following:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">PersonSchema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">age</span><span class="p">:</span> <span class="s1">'int'</span>
  <span class="p">}</span>
<span class="p">}</span></code></pre> 
      <p>At this point if you had saved any data with the previous model version there will be a mismatch between the new code and the old data Realm has stored on disk. When this occurs, an exception will be thrown when you try to open the existing Realm with the new schema unless you run a migration.</p> 
      <h3 id="performing-a-migration">Performing a Migration</h3> 
      <p>You define a migration and the associated schema version by updating the <a href="#schema-version">schemaVersion</a> and defining an optional <code>migration</code> function. Your migration function provides any logic needed to convert data models from previous schemas to the new schema. When opening a <code>Realm</code> the migration function will be applied to update the <code>Realm</code> to the given schema version only if a migration is needed.</p> 
      <p>If no migration function is supplied then any new properties an automatically added and old properties are removed from the database when updating to the new <code>schemaVersion</code>. If you need to update old or populate new properties when upgrading your version you can do this in the migration function. For example, suppose we want to migrate the <code>Person</code> model declared earlier. You can populate the <code>name</code> property of the new schema using the old <code>firstName</code> and <code>lastName</code> properties:</p> 
      <pre class="highlight javascript"><code><span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span>
  <span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">PersonSchema</span><span class="p">],</span>
  <span class="na">schemaVersion</span><span class="p">:</span> <span class="mi">1</span><span class="p">,</span>
  <span class="na">migration</span><span class="p">:</span> <span class="p">(</span><span class="nx">oldRealm</span><span class="p">,</span> <span class="nx">newRealm</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// only apply this change if upgrading to schemaVersion 1</span>
    <span class="k">if</span> <span class="p">(</span><span class="nx">oldRealm</span><span class="p">.</span><span class="nx">schemaVersion</span> <span class="o">&lt;</span> <span class="mi">1</span><span class="p">)</span> <span class="p">{</span>
      <span class="kr">const</span> <span class="nx">oldObjects</span> <span class="o">=</span> <span class="nx">oldRealm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">);</span>
      <span class="kr">const</span> <span class="nx">newObjects</span> <span class="o">=</span> <span class="nx">newRealm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">);</span>

      <span class="c1">// loop through all objects and set the name property in the new schema</span>
      <span class="k">for</span> <span class="p">(</span><span class="kd">let</span> <span class="nx">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="nx">i</span> <span class="o">&lt;</span> <span class="nx">oldObjects</span><span class="p">.</span><span class="nx">length</span><span class="p">;</span> <span class="nx">i</span><span class="o">++</span><span class="p">)</span> <span class="p">{</span>
        <span class="nx">newObjects</span><span class="p">[</span><span class="nx">i</span><span class="p">].</span><span class="nx">name</span> <span class="o">=</span> <span class="nx">oldObjects</span><span class="p">[</span><span class="nx">i</span><span class="p">].</span><span class="nx">firstName</span> <span class="o">+</span> <span class="s1">' '</span> <span class="o">+</span> <span class="nx">oldObjects</span><span class="p">[</span><span class="nx">i</span><span class="p">].</span><span class="nx">lastName</span><span class="p">;</span>
      <span class="p">}</span>
    <span class="p">}</span>
  <span class="p">}</span>
<span class="p">}).</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="kr">const</span> <span class="nx">fullName</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Person'</span><span class="p">)[</span><span class="mi">0</span><span class="p">].</span><span class="nx">name</span><span class="p">;</span>
<span class="p">});</span></code></pre> 
      <p>Once the migration is successfully completed the Realm and all of its objects can be accessed as usual by your app.</p> 
      <h3 id="linear-migrations">Linear Migrations</h3> 
      <p>With the migration pattern described above you can potentially run into issues when migrating over multiple versions. This could happen if a user skips an app update and a property has been changed multiple times in the versions being skipped. In this case you may need to edit old migration code to correctly update data from old schema to the latest schema.</p> 
      <p>It’s possible to avoid this issue by running multiple migrations sequentially, making sure that the database is upgraded to each previous version and that the associated migration code is run. When following this pattern old migration code should never have to be modified, although you will need to keep all old schema and migration blocks for future use. An example of what this would look like:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">schemas</span> <span class="o">=</span> <span class="p">[</span>
  <span class="p">{</span> <span class="na">schema</span><span class="p">:</span> <span class="nx">schema1</span><span class="p">,</span> <span class="na">schemaVersion</span><span class="p">:</span> <span class="mi">1</span><span class="p">,</span> <span class="na">migration</span><span class="p">:</span> <span class="nx">migrationFunction1</span> <span class="p">},</span>
  <span class="p">{</span> <span class="na">schema</span><span class="p">:</span> <span class="nx">schema2</span><span class="p">,</span> <span class="na">schemaVersion</span><span class="p">:</span> <span class="mi">2</span><span class="p">,</span> <span class="na">migration</span><span class="p">:</span> <span class="nx">migrationFunction2</span> <span class="p">},</span>
  <span class="p">...</span>
<span class="p">]</span>

<span class="c1">// the first schema to update to is the current schema version</span>
<span class="c1">// since the first schema in our array is at</span>
<span class="kd">let</span> <span class="nx">nextSchemaIndex</span> <span class="o">=</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">schemaVersion</span><span class="p">(</span><span class="nx">Realm</span><span class="p">.</span><span class="nx">defaultPath</span><span class="p">);</span>
<span class="k">while</span> <span class="p">(</span><span class="nx">nextSchemaIndex</span> <span class="o">&lt;</span> <span class="nx">schemas</span><span class="p">.</span><span class="nx">length</span><span class="p">)</span> <span class="p">{</span>
  <span class="kr">const</span> <span class="nx">migratedRealm</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Realm</span><span class="p">(</span><span class="nx">schemas</span><span class="p">[</span><span class="nx">nextSchemaIndex</span><span class="o">++</span><span class="p">]);</span>
  <span class="nx">migratedRealm</span><span class="p">.</span><span class="nx">close</span><span class="p">();</span>
<span class="p">}</span>

<span class="c1">// open the Realm with the latest schema</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">(</span><span class="nx">schemas</span><span class="p">[</span><span class="nx">schemas</span><span class="p">.</span><span class="nx">length</span><span class="o">-</span><span class="mi">1</span><span class="p">]);</span></code></pre> 
      <h2 id="notifications">Notifications</h2> 
      <p>The <code>Realm</code>, <code>Results</code> and <code>List</code> objects provide <code>addListener</code> methods to register notification callbacks. Whenever the object is updated, the change notification callback will be called.</p> 
      <p>There are two kinds of notifications, “Realm Notifications” (simple callbacks notified when write transactions are committed) and “Collection Notifications” (more sophisticated callbacks which receive change metadata on insertions, deletions and updates).</p> 
      <p>In addition, the Professional Edition and Enterprise Edition provide event handling notifications. Read “The Realm Mobile Platform” for more information.</p> 
      <h3 id="realm-notifications">Realm Notifications</h3> 
      <p>Realm instances send out notifications to other instances every time a write transaction is committed. To register for notifications:</p> 
      <pre class="highlight javascript"><code><span class="kd">function</span> <span class="nx">updateUI</span><span class="p">()</span> <span class="p">{</span>
  <span class="c1">// ...</span>
<span class="p">}</span>

<span class="c1">// Observe Realm Notifications</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">addListener</span><span class="p">(</span><span class="s1">'change'</span><span class="p">,</span> <span class="nx">updateUI</span><span class="p">);</span>

<span class="c1">// ..later remove the listener</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">removeListener</span><span class="p">(</span><span class="s1">'change'</span><span class="p">,</span> <span class="nx">updateUI</span><span class="p">);</span>

<span class="c1">// ..or unregister all listeners</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">removeAllListeners</span><span class="p">();</span></code></pre> 
      <h3 id="collection-notifications">Collection Notifications</h3> 
      <p>Collection notifications contain information that describe what changes have occurred at a fine-grained level. This consists of the indices of objects that have been inserted, deleted, or modified since the last notification. Collection notifications are delivered asynchronously: first with the initial results, and then after any write transaction which modifies any of the objects in the collection, deletes objects from the collection, or adds new objects to the collection.</p> 
      <p>The notification callback function given to <code>addListener</code> receives two parameters when these changes occur. The first one is the collection that changed, and the second one is a <code>changes</code> object with information about the collection indices affected by deletions, insertions and modifications.</p> 
      <p>The former two, <strong>deletions</strong> and <strong>insertions</strong>, record the indices whenever objects start and stop being part of the collection. This takes into account when you add objects to the Realm or delete them from the Realm. For <code>Results</code> this also applies when you filter for specific values and the object was changed so that it is now matching the query or not matching anymore. For collections based on <code>List</code>, this applies when objects are added or removed from the relationship.</p> 
      <p>Your application is notified about <strong>modifications</strong> whenever a property of an object has changed, which was previously part of the collection and is still part of it. This happens as well when to-one and to-many relationships change, but doesn’t take changes on inverse relationships into account.</p> 
      <pre class="highlight javascript"><code><span class="kr">class</span> <span class="nx">Dog</span> <span class="p">{}</span>
<span class="nx">Dog</span><span class="p">.</span><span class="nx">schema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Dog'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span>  <span class="s1">'string'</span><span class="p">,</span>
    <span class="na">age</span><span class="p">:</span> <span class="s1">'int'</span><span class="p">,</span>
  <span class="p">}</span>
<span class="p">};</span>
<span class="kr">class</span> <span class="nx">Person</span> <span class="p">{}</span>
<span class="nx">Person</span><span class="p">.</span><span class="nx">schema</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">name</span><span class="p">:</span> <span class="s1">'Person'</span><span class="p">,</span>
  <span class="na">properties</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">name</span><span class="p">:</span>    <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">},</span>
    <span class="na">dogs</span><span class="p">:</span>    <span class="p">{</span><span class="na">type</span><span class="p">:</span> <span class="s1">'list'</span><span class="p">,</span> <span class="na">objectType</span><span class="p">:</span> <span class="s1">'Dog'</span><span class="p">},</span>
  <span class="p">}</span>
<span class="p">};</span></code></pre> 
      <p>Let’s assume you’re observing a list of dog owners as given by the model code above. You will be notified about modifications for a matched <code>Person</code> object when:</p> 
      <ul> 
       <li>You modify the <code>Person</code>’s <code>name</code> property.</li> 
       <li>You add or remove a <code>Dog</code> to the <code>Person</code>’s <code>dogs</code> property.</li> 
       <li>You modify the <code>age</code> property of a <code>Dog</code> belonging to that <code>Person</code>.</li> 
      </ul> 
      <p>This makes it possible to discretely control the animations and visual updates made to the content inside your UI, instead of arbitrarily reloading everything each time a notification occurs.</p> 
      <pre class="highlight javascript"><code><span class="c1">// Observe Collection Notifications</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Dog'</span><span class="p">).</span><span class="nx">filtered</span><span class="p">(</span><span class="s1">'age &lt; 2'</span><span class="p">).</span><span class="nx">addListener</span><span class="p">((</span><span class="nx">puppies</span><span class="p">,</span> <span class="nx">changes</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
  
  <span class="c1">// Update UI in response to inserted objects</span>
  <span class="nx">changes</span><span class="p">.</span><span class="nx">insertions</span><span class="p">.</span><span class="nx">forEach</span><span class="p">((</span><span class="nx">index</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="kd">let</span> <span class="nx">insertedDog</span> <span class="o">=</span> <span class="nx">puppies</span><span class="p">[</span><span class="nx">index</span><span class="p">];</span>
    <span class="p">...</span>
  <span class="p">});</span>
  
  <span class="c1">// Update UI in response to modified objects</span>
  <span class="nx">changes</span><span class="p">.</span><span class="nx">modifications</span><span class="p">.</span><span class="nx">forEach</span><span class="p">((</span><span class="nx">index</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="kd">let</span> <span class="nx">modifiedDog</span> <span class="o">=</span> <span class="nx">puppies</span><span class="p">[</span><span class="nx">index</span><span class="p">];</span>
    <span class="p">...</span>
  <span class="p">});</span>
  
  <span class="c1">// Update UI in response to deleted objects</span>
  <span class="nx">changes</span><span class="p">.</span><span class="nx">deletions</span><span class="p">.</span><span class="nx">forEach</span><span class="p">((</span><span class="nx">index</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// Deleted objects cannot be accessed directly</span>
    <span class="c1">// Support for accessing deleted objects coming soon...</span>
    <span class="p">...</span>
  <span class="p">});</span>
  
  
<span class="p">});</span>

<span class="c1">// Unregister all listeners</span>
<span class="nx">realm</span><span class="p">.</span><span class="nx">removeAllListeners</span><span class="p">();</span></code></pre> 
      <h2 id="sync">Sync</h2> 
      <div class="alert alert-warning"> 
       <p>Sync for Node.js on Linux is a Professional Edition feature and is not available without a license.</p> 
      </div> 
      <p>If you are working with Node.js on Linux, you need to specify your license access token to enable the features here. Do this like so:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">token</span> <span class="o">=</span> <span class="s2">"eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9..."</span><span class="p">;</span>

<span class="c1">// Unlock Professional Edition APIs</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">setAccessToken</span><span class="p">(</span><span class="nx">token</span><span class="p">);</span></code></pre> 
      <p>The Realm Mobile Platform (RMP) extends the Realm Mobile Database across the network, enabling automatic synchronization of data across devices. In order to do this a new set of types and classes are provided that support these synchronized Realms; these new classes are additive to the existing Realm Mobile Database.</p> 
      <h3 id="users">Users</h3> 
      <p>The central object in the Realm Object Server is the Realm User (<code>Realm.Sync.User</code>) associated with a synchronized Realm. A <code>User</code> can be authenticated to a shared Realm via a username/password scheme, or through a number of third-party authentication methods.</p> 
      <p>Creating and logging in a user requires two things:</p> 
      <ul> 
       <li>A URL of a Realm Object Server to connect to.</li> 
       <li>Credentials for an authentication mechanism that describes the user as appropriate for that mechanism (i.e., username/password, access key, etc).</li> 
      </ul> 
      <h4 id="authentication">Authentication</h4> 
      <p>Authentication is used to establish the identity of users and log them in. Refer to our <a href="/docs/realm-object-server/#authentication">authentication documentation</a> for a list of authentication providers supported by the Realm Mobile Platform.</p> 
      <p>The credential information for a given user can be created in one of several ways:</p> 
      <ul> 
       <li>Providing a valid username/password combination</li> 
       <li>Providing a token obtained from a supported third-party authentication service</li> 
       <li>Providing a token and a custom authentication provider (see <a href="/docs/realm-object-server/#custom-authentication">Custom Authentication</a>)</li> 
      </ul> 
      <p>The username and password authentication is entirely managed by the Realm Object Server, giving you full control over your application’s user management. For other authentication methods, your application is responsible for logging into the external service and obtaining the authentication token.</p> 
      <p>Here are some examples of setting credentials with various providers.</p> 
      <h5 id="usernamepassword">Username/Password</h5> 
      <pre class="highlight javascript"><code><span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">login</span><span class="p">(</span><span class="s1">'http://my.realm-auth-server.com:9080'</span><span class="p">,</span> <span class="s1">'username'</span><span class="p">,</span> <span class="s1">'<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="0e7e4e7d">[email&nbsp;protected]</a>$w0rd'</span><span class="p">,</span> <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">});</span></code></pre> 
      <p>Before a user can log in, the account must be created. You can either do that in advance on the server using the admin dashboard, or by calling <code>register</code>:</p> 
      <pre class="highlight javascript"><code><span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">register</span><span class="p">(</span><span class="s1">'http://my.realm-auth-server.com:9080'</span><span class="p">,</span> <span class="s1">'username'</span><span class="p">,</span> <span class="s1">'<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="01714172">[email&nbsp;protected]</a>$w0rd'</span><span class="p">,</span> <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">});</span></code></pre> 
      <h5 id="google">Google</h5> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">googleAccessToken</span> <span class="o">=</span> <span class="s1">'acc3ssT0ken...'</span><span class="p">;</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">registerWithProvider</span><span class="p">(</span><span class="s1">'http://my.realm-auth-server.com:9080'</span><span class="p">,</span> <span class="s1">'google'</span><span class="p">,</span> <span class="nx">googleAccessToken</span><span class="p">,</span> <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">});</span></code></pre> 
      <h5 id="facebook">Facebook</h5> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">fbAccessToken</span> <span class="o">=</span> <span class="s1">'acc3ssT0ken...'</span><span class="p">;</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">registerWithProvider</span><span class="p">(</span><span class="s1">'http://my.realm-auth-server.com:9080'</span><span class="p">,</span> <span class="s1">'facebook'</span><span class="p">,</span> <span class="nx">fbAccessToken</span><span class="p">,</span> <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">});</span></code></pre> 
      <h5 id="custom-auth">Custom Auth</h5> 
      <pre class="highlight javascript"><code><span class="c1">// The user token provided by your authentication server</span>
<span class="kr">const</span> <span class="nx">accessToken</span> <span class="o">=</span> <span class="s1">'acc3ssT0ken...'</span><span class="p">;</span>

<span class="kr">const</span> <span class="nx">user</span> <span class="o">=</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">registerWithProvider</span><span class="p">(</span>
  <span class="s1">'http://my.realm-auth-server.com:9080'</span><span class="p">,</span>
  <span class="s1">'custom/fooauth'</span><span class="p">,</span>
  <span class="nx">accessToken</span><span class="p">,</span>
  <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">}</span>
<span class="p">);</span></code></pre> 
      <p><em>Note:</em> the JavaScript SDK does not currently allow you to send additional data. If you need to send more than a single token, please encode the additional data as JSON and pass it through the accessToken parameter, and decode this string on the server side.</p> 
      <h5 id="logging-out">Logging Out</h5> 
      <p>Logging out of a synced Realm is simple:</p> 
      <pre class="highlight javascript"><code><span class="nx">user</span><span class="p">.</span><span class="nx">logout</span><span class="p">();</span></code></pre> 
      <p>When a user is logged out, the synchronization will stop. A logged out user can no longer open a synced Realm.</p> 
      <h4 id="working-with-users">Working with Users</h4> 
      <p>The sync server URL may contain the tilde character (“~”) which will be transparently expanded to represent the user’s unique identifier. This scheme easily allows you to write your app to cater to its individual users. The location on disk for shared Realms is managed by the framework, but can be overridden if desired.</p> 
      <pre class="highlight javascript"><code><span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">login</span><span class="p">(</span><span class="cm">/* ... */</span><span class="p">,</span> <span class="p">(</span><span class="nx">error</span><span class="p">,</span> <span class="nx">user</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="k">if</span> <span class="p">(</span><span class="o">!</span><span class="nx">error</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span>
      <span class="na">sync</span><span class="p">:</span> <span class="p">{</span>
        <span class="na">user</span><span class="p">:</span> <span class="nx">user</span><span class="p">,</span>
        <span class="na">url</span><span class="p">:</span> <span class="s1">'realm://object-server-url:9080/~/my-realm'</span><span class="p">,</span>
      <span class="p">},</span>
      <span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="cm">/* ... */</span><span class="p">]</span>
    <span class="p">}).</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
      <span class="cm">/* ... */</span>
    <span class="p">});</span>
  <span class="p">}</span>
<span class="p">});</span></code></pre> 
      <p><code>Realm.Sync.User.current</code> can be used to obtain the currently logged in user. If no users have logged in or all have logged out, it will return <code>undefined</code>. If there are more than one logged in users, an error will be thrown.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">user</span> <span class="o">=</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">current</span><span class="p">;</span></code></pre> 
      <p>If there are likely to be multiple users logged in, you can get a collection of them by calling <code>Realm.Sync.User.all</code>. This will be empty if no users have logged in.</p> 
      <pre class="highlight javascript"><code><span class="kd">let</span> <span class="nx">users</span> <span class="o">=</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">Sync</span><span class="p">.</span><span class="nx">User</span><span class="p">.</span><span class="nx">all</span><span class="p">;</span>

<span class="k">for</span><span class="p">(</span><span class="kr">const</span> <span class="nx">key</span> <span class="k">in</span> <span class="nx">users</span><span class="p">)</span> <span class="p">{</span>
  <span class="kr">const</span> <span class="nx">user</span> <span class="o">=</span> <span class="nx">users</span><span class="p">[</span><span class="nx">key</span><span class="p">];</span>

  <span class="c1">// do something with the user. </span>
<span class="p">})</span></code></pre> 
      <h3 id="opening-a-synchronized-realm">Opening a Synchronized Realm</h3> 
      <p>You open a synchroized Realm the same say as you open any other Realm. The <a href="https://realm.io/docs/javascript/latest/api/Realm.html#~Configuration">configuration</a> can be extended with a <code>sync</code> property if you need to configure the synchronization. The optional properties of <code>sync</code> include:</p> 
      <ul> 
       <li><code>error</code> - a callback for error handling/reporting</li> 
       <li><code>validate_ssl</code> - indicating if SSL certificates must be validated</li> 
       <li><code>ssl_trust_certificate_path</code> - a path where to find trusted SSL certificates</li> 
      </ul> 
      <p>The error handling is set up by registering a callback (<code>error</code>) as part of the configuration:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">config</span> <span class="o">=</span> <span class="p">{</span>
  <span class="na">sync</span><span class="p">:</span> <span class="p">{</span> <span class="na">user</span><span class="p">:</span> <span class="nx">userA</span><span class="p">,</span>
          <span class="na">url</span><span class="p">:</span> <span class="nx">realmUrl</span><span class="p">,</span> 
          <span class="na">error</span><span class="p">:</span> <span class="nx">err</span> <span class="o">=&gt;</span> <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">err</span><span class="p">)</span> 
        <span class="p">},</span>
  <span class="na">schema</span><span class="p">:</span> <span class="p">[{</span> <span class="na">name</span><span class="p">:</span> <span class="s1">'Dog'</span><span class="p">,</span> <span class="na">properties</span><span class="p">:</span> <span class="p">{</span> <span class="na">name</span><span class="p">:</span> <span class="s1">'string'</span> <span class="p">}</span> <span class="p">}]</span>
<span class="p">};</span>

<span class="kd">var</span> <span class="nx">realm</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Realm</span><span class="p">(</span><span class="nx">config</span><span class="p">);</span></code></pre> 
      <h3 id="working-with-a-synchronized-realm">Working with a Synchronized Realm</h3> 
      <p>Once you have opened a Realm using a URL to a Realm Object Server and a <code>User</code> object, you can interact with it as you would any other Realm in JavaScript.</p> 
      <pre class="highlight javascript"><code><span class="nx">realm</span><span class="p">.</span><span class="nx">write</span><span class="p">(()</span> <span class="o">=&gt;</span> <span class="p">{</span>
  <span class="nx">realm</span><span class="p">.</span><span class="nx">create</span><span class="p">(</span><span class="s1">'MyObject'</span><span class="p">,</span> <span class="nx">jsonData</span><span class="p">);</span>
<span class="p">});</span>

<span class="kd">var</span> <span class="nx">objects</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'MyObject'</span><span class="p">);</span></code></pre> 
      <h3 id="sync-sessions">Sync Sessions</h3> 
      <p>A synced Realm’s connection to the Realm Object Server is represented by a <code>Session</code> object. Session objects can be retrieved by calling <code>realm.syncSession</code>.</p> 
      <p>The state of the underlying session can be retrieved using the <code>state</code> property. This can be used to check whether the session is active, not connected to the server, or in an error state.</p> 
      <h3 id="access-control">Access Control</h3> 
      <p>The Realm Mobile Platform provides flexible access control mechanisms to restrict which users are allowed to sync against which Realm files. This can be used, for example, to create collaborative apps where multiple users write to the same Realm. It can also be used to share data in a publisher/subscriber scenario where a single writing user shares data with many users with read permissions.</p> 
      <p>There are three permissions that control the access level of a given Realm for a User:</p> 
      <ul> 
       <li><code>mayRead</code> indicates that the user is allowed to read from the Realm.</li> 
       <li><code>mayWrite</code> indicates that the user is allowed to write to the Realm.</li> 
       <li><code>mayManage</code> indicates that the user is allowed to change the permissions for the Realm.</li> 
      </ul> 
      <p>Unless permissions are explicitly modified, only the owner (creator) of a Realm can access it. The only exception is admin users: They are always granted all permissions to all Realms on the server.</p> 
      <div class="alert alert-warning"> 
       <p><b>Write-only</b> permissions (i.e., <code>mayWrite</code> set without <code>mayRead</code>) are not currently supported.</p> 
      </div> 
      <p>Please refer to the general Realm Object Server documentation on <a href="/docs/realm-object-server/#access-control">Access Control</a> to learn more about the concept.</p> 
      <h4 id="retrieving-permissions">Retrieving Permissions</h4> 
      <p>To get a collection of all the Permissions a user has been granted, use the <code>User.getGrantedPermissions</code> method:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">permissions</span> <span class="o">=</span> <span class="nx">user</span><span class="p">.</span><span class="nx">getGrantedPermissions</span><span class="p">(</span><span class="s2">"currentUser"</span><span class="p">);</span>

<span class="c1">// Permissions is a regular query</span>
<span class="kr">const</span> <span class="nx">writePermissions</span> <span class="o">=</span> <span class="nx">permissions</span><span class="p">.</span><span class="nx">filtered</span><span class="p">(</span><span class="s2">"mayWrite = true"</span><span class="p">);</span>

<span class="c1">// Queries are live and emit notifications</span>
<span class="nx">writePermissions</span><span class="p">.</span><span class="nx">addListener</span><span class="p">((</span><span class="nx">collection</span><span class="p">,</span> <span class="nx">changes</span><span class="p">)</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// handle permission changes</span>
<span class="p">});</span></code></pre> 
      <p>To get permissions granted <strong>by</strong> a user, pass in <code>"otherUser"</code>.</p> 
      <h4 id="modifying-permissions">Modifying Permissions</h4> 
      <p>Modifying the access control settings for a Realm file is performed by either applying permissions directly or offering them.</p> 
      <h5 id="granting-permissions">Granting Permissions</h5> 
      <p>Permission changes can be applied (i.e. granted or revoked) via the <code>User.applyPermissions</code> method in order to directly increase or decrease other users’ access level to a Realm.</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">condition</span> <span class="o">=</span> <span class="p">{</span> <span class="na">userId</span><span class="p">:</span> <span class="s1">'some-user-id'</span> <span class="p">};</span>
<span class="kr">const</span> <span class="nx">realmUrl</span> <span class="o">=</span> <span class="s2">"realm://my-server.com/~/myRealm"</span><span class="p">;</span>
<span class="nx">user</span><span class="p">.</span><span class="nx">applyPermissions</span><span class="p">(</span><span class="nx">condition</span><span class="p">,</span> <span class="nx">realmUrl</span><span class="p">,</span> <span class="s1">'read'</span><span class="p">);</span></code></pre> 
      <p>The <code>condition</code> parameter must be an object containing either</p> 
      <ul> 
       <li><code>userId</code> - use this to apply permissions based on a user’s identity (the internal Id that Realm generates).</li> 
       <li><code>metadataKey</code> and <code>metadataValue</code> - use the key <code>'email'</code> and an email address as the value to specify a user via the Username/Password provider.</li> 
      </ul> 
      <p>The last argument controls the access level that the user will be granted. Higher access implies all lower tiers, e.g. <code>write</code> implies <code>read</code>, <code>admin</code> implies <code>read</code> and <code>write</code>. If <code>none</code> is passed, this will revoke the user’s permissions for this Realm.</p> 
      <h5 id="offerresponse">Offer/Response</h5> 
      <p>A user can offer permissions to their Realm by sharing the opaque token returned by <code>offerPermissionsAsync</code>:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">realmUrl</span> <span class="o">=</span> <span class="s1">'realm://my-server.com/~/myRealm'</span><span class="p">;</span>
<span class="kr">const</span> <span class="nx">oneDay</span> <span class="o">=</span> <span class="mi">1000</span> <span class="o">*</span> <span class="mi">60</span> <span class="o">*</span> <span class="mi">60</span> <span class="o">*</span> <span class="mi">24</span><span class="p">;</span>
<span class="kr">const</span> <span class="nx">expiration</span> <span class="o">=</span> <span class="k">new</span> <span class="nb">Date</span><span class="p">(</span><span class="nb">Date</span><span class="p">.</span><span class="nx">now</span><span class="p">()</span> <span class="o">+</span> <span class="mi">7</span> <span class="o">*</span> <span class="nx">oneDay</span><span class="p">);</span>
<span class="nx">userA</span><span class="p">.</span><span class="nx">offerPermissions</span><span class="p">(</span><span class="nx">realmUrl</span><span class="p">,</span> <span class="s1">'write'</span><span class="p">,</span> <span class="nx">expiration</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">token</span> <span class="o">=&gt;</span> <span class="p">{</span> <span class="cm">/* ... */</span> <span class="p">});</span></code></pre> 
      <p>The optional <code>expiresAt</code> argument controls when the offer expires - i.e. using the token after that date will no longer grant permissions to that Realm. Users who have already consumed the token to obtain permissions will not lose their access after that date. If you want to revoke permissions, use <code>applyPermissionsAsync</code>.</p> 
      <p>Once a user has received a token, e.g. by sharing it via messaging app, or scanning a QR code, they can consume it to obtain the permissions offered:</p> 
      <pre class="highlight javascript"><code><span class="kr">const</span> <span class="nx">token</span> <span class="o">=</span> <span class="s2">"..."</span><span class="p">;</span>
<span class="nx">userB</span><span class="p">.</span><span class="nx">acceptPermissionOffer</span><span class="p">(</span><span class="nx">token</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">realmUrl</span> <span class="o">=&gt;</span> <span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span> <span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="cm">/* ... */</span><span class="p">],</span> <span class="na">sync</span><span class="p">:</span> <span class="p">{</span> <span class="na">user</span><span class="p">:</span> <span class="nx">userB</span><span class="p">,</span> <span class="na">url</span><span class="p">:</span> <span class="nx">realmUrl</span> <span class="p">}}))</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// ..use the realm</span>
  <span class="p">});</span></code></pre> 
      <h2 id="encryption">Encryption</h2> 
      <div class="alert alert-info"> 
       <p>Please take note of the Export Compliance section of our LICENSE, as it places restrictions against the usage of Realm if you are located in countries with an export restriction or embargo from the United States.</p> 
      </div> 
      <p>Realm supports encrypting the database file on disk with AES-256+SHA2 by supplying a 64-byte encryption key when creating a Realm.</p> 
      <pre class="highlight javascript"><code><span class="kd">var</span> <span class="nx">key</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Int8Array</span><span class="p">(</span><span class="mi">64</span><span class="p">);</span>  <span class="c1">// pupulate with a secure key</span>
<span class="nx">Realm</span><span class="p">.</span><span class="nx">open</span><span class="p">({</span><span class="na">schema</span><span class="p">:</span> <span class="p">[</span><span class="nx">CarObject</span><span class="p">],</span> <span class="na">encryptionKey</span><span class="p">:</span> <span class="nx">key</span><span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="nx">realm</span> <span class="o">=&gt;</span> <span class="p">{</span>
    <span class="c1">// Use the Realm as normal</span>
    <span class="kd">var</span> <span class="nx">dogs</span> <span class="o">=</span> <span class="nx">realm</span><span class="p">.</span><span class="nx">objects</span><span class="p">(</span><span class="s1">'Car'</span><span class="p">);</span>
  <span class="p">});</span></code></pre> 
      <p>This makes it so that all of the data stored on disk is transparently encrypted and decrypted with AES-256 as needed, and verified with a SHA-2 HMAC. The same encryption key must be supplied every time you obtain a Realm instance.</p> 
      <p>There is a small performance hit (typically less than 10% slower) when using encrypted Realms.</p> 
      <p><span id="realm-sync"></span></p> 
      <h2 id="troubleshooting">Troubleshooting</h2> 
      <h3 id="missing-realm-constructor">Missing Realm Constructor</h3> 
      <p>If your app crashes, telling you that the Realm constructor was not found, there are a few things you can try:</p> 
      <p>First of all, run <code>react-native link realm</code></p> 
      <p>If that doesn’t help, and your problem is on Android, try:</p> 
      <p>Add the following in your <code>MainApplication.java</code> file: <code>java import io.realm.react.RealmReactPackage; </code></p> 
      <p>And add the <code>RealmReactPackage</code> to the list of packages:</p> 
      <pre class="highlight java"><code><span class="kd">protected</span> <span class="n">List</span> <span class="nf">getPackages</span><span class="o">()</span> <span class="o">{</span>
    <span class="k">return</span> <span class="n">Arrays</span><span class="o">.</span><span class="na">asList</span><span class="o">(</span>
        <span class="k">new</span> <span class="nf">MainReactPackage</span><span class="o">(),</span>
        <span class="k">new</span> <span class="nf">RealmReactPackage</span><span class="o">()</span> <span class="c1">// add this line</span>
    <span class="o">);</span>
<span class="o">}</span></code></pre> 
      <p>add following two lines in <code>settings.gradle</code>:</p> 
      <pre class="highlight plaintext"><code>include ':realm'
project(':realm').projectDir = new File(settingsDir, '../node_modules/realm/android')</code></pre> 
      <p>If your problem is on iOS, try: 1. Close all simulators/device builds 2. Stop the package manager running in terminal (or better yet, just restart terminal) 3. Open the ios folder in your app root in finder 4. Go into the build folder (note: you won’t see this build folder in atom, so just right click ios and click open in finder) 5. Delete everything inside of the build folder (just move to trash and keep trash around in case you’re worried) 6. Run <code>react-native run-ios</code> to rebuild the whole thing</p> 
      <h3 id="chrome-debugging-is-slow">Chrome Debugging is slow</h3> 
      <p>We are aware of this. The reason for this is that since Realm is written in C++ and runs native code, it has to run on the device/simulator. But given the zero-copy architecture, we need to send values in realm objects over the RPC wire every time you inspect an object that is stored in a Realm.</p> 
      <p>We are investigating various potential solutions for this problem. If you want to keep track of it, you can follow the <a href="https://github.com/realm/realm-js/issues/491">GitHub issue</a>.</p> 
      <h3 id="crash-reporting">Crash Reporting</h3> 
      <p>We encourage you to use a crash reporter in your application. Many Realm operations could potentially fail at runtime (like any other disk IO), so collecting crash reports from your application will help identify areas where either you (or us) can improve error handling and fix crashing bugs.</p> 
      <p>Most commercial crash reporters have the option of collecting logs. We strongly encourage you to enable this feature. Realm logs metadata information (but no user data) when throwing exceptions and in irrecoverable situations, and these messages can help debug when things go wrong.</p> 
      <h3 id="reporting-realm-issues">Reporting Realm Issues</h3> 
      <p>If you’ve found an issue with Realm, please either <a href="https://github.com/realm/realm-js/issues/new">file an issue on GitHub</a> or email us at <a href="mailto:help@realm.io">help@realm.io</a> with as much information as possible for us to understand and reproduce your issue.</p> 
      <p>The following information is very useful to us:</p> 
      <ol> 
       <li>Goals.</li> 
       <li>Expected results.</li> 
       <li>Actual results.</li> 
       <li>Steps to reproduce.</li> 
       <li>Code sample that highlights the issue <em>(full working projects that we can compile ourselves are ideal)</em>.</li> 
       <li>Version of Realm.</li> 
       <li>Crash logs &amp; stack traces. See <a href="#crash-reporting">Crash Reporting</a> above for details.</li> 
      </ol>
