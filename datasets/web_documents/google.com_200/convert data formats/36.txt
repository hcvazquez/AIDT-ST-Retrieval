       <p>In the above code:</p> 
       <ul> 
        <li> <p>We have created an instance of the <code>FusionCharts</code> object in the <code>revenueChart</code> variable. Each chart or gauge in your HTML page needs to have a separate variable. The initialization code is wrapped within the <code>FusionCharts.ready</code> method. This safeguards your chart instantiation code from being called before the FusionCharts Suite XT library is loaded and is ready to be used on the page.</p> </li> 
        <li> <p>Next, we created an instance of the <code>column2d</code> chart. Each chart type in FusionCharts Suite XT has a unique alias, using which you can create instance of that chart. In this case, we are creating an instance of a Column 2D chart with dimensions of 500x300 pixels, and providing JSON data as a string.</p> </li> 
        <li> <p>To specify the data format as JSON, we set the <code>dataFormat</code> parameter to <code>json</code>. If you provided data to the chart in XML format, as explained in a later section, the value of this attribute would be <code>xml</code>.</p> </li> 
        <li> <p>The actual JSON data is embedded as string as value of the <code>dataSource</code> parameter. The <code>chart</code> object contains a list of key-value pairs that let you configure the functional and cosmetic attributes of your chart. For example, the chart caption is set using <code>"caption"</code>: <code>"Harry's SuperMart"</code>. Each row of the tabular data is present within the <code>data</code> array, with name of month with key as <code>label</code>, and revenue with key as <code>value</code>.</p> </li> 
        <li> <p>If you noted, we have used the <code>theme</code> attribute in the charts JSON data and provided <code>fint</code> (FusionCharts Suite XT Internal theme) as the value for it. Themes let you centralize your cosmetic and functional properties across various charts in your web application. FusionCharts Suite XT is shipped with three default themes - <code>ocean</code>, <code>zune</code> and <code>carbon</code>. You can also create your own themes, as detailed in a later section.</p> </li> 
        <li> <p>Call the <code>render</code> method to draw the chart in the chartContainer defined using the <code>&lt;div&gt;</code> element.</p> </li> 
       </ul>

       <h2 id="was-there-a-problem-rendering-the-chart">Was there a problem rendering the chart?</h2> 
       <p>In case something went wrong and you are unable to see the chart, check for the following:</p> 
       <ul> 
        <li> <p>If you are getting a JavaScript error on your page, check your browser console for the exact error and fix accordingly.</p> </li> 
        <li> <p>If the chart does not show up at all, but there are no JavaScript errors, check if the FusionCharts Suite XT JavaScript library has loaded correctly. You can use developer tools within your browser to see if <code>fusioncharts.js</code> was loaded. Check if the path to <code>fusioncharts.js</code> file is correct, and whether the file exists in that location.</p> </li> 
        <li> <p>If you get a <strong>Loading Data</strong> or <strong>Error in loading data</strong> message, check whether your JSON data structure is correct, and there are no conflicts related to quotation marks in your code.</p> </li> 
       </ul>
