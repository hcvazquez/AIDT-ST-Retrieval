               <h2>You Asked </h2> 
               <div class="ate-Question-youAsked">
                 hi Tom, I am stuck with the following question and would appriciate your help, 
                <br> 
                <br>What 2 methods of user authentication are available in Oracle??? What are some advantages and disadvantages of each??? 
                <br> 
                <br>Thank u 
                <br> 
                <br>Tan 
                <br> 
                <br> 
                <br>

               <h2>and we said...</h2> 
               <div class="ate-Question-weSaid">
                 Well, there are alot more then two - the two most basic ones are: 
                <br> 
                <br>USERNAME/PASSWORD authentication. 
                <br> 
                <br>OS Authentication. 
                <br> 
                <br> 
                <br>Username/password -- you know that one well, it has the same strengths/weakness as your OS username/password does. 
                <br> 
                <br>OS Authentication lets me log in like this: 
                <br> 
                <br>$ sqlplus / 
                <br> 
                <br>SQL*Plus: Release 8.1.6.0.0 - Production on Sun Apr 1 17:34:47 2001 
                <br> 
                <br>(c) Copyright 1999 Oracle Corporation. All rights reserved. 
                <br> 
                <br> 
                <br>Connected to: 
                <br>Oracle8i Enterprise Edition Release 8.1.6.2.0 - Production 
                <br>With the Partitioning option 
                <br>JServer Release 8.1.6.2.0 - Production 
                <br> 
                <br>ops$tkyte@ORA8I.WORLD&gt; 
                <br> 
                <br>The database pickes up my identity from the OS. It works well on a single machine but not over a network. Over a network it can be very insecure as the machines just "trust" eachother. All I would need to do is put a machine in your network, create a user account in the OS named after some database account and I would be in. Never set remote_os_authent=true for this reason. 
                <br> 
                <br>On NT it is more secure then on Unix with domains and such. 
                <br> 
                <br>I use the OS identified accounts for all of my cron jobs -- that way I do not have a username/password stored in a script anywhere. Thats the strength of it. 
                <br> 
                <br> 
                <br> 
                <br> 
                <br> 
                <br>

              <div class="ate-Question-review">
                That is very helpful. Then, I have a slightly different situation. If I have a web page running Java Servlet or JSP connected to our Oracle database (8.1.7), and I want to let only our NT domain user be able to log in. How should I set it? Also, which one is better for this purpose, Java Servlet or JSP?

              <div class="ate-Question-followup"> 
               <div class="ate-Expert-photo"> 
                <img title="Tom Kyte" alt="Tom Kyte" src="f?p=100:DOWNLOAD::APPLICATION_PROCESS=GET_IMAGE:::GET_TYPE,GET_ID:ADMIN_IMG,40395599290472004466352335132759465319">

               <p><span style="font-size: 1.05em;font-weight:bold;font-style:italic">Followup</span>&nbsp;&nbsp; </p>
               <p>October 15, 2001 - 4:24 pm UTC <a name="followup-1623302047418"></a><br></p> Well, that isn't going to work. You have a client connected via the web to the app server. The app server logs into the database from its machine. The database client here is the app server, not the client running a web browser. You cannot log into the database over the web using an app server and have the credentials passed all the way through. 
               <br> 
               <br>the java code will have to log in "as itself", you cannot pass the NT user through a "man in the middle". 
               <br> 
               <br>In 9i, something that may be of interest to you is: 
               <br>&lt;/code&gt; 
               <pre><a href="http://www.oracle.com/pls/db901/db901.to_toc?pathname=java.901/a90211/oci_func.htm#1014235">http://www.oracle.com/pls/db901/db901.to_toc?pathname=java.901/a90211/oci_func.htm#1014235</a></pre> &lt;code&gt;
               <br> 
               <br>You can combine this with your desire to authenticate all of the way through. The client would identify itself to the middle tier, you can then connect on their behalf to the database. 
               <br> 
               <br>As servelet or JSP -- it is pretty much the same to me. JSPs are mixing html + java, a servelet is something that generates html (generally, could be xml or anything really) from scratch. Depends on what tools you are using and how comfortable you are with either technology.
