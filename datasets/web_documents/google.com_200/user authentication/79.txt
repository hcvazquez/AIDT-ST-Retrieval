               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="_introduction"></a>9.1.&nbsp;Introduction</h2> 
               </div> 
              </div> 
             </div> 
             <p>The command line tool <code class="literal">npm</code> is a package management solution for Javascript-based development. It is used to create and use <span class="emphasis"><em>node packaged modules</em></span> and is built into the popular Javascript platform <a class="ulink" href="http://www.nodejs.org/" target="_top">Node.js</a>, which is mostly used for server-side application development.</p> 
             <p>The npmjs website, available at <a class="ulink" href="https://www.npmjs.org" target="_top">https://www.npmjs.org</a>, provides search and other convenience features to access the public registry at <a class="ulink" href="https://registry.npmjs.org" target="_top">https://registry.npmjs.org/</a>. It is the default package registry, from which components can be retrieved. It contains a large number of open source packages for Node.js based server-side application development, build tools like <span class="emphasis"><em>bower</em></span> or <span class="emphasis"><em>grunt</em></span> and many other packages for a variety of use cases.</p> 
             <p>Nexus Repository Manager Pro and Nexus Repository Manager OSS support the npm registry format for proxy repositories. This allows you to take advantage of the packages in the npm registry and other public registries without incurring repeated downloads of packages, since they will be proxied in the repository manager.</p> 
             <p>In addition, Nexus Repository Manager supports running your own private registry - also known as a hosted repository using the <span class="emphasis"><em>npm</em></span> format. You can share internally developed, proprietary packages within your organization via these private registries allowing you to collaborate efficiently across development teams with a central package exchange and storage location.</p> 
             <p>To simplify configuration Nexus Repository Manager supports aggregation of npm registries. This allows you to expose all the external packages from the npm registry and other public registries as well as the private registries as one registry, which greatly simplifies client configuration.</p> 
             <p>To share a package or tool with npm, you create a npm package and store it in the npm registry hosted by the repository manager. Similarly, you can use packages others have created and made available in their NPM repositories by proxying them or downloading the packages and installing them in your own private registry for third party packages.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-proxying-registries"></a>9.2.&nbsp;Proxying npm Registries</h2> 
               </div> 
              </div> 
             </div> 
             <p>To reduce duplicate downloads and improve download speeds for your developers and CI servers, you should proxy the registry hosted at <a class="ulink" href="https://registry.npmjs.org" target="_top">https://registry.npmjs.org</a>. By default npm accesses this registry directly. You can also proxy any other registries you require.</p> 
             <p>To proxy an external npm registry, you simply create a new <span class="emphasis"><em>npm (proxy)</em></span> as documented in <a class="xref" href="admin.html#admin-repositories" title="4.3.&nbsp;Repository Management">Section&nbsp;4.3, “Repository Management”</a>.</p> 
             <p>Minimal configuration steps are:</p> 
             <div class="itemizedlist"> 
              <ul class="itemizedlist" style="list-style-type: disc; "> 
               <li class="listitem"> Define <span class="emphasis"><em>Name</em></span> </li> 
               <li class="listitem"> Define URL for <span class="emphasis"><em>Remote storage</em></span> e.g. <code class="literal">https://registry.npmjs.org</code> </li> 
               <li class="listitem"> Select <span class="emphasis"><em>Blob store</em></span> for <span class="emphasis"><em>Storage</em></span> </li> 
              </ul> 
             </div> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-private-registries"></a>9.3.&nbsp;Private npm Registries</h2> 
               </div> 
              </div> 
             </div> 
             <p>A private npm registry can be used to upload your own packages as well as third-party packages. You can create a private npm registry by setting up a hosted repository with the npm format in the repository manager. It is good practice to create two separate hosted repositories for these purposes.</p> 
             <p>To create a hosted repository with npm format, simply create a new <span class="emphasis"><em>npm (hosted)</em></span> as documented in <a class="xref" href="admin.html#admin-repositories" title="4.3.&nbsp;Repository Management">Section&nbsp;4.3, “Repository Management”</a>.</p> 
             <p>Minimal configuration steps are:</p> 
             <div class="itemizedlist"> 
              <ul class="itemizedlist" style="list-style-type: disc; "> 
               <li class="listitem"> Define <span class="emphasis"><em>Name</em></span> </li> 
               <li class="listitem"> Select <span class="emphasis"><em>Blob store</em></span> for <span class="emphasis"><em>Storage</em></span> </li> 
              </ul> 
             </div> 
             <p>The npm registry information is immediately updated as packages are deployed or deleted from the repository.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-grouping-registries"></a>9.4.&nbsp;Grouping npm Registries</h2> 
               </div> 
              </div> 
             </div> 
             <p>A repository group is the recommended way to expose all your npm registries repositories from the repository manager to your users, without needing any further client side configuration. A repository group allows you to expose the aggregated content of multiple proxy and hosted repositories with one URL to npm and other tools. This is possible for npm repositories by creating a new <span class="emphasis"><em>npm (group)</em></span> as documented in <a class="xref" href="admin.html#admin-repositories" title="4.3.&nbsp;Repository Management">Section&nbsp;4.3, “Repository Management”</a>.</p> 
             <p>Minimal configuration steps are:</p> 
             <div class="itemizedlist"> 
              <ul class="itemizedlist" style="list-style-type: disc; "> 
               <li class="listitem"> Define <span class="emphasis"><em>Name</em></span> </li> 
               <li class="listitem"> Select <span class="emphasis"><em>Blob store</em></span> for <span class="emphasis"><em>Storage</em></span> </li> 
               <li class="listitem"> Add npm repositories to the <span class="emphasis"><em>Members</em></span> list in the desired order </li> 
              </ul> 
             </div> 
             <p>A typical, useful example would be to group the proxy repository that: proxies the npm registry, a npm, hosted repository with internal software packages and another npm, hosted repository with third-party packages.</p> 
             <p>Using the <span class="emphasis"><em>URL</em></span> of the repository group as your npm repository URL in your client tool will give you access to the packages in all three repositories with one URL. Any new packages added as well as any new repositories added to the group will automatically be available.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="_browsing_npm_registries_and_searching_modules"></a>9.5.&nbsp;Browsing npm Registries and Searching Modules</h2> 
               </div> 
              </div> 
             </div> 
             <p>You can browse npm repositories in the user interface inspecting the components and assets and their details as documented in <a class="xref" href="using.html#browse-browse" title="3.4.&nbsp;Browsing Repositories and Repository Groups">Section&nbsp;3.4, “Browsing Repositories and Repository Groups”</a>.</p> 
             <p>Searching for npm modules can be performed in the user interface as described in <a class="xref" href="using.html#search-components" title="3.3.&nbsp;Searching for Components">Section&nbsp;3.3, “Searching for Components”</a>. This search will find all npm modules images that are currently stored in the repository manager, either because they have been pushed to a hosted repository or they have been proxied from an upstream repository and cached in the repository manager.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-configuring"></a>9.6.&nbsp;Configuring npm</h2> 
               </div> 
              </div> 
             </div> 
             <p>Once you have set up your hosted and proxy repositories for npm packages, and created a repository group to merge them, you can access them with the <code class="literal">npm</code> tool on the command line as one registry.</p> 
             <p>You can configure the registry used by <code class="literal">npm</code> in your <code class="literal">.npmrc</code> file located in your user’s home directory with the <code class="literal">npm config</code> command and the public URL of your repository group in the repository list in the <span class="emphasis"><em>Repository Path</em></span> column.</p> 
             <pre class="screen">npm config set registry http://localhost:8081/repository/npm-all/</pre> 
             <p>The command inserts the configuration in the <code class="literal">.npmrc</code> file in your users home directory.</p> 
             <p><strong>Registry configuration in <code class="literal">.npmrc</code>.&nbsp;</strong> </p> 
             <pre class="screen">registry = http://localhost:8081/repository/npm-all/</pre> 
             <p> </p> 
             <p>With this configuration any npm commands will use the new registry from the repository manager. The command line output will reference the URLs in <code class="literal">--verbose</code> mode or with <code class="literal">info</code> logging for the downloads of the requested packages:</p> 
             <pre class="screen">$ npm --loglevel info install grunt
...
npm http fetch GET http://localhost:8081/repository/npmjs-org/grunt/-/grunt-0.4.5.tgz
npm http fetch 200 http://localhost:8081/repository/npmjs-org/grunt/-/grunt-0.4.5.tgz
...
npm http fetch GET http://localhost:8081/repository/npm-all/underscore/-/underscore-1.7.0.tgz
npm http fetch 200 http://localhost:8081/repository/npm-all/underscore/-/underscore-1.7.0.tgz

...</pre> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="_npm_security"></a>9.7.&nbsp;npm Security</h2> 
               </div> 
              </div> 
             </div> 
             <p>By default any anonymous user has read access to the repositories and repository groups. If anonymous access, as documented in <a class="xref" href="security.html#anonymous" title="5.6.&nbsp;Anonymous Access">Section&nbsp;5.6, “Anonymous Access”</a> is disabled, or write access is required for publishing a package, the user needs to authenticate to the repository manager.</p> 
             <p>This authentication requires the <code class="literal">npm Bearer Token Realm</code>. Simply add the realm to the active realms in the <code class="literal">Realms</code> feature of the <code class="literal">Security</code> menu from the <code class="literal">Administration</code> menu to activate it as documented in <a class="xref" href="security.html#realms" title="5.2.&nbsp;Realms">Section&nbsp;5.2, “Realms”</a>.</p> 
             <p>Once the realm is activated, a user can establish the authentication to a repository with the <code class="literal">npm login</code> command.</p> 
             <pre class="screen">npm login --registry=http://localhost:8081/repository/npm-internal</pre> 
             <p>Provide your repository manager username and password as well as your email address when prompted. Upon successful completion, a line for authentication of this combination is automatically added to your <code class="literal">.npmrc</code> configuration file for the specific repository.</p> 
             <p>Further details on <code class="literal">npm login</code> can be found on the <a class="ulink" href="https://docs.npmjs.com/cli/adduser" target="_top">npm website</a>.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-deploying-packages"></a>9.8.&nbsp;Publishing npm Packages</h2> 
               </div> 
              </div> 
             </div> 
             <p>Publishing your own packages to a npm hosted repository allows you to share packages across your organization or with external partners. With authentication configured you can publish your packages with the <code class="literal">npm publish</code> command.</p> 
             <p>The <code class="literal">npm publish</code> command uses a <code class="literal">registry</code> configuration value to know where to publish your package. There are several ways to change the registry value to point at your hosted npm repository.</p> 
             <p>Since the <code class="literal">.npmrc</code> file usually contains a registry value intended only for getting new packages, a simple way to override this value is to provide a registry to the <code class="literal">publish</code> command:</p> 
             <pre class="screen">npm publish --registry http://localhost:8081/repository/npm-internal/</pre> 
             <p>Alternately, you can edit your package.json file and add a <code class="literal">publishConfig</code> section:</p> 
             <pre class="screen">  "publishConfig" : {
    "registry" : "http://localhost:8081/repository/npm-internal/"
  },</pre> 
             <p>Detailed information about package creation can be found on the <a class="ulink" href="https://docs.npmjs.com/cli/publish" target="_top">npm website</a>.</p> 
             <p>If your package requires the use of <code class="literal">npm scope</code>, the repository manager supports this functionality. Packages published to the repository manager with a defined scope are reflected with the scope value populating the repository group field in Browse and Search. Details on scoping are available on the <a class="ulink" href="https://docs.npmjs.com/misc/scope" target="_top">npm website</a> also.</p> 
             <p>Once a package is published to the private registry in the repository manager, any other developers or build servers that access the repository manager via the repository group have instant access to it.</p> 
            </div> 
            <div class="section"> 
             <div class="titlepage"> 
              <div> 
               <div> 
                <h2 xmlns="" class="title" style="clear: both"><a xmlns="http://www.w3.org/1999/xhtml" id="npm-deprecating-packages"></a>9.9.&nbsp;Deprecating npm Packages</h2> 
               </div> 
              </div> 
             </div> 
             <p>Once your packages have been pushed to an npm hosted repository, you can mark them as deprecated. This is useful when a newer version of the package is available, and you want to warn people that the old package has reached end of life or you want to avoid usage and warn your users for some other reason.</p> 
             <p>The <code class="literal">npm deprecate</code> command uses a <code class="literal">registry</code> configuration value to inform where the package lives. To deprecate an existing package, use a command like the following:</p> 
             <pre class="screen">npm deprecate --registry http://localhost:8081/repository/npm-internal/ testproject1@0.0.1 "This package is deprecated"</pre> 
             <p>If you change your mind, you can reverse this action using the same command. To undeprecate a package, pass an empty string to the deprecate command:</p> 
             <pre class="screen">npm deprecate --registry http://localhost:8081/repository/npm-internal/ testproject1@0.0.1 ""</pre> 
             <p>The message text is persisted in the <code class="literal">deprecated</code> attribute of the <span class="emphasis"><em>packageJson</em></span> section for the asset and can be viewed in the user interface.</p> 
            </div>
