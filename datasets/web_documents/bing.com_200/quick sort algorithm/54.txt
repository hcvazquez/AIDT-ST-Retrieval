       <p>The Collection Framework is perhaps the most used library among the entire core API group in <a href="http://www.developer.com/java">Java</a>. A Collection is basically a container to store objects as an element of that collection. The thing that sets it apart from other primitive repositories such as simple arrays is the safety and convenience of many of its utilities. A programmer would find the complimentary services of this framework quite useful. The Collection Framework also provides features to apply different algorithms on all or a few elements of a collection, such as searching through a collection, sorting or shuffling elements of a collection, fetch a read only view of a collection, and so forth. This article focuses on some of its common algorithmic application to collection elements in Java.</p> 
       <h2>Collection and Collections</h2> 
       <p>The algorithms that are typically applied to a collection are provided in a class named <em>Collections</em>. To allay confusion, remember that the names <strong><em>Collection</em></strong> and <strong><em>Collections</em></strong> are distinct; one is an interface and another is a pure class defined both within the <strong><em>java.util</em></strong> package.</p> 
       <p>A <em>collection</em> is an interface that is the parent of most of the collection interfaces defined in the collection framework. This interface implements <em>Iterable</em>, another interface primary used for traversing through the collection elements. There are many sub interfaces and sub classes that extend and implement the <em>Collection</em> interface respectively. These inherited sub classes and sub interfaces add a few for specialized mechanisms to serve a specific purpose. For example, <em>Collection</em> extended by the <em>Set</em> interface ensures that the collection elements do not contain any duplicates; similarly, <em>List</em> maintains an ordered collection of elements that can be referred to by their positional indexes.</p> 
       <div id="olal-container"> 
        <script language="javascript" src="http://www.developer.com/sl/assetlisting/?wsn=www.developer.com&amp;wpu=/&amp;lurl=http://o1.qnsr.com/cgi/r?WT.qs_dlk=Wb7HxwrIZ6wAAGbVpoQAAAAE;;n=203;c=1397070;s=9487;x=7936;f=201309171321110;u=j;z=TIMESTAMP;k=http://assetform.developer.com/controller&amp;wpos=onelineassetlisting&amp;lyt=l6&amp;css=http://www.developer.com/css/onePaidAsset.css&amp;wi=19737410">
</script>

       <div style="clear:both;"></div> 
       <p><em>Collections</em>, on the other hand, is a class with an exclusive collection of static methods and fields. This means that these methods can be directly referred to as and when required by the class name itself (without creating an object instance of the <em>Collections</em> class). These methods are polymorphic implementation of some common algorithms frequently required by the programmer, such as sorting, searching, swapping, shuffling, and so on. The methods are invoked to operate on the elements of <em>Collection</em>. However, they are defined in such a manner that they do not to change the actual data, but operate on the collection wrappers that return a new collection backed by a specified collection. This keeps your original collection safe.</p>

       <p>Let's check out some common usages.</p> 
       <h2>Sorting</h2> 
       <p>Suppose we want to sort a list of data. We can use one of two methods:</p> 
       <ul> 
        <li><tt>static &lt;T extends Comparable&lt;? super T&gt;&gt; void sort(List&lt;T&gt; list)</tt></li> 
        <li><tt>static &lt;T&gt; void sort(List&lt;T&gt; list, Comparator&lt;? super T&gt; c)&lt;T&gt; void sort</tt></li> 
       </ul> 
       <p>By default, sorting follows a natural order (ascending). The natural order is defined by the Comparable interface. Java relies on a (modified) Merge Sort algorithm because of its stability and performance (time required to sort a given list of elements) in even the worst case scenario. The performance can be measured in <em>O(n log n)</em>, where <em>n</em> is the number of elements in the list. What it does internally is copy all the elements of the list into an array, apply the sorting technique, and then copy back to the List. So, any exception that occurred during sorting does not really affect the actual data.</p> 
       <pre><strong>package</strong> org.algo.example;

<strong>import</strong> java.util.ArrayList;
<strong>import</strong> java.util.Collections;
<strong>import</strong> java.util.List;

<strong>public class</strong> Main {

   <strong>public static void</strong> main(String[] args) {

      List&lt;String&gt; list = <strong>new</strong> ArrayList&lt;&gt;();
      list.add("Cumin");
      list.add("Raisin");
      list.add("Chickpeas");
      list.add("Strawberry");
      list.add("Fenugreek");
      list.add("Nutmeg");
      list.add("Cloves");
      list.add("Parsley");
      list.add("Corriander");
      list.add("Fennel");
      System.<strong><em>out</em></strong>.<span style="background: silver;">println</span>("Before sorting: "+list);
      Collections.<em>sort</em>(list);
      System.<strong><em>out</em></strong>.<span style="background: silver;">println</span>("After sorting: "+list);
   }
}
</pre>

       <p>The <tt>List&lt;E&gt;</tt> interface, however, has its own sort method called:</p> 
       <pre>sort(Comparator&lt;? super E&gt; c)</pre> 
       <p>This method is introduced in Java 8. This method can be applied conveniently without the need to use a sort method from the <em>Collections</em> class.</p> 
       <pre><strong>package</strong> org.algo.example;

       <h2>Searching</h2> 
       <p>Similarly, there are two methods to search an element from the list, such as</p> 
       <ul> 
        <li><tt>static &lt;T&gt; int binarySearch(List&lt;? extends Comparable&lt;? super T&gt;&gt; list, T key)</tt></li> 
        <li><tt>static &lt;T&gt; int binarySearch(List&lt;? extends T&gt; list, T key, Comparator&lt;? super T&gt; c)</tt></li> 
       </ul> 
       <p>The binary search algorithm is the fastest searching technique known to man, but it can be applied only in a sorted list. According to the Java documentation, the list must be sorted in ascending order according to the natural ordering of the elements before calling this method. Otherwise, the result of a binary search method applied to an unsorted list is undefined. It normally takes Order of, <em>O(log n)</em> on random access list; also, it may take Order <em>O(n)</em>, time depending upon how large the list is or whether the list implements a <em>RandomAccess</em> interface or not. In this case, the method uses an iterative based binary search. Iterative based binary searching is done on links that take <em>O(n)</em> time whereas the actual element comparison takes <em>O(log n)</em> as we know. So, <em>O(n) + O(log n) = O(n) time</em>, according to big O. Let's see a quick example.</p> 
       <pre><strong>package</strong> org.algo.example;

<strong>import</strong> java.util.ArrayList;
<strong>import</strong> <span style="background: silver;">java.util.Collections</span>;
<strong>import</strong> java.util.List;

<strong>public class</strong> Main {

   <strong>public static void</strong> main(String[] args) {

      List&lt;String&gt; list = <strong>new</strong> ArrayList&lt;&gt;();
      list.add("Cumin");
      // ...
      list.add("Fennel");

      <span style="background: silver;">Collections</span>.<em>sort</em>(list);
      <strong>int</strong> index=<span style="background: silver;">Collections</span>.<em>binarySearch</em>(list,
         "Parsley");

      System.<strong><em>out</em></strong>.println("After searching: "
         +list.get(index));
   }
}
</pre>

       <p>If the searched element is found, the method returns the index; otherwise, it returns a negative value defined by ( - (insertion_point) - 1 ). The <em>insertion point</em> is the point at which the key would be inserted into the list, meaning that, by the negative number returned, you'll know exactly where the (not found) value would be inserted (if you insert it into the list subsequently). If the returned negative value is -7, the value will be inserted at the 6<sup>th</sup> index; similarly for -3, at the index 2<sup>nd</sup>, and so on.</p> 
       <h2>Some Other Algorithms</h2> 
       <p>Shuffling provides a random permutation of elements in the List. This can be applied with the help of two methods, one with default sources of randomness and another where a custom random source can be supplied.</p> 
       <ul> 
        <li><tt>public static void shuffle(List&lt;?&gt; list)</tt></li> 
        <li><tt>public static void shuffle(List&lt;?&gt; list, Random rnd)</tt></li> 
       </ul> 
       <p>Internally, what it does is traverse the list backward from the last element to the second and swap the randomly selected element into the current position, inclusively. Let's see an example:</p> 
       <pre><strong>package</strong> org.algo.example;

<strong>import</strong> java.util.ArrayList;
<strong>import</strong> java.util.Collections;
<strong>import</strong> java.util.List;

<strong>public class</strong> Main {

   <strong>public static void</strong> main(String[] args) {

      List&lt;String&gt; list = <strong>new</strong> ArrayList&lt;&gt;();
      list.add("Cumin");
      // ...
      list.add("Fennel");

      System.<strong><em>out</em></strong>.println("Before shuffle: "+list);
      Collections.<em>shuffle</em>(list);
      System.<strong><em>out</em></strong>.println("After shuffle: "+list);
   }
}
</pre> 
       <p>Similarly, we can <strong>reverse</strong> a list.</p> 
       <pre>public static void reverse(List&lt;?&gt; list)

System.<strong><em>out</em></strong>.println("Before Reverse "+list);
Collections.<em>reverse</em>(list);
System.<strong><em>out</em></strong>.println("After Reverse "+list);
</pre> 
       <p><strong>Swap</strong> elements at list index.</p> 
       <pre>public static void swap(List&lt;?&gt; list, int i, int j)

System.<strong><em>out</em></strong>.println("Before swap "+list);
Collections.<em>swap</em>(list,4,5);
System.<strong><em>out</em></strong>.println("After swap "+list);
</pre> 
       <p><strong>Rotating</strong> elements in the list.</p> 
       <p>The rotations will be done in a manner where the element at index say, i, will be the element previously at <em>( i - distance) mod list.size()</em> for all values <em>0</em> to <em>list.size()-1</em>, inclusive. The method signature is as follows.</p> 
       <pre>public static void rotate(List&lt;?&gt; list,int distance)</pre> 
       <p>And can be implemented in the following manner:</p> 
       <pre>System.<em>out</em>.println("Before rotate "+list);
Collections.<em>rotate</em>(list,3);

System.<em>out</em>.println("After rotate "+list);
</pre> 
       <p>This will make the third element from the end of the list as the first element, the last second element will be the second element, and the last element will be the first element.</p>

       <h2>Conclusion</h2> 
       <p>The <em>java.util</em> package contains utility classes that form the base of the Java Collection Framework. Java.util.Collections is one of the utility classes that contains only static methods. These methods define many common algorithms, such as shuffling elements in a collection, rotating its elements, sorting elements of a list, and the like. The Collections class also contains methods to obtain different views of collections, such as read-only view, synchronized view, un-modifiable view, and so forth.</p>
