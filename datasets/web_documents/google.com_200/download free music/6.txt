            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>I used quite @Eyuel method: </p> 
              <ul> 
               <li>Download the nodejs msi from <a href="https://nodejs.org/en/#download" rel="nofollow noreferrer">https://nodejs.org/en/#download</a></li> 
               <li>Download npm zip from github <a href="https://github.com/npm/npm" rel="nofollow noreferrer">https://github.com/npm/npm</a> </li> 
               <li>Extract the msi (with 7 Zip) in a directory "node"</li> 
               <li>Set the PATH environment variable to add the "node" directory</li> 
               <li>Extract the zip file from npm in a different directory (not under node directory)</li> 
               <li>CD to the npm directory and run the command <code>node cli.js install npm -gf</code></li> 
              </ul> 
              <p>Now you should have node + npm working, use theses commands to check: <code>node --version</code> and <code>npm --version</code></p> 
              <p><strong>Update 27/07/2017</strong> : I noticed that the latest version of node 8.2.1 with the latest version of npm are quite different from the one I was using at the time of this answer. The install with theses versions won't work. It is working with node 6.11.1 and npm 5.2.3. Also if you are running with a proxy don't forget this to connect on internet :</p> 
              <ul> 
               <li>export http_proxy=<a href="http://proxy:8080" rel="nofollow noreferrer">http://proxy:8080</a> </li> 
               <li>export https_proxy=<a href="http://proxy:8080" rel="nofollow noreferrer">http://proxy:8080</a></li> 
               <li>npm config set proxy <a href="http://proxy:8080" rel="nofollow noreferrer">http://proxy:8080</a></li> 
              </ul>

             <div class="post-text" itemprop="text"> 
              <p>I also needed to install npm in Windows and got it through the Chocolatey pacakage manager. For those who haven't heard about it, Chocolatey is a package manager for Windows, that gives you the convenience of an apt-get in Windows environments. To get it go to <a href="https://chocolatey.org/" rel="nofollow">https://chocolatey.org/</a> where there's a PowerShell script to download it and install it. After that you can run:</p> 
              <pre><code>chocolatey install npm
</code></pre> 
              <p>and you're good to go.</p> 
              <p>Note that the standalone npm is no longer being updated and the last version that is out there is known to have problems on Windows. Another option you can look at is extracting npm from the MSI using <a href="https://chocolatey.org/packages/lessmsi" rel="nofollow">LessMSI</a>.</p>

             <div class="post-text" itemprop="text"> 
              <p>Just download "node.exe" from <a href="http://nodejs.org/dist/" rel="nofollow">http://nodejs.org/dist/</a>, select your favorite "node.js" version or take the <a href="http://nodejs.org/dist/latest/" rel="nofollow">latest</a>. You can also take 64-bits version from "x64" sub-directory.</p> 
              <p>Then, go to <a href="http://nodejs.org/dist/npm/" rel="nofollow">http://nodejs.org/dist/npm/</a> to retrieve Zip-archive of your favorite "npm" version (recommanded : <code>1.4.10</code>). Extract the archive along "node.exe".</p> 
              <p>Finally, it is recommanded to add "node.js" directory to the PATH for convenience.</p> 
              <p>EDIT: I recommande to update npm using <code>npm install npm -g</code> because versions provided by nodejs.org are very old.</p> 
              <p>If you want to keep original npm version, don't put npm alongside <code>"node.exe"</code>. Just create a directory and use the same command with "global" flag, then copy <code>.\node_modules\.bin\npm.cmd</code> to the new directory :</p> 
              <pre><code>mkdir c:\app\npm\_latest
cd c:\app\npm\_latest
&lt;NPM_ORIGINAL_PATH&gt;\npm install npm
cp node_modules\.bin\npm.cmd npm.cmd
</code></pre> 
              <p>Finally change your PATH to use <code>c:\app\npm\_latest</code></p>

             <div class="post-text" itemprop="text"> 
              <p>Download the latest Node.js MSI (4.x or 5.x) installer and run the following via command line:</p> 
              <pre><code>msiexec /a node-v4.4.3-x64.msi /qb TARGETDIR="C:\Node.js"
</code></pre> 
              <p>This will extract the binaries into <code>C:\Node.js\nodejs</code>.</p> 
              <p>Then you will want to add <code>C:\Node.js\nodejs</code> <code>PATH</code> environment variable.</p> 
              <p>To update NPM, do the following:</p> 
              <pre><code>cd C:\Node.js\nodejs
npm install npm@latest
</code></pre> 
              <p>After that completes, you should be able to check the versions:</p> 
              <pre><code>node --version
npm --version
</code></pre> 
              <p>Node should be 4.4.3+ (whichever you installed) and npm should be 3.8.7+.</p>

              <h1>"Chocolatey installs in seconds"</h1> 
              <p>If you don't care about sarcasm or lamenting engineers on a Sunday afternoon, skip ahead to the installation instructions in the <strong>TL;DR</strong> section below.</p> 
              <p>For everyone else: I want to amend <a href="https://stackoverflow.com/a/25980740/2228771">this answer</a> which recommends using (what seems to be the hottest package management solution for Windows right now): <a href="https://chocolatey.org/" rel="nofollow noreferrer">Chocolatey</a></p> 
              <p>It gets the job done nice and quick. However, when I gave it a first try, it took me a while to make sense of the install instructions which are kinda convoluted. The install instructions go a bit like this (complete with what went through my head while going through it):</p> 
              <ul> 
               <li><strong>NOTE:</strong></li> 
               <li><strong>NOTE:</strong></li> 
               <li><strong>NOTE:</strong> 
                <blockquote> 
                 <p>three big-ass NOTEs before even knowing the basics... this makes me anxious... how wrong could things go!?</p> 
                </blockquote></li> 
               <li><strong>Installing With Restricted TLS</strong> 
                <blockquote> 
                 <p>not even sure what <code>TLS</code> is... Oh it's a good friend of <code>SSL</code> - Shouldn't this just be the default and just work out of the box? My browser can do <code>HTTPS</code>, NO PROBLEM! (just kidding... I know that <code>SSL</code> and <code>TSL</code> frequently cause a lot of pain in environments that have high security needs)</p> 
                </blockquote></li> 
               <li><strong>Option 1</strong></li> 
               <li><strong>Option 2</strong> 
                <blockquote> 
                 <p>eeh... great... Can't I just install?</p> 
                </blockquote></li> 
               <li><strong>Installing Behind a Proxy?</strong> 
                <blockquote> 
                 <p>Just... no...</p> 
                </blockquote></li> 
               <li><strong>Requirements</strong> 
                <blockquote> 
                 <p>uh boi...</p> 
                </blockquote></li> 
               <li><strong>Why does Chocolatey install where it does by default?</strong> 
                <blockquote> 
                 <p>Seriously!?!</p> 
                </blockquote></li> 
               <li><strong>Before You Install</strong> 
                <blockquote> 
                 <p><a href="http://www.clipartbest.com/cliparts/jix/Ekx/jixEkxaGT.png" rel="nofollow noreferrer">sad http://www.clipartbest.com/cliparts/jix/Ekx/jixEkxaGT.png</a></p> 
                </blockquote></li> 
               <li><strong>Can I install with a proxy?</strong> 
                <blockquote> 
                 <p>again with the proxy...</p> 
                </blockquote></li> 
               <li><strong>Can I install a particular version of Chocolatey?</strong> 
                <blockquote> 
                 <p>just any version would be fine, thank you...</p> 
                </blockquote></li> 
               <li><strong>Can I use Windows built-in compression instead of downloading 7zip?</strong> 
                <blockquote> 
                 <p>7zip?! Why do you even mention this?!</p> 
                </blockquote></li> 
               <li><strong>Non-Administrative Install</strong></li> 
               <li>
                <blockquote> 
                 <p>Now that sounds great!</p> 
                </blockquote> 
                <ul> 
                 <li>"<strong>NOTE</strong>: This option should be a last resort and is considered to be an advanced scenario." 
                  <ul> 
                   <li>
                    <blockquote> 
                     <p>Sh$%!@T.</p> 
                    </blockquote></li> 
                  </ul></li> 
                </ul></li> 
               <li><strong>Alternative Installation Options</strong> 
                <blockquote> 
                 <p><em>giddy</em></p> 
                </blockquote></li> 
               <li><strong>Command Line</strong> 
                <ul> 
                 <li>"This really is the easiest method because it requires no configuration of PowerShell prior to executing it." 
                  <blockquote> 
                   <p>And there you go!</p> 
                  </blockquote></li> 
                </ul></li> 
              </ul> 
              <p>While I really appreciate the fact that pitfalls and their possible solutions are discussed so extensively, maybe re-organizing them as such, and putting the <code>Chocolatey installs in seconds</code> promise to work by putting the "easiest method" first would be just awesome!</p> 
              <h1>TL;DR: Install Chocolatey, Node (and NPM)</h1> 
              <h2>Install Chocolatey</h2> 
              <p>NOTE: You might want to copy the exact command from <a href="https://chocolatey.org/install" rel="nofollow noreferrer">their install page</a> since it might change over time.</p> 
              <ol> 
               <li>Open your standard Windows command line</li> 
               <li><code>@powershell -NoProfile -ExecutionPolicy unrestricted -Command "(iex ((new-object net.webclient).DownloadString('https://chocolatey.org/install.ps1'))) &gt;$null 2&gt;&amp;1" &amp;&amp; SET PATH="%PATH%;%ALLUSERSPROFILE%\chocolatey\bin"</code></li> 
               <li>Restart console</li> 
               <li>Done!</li> 
              </ol> 
              <h2>Install Node (and NPM)</h2> 
              <ol> 
               <li>Because Chocolatey installs a directory different from the MSI installation, go to your system configuration and delete your previous <code>node</code> installation (if you have one)</li> 
               <li>Install Chocolatey as described above</li> 
               <li><code>choco install nodejs</code> 
                <blockquote> 
                 <p>NOTE I used <code>nodejs</code>. I am not even sure what <code>node</code> is, but having had my fair share of troubles with <code>node</code> already being taken by something else in other systems, I thought, <code>nodejs</code> would be the safer bet right away</p> 
                </blockquote></li> 
               <li>Restart your console</li> 
               <li>Beware: <code>node -v</code> works! 
                <ul> 
                 <li>And <code>npm -v</code> also works! Awesome.</li> 
                </ul></li> 
              </ol> 
              <p>After this, I was able to install <code>firebase-tools</code> without problems (which I was not able to do previously), so something must have gone terribly right! :)</p>
