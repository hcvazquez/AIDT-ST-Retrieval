        <figure> 
         <div>
          <img src="https://media.scmagazine.com/images/2014/04/21/179216716_583109.jpg?format=jpg&amp;zoom=1&amp;quality=70&amp;anchor=middlecenter&amp;mode=pad" alt="On Aug. 1, 2017, npm removed around 40 malicious, fraudulent JavaScript packages that were designed to collect data from unknowing users." title="On Aug. 1, 2017, npm removed around 40 malicious, fraudulent JavaScript packages that were designed to collect data from unknowing users." border="0">

          On Aug. 1, 2017, npm removed around 40 malicious, fraudulent JavaScript packages that were designed to collect data from unknowing users.
         </figcaption>

        <p>The developers of the JavaScript&nbsp;programming language package manager "npm" have disclosed that they recently removed&nbsp;approximately 40 fraudulent, malware-spiked packages that were designed to steal environment variables upon installation.</p> 
        <p>In&nbsp;a classic case of typosquatting intended to fool inattentive users, the fake packages featured names that were just slightly different than actual, genuine packages offered by npm. "The package naming was both deliberate and malicious –&nbsp;the intent was to collect useful data from tricked users," npm explained in a <a href="http://blog.npmjs.org/post/163723642530/crossenv-malware-on-the-npm-registry">blog post</a>.</p> 
        <p>According to npm, a user by the handle of "hacktask" published the malicious libraries on July 19, including two that mimicked the popular "cross-env," &nbsp;which between then were downloaded nearly 700 times before they were removed on Aug. 1. Fortunately, only about 50 of these downloads appear to be genuine installations from real users, while the rest came from &nbsp;registry mirrors that automatically downloaded copies, npm explained.</p> 
        <p>It was a Swedish npm user who initially alerted npm to the problem, <a href="https://twitter.com/o_cee/status/892306836199800836">reporting via Twitter</a> that a false cross-env package was engaged in suspicious activity.</p> 
        <p>"If you downloaded and installed any of these packages, you should immediately revoke and replace any credentials you might have had in your shell environment," npm advised.</p> 
        <p>In response to the incident, npm banned the user "hacktask." Additionally, npm said that its developers are discussing taking various approaches to detecting and preventing future instances of accidental or malicious typosquatting.</p> 
        <p>"There are programmatic ways to detect this, and we might use them to block publication," the npm blog post reads. "We're using the&nbsp;Smyte&nbsp;service&nbsp;[a trust and safety&nbsp;SaaS offering] to detect spam as it is published to the registry, and will be experimenting with using it to detect other kinds of violations of our terms of service."</p>
