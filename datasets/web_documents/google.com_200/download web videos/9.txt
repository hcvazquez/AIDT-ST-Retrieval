       <h2> <span class="mw-headline" id="Bower_and_RequireJS"> Bower and RequireJS </span></h2> 
       <p>If you use bower package manager for your project, you can install JS SDK through bower: </p> 
       <pre>bower install quickblox --save
</pre> 
       <p>When you use <b>RequireJS</b>, you are able to use quickblox as AMD module. SDK supports <a rel="nofollow" class="external text" href="https://github.com/umdjs/umd">UMD (Universal Module Definition)</a> pattern for JavaScript modules. So it is possible to use SDK everywhere (as browser global variable, with AMD module loader like RequireJS or as CommonJS module for Node.js environment). </p> 
       <h2> <span class="mw-headline" id="Node.js_and_NPM_integration"> Node.js and NPM integration </span></h2>

       <pre>npm install quickblox --save
</pre>

       <h1> <span class="mw-headline" id="Getting_started"> Getting started </span></h1> 
       <p>To use inside a browser, a window scoped variable called <code>QB</code> is created. Most, with the notable exception of init, take a callback parameter which gets called with an error and result parameter (ala <code>node.js</code>). </p>
       <p>The common way to interact with QuickBlox can be presented with the following sequence of actions: </p> 
       <ol>
        <li> Initialize framework </li>
        <li> Authorisation </li>
        <li> Perform actions </li>
       </ol> 
       <h2> <span class="mw-headline" id="Initialize_framework"> Initialize framework </span></h2> 
       <p>In addition, there is helpful <a href="/developers/5_Minute_Guide" title="5 Minute Guide">5 minute guide</a> </p>
       <p><br> Initialize framework with application credentials: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1"><span class="kw2">var</span> CREDENTIALS <span class="sy0">=</span> <span class="br0">{</span>
  appId<span class="sy0">:</span> <span class="nu0">28287</span><span class="sy0">,</span>
  authKey<span class="sy0">:</span> <span class="st0">'XydaWcf8OO9xhGT'</span><span class="sy0">,</span>
  authSecret<span class="sy0">:</span> <span class="st0">'JZfqTspCvELAmnW'</span>
<span class="br0">}</span><span class="sy0">;</span>
&nbsp;
QB.<span class="me1">init</span><span class="br0">(</span>CREDENTIALS.<span class="me1">appId</span><span class="sy0">,</span> CREDENTIALS.<span class="me1">authKey</span><span class="sy0">,</span> CREDENTIALS.<span class="me1">authSecret</span><span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>Initialize framework with existing QB token: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1"><span class="kw2">var</span> sessionToken <span class="sy0">=</span> <span class="st0">'1b785b603a9ae88d9dfbd1fc0cca0335086927f1'</span><span class="sy0">;</span>
<span class="kw2">var</span> appId <span class="sy0">=</span> <span class="nu0">3451</span><span class="sy0">;</span>
&nbsp;
QB.<span class="me1">init</span><span class="br0">(</span>sessionToken<span class="sy0">,</span> appId<span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>This means you can generate a token elsewhere (e.g. on the server side), and then use this token and application ID to initialise the SDK. </p> 
       <h2> <span class="mw-headline" id="Authorization"> Authorization </span></h2> 
       <p>To be able to use QuickBlox API you have to create a session. </p>
       <p>There are 2 types of session: </p> 
       <ul>
        <li> <b>Application session</b>. It provides only READ access to data. </li>
        <li> <b>User session</b>. It provides CRUD (Create, Read, Update, Delete) access to data. </li>
       </ul> 
       <p><br> To create an application session use this code: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1">QB.<span class="me1">createSession</span><span class="br0">(</span><span class="kw2">function</span><span class="br0">(</span>err<span class="sy0">,</span> result<span class="br0">)</span> <span class="br0">{</span>
  <span class="co1">// callback function</span>
<span class="br0">}</span><span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>With an Application session you can READ any data you need and only have to do one Create operation — <a rel="nofollow" class="external text" href="http://quickblox.com/developers/Sample-users-javascript#Signing_Up">User Sign Up</a>). </p>
       <p>To update an Application session to a User session you have to login to QuickBlox: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1"><span class="kw2">var</span> params <span class="sy0">=</span> <span class="br0">{</span>login<span class="sy0">:</span> <span class="st0">'garry'</span><span class="sy0">,</span> password<span class="sy0">:</span> <span class="st0">'garry5santos'</span><span class="br0">}</span><span class="sy0">;</span>
&nbsp;
<span class="co1">// or through email</span>
<span class="co1">// var params = {email: 'garry@gmail.com', password: 'garry5santos'};</span>
&nbsp;
<span class="co1">// or through social networks (Facebook / Twitter)</span>
<span class="co1">// var params = {provider: 'facebook', keys: {token: 'AM46dxjhisdffgry26282352fdusdfusdfgsdf'}};</span>
&nbsp;
QB.<span class="me1">login</span><span class="br0">(</span>params<span class="sy0">,</span> <span class="kw2">function</span><span class="br0">(</span>err<span class="sy0">,</span> result<span class="br0">)</span> <span class="br0">{</span>
  <span class="co1">// callback function</span>
<span class="br0">}</span><span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>Or, if the user is already exist, you can create a User session in single query as well: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1"><span class="kw2">var</span> params <span class="sy0">=</span> <span class="br0">{</span>login<span class="sy0">:</span> <span class="st0">'garry'</span><span class="sy0">,</span> password<span class="sy0">:</span> <span class="st0">'garry5santos'</span><span class="br0">}</span><span class="sy0">;</span>
&nbsp;
<span class="co1">// or through email</span>
<span class="co1">// var params = {email: 'garry@gmail.com', password: 'garry5santos'};</span>
&nbsp;
<span class="co1">// or through social networks (Facebook / Twitter)</span>
<span class="co1">// var params = {provider: 'facebook', keys: {token: 'AM46dxjhisdffgry26282352fdusdfusdfgsdf'}};</span>
&nbsp;
QB.<span class="me1">createSession</span><span class="br0">(</span>params<span class="sy0">,</span> <span class="kw2">function</span><span class="br0">(</span>err<span class="sy0">,</span> result<span class="br0">)</span> <span class="br0">{</span>
  <span class="co1">// callback function</span>
<span class="br0">}</span><span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>A session will remain valid for <b>2 hours</b> after the last request to QuickBlox was performed. </p> 
       <h3> <span class="mw-headline" id="Session_expiration">Session expiration</span></h3> 
       <p>You may now specify a session expiration listener in this configuration. It looks like this: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1">config <span class="sy0">=</span> <span class="br0">{</span>
  on<span class="sy0">:</span> <span class="br0">{</span>
    sessionExpired<span class="sy0">:</span> <span class="br0">[</span><span class="kw2">Function</span><span class="br0">]</span>
  <span class="br0">}</span>
<span class="br0">}</span>
...
&nbsp;
<span class="me1">QB</span>.<span class="me1">init</span><span class="br0">(</span><span class="nu0">3477</span><span class="sy0">,</span> <span class="st0">'ChRnwEJ3WzxH9O4'</span><span class="sy0">,</span> <span class="st0">'AS546kpUQ2tfbvv'</span><span class="sy0">,</span> config<span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>It has been placed inside the "on" field so that we can add more listeners in the future. <b>This function fires when the SDK makes a request and recognises an expired session error.</b> It will execute this function before it executes the original callback for the request. You could use it to create a new session, request a new token from your backend, or simply tell the user to refresh the page. </p>
       <p>This function accepts 2 parameters, "<code>next</code>" and "<code>retry</code>". Calling <code>next()</code> will cause the original callback of the request to be fired - you would use this if your session creation/regeneration fails. Retry will make the original request again with your newly regenerated token, meaning minimal interruption to the operation of your code. </p> 
       <h2> <span class="mw-headline" id="Configuration"> Configuration </span></h2> 
       <p>We recommend you to create <code>config.js</code> configuration file in your js-application and insert your QB application credentials and other ancillary data into it. </p>
       <p>A custom configuration can look like this: </p> 
       <div dir="ltr" class="mw-geshi" style="text-align: left;">
        <div class="javascript source-javascript">
         <pre class="de1"><span class="kw2">var</span> CONFIG <span class="sy0">=</span> <span class="br0">{</span>
  endpoints<span class="sy0">:</span> <span class="br0">{</span>
    api<span class="sy0">:</span> <span class="st0">"apicustomdomain.quickblox.com"</span><span class="sy0">,</span> <span class="co1">// set custom API endpoint</span>
    chat<span class="sy0">:</span> <span class="st0">"chatcustomdomain.quickblox.com"</span> <span class="co1">// set custom Chat endpoint</span>
  <span class="br0">}</span><span class="sy0">,</span>
  chatProtocol<span class="sy0">:</span> <span class="br0">{</span>
    active<span class="sy0">:</span> <span class="nu0">2</span> <span class="co1">// set 1 to use BOSH, set 2 to use WebSockets (default)</span>
  <span class="br0">}</span><span class="sy0">,</span>
  debug<span class="sy0">:</span> <span class="br0">{</span>mode<span class="sy0">:</span> <span class="nu0">1</span><span class="br0">}</span> <span class="co1">// set DEBUG mode</span>
<span class="br0">}</span><span class="sy0">;</span>
QB.<span class="me1">init</span><span class="br0">(</span><span class="nu0">3477</span><span class="sy0">,</span> <span class="st0">'ChRnwEJ3WzxH9O4'</span><span class="sy0">,</span> <span class="st0">'AS546kpUQ2tfbvv'</span><span class="sy0">,</span> CONFIG<span class="br0">)</span><span class="sy0">;</span></pre>
        </div>
       </div> 
       <p>The above would enable debugging, disable ssl and change the endpoints so that requests would go to another server. </p>
       <p>There are 3 debug mods: </p> 
       <ul>
        <li> <b>debug: {mode: 0}</b> - logs are off </li>
        <li> <b>debug: {mode: 1}</b> - the SDK will be printing logs to <b>console.log()</b> </li>
        <li> <b>debug: {mode: 2, file: "appname_debug.log"}</b> - the SDK will be printing logs into file with name "appname_debug.log". Works only on Node.js. </li>
       </ul>
