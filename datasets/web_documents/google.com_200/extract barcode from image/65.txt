          <div class="blog" style="width: 75%;"> 
           <p>A really handy thing to be able to do is to control&nbsp;USB devices using JavaScript. This is a powerful capability, made easily possible by&nbsp;<a title="Node" href="https://nodejs.org/en/" target="_blank">NodeJS</a> (better known, simply, as Node). To get Node, go <a title="Node downloads page" href="https://nodejs.org/en/download/" target="_blank">here</a>. &nbsp;&nbsp;<img style="width: 50%; height: 50%;" title="b2ap3_thumbnail_nodejs-logo.png" src="/images/easyblog_images/912/b2ap3_thumbnail_nodejs-logo.png" alt="b2ap3_thumbnail_nodejs-logo.png" align="right"></p> 
           <p>Node is an increasingly vital piece of infrastructure for companies that do business via the Web. You've heard of many&nbsp;of the companies I'm talking about. (Check out their logos below.)</p> 
           <p>Most companies that use Node run it on the server side.&nbsp;But it's easily possible (and definitely worthwhile) to run it on the client side, too.&nbsp;</p> 
           <p>Unlike scripts written for web pages, scripts that run on Node have no security restrictions that would keep them from performing file I/O,&nbsp;opening network connections across domains, shelling out to the OS, or calling native code modules, among other things. Since they're unhampered by "sandbox security" restrictions, scripts that run on Node can do amazing, powerful things that browser scripts could never do.</p> 
           <div style="margin: 0 0 0 80px; width: 100%;">
            <img title="b2ap3_thumbnail_WhoUsesNode.png" src="/images/easyblog_images/912/b2ap3_thumbnail_WhoUsesNode.png" alt="b2ap3_thumbnail_WhoUsesNode.png">
            <br>
            <i style="width: 100%;">Companies that use Node for critical infrastructure.</i>
           </div> 
           <br> 
           <p>When I wanted to connect to ID TECH's USB-based credit card readers using JavaScript, I looked to see what kind of USB connectivity modules might already exist for Node. Turns out there are several. I chose one called <a title="node-hid module" href="https://github.com/node-hid/node-hid" target="_blank">node-hid</a>, which is based on the cross-platform <a title="HIDAPI" href="http://www.signal11.us/oss/hidapi/" target="_blank">HIDAPI&nbsp;</a>library.</p> 
           <p>I chose <strong>node-hid</strong> mostly because of its brain-dead-simple API, but also based on the fact that it <span style="text-decoration: underline;">installs</span> super-easily. Just fire up a command line console and run <strong>npm install node-hid</strong>. Node's <strong>npm</strong> package manager will go to work downloading all the bits you need from the Web, and it will resolve dependencies and build binaries automatically. (Allow a minute or two for this.) When you're done, you'll have the ability to put <code>require 'node-hid'</code> in your Node scripts and connect to USB devices!</p> 
           <p>Below is&nbsp;an example of a script you can run with Node (and node-hid) that will let you connect to USB devices having the ID TECH vendor ID (hex 0xACD):</p> 
           <pre>// ==========  usb-provider ===========
'use strict';

var HID = require("node-hid");
const EventEmitter = require('events');

class USBProvider extends EventEmitter {

    constructor() {
        super();
        var self = this;
        this.onerror = function(e) {
            console.log('error: ' + e);
        };
        this.getDeviceHandle = function() {
            return deviceHandle;
        }

        var SCAN_INTERVAL = 2000; // scan every 2 secs
        var VENDOR_ID = 0xACD; // default ID TECH vid
        var deviceHandle = null; // stores our handle
        var deviceRecord = null; // stores device record
        var stopKey = null; // to stop polling (if needed)

        // This will be called repeatedly by poll(), below
        function cycle() {

            var deviceFound = false;
            HID.devices().forEach(function(device, index, records) {

                deviceFound = (device.vendorId == VENDOR_ID);

                if (device.vendorId == VENDOR_ID &amp;&amp; deviceRecord == null) {
                    deviceRecord = device;
                    try {
                        // Try to connect.
                        deviceHandle =
                            new HID.HID(device.vendorId, device.productId);

                        deviceHandle.on('error', self.onerror);

                        self.emit('usbconnect', deviceHandle);

                        console.log("usbprovider: connect");
                    } catch (e) {
                        self.onerror("Exception caught:\n" + e);
                        self.emit('usbexception', device);
                    }
                } // if

                if (index == records.length - 1 &amp;&amp; !deviceFound) {

                    // HANDLE DISCONNECT EVENT
                    if (deviceRecord != null) {
                        deviceRecord = deviceHandle = null; // nullify record	
                        // self.ondisconnect();
                        self.emit('usbdisconnect');

                        console.log("usbprovider: disconnect");
                    } // if 

                } // if <br>
            }); // forEach
        } // cycle

        this.poll = function() {
            this.stopKey = setInterval(cycle, SCAN_INTERVAL);
        }
    }
}<br>
// Allow other modules to use this one:
module.exports = USBProvider;</pre> 
           <p>We want to inherit the methods of Node's EventEmitter class (so that we can emit custom events). Therefore we start right out by declaring a class that extends <code>EventEmitter</code>. (Note that Node allows the use of <a title="ECMA 6 Classes" href="http://es6-features.org/#ClassDefinition" target="_blank">EMCA 6</a> class-declaration syntax.)</p> 
           <p>We have a <code>poll()</code> method (near the bottom) that uses <code>setInterval()</code> to start an endless polling cycle that has us checking the device list once every two seconds. The <code>cycle()</code> method carries out our poll. It checks every connected device using two 'if' statements. The first 'if' says: If any device with the ID TECH VendorID (0x0ACD) is found, and we're not already connected to a device of interest, we go ahead and try connecting to the device. The second 'if' says: If the poll finds no ID TECH device and yet we already have a non-null <code>deviceRecord</code>, it means the record is stale because the device is unplugged. (Hence we pop a 'usbdisconnect' event.)&nbsp;</p> 
           <p>The USBProvider emits 'usbconnect' and 'usbdisconnect' events, plus 'data' events. When you get a 'usbconnect' event, it's important to realize you're being passed the <code>deviceHandle</code> for the connected device. You need to <span style="text-decoration: underline;">store a reference</span> to that handle right away, so you can later use it to <span style="text-decoration: underline;">write</span> to the device. But there's also another thing you need to do right away, which is register your data handler. This lets you <span style="text-decoration: underline;">read </span>from the device. (Reading happens via 'data' events.) Register your 'data' handler at connect time.</p> 
           <p>Code that uses the <code>USBProvider</code> class might look similar to this:</p> 
           <pre>// First, instantiate the provider
var usb = new USBProvider();

var deviceHandle = null; // We will store the device handle here

// Set up a connection handler. Inside it, set the data handler.
usb.on('usbconnect', function(h) {

    deviceHandle = h; // cache the handle

    // set up a data handler (for reading data)
    deviceHandle.on('data', (data) =&gt; {

        var hex = data.toString('hex');
        // do something with data...
    });

});</pre> 
           <div class="blog" style="width: 80%;"> 
            <p>Notice that we can easily get a hex string version of our data using <code>toString('hex')</code>. That's because the data is in a <a title="Node Buffer API" href="https://nodejs.org/api/buffer.html" target="_blank">Buffer object</a>. The Buffer object is <span style="text-decoration: underline;">not</span> a JavaScript array.&nbsp;</p> 
            <p>Elsewhere in your code, you can <span style="text-decoration: underline;">write</span> to the device very easily, as follows:</p> 
           </div> 
           <pre>    deviceHandle.write( cmdArray );
</pre> 
           <div class="blog" style="width: 80%;"> 
            <p>The <code>cmdArray</code>&nbsp;isn't a Buffer object. It is simply a JavaScript array containing integer values.</p> 
            <p>But don't forget, you can't connect to anything at all&nbsp;<span style="text-decoration: underline;">unless you start up the <strong>poll()</strong> method! </span>So after setting up your event handlers and initializing your code, remember to begin polling:</p> 
           </div> 
           <pre>    usb.poll(); &nbsp;// Be sure to do this! This starts auto-detect &amp; connect</pre> 
           <div class="blog" style="width: 80%;"> 
            <p>With the various bits of code shown above, you'll be able to read and write USB-based ID TECH devices in no time. And if you want, you can use Node's wonderful Websockets capability to send USB data straight to the browser, so that your HTML page can control your payment device. I'll have a lot more to say on that very subject in future posts. So check back here soon!</p> 
           </div>
