The content displaced the layout of the Web site, and Japanese kanji characters displayed as a strange set of characters. 
             <b>Figure A</b> shows an example of the screen layout with and without Japanese support. To solve this problem, I had to use JavaScript to determine whether the language was supported.
             <br>
             <div align="center">

             <br>
             <br>
             <span class="subhead1">Is it supported?</span>
             <br>To determine whether Japanese is supported in the browser, I created a JavaScript application that tests whether the user has sufficient resources to view a Japanese site. If so, the user is redirected to the Web site. If not, the user is redirected to a help page that contains language support download links and troubleshooting information.
             <br>
             <br>I designed an object-oriented JavaScript API to detect Japanese language support. This API is easily modified to detect other language types, especially those that use extended character sets (such as Cyrillic and Chinese). The language detection API uses three primary functions:
             <br>
             <ul>
              <li>Browser localization</li>
              <li>Language component detection</li>
              <li>Unicode Double-Byte character set detection</li>
             </ul>
             <br>Let's take a closer look at each.
             <br>
             <br>
             <span class="subhead1">Browser localization function</span>
             <br>JavaScript's navigator object has several properties to detect language settings, including:
             <br>
             <ul>
              <li>navigator.language (Netscape - Browser Localization)</li>
              <li>navigator.browserLanguage (IE-Specific - Browser Localized Language)</li>
              <li>navigator.systemLanguage (IE-Specific - Windows OS - Localized Language)</li>
              <li>navigator.userLanguage (IE-Specific - Windows OS - Regional Settings)</li>
             </ul>
             <br>The navigatorDetect() function is designed to detect Japanese versions of Netscape or Internet Explorer. It determines whether the user has a Japanese Windows version or has localized either the browser or operating system to Japanese. The following chunk of code determines what browser and operating system the user has and adds a Boolean value to each variable:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">functionjpDetect(){}</span>
             <br>
             <span class="code">var uA=navigator.userAgent.toLowerCase();</span>
             <br>
             <span class="code">var nA=navigator.appName.toLowerCase();</span>
             <br>
             <span class="code">var majV=parseInt(navigator.appVersion);</span>
             <br>
             <span class="code">jpDetect.prototype.IE=((uA.indexOf("msie")!=-1) &amp;&amp; (uA.indexOf("opera")==-1));</span>
             <br>
             <span class="code">jpDetect.prototype.IE4=(uA.indexOf("msie")!=-1);</span>
             <br>
             <span class="code">jpDetect.prototype.IE5=((uA.indexOf("msie")!=-1) &amp;&amp; ((majV="5")||(majV="5.5")||(majV="6")));</span>
             <br>
             <span class="code">jpDetect.prototype.NS4=(nA.indexOf("netscape")!=-1);</span>
             <br>
             <span class="code">jpDetect.prototype.isMac=(navigator.appVersion.indexOf("Mac")!=-1);</span>
             <br>
             <span class="code">jpDetect.prototype.isWin=(navigator.appVersion.indexOf("Win")!=-1);</span>
             <br>
             <br>Next, default values are added to the navigator detection variables:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">jpDetect.prototype.navigatorBrowserLanguage="false";</span>
             <br>
             <span class="code">jpDetect.prototype.navigatorSystemLanguage="false";</span>
             <br>
             <span class="code">jpDetect.prototype.navigatorUserLanguage="false";</span>
             <br>
             <span class="code">jpDetect.prototype.navigatorLanguage="false";</span>
             <br>
             <br>Finally, the navigatorDetect() function reveals whether the default browser language (and Windows system language) has been set to Japanese. Each variable contains a Boolean value:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">jpDetect.prototype.navigatorDetect=function() {</span>
             <br>
             <span class="code">if (this.NS4){</span>
             <br>
             <span class="code">this.navigatorLanguage=(navigator.language=="jp");}</span>
             <br>
             <span class="code">if (this.IE4){</span>
             <br>
             <span class="code">this.navigatorBrowserLanguage=(navigator.browserLanguage=="ja-jp");</span>
             <br>
             <span class="code">this.navigatorSystemLanguage=(navigator.systemLanguage=="ja-jp");</span>
             <br>
             <span class="code">this.navigatorUserLanguage=(navigator.userLanguage=="ja-jp");}}</span>
             <br>
             <br>To detect language support in other browsers, you should examine ja-jp in the client's User Agent using server-side variables.
             <br>
             <br>
             <span class="subhead1">Japanese support component detection</span>
             <br>This functionality works only in Internet Explorer 5.0+. It does not work with Netscape and operating systems other than Windows, since it relies on the Windows registry. The comDetect() function looks for the following four components:
             <br>
             <ul>
              <li>Uniscribe (built-in Japanese support for W2K/IE5.0+)</li>
              <li>Language Auto-Selection (available in IE5+/IE6)</li>
              <li>Japanese Text Display Support (IE5+)</li>
              <li>Microsoft Global IME for Japanese (IE5+)</li>
             </ul>
             <br>The Uniscribe component is integrated in Windows 2000 and provides basic Japanese language support. The Language Auto-Selection component determines whether a download prompt will appear if language support is not present. Japanese Text Display Support is a stand-alone component that can be loaded into the IE browser. If you want to add global Japanese Language support in Windows, I would recommend downloading the 
             <a target="_blank" href="http://www.microsoft.com/windows/ie/downloads/recommended/ime/">Microsoft Global IME</a> (Input Method Editors).
             <br>
             <br>Let's examine the component detection function in detail. First, we define two arrays: comJapanese holds the registry keys we want to check and comStatus holds a Boolean value confirming whether the component was found:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">jpDetect.prototype.comJapanese=new Array();</span>
             <br>
             <span class="code">jpDetect.prototype.comStatus=new Array();</span>
             <br>
             <br>Next, the comJapanese array is populated with the key values for each of the desired components. You can discover a lot more about these components by looking at the Windows registry and searching for the following keys:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">jpDetect.prototype.comJapanese[0]="{76C19B30-F0C8-11cF-87CC-0020AFEECF20}";</span>
             <br>
             <span class="code">jpDetect.prototype.comJapanese[1]="{01679A11-5C6C-11d1-83FF-00C04FD25153}";</span>
             <br>
             <span class="code">jpDetect.prototype.comJapanese[2]="{3bf42070-b3b1-11d1-b5c5-0000f8051515}";</span>
             <br>
             <span class="code">jpDetect.prototype.comJapanese[3]="{76C19B50-F0C8-11CF-87CC-0020AFEECF20}";</span>
             <br>
             <br>The comDetect() function loops through the comJapanese array and checks each component using IE's oClientCaps behavior. The result is passed into the comStatus array (which can be displayed or evaluated):
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">jpDetect.prototype.comDetect=function(){</span>
             <br>
             <span class="code">if (this.IE5 &amp;&amp; !(this.isMac)){</span>
             <br>
             <span class="code">for(i=0; i&lt;=3; i++){</span>
             <br>
             <span class="code">this.comStatus[i]=oClientCaps.isComponentInstalled(this.comJapanese[i],"componentid");}}}</span>
             <br>
             <br>
             <span class="subhead1">Unicode Double-Byte character set (DBCS) detection</span>
             <br>Japanese kanji characters (also known as glyphs) differ in many ways from the standard English alphabet. The Japanese alphabet is part of the Unicode specifications, and each character is represented by Double Bytes in the Web browser. (The English alphabet is Single Byte.) We can use this unique characteristic to detect the existence of Japanese language support. For more information about Unicode and character sets, please refer to this 
             <a target="_blank" href="http://www.microsoft.com/typography/unicode/cscp.htm">useful Microsoft primer</a>.
             <br>
             <br>First, we set the Web page content-type to Japanese. Next, using JavaScript, we test a string containing a Japanese character. If the browser has the appropriate language support, the string length will be equal to 1 (the single Japanese character). If the browser cannot render the Japanese character, the string length will be equal to 2the two characters (or bytes) representing the single Japanese character. Here is the code:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">&lt;html&gt;</span>
             <br>
             <span class="code">&lt;head&gt;</span>
             <br>
             <span class="code">&lt;META http-equiv=content-type content=text/html;charset=x-sjis&gt;</span>
             <br>
             <br>We are setting the content-type to Shift-JIS. For more information about this character set, refer to 
             <a target="_blank" href="http://www.microsoft.com/globaldev/reference/dbcs/932.htm">this table</a>
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">&lt;/head&gt;</span>
             <br>
             <span class="code">&lt;body&gt;</span>
             <br>
             <span class="code">&lt;script language="Javascript"&gt;</span>
             <br>
             <span class="code">dbcsTest="&nbsp;";</span>
             <br>
             <br>The dbcsTest variable contains two characters &nbsp;, which represent a single Japanese character. We test the string length to find out whether Japanese language support is present:
             <br>
             <span class="code">&nbsp;</span>
             <br>
             <span class="code">if ((dbcsTest.length)==1){</span>
             <br>
             <span class="code">document.write("Client has Japanese language support.");</span>
             <br>
             <span class="code">}else{</span>
             <br>
             <span class="code">document.write("Client does not have Japanese language support."); }</span>
             <br>
             <span class="code">&lt;/script&gt;&lt;/body&gt;&lt;/html&gt;</span>
             <br>
             <br>You can test this application by changing the language encoding on the Web page. 
             <b>Figure B</b> shows what's displayed if we change the language setting to Western European.

             <br>
             <br>
             <span class="subhead1">A caveat</span>
             <br>If you try to change the Web page encoding, certain versions of Internet Explorer may automatically prompt the user with the question 
             <i>Do you want to download Japanese Language pack?</i> Unfortunately, there is no Web-based method of turning off this feature in the browser. However, there is a workaround: You can use the JavaScript component check for Language Auto-Detection. If the component is present, you can skip the DBCS test and thus avoid having the download prompt disrupt JavaScript execution.
