            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <ol> 
               <li><p>Remove all the Java related packages (Sun, Oracle, OpenJDK, IcedTea plugins, GIJ):</p> <pre><code>dpkg-query -W -f='${binary:Package}\n' | grep -E -e '^(ia32-)?(sun|oracle)-java' -e '^openjdk-' -e '^icedtea' -e '^(default|gcj)-j(re|dk)' -e '^gcj-(.*)-j(re|dk)' -e '^java-common' | xargs sudo apt-get -y remove
sudo apt-get -y autoremove
</code></pre></li> 
               <li><p>Purge config files:</p> <pre><code>dpkg -l | grep ^rc | awk '{print($2)}' | xargs sudo apt-get -y purge
</code></pre></li> 
               <li><p>Remove Java config and cache directory:</p> <pre><code>sudo bash -c 'ls -d /home/*/.java' | xargs sudo rm -rf
</code></pre></li> 
               <li><p>Remove manually installed JVMs:</p> <pre><code>sudo rm -rf /usr/lib/jvm/*
</code></pre></li> 
               <li><p>Remove Java entries, if there is still any, from the <em>alternatives</em>:</p> <pre><code>for g in ControlPanel java java_vm javaws jcontrol jexec keytool mozilla-javaplugin.so orbd pack200 policytool rmid rmiregistry servertool tnameserv unpack200 appletviewer apt extcheck HtmlConverter idlj jar jarsigner javac javadoc javah javap jconsole jdb jhat jinfo jmap jps jrunscript jsadebugd jstack jstat jstatd native2ascii rmic schemagen serialver wsgen wsimport xjc xulrunner-1.9-javaplugin.so; do sudo update-alternatives --remove-all $g; done
</code></pre></li> 
               <li><p>Search for possible remaining Java directories:</p> <pre><code>sudo updatedb
sudo locate -b '\pack200'
</code></pre> <p>If the command above produces any output like <code>/path/to/jre1.6.0_34/bin/pack200</code> remove the directory that is parent of <strong>bin</strong>, like this: <code>sudo rm -rf /path/to/jre1.6.0_34</code>.</p></li> 
              </ol>

              <p>You may not need to completely remove the OpenJDK to resolve your problem. Install the sun-java6 packages. Then use <code>update-java-alternatives</code> to switch to the Sun java packages. </p> 
              <p>If you do want to completely remove OpenJDK remove the default-jdk and/or default-jre packages. You may need to remove some java packages but most of them should be happy once you have the Sun JDK packages installed. </p> 
              <p>You may want to follow one of these <a href="http://raphaelhertzog.com/2011/01/31/debian-cleanup-tip-1-get-rid-of-useless-configuration-files/">cleanup tips</a> once you are done removing packages.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <blockquote> 
               <p>To uninstall Oracle Java 7, just press <kbd>Ctrl</kbd>+<kbd>Alt</kbd>+<kbd>T</kbd> on your keyboard to open Terminal. When it opens, run the command below.</p> 
               <pre><code>sudo update-alternatives --display java
</code></pre> 
               <p>To check the setup before uninstalling Java.</p> 
               <p>Next, remove symlinks </p> 
              </blockquote> 
              <p><strong>(replace the word (version)with your Java version. DO <code>java -version</code> to get yours. So if your version is 1.7.0_03, you would <code>type sudo update-alternatives --remove "java" "/usr/lib/jvm/jdk1.7.0_03/bin/java"</code>)</strong></p> 
              <blockquote> 
               <pre><code>sudo update-alternatives --remove "java" "/usr/lib/jvm/jdk&lt;version&gt;/bin/java"
sudo update-alternatives --remove "javac" "/usr/lib/jvm/jdk&lt;version&gt;/bin/javac"
sudo update-alternatives --remove "javaws" "/usr/lib/jvm/jdk&lt;version&gt;/bin/javaws"
</code></pre> 
               <p>verify that the symlinks were removed</p> 
               <pre><code>java -version
javac -version
which javaws
</code></pre> 
               <p><strong>The next 2 commands must be type excatly perfectly to avoid permanently destroying your system</strong>. </p> 
               <pre><code>cd /usr/lib/jvm
sudo rm -rf jdk&lt;version&gt;
</code></pre> 
               <p>Then do</p> 
               <pre><code>sudo update-alternatives --config java
sudo update-alternatives --config javac
sudo update-alternatives --config javaws
</code></pre> 
               <p>Then do</p> 
               <pre><code>sudo vi  /etc/environment
</code></pre> 
               <p>Delete the line with JAVA_HOME <sup>1</sup></p> 
              </blockquote> 
              <hr> 
              <hr> 
              <p>To uninstall OpenJDK (if installed). First check which OpenJDK packages are installed.</p> 
              <pre><code>sudo dpkg --list | grep -i jdk
</code></pre> 
              <p>To remove openjdk:</p> 
              <pre><code>sudo apt-get purge openjdk*
</code></pre> 
              <p>Uninstall OpenJDK related packages.</p> 
              <pre><code>sudo apt-get purge icedtea-* openjdk-*
</code></pre> 
              <p>Check that all OpenJDK packages have been removed.</p> 
              <pre><code>sudo dpkg --list | grep -i jdk
</code></pre> 
              <p><sup><sup><strong>1</strong></sup>Source:<a href="http://akbarahmed.com/2012/06/24/uninstall-java-from-ubuntu-linux/">akbarahmed.com</a></sup></p>

              <p>Try this command:</p> 
              <pre><code>java -version
</code></pre> 
              <p>If 1.6* comes then try:</p> 
              <pre><code>sudo apt-get autoremove openjdk-6-jre
</code></pre> 
              <p>If 1.7* comes then try:</p> 
              <pre><code>sudo apt-get autoremove openjdk-7-jre
</code></pre> 
              <p>Assuming that you don't have jdk in your system. If you have use this command:</p> 
              <pre><code>sudo apt-get autoremove openjdk-`&lt;version&gt;`-jdk
</code></pre> 
              <p>replace with the version like we did it in previous example.</p>

              <p>Assuming you have previously installed a copy of java-6-sun package, a followup will help getting rid of broken symbolic link:</p> 
              <pre><code>sudo update-alternatives --config java
</code></pre> 
              <p>And then select a correct path which you want to link up as default Java calling path.</p> 
              <p>Then have a check on the version of Java with this command:</p> 
              <pre><code>java -version
</code></pre>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Just remove all files of JDK or JRE, for example it's usually installed in this location: </p> 
              <p><code>/usr/lib/jvm/java-7-oracle</code></p> 
              <p>So remove all files resides in "java-7-oracle" folder with root permission and extract the latest JDK or JRE files in it. That's it now you would have the latest Java version Installed.</p> 
              <p>P.S. Your directory name may differ from <strong>java-7-oracle</strong>.</p> 
              <p>Reference: <a href="https://www.java.com/en/download/help/linux_x64_install.xml" rel="nofollow">Official installation instruction from Oracle</a></p>
