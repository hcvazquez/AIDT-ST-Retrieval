   <h2><a name="Package_Overview"></a>Overview</h2> 
   <p>IDAutomation's Java Barcode Packages are mature barcode generator components that have been in use since the year 2000. Multiple capabilities of this product are provided in a single JAR file to ease installation, use and upgrades. All IDAutomation Java Barcode Packages include the following components:</p> 
   <ol> 
    <li><b>Barcode class library</b> - includes all necessary class files for Java barcode generation.</li> 
    <li><b>Applet</b> - allows easy <a href="#Java_Barcode_Applet">applet implementation</a>.</li> 
    <li><b>Servlet</b> - enables <a href="#Java_Barcode_Servlet">servlet</a> use with applet <a href="#Properties">parameters</a>.</li> 
    <li><b>Encoder</b> - used to convert java.awt images obtained from the barcode class into JPEG or GIF files.</li> 
    <li><b>Source code</b> - includes source code to the applet, servlet, encoder and examples. The source code for the Java barcode generator class should not be needed; however, it is available with the purchase of the <a href="http://www.idautomation.com/barcode-components/java/#Licensing">Unlimited Developer License</a> and a signed <a href="http://www.idautomation.com/licensing/agreements/source-code.html">Source Code License Agreement</a>. </li> 
   </ol> 
   <h2><a name="Installation"></a>Installation</h2> 
   <p>This barcode Java library is easily utilized after the JAR file is installed in the CLASSPATH environment variable. There are some exceptions, such as when integrating source code or class files directly. However, for most purposes, the JAR file must be installed and added to the CLASSPATH environment variable.</p> 
   <h4>Installing &amp; <a name="Testing_Barcode_JAR"></a>Testing the Barcode JAR File:</h4> 
   <p>The following example uses the Linear Java Barcode Package. When using other packages, the associated file and its Classpath should be substituted accordingly.</p> 
   <ol> 
    <li>Install Java and ensure the Java bin files are included in the "Path" environment variable. For example: after installing Java on Windows, <span style="color: #008000; font-family:'Courier New', Courier, monospace">C:\Program Files (x86)\Java\jre1.8.0_66\bin:\jdk1.3\bin</span> is automatically appended to the end of the <span style="color: #008000; font-family:'Courier New', Courier, monospace">path</span> environment variable by the installer. In Windows®, the path environment is created or edited by choosing My Computer - Properties - Advanced System Settings - Environment Variables. </li> 
    <li>Copy <span style="color: #008000; font-family:'Courier New', Courier, monospace">LinearBarCode.jar</span> from the package to C:\IDAutomation or a production directory of choice. </li> 
    <li>Add both the JAR file and the period to the CLASSPATH environment variable. The specification of the JAR file within the Classpath enables the barcode library to be available for Java programs. The period allows Java applications to run in the current directory, which is convenient for testing. In Windows, the CLASSPATH environment variable may be created or edited by choosing My Computer - Properties - Advanced System Settings - Environment Variables. In this example the text <span class="auto-style4">;C:\IDAutomationJava\LinearBarCode.jar;.</span> is appended to the end of the CLASSPATH.<br> <img class="whitespace" alt="Java Barcode JAR in the Classpath" height="235" src="java-jar-in-classpath.png" width="376"><br>The "SET CLASSPATH" command may be used from the command prompt to verify the CLASSPATH variable. </li> 
    <li>Download <a href="IDAutomation_LinearJavaEncoderTestApplication.zip">IDAutomation_LinearJavaEncoderTestApplication.zip</a> and extract the files into a temp directory such as c:\temp. From the temp files extracted, execute the CreateJPEG class file in Java; Windows users may simply execute run.bat instead. When asked, enter the data of "IDAUTOMATION" and verify a JPEG file is created.</li> 
    <li>If problems are encountered, verify the above steps were completed properly and ensure the Java.awt GUI functions are available. If the Java environment has the java.awt library stripped out or the system running Java does not have an active GUI environment, either (1) use Headless Java or (2) connect to a remote X Server or (3) install the X Windows environment and run an X Windows session or (4) install an emulator, PJA library or virtual frame buffer as described in the <a href="http://www.idautomation.com/kb/xwindow-error.html">XWindow Error KB Document</a>. </li> 
   </ol> 
   <p>If a barcode scanner is needed, IDAutomation provides several types of <a href="http://www.idautomation.com/scanners/">high-quality USB Barcode Scanners</a>. </p> 
   <h2><a name="Java_Barcode_Applet"></a>Applet Implementation</h2> 
   <p>Java Applets are included with all IDAutomation JAR files in the <a href="http://www.idautomation.com/barcode-components/java/">Java Barcode Packages</a>. An applet may be used to easily display barcodes on WebPages with <a href="http://www.idautomation.com/kb/applet-probs.html">compatible Web browsers</a>. An example of using the applet in HTML is provided in the package download. Properties are modified by changing the appropriate <a href="#Properties">Applet Parameter</a>.</p> 
   <div id="main"> 
    <p class="rollup"><strong>HTML Code Example:</strong></p> 
    <div class="answer"> 
     <blockquote> 
      <pre class="brush: java; gutter: false;">&lt;APPLET CODEBASE = "./" ARCHIVE = "LinearBarCode.jar" CODE = "com.idautomation.linear.BCApplet.class"NAME = 
"TestApplet" WIDTH = 500 HEIGHT = 200 HSPACE = 0 VSPACE = 0 ALIGN = middle &gt; 
    &lt;PARAM NAME = "BARCODE" VALUE = "123456789012"&gt; 
    &lt;PARAM NAME = "CODE_TYPE" VALUE = "CODE39"&gt; 
    &lt;PARAM NAME = "LEFT_MARGIN" VALUE = "1"&gt;
    &lt;PARAM NAME = "TOP_MARGIN" VALUE = "1"&gt;
    &lt;PARAM NAME = "FONT_COLOR" VALUE = "BLUE"&gt;
    &lt;PARAM NAME = "TEXT_FONT" VALUE = "ARIAL|BOLD|14"&gt; 
    &lt;PARAM NAME = "CHECK_CHAR" VALUE = "N"&gt; 
    &lt;PARAM NAME = "BAR_HEIGHT" VALUE = "2"&gt; 
&lt;/APPLET&gt; </pre>
     </blockquote> 
    </div> 
    <p>The size must be specified when using the applet. Parameters may be provided in the Applet PARAM tag or from<b> Javascript</b>. For example, the following code will set a new value for the barcode:</p> 
    <blockquote> 
     <pre class="brush: java; gutter: false;">TestApplet.setParameter(BARCODE, "new value");
TestApplet.refresh();</pre>
    </blockquote> 
   </div>

    <p class="rollup"><a name="Pre-Loading"></a><strong>Applet Pre-Loading</strong></p>

     <blockquote> 
      <pre class="brush: java; gutter: false;">&lt;APPLET CODE = "com.idautomation.linear.BCApplet"

NAME = "PreloadApplet"

&lt;/APPLET&gt; </pre> 
     </blockquote> 
    </div>

   <h5>Inserting Java Barcodes in HTML:</h5> 
   <p>&nbsp;IDAutomation's Java Servlets generate <a href="http://www.idautomation.com/barcode-faq/internet-barcode-faq.html#Streaming_Barcode_Products"> streaming barcodes</a> which may be <a href="http://www.idautomation.com/hosted-saas-barcode-generator/streaming-integration/"> easily utilized within several applications</a>.    <h5>Auto-sizing and Performance Considerations:</h5> 
   <p>When using the Servlet, auto sizing of the image is disabled if the HEIGHT and WIDTH are manually specified. Because a temp image is created to determine the size, performance is improved by specifying the size.</p> 
   <p>If a barcode scanner is needed, IDAutomation provides several types of <a href="http://www.idautomation.com/scanners/">high-quality USB Barcode Scanners</a>. </p> 
   <h2><a name="java.awt.Image"></a>java.awt.Image Object Implementation</h2> 
   <p>The following code illustrates how to create a barcode in a java.awt.Image object:</p> 
   <blockquote> 
    <pre class="brush: java; gutter: false;">
BarCode bc = new BarCode();
bc.setSize(400, 200);
// create image
java.awt.image.BufferedImage
image = new java.awt.image.BufferedImage(
    bc.getSize().width, bc.getSize()
    .height, java.awt.image.BufferedImage.TYPE_BYTE_INDEXED
);
// get graphic context of image
java.awt.Graphics imgGraphics = image.createGraphics();
// paint barcode in graphics 
context of image
bc.paint(imgGraphics);
</pre> 
   </blockquote>
