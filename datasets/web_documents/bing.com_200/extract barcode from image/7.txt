       <div style="color:#000080;font-family:Arial;font-size:10pt">
         The 
        <b>String  </b>class provides a number of ways in which characters can be extracted from a 
        <b>String </b>object. Each is examined here. Although the characters that comprise a string within a 
        <b>String  </b>object cannot be indexed as if they were a character array, many of the 
        <b>String </b>methods employ an index (or offset) into the string for their operation. Like arrays, the string indexes begin at zero. 
        <p> <b> charAt( ) </b> </p> 
        <p> To extract a single character from a <b>String</b>, you can refer directly to an individual character via the <b>charAt( ) </b>method. It has this general form:</p> 
        <p> char charAt(int <i>where</i>)</p> 
        <p> Here, <i>where </i>is the index of the character that you want to obtain. The value of <i>where </i> must be nonnegative and specify a location within the string. <b>charAt( ) </b>returns the character at the specified location. For example,</p> 
        <p> char ch; <br>ch = "abc".charAt(1); <br>assigns the v0alue "<b>b</b>" to <b>ch</b>.</p> 
        <p> <b> getChars( ) </b> </p> 
        <p> If you need to extract more than one character at a time, you can use the <b>getChars( ) </b> method. It has this general form:</p> 
        <p> void getChars(int <i>sourceStart</i>, int <i>sourceEnd</i>, char <i>target</i>[ ], int <i>targetStart</i>)</p> 
        <p> Here, <i>sourceStart </i>specifies the index of the beginning of the substring, and <i>sourceEnd </i> specifies an index that is one past the end of the desired substring. Thus, the substring contains the characters from <i>sourceStart </i>through <i>sourceEnd</i>â??1. The array that will receive the characters is specified by <i>target. </i>The index within <i>target </i>at which the substring will be copied is passed in <i>targetStart. </i>Care must be taken to assure that the <i> target </i>array is large enough to hold the number of characters in the specified substring. The following program demonstrates <b>getChars( )</b>:</p> 
        <p> class getCharsDemo { <br>public static void main(String args[]) { <br>String s = "This is a demo of the getChars method."; <br>int start = 10; <br>int end = 14; <br>char buf[] = new char[end - start]; <br>s.getChars(start, end, buf, 0); <br>System.out.println(buf); <br>} <br>}</p> 
        <p> Here is the output of this program:</p> 
        <p> demo</p> 
        <p> <b> getBytes( ) </b> </p> 
        <p> There is an alternative to <b>getChars( ) </b>that stores the characters in an array of bytes. This method is called <b>getBytes( )</b>, and it uses the default character-to-byte conversions provided by the platform. Here is its simplest form:</p> 
        <p> byte[ ] getBytes( )</p> 
        <p> Other forms of <b>getBytes( ) </b>are also available. <b>getBytes( ) </b>is most useful when you are exporting a <b>String </b>value into an environment that does not support 16-bit Unicode characters. For example, most Internet protocols and text file formats use 8-bit ASCII for all text interchange.</p> 
        <p> <b> toCharArray( ) </b> </p> 
        <p> If you want to convert all the characters in a <b>String </b>object into a character array, the easiest way is to call <b>toCharArray( )</b>. It returns an array of characters for the entire string. It has this general form:</p> 
        <p> char[ ] toCharArray( )</p> 
        <p> This function is provided as a convenience, since it is possible to use <b>getChars( ) </b>to achieve the same result.</p>
