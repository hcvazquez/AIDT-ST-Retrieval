         <p align="justify">Speech synthesis and recognition were both introduced in .NET 3.0. They both live in <strong>System.Speech.dll</strong>.          <p align="justify">.NET has in fact two APIs for that:</p> 
         <ul> 
          <li>

            <strong>System.Speech</strong>, which comes with the core framework;

           </div> </li> 
         </ul>

         <p align="justify"><strong>SpeechRecognition</strong> implements <a href="http://msdn.microsoft.com/en-us/library/system.web.ui.icallbackeventhandler%28v=vs.110%29.aspx" target="_blank">ICallbackEventHandler</a> for a self-contained AJAX experience; it registers a couple of JavaScript functions and also an embedded JavaScript file, for some useful sound manipulation and conversion. Only one instance is allowed on a page. On the client-side, this JavaScript uses <a href="http://w3c.github.io/mediacapture-main/getusermedia.html" target="_blank">getUserMedia </a>to access an audio source and uses a clever mechanism to pack them as a <a href="http://en.wikipedia.org/wiki/WAV" target="_blank">.WAV</a> file in a <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/data_URIs" target="_blank">Data URI</a>. I got these functions from <a title="http://typedarray.org/from-microphone-to-wav-with-getusermedia-and-web-audio/" href="http://typedarray.org/from-microphone-to-wav-with-getusermedia-and-web-audio/">http://typedarray.org/from-microphone-to-wav-with-getusermedia-and-web-audio/</a> and made some changes to them. I like them because they don’t require any external library, which makes all this pretty much self-contained.</p> 
         <p align="justify">The control exposes some custom properties:</p> 
         <ul> 
          <li> 
           <div align="justify">
            <strong>Culture</strong>: an optional culture name, such as “
            <strong>pt-PT</strong>” or “
            <strong>en-US</strong>”; if not specified, it defaults to the current culture in the server machine;
           </div> </li> 
          <li> 
           <div align="justify">
            <strong>Mode</strong>: one of the two providers: 
            <strong>Desktop</strong> (for 
            <strong>System.Speech</strong>) or 
            <strong>Server</strong> (for 
            <strong>Microsoft.Speech</strong>, of Microsoft Speech Platform);
           </div> </li> 
          <li> 
           <div align="justify">
            <strong>OnClientSpeechRecognized</strong>: the name of a callback JavaScript version that will be called when there are results (more on this later);
           </div> </li> 
          <li> 
           <div align="justify">
            <strong>SampleRate</strong>: a sample rate, by default, it is 44100;
           </div> </li> 
          <li> 
           <div align="justify">
            <strong>Grammars</strong>: an optional collection of additional grammar files, with extension 
            <strong>.grxml</strong> (
            <a href="http://www.w3.org/TR/speech-grammar/" target="_blank">Speech Recognition Grammar Specification</a>), to add to the engine;
           </div> </li> 
          <li> 
           <div align="justify">
            <strong>Choices</strong>: an optional collection of choices to recognize, if we want to restrict the scope, such as “
            <strong>yes</strong>”/”
            <strong>no</strong>”, “
            <strong>red</strong>”/”
            <strong>green</strong>”, etc.
           </div> </li> 
         </ul>

         <p align="justify">And that’s it. You start recognition by calling <strong>startRecording()</strong>, get results in <strong>onSpeechRecognized()</strong> (or any other function set in the <strong>OnClientSpeechRecognized</strong> property) and stop recording with <strong>stopRecording()</strong>. The values passed to <strong>onSpeechRecognized()</strong> are those that may have been filtered by the server-side <strong>SpeechRecognized</strong> event handler.</p> 
         <p align="justify">A final word of advisory: because generated sound files may become very large, do keep the recordings as short as possible.</p>
