        <h1> <a href="/java-ee/web-services/java-web-services-binary-data-transfer-example-base64-encoding"> Java Web Services Binary Data Transfer Example (base64 encoding)</a></h1>
       </div>
       <div class="article-info muted">
        <dl class="article-info">
         <dt class="article-info-term">
          Details
         </dt>
         <dd class="modified"> 
          <span class="icon-calendar"></span> Last Updated on 21 July 2015 &nbsp; | &nbsp; 
          <a href="/java-ee/web-services/java-web-services-binary-data-transfer-example-base64-encoding?tmpl=component&amp;print=1&amp;page=" title="Print" onclick="window.open(this.href,'win2','status=no,toolbar=no,scrollbars=yes,titlebar=no,menubar=no,resizable=yes,width=640,height=480,directories=no,location=no'); return false;" rel="nofollow"><span class="icon-print"></span>&nbsp;Print&nbsp;</a> 
          <a href="/component/mailto/?tmpl=component&amp;template=protostar&amp;link=0cc04046339233b892e981f86a13d2385042831c" title="Email" onclick="window.open(this.href,'win2','width=400,height=350,menubar=yes,resizable=yes'); return false;"><span class="icon-envelope"></span> Email</a>
         </dd>
        </dl>
       </div>
       <div>
        <a href="https://click.linksynergy.com/link?id=d1Fh24*EiQI&amp;offerid=323058.1344876&amp;type=2&amp;murl=https%3A%2F%2Fwww.udemy.com%2Fwhats-new-in-java-9%2F?couponCode=OPENING&amp;u1=Java9" target="_blank" rel="nofllow"><b><font size="+1" color="red">NEW COURSE: </font><font size="+1" color="black">What's New In Java 9</font></b></a>
       </div>In this article, we are going to provide an example of a web services-based application that transfers binary data (e.g. content of an arbitrary file) using base64 encoding/decoding approach. The web services library to be used is 
       <a href="https://jax-ws.java.net" target="_blank">JAX-WS</a> (
       <i>Java API for XML Web Services</i>) which is a built-in technology in Java EE family and is also available in Java SE 6 or later.
       <p></p> &nbsp;
       <p></p>
       <h1>1. Background</h1>
       <p style="margin-left: 0in;">In the approach employed by this article, the binary data is embedded directly in the SOAP envelop using base64 text encoding. In other words, the raw binary data is converted to an encoded String which is value of an XML element in the SOAP message. Upon receiving the SOAP message, the receiver decodes the encoded String in order to re-construct the original binary data.</p>
       <p style="margin-left: 0in;">The following picture depicts this process:</p> 
       <img style="display: block; margin-left: auto; margin-right: auto;" src="/images/articles/javaee/webservices/binary-transfer-base64/SOAP%20with%20binary%20data%20process.png" alt="SOAP with binary data process" width="460" height="122">
       <p></p>
       <p style="margin-left: 0in;">This approach is the simplest way and is only suitable for transferring a small amount of binary data. It becomes very inefficient when transferring a large amount of binary data because the base64 text encoding technique bloats the data by a factor of 1.33x (UTF-8 text encoding) or 2.66x (UTF-16 text encoding) of the original size. Also, the encoding/decoding process slows down the application performance.</p> This approach is also often referred as “by value” or “inline attachment” method. Now, let’s go through an example application that is used to transfer small binary files.
       <p></p> 
       <strong><span style="font-size: 12pt; line-height: 115%;">NOTE:</span></strong> To optimize the binary data transfer, see: 
       <a href="/java-ee/web-services/using-mtom-to-optimize-binary-data-transfer-with-jax-ws-web-services" target="_blank">Using MTOM to optimize binary data transfer with JAX-WS web services</a>.
       <p></p>
       <h1><br>2. Coding the Web Service Endpoint Interface</h1> Let’s define an endpoint interface as follows:
       <p></p>
       <pre class="brush:java;toolbar:false">package net.codejava.ws.binary.server;

import javax.jws.WebMethod;
import javax.jws.WebService;

/**
 * A web service endpoint interface.
 * @author www.codejava.net
 *
 */
@WebService
public interface FileTransferer {
	@WebMethod
	public void upload(String fileName, byte[] imageBytes);
	
	@WebMethod
	public byte[] download(String fileName);	
} </pre>This endpoint defines two web methods, 
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">upload()</span> and 
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">download()</span> which allows the client to upload/download a file by sending/receiving a chunk of bytes.
       <p></p>
       <h1><br>3. Coding Web Service Endpoint Implementation</h1> Write an implementation for the 
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">FileTransferer</span> interface as follows:
       <p></p>
       <pre class="brush:java;toolbar:false">package net.codejava.ws.binary.server;

import java.io.BufferedInputStream;
import java.io.BufferedOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

import javax.jws.WebMethod;
import javax.jws.WebService;
import javax.xml.ws.WebServiceException;

/**
 * A web service implementation of an endpoint interface.
 * @author www.codejava.net
 *
 */
@WebService
public class FileTransfererImpl implements FileTransferer {
	@WebMethod
	public void upload(String fileName, byte[] imageBytes) {
		
		String filePath = "e:/Test/Server/Upload/" + fileName;
		
		try {
			FileOutputStream fos = new FileOutputStream(filePath);
			BufferedOutputStream outputStream = new BufferedOutputStream(fos);
			outputStream.write(imageBytes);
			outputStream.close();
			
			System.out.println("Received file: " + filePath);
			
		} catch (IOException ex) {
			System.err.println(ex);
			throw new WebServiceException(ex);
		}
	}
	
	@WebMethod
	public byte[] download(String fileName) {
		String filePath = "e:/Test/Server/Download/" + fileName;
		System.out.println("Sending file: " + filePath);
		
		try {
			File file = new File(filePath);
			FileInputStream fis = new FileInputStream(file);
			BufferedInputStream inputStream = new BufferedInputStream(fis);
			byte[] fileBytes = new byte[(int) file.length()];
			inputStream.read(fileBytes);
			inputStream.close();
			
			return fileBytes;
		} catch (IOException ex) {
			System.err.println(ex);
			throw new WebServiceException(ex);
		}		
	}
} </pre>As we can see, the
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;"> upload()</span> method saves the received bytes array to file specified by the given 
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">fileName</span>; and the 
       <span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">download()</span> method reads the requested file and returns its content as a bytes array.
       <p></p>
       <h1><br>4. Coding the Server Program</h1>
       <div>
        <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script> 
        <!-- CJ Inside Content Rectangle --> 
        <ins class="adsbygoogle" style="display:inline-block;width:336px;height:280px" data-ad-client="ca-pub-8129714482020982" data-ad-slot="4442193350"></ins> 
        <script>(adsbygoogle = window.adsbygoogle || []).push({});</script>
       </div>Let’s create a server program that publishes the above endpoint implementation as follows:
       <p></p>
       <pre class="brush:java;toolbar:false">package net.codejava.ws.binary.server;

import javax.xml.ws.Endpoint;

/**
 * A simple web service server.
 * @author www.codejava.net
 *
 */
public class WebServiceServer {

	public static void main(String[] args) {
		String bindingURI = "http://localhost:9898/codejava/fileService";
		FileTransferer service = new FileTransfererImpl();
		Endpoint.publish(bindingURI, service);
		System.out.println("Server started at: " + bindingURI);
	}
} </pre>Now run this program and we should see the following message in the console.
       <p></p>
       <p style="text-align: center;" align="center"><span style="font-size: 9pt; line-height: 115%; font-family: 'Courier New';">Server started at: http://localhost:9898/codejava/fileService</span></p> The server is now waiting for request. Next, we are going to generate the web service client code.
       <p></p> &nbsp;
       <p></p>
       <h1>5. Generating Client Code</h1> It’s recommended to use the 
       <span style="color: #800000;"><strong><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">wsimport</span></strong></span>
       <span style="font-size: 10pt; line-height: 115%;"> </span>tool in order to generate web services client code in Java. Execute the following command at the command prompt:
       <p></p>
       <p style="text-align: center;" align="center"><span style="color: #800000;"><strong><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">wsimport -keep -p net.codejava.ws.binary.client http://localhost:9898/codejava/fileService?wsdl</span></strong></span></p> The 
       <strong><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">wsimport</span></strong>
       <span style="font-size: 10pt; line-height: 115%;"> </span>tool generates necessary client code and puts .java source files under the package 
       <span style="font-size: 9pt; line-height: 115%; font-family: 'Courier New'; color: #800000;">net.codejava.ws.binary.client</span>. Here’s the list of generated files:
       <p></p>
       <ul>
        <li style="list-style-type: none;">
         <ul>
          <li><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">Download.java</span></li>
          <li><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">DownloadResponse.java</span></li>
          <li><span style="font-size: 10pt; line-height: 115%; font-family: 'Courier New';">FileTransfererImpl.java</span></li>

       <h1><br>6. Coding the Client Program</h1> Based on the generated web services client code, we can write a client program as follows:
       <p></p>
       <pre class="brush:java;toolbar:false">package net.codejava.ws.binary.client;

import java.io.BufferedInputStream;
import java.io.BufferedOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

/**
 * A client program that connects to a web service in order to upload
 * and download files.
 * @author www.codejava.net
 *
 */
public class WebServiceClient {

	public static void main(String[] args) {
		// connects to the web service
		FileTransfererImplService client = new FileTransfererImplService();
		FileTransfererImpl service = client.getFileTransfererImplPort();
		
		String fileName = "binary.png";
		String filePath = "e:/Test/Client/Upload/" + fileName;
		File file = new File(filePath);
		
		// uploads a file
		try {
			FileInputStream fis = new FileInputStream(file);
			BufferedInputStream inputStream = new BufferedInputStream(fis);
			byte[] imageBytes = new byte[(int) file.length()];
			inputStream.read(imageBytes);
			
			service.upload(file.getName(), imageBytes);

			inputStream.close();
			System.out.println("File uploaded: " + filePath);
		} catch (IOException ex) {
			System.err.println(ex);
		}		
		
		// downloads another file
		fileName = "camera.png";
		filePath = "E:/Test/Client/Download/" + fileName;
		byte[] fileBytes = service.download(fileName);
		
		try {
			FileOutputStream fos = new FileOutputStream(filePath);
			BufferedOutputStream outputStream = new BufferedOutputStream(fos);
			outputStream.write(fileBytes);
			outputStream.close();
			
			System.out.println("File downloaded: " + filePath);
		} catch (IOException ex) {
			System.err.println(ex);
		}
	}
} </pre>As we can see, this client program connects to the web service then uploads an image file to the service, and finally downloads another image file from the service.
