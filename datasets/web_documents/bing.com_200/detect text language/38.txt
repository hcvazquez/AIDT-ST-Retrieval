      <h1>guess-language 0.2</h1> 
      <div id="download-button"> 
       <a class="button green" style="float:right;" href="https://pypi.python.org/packages/e4/d0/df50192865e44af5c934504d413514158c5e522b00d8ec631d818b6d8b6b/guess-language-0.2.tar.gz">Download<br> <span style="font-size: 75%">guess-language-0.2.tar.gz</span> </a> 
      </div> 
      <p style="font-style: italic">Guess the natural language of a text</p> 
      <p>Attempts to determine the natural language of a selection of Unicode (utf-8) text.</p> 
      <p>Based on <a href="http://websvn.kde.org/branches/work/sonnet-refactoring/common/nlp/guesslanguage.cpp?view=markup" rel="nofollow">guesslanguage.cpp</a> by Jacob R Rideout for KDE which itself is based on <a href="http://languid.cantbedone.org/" rel="nofollow">Language::Guess</a> by Maciej Ceglowski.</p> 
      <p>Detects over 60 languages - all languages listed in the <a href="http://code.google.com/p/guess-language/source/browse/trunk/guess_language/trigrams/" rel="nofollow">trigrams</a> directory plus Japanese, Chinese, Korean and Greek.</p> 
      <p>guess_language uses heuristics based on the character set and trigrams in a sample text to detect the language. It works better with longer samples and will be confused if the sample text includes markup such as HTML tags.</p> 
      <div id="usage"> 
       <h2>Usage</h2> 
       <p>The main entry points all take a single string as input and return a language identifier. The string must be Unicode or UTF-8 text. The language identifer can be the language name in English, the two- or three-letter IANA language code, a language ID or a tuple containing all three codes.</p> 
       <p>The primary entry points, and the return values, are as follows:</p> 
       <pre>guessLanguage(txt) - IANA language code
guessLanguageTag(txt) - IANA language code (same as guessLanguage)
guessLanguageName(txt) - Language name in English
guessLanguageId(txt) - language ID
guessLanguageInfo(txt) - tuple of (IANA code, id, name)
</pre>
