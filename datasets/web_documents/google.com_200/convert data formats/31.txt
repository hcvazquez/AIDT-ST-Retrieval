     <section class="entry cf" itemprop="description articleBody"> 
      <p>Date manipulation is one of those development hurdles that we all must jump at one point in our career, and great tools go a long way in taming the beast.</p> 
      <p>The Moment.js library is <em>that</em> tool for JavaScript developers.</p> 
      <p>Since discovering Moment.js, an open source project, I’ve used it religiously to simplify validation, parsing and manipulation of dates on the client-side.</p> 
      <p>In this tutorial, I’ll help you get up and running with this ultra-useful JavaScript date library.</p> 
      <h3>What is Moment.js?</h3> 
      <p><a href="http://momentjs.com/" target="_blank">Moment.js</a> is a free and open source JavaScript library that removes the need to use the native JavaScript <code>Date</code> object directly. The library is a wrapper for the <code>Date</code> object (in the same way that jQuery is a wrapper for JavaScript) making the object a whole lot easier to work with.</p> 
      <p>The problem with JavaScript’s <code>Date</code> object isn’t the functionality it brings; it’s just extremely cumbersome to use as a developer. If you want to do complex parsing, validation and displaying of dates, you’ll end up writing a lot of code.</p> 
      <p>Moment.js also extends native JavaScript date capabilities with a variety of features, such as relative time, calendar time, durations and multi-language support. It has a seemingly endless list of plugins that allow for additional features like time-zone support, recurrence and Twitter integration.</p> 
      <h3>Tutorial Information</h3> 
      <p>In this tutorial, we’re going to take a look at Moment.js. The following are the things we’ll tackle.</p> 
      <ul> 
       <li>Referencing Moment.js and displaying basic date outputs</li> 
       <li>Displaying formatted dates</li> 
       <li>Parsing dates</li> 
       <li>Detecting invalid dates</li> 
       <li>Correcting invalid dates</li> 
       <li>Adding and subtracting dates</li> 
      </ul> 
      <p>There are demo pages for almost all the sections of this tutorial that demonstrate the code being discussed.</p> 
      <p>Alternatively, you can download all the demo files associated with this tutorial as a ZIP archive.</p> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/momentjs-tutorial-demos-v1-1.zip" download="momentjs-tutorial-demos.zip">Download All Demo Files</a></p> 
      <p>Let’s dive in.</p> 
      <h3>Referencing Moment.js and Displaying Basic Outputs</h3> 
      <p>To get started, you’ll need to <a href="http://momentjs.com/downloads/moment.js" target="_blank">download</a> Moment.js and reference it in your HTML document. You can also find the library at <a href="http://cdnjs.com/" target="_blank">cdnjs.com</a> (a public CDN for open source projects). Just search for the library using the term <em>"moment.js"</em>.</p> 
      <p>The first thing we’re going to do in this tutorial is to reference the Moment.js library. </p> 
      <p>Then, to make sure it’s working properly, we’re going to instantiate a <code>moment</code> object and a JavaScript <code>Date</code> object, and then output their values.</p> 
      <p>The following shows you how to reference the Moment.js library in your HTML documents, as well as how to instantiate and display the values of the <code>moment</code> object and the JavaScript <code>Date</code> object.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;">
&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
  <strong>&lt;!-- reference Moment.js library --&gt;</strong>
  &lt;script src="//cdnjs.cloudflare.com/ajax/libs/moment.js/2.7.0/moment.min.js" type="text/javascript"&gt;&lt;/script&gt;
&lt;/head&gt;
&lt;body&gt;

&lt;h2&gt;Moment Date&lt;/h2&gt;<strong>
&lt;!-- container for Moment.js output --&gt;</strong>
&lt;div id="displayMoment"&gt;&lt;/div&gt;

&lt;h2&gt;JavaScript Date&lt;/h2&gt;
<strong>&lt;!-- container for JavaScript Date output --&gt;</strong>
&lt;div id="displayJsDate"&gt;&lt;/div&gt;

&lt;script type="text/javascript"&gt;
(function()
{
  <strong>// instantiate a moment object</strong>
  var NowMoment = moment();
  
  <strong>// instantiate a JavaScript Date object</strong>
  var NowDate = new Date();
  
  <strong>// display value of moment object in #displayMoment div</strong>
  var eDisplayMoment = document.getElementById('displayMoment');
  eDisplayMoment.innerHTML = NowMoment;
  
  <strong>// display value of Date object in #displayJsDate div</strong>
  var eDisplayDate = document.getElementById('displayJsDate');
  eDisplayDate.innerHTML = NowDate;
})();
&lt;/script&gt;

&lt;/body&gt;
&lt;/html&gt;
</pre> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/demo01.html" target="_blank">View Demo 1</a></p> 
      <p>At this point, the code-lengths of both the <code>moment</code> and native <code>Date</code> object look on par.</p> 
      <h3>Displaying Formatted Dates</h3> 
      <p>Things between <code>moment</code> and JavaScript <code>Date</code> quickly change, however, when we want to output even a simple, human-readable date display with the format of ‘Y-m-d’ (e.g. ‘2014-7-1’).</p> 
      <p>Using JavaScript <code>Date</code>, this would be our code.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;"><strong>// use JavaScript Date to display current date in a div (#displayJsDate)</strong>
var NowDate = new Date();
var months = ['January', 'February', 'March', 'April', 'May', 'June', 'July', 'August', 'September', 'October', 'November', 'December'];
var sDay = NowDate.getDate();
var sMonth = NowDate.getMonth() + 1;
var sYear = NowDate.getFullYear();
var eDisplayDate = document.getElementById('displayJsDate');
eDisplayDate.innerHTML = sYear+'-'+sMonth+'-'+sDay;
</pre> 
      <p>Moment.js makes our code terser.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;"><strong>// use Moment.js to display current date in a div (#displayMoment)</strong>
var NowMoment = moment();
var eDisplayMoment = document.getElementById('displayMoment');
eDisplayMoment.innerHTML = NowMoment.format('YYYY-M-D');
</pre> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/demo02.html" target="_blank">View Demo 2</a></p> 
      <p>You’ll notice we had to add a lot of set-up code for our specific date format output in the native solution, and only a simple method called <code>.format()</code> in the Moment solution.</p> 
      <p>The string argument that we passed into <code>.format()</code>, which is <code>'YYYY-M-D'</code>, is the format that we’re asking the Moment.js library to return the date in. The Moment.js website has many format options conveniently listed in a table here:</p> 
      <ul> 
       <li><a href="http://momentjs.com/docs/#/displaying/format/" target="_blank">http://momentjs.com/docs/#/displaying/format/</a></li> 
      </ul> 
      <p>Now that we’re familiar with the process of displaying and formatting the current date, let’s move on.</p> 
      <h3>Parsing Dates</h3> 
      <p>The Moment initializer will take a parameter and set the <code>Date</code> object to that date.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;">
&lt;h2&gt;Output Date&lt;/h2&gt;
&lt;div id="output"&gt;&lt;/div&gt;

&lt;script type="text/javascript"&gt;
(function()
{
  var Date = moment('July 1, 2014');
  var eDisplayMoment = document.getElementById('output');
  <strong>// display output in the preferred format</strong>
  eDisplayMoment.innerHTML = Date.format('YYYY-M-D');
})();
&lt;/script&gt;
</pre> 
      <p>One detail to note is there’s no standard that browsers adhere to when parsing dates. Fortunately, the Moment.js library provides additional parameters that allow us to specify the exact pattern we’re passing in to help us avoid cross-browser inconsistencies.</p> 
      <p>For consistency across all browsers and platforms, the above code should be parsed like the fowlloing.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;">
&lt;h2&gt;Input Date&lt;/h2&gt;
&lt;div id="input"&gt;July 1, 2014&lt;/div&gt;

&lt;h2&gt;Output Date&lt;/h2&gt;
&lt;div id="output"&gt;&lt;/div&gt;

&lt;script type="text/javascript"&gt;
(function()
{
  <strong>// get the input date from #input div element</strong>
  var input = document.getElementById('input').innerHTML;
  
  <strong>// use moment() with input value and a string format pattern as arguments</strong>
  var Date = moment(input, 'MMMM D, YYYY');
  
  <strong>// display the parsed date in a div (#output)</strong>
  var eDisplayMoment = document.getElementById('output');
  eDisplayMoment.innerHTML = Date.format('YYYY-M-D');
})();
&lt;/script&gt;
</pre> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/demo03.html" target="_blank">View Demo 3</a></p> 
      <p>The Moment.js website has another table displaying the available parameters that we can use for parsing:</p> 
      <ul> 
       <li><a href="http://momentjs.com/docs/#/parsing/string-format/" target="_blank">http://momentjs.com/docs/#/parsing/string-format/</a></li> 
      </ul> 
      <p>As the value we’re parsing is rarely entered by hand, it’s important to know that we’re actually parsing a date value.</p> 
      <p>Consider the input <code>'2014-14-14'</code> — this is an impossible date. In this case the <code>moment()</code> function will actually return the string <code>'Invalid Date'</code> if you try to parse it.</p> 
      <p>However, there are additional tools available to us for detecting and correcting an invalid date values.</p> 
      <h3>Detecting Invalid Dates</h3> 
      <p>To detect an invalid date, Moment.js provides us with the <code>isValid()</code> method. It returns a simple boolean value (<code>true</code> or <code>false</code>), enabling us to determine the next course of action in our code logic in case the date value is not a valid one.</p> 
      <p>The other detection method available is <code>invalidAt()</code>. This method is more specific, it will tell us <em>where</em> our date value is invalid. It will return an integer between <code>0</code> and <code>6</code>, the meanings of which can be found in the following table.</p> 
      <table width="550" border="0" cellspacing="0" cellpadding="0"> 
       <tbody>
        <tr> 
         <th scope="col">Integer</th> 
         <th scope="col">Error location</th> 
        </tr> 
        <tr> 
         <td>0</td> 
         <td>Years</td> 
        </tr> 
        <tr> 
         <td>1</td> 
         <td>Months</td> 
        </tr> 
        <tr> 
         <td>2</td> 
         <td>Days</td> 
        </tr> 
        <tr> 
         <td>3</td> 
         <td>Hours</td> 
        </tr> 
        <tr> 
         <td>4</td> 
         <td>Minutes</td> 
        </tr> 
        <tr> 
         <td>5</td> 
         <td>Seconds</td> 
        </tr> 
        <tr> 
         <td>6</td> 
         <td>Milliseconds</td> 
        </tr> 
       </tbody>
      </table> 
      <p>So, in the example of <code>'2014-14-14'</code>, the <code>invalidAt()</code> function would return <code>1</code> because 14 is an invalid month (only 1-12 are valid month values).</p> 
      <h3>Correcting Invalid Dates</h3> 
      <p>When would you ever be able to correct an invalid date? If you haven’t worked with dates at length before, this one might stump you. However, there are more variables at play than just the date itself: We must also consider how we’re getting the date and who is providing it.</p> 
      <p>Often, we ask people to enter dates and, despite our best efforts and flashy tooltips asking them to input in a specific date format like "d/m/y", they ignore our instructions and provide "m/d/y" anyways.</p> 
      <p>We can go back to the date-parsing call we made earlier and add a second parsing parameter (you can have more than two options) by passing in a string array of various possible date formats as a fallback.</p> 
      <p>Let’s use the date <em>January 1, 2014</em> as our input example. We’ll input <code>14/1/2014</code> and <code>1/14/2014</code> expecting an input of <code>D/M/YYYY</code>.</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;">
&lt;h2&gt;Input Dates&lt;/h2&gt;
&lt;div id="input1"&gt;14/1/2014&lt;/div&gt;
&lt;div id="input2"&gt;1/14/2014&lt;/div&gt;

&lt;h2&gt;Output Dates (1 format)&lt;/h2&gt;
&lt;div id="output1"&gt;&lt;/div&gt;
&lt;div id="output2"&gt;&lt;/div&gt;

&lt;h2&gt;Output (multiple formats)&lt;/h2&gt;
&lt;div id="output3"&gt;&lt;/div&gt;
&lt;div id="output4"&gt;&lt;/div&gt;

&lt;script type="text/javascript"&gt;
(function()
{
  var correct = document.getElementById('input1').innerHTML;
  var incorrect = document.getElementById('input2').innerHTML;
  
  var Date1 = moment(correct, 'D/M/YYYY');
  document.getElementById('output1').innerHTML = Date1.format('MMMM D, YYYY');
  
  var Date2 = moment(incorrect, 'D/M/YYYY');
  <strong>// will display 'Invalid Date' in #output2 because input format is incorrect</strong>
  document.getElementById('output2').innerHTML = Date2.format('MMMM D, YYYY');

  var Date3 = moment(correct, ['D/M/YYYY','M/D/YYYY']);
  document.getElementById('output3').innerHTML = Date3.format('MMMM D, YYYY');

  <strong>// will try to parse using two potential date formats</strong>
  var Date4 = moment(incorrect, ['D/M/YYYY','M/D/YYYY']);
  <strong>// will display correct date format</strong>
  document.getElementById('output4').innerHTML = Date4.format('MMMM D, YYYY');
})();
&lt;/script&gt;
</pre> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/demo04.html" target="_blank">View Demo 4</a></p> 
      <p>You’ll notice that with just one string argument, if the input doesn’t match, you will output <code>'Invalid Date'</code>.</p> 
      <p>However, if you provide an array of string formats as the argument, the first parsing option that works will be used. This is great for <a href="https://www.webpagefx.com/blog/web-design/poka-yoke-ui-design/" title="Mistake-proofing (Poka-Yoke) in User Interface Design">removing</a> or anticipating possible user errors.</p> 
      <h3>Adding and Subtracting Dates</h3> 
      <p>One of my favorite features in the Moment.js library is it conveniently has date-manipulation methods, such as <code>.add()</code> and <code>.subtract()</code>.</p> 
      <p>For example:</p> 
      <pre style="white-space:pre-wrap;word-wrap:break-word;"><strong>// current date</strong>
var Date = moment();

<strong>// add 7 days to current date</strong>
document.getElementById('currDatePlus7d').innerHTML = Date<strong>.add('days', 7);</strong><strong>// subtract 8 hours from current date</strong>
document.getElementById('currDateSub8h').innerHTML = Date<strong>.subtract('hours', 8);</strong><strong>// add 5 years and 3 months to current date</strong>
document.getElementById('currDatePlus8h').innerHTML = Date<strong>.add({years: 5, months: 3});</strong></pre> 
      <p><a style="display: inline-block; padding: 6px 10px; color: #ffffff; border-radius: 3px; text-decoration: none; text-align: center; text-transform: uppercase; background-color:#363636;" href="https://www.webpagefx.com/blog/images/assets/cdn.sixrevisions.com/0438-01_javascript-dates-moment-js/demo05.html" target="_blank">View Demo 5</a></p> 
      <p>The Moment.js library will handle edge-cases elegantly. For example, using the native JavaScript <code>Date</code> object, adding one month to August 31 will result in October 1. This same operation performed by Moment.js will correctly produce September 30.</p> 
      <h3>Conclusion</h3> 
      <p>Moment.js is an excellent tool to have if you regularly end up working with client-side date parsing, formatting and validation. It has plenty to offer in terms of speeding up JavaScript development in this area. To see the project’s source code, head over to its <a href="https://github.com/moment/moment/" target="_blank">GitHub repository</a>. To learn more about its features, check out the <a href="http://momentjs.com/docs/" target="_blank">official docs</a>.</p>
