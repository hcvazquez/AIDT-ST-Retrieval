      <span> <a href="/u/badita"><b itemprop="author">badita</b></a> <time datetime="2017-02-08T17:40:54Z" itemprop="datePublished"> 2017-02-08 17:40:54 UTC </time> </span>

     <div class="post" itemprop="articleBody"> 
      <p>Not an easy task, but attached is an experimental activity that transforms an image into a string (QRCode Reader).</p> 
      <p>Your task will be to take a screenshot of the QRCode from the PDF and save it as an image.<br>Pass that image to the below QRCode reader activity that transforms an image into a string.</p> 
      <p><a class="attachment" href="https://discourse-cdn-sjc1.com/business2/uploads/uipath/original/1X/7d80b17898678084845e12f90ef27503273496c4.xaml">Main.xaml</a> (5.0 KB)<br><a class="attachment" href="https://discourse-cdn-sjc1.com/business2/uploads/uipath/original/1X/da76ab4c032edd961dfad4e7f7ff79d0f6faeac2.nupkg">QRCodeActivities.1.0.0.nupkg</a> (980.1 KB)<br><a class="attachment" href="https://discourse-cdn-sjc1.com/business2/uploads/uipath/original/1X/1c4323518dbf63ebe74adab7b6abef01671761dd.zip">QRCodeReader.zip</a> (2.9 MB)</p> 
      <hr> 
      <p>You can construct your own activity which decodes QR Code from an image.</p> 
      <p>In Visual Studio<br>- create a new Class Library project<br>- rename the class Class1.cs created by default (or delete it and create a new class)<br>- add a reference to System.Activities and make the class mentioned above inherit from CodeActivity<br>- create an input argument and an output argument (both of type string) and also override the Execute method<br>- build your project</p> 
      <p>Finally, you need to follow the steps described here to build a NuGet package and make it available in UiPath Studio. Make sure that you have NuGet Package Explorer installed.</p> 
      <p>You can find in attachments the VS project archive and the NuGet package, as well as a workflow that uses a “Select File” activity and passes the file path to our QRCode Reader activity.</p> 
      <p>You can modify this activity to suit your needs (but you’ll have to rebuild and recreate the NuGet package). Maybe you want use another library which can decode both, barcodes and QRCodes. Or you can simply add an input value to tell you if it is QRCode or a barcode and based on that you can keep the MessagingToolkit.QRCode library and simply add another one for reading barcodes.</p>
