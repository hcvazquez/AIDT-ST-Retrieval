               <li class="libr"> <h5>What is Foxit PDF IFilter?</h5> Foxit PDF IFilter is an application designed to help users index a large amount of PDF documents and then quickly find text within these documents. It acts as a plug-in for full-text search engines that scans documents for text and properties (also called attributes), extracts text from documents, filters out formatting and retaining information about text position, and extracts properties of an entire document. </li> 
               <li class="libr"> <h5>What’s new in Foxit PDF IFilter 3.0?</h5> Foxit PDF IFilter 3.0 adds the following features:<br> 
                <ul>

                 <li>Improved the support for Right-to-Left languages, including Hebrew and Arabic.</li> 
                 <li>Fixed the "Unable to access object" error reported by SharePoint.</li> 
                 <li>Fixed the issue where Foxit PDF IFilter indexes multiple layers of duplicated text.</li> 
                </ul> </li>

               <li class="libr"> <h5>How can I get the margin size of a page in Foxit PDF SDK?</h5> Foxit PDF SDK provides applications to get the content box of a page by calling the API FSPDF_Page_CalcContentBBox. After the content box is returned by this API, the margin can be calculated by the following formula.<br> Margin_h = (page width – box width)/ 2<br> Margin_v = (page height – box height) / 2<br> Margin_h means the margin in horizontal direction.<br> Margin_v means the margin in vertical direction.<br> The details about how to call this API, please refer to Foxit PDF SDK API reference.<br> </li> 
               <li class="libr"> <h5>How to transform the coordinate between page coordinate system and device coordinate system?</h5> In PDF, when contents of a page are rendered on a device like a printer or display, their coordinates are determined by the device coordinate system, which could vary a lot. Developers have to transform the coordinates from a page to the device or vice versa. The transformation can be fully described by a transformation matrix (FSCRT_MATRIX). Foxit PDF SDK provides APIs to do the transformation as explained in the example below.<br> <img border="0" height="300" width="300" src="/resource/images/support/faq_pdfsdk1.png"> <img border="0" height="300" width="300" src="/resource/images/support/faq_pdfsdk2.png"><br> Figure 1 shows a page in the page coordinates and Figure2 shows the same page in device coordinates. The width and height of the page is 400 and 300. Assuming the lower left corner of the page A(0,0) corresponds to A’(100,100) in the device coordinate.<br> <strong style="text-decoration: underline;">PAGE TO DEVICE:</strong><br> First, obtain the page-to-device transformation matrix:<br> <strong>FSCRT_MATRIX matrix_p2d;</strong><br> <strong>FSPDF_Page_GetMatrix(page, 100, 100, 400, 300, 0, &amp;matrix_p2d);</strong><br> Once this matrix is obtained, we could call other transformation functions to get the coordinate in device coordinates. For example, for page coordinate B(50,50), to get the device coordinate B’:<br> <strong>FS_INT32 page_x = 50;</strong><br> <strong>FS_INT32 page_y = 50;</strong><br> <strong>FSPDF_Matrix_TransformPoint(&amp;matrix_p2d, &amp;page_x, &amp;page_y);</strong><br> For a rectangle C rect(100,200,200,100) in page coordinate, to get the rectangle C’ in device coordinate:<br> <strong>FSPDF_Matrix_TransformRect(&amp;matrix_p2d, &amp;rect);</strong><br> <strong style="text-decoretion:underline">DEVICE TO PAGE:</strong><br> There are occasions that we need to go from device to page coordinate. To do that, obtain the device-to-page transformation matrix: <br> <strong>FSCRT_MATRIX matrix_d2p;</strong><br> <strong>FSPDF_Page_GetReverse(&amp;matrix_p2d, &amp;matrix_d2p);</strong><br> To transform a point D(150,150) in the device coordinate to the page coordinate D’, we can do:<br> <strong>FS_INT32 device_x = 150;</strong><br> <strong>FS_INT32 device_y = 150;</strong><br> <strong>FSPDF_Matrix_TransformPoint(&amp;matrix_d2p, &amp;device_x, &amp;device_y);</strong><br> For a rectangle E’ rect(400,100,500,200) in device coordinate, to get the rectangle E in page coordinate:<br> <strong>FSPDF_Matrix_TransformRect(&amp;matrix_d2p, &amp;rect);</strong> </li> 
               <li class="libr"> <h5>Does Foxit PDF SDK support multi-thread?</h5> Currently, most applications support multi-threads. It requires PDF SDK to provide thread safe APIs to support applications. Since SDK 4.0, Foxit PDF SDK introduces thread safety mechanism to ensure all APIs are thread safety. It frees developers from multi-thread supporting details. Based on PDF SDK3.0, developers can concentrate on multi-thread applications and don’t need to worry about thread safety in PDF API level. </li> 
               <li class="libr"> <h5>How to save PDF attachment to a file/How to added attachment to PDF file?</h5> A PDF document may contain attachments. To access to an attachment, call function FSPDF_Doc_LoadAttachments to get a handle that receives an FSPDF_ATTACHMENTS object and call FSPDF_Attachments_GetAttachment to get a specific FSPDF_ATTACHMENT object by index. Alternatively, you can call FSPDF_Attachment_Create to create a new attachment and call FSPDF_Attachments_InsertAttachment to insert the attachment to the PDF document. Call FSPDF_Attachment_SetFile to associate the attachment to an existing file. Call FSPDF_Attachment_WriteToFile to save an attachment to a file.<br> There is a set of functions like FSPDF_Attachment_GetFileName, FSPDF_Attachment_GetDescription, etc. to get the property of the specific attachment. Correspondingly, use FSPDF_Attachment_SetFileName, FSPDF_Attachment_SetDescription, etc. to set the properties of an attachment.<br> After calling APIs related to attachment, make sure to call FSPDF_Attachments_Release and FSCRT_File_Release to release the resources of attachment and file. </li> 
               <li class="libr"> <h5>What’s ‘wrapper’ in Foxit PDF SDK? What can I use ‘wrapper’ for? How can I use the ‘wrapper’ functionality in my applications?</h5> The ‘wrapper’ is a newly introduced functionality in Foxit SDK . It provides a way for users to save their own data related to a PDF document. For example, when opening an encrypted unauthorized PDF document, users may get an error message. In this case, users can still access wrapper data even they do not have permissions to the PDF content. The wrapper data can be used to inform users that which applications are used to open PDF content or where users can get some useful information and so on. More information about how to use ‘wrapper’ in Foxit SDK, please refer to Foxit PDF SDK developer guide. </li> 
               <li class="libr"> <h5>How to use private data? What is module ID?</h5> PDF SDK provides developers the flexibility to use private data in their applications. There are three APIs: FSCRT_Library_GetPrivateData, FSCRT_Library_RemovePrivateData, FSCRT_Library_SetPrivateData to accomplish operations with their private data. The module ID is unique to each private data. These APIs provide the capability for customers to set or obtain global data or data from a specific thread.<br> Here is one scenario of using private data in PDF SDK. </li>
