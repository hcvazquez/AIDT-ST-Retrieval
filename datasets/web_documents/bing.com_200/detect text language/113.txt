       <h1 itemprop="name" class="devsite-page-title"> Load the Libraries </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p>This page shows how to load the Google Chart libraries.</p> 
        <h2><a name="basicloading"></a>Basic Library Loading</h2> 
        <p>With few exceptions, all web pages with Google Charts should include the following lines in the <code>&lt;head&gt;</code> of the web page:</p> 
        <pre class="prettyprint lang-html">
&lt;script type="text/javascript" <b>src="https://www.gstatic.com/charts/loader.js"</b>&gt;&lt;/script&gt;
&lt;script type="text/javascript"&gt;
  <b>google.charts.load('current', {packages: ['corechart']});</b>
  <b>google.charts.setOnLoadCallback(drawChart);</b>
  ...
&lt;/script&gt;
</pre> 
        <p>The first line of this example loads the loader itself. You can only load the loader one time no matter how many charts you plan to draw. After loading the loader, you can call the <code>google.charts.load</code> function one or more times to load packages for particular chart types. </p> 
        <p>The first argument to <code>google.charts.load</code> is the version name or number, as a string. If you specify <code>'current'</code>, this causes the latest official release of Google Charts to be loaded. If you want to try the candidate for the next release, use <code>'upcoming'</code> instead. In general there will be very little difference between the two, and they'll be completely identical except when a new release is underway. A common reason to use <code>upcoming</code> is that you want to test a new chart type or feature that Google is about to release in the next month or two. (We announce upcoming releases on our <a href="https://groups.google.com/group/google-visualization-api">forum</a> and recommend that you try them out when announced, to be sure that any changes to your charts are acceptable.)</p> 
        <p>The example above assumes you want to display a <code>corechart</code> (bar, column, line, area, stepped area, bubble, pie, donut, combo, candlestick, histogram, scatter). If you want a different or additional chart type, substitute or add the appropriate package name for <code>corechart</code> above (e.g., <code>{packages: ['corechart', 'table', 'sankey']}</code>. You can find the package name in the 'Loading' section of the documentation page for each chart.</p> 
        <p>This example also assumes that you have a JavaScript function named <code>drawChart</code> defined somewhere in your web page. You can name that function whatever you like, but be sure it is globally unique and that it is defined before you reference it in your call to <code>google.charts.setOnLoadCallback</code>. </p> 
        <p class="note"><b>Note:</b> Previous versions of Google Charts used code that differs from the above to load the libraries. To update your existing charts to use the new code, see <a href="#updateloader">Update Library Loader Code</a>. </p> 
        <p>Here is a complete example of drawing a pie chart using the basic loading technique:</p> 
        <pre class="prettyprint lang-html">
&lt;head&gt;
  &lt;script type="text/javascript" <b>src="https://www.gstatic.com/charts/loader.js"</b>&gt;&lt;/script&gt;
  &lt;script type="text/javascript"&gt;
    google.charts.load('current', {packages: ['corechart']});
    google.charts.setOnLoadCallback(drawChart);

    function drawChart() {
      // Define the chart to be drawn.
      var data = new google.visualization.DataTable();
      data.addColumn('string', 'Element');
      data.addColumn('number', 'Percentage');
      data.addRows([
        ['Nitrogen', 0.78],
        ['Oxygen', 0.21],
        ['Other', 0.01]
      ]);

      // Instantiate and draw the chart.
      var chart = new google.visualization.PieChart(document.getElementById('myPieChart'));
      chart.draw(data, null);
    }
  &lt;/script&gt;
&lt;/head&gt;
&lt;body&gt;
  &lt;!-- Identify where the chart should be drawn. --&gt;
  &lt;div id="myPieChart"/&gt;
&lt;/body&gt;
</pre> 
        <h2><a name="details" id="details"></a>Loading Details</h2> 
        <p> First you must load the loader itself, which is done in a separate <code>script</code> tag with <code>src="https://www.gstatic.com/charts/loader.js"</code>. This tag can be either in the <code>head</code> or <code>body</code> of the document, or it can be inserted dynamically into the document while it is being loaded or after loading is completed. </p> 
        <pre class="prettyprint lang-html">
&lt;script type="text/javascript" <b>src="https://www.gstatic.com/charts/loader.js"</b>&gt;&lt;/script&gt;
</pre> 
        <p> After the loader is loaded, you are free to call <code>google.charts.load</code>. Where you call it can be in a <code>script</code> tag in the <code>head</code> or <code>body</code> of the document, and you could call it either while the document is still loading or any time after it has finished loading. </p> 
        <p> As of version 45, you can call <code>google.charts.load</code> more than one time in order to load additional packages, but if you do so, you must provide the same version number and language setting each time. </p> 
        <div class="note"> 
         <h3><a name="limitations" id="limitations"></a>Limitations</h3> 
         <p>There are a couple minor but important limitations with how you load Google Charts in versions prior to 45:</p> 
         <ol> 
          <li>You can only call <code>google.charts.load</code> <i>once</i>. But you can list all the packages that you'll need in one call, so there's no need to make separate calls.</li> 
          <li>You can't <code>autoload</code> the library.</li> 
          <li>If you're using a ChartWrapper, you must explicitly load all the packages you'll need, rather than relying on the ChartWrapper to automatically load them for you.</li> 
          <li>For <a href="https://developers.google.com/chart/interactive/docs/gallery/geochart">Geochart</a> and <a href="https://developers.google.com/chart/interactive/docs/gallery/map">Map Chart</a>, you must load both the old library loader and the new library loader. <pre>
&lt;script type="text/javascript" <b>src="https://www.gstatic.com/charts/loader.js"</b>&gt;&lt;/script&gt;
&lt;script type="text/javascript" <b>src="https://www.google.com/jsapi"</b>&gt;&lt;/script&gt;</pre> </li> 
         </ol> 
        </div> 
        <h3><a name="version" id="version"></a>Load Version Name or Number</h3> 
        <p> The first argument of your <code>google.charts.load</code> call is the version name or number. There are only two special version names at this time, and several frozen versions. </p> 
        <dl> 
         <dt>
          current
         </dt> 
         <dd>
          This is for the latest official release which changes each time we push out a new release. This version is ideally well tested and bug free, but you may want to specify a particular frozen version instead once you are satisfied it is working.
         </dd> 
         <dt>
          upcoming
         </dt> 
         <dd>
          This is for the next release, while it is still being tested and before it becomes the official current release. Please test this version regularly so that you know as soon as possible whether there are any problems that should be addressed before this version becomse the official release.
         </dd> 
        </dl> 
        <p>When we release new versions of Google Charts, some of the changes are big, like entirely new chart types. Other changes are small, like enhancements to the appearance or behavior of existing charts.</p> 
        <p>Many Google Chart creators fine-tune the look and feel of their charts until it's exactly what they want. Some creators might feel more comfortable knowing that their charts will never change, regardless of what improvements we make in the future. For those users, we support <i>frozen</i> Google Charts.</p> 
        <p>Frozen chart versions are identified by number, and they're described in our <a href="https://developers.google.com/chart/interactive/docs/release_notes#Releases">Official Releases</a>. To load a frozen version, replace <code>current</code> or <code>upcoming</code> in your call of <code>google.charts.load</code> with the frozen version number:</p> 
        <pre class="prettyprint lang-html">
&lt;script type="text/javascript" src="https://www.gstatic.com/charts/loader.js"&gt;&lt;/script&gt;
&lt;script type="text/javascript"&gt;
    google.charts.load(<b>'43'</b>, {packages: ['corechart']});
    google.charts.setOnLoadCallback(drawChart);
    ...
&lt;/script&gt;
</pre> 
        <p>We expect that frozen versions will remain available indefinitely, though we may retire frozen versions that have security concerns. We will typically not provide support for frozen versions, except to unhelpfully suggest that you upgrade to a newer version.</p> 
        <h3><a name="settings" id="settings"></a>Load Settings</h3> 
        <p>The second parameter in your call of <code>google.charts.load</code> is an object for specifying settings. The following properties are supported for settings. </p> 
        <dl> 
         <dt>
          packages
         </dt> 
         <dd>
          An array of zero or more packages. Each package that is loaded will have the code required to support a set of functionality, typically a type of chart. The package or packages you need to load are listed in the documentation for each type of chart. You can avoid specifying any packages if you use a 
          <a href="https://developers.google.com/chart/interactive/docs/reference#chartwrapper-class">ChartWrapper</a> to automatically load what will be required. 
         </dd>
         <dt>
          language
         </dt> 
         <dd>
          The code for the language or locale that should be to customize text that might be part of the chart. See 
          <a href="#Locales>">Locales</a> for more details. 
         </dd>
         <dt>
          callback
         </dt> 
         <dd>
          A function that will be called once the packages have been loaded. Alternatively, you can specify this function by calling 
          <code>google.charts.setOnLoadCallback</code> as demonstrated in the example above. See 
          <a href="#Callback">Callback</a> for more details. 
          <pre class="prettyprint lang-html">
  google.charts.load('current', { packages: [ 'corechart'], <b>callback: drawChart</b> });
</pre> 
         </dd> 
         <dt>
          mapsApiKey
         </dt> 
         <dd>
          For version 45 and later, this setting lets you specify a key that you may use with 
          <a href="https://developers.google.com/chart/interactive/docs/gallery/geochart">Geochart</a> and 
          <a href="https://developers.google.com/chart/interactive/docs/gallery/map">Map Chart</a>. You may want to do this rather than use the default behavior which may result in occassional throttling of service for your users. Learn how to set up your own key for using the 'Google Maps JavaScript API' service here: 
          <a href="https://developers.google.com/maps/documentation/javascript/get-api-key"> Get a Key/Authentication</a>. Your code will look something like this: 
          <pre class="prettyprint lang-html">
  var myMapsApiKey = 'SomeMagicToSetThis';
  google.charts.load('45', { packages: [ 'geochart'], <b>mapsApiKey: myMapsApiKey</b>  });
</pre> 
         </dd> 
        </dl> 
        <h3 name="Locales" id="loadwithlocale">Locales</h3> 
        <p><i>Locales</i> are used to customize text for a country or language, affecting the formatting of values such as currencies, dates, and numbers.</p> 
        <p>By default, the Google Charts is loaded with the "en" locale. You can override this default by explicitly specifying a locale in the loading settings. </p> 
        <p>To load a chart formatted for a specific locale, use the <code>language</code> setting like so: </p> 
        <pre class="prettyprint lang-js">
  // Load Google Charts for the Japanese locale.
  google.charts.load('current', {'packages':['corechart'], 'language': 'ja'});
</pre> 
        <p><a href="https://developers.google.com/chart/interactive/docs/examples/specificlocale">Follow this link for a live example.</a></p> 
        <h3 name="Callback" id="Callback">Callback</h3> 
        <p> Before you can use any of the packages loaded by <code>google.charts.load</code> you have to wait for the loading to finish. It is not enough to just wait for the document to finish loading. Since it can take some time before this loading is finished, you need to register a callback function. There are two ways this can be done. Either specify a <code>callback</code> setting in your <code>google.charts.load</code> call or call <code>setOnLoadCallback</code> passing a function as the argument. </p> 
        <p> Note that you need to provide a function definition, rather than call the function. The function you provide can be either a named function or an anonymous function. When the packages have finished loading, this callback function will be called with no arguments. The loader will also wait for the document to finish loading before calling the callback. </p> 
        <p> If you want to draw more than one chart, you can either register more than one callback function, or you can combine them into one function. Learn more about how to <a href="https://developers.google.com/chart/interactive/docs/basic_multiple_charts"> Draw Multiple Charts on One Page</a>. </p> 
        <pre class="prettyprint lang-html">
  google.charts.setOnLoadCallback(<b>drawChart1</b>);
  google.charts.setOnLoadCallback(<b>drawChart2</b>);
  // OR
  google.charts.setOnLoadCallback(
    <b>function() { // Anonymous function that calls drawChart1 and drawChart2
         drawChart1();
         drawChart2();
      }</b>);
</pre> 
        <h2><a name="updateloader">Update Library Loader Code</a></h2>
        <a name="updateloader"> <p>Previous versions of Google Charts used different code to load the libraries. The table below shows the old library loader code versus the new.</p> 
         <table> 
          <tbody>
           <tr> 
            <th>Old Library Loader Code</th> 
           </tr> 
           <tr>
            <td> <pre class="prettyprint lang-js">&lt;script type="text/javascript" src="https://www.google.com/jsapi"&gt;&lt;/script&gt;
&lt;script type="text/javascript"&gt;
  google.load("visualization", "1", {packages:["corechart"]});
  google.setOnLoadCallback(drawChart);
&lt;/script&gt;
      </pre> </td> 
           </tr> 
           <tr> 
            <th>New Library Loader Code</th>
