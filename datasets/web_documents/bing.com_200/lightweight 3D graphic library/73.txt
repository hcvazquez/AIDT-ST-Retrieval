    <h1 id="Useful_Modules.2C_Packages_and_Libraries">Useful Modules, Packages and Libraries</h1> 
    <span class="anchor" id="line-4"></span>
    <p class="line862">The intent of this page is to list some of the most commonly used Python modules, in the hope that it will provide useful recommendations for other programmers (especially beginners). Remember that in addition to the listings below, there are other directories of Python modules - see <a href="/moin/PublishingPythonModules">PublishingPythonModules</a> for details. Another collection of library details can be found on the <a href="/moin/Libraries">Libraries</a> page. <span class="anchor" id="line-5"></span><span class="anchor" id="line-6"></span></p>
    <p class="line874">Be warned that this list is subjective by its very nature - it is only intended as a helpful guide. It is not definitive in any way, nor should it discourage developers from developing their own modules. <span class="anchor" id="line-7"></span><span class="anchor" id="line-8"></span></p>

    <h2 id="ID3_Handling">ID3 Handling</h2> 
    <span class="anchor" id="line-45"></span>
    <ul>
     <li><p class="line891"><a class="http" href="http://code.google.com/p/mutagen/">Mutagen</a> - Mutagen is a Python module to handle audio metadata. It supports FLAC, M4A, Musepack, MP3, Ogg FLAC, Ogg Speex, Ogg Theora, Ogg Vorbis, True Audio, and <a class="nonexistent" href="/moin/WavPack">WavPack</a> audio files. All versions of ID3v2 are supported, and all standard ID3v2.4 frames are parsed. It can read Xing headers to accurately calculate the bitrate and length of MP3s. ID3 and APEv2 tags can be edited regardless of audio format. It can also manipulate Ogg streams on an individual packet/page level. <strong>It only writes ID3v2.4</strong> (introduced in 2000, but as of Windows 8 still not supported by <a class="nonexistent" href="/moin/MediaPlayer">MediaPlayer</a>). <span class="anchor" id="line-46"></span></p></li>
     <li><p class="line891"><a class="http" href="http://www.nedbatchelder.com/code/modules/id3reader.html">ID3Reader</a> - "Id3reader.py is a Python module that reads ID3 metadata tags in MP3 files. It can read ID3v1, ID3v2.2, ID3v2.3, or ID3v2.4 tags. It does not write tags at all" (from site). Used in ID3Writer. Does not work with Python 3000. Not maintained since 2006. <span class="anchor" id="line-47"></span></p></li>
     <li><p class="line891"><a class="http" href="http://sourceforge.net/projects/pyid3">PyID3</a> - ""(Appears to be inactive)""Module for manipulating ID3 informational tags in MP3 audio files. Not as good as ID3Writer, but no issues w/ genre, unlike ID3Writer. Not maintained since 2007. <span class="anchor" id="line-48"></span></p></li>
     <li><p class="line891"><a class="http" href="http://www.liquidx.net/pytagger/">pytagger</a> - tag reader and writer implemented purely in Python. Supports ID3v1, ID3v1.1, ID3v2.2, ID3v2.3 and ID3v2.4 <span class="anchor" id="line-49"></span></p></li>
     <li><p class="line891"><a class="http" href="http://eyed3.nicfit.net/">eyeD3</a> - is a Python module and program for processing ID3 tags. It can extract information such as bit rate, sample frequency, play time, etc. It supports ID3 v1.0/v1.1 and v2.3/v2.4. <span class="anchor" id="line-50"></span></p></li>
     <li><p class="line891"><a class="http" href="http://pypi.python.org/pypi/hsaudiotag3k#downloads">hsaudiotag</a> - Py3k - hsaudiotag is a pure Python library that lets you read metadata (bitrate, sample rate, duration and tags) from mp3, mp4, wma, ogg, flac and aiff files. It can only read tags, not write to them, but unlike more complete libraries (like Mutagen), it is BSD licensed. <span class="anchor" id="line-51"></span></p></li>
     <li><p class="line891"><a class="https" href="https://pypi.python.org/pypi/pytaglib">pytaglib</a> - Python 3.x and 2.x support - bindings to the C++ <a class="http" href="http://taglib.github.io">taglib</a> library, reads and writes mp3, ogg, flac, mpc, speex, opus, <a class="nonexistent" href="/moin/WavPack">WavPack</a>, <a class="nonexistent" href="/moin/TrueAudio">TrueAudio</a>, wav, aiff, mp4 and asf files. <span class="anchor" id="line-52"></span><span class="anchor" id="line-53"></span></p></li>
    </ul>

    <h2 id="Editorial_Notes">Editorial Notes</h2> 
    <span class="anchor" id="line-162"></span>
    <p class="line874">Please avoid listing modules where... <span class="anchor" id="line-163"></span><span class="anchor" id="line-164"></span></p>
    <ul>
     <li>You are one of the developers (and you just want to promote your work somewhere). <span class="anchor" id="line-165"></span></li>
     <li>The modules in question do not have widespread approval or usage. <span class="anchor" id="line-166"></span><span class="anchor" id="line-167"></span></li>
    </ul>
    <p class="line862">If you do want to make people aware of a module, package or library (perhaps your own), consider submitting it to one or more of the directories mentioned in <a href="/moin/PublishingPythonModules">PublishingPythonModules</a>, perhaps also adding it to the topic-specific part of this site (eg. <a href="/moin/WebProgramming">WebProgramming</a>, <a href="/moin/GuiProgramming">GuiProgramming</a>). <span class="anchor" id="line-168"></span><span class="anchor" id="line-169"></span></p>
    <p class="line874">Do list modules that: <span class="anchor" id="line-170"></span><span class="anchor" id="line-171"></span></p>
    <ul>
     <li>Get recommended often in the Tutor or comp.lang.python lists <span class="anchor" id="line-172"></span><span class="anchor" id="line-173"></span></li>
    </ul>
    <p class="line874">Please provide a short description for each module. Try to put each module in the category that matches its "main" audience, since a module might fall into a number of categories. <span class="anchor" id="line-174"></span><span class="anchor" id="bottom"></span></p>
