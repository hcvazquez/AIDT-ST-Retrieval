    <h2 id="introduction-to-java"><a class="link" href="#introduction-to-java">1. Introduction to Java</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="a-small-history-of-java"><a class="link" href="#a-small-history-of-java">1.1. A small history of Java</a></h3> 
      <div class="paragraph"> 
       <p>Java is a programming language created by James Gosling from Sun Microsystems (Sun) in 1991. The target of Java is to write a program once and then run this program on multiple operating systems. The first publicly available version of Java (Java 1.0) was released in 1995. Sun Microsystems was acquired by the Oracle Corporation in 2010. Oracle has now the steermanship for Java. In 2006 Sun started to make Java available under the GNU General Public License (GPL). Oracle continues this project called <em>OpenJDK</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Over time new enhanced versions of Java have been released. The current version of Java is Java 1.8 which is also known as <em>Java 8</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Java is defined by a specification and consists of a programming language, a compiler, core libraries and a runtime (Java virtual machine) The Java runtime allows software developers to write program code in other languages than the Java programming language which still runs on the Java virtual machine. The <em>Java platform</em> is usually associated with the <em>Java virtual machine</em> and the <em>Java core libraries</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Java language was designed with the following properties:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>Platform independent: Java programs use the Java virtual machine as abstraction and do not access the operating system directly. This makes Java programs highly portable. A Java program (which is standard-compliant and follows certain rules) can run unmodified on all supported platforms, e.g., Windows or Linux.</p> </li> 
        <li> <p>Object-orientated programming language: Except the primitive data types, all elements in Java are objects.</p> </li> 
        <li> <p>Strongly-typed programming language: Java is strongly-typed, e.g., the types of the used variables must be pre-defined and conversion to other objects is relatively strict, e.g., must be done in most cases by the programmer.</p> </li> 
        <li> <p>Interpreted and compiled language: Java source code is transferred into the bytecode format which does not depend on the target platform. These bytecode instructions will be interpreted by the Java Virtual machine (JVM). The JVM contains a so called Hotspot-Compiler which translates performance critical bytecode instructions into native code instructions.</p> </li> 
        <li> <p>Automatic memory management: Java manages the memory allocation and de-allocation for creating new objects. The program does not have direct access to the memory. The so-called garbage collector automatically deletes objects to which no active pointer exists.</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>The Java syntax is similar to C++. Java is case-sensitive, e.g., variables called <code>myValue</code> and <code>myvalue</code> are treated as different variables.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="hello-world-java-program"><a class="link" href="#hello-world-java-program">1.2. Hello world Java program</a></h3> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// a small Java program</span>
<span class="directive">public</span> <span class="type">class</span> <span class="class">HelloWorld</span> {
    <span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {
        <span class="predefined-type">System</span>.out.println(<span class="string"><span class="delimiter">"</span><span class="content">Hello World</span><span class="delimiter">"</span></span>);
    }
}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="java-virtual-machine"><a class="link" href="#java-virtual-machine">1.3. Java virtual machine</a></h3> 
      <div class="paragraph"> 
       <p>The Java virtual machine (JVM) is a software implementation of a computer that executes programs like a real machine.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Java virtual machine is written specifically for a specific operating system, e.g., for Linux a special implementation is required as well as for Windows.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xjvmintroduction10.png.pagespeed.ic.yN5YT6-7_6.png" alt="jvmintroduction10" data-pagespeed-url-hash="2029914335" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Java programs are compiled by the Java compiler into <em>bytecode</em>. The Java virtual machine interprets this <em>bytecode</em> and executes the Java program.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="java-runtime-environment-vs-java-development-kit"><a class="link" href="#java-runtime-environment-vs-java-development-kit">1.4. Java Runtime Environment vs. Java Development Kit</a></h3> 
      <div class="paragraph"> 
       <p>A Java distribution typically comes in two flavors, the <em>Java Runtime Environment</em> (JRE) and the <em>Java Development Kit</em> (JDK).</p> 
      </div> 
      <div class="paragraph"> 
       <p>The JRE consists of the JVM and the Java class libraries. Those contain the necessary functionality to start Java programs.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The JDK additionally contains the development tools necessary to create Java programs. The JDK therefore consists of a Java compiler, the Java virtual machine and the Java class libraries.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="development-process-with-java"><a class="link" href="#development-process-with-java">1.5. Development Process with Java</a></h3> 
      <div class="paragraph"> 
       <p>Java source files are written as plain text documents. The programmer typically writes Java source code in an <em>Integrated Development Environment</em> (IDE) for programming. An IDE supports the programmer in the task of writing code, e.g., it provides auto-formating of the source code, highlighting of the important keywords, etc.</p> 
      </div> 
      <div class="paragraph"> 
       <p>At some point the programmer (or the IDE) calls the Java compiler ( <code>javac</code> ). The Java compiler creates the <em>bytecode</em> instructions. These instructions are stored in <code>.class</code> files and can be executed by the Java Virtual Machine.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="garbage-collector"><a class="link" href="#garbage-collector">1.6. Garbage collector</a></h3> 
      <div class="paragraph"> 
       <p>The JVM automatically re-collects the memory which is not referred to by other objects. The Java <em>garbage collector</em> checks all object references and finds the objects which can be automatically released.</p> 
      </div> 
      <div class="paragraph"> 
       <p>While the garbage collector relieves the programmer from the need to explicitly manage memory, the programmer still need to ensure that he does not keep unneeded object references, otherwise the garbage collector cannot release the associated memory. Keeping unneeded object references are typically called <em>memory leaks</em>.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="classpath"><a class="link" href="#classpath">1.7. Classpath</a></h3> 
      <div class="paragraph"> 
       <p>The <em>classpath</em> defines where the Java compiler and Java runtime look for <code>.class</code> files to load. These instructions can be used in the Java program.</p> 
      </div> 
      <div class="paragraph"> 
       <p>For example, if you want to use an external Java library you have to add this library to your classpath to use it in your program.</p> 
      </div> 
     </div> 
    </div> 
   </div>

     <div class="sect2"> 
      <h3 id="check-installation"><a class="link" href="#check-installation">2.1. Check installation</a></h3> 
      <div class="paragraph"> 
       <p>To run Java programs on your computer you must at least have the Java runtime environment (JRE) installed. This might already be the case on your machine. You can test is the JRE is installed and in your current path by opening a console (if you are using Windows: Win+R, enter <em>cmd</em> and press Enter) and by typing in the following command:</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="bourne">java -version</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>If the JRE is installed and within your path, this commands prints information about your Java installation. In this case you can skip the Java installation description.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If the command line returns the information that the program could not be found, you have to install Java.</p> 
      </div>

    <h2 id="exercise-write-compile-and-run-a-java-program"><a class="link" href="#exercise-write-compile-and-run-a-java-program">3. Exercise: Write, compile and run a Java program</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="write-source-code"><a class="link" href="#write-source-code">3.1. Write source code</a></h3> 
      <div class="paragraph"> 
       <p>The following Java program is developed under Linux using a text editor and the command line. The process on other operating system should be similar, but is not covered in this description.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Select or create a new directory which will be used for your Java development. In this description the path <code>\home\vogella\javastarter</code> is used. On Microsoft Windows you might want to use <code>c:\temp\javastarter</code>. This path is called <em>javadir</em> in the following description.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Open a text editor which supports plain text, e.g., <em>gedit</em> under Linux or <em>Notepad</em> under Windows and write the following source code.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// a small Java program</span>
<span class="directive">public</span> <span class="type">class</span> <span class="class">HelloWorld</span> {
    <span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {
        <span class="predefined-type">System</span>.out.println(<span class="string"><span class="delimiter">"</span><span class="content">Hello World</span><span class="delimiter">"</span></span>);
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="admonitionblock warning"> 
       <table> 
        <tbody>
         <tr> 
          <td class="icon"> <i class="fa icon-warning" title="Warning"></i> </td> 
          <td class="content"> Do not use a rich editor like Microsoft Word or LibreOffice for writing Java code. If in doubt, google for "Plain text editor for [your_OS]". </td> 
         </tr> 
        </tbody>
       </table> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xfirstjava10.png.pagespeed.ic.-1pEz75_iy.png" alt="Writing Java programs in a text editor" data-pagespeed-url-hash="4060528462" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Save the source code in your <em>javadir</em> directory with the <code>HelloWorld.java</code> filename. The name of a Java source file must always equal the class name (within the source code) and end with the <code>.java</code> extension. In this example the filename must be <code>HelloWorld.java</code>, because the class is called <code>HelloWorld</code>.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="compile-and-run-your-java-program"><a class="link" href="#compile-and-run-your-java-program">3.2. Compile and run your Java program</a></h3> 
      <div class="paragraph"> 
       <p>Open a shell for command line access.</p> 
      </div> 
      <div class="admonitionblock tip"> 
       <table> 
        <tbody>
         <tr> 
          <td class="icon"> <i class="fa icon-tip" title="Tip"></i> </td> 
          <td class="content"> If you don’t know how to do this, google for "How to open a shell under [your_OS]". </td> 
         </tr> 
        </tbody>
       </table> 
      </div> 
      <div class="paragraph"> 
       <p>Switch to the <em>javadir</em> directory with the command <code>cd javadir</code>, for example, in the above example via the <code>cd \home\vogella\javastarter</code> command. Use the <code>ls</code> command (<code>dir</code> under Microsoft Windows) to verify that the source file is in the directory.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Compile your Java source file into a class file with the following command.</p> 
      </div>

   <div class="sect1"> 
    <h2 id="base-java-language-structure"><a class="link" href="#base-java-language-structure">4. Base Java language structure</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="class"><a class="link" href="#class">4.1. Class</a></h3> 
      <div class="paragraph"> 
       <p>A class is a template that describes the data and behavior associated with an instance of that class.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A class is defined by the <em>class</em> keyword and must start with a capital letter. The body of a class is surrounded by {}.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">test</span>;

<span class="type">class</span> <span class="class">MyClass</span> {

}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>The data associated with a class is stored in <em>variables</em> ; the behavior associated to a class or object is implemented with <em>methods</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A class is contained in a text file with the same name as the class plus the <code>.java</code> extension. It is also possible to define inner classes, these are classes defined within another class, in this case you do not need a separate file for this class.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="object"><a class="link" href="#object">4.2. Object</a></h3> 
      <div class="paragraph"> 
       <p>An object is an instance of a class.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The object is the real element which has data and can perform actions. Each object is created based on the class definition. The class can be seen as the blueprint of an object, i.e., it describes how an object is created.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="package"><a class="link" href="#package">4.3. Package</a></h3> 
      <div class="paragraph"> 
       <p>Java groups classes into functional <em>packages</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Packages are typically used to group classes into logical units. For example, all graphical views of an application might be placed in the same package called <code>com.vogella.webapplication.views</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>It is common practice to use the reverse domain name of the company as top level package. For example, the company might own the domain, vogella.com and in this example the Java packages of this company starts with <code>com.vogella</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Other main reason for the usage of packages is to avoid name collisions of classes. A name collision occurs if two programmers give the same <em>fully qualified name</em> to a class. The <em>fully qualified name</em> of a class in Java consists of the package name followed by a dot (.) and the class name.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Without packages, a programmer may create a Java class called <code>Test</code>. Another programmer may create a class with the same name. With the usage of packages you can tell the system which class to call. For example, if the first programmer puts the <code>Test</code> class into package <code>report</code> and the second programmer puts his class into package <code>xmlreader</code> you can distinguish between these classes by using the <em>fully qualified name</em>, e.g, <code>xmlreader.Test</code> or <code>report.Test</code>.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_inheritance"><a class="link" href="#javadef_inheritance">4.4. Inheritance</a></h3> 
      <div class="paragraph"> 
       <p>A class can be derived from another class. In this case this class is called a <em>subclass</em>. Another common phrase is that <em>a class extends another class.</em></p> 
      </div> 
      <div class="paragraph"> 
       <p>The class from which the subclass is derived is called a <em>superclass</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Inheritance allows a class to inherit the behavior and data definitions of another class.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following codes demonstrates how a class can extend another class. In Java a class can only extend a maximum of one class.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="type">class</span> <span class="class">MyBaseClass</span> {

    <span class="directive">public</span> <span class="type">void</span> hello() {
        <span class="predefined-type">System</span>.out.println(<span class="string"><span class="delimiter">"</span><span class="content">Hello from MyBaseClass</span><span class="delimiter">"</span></span>);
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="type">class</span> <span class="class">MyExtensionClass</span> <span class="directive">extends</span> MyBaseClass {
}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_objectsuperclass"><a class="link" href="#javadef_objectsuperclass">4.5. Object as superclass</a></h3> 
      <div class="paragraph"> 
       <p>Every object in Java implicitly extends the <code>Object</code> class. The class defines the following methods for every Java object: * <code>equals(o1)</code> allows checking if the current object is equal to o1</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p><code>getClass()</code> returns the class of the object</p> </li> 
        <li> <p><code>hashCode()</code> returns an identifier of the current object</p> </li> 
        <li> <p><code>toString()</code> gives a string representation of the current object</p> </li> 
       </ul> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="exception-handling-in-java"><a class="link" href="#exception-handling-in-java">4.6. Exception handling in Java</a></h3> 
      <div class="paragraph"> 
       <p>In Java an exception is an event to indicate an error during the runtime of an application. So this disrupts the usual flow of the application’s instructions.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In general exceptions are thrown up in the call hierarchy until they get catched.</p> 
      </div> 
      <div class="imageblock"> 
       <div class="content"> 
        <img src="img/xexception_throw_callstack.png.pagespeed.ic.W_hZMu07dI.png" alt="exception throw callstack" data-pagespeed-url-hash="2685211814" onload="pagespeed.CriticalImages.checkImageForCriticality(this);"> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="checked-exceptions"><a class="link" href="#checked-exceptions">4.7. Checked Exceptions</a></h3> 
      <div class="paragraph"> 
       <p>Checked Exceptions are explicitly thrown by methods, which might cause the exception or re-thrown by methods in case a thrown Exception is not caught.</p> 
      </div> 
      <div class="paragraph"> 
       <p>So when calling methods, which throw checked Exceptions the Exceptions have either to be caught or to be re-thrown.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="directive">public</span> <span class="type">void</span> fileNotFoundExceptionIsCaughtInside() {
    <span class="keyword">try</span> {
        createFileReader();
    } <span class="keyword">catch</span> (<span class="exception">FileNotFoundException</span> e) {
        logger.error(e.getMessage(), e);
    }
}

<span class="directive">public</span> <span class="type">void</span> fileNotFoundExceptionIsReThrown() <span class="directive">throws</span> <span class="exception">FileNotFoundException</span> {
    createFileReader();
}

<span class="directive">public</span> <span class="type">void</span> createFileReader() <span class="directive">throws</span> <span class="exception">FileNotFoundException</span> {
    <span class="predefined-type">File</span> file = <span class="keyword">new</span> <span class="predefined-type">File</span>(<span class="string"><span class="delimiter">"</span><span class="content">/home/Documents/JavaTraining.txt</span><span class="delimiter">"</span></span>);

    <span class="comment">// creating a new FileReader can cause a FileNotFoundException</span>
    <span class="keyword">new</span> <span class="predefined-type">FileReader</span>(file);
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Checked Exceptions are used when an error can be predicted under certain circumstances, e.g., a file which cannot be found.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="runtime-exceptions"><a class="link" href="#runtime-exceptions">4.8. Runtime Exceptions</a></h3> 
      <div class="paragraph"> 
       <p>Runtime Exceptions are Exceptions, which are not explicitly mentioned in the method signature and therefore also do not have to be catched explicitly.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The most famous runtime exception is the NullPointerException, which occurs during runtime, when a method is invoked on an object, which actually is <code>null</code>.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="directive">public</span> <span class="type">void</span> causeANullPointerException() {
    <span class="predefined-type">String</span> thisStringIsNull = getMessage(<span class="predefined-constant">false</span>);

    <span class="comment">// because the thisStringIsNull object is null</span>
    <span class="comment">// this will cause a NullPointerException</span>
    thisStringIsNull.toLowerCase();
}

<span class="directive">public</span> <span class="predefined-type">String</span> getMessage(<span class="type">boolean</span> messageIsAvailable) {
    <span class="keyword">if</span>(messageIsAvailable) {
        <span class="keyword">return</span> message;
    }

    <span class="keyword">return</span> <span class="predefined-constant">null</span>;
}</code></pre> 
       </div>

    <h2 id="java_interfaces"><a class="link" href="#java_interfaces">5. Java interfaces</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="javadef_interface"><a class="link" href="#javadef_interface">5.1. What is an interface in Java?</a></h3> 
      <div class="paragraph"> 
       <p>An <em>interfaces</em> is a type similar to a class and is defined via the <code>interface</code> keyword. Interfaces are used to define common behavior of implementing classes. If two classes implement the same interface, other code which work on the interface level, can use objects of both classes.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Like a class an interface defines methods. Classes can implement one or several interfaces. A class which implements an interface must provide an implementation for all abstract methods defined in the interface.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_interfacemethods"><a class="link" href="#javadef_interfacemethods">5.2. Abstract, default and static methods in Interfaces</a></h3> 
      <div class="paragraph"> 
       <p>An interface can have abstract methods and _default_methods. A default method is defined via the <code>default</code> keyword at the beginning of the method signature. All other methods defined in an interfaces are public and abstract; explicit declaration of these modifiers is optional.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Interfaces can have constants which are always implicitly public, static and final.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following code shows an example implementation of an interface.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">testing</span>;

<span class="directive">public</span> <span class="type">interface</span> <span class="class">MyInterface</span> {

        <span class="comment">// constant definition</span>
        <span class="predefined-type">String</span> <span class="predefined-type">URL</span> = <span class="string"><span class="delimiter">"</span><span class="content">http://www.vogella.com</span><span class="delimiter">"</span></span>;

        <span class="comment">// public abstract methods</span>
        <span class="type">void</span> test();
        <span class="type">void</span> write(<span class="predefined-type">String</span> s);

        <span class="comment">// default method</span>
        <span class="keyword">default</span> <span class="predefined-type">String</span> reserveString(<span class="predefined-type">String</span> s){
            <span class="keyword">return</span> <span class="keyword">new</span> <span class="predefined-type">StringBuilder</span>(s).reverse().toString();
        }
}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_interfaceimplementing"><a class="link" href="#javadef_interfaceimplementing">5.3. Implementing Interfaces</a></h3> 
      <div class="paragraph"> 
       <p>A class can implement an interface. In this case it must provide concrete implementations of the abstract interface methods. If you implement a method defined by an interface, you can use <code>@Override</code> annotation. This indicates to the Java compiler that you actually want to implement a method defined by this interface. This way the compiler can give you an error in you mis-typed the name of the method or in the number of arguments. The following class implements the <code>MyInterface</code> interface, its must therefore implement the abstract method and can use the default methods.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyClassImpl</span> <span class="directive">implements</span> MyInterface {

    <span class="annotation">@Override</span>
    <span class="directive">public</span> <span class="type">void</span> test() {
    }

    <span class="annotation">@Override</span>
    <span class="directive">public</span> <span class="type">void</span> write(<span class="predefined-type">String</span> s) {
    }

    <span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {
        MyClassImpl impl = <span class="keyword">new</span> MyClassImpl();
        <span class="predefined-type">System</span>.out.println(impl.reserveString(<span class="string"><span class="delimiter">"</span><span class="content">Lars Vogel</span><span class="delimiter">"</span></span>));
    }

}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_interfaceevolving"><a class="link" href="#javadef_interfaceevolving">5.4. Evolving interfaces with default methods</a></h3> 
      <div class="paragraph"> 
       <p>Before Java 8 evolving interfaces, e.g., adding new methods to an interface, was not possible without breaking existing clients. Java 8 introduced default methods, now you can extend an interface without breaking clients by simply suppling a default implementation with it. Adding such a default method is a source and binary compatible change.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A class can always override a default method to supply a better behavior.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_interface_multiple"><a class="link" href="#javadef_interface_multiple">5.5. Multiple inheritance of methods</a></h3> 
      <div class="paragraph"> 
       <p>If a class implements two interfaces and if these interfaces provide the same default method, Java resolves the correct method for the class by the following rules:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>Superclass wins always against the superinterface - If a class can inherit a method from a superclass and a superinterface, the class inherits the superclass method. This is true for concrete and abstract superclass methods. This rule implies that default methods are not used if this method is also declared in the superclass chain.</p> </li> 
        <li> <p>Subtypes win over Supertypes - If a class can inherit a method from two interfaces, and one is a subtype of the other, the class inherts the method from the subtype</p> </li> 
        <li> <p>In all other cases the class needs to implement the default method</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>The following listing demonstrates listing number 3.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="directive">public</span> <span class="type">interface</span> <span class="class">A</span> {
    <span class="keyword">default</span> <span class="type">void</span> m() {}
}

<span class="directive">public</span> <span class="type">interface</span> <span class="class">B</span> {
    <span class="keyword">default</span> <span class="type">void</span> m() {}
}

<span class="directive">public</span> <span class="type">class</span> <span class="class">C</span> <span class="directive">implements</span> A, B {
    <span class="annotation">@Override</span>
    <span class="directive">public</span> <span class="type">void</span> m() {}
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>In your implementation you can also call the super method you prefer.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="directive">public</span> <span class="type">class</span> <span class="class">C</span> <span class="directive">implements</span> A, B {
    <span class="annotation">@Override</span>
    <span class="directive">public</span> <span class="type">void</span> m() {A.super.m();}
}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_functionalinterfaces"><a class="link" href="#javadef_functionalinterfaces">5.6. Functional interfaces</a></h3> 
      <div class="paragraph"> 
       <p>All interfaces that have only one method are called <em>functional interfaces</em>. Functional interfaces have the advantage that they can be used together with <em>lambda expressions</em>. See <a href="#lambdas_definition">What are lambdas?</a> to learn more about lambdas, e.g., the type of lambdas is a functional interface.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The Java compiler automatically identifies functional interfaces. The only requirement is that they have only one abstract method. However, is possible to capture the design intent with a <code>@FunctionalInterface</code> annotation.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Several default Java interfaces are functional interfaces:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p><code>java.lang.Runnable</code></p> </li> 
        <li> <p><code>java.util.concurrent.Callable</code></p> </li> 
        <li> <p><code>java.io.FileFilter</code></p> </li> 
        <li> <p><code>java.util.Comparator</code></p> </li> 
        <li> <p><code>java.beans.PropertyChangeListener</code></p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>Java also contains the <code>java.util.function</code> package which contains functional interfaces which are frequently used such as:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>Predicate&lt;T&gt; - a boolean-valued property of an object</p> </li> 
        <li> <p>Consumer&lt;T&gt; - an action to be performed on an object</p> </li> 
        <li> <p>Function&lt;T, R&gt; - a function transforming a T to a R</p> </li> 
        <li> <p>Supplier&lt;T&gt; - provides an instance of T (such as a factory)</p> </li> 
        <li> <p>UnaryOperator&lt;T&gt; - a function from T to T</p> </li> 
        <li> <p>BinaryOperator&lt;T&gt; - a function from (T, T) to T</p> </li> 
       </ul> 
      </div> 
     </div> 
    </div> 
   </div> 
   <div class="sect1"> 
    <h2 id="javaannotations"><a class="link" href="#javaannotations">6. Annotations in Java</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="annotations_definition"><a class="link" href="#annotations_definition">6.1. Annotations in Java</a></h3> 
      <div class="paragraph"> 
       <p>Annotations provide data about a class that is not part of the programming logic itself. They have no direct effect on the code they annotate. Other components can use this information.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Annotations can be preserved at runtime (<code>RetentionPolicy.RUNTIME</code>) or are only available at development time (<code>RetentionPolicy.SOURCE</code>).</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef"><a class="link" href="#javadef">6.2. Override methods and the @Override annotation</a></h3> 
      <div class="paragraph"> 
       <p>If a class extends another class, it inherits the methods from its superclass. If it wants to change these methods, it can <em>override</em> these methods, i.e., redeclare the methods. This is necessary for an abstract method unless the class itself is defined as abstract.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The <code>@Override</code> annotation can be added to such a method. It is used by the Java compiler to check if the annotated method really overrides a method of an interface or the extended class.</p> 
      </div> 
      <div class="paragraph"> 
       <p>To override a method, you use the same method signature in the source code of the subclass.</p> 
      </div> 
      <div class="paragraph"> 
       <p>To indicate to the reader of the source code and the Java compiler that you have the intention to override a method, you can use the <code>@Override</code> annotation.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following code demonstrates how you can override a method from a superclass.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

    <h2 id="javadef_variables"><a class="link" href="#javadef_variables">7. Variables and methods</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="javadef_variable"><a class="link" href="#javadef_variable">7.1. Variable</a></h3> 
      <div class="paragraph"> 
       <p><em>Variables</em> allow the Java program to store values during the runtime of the program.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A variable can either be a: * <em>primitive variable</em> * <em>reference variable</em></p> 
      </div> 
      <div class="paragraph"> 
       <p>A primitive variable contains the value. A reference variable contains a reference (pointer) to the object. Hence, if you compare two reference variables, you compare if both point to the same object. To identify if objects contain the same data, use the <code>object1.equals(object2)</code> method call.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_instancevariable"><a class="link" href="#javadef_instancevariable">7.2. Instance variable</a></h3> 
      <div class="paragraph"> 
       <p>Instance variable is associated with an instance of the class (also called object). Access works over these objects.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Instance variables can have any access control and can be marked <code>final</code> or <code>transient</code>. Instance variables marked as final cannot be changed after a value has been assigned to them.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_localvariable"><a class="link" href="#javadef_localvariable">7.3. Local variable</a></h3> 
      <div class="paragraph"> 
       <p>Local (stack) variable declarations cannot have access modifiers. Local variables do not get default values, so they must be initialized before they can be used.</p> 
      </div> 
      <div class="paragraph"> 
       <p><em>final</em> is the only modifier available to local variables. This modifier defines that the variable cannot be changed after the first assignment.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_method"><a class="link" href="#javadef_method">7.4. Methods</a></h3> 
      <div class="paragraph"> 
       <p>A method is a block of code with parameters and a return value. It can be called on the object.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyMethodExample</span> {
    <span class="type">void</span> tester(<span class="predefined-type">String</span> s) {
        <span class="predefined-type">System</span>.out.println(<span class="string"><span class="delimiter">"</span><span class="content">Hello World</span><span class="delimiter">"</span></span>);
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Methods can be declared with var-args. In this case the method declares a parameter which accepts everything from zero to many arguments (syntax: type …? name;) A method can only have one var-args parameter and this must be the last parameter in the method.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Overwrite of a superclass method: A method must be of the exact same return parameter and the same arguments. Also the return parameter must be the same. Overload methods: An overloaded method is a method with the same name, but different arguments. The return type can not be used to overload a method.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_mainmethod"><a class="link" href="#javadef_mainmethod">7.5. Main method</a></h3> 
      <div class="paragraph"> 
       <p>A public static method with the following signature can be used to start a Java application. Such a method is typically called <code>main</code> method.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {

}</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javadef_constructor"><a class="link" href="#javadef_constructor">7.6. Constructor</a></h3> 
      <div class="paragraph"> 
       <p>A class contains <em>constructors</em> that are invoked by the Java runtime to create objects based on the class definition.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Constructor declarations look like method declarations except that they use the name of the class and have no return type.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A class can have several constructors with different parameters.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In the following example the constructor of the class expects a parameter.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyConstructorExample2</span> {

    <span class="predefined-type">String</span> s;

    <span class="directive">public</span> MyConstructorExample2(<span class="predefined-type">String</span> s) {
        <span class="local-variable">this</span>.s = s;
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>Each class must define at least one constructor. If no explicit constructor is defined in the Java source file, the compiler implicitly adds a constructor. If the class is sub-classed, then the constructor of the super class is always called implicitly in this case.</p> 
      </div> 
      <div class="paragraph"> 
       <p>In the following example the definition of the constructor without parameters (also known as the <em>empty constructor</em>) is unnecessary. If not specified, the compiler would create one.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyConstructorExample</span> {

    <span class="comment">// unnecessary: would be created by the compiler if left out</span>
    <span class="directive">public</span> MyConstructorExample() {
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="paragraph"> 
       <p>The naming convention for creating a constructor is the following: <code>classname (Parameter p1, …?) { }</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Every object is created based on a constructor. This constructor method is the first statement called before anything else can be done with the object.</p> 
      </div>

      </div>

      </div>

      </div>

      <h3 id="javaimports_definition"><a class="link" href="#javaimports_definition">9.1. Usage of import statements</a></h3>

      </div> 
      <div class="paragraph"> 
       <p>The feature provides a typesafe mechanism to include constants into code. It also improves code readability and allows Java API designers to write a concise API.</p>

    <h2 id="java_additionaldefinition"><a class="link" href="#java_additionaldefinition">10. More Java language constructs</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="javadef_classmethod"><a class="link" href="#javadef_classmethod">10.1. Class methods and class variables</a></h3> 
      <div class="paragraph"> 
       <p>Class methods and class variables are associated with the class and not an instance of the class, i.e., objects. To refer to these elements, you can use the classname and a dot (".") followed by the class method or class variable name.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Class methods and class variables are declared with the <code>static</code> keyword. Class methods are also called <em>static methods</em> and class variables are also called <em>static variables</em> or <em>static fields</em>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>An example for the usage of a static field is <code>println</code> of the following statement: <code>System.out.println("Hello World")</code>. Hereby <code>out</code> is a static field, an object of type <code>PrintStream</code> and you call the <code>println()</code> method on this object.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If you define a static variable, the Java runtime environment associates one class variable for a class no matter how many instances (objects) exist. The static variable can therefore be seen as a global variable.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following code demonstrates the usage of <code>static</code> fields.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyStaticExample</span> {

    <span class="directive">static</span> <span class="predefined-type">String</span> PLACEHOLDER = <span class="string"><span class="delimiter">"</span><span class="content">TEST</span><span class="delimiter">"</span></span>;

    <span class="directive">static</span> <span class="type">void</span> test() {
        <span class="predefined-type">System</span>.out.println(<span class="string"><span class="delimiter">"</span><span class="content">Hello</span><span class="delimiter">"</span></span>);
    }
}</code></pre> 
       </div> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">com.vogella.javaintro.base</span>;

       </div> 
      </div> 
      <div class="paragraph"> 
       <p>If a variable should be defined as constant, you declare it with the <code>static</code> and the <code>final</code> keyword.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The static method runs without any instance of the class, it cannot directly access non-static variables or methods.</p> 
      </div>

      </div> 
      <div class="paragraph"> 
       <p>If a class has at least one method, which only contains the declaration of the method, but not the implementation, then this class is <code>abstract</code> and can not be instantiated. Sub-classes need then to define the methods except if they are also declared as abstract.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If a class contains an abstract method, it also needs to get defined with the <code>abstract</code> keyword.</p> 
      </div>

      </div>

     <div class="sect2"> 
      <h3 id="javastringpool"><a class="link" href="#javastringpool">19.2. String pool in Java</a></h3> 
      <div class="paragraph"> 
       <p>For memory efficiency Java uses a <code>String</code> pool. The string pool allows string literals to be reused. This is possible because strings in Java are immutable.</p> 
      </div> 
      <div class="paragraph"> 
       <p>If the same string literal is used in several places in the Java code, only one copy of that string is created.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Whenever a <code>String</code> object is created and gets a string literal assigned the string pool is used. For example, <code>String s = "constant"</code>. However, the <code>new</code> operator forces a new <code>String</code> copy to be allocated, for example, in <code>String s = new String("constant");</code>.</p> 
      </div> 
     </div>

    <h2 id="lambdas"><a class="link" href="#lambdas">20. Lambdas</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="lambdas_definition"><a class="link" href="#lambdas_definition">20.1. What are lambdas?</a></h3> 
      <div class="paragraph"> 
       <p>The Java programming language supports <em>lambdas</em> as of Java 8. A lambda expression is a block of code with parameters. Lambdas allows to specify a block of code which should be executed later. If a method expects a <em>functional interface</em> as parameter it is possible to pass in the lambda expression instead.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The type of a lambda expression in Java is a functional interface.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="lambdas_purpose"><a class="link" href="#lambdas_purpose">20.2. Purpose of lambda expressions</a></h3> 
      <div class="paragraph"> 
       <p>Using lambdas allows to use a condensed syntax compared to other Java programming constructs. For example the <code>Collection</code> interfaces has <code>forEach</code> method which accepts a lambda expression.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="predefined-type">List</span>&lt;<span class="predefined-type">String</span>&gt; list = <span class="predefined-type">Arrays</span>.asList(<span class="string"><span class="delimiter">"</span><span class="content">vogella.com</span><span class="delimiter">"</span></span>,<span class="string"><span class="delimiter">"</span><span class="content">google.com</span><span class="delimiter">"</span></span>,<span class="string"><span class="delimiter">"</span><span class="content">heise.de</span><span class="delimiter">"</span></span> )
list.forEach(s-&gt; <span class="predefined-type">System</span>.out.println(s));</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="lambdas_methodreferences"><a class="link" href="#lambdas_methodreferences">20.3. Using method references</a></h3> 
      <div class="paragraph"> 
       <p>You can use method references in a lambda expression. Method reference define the method to be called via <code>CalledFrom::method</code>. CalledFrom can be * instance::instanceMethod * SomeClass::staticMethod * SomeClass::instanceMethod</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="java"><span class="predefined-type">List</span>&lt;<span class="predefined-type">String</span>&gt; list = <span class="keyword">new</span> <span class="predefined-type">ArrayList</span>&lt;&gt;();
list.add(<span class="string"><span class="delimiter">"</span><span class="content">vogella.com</span><span class="delimiter">"</span></span>);
list.add(<span class="string"><span class="delimiter">"</span><span class="content">google.com</span><span class="delimiter">"</span></span>);
list.add(<span class="string"><span class="delimiter">"</span><span class="content">heise.de</span><span class="delimiter">"</span></span>);
list.forEach(<span class="predefined-type">System</span>.out::println);</code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="lambdas_closures"><a class="link" href="#lambdas_closures">20.4. Difference between a lambda expression and a closure</a></h3> 
      <div class="paragraph"> 
       <p>The Java programming language supports lambdas but not closures. A lambda is an anonymous function, e.g., it can be defined as parameter. Closures are <em>code fragments</em> or <em>code blocks</em> which can be used without being a method or a class. This means that a closure can access variables not defined in its parameter list and that it can also be assigned to a variable.</p> 
      </div> 
     </div> 
    </div> 
   </div>

      <div class="paragraph"> 
       <p>A stream from the <code>java.util.stream</code> package is a sequence of elements from a source that supports aggregate operations.</p> 
      </div>

   <div class="sect1"> 
    <h2 id="optional"><a class="link" href="#optional">22. Optional</a></h2> 
    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>If you call a method or access a field on an object which is not initialized (null) you receive a NullPointerException (NPE). The <code>java.util.Optional</code> class can be used to avoid these NPEs.</p> 
     </div> 
     <div class="paragraph"> 
      <p><code>java.util.Optional</code> is a good tool to indicate that a return value may be absent, which can be seen directly in the method signature rather than just mentioning that <code>null</code> may be returned in the JavaDoc.</p> 
     </div> 
     <div class="paragraph"> 
      <p>If you want to call a method on an Optional object and check some property you can use the <code>filter</code> method. The filter method takes a predicate as an argument. If a value is present in the Optional object and it matches the predicate, the filter method returns that value; otherwise, it returns an empty Optional object.</p> 
     </div> 
     <div class="paragraph"> 
      <p>You can create an <code>Optional</code> in different ways:</p> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// use this if the object is not null</span>
opt = Optional.of(o);
<span class="comment">// creates an empty Optional, if o is null</span>
opt = Optional.ofNullable(o);
<span class="comment">// create an empty Optional</span>
opt = Optional.empty();</code></pre> 
      </div> 
     </div> 
     <div class="paragraph"> 
      <p>The <code>ifPresent</code> method can be used to execute some code on an object if it is present. Assume you have a <code>Todo</code> object and want to call the <code>getId()</code> method on it. You can do this via the following code.</p> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java">Todo todo = <span class="keyword">new</span> Todo(-<span class="integer">1</span>);
Optional&lt;Todo&gt; optTodo = Optional.of(todo);
<span class="comment">// get the id of the todo or a default value</span>
optTodo.ifPresent(t-&gt; <span class="predefined-type">System</span>.out.println(t.getId()));</code></pre> 
      </div> 
     </div> 
     <div class="paragraph"> 
      <p>Via the <code>map</code> method you can transform the object if it is present and via the <code>filter</code> method you can filter for certain values.</p> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java">Todo todo = <span class="keyword">new</span> Todo(-<span class="integer">1</span>);
Optional&lt;Todo&gt; optTodo = Optional.of(todo);

<span class="comment">// get the summary (trimmed) of todo if the id is higher than 0</span>
Optional&lt;<span class="predefined-type">String</span>&gt; map = optTodo.filter(o -&gt; o.getId() &gt; <span class="integer">0</span>).map(o -&gt; o.getSummary().trim());

<span class="comment">// same as above but print it out</span>
optTodo.filter(o -&gt; o.getId() &gt; <span class="integer">0</span>).map(o -&gt; o.getSummary().trim()).ifPresent(<span class="predefined-type">System</span>.out::println);</code></pre> 
      </div> 
     </div> 
     <div class="paragraph"> 
      <p>To get the real value of an Optional the get() method can be used. But in case the Optional is empty this will throw a NoSuchElementException. To avoid this NoSuchElementException the <code>orElse</code> or the <code>orElseGet</code> can be used to provide a default in case of absence.</p> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java"><span class="comment">// using a String</span>
<span class="predefined-type">String</span> s = <span class="string"><span class="delimiter">"</span><span class="content">Hello</span><span class="delimiter">"</span></span>;
Optional&lt;<span class="predefined-type">String</span>&gt; maybeS = Optional.of(s);
<span class="comment">// get length of the String or -1 as default</span>
<span class="type">int</span> len = maybeS.map(<span class="predefined-type">String</span>::length).orElse(-<span class="integer">1</span>);

<span class="comment">// orElseGet allows to construct an object / value with a Supplier</span>
<span class="type">int</span> calStringlen = maybeS.map(<span class="predefined-type">String</span>::length).orElseGet(()-&gt; <span class="string"><span class="delimiter">"</span><span class="content">Hello</span><span class="delimiter">"</span></span>.length());</code></pre> 
      </div> 
     </div> 
    </div>

   <div class="sect1"> 
    <h2 id="systemproperties"><a class="link" href="#systemproperties">23. System properties</a></h2> 
    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>The <code>System</code> class provides access to the configuratoin of the current working environment. You can access them, via <code>System.getProperty("property_name")</code>, for example <code>System.getProperty("path.separator");</code> The following lists describes the most important properties.</p> 
     </div> 
     <div class="ulist"> 
      <ul> 
       <li> <p>"line.separator" - Sequence used by operating system to separate lines in text files</p> </li> 
       <li> <p>"user.dir" - User working directory</p> </li> 
       <li> <p>"user.home" - User home directory == Schedule tasks</p> </li> 
      </ul> 
     </div> 
     <div class="paragraph"> 
      <p>Java allows you to schedule tasks. A scheduled tasks can perform once or several times.</p> 
     </div> 
     <div class="paragraph"> 
      <p><code>java.util.Timer</code> and <code>java.util.TimerTask</code> can be used to schedule tasks. The object which implements <code>TimeTask</code> will then be performed by the <code>Timer</code> based on the given interval.</p> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">schedule</span>;

<span class="keyword">import</span> <span class="include">java.util.TimerTask</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">MyTask</span> <span class="directive">extends</span> <span class="predefined-type">TimerTask</span> {
    <span class="directive">private</span> <span class="directive">final</span> <span class="predefined-type">String</span> string;
    <span class="directive">private</span> <span class="type">int</span> count = <span class="integer">0</span>;

    <span class="directive">public</span> MyTask(<span class="predefined-type">String</span> string) {
        <span class="local-variable">this</span>.string = string;
    }

    <span class="annotation">@Override</span>
    <span class="directive">public</span> <span class="type">void</span> run() {
        count++;
        <span class="predefined-type">System</span>.out.println(string + <span class="string"><span class="delimiter">"</span><span class="content"> called </span><span class="delimiter">"</span></span> + count);
    }

}</code></pre> 
      </div> 
     </div> 
     <div class="listingblock"> 
      <div class="content"> 
       <pre class="CodeRay highlight"><code data-lang="java"><span class="keyword">package</span> <span class="namespace">schedule</span>;

<span class="keyword">import</span> <span class="include">java.util.Timer</span>;

<span class="directive">public</span> <span class="type">class</span> <span class="class">ScheduleTest</span> {

    <span class="directive">public</span> <span class="directive">static</span> <span class="type">void</span> main(<span class="predefined-type">String</span><span class="type">[]</span> args) {
        <span class="predefined-type">Timer</span> timer = <span class="keyword">new</span> <span class="predefined-type">Timer</span>();

        <span class="comment">// wait 2 seconds (2000 milli-secs) and then start</span>
        timer.schedule(<span class="keyword">new</span> MyTask(<span class="string"><span class="delimiter">"</span><span class="content">Task1</span><span class="delimiter">"</span></span>), <span class="integer">2000</span>);

        <span class="keyword">for</span> (<span class="type">int</span> i = <span class="integer">0</span>; i &lt; <span class="integer">100</span>; i++) {
            <span class="comment">// wait 1 seconds and then again every 5 seconds</span>
            timer.schedule(<span class="keyword">new</span> MyTask(<span class="string"><span class="delimiter">"</span><span class="content">Task </span><span class="delimiter">"</span></span> + i), <span class="integer">1000</span>, <span class="integer">5000</span>);
        }
    }
}</code></pre> 
      </div> 
     </div> 
     <div class="paragraph"> 
      <p>TIP: Improved job scheduling is available via the open source framework <em>quartz</em>. See <a href="http://www.onjava.com/lpt/a/4637" class="bare">http://www.onjava.com/lpt/a/4637</a> <a href="http://www.onjava.com/lpt/a/4637" class="bare">http://www.onjava.com/lpt/a/4637</a> or <a href="http://www.quartz-scheduler.org/" class="bare">http://www.quartz-scheduler.org/</a> <a href="http://www.quartz-scheduler.org/" class="bare">http://www.quartz-scheduler.org/</a> for an explanation.</p>
